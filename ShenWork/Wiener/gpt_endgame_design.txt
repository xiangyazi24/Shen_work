# 1. The `hdecay` issue is decisive

A pure summable envelope

\[
\sum_k E_k<\infty,\qquad |\hat f_k|\le E_k
\]

does **not** imply

\[
|\hat f_k|\le C/k^2.
\]

A sparse summable sequence can have spikes with \(k^2E_k\to\infty\). So `SourceEnvelope` cannot fill

```lean
hdecay :
  |cosineCoeffs ... k| ≤ Cchem / ((k : ℝ) * Real.pi)^2
```

in `CoupledChemDivTimeC1Fields`.

If you insist on constructing the old struct, you need something like:

```lean
chemDivEWA ∈ E_T^2
```

because then each coefficient has a uniform \(k^{-2}\) bound. But that is a much stronger target and will push the weight requirement up.

The clean move is to bypass the old struct.

You already have the generic constructor:


which packages `adot`, coefficient differentiability, `hadotcont`, an arbitrary summable `envelope`, `henv_bound`, and a single `derivBound`. fileciteturn91file0L59-L79

And the actual `DuhamelSourceTimeC1` structure only asks for the ℓ¹ envelope, not quadratic decay. fileciteturn83file0L15-L31

Build this:

```lean
noncomputable def coupledChemDivSource_timeC1_of_l1Envelope
```

## Recommended theorem

Build:

Use `Ioo` plus an a.e. congruence for cosine coefficients, because your committed source eval bridge is interior-only. The current `evalST_sourceEWA_eq_intervalCoupledSource` proves the source equality at every \(x\in(0,1)\), not on the closed endpoints. fileciteturn88file0L11-L26

1. Let

   ```lean

   ```

2. From `heven` and `hreal`, prove

   ```lean

   ```

   coefficientwise.

3. From the already-built intrinsic envelope:


   prove


This theorem replaces the circular bridge.

## Is full parity propagation needed?

Yes, in some form.

You cannot get the coefficient identity from “real-valued eval” or conjugate symmetry alone. A real sine series is conjugate-symmetric as a Fourier series, but its paired extractor is not the Neumann cosine coefficient family of its restriction to `[0,1]`.

So you need either:

```lean
(sliceWA τ F).toFun = ofCosineCoeffs c
```

or enough to prove it: evenness plus real coefficients.

For the chem-div source, the parity chain is:

\[
u\ \text{even},\quad
u^\gamma\ \text{even},\quad
v=R_\mu(u^\gamma)\ \text{even},
\]

\[
v_x\ \text{odd},\quad
q=(1+v)^{-\beta}\ \text{even},
\]

\[
B=u\,v_x\,q\ \text{odd},
\qquad
\partial_xB\ \text{even},
\qquad
G(u)\ \text{even}.
\]

You do not need to prove a huge bespoke parity theorem for every final coefficient identity, but you do need reusable parity lemmas for the EWA operations or a direct theorem proving `sourceEWA` is even/real from `U` even/real.

A good minimal record is:

```lean
structure EvenRealEWA (U : EWA T r) : Prop where
  even : ∀ τ n, (sliceWA τ U).toFun (-n) = (sliceWA τ U).toFun n
  real : ∀ τ n, ((sliceWA τ U).toFun n).im = 0
```

Then prove closure:

```lean

EvenRealEWA.qFactor
EvenRealEWA.growthEWA

OddImagEWA.gDeriv_of_even

```

This is less brittle than threading `EWARealizesOn`.

# 4. Q3 — is `u ∈ A¹` truly required?

For the current Wiener product route, yes.

To get

\[
\partial_xB\in A^0
\]

through

\[
B=u\,v_x\,q,
\]

your construction uses

\[
B\in A^1.
\]

The bare factor \(u\) is the bottleneck. Resolver smoothing of \(v\) does not remove the high frequencies of \(u\) when multiplying by a nontrivial low mode of \(v_xq\).

A Fourier obstruction is simple: if \(h=v_xq\) has some nonzero low mode, then the high modes of \(u h\) contain shifted copies of the high modes of \(u\). Differentiating then costs one power of \(k\). So without a first moment

\[
\sum_k k|\hat u_k|<\infty,
\]

you cannot generally force

\[
\partial_x(uh)\in A^0.
\]

Thus a paraproduct/commutator shortcut is not the shortest sound Lean path. It would be a new analytic route, not a small bridge.

## Positive-time restart

A positive-time restart is the right device **if** the committed development exposes enough regularity to produce

\[
\sum_k (1+k)\sup_{s\in[\tau_0,T]}
|\widehat u_k(s)|<\infty.
\]

But the currently exposed joint iterate data is only `C²`-level. It explicitly records the iterate coefficient family as `C²` in time and the spatial regularity route as joint `C²`, not a higher `C³`/smooth package. fileciteturn93file0L5-L23

Plain spatial `C²` gives roughly

\[
|\hat u_k|\lesssim k^{-2},
\]

which yields

\[
\sum_k k|\hat u_k|\sim\sum_k 1/k,
\]

not enough.

You need one of:

1. a committed or new positive-time smoothing theorem giving `EWA T 1` directly;
2. a uniform `C³`/bounded-variation style coefficient decay theorem on `[τ₀,T]`;
3. an EWA fixed-point/shadow construction started from already `A¹` data;
4. a stronger weighted source/bootstrap theorem.

If the repo only exposes the current `C²` layer, then the positive-time restart still requires a new brick:

```lean

```

or

```lean
picardLimit_coeff_envelope_A1_on_Icc_pos
```

This is now the real analytic join brick if your initial data is not already in `A¹`.

# 6. Global vs windowed warning

Your `SourceEnvelope` is for `F : EWA T 0`, hence a finite time window. It naturally gives:

```lean
∀ s ∈ [0,T], |a s n| ≤ envelope n
```

The global `DuhamelSourceTimeC1` wants:

```lean
∀ s, 0 ≤ s → ...
```

fileciteturn83file0L22-L31

So the most faithful endpoint is the windowed package:

```lean

```

If a downstream theorem still insists on global `DuhamelSourceTimeC1`, you have two choices:

1. Add a windowed PAR theorem.
2. Use a soft-clamp/extension trick to create a global source package that agrees with the real source on `[0,T]`.

Do not silently use a finite-window envelope as a global one.
