## Verdict

The clean route is **(b): a trajectory-envelope continuation / bootstrap closure**, not a direct closed-form majorant from the endpoint-uniform Duhamel engine plus the uniform `L∞` ball.

More precisely:

* The endpoint-uniform engine is real and important, but it is a **propagator**: given a time-uniform per-mode source envelope `Msup ∈ H^σ`, it produces an endpoint-uniform `H^{σ+α}` Duhamel bound. It does **not** create the `Msup` envelope.
* The uniform `L∞` ball is only the base size/cone control. It does not imply `H^σ` for `σ > 0`; even at the level of coordinatewise envelopes, a scalar bound on `sup_τ ‖u(τ)‖_{H^σ}` is weaker than the field `genv σ : ℕ → ℝ` demanded by `SliceMildStepData`.
* Once a τ-uniform **coefficient envelope** `Uσ` for the trajectory `u(τ)` is available, the flux/source envelopes are produced cleanly by the repo's envelope algebra: resolver relay + denominator composition + mixed cosine×sine product. This is the part that is wiring.

So the residual is not a Gronwall blow-up and not classical-regularity circularity. It is the monotone production of a **single coordinatewise H^σ envelope over the whole trajectory window**.

## Why (a) is not enough

The fields in `SliceMildStepData` are deliberately stronger than a norm estimate. For the chemotaxis leg they require:

```lean
genv : ℝ → ℕ → ℝ
hg : ∀ {σ}, MemHSigma σ (cosineCoeffs ut) → MemHSigma σ (genv σ)
hg_dom : ∀ σ, ∀ τ ∈ Set.Icc (0:ℝ) t, ∀ k,
  |sineCoeffs (Q τ) k| ≤ genv σ k
```

and analogously for `glenv`/`hgl_dom`. This is a single sequence dominating every time slice, mode by mode. The new Key B file explicitly identifies these fields as the single remaining analytic input, not as data generated by the endpoint slice alone.

The engine lemma `duhamelEnergy_mode_endpoint_uniform` has the right endpoint behavior, but its hypotheses already include `hFbd : ∀ k, ∀ τ ∈ Icc 0 s, |F k τ| ≤ Msup k` and `Msup ∈ H^σ`. It turns that input into an endpoint-uniform output; it does not build `Msup` from `L∞` data.

Therefore there is no valid formula of the form

```text
genv σ k = C(M,p,t,σ) * simple_decay(k)
```

coming only from the `L∞` ball, unless an additional high-frequency estimate is smuggled into `simple_decay`. `L∞` alone has no such high-frequency decay.

## What is explicit once a trajectory envelope is available

Let `Uσ : ℕ → ℝ` be a τ-uniform cosine envelope for the solution trajectory:

```lean
HUσ : MemHSigma σ Uσ
HUdom : ∀ τ ∈ Set.Icc (0:ℝ) t, ∀ k,
  |cosineCoeffs (u τ) k| ≤ Uσ k
```

Then the chemotaxis flux envelope is explicit, in the repo's envelope language.

For the simplified resolver `(-∂ₓₓ + 1)v = u`, define the resolver envelope

```text
Vσ₂ k := Uσ k / (1 + λ_k)
Vxσ k := sqrt(λ_k) * Vσ₂ k
```

In the parameterized repo this is the same pattern as `resolverCoeff μ Uσ` and the derivative multiplier. The resolver gain is already formalized in `IntervalHSigmaScale.lean`.

Let `Gdenσ` be the denominator envelope for `(1+v)^(-β)` supplied by the existing denominator/composition envelope chain. Then

```text
GWσ    := trueCosProd Uσ Gdenσ
GENVσ  := trueCosProd GWσ Vxσ
```

is the natural `genv σ`. The reason is exactly `fluxSineEnvelope_uniform`: it takes τ-independent factor envelopes `gW` and `gvx`, plus the mixed bridge, and returns both `MemHSigma σ (trueCosProd gW gvx)` and the uniform domination of `sineCoeffs(Q τ)` over `τ ∈ [0,t]`.

The logistic/source envelope is lower-order. For a polynomial/integer-power presentation one can write schematically

```text
GLENVσ := |a| • Uσ + b • PowEnvelope_{α+1}(Uσ)
```

and for the repo's real-power form it should be the corresponding bounded-range Nemytskii envelope. This is the same kind of `H^σ ∩ L∞` algebra/composition input as the denominator factor, without the mixed sine product.

## Norm inequality chain

At the norm level, once the trajectory has a uniform `H^σ` bound and `σ > 1/2`, the standard chain is:

```text
sup_τ ‖v(τ)‖_{H^{σ+2}}
  ≤ C_R sup_τ ‖u(τ)‖_{H^σ}

sup_τ ‖v_x(τ)‖_{H^σ_sine}
  ≤ C_R' sup_τ ‖u(τ)‖_{H^σ}

sup_τ ‖(1+v(τ))^{-β}‖_{H^σ}
  ≤ C_den(M,p,σ, sup_τ ‖u(τ)‖_{H^σ})

sup_τ ‖W(τ)‖_{H^σ},  W = u(1+v)^{-β}
  ≤ C_alg ‖u‖_{L∞_τ H^σ} · ‖(1+v)^{-β}‖_{L∞_τ H^σ}

sup_τ ‖Q(τ)‖_{H^σ_sine},  Q = W v_x
  ≤ C_mix ‖W‖_{L∞_τ H^σ} · ‖v_x‖_{L∞_τ H^σ_sine}
```

Thus, schematically,

```text
sup_τ ‖Q(τ)‖_{H^σ_sine}
  ≤ C_Q(M,p,σ, sup_τ ‖u(τ)‖_{H^σ})
```

and similarly

```text
sup_τ ‖L(u(τ))‖_{H^σ}
  ≤ C_L(M,p,σ, sup_τ ‖u(τ)‖_{H^σ}).
```

This norm chain is correct and useful, but it is not literally the `genv` field. The field wants a coordinatewise envelope. The repo's envelope-monotone algebra is exactly the upgrade from a norm-level estimate to a concrete single sequence.

## Continuation / closure route

Define a property `P(σ,s)` for `0 ≤ s ≤ t`:

> There exists a τ-uniform coefficient envelope `Uσ` for `u(τ)` on `[0,s]`, with `MemHSigma σ Uσ`, and the induced `genv σ`, `glenv σ` constructed from `Uσ` satisfy the `SliceMildStepData` domination fields on `[0,s]`.

The closure should be carried out with coordinatewise envelopes, not merely scalar norms.

### 1. Base

For the formal mild trajectory, the producer should not require the raw initial datum to be `H^σ` for `σ>0`. Either:

* use the actual zero/off-window convention of the mild object at `τ = 0`, and start the positive-time smoothing immediately; or
* weaken the envelope domination fields from `Icc 0 t` to `Ioc 0 t` plus an interval-integral congruence, since the endpoint is invisible to the Duhamel integral.

At `σ = 0`, the seed is non-circular: continuous slices give `MemHSigma 0`, and the mild ladder starts from that seed. For `σ > 0`, the base is the first positive-time smoothing output, not an assumption on `u₀`.

### 2. Openness / local extension

Assume `P(σ,s)` on `[0,s]`. The induced `genv σ`, `glenv σ` feed the endpoint-uniform Duhamel engine. Because the engine constant has no elapsed-time blow-up (`R(r) ≤ Rbar` for endpoints `r ≤ 1`), the same envelope recurrence works on a small extension `[0,s+δ]` or, after a Chapman-Kolmogorov/restart split, on `[s,s+δ]`.

This step uses:

* the endpoint-uniform Duhamel gain;
* the mixed-product envelope construction for the flux;
* the lower-order source envelope construction for the logistic leg;
* time-continuity/measurability of the mild trajectory to keep the factor bridges valid locally.

### 3. Closedness

Let `s_n ↑ s` with `P(σ,s_n)`. Closedness should not be proved by taking an arbitrary coordinatewise supremum of a uniformly bounded `H^σ` family; that can fail in ℓ². Instead, use the monotone envelope recurrence itself: the same explicit majorant sequence produced by the recurrence dominates all earlier windows and has a uniform `H^σ` summability proof. Then pass to the limit in `τ ≤ s` by the already carried coefficient/source continuities.

Endpoint-uniformity is what makes this closedness sharp: the Duhamel constants stay independent of the endpoint `s_n`, so the limiting envelope remains in `H^σ`.

## Minimal Lean shape

```lean
namespace ShenWork.Paper2.IntervalMildBootstrapStep

open ShenWork.Paper2.HSigmaScale
open ShenWork.IntervalNeumannFullKernel (cosineCoeffs)
open ShenWork.Paper2.IntervalDivergenceModeIdentity (sineCoeffs)

/-- A τ-uniform coordinatewise `H^σ` envelope for the trajectory on `[0,t]`.
This is stronger than `sup_τ ‖u τ‖_{H^σ} < ∞`; it is the object needed to
construct `SliceMildStepData.genv/glenv`. -/
structure TrajectoryHSigmaEnvelope
    (σ t : ℝ) (u : ℝ → ℝ → ℝ) : Prop where
  env : ℕ → ℝ
  henv : MemHSigma σ env
  hdom : ∀ τ ∈ Set.Icc (0:ℝ) t, ∀ k,
    |cosineCoeffs (u τ) k| ≤ env k

/-- Desired flux/source envelope producer from a trajectory envelope.
The proof should be the resolver relay + denominator composition +
`fluxSineEnvelope_uniform` for the chemotaxis leg, and the analogous
bounded-range Nemytskii/Wiener envelope for the logistic leg. -/
structure ProducedTrajectorySourceEnvelopes
    (σ t : ℝ) (Q : ℝ → ℝ → ℝ) (Fl : ℝ → ℕ → ℝ) : Prop where
  genv : ℕ → ℝ
  hg : MemHSigma σ genv
  hg_dom : ∀ τ ∈ Set.Icc (0:ℝ) t, ∀ k,
    |sineCoeffs (Q τ) k| ≤ genv k
  glenv : ℕ → ℝ
  hgl : MemHSigma σ glenv
  hgl_dom : ∀ τ ∈ Set.Icc (0:ℝ) t, ∀ k,
    |Fl τ k| ≤ glenv k

/-- The genuine remaining analytic atom: trajectory envelope continuation.
This is the continuation/fixed-point closure over `[0,t]`, not a consequence
of the endpoint slice alone and not a classical-regularity assumption. -/
def TrajectoryEnvelopeContinuationAtom
    (α σ t : ℝ) (u Q : ℝ → ℝ → ℝ) (Fl : ℝ → ℕ → ℝ) : Prop :=
  ∀ s, 0 ≤ s → s ≤ t →
    TrajectoryHSigmaEnvelope σ s u →
      ∃ U_next : TrajectoryHSigmaEnvelope (σ + α) s u,
        ProducedTrajectorySourceEnvelopes σ s Q Fl

end ShenWork.Paper2.IntervalMildBootstrapStep
```

## Practical answer to the route question

* Do **not** try to manufacture `genv σ` from the `L∞` ball alone.
* Do **not** use a downstream classical/C² solution.
* Do use a trajectory-envelope continuation argument.
* Inside each continuation step, use the explicit envelope formula

```text
genv σ = trueCosProd gWσ gvxσ
```

with `gWσ` built from the `u` and denominator envelopes, and `gvxσ` built from the elliptic resolver derivative envelope. That is the clean split: continuation produces the trajectory `u` envelope; the repo's algebra produces `genv/glenv` from it.
