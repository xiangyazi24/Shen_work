# Verdict

Yes: the HSpectral plan is non-circular provided Codex keeps the WCE/WSE ladder as a local coefficient-envelope induction and does not obtain any WCE from BFormMildSpectralBootstrapData itself.

The key point is that SourceFromSolutionEnvelopePass is not a producer of solution envelopes. It is a one-step nonlinear source pass:

```javascript
def SourceFromSolutionEnvelopePass
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (S : ConjugateMildSolutionData p u₀) : Type :=
  ∀ {m : ℕ} {c T' : ℝ}, 0 < c → T' < S.T → 1 ≤ m →
    WindowCoefficientEnvelope m c T'
      (fun σ n => cosineCoeffs (intervalDomainLift (S.u σ)) n) →
    WindowSourceEnvelope (m - 1) c T'
      (bFormSourceCoeffs p S.u)
```

So the pass consumes WCE m for S.u and returns WSE (m-1) for the B-form source. The source pass itself should never be used to justify its own WCE m input.

# Why the input WCE is not HSpectral-dependent

The base solution envelope is already available directly from the mild-solution data:

```javascript
def windowCoefficientEnvelope_zero_of_bound
    (S : ConjugateMildSolutionData p u₀)
    {c T' : ℝ} (hc : 0 < c) (hT' : T' < S.T) :
    WindowCoefficientEnvelope 0 c T'
      (fun σ n => cosineCoeffs (intervalDomainLift (S.u σ)) n)
```

Its proof uses only:

```javascript
S.hcont   -- continuous slices
S.hbound  -- |S.u σ x| ≤ S.M
S.hM      -- positivity/nonnegativity of the ball bound
```

The code comment says this explicitly: bounded continuous slices give coefficients bounded by 2 * S.M on every interior window, this is “Unconditional; no source stub, no cq31 dependency.” The proof then calls restartBase_coeff_bound, which itself is just S.hcont + S.hbound + the generic bounded-continuous cosine coefficient estimate.

Concretely, the base is:

```javascript
WCE0 : WindowCoefficientEnvelope 0 c T' coeffU :=
  windowCoefficientEnvelope_zero_of_bound S hc hT'
```

where

```javascript
coeffU σ n := cosineCoeffs (intervalDomainLift (S.u σ)) n
```

This does not use HSpectral, hsrcB, hB_restart, BFormMildSpectralBootstrapData, or any final spectral bootstrap field.

# Where higher WCEs come from

Higher WCEs must come from heat/Duhamel smoothing inside the local ladder, not from the final HSpectral output. The intended order is:

```javascript
-- base from boundedness / continuity only
WCE0 : WindowCoefficientEnvelope 0 c T' coeffU :=
  windowCoefficientEnvelope_zero_of_bound S hc hT'

-- bounded source base, or direct weak-H² source producer depending on the route
WSE0 : WindowSourceEnvelope 0 c T' sourceCoeff :=
  bounded_source_envelope_from_S_bound_and_resolver_bounds ...

-- heat smoothing / parabolic gain, independent of HSpectral
WCE2 : WindowCoefficientEnvelope 2 c T' coeffU :=
  ladder_pass_gain_envelope ... WSE0 ... hhom ... hzero ...

-- nonlinear source pass, consuming an already-produced WCE
WSE1 : WindowSourceEnvelope 1 c T' sourceCoeff :=
  sourcePass hc hT' (by norm_num : 1 ≤ 2) WCE2

-- heat smoothing again
WCE3 : WindowCoefficientEnvelope 3 c T' coeffU :=
  ladder_pass_gain_envelope ... WSE1 ... hhom ... hzero ...

-- final source pass
WSE2 : WindowSourceEnvelope 2 c T' sourceCoeff :=
  sourcePass hc hT' (by norm_num : 1 ≤ 3) WCE3

-- then R
R := restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope ... WSE2 ...
```

The important invariant is: every WCE used by sourcePass is produced earlier in the induction by either the base WCE0 or a heat-pass from a previously produced WSE. None is projected from the final HSpectral result.

If using the fork’s direct weak-H² witness route to produce WSE2 directly, the situation is even less circular: the direct weak-H² witness plus quadratic coefficient decay can produce the WindowSourceEnvelope 2 source envelope without calling SourceFromSolutionEnvelopePass at all. Then R follows directly from pass-2. In that shortcut, SourceFromSolutionEnvelopePass becomes unnecessary for the endpoint, not circular.

# Role of u ≥ 0, strict positivity, and δ-floor

The source pass may need positivity/floor data for the Nemytskii side, especially when differentiating powers like u^γ with γ ∈ [1,2). The repo’s uniform_positive_lower_bound obtains a per-slice closed-interval floor from S.hpos and S.hcont by compactness:

```javascript
theorem uniform_positive_lower_bound
    (S : ConjugateMildSolutionData p u₀)
    {σ : ℝ} (hσ : 0 < σ) (hσT : σ ≤ S.T) :
    ∃ δ : ℝ, 0 < δ ∧
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        δ ≤ intervalDomainLift (S.u σ) x
```

That dependency is Jensen/energy side, not HSpectral. In the V6 assembly, S.hpos must be supplied before HSpectral is built; once S.hpos is in ConjugateMildSolutionData, using it in the source pass is not a spectral self-reference.

# What would be circular

The following would be circular and should be avoided:

```javascript
-- BAD: deriving WCE/R from the final HSpectral object that is being built
have WCE_m := some_projection_from (bFormMildSpectralBootstrapData_of_leaves H)
have WSE := sourcePass hc hT' hm WCE_m
-- then use WSE to build H.hsrcB and finally the same HSpectral object
```

That would make the source pass depend on the result it is supposed to help construct.

The correct route is:

```javascript
-- GOOD: all before HSpectral
have WCE0 := windowCoefficientEnvelope_zero_of_bound S hc hT'
have WSE0 := bounded_source_envelope ...
have WCE2 := heat_gain WSE0 ...
have WSE1 := sourcePass hc hT' hm WCE2
have WCE3 := heat_gain WSE1 ...
have WSE2 := sourcePass hc hT' hm WCE3
have R := restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope ... WSE2 ...
-- then use R to build L/hsrcB and finally HSpectral
```

# Final check

No self-reference is forced by the API. SourceFromSolutionEnvelopePass merely consumes a WindowCoefficientEnvelope m argument; it does not say where that argument came from. The repo already provides a non-spectral base WCE0 from S.hcont/S.hbound, and the heat-pass ladder is the intended non-HSpectral mechanism for higher WCEs. Thus HSpectral remains acyclic as long as Codex wires the ladder in topological order:

```plain text
S.hcont + S.hbound
  → WCE0 / bounded source base
  → heat gain WCEs
  → SourceFromSolutionEnvelopePass source envelopes
  → WSE2
  → pass-2 eigenvalue-weighted summability R
  → L / hsrcB / hB_restart / Fourier leaves
  → bFormMildSpectralBootstrapData_of_leaves
  → HSpectral
```
