# Executive verdict

There are three distinct tail architectures, and they have very different case counts.

1. Direct prime-torsion theorem. A published Oesterlé/Parent-style uniform bound gives, for a degree-d number field, roughly

For d=1 this gives p\le 7. Thus one uniform theorem can eliminate every prime p\ge 11. In this strongest formulation the optimal bound is effectively B=7, so there are zero fixed prime cases in the tail. This theorem is not cheap: it is essentially the deep modular-curve/formal-immersion core.

1. Lean-pragmatic split. Handle p=17,19 by fixed elliptic curves X_0(17),X_0(19), then expose one weakened capstone

```javascript
theorem no_prime_torsion_of_gt_19
    {p : ℕ} [Fact p.Prime] (hp : 19 < p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E p
```

This has two fixed cases plus one uniform theorem. The threshold 19 is chosen for engineering convenience; it does not make the uniform theorem conceptually shallow.

1. Detour through Mazur's prime-isogeny theorem. Since rational p-torsion gives a rational p-isogeny, Mazur's theorem leaves

```plain text
p ∈ {2,3,5,7,11,13,17,19,37,43,67,163}.
```

After 11,13 are handled, the residual fixed tail is

```plain text
{17,19,37,43,67,163},
```

so six explicit cases. If one uses only the coarse corollary “no p-isogeny for p>163” and does not formalize the exact exceptional-set classification, there are 32 primes from 17 through 163 to check. Therefore the isogeny detour is not the smallest route for the torsion theorem.

Recommended scope: fixed 17,19 plus a direct uniform theorem for p>19. Do not build the full J_1(p) for each prime, and do not use the full rational-isogeny classification unless it is independently needed elsewhere.

# 2. The cases p=17 and p=19

## 2.1 Use X_0(p), not the full J_1(p)

The natural quotient is

induced by pushforward along X_1(p)\to X_0(p). For p=17,19, J_0(p)=X_0(p) is itself an elliptic curve of rank zero.

Concrete models are:

```plain text
X_0(17) = 17a1:
  y² + xy + y = x³ - x² - x - 14,
  rank 0, torsion Z/4Z.

X_0(19) = 19a1:
  y² + y = x³ + x² - 9x - 15,
  rank 0, torsion Z/3Z.
```

These coefficient tuples are respectively

```plain text
[1,-1,1,-1,-14]
[0, 1,1,-9,-15].
```

The maps X_1(17)\to X_0(17) and X_1(19)\to X_0(19) have degrees 8 and 9.

## 2.2 Rank zero of the quotient is not by itself enough

A rational point on X_0(p) represents a rational cyclic subgroup of order p. It need not represent a rational generator. Equivalently, the kernel has a Galois character

A rational point of exact order p requires \lambda=1. A general rational p-isogeny only requires the line to be Galois-stable.

Therefore the fixed proof is:

```plain text
1. enumerate X_0(p)(Q) using rank zero;
2. identify the cusps and the finitely many noncuspidal isogeny points;
3. for each noncuspidal point, prove that its fiber under X_1(p)->X_0(p)
   has no Q-rational point;
4. equivalently, prove that the kernel character is nontrivial.
```

This fiber step is finite and explicit. It can be implemented by:

- a Tate-normal-form fiber polynomial;

- a division-polynomial/kernel-generator polynomial;

- or a direct computation of the Galois character for each listed isogeny.

Thus the elliptic quotient captures the finite list of possible images, but rank zero alone does not close the order-p problem.

## 2.3 What about the full J_1(17) and J_1(19)?

Published modular-form computations place these small modular Jacobians among the rank-zero cases. But proving the rank of the entire dimension-5 or dimension-7 Jacobian is the wrong Lean target:

```plain text
J_1(17): dimension 5;
J_1(19): dimension 7.
```

A full proof requires decomposing every newform factor and proving every factor has rank zero. Even after that, one still needs a finite rational-point enumeration on a genus-5 or genus-7 curve.

The quotient route replaces that with:

```plain text
one elliptic rank-zero computation
+ one finite list
+ finitely many fiber checks.
```

That is substantially smaller.

# 3. What happens at 23,29,31,37,41?

If the uniform theorem is postponed, the next fixed cases can still be attacked through X_0(p):

- p=23,29,31: genus-2 X_0(p); prove all rational points are cusps, then no rational p-isogeny and hence no rational p-torsion.

- p=37: genus-2 X_0(37) has noncuspidal rational points, so one must enumerate them and check the fibers in X_1(37).

- p=41: X_0(41) has genus 3, so this begins a new higher-genus fixed-curve engine.

This gives a practical boundary:

If a genus-2 rational-point engine already exists, fixed arithmetic remains plausible through p=37. Continuing prime by prime beyond that does not scale.

The enormous genera of X_1(p) show why direct fixed-X_1 arithmetic is not a viable tail strategy:

```plain text
p=23: genus 12
p=29: genus 22
p=31: genus 26
p=37: genus 40
p=41: genus 51.
```

# 4. The uniform rank-zero quotient

There is a uniform low-rank object, but it is not a fixed elliptic curve.

Mazur constructs the Eisenstein quotient

of J_0(p). The relevant theorem is:

For prime p, the Eisenstein quotient is nontrivial and has Mordell–Weil rank zero over \mathbf Q.

One composes

The crucial additional statement is that the resulting map is a formal immersion at the cusp. If a rational modular point reduces to the same cusp and has the same image in the rank-zero quotient, formal immersion forces equality with the cusp.

The algebraic skeleton is:

```plain text
rank-zero quotient
+ torsion injectivity under reduction
+ formal immersion at infinity
+ local analysis of the modular point
=> the modular point is a cusp.
```

This is the true uniform replacement for full rank zero of J_1(p).

For p=17,19, J_0(p) itself is already a rank-zero elliptic quotient. For large p, J_e(p) generally has growing dimension, but its construction and proof are uniform in p.

The most economical public split is:

    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E p

Then the complete prime tail is a short case split:

```javascript
theorem no_rational_point_of_prime_order_ge_17
    {p : ℕ} [Fact p.Prime] (hp : 17 ≤ p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E p := by
  by_cases h19 : p ≤ 19
  · have : p = 17 ∨ p = 19 := by
      -- finite prime arithmetic
      omega
    rcases this with rfl | rfl
    · exact no_rational_point_of_order_17 E
    · exact no_rational_point_of_order_19 E
  · exact no_rational_point_of_prime_order_gt_19
      (Nat.lt_of_not_ge h19) E
```

The exact finite-prime proof will use primality to remove 18.

# 6. Is the uniform theorem reducible to a finite computation?

Not as a uniform theorem. It is the irreducible research core of the prime tail.

For a fixed p, many subclaims can be turned into finite certificates:

- modular-symbol matrices;

- Hecke-operator calculations;

- dimensions of chosen quotients;

- q-expansion coefficients proving a cotangent map is nonzero;

- finite reduction groups;

- local cusp and component-group calculations.

But uniformity in p requires genuine theorems:

1. construction of X_0(p), J_0(p) and Hecke correspondences for variable prime p;

1. the Eisenstein ideal and Eisenstein/winding quotient;

1. proof that the quotient is nontrivial and rank zero;

1. rational cuspidal torsion and reduction injectivity;

1. formal immersion at the cusp;

1. local/Galois analysis turning a rational p-torsion point into a point reducing to that cusp;

1. uniform inequalities that close the contradiction.

This is not a database lookup and not a bounded decide. It is most of the conceptual content of Mazur's modular proof.

A Lean implementation may use finite modular-symbol certificates internally, but it still needs a theorem proving that those certificates exist or have the required shape for every prime in the range.

# 7. Honest mountain-size estimate

## Route A: direct uniform prime-torsion theorem

```plain text
fixed cases:       0, or optionally 2 (17 and 19)
uniform core:      one very deep modular theorem
new infrastructure: modular curves, Hecke algebra, modular symbols,
                    rank-zero Eisenstein/winding quotient,
                    formal immersion, local reduction
```

This is mathematically optimal but formally the largest single block.

## Route B: full prime-isogeny theorem plus fibers

```plain text
uniform core:      full Mazur prime-isogeny theorem
explicit residual: 17,19,37,43,67,163
extra work:         prove the known rational isogenies do not have
                    a rational kernel generator
```

This is strictly more than is needed for the torsion bound unless the isogeny theorem is a separate project goal.

## Route C: fixed curves as far as possible

```plain text
17,19:             elliptic X0
23,29,31,37:       genus-2 X0
41 onward:         new higher-genus cases with unbounded complexity
```

This route hits a wall immediately after 37 and cannot replace a uniform theorem.

Bottom line: the prime tail is not “do dozens of X_1(p) computations.” It is either

```plain text
2 small fixed cases + 1 deep uniform theorem,
```

or, via the isogeny detour,

```plain text
6 exceptional fixed cases + 1 even broader deep uniform theorem.
```

The first architecture is the smaller mountain.
