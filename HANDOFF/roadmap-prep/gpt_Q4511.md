# N18 Block 5: the strict \(3\)-adic formal-kernel filtration and the \([21]\)-annihilator

## Executive verdict

The Block-5 argument is correct, but two statements in the proposed proof need correction.

1. The reduction of the good \(E_0\)-model at \(\pi=a-1\) is supersingular in characteristic \(3\). Therefore the reduced multiplication series has height \(2\): modulo \(\pi\), its first nonzero term is in degree \(3^2=9\), not degree \(3\). There is no load-bearing \(z^3\) term.
2. The nonarchimedean triangle inequality gives a **lower bound**
   
   
   
   \[
   v([3]z)\ge \min\{3+v(z),\,2v(z)\},
   \]
   
   not equality in general; cancellation may make the valuation larger.

Neither correction hurts the proof. The universal formal-group identity

\[
[3]_{\widehat E}(T)=3T+T^2A(T),\qquad A(T)\in\mathcal O_K[[T]],
\]

already gives, for every nonzero \(z\in\mathfrak m_K\) with \([3]z
e0\),

\[
egin{aligned}
 v([3]z)
 &\ge \minigl(v(3z),v(z^2A(z))igr) \\
 &\ge \minigl(3+v(z),2v(z)igr) \\
 &\ge v(z)+1.
\end{aligned}
\]

This is the entire local estimate. It proves that the reduction kernel has no nonzero element divisible by \(3^n\) for every \(n\). Combined with

```text
E0(L) = H3 + 3 E0(L),          3 H3 = 0,
7 · E0(F3) = 0,
```

it gives

```text
21 · E0(L) = 0
```

without Mordell--Weil finite generation.

The generic filtration and annihilator code below was compiled against the repository's pinned Lean/Mathlib environment; the validation ended with `EXIT_STATUS=0`. The remaining curve-specific work is exactly the adapter from the good Weierstrass model to the local parameter and its additive valuation. Current Mathlib has a generic `FormalGroup`, but does not yet have `WeierstrassCurve.formalGroup` or a ready reduction-kernel/formal-parameter equivalence, so those adapter declarations must be project-local.

# 1. Local arithmetic and the good model

Let

```text
L = Q(a),       a^3 = 3a + 1,
pi = a - 1,
K = L_pi,
O = O_K,
k = O / (pi) = F_3.
```

Substituting \(a=\pi+1\) into \(a^3-3a-1=0\) gives

\[
\pi^3+3\pi^2-3=0,
\]

hence

\[
3(1-\pi^2)=\pi^3.
\]

Since \(1-\pi^2\equiv1\pmod\pi\), it is a unit of \(\mathcal O_K\). With the additive normalization \(v(\pi)=1\),

\[
v(3)=3.
\]

Use the integral model already identified in Block 5:

\[
egin{aligned}
Y^2+(a^2+a-1)XY+(a^2-2)Y
={}&X^3+(-a^2+a+2)X^2 \\
&+(13a^2+2a-44)X-22a^2-3a+88.
\end{aligned}
\]

Its discriminant is \(-8\), a \(\pi\)-adic unit, so it has good reduction. Reducing at \(\pi\) gives

\[
y^2+xy-y=x^3-x^2+x
\]

over \(\mathbf F_3\). Its rational point group has exactly seven elements, so every reduced rational point is killed by \(7\).

**Codomain correction.** For the annihilator proof, define reduction with codomain the rational residue group

```lean
redLocal : E0KPoint →+ E0ReductionF3Point
```

or pull this map back to \(E_0(L)\). Do not use \(E_0(\overline{\mathbf F}_3)\) as the codomain of the exponent-\(7\) statement: the full algebraic-closure point group is not killed by \(7\). The image of a \(K\)-rational integral point lies in \(E_0(\mathbf F_3)\), so the smaller codomain is the correct one.

# 2. The formal group and its multiplication-by-three series

For a good integral Weierstrass equation

\[
y^2+a_1xy+a_3y=x^3+a_2x^2+a_4x+a_6,
\]

use the standard parameter at the identity

\[
t=-x/y.
\]

One concrete construction introduces \(w=-1/y\), so that

\[
x=t/w,\qquad y=-1/w,
\]

where \(w=w(t)\in t^3\mathcal O_K[[t]]\) is the unique solution of

\[
w=t^3+a_1tw+a_2t^2w+a_3w^2+a_4tw^2+a_6w^3.
\]

Transporting the elliptic group law through this parameter gives the commutative one-dimensional formal group law

\[
F(T_1,T_2)=T_1+T_2+	ext{terms of total degree at least }2.
\]

Define its \(n\)-series recursively by

\[
[0]_F(T)=0,\qquad [n+1]_F(T)=F([n]_F(T),T).
\]

## 2.1 Linear coefficient

Induction on \(n\), using \(F(X,Y)=X+Y+O((X,Y)^2)\), gives

\[
[n]_F(T)=nT+O(T^2).
\]

In particular, the constant coefficient of \([3]_F\) is zero and its coefficient of \(T\) is \(3\). Therefore the series

\[
[3]_F(T)-3T
\]

has zero coefficients in degrees \(0\) and \(1\), and hence is divisible by \(T^2\). Thus there is a unique \(A(T)\in\mathcal O_K[[T]]\) such that

\[
oxed{[3]_F(T)=3T+T^2A(T).}
\]

A direct Lean definition of the quotient series is coefficient shifting:

```lean
noncomputable def threeRemainderQuotient
    (threeSeries : O⟦X⟧) : O⟦X⟧ :=
  PowerSeries.mk fun n ↦
    PowerSeries.coeff (n + 2)
      (threeSeries - PowerSeries.C (3 : O) * PowerSeries.X)
```

After proving the constant and linear coefficients of `threeSeries`, the identity

```lean
threeSeries =
  PowerSeries.C (3 : O) * PowerSeries.X +
    PowerSeries.X ^ 2 * threeRemainderQuotient threeSeries
```

is coefficient extensionality, splitting the coefficient index into \(0\), \(1\), and \(n+2\). Exact names around `PowerSeries.mk` and `coeff` may require the repository's selected formal-series wrapper, but no elliptic arithmetic remains in this factorization.

## 2.2 Supersingular refinement, not needed for strictness

The reduced curve has

```text
#E(F3) = 7,
trace(Frob_3) = 3 + 1 - 7 = -3,
```

so it is supersingular. Consequently its formal group has height \(2\), and modulo \(\pi\)

\[
[3]_{\widetilde E}(T)=uT^9+	ext{higher terms},\qquad u
e0.
\]

This stronger statement implies that the coefficients of degrees \(2,\ldots,8\) in the integral \([3]\)-series lie in the maximal ideal. It is useful for sharper ramification estimates but should **not** be used as the basic Block-5 proof. The coarse factorization \(3T+T^2A(T)\) is universal and enough.

# 3. Evaluation in the formal kernel

Let

```text
E1(K) = ker(E0(K) -> E0(F3)).
```

Good reduction gives a group isomorphism between \(E_1(K)\) and the maximal ideal equipped with the formal group law:

```lean
formalParam : E1K ≃+ FormalMaxIdealPoint e0FormalGroup
```

with underlying coordinate \(P\mapsto t(P)\). It must satisfy

```lean
formalParam (n • P) = n • formalParam P
```

and, after unfolding the formal group action,

```lean
t ([3]P) = PowerSeries.eval₂ (algebraMap O K) (t P) threeSeries.
```

For \(P
e O\) in the formal kernel, write

\[
z=t(P),\qquad r=v(z).
\]

Then \(z\in\mathfrak m_K\), so \(r\ge1\). Completeness of \(\mathcal O_K\) makes evaluation of every integral power series at \(z\) well-defined, and

\[
A(z)\in\mathcal O_K,\qquad v(A(z))\ge0.
\]

If \([3]P
e O\), then \([3]_F(z)
e0\), and

\[
egin{aligned}
v(t([3]P))
 &=v(3z+z^2A(z)) \\
 &\ge\min\{v(3)+v(z),\,2v(z)+v(A(z))\} \\
 &\ge\min\{3+r,\,2r\}.
\end{aligned}
\]

For \(r\ge1\), both terms are at least \(r+1\):

\[
3+r\ge r+1,\qquad 2r\ge r+1.
\]

Therefore

\[
oxed{v(t([3]P))\ge v(t(P))+1.}
\]

### Lean valuation-order warning

Mathlib's `Valuation K ℤᵐ⁰` is multiplicative and orders more-divisible elements in the opposite-looking direction from the usual additive order. Do not state the strictness theorem directly using that order unless every conversion is explicit. The clean implementation is to expose a natural-valued additive level

```lean
formalLevel : E1K → ℕ
```

with `formalLevel P = v_pi (t P)` for `P ≠ 0`, and prove the natural-number inequality. An equally robust implementation defines the filtration by ideal powers

```text
F_r = {P | t(P) ∈ maximalIdeal^r};
```

then proves `[3](F_r) ⊆ F_{r+1}` for `r ≥ 1`.

# 4. Exact strict-filtration interface

The name `StrictNSmulFiltration` is not currently landed on `scratch`. The following definition is sufficient and avoids assigning an infinite natural level to zero:

```lean
/-- A strict `m`-divisibility filtration on an additive subgroup.
Strictness is required only when the `m`-multiple is nonzero. -/
structure StrictNSmulFiltration
    {G : Type*} [AddCommGroup G] (H : AddSubgroup G) (m : ℕ) where
  level : H → ℕ
  strict : ∀ x : H, m • x ≠ 0 → level x < level (m • x)
```

Package the curve-specific valuation result in the following minimal data:

```lean
structure FormalKernelThreeData
    {G Q : Type*} [AddCommGroup G] [AddCommGroup Q]
    (red : G →+ Q) where
  level : red.ker → ℕ
  level_pos : ∀ P : red.ker, P ≠ 0 → 1 ≤ level P
  three_bound : ∀ P : red.ker, (3 : ℕ) • P ≠ 0 →
    min (3 + level P) (2 * level P) ≤ level ((3 : ℕ) • P)
```

The numerical step and constructor are fully elementary:

```lean
lemma min_three_add_two_mul_ge_succ {r : ℕ} (hr : 1 ≤ r) :
    r + 1 ≤ min (3 + r) (2 * r) := by
  omega

namespace FormalKernelThreeData

variable {G Q : Type*} [AddCommGroup G] [AddCommGroup Q]
variable {red : G →+ Q}

/-- The valuation estimate builds the strict `3`-filtration. -/
def strictNSmulFiltration (D : FormalKernelThreeData red) :
    StrictNSmulFiltration red.ker 3 where
  level := D.level
  strict := by
    intro P h3P
    have hP : P ≠ 0 := by
      intro hP0
      apply h3P
      rw [hP0, nsmul_zero]
    have hmin := min_three_add_two_mul_ge_succ (D.level_pos P hP)
    have hbound := D.three_bound P h3P
    omega

end FormalKernelThreeData
```

For the local curve, instantiate `three_bound` by the displayed valuation calculation. For the global \(L\)-rational group, first define

```lean
loc : E0LPoint →+ E0KPoint
redGlobal : E0LPoint →+ E0ReductionF3Point := redLocal.comp loc
```

and pull the local level back along `loc`. Injectivity of the field embedding \(L\hookrightarrow K\) gives injectivity of `loc`; hence `3 • P ≠ 0` globally implies `3 • loc P ≠ 0` locally.

A project-facing adapter has the shape

```lean
noncomputable def e0GlobalFormalKernelThreeData :
    FormalKernelThreeData redGlobal where
  level := fun P ↦
    localFormalLevel
      ⟨loc P.1, by simpa [redGlobal] using P.2⟩
  level_pos := by
    intro P hP
    apply localFormalLevel_pos
    intro hloc
    apply hP
    apply Subtype.ext
    exact loc_injective hloc
  three_bound := by
    intro P h3P
    let Ploc : redLocal.ker :=
      ⟨loc P.1, by simpa [redGlobal] using P.2⟩

      apply h3P
      apply Subtype.ext
      apply loc_injective
      simpa [Ploc] using congrArg Subtype.val hzero
    simpa [Ploc, localFormalLevel, map_nsmul] using
      local_three_level_bound Ploc h3Ploc

```

The identifiers `localFormalLevel`, `local_three_level_bound`, and the concrete point types are the only names to adapt to the landed local model.

# 5. No nonzero infinitely \(3\)-divisible point

The strict-level argument is finite and does not use topology.

If

\[
x=3^ny_n
e0,
\]

then none of the intermediate multiples \(3^iy_n\), \(0\le i\le n\), can be zero. Repeated strictness gives

\[
\operatorname{level}(y_n)+n\le\operatorname{level}(x).
\]

Taking \(n=\operatorname{level}(x)+1\) is impossible. Here is the compiled Lean proof:

```lean
namespace StrictNSmulFiltration

variable {G : Type*} [AddCommGroup G]
variable {H : AddSubgroup G} {m : ℕ}

lemma level_add_le_of_eq_pow_nsmul
    (F : StrictNSmulFiltration H m)
    {n : ℕ} {x y : H} (hx : x ≠ 0) (hxy : x = (m ^ n) • y) :
    F.level y + n ≤ F.level x := by
  induction n generalizing x with
  | zero =>
      simpa [hxy]
  | succ n ih =>
      let z : H := (m ^ n) • y
      have hxz : x = m • z := by
        calc
          x = (m ^ (n + 1)) • y := hxy
          _ = (m ^ n * m) • y := by rw [pow_succ]
          _ = (m * m ^ n) • y := by rw [Nat.mul_comm]
          _ = m • ((m ^ n) • y) := by rw [smul_smul]
          _ = m • z := rfl
      have hz : z ≠ 0 := by
        intro hz0
        apply hx
        rw [hxz, hz0, nsmul_zero]
      have hi : F.level y + n ≤ F.level z := ih hz rfl
      have hmz : m • z ≠ 0 := by
        intro hmz0
        exact hx (hxz.trans hmz0)
      have hs : F.level z < F.level x := by
        simpa [hxz] using F.strict z hmz
      omega

/-- A nonzero element cannot be divisible by every power of `m`. -/
theorem eq_zero_of_infinitely_divisible
    (F : StrictNSmulFiltration H m) (x : H)
    (hdiv : ∀ n : ℕ, ∃ y : H, x = (m ^ n) • y) :
    x = 0 := by
  by_contra hx
  obtain ⟨y, hy⟩ := hdiv (F.level x + 1)
  have hle := F.level_add_le_of_eq_pow_nsmul hx hy
  omega

end StrictNSmulFiltration
```

# 6. The visible order-\(3\) point

Let \(T\in E_0(L)\) be the known point of exact order \(3\). Its reduction satisfies

```text
3 · red(T) = red(3T) = 0,
7 · red(T) = 0.
```

Since \(3\) and \(7\) are coprime, `red(T)=0`. Thus \(T\) lies in the formal kernel. Its multiplication-by-three value is the identity:

```text
[3]T = O.
```

In extended additive-valuation notation this reads

```text
v(t([3]T)) = v(0) = infinity > v(t(T)).
```

Our `StrictNSmulFiltration` deliberately asks for strictness only when `3 • P ≠ 0`, so the torsion point requires no artificial natural value for infinity.

**Do not claim that strictness proves \(T\) is not once \(3\)-divisible.** If \(T=3Q\), strictness merely says `level Q < level T`; that is possible. What strictness proves, and what Block 5 needs, is that no nonzero formal-kernel point is divisible by every \(3^n\).

# 7. The \([21]\)-annihilator

Define weak descent abstractly:

```lean
structure WeakDescent
    {G : Type*} [AddCommGroup G] (m : ℕ) (T : AddSubgroup G) : Prop where
  decompose : ∀ x : G, ∃ t : T, ∃ y : G, x = (t : G) + m • y
```

Assume

```text
hweak : WeakDescent 3 H3,
hH3   : forall t : H3, 3 • t = 0,
hred  : forall q : E0ReductionF3Point, 7 • q = 0.
```

For each \(x\in G=E_0(L)\), recursively applying weak descent gives, for every \(n\), some \(y_n\) with

\[
21x=3^n(21y_n).
\]

Indeed, multiplying `x = t + 3y` by \(21\) kills \(t\), because \(21t=7(3t)=0\). Moreover

\[
\operatorname{red}(21y_n)=3igl(7\operatorname{red}(y_n)igr)=0,
\]

so every \(21y_n\) belongs to the reduction kernel. Thus \(21x\) is infinitely \(3\)-divisible **inside the formal kernel**, and strictness forces \(21x=0\).

The following complete generic proof was compiled with `EXIT_STATUS=0`:

```lean
namespace WeakDescent

variable {G Q : Type*} [AddCommGroup G] [AddCommGroup Q]
variable {T : AddSubgroup G}

lemma twentyOne_smul_torsion_zero
    (hT : ∀ t : T, (3 : ℕ) • (t : G) = 0) (t : T) :
    (21 : ℕ) • (t : G) = 0 := by
  calc
    (21 : ℕ) • (t : G) = (7 : ℕ) • ((3 : ℕ) • (t : G)) := by
      norm_num [smul_smul]
    _ = 0 := by rw [hT t, nsmul_zero]

lemma twentyOne_smul_three_pow_divisible
    (hweak : WeakDescent 3 T)
    (hT : ∀ t : T, (3 : ℕ) • (t : G) = 0)
    (n : ℕ) (x : G) :
    ∃ y : G, (21 : ℕ) • x = (3 ^ n) • ((21 : ℕ) • y) := by
  induction n with
  | zero =>
      exact ⟨x, by simp⟩
  | succ n ih =>
      obtain ⟨y, hy⟩ := ih
      obtain ⟨t, z, hz⟩ := hweak.decompose y
      refine ⟨z, ?_⟩
      have h21t : (21 : ℕ) • (t : G) = 0 :=
        twentyOne_smul_torsion_zero hT t
      have hinner : (21 : ℕ) • ((t : G) + (3 : ℕ) • z) =
          (3 : ℕ) • ((21 : ℕ) • z) := by
        rw [nsmul_add, h21t, zero_add]
        simp [smul_smul, Nat.mul_comm]
      calc
        (21 : ℕ) • x = (3 ^ n) • ((21 : ℕ) • y) := hy
        _ = (3 ^ n) • ((21 : ℕ) • ((t : G) + (3 : ℕ) • z)) := by rw [hz]
        _ = (3 ^ n) • ((3 : ℕ) • ((21 : ℕ) • z)) := by rw [hinner]
        _ = (3 ^ n * 3) • ((21 : ℕ) • z) := by rw [smul_smul]
        _ = (3 ^ (n + 1)) • ((21 : ℕ) • z) := by rw [pow_succ]

lemma twentyOne_smul_mem_ker
    (red : G →+ Q) (hred : ∀ q : Q, (7 : ℕ) • q = 0) (x : G) :
    red ((21 : ℕ) • x) = 0 := by
  rw [map_nsmul]
  calc
    (21 : ℕ) • red x = (3 : ℕ) • ((7 : ℕ) • red x) := by
      norm_num [smul_smul]
    _ = 0 := by rw [hred (red x), nsmul_zero]

/-- Weak `3`-descent modulo `3`-torsion, exponent `7` on the reduction,
and a strict `3`-filtration on the reduction kernel imply exponent `21`. -/
theorem twentyOne_annihilator
    (red : G →+ Q)
    (hweak : WeakDescent 3 T)
    (hT : ∀ t : T, (3 : ℕ) • (t : G) = 0)
    (hred : ∀ q : Q, (7 : ℕ) • q = 0)
    (F : StrictNSmulFiltration red.ker 3) :
    ∀ x : G, (21 : ℕ) • x = 0 := by
  intro x
  let X : red.ker :=
    ⟨(21 : ℕ) • x, twentyOne_smul_mem_ker red hred x⟩
  have hdiv : ∀ n : ℕ, ∃ Y : red.ker, X = (3 ^ n) • Y := by
    intro n
    obtain ⟨y, hy⟩ :=
      twentyOne_smul_three_pow_divisible hweak hT n x
    let Y : red.ker :=
      ⟨(21 : ℕ) • y, twentyOne_smul_mem_ker red hred y⟩
    refine ⟨Y, ?_⟩
    apply Subtype.ext
    exact hy
  have hX : X = 0 := F.eq_zero_of_infinitely_divisible X hdiv
  exact congrArg Subtype.val hX

end WeakDescent
```
