# N18 Route C: exact Mathlib declaration survey

**Mathlib revision checked:** `96fd0fff3b8837985ae21dd02e712cb5df72ec05`, the revision pinned by `xiangyazi24/FLT` at the time of this survey.

This is an import-and-reuse map, not a mathematical roadmap. For each block, declarations are divided into:

- **HAS:** Mathlib already provides the named object or theorem.
- **PARTIAL:** related infrastructure exists, but not the N18 theorem.
- **BUILD:** the project must construct it.

The most important negative results are:

1. Mathlib has no genus-two Jacobian/Picard-functoriality API that supplies the N18 push-pull theorem.
2. Mathlib has no elliptic-curve isogeny/Vélu/dual-isogeny API or Selmer computation package.
3. Mathlib has a generic one-dimensional `FormalGroup`, but no `WeierstrassCurve.formalGroup` or elliptic formal-parameter/reduction-kernel API.
4. `FractionalIdeal.mk0` and `ClassGroup.mk0` require `IsDedekindDomain`; the no-Dedekind Block 2 must instead exhibit the relevant fractional ideals as units explicitly and use `ClassGroup.mk`.

## Critical logical warning

A bounded exponent implies rational rank zero, but it does **not** imply finiteness for an arbitrary group. To prove finiteness, Route C still needs an injection into a finite group, an exact finite enumeration, or an independent finite-generation theorem. Keep these as separate capstones:

```lean
n18_pic_exponent       : ∀ D, 126 • D = 0
n18_pic_finite         : Finite (Pic18 ℚ)
```

Do not derive the second from the first alone.

# Builder-facing conclusions

1. **Use `AdjoinRoot`, not `SplittingField`, for every ring certificate.**
2. **Use `ClassGroup.mk`, not `ClassGroup.mk0`, in Block 2.** The latter secretly reintroduces the Dedekind obligation.
3. **Copy the structure of `EllipticCurve/Affine/Point.lean`.** `XYIdeal'`, `Point.toClass`, and `Point.toClass_injective` are the closest exact precedents in Mathlib.
4. **Stop searching for genus-two Jacobian functoriality or a Vélu API.** Neither is present in a usable form at the checked revision.
5. **Use `HeightOneSpectrum.valuation` for the prime above `3`.** `padicValRat` is a rational-only convenience function.
6. **Treat generic `FormalGroup` as power-series infrastructure, not an elliptic formal group implementation.**
7. **Keep exponent and finiteness separate.** Mathlib's APIs reflect the same distinction.
