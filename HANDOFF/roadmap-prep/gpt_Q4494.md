# Executive verdict

For rational prime torsion, the smallest classical route is not to formalize the full higher-degree Kamienny–Merel machine on X_1(p). It is the one-dimensional d=1 argument on X_0(p):

```plain text
P ∈ E(Q), order(P)=p
  -> the cyclic subgroup <P> gives x ∈ Y_0(p)(Q)
  -> at ℓ=3, x specializes to a rational cusp c
  -> map x-c into the winding quotient A_p of J_0(p)
  -> A_p(Q) is finite
  -> reduction on A_p(Q) at 3 is injective
  -> X_0(p) -> A_p is a formal immersion at c mod 3
  -> x=c
  -> contradiction, since x was noncuspidal.
```

This works uniformly for every prime p ≥ 11, hence certainly for p > 19. There is no necessary “check primes up to a large B, then invoke a tail theorem” stage in degree one. That finite-then-uniform architecture is relevant to higher-degree number fields, not to Q.

The exact formal-immersion test is:

```plain text
rank over F_ℓ of the first d q-expansion coefficient rows = d,
```

or equivalently

```plain text
T_1 e, ..., T_d e are F_ℓ-linearly independent in T e / ℓ T e.
```

For d=1, this collapses to one nonvanishing statement:

```plain text
[e] ≠ 0 in T e / 3 T e.
```

The hard mathematical input is the finiteness of the winding quotient. The formal-immersion criterion is local linear algebra once the modular-curve, Jacobian, Hecke, modular-symbol, and q-expansion interfaces exist. Unfortunately, those interfaces are almost entirely absent from current Mathlib, so the formalization engineering surrounding the local criterion is still very large.

The most honest practical Lean architecture is:

1. formalize a generic formal-immersion/unique-lift theorem;

1. isolate the deep tail as a named PrimeTailPackage p assumption, preferably split into three independently replaceable assumptions;

1. derive no_order_p from that package in a short theorem;

1. replace the assumptions only as modular-curve infrastructure matures.

# 1. Correct curve and quotient for the rational case

A rational point of exact order p gives more than a rational cyclic subgroup, so forgetting the generator produces

```plain text
x(E,<P>) ∈ Y_0(p)(Q).
```

Therefore X_0(p) is sufficient. This is substantially smaller than X_1(p):

- the cusps 0 and ∞ of X_0(p) are rational;

- the classical winding element naturally lives in the modular-symbol homology of X_0(p);

- no diamond-character bookkeeping is needed;

- the degree-one formal-immersion matrix has one row.

Do not identify the winding quotient with a generic object called J_1(p)^-. For the shortest rational proof, define

```plain text
J = J_0(p),
e = winding element {0,∞},
I_e = Ann_T(e),
A_p = J / I_e J.
```

For higher-degree variants one may use an intermediate X_H(p) or a quotient of J_1(p), but that extra generality buys nothing here.

# 2. Formal immersion: exact statement

## 2.1 Scheme-theoretic definition

Let R=Z_(ℓ), let X and Y be locally Noetherian R-schemes, and let

```plain text
f : X -> Y,
x ∈ X(k),
y=f(x),
k=F_ℓ.
```

The map f is a formal immersion at x if the induced map on completed local rings

```plain text
Ohat_{Y,y} -> Ohat_{X,x}
```

is surjective.

When X and Y are smooth at the relevant points, this is equivalent to surjectivity on cotangent spaces

```plain text
m_y/m_y^2 -> m_x/m_x^2,
```

or equivalently injectivity on tangent spaces.

Compile-shaped interface:

```javascript
structure IsFormalImmersionAt
    {R X Y : Type*}
    (f : X → Y) (x : X) : Prop where
  completeLocalRing_surjective :
    Function.Surjective (completedLocalRingMap f x)

-- Smooth finite-type version used computationally.
theorem isFormalImmersionAt_iff_cotangent_surjective
    (hf : IsMorphismOfSmoothSchemes f)
    (hx : SmoothAt X x)
    (hy : SmoothAt Y (f x)) :
    IsFormalImmersionAt f x ↔
      Function.Surjective (cotangentMap f x) := by
  sorry
```

The first generic theorem to formalize should be the uniqueness consequence, not the modular arithmetic:

```javascript
/-- A finite target image and a formal immersion prevent two integral points
in the same residue disk from having the same target image. -/
theorem eq_of_same_reduction_of_formalImmersion
    {R K k X A : Type*}
    (f : X → A)
    (x y : X)
    (hred : reduction x = reduction y)
    (hformal : IsFormalImmersionAt f (reduction x))
    (hinj : Function.Injective
      (reductionOnFiniteSubgroup (f '' rationalPoints X)))
    (himage : f x = f y) :
    x = y := by
  sorry
```

A more faithful implementation uses sections of proper smooth models over Z_(ℓ) and equality of formal completions, rather than raw functions between point types.

## 2.2 Symmetric-power q-expansion criterion

Let C=X_0(p), let A be a quotient of J_0(p), and define

```plain text
ι_d : Sym^d(C) -> A,
D |-> π([D-d∞]),
```

where π:J_0(p)->A is the quotient map.

Let

```plain text
V_A = π^* H^0(A, Ω^1) ⊆ S_2(Γ_0(p), Z_(ℓ)).
```

Choose an R-basis f_1,...,f_r of V_A, with

```plain text
f_j(q)=Σ_{n≥1} a_n(f_j) q^n,
ω_j=f_j(q) dq/q
   =Σ_{n≥1} a_n(f_j) q^{n-1} dq.
```

At the point d∞ of Sym^d(C), the cotangent map is represented by the d x r matrix

```plain text
M_d = (a_i(f_j) mod ℓ)_{1≤i≤d, 1≤j≤r}.
```

The precise criterion is

```plain text
ι_d is a formal immersion at d∞ mod ℓ
  iff rank_Fℓ(M_d)=d.
```

Thus dim A only has to be at least d; the matrix need not be square.

Compile-shaped statement:

```javascript
theorem formalImmersionAt_dInfinity_iff_qExpansion_rank
    (basis : Basis (Fin r) R (pullbackDifferentials A)) :
    IsFormalImmersionAt (symmetricAbelJacobi d A) (d • infinity) ↔
      Matrix.rank (qExpansionCoefficientMatrix basis d ℓ) = d := by
  sorry
```

## 2.3 Several cusps / the full Kamienny matrix

For a divisor

```plain text
D=n_1 <d_1>∞ + ... + n_m <d_m>∞,
Σ n_j=d,
```

the rows are indexed by (j,i) with 1≤i≤n_j. The coefficient matrix is

```plain text
(a_i(<d_j> f_k) mod ℓ)_{(j,i),k}.
```

In Hecke notation the equivalent condition is the linear independence of

```plain text
T_i <d_j> e,
1≤j≤m,
1≤i≤n_j,
```

in the appropriate winding module modulo ℓ.

This is the criterion needed for degree d>1; it is unnecessary for rational torsion.

# 3. Hecke/winding formulation

Let T be the integral Hecke algebra acting on the relevant modular-symbol module, and let

```plain text
e = image of the path {0,∞}
```

be the winding element. Put

```plain text
T e = {t e : t∈T}.
```

The q-expansion pairing identifies the n-th coefficient functional with T_n e. Consequently:

```plain text
rank(M_d)=d
  iff [T_1 e],...,[T_d e]
      are linearly independent in T e / ℓ T e.
```

For degree one:

```plain text
ι_1 : X_0(p) -> A_p
```

is a formal immersion at ∞ mod ℓ if and only if

```plain text
[e] ≠ 0 in T e / ℓ T e.
```

Equivalently, one may use a basis of differentials and check that at least one pullback differential has nonzero first q-coefficient modulo ℓ.

```javascript
theorem windingMap_formalImmersion_iff
    (p ℓ : ℕ) [Fact p.Prime] [Fact ℓ.Prime] :
    IsFormalImmersionAt (windingMap p) (cuspInfinityMod p ℓ) ↔
      windingElement p ∉ ℓ • windingCyclicModule p := by
  sorry
```

For ℓ=3 and prime p≥11, the needed nonvanishing is a uniform theorem in p, not a separate matrix computation for every prime.

# 4. The local specialization theorem

The formal immersion only controls the residue disk around a cusp. One separately proves that a rational p-torsion point lands in such a disk.

For a prime p≠3, let (E,P) be defined over Q with P of exact order p. Forgetting the generator gives x∈Y_0(p)(Q). The integral modular-curve analysis at 3 proves that, after possibly applying the Atkin–Lehner involution, the section x specializes to ∞.

The numerical threshold comes from

```plain text
p > (sqrt(3)+1)^2.
```

Since the right side is less than 8, every prime p≥11 qualifies. The proof combines:

- injectivity of prime-to-3 torsion under good reduction;

- the Hasse bound over F_3;

- the description of bad semistable fibers via Tate curves / the Deligne–Rapoport model;

- the two rational cusps 0,∞ of X_0(p).

It should be exposed as one uniform lemma:

```javascript
theorem orderPPoint_specializes_to_cusp_mod3
    {p : ℕ} (hp : p.Prime) (hp11 : 11 ≤ p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (P : (E⁄ℚ).Point)
    (hP : addOrderOf P = p) :
    let x := X0Point.ofCyclicSubgroup E P hp hP
    reductionAt3 x = cuspInfinity p ∨
      reductionAt3 (atkinLehner p x) = cuspInfinity p := by
  sorry
```

This local theorem is substantial algebraic geometry, but it is elementary relative to the winding-quotient finiteness theorem.

# 5. Finiteness of the winding quotient

Define

```plain text
A_p = J_0(p) / Ann_T(e) J_0(p).
```

The classical theorem is

```plain text
A_p(Q) is finite.
```

The mechanism is:

1. A_p is isogenous to the product of modular abelian factors whose associated newforms satisfy L(f,1)≠0;

1. Kolyvagin–Logachev prove finiteness/rank zero for the relevant J_0 factors;

1. Kato provides a more general route for quotients arising from J_1(N).

For Lean, the black-box statement should be finiteness, not merely “rank zero”:

```javascript
axiom windingQuotient_rationalPoints_finite
    (p : ℕ) (hp : p.Prime) (hp11 : 11 ≤ p) :
    Finite (WindingQuotient p).RationalPoint
```

If one axiomatizes only a rank equality, one still needs Mordell–Weil finite generation to turn rank zero into a finite group. The cited theorem already supplies the exact property used by formal immersion, so state it at that strength.

At the odd prime 3, the formal group of an abelian scheme has no nonzero torsion in the reduction kernel. Therefore reduction is injective on any finite subgroup of A_p(Q):

```javascript
theorem reduction_injective_on_finite_abVar_subgroup_at_odd_prime
    (A : AbelianSchemeOver ZLocalizedAt3)
    (G : AddSubgroup A.QPoints)
    [Finite G] :
    Function.Injective (reductionHom A G) := by
  sorry
```

This local lemma is a higher-dimensional analogue of the elliptic formal-kernel separatedness already being built for N15/N18.

# 6. Which part is mathematically hard?

## 6.1 Mathematical depth

The deepest imported theorem is:

```plain text
A_p(Q) is finite.
```

It packages modularity, special values L(f,1), and the Kolyvagin–Logachev/Kato rank theorem.

The formal-immersion criterion itself is local:

```plain text
completed local rings
  -> cotangent spaces
  -> first q-expansion coefficients
  -> one Hecke-module nonvanishing when d=1.
```

For Q, the matrix is one row. There is no high-dimensional determinant calculation.

## 6.2 Formalization depth

The engineering picture is less favorable. Even if finiteness is cited as an axiom, the following objects must exist before one can state the concrete formal-immersion theorem:

```plain text
- integral compactified X_0(p) over Z_(3);
- its cusps and special fiber;
- J_0(p) as a Picard/Jacobian abelian scheme;
- Abel–Jacobi map X_0(p)->J_0(p);
- Hecke correspondences and their action on J_0(p), differentials, and homology;
- modular symbols and the winding element;
- Ann(e) and the quotient abelian variety;
- integral q-expansion/cotangent comparison;
- Atkin–Lehner and specialization of the torsion point.
```

Thus the formal immersion is not the deepest theorem, but the missing infrastructure around it is enormous.

# 7. Uniformity: no finite matrix valid for all p

There is no single fixed numerical matrix whose rank proves the result for all p>19. The spaces

```plain text
S_2(Γ_0(p)), J_0(p), T_p, T_p e
```

change with p.

What is uniform is a theorem schema:

```plain text
∀ prime p≥11,
  [e_p] ≠ 0 in T_p e_p / 3 T_p e_p.
```

Its proof is symbolic in p, using the modular-symbol/Hecke structure. Once formalized, it is applied without running a per-prime computation.

The “finite cases followed by a uniform large-p bound” architecture appears in Kamienny–Merel arguments over number fields of degree d>1. There one checks linear independence of

```plain text
T_1e,...,T_de
```

and combines it with bounds such as an Oesterlé/Merel threshold. For d=1, that machinery collapses to the single class [e], and the local Hasse threshold is already below 11.

Therefore, for this project:

```plain text
p=11 and p=13 may be retained as independent fixed-curve proofs,
but the uniform tail theorem can naturally be stated for every p≥11
and would subsume them.
```

# 8. Minimal theorem package

## 8.1 Generic reusable local core

The genuinely reusable formalized core can be small:

```javascript
namespace FormalImmersion

structure IntegralPointData where
  -- Proper smooth models, generic and special points.
  sorry

structure FiniteTargetData (X A : Type*) where
  map : X → A
  finite_image : Set.Finite (map '' RationalPoints X)
  reduction_injective :
    Function.Injective (reductionOnImage map finite_image)

/-- Unique lifting from a residue disk through a finite target. -/
theorem unique_lift
    (D : FiniteTargetData X A)
    {x c : RationalPoints X}
    (hred : reduce x = reduce c)
    (hformal : IsFormalImmersionAt D.map (reduce c))
    (hmap : D.map x = D.map c) :
    x = c := by
  sorry

end FormalImmersion
```

This generic theorem is plausibly a bounded project by itself.

## 8.2 Modular theorem boundaries

The minimal concrete prime-tail layer should expose three named inputs:

```javascript
/-- Deep analytic/arithmetic input. -/
axiom windingQuotient_finite
    (p : ℕ) (hp : p.Prime) (hp11 : 11 ≤ p) :
    Finite (WindingQuotient p).QPoints

/-- Uniform Hecke/modular-symbol/q-expansion input. -/
axiom windingMap_formalImmersion_mod3
    (p : ℕ) (hp : p.Prime) (hp11 : 11 ≤ p) :
    IsFormalImmersionAt
      (windingMap p)
      (cuspInfinityMod3 p)

/-- Integral modular-curve reduction input. -/
axiom pTorsion_specializes_to_cusp_mod3
    (p : ℕ) (hp : p.Prime) (hp11 : 11 ≤ p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (P : (E⁄ℚ).Point)
    (hP : addOrderOf P = p) :
    SpecializesToInfinityOrAtkinLehner p E P
```

The final theorem is then short:

```javascript
theorem no_rational_prime_torsion_ge_11
    (p : ℕ) (hp : p.Prime) (hp11 : 11 ≤ p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ ∃ P : (E⁄ℚ).Point, addOrderOf P = p := by
  rintro ⟨P,hP⟩

  let x : X0 p |>.QPoint :=
    X0Point.ofPointOfExactOrder E P hp hP

  obtain hx | hx :=
    pTorsion_specializes_to_cusp_mod3 p hp hp11 E P hP

  · have himage : windingMap p x = windingMap p (cuspInfinity p) := by
      apply reduction_injective_on_finite_target
      simpa [hx]

    have hxcusp : x = cuspInfinity p :=
      FormalImmersion.unique_lift
        (windingTargetData p hp hp11)
        hx
        (windingMap_formalImmersion_mod3 p hp hp11)
        himage

    exact X0Point.noncuspidal_of_exactOrder hP
      (by simpa [hxcusp])

  · -- Apply the same argument to `w_p x`.
    exact no_torsion_from_atkinLehner_branch hp hp11 E P hP hx
```

The requested theorem follows immediately:

```javascript
theorem no_rational_prime_torsion_gt_19
    (p : ℕ) (hp : p.Prime) (hp19 : 19 < p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ ∃ P : (E⁄ℚ).Point, addOrderOf P = p := by
  exact no_rational_prime_torsion_ge_11
    p hp (by omega) E
```

# 9. Can only the rank-zero theorem be cited?

Yes, but phrase it as finiteness:

```javascript
Finite (A_p(Q)).
```

This legitimately cites the Kolyvagin–Logachev/Kato part. It leaves a conceptually elementary formalization project:

```plain text
modular curve + winding quotient construction
+ formal immersion
+ specialization at 3.
```

# 10. A more practical assumption boundary

For the current FLT repository, the cleanest interim assumption is a package rather than a vague “Mazur tail” axiom:

```javascript
structure PrimeTailPackage (p : ℕ) [Fact p.Prime] where
  X0QPoint : Type
  cuspInfinity : X0QPoint
  windingTarget : Type
  map : X0QPoint → windingTarget

  targetFinite : Finite windingTarget
  reduction : X0QPoint → X0SpecialPoint p 3
  targetReduction : windingTarget → WindingSpecialPoint p 3

  map_reduction_compat : ∀ x,
    targetReduction (map x) = specialMap (reduction x)

  reduction_injective_on_target :
    Function.Injective targetReduction

  formalImmersion_unique :
    ∀ x,
      reduction x = reduction cuspInfinity →
      map x = map cuspInfinity →
      x = cuspInfinity

  pointOfExactOrder :
    ∀ E : WeierstrassCurve ℚ, E.IsElliptic →
      ∀ P : (E⁄ℚ).Point,
        addOrderOf P = p → X0QPoint

  specializes :
    ∀ E hE P hP,
      reduction (pointOfExactOrder E hE P hP) =
        reduction cuspInfinity

  noncuspidal :
    ∀ E hE P hP,
      pointOfExactOrder E hE P hP ≠ cuspInfinity
```

Then the tail theorem is ten lines and the exact missing content is visible. This package is stronger than axiomatizing only rank zero, but it avoids pretending the surrounding modular geometry is already formalized.

A better long-term decomposition is the three axioms in section 8.2, because each can eventually be replaced independently.

# 11. Scale assessment

Approximate scales, assuming active expert development:

Even with winding-quotient finiteness cited, a fully concrete proof is likely a year-scale or multi-year infrastructure effort in the current ecosystem. Formalizing the analytic rank theorem itself would be much larger.

Therefore:

```plain text
- the mathematical core is minimal and elegant;
- the current Lean substrate is not close to supporting it;
- formalizing only the generic formal-immersion implication is tractable;
- an axiom-free uniform prime tail is the genuine research-scale obstruction
  in the full Mazur project.
```

# 12. Recommended project decision

## Near-term

1. Formalize the generic FormalImmersion.unique_lift theorem.

1. State the three named assumptions:

1. Derive the uniform p≥11 theorem and use it for the p>19 tail.

1. Keep the fixed p=11 and p=13 proofs as independent cross-checks and as usable results without the tail package.

## Long-term

Build infrastructure in this order:

```plain text
integral modular curves
 -> q-expansion/cotangent comparison
 -> Hecke correspondences and modular symbols
 -> winding quotient
 -> formal immersion theorem
 -> replace the finiteness black box only if a much larger program is desired.
```

Do not start by formalizing Merel's full number-field theorem or Kamienny's degree-d matrices. For the rational bound they add complexity without reducing the one irreducible deep input.
