# Executive verdict

Building N18 Route C as generic genus-two infrastructure is a very good investment, but the reuse boundary is narrower than “all modular seams of roughly genus two.” The correct split is:

1. Nearly verbatim reusable: balanced Mumford representatives for a squarefree monic sextic with two rational points at infinity; the oriented fractional-ideal realization of Pic⁰; uniqueness of reduced representatives; the abstract group law; base change; the hyperelliptic involution theorem h_* = -1; the dimension-free weak-descent/separatedness group glue; primary decomposition; finite-cardinality arguments.

1. Reusable after supplying per-curve data: push-pull for a degree-two elliptic quotient, the identity q^* q_* = 1 + σ_*, and the resulting (2,2)-isogeny identity Ψ ∘ Φ = [2]. The theorem is generic, but the involutions, quotient maps, elliptic models, exceptional charts, and polynomial identities are curve-specific.

1. Substantial new arithmetic for every curve: the Selmer/Kummer image certificate, local images, the relevant number-field arithmetic, torsion generators, good-reduction tables, and the final rational-point fiber enumeration.

1. Important correction for N25: the full modular curve X₁(25) has genus 12, not genus 2. The commonly used bivariate P₂₅=0 model is a genus-4 quotient of X₁(25). Therefore the N18 genus-two module does not directly instantiate on N25 unless a further explicit genus-two quotient or auxiliary curve is first proved to control the desired rational points.

For X₁(13), the payoff is large. It is genus two and becomes bielliptic over

Its Jacobian is irreducible over ℚ but splits up to isogeny over K into the two elliptic quotient curves. Thus the N18 Picard and push-pull architecture ports very well. The rank-zero arithmetic does not port verbatim: either perform full elliptic 2-descent on the two quotient factors over K, or bypass the splitting and apply one generic genus-two Jacobian 2-descent directly to J₁(13).

# 1. What the N18 genus-two core should actually abstract

## 1.1 The right generic model

The exact N18 balanced representation is generic for an even-degree split-infinity genus-two model:

```javascript
structure EvenGenusTwoModel (K : Type*) [Field K] where
  f              : K[X]
  monic_f        : f.Monic
  natDegree_f    : f.natDegree = 6
  squarefree_f   : f.Squarefree
  two_ne_zero    : (2 : K) ≠ 0
```

The smooth projective completion of

has two points at infinity because f is monic of degree six. The computational representative can be kept in the N18 shape:

```javascript
structure BalancedMumford (C : EvenGenusTwoModel K) where
  u             : K[X]
  v             : K[X]
  nInf          : ℤ
  u_monic       : u.Monic
  u_degree      : u.natDegree ≤ 2
  v_reduced     : v % u = v
  curve_dvd     : u ∣ C.f - v^2
  balanced      : BalancedInfinityCondition u.natDegree nInf
```

The exact inequality or centered-counter convention in balanced may differ, but it must be frozen once and used consistently in negation, reduction, base change, and Abel–Jacobi.

This is curve-agnostic in the meaningful Lean sense: all proofs should depend only on monic_f, natDegree_f, squarefree_f, and characteristic not two. The N18 polynomial should appear only in an instance file.

There should be a separate adapter for an odd-degree model y²=f(x) with deg f=5, where there is one rational point at infinity and no orientation integer is needed. Trying to force both cases into one low-level data type will make reduction proofs harder. A shared semantic PicZero layer can sit above two computational front ends:

```plain text
OddGenusTwoMumford      -- one point at infinity
EvenSplitMumford        -- two points at infinity, oriented/balanced
```

## 1.2 The reusable semantic theorem

The central theorem should be parameterized by the model, not by N18:

```javascript
noncomputable def reducedEquivPicZero
    (C : EvenGenusTwoModel K) :
    BalancedMumford C ≃+ OrientedPicZero C
```

Its proof contains the hard generic facts already being built for N18:

- I(u,v)=⟨u,y-v⟩ is invertible;

- principal oriented pairs define the quotient;

- every oriented class has a reduced balanced representative;

- that representative is unique;

- Cantor composition plus reduction computes multiplication of ideal classes.

Once this equivalence exists, associativity and commutativity are inherited from the quotient group and never reproved by branchwise Cantor algebra.

## 1.3 Hyperelliptic involution

For a model

the generic involution is

On the Mumford polynomial pair this is

```plain text
(u,v) ↦ (u, (-h-v) mod u).
```

For the N18-style equation y²=f(x) it is simply

```plain text
(u,v) ↦ (u, (-v) mod u).
```

The orientation counter changes according to the chosen convention because the involution swaps the two points at infinity. Do not make the downstream proof depend on the raw counter formula. Prove once at the semantic level:

```javascript
theorem hyperellipticStar_eq_neg
    (D : OrientedPicZero C) :
    hyperellipticStar C D = -D
```

and then derive the computational Mumford formula through reducedEquivPicZero. This theorem ports verbatim to every genus-two hyperelliptic curve with the corresponding infinity convention.

## 1.5 Separatedness and finite exponent

The group-theory part ports literally:

```javascript
def WeakDescent (p : ℕ) (H : AddSubgroup G) : Prop :=
  ∀ x : G, ∃ h : H, ∃ y : G, x = h + p • y

def InfinitelyPDivisible (p : ℕ) (x : G) : Prop :=
  ∀ n : ℕ, ∃ y : G, (p^n) • y = x
```

From weak descent, an exponent e for H, and p-adic separatedness, one gets e • x = 0 for every x, with no finite-generation theorem. The iterated-decomposition and primary-decomposition modules are dimension-free and should be imported unchanged by N18, X13, and any later abelian variety.

The local formal-group implementation is only partly generic. The reusable theorem should accept a filtration and a strict multiplication estimate. Constructing that filtration and proving the valuation estimate is per local model:

- for an elliptic quotient, reuse the one-dimensional elliptic formal-group layer;

- for an unsplit genus-two Jacobian, either build a two-dimensional formal-group chart, or use an alternative prime-to-residue-characteristic reduction argument after a direct descent.

# 2. X₁(13): the high-reuse case

## 2.1 Geometry to freeze as per-curve data

X₁(13) is genus two. Over

it is bielliptic: the extra involutions are defined over K, and quotienting by one of them gives an elliptic curve over K. Its Jacobian is irreducible over ℚ, but over K it is isogenous to the product of the two elliptic quotient curves.

This is exactly the geometric situation for which the N18 Φ/Ψ architecture was designed. The per-curve file must nevertheless certify all of the following explicitly:

```plain text
X13Model.lean
  squarefree monic sextic model
  the two points at infinity
  birational map from the modular/Tate model

X13CubicField.lean
  K = Q(ζ13)^+
  integral basis and embeddings needed downstream

X13Involutions.lean
  σ and τ = hσ
  σ² = τ² = 1
  preservation of the sextic equation

X13Quotients.lean
  invariant functions for q₊ and q₋
  equations of E₊ and E₋
  degree-two fiber identities
  exceptional-chart and infinity branches
```

The published model has two points at infinity, so the N18 even-degree balanced representation is the right front end after one explicit model-conversion certificate.

## 2.2 What ports verbatim

The following should require no X13-specific proof beyond instance parameters:

- mumfordIdeal_mul_conjugateIdeal;

- construction of the oriented fractional-ideal quotient;

- reduced representative existence and uniqueness;

- inherited AddCommGroup structure;

- coefficient extension and base-change injectivity via reduced uniqueness;

- h_*=-1;

- abstract q^*q_*=1+σ_* once the fiber certificate is supplied;

- abstract Ψ∘Φ=[2];

- iterated weak descent;

- no-infinite-p-divisibility implies a fixed annihilator;

- primary decomposition of a bounded-exponent group;

- finite-cardinality transfer through an injective reduction map;

- Abel–Jacobi injectivity and finite candidate-set packaging.

## 2.3 The rank-zero arithmetic is new

There are two coherent routes. They should not be mixed.

### Route X13-A: use the bielliptic splitting

After constructing

prove both elliptic groups finite. If the quotient curves have no usable rational 2-isogeny, the required arithmetic is a full elliptic 2-descent over K, not a genus-two Jacobian descent. The new module consists of:

- the cubic étale algebra of the 2-division polynomial of each elliptic factor;

- the Kummer map for E(K)/2E(K);

- the finite S-squareclass group;

- local images at the primes over 2, the discriminant, and infinity;

- a finite linear-algebra certificate spanning the global image;

- a weak-descent theorem E(K)=H+2E(K);

- elliptic 2-adic separatedness;

- good-reduction bounds and an explicit torsion generator.

The N18 separatedness and group glue still apply. What is new is the Kummer arithmetic and the local tables.

This route reuses the push-pull infrastructure most fully, but it pays the cost of arithmetic in a degree-six field twice.

### Route X13-B: direct genus-two 2-descent on J₁(13)

If a generic genus-two 2-descent module is available, the conceptually smaller route for the rational seam is often:

```plain text
J₁(13)(Q)/2J₁(13)(Q)
  → finite global squareclass target
  → weak descent on J₁(13)(Q)
  + 2-adic separatedness
  → fixed annihilator
  + good reduction
  → J₁(13)(Q) finite
  → enumerate Abel–Jacobi images
  → only cusps.
```

This route does not need the splitting field or the elliptic factors at all. It is especially attractive if the final theorem concerns only X₁(13)(ℚ), rather than all K-rational points.

### Recommendation for X13

- If the explicit involutions and elliptic quotient equations are already available and the two elliptic 2-descents over K are small, use Route X13-A.

- If the generic genus-two Jacobian 2-descent is being built for several seams, use Route X13-B over ℚ. It avoids the largest per-curve number-field layer.

The statement “the elliptic factors need genus-two 2-descent” should be corrected: elliptic factors need elliptic full 2-descent. The generic genus-two descent is the alternative applied directly to the Jacobian.

## 2.4 Expected reuse fraction for X13

With only the completed N18 Picard/push-pull/separatedness infrastructure, but no generic genus-two descent yet:

```plain text
about 45–60% reusable,
about 40–55% new curve/number-field/descent arithmetic.
```

After a generic genus-two 2-descent module exists:

```plain text
about 70–80% reusable for the direct-over-Q route,
about 20–30% per-curve model, local images, and enumeration.
```

The split-over-K route is more like 55–70% reuse because its field and elliptic-factor arithmetic remain substantial.

# 3. N25: the genus correction changes the architecture

## 3.1 The standard objects are not genus two

The full curve X₁(25) has genus 12. A standard bivariate model used in explicit computations is a quotient

of genus 4. Therefore neither object can be represented by the genus-two reduced condition deg u ≤ 2.

Consequences:

- the N18 balanced-Mumford type is not a model of Pic⁰(X₁(25));

- the genus-two uniqueness theorem does not apply;

- the 16-element geometric 2-torsion bound becomes wrong;

- a (2,2) split into two elliptic curves cannot account for a genus-4 or genus-12 Jacobian;

- the final ≤42 degree-two-fiber pattern from N18 has no automatic analogue.

## 3.2 What still ports

The following higher-level abstractions remain reusable:

- weak descent plus separatedness implies a fixed exponent;

- primary decomposition of bounded-exponent groups;

- injective prime-to-residue-characteristic reduction on torsion;

- finite-cardinality comparison;

- generic push-pull identities for any explicitly supplied finite morphism;

- the local one-dimensional formal-group layer for any elliptic factors that arise;

- some ideal and fractional-ideal helper lemmas.

But the genus-two canonical-representative implementation itself does not port to the full N25 curve or its genus-four quotient.

## 3.3 Three viable N25 strategies

### Strategy N25-A: find and certify a genus-two quotient

This is the only route in which the existing N18 module applies directly. One must provide:

```javascript
Y25 : EvenGenusTwoModel ℚ
pi25 : X25Point → Y25Point
```

and then prove a theorem strong enough for the modular application, for example:

```javascript
theorem noncusp_maps_to_relevant_point
    (P : X25Point)
    (hP : ¬ IsCusp P) :
    ¬ IsBoundary (pi25 P)
```

or an explicit finite fiber theorem that reconstructs every possible noncuspidal point from points on Y25.

The existence of a genus-two quotient must not be inferred merely from a bivariate equation. It needs an automorphism subgroup, a genus computation, and explicit invariant functions. This quotient/lifting theorem may be the dominant new N25 seam.

### Strategy N25-B: decompose the genus-four quotient

Use the automorphism/newform decomposition of the genus-four quotient and reduce its rational points to elliptic or abelian-surface factors. The generic push-pull algebra may help, but all quotient maps and factor descents are new per-curve mathematics.

### Strategy N25-C: generalize the divisor-class engine beyond genus two

A hyperelliptic genus-four curve would use reduced u of degree at most four, and the oriented reduction/uniqueness theorem would have to be generalized. If the genus-four quotient is not hyperelliptic, Mumford pairs are not the right representation at all. This is a much larger project than instantiating N18.

## 3.4 Expected reuse fraction for N25

For the full standard N25 seam:

```plain text
about 15–30% direct reuse,
about 70–85% new geometry, factorization, and arithmetic.
```

If a verified genus-two quotient is found and proved to control the relevant rational points, then the quotient subproblem becomes roughly 60–75% instantiation. The quotient construction and lifting/fiber theorem remain major new work and should be budgeted separately.

## 4.4 Finite computation versus genuine theory

The following are finite certificate checks:

- the factorization of f and the étale-algebra basis;

- an S-unit/squareclass basis;

- valuation vectors modulo 2;

- residue square tests and local image tables;

- matrices over F₂;

- intersection of global and local subspaces;

- the list of candidate Selmer classes;

- good-reduction group cardinalities;

- final Mumford or fiber enumeration.

The following are genuine reusable theory and should not be hidden behind decide:

- well-definedness of δ under principal divisors;

- compatibility of δ with Cantor/ideal-class addition;

- ker δ = 2J(K);

- the even-degree infinity correction;

- unramifiedness outside S;

- soundness of localization and local Kummer images;

- the theorem converting a finite Selmer span into weak descent.

The single hardest generic lemma is:

```javascript
theorem delta_ker_eq_doubleRange
```

for the true even-degree target. It is the point where divisor theory, the norm witness, and the infinity orientation all meet.

# 6. Bottom line

## X₁(13)

N18 pays off strongly. The Mumford/Picard machinery, hyperelliptic involution, push-pull algebra, separatedness, primary decomposition, and finite-exhaustion framework all port. The new mathematics is the explicit X13 model and quotient data plus a rank-zero certificate. With a generic genus-two 2-descent already available, X13 becomes mostly an instance file.

## N25

N18 does not make the standard N25 seam cheap because the relevant curves are genus 12 and genus 4, not genus 2. The reusable abstract group theory remains valuable, but a quotient/factorization theorem is needed before the genus-two engine can enter. Treat “N25 is genus two” as a hypothesis to be proved for a specific auxiliary quotient, not as a property of X₁(25) or its usual bivariate quotient.

## Strategic recommendation

Finish N18 with the generic interfaces above, then build the true even-degree genus-two 2-descent. That pair of investments makes X₁(13) substantially cheaper and will cover future genuine genus-two seams. Do not use N25 as the justification for the generic genus-two descent until an explicit genus-two quotient controlling its rational points has been certified.
