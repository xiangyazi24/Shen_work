## The upper bound is separate

The barrier proves a lower bound only:

```javascript
0 < w t x ≤ u t x
```

It says nothing about u t x ≤ M. The upper/sup bound must come from the Picard ball/map-certificate side. In the B-form Picard core, the relevant form is the limit-ball lemma:

```javascript
theorem conjugatePicardLimit_bounded ... :
  ∀ t, 0 < t → t ≤ T → ∀ x : intervalDomainPoint,
    |conjugatePicardLimit p u₀ T t x| ≤ M
```

This is the correct source of the hbound field. From it, an ordinary one-sided upper bound is immediate:

```javascript
have hAbs : |u t x| ≤ M := hbound t ht htT x
have hu_le_M : u t x ≤ M :=
  (le_abs_self (u t x)).trans hAbs
```

If the downstream field wants exactly |u| ≤ R, use hbound directly. If it wants u ≤ M, derive it as above. If it wants both 0 ≤ u and u ≤ M, combine:

```javascript
have hpos : 0 < u t x := hbarrier_pos t x ht htT
have hnonneg : 0 ≤ u t x := le_of_lt hpos
have hupper : u t x ≤ M := (le_abs_self (u t x)).trans (hbound t ht htT x)
```

## Composition into Jensen / assembly fields

For truncatedJensenStrictPosDataFor_of_strictPos, the relevant inputs are structurally independent:

```javascript

(hbound : ∀ s, 0 < s → s ≤ T → ∀ x, |u s x| ≤ R)
(hpos   : ∀ t, 0 < t → t ≤ T → ∀ x, 0 < u t x)
```

Use:

- hnonneg from the negative-part energy result, or from hpos by le_of_lt after strict positivity is available.

- hpos from the matched-divergence barrier comparison.

- hbound from the Picard ball/map certificate, not from the barrier.

This is not circular: the seed and comparison can be built from nonnegativity plus nontrivial mass/seed data; the upper bound is inherited from the already-built truncated Picard ball. The barrier is only responsible for the strict lower side.

## One small time-horizon caution

Some existing square-heat comparison lemmas are stated with t < T, while some Jensen data interfaces use t ≤ T. If a call site really needs positivity at the terminal time T, either run the positivity theorem on a slightly larger ambient horizon, or add the standard closed-time continuity endpoint upgrade. This is a time-endpoint issue, not a spatial endpoint issue. The spatial closed interval [0,1] is handled by the pointwise comparison plus squareHeatBarrier_pos.
