# Exact declaration

In ShenWork/Paper2/Statements.lean, IsPaper2ClassicalSolution is a def whose value is a seven-part right-associated conjunction. It is not a Lean structure with named fields.

```javascript
def IsPaper2ClassicalSolution
    (D : BoundedDomainData) (p : CM2Params) (T : ℝ)
    (u v : ℝ → D.Point → ℝ) : Prop :=
  0 < T ∧
    D.classicalRegularity T u v ∧
    (∀ t x, 0 < t → t < T → 0 < u t x) ∧
    (∀ t x, 0 < t → t < T → 0 ≤ v t x) ∧
    (∀ t x, 0 < t → t < T → x ∈ D.inside →
      D.timeDeriv u t x =
        D.laplacian (u t) x
          - p.χ₀ * D.chemotaxisDiv p (u t) (v t) x
          + u t x * (p.a - p.b * (u t x) ^ p.α)) ∧
    (∀ t x, 0 < t → t < T → x ∈ D.inside →
      0 = D.laplacian (v t) x - p.μ * v t x
        + p.ν * (u t x) ^ p.γ) ∧
    (∀ t x, 0 < t → t < T → x ∈ D.boundary →
      D.normalDeriv (u t) x = 0 ∧
      D.normalDeriv (v t) x = 0)
```

# Flattened component types

For

```javascript
hsol : IsPaper2ClassicalSolution D p T u v
```

the seven components are:

```javascript
-- 1. Positive lifespan
hT : 0 < T

-- 2. Classical regularity package
hreg : D.classicalRegularity T u v

-- 3. Strict positivity of u on the closed spatial domain
hpos : ∀ t x, 0 < t → t < T → 0 < u t x

-- 4. Nonnegativity of v on the closed spatial domain
hv_nonneg : ∀ t x, 0 < t → t < T → 0 ≤ v t x

-- 5. Parabolic u-equation, only at interior spatial points
hpde_u :
  ∀ t x, 0 < t → t < T → x ∈ D.inside →
    D.timeDeriv u t x =
      D.laplacian (u t) x
        - p.χ₀ * D.chemotaxisDiv p (u t) (v t) x
        + u t x * (p.a - p.b * (u t x) ^ p.α)

-- 6. Elliptic v-equation, only at interior spatial points
hpde_v :
  ∀ t x, 0 < t → t < T → x ∈ D.inside →
    0 = D.laplacian (v t) x - p.μ * v t x
      + p.ν * (u t x) ^ p.γ

-- 7. Homogeneous Neumann conditions at boundary points
hneumann :
  ∀ t x, 0 < t → t < T → x ∈ D.boundary →
    D.normalDeriv (u t) x = 0 ∧
    D.normalDeriv (v t) x = 0
```

The positivity and PDE/Neumann fields all use the open time interval 0 < t ∧ t < T; they do not include t = T.

# Paste-ready elimination

The cleanest way to unpack the conjunction is:

```javascript
rcases hsol with
  ⟨hT, hreg, hpos, hv_nonneg, hpde_u, hpde_v, hneumann⟩
```

The raw projection paths are:

```javascript
have hT         : 0 < T := hsol.1
have hreg       : D.classicalRegularity T u v := hsol.2.1
have hpos       := hsol.2.2.1
have hv_nonneg  := hsol.2.2.2.1
have hpde_u     := hsol.2.2.2.2.1
have hpde_v     := hsol.2.2.2.2.2.1
have hneumann   := hsol.2.2.2.2.2.2
```

Existing accessor lemmas include:

```javascript
hsol.T_pos
hsol.regularity
hsol.u_pos' ht0 htT
hsol.u_pos ht0 htT hx
hsol.v_nonneg ht0 htT
```

u_pos' is the closed-domain accessor and does not require an inside hypothesis. u_pos is the backward-compatible accessor that accepts an unused interior-membership argument.

# Initial trace is not a field

InitialTrace is a separate predicate involving an initial datum u₀:

```javascript
htrace : InitialTrace D u₀ u
```

It is not contained in IsPaper2ClassicalSolution, because that predicate has no u₀ argument. For the interval API the usual package therefore carries both:

```javascript
hsol   : IsPaper2ClassicalSolution intervalDomain p T u v
htrace : InitialTrace intervalDomain u₀ u
```

InitialTrace is the right-time sup-norm convergence of u(t) to u₀; it does not assert an initial trace for the elliptic variable v.

# Step-0 projection recommendation

A stable theorem-level Paper 2 API should expose these as separate named outputs:

```javascript
classicalSolution : IsPaper2ClassicalSolution intervalDomain p T u v
initialTrace      : InitialTrace intervalDomain u₀ u
```

and then project the classical-solution components through either the accessor lemmas or one local rcases, rather than hard-coding deep .2.2... paths throughout downstream Paper 3 files.
