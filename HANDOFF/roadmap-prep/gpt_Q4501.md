# Rational elliptic-curve torsion is finite via two good primes

## Executive verdict

The proposed argument is correct and does **not** use Mordell–Weil finite generation.

The proof separates cleanly into two layers:

1. **Pure group theory**, now given below as compile-checked Lean with no `sorry`:
   two homomorphisms from a group `T` to finite groups, with respectively `p`-primary and `q`-primary kernels for coprime `p,q`, combine to an injective map into a finite product. Hence `T` is finite.
2. **Local elliptic-curve input**, which is not yet fully present in Mathlib:
   for a good prime `p`, construct the reduction homomorphism on points and prove that its torsion kernel is `p`-primary.

At the repository's pinned Mathlib revision

```text
96fd0fff3b8837985ae21dd02e712cb5df72ec05
```

Mathlib already has:

```lean
WeierstrassCurve.IsIntegral
WeierstrassCurve.integralModel
WeierstrassCurve.IsMinimal
WeierstrassCurve.minimal
WeierstrassCurve.reduction
WeierstrassCurve.HasGoodReduction
WeierstrassCurve.hasGoodReduction_iff_isElliptic_reduction
```

but it does **not** yet have either of the two point-level results needed here:

```lean
red : E(K) →+ Ẽ(k)
```

for all points over a discretely valued field, or

```lean
P ∈ ker red ∧ IsOfFinAddOrder P ∧ Nat.Coprime (addOrderOf P) p → P = 0.
```

The generic `WeierstrassCurve.Affine.Point.map` is not a reduction map. It maps points along an algebra homomorphism **between fields**. The residue map is from a DVR to its residue field, while the source points live over its fraction field. Nonintegral affine points also reduce to the point at infinity, so coefficientwise affine reduction is insufficient.

Thus the right near-term implementation is:

```text
local point reduction + prime-to-p kernel theorem
                         ↓
PrimePowerKernelMap at p and q
                         ↓
fully formal group-theoretic theorem below
                         ↓
Finite (AddCommGroup.torsion (E⁄ℚ).Point)
```

```bash

```

and compiled successfully. Its only `sorry` is the deliberately isolated construction of the two good-reduction packages.

# 2. Compile-checked abstract proof

The following draft has only one `sorry`, in `exists_rational_two_good_primes`. Everything from the two local packages to finiteness is fully elaborated.

```lean
import Mathlib.AlgebraicGeometry.EllipticCurve.Affine.Point
import Mathlib.AlgebraicGeometry.EllipticCurve.Reduction
import Mathlib.GroupTheory.Torsion

open scoped WeierstrassCurve.Affine

namespace RationalTorsionTwoGoodPrimes

universe u

/-- A homomorphism to a finite group whose kernel is `p`-primary.

For good reduction of an elliptic curve, `Target` will be the point group
of the reduced curve over the finite residue field. -/
structure PrimePowerKernelMap (T : Type u) [AddCommGroup T] (p : ℕ) where
  Target : Type
  instAddCommGroupTarget : AddCommGroup Target
  instFiniteTarget : Finite Target
  red : T →+ Target
  kernel_pow : ∀ x : T, red x = 0 → ∃ n : ℕ, (p ^ n : ℕ) • x = 0

attribute [instance] PrimePowerKernelMap.instAddCommGroupTarget
attribute [instance] PrimePowerKernelMap.instFiniteTarget

/-- Two coprime integers cannot both kill a nonzero element of an
additive group. -/
theorem eq_zero_of_coprime_nsmul_eq_zero
    {T : Type u} [AddCommGroup T]
    {m n : ℕ} (hmn : m.Coprime n)
    {x : T} (hm : m • x = 0) (hn : n • x = 0) :
    x = 0 := by
  have hom : addOrderOf x ∣ m :=
    addOrderOf_dvd_iff_nsmul_eq_zero.mpr hm
  have hon : addOrderOf x ∣ n :=
    addOrderOf_dvd_iff_nsmul_eq_zero.mpr hn
  have hog : addOrderOf x ∣ Nat.gcd m n :=
    Nat.dvd_gcd hom hon
  rw [hmn.gcd_eq_one] at hog
  exact AddMonoid.addOrderOf_eq_one_iff.mp (Nat.dvd_one.mp hog)

/-- The product of two reduction maps with coprime-primary kernels is
injective. -/
theorem pair_reduction_injective
    {T : Type u} [AddCommGroup T]
    {p q : ℕ} (hpq : p.Coprime q)
    (Rp : PrimePowerKernelMap T p)
    (Rq : PrimePowerKernelMap T q) :
    Function.Injective (Rp.red.prod Rq.red) := by
  intro x y hxy
  have hp_xy : Rp.red x = Rp.red y := by
    simpa only [AddMonoidHom.prod_apply] using
      congrArg Prod.fst hxy
  have hq_xy : Rq.red x = Rq.red y := by
    simpa only [AddMonoidHom.prod_apply] using
      congrArg Prod.snd hxy

  let z : T := x - y
  have hp_zero : Rp.red z = 0 := by
    dsimp [z]
    simpa using sub_eq_zero.mpr hp_xy
  have hq_zero : Rq.red z = 0 := by
    dsimp [z]
    simpa using sub_eq_zero.mpr hq_xy

  obtain ⟨a, ha⟩ := Rp.kernel_pow z hp_zero
  obtain ⟨b, hb⟩ := Rq.kernel_pow z hq_zero

  have hz : z = 0 :=
    eq_zero_of_coprime_nsmul_eq_zero (hpq.pow a b) ha hb
  exact sub_eq_zero.mp hz

/-- Pure group theory: two finite reduction targets with coprime-primary
kernels make the source finite. -/
theorem finite_of_two_prime_power_kernel_maps
    {T : Type u} [AddCommGroup T]
    {p q : ℕ} (hpq : p.Coprime q)
    (Rp : PrimePowerKernelMap T p)
    (Rq : PrimePowerKernelMap T q) :
    Finite T :=
  Finite.of_injective (Rp.red.prod Rq.red)
    (pair_reduction_injective hpq Rp Rq)

/-- The rational torsion subgroup of an elliptic curve. -/
abbrev RationalTorsion
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :=
  AddCommGroup.torsion (E⁄ℚ).Point

/-- The complete local input at one good rational prime.

In the production implementation, `mapData.Target` is the point group of
`E`'s reduction at `p`, after choosing a suitable integral/minimal local
model and transporting points through the variable change. -/
structure RationalGoodReductionData
    (E : WeierstrassCurve ℚ) [E.IsElliptic] (p : ℕ) where
  prime : Nat.Prime p
  mapData : PrimePowerKernelMap (RationalTorsion E) p

/-- Two distinct good primes, packaged in exactly the form needed by the
finite group-theoretic argument. -/
structure RationalTwoGoodPrimes
    (E : WeierstrassCurve ℚ) [E.IsElliptic] where
  p : ℕ
  q : ℕ
  hp : Nat.Prime p
  hq : Nat.Prime q
  coprime : Nat.Coprime p q
  atP : PrimePowerKernelMap (RationalTorsion E) p
  atQ : PrimePowerKernelMap (RationalTorsion E) q

/-- The only unfinished arithmetic/geometric package.

Its proof chooses two primes away from the coefficient denominators and
from the discriminant, constructs point reduction at both primes, and
uses the formal group to prove that each torsion kernel is primary.
No Mordell--Weil finite generation is involved. -/
theorem exists_rational_two_good_primes
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Nonempty (RationalTwoGoodPrimes E) := by
  sorry

/-- Rational elliptic-curve torsion is finite via two good primes. -/
theorem rational_torsion_finite_via_two_good_primes
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Finite (AddCommGroup.torsion (E⁄ℚ).Point) := by
  obtain ⟨D⟩ := exists_rational_two_good_primes E
  exact finite_of_two_prime_power_kernel_maps
    D.coprime D.atP D.atQ

end RationalTorsionTwoGoodPrimes
```

The key line in the injectivity proof is:

```lean
have hz : z = 0 :=
  eq_zero_of_coprime_nsmul_eq_zero (hpq.pow a b) ha hb
```

It avoids manipulating prime factorizations or element orders by hand. If `z` is in both kernels, then `p^a • z = 0` and `q^b • z = 0`; `Nat.Coprime.pow` proves the two annihilators are coprime, and `addOrderOf_dvd_iff_nsmul_eq_zero` finishes.

# 3. What Mathlib currently provides for reduction

## 3.1 Curve-level reduction: present

`Mathlib/AlgebraicGeometry/EllipticCurve/Reduction.lean` currently defines the following chain.

For a fraction field `K` of a DVR `R`:

```lean
class WeierstrassCurve.IsIntegral
    (R : Type*) (W : WeierstrassCurve K) : Prop

noncomputable def WeierstrassCurve.integralModel
    (R : Type*) (W : WeierstrassCurve K) [W.IsIntegral R] :
    WeierstrassCurve R

class WeierstrassCurve.IsMinimal
    (R : Type*) (W : WeierstrassCurve K) : Prop

noncomputable def WeierstrassCurve.minimal
    (R : Type*) (W : WeierstrassCurve K) :
    WeierstrassCurve K

noncomputable def WeierstrassCurve.reduction
    (R : Type*) (W : WeierstrassCurve K) [W.IsMinimal R] :
    WeierstrassCurve (IsLocalRing.ResidueField R)

class WeierstrassCurve.HasGoodReduction
    (R : Type*) (W : WeierstrassCurve K) : Prop

lemma WeierstrassCurve.hasGoodReduction_iff_isElliptic_reduction ... :
  W.HasGoodReduction R ↔ (W.reduction R).IsElliptic
```

This is enough to define the **reduced Weierstrass equation** and certify that it is elliptic.

## 3.2 Field base change on points: present, but not reduction

Mathlib has:

```lean
WeierstrassCurve.Affine.Point.map
  (f : F →ₐ[S] K) :
  (W'.baseChange F).Point →+
  (W'.baseChange K).Point
```

and an injectivity theorem when the field map is injective:

```lean
WeierstrassCurve.Affine.Point.map_injective
```

This is useful for the embedding

```text
E(ℚ) → E(ℚ_p),
```

but not for

```text
E(ℚ_p) → E(𝔽_p).
```

There is no field homomorphism `ℚ_p → 𝔽_p`. Reduction is defined on the valuation ring, and a point with nonintegral affine coordinates must be handled projectively and may reduce to infinity.

## 3.3 Point-level reduction: missing

A reusable API should eventually look approximately like this:

```lean
namespace WeierstrassCurve

variable
  {R K : Type*}
  [CommRing R] [IsDomain R] [IsDiscreteValuationRing R]
  [Field K] [Algebra R K] [IsFractionRing R K]
  (W : WeierstrassCurve K) [W.HasGoodReduction R]

/-- Reduction on all points of a good minimal model. -/
noncomputable def Point.reduction :
    (W⁄K).Point →+ ((W.reduction R)⁄(ResidueField R)).Point :=
  ...

end WeierstrassCurve
```

The exact target may simplify because `W.reduction R` is already over `ResidueField R`; the displayed notation is schematic. The important feature is that the map must be an additive homomorphism on **all** local points.

A robust construction uses primitive projective coordinates. An affine-coordinate-only definition will have avoidable integrality side conditions.

# 4. Kernel of reduction and the formal-group gap

## 4.1 Desired theorem

The most directly reusable local theorem is:

```lean
/-- The torsion in the kernel of good reduction is residue-characteristic-primary. -/
theorem Point.kernel_reduction_torsion_primary
    {p : ℕ} [Fact p.Prime]
    (P : AddCommGroup.torsion (W⁄K).Point)
    (hP : Point.reduction R W P = 0) :
    ∃ a : ℕ, (p ^ a : ℕ) • P = 0 := by
  ...
```

An equivalent and often more standard interface is prime-to-`p` injectivity:

```lean
/-- Good reduction is injective on `n`-torsion when `n` is prime to the
residue characteristic. -/
theorem Point.reduction_injective_on_nTorsion
    {n p : ℕ} [Fact p.Prime]
    (hnp : Nat.Coprime n p) :
    Function.Injective
      (fun P : W.nTorsion n => Point.reduction R W P) := by
  ...
```

The first statement is more convenient for the two-prime proof; the second is a stronger library theorem.

## 4.2 Mathematical proof

For a minimal good model over a DVR:

1. Let `E₁(K)=ker(E(K)→Ẽ(k))`.
2. The local parameter `z=-x/y` identifies `E₁(K)` with the maximal ideal equipped with the elliptic formal-group law.
3. If `n` is a unit in `R`, the formal multiplication series satisfies
   
   ```text
   [n](T)=nT+terms of degree at least 2.
   ```
   
   Hence `[n]` is an automorphism of the formal group, or at minimum has trivial kernel.
4. Therefore `E₁(K)` contains no nonzero torsion of order prime to the residue characteristic.
5. A finite-order element of `E₁(K)` consequently has `p`-power order.

This argument works at `p=2` as well, but the first implementation can deliberately choose two good primes `p,q ≥ 5`. There are infinitely many good primes, and avoiding `2` and `3` simplifies coefficient and formal-series lemmas.

## 4.3 Current Mathlib status

Mathlib has a new abstract one-dimensional formal-group-law file:

```text
Mathlib/RingTheory/FormalGroup/Basic.lean
```

but it does not currently connect a Weierstrass curve to:

- its elliptic formal group;
- evaluation of that formal group on a complete DVR's maximal ideal;
- the formal kernel of point reduction;
- the prime-to-residue-characteristic torsion theorem.

So this is a genuine library gap rather than a missing theorem-name lookup.

For the present FLT project, the smallest useful implementation need not build the whole general theory. A focused theorem for `ℚ_p`, or even for localizations/completions of `ℚ` at odd good primes, is enough to prove rational torsion finite. The existing N15/N21/N18 separatedness work may supply much of the local-parameter and valuation infrastructure, but the statement needed here is weaker than full separatedness: only the absence of prime-to-`p` torsion in the formal kernel.

# 5. Finiteness of the reduced point groups

Once a good prime `p` and reduction map are available, the target is finite for elementary type-theoretic reasons.

For a finite residue field `k`, Mathlib's affine point type has an equivalence of the form

```lean
W.nonsingularPointEquiv :
  W.Point ≃ WithZero {xy : k × k // W.Nonsingular xy.1 xy.2}
```

or the corresponding elliptic `pointEquivSubtype`. Since `k × k` is finite, the subtype is finite, and adjoining the point at infinity remains finite.

Thus one should add or invoke an instance resembling:

```lean
noncomputable instance instFinitePoint
    {k : Type*} [Field k] [Finite k]
    (W : WeierstrassCurve k) [W.IsElliptic] :
    Finite W.toAffine.Point :=
  Finite.of_injective W.toAffine.nonsingularPointEquiv
    W.toAffine.nonsingularPointEquiv.injective
```

The precise namespace and equivalence argument should follow the current `Affine/Point.lean` API. This is small plumbing, not arithmetic.

In the compile-checked draft, `PrimePowerKernelMap` stores

```lean
instFiniteTarget : Finite Target
```

so the group-theory layer is independent of how that instance is obtained.

# 6. Choosing two good rational primes

The existence of two good primes is elementary.

A clean route is:

1. Replace `E/ℚ` by an isomorphic integral Weierstrass model. Clearing denominators gives one explicitly; alternatively use the existing local `exists_isMinimal` machinery separately at each prime.
2. Let `S` contain:
   - primes occurring in denominators of the chosen coefficients;
   - primes dividing the nonzero discriminant;
   - optionally `2` and `3`.
3. Choose distinct primes `p,q ∉ S` by infinitude of primes.
4. At each such prime, the coefficients are integral and the discriminant is a unit, so the reduction is elliptic and hence good.

For formalization, choosing an integral global model first is likely easier than choosing unrelated minimal local models, because it avoids transporting the source group through two separate variable changes. Minimality is automatic at a prime where the discriminant valuation is zero.

The prime-selection lemma can be packaged independently:

```lean
lemma exists_two_distinct_primes_not_mem
    (S : Finset ℕ) :
    ∃ p q : ℕ,
      p.Prime ∧ q.Prime ∧ p ≠ q ∧ p ∉ S ∧ q ∉ S := by
  ...
```

No analytic prime theorem is required.

# 9. Direct answers

## (1) Exact Lean statement

```lean
theorem WeierstrassCurve.rational_torsion_finite_via_two_good_primes
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Finite (AddCommGroup.torsion (E⁄ℚ).Point)
```

The statement is MW-FG-free.

## (2) Does Mathlib have the reduction map?

**Curve reduction: yes. Point reduction: no.**

Existing API:

```lean
WeierstrassCurve.reduction
WeierstrassCurve.HasGoodReduction
WeierstrassCurve.hasGoodReduction_iff_isElliptic_reduction
```

`WeierstrassCurve.Affine.Point.map` is only a field-base-change map and cannot serve as residue reduction. A new additive homomorphism on local points is required.

## (3) Is prime-to-p torsion injectivity present?

No theorem matching it was found at the pinned revision. The abstract formal-group-law library does not yet construct the elliptic formal group or identify it with the kernel of reduction. This is the main mathematical API gap.

The recommended theorem is either:

```lean
kernel_reduction_torsion_primary
```

or the stronger

```lean
reduction_injective_on_nTorsion
```

for `Nat.Coprime n p`.

## (4) Product injectivity and finiteness

This part is complete and compile-checked:

```lean
Finite.of_injective (Rp.red.prod Rq.red)
  (pair_reduction_injective hpq Rp Rq)
```

The proof uses only:

```lean
Nat.Coprime.pow
addOrderOf_dvd_iff_nsmul_eq_zero
AddMonoid.addOrderOf_eq_one_iff
AddMonoidHom.prod
Finite.of_injective
```

No descent, height, Mordell–Weil, analytic rank, or finite-generation theorem appears.

# Bottom line

This is the right elementary route and should be used to remove the torsion-finiteness dependence on Mordell–Weil finite generation. The final theorem is not blocked by difficult global arithmetic. It is blocked by a focused local API seam:

```text
reduction on all elliptic-curve points
+
formal-kernel has no prime-to-p torsion.
```

Once that seam is supplied at two odd good primes, the rest of the proof is already a small, checked Lean argument.
