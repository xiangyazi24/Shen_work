# 2. Explicit bielliptic split over K13

## 2.1 Möbius involution

Put

```plain text
b0 = -a^5+5a^3-6a,
cMob = b0-13.
```

On the original sextic x-coordinate, one elliptic involution has

```plain text
x |-> (x+b0)/(cMob*x-1).
```

For Lean, do not use this form to construct the quotient. Use the following birationally equivalent even-sextic coordinates, where the involutions become sign changes.

## 2.2 Even-sextic coordinates

Define

```plain text
d1 = -a^5+2a^4+3a^3-6a^2+1,
d2 =  a^5-2a^4-5a^3+8a^2+6a-5,

X = (x+d1)/(x+d2),
Y = y/(x+d2)^3.
```

Then

```plain text
Y^2 = c6*X^6+c4*X^4+c2*X^2+c0,
```

```plain text

c4 = (40a^5-238a^3-9a^2+296a+69)/208,
```


# 4. The shortest rank-zero proof without deep modularity

## 4.1 Why separatedness alone is insufficient

The p-adic formal kernel theorem only says that a point inside the formal kernel cannot be nonzero and infinitely 2-divisible. It does not make an arbitrary Mordell–Weil point infinitely divisible. A weak descent is still required.

Therefore this is not a valid standalone route:

```plain text
reduction + separatedness => rank zero.             -- false
```

The valid route is:

```plain text
weak 2-descent
  => 19*D is infinitely 2-divisible
  => Phi(19*D) is infinitely 2-divisible in Eplus x Eminus
  => a finite reduction exponent moves it into formal kernels
  => separatedness kills that multiple
  => Psi Phi=[2] gives an explicit annihilator of D.
```

## 4.2 Fixed fake-2-Selmer certificate

Let

```plain text
L13=Q[T]/(f13(T)).
```

For the even sextic descent the target is the norm-compatible quotient

```plain text
L13^*/(L13^{*2} Q^*).
```

The fixed implementation should not attempt generic class-group or S-unit algorithms in Lean. Store their output as a certificate:

```javascript
structure J13FakeSelmerCertificate where
  Candidate : Type
  candidateFintype : Fintype Candidate
  candidateDecEq : DecidableEq Candidate

  decode : Candidate → L13

  -- Every global square class unramified outside S is represented.
  global_complete :
    ∀ ξ : L13, GlobalAdmissible13 ξ →
      ∃ c : Candidate, SameSquareClassModQ ξ (decode c)

  badPlaces : Finset Place13

  locallyAdmissible : Place13 → Candidate → Prop
  locallyAdmissible_decidable :
    ∀ v c, Decidable (locallyAdmissible v c)

  -- Finite local enumeration result.
  surviving : Finset Candidate
  surviving_spec :
    ∀ c, c ∈ surviving ↔
      ∀ v ∈ badPlaces, locallyAdmissible v c

  -- The survivors are represented by known cuspidal classes.
  survivor_from_H13 :
    ∀ c ∈ surviving,
      ∃ h : H13, kummer13 (h : J13Q) = classOf (decode c)
```

The descent theorem extracted from it is:

```javascript
theorem J13_weakTwoDescent
    (cert : J13FakeSelmerCertificate) :
    ∀ D : J13Q,
      ∃ h : H13, ∃ Q : J13Q,
        D = (h : J13Q) + 2 • Q := by
  intro D
  let c := cert.classify (kummer13 D)
  obtain ⟨h, hh⟩ := cert.survivor_from_H13 c (localSolubility_of_point D)
  have hker : kummer13 (D-(h:J13Q)) = 0 := by
    simpa [map_sub, hh]
  obtain ⟨Q,hQ⟩ := kummer13_kernel_fixed hker
  exact ⟨h,Q,by linarith⟩
```

The theorem kummer13_kernel_fixed may be packaged as part of the fixed descent certificate if a generic ker(delta)=2J theorem has not landed.

What specifically remains computational: the integral basis of L13, its class group modulo squares, an S-unit basis, the finite candidate list, and the local-image matrices at the primes dividing 2*13*disc(f13) and at infinity. These numbers are not contained in the current prompt or repository search. The Lean draft therefore leaves only J13FakeSelmerCertificate as arithmetic input.
