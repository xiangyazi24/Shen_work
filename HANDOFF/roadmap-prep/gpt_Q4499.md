# `PrimeTailPackage p`: a precise replacement for the opaque prime-tail axiom

## Verdict

The clean formalization has **two layers**.

1. `PrimeTailLevelData p` is reusable geometric infrastructure for the level-`p` formal-immersion argument: the integral model of `X₀(p)` at `3`, the cusp, the winding Abel–Jacobi map, reduction, the local reduction theorem for a rational `p`-torsion point, the good-reduction torsion-kernel lemma, and the generic unique-lift consequence of formal immersion.
2. `PrimeTailPackage p` contains exactly the two genuinely deep, per-level arithmetic inputs:

   ```text
   A_p(Q) is finite;
   [e] ≠ 0 in T e / 3 T e.
   ```

This separation is important. Putting the whole conclusion

```lean
¬ HasRationalPointOfOrder E p
```

inside a package would merely rename the old axiom. The package below does not contain that conclusion, nor any statement equivalent to it.

The theorem requested in the question applies to `19 < p`, hence to prime `p ≥ 23`. The fixed primes `11`, `13`, `17`, and `19` remain separate files unless the local-reduction theorem is strengthened and its exceptional cases are handled uniformly.

# 1. Exact Lean interface

The following is the recommended repository contract. It is intentionally independent of a not-yet-existing Mathlib Picard/Jacobian/modular-curve API.

```lean
import Mathlib.AlgebraicGeometry.EllipticCurve.Projective.Point
import Mathlib.GroupTheory.OrderOfElement

set_option autoImplicit false

namespace MazurProof

/-!
Use the repository's existing declaration instead of this adapter if it has
already been frozen elsewhere.
-/

def HasRationalPointOfOrder
    (E : WeierstrassCurve ℚ) [E.IsElliptic] (n : ℕ) : Prop :=
  ∃ P : WeierstrassCurve.Projective.Point E,
    addOrderOf P = n

namespace PrimeTail

universe u

/-- A small bundled additive commutative group.

This avoids dependent instance-plumbing in `PrimeTailLevelData`. Replace it by
whatever bundled-group object the project ultimately standardizes on.
-/
structure AddCommGroupObj where
  carrier : Type u
  str : AddCommGroup carrier

namespace AddCommGroupObj

instance : CoeSort (AddCommGroupObj.{u}) (Type u) where
  coe G := G.carrier

instance (G : AddCommGroupObj.{u}) : AddCommGroup G :=
  G.str

end AddCommGroupObj

/--
Reusable geometry and local algebra for the degree-one prime-tail argument.

This is *not* the deep per-prime assumption package. It is an implementation
interface to be proved once for the family `X₀(p)` and its winding quotient.

Intended meanings:

* `X0Q` and `X0F3` are the rational and `F₃` point types of the proper
  integral model of `X₀(p)`;
* `AQ` and `AF3` are the rational and special-fibre point groups of the
  winding quotient `A_p`;
* `Te` is the integral Hecke lattice `T · e` dual to the cotangent lattice of
  `A_p`; its normalization must be the one used in the formal-immersion
  criterion;
* `tailPoint` is the image on `X₀(p)` of the rational order-`p` subgroup,
  after applying the Atkin--Lehner involution when necessary so that it
  reduces to `∞` modulo `3`.
-/
class PrimeTailLevelData (p : ℕ) [Fact p.Prime] where
  X0Q  : Type u
  X0F3 : Type u

  AQ   : AddCommGroupObj.{u}
  AF3  : AddCommGroupObj.{u}
  Te   : AddCommGroupObj.{u}

  infinityQ  : X0Q
  infinityF3 : X0F3

  redX : X0Q → X0F3
  redA : AQ →+ AF3

  /-- The normalized map `x ↦ π([x]-[∞])` to the winding quotient. -/
  windingMapQ  : X0Q → AQ
  windingMapF3 : X0F3 → AF3

  red_infinity : redX infinityQ = infinityF3
  windingMapQ_infinity : windingMapQ infinityQ = 0
  windingMapF3_infinity : windingMapF3 infinityF3 = 0
  windingMap_reduce :
    ∀ x : X0Q,
      redA (windingMapQ x) = windingMapF3 (redX x)

  /-- The integral winding element `e = {0,∞}` in the correct Hecke lattice. -/
  windingElement : Te

  /--
  Temporary pointwise formal-immersion predicate.

  When a completed-local-ring API is available, replace this abstract `Prop`
  by surjectivity of the induced map on completed local rings at `∞`, or by
  injectivity of the tangent map plus the required smoothness hypotheses.
  -/
  formalImmersionAtInfinity : Prop

  /--
  The modular-symbol/q-expansion criterion. For a one-dimensional source,
  nonvanishing of the differential is enough for formal immersion.
  -/
  formalImmersion_of_windingClass :
    (¬ ∃ t : Te, (3 : ℕ) • t = windingElement) →
      formalImmersionAtInfinity

  /--
  Good reduction at `3`: the kernel of reduction on `A_p(Q₃)` has no
  nonzero torsion. Since `AQ` is embedded in the local points, this is the
  exact form consumed below.
  -/
  reductionKernel_torsionFree :
    ∀ a : AQ,
      IsOfFinAddOrder a →
      redA a = 0 →
      a = 0

  /--
  Generic formal-immersion unique-lift lemma.

  Two `Z₃`-sections with the same special point and the same image in `A_p`
  agree when the map is a formal immersion at that special point.
  Properness supplies the sections from rational points.
  -/
  uniqueLiftAtInfinity :
    formalImmersionAtInfinity →
    ∀ x : X0Q,
      redX x = infinityF3 →
      windingMapQ x = windingMapQ infinityQ →
      x = infinityQ

  /--
  Modular interpretation plus local orientation. The returned point is the
  `X₀(p)` point of the cyclic subgroup generated by the rational torsion
  point, possibly after `w_p`.
  -/
  tailPoint :
    ∀ (E : WeierstrassCurve ℚ) [E.IsElliptic],
      HasRationalPointOfOrder E p →
      X0Q

  /-- The modular point of a genuine elliptic curve is not the cusp `∞`. -/
  tailPoint_ne_infinity :
    ∀ (E : WeierstrassCurve ℚ) [E.IsElliptic]
      (hP : HasRationalPointOfOrder E p),
      tailPoint E hP ≠ infinityQ

  /--
  Uniform local reduction theorem used in the tail: for prime `p > 19`, one
  of the two Atkin--Lehner orientations reduces to `∞` modulo `3`.
  -/
  tailPoint_reduces :
    ∀ (E : WeierstrassCurve ℚ) [E.IsElliptic]
      (hP : HasRationalPointOfOrder E p),
      19 < p →
      redX (tailPoint E hP) = infinityF3

/--
The literal membership-free form of

    [e] ≠ 0 in (T · e) / 3(T · e).

It says exactly that the winding element is not in the image of
multiplication by `3` on the chosen integral Hecke lattice.
-/
def WindingClassNonzeroModThree
    {p : ℕ} [Fact p.Prime] [L : PrimeTailLevelData p] : Prop :=
  ¬ ∃ t : L.Te, (3 : ℕ) • t = L.windingElement

/--
The sharp deep-input bundle for one prime level.

There are exactly two fields:

1. finiteness of the rational points of the winding quotient;
2. the mod-`3` winding-element nonvanishing certificate.

Everything else belongs to the reusable `PrimeTailLevelData` layer.
-/
class PrimeTailPackage
    (p : ℕ) [Fact p.Prime] [L : PrimeTailLevelData p] : Prop where
  windingQuotientFinite : Finite L.AQ
  windingClassNeZeroModThree : WindingClassNonzeroModThree (p := p)

/--
A finite additive group consists entirely of finite-order elements.

Current Mathlib already provides the exact theorem
`isOfFinAddOrder_of_finite`; this local wrapper records the only use made of
finiteness in the assembly.
-/
private theorem finitePoint_isOfFinAddOrder
    {G : Type u} [AddCommGroup G] [Finite G] (x : G) :
    IsOfFinAddOrder x := by
  exact isOfFinAddOrder_of_finite x

/--
Formal-immersion prime tail.

The proof contains no Mordell--Weil finite-generation argument and no opaque
prime-torsion theorem. It only composes the two package fields with the
reusable level infrastructure.
-/
theorem no_order_p_of_primeTailPackage
    {p : ℕ} [Fact p.Prime] [L : PrimeTailLevelData p]
    (pkg : PrimeTailPackage p)
    (hp : 19 < p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E p := by
  intro hP

  -- Deep input (a): `A_p(Q)` is finite.
  letI : Finite L.AQ := pkg.windingQuotientFinite

  -- Choose the `X₀(p)` orientation that reduces to `∞` at 3.
  let x : L.X0Q := L.tailPoint E hP

  have hxred : L.redX x = L.infinityF3 := by
    simpa [x] using L.tailPoint_reduces E hP hp

  have hxne : x ≠ L.infinityQ := by
    simpa [x] using L.tailPoint_ne_infinity E hP

  -- The normalized winding-map image reduces to zero.
  have hred0 : L.redA (L.windingMapQ x) = 0 := by
    calc
      L.redA (L.windingMapQ x)
          = L.windingMapF3 (L.redX x) :=
        L.windingMap_reduce x
      _ = L.windingMapF3 L.infinityF3 := by
        rw [hxred]
      _ = 0 :=
        L.windingMapF3_infinity

  -- Finiteness makes the image torsion; the good-reduction formal kernel is
  -- torsion-free, so the image itself is zero.
  have htor : IsOfFinAddOrder (L.windingMapQ x) :=
    finitePoint_isOfFinAddOrder (L.windingMapQ x)

  have hmap0 : L.windingMapQ x = 0 :=
    L.reductionKernel_torsionFree
      (L.windingMapQ x) htor hred0

  have hsame :
      L.windingMapQ x = L.windingMapQ L.infinityQ := by
    calc
      L.windingMapQ x = 0 := hmap0
      _ = L.windingMapQ L.infinityQ :=
        L.windingMapQ_infinity.symm

  -- Deep input (b): `[e]` is nonzero modulo 3; the generic q-expansion
  -- criterion turns this into formal immersion.
  have hformal : L.formalImmersionAtInfinity :=
    L.formalImmersion_of_windingClass
      pkg.windingClassNeZeroModThree

  -- Formal immersion gives uniqueness of the lift of the cusp.
  have hxinf : x = L.infinityQ :=
    L.uniqueLiftAtInfinity hformal x hxred hsame

  exact hxne hxinf

end PrimeTail
end MazurProof
```

## Optional typeclass-style invocation

Because `PrimeTailPackage` is declared as a class, a canonical per-prime certificate may be installed as an instance. The explicit theorem above is preferable at audit boundaries, but the call site can be shortened to:

```lean
example
    {p : ℕ} [Fact p.Prime]
    [MazurProof.PrimeTail.PrimeTailLevelData p]
    [pkg : MazurProof.PrimeTail.PrimeTailPackage p]
    (hp : 19 < p)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ MazurProof.HasRationalPointOfOrder E p := by
  exact MazurProof.PrimeTail.no_order_p_of_primeTailPackage
    pkg hp E
```

# 2. What the proof is actually doing

Given an exact rational point of order `p`, the generic level layer constructs a noncuspidal rational point

```text
x ∈ X₀(p)(Q)
```

corresponding to the cyclic subgroup it generates. After possibly applying the Atkin–Lehner involution `w_p`, the local reduction theorem gives

```text
red₃(x) = ∞.
```

Let

```text
f_p : X₀(p) → A_p,
       x ↦ π([x] - [∞])
```

be the normalized winding-quotient map. Reduction compatibility gives

```text
red₃(f_p(x)) = 0.
```

The package field `Finite A_p(Q)` makes `f_p(x)` torsion. Since `p ≠ 3`, `A_p` has good reduction at `3`; the formal kernel over the unramified local field `Q₃` has no nonzero torsion. Hence

```text
f_p(x) = f_p(∞) = 0.
```

The second package field says that the winding element has nonzero image in

```text
(T · e) / 3(T · e).
```

The q-expansion/modular-symbol identification of the cotangent map converts this into formal immersion of `f_p` at the cusp in characteristic `3`. A formal immersion admits at most one `Z₃`-section above the cusp with a given image. Thus `x=∞`, contradicting noncuspidality.

That is the whole derivation. The two arithmetic package fields are used at visibly separate lines.

# 3. Classification of the obligations

## 3.1 Deep citable theorem: finiteness of the winding quotient

```lean
windingQuotientFinite : Finite L.AQ
```

This is the right formal endpoint. Do **not** package it as

```lean
AddGroup.FG L.AQ ∧ rank L.AQ = 0
```

because the formal-immersion proof only needs every rational point of `A_p` to be torsion, and `Finite` gives exactly that without exposing a Mordell–Weil decomposition.

Mathematically, the winding quotient is built from weight-two newform factors with nonzero central `L`-value. Finiteness of its rational Mordell–Weil group is the standard analytic-rank-zero input obtained from the modular analytic-rank-zero theorems of Kolyvagin–Logachev, or from Kato's Euler-system theorem in an appropriate formulation, together with the standard construction of the winding quotient. Merel and Parent use this kind of quotient in the uniform torsion argument.

For Lean, cite and import the exact theorem directly as

```lean
Finite (WindingQuotient p).QPoint
```

rather than importing a vague sentence saying “the winding quotient has rank zero.”

## 3.2 Finite certificate: winding element modulo `3`

```lean
windingClassNeZeroModThree :
  ¬ ∃ t : L.Te, 3 • t = L.windingElement
```

This is the finite modular-symbol/Hecke-lattice computation. Once a basis for the finite-rank lattice `T·e` has been fixed, it is a row-reduction or Smith-normal-form calculation over `ZMod 3`.

The normalization matters. `L.Te` must be the integral lattice that is dual to the cotangent lattice of the chosen quotient `A_p`; using an arbitrary unsaturated copy of the rational Hecke orbit can make “not divisible by three” the wrong statement. The generic theorem

```lean
formalImmersion_of_windingClass
```

is where that lattice-identification theorem is recorded.

A generated certificate file should ideally expose:

```text
basis of T·e;
coordinates of e in that basis;
a coordinate that is nonzero modulo 3;
proof that the basis identifies the intended integral lattice.
```

Only the last line is theoretical; the nonzero-vector check itself is finite and decidable.

## 3.3 Generic reusable theory

These are fields of `PrimeTailLevelData`, not fields of the deep package:

- exact-order `p` point → noncuspidal `X₀(p)(Q)` point;
- Atkin–Lehner orientation and reduction to `∞` modulo `3` for `p>19`;
- extension of rational points to the proper integral model;
- compatibility of the winding map with reduction;
- good reduction of `A_p` at `3` and torsion-freeness of the formal kernel;
- nonzero winding differential → formal immersion;
- formal immersion → uniqueness of a lift of the cusp.

The last implication is completely general. On completed local rings it is the elementary statement that, if

```text
Ô_{A,0} → Ô_{X,∞}
```

is surjective, then two maps out of `Ô_{X,∞}` agreeing after precomposition with `Ô_{A,0}` agree everywhere.

Pinned Mathlib has

```text
Mathlib.AlgebraicGeometry.Morphisms.FormallyUnramified
```

and the local-ring/unramified infrastructure, but no ready-made pointwise declaration named `FormalImmersionAt`. The abstract predicate in `PrimeTailLevelData` is therefore a deliberate temporary seam, not an attempt to identify formal immersion with global `FormallyUnramified`—those are different notions and directions.

# 4. Why this is a better assumption state

It is strictly better **as a formalization interface and audit state**:

- the old axiom directly asserts the desired prime-torsion conclusion;
- the new package has two fields with standard mathematical names and independent references;
- the finite field can be replaced by an explicit certificate without touching the proof;
- the winding-finiteness field can later be discharged by a theorem about modular abelian varieties;
- all local geometry and the contradiction are checked once in reusable Lean code;
- failures are localized: one can see whether the missing item is analytic rank zero, the mod-`3` modular-symbol calculation, or generic formal-immersion infrastructure.

It is **not yet axiom-free** merely because it has a better name. If one assumes

```lean
∀ p, p.Prime → 19 < p → PrimeTailPackage p
```

without constructing those packages, one still assumes an infinite family of deep statements. Nor is the package automatically logically weaker than the old tail theorem when assumed for every prime. The improvement is that the assumptions are minimal, citable, independently replaceable, and do not contain the target conclusion.

For the complete `|E(Q)_tors| ≤ 16` build, the recommended assembly is:

```text
p = 11        fixed X₁(11) calculation;
p = 13        fixed genus-two X₁(13) calculation;
p = 17, 19    fixed X₀(p) plus X₁-fiber calculations;
p > 19        `PrimeTailPackage p` + the theorem above.
```

# 5. References to attach to the package fields

- B. Mazur, *Rational isogenies of prime degree*, Invent. Math. **44** (1978), 129–162. This is the classical degree-one formal-immersion/Eisenstein-quotient setting.
- S. Kamienny, *Torsion points on elliptic curves and q-coefficients of modular forms*, Invent. Math. **109** (1992), 221–229. This gives the formal-immersion/modular-symbol criterion in the small-degree program.
- P. Parent, *Bornes effectives pour la torsion des courbes elliptiques sur les corps de nombres*, arXiv:alg-geom/9611022. This supplies a detailed effective Mazur–Kamienny–Merel formal-immersion framework.
- V. Kolyvagin and D. Logachev, the analytic-rank-zero theorem for modular abelian varieties; or K. Kato, *p-adic Hodge theory and values of zeta functions of modular forms*, Astérisque **295** (2004), for the theorem used to discharge finiteness of the relevant modular quotient.

The repository should cite the exact chosen finiteness theorem in the constructor of each `PrimeTailPackage p`; the generic assembly file should not commit to one analytic proof.
