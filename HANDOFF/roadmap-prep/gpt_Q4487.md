```javascript
    (trap : (ℝ → ℝ) → Prop)
    (Tmap : (ℝ → ℝ) → ℝ → ℝ) where
  dim : ℕ → ℕ
  proj : ∀ N, (ℝ → ℝ) → Fin (dim N) → ℝ
  lift : ∀ N, (Fin (dim N) → ℝ) → ℝ → ℝ
  eps : ℕ → ℝ
  localError : ℕ → ℝ → ℝ

  proj_trap :
    ∀ N u, trap u → proj N u ∈ Freudenthal.unitCube (dim N)
  maps :

      (fun a => proj N (Tmap (lift N a)))
      (Freudenthal.unitCube (dim N))
      (Freudenthal.unitCube (dim N))
  cont :
    ∀ N, ContinuousOn
      (fun a => proj N (Tmap (lift N a)))
      (Freudenthal.unitCube (dim N))
  lift_trap :
    ∀ N a, a ∈ Freudenthal.unitCube (dim N) → trap (lift N a)
  localError_nonneg : ∀ N R, 0 ≤ localError N R
  localError_tendsto :

  residual_le :
    ∀ N a, a ∈ Freudenthal.unitCube (dim N) →
      ‖proj N (Tmap (lift N a)) - a‖ ≤ eps N →
```

The record says exactly this:

1. encode a source profile by finitely many cube coordinates (proj);

1. reconstruct a source profile from cube coordinates (lift);

1. obtain a continuous finite-dimensional self-map

1. turn an eps N-approximate fixed point of that cube map into a local-uniform approximate fixed point of the original source map;

1. make the reconstruction error tend to zero on every fixed compact interval.

simply defines


# 2. The concrete source-box mesh already in the repository

The source-space discretization is in WavePaperRotheProducer.lean. It is not the outer monotone-wave cube reused literally; it is a separate weighted-source-box cube with a clipped McShane reconstruction.

The dimensions and mesh are:

```javascript
def sourceCubeSampleDim (N : ℕ) : ℕ :=
  2 * (N + 1) * (N + 1) + 1

def sourceCubeDim (N : ℕ) : ℕ :=
  sourceCubeSampleDim N + 1

def sourceCubeRadius (N : ℕ) : ℝ :=
  (N + 1 : ℝ)

def sourceCubeMesh (N : ℕ) : ℝ :=
  ((N + 1 : ℝ))⁻¹

def sourceCubeNode
    (N : ℕ) (i : Fin (sourceCubeSampleDim N)) : ℝ :=
  -sourceCubeRadius N + (i : ℕ) * sourceCubeMesh N

def sourceCubeEps (β : ℝ) (N : ℕ) : ℝ :=
  sourceCubeMesh N ^ β
```

There is one extra coordinate beyond the samples. It stores the left-tail limit. The other coordinates encode sampled values on the expanding interval

```plain text
[-(N+1), N+1]
```

at spacing (N+1)⁻¹.

The concrete objects are already defined:

```javascript

```

Conceptually:

```plain text

  the weighted upper/lower source obstacles
```

# 5. Brouwer/Schauder mechanism

Yes, this is the discretization that reduces the infinite-dimensional Schauder problem to finite-dimensional cube problems. The exact bridge is:

```javascript
theorem localUniformApproxFixedPointSequence_of_cubeApproxData
    (D : LocalUniformCubeApproxData trap Tmap) :
    ∃ seq : ℕ → ℝ → ℝ,
      (∀ N, trap (seq N)) ∧
        LocallyUniformApproxFixed Tmap seq
```

For each N, it chooses a N from:

```javascript
Freudenthal.brouwer_cube_approx
  (D.maps N) (D.cont N) (D.eps N) (D.eps_pos N)
```

and sets

```javascript
seq N := D.lift N (a N).
```

The residual field converts

```javascript
‖D.Tfin N (a N) - a N‖ ≤ D.eps N
```

into local-uniform approximate fixedness of seq N.

Finally:

```javascript
theorem localUniformFixedPoint_of_cubeApproxData
    (hcont : LocalUniformContinuousOn trap Tmap)
    (hcompact : LocalUniformSequentiallyCompactRange trap Tmap)
    (D : LocalUniformCubeApproxData trap Tmap) :
    ∃ U, trap U ∧ Tmap U = U
```

uses compactness to extract a locally uniformly convergent subsequence and closedness/continuity to identify the limit as a fixed point.

## “Constructive” clarification

The construction is constructive in the architectural sense:

```plain text
explicit finite mesh
→ explicit finite cube map
→ finite-dimensional Brouwer approximate fixed point
→ explicit reconstructed profile
→ compactness limit.
```

It is not computationally constructive in Lean’s logic. The definitions are noncomputable, and the proof uses Classical.choose, finite argmin witnesses, compact subsequences, and classical real analysis. This is normal and axiom-clean in Mathlib; it should result only in the standard classical axioms, not sorryAx.
