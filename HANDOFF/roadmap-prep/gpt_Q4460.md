# Adversarial audit: Vélu 2-isogeny formulas

## Executive verdict

Let

```text
E : y^2 + a1*x*y + a3*y = x^3 + a2*x^2 + a4*x + a6
```

over a field of characteristic different from `2` (in particular over `ℚ`), and let

```text
Q = (x0,y0) ∈ E(K),
2*y0 + a1*x0 + a3 = 0.
```

Thus `Q = -Q`; since `Q` is affine on an elliptic curve, it has exact order `2`.

Define

```text
b2 := a1^2 + 4*a2,
t  := 3*x0^2 + 2*a2*x0 + a4 - a1*y0,
w  := x0*t.
```

Then the claimed **codomain coefficients are correct**:

```text
E' = E/<Q> :
  Y^2 + a1*X*Y + a3*Y
    = X^3 + a2*X^2 + (a4 - 5*t)*X
      + (a6 - b2*t - 7*w).
```

In particular:

```text
a1' = a1,
a2' = a2,
a3' = a3,
a4' = a4 - 5*t,
a6' = a6 - (a1^2 + 4*a2)*t - 7*x0*t.
```

The constants `-5`, `-(a1^2+4*a2)`, and `-7` are exactly the standard Vélu constants.

However, the proposed rational map

```text
X = x + t/(x-x0) + w/(x-x0)^2
```

is **wrong**. For a kernel generated by a 2-torsion point, the exact normalized Vélu map is

```text
X = x + t/(x-x0),
Y = y - t*(a1*(x-x0) + y-y0)/(x-x0)^2.
```

There is no `w/(x-x0)^2` term. The parameter `w=x0*t` enters the coefficient `a6'`; it is not the coefficient of a second-order pole in the `x`-map.

Equivalently, using the 2-torsion relation,

```text
Y = y - t*(2*y + 2*a1*x - a1*x0 + a3)
          /(2*(x-x0)^2).
```

This is the formula that should be formalized.

# 1. Cross-check against the standard Vélu formulas

For a general kernel point `Q=(xQ,yQ)`, standard Vélu notation uses

```text
gxQ := 3*xQ^2 + 2*a2*xQ + a4 - a1*yQ,
gyQ := -2*yQ - a1*xQ - a3,
uQ  := gyQ^2.
```

The coefficient called `vQ` is defined by cases:

```text
vQ = gxQ                         if Q is 2-torsion,
vQ = 2*gxQ - a1*gyQ             otherwise.
```

Then

```text
v = Σ vQ,
w = Σ (uQ + xQ*vQ),
```

where the sum is over kernel points modulo sign in the standard implementation. The codomain is

```text
[a1, a2, a3, a4 - 5*v,
 a6 - (a1^2 + 4*a2)*v - 7*w].
```

For the subgroup `{O,Q}` with `Q` of order `2`, one has

```text
gyQ = 0,
uQ  = 0,
vQ  = gxQ = t,
w   = x0*t.
```

Therefore the exact quotient coefficients are precisely

```text
[a1, a2, a3, a4-5*t, a6-b2*t-7*w].
```

This is also exactly what SageMath implements in the functions

```text
compute_codomain_formula
compute_vw_kohel_even_deg1
__update_kernel_data
```

in `sage/schemes/elliptic_curves/ell_curve_isogeny.py`.

## Important factor-of-two trap

For a non-2-torsion pair `{Q,-Q}`, the contribution is

```text
2*gxQ - a1*gyQ.
```

For a self-opposite 2-torsion point, it is only

```text
gxQ.
```

Thus the present definition of `t` is correct as written. Replacing it by `2*t` would make both `a4'` and `a6'` wrong.

# 2. Exact rational-function isogeny

Put

```text
d := x - x0.
```

The normalized Vélu map away from `Q` is

```text
phi_x(x,y) = x + t/d,

phi_y(x,y) = y - t*(a1*d + y-y0)/d^2.
```

Thus

```text
phi(x,y)
  = (x + t/(x-x0),
     y - t*(a1*(x-x0)+y-y0)/(x-x0)^2).
```

A denominator-cleared presentation is

```text
Xnum := x*(x-x0) + t,
Xden := x-x0,

Ynum := y*(x-x0)^2 - t*(a1*(x-x0)+y-y0),
Yden := (x-x0)^2.
```

so that

```text
X = Xnum/Xden,
Y = Ynum/Yden.
```

Using

```text
2*y0 + a1*x0 + a3 = 0,
```

the `Y`-formula may also be written

```text
Y = y - t*(2*y + 2*a1*x - a1*x0 + a3)
          /(2*(x-x0)^2).
```

The first version is better for Lean because it avoids introducing `1/2` into the principal map identity.

## Why no `w/(x-x0)^2` occurs

The general Vélu `x`-correction for a kernel representative is

```text
vQ/(x-xQ) + uQ/(x-xQ)^2.
```

For a 2-torsion point,

```text
uQ = (-2*y0-a1*x0-a3)^2 = 0.
```

Hence

```text
X = x + t/(x-x0).
```

The term `w=x0*t` is a sum used only in the codomain coefficient formula.

A minimal sanity check is

```text
E : y^2 = x^3 + A*x^2 + B*x,
Q = (0,0).
```

Here `t=B`, `w=0`, and the formulas give

```text
E' : Y^2 = X^3 + A*X^2 - 4*B*X - 4*A*B,
X = x + B/x,
Y = y*(1-B/x^2).
```

This is the classical normalized 2-isogeny. Adding `w/x^2` would be invisible in this special case because `w=0`, which is why a translated test with `x0 ≠ 0` is needed to catch the error.

# 3. Ring-checkable centered form

The cleanest proof of every formula is to translate the 2-torsion point to the origin.

Set

```text
u := x-x0,
v := y-y0,
A := a2 + 3*x0,
B := a1^2 + 4*A = b2 + 12*x0.
```

Using the point equation for `Q` and

```text
2*y0 + a1*x0 + a3 = 0,
```

the original curve becomes

```text
E0 : v^2 + a1*u*v = u^3 + A*u^2 + t*u.
```

The quotient becomes

```text
E0' : V^2 + a1*U*V
        = U^3 + A*U^2 - 4*t*U - B*t.
```

The map is

```text
U = u + t/u,
V = v - t*(v+a1*u)/u^2.
```

Returning by

```text
X = U+x0,
Y = V+y0
```

gives exactly the formulas in Section 2 and the coefficients

```text
a4' = a4-5*t,
a6' = a6-b2*t-7*x0*t.
```

This centered form is the recommended Lean proof architecture:

```lean
-- first prove by ring_nf that translating Q gives E0
-- then verify the map E0 -> E0' by field_simp; ring_nf
-- finally translate back and ring_nf the five coefficients
```

No appeal to a database is needed.

# 4. Discriminant certificate

Use the standard generalized-Weierstrass discriminant `Δ`.

For

```text
E0 : v^2+a1*u*v = u^3+A*u^2+t*u
```

one has

```text
b2(E0) = B,
b4(E0) = 2*t,
b6(E0) = 0,
b8(E0) = -t^2,
```

and therefore

```text
Δ(E) = Δ(E0) = t^2*(B^2-64*t).
```

For

```text
E0' : V^2+a1*U*V = U^3+A*U^2-4*t*U-B*t
```

one has

```text
b2(E0') = B,
b4(E0') = -8*t,
b6(E0') = -4*B*t,
b8(E0') = -B^2*t-16*t^2,
```

hence

```text
Δ(E') = Δ(E0') = t*(B^2-64*t)^2.
```

The strongest denominator-free relation is

```text
t^3 * Δ(E') = Δ(E)^2.
```

This is a pure polynomial identity in

```text
a1,a2,a3,a4,a6,x0,y0
```

under the two relations

```text
Q ∈ E,
2*y0+a1*x0+a3=0.
```

It is ideal for Lean:

```lean
theorem velu2_discriminant_relation :
    t ^ 3 * Eprime.Δ = E.Δ ^ 2 := by
  -- translate to the centered model
  -- simp [WeierstrassCurve.Δ, WeierstrassCurve.b₂,
  --       WeierstrassCurve.b₄, WeierstrassCurve.b₆,
  --       WeierstrassCurve.b₈]
  -- ring_nf
  sorry
```

Then ellipticity of `E'` follows immediately. If `E'.Δ=0`, the relation gives `E.Δ^2=0`, contradicting `E.IsElliptic`.

One may also record

```text
Δ(E') = Δ(E)^2/t^3,
```

but the multiplied identity is better because it requires no prior proof that `t ≠ 0`.

For completeness, ellipticity of `E` also implies

```text
t ≠ 0,
B^2-64*t ≠ 0,
```

from `Δ(E)=t^2*(B^2-64*t)`.

# 5. Kernel

The map extends to a morphism of smooth projective curves by setting

```text
phi(O) = O,
phi(Q) = O.
```

Its kernel is exactly

```text
ker(phi) = {O,Q}.
```

There are several rigorous proofs.

## Direct pole proof

The function

```text
X = x + t/(x-x0)
```

has poles only at:

1. `O`, because `x` has its usual order-2 pole there;
2. `Q`, because `x-x0` vanishes there.

Thus the only points mapping to the point at infinity on `E'` are `O` and `Q`.

If an affine point `P` has `x(P)=x0`, then the two possible points above `x0` are `Q` and `-Q`; but `Q=-Q`, so `P=Q`.

## Degree proof

The `x`-coordinates satisfy

```text
U = u+t/u,
```

so `u` satisfies

```text
u^2-U*u+t=0.
```

Hence the induced function-field extension has degree `2`. In characteristic zero the isogeny is separable, so its geometric kernel has two elements. Since `O` and `Q` are in the kernel, they are the entire kernel.

No additional rational kernel point can occur.

# 6. Correct Lean-facing statement

A safe project-local theorem should look schematically like this:

```lean
variable {K : Type*} [Field K] [CharZero K]

structure RationalTwoTorsion (E : WeierstrassCurve K) where
  x0 y0 : K
  onCurve : E.Equation x0 y0
  twoY : 2*y0 + E.a₁*x0 + E.a₃ = 0

namespace RationalTwoTorsion

variable (Q : RationalTwoTorsion E)

noncomputable def t : K :=
  3*Q.x0^2 + 2*E.a₂*Q.x0 + E.a₄ - E.a₁*Q.y0

noncomputable def w : K := Q.x0 * Q.t

noncomputable def quotient : WeierstrassCurve K :=
  { a₁ := E.a₁
    a₂ := E.a₂
    a₃ := E.a₃
    a₄ := E.a₄ - 5*Q.t
    a₆ := E.a₆ - E.b₂*Q.t - 7*Q.w }

noncomputable def mapX (x : K) : K :=
  x + Q.t/(x-Q.x0)

noncomputable def mapY (x y : K) : K :=
  y - Q.t*(E.a₁*(x-Q.x0) + y-Q.y0)/(x-Q.x0)^2

 theorem map_equation
    {x y : K} (hE : E.Equation x y) (hx : x ≠ Q.x0) :
    Q.quotient.Equation (Q.mapX x) (Q.mapY x y) := by
  field_simp [mapX, mapY, quotient, t, w]
  -- use hE, Q.onCurve, Q.twoY
  ring_nf at *
  sorry

 theorem discriminant_relation :
    Q.t^3 * Q.quotient.Δ = E.Δ^2 := by
  -- pure ring identity after Q.onCurve and Q.twoY
  sorry

 theorem quotient_isElliptic [E.IsElliptic] : Q.quotient.IsElliptic := by
  rw [WeierstrassCurve.isElliptic_iff]
  intro hΔ
  have := Q.discriminant_relation
  rw [hΔ, mul_zero] at this
  exact E.isUnit_Δ.ne_zero (sq_eq_zero_iff.mp this)

-- The projective extension is the degree-2 normalized Vélu isogeny.
-- Its geometric kernel is exactly {O,Q}.
```

The exact last proof syntax will depend on the landed `IsElliptic` API, but the formulas above are the ones to freeze.

# 7. Final answers to the four questions

1. **Codomain coefficients:** yes, exactly
   ```text
   [a1,a2,a3,a4-5*t,a6-(a1^2+4*a2)*t-7*w]
   ```
   with `t=3*x0^2+2*a2*x0+a4-a1*y0` and `w=x0*t`. The `-7*w` is correct.

2. **Map:** the claimed `+w/(x-x0)^2` is incorrect. The exact map is
   ```text
   X=x+t/(x-x0),
   Y=y-t*(a1*(x-x0)+y-y0)/(x-x0)^2.
   ```

3. **Discriminant:** with `B=b2+12*x0`,
   ```text
   Δ(E)=t^2*(B^2-64*t),
   Δ(E')=t*(B^2-64*t)^2,
   t^3*Δ(E')=Δ(E)^2.
   ```
   Therefore `E'.IsElliptic` follows unconditionally from `E.IsElliptic`.

4. **Kernel:** in characteristic zero,
   ```text
   ker(phi)={O,Q}.
   ```
   The map has degree `2`, is separable, and its only poles/preimages of `O` are `O` and `Q`.
