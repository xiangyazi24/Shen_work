# Definitive verdict

Yes. The exact ConjugateMildSolutionData.hpos obligation is

```javascript
hpos : ∀ t, 0 < t → t ≤ T → ∀ x, 0 < u t x
```

where Lean infers

```javascript
x : intervalDomainPoint
```

and intervalDomainPoint is the subtype of the closed interval Set.Icc (0 : ℝ) 1. Thus the field is:

- pointwise, not almost everywhere;

- valid for every positive time through the terminal time T (t ≤ T, not merely t < T);

- valid at every closed-domain point, including x = 0 and x = 1.

The forward V6 adapter

```javascript
strictPos_of_truncatedJensenStrictPosDataFor
```

has exactly the logical output needed by this field. A TruncatedJensenStrictPosDataFor T u can therefore be inserted into the record with no time-endpoint, spatial-endpoint, or a.e.-to-pointwise conversion.

# 3. How the Jensen data yields strict positivity

Unfolded at a fixed

```javascript
t : ℝ
ht : 0 < t
htT : t ≤ T
x : intervalDomainPoint
```

TruncatedJensenStrictPosDataFor T u supplies witnesses of the following form:

```javascript
D s σ : ℝ
f : ℝ → ℝ
hσ : 0 < σ
htime : s + σ = t
hjensen : FullKernelJensenInequality f
hseed_after_heat :
  intervalFullSemigroupOperator σ (fun y => f y ^ 2) x.1 ≤
    intervalFullSemigroupOperator σ
      (fun y => u s (unitClip y)) x.1
hdiscount :
  Real.exp (-D * σ) *
      intervalFullSemigroupOperator σ
        (fun y => u s (unitClip y)) x.1 ≤
    u t x
hS_pos :
  0 < intervalFullSemigroupOperator σ f x.1
```

The forward theorem performs this monotone chain:

```plain text
0 < S(σ)f(x)
  ⇒ 0 < (S(σ)f(x))²
  ≤ S(σ)(f²)(x)                       by Jensen
  ≤ S(σ)(u(s,·))(x)                  by the seed comparison
  ⇒ 0 < exp(-Dσ) S(σ)(u(s,·))(x)     because exp(-Dσ) > 0
  ≤ u(t,x)                            by the discounted lower comparison.
```

Hence:

```javascript
0 < u t x
```

for the exact t, including t = T, and the exact subtype point x, including either boundary endpoint.

A proof-shaped expansion is:

```javascript
intro t ht htT x
rcases H /* pointwise field */ t ht htT x with
  ⟨D, s, σ, f, hσ, htime, hjensen,
    hseed_after_heat, hdiscount, hS_pos⟩

have hsq_pos :
    0 < (intervalFullSemigroupOperator σ f x.1) ^ 2 :=
  sq_pos_of_pos hS_pos

have hSf2_pos :
    0 < intervalFullSemigroupOperator σ (fun y => f y ^ 2) x.1 :=
  lt_of_lt_of_le hsq_pos (hjensen /* σ, x */)

have hSrestart_pos :
    0 < intervalFullSemigroupOperator σ
      (fun y => u s (unitClip y)) x.1 :=
  lt_of_lt_of_le hSf2_pos hseed_after_heat

have hdiscounted_pos :
    0 < Real.exp (-D * σ) *
      intervalFullSemigroupOperator σ
        (fun y => u s (unitClip y)) x.1 :=
  mul_pos (Real.exp_pos _) hSrestart_pos

exact lt_of_lt_of_le hdiscounted_pos hdiscount
```

The comment /* pointwise field */ and the argument display of hjensen depend on how those two structures are projected, but the inequalities and the final proposition are the exact consumer logic.

# 4. Do not confuse the forward and reverse adapters

The repository also has:

```javascript
theorem truncatedJensenStrictPosDataFor_of_strictPos
    {T R : ℝ} {u₀ : intervalDomainPoint → ℝ}
    {u : ℝ → intervalDomainPoint → ℝ}
    ...
    (hpos :
      ∀ t, 0 < t → t ≤ T →
        ∀ x : intervalDomainPoint, 0 < u t x) :
    TruncatedJensenStrictPosDataFor T u
```

This goes in the opposite direction:

```plain text
already-known hpos → Jensen witness package.
```

It cannot be used to prove ConjugateMildSolutionData.hpos without circularity. The unconditional chain must be:

```plain text
energy/nonnegativity
  → matched-divergence barrier comparison
  → TruncatedJensenStrictPosDataFor T u
  → strictPos_of_truncatedJensenStrictPosDataFor
  → ConjugateMildSolutionData.hpos.
```

The reverse helper is useful only after an independent theorem has already established the same pointwise strict positivity.

Reject any candidate producer whose output is only one of the following:

```javascript

∀ x ∈ Set.Ioo (0 : ℝ) 1, 0 < ...         -- spatial endpoints missing
∀ᵐ x, 0 < ...                              -- only almost everywhere
∀ K ⊆ Set.Ioo (0 : ℝ) 1, ...              -- only compact interior sets
0 ≤ u t x                                  -- nonnegative, not strict
```

Strict positivity on 0 < t < T plus continuity does not automatically imply strict positivity at t = T, so the ≤ T requirement is load-bearing. Likewise, interior positivity alone does not fill a field quantified over intervalDomainPoint.

# Final conclusion

The exact Jensen output required by the record is:

```javascript
∀ t, 0 < t → t ≤ T →
  ∀ x : intervalDomainPoint, 0 < u t x
```

It is terminal-time inclusive and closed-spatial-domain inclusive. strictPos_of_truncatedJensenStrictPosDataFor HJ has exactly this result and can be assigned directly to ConjugateMildSolutionData.hpos. Codex should therefore prove the Jensen witness package for every t ∈ (0,T] and every x : intervalDomainPoint; no weaker interior/open-time/a.e. statement is sufficient.
