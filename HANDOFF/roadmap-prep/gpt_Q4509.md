# `A4Package` for the order-25 seam

## Executive verdict

The sharp deep input is

```text
A_4(Q) is finite.
```

Here `A_4` is the four-dimensional Q-simple modular abelian variety identified with the Jacobian of the genus-four diamond quotient

```text
q_H : X_1(25) -> X_H,
H={1,7,18,24} subset (Z/25Z)^*.
```

Since `H/{±1}` has order two, `q_H` has degree two. The quotient has genus four, and its Jacobian is Q-isogenous to `A_4`.

The package must contain more than the finiteness theorem. Finiteness of `A_4(Q)` alone does **not** imply that every rational point of `X_H` is a cusp. The remaining necessary inputs are:

1. the explicit diamond quotient and its cusp fibers;
2. a reduction theorem placing the image of an order-25 point in the residue disk of a known cusp;
3. formal immersion of the Abel-Jacobi/modular-quotient map at that cusp;
4. a finite-group annihilator prime to the residue characteristic.

Only item `A_4(Q) finite` is the Kato-type citable assumption. Items 1, 2, and 4 are fixed finite geometry/arithmetic. Item 3 is an instance of a reusable formal-immersion theorem.

The logical chain is

```text
ExplicitOrder25Obstruction
 -> noncuspidal P in X_1(25)(Q)
 -> Q=q_H(P) in X_H(Q)
 -> Q and a fixed cusp c have the same reduction
 -> m*(AJ_A4(Q)-AJ_A4(c))=0 because m kills A_4(Q)
 -> formal immersion, with gcd(m,l)=1, gives Q=c
 -> the degree-two fiber above c consists only of cusps
 -> P is a cusp, contradiction.
```

# 1. Verified fixed geometry

Use

```text
H25={1,7,18,24}={±1,±7}.
```

Because

```text
7^2 = 49 = -1 mod 25,
```

the additional diamond operator has order two modulo `±1`. Thus

```text
X_1(25)=X_{±1}(25) -> X_H(25)
```

is a degree-two quotient.

The relevant genera are

```text
g(X_1(25))=12,
g(X_H)=4.
```

A modular-symbol computation gives a four-dimensional space of regular differentials on `X_H`. One explicit canonical model in `P^3` has quadric

```text
-z1^2 + z0*z2 + z1*z3 - 2*z2*z3 + z3^2 = 0
```

and cubic

```text
-4*z0*z1^2 + 5*z1^3 + 4*z0^2*z2 - 9*z0*z1*z2
+2*z1^2*z2 + 2*z0*z2^2 + 4*z0^2*z3 + 4*z0*z1*z3
-6*z1^2*z3 - 11*z0*z2*z3 + 8*z1*z2*z3 + z3^3 = 0.
```

The dimension-four Jacobian is Q-simple; call it `A4`. For the Lean endgame, an explicit pair of isogeny maps is preferable to a bare proposition `Jac(X_H) ~ A4`, but the contradiction only consumes the induced map

```text
AJ_A4 : X_H(Q) -> A4(Q)
```

normalized to send the chosen cusp to zero.

# 2. The one reusable Kato-type assumption

Do not make `analyticRank A4 = 0` the field consumed by the arithmetic proof. Analytic rank zero implies the needed algebraic finiteness only through the modular-abelian-variety theorem being cited. The sharp interface is therefore direct finiteness.

```lean
namespace FLT.Mazur.ModularAV

universe u

/-- The theorem-level output imported from Kato for a fixed modular
abelian variety whose central L-value is nonzero. -/
structure FiniteMWInput (A : Type u) [AddCommGroup A] : Prop where
  finite_mw : Finite A

end FLT.Mazur.ModularAV
```

If the project later develops modular L-functions, use one generic axiom/theorem schema:

```lean
/-- Citable analytic-to-algebraic input.  The precise hypotheses should say
that `A` is the newform quotient attached to a weight-two modular form and
that its central L-value is nonzero. -/
axiom modularAV_finite_mw_of_L_one_ne_zero
    (A : ModularAbelianVarietyQ)
    (hL : A.LValueAtOne ≠ 0) :
    Finite A.RationalPoints
```

Then the N25 package uses it with `A=A4`, while the prime-tail package uses it with the relevant winding/Eisenstein quotient `B_p`. This is the exact sense in which N25 and the uniform prime tail share one deep axiom schema.

For the current formalization, the smaller and safer assumption is simply

```lean
FiniteMWInput A4Q
```

rather than importing an incompletely formalized analytic-rank vocabulary.

# 3. A compiling abstract core

The following structure deliberately avoids depending on a nonexistent general modular-curve API. It packages the exact logical interface needed by the contradiction. The geometric metadata—degree two, genus four, and `Jac(X_H)~A4`—is represented by explicit maps and numerical certificates rather than opaque prose.

```lean
import Mathlib.GroupTheory.OrderOfElement
import Mathlib.GroupTheory.Torsion

set_option autoImplicit false

namespace FLT.Mazur.N25

universe u

/-- Abstract fixed-instance package closing the order-25 obstruction.

`Obs` is intended to be the repository's `ExplicitOrder25Obstruction`.
Only `kato.finite_mw` is a deep citable theorem. -/
structure A4Package (Obs : Prop) where
  /- Rational-point types. -/
  X1Q : Type u
  XHQ : Type u
  XHRed : Type u

  /- Rational groups of Jac(X_H) and A4. -/
  JacXHQ : Type u
  A4Q : Type u
  [instAddCommGroupJacXHQ : AddCommGroup JacXHQ]
  [instAddCommGroupA4Q : AddCommGroup A4Q]

  /- Cusps and the Tate-normal-form bridge. -/
  IsCuspX1 : X1Q → Prop
  pointOfObstruction : Obs → X1Q
  pointOfObstruction_nonCusp :
    ∀ h : Obs, ¬ IsCuspX1 (pointOfObstruction h)

  /- The order-two diamond action and its quotient.  These two fiber laws
  are the point-set form of a degree-two quotient. -/
  diamond : X1Q → X1Q
  diamond_involutive : Function.Involutive diamond
  quotient : X1Q → XHQ
  quotient_diamond : ∀ P : X1Q, quotient (diamond P) = quotient P
  quotient_fiber : ∀ P Q : X1Q,
    quotient P = quotient Q ↔ Q = P ∨ Q = diamond P

  /- Numerical geometric certificates. -/
  X1_genus : ℕ
  X1_genus_eq : X1_genus = 12
  XH_genus : ℕ
  XH_genus_eq : XH_genus = 4

  /- Explicit isogeny certificate Jac(X_H) ~ A4.  The nonzero multiplier
  identities imply finite kernels on rational points once the usual torsion
  facts are available. -/
  jacToA4 : JacXHQ →+ A4Q
  a4ToJac : A4Q →+ JacXHQ
  isogenyMultiplier : ℕ
  isogenyMultiplier_pos : 0 < isogenyMultiplier
  jacToA4_a4ToJac : ∀ a : A4Q,
    jacToA4 (a4ToJac a) = isogenyMultiplier • a
  a4ToJac_jacToA4 : ∀ j : JacXHQ,
    a4ToJac (jacToA4 j) = isogenyMultiplier • j

  /- Abel-Jacobi map followed by the A4 quotient. -/
  baseCusp : XHQ
  abelJacobi : XHQ → JacXHQ
  abelJacobi_baseCusp : abelJacobi baseCusp = 0
  abelJacobiA4 : XHQ → A4Q
  abelJacobiA4_eq : ∀ Q : XHQ,
    abelJacobiA4 Q = jacToA4 (abelJacobi Q)

  /- The sole deep citable input. -/
  kato : FLT.Mazur.ModularAV.FiniteMWInput A4Q

  /- Fixed finite-group certificate.  It may be obtained from `kato` by
  taking the exponent of the finite group, then reduced to a convenient
  explicit annihilator by finite reduction computations. -/
  a4Annihilator : ℕ
  a4Annihilator_pos : 0 < a4Annihilator
  a4Annihilator_kills : ∀ a : A4Q, a4Annihilator • a = 0

  /- Reduction/formal-immersion data. -/
  auxPrime : ℕ
  auxPrime_prime : Nat.Prime auxPrime
  annihilator_coprime_auxPrime :
    Nat.Coprime a4Annihilator auxPrime
  redXH : XHQ → XHRed

  /-- Every order-25 obstruction lands in the residue disk of the chosen cusp. -/
  obstruction_reduces_to_baseCusp : ∀ h : Obs,
    redXH (quotient (pointOfObstruction h)) = redXH baseCusp

  /-- Instantiated generic formal-immersion rigidity theorem.

  In production this is proved from:
  * formal immersion of `[a4Annihilator] ∘ abelJacobiA4` at `baseCusp`;
  * `gcd(a4Annihilator, auxPrime)=1`, so multiplication by the
    annihilator has nonzero differential in the residue characteristic;
  * equality of reductions and equality of target points. -/
  formalImmersionRigidity : ∀ Q : XHQ,
    redXH Q = redXH baseCusp →
    a4Annihilator •
        (abelJacobiA4 Q - abelJacobiA4 baseCusp) = 0 →
    Q = baseCusp

  /-- The two points in the degree-two fiber above the selected quotient cusp
  are cusps of X_1(25).  This is a finite explicit cusp calculation. -/
  preimage_baseCusp_is_cusp : ∀ P : X1Q,
    quotient P = baseCusp → IsCuspX1 P

attribute [instance] A4Package.instAddCommGroupJacXHQ
attribute [instance] A4Package.instAddCommGroupA4Q
```

The finiteness field is intentionally present even though the final three-line contradiction consumes the extracted annihilator. This keeps the assumption auditable: the annihilator is finite arithmetic derived from the one citable theorem, not a second analytic assumption.

# 4. The derivation theorem

```lean
/-- The fixed genus-four modular quotient package rules out the explicit
order-25 Tate obstruction. -/
theorem no_explicit_order25_obstruction_of_A4Package
    {ExplicitOrder25Obstruction : Prop}
    (pkg : A4Package ExplicitOrder25Obstruction) :
    ¬ ExplicitOrder25Obstruction := by
  intro h

  let P : pkg.X1Q := pkg.pointOfObstruction h
  let Q : pkg.XHQ := pkg.quotient P

  have hred : pkg.redXH Q = pkg.redXH pkg.baseCusp := by
    simpa [P, Q] using pkg.obstruction_reduces_to_baseCusp h

  have hkill :
      pkg.a4Annihilator •
          (pkg.abelJacobiA4 Q - pkg.abelJacobiA4 pkg.baseCusp) = 0 :=
    pkg.a4Annihilator_kills _

  have hQ : Q = pkg.baseCusp :=
    pkg.formalImmersionRigidity Q hred hkill

  have hPcusp : pkg.IsCuspX1 P :=
    pkg.preimage_baseCusp_is_cusp P (by simpa [Q] using hQ)

  exact pkg.pointOfObstruction_nonCusp h hPcusp

end FLT.Mazur.N25
```

This proof is logically complete once the package is constructed. It does not use Mordell-Weil finite generation as a separate theorem: finiteness of this one modular abelian variety is the imported conclusion.

# 6. Generic formal-immersion theorem to share with the tail

The reusable theorem is not specific to level 25.

```lean
/-- Formal immersion plus a finite target forces a rational point in the cusp
residue disk to equal the cusp. -/
theorem eq_cusp_of_finite_target_formalImmersion
    {X A : Type*}
    [AddCommGroup A]
    [Finite A]
    (ell m : ℕ)
    (hell : Nat.Prime ell)
    (hcop : Nat.Coprime m ell)
    (map : X → A)
    (c P : X)
    (hred : SameReductionAt ell P c)
    (hformal : IsFormalImmersionAt ([m] ∘ map) c ell)
    (hkill : ∀ a : A, m • a = 0) :
    P = c := by
  -- 1. `hkill` gives `[m]map(P)=[m]map(c)=0`.
  -- 2. `hcop` says the differential of `[m]` is invertible mod `ell`.
  -- 3. Therefore `[m]∘map` is a formal immersion at `c`.
  -- 4. The formal-immersion uniqueness lemma for two sections with the same
  --    special fiber and the same image gives `P=c`.
  sorry
```

This is the generic geometric theorem shared by N25 and any prime-tail package. The actual difficult infrastructure is the scheme/DVR statement in step 4, not the final group calculation.

# 8. Field-by-field audit

## 8.1 Citable Kato field

Exactly one field:

```lean
kato : FiniteMWInput A4Q
```

Mathematical provenance:

```text
L(A_4,1) != 0
  + Kato's Euler-system theorem for the weight-two modular quotient
  => A_4(Q) is finite.
```

For trustworthiness, cite the theorem as `A_4(Q) finite`; do not silently replace it by a database entry saying “analytic rank 0.”

## 8.2 Geometric and finite fields

These are axiom-free fixed-instance work:

```text
H={1,7,18,24};
diamond <7> has order 2 modulo ±1;
degree(q_H)=2;
g(X_H)=4;
the explicit quadric/cubic model;
Jac(X_H) ~ A_4;
the Tate-obstruction-to-X_1(25) map;
which quotient cusp the obstruction reduces to;
the two-point cusp fiber above that cusp;
an explicit finite annihilator m of A_4(Q);
gcd(m,ell)=1.
```

The annihilator is not a second deep assumption. Once `A_4(Q)` is finite, a group exponent exists. In practice one should pin a small `m` by reduction at two good primes and check it finitely.

## 8.3 Generic reusable field

The generic theorem is:

```text
formal immersion at a cusp
+ same reduction
+ equal target image
=> equal rational sections.
```

It belongs in shared modular-curve infrastructure and should also be used by `PrimeTailPackage`.

# 11. Important cautions

1. **Do not assume only `analyticRank A4 = 0`.** Either formalize the Kato implication or assume `Finite A4Q` directly.
2. **Do not infer cuspidality from finiteness alone.** The reduction and formal-immersion fields are indispensable.
3. **Do not omit coprimality of the annihilator and the auxiliary residue characteristic.** Otherwise multiplication by the annihilator may destroy formal immersion.
4. **Do not merely state `Jac(X_H) ~ A4`.** Export the actual map used in Abel-Jacobi and the isogeny identities.
5. **Do not assume that the image of a noncusp under a degree-two quotient is automatically a noncusp.** Prove the relevant cusp fiber explicitly; the package uses the safer contrapositive field `preimage_baseCusp_is_cusp`.
6. **Do not conflate X_1(25) with the genus-four quotient.** `X_1(25)` has genus 12; `X_H` has genus 4.
