# Verdict

The weak-H² route is the shortest route to the **specific endpoint** needed by the spectral bootstrap:

```lean
WindowSourceEnvelope 2 c T' (bFormSourceCoeffs p S.u)
```

Once this is available on the restart window, it can feed the pass-2 eigenvalue-weighted summability lemma directly. In that sense, yes: the weak-H² witness plus the quantitative quadratic-decay lemma skips the alternating finite ladder **for the endpoint R**.

But it does **not** automatically inhabit the currently typed generic pass

```lean
SourceFromSolutionEnvelopePass S
```

because that pass is polymorphic in `m`:

```lean
∀ {m : ℕ} {c T' : ℝ}, 0 < c → T' < S.T → 1 ≤ m →
  WindowCoefficientEnvelope m c T' (solutionCoeffs S) →
  WindowSourceEnvelope (m - 1) c T' (bFormSourceCoeffs p S.u)
```

A single weak-H² witness gives quadratic decay, i.e. order `2`. It can be weakened to orders `0`, `1`, and `2`, but it cannot be promoted to arbitrary `m - 1 > 2`. Therefore it is a direct replacement for the **finite ladder endpoint**, not a direct inhabitant of the full generic `SourceFromSolutionEnvelopePass` unless that interface is narrowed or wrapped to the finite orders actually used.

# Exact weak-H² → quadratic decay reduction

The quantitative lemma is in `ShenWork/PDE/IntervalSourceDecayQuantitative.lean`:

```lean
theorem intervalWeakH2Neumann_cosineCoeff_quadratic_decay_of_bound
    {f : ℝ → ℝ} (hf : IntervalWeakH2Neumann f) {B : ℝ}
    (hB : (∫ x in (0:ℝ)..1, |hf.secondDeriv x|) ≤ B) :
    ∀ k : ℕ, 1 ≤ k →
      |cosineCoeffs f k| ≤ 2 * B / ((k : ℝ) * Real.pi) ^ 2
```

The file comment states the point of the lemma: the explicit constant is `2 * B`, where `B` bounds `∫₀¹ |f''|`, because the older existential decay constant was not useful for uniform windows. The proof first bounds the weak Laplacian coefficient by `B`, then uses the weak cosine Laplacian identity to divide by `(kπ)^2`, and finally uses the nonzero-mode normalization `cosineCoeffs f k = 2 * raw` for `k ≠ 0`.

For a source slice

```lean
f_s := coupledChemDivSourceLift p S.u s
```

you need:

```lean
hH2 : ∀ s, s ∈ Icc c T' → IntervalWeakH2Neumann (coupledChemDivSourceLift p S.u s)
hB  : ∃ B, 0 ≤ B ∧ ∀ s, s ∈ Icc c T' →
        (∫ x in (0:ℝ)..1, |(hH2 s hs).secondDeriv x|) ≤ B
```

Then define the quadratic source envelope by

```lean
E k :=
  if hk : k = 0 then C0 else 2 * B / ((k : ℝ) * Real.pi) ^ 2
```

or, if the `WindowSourceEnvelope` structure stores a global constant `C` and a decay inequality rather than an explicit `env`, use

```lean
C := max C0 (2 * B / Real.pi ^ 2)
```

with the actual high-mode proof

```lean
have hdecay_high :
    ∀ s ∈ Icc c T', ∀ k : ℕ, 1 ≤ k →
      |cosineCoeffs (coupledChemDivSourceLift p S.u s) k| ≤
        2 * B / ((k : ℝ) * Real.pi) ^ 2 := by
  intro s hs k hk
  exact intervalWeakH2Neumann_cosineCoeff_quadratic_decay_of_bound
    (hH2 s hs) (hB_bound s hs) k hk
```

and a separate zero-mode bound

```lean
hzero : ∀ s ∈ Icc c T', |cosineCoeffs (coupledChemDivSourceLift p S.u s) 0| ≤ C0
```

The repo already has this shape for the chem-div piece in `IntervalChemDivFluxFACSourceDecay.lean`:

# To get `bFormSourceCoeffs`, not just chem-div

The full B-form source coefficient is the scaled chem-div source plus logistic source. The repo’s coupled-source layer has exactly this decomposition shape:

```lean
def coupledChemDivSourceCoeffs p u : ℝ → ℕ → ℝ :=
  fun s n => cosineCoeffs (coupledChemDivSourceLift p u s) n

def coupledLogisticSourceCoeffs p u : ℝ → ℕ → ℝ :=
  fun s n => cosineCoeffs (coupledLogisticSourceLift p u s) n

def coupledChemicalSourceCoeffs p u : ℝ → ℕ → ℝ :=
  fun s n => cosineCoeffs (coupledChemicalSourceLift p u s) n
```

and it packages source time-C¹ by scaling chem-div and adding logistic:

```lean
coupledChemicalSource_duhamelSourceTimeC1
  (hlog : DuhamelSourceTimeC1 (coupledLogisticSourceCoeffs p u))
  (hchem : DuhamelSourceTimeC1 (coupledChemDivSourceCoeffs p u))
  (hsplit : coupledChemicalSourceCoeffs p u =
    fun s n => -(p.χ₀ * coupledChemDivSourceCoeffs p u s n)
      + coupledLogisticSourceCoeffs p u s n)
```

For a `WindowSourceEnvelope 2` of `bFormSourceCoeffs p S.u`, the analogous coefficient envelope should be built as:

```lean
E_total k := |p.χ₀| * E_chem k + E_log k
```

or, with global constants:

```lean
C_total := |p.χ₀| * C_chem + C_log
```

where:

```lean
|chem_k s| ≤ C_chem / ((k : ℝ) * Real.pi)^2
|log_k  s| ≤ C_log  / ((k : ℝ) * Real.pi)^2
```

Then:

```lean
|-(p.χ₀ * chem_k s) + log_k s|
  ≤ |p.χ₀| * |chem_k s| + |log_k s|
  ≤ (|p.χ₀| * C_chem + C_log) / ((k : ℝ) * Real.pi)^2
```

The zero mode is similarly:

```lean
|-(p.χ₀ * chem_0 s) + log_0 s|
  ≤ |p.χ₀| * Cchem0 + Clog0
```

This is the shortest direct source envelope for the **full** source. Do not feed only chem-div into `bFormSourceCoeffs`: the logistic branch still must be included or separately shown to be part of the source split.

# Can `SourceFromSolutionEnvelopePass` be inhabited directly from this?

Not as the current fully polymorphic interface is written.

The current interface in `IntervalMildPositiveTimeRegularityV6.lean` is:

```lean
def SourceFromSolutionEnvelopePass
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (S : ConjugateMildSolutionData p u₀) : Type :=
  ∀ {m : ℕ} {c T' : ℝ}, 0 < c → T' < S.T → 1 ≤ m →
    WindowCoefficientEnvelope m c T'
      (fun σ n => cosineCoeffs (intervalDomainLift (S.u σ)) n) →
    WindowSourceEnvelope (m - 1) c T'
      (bFormSourceCoeffs p S.u)
```

A direct weak-H² proof gives order `2` only. It can close the cases:

```lean
m = 1  -- need WSE 0, weaken from WSE 2
m = 2  -- need WSE 1, weaken from WSE 2
m = 3  -- need WSE 2, exact
```

but it does not close:

```lean
m ≥ 4  -- need WSE 3 or higher; WSE 2 is not enough
```

unless there is an additional theorem saying `WindowSourceEnvelope 2` is accepted as every higher order, which would be mathematically false for polynomial decay envelopes. So the direct weak-H² route is not a legitimate inhabitant of that exact `∀ m` pass.

# Bottom line

Use the weak-H² witness to build `WindowSourceEnvelope 2` directly and feed `restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope`. That is the shortest R closure.

Do **not** claim this single proof inhabits the current `SourceFromSolutionEnvelopePass S` unless you either:

1. narrow the pass interface to the finite endpoint actually needed, or
2. prove/order-weaken only the finitely used cases, or
3. retain the original source-from-solution machinery for arbitrary `m`.
