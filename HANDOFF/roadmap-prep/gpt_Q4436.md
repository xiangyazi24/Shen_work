# Verdict

The current repository’s Paper 3 “P3.1” global-existence floor is not a semiflow theorem. The exact public target is

```javascript
ShenWork.Paper3.Proposition_1_2 intervalDomain p
```

and it says only:

```plain text
χ₀ ≤ 0 and 1 ≤ m
→ for every paper-positive initial datum
```

There is currently no repository declaration named exactly P3_1, P3_1_GlobalExistence, or paper3_P3_1. Those are roadmap labels. The closest exact Lean target is Proposition_1_2 intervalDomain p.

## What its three conclusion fields mean

### A. Global classical solution

```javascript
IsPaper2GlobalClassicalSolution intervalDomain p u v
```

is

```javascript
∀ T > 0, IsPaper2ClassicalSolution intervalDomain p T u v
```

and each finite-horizon IsPaper2ClassicalSolution already packages:

```plain text
classical regularity,
strict positivity of u at positive times,
nonnegativity of v,
the u PDE,
the elliptic v equation,
homogeneous Neumann boundary conditions.
```

Thus positivity/nonnegativity and per-solution regularity are already inside the global-solution conjunct. They are not separate P3.1 payloads.

### B. Initial trace

```javascript
InitialTrace intervalDomain u₀ u
```

is a separate conjunct. It is not contained in IsPaper2GlobalClassicalSolution.

### C. Eventual boundedness

```javascript
IsPaper2Bounded intervalDomain u
```

is defined by

```javascript
∃ M, ∀ᶠ t in atTop, intervalDomain.supNorm (u t) ≤ M
```

so the target needs an eventual-in-time L∞ bound for the same chosen global trajectory.

# 2. What is absent: no semiflow object

None of the following occurs in Proposition_1_2:

```javascript
Φ : ℝ≥0 → X → X
Φ 0 = id
Φ (t + s) = Φ t ∘ Φ s
Continuous (Φ t)
Continuous fun q : ℝ≥0 × X => Φ q.1 q.2
```

Nor does the target contain:

```plain text
uniqueness of the global solution,
a canonical choice of solution for each datum,
restart compatibility,
continuous dependence on arbitrary initial data,
local Lipschitz dependence,
a cocycle or semigroup identity.
```

The proposition is purely existential:

```plain text
∀ u₀, ∃ u v, ...
```

Without uniqueness, it does not even define a mathematical solution map u₀ ↦ u; therefore it cannot, as stated, assert a semiflow.

The phrase “global classical solution” here means global existence in time for an individual orbit. It does not mean a continuous dynamical-system semiflow.

# 7. Is there a genuine P3.1 gap beyond Paper 2?

Yes, under the current Lean statements, but it is not semiflow continuity.

## Gap A: eventual boundedness for the same global orbit

Paper 3 Proposition 1.2 asks for:

```javascript
IsPaper2Bounded intervalDomain u
```

which is an atTop assertion. The current Paper 2 Theorem_1_1 statement exposes a bound only on its selected finite interval 0<t<Tmax, even though it also states that the witness is global when 1≤m.

That finite-horizon bound does not definitionally become an eventual bound on the same global witness. The repository records this deliberately with:

```javascript
not_paper2_theorem_1_1_implies_paper3_proposition_1_2
```

for the abstract domain interface.

The interval roadmap therefore needs a separate theorem of the shape:

```javascript
p.χ₀ ≤ 0 → 1 ≤ p.m →
∀ u₀ u v,
  PositiveInitialDatum intervalDomain u₀ →
  IsPaper2GlobalClassicalSolution intervalDomain p u v →
  InitialTrace intervalDomain u₀ u →
  IsPaper2Bounded intervalDomain u
```

or the equivalent explicit ∃ T₀ M, ∀ t≥T₀, ... form.

## Gap B: initial-datum class

The final target quantifies over:

```javascript
PaperPositiveInitialDatum intervalDomain u₀
```

whereas the named residual and the interval two-field frontier quantify over the weaker, larger class:

```javascript
PositiveInitialDatum intervalDomain u₀
```

A Paper 2 theorem proved only for PaperPositiveInitialDatum cannot by itself produce the stronger residual. Either:

```plain text
prove the stronger interval producer for all PositiveInitialDatum,
```

or

```plain text
prove Proposition_1_2 intervalDomain p directly at the final PaperPositiveInitialDatum quantifier.
```

## Gap C: parameter-branch coverage

Proposition_1_2 assumes only

```javascript
p.χ₀ ≤ 0,
1 ≤ p.m.
```

The current Paper 2 Theorem 1.1 API is split into strict-logistic and zero-logistic branches. A bridge must ensure that its parameter coverage really matches the unrestricted fixed p appearing in Proposition 1.2, or intentionally state a restricted P3.1 wrapper.

None of these is a semiflow-continuity issue.

# 8. If a true semiflow API is desired later

A future, stronger dynamical API would need more than the current existential theorem. Schematically, after choosing a state space X, it would require:

```javascript
structure ChemotaxisSemiflow where
  Φ : ℝ≥0 → X → X
  zero : ∀ u₀, Φ 0 u₀ = u₀
  add  : ∀ s t u₀, Φ (s+t) u₀ = Φ s (Φ t u₀)
  solves : ∀ u₀, trajectory (fun t => Φ t u₀) solves the PDE
  continuous_at_time : ∀ t, Continuous (Φ t)
```

Joint continuity in (t,u₀) or a local Lipschitz estimate would be stronger optional fields.

Analytically, such a structure normally requires:

```plain text
existence,
uniqueness,
restart/concatenation compatibility,
continuous dependence on initial data.
```

For the local fixed-point construction, a typical sufficient estimate would be, on bounded sets and finite windows,

```plain text
sup_{0≤t≤T} ‖u(t;u₀)-u(t;v₀)‖∞
  ≤ C(T,M) ‖u₀-v₀‖∞,
```

or the analogous L² estimate followed by smoothing. But this is not required by the current P3.1 target. It would be a new theorem-level strengthening for later dynamical work.

# Final answer

1. P3.1 is existential global bounded existence, not a semiflow theorem. Its exact type is Proposition_1_2 intervalDomain p as displayed above.

1. It consumes global classical existence, initial trace, and eventual sup-norm boundedness. Positivity/nonnegativity and classical regularity are already bundled inside IsPaper2GlobalClassicalSolution.

1. It does not consume continuous dependence, uniqueness, a solution map, or a semigroup law.

1. The initialContinuity field in IntervalDomainActualLinearStatementAssembly.lean is a separate later Theorem 2.2/Lemma 3.3 input, localized near constant equilibria; it is not a hidden P3.1 field.

1. The exact bridge is

```javascript

  → Proposition_1_2 intervalDomain p.
```

The real current bridge gap beyond bare Paper 2 existence is the eventual L∞ bound for the same global orbit, plus datum-class and parameter-coverage alignment—not semiflow continuity.
