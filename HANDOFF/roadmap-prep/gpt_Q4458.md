# 1. N14 — `Z2xZ14_gives_non_degenerate_N14_point`

## Classification

**Elementary bridge. No rank-zero descent is needed.**

The group-theoretic input is unusually favorable. In

\[
\mathbf Z/2\mathbf Z\times\mathbf Z/14\mathbf Z,
\]

the rational 2-torsion subgroup is

\[
(\mathbf Z/2\mathbf Z)^2.
\]

Choose

\[
P=(0,1),\qquad T=(1,0).
\]

Then `P` has exact order 14, `7P=(0,7)` has order 2, `T` has order 2, and

\[
T\ne 7P.
\]

Thus the three nonzero rational 2-torsion points are

\[
7P,\quad T,\quad T+7P.
\]

This is precisely the situation that was *not* available in the pure cyclic-order-16 case: here the extra `ℤ/2` factor genuinely supplies full rational 2-torsion.

After choosing the order-14 generator and transporting it to Tate normal form, the remaining work should be:

1. derive the fixed N14 Tate/Kubert relation;
2. use the independent rational 2-torsion point to obtain the required square/discriminant datum, if the target N14 affine model is presented that way;
3. evaluate the explicit rational map to the N14 obstruction curve;
4. prove all denominator and cusp exclusions from exact order, independence, and nonsingularity.

Those last steps are `field_simp`, `ring_nf`, and short group-order contradictions.

## Hardest sublemma

The hardest sublemma is not a polynomial identity. It is the **generator-and-independence bridge**:

```lean
lemma full_two_torsion_of_Z2xZ14
    (ι : Multiplicative (ZMod 2 × ZMod 14) ↪* E.Point) :
    ∃ P T : E.Point,
      orderOf P = 14 ∧
      orderOf T = 2 ∧
      T ≠ 7 • P ∧
      /* the three nonzero 2-torsion points are rational */ := by
  ...
```

The key inequality is discharged before elliptic geometry enters:

```text
(1,0) ≠ (0,7) in Z/2 × Z/14.
```

After applying injectivity of the torsion-structure embedding, Lean obtains `T ≠ 7 • P`.

The second likely friction point is proving that every denominator vanishing would force one of:

```text
P = 0,
2P = 0,
7P = 0,
T = 7P,
Δ(E) = 0,
```

all contradicted by stored hypotheses.

## Shared infrastructure

N14 should reuse:

- the generic exact-order-to-Tate-normal-form bridge;
- the same generalized-Weierstrass coordinate-change lemmas used for N16/N18;
- the “independent second 2-torsion point gives a square discriminant” lemma from the `ℤ/2 × ℤ/16` style argument;
- primitive/pre-`Ψ` division-polynomial identities only if the current N14 model is defined that way.

## Tactic profile

Mostly pure computation:

```text
simp / norm_num          group-coordinate facts
field_simp               rational maps and denominator clearing
ring / ring_nf           Tate and obstruction-curve identities
linear_combination       eliminate the Kubert relation
```

No local field, Selmer group, Mordell-Weil rank, or separatedness theorem should appear in this file.

## Verdict

**Do this first.** It is the cleanest isolated win and may also produce a reusable `fullTwoTorsionOfProductTorsion` helper for other composite cases.

# 2. N20 — `exists_rational_two_isogeny_quotient`

## Classification

**Elementary explicit isogeny construction. No rank-zero descent is needed for this helper itself.**

The generic algebra is the classical rational-2-torsion situation

\[
E_{A,B}: y^2=x^3+Ax^2+Bx,
\]

with kernel point `(0,0)`. The quotient is

\[
E'_{A,B}:Y^2=X^3-2AX^2+(A^2-4B)X.
\]

Away from `x=0`, the forward map can be written

\[
X=\frac{y^2}{x^2},
\qquad
Y=\frac{y(B-x^2)}{x^2}.
\]

For the concrete N20 curve in the repository,

\[
A=1,\qquad B=-1,
\]

so

\[
E:y^2=x^3+x^2-x,
\]

\[
E':Y^2=X^3-2X^2+5X,
\]

and the checked scratch formula is

\[
\phi(x,y)=\left(rac{y^2}{x^2},-rac{y(x^2+1)}{x^2}ight).
\]

The repository already proves the symbolic landing equation by rewriting the source curve equation, clearing denominators, and using `ring`.

## Hardest sublemma

The hard part is **totalizing the affine rational formula as a group homomorphism**, not checking the displayed equation.

One must handle three branches:

```text
O       ↦ O,
(0,0)   ↦ O,
(x,y), x ≠ 0 ↦ the displayed affine formula.
```

Then prove:

```lean
phi.map_add
kernel_phi_eq : phi.ker = AddSubgroup.zmultiples T
```

and, if the downstream theorem needs the full quotient package,

```lean
phihat_comp_phi : phihat.comp phi = 2 • AddMonoidHom.id _
phi_comp_phihat : phi.comp phihat = 2 • AddMonoidHom.id _
```

The dual map in the current scratch file is already written explicitly. The only genuinely tedious part is the finite case split where an input or sum lies at infinity or at `(0,0)`.

A good implementation strategy is to avoid designing a general scheme-theoretic isogeny API. Define the maps directly on Mathlib's existing elliptic-point type, exactly as `Affine/Point.lean` defines addition by cases, and prove the required group-homomorphism fields.

## Shared infrastructure

N20 should be used to land a reusable package:

```lean
structure ExplicitTwoIsogenyData where
  E Ehat : WeierstrassCurve K
  T : E.Point
  phi : E.Point →+ Ehat.Point
  phihat : Ehat.Point →+ E.Point
  order_T : orderOf T = 2
  kernel_phi : phi.ker = AddSubgroup.zmultiples T
  dual_comp : phihat.comp phi = 2 • AddMonoidHom.id E.Point
  comp_dual : phi.comp phihat = 2 • AddMonoidHom.id Ehat.Point
```

That package can be reused almost verbatim for:

- the rank-zero quotient in N49;
- the existing N15/N21 2-isogeny descents;
- any later elliptic quotient with rational 2-torsion.

## Tactic profile

The map equations and dual-composition coordinates are pure computation:

```text
field_simp [hx]
ring_nf
rw [source_equation]
```

The homomorphism and kernel proofs are real structural Lean work, but not deep mathematics. `decide` is useful only for finite branch tags; it does not prove the rational-function identities.

## Verdict

**Do this second**, and make it generic enough that N49 can consume it. The generic extraction is worth more than proving only the concrete 20a4 theorem.

# 3. N49 — `no_raw_order49_tate_obstruction`

## Classification

**Real arithmetic, but only elliptic rank-zero arithmetic. It should not require anything comparable to the N18 genus-two Jacobian.**

The clean route is not to attack the raw high-degree Tate system by a giant resultant. A rational point of exact order 49 determines a rational cyclic subgroup of order 49, hence a rational point on `X₀(49)`. A convenient elliptic model is

\[
X_0(49):
 y^2+xy=x^3-x^2-2x-1.
\]

It has the rational 2-torsion point

\[
(2,-1).
\]

Indeed this point lies on the curve and is fixed by the generalized-Weierstrass negation

\[
(x,y)\longmapsto(x,-y-x).
\]

There is an especially useful 2-isogeny model. Put

\[
U=4(x-2),
\qquad
V=4(2y+x).
\]

Then a direct ring calculation gives

\[
E_{49}:V^2=U^3+21U^2+112U.
\]

The 2-isogenous companion is

\[
\widehat E_{49}:
\widehat V^2=\widehat U^3-42\widehat U^2-7\widehat U.
\]

Thus the relevant squareclasses are supported only at the very small set of primes `{2,7,∞}`. This is an ordinary finite 2-isogeny descent, not a sextic number-field descent.

Depending on how the current rank API is organized, there are two ways to finish:

1. compute the two isogeny-descent images and invoke an existing rank bound; or
2. follow the current N15/N21 pattern: prove the required 2-divisibility statement, then use the already-built 2-adic separatedness/reduction-injectivity layer to rule out a free part.

Either way, the local arithmetic is much smaller than N18.

## Hardest sublemma

The hardest sublemma is the **raw-Tate-system to noncuspidal `X₀(49)` bridge**:

```lean
lemma X0_49_point_of_raw_order49
    (hraw : RawOrder49TateSystem b c ...)
    (hnd : Nondegenerate49 b c ...) :
    ∃ P : X0_49.Point,
      P ≠ cusp0 ∧ P ≠ cuspInf := by
  ...
```

The elliptic rank-zero proof is finite and reusable. The risk is producing an explicit rational map from the current raw variables and proving that a nondegenerate order-49 datum cannot collapse to a cusp.

There are three possible bridge implementations, in decreasing order of preference:

1. an existing explicit forgetful map from the chosen `X₁(49)` Tate model to the model of `X₀(49)`;
2. a low-degree invariant under the diamond-operator action, with a stored `field_simp`/`ring_nf` certificate;
3. a direct quotient construction by the cyclic subgroup, only if a usable finite Vélu-sum API already exists.

Do **not** begin by fully normalizing a gigantic resultant in Lean. Generate a compact rational-map certificate externally once, then verify it in Lean.

## Shared infrastructure

N49 reuses two independent stacks:

### From the high-order Tate cases

- exact-order Tate normal form;
- primitive/pre-`Ψ` division factors;
- denominator and cusp-locus bookkeeping;
- resultant and rational-root certificates for finite fibers.

### From N20/N15/N21

- the generic rational-2-torsion isogeny package;
- local squareclass enumeration;
- reduction maps and prime-to-`p` torsion injectivity;
- 2-adic separatedness, if the repository uses divisibility rather than a packaged Mordell-Weil rank theorem.

## Tactic profile

The bridge and finite fibers are computation:

```text
field_simp
ring_nf
linear_combination
resultant identities
Polynomial rational-root lemmas
```

The rank-zero statement is real mathematics, but it is a small elliptic 2-isogeny descent. Final local tables can be `native_decide` over finite residue rings once the global-to-local theorem is in place.

## N18 comparison

N49 is **not secretly N18-hard** if routed through `X₀(49)`:

```text
N18: genus-2 Picard/Jacobian + nontrivial endomorphism/descent infrastructure.
N49: one elliptic curve with rational 2-torsion + a finite {2,7}-supported descent.
```

It becomes unnecessarily difficult only if the implementation insists on proving unsolvability directly on the raw `X₁(49)` equations.

## Verdict

**Do this third**, immediately after extracting the reusable 2-isogeny framework from N20.

# 4. N25 — `no_explicit_order25_obstruction`

## Classification

**This declaration needs a syntactic triage before anyone calls it a finite computation.**

The phrase “degree-40 factor” is ambiguous in exactly the dangerous way.

## Case A: genuinely univariate

Suppose the statement is essentially

```lean
∀ t : ℚ, H25.eval t = 0 → t ∈ explicitCuspValues
```

for a fixed primitive `H25 : ℤ[X]` or `ℚ[X]`. Then it is elementary:

1. clear denominators and make the polynomial primitive;
2. apply the rational-root theorem;
3. enumerate divisors of the constant and leading coefficients;
4. evaluate all candidates;
5. identify the surviving roots with cusp values.

The degree 40 is then a performance issue, not a mathematical issue. The final finite list can be checked by `native_decide`/`norm_num`, and N25 should be moved to **second place**, before N20 or immediately after N14.

## Case B: bivariate Tate/Kubert factor

Suppose instead the statement is

```lean
∀ b c : ℚ,
  F25 b c = 0 → Degenerate25 b c
```

where “degree 40” means total degree or degree in one variable of a plane equation. Then this is not a rational-root problem. It is the rational-point problem on the exact-order component of `X₁(25)`, a high-genus modular curve (the smooth `X₁(25)` has genus 12).

In that case:

- `gcd`, `resultant`, and factorization certify the model;
- they do **not** classify its rational points;
- `decide` cannot quantify over `ℚ²`;
- a finite mod-`p` search only proves a local obstruction if every noncuspidal global point would reduce into the searched locus and the bad-denominator cases are separately controlled.

The likely successful route is a quotient-and-fiber argument:

```text
exact-order Tate component
  → explicit lower-genus quotient
  → rank-zero or finite rational-point theorem on the quotient
  → finite fiber/root checks
  → cusps only.
```

A map merely to `X₀(25)` will not suffice: `X₀(25)` is genus zero and has noncuspidal rational points corresponding to rational 25-isogenies. Forgetting the generator loses exactly the information needed to rule out rational 25-torsion.

## Hardest sublemma

In Case B, the hard sublemma is the whole arithmetic core:

```lean
lemma rational_points_on_exact_order25_component_are_cuspidal :
    ∀ P : X1_25_component.Point ℚ, IsCusp P := by
  ...
```

For a Lean-minimal proof, replace that abstract statement by explicit data:

```lean
structure N25QuotientCertificate where
  Q : ExplicitCurve
  forward : Tate25Point → Q.Point
  forward_equation : ...
  Q_rational_points : FiniteExplicitList Q.Point
  fiber_polynomials : ...
  fiber_no_nonCusp_root : ...
```

The single most important reconnaissance question is therefore:

```text
What is the arity of `no_explicit_order25_obstruction`'s degree-40 factor?
```

That answer changes the build estimate by an order of magnitude.

## Shared infrastructure

N25 can reuse:

- Tate normal form and exact-order predicates;
- pre-`Ψ`/primitive division-factor production;
- denominator/cusp-locus APIs;
- resultant and finite-fiber certification;
- local separatedness only **after** an elliptic quotient has been identified.

It cannot be closed merely by importing N18 separatedness. Separatedness is the last step of a descent; it does not manufacture a finite quotient or a Selmer bound.

## Tactic profile

Purely computational pieces:

```text
factor/multiplicity certificates
resultant identities
field_simp + ring_nf
finite-field residue tables
rational-root checks in individual fibers
```

Real mathematics in the likely bivariate case:

```text
construction of a useful quotient
rank-zero/Selmer or another rational-point theorem on that quotient
proof that every rational point descends and every fiber is exhausted
```

## N18 comparison

**N25 is the only listed helper that may be as hard as, or harder than, N18.**

It is not because degree 40 is large. It is because a high-genus curve can have a compact defining polynomial while its rational-point theorem remains the entire problem.

## Verdict

Perform the arity/model audit immediately, but schedule the actual N25 proof **last** unless it turns out to be univariate or admits a one-prime local obstruction.

# 7. What `decide`, `norm_num`, and `ring` can and cannot close

## Directly closeable by computation

- equality of explicit Tate/division polynomials;
- verification of rational maps;
- denominator-cleared curve equations;
- resultant/Bézout certificates;
- finite-field point tables;
- a finite list of rational-root candidates;
- finite local squareclass tables;
- finite exceptional-point branches of an explicit isogeny.

Typical endpoint:

```lean
native_decide   -- only after all variables range over a finite type
norm_num        -- concrete integer/rational arithmetic
ring_nf         -- polynomial identities
field_simp      -- rational identities with explicit nonzero hypotheses
```

## Not directly closeable by computation

- “a bivariate polynomial curve has no rational points”;
- rank zero of an elliptic curve without a descent/height theorem;
- surjectivity of a Kummer map;
- local-to-global Selmer reasoning;
- injectivity/separatedness of an infinite local filtration;
- classification of rational points on a genus-12 curve.

A useful review rule is:

> `native_decide` is the **last line** of a finite certificate, not the theorem that turns a rational-point problem into a finite certificate.
