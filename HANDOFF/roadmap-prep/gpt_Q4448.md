# Eliminating `exists_rational_two_isogeny_quotient`

## Executive answer

The axiom can be replaced by one generic explicit Vélu theorem over any field of characteristic different from `2`, specialized to `ℚ`.

For

\[
E:\quad y^2+a_1xy+a_3y=x^3+a_2x^2+a_4x+a_6
\]

and a nonzero rational point of order two

\[
Q=(x_0,y_0),
\]

one has

\[
2y_0+a_1x_0+a_3=0.
\tag{1}
\]

Define

\[
\boxed{
 t=3x_0^2+2a_2x_0+a_4-a_1y_0
}
\tag{2}
\]

and

\[
\boxed{w=x_0t.}
\tag{3}
\]

Equivalently, with the usual Weierstrass quantities

\[
b_2=a_1^2+4a_2,
\qquad
b_4=2a_4+a_1a_3,
\]

relation (1) gives

\[
t=3x_0^2+\frac{b_2}{2}x_0+\frac{b_4}{2}.
\tag{4}
\]

The Vélu quotient by `\{O,Q\}` is

\[
\boxed{
E':\quad
 y^2+a_1xy+a_3y
 =x^3+a_2x^2+(a_4-5t)x+
 \bigl(a_6-b_2t-7x_0t\bigr).
}
\tag{5}
\]

Away from `O` and `Q`, put `u=x-x_0`.  The quotient map is

\[
\boxed{
 x'=x+\frac{t}{u},
}
\tag{6}
\]

\[
\boxed{
 y'=y-
 \frac{t(2y+a_1x+a_3)}{2u^2}
 -\frac{a_1t}{2u}.
}
\tag{7}
\]

Define `φ(O)=O` and `φ(Q)=O`.  Then `φ:E→E'` is a degree-two isogeny with

\[
\ker\phi=\{O,Q\}.
\]

The formulas in (2), (5), (6), and (7) are the general-Weierstrass formulas that should replace the axiom.

The current Mathlib elliptic-curve development supplies Weierstrass coefficients, admissible variable changes, normal forms, affine and projective point group laws, and division polynomials.  It does **not** currently supply a bundled elliptic-curve isogeny type, a quotient by a finite subgroup, or Vélu's construction.  Consequently the missing project module should bundle the explicit point map as an `AddMonoidHom` and prove additivity by the existing affine group-law formulas.

# 1. The familiar centered formulas

The general formulas become transparent after one coordinate change.

Set

\[
u=x-x_0,
\qquad
v=y-y_0+\frac{a_1}{2}u
  =y+\frac{a_1x+a_3}{2}.
\tag{8}
\]

Define

\[
A=3x_0+\frac{b_2}{4},
\qquad
B=t.
\tag{9}
\]

Using the equation of `Q` and (1), the curve becomes

\[
\boxed{
E_c:\quad v^2=u^3+Au^2+Bu.
}
\tag{10}
\]

There are two convenient equivalent quotient models.

## 1.1 The usual descent model

Let

\[
D=A^2-4B.
\]

Then

\[
\boxed{
E'_0:\quad V^2=U^3-2AU^2+DU
}
\tag{11}
\]

and

\[
\boxed{
U=\frac{v^2}{u^2},
\qquad
V=\frac{v(B-u^2)}{u^2}.
}
\tag{12}
\]

This is the form most often used in a two-isogeny descent.

## 1.2 The literal Vélu-sum model

An equivalent quotient is

\[
\boxed{
E'_V:\quad
W^2=X^3+AX^2-4BX-4AB
}
\tag{13}
\]

with

\[
\boxed{
X=u+\frac Bu,
\qquad
W=v\left(1-\frac{B}{u^2}\right).
}
\tag{14}
\]

The two target models are related by

\[
U=X+A,
\qquad
V=-W.
\tag{15}
\]

Transporting (13) and (14) back through (8) gives exactly (5)--(7).

## 1.3 Specialization to a factored cubic

Suppose the starting equation is

\[
y^2=(x-x_0)(x^2+ax+b).
\]

Then after `u=x-x_0`,

\[
A=2x_0+a,
\qquad
B=x_0^2+ax_0+b,
\]

and (11)--(14) apply verbatim.

# 2. Discriminants and nonsingularity of the quotient

For the centered source (10),

\[
\boxed{
\Delta(E_c)=16B^2(A^2-4B).
}
\tag{16}
\]

For the centered Vélu target (13),

\[
\boxed{
\Delta(E'_V)=256B(A^2-4B)^2.
}
\tag{17}
\]

Since the coordinate change (8) has scaling unit `u=1`, it does not change the discriminant.  Hence ellipticity of `E` implies

\[
B\ne0,
\qquad
A^2-4B\ne0,
\]

and therefore (17) proves that `E'` is elliptic.

This is the cleanest Lean proof of `E'.IsElliptic`: do not expand the general target discriminant from scratch in the final theorem.  Prove that the original curve is changed to (10), prove (16) and (17) by `ring`, then transport ellipticity through `WeierstrassCurve.VariableChange`.

# 3. Denominator-free map certificate

For the centered source, define the homogeneous cubic

\[
F(X,Y,Z)=Y^2Z-X^3-AX^2Z-BXZ^2.
\]

For the target (13), define

\[
F'(X,Y,Z)=Y^2Z-X^3-AX^2Z+4BXZ^2+4ABZ^3.
\]

The rational map (14) is represented, away from the two base points, by

\[
\boxed{
\Phi[X:Y:Z]
=
\bigl[
 X(X^2+BZ^2):
 Y(X^2-BZ^2):
 X^2Z
\bigr].
}
\tag{18}
\]

The exact polynomial certificate is

\[
\boxed{
F'(\Phi(X,Y,Z))
=
X^2(X^2-BZ^2)^2F(X,Y,Z).
}
\tag{19}
\]

This is a single `ring` proof.  The displayed triple is zero precisely at the two points where the morphism must be filled in separately:

```text
O       -> O,
(0,0)   -> O.
```

On every other point of the source, (19) constructs a target point.

An affine denominator-cleared version is

\[
\boxed{
 u^4
 \left(
 W^2-X^3-AX^2+4BX+4AB
 \right)
 =
 (u^2-B)^2
 \left(
 v^2-u^3-Au^2-Bu
 \right),
}
\tag{20}
\]

where `X=u+B/u` and `W=v(1-B/u²)`.

# 4. Optional explicit dual isogeny

For proving the strongest interface, it is useful to include the dual map.

On the target (13), the nonzero dual-kernel point is

\[
(-A,0).
\]

Away from this point and `O`, define

\[
\boxed{
\widehat u=\frac{W^2}{4(X+A)^2},
}
\tag{21}
\]

\[
\boxed{
\widehat v=
\frac{W(X^2+2AX+4B)}{8(X+A)^2}.
}
\tag{22}
\]

Then

\[
\widehat\phi:E'_V\longrightarrow E_c,
\qquad
\ker\widehat\phi=\{O,(-A,0)\},
\]

and

\[
\widehat\phi\circ\phi=[2]_{E_c},
\qquad
\phi\circ\widehat\phi=[2]_{E'_V}.
\tag{23}
\]

Transporting (21) and (22) through (8) gives a dual map into the original generalized Weierstrass model.  For the existence axiom it is enough to construct `φ`; for later descent code, bundling the dual identities is usually worth the additional finite algebra.

  \]

There is currently no Mathlib declaration for:

```text
EllipticCurve.Isogeny
Velu / Vélu
quotient by a finite subgroup
kernel subgroup -> quotient elliptic curve
```

and `VariableChange` currently acts on curve coefficients, not as a ready-made additive equivalence of point groups.  Thus the missing implementation consists of two reusable adapters:

1. a point-level additive equivalence associated to a `VariableChange`;
2. the generic centered two-isogeny map (14), proved additive by the explicit point formulas.

Neither requires Mordell--Weil theory, descent, schemes, or a quotient existence theorem.

# 6. Recommended project-local interface

The project does not need to formalize a general category of isogenies first.  A small structure containing exactly the needed properties is enough.

```lean
import Mathlib.AlgebraicGeometry.EllipticCurve.NormalForms
import Mathlib.AlgebraicGeometry.EllipticCurve.Projective.Point
import Mathlib.AlgebraicGeometry.EllipticCurve.DivisionPolynomial.Basic

set_option autoImplicit false

namespace FLT.RationalTwoIsogeny

noncomputable section

open WeierstrassCurve
open WeierstrassCurve.Affine
open WeierstrassCurve.Affine.Point

abbrev ECPoint (E : WeierstrassCurve ℚ) := E.toAffine.Point

/-- The exact package replacing the old existence axiom. -/
structure QuotientData
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (Q : ECPoint E) where
  E' : WeierstrassCurve ℚ
  hE' : E'.IsElliptic
  phi : ECPoint E →+ ECPoint E'
  phi_Q : phi Q = 0
  ker_phi : ∀ P : ECPoint E,
    phi P = 0 ↔ P = 0 ∨ P = Q

  /-- Optional, but useful for descent. -/
  dual : ECPoint E' →+ ECPoint E
  dual_phi : ∀ P : ECPoint E,
    dual (phi P) = 2 • P
  phi_dual : ∀ R : ECPoint E',
    phi (dual R) = 2 • R

end
end FLT.RationalTwoIsogeny
```

If the old axiom only requested a quotient curve, a map, and `Q∈ker φ`, the last three fields may be omitted.  Keeping them gives a robust definition of “the relevant properties” without first building an abstract algebraic-geometric isogeny API.

# 7. Coefficient definitions in Lean

```lean
namespace FLT.RationalTwoIsogeny

noncomputable section

open WeierstrassCurve

/-- Vélu's `t_Q` for a rational point of order two. -/
def veluT
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) : ℚ :=
  3 * x0 ^ 2 + 2 * E.a₂ * x0 + E.a₄ - E.a₁ * y0

/-- Vélu's `w_Q`; for an order-two point, the other summand is zero. -/
def veluW
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) : ℚ :=
  x0 * veluT E x0 y0

/-- The quotient coefficients in the original Weierstrass gauge. -/
def quotientCurve
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    WeierstrassCurve ℚ :=
  let t := veluT E x0 y0
  let w := veluW E x0 y0
  ⟨E.a₁,
   E.a₂,
   E.a₃,
   E.a₄ - 5 * t,
   E.a₆ - E.b₂ * t - 7 * w⟩

/-- Centered `A`. -/
def centerA (E : WeierstrassCurve ℚ) (x0 : ℚ) : ℚ :=
  3 * x0 + E.b₂ / 4

/-- The coordinate change from centered coordinates to the original model. -/
def centerChange
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    WeierstrassCurve.VariableChange ℚ :=
  ⟨1, x0, -E.a₁ / 2, y0⟩

/-- The centered source. -/
def centeredSource
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    WeierstrassCurve ℚ :=
  centerChange E x0 y0 • E

/-- The centered literal Vélu quotient. -/
def centeredTarget
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    WeierstrassCurve ℚ :=
  let A := centerA E x0
  let B := veluT E x0 y0
  ⟨0, A, 0, -4 * B, -4 * A * B⟩

end
end FLT.RationalTwoIsogeny
```

```lean
    2 * y0 + E.a₁ * x0 + E.a₃ = 0 := by
  -- `addOrderOf_eq_prime_iff.mp hQ` gives `2 • Q = 0` and `Q ≠ 0`.
  -- Rewrite `2 • Q = 0` as `Q = -Q`, then use `Point.neg_some`.
  ...

lemma veluT_eq_b
    (E : WeierstrassCurve ℚ)
    (x0 y0 : ℚ)
    (h2 : 2 * y0 + E.a₁ * x0 + E.a₃ = 0) :
    veluT E x0 y0 =
      3 * x0 ^ 2 + E.b₂ * x0 / 2 + E.b₄ / 2 := by
```

The centered-source identity can be proved coefficientwise with

```lean
ext <;>
simp [centeredSource, centerChange,
  WeierstrassCurve.variableChange_a₁,
  WeierstrassCurve.variableChange_a₂,
  WeierstrassCurve.variableChange_a₃,
  WeierstrassCurve.variableChange_a₄,
  WeierstrassCurve.variableChange_a₆,
  centerA, veluT, WeierstrassCurve.b₂,
  WeierstrassCurve.b₄] <;>
ring_nf
```

using:

```text
h2   : 2*y0 + a1*x0 + a3 = 0
hEq  : y0^2 + a1*x0*y0 + a3*y0
       = x0^3 + a2*x0^2 + a4*x0 + a6
```

The result is

```lean
centerChange E x0 y0 • E
  = ⟨0, centerA E x0, 0, veluT E x0 y0, 0⟩.
```

Likewise, applying `(centerChange E x0 y0)⁻¹` to `centeredTarget` gives exactly `quotientCurve E x0 y0`.  This coefficient equality is another finite `ext <;> field_simp <;> ring_nf` proof using `h2` and `hEq`.

# 8. Ring certificates to put in the implementation

The following lemmas are independent of elliptic-curve APIs and should compile as short `ring` proofs.

```lean
namespace FLT.RationalTwoIsogeny

/-- Homogeneous source residual for `v²=u³+Au²+Bu`. -/
def sourceHom (A B X Y Z : ℚ) : ℚ :=
  Y ^ 2 * Z - X ^ 3 - A * X ^ 2 * Z - B * X * Z ^ 2

/-- Homogeneous target residual for
`W²=X³+AX²-4BX-4AB`. -/
def targetHom (A B X Y Z : ℚ) : ℚ :=
  Y ^ 2 * Z - X ^ 3 - A * X ^ 2 * Z
    + 4 * B * X * Z ^ 2 + 4 * A * B * Z ^ 3

lemma velu_homogeneous_certificate
    (A B X Y Z : ℚ) :
    targetHom A B
      (X * (X ^ 2 + B * Z ^ 2))
      (Y * (X ^ 2 - B * Z ^ 2))
      (X ^ 2 * Z)
      =
    X ^ 2 * (X ^ 2 - B * Z ^ 2) ^ 2 *
      sourceHom A B X Y Z := by
  simp [sourceHom, targetHom]
  ring

lemma source_discriminant
    (A B : ℚ) :
    (⟨0, A, 0, B, 0⟩ : WeierstrassCurve ℚ).Δ
      = 16 * B ^ 2 * (A ^ 2 - 4 * B) := by
  simp [WeierstrassCurve.Δ, WeierstrassCurve.b₂,
    WeierstrassCurve.b₄, WeierstrassCurve.b₆,
    WeierstrassCurve.b₈]
  ring

lemma target_discriminant
    (A B : ℚ) :
    (⟨0, A, 0, -4 * B, -4 * A * B⟩ :
      WeierstrassCurve ℚ).Δ
      = 256 * B * (A ^ 2 - 4 * B) ^ 2 := by
  simp [WeierstrassCurve.Δ, WeierstrassCurve.b₂,
    WeierstrassCurve.b₄, WeierstrassCurve.b₆,
    WeierstrassCurve.b₈]
  ring

lemma velu_affine_certificate
    (A B u v : ℚ) (hu : u ≠ 0) :
    let X := u + B / u
    let Y := v * (1 - B / u ^ 2)
    u ^ 4 *
      (Y ^ 2 - (X ^ 3 + A * X ^ 2 - 4 * B * X - 4 * A * B))
      =
    (u ^ 2 - B) ^ 2 *
      (v ^ 2 - (u ^ 3 + A * u ^ 2 + B * u)) := by
  dsimp
  field_simp [hu]
  ring

end FLT.RationalTwoIsogeny
```

The homogeneous certificate is preferable in the final point-map proof because it avoids repeated denominator manipulation.

# 9. Total point map

Use the affine point type, since it already separates `O` from affine points.

```lean
noncomputable def veluPointMap
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (x0 y0 : ℚ)
    (hEq0 : E.Equation x0 y0)
    (h2 : 2 * y0 + E.a₁ * x0 + E.a₃ = 0) :
    E.toAffine.Point →
      (quotientCurve E x0 y0).toAffine.Point
  | 0 => 0
  | .some x y h =>
      if hx : x = x0 then
        0
      else
        let u := x - x0
        let t := veluT E x0 y0
        let xp := x + t / u
        let yp :=
          y - t * (2 * y + E.a₁ * x + E.a₃) / (2 * u ^ 2)
            - E.a₁ * t / (2 * u)
        .mk (by
          -- Prove `(quotientCurve ...).Equation xp yp`.
          -- Move to `(u,v)` using the centered-source identity,
          -- then apply `velu_affine_certificate`.
          ...)
```

The branch `x=x0` is genuinely only the point `Q`.  In centered coordinates the equation is

\[
v^2=u(u^2+Au+B),
\]

so `u=0` implies `v=0`, and hence `y=y0`.

The kernel theorem is therefore short:

```lean
lemma veluPointMap_eq_zero_iff
    (P : E.toAffine.Point) :
    veluPointMap E x0 y0 hEq0 h2 P = 0
      ↔ P = 0 ∨ P = Point.mk hEq0 := by
  rcases P with (_ | ⟨x, y, hP⟩)
  · simp [veluPointMap]
  · by_cases hx : x = x0
    · -- Centered equation gives `y=y0`.
      ...
    · simp [veluPointMap, hx]
```

# 10. The one substantial proof: additivity

Curve preservation and the kernel computation are short ring proofs.  To obtain an actual `AddMonoidHom`, one must additionally prove

```lean
veluPointMap (P + R) = veluPointMap P + veluPointMap R.
```

Mathlib currently has no theorem saying that a rational map of smooth cubic curves carrying `O` to `O` is automatically a group homomorphism.  The shortest axiom-free implementation is therefore one generic theorem for the centered formula.

```lean
/-- Generic centered Vélu map as an additive homomorphism. -/
noncomputable def centeredVeluHom
    (A B : ℚ)
    (hB : B ≠ 0)
    (hD : A ^ 2 - 4 * B ≠ 0) :
    (⟨0, A, 0, B, 0⟩ : WeierstrassCurve ℚ).toAffine.Point →+
      (⟨0, A, 0, -4 * B, -4 * A * B⟩ :
        WeierstrassCurve ℚ).toAffine.Point :=
by
  refine
    { toFun := centeredVeluPointMap A B
      map_zero' := rfl
      map_add' := ?_ }
  intro P R
  -- Cases:
  -- 1. P=O or R=O;
  -- 2. P=(0,0) or R=(0,0);
  -- 3. R=-P, so both sides are O;
  -- 4. P=R, use the doubling formulas;
  -- 5. generic distinct x-coordinates, use the secant formulas.
  -- Every nontrivial branch is `field_simp` followed by `ring_nf`,
  -- using the two source curve equations.
  ...
```

This is the only lengthy part of the replacement theorem.  It is finite algebra, not a new mathematical assumption.  Write it once over a general field with `[Field K] [NeZero (2:K)]` if reuse is valuable; writing it first over `ℚ` is simpler.

A second reusable adapter should package a variable change as a point-group equivalence:

```lean
noncomputable def variableChangePointAddEquiv
    (C : WeierstrassCurve.VariableChange ℚ)
    (E : WeierstrassCurve ℚ) :
    (C • E).toAffine.Point ≃+ E.toAffine.Point := by
  -- Coordinates:
  -- (x,y) |-> (C.u^2*x + C.r,
  --            C.u^3*y + C.u^2*C.s*x + C.t).
  -- The inverse uses `C⁻¹`.
  -- Equation preservation and additivity follow from the existing
  -- affine formulas and the variable-change coefficient identities.
  ...
```

Then the desired homomorphism is the composition

```text
E points
  -- inverse center variable change -->
centered source points
  -- centeredVeluHom -->
centered target points
  -- inverse target variable change -->
quotientCurve points.
```

If avoiding the point-level variable-change adapter, prove additivity directly for formulas (6)--(7); the case split is the same but the expressions are larger.

# 11. Final theorem replacing the axiom

The core affine theorem should have the following shape.

```lean
theorem exists_rational_two_isogeny_quotient_affine
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    {Q : E.toAffine.Point}
    (hQ : addOrderOf Q = 2) :
    Nonempty (QuotientData E Q) := by
  have hprime : Nat.Prime 2 := by norm_num
  letI : Fact (Nat.Prime 2) := ⟨hprime⟩

  have h2Q : 2 • Q = 0 :=
    (addOrderOf_eq_prime_iff.mp hQ).1
  have hQ0 : Q ≠ 0 :=
    (addOrderOf_eq_prime_iff.mp hQ).2

  rcases Q with (_ | ⟨x0, y0, hns⟩)
  · exact (hQ0 rfl).elim
  · have hEq0 : E.Equation x0 y0 := hns.1
    have htwo : 2 * y0 + E.a₁ * x0 + E.a₃ = 0 := by
        -- Rewrite `h2Q` using `two_nsmul`, `Point.neg_some`,
        -- and injectivity of the `.some` constructor.
        ...

    let A := centerA E x0
    let B := veluT E x0 y0

    have hsrcΔ : E.Δ ≠ 0 := E.isUnit_Δ.ne_zero
    have hfactor : E.Δ = 16 * B ^ 2 * (A ^ 2 - 4 * B) := by
      -- Either transport `source_discriminant` through `centerChange`,
      -- using `variableChange_Δ`, or expand and use `hEq0, htwo`.
      ...

    have hB : B ≠ 0 := by
      intro h
      apply hsrcΔ
      rw [hfactor, h]
      norm_num

    have hD : A ^ 2 - 4 * B ≠ 0 := by
      intro h
      apply hsrcΔ
      rw [hfactor, h]
      ring

    have htarget :
        (quotientCurve E x0 y0).IsElliptic := by
      -- Use `target_discriminant`, then transport through the
```
