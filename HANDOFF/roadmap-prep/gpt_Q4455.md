# Explicit Vélu degree-two quotient: Lean 4 reference draft

## Executive correction

The coefficient formulas in the question are the standard normalized Vélu formulas.  Put

\[
\tau=3x_0^2+2a_2x_0+a_4-a_1y_0,
\qquad \omega=x_0\tau.
\]

Then the normalized quotient is

\[
\boxed{
E'=[a_1,a_2,a_3,a_4-5\tau,
             a_6-(a_1^2+4a_2)\tau-7\omega].
}
\]

Sage uses exactly these formulas: its degree-two helper returns
`v = 3*x0^2 + 2*a2*x0 + a4 - a1*y0`, `w = x0*v`, and the common Vélu codomain routine returns
`[a1,a2,a3,a4-5*v,a6-(a1^2+4*a2)*v-7*w]`.

There are two corrections to common informal statements.

1. On a generalized Weierstrass equation, an affine point of order two need not have `y0 = 0`.  The correct fixed-by-negation equation is
   \[
   2y_0+a_1x_0+a_3=0.
   \]
2. In general `Delta(E')` is **not** a constant degree-dependent multiple of `Delta(E)`.  For this kernel, define
   \[
   A=b_2+12x_0,
   \qquad D=A^2-64\tau.
   \]
   Then the exact factorizations are
   \[
   \boxed{\Delta(E)=\tau^2D,\qquad \Delta(E')=\tau D^2.}
   \]
   These factorizations, rather than a false constant-ratio assertion, prove that `E'` is elliptic.

The normalized affine Vélu map is

\[
\boxed{
\begin{aligned}
X&=x+\frac{\tau}{x-x_0},\\[1mm]
Y&=y-
 \frac{\tau\bigl(y-y_0+a_1(x-x_0)\bigr)}{(x-x_0)^2}.
\end{aligned}}
\tag{V}
\]

It sends `O` and `Q=(x0,y0)` to `O`.  The second formula is equivalently characterized by

\[
2Y+a_1X+a_3
 = (2y+a_1x+a_3)
   \left(1-\frac{\tau}{(x-x_0)^2}\right).
\tag{Vcs}
\]

That completed-square form is the shortest route to the Lean curve-preservation proof.

# 2. Algebra behind the formulas

Let

\[
E:\quad y^2+a_1xy+a_3y
=x^3+a_2x^2+a_4x+a_6.
\]

Write

\[
\begin{aligned}
b_2&=a_1^2+4a_2,\\
b_4&=a_1a_3+2a_4,\\
b_6&=a_3^2+4a_6.
\end{aligned}
\]

For the completed-square coordinate

\[
s=2y+a_1x+a_3
\]

one has the exact identity

\[
s^2=4x^3+b_2x^2+2b_4x+b_6.
\tag{CS}
\]

If `Q=(x0,y0)` is fixed by negation, then

\[
2y_0+a_1x_0+a_3=0
\]

and therefore

\[
4x_0^3+b_2x_0^2+2b_4x_0+b_6=0.
\tag{R}
\]

Set

\[
u=x-x_0,
\qquad A=b_2+12x_0.
\]

The definition of `tau`, together with the fixed-by-negation equation, gives

\[
4\tau=12x_0^2+2b_2x_0+2b_4.
\tag{T}
\]

Consequently the source completed-square equation translates to

\[
\boxed{s^2=4u^3+Au^2+4\tau u.}
\tag{Slocal}
\]

Under the map `(V)`, put

\[
z=X-x_0=u+\frac{\tau}{u},
\qquad
s'=2Y+a_1X+a_3=s\left(1-\frac{\tau}{u^2}\right).
\]

The quotient completed-square equation translates to

\[
\boxed{(s')^2=4z^3+Az^2-16\tau z-4A\tau.}
\tag{Tlocal}
\]

The denominator-cleared identity certifying this is

\[
\boxed{
\begin{aligned}
&u^4\left[
  \left(s\left(1-\frac{\tau}{u^2}\right)\right)^2
  -\left(4\left(u+\frac{\tau}{u}\right)^3
     +A\left(u+\frac{\tau}{u}\right)^2
     -16\tau\left(u+\frac{\tau}{u}\right)-4A\tau
   \right)
 \right]\\
&\qquad=(u^2-\tau)^2
 \left[s^2-(4u^3+Au^2+4\tau u)\right].
\end{aligned}}
\tag{CERT}
\]

It is a single `field_simp; ring` calculation.

The target local cubic factors as

\[
4z^3+Az^2-16\tau z-4A\tau
=(4z+A)(z^2-4\tau).
\]

Let

\[
D=A^2-64\tau.
\]

The discriminants of the source and target completed-square cubics are respectively

\[
16\tau^2D,
\qquad
16\tau D^2.
\]

Since Mathlib's two-torsion-cubic discriminant equals `16 * Delta`, this gives

\[
\Delta(E)=\tau^2D,
\qquad
\Delta(E')=\tau D^2.
\]

# 3. Reference Lean file

The following is intended as the Codex build specification.  The coefficient and discriminant portions are direct kernel-checkable algebra.  The only intentional `sorry` is the additive-map theorem, where the pinned Mathlib lacks a bundled elliptic-curve morphism/isogeny API.

```lean
import Mathlib.AlgebraicGeometry.EllipticCurve.Affine.Point
import Mathlib.AlgebraicGeometry.EllipticCurve.Projective.Point
import Mathlib.AlgebraicGeometry.EllipticCurve.VariableChange
import Mathlib.GroupTheory.OrderOfElement

set_option autoImplicit false
set_option maxHeartbeats 0

namespace RationalTwoIsogeny

noncomputable section

open WeierstrassCurve
open WeierstrassCurve.Affine
open WeierstrassCurve.Affine.Point

/-! ----------------------------------------------------------------------
    Affine order-two data
    ---------------------------------------------------------------------- -/

/-- The coordinate conditions needed from an affine point of order two.

`fixedByNeg` says `(x0,y0) = -(x0,y0)` on the generalized
Weierstrass equation. -/
structure AffineTwoTorsionData (E : WeierstrassCurve ℚ) where
  x0 : ℚ
  y0 : ℚ
  onCurve : E.toAffine.Equation x0 y0
  fixedByNeg : 2 * y0 + E.a₁ * x0 + E.a₃ = 0

namespace AffineTwoTorsionData

variable {E : WeierstrassCurve ℚ}

/-- The actual affine point corresponding to the coordinate package. -/
noncomputable def point
    [E.IsElliptic] (D : AffineTwoTorsionData E) : E.toAffine.Point :=
  .mk D.onCurve

end AffineTwoTorsionData

/-! ----------------------------------------------------------------------
    Vélu coefficients
    ---------------------------------------------------------------------- -/

/-- Vélu's degree-two parameter, called `v` in Sage and often `t` in
printed formulas. -/
def veluT
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) : ℚ :=
  3 * x0 ^ 2 + 2 * E.a₂ * x0 + E.a₄ - E.a₁ * y0

/-- Vélu's second parameter. -/
def veluW
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) : ℚ :=
  x0 * veluT E x0 y0

/-- The normalized Vélu quotient by the order-two subgroup generated by
`(x0,y0)`.

The proof that `(x0,y0)` is order two is deliberately not an argument of
this definition: quotient coefficients should not depend definitionally on
which proof object establishes the point conditions. -/
def twoIsogenyQuotient
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    WeierstrassCurve ℚ :=
  let t := veluT E x0 y0
  let w := veluW E x0 y0
  ⟨E.a₁,
   E.a₂,
   E.a₃,
   E.a₄ - 5 * t,
   E.a₆ - E.b₂ * t - 7 * w⟩

@[simp] lemma twoIsogenyQuotient_a₁
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    (twoIsogenyQuotient E x0 y0).a₁ = E.a₁ := rfl

@[simp] lemma twoIsogenyQuotient_a₂
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    (twoIsogenyQuotient E x0 y0).a₂ = E.a₂ := rfl

@[simp] lemma twoIsogenyQuotient_a₃
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    (twoIsogenyQuotient E x0 y0).a₃ = E.a₃ := rfl

@[simp] lemma twoIsogenyQuotient_a₄
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    (twoIsogenyQuotient E x0 y0).a₄ =
      E.a₄ - 5 * veluT E x0 y0 := rfl

@[simp] lemma twoIsogenyQuotient_a₆
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) :
    (twoIsogenyQuotient E x0 y0).a₆ =
      E.a₆ - E.b₂ * veluT E x0 y0
        - 7 * veluW E x0 y0 := rfl

/-! ----------------------------------------------------------------------
    Completed-square algebra
    ---------------------------------------------------------------------- -/

/-- `2y + a₁x + a₃`. -/
def completedY
    (E : WeierstrassCurve ℚ) (x y : ℚ) : ℚ :=
  2 * y + E.a₁ * x + E.a₃

/-- Right side of the completed-square equation. -/
def twoTorsionCubic
    (E : WeierstrassCurve ℚ) (x : ℚ) : ℚ :=
  4 * x ^ 3 + E.b₂ * x ^ 2 + 2 * E.b₄ * x + E.b₆

/-- Universal completed-square residual identity. -/
lemma completedSquare_residual
    (E : WeierstrassCurve ℚ) (x y : ℚ) :
    completedY E x y ^ 2 - twoTorsionCubic E x
      = 4 *
        (y ^ 2 + E.a₁ * x * y + E.a₃ * y
          - (x ^ 3 + E.a₂ * x ^ 2 + E.a₄ * x + E.a₆)) := by
  simp only [completedY, twoTorsionCubic,
    WeierstrassCurve.b₂, WeierstrassCurve.b₄,
    WeierstrassCurve.b₆]
  ring

lemma completedSquare_of_equation
    (E : WeierstrassCurve ℚ) {x y : ℚ}
    (h : E.toAffine.Equation x y) :
    completedY E x y ^ 2 = twoTorsionCubic E x := by
  rw [WeierstrassCurve.Affine.equation_iff] at h
  have hzero :
      y ^ 2 + E.a₁ * x * y + E.a₃ * y
        - (x ^ 3 + E.a₂ * x ^ 2 + E.a₄ * x + E.a₆) = 0 :=
    sub_eq_zero.mpr h
  have hres := completedSquare_residual E x y
  rw [hzero, mul_zero] at hres
  exact sub_eq_zero.mp hres

lemma equation_of_completedSquare
    (E : WeierstrassCurve ℚ) {x y : ℚ}
    (h : completedY E x y ^ 2 = twoTorsionCubic E x) :
    E.toAffine.Equation x y := by
  rw [WeierstrassCurve.Affine.equation_iff]
  have hres := completedSquare_residual E x y
  have hcs : completedY E x y ^ 2 - twoTorsionCubic E x = 0 :=
    sub_eq_zero.mpr h
  rw [hcs] at hres
  have hsource :
      y ^ 2 + E.a₁ * x * y + E.a₃ * y
        - (x ^ 3 + E.a₂ * x ^ 2 + E.a₄ * x + E.a₆) = 0 := by
    linarith
  exact sub_eq_zero.mp hsource

variable {E : WeierstrassCurve ℚ}

/-- The chosen order-two x-coordinate is a root of Mathlib's
`twoTorsionPolynomial`. -/
lemma twoTorsionCubic_at_point
    (D : AffineTwoTorsionData E) :
    twoTorsionCubic E D.x0 = 0 := by
  have h := completedSquare_of_equation E D.onCurve
  have hleft : completedY E D.x0 D.y0 = 0 := by
    simpa [completedY] using D.fixedByNeg
  rw [hleft, zero_pow (by norm_num : 2 ≠ 0)] at h
  exact h.symm

/-- Four times Vélu's `t` is the derivative of the completed-square cubic
at `x0`. -/
lemma four_mul_veluT
    (D : AffineTwoTorsionData E) :
    4 * veluT E D.x0 D.y0 =
      12 * D.x0 ^ 2 + 2 * E.b₂ * D.x0 + 2 * E.b₄ := by
  simp only [veluT, WeierstrassCurve.b₂, WeierstrassCurve.b₄]
  linear_combination -2 * E.a₁ * D.fixedByNeg

/-- Coefficient of `u²` after translating `x = x0 + u`. -/
def veluA
    (E : WeierstrassCurve ℚ) (x0 : ℚ) : ℚ :=
  E.b₂ + 12 * x0

/-- The second discriminant factor. -/
def veluD
    (E : WeierstrassCurve ℚ) (x0 y0 : ℚ) : ℚ :=
  veluA E x0 ^ 2 - 64 * veluT E x0 y0

lemma source_b₄_eq
    (D : AffineTwoTorsionData E) :
    E.b₄ =
      2 * veluT E D.x0 D.y0
        - 6 * D.x0 ^ 2 - E.b₂ * D.x0 := by
  linarith [four_mul_veluT D]

lemma source_b₆_eq
    (D : AffineTwoTorsionData E) :
    E.b₆ =
      -4 * D.x0 ^ 3 - E.b₂ * D.x0 ^ 2 - 2 * E.b₄ * D.x0 := by
  have h := twoTorsionCubic_at_point D
  simp only [twoTorsionCubic] at h
  linarith

/-! ----------------------------------------------------------------------
    Exact discriminant factorizations
    ---------------------------------------------------------------------- -/

/-- Source discriminant factorization at a rational order-two root. -/
lemma source_delta_factor
    (D : AffineTwoTorsionData E) :
    E.Δ =
      veluT E D.x0 D.y0 ^ 2 * veluD E D.x0 D.y0 := by
  have hb4 := source_b₄_eq D
  have hb6 := source_b₆_eq D
  rw [hb4] at hb6
  have hb8 : E.b₈ = (E.b₂ * E.b₆ - E.b₄ ^ 2) / 4 := by
    linarith [E.b_relation]
  rw [hb4, hb6] at hb8
  rw [WeierstrassCurve.Δ, hb4, hb6, hb8]
  simp only [veluD, veluA]
  field_simp
  ring

@[simp] lemma quotient_b₂
    (D : AffineTwoTorsionData E) :
    (twoIsogenyQuotient E D.x0 D.y0).b₂ = E.b₂ := by
  simp [twoIsogenyQuotient, WeierstrassCurve.b₂]

@[simp] lemma quotient_b₄
    (D : AffineTwoTorsionData E) :
    (twoIsogenyQuotient E D.x0 D.y0).b₄ =
      E.b₄ - 10 * veluT E D.x0 D.y0 := by
  simp [twoIsogenyQuotient, WeierstrassCurve.b₄]
  ring

@[simp] lemma quotient_b₆
    (D : AffineTwoTorsionData E) :
    (twoIsogenyQuotient E D.x0 D.y0).b₆ =
      E.b₆ - 4 * E.b₂ * veluT E D.x0 D.y0
        - 28 * D.x0 * veluT E D.x0 D.y0 := by
  simp [twoIsogenyQuotient, veluW,
    WeierstrassCurve.b₂, WeierstrassCurve.b₆]
  ring

/-- Target discriminant factorization. -/
lemma quotient_delta_factor
    (D : AffineTwoTorsionData E) :
    (twoIsogenyQuotient E D.x0 D.y0).Δ =
      veluT E D.x0 D.y0 * veluD E D.x0 D.y0 ^ 2 := by
  let E' := twoIsogenyQuotient E D.x0 D.y0
  have hb2' : E'.b₂ = E.b₂ := quotient_b₂ D
  have hb4' : E'.b₄ = E.b₄ - 10 * veluT E D.x0 D.y0 :=
    quotient_b₄ D
  have hb6' : E'.b₆ =
      E.b₆ - 4 * E.b₂ * veluT E D.x0 D.y0
        - 28 * D.x0 * veluT E D.x0 D.y0 :=
    quotient_b₆ D
  have hb8' : E'.b₈ = (E'.b₂ * E'.b₆ - E'.b₄ ^ 2) / 4 := by
    linarith [E'.b_relation]
  have hb4 := source_b₄_eq D
  have hb6 := source_b₆_eq D
  rw [hb4] at hb6
  rw [WeierstrassCurve.Δ, hb2', hb4', hb6', hb8', hb4, hb6]
  simp only [veluD, veluA]
  field_simp
  ring

/-- The quotient is elliptic.  This is the concrete replacement for the
nonzero-discriminant part of the old axiom. -/
noncomputable instance quotient_isElliptic
    [E.IsElliptic] (D : AffineTwoTorsionData E) :
    (twoIsogenyQuotient E D.x0 D.y0).IsElliptic := by
  refine ⟨isUnit_iff_ne_zero.mpr ?_⟩
  have hΔ : E.Δ ≠ 0 := E.isUnit_Δ.ne_zero
  have ht : veluT E D.x0 D.y0 ≠ 0 := by
    intro ht
    apply hΔ
    rw [source_delta_factor D, ht]
    norm_num
  have hD : veluD E D.x0 D.y0 ≠ 0 := by
    intro hD
    apply hΔ
    rw [source_delta_factor D, hD]
    simp
  rw [quotient_delta_factor D]
  exact mul_ne_zero ht (pow_ne_zero 2 hD)

/-! ----------------------------------------------------------------------
    Rational map
    ---------------------------------------------------------------------- -/

/-- The normalized Vélu x-coordinate. -/
def veluPhiX
    (E : WeierstrassCurve ℚ) (x0 y0 x : ℚ) : ℚ :=
  x + veluT E x0 y0 / (x - x0)

/-- The normalized Vélu y-coordinate. -/
def veluPhiY
    (E : WeierstrassCurve ℚ)
    (x0 y0 x y : ℚ) : ℚ :=
  y - veluT E x0 y0 *
      (y - y0 + E.a₁ * (x - x0)) / (x - x0) ^ 2

/-- The explicit y-coordinate is equivalent to the especially convenient
completed-square formula. -/
lemma completedY_veluPhi
    (D : AffineTwoTorsionData E)
    {x y : ℚ} (hu : x - D.x0 ≠ 0) :
    completedY (twoIsogenyQuotient E D.x0 D.y0)
        (veluPhiX E D.x0 D.y0 x)
        (veluPhiY E D.x0 D.y0 x y)
      = completedY E x y *
          (1 - veluT E D.x0 D.y0 / (x - D.x0) ^ 2) := by
  simp only [completedY, veluPhiX, veluPhiY,
    twoIsogenyQuotient_a₁, twoIsogenyQuotient_a₃]
  field_simp [hu]
  linear_combination veluT E D.x0 D.y0 * D.fixedByNeg

/-- Source equation translated to `u=x-x0`. -/
lemma source_local_equation
    (D : AffineTwoTorsionData E)
    {x y : ℚ} (hP : E.toAffine.Equation x y) :
    completedY E x y ^ 2 =
      4 * (x - D.x0) ^ 3
      + veluA E D.x0 * (x - D.x0) ^ 2
      + 4 * veluT E D.x0 D.y0 * (x - D.x0) := by
  rw [completedSquare_of_equation E hP]
  have hroot := twoTorsionCubic_at_point D
  have ht := four_mul_veluT D
  calc
    twoTorsionCubic E x
        = twoTorsionCubic E x - twoTorsionCubic E D.x0 := by
            rw [hroot]
            ring
    _ = 4 * (x - D.x0) ^ 3
        + veluA E D.x0 * (x - D.x0) ^ 2
        + (12 * D.x0 ^ 2 + 2 * E.b₂ * D.x0 + 2 * E.b₄)
            * (x - D.x0) := by
          simp only [twoTorsionCubic, veluA]
          ring
    _ = 4 * (x - D.x0) ^ 3
        + veluA E D.x0 * (x - D.x0) ^ 2
        + 4 * veluT E D.x0 D.y0 * (x - D.x0) := by
          rw [← ht]

/-- Target completed-square cubic after translating `X=x0+z`. -/
lemma quotient_cubic_translate
    (D : AffineTwoTorsionData E) (z : ℚ) :
    twoTorsionCubic (twoIsogenyQuotient E D.x0 D.y0) (D.x0 + z)
      = 4 * z ^ 3 + veluA E D.x0 * z ^ 2
          - 16 * veluT E D.x0 D.y0 * z
          - 4 * veluA E D.x0 * veluT E D.x0 D.y0 := by
  have hb4 := source_b₄_eq D
  have hb6 := source_b₆_eq D
  rw [hb4] at hb6
  simp only [twoTorsionCubic, quotient_b₂, quotient_b₄, quotient_b₆,
    hb4, hb6, veluA]
  ring

/-- The entire rational-function certificate. -/
lemma velu_local_residual
    (A t u s : ℚ) (hu : u ≠ 0) :
    u ^ 4 *
      ((s * (1 - t / u ^ 2)) ^ 2
        - (4 * (u + t / u) ^ 3
          + A * (u + t / u) ^ 2
          - 16 * t * (u + t / u)
          - 4 * A * t))
      = (u ^ 2 - t) ^ 2 *
          (s ^ 2 - (4 * u ^ 3 + A * u ^ 2 + 4 * t * u)) := by
  field_simp [hu]
  ring

/-- Generic affine points are sent to the quotient curve. -/
lemma veluPhi_equation
    (D : AffineTwoTorsionData E)
    {x y : ℚ}
    (hP : E.toAffine.Equation x y)
    (hu : x - D.x0 ≠ 0) :
    (twoIsogenyQuotient E D.x0 D.y0).toAffine.Equation
      (veluPhiX E D.x0 D.y0 x)
      (veluPhiY E D.x0 D.y0 x y) := by
  let u := x - D.x0
  let t := veluT E D.x0 D.y0
  let A := veluA E D.x0
  let s := completedY E x y
  have hsource : s ^ 2 = 4 * u ^ 3 + A * u ^ 2 + 4 * t * u := by
    simpa [u, t, A, s] using source_local_equation D hP
  have hres := velu_local_residual A t u s (by simpa [u] using hu)
  rw [hsource, sub_self, mul_zero] at hres
  have hu4 : u ^ 4 ≠ 0 := pow_ne_zero 4 (by simpa [u] using hu)
  have htargetLocal :
      (s * (1 - t / u ^ 2)) ^ 2 =
        4 * (u + t / u) ^ 3
          + A * (u + t / u) ^ 2
          - 16 * t * (u + t / u)
          - 4 * A * t := by
    apply sub_eq_zero.mp
    exact (mul_eq_zero.mp hres).resolve_left hu4
  apply equation_of_completedSquare
  rw [completedY_veluPhi D hu]
  rw [show veluPhiX E D.x0 D.y0 x =
      D.x0 + (u + t / u) by
        simp only [veluPhiX, u, t]
        field_simp [hu]
        ring]
  rw [quotient_cubic_translate D]
  simpa [u, t, A, s] using htargetLocal

/-! ----------------------------------------------------------------------
    Total point map and exact kernel
    ---------------------------------------------------------------------- -/

variable [E.IsElliptic]

/-- The total point map.  The point at infinity and the kernel point are
both sent to the point at infinity. -/
noncomputable def veluMapFn
    (D : AffineTwoTorsionData E) :
    E.toAffine.Point →
      (twoIsogenyQuotient E D.x0 D.y0).toAffine.Point
  | 0 => 0
  | .some x y h =>
      if hx : x = D.x0 then
        0
      else
        .mk (veluPhi_equation D h.1 (sub_ne_zero.mpr hx))

@[simp] lemma veluMapFn_zero
    (D : AffineTwoTorsionData E) :
    veluMapFn D 0 = 0 := rfl

@[simp] lemma veluMapFn_kernelPoint
    (D : AffineTwoTorsionData E) :
    veluMapFn D D.point = 0 := by
  simp [veluMapFn, AffineTwoTorsionData.point]

/-- There is only one affine point above `x=x0`, because the two roots of
the quadratic in `y` coincide at a two-torsion point. -/
lemma point_eq_kernelPoint_of_x_eq
    (D : AffineTwoTorsionData E)
    {y : ℚ} (h : E.toAffine.Equation D.x0 y) :
    (Affine.Point.mk h : E.toAffine.Point) = D.point := by
  have h0 := D.onCurve
  rw [WeierstrassCurve.Affine.equation_iff] at h h0
  have hdifference :
      y ^ 2 + E.a₁ * D.x0 * y + E.a₃ * y
        - (D.y0 ^ 2 + E.a₁ * D.x0 * D.y0 + E.a₃ * D.y0) = 0 := by
    linarith
  have hsquare : (y - D.y0) ^ 2 = 0 := by
    calc
      (y - D.y0) ^ 2
          = (y - D.y0) *
              (y + D.y0 + E.a₁ * D.x0 + E.a₃) := by
              congr 1
              linarith [D.fixedByNeg]
      _ = y ^ 2 + E.a₁ * D.x0 * y + E.a₃ * y
          - (D.y0 ^ 2 + E.a₁ * D.x0 * D.y0 + E.a₃ * D.y0) := by
              ring
      _ = 0 := hdifference
  have hy : y = D.y0 := by nlinarith
  subst y
  rfl

/-- Exact set-theoretic kernel of the total map. -/
lemma veluMapFn_eq_zero_iff
    (D : AffineTwoTorsionData E)
    (P : E.toAffine.Point) :
    veluMapFn D P = 0 ↔ P = 0 ∨ P = D.point := by
  rcases P with (_ | ⟨x, y, h⟩)
  · simp
  · by_cases hx : x = D.x0
    · subst x
      have hQ :
          (Affine.Point.mk h.1 : E.toAffine.Point) = D.point :=
        point_eq_kernelPoint_of_x_eq D h.1
      simp [veluMapFn, hQ]
    · have hne :
          (Affine.Point.mk
            (veluPhi_equation D h.1 (sub_ne_zero.mpr hx)) :
              (twoIsogenyQuotient E D.x0 D.y0).toAffine.Point) ≠ 0 :=
        Affine.Point.some_ne_zero _
      simp [veluMapFn, hx, hne]

/-! ----------------------------------------------------------------------
    Additivity seam
    ---------------------------------------------------------------------- -/

/--
`SORRY[MATHLIB-ISOGENY-API]`.

The statement is elementary but long.  At the pinned Mathlib revision there
is no bundled theorem that an origin-preserving morphism between elliptic
curves is a homomorphism.  Fill this by case-splitting on Mathlib's explicit
affine addition law:

* either input is `O`;
* either input is the kernel point;
* `R = -P`;
* generic secant branch;
* doubling branch.

After unfolding `Affine.Point.add`, every generic branch is
`field_simp` followed by the two source equations and `ring`.
-/
theorem veluMapFn_map_add
    (D : AffineTwoTorsionData E)
    (P R : E.toAffine.Point) :
    veluMapFn D (P + R) = veluMapFn D P + veluMapFn D R := by
  sorry

/-- The normalized Vélu isogeny as the additive map consumed downstream. -/
noncomputable def veluHom
    (D : AffineTwoTorsionData E) :
    E.toAffine.Point →+
      (twoIsogenyQuotient E D.x0 D.y0).toAffine.Point where
  toFun := veluMapFn D
  map_zero' := veluMapFn_zero D
  map_add' := veluMapFn_map_add D

@[simp] theorem veluHom_kernelPoint
    (D : AffineTwoTorsionData E) :
    veluHom D D.point = 0 :=
  veluMapFn_kernelPoint D

/-- Exact kernel in the form most consumers need. -/
theorem mem_ker_veluHom_iff
    (D : AffineTwoTorsionData E)
    (P : E.toAffine.Point) :
    P ∈ (veluHom D).ker ↔ P = 0 ∨ P = D.point := by
  change veluMapFn D P = 0 ↔ _
  exact veluMapFn_eq_zero_iff D P

/-! ----------------------------------------------------------------------
    Optional projective adapter
    ---------------------------------------------------------------------- -/

/-- Transport the affine map to Mathlib's projective point group. -/
noncomputable def veluHomProjective
    (D : AffineTwoTorsionData E) :
    E.toProjective.Point →+
      (twoIsogenyQuotient E D.x0 D.y0).toProjective.Point :=
  (WeierstrassCurve.Projective.Point.toAffineAddEquiv
      (twoIsogenyQuotient E D.x0 D.y0)).symm.toAddMonoidHom.comp
    ((veluHom D).comp
      (WeierstrassCurve.Projective.Point.toAffineAddEquiv E).toAddMonoidHom)

/-! ----------------------------------------------------------------------
    Strong replacement structure for the old axiom
    ---------------------------------------------------------------------- -/

structure RationalTwoIsogenyData
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (Q : E.toAffine.Point) where
  E' : WeierstrassCurve ℚ
  elliptic_E' : E'.IsElliptic
  phi : E.toAffine.Point →+ E'.toAffine.Point
  phi_Q : phi Q = 0
  ker_phi : ∀ P, phi P = 0 ↔ P = 0 ∨ P = Q

/-- Affine-coordinate endpoint. -/
noncomputable def rationalTwoIsogenyDataOfAffine
    (D : AffineTwoTorsionData E) :
    RationalTwoIsogenyData E D.point where
  E' := twoIsogenyQuotient E D.x0 D.y0
  elliptic_E' := inferInstance
  phi := veluHom D
  phi_Q := veluHom_kernelPoint D
  ker_phi := fun P => by
    change veluMapFn D P = 0 ↔ _
    exact veluMapFn_eq_zero_iff D P

end

end RationalTwoIsogeny
```

# 4. Extracting affine data from `addOrderOf Q = 2`

If the old axiom takes a projective point, use Mathlib's additive equivalence

```lean
WeierstrassCurve.Projective.Point.toAffineAddEquiv E
```

and prove the following small adapter once.

```lean
namespace RationalTwoIsogeny

noncomputable section

open WeierstrassCurve

/-- Routine point-API adapter: a projective point of order two has affine
coordinates satisfying the fixed-by-negation equation. -/
theorem affineTwoTorsionData_of_projective
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    {Q : E.toProjective.Point}
    (hQ : addOrderOf Q = 2) :
    ∃ D : AffineTwoTorsionData E,
      (WeierstrassCurve.Projective.Point.toAffineAddEquiv E) Q = D.point := by
  let e := WeierstrassCurve.Projective.Point.toAffineAddEquiv E
  let Qa : E.toAffine.Point := e Q
  have horder : addOrderOf Qa = 2 := by
    simpa [Qa, e] using
      addOrderOf_injective e.toAddMonoidHom e.injective Q
  have hQa : Qa ≠ 0 := by
    intro h0
    rw [h0, AddMonoid.addOrderOf_eq_one_iff.mpr rfl] at horder
    omega
  rcases hQaDef : Qa with (_ | ⟨x0, y0, hns⟩)
  · exact (hQa hQaDef).elim
  · have htwo : 2 • Qa = 0 := by
      have hz := addOrderOf_nsmul_eq_zero Qa
      simpa [horder] using hz
    have hneg : Qa = -Qa := by
      apply eq_neg_of_add_eq_zero_left
      simpa [two_nsmul] using htwo
    have hfixed : 2 * y0 + E.a₁ * x0 + E.a₃ = 0 := by
      -- After rewriting `Qa` by `hQaDef`, `Affine.Point.neg_some`
      -- turns `hneg` into `y0 = -y0-a1*x0-a3`.
      rw [hQaDef] at hneg
      simpa [WeierstrassCurve.Affine.Point.neg_some,
        WeierstrassCurve.Affine.negY] using
        congrArg (fun P => (E.toAffine.pointEquiv P)) hneg
    let D : AffineTwoTorsionData E :=
      { x0 := x0
        y0 := y0
        onCurve := hns.1
        fixedByNeg := hfixed }
    refine ⟨D, ?_⟩
    simpa [D, AffineTwoTorsionData.point] using hQaDef

end

end RationalTwoIsogeny
```

The exact `simp` line extracting `hfixed` may need a minor adjustment to the project's chosen affine/projective abbreviations, but it contains no mathematical gap.  An alternative is to prove a helper lemma directly from `P = -P` for `Affine.Point.some`.

# 5. Final theorem replacing the axiom

For an affine-point consumer, the final result is:

```lean
namespace RationalTwoIsogeny

noncomputable section

open WeierstrassCurve

/-- Axiom-free explicit degree-two Vélu quotient. -/
theorem exists_rational_two_isogeny_quotient_affine
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (D : AffineTwoTorsionData E) :
    ∃ (E' : WeierstrassCurve ℚ)
      (_ : E'.IsElliptic)
      (phi : E.toAffine.Point →+ E'.toAffine.Point),
      phi D.point = 0 ∧
      (∀ P, phi P = 0 ↔ P = 0 ∨ P = D.point) := by
  letI : (twoIsogenyQuotient E D.x0 D.y0).IsElliptic :=
    quotient_isElliptic D
  refine ⟨twoIsogenyQuotient E D.x0 D.y0,
    inferInstance, veluHom D, ?_, ?_⟩
  · exact veluHom_kernelPoint D
  · exact fun P => (mem_ker_veluHom_iff D P)

end

end RationalTwoIsogeny
```

For the projective signature displayed in the question:

```lean

theorem exists_rational_two_isogeny_quotient

    {Q : E.toProjective.Point}
    (hQ : addOrderOf Q = 2) :
    ∃ (E' : WeierstrassCurve ℚ)
      (_ : E'.IsElliptic)
      (phi : E.toProjective.Point →+ E'.toProjective.Point),
      phi Q = 0 := by
  obtain ⟨D, hDQ⟩ := affineTwoTorsionData_of_projective E hQ
```

# 6. Why this is genuinely a two-isogeny

Beyond the exact two-element kernel, the function-field degree is visible immediately.  Put

\[
u=x-x_0.
\]

From

\[
X=x_0+u+\frac{\tau}{u}
\]

one obtains

\[
\boxed{u^2-(X-x_0)u+\tau=0.}
\]

Thus the source function field has degree at most two over the image.  Since `tau != 0`, the map is nonconstant; since its geometric kernel is exactly `{O,Q}`, its degree is two in characteristic zero.

It is also normalized.  In completed-square coordinates,

\[
\frac{dX}{2Y+a_1X+a_3}
=
\frac{(1-\tau/u^2)\,dx}
     {(2y+a_1x+a_3)(1-\tau/u^2)}
=
\frac{dx}{2y+a_1x+a_3}.
\]

No analytic, rank, descent, or finite-generation input occurs anywhere in this construction.

# 7. Implementation status by obligation

| Obligation | Lean proof style |
|---|---|
| quotient coefficients | definitional |
| completed-square equation | `simp; ring` |
| order-two cubic root | completed-square identity + `fixedByNeg` |
| `4*t = derivative` | `linear_combination` |
| `Delta(E)=t^2*D` | substitute `b4,b6,b8`; `field_simp; ring` |
| `Delta(E')=t*D^2` | target `b`-invariants; `field_simp; ring` |
| `E'.IsElliptic` | nonzero factors from source discriminant |
| generic map preserves curve | `(CERT)` by `field_simp; ring` |
| `phi(Q)=O` | definitional branch |
| exact kernel | uniqueness of the affine point above `x0` |
| projective wrapper | existing `toAffineAddEquiv` |
| `map_add` | finite explicit group-law case split; only marked API seam |

The old geometric axiom can therefore be reduced to one explicit, local engineering lemma—`veluMapFn_map_add`—rather than an abstract quotient-of-elliptic-curves development.
