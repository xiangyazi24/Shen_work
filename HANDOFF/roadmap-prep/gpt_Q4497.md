# Paper 1 Rothe step+tail continuous-dependence: paste-ready target

## Executive verdict

The hard block is **not** a single Aubin-Lions/Minty lemma. For the actual repo architecture, the correct block is the whole-line **Green-kernel fixed-point closed-graph package**:

1. **Per-step choice stability**: fixed finite Rothe step `k` depends continuously on the frozen profile `u`, despite the current `rotheSeqOf` being built with `Classical.choose`.
2. **Uniform `k → ∞` tail**: the convergence of the monotone Rothe sequence to `rotheLimit` must be uniform over all trapped frozen profiles, not merely per profile.
3. **Spatial tails / max-principle tails**: the Green-produced steps must have the correct two-sided tails at `±∞` so max-principle comparisons and whole-line compactness do not leak mass at infinity.

The repo already isolates this accurately in `WaveRotheDep.lean`: `RotheSeqStepDependence` + `RotheTailUniform` imply the closed-graph/continuity theorem `rotheLimit_dep_of_step_and_tail`, and then `rotheContinuousDependence`. The remaining genuinely analytic work is to **produce** those hypotheses, plus any still-unclosed per-step floor fields feeding `RotheStepProducer`/`RotheStepFloor`.

## 1. Exact TAIL statements

There are two different “tail” notions. Do not conflate them.

### 1A. Spatial whole-line tails, used by the max principle and compactness

At the per-step max-principle level, the exact in-repo interface is not simply `u^τ(t,x) → 0` as `|x| → ∞`. It is the **relative tail** of a comparison function:

```lean
structure RotheStepTails (W B : ℝ → ℝ) where
  φcont : Continuous (fun x => W x - B x)
  La : ℝ
  Lb : ℝ
  hbot : Tendsto (fun x => W x - B x) atBot (𝓝 La)
  hLa : La ≤ 0
  htop : Tendsto (fun x => W x - B x) atTop (𝓝 Lb)
  hLb : Lb ≤ 0
```

This is the right max-principle tail: it says `W - B` cannot have a positive maximum escaping to `±∞`.

For absolute spatial decay of the orbit, the correct statement depends on the chosen wave asymptotes. In the barrier file, `upperBarrier κ x = min 1 (exp (-κx))`; it tends to `0` as `x → +∞` and is eventually `1` as `x → -∞`. So the literal statement

```lean
∀ ε > 0, ∃ R, ∀ |x| ≥ R, |u^τ(t,x)| ≤ ε
```

is generally **too strong** for a front profile unless the left asymptote is also zero. The robust version is asymptote-aware:

```lean
/-- Spatial tightness/asymptote control for the Rothe family.  Use this only if
    the compactness topology is global, not merely local-uniform. -/
def RotheOrbitSpatialTailUniform
    (p : CMParams) (c lam M κ Λ : ℝ)
    (hprodTrap : ∀ v, InMonotoneWaveTrapSet κ M v →
      RotheStepProducer p c lam M κ Λ v)
    (hκ : 0 ≤ κ) (hM : 0 ≤ M) : Prop :=
  ∀ ε > 0, ∃ R > 0,
    ∀ v : ℝ → ℝ, (hv : InMonotoneWaveTrapSet κ M v) →
      ∀ k : ℕ,
        -- right tail: the front decays to zero
        (∀ x, R ≤ x →
          |rotheSeqOf p c lam M κ Λ v (hprodTrap v hv) hκ hM k x| < ε) ∧
        -- left tail: replace `leftAsymptote` by the actual trap asymptote,
        -- e.g. `M` or `1` depending on the normalized Paper 1 interface.
        (∀ x, x ≤ -R →
          |rotheSeqOf p c lam M κ Λ v (hprodTrap v hv) hκ hM k x - leftAsymptote| < ε)
```

What gives this analytically: the **frozen-profile supersolution barrier** and Green-kernel representation. On the right, `W ≤ upperBarrier` plus `upperBarrier → 0` gives the zero tail. On the left, use the corresponding left asymptote of the barrier/trap; do not assert zero unless the trap is actually zero there. For max-principle use, `RotheStepTails W B` is the lighter and already-shaped object.

The repo also has `GreenConvTails.lean`, which is exactly the landed route for turning bounded-source Green limits into `RotheStepTails`: `greenConv_tendsto_atTop_of_source_tendsto`, `rotheStepTails_of_limits`, and `rotheStepTails_greenConv_of_barrier_limits`.

### 1B. Uniform `k`-tail for the Rothe limit map

This is the true tail needed for **continuous dependence of the limit map**. The exact in-repo statement is:

```lean
def RotheTailUniform
    (p : CMParams) (c lam M κ Λ : ℝ)
    (hprodTrap : ∀ v, InMonotoneWaveTrapSet κ M v →
      RotheStepProducer p c lam M κ Λ v)
    (hκ : 0 ≤ κ) (hM : 0 ≤ M) : Prop :=
  ∀ R > 0, ∀ ε > 0,
    ∃ K : ℕ, ∀ v : ℝ → ℝ, (hv : InMonotoneWaveTrapSet κ M v) →
      ∀ k : ℕ, K ≤ k → ∀ x ∈ Set.Icc (-R) R,
        |rotheSeqOf p c lam M κ Λ v (hprodTrap v hv) hκ hM k x
            - rotheLimit
              (rotheSeqOf p c lam M κ Λ v (hprodTrap v hv) hκ hM) x| < ε
```

This is a **uniform Dini tail over the whole trapped family**. It is stronger than the already-available per-profile locally-uniform convergence of `rotheSeqOf v k → rotheLimit ...`. It is the exact missing uniformity needed to commute:

```text
seq n → u, then k→∞
```

without choosing `K` depending on `n`.

Analytic route to produce it: monotone-in-`k` descent, equi-Lipschitz/equi-bounded on each compact interval, and Dini/Arzelà-style compactness on the trapped family. If the topology is strengthened to a weighted/global sup topology, combine this compact-window uniformity with the spatial tail in 1A.

## 2. Exact STEP continuous-dependence statement

The precise in-repo statement is:

```lean
def RotheSeqStepDependence
    (p : CMParams) (c lam M κ Λ : ℝ)
    (hprodTrap : ∀ v, InMonotoneWaveTrapSet κ M v →
      RotheStepProducer p c lam M κ Λ v)
    (hκ : 0 ≤ κ) (hM : 0 ≤ M) : Prop :=
  ∀ (seq : ℕ → ℝ → ℝ) (u : ℝ → ℝ),
    (hseq : ∀ n, InMonotoneWaveTrapSet κ M (seq n)) →
      (hu : InMonotoneWaveTrapSet κ M u) →
      LocallyUniformConverges seq u →
        ∀ k : ℕ,
          LocallyUniformConverges
            (fun n => rotheSeqOf p c lam M κ Λ (seq n)
              (hprodTrap (seq n) (hseq n)) hκ hM k)
            (rotheSeqOf p c lam M κ Λ u
              (hprodTrap u hu) hκ hM k)
```

The proof mechanism should be an induction on `k`.

Base `k=0`: all profiles start from the same `upperBarrier κ M`, so convergence is trivial.

Step `k → k+1`: prove convergence of the fixed point of the step map. The load-bearing analytic lemma is:

```lean
/-- Canonical implicit-step stability, before taking the `k`-limit. -/
theorem crossImplicitStep_continuous_in_frozen_profile
    -- schematic inputs
    (hZn : LocallyUniformConverges Zn Z)
    (hun : LocallyUniformConverges un u)
    (hVn : LocallyUniformConverges
      (fun n => deriv (frozenElliptic p (un n)))
      (deriv (frozenElliptic p u)))
    (htrap : uniform_trap_bounds)
    (hcontract : uniform_contraction_constant_lt_one) :
    LocallyUniformConverges Wn W := ...
```

where `Wn` and `W` are the unique fixed points of the corresponding per-step maps.

This is exactly where `frozenEllipticDerivDependence` enters: the only dependence on the frozen profile `u` in the per-step Green map is through `V'_u = deriv (frozenElliptic p u)` in the chemotaxis flux. The reaction and `λ Z` terms do not depend on the frozen profile except through `Z`.

The key issue is **not** ordinary dominated convergence. It is the `Classical.choose` issue named in `WaveRotheDep.lean`: `rotheSeqOf (seq n) k` and `rotheSeqOf u k` are independently chosen from producer witnesses. So the proof must use **uniqueness of the fixed point** (from `crossStep_exists_unique_concrete`) or refactor the producer so `rotheSeqOf` is definitionally the canonical unique fixed point. Without uniqueness/canonicalization, continuity of the set-valued existence statement is not enough.

## 3. Combined closed-graph / continuity statement

Once `RotheSeqStepDependence` and `RotheTailUniform` are supplied, the repo already has the core epsilon/3 theorem:

```lean
theorem rotheLimit_dep_of_step_and_tail
    {hprodTrap : ∀ v, InMonotoneWaveTrapSet κ M v →
      RotheStepProducer p c lam M κ Λ v} {hκ : 0 ≤ κ} {hM : 0 ≤ M}
    (hstep : RotheSeqStepDependence p c lam M κ Λ hprodTrap hκ hM)
    (htail : RotheTailUniform p c lam M κ Λ hprodTrap hκ hM)
    (seq : ℕ → ℝ → ℝ) (u : ℝ → ℝ)
    (hseq : ∀ n, InMonotoneWaveTrapSet κ M (seq n))
    (hu : InMonotoneWaveTrapSet κ M u)
    (hconv : LocallyUniformConverges seq u) :
    LocallyUniformConverges
      (fun n =>
        rotheLimit ((rotheSeqFromTrap p c lam M κ Λ hprodTrap hκ hM) (seq n)))
      (rotheLimit ((rotheSeqFromTrap p c lam M κ Λ hprodTrap hκ hM) u))
```

and the exact final dependency field:

```lean
theorem rotheContinuousDependence
```

## Genuine analytic input vs mechanical wiring

**Genuine remaining analytic input:**

1. `RotheSeqStepDependence`: prove chosen finite-step orbit continuity from fixed-point uniqueness + `frozenEllipticDerivDependence` + dominated convergence of the Green map + uniform contraction.
2. `RotheTailUniform`: prove uniform-in-profile `k`-tail convergence of `rotheSeqOf v k → rotheLimit ...` on compact windows, using monotone descent + equicontinuity/equi-boundedness, i.e. Dini/Arzelà-style compactness.
3. If `hprodAll` is not yet closed, the per-step floor still has honest residual packets: `RotheStepFluxData`, `RotheStepChemData`, and `RotheStepTails`/`RotheStepFloor`. The Green tail part is now well-shaped by `GreenConvTails.lean`, but flux/chem data may still require field-by-field production.

**Mechanical after those inputs:**

- `RotheSeqStepDependence + RotheTailUniform` → `rotheLimit_dep_of_step_and_tail`.
- `rotheLimit_dep_of_step_and_tail` → `rotheContinuousDependence`.
- `rotheContinuousDependence` supplies the `hdep`/continuous-map field for the Schauder/Rothe fixed-point construction.
- Per-step floor data supplies `RotheStepProducer`, hence `hprodTrap/hprodAll`.

Bottom line: the paste-ready target for the hardest Paper 1 block is:

```lean
-- Real analytic production targets:
hstep : RotheSeqStepDependence p c lam M κ Λ hprodTrap hκ hM
htail : RotheTailUniform p c lam M κ Λ hprodTrap hκ hM

-- Already-built assembly target:

```

Do **not** try to close the block by Minty monotonicity. Discrete Gronwall may help prove finite-step stability, but it does not by itself solve the `Classical.choose`/unique fixed-point continuity problem or the uniform `k`-tail problem. The correct route is Green-kernel fixed-point stability + uniform Dini tail + asymptote-aware spatial tightness.
