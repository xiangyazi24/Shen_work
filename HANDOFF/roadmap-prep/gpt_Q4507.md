
That is a producer for every frozen profile `u`. Its `.producer` field is then fed to the concrete paper Rothe sequence:

```lean

```

and the sequence becomes:

```lean
rotheSeqOfPaperFromCond p c lam M κ κtilde Λ hcond
  (fun u => (hprodAll u).producer)
```

This is the exact `hprodAll` object that older notes call the per-step Rothe producer.

At the lower level, the ordinary Rothe producer interface is a step producer from a trapped continuous antitone supersolution `Z` to a next iterate `W` with the full `RotheStepFacts` bundle:

```lean
structure RotheStepProducer (p : CMParams) (c lam M κ Λ : ℝ) (u : ℝ → ℝ) : Prop where
  baseSuper : ∀ x, frozenWaveOperator p c u (upperBarrier κ M) x ≤ 0
  produce : ∀ Z : ℝ → ℝ, Continuous Z → Antitone Z → (∀ x, 0 ≤ Z x) →
      (∀ x, Z x ≤ upperBarrier κ M x) →
      (∀ x, frozenWaveOperator p c u Z x ≤ 0) →
      ∃ W : ℝ → ℝ, RotheStepFacts p c lam M κ Λ u Z W
```

and `RotheStepFacts` includes:

```lean
step_eq   : W = crossImplicitMap p c lam u Z W
cont      : Continuous W
diff      : Differentiable ℝ W
deriv_le  : ∀ x, |deriv W x| ≤ Λ
nonneg    : ∀ x, 0 ≤ W x
le_barrier: ∀ x, W x ≤ upperBarrier κ M x
le_old    : ∀ x, W x ≤ Z x
anti      : Antitone W
supersol  : ∀ x, frozenWaveOperator p c u W x ≤ 0
```

So `hprodAll` is not merely “existence of a step”; it must produce a whole trapped, monotone, differentiable implicit-Euler step with the supersolution invariant.
