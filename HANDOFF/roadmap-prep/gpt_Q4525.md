# N18 Block 2: the two infinity orders and the oriented-class bridge

## Executive verdict

The clean project-local solution has two layers.

1. Semantic layer: construct the two embeddings

1. Computational layer: prove a closed polynomial-degree formula for the value of either order on every numerator (a(x)+b(x)y). All downstream Mumford-reduction and uniqueness proofs use only this formula, polynomial division, and ideal equalities; they never unfold power series.

There is one important correction to the naive degree rule:

If the degree-(d) terms of (a(x)) and (pm b(x)x^3) cancel, the order is not determined by deg a and deg b alone. The exact correction is the degree of the norm

[

N(a,b)=a^2-b^2f.

]

For a nonzero numerator, this gives a completely exact formula and handles arbitrarily deep cancellation.

No Riemann–Roch theorem, general divisor API, Picard scheme, or general theory of places is required. Current pinned Mathlib already supplies the relevant AdjoinRoot, power-series order, Laurent-series valuation, localization, fractional-ideal, and quotient-group APIs. The only genuinely new infrastructure is a small formal square-root construction in K⟦t⟧, the two explicit embeddings, and the bridge lemma that derives the second infinity equation from the first one plus the affine ideal equality.

## 2.1 Generic pole degree and branch-leading coefficients

Let

[

d=maxbigl(deg a,deg b+3bigr),

]

with the convention (deg 0=-infty). Define

[

lambda_+(a,b)= [x^d]a+[x^{d-3}]b,

qquad

lambda_-(a,b)= [x^d]a-[x^{d-3}]b.

]

At least one of (lambda_+,lambda_-) is nonzero. If both vanished, adding and subtracting would force the relevant leading coefficients of both (a) and (b) to vanish; characteristic zero is used only to divide by (2).

A Nat-valued Lean implementation that handles zero polynomials without WithBot bookkeeping is:

```javascript
inductive InfinityBranch
  | plus
  | minus

def shiftedYDegree (b : K[X]) : ℕ :=
  if b = 0 then 0 else b.natDegree + 3

def poleDegree (a b : K[X]) : ℕ :=
  max a.natDegree (shiftedYDegree b)

def branchLead : InfinityBranch → K[X] → K[X] → K
  | .plus,  a, b =>
      let d := poleDegree a b
      a.coeff d + b.coeff (d - 3)
  | .minus, a, b =>
      let d := poleDegree a b
      a.coeff d - b.coeff (d - 3)

def normNumerator (a b : K[X]) : K[X] :=
  a^2 - b^2 * n18F K
```

The expression b.coeff (d - 3) is harmless when d < 3: in that case b = 0, so the coefficient is zero.

## 2.3 Proof of the formula

Scale (a+by) by (t^d), with (t=1/x). Its constant coefficient on the (arepsilon)-branch is precisely (lambda_varepsilon(a,b)).

- If that coefficient is nonzero, the scaled Laurent series has order (0), so the original function has order (-d).

- If it vanishes, the opposite branch does not vanish in degree (-d), hence its order is (-d). Use

## 3.2 The only new generic power-series lemma

Current pinned Mathlib does not expose a ready-made square-root/Hensel theorem specialized to a unit power series with constant coefficient (1). Add one project-local theorem:

```javascript
theorem existsUnique_sq_eq_of_constantCoeff_one
    (F : K⟦X⟧)
    (hF0 : F.constantCoeff = 1) :
    ∃! W : K⟦X⟧,
      W^2 = F ∧ W.constantCoeff = 1
```

It is elementary and constructive. Define coefficients recursively:

[

w_0=1,

]

and for (n>0),

[

w_n=frac{1}{2}

left(F_n-sum_{i=1}^{n-1}w_iw_{n-i}right).

]

A Lean-shaped definition is:

```javascript
noncomputable def sqrtCoeff
    (F : K⟦X⟧) : ℕ → K :=
  fun n => Nat.strongRecOn n fun n ih =>
    if hn : n = 0 then 1 else
      ((PowerSeries.coeff n F) -
        ∑ i in Finset.Icc 1 (n - 1),
          ih i (by omega) * ih (n - i) (by omega)) / 2

noncomputable def sqrtOne (F : K⟦X⟧) : K⟦X⟧ :=
  PowerSeries.mk (sqrtCoeff K F)
```

The proof of sqrtOne ^ 2 = F is coefficientwise: expand PowerSeries.coeff_mul, split the two endpoint terms of Finset.antidiagonal n, and use the recursive equation. Uniqueness is induction on the coefficient index; the coefficient of degree (n) in a square is (2w_n) plus terms already known from lower indices.

This generic helper is approximately 150–250 Lean lines and is reusable for every split unramified branch of this form.

Define

## 3.3 Affine ring and rank-two coefficients

Use


```javascript

  AdjoinRoot.modByMonicHom (curvePoly_monic K)

lemma recompose (z : N18AffineRing K) :
```

## 3.4 Two embeddings into Laurent series

Let

```javascript
abbrev LS := LaurentSeries K

def tInv : LS K := HahnSeries.single (-1) 1

def yPlusSeries : LS K :=
  HahnSeries.single (-3) 1 * (wPlus K : LS K)

def yMinusSeries : LS K :=
  HahnSeries.single (-3) 1 * (wMinus K : LS K)
```

Evaluate coefficient polynomials at (t^{-1}):

```javascript
noncomputable def polyAtInfinity : K[X] →+* LS K :=
  Polynomial.eval₂RingHom (algebraMap K (LS K)) (tInv K)
```

The relation (y^2=f(x)) follows from wPlus_sq or wMinus_sq after multiplying the reversed equation by (t^{-6}). Therefore:

```javascript
noncomputable def affineAtPlus :
    N18AffineRing K →+* LS K :=
  AdjoinRoot.lift (polyAtInfinity K) (yPlusSeries K)
    (by simp [curvePoly, yPlusSeries, polyAtInfinity, wPlus_sq]; ring)

noncomputable def affineAtMinus :
    N18AffineRing K →+* LS K :=
  AdjoinRoot.lift (polyAtInfinity K) (yMinusSeries K)
    (by simp [curvePoly, yMinusSeries, polyAtInfinity, wMinus_sq]; ring)
```

Both maps are injective. If the image of (z=a+by) were zero, then the image of

[

zbar z=a^2-b^2f

]

would be zero. Evaluation of a nonzero polynomial at (t^{-1}) is injective because its lowest Laurent exponent is the negative of its degree and has nonzero leading coefficient. Thus the norm polynomial is zero, and the domain property of the affine ring forces (z=0).

Extend the maps to the fraction field by the localization universal property:

```javascript
noncomputable def fieldAtPlus :
    N18FunctionField K →+* LS K :=
  IsLocalization.lift
    (affineAtPlus K)
    (fun s => isUnit_iff_ne_zero.mpr
      ((affineAtPlus_injective K) s.property))

noncomputable def fieldAtMinus :
    N18FunctionField K →+* LS K :=
  IsLocalization.lift
    (affineAtMinus K)
    (fun s => isUnit_iff_ne_zero.mpr
      ((affineAtMinus_injective K) s.property))
```

The exact argument order of IsLocalization.lift may require a small adapter around the chosen FractionRing; no new mathematics is involved.

# 4. The polynomial part (A_infty) and balancing

The identity

[

f-A_infty^2=8x

]

implies

[

(y-A_infty)(y+A_infty)=8x.

]

At (infty_+), (y+A_infty) has leading term (2x^3), hence order (-3). Therefore

[

v_+(y-A_infty)=v_+(8x)-v_+(y+A_infty)=-1-(-3)=2.

]

At (infty_-), the roles reverse. Thus

[

boxed{

begin{array}{c|cc}

end{array}}

]

These four values are immediate instances of ord_num_formula:

```javascript
@[simp] theorem ordPlus_y_sub_AInf :
    ordPlusZ K (unitOf (y - AInf)) = 2

@[simp] theorem ordMinus_y_sub_AInf :
    ordMinusZ K (unitOf (y - AInf)) = -3

@[simp] theorem ordPlus_y_add_AInf :
    ordPlusZ K (unitOf (y + AInf)) = -3

@[simp] theorem ordMinus_y_add_AInf :
    ordMinusZ K (unitOf (y + AInf)) = 2
```

Their finite zeros are

[

P_-=(0,-1),qquad P_+=(0,1).

]

Consequently

[

operatorname{div}(y-A_infty)=P_-+2infty_+-3infty_-,

]

[

operatorname{div}(y+A_infty)=P_+-3infty_++2infty_-.

]

With the divisor convention used by the corrected Mumford type, these are the zero-class semi-representatives

```plain text
(X, -1,  3)
(X,  1, -2)
```

respectively. They are the two fixed “baby steps” that move an out-of-range infinity counter back toward

[

0le n_inftyle2-deg u.

The degree-three reduction branch also uses the sign choice (y-A_infty) or (y+A_infty): replace (v) by (v+c,u) so its cubic leading coefficient is (+1) or (-1), causing the degree-six term of (f-(v+cu)^2) to cancel. The complement polynomial then has degree at most two.

g(y-v_1)=frac{a(y-v_1)}{u_1}in I(D_2)subset A.

]

The (y)-coefficient of this element is (a/u_1). Every integral affine-ring element has polynomial (y)-coefficient, so

[

frac{a}{u_1}in K[x],

qquad

u_1mid a.

]

```javascript
theorem dvd_of_div_yCoeff_integral
    {a u v : K[X]} (hu : u ≠ 0)
    (h :
      ((algebraMap _ _ a / algebraMap _ _ u) *
          (y K - algebraMap _ _ v))
        ∈ algebraMap (N18AffineRing K) (N18FunctionField K) '' Set.univ) :

```

After writing (a=u_1h),

[

boxed{g=h(x),qquad hin K[x]^times.}

]

## 6.4 The polynomial (h) is constant

Let (k=deg h). Since a nonzero polynomial has the same order at both infinities,

[

v_+(g)=v_-(g)=-k.

]

The bridge equations give

[

n_2=n_1-k,

qquad

m_2=m_1-k.

]

Hence (kle n_1) and (kle m_1), so

[

2kle n_1+m_1=2-r_1.

]

If (k>0), necessarily

[

k=1,qquad r_1=0,qquad n_1=m_1=1.

]

Thus (D_1) is the identity representative, (I(D_1)=top), while

[

n_2=m_2=0,qquad r_2=2.

]

The affine ideal equality becomes

[

I(D_2)=(h).

]

Comap both sides to (K[x]):

[

(u_2)=(h).

]

But (u_2) is monic of degree (2) and (h) has degree (1), impossible. Therefore (k=0), so (hin K^times).

This argument is entirely elementary; the exceptional nonconstant function (x-c) is exactly the hyperelliptic-fiber case, and the comap-degree contradiction excludes it from the semireduced balanced representatives.

## 6.5 Recover (u,v,n_infty)

A nonzero constant is a unit of the affine ring, so the principal ideal factor disappears and

[

I(D_1)=I(D_2).

]

Use the standard kernel/evaluation facts:

```javascript
theorem mumfordIdeal_comap_base (D : BalancedMumford K) :
    (mumfordIdeal K D).comap
        (algebraMap K[X] (N18AffineRing K)) =
      Ideal.span {D.u}
```

Therefore (u₁)=(u₂). Both are monic, hence (u_1=u_2).

With the common (u), equality of the two graph ideals gives

[

umid v_1-v_2.

]

Both (v_i) are canonical remainders modulo the monic (u), so (v_1=v_2). Finally plus_eq with (k=0) yields (n_1=n_2).

The finished theorem is:

```javascript
theorem reduced_unique_of_principal
    {D₁ D₂ : BalancedMumford K}
    (B : PrincipalBetween K D₁ D₂) :
    D₁ = D₂ := by
  obtain ⟨a, b, hrep⟩ :=
    numerator_over_u_of_principal K B

  have hnum_ne : a ≠ 0 ∨ b ≠ 0 := by
    -- Otherwise `B.g = 0`, impossible for a unit.
    ...

  let q : NonzeroQuadNum K := ⟨a, b, hnum_ne⟩

  have hp : -2 ≤ ordNum K .plus q := by
    rw [← ord_fraction_formula K .plus D₁.u_monic.ne_zero hnum_ne,
      hrep, B.plus_eq]
    omega

  have hm : -2 ≤ ordNum K .minus q := by
    rw [← ord_fraction_formula K .minus D₁.u_monic.ne_zero hnum_ne,
      hrep, B.minus_eq]
    omega

  have hd : poleDegree a b ≤ 2 := by
    have hmin := min_ord_eq_neg_poleDegree K q
    omega

  have hb : b = 0 := by
    by_contra hb0
    have : 3 ≤ poleDegree a b := by
      simp [poleDegree, shiftedYDegree, hb0]
    omega

  subst b

  obtain ⟨h, ha⟩ : ∃ h : K[X], a = D₁.u * h := by
    exact dvd_of_div_yCoeff_integral K
      D₁.u_monic.ne_zero
      (principal_times_y_sub_mem K B hrep)

  have hg : B.g = polynomialUnit K h := by
    -- Substitute `a = u₁*h` into `hrep` and cancel `u₁`.
    ...

  have hk_bounds :
      h.natDegree ≤ D₁.nInf ∧
      h.natDegree ≤ D₁.nMinus := by
    have hp' := B.plus_eq
    have hm' := B.minus_eq
    simp [hg, ordPlus_polynomial, ordMinus_polynomial] at hp' hm'
    omega

    by_contra hk
    have hkpos : 0 < h.natDegree := Nat.pos_of_ne_zero hk
    have hcase :
        h.natDegree = 1 ∧
        D₁.u.natDegree = 0 ∧
        D₁.nInf = 1 ∧
        D₁.nMinus = 1 := by
      have hbal := D₁.degree_balance
      omega
    rcases hcase with ⟨hk1, hr1, hn1, hm1⟩
    have hr2 : D₂.u.natDegree = 2 := by
      have hp' := B.plus_eq
      have hm' := B.minus_eq
      have hbal₂ := D₂.degree_balance
      simp [hg, ordPlus_polynomial, ordMinus_polynomial,
        hk1, hn1, hm1] at hp' hm'
      omega
    have hspan := comap_principal_contradiction K B hg hr1
    have := natDegree_eq_of_span_singleton_eq hspan
    omega

  have hconst : IsUnit (algebraMap K[X] (N18AffineRing K) h) := by
    obtain ⟨c, rfl⟩ := Polynomial.eq_C_of_natDegree_eq_zero hk0
    -- `h ≠ 0` follows from `B.g` being a unit.
    simpa using isUnit_iff_ne_zero.mpr ...

  have hideal : mumfordIdeal K D₁ = mumfordIdeal K D₂ := by
    -- Cancel the constant principal ideal from `B.ideal_eq`.
    ...

  have hu : D₁.u = D₂.u := by
    apply Polynomial.eq_of_monic_of_associated
      D₁.u_monic D₂.u_monic
    exact associated_of_span_singleton_eq <|
      congrArg (Ideal.comap (algebraMap K[X] (N18AffineRing K))) hideal

  have hv : D₁.v = D₂.v := by
    apply modByMonic_unique D₁.u_monic
    · exact D₁.v_mod
    · simpa [hu] using D₂.v_mod
    · exact v_sub_v_dvd_of_mumfordIdeal_eq K hideal hu

  have hn : D₁.nInf = D₂.nInf := by
    have := B.plus_eq
    simpa [hg, hk0, ordPlus_polynomial] using this.symm

  cases D₁
  cases D₂
  simp_all
```

# Bottom line

The two infinity valuations can be implemented without a general divisor or Riemann–Roch theory. Define them semantically by the two Laurent-series branches, then expose the exact computational rule

[

v_varepsilon(a+by)=

begin{cases}

-d,&lambda_varepsilonne0,\

d-deg(a^2-b^2f),&lambda_varepsilon=0.

end{cases}

]

The oriented-class quotient stores only (v_+). The affine ideal equality, conjugation, and norm recover

[

v_+(g)+v_-(g)=deg u_1-deg u_2,

]

which supplies the missing (v_-) equation. Those two inequalities force the numerator of a principal relation between reduced balanced triples to contain no (y)-term; integrality then makes it a polynomial in (x), and the balance condition forces that polynomial to be constant. This closes the load-bearing uniqueness lemma by polynomial-degree arithmetic alone.
