# Definitive verdict

The smallest correct Block-2 architecture is:

```plain text
(C) balanced/generalized Mumford normal forms as the public Jacobian interface,
    implemented with
(B) direct invertibility of the affine Mumford ideals as an internal proof device,
    but NOT mapped merely to ClassGroup(A).
```

The current Codex attempt to prove

```javascript
IsIntegrallyClosed A
```

is mathematically legitimate but strategically a detour. Stop it unless that proof is needed independently elsewhere.

The crucial distinction is:

```plain text
ClassGroup(A) only sees the affine curve U=C\{∞+,∞−}.
```

For this even-degree sextic there are two rational points at infinity. Restricting divisor classes from the complete curve to U kills the degree-zero boundary class

```plain text
delta = [∞+ - ∞−],
```

which is precisely the order-21 class central to N18. Therefore route (B), if its target is only ClassGroup A, forgets nInf and cannot model the full Jacobian.

So:

- (A) is true but too expensive.

- (B) successfully avoids normality, but ClassGroup A is the wrong final target.

- (C) is the right public architecture, with direct invertible ideals and explicit infinity data underneath.

# 1. What Mathlib's ClassGroup actually requires

At current/recent Mathlib, in

```plain text
Mathlib/RingTheory/ClassGroup/Basic.lean
```

one has, schematically,

```javascript
variable (R) [CommRing R] [IsDomain R]

def ClassGroup :=
  (FractionalIdeal R⁰ (FractionRing R))ˣ ⧸
    (toPrincipalIdeal R (FractionRing R)).range
```

Thus merely defining and using ClassGroup R requires:

```plain text
CommRing R,
IsDomain R.
```

It does not require:

```plain text
IsIntegrallyClosed R,
IsNoetherianRing R,
IsDedekindDomain R.
```

The relevant existing API is:

```javascript
ClassGroup.mk
ClassGroup.mk_def
ClassGroup.mk_eq_mk
ClassGroup.mk_eq_mk_of_coe_ideal
ClassGroup.toPrincipalIdeal
```

The constructor

```javascript
ClassGroup.mk : (FractionalIdeal R⁰ K)ˣ →* ClassGroup R
```

takes an actual unit in the monoid of fractional ideals. Therefore an explicit proof that a Mumford ideal is invertible is sufficient; no integrally-closed instance is needed.

This is exactly the pattern already used by Mathlib in

```plain text
Mathlib/AlgebraicGeometry/EllipticCurve/Affine/Point.lean.
```

There, Mathlib constructs the affine point ideal

```plain text
<X-x, Y-y>,
```

proves it is a unit among fractional ideals by an explicit inverse calculation, and defines

```javascript
WeierstrassCurve.Affine.Point.toClass :
  W.Point →+ Additive (ClassGroup W.CoordinateRing).
```

So Q4402 was correct about this narrow point: direct invertibility bypasses IsIntegrallyClosed.

# 2. Is the affine coordinate ring integrally closed?

Yes, mathematically.

Let K be a characteristic-zero field and

```plain text
A=K[x,y]/(y^2-f(x)),
```

where f is squarefree. For the fixed N18 polynomial,

```plain text
f=x^6+4x^5+10x^4+10x^3+5x^2+2x+1,
```

f is squarefree.

The affine plane curve is smooth because a singular affine point would satisfy

```plain text
y^2=f(x),
2y=0,
f'(x)=0.
```

In characteristic zero this gives y=0, and therefore

```plain text
f(x)=f'(x)=0,
```

contradicting squarefreeness.

Hence A is a one-dimensional regular Noetherian domain. Consequently it is normal, hence integrally closed; equivalently it is a Dedekind domain.

## 2.1 No subtle failure caused by the two infinities

The two points at infinity do not make the affine ring nonnormal. They are omitted from Spec A.

There is, however, a common source of confusion: the naive closure of

```plain text
y^2=f(x)
```

inside ordinary projective P^2 has a bad-looking point at infinity. The correct smooth completion is the normalized/weighted hyperelliptic model, which has two points ∞+ and ∞−. This projective-model issue does not affect normality of the affine coordinate ring.

## 2.2 Why proving this in Lean is still the wrong route

Mathlib does not currently offer a short theorem with the practical shape

```javascript
squarefree f → IsIntegrallyClosed (K[x,y]/(y^2-f)).
```

A proof through smoothness would force the development to formalize or connect:

```plain text
Jacobian criterion,
regular local rings,
regular implies normal,
dimension-one Noetherian normal implies Dedekind,
quotient-coordinate-ring dimension.
```

A custom direct normality proof would also be substantial. None of this helps with the two-infinity orientation, Cantor reduction, or reduced Mumford uniqueness.

Therefore route (A) solves a theorem which is true but does not solve the actual N18 representation problem.

# 3. Why route (B) is not sufficient as stated

Let

```plain text
U=C\{∞+,∞−} = Spec A.
```

Then ClassGroup A is the divisor/Picard class group of the affine curve U, not Pic^0(C) of the complete genus-two curve.

The restriction map has the schematic form

```plain text
Pic(C) → Pic(U),
```

and its kernel is generated by divisors supported on the omitted boundary points. Intersecting this boundary subgroup with degree zero gives

```plain text
Z · (∞+ - ∞−).
```

Consequently the class

```plain text
delta=[∞+−∞−]
```

maps to zero in ClassGroup A.

For N18, delta is not a harmless kernel element: it is the explicit order-21 generator used in the torsion table and rational-point enumeration.

Thus this proposed architecture is incorrect:

```plain text
(u,v,nInf)
   ↦ I(u,v)
   ↦ ClassGroup A.
```

The value of nInf disappears. In particular,

```plain text
(1,0,0)
and
(1,0,1)
```

would both map to the identity affine ideal class, although they represent 0 and delta in the complete Jacobian.

## 3.1 Correct use of direct ideals

Direct ideal invertibility remains very useful, but only as the affine component of an oriented/balanced class that also remembers orders at ∞+ and ∞−.

So the implementation recommendation is:

```plain text
C externally, B internally.
```

Do not expose ClassGroup A as J(C).

# 4. Direct invertibility of the Mumford ideal

For a valid Mumford pair set

```plain text
I(u,v)=<u(x), y-v(x)> ⊂ A,
```

with

```plain text
u monic,
deg v<deg u,
u | f-v^2.
```

Write

```plain text
w=(f-v^2)/u.
```

Let the conjugate ideal be

```plain text
Ibar(u,v)=<u(x), y+v(x)>.
```

Multiplying generators gives

```plain text
I(u,v) Ibar(u,v)
 = (u) · <u, y-v, y+v, w>.
```

Indeed,

```plain text
(y-v)(y+v)=y^2-v^2=f-v^2=u*w.
```

For a semireduced divisor on a squarefree curve, one proves

```plain text
<u, 2v, w> = (1)
```

in K[x]. If a common root killed u, 2v, and w, then the equations imply that the same root kills both f and f', contradicting squarefreeness. For fixed N18 this can be packaged as a polynomial Bézout certificate.

From

```plain text
A*u+B*(2v)+C*w=1
```

one gets 1 in <u,y-v,y+v,w>, because

```plain text
2v=(y+v)-(y-v).
```

Hence

```plain text
I(u,v) Ibar(u,v)=(u).
```

Therefore an explicit fractional-ideal inverse is

```plain text
I(u,v)^(-1) = (1/u) · Ibar(u,v).
```

This is the exact genus-two analogue of Mathlib's elliptic XYIdeal proof.

# 5. The corrected balanced Mumford type

Choose ∞− as the base point. The triple

```plain text
(u,v,nInf)
```

represents

```plain text
D_aff(u,v)
  - deg(u)*∞−
  + nInf*(∞+−∞−).
```

The raw integer nInf is unbounded for intermediate composition data. The canonical reduced form must impose the missing bound.

```javascript
structure Mumford18 where
  u v : Polynomial K
  nInf : ℤ
  u_monic : u.Monic
  u_ne_zero : u ≠ 0
  v_canonical : v %ₘ u = v
  quotient : Polynomial K
  factorization : f18 - v^2 = u * quotient

namespace Mumford18

def affineDegree (D : Mumford18) : ℕ := D.u.natDegree

def weight (D : Mumford18) : ℤ :=
  (D.affineDegree : ℤ) + D.nInf

def IsReduced (D : Mumford18) : Prop :=
  0 ≤ D.nInf ∧ D.weight ≤ 2

structure Reduced where
  toMumford : Mumford18
  reduced : toMumford.IsReduced
```

Equivalently, the reduced constraints are

```plain text
0 ≤ nInf,
deg(u)+nInf ≤ genus = 2.
```

This fixes the Q4510 type bug. In particular, the pure infinity classes have only

```plain text
(1,0,0),
(1,0,1),
(1,0,2)
```

as reduced representatives; an unbounded integer orientation would destroy uniqueness immediately.

# 6. The correct semantic target: oriented affine ideal classes

The minimum robust replacement for ClassGroup A is an equivalence relation which combines:

```plain text
1. an invertible affine fractional ideal;
2. the two infinity orders, or one balanced orientation coordinate;
3. principal multiplication in the function field.
```

A practical fixed-instance relation can be defined directly on valid Mumford triples.

## 6.1 Infinity orders

Every element of the function field has the form

```plain text
g=(a(x)+b(x)y)/c(x).
```

For the monic sextic, near the two infinity branches,

```plain text
y ~ +x^3 at ∞+,
y ~ -x^3 at ∞−.
```

Define integer orders

```javascript
def ordInfPlus  : FracAˣ → ℤ
def ordInfMinus : FracAˣ → ℤ
```

by degree and leading-coefficient cancellation in a±b*q18, where

```plain text
q18=x^3+2x^2+3x-1,
f18=q18^2+8x.
```

The required API is only:

```javascript
ordInfPlus_mul
ordInfMinus_mul
ordInfPlus_inv
ordInfMinus_inv
principal_degree_zero
```

and exact evaluations for x-r and y-w(x) used by Cantor reduction.

## 6.2 Relation on triples

Let D1,D2 : Mumford18. Define D1 ~ D2 iff there exists a nonzero function g such that:

```plain text
- the affine ideals differ by the principal fractional ideal of g;
- the coefficient difference at ∞+ equals ordInfPlus(g);
- the coefficient difference at ∞− equals ordInfMinus(g).
```

With the chosen triple convention, the infinity coefficients are

```plain text
coeffPlus(D)  = nInf,
coeffMinus(D) = -deg(u)-nInf.
```

Compile-shaped relation:

```javascript
def Mumford18.Equivalent (D1 D2 : Mumford18) : Prop :=
  ∃ g : FracAˣ,
    mumfordFractionalIdeal D1.toValid =
      FractionalIdeal.spanSingleton A⁰ (g : FracA) *
        mumfordFractionalIdeal D2.toValid ∧
    D1.nInf - D2.nInf = ordInfPlus g ∧
    (-(D1.u.natDegree : ℤ) - D1.nInf)
      - (-(D2.u.natDegree : ℤ) - D2.nInf)
      = ordInfMinus g
```

Depending on the ideal/divisor sign convention, the principal ideal factor may be g or g⁻¹. Choose the convention once and test it against

```plain text
div(y-q18)
and
div(y+q18).
```

Do not change it later.

Then define

```javascript
instance mumford18Setoid : Setoid Mumford18 where
  r := Mumford18.Equivalent
  iseqv := by
    -- identity, inverse and multiplication of principal functions
    sorry

abbrev Jac18Class := Quotient mumford18Setoid
```

This quotient retains delta; unlike ClassGroup A, it distinguishes (1,0,0) from (1,0,1).

## 6.3 Why this remains elementary

The relation only needs:

```plain text
fractional-ideal multiplication,
explicit principal ideals,
two fixed degree valuations at infinity.
```

It does not need:

```plain text
IsIntegrallyClosed A,
Dedekind factorization,
Riemann–Roch,
general algebraic curves,
general divisor theory.
```

# 7. Cantor composition and reduction in this architecture

Define raw Cantor composition exactly as planned:

```plain text
d=gcd(u1,u2,v1+v2),
u0=u1*u2/d^2,
v0=(s1*u1*v2+s2*u2*v1+s3*(v1*v2+f))/d mod u0,
n0=n1+n2+deg(d).
```

The proof obligation is not “the algorithm is the Jacobian law” in one theorem. Split it into two elementary statements:

```javascript
theorem compose_ideal_identity :
  mumfordFractionalIdeal (composeRaw D1 D2) =
    principalCorrection *
      (mumfordFractionalIdeal D1 * mumfordFractionalIdeal D2)

theorem compose_infinity_identity :
  -- The correction function has the exact two infinity orders
  -- needed for n0=n1+n2+deg(d).
```

Together they give:

```javascript
theorem compose_equivalent_product :
  classOf (composeRaw D1 D2) = classOf D1 + classOf D2
```

For a reduction step with lift w ≡ v mod u, use

```plain text
u'=(f-w^2)/u,
v'=-w mod u'.
```

The principal function is y-w. Prove:

```javascript
theorem reduction_ideal_identity
    (cert : ReductionCert D D') :
  -- affine relation from (y-w)

theorem reduction_infinity_identity
    (cert : ReductionCert D D') :
  -- ord∞±(y-w) from degree/leading-term calculation

theorem reduction_sameClass
    (cert : ReductionCert D D') :
  classOf D = classOf D'
```

This is enough to define addition by composing and reducing.

Associativity should be inherited from fractional-ideal multiplication at the quotient level. Do not prove associativity by expanding Cantor formulas on reduced triples.

# 8. Existence and uniqueness of reduced representatives

## 8.1 Existence

For this fixed genus-two curve, existence is the termination of explicit Cantor reduction.

Use a lexicographic measure such as

```plain text
(deg u,
 distance of nInf from the interval [0,2-deg u]).
```

Each liftPlus or liftMinus reduction step strictly decreases that measure. The fixed identity

```plain text
f18=q18^2+8x
```

makes the infinity-order cases finite and explicit.

Target theorem:

```javascript
theorem exists_reduced (D : Mumford18) :
  ∃ R : Mumford18.Reduced,
    classOf R.toMumford = classOf D
```

No Dedekind theorem is required.

## 8.2 Uniqueness

Uniqueness is a degree/pole-order proof, but it is not automatic merely from the corrected type.

Suppose reduced triples D1,D2 represent the same class. Then there is a nonzero

```plain text
g=a(x)+b(x)y
```

after clearing denominators, whose divisor realizes their difference. Since both divisors have reduced weight at most 2, the positive and negative parts of div(g) have very small total degree.

For a nonconstant function:

```plain text
pole order at an infinity branch
  is controlled by max(deg a, deg b+3),
```

except for an explicitly handled leading cancellation in a±b*q18.

The genus-two bound shows that a nonconstant g cannot have the required pole divisor of total degree at most 2. Therefore g is constant, so:

```plain text
u1=u2,
v1=v2,
nInf1=nInf2.
```

Lean theorem split:

```javascript

    {g : FracAˣ}
    (hg : ¬ IsConstant g) :
    3 ≤ poleDegreeAtInfinity g + affinePoleDegree g := by

    {D1 D2 : Mumford18.Reduced}

    omega
  omega
```

This is elementary fixed-instance algebra, not Riemann–Roch. But it still requires the explicit pole-order lemmas; Codex should not replace them with a comment saying “degree argument.”
