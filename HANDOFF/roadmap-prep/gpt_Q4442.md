# N18 Route C: reference Lean 4 base-change interface

## Executive verdict

The seam should be implemented in two layers:

1. **Concrete coefficient extension on canonical balanced Mumford representatives**
   ```text
   N18Mumford ‚Ąö  ‚Üí  N18Mumford L,
   (u,v,orientation) ‚Ü¶ (u.map őĻ, v.map őĻ, orientation),
   őĻ = algebraMap ‚Ąö L.
   ```
2. **Extension of invertible fractional ideals before taking the oriented quotient**
   ```text
   Pic18 ‚Ąö ‚Üí+ Pic18 L.
   ```

The key recommendation is:

> Prove injectivity through uniqueness of reduced oriented Mumford representatives, not by formalizing a general Picard-descent theorem first.

If a reduced triple over `‚Ąö` becomes the zero class over `L`, its coefficient extension is still a reduced triple over `L`. Uniqueness of the `L`-reduced representative makes it the zero triple; injectivity of `algebraMap ‚Ąö L` then makes the original triple zero. This proof is substantially lighter than constructing line bundles and descent data.

The Hilbert-90 argument is nevertheless mathematically correct and is given below as the abstract backup. Current pinned Mathlib has the needed Hilbert-90 result

```lean
groupCohomology.isMulCoboundary‚āĀ_of_isMulCocycle‚āĀ_of_aut_to_units
```

and the affine class-group extension API

```lean
ClassGroup.extendedHom
ClassGroup.extendedHom_mk
ClassGroup.extendedHom_mk0
FractionalIdeal.extendedHom
FractionalIdeal.extendedHom_spanSingleton
```

at revision `96fd0fff...`.

There is one important warning:

> Do **not** try to prove that `ClassGroup.extendedHom A‚Ąö AL` is injective as a general class-group theorem. Extension of affine ideal classes can capitulate. Injectivity here is a statement about the **projective degree-zero oriented group**, or equivalently about its unique balanced Mumford representatives.

The two rational points at infinity are used to define the orientation/basepoint and identify the concrete group with `Pic^0(C)`. They are not needed for Hilbert 90 itself.

# 2. Coefficient extension on Mumford triples

## 2.1 Plain coefficient map

The direct map is structurally

```lean
namespace N18Mumford

/-- Extend every polynomial coefficient; keep the infinity orientation unchanged. -/
noncomputable def mapCoeffs
    {K K' : Type*} [Field K] [Field K']
    (őĻ : K ‚Üí+* K') (hőĻ : Function.Injective őĻ)
    (D : N18Mumford K) : N18Mumford K' := by
  refine
    { u := D.u.map őĻ
      v := D.v.map őĻ
      orientation := D.orientation
      u_monic := ?_
      u_degree := ?_
      v_degree := ?_
      divides_curve := ?_
      orientation_bound := ?_ }
  ¬∑ exact D.u_monic.map őĻ
  ¬∑ -- `natDegree` is preserved by an injective coefficient map.
    simpa [Polynomial.natDegree_map_eq_of_injective őĻ hőĻ] using D.u_degree
  ¬∑ simpa [Polynomial.natDegree_map_eq_of_injective őĻ hőĻ] using D.v_degree
  ¬∑ -- Map `u ‚ą£ f-v¬≤`; the sextic over `K'` is the coefficient map of the sextic over `K`.
    rcases D.divides_curve with ‚ü®q, hq‚ü©
    refine ‚ü®q.map őĻ, ?_‚ü©
    simpa [N18Polynomial.map, map_sub, map_pow] using congrArg (Polynomial.map őĻ) hq
  ¬∑ simpa using D.orientation_bound

/-- Coefficient extension is injective when the scalar map is injective. -/
theorem mapCoeffs_injective
    {K K' : Type*} [Field K] [Field K']
    (őĻ : K ‚Üí+* K') (hőĻ : Function.Injective őĻ) :
    Function.Injective (mapCoeffs őĻ hőĻ : N18Mumford K ‚Üí N18Mumford K') := by
  intro D‚āĀ D‚āā h
  -- Extensionality reduces to `u`, `v`, and the integer orientation.
  -- For the polynomial fields, compare coefficients and use `hőĻ`.
  ext n <;> simpa [mapCoeffs] using congrArg (fun D => D) h

end N18Mumford
```

The exact field names in `N18Mumford` will differ. The mathematical content is only:

```text
u' = Polynomial.map őĻ u,
v' = Polynomial.map őĻ v,
orientation' = orientation.
```

The orientation integer is unchanged because this is constant-field extension and both infinity sections are already rational over `‚Ąö`.

For N18, define the plain function

```lean
noncomputable def mumfordMapCoeffs : MQ ‚Üí ML :=

 theorem mumfordMapCoeffs_injective :
    Function.Injective mumfordMapCoeffs :=
  N18Mumford.mapCoeffs_injective
    (algebraMap ‚Ąö L) (algebraMap ‚Ąö L).injective
```

## 2.2 Why the plain map should not initially be declared additive

Proving directly that coefficient extension commutes with the balanced Cantor/reduction group law is possible, but unnecessary. Instead:

1. construct the quotient-level additive map `orientedClassBaseChange`;
2. transport it through `mumfordEquivQ` and `mumfordEquivL`;
3. prove that the transported map evaluates to `mumfordMapCoeffs`.

This makes additivity automatic and leaves only one representative-compatibility theorem.

# 4. The one compatibility theorem

This is the load-bearing B1‚ÜíB2 seam:

```lean
/-- Extending the Mumford triple and then taking its class agrees with extending
its oriented fractional-ideal class. -/
@[simp] theorem classOf_mumfordMapCoeffs (D : MQ) :
    mumfordEquivL (mumfordMapCoeffs D) =
      orientedClassBaseChange (mumfordEquivQ D) := by
  -- Unfold the Mumford ideal `<u, y-v>` and its infinity orientation.
  -- Extension sends the generators to `<map u, y-map v>`.
  -- The integer orientation is unchanged.
  -- Finish with the quotient compatibility lemmas above.
  sorry
```

This theorem is not Hilbert 90. It is a finite algebra calculation involving ideal generators and the chosen infinity coordinate.

Now define the additive map on Mumford representatives by transport:

```lean
/-- Additive base change on canonical reduced Mumford representatives. -/
noncomputable def baseChangeMumford : MQ ‚Üí+ ML :=
  mumfordEquivL.symm.toAddMonoidHom.comp
    (orientedClassBaseChange.comp mumfordEquivQ.toAddMonoidHom)

@[simp] theorem baseChangeMumford_apply (D : MQ) :
    baseChangeMumford D = mumfordMapCoeffs D := by
  apply mumfordEquivL.injective
  change orientedClassBaseChange (mumfordEquivQ D) =
    mumfordEquivL (mumfordMapCoeffs D)
  exact (classOf_mumfordMapCoeffs D).symm
```

This construction avoids a separate theorem that the explicit Cantor/reduction formulas commute with base change.

The requested class compatibility becomes a simp theorem:

```lean
@[simp] theorem classOf_baseChangeMumford (D : MQ) :
    mumfordEquivL (baseChangeMumford D) =
      orientedClassBaseChange (mumfordEquivQ D) := by
  simpa only [baseChangeMumford_apply] using classOf_mumfordMapCoeffs D
```

# 5. Injectivity: recommended reduced-Mumford proof

First prove the representative map injective:

```lean
theorem baseChangeMumford_injective :
    Function.Injective baseChangeMumford := by
  intro D‚āĀ D‚āā h
  apply mumfordMapCoeffs_injective
  simpa only [baseChangeMumford_apply] using h
```

Then transport injectivity to the class groups:

```lean
/-- Base extension `Pic^0(C)(‚Ąö) ‚Üí Pic^0(C)(L)` is injective. -/
theorem baseChangePic_injective : Function.Injective baseChangePic := by
  intro x y hxy
  have hM :
      baseChangeMumford (mumfordEquivQ.symm x) =
        baseChangeMumford (mumfordEquivQ.symm y) := by
    apply mumfordEquivL.injective
    change baseChangePic (mumfordEquivQ (mumfordEquivQ.symm x)) =
      baseChangePic (mumfordEquivQ (mumfordEquivQ.symm y))
    simpa using hxy
  exact mumfordEquivQ.symm.injective
    (baseChangeMumford_injective hM)
```

This proof needs only:

```text
‚ÄĘ unique reduced oriented Mumford representatives over ‚Ąö and L;
‚ÄĘ coefficient extension preserves the reduced conditions;
‚ÄĘ algebraMap ‚Ąö L is injective;
‚ÄĘ the class-compatibility theorem.
```

It does **not** need an abstract Picard scheme, Galois cohomology, or a general divisor-descent library.

# 6. Abstract Hilbert-90 proof, as a backup

The exact mathematical lemma is:

> Let `X/k` be a proper geometrically integral curve and `K/k` a finite Galois extension. If `őď(X_K, O_X)^√ó = K^√ó`, then `Pic(X) ‚Üí Pic(X_K)` is injective.

For a proper geometrically integral curve, every global regular unit is constant, so the global-unit hypothesis holds.

A project-local statement can be exposed as

```lean
/-- Abstract descent proof of the same injectivity theorem. -/
theorem orientedClassBaseChange_injective_hilbert90 :
    Function.Injective orientedClassBaseChange := by
  intro x y hxy
  -- Reduce to the class `d = x-y` whose base change is zero.
  -- Interpret `d` as a line bundle `M` on the smooth projective curve C/‚Ąö.
  -- Choose a trivialization `t : M_L ‚ČÉ O_{C_L}`.
  --
  -- For each g : Gal(L/‚Ąö), compare `g(t)` with `t`.  Their ratio is a
  -- global unit on C_L, hence a constant `c g : Lň£`.
  -- The descent datum gives:
  --
  --     c (g*h) = g(c h) * c g.
  --
  -- Thus `c` is an `IsMulCocycle‚āĀ`.
  --
  -- Hilbert 90:
  --
  --   obtain ‚ü®ő≤, hő≤‚ü© :=
  --     groupCohomology.isMulCoboundary‚āĀ_of_isMulCocycle‚āĀ_of_aut_to_units
  --       c hc
  --
  -- Rescale `t` by ő≤.  The new trivialization is Galois invariant and
  -- descends to a trivialization over ‚Ąö.  Therefore d=0.
  sorry
```

At the rational-function/divisor level the same argument is:

1. Suppose an oriented class represented over `‚Ąö` becomes principal over `L`, generated by `őĪ ‚ąą L(C)^√ó`.
2. For `g ‚ąą Gal(L/‚Ąö)`, the function `g(őĪ)/őĪ` has zero divisor on the projective curve, hence is a constant `c_g ‚ąą L^√ó`.
3. The constants `c_g` form a multiplicative 1-cocycle.
4. Hilbert 90 gives `ő≤ ‚ąą L^√ó` with `c_g = g(ő≤)/ő≤` (up to the convention used by the theorem).
5. Then `őĪ/ő≤` is Galois invariant, hence lies in `‚Ąö(C)^√ó`.
6. Its principal divisor and its infinity order are the original data, so the original oriented class was already zero.

Two rational points at infinity are not what proves Hilbert 90. Their roles are:

```text
‚ÄĘ making the orientation divisor ‚ąě+‚ąí‚ąě‚ąí defined over ‚Ąö;
‚ÄĘ giving a rational base point for Abel‚ÄďJacobi;
‚ÄĘ identifying the concrete divisor-class quotient with Pic^0(C)(‚Ąö), rather
  than merely with a Picard torsor.
```

Also note that only injectivity is claimed. The map

```text
Pic^0(C)(‚Ąö) ‚Üí Pic^0(C_L)(L)^Gal
```

need not be surjective without addressing the usual Brauer obstruction. Route C never needs surjectivity.

theorem picQ_annihilated_42
    (hL : ‚ąÄ D : JL, (42 : ‚Ąē) ‚ÄĘ D = 0) :
    ‚ąÄ D : JQ, (42 : ‚Ąē) ‚ÄĘ D = 0 :=

If the final B5 capstone is `[126]JL=0` rather than `[42]JL=0`, use exactly the same theorem with `126`. No finite-generation hypothesis occurs.

A uniform annihilator alone does **not** prove finiteness of an arbitrary group. Finiteness must be supplied separately by the finite reduction/enumeration block or by an explicit finite target group.
