# N18 Route C: top-level Lean assembly contract

## Executive contract

The final assembly should depend on one capstone theorem per block. The only logical work in the final theorem is:

```plain text
order-18 point on E/Q
  -- B6 -->
noncuspidal P : C(Q)
  -- B7 -->
P is a cusp
  --> contradiction.
```

Blocks B1–B5 enter through the arguments of the B7 capstone. This is deliberate: the final theorem does not reproach the internal descent, push-pull, or enumeration mathematics, but Lean still verifies that B7 was constructed from the exact earlier certificates.

The generic interface and the final composition proof below contain no sorry. Each concrete block capstone has a by sorry body until that block is implemented.

One convention should be fixed project-wide:

```javascript
/-- A rational point of exact additive order `n`. -/
def HasRationalPointOfOrder
    (E : WeierstrassCurve ℚ) [E.IsElliptic] (n : ℕ) : Prop :=
  ∃ P : WeierstrassCurve.Projective.Point E,
    addOrderOf P = n
```

If the repository already defines this predicate, import that declaration and delete the duplicate.

# 1. Generic, dimension-free assembly interfaces

This part can live in N18/RouteC/Contracts.lean. It deliberately does not know the implementation of the cubic field, Mumford triples, or the two elliptic curves.

```javascript
import Mathlib.AlgebraicGeometry.EllipticCurve.Projective.Point
import Mathlib.Data.ZMod.Basic

set_option autoImplicit false

namespace MazurProof

/-- A rational projective point of exact additive order `n`. -/
def HasRationalPointOfOrder
    (E : WeierstrassCurve ℚ) [E.IsElliptic] (n : ℕ) : Prop :=
  ∃ P : WeierstrassCurve.Projective.Point E,
    addOrderOf P = n

namespace N18RouteC

/-- The concrete types supplied by `N18.Basic`.

Intended bindings:
* `CQ`    = points of the fixed smooth projective sextic over `ℚ`;
* `CL`    = points after base change to `L = ℚ(ζ₉)⁺`;
* `E0L`   = `E₀(L)`, where `E₀ = 162.c3`;
* `EhatL` = the rational points on the 3-isogenous curve;
* `JQ`    = the balanced-Mumford model of `Pic⁰(C)(ℚ)`;
* `JL`    = the balanced-Mumford model of `Pic⁰(C_L)(L)`.
-/
structure Types where
  CQ    : Type
  CL    : Type
  E0L   : Type
  EhatL : Type
  JQ    : Type
  JL    : Type

/-- Common curve-level data used by B1, B6, and B7. -/
structure CurveData (T : Types) where
  baseChangePoint : T.CQ → T.CL
  baseChangePoint_injective : Function.Injective baseChangePoint
  IsCusp : T.CQ → Prop

/-- B1: the two quotient maps after composing with
`E₊ ≃ E₀` and `E₋ ≃ Ehat₀`.

The fiber bound is the effective degree-two statement consumed by B7.
-/
structure B1Split (T : Types) where
  sigma : T.CL ≃ T.CL
  tau   : T.CL ≃ T.CL
  qPlus  : T.CL → T.E0L
  qMinus : T.CL → T.EhatL
  qPlus_sigma  : ∀ P, qPlus (sigma P) = qPlus P
  qMinus_tau   : ∀ P, qMinus (tau P) = qMinus P
  qPlus_fiber_finite :
    ∀ Q : T.E0L, Set.Finite {P : T.CL | qPlus P = Q}
  qPlus_fiber_card_le_two :
    ∀ Q : T.E0L, Nat.card {P : T.CL // qPlus P = Q} ≤ 2

/-- B2: the oriented fractional-ideal/Mumford realization of `Pic⁰`,
together with the two injective maps needed at integration seams. -/
structure B2Pic0
    (T : Types)
    [AddCommGroup T.JQ]
    [AddCommGroup T.JL]
    (C : CurveData T) where
  baseChangePic : T.JQ →+ T.JL
  baseChangePic_injective : Function.Injective baseChangePic
  abelJacobiQ : T.CQ → T.JQ
  abelJacobiQ_injective : Function.Injective abelJacobiQ
  abelJacobiL : T.CL → T.JL
  abelJacobi_baseChange :
    ∀ P : T.CQ,
      baseChangePic (abelJacobiQ P) =
        abelJacobiL (C.baseChangePoint P)

/-- B3: explicit push-pull maps on the balanced-Mumford group.

`psi_phi` is the sole identity needed by later blocks:
`Ψ ∘ Φ = [2]` on `Pic⁰(C_L)(L)`.
-/
structure B3PushPull
    (T : Types)
    [AddCommGroup T.E0L]
    [AddCommGroup T.EhatL]
    [AddCommGroup T.JL] where
  Phi : T.JL →+ T.E0L × T.EhatL
  Psi : T.E0L × T.EhatL →+ T.JL
  psi_phi : ∀ D : T.JL, Psi (Phi D) = 2 • D

/-- B4: the exact output of the finite 3-isogeny descent.

The weak-descent statement—not merely a Selmer dimension—is the input to
3-adic separatedness. The isogeny fields are included because B5 and B7 need
the same concrete curves and kernel.
-/
structure B4ThreeDescent
    (T : Types)
    [AddCommGroup T.E0L]
    [AddCommGroup T.EhatL] where
  H3 : AddSubgroup T.E0L
  T3 : H3
  order_T3 : addOrderOf (T3 : T.E0L) = 3
  card_H3 : Nat.card H3 = 3
  H3_annihilated : ∀ t : H3, 3 • (t : T.E0L) = 0

  phi    : T.E0L →+ T.EhatL
  phiHat : T.EhatL →+ T.E0L
  ker_phi : phi.ker = H3
  phi_surjective : Function.Surjective phi
  phiHat_injective : Function.Injective phiHat
  phiHat_phi : ∀ P : T.E0L, phiHat (phi P) = 3 • P
  phi_phiHat : ∀ Q : T.EhatL, phi (phiHat Q) = 3 • Q

  weak_three_descent :
    ∀ P : T.E0L,
      ∃ t : H3, ∃ Q : T.E0L,
        P = (t : T.E0L) + 3 • Q

/-- B5: the shared-filtration/3-adic-separatedness conclusions.

The sharp chain currently justified is
`[21]E₀(L)=0`, `[21]Ehat₀(L)=0`, and `[42]Pic⁰(C_L)(L)=0`.
-/
structure B5Separated
    (T : Types)
    [AddCommGroup T.E0L]
    [AddCommGroup T.EhatL]
    [AddCommGroup T.JL] where
  e0_annihilated_21   : ∀ P : T.E0L, 21 • P = 0
  ehat_annihilated_21 : ∀ Q : T.EhatL, 21 • Q = 0
  pic_annihilated_42  : ∀ D : T.JL, 42 • D = 0

/-- B7: the effective finite exhaustion.

The certificate stores the finite 21-point target, a list of at most 42
points in the degree-two fibers, and the two coverage/checking theorems.
The final `all_rational_points_are_cusps` theorem is derived without any
additional arithmetic.
-/
structure B7Enumeration
    (T : Types)
    [AddCommGroup T.E0L]
    [AddCommGroup T.JL]
    [DecidableEq T.CL]
    (C : CurveData T) where
  e0EquivZMod21 : ZMod 21 ≃+ T.E0L
  pic0L_finite : Finite T.JL
  rationalPoints_finite : Set.Finite (Set.univ : Set T.CQ)

  candidates : Finset T.CL
  candidates_card_le_42 : candidates.card ≤ 42
  rational_point_mem_candidates :
    ∀ P : T.CQ, C.baseChangePoint P ∈ candidates
  rational_candidate_is_cusp :
    ∀ P : T.CQ,
      C.baseChangePoint P ∈ candidates → C.IsCusp P

/-- The exact B7 conclusion consumed by the top-level theorem. -/
theorem B7Enumeration.all_rational_points_are_cusps
    {T : Types}
    [AddCommGroup T.E0L]
    [AddCommGroup T.JL]
    [DecidableEq T.CL]
    {C : CurveData T}
    (cert : B7Enumeration T C) :
    ∀ P : T.CQ, C.IsCusp P := by
  intro P
  exact cert.rational_candidate_is_cusp P
    (cert.rational_point_mem_candidates P)

/-- B6: Tate/Kubert order-18 data produce a noncuspidal rational point
on the fixed genus-two curve. -/
def TateProducesNoncusp (T : Types) (C : CurveData T) : Prop :=
  ∀ (E : WeierstrassCurve ℚ) [E.IsElliptic],
    HasRationalPointOfOrder E 18 →
      ∃ P : T.CQ, ¬ C.IsCusp P

/-- Pure final composition. This proof is independent of every internal
representation choice in B1–B7. -/
theorem assemble_no_order_18
    (T : Types)
    (C : CurveData T)
    (b6 : TateProducesNoncusp T C)
    (allCusps : ∀ P : T.CQ, C.IsCusp P)
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E 18 := by
  intro h18
  obtain ⟨P, hPnoncusp⟩ := b6 E h18
  exact hPnoncusp (allCusps P)

end N18RouteC
end MazurProof
```

The only potentially project-specific line above is the chosen definition of HasRationalPointOfOrder. WeierstrassCurve.Projective.Point E is the current Mathlib projective point group, and addOrderOf is the standard exact-order function.

# 3. Exact top-level theorem

This is the final file, for example MazurProof/NoOrder18.lean. Its proof has no placeholder.

```javascript
import Mathlib.AlgebraicGeometry.EllipticCurve.Projective.Point

-- Replace these paths by the repository's final module paths.
import FLT.N18.Block1
import FLT.N18.Block2
import FLT.N18.Block3
import FLT.N18.Block4
import FLT.N18.Block5
import FLT.N18.Block6
import FLT.N18.Block7

set_option autoImplicit false

namespace MazurProof

open N18RouteC

/-- No elliptic curve over `ℚ` has a rational point of exact order `18`. -/
theorem no_rational_point_of_order_18
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E 18 := by
  -- Materialize every seam exactly once. The dependent arguments ensure that
  -- B7 was built from the same curve, Picard group, quotient maps, and E₀.
  let b1 : B1Split N18.types :=
    N18.Block1.capstone

  let b2 : B2Pic0 N18.types N18.curveData :=
    N18.Block2.capstone b1

  let b3 : B3PushPull N18.types :=
    N18.Block3.capstone b1 b2

  let b4 : B4ThreeDescent N18.types :=
    N18.Block4.capstone b1

  let b5 : B5Separated N18.types :=
    N18.Block5.capstone b1 b2 b3 b4

  let b7 : B7Enumeration N18.types N18.curveData :=
    N18.Block7.capstone b1 b2 b3 b4 b5

  have allCusps :
      ∀ P : N18.types.CQ, N18.curveData.IsCusp P :=
    b7.all_rational_points_are_cusps

  exact assemble_no_order_18
    N18.types
    N18.curveData
    N18.Block6.capstone
    allCusps
    E

end MazurProof
```

The same final proof can be written as the transparent contradiction:

```javascript
theorem no_rational_point_of_order_18'
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E 18 := by
  let b1 := N18.Block1.capstone
  let b2 := N18.Block2.capstone b1
  let b3 := N18.Block3.capstone b1 b2
  let b4 := N18.Block4.capstone b1
  let b5 := N18.Block5.capstone b1 b2 b3 b4
  let b7 := N18.Block7.capstone b1 b2 b3 b4 b5

  intro h18
  obtain ⟨P, hPnoncusp⟩ := N18.Block6.capstone E h18
  exact hPnoncusp (b7.all_rational_points_are_cusps P)
```

This second version makes the mathematical endpoint especially clear.

# 5. Integration acceptance checklist

Before removing the final sorrys, verify these definitional seams:

- B1, B2, B3, and B7 literally share the same N18.types.CL curve-point type.

- The B1 elliptic isomorphisms preserve the selected origins, so B3 maps are additive homomorphisms rather than translated maps.

- B2 coefficient extension is injective by canonical Mumford representatives, not by an invalid generic claim that class groups never capitulate.

- B3's Phi targets the exact E0L and EhatL used by B4/B5.

- B4 exports weak descent, not only a Selmer cardinality.

- B5 uses the zero-aware formal valuation statement and returns the sharp [42] Picard annihilator.

- B7 proves an actual finite coverage table; bounded exponent alone is not accepted as finiteness.

- B6 returns a point of N18.types.CQ and proves ¬ IsCusp, including all birational exceptional branches.

Once these contracts elaborate, the final contradiction cannot hide any further arithmetic or geometric assumption.
