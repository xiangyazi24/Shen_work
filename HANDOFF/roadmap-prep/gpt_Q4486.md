# Paper 2 Lᵖ → Paper 3 bridge map

## Verdict

Yes: the repo has direct bridge routes

- Paper2.Theorem_1_3 D p C → Paper3.Proposition_1_3 D p C

- Paper2.Theorem_1_2 D p → Paper3.Proposition_1_4 D p

These are direct statement-layer implications, not new analytic estimates. They reuse the global bounded-solution branches already present in Paper 2 Theorems 1.2 and 1.3.

Caveat: these bridges unlock Paper3 Prop 1.3/1.4, not Prop 1.2. The bundled Paper3Proposition1Targets still needs the independent NegativeSensitivityGlobalEventualBound D p field for Proposition_1_2 D p.

So, strategically, Paper 2 Lᵖ should come before the Paper 3 proposition block if the goal is maximum reuse. Logically, Prop 1.3/1.4 can also be proved independently by filling the raw Paper 3 frontier fields, but the existing bridge avoids duplicating the same global boundedness proof.

## Paper 3 proposition target statements

Source: ShenWork/Paper3/Statements.lean.

### Proposition_1_3

```javascript
def Proposition_1_3
    (D : BoundedDomainData) (p : CM2Params) (C : Paper2Constants p) : Prop :=
  0 < p.a → 0 < p.b → 1 ≤ p.m → StrongLogisticCondition p C →
    ∀ u₀ : D.Point → ℝ, PaperPositiveInitialDatum D u₀ →
      ∃ u v : ℝ → D.Point → ℝ,
        IsPaper2GlobalClassicalSolution D p u v ∧
        InitialTrace D u₀ u ∧
        IsPaper2Bounded D u
```

Meaning: under strict logistic signs, 1 ≤ m, and the strong logistic condition, every paper-positive datum has a global classical solution with initial trace and eventual boundedness.

### Proposition_1_4

```javascript
def Proposition_1_4 (D : BoundedDomainData) (p : CM2Params) : Prop :=
  p.m = 1 → 1 ≤ p.β →
    ((p.a = 0 ∧ p.b = 0) ∨ (0 ≤ p.a ∧ 0 < p.b)) →
      p.χ₀ < chiBeta p →
        ∀ u₀ : D.Point → ℝ, PaperPositiveInitialDatum D u₀ →
          ∃ u v : ℝ → D.Point → ℝ,
            IsPaper2GlobalClassicalSolution D p u v ∧
            InitialTrace D u₀ u ∧
            IsPaper2Bounded D u
```

Meaning: in the m = 1, β ≥ 1, small-χ regime, with either the minimal case a=b=0 or a nonnegative/strict logistic case 0≤a,0<b, every paper-positive datum has a global classical solution with initial trace and eventual boundedness.

## Exact bridge theorem signatures

Source: ShenWork/Paper3/StatementAssembly.lean.

### Bridge A: Paper 2 Theorem 1.3 → Paper 3 Proposition 1.3

```javascript
/-- Bridge Paper2 Theorem 1.3's global bounded branch to Paper3 Proposition
1.3. -/
theorem paper3_Proposition_1_3_of_Paper2_Theorem_1_3
    {D : BoundedDomainData} {p : CM2Params} {C : Paper2Constants p}
    (h13 : Paper2.Theorem_1_3 D p C) :
    Proposition_1_3 D p C := by
  intro ha hb hm hstrong u₀ hu₀
  have hm_pos : 0 < p.m := lt_of_lt_of_le zero_lt_one hm
  exact (h13 ha hb hm_pos hstrong).2 hm u₀ hu₀
```

This is a direct implication. It uses the second/global branch of Paper2.Theorem_1_3, converting 1 ≤ p.m into 0 < p.m via zero_lt_one ≤ p.m, then applying the branch with the same ha hb hstrong u₀ hu₀.

No extra analytic hypotheses are added beyond Paper2.Theorem_1_3 D p C and the exact parameters required by Proposition_1_3.

### Bridge B: Paper 2 Theorem 1.2 → Paper 3 Proposition 1.4

```javascript
/-- Bridge Paper2 Theorem 1.2's `m = 1` global bounded branch to Paper3
Proposition 1.4. -/
theorem paper3_Proposition_1_4_of_Paper2_Theorem_1_2
    {D : BoundedDomainData} {p : CM2Params}
    (h12 : Paper2.Theorem_1_2 D p) :
    Proposition_1_4 D p := by
  intro hm hβ hcase hχ u₀ hu₀
  have ha_nonneg : 0 ≤ p.a := by
    rcases hcase with hzero | hpos
    · simp [hzero.1]
    · exact hpos.1
  have hb_nonneg : 0 ≤ p.b := by
    rcases hcase with hzero | hpos
    · simp [hzero.2]
    · exact hpos.2.le
  exact (h12 ha_nonneg hb_nonneg hβ).2 hm hχ u₀ hu₀
```

This is also a direct implication. It uses the m = 1 global branch of Paper2.Theorem_1_2.

The only local bookkeeping is extracting the Paper 2 assumptions 0 ≤ p.a and 0 ≤ p.b from the Paper 3 case split:

```javascript
((p.a = 0 ∧ p.b = 0) ∨ (0 ≤ p.a ∧ 0 < p.b))
```

Then it applies (h12 ha_nonneg hb_nonneg hβ).2 hm hχ u₀ hu₀.

## Dependency / attack-order conclusion

### Prop 1.3

For the existing bridge route, Paper 2 Thm 1.3 is effectively the producer:

```javascript
Paper2.Theorem_1_3 D p C
  ⟹ Proposition_1_3 D p C
```

It is not a logical prerequisite if you instead prove Paper3Proposition1FrontierData.proposition13 directly, but that would duplicate the same global existence + boundedness analysis.

### Prop 1.4

For the existing bridge route, Paper 2 Thm 1.2 is effectively the producer:

```javascript
Paper2.Theorem_1_2 D p
  ⟹ Proposition_1_4 D p
```

Again, it is not logically forced if Paper3Proposition1FrontierData.proposition14 is supplied directly, but the bridge is the intended reuse path.

### Prop 1.2

Paper 2 Thm 1.1 does not supply Prop 1.2 in the current abstract API. The repo explicitly keeps:

```javascript
negativeBound : NegativeSensitivityGlobalEventualBound D p
```

as an independent residual in every proposition-1 bundle. The StatementAssembly.lean comment also says this is not derived from Paper 2 Theorem 1.1.
