# Executive verdict

The clean Lean endpoint is stronger than an abstract rank statement:

```javascript
theorem x1_11_five_annihilator (P : X1_11Point) :
    (5 : ℕ) • P = 0
```

It follows from:

```plain text
weak 2-descent:       P = 2 • Q for every P,
reduction mod 2:      #E11(F₂)=5, hence [5] enters the formal kernel,
2-adic separatedness: ⋂ₙ [2ⁿ]E11₁(Q₂) = {0}.
```

This is the same shared-filtration endgame as the composite-order cases and does not use Mordell–Weil finite generation. A finite reduction calculation then proves that every rational point is one of

```plain text
O, (0,0), (0,-1), (1,0), (1,-1),
```

all of which are cusps. The final contradiction is only three lines.

The one genuinely new arithmetic input is the curve-specific cubic étale 2-descent for 11a3; this curve has no rational 2-torsion, so a rational-2-isogeny descent package does not apply.

# 3. The exact Tate-normal-form bridge

Use the standard Tate normal form

```plain text
y² + (1-c)xy - by = x³ - bx²,
P=(0,0).
```

For an exact order-11 point define

```plain text
r = b/c,
s = c²/(b-c).
```

The primitive order-11 equation is

```plain text
F₁₁(r,s) = r² - r s³ + 3 r s² - 4 r s + s = 0.
```

The optimized model is

```plain text
v² + (u²+1)v + u = 0,
u = 1-s,
v = (r-1)/(1-s).
```

The map to 11a3 is

```plain text
X = -u/v = -(1-s)²/(r-1),
Y =  u²/v =  (1-s)³/(r-1).
```

## Lean definitions

```javascript
def F11 (r s : ℚ) : ℚ :=
  r ^ 2 - r * s ^ 3 + 3 * r * s ^ 2 - 4 * r * s + s

/-- The noncuspidal raw Tate chart. The generic Tate-normal-form theorem
must construct this from an exact order-eleven point. -/
structure Tate11Chart where
  r : ℚ
  s : ℚ
  hr0 : r ≠ 0
  hr1 : r ≠ 1
  hs0 : s ≠ 0
  hs1 : s ≠ 1
  hF : F11 r s = 0

def tate11X (T : Tate11Chart) : ℚ :=
  -(1 - T.s) ^ 2 / (T.r - 1)

def tate11Y (T : Tate11Chart) : ℚ :=
  (1 - T.s) ^ 3 / (T.r - 1)
```

## Ring certificate for the birational map

The direct denominator-cleared identity is

```plain text
(r-1)³ · (Y²+Y-(X³-X²)) = (1-s)³ · F₁₁(r,s).
```

It is preferable to formalize this direct certificate instead of introducing the intermediate (u,v) type.

```javascript
lemma tate11_direct_certificate (r s : ℚ) (hr1 : r ≠ 1) :
    let X := -(1 - s) ^ 2 / (r - 1)
    let Y :=  (1 - s) ^ 3 / (r - 1)
    (r - 1) ^ 3 * (Y ^ 2 + Y - (X ^ 3 - X ^ 2)) =
      (1 - s) ^ 3 * F11 r s := by
  dsimp
  field_simp [sub_ne_zero.mpr hr1]
  ring

lemma tate11_equation (T : Tate11Chart) :
    tate11Y T ^ 2 + tate11Y T =
      tate11X T ^ 3 - tate11X T ^ 2 := by
  have hcert := tate11_direct_certificate T.r T.s T.hr1
  have hd : (T.r - 1) ^ 3 ≠ 0 :=
    pow_ne_zero _ (sub_ne_zero.mpr T.hr1)
  have hmul :
      (T.r - 1) ^ 3 *
        (tate11Y T ^ 2 + tate11Y T -
          (tate11X T ^ 3 - tate11X T ^ 2)) = 0 := by
    simpa [tate11X, tate11Y, T.hF] using hcert
  have hres := (mul_eq_zero.mp hmul).resolve_left hd
  exact sub_eq_zero.mp hres

noncomputable def tate11ToX1 (T : Tate11Chart) : X1_11Point :=
  mkX1_11Point (tate11X T) (tate11Y T) (tate11_equation T)
```

## The point is noncuspidal

The proof uses only the chart inequalities:

- s ≠ 1 implies 1-s ≠ 0, hence X ≠ 0 and Y ≠ 0;

- this excludes (0,0), (0,-1), and (1,0);

- if (X,Y)=(1,-1), clearing denominators gives

A project-facing proof can be written as follows.

```javascript
theorem tate11ToX1_noncuspidal (T : Tate11Chart) :
    Noncuspidal (tate11ToX1 T) := by
  let u : ℚ := 1 - T.s
  let d : ℚ := T.r - 1
  have hu : u ≠ 0 := by
    dsimp [u]
    exact sub_ne_zero.mpr T.hs1
  have hd : d ≠ 0 := by
    dsimp [d]
    exact sub_ne_zero.mpr T.hr1

  have hx0 : tate11X T ≠ 0 := by
    exact div_ne_zero (neg_ne_zero.mpr (pow_ne_zero _ hu)) hd

  have hy0 : tate11Y T ≠ 0 := by
    exact div_ne_zero (pow_ne_zero _ hu) hd

  have hOneMinusOne :
      ¬(tate11X T = 1 ∧ tate11Y T = -1) := by
    rintro ⟨hx, hy⟩
    have hx' : -u ^ 2 = d := by
      apply (div_eq_iff hd).mp
      simpa [tate11X, u, d] using hx
    have hy' : u ^ 3 = -d := by
      apply (div_eq_iff hd).mp
      simpa [tate11Y, u, d] using hy
    have hu23 : u ^ 3 = u ^ 2 := by
      nlinarith [hx', hy']
    have hfac : u ^ 2 * (u - 1) = 0 := by
      nlinarith [hu23]
    rcases mul_eq_zero.mp hfac with hsq | hOne
    · have : u = 0 := by simpa using hsq
      exact hu this
    · have hs0 : T.s = 0 := by
        dsimp [u] at hOne
        linarith
      exact T.hs0 hs0

  -- Equality of affine `Point.some` values reduces to equality of coordinates.
  simp [Noncuspidal, IsCusp, cuspFinset, cuspInf,
    cusp00, cusp0m1, cusp10, cusp1m1,
    tate11ToX1, mkX1_11Point, hx0, hy0, hOneMinusOne]
```

The final simp may need the local lemma

```javascript
WeierstrassCurve.Affine.Point.some.injEq
```

explicitly added to its simp set, depending on how the cusp wrappers are implemented.

Completing the square gives


```plain text
4θ³ - 4θ² + 1,
```


# 5. Finite classification of all rational points

The finite-field check is:

```plain text
E11(F₂) = {O,(0,0),(0,1),(1,0),(1,1)},
#E11(F₂)=5.
```

Reduction is injective on rational points killed by 5, because 5 is prime to the residue characteristic 2. The five visible rational cusps have distinct reductions and exhaust the reduction group.

Package that finite calculation as:


# 6. Final theorem

