# Reference Lean 4 draft: N18 Route C, Block 1

This is a reference skeleton for the independent build. The field relation, involution identities, even-sextic identity, quotient equations, and the two Weierstrass changes are written as explicit algebraic certificates. The only remaining implementation seams are the project-level adapters:

1. constructing the smooth projective model of the genus-two curve and extending the affine rational involution across its pole;

1. packaging a WeierstrassCurve.VariableChange as a point-level additive equivalence, which the pinned Mathlib API does not provide directly.

The important design choice is to prove the quotient maps first as homogeneous projective triples. Their curve equations are denominator-free ring_nf computations, so the exceptional fibers are not hidden behind division-by-zero branches.

```javascript
a ^ n.succ.succ.succ = 3 * a ^ n.succ + a ^ n
```

is important: a bare simp [a_cubic] only sees literal cubes and can leave powers such as a^12. The recurrence makes the normalizer robust for all fixed identities in this block.

## Project-level seams

Only these require new infrastructure rather than polynomial algebra:

1. the smooth projective hyperelliptic point type and extension of sigma across x=A0;

1. point-level maps attached to VariableChange and proof of additivity;

1. nonsingularity/nonzero packaging of the homogeneous quotient triples.

The code deliberately does not use sorry for any coefficient identity. A build may use one sorry for each of the three structural adapters while retaining all arithmetic certificates verbatim.
