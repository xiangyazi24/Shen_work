# Restart-slice square-root seed: what it gives and how to use it

## Verdict

Yes: the square-root restart seed is exactly the right initialization for the squared-heat Jensen barrier.

For a restart time `s`, set

```lean
let f : ℝ → ℝ := restartSliceSqrtSeed (u s)
```

The intended definitional shape is

```lean
f y = Real.sqrt (u s (unitClip y))
```

and the existing Jensen producers already use it in this exact form. In both `IntervalTruncatedJensenLocalProducer.lean` and `IntervalTruncatedStrictPositivityProducerV6.lean`, the key square identity is proved inline as:

```lean
have hsq :
    (fun y : ℝ => (f y) ^ 2) = fun y => u s (unitClip y) := by
  funext y
  change (Real.sqrt (u s (unitClip y))) ^ 2 = u s (unitClip y)
  exact Real.sq_sqrt (hnonneg s hs hsT (unitClip y))
```

So the seed gives exactly

```lean
f^2 = clipped restart slice
```

provided the restart slice is nonnegative.

## The exact producer lemma

The relevant existing producer is:

```lean
ShenWork.Paper2.BFormPositiveDatumNegPart.exists_restartSliceSqrtSeed_of_initialTrace
```

It is used in the Jensen producers with this shape:

```lean
rcases
    ShenWork.Paper2.BFormPositiveDatumNegPart.exists_restartSliceSqrtSeed_of_initialTrace
      hu₀ htrace hcont hnonneg hbound ht htT with
  ⟨s, hs, hst, hsT, hseed⟩
```

The returned object is:

```lean
hseed : SquareHeatSeed (intervalDomainLift (u s)) (restartSliceSqrtSeed (u s))
```

up to the local `let f := restartSliceSqrtSeed (u s)` / `simpa [f]` wrapper:

```lean
let f : ℝ → ℝ := restartSliceSqrtSeed (u s)

have hseed' : SquareHeatSeed (intervalDomainLift (u s)) f := by
  simpa [f] using hseed
```

This is the right abstraction. `SquareHeatSeed u₀ f` contains exactly the four fields needed for the barrier:

```lean
structure SquareHeatSeed (u₀ f : ℝ → ℝ) : Prop where
  continuousOn : ContinuousOn f (Set.Icc (0 : ℝ) 1)
  nonneg : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ f y
  pos_somewhere : ∃ y₀ ∈ Set.Icc (0 : ℝ) 1, 0 < f y₀
  square_le_initial : ∀ y ∈ Set.Icc (0 : ℝ) 1, f y ^ 2 ≤ u₀ y
```

Instantiated with `u₀ = intervalDomainLift (u s)`, the last field is the initial comparison:

```lean
have hinit : ∀ y ∈ Set.Icc (0 : ℝ) 1,
    f y ^ 2 ≤ intervalDomainLift (u s) y :=
  hseed'.square_le_initial
```

If Codex wants the comparison against the subtype slice, use:

```lean
have hinit_subtype : ∀ y (hy : y ∈ Set.Icc (0 : ℝ) 1),
    f y ^ 2 ≤ u s ⟨y, hy⟩ := by
  intro y hy
  have h := hseed'.square_le_initial y hy
  simpa [intervalDomainLift, hy] using h
```

or, for the clipped real-line slice:

```lean
have hsq :
    (fun y : ℝ => f y ^ 2) = fun y => u s (unitClip y) := by
  funext y
  change (Real.sqrt (u s (unitClip y))) ^ 2 = u s (unitClip y)
  exact Real.sq_sqrt (hnonneg s hs hsT (unitClip y))
```

That is the cleanest way to turn `w(restart) = f² ≤ u(s)` into Lean.

## Barrier initialization

For the squared heat barrier

```lean
def squareHeatBarrier (M : ℝ) (f : ℝ → ℝ) (t x : ℝ) : ℝ :=
  Real.exp (-M * t) * (intervalFullSemigroupOperator t f x) ^ 2
```

there are two equivalent initialization routes.

### Route A: through `SquareHeatSeed.square_le_initial`

For a restarted comparison where local barrier time is `τ = 0`, use the seed field directly:

```lean
have hinit : ∀ y ∈ Set.Icc (0 : ℝ) 1,
    f y ^ 2 ≤ intervalDomainLift (u s) y :=
  hseed'.square_le_initial
```

Then combine this with the calculus package’s initial trace field, when you are in the subsolution calculus route:

```lean
hcalc.initial_eq :
  ∀ x ∈ Set.Icc (0 : ℝ) 1,
    squareHeatBarrier M f 0 x = f x ^ 2
```

so the comparison initial condition is:

```lean
have hbar_init : ∀ x ∈ Set.Icc (0 : ℝ) 1,
    squareHeatBarrier M f 0 x ≤ intervalDomainLift (u s) x := by
  intro x hx
  rw [hcalc.initial_eq x hx]
  exact hseed'.square_le_initial x hx
```

### Route B: through the explicit square identity

For the Jensen witness path, the code avoids a separate `S 0` argument and uses the pointwise square identity:

```lean
have hsq :
    (fun y : ℝ => (f y) ^ 2) = fun y => u s (unitClip y) := by
  funext y
  change (Real.sqrt (u s (unitClip y))) ^ 2 = u s (unitClip y)
  exact Real.sq_sqrt (hnonneg s hs hsT (unitClip y))
```

Then the post-heat comparison is just rewriting:

```lean
have hseed_after_heat :
    intervalFullSemigroupOperator σ (fun y => (f y) ^ 2) x.1 ≤
      intervalFullSemigroupOperator σ (fun y => u s (unitClip y)) x.1 := by
  rw [hsq]
```

This is exactly what the committed Jensen producers already do.

## Heat positivity on the closed interval

The strict positivity step is already packaged in the imported name:

```lean
heat_seed_strict_pos_of_squareHeatSeed
```

and it is used in the producers as:

```lean
have hS_pos : 0 < intervalFullSemigroupOperator σ f x.1 :=
  heat_seed_strict_pos_of_squareHeatSeed hσ hseed'
```

This gives positivity at `x.1` for any `x : intervalDomainPoint`, including endpoints `x = 0` and `x = 1`, because the underlying semigroup positivity lemma is real-line-targeted:

```lean
theorem intervalFullSemigroupOperator_pos_of_nonneg_nonzero
    {t : ℝ} (ht : 0 < t) {f : ℝ → ℝ}
    (hf_cont : ContinuousOn f (Set.Icc (0 : ℝ) 1))
    (hf_nonneg : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ f y)
    (hf_pos_somewhere : ∃ y₀ ∈ Set.Icc (0 : ℝ) 1, 0 < f y₀)
    (x : ℝ) :
    0 < intervalFullSemigroupOperator t f x
```

There is no `x ∈ Ioo 0 1` hypothesis. Thus applying it to `x.1` closes the closed-endpoint issue.

## Paste-ready chain

This is the compact form Codex can paste into the Jensen endpoint bridge, modulo local hypothesis names:

```lean
rcases
    ShenWork.Paper2.BFormPositiveDatumNegPart.exists_restartSliceSqrtSeed_of_initialTrace
      hu₀ htrace hcont hnonneg hbound ht htT with
  ⟨s, hs, hst, hsT, hseed⟩

let σ : ℝ := t - s
let f : ℝ → ℝ := restartSliceSqrtSeed (u s)

have hσ : 0 < σ := by
  dsimp [σ]
  linarith

have htime : s + σ = t := by
  dsimp [σ]
  ring

have hf_cont : Continuous f := by
  simpa [f] using restartSliceSqrtSeed_continuous (hcont s hs hsT)

have hseed' : SquareHeatSeed (intervalDomainLift (u s)) f := by
  simpa [f] using hseed

-- Initial comparison at the restart time: f² ≤ u(s).
have hinit : ∀ y ∈ Set.Icc (0 : ℝ) 1,
    f y ^ 2 ≤ intervalDomainLift (u s) y :=
  hseed'.square_le_initial

-- Stronger real-line clipped equality, useful before applying the full semigroup.
have hsq :
    (fun y : ℝ => f y ^ 2) = fun y => u s (unitClip y) := by
  funext y
  change (Real.sqrt (u s (unitClip y))) ^ 2 = u s (unitClip y)
  exact Real.sq_sqrt (hnonneg s hs hsT (unitClip y))

-- Heat propagation of the seed square is the heat propagation of the restart slice.
have hseed_after_heat :
    intervalFullSemigroupOperator σ (fun y => f y ^ 2) x.1 ≤
      intervalFullSemigroupOperator σ (fun y => u s (unitClip y)) x.1 := by
  rw [hsq]

-- Strict positivity of the heat evolution of f, on the closed interval too.
have hS_pos : 0 < intervalFullSemigroupOperator σ f x.1 :=
  heat_seed_strict_pos_of_squareHeatSeed hσ hseed'
```

If the comparison is written with the squared barrier at restarted time `0`, add:

```lean
have hbar_init : ∀ y ∈ Set.Icc (0 : ℝ) 1,
    squareHeatBarrier M f 0 y ≤ intervalDomainLift (u s) y := by
  intro y hy
  rw [hcalc.initial_eq y hy]
  exact hseed'.square_le_initial y hy
```

## Non-circularity check

This seed construction does **not** assume strict positivity of `u(s,·)` everywhere. It only needs:

1. `u(s,·) ≥ 0`, for `Real.sq_sqrt` and `f ≥ 0`;
2. a nontrivial positive slice somewhere, supplied by `exists_restartSliceSqrtSeed_of_initialTrace` from positive initial datum plus initial trace;
3. continuity of the slice, for `restartSliceSqrtSeed_continuous` and for the heat positivity lemma.

Therefore it is safe for the Jensen strict-positivity producer: the seed is built from nonnegativity + initial trace, then heat-kernel positivity upgrades it to `Sν(τ) f > 0` at every closed-domain point for `τ > 0`.
