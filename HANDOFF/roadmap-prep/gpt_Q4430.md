## Why there is no correction term

Define pushforward and pullback first on degree-zero divisor classes. For a degree-two quotient q : C -> C/<rho>, the raw divisor identity is exactly

At a ramification point fixed by rho, the left side is the pullback of the branch point and equals 2P; the right side is P+P. Thus ramification creates no correction.

A correction may appear if Codex compares normalized Mumford representatives before quotienting principal divisors. That correction is principal and disappears in Pic^0. State the theorem as an equality in JL, not as syntactic equality of triples.

Also define qPlusPush through Pic^0(E+) or explicitly include all basepoint terms in the elliptic sum. A point-level shortcut that forgets the negative/basepoint part of the degree-zero divisor can manufacture a false translation term.

  calc
    d.hStar D = -D + (D + d.hStar D) := by abel
    _ = -D := by rw [d.h_add D]; simp

    d.hStar = -(AddMonoidHom.id J) := by

  ext D

  simpa using h

# 4. Exact mathematical meaning of the four geometric fields

The generic structure has four nonformal inputs.

## h_add

```javascript
h_add : ∀ D : J, D + hStar D = 0
```

On reduced Mumford classes this follows from hMumford M = -M. Geometrically, each pair P+h(P) is a fiber of the hyperelliptic map. Differences of fibers are principal, so the sum vanishes in degree zero.

The statement is special to Pic^0. On a fixed positive-degree Picard component, the hyperelliptic action may be expressed as negation followed by a degree-dependent translation. Do not generalize this exact equation to all divisor degrees.

## tau_comp

```javascript
tau_comp : tauStar = hStar.comp sigmaStar
```

The composition order matters. From tau = h ∘ sigma, covariance of pushforward gives

```plain text
tau_* = h_* ∘ sigma_*.
```

Do not reverse the composition. No commutativity lemma is needed for the [2] proof.

## plus_push_pull and minus_push_pull

```javascript
pullPlus.comp pushPlus  = id + sigmaStar
pullMinus.comp pushMinus = id + tauStar
```

These are the divisor trace identities. They are valid at branch points with multiplicity and descend because

```plain text
push(div f) = div(Norm f),
pull(div g) = div(g ∘ q).
```
