# Executive verdict

**`p=19` is a genuine fixed-curve case. It does not need the uniform Eisenstein/formal-immersion tail.**

But the proposed one-line argument

```text
X_0(19) has rank 0, so all rational points are cusps
```

is false. The correct picture is:

```text
X_0(19)(Q) has exactly three points:
  two cusps,
  one noncuspidal CM point.
```

A rational point of exact order `19` certainly gives a rational cyclic subgroup, hence a rational noncuspidal point of `X_0(19)`. Rank-zero enumeration forces that point to be the unique CM point. One must then prove that the CM `19`-isogeny has **no rational generator**.

That final fiber calculation is extremely small. A representative of the CM point is

```text
Ecm : y^2+y=x^3-38x+90              (Cremona 361a1),
```

with CM discriminant `-19` and `j=-884736`. Its rational `19`-isogeny has kernel `x`-polynomial

```text
k19(X)=
  X^9 - 38X^8 + 437X^7 - 1444X^6 - 7942X^5
      + 82308X^4 - 274360X^3 + 390963X^2
      - 130321X - 130321.
```

Modulo `2`,

```text
k19(X)=X^9+X^7+X^2+X+1.
```

This polynomial is `1` at both elements of `F_2`, so it has no root in `F_2`. Since `k19` is monic integral, it has no rational root. Therefore the CM kernel has no rational nonzero point, even after quadratic twisting.

Thus the fixed proof is:

```text
exact order-19 point
  -> rational noncuspidal X_0(19)-point
  -> unique CM point of X_0(19)(Q)
  -> rational root of k19
  -> contradiction modulo 2.
```

The degree-nine polynomial is exactly the fiber of

```text
X_1(19) -> X_0(19)
```

over the CM point: the `18` nonzero kernel points are paired by `P ~ -P`.

# 1. `X_1(19)` versus `X_0(19)`

## 1.1 The forgetful map is legitimate

A rational point `P` of exact order `19` on an elliptic curve `E/Q` gives the Galois-stable cyclic subgroup

```text
C=<P> subset E[19].
```

Therefore it gives a rational noncuspidal point

```text
(E,C) in X_0(19)(Q).
```

There is no descent-of-field issue here: `P` itself is rational, so every element of `<P>` is rational and the subgroup scheme is constant.

The natural map

```text
pi : X_1(19) -> X_0(19)
```

has degree

```text
phi(19)/2=9.
```

The division by `2` occurs because the pairs `(E,P)` and `(E,-P)` are isomorphic through `[-1]` in the coarse modular problem.

## 1.2 What `X_0(19)` does and does not prove

The curve `X_0(19)` is genus one. Its minimal model is

```text
X019 : y^2+y=x^3+x^2-9x-15,
```

or in Mathlib coefficient order

```text
[a1,a2,a3,a4,a6]=[0,1,1,-9,-15].
```

This is Cremona `19a1`. Its discriminant is

```text
Delta=-19^3=-6859.
```

Its rational group is

```text
X_0(19)(Q) = {O,(5,9),(5,-10)} ~= Z/3Z.
```

The two finite points are inverses. After fixing the modular parametrization normalization, one is the second cusp and the other is the unique noncuspidal CM point. Interchanging them amounts only to replacing the chosen generator of the order-three group by its negative; none of the proof depends on that orientation.

The modular `j`-map—not the elliptic curve's own `j`-invariant—takes the noncuspidal point to

```text
j=-884736=-96^3,
```

the class-number-one CM value of discriminant `-19`.

So `X_0(19)` does **not** say that rational `19`-isogenies do not exist. They do exist. It says there is exactly one noncuspidal rational isogeny datum, and it is CM.

# 2. MW-FG-free proof of `X_0(19)(Q)=Z/3`

The rank-zero computation is a small fixed elliptic descent and can use the same separatedness layer already developed for N15/N21/N18.

Let

```text
G=X_0(19)(Q),
T=(5,9),
H=<T>.
```

Then

```text
order(T)=3,
exp(H)=3.
```

A fixed `2`-descent proves the weak-descent statement

```text
G=H+2G.
```

Reduction at `2` is good because `2` does not divide `19^3`. The reduced group is

```text
X_0(19)(F_2)={O,(1,0),(1,1)} ~= Z/3Z.
```

Thus `3` kills both `H` and the reduction group. The shared formal-kernel theorem says that the kernel of reduction contains no nonzero infinitely `2`-divisible point. The common-annihilator version of the weak-descent theorem therefore gives

```text
3*G=0.
```

Since `3` is prime to `2`, good-reduction injectivity on `3`-torsion injects all of `G` into the three-element reduction group. The explicit subgroup `H` already has three elements, so

```text
G=H={O,T,-T}.
```

No Mordell-Weil finite-generation theorem is used.

# 3. The unique noncuspidal fiber

## 3.1 CM representative

Use

```text
Ecm19 : y^2+y=x^3-38x+90,
```

with coefficients

```text
[0,0,1,-38,90].
```

This is Cremona `361a1`. Its invariants are

```text
Delta=-19^3,
j=-884736,
CM discriminant=-19.
```

It has one rational `19`-isogeny to

```text
Ecm19' : y^2+y=x^3-13718x-619025
```

(Cremona `361a2`). Both curves have the same CM `j`-invariant.

The rank of `361a1` is `1`; this is irrelevant. The question is whether the *isogeny kernel* contains a rational generator, not whether the source curve has other rational points.

## 3.2 Exact kernel polynomial

Define

```lean
def kernelPoly19 : Polynomial ℤ :=
    X^9
  - C 38 * X^8
  + C 437 * X^7
  - C 1444 * X^6
  - C 7942 * X^5
  + C 82308 * X^4
  - C 274360 * X^3
  + C 390963 * X^2
  - C 130321 * X
  - C 130321
```

The Vélu certificate should expose:

```lean
structure CM19IsogenyCertificate where
  target : WeierstrassCurve ℚ
  phi : Ecm19.Point →+ target.Point
  degree_phi : IsogenyDegree phi 19
  kernel_x : ∀ {P : Ecm19.Point},
    P ≠ 0 → phi P = 0 →
    (kernelPoly19.map (Int.castRingHom ℚ)).eval P.x = 0
```

For this fixed proof, it is enough to verify the curve equation and the rational functions of the Vélu map by `field_simp; ring`; no general isogeny library is required.

## 3.3 No rational root

Modulo `2`:

```lean
lemma kernelPoly19_mod_two :
    kernelPoly19.map (Int.castRingHom (ZMod 2)) =
      X^9 + X^7 + X^2 + X + 1 := by
  native_decide
```

The finite check is:

```lean
lemma kernelPoly19_mod_two_no_root (z : ZMod 2) :
    (X^9 + X^7 + X^2 + X + 1).eval z ≠ 0 := by
  fin_cases z <;> native_decide
```

Use the standard monic rational-root lemma:

```lean
lemma kernelPoly19_no_rational_root (x : ℚ) :
    (kernelPoly19.map (Int.castRingHom ℚ)).eval x ≠ 0 := by
  intro hx
  have hxInt : ∃ z : ℤ, x = z := by
    exact rational_root_of_monic_integer_polynomial
      (by simp [kernelPoly19]) hx
  obtain ⟨z,rfl⟩ := hxInt
  have hmod := congrArg
    (fun n : ℤ => (n : ZMod 2))
    (integer_eval_eq_zero_of_rat_eval_eq_zero hx)
  exact kernelPoly19_mod_two_no_root (z : ZMod 2) (by
    simpa [kernelPoly19_mod_two] using hmod)
```

The helper `rational_root_of_monic_integer_polynomial` is elementary Gauss-lemma infrastructure and should be reusable.

# 4. Why this excludes twists as well

For `j` different from `0` and `1728`, all rational elliptic curves with that `j` are quadratic twists of one fixed rational model. Here

```text
-884736 ≠ 0,1728.
```

Under the standard untwisting isomorphism over `Q(sqrt(d))`, the `x`-coordinate changes by an affine expression with rational coefficients. Consequently, a rational generator of the `19`-kernel on any rational quadratic twist would give a rational root of the kernel polynomial on `Ecm19`.

Package this as the fixed fiber theorem:

```lean
/-- The fiber of X_1(19)->X_0(19) over the CM point is the
    degree-nine kernel polynomial. -/
theorem cm19_rational_generator_gives_root
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (P : (E⁄ℚ).Point)
    (hP : addOrderOf P = 19)
    (hj : E.j = (-884736 : ℚ)) :
    ∃ x : ℚ,
      (kernelPoly19.map (Int.castRingHom ℚ)).eval x = 0 := by
  -- 1. Classify E as a quadratic twist of Ecm19.
  -- 2. Exact order 19 identifies <P> with the unique rational
  --    19-isogeny kernel.
  -- 3. Untwist the x-coordinate.
  -- 4. Apply CM19IsogenyCertificate.kernel_x.
  sorry
```

An alternative implementation is to define the fiber of the fixed degree-nine map `X_1(19)->X_0(19)` directly and prove that its coordinate ring is

```text
Q[X]/(k19).
```

That version automatically includes the twisting argument.

# 5. Fixed modular bridge

## 5.1 Abstract minimal interface

The clean reusable object is a rational cyclic isogeny datum:

```lean
structure RationalCyclicIsogenyDatum (p : ℕ) where
  E : WeierstrassCurve ℚ
  isElliptic : E.IsElliptic
  C : AddSubgroup (E⁄ℚ).Point
  cyclic : IsCyclic C
  card_eq : Nat.card C = p
```

An exact-order point produces one:

```lean
noncomputable def cyclicIsogenyDatumOfPoint
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (P : (E⁄ℚ).Point)
    (hP : addOrderOf P = 19) :
    RationalCyclicIsogenyDatum 19 := by
  refine
    { E := E
      isElliptic := inferInstance
      C := AddSubgroup.zmultiples P
      cyclic := AddSubgroup.isCyclic_zmultiples P
      card_eq := ?_ }
  simpa [Nat.card_zmultiples, hP]
```

## 5.2 Fully explicit Tate alternative

If the repository does not want an abstract modular-curve object, use Tate normal form.

Sutherland's optimized affine model of `Y_1(19)` is

```text
v^5 -(u^2+2)v^4 -(2u^3+2u^2+2u-1)v^3
 +(u^5+3u^4+7u^3+6u^2+2u)v^2
 -(u^5+2u^4+4u^3+3u^2)v
 +u^3+u^2=0.
```

The fixed bridge consists of:

```text
exact-order-19 point
 -> Tate normal form E(b,c),(0,0)
 -> rational point on the displayed Y_1(19) model
 -> fixed degree-nine diamond quotient
 -> point on y^2+y=x^3+x^2-9x-15.
```

Every arrow is a finite rational-function identity. The quotient map is the one additional fixed formula that must be extracted if no general `X_1->X_0` moduli map is built. This is still ordinary engineering; it is not a formal-immersion or Eisenstein-ideal theorem.

# 6. Final Lean theorem

Use the repository's final definition if it has landed; otherwise:

```lean
def HasRationalPointOfOrder
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (n : ℕ) : Prop :=
  ∃ P : (E⁄ℚ).Point, addOrderOf P = n
```

The assembly is short:

```lean
/-- No elliptic curve over Q has a rational point of exact order 19. -/
theorem no_rational_point_of_order_19
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    ¬ HasRationalPointOfOrder E 19 := by
  rintro ⟨P,hP⟩

  let D : RationalCyclicIsogenyDatum 19 :=
    cyclicIsogenyDatumOfPoint E P hP

  have hj : E.j = (-884736 : ℚ) := by
    simpa [D] using cyclic19_j_eq_cm D

  obtain ⟨x,hx⟩ :=
    cm19_rational_generator_gives_root E P hP hj

  exact kernelPoly19_no_rational_root x hx
```

```text

2. x0Point19 / the fixed modular j-map table
3. the fixed CM fiber/twist identification
```

The third is reduced to the explicit polynomial above; its arithmetic part is two evaluations in `ZMod 2`.

# 7. Difficulty and boundary verdict

## Fixed-case work

```text
X_0(19) model and point arithmetic:             tiny
fixed weak 2-descent on 19a1:                   small/medium
2-adic formal-kernel separatedness:             already shared
three-point enumeration:                        tiny
Tate/moduli map to X_0(19):                     medium fixed identity
CM kernel polynomial and mod-2 obstruction:     tiny
```

Estimated new code after the shared elliptic infrastructure exists:

```text
700-1,500 lines,
```

mostly for the fixed modular bridge and descent certificate.

## Uniform-tail work avoided

This proof does **not** require:

```text
- the Jacobian of genus-seven X_1(19),
- the Eisenstein ideal,
- a winding quotient,
- Mazur's formal immersion theorem in prime level,
- a uniform theorem in p.
```

Therefore `p=19` should be removed from the mountain and implemented as a fixed case. The correct boundary is:

```text
p=19: fixed X_0 curve + one CM fiber polynomial.
large-prime tail: uniform formal immersion/Eisenstein package.
```

The only correction to the proposed route is that the three rational points of `X_0(19)` are **not all cusps**. Rank zero narrows the problem to one CM fiber; the explicit degree-nine obstruction closes that fiber.
