# What the assembler does internally

The dependency flow forced by the types is:

```plain text

  → faithful truncated Picard/existence data DT

I.energy ... DT
  → u ≥ 0
  → positive-part truncations become inactive
  → full physical conjugate mild identity

I.jensenStrictPos ... DT HE
  → u > 0 on (0,T] × [0,1]

  → S : ConjugateMildSolutionData p u₀

HSpectral S
  → BFormMildSpectralBootstrapData p S
  → classical local solution
  → uniform-local/global continuation
  → Theorem_1_1 intervalDomain p
```

In particular, the newly landed Jensen producer fills the exact field consumed by the assembly. It is not necessary to call strictPos_of_truncatedJensenStrictPosDataFor manually in the final theorem; that conversion belongs inside the Jensen producer/assembly chain.
