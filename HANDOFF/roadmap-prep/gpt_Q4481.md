# Executive verdict

For the specific theorem |E(ℚ)_tors| ≤ 16, the strong axiom

```javascript
mordell_weil_fg (E : WeierstrassCurve ℚ) [E.IsElliptic]
```

is not needed and should be removed from Layer 3.

What is actually needed is:

```plain text
1. E(ℚ)_tors is finite;
2. every finite rational torsion subgroup has shape
      Z/mZ × Z/nZ,  m | n,  m ≤ 2;
3. the Layer-2 modular-curve exclusions bound the remaining n.
```

The honest current status is:

- Generic finite-abelian decomposition: Mathlib has it.

- Roots-of-unity arithmetic in ℚ: Mathlib has enough infrastructure; the final m ≤ 2 rational-root lemma is small.

- Elliptic n-torsion is rank two and Weil pairing: not currently packaged in Mathlib; this is the main structural gap.

- Mordell–Weil finite generation over ℚ: not currently in Mathlib.

- Torsion finiteness by good reduction: mathematically much weaker and sufficient, but Mathlib currently has curve reduction, not a complete point-reduction/torsion-injectivity API. This should be built instead of Mordell–Weil.

Thus the recommended Layer-3 axiom budget is:

Eliminate axiom B entirely. If a temporary axiom is unavoidable, isolate only the narrow structural statement first invariant factor m ≤ 2, rather than assuming Mordell–Weil finite generation.

# 1. Exact Weil-pairing structure reduction

Let

```plain text
T := E(ℚ)_tors.
```

Assume first that T is finite. The desired theorem is:

```javascript
/-- Rational torsion has the usual two-invariant-factor form. -/
theorem rational_torsion_structure
    (E : WeierstrassCurve ℚ) [E.IsElliptic] [Finite (RatTorsion E)] :
    ∃ m n : ℕ,
      0 < m ∧ m ∣ n ∧
      Nonempty (RatTorsion E ≃+ ZMod m × ZMod n) ∧
      m ≤ 2
```

Here ZMod 1 is trivial, so:

```plain text
m = 1  => T is cyclic;
m = 2  => n = 2k and T ≅ Z/2 × Z/2k.
```

This is exactly the reduction needed for Mazur's two families.

## 1.1 Why the invariant factors are m | n

The generic finite-abelian theorem in Mathlib is:

```javascript
AddCommGroup.equiv_directSum_zmod_of_finite
AddCommGroup.equiv_directSum_zmod_of_finite'
```

from

```plain text
Mathlib/GroupTheory/FiniteAbelian/Basic.lean
```

It decomposes a finite abelian group as a finite direct sum of prime-power cyclic groups.

However, it does not by itself prove that an elliptic torsion subgroup has at most two cyclic factors. One still needs the elliptic theorem

```plain text
E[N](ℚbar) ≅ (Z/NZ)^2
```

for characteristic zero. Once that is available, any finite subgroup of E(ℚbar) has generator rank at most two, and ordinary Smith-normal-form regrouping gives

```plain text
T ≅ Z/mZ × Z/nZ,  m | n.
```

So there are really two elliptic inputs before the Weil pairing:

1. E[N] is finite free of rank two over ZMod N;

1. the Weil pairing on E[N] is perfect.

Mathlib's division-polynomial files are useful computational infrastructure, but they do not currently export this complete module theorem.

## 1.2 Why the Weil pairing forces m ≤ 2

Suppose

```plain text
T ≅ Z/mZ × Z/nZ,  m | n.
```

Then T[m] has order m². Since

```plain text
T[m] ⊆ E[m](ℚbar)
```

and E[m](ℚbar) also has order m², one obtains

```plain text
T[m] = E[m](ℚbar).
```

Therefore every m-torsion point is rational:

```plain text
E[m](ℚbar) = E[m](ℚ).
```

Choose a ZMod m-basis P,Q of E[m]. The Weil pairing

```plain text
e_m : E[m] × E[m] → μ_m
```

is alternating, bilinear, nondegenerate, and defined over ℚ. For a basis, e_m(P,Q) is a primitive m-th root of unity. Since both P and Q are rational and the pairing is defined over ℚ,

```plain text
e_m(P,Q) ∈ μ_m(ℚ).
```

But the only roots of unity in ℚ are 1 and -1. Hence a primitive m-th root in ℚ exists only for

```plain text
m = 1 or m = 2.
```

Therefore m ≤ 2.

## 1.4 Current Mathlib status: Weil pairing

A source/declaration search at the current Mathlib revision found no elliptic declaration such as

```plain text
EllipticCurve.weilPairing
WeierstrassCurve.weilPairing
```

and no complete E[n] ≅ (ZMod n)^2 theorem.

Therefore axiom A is not elementary from current Mathlib. The generic group theory and roots-of-unity endpoint are present, but the load-bearing elliptic geometry is missing.

A full Weil-pairing build would need some combination of:

```plain text
- divisors or Miller functions on an elliptic curve;
- principal-divisor calculations;
- bilinearity and independence of auxiliary divisors;
- nondegeneracy/perfectness;
- base-field/Galois equivariance;
- the rank-two structure of E[n].
```

That is a real infrastructure project, not a 200-line wrapper.

A rough estimate is several thousand Lean lines if built from scratch, with the exact size dominated by the missing divisor/function-field API rather than by the final group-theoretic argument.

# 2. Mordell–Weil finite generation

## 2.1 Current Mathlib status

Mathlib currently has the affine elliptic point group and substantial height infrastructure in progress, but no theorem giving

```javascript
AddGroup.FG ((E⁄ℚ).Point)
```

or

```javascript
Module.Finite ℤ ((E⁄ℚ).Point).
```

The current development status is confirmed by open Mathlib work:

```plain text
PR #25986: heights on Weierstrass curves — open draft;
PR #40303: affine x-coordinate formulas explicitly described as
           needed "on the way to the Mordell-Weil Theorem".
```

Thus mordell_weil_fg is still an axiom/project theorem, not a current Mathlib declaration.

## 2.2 It is unnecessary for |T| ≤ 16

The full Mordell–Weil theorem proves

```plain text
E(ℚ) ≅ ℤ^r × finite torsion.
```

For the torsion-order theorem, the free part is irrelevant. One only needs the subgroup

```plain text
T = {P : E(ℚ) | P has finite additive order}
```

to be finite.

Therefore replace

```javascript
mordell_weil_fg
```

with the much weaker statement

```javascript
theorem rational_torsion_finite
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Finite (RatTorsion E)
```

No rank, height, canonical height, or finite generation of E(ℚ) is required.

# 3. Torsion finiteness by good reduction

## 3.1 Important correction: one-prime injectivity needs a qualification

The commonly used theorem is:

At a prime p of good reduction, reduction is injective on torsion of order prime to p.

It is unsafe to state, without an additional formal-group theorem, that reduction at one arbitrary good prime is injective on all torsion.

There are two safe routes.

### Route A: one odd prime plus a torsion-free formal kernel

For a good prime p ≥ 3, prove that the kernel of

```plain text
E(ℚ_p) → Ē(𝔽_p)
```

is torsion-free. Then reduction is injective on all rational torsion.

This uses the formal logarithm/formal parameter and is stronger than the standard prime-to-p statement.

### Route B: two distinct good primes — recommended

Choose two distinct good primes p ≠ q. For each prime, use only prime-to-residue-characteristic injectivity.

If a torsion point reduces to zero at p, its order is a power of p. If it also reduces to zero at q, its order is a power of q. Since p ≠ q, its order is 1.

Therefore the combined map

```plain text
T → Ē_p(𝔽_p) × Ē_q(𝔽_q)
```

is injective. The target is finite, so T is finite.

This avoids proving that a single formal kernel is torsion-free and is the cleanest generic finiteness theorem.

## 3.2 Lean abstraction for the two-prime proof

Bundle the local theorem as:

```javascript
structure GoodReductionTorsionMap
    (G : Type*) [AddCommGroup G] (p : ℕ) where
  target : Type*
  targetGroup : AddCommGroup target
  targetFinite : Finite target
  red : G →+ target
  kernel_torsion_order_is_pPower :
    ∀ x : G, IsOfFinAddOrder x → red x = 0 →
      ∃ k : ℕ, addOrderOf x = p ^ k
```

Then the global group-theoretic theorem is elementary:

```javascript
theorem torsion_finite_of_two_reductions
    {G : Type*} [AddCommGroup G]
    {p q : ℕ} (hpq : p ≠ q)
    (Rp : GoodReductionTorsionMap G p)
    (Rq : GoodReductionTorsionMap G q) :
    Finite (AddCommMonoid.addTorsion G) := by
  let redPair : AddCommMonoid.addTorsion G →
      Rp.target × Rq.target := fun x => (Rp.red x, Rq.red x)
  apply Finite.of_injective redPair
  intro x y hxy
  -- Apply the kernel statements to x-y.
  -- Its order is both a p-power and a q-power.
  -- Distinct-prime powers can agree only at 1.
  -- Hence x-y=0 and x=y.
  sorry
```

Once the local bundles exist, this proof is a few hundred lines at most, mostly generic finite-group arithmetic.

## 3.3 Existence of two good primes

After clearing denominators, choose an integral Weierstrass model with nonzero integer discriminant Δ. Every prime not dividing Δ is a good-reduction prime for that integral model. Since a nonzero integer has only finitely many prime divisors, there exist two distinct odd primes not dividing Δ.

This is elementary number theory. It does not use Mordell–Weil.

## 3.4 Current Mathlib support and missing part

Mathlib currently has, in

```plain text
Mathlib/AlgebraicGeometry/EllipticCurve/Reduction.lean
```

the curve-level declarations

```javascript
WeierstrassCurve.IsIntegral
WeierstrassCurve.integralModel
WeierstrassCurve.exists_isIntegral
WeierstrassCurve.IsMinimal

WeierstrassCurve.reduction

```

But this file does not currently construct:

```plain text
- a reduction homomorphism on elliptic points;
- the reduction kernel filtration;
- injectivity on prime-to-p torsion;
- a theorem that rational torsion embeds into a finite reduction group.
```

WeierstrassCurve.Affine.Point.map is not a substitute: it maps points along a ring/algebra homomorphism between fields, while there is no ring homomorphism ℚ → 𝔽_p.

Therefore rational_torsion_finite is not currently a one-line Mathlib consequence. The required new infrastructure is nevertheless far smaller than Mordell–Weil:

```plain text
1. integral/projective local point representation;
2. reduction of points modulo a good prime;
3. compatibility with the group law;
4. prime-to-p torsion injectivity;
5. the two-prime generic finiteness lemma.
```

The repository's existing local formal-parameter and separatedness work should help with steps 1–4, but it must be generalized from fixed curves to an arbitrary good-reduction model.

# 4. Alternative finiteness route after a uniform exponent bound

There is another way to avoid Mordell–Weil and possibly avoid generic point reduction.

If Layer 2 eventually proves a uniform annihilator

```javascript
∀ P : RatTorsion E, N • P = 0
```

for one explicit nonzero N, then

```plain text
RatTorsion E ⊆ E[N](ℚ).
```

It is enough to prove E[N](ℚ) finite. Division polynomials give finite possible x-coordinates, and each x-coordinate has at most two points.

Mathlib already contains substantial ingredients:

```plain text
Mathlib/AlgebraicGeometry/EllipticCurve/DivisionPolynomial/Basic.lean
Mathlib/AlgebraicGeometry/EllipticCurve/DivisionPolynomial/Degree.lean
WeierstrassCurve.Affine.Point.xRep_eq_xRep_iff
```

but a complete public theorem

```javascript
Finite {P : E.Point | N • P = 0}
```

is not presently packaged.

This route may be shorter if the cyclic-order layer already outputs a global exponent. If Layer 3 needs finiteness before any uniform order theorem, the two-good-prime route is logically cleaner.

# 6. Recommended Layer-3 architecture

Use three modules.

## 6.1 RationalTorsionFinite.lean

Public capstone:

```javascript
theorem rational_torsion_finite
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Finite (RatTorsion E)
```

Preferred proof: two distinct good primes and prime-to-prime reduction injectivity.

This replaces mordell_weil_fg everywhere in the torsion-order proof.

## 6.2 EllipticTorsionRankTwo.lean

Public capstone:

```javascript
theorem torsion_invariant_factors
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    [Finite (RatTorsion E)] :
    ∃ m n : ℕ, 0 < m ∧ m ∣ n ∧
      Nonempty (RatTorsion E ≃+ ZMod m × ZMod n)
```

This packages E[N] rank two plus finite-group regrouping.

## 6.3 RationalWeilPairing.lean

Public capstone:

```javascript
theorem first_invariant_factor_le_two
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    [Finite (RatTorsion E)]
    {m n : ℕ}
    (hmn : m ∣ n)
    (hT : Nonempty (RatTorsion E ≃+ ZMod m × ZMod n)) :
    m ≤ 2
```

The final structure theorem then becomes a short composition.

If the full Weil-pairing build is deferred, make only this capstone the temporary axiom. Do not assume finite generation of the entire Mordell–Weil group.

# 7. Bottom line

1. Weil-pairing reduction: mathematically precise and correct. It gives m≤2, hence exactly the cyclic family and Z/2 × Z/2n family. Current Mathlib does not supply the elliptic Weil pairing or the full rank-two E[n] package, so this remains the principal Layer-3 infrastructure gap.

1. Mordell–Weil finite generation: not currently in Mathlib. More importantly, it is unnecessary for |T|≤16.

1. Torsion finiteness: can be proved independently by good reduction. Use two distinct good primes to avoid the hidden mistake that one reduction map is automatically injective on all torsion. This is a moderate local-reduction build, but much smaller than Mordell–Weil.

1. Honest Layer-3 scope:

```plain text
Elementary/already present:
  finite abelian decomposition;
  torsion subgroup definitions;
  roots-of-unity arithmetic;
  generic two-prime kernel argument.

Moderate new infrastructure:
  point reduction at good primes;
  prime-to-p torsion injectivity;
  generic rational torsion finiteness.

Major new infrastructure:
  E[n] as a rank-two ZMod n module;
  Weil pairing, perfectness, and rationality.

Not needed:
  Mordell–Weil finite generation;
  rank computations;
  Sha finiteness;
  database facts.
```

Therefore the correct planning decision is:

Delete axiom B from the Mazur |T|≤16 proof. Keep Layer 3 focused on finite torsion by reduction plus the single genuine structural gap: rank-two n-torsion and the Weil pairing.
