# Executive verdict

The uniform prime tail

```plain text
∀ p prime, 17 ≤ p →
  no elliptic curve E/Q has a Q-rational point of exact order p
```

is the one part of the proposed Mazur formalization that is not reducible to a finite collection of fixed low-genus arithmetic certificates.

There is no known elementary “check primes up to a modest B, then use a soft argument above B” proof. Every viable uniform proof contains essentially Mazur–Kamienny–Merel-level modular-curve machinery: integral models of modular curves, Jacobians or abelian quotients, Hecke operators/modular symbols, a rank-zero or finite-Mordell–Weil quotient, and a formal-immersion argument at a cusp.

The shortest genuinely axiom-free route for Lean is likely a specialized version of Mazur’s Eisenstein-quotient proof, proving only the theorem needed for rational prime torsion. It is still a multi-year research-formalization project from the current Mathlib/FLT baseline.

The winding-quotient/Kamienny route has a cleaner final formal-immersion statement, but its rank-zero input uses Kato or Kolyvagin–Logachev-type results. Fully formalizing that route is probably larger than the specialized Eisenstein route.

Merel’s theorem is not a shortcut to the sharp bound p < 17: it supplies uniform boundedness, but converting it into the exact rational prime list either uses a very weak explicit bound followed by thousands of impossible case computations, or reintroduces the same winding-quotient machinery.

# 1. Is the standard proof “finite primes up to B, then a uniform tail”?

## 1.1 Mazur’s original proof: no small cutoff B

Mazur’s 1977 theorem is itself uniform in the prime. It does not proceed by determining all rational points on

```plain text
X_1(17), X_1(19), X_1(23), ... , X_1(B)
```

and then invoking an elementary estimate above B.

The uniform content is encoded in the Eisenstein ideal and the resulting quotient of a modular Jacobian. In modern schematic language, the core pattern is:

```plain text
rational p-torsion point
  -> integral noncuspidal section of X_1(p), hence X_0(p)
  -> the section specializes to a cusp at a small prime
  -> map X_0(p) to a finite-MW Eisenstein quotient A_p of J_0(p)
  -> the section and cusp have the same image in A_p
  -> the map is a formal immersion at the cusp
  -> the section equals the cusp
  -> contradiction.
```

The details use the integral modular curve, Néron models, Hecke operators, the Eisenstein quotient, component groups, and finite flat group schemes. This is the hard uniform theorem, not a finite arithmetic cleanup.

## 1.2 Parent’s explicit general bound gives B=8320 over Q, but is useless here

Parent’s effective bound for a prime-power torsion point of order p^n, for p>3, is

```plain text
p^n ≤ 65*(3^d-1)*(2d)^6
```

for a number field of degree d. At d=1 this gives

```plain text
p ≤ 65*(3-1)*2^6 = 8320.
```

Thus a logically possible route is:

```plain text
uniform boundedness -> p ≤ 8320
then check every prime 17 ≤ p ≤ 8320.
```

This is not viable. There are over a thousand primes in that interval, the genera of X_1(p) grow quadratically, and the required Jacobians rapidly become enormous. There is no realistic fixed-curve Lean strategy for that list.

## 1.3 Oesterlé’s sharp-looking formula is not an independent d=1 shortcut

Oesterlé’s unpublished notes, later included in the appendix of Derickx–Kamienny–Stein–Stoll, prove bounds of the shape

```plain text
p ≤ (3^(d/2)+1)^2
```

in the higher-degree range treated there, together with special bounds for small degrees. The published exposition explicitly cites Mazur for d=1; it does not provide an independent one-page proof of Mazur by substituting d=1 into the displayed formula.

So one must not argue:

```plain text
plug d=1 into Oesterlé -> p≤7 -> Mazur tail solved.
```

That would hide the same modular-symbol/winding-quotient theorem whose degree-one case is Mazur’s result.

## 1.4 Rational-isogeny classification gives a finite list, but the classification is itself the hard theorem

A rational point of order p generates a rational cyclic subgroup, hence a rational p-isogeny. Mazur’s rational-isogeny theorem restricts p to

```plain text
2,3,5,7,11,13,17,19,37,43,67,163.
```

After separately handling 11 and 13, this would leave only

```plain text
17,19,37,43,67,163.
```

But formalizing the rational-isogeny theorem is not easier than the prime-torsion tail. It is another major Eisenstein-ideal theorem. Moreover, several of these primes really do occur as rational isogeny degrees, so X_0(p)(Q) is not merely cuspidal; a further argument is needed to show that the kernel character cannot be trivial.

Therefore this is a useful logical factorization only if the isogeny theorem is already available. It is not a shortcut for an axiom-free build.

## 1.5 Gonality does not supply a soft uniform tail

Large gonality proves that low-degree maps to P^1 do not exist and is powerful in uniform boundedness arguments. It does not imply that a high-genus modular curve has no isolated rational point. Thus a proof of

```plain text
gonality(X_1(p)) -> infinity as p -> infinity
```

cannot by itself exclude rational p-torsion.

A genuinely arithmetic quotient/formal-immersion theorem remains necessary.

# 2. The winding-quotient/Kamienny route

## 2.1 Correct object and construction

The relevant object is usually denoted something like

```plain text
J_1(p)^e,  J_mu^e(p), or the winding quotient,
```

not simply the - eigenspace of an involution.

Let e be the winding element, represented analytically by the modular-symbol path from 0 to infinity. The Hecke algebra acts on modular symbols. The annihilator of e defines an optimal quotient of the modular Jacobian:

```plain text
J_1(p) -> J_1(p)^e.
```

The pairing with a weight-two cusp form is

```plain text
<e,f> = integral_0^{i infinity} f(z) dz
      = a nonzero scalar times L(f,1).
```

Thus the winding quotient retains precisely the newform factors whose central value L(f,1) is nonzero.

## 2.2 Why its Mordell–Weil group is finite

The implication

```plain text
L(f,1) != 0 -> associated modular abelian variety has rank 0
```

is deep. In modern treatments it is supplied by Kolyvagin–Logachev in suitable cases or by Kato’s Euler-system results. Derickx–Kamienny–Stein–Stoll explicitly explain that Parent’s hypothesis that the winding quotient has rank zero follows from Kato.

This is not a routine analytic lemma and is not present in Mathlib.

## 2.3 Formal immersion and q-coefficients

The Abel–Jacobi map from X_1(p) or a symmetric power to the winding quotient is tested at a cusp. Its cotangent map can be expressed using the first Fourier coefficients of weight-two cusp forms, or equivalently Hecke translates of the winding element.

Kamienny’s criterion turns formal immersion into a finite linear-independence statement involving

```plain text
T_1 e, T_2 e, ..., T_d e
```

modulo a chosen residue prime. For degree d=1, the matrix is much smaller than in higher-degree applications, but the surrounding infrastructure is still substantial:

```plain text
integral X_1(p), its cusp and completed local ring;
Jacobian/Néron model;
Hecke correspondences;
modular symbols and the winding element;
q-expansion/cotangent identification;
finite-MW winding quotient;
reduction of a p-torsion moduli point to a cusp.
```

## 2.4 The final contradiction

At a small prime, usually 2 or 3, a point corresponding to rational p-torsion reduces to a cusp once p is larger than the Hasse-bound range. Roughly:

```plain text
if the curve had good reduction, p would divide #E(F_l),
which is impossible for large p;
therefore the modular point lies on the cuspidal boundary.
```

The difference between the rational point and the cusp maps to zero in the finite-MW quotient. Formal immersion then says that two integral sections with the same reduction and the same quotient image must coincide. This contradicts noncuspidality.

## 2.5 Is this more Lean-tractable than the Eisenstein ideal?

Not fully axiom-free.

It has two advantages:

```plain text
- the final uniqueness lemma is a clean general formal-immersion theorem;
- the Hecke/q-expansion condition can be phrased as finite linear algebra.
```

But it introduces a major disadvantage:

```plain text
- proving the winding quotient has finite Mordell–Weil group requires
  Kato/Kolyvagin-scale arithmetic.
```

Given current Mathlib, this makes the complete winding route at least as large as, and likely larger than, a specialized algebraic Eisenstein-quotient proof.

A practical compromise would be to assume one theorem

```javascript
windingQuotient_finite_Q
```

and formalize everything else. That would be a meaningful project, but it would not be axiom-free.

# 3. The genuinely minimal axiom-free theorem package

The shortest real target is not “formalize all of Mazur’s paper.” It is to isolate a specialized prime-tail package with only the following outputs.

## 3.1 Moduli and integral reduction

```javascript
/-- Exact rational p-torsion gives a noncuspidal integral modular section. -/
theorem modularSection_of_primeTorsion
    (p : Nat) (hp : p.Prime)
    (E : WeierstrassCurve Q) [E.IsElliptic]
    (P : (E/Q).Point)
    (hP : addOrderOf P = p) :
    NoncuspidalIntegralSection (X1 p)

/-- For p>=17, the section reduces at 2 to the distinguished cusp. -/
theorem primeTorsion_reducesToCusp_two
    {p : Nat} (hp : p.Prime) (h17 : 17 <= p)
    (x : NoncuspidalIntegralSection (X1 p)) :
    reductionAtTwo x = cuspInfinity
```

The second theorem packages Tate-curve/stable-reduction and Hasse-bound arguments.

## 3.2 Eisenstein quotient

For each prime p, construct an abelian scheme or Néron-model quotient

```plain text
J_0(p) -> A_p
```

and prove:

```javascript
/-- The Eisenstein quotient has finite rational points. -/
theorem eisensteinQuotient_finite_Q
    (p : Nat) (hp : p.Prime) :
    Finite (A p Q)

/-- The modular curve map is a formal immersion at the cusp. -/
theorem eisensteinMap_formalImmersion_atInfinity
    (p : Nat) (hp : p.Prime) (h17 : 17 <= p) :
    FormalImmersionAt (x0ToEisenstein p) cuspInfinity
```

The finite-flat and Hecke-theoretic part must then prove that the modular section and the cusp have the same image:

```javascript
theorem primeTorsion_sameEisensteinImage_asCusp
    {p : Nat} (hp : p.Prime) (h17 : 17 <= p)
    (x : NoncuspidalIntegralSection (X1 p)) :
    x0ToEisenstein p (forgetGenerator x) =
      x0ToEisenstein p cuspInfinity
```

This is where the Eisenstein ideal, Atkin–Lehner action, cuspidal subgroup, component groups, and classification of the relevant finite flat subgroup schemes enter.

## 3.3 General formal-immersion uniqueness

This part is general algebraic geometry and reusable:

```javascript
theorem eq_of_sameReduction_sameImage_of_formalImmersion
    {R X Y : Type*}
    [DVR R]
    (f : X -> Y)
    {x0 : SpecialFiberPoint X}
    (hf : FormalImmersionAt f x0)
    {P Q : IntegralPoint R X}
    (hredP : P.red = x0)
    (hredQ : Q.red = x0)
    (himage : f P = f Q) :
    P = Q
```

## 3.4 Final theorem

```javascript
theorem no_rational_prime_torsion_ge_17
    (p : Nat) (hp : p.Prime) (h17 : 17 <= p)
    (E : WeierstrassCurve Q) [E.IsElliptic] :
    ¬ exists P : (E/Q).Point, addOrderOf P = p := by
  rintro ⟨P,hP⟩
  let x := modularSection_of_primeTorsion p hp E P hP
  have hred := primeTorsion_reducesToCusp_two hp h17 x
  have himg := primeTorsion_sameEisensteinImage_asCusp hp h17 x
  have hx := eq_of_sameReduction_sameImage_of_formalImmersion
    (eisensteinMap_formalImmersion_atInfinity p hp h17)
    hred rfl himg
  exact x.noncuspidal (by simpa using hx)
```

This is the smallest honest top-level dependency graph. The proof hidden behind the three modular theorems remains large.

# 4. Why the alternatives are not smaller

## 4.1 Full Merel uniform boundedness

Merel proves finiteness of possible torsion over every fixed degree. This is much more general than needed. The proof uses symmetric powers of modular curves, modular symbols, and formal immersion/winding arguments.

For the rational case, using Merel or Parent gives either:

```plain text
- a non-sharp existence bound, which does not prove p<17; or
- the explicit bound p<=8320, followed by an impossible case sweep.
```

Formalizing full Merel is strictly larger than formalizing the specialized Mazur tail.

## 4.2 Case-by-case up to a small B plus gonality

No known soft theorem says:

```plain text
p>B -> X_1(p)(Q) contains only cusps
```

using genus or gonality alone. Any theorem strong enough to give this conclusion is already a uniform boundedness/formal-immersion theorem.

## 4.3 Rational-isogeny theorem plus six residual primes

This reduces the visible list to

```plain text
17,19,37,43,67,163,
```

but the rational-isogeny theorem is itself a major Mazur theorem. If that theorem is formalized for another purpose, this is a reasonable reuse strategy. Otherwise it does not reduce total effort.

## 4.4 Explicit Tate obstruction polynomials for every p

For each fixed p, Tate normal form produces an algebraic curve X_1(p) and explicit equations. As p varies, their degrees and genera grow without bound. There is no finite symbolic resultant that simultaneously excludes all primes.

# 5. Current Mathlib and FLT infrastructure audit

## 5.1 What exists

Current Mathlib contains substantial complex-analytic modular-form infrastructure, including modules such as

```plain text
Mathlib/NumberTheory/ModularForms/Basic.lean
Mathlib/NumberTheory/ModularForms/CongruenceSubgroups.lean
Mathlib/NumberTheory/ModularForms/Cusps.lean
Mathlib/NumberTheory/ModularForms/QExpansion.lean
Mathlib/NumberTheory/ModularForms/CuspFormSubmodule.lean
Mathlib/NumberTheory/ModularForms/EisensteinSeries/...
```

There is also

```plain text
Mathlib/NumberTheory/HeckeRing/Defs.lean
```

but this is not an implementation of the classical Hecke algebra acting through correspondences on J_0(p) with the Eisenstein ideal needed by Mazur.

Mathlib and the FLT repository also provide useful general foundations:

```plain text
schemes and algebraic geometry;
elliptic curves and their point groups;
number fields and local fields;
some group-scheme infrastructure;
FLT/GroupScheme/FiniteFlat.lean;
Galois representations and deformation theory;
complex analytic modular forms.
```

These reduce the foundational cost, but not the central modular-curve gap.

## 5.2 What is absent

A current code search found no implementation of:

```plain text
X_0(N) or X_1(N) as algebraic/moduli schemes;
Deligne–Rapoport or Katz–Mazur integral models;
universal generalized elliptic curves at the cusps;
Jacobians J_0(N), J_1(N) or their Néron models;
Picard functors specialized to modular curves;
Hecke correspondences on modular curves and Jacobians;
Atkin–Lehner operators on those Jacobians;
Manin symbols or modular-symbol homology;
the winding element and winding quotient;
the Eisenstein ideal or Eisenstein quotient;
the rational cuspidal subgroup theorem;
component-group calculations at p;
the q-expansion-to-cotangent formal-immersion criterion;
Kato/Kolyvagin rank-zero theorems for modular abelian varieties.
```

Searching for ModularCurve, ModularSymbol, Winding, and the modular meaning of Eisenstein ideal in current Mathlib returns no relevant implementation.

## 5.3 Status in this FLT repository

FLT/Assumptions/Mazur.lean currently states Mazur’s bound as an axiom and explicitly describes the original proof as a 154-page theorem using modular curves and the Eisenstein ideal.

The repository’s FLT/EllipticCurve/Torsion.lean has useful definitions of n-torsion, but still contains unfinished theorems even for general finiteness and cardinality of geometric n-torsion. Those gaps are much smaller than the Eisenstein-ideal project but confirm that the needed stack is not already present.

# 6. True scale

The following are order-of-magnitude assessments, not precise line estimates.

## 6.1 Specialized Eisenstein route

Major work packages:

```plain text
1. integral modular curves X_0(p), X_1(p) and their moduli property;
2. cusps, degenerations, and reduction of a p-torsion section;
3. Jacobian/Picard and Néron-model interfaces;
4. Hecke correspondences and Atkin–Lehner actions;
5. Eisenstein ideal, cuspidal subgroup, and optimal quotient;
6. component groups and finite flat subgroup-scheme arguments;
7. formal immersion at the cusp;
8. final uniform theorem in the variable prime p.
```

Even aggressively specialized, this is plausibly

```plain text
50,000–150,000+ new Lean lines
```

and multiple expert-years. The uncertainty is large because several missing objects—especially integral modular curves and Jacobians/Néron models—are reusable libraries, not local lemmas.

## 6.2 Winding quotient route

In addition to much of the preceding geometry, this requires:

```plain text
modular symbols and Manin relations;
Hecke action on homology;
winding element and quotient;
L(f,1) pairing;
Kato or Kolyvagin–Logachev finiteness of the quotient.
```

A fully axiom-free implementation is likely larger than the Eisenstein route and could easily become a several-year project by itself.

## 6.3 Merel route

Full uniform boundedness over arbitrary number fields is broader still. It is not a rational-tail shortcut and should not be selected for this project.

# 7. Recommended project decision

## If the immediate goal is an axiom-free full Mazur theorem

Treat the prime tail as a dedicated major project with its own blueprint. The recommended mathematical route is:

```plain text
specialized Mazur Eisenstein quotient
  + general formal-immersion uniqueness lemma
  + only the moduli and group-scheme facts needed for rational p-torsion.
```

Do not first formalize full rational-isogeny classification, full Merel, or Kato.

## If the immediate goal is to remove all fixed-level axioms

Complete the N18/N13/fixed composite orders and leave precisely one named axiom:

```javascript
axiom no_rational_prime_torsion_ge_17
    (p : Nat) (hp : p.Prime) (h17 : 17 <= p)
    (E : WeierstrassCurve Q) [E.IsElliptic] :
    ¬ exists P : (E/Q).Point, addOrderOf P = p
```

This is a clean and mathematically honest boundary. It isolates exactly Mazur’s uniform Eisenstein-ideal content rather than mixing it with the fixed-curve engineering.

## Bottom line

There is no known Lean-tractable elementary replacement for the uniform tail. The tail is not a finite collection of more N18-style rank calculations. It is the genuinely research-scale component of Mazur’s theorem.

The rest of the proposed project can reasonably be described as difficult arithmetic engineering once the shared genus-two and separatedness infrastructure lands. The prime tail cannot.
