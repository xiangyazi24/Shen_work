
The first three parts are finite and, in principle, engineering. The last part is not.

# 1. What theorem is actually being formalized?

The repository axiom is only the cardinality bound

```javascript

```

not the full 15-group classification.

That distinction helps. We do not need to prove that every allowed group occurs, and we do not need a complete classification of all groups of order at most 16. We only need enough exact-order exclusions and two noncyclic bridge lemmas to force the cardinality bound.

The current FLT/Assumptions/Mazur.lean explicitly notes that its use of ncard still requires a separate proof that rational torsion is finite. The current FLT/EllipticCurve/Torsion.lean also still contains sorry for finiteness/cardinality of n-torsion over algebraically closed fields. These are real prerequisites, not bookkeeping.

# 2. The exact finite group-theory reduction

## 2.1 Finiteness of rational torsion: no Mordell–Weil theorem needed

For an arbitrary elliptic curve E/ℚ, choose two distinct good primes p and q. Reduction gives

```plain text
red_p : E(ℚ)_tors → E(F_p),
red_q : E(ℚ)_tors → E(F_q).
```

The kernel of good reduction has no torsion of order prime to the residue characteristic. Therefore

```plain text
T → E(F_p) × E(F_q),
P ↦ (red_p(P),red_q(P))
```

is injective: a point in both kernels would have simultaneously p-power and q-power order.

Hence T=E(ℚ)_tors is finite. This uses no height and no Mordell–Weil finite generation.

Recommended theorem:

```javascript
theorem rational_torsion_finite_via_two_good_primes
    (E : WeierstrassCurve ℚ) [E.IsElliptic] :
    Finite (AddCommGroup.torsion (E⁄ℚ).Point) := by
  -- choose two primes not dividing an integral discriminant;
  -- construct the two reduction maps;
  -- prove injectivity into the product of finite groups.
  sorry
```

## 2.2 Invariant factors and Weil pairing

Once T is finite, finite abelian group theory and the fact that E[n](Qbar) has rank two give

```plain text
T ≅ Z/mZ × Z/nZ,
m | n.
```

If m>1, the two invariant-factor generators give all of E[m] rational. Nondegeneracy of the Weil pairing then puts a primitive m-th root of unity in ℚ. Since the only roots of unity in ℚ are ±1,

```plain text
m ≤ 2.
```

Thus

```plain text
T is cyclic, or T ≅ Z/2Z × Z/2nZ.
```

This also requires no Mordell–Weil finite generation.

The repository's n_torsion_card, n_torsion_dimension, and Weil-pairing layer must be completed or replaced by a fixed theorem with exactly this consequence.

# 3. The exact finite obstruction list for cyclic torsion

Suppose T is cyclic of order N>16.

If a prime p≥17 divides N, the uniform prime-tail theorem excludes N.

If 11|N or 13|N, the p=11 or p=13 theorem excludes N.

Otherwise every prime divisor of N lies in {2,3,5,7}. Then an elementary exponent case split shows that one of

```plain text
14, 15, 16, 18, 20, 21, 24, 25, 27, 35, 49
```

divides N.

Indeed:

```plain text
v2(N)≥4       -> 16|N
v3(N)≥3       -> 27|N
v5(N)≥2       -> 25|N
v7(N)≥2       -> 49|N
5 and 7 occur -> 35|N
3 and 7 occur -> 21|N
2 and 7 occur -> 14|N
3 and 5 occur -> 15|N
v2≥2 and 5    -> 20|N
v2≥3 and 3    -> 24|N
v3≥2 and 2    -> 18|N
```

with the remaining exponent combinations giving N≤16.

So the finite cyclic obstruction list is logically sufficient.

Lean target:

```javascript
theorem large_nat_has_mazur_forbidden_divisor
    {N : ℕ} (hN : 16 < N) :
    (∃ p, Nat.Prime p ∧ 17 ≤ p ∧ p ∣ N) ∨
    11 ∣ N ∨ 13 ∣ N ∨
    14 ∣ N ∨ 15 ∣ N ∨ 16 ∣ N ∨
    18 ∣ N ∨ 20 ∣ N ∨ 21 ∣ N ∨ 24 ∣ N ∨
    25 ∣ N ∨ 27 ∣ N ∨ 35 ∣ N ∨ 49 ∣ N := by
  omega
  -- More realistically: factor N and split on valuations at 2,3,5,7.
```

The fact that 14,15,16≤16 is not a problem: excluding them is stronger than the cardinality theorem and is useful because they divide larger forbidden orders.

# 4. The noncyclic case and the two genuinely special bridges

Let

```plain text
T ≅ Z/2Z × Z/2nZ.
```

Then #T=4n.

If n≤4, the desired bound holds.

For n=7 and n=8, T already contains points of order 14 and 16.

For n≥9, the exponent 2n>16; the same finite cyclic-divisor lemma applies to an element of order 2n.

The exceptional cases are exactly

```plain text
n=5 : T ≅ Z/2 × Z/10, #T=20,
n=6 : T ≅ Z/2 × Z/12, #T=24.
```

These groups do not necessarily contain points of order 20 or 24. Therefore the N20 and N24 helpers must be stated as isogeny bridges, not merely exact-order exclusions on the same curve:

```javascript
theorem order20_on_isogenous_curve_of_two_by_ten
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (h : ContainsSubgroup E (ZMod 2 × ZMod 10)) :
    ∃ E' : WeierstrassCurve ℚ,
      E'.IsElliptic ∧ HasRationalPointOfOrder E' 20


theorem order24_on_isogenous_curve_of_two_by_twelve
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    (h : ContainsSubgroup E (ZMod 2 × ZMod 12)) :
    ∃ E' : WeierstrassCurve ℚ,
      E'.IsElliptic ∧ HasRationalPointOfOrder E' 24
```

The degree-two Vélu quotient is the correct elementary mechanism.

N14 does not require an analogous sophisticated bridge: independent rational 2- and 7-torsion points already sum to a point of order 14.

# 5. Layer 1 audit

## 5.1 Fixed orders other than 25 and 49

Conceptually, the following can all be fixed-instance arithmetic:

```plain text
14,15,16,18,20,21,24,27,35
```

provided every block contains:

```plain text
Tate-normal-form forward map,
all denominator/noncusp conditions,
rank/bounded-exponent proof for the obstruction curve,
finite reduction to pin torsion,
complete rational-point enumeration,
cusp evaluation.
```

A proof sketch or a database rank label is not enough. The endgame must enumerate the rational points and identify all of them as exceptional/cuspidal.

## 5.2 N18 consistency correction

There are two valid annihilator strengths:

```plain text
safe baseline:
  [21]E0(L)=0,
  [63]Ehat0(L)=0,
  Psi Phi=[2]
  => [126]J_1(18)(L)=0;

sharpened version:
  E0(L)=Z/21,
  Ehat0(L)=Z/7,
  Psi Phi=[2]
  => [42]J_1(18)(L)=0.
```

The plan must choose one and prove every input. It must not infer [42] merely from the dual three-isogeny; that alone only gives the weaker [63] bound on the second factor.

Either annihilator suffices. Then one still needs finite good-reduction injection to deduce that J_1(18)(Q) is finite and to pin it to Z/21. Bounded exponent by itself does not imply finiteness.

## 5.3 N25 is currently mischaracterized

X_1(25) is not genus zero. Its invariants are

```plain text
genus(X_1(25)) = 12,
Q-gonality(X_1(25)) = 5.
```

Therefore a degree-40 bivariate Tate factor is not discharged by parametrizing a conic or by a simple rational-root test.

There is promising known finite arithmetic: modern work proves J_1(25)(Q) has rank zero and treats X_1(25) by a local method at 3. But this is a genus-12 modular-Jacobian/local-reduction certificate, not an N18 genus-two clone and not a gcd/resultant calculation.

A valid Lean block needs one of:

```plain text
A. a direct local obstruction on every noncuspidal rational Tate solution;
B. a finite cuspidal-Jacobian/reduction argument specialized to X_1(25);
C. an explicit intermediate quotient whose rational points can be enumerated.
```

Until that concrete certificate is written, N25 is unresolved.

## 5.4 N49 is also not a direct rank-zero elliptic case

```plain text
genus(X_1(49)) = 69.
```

The published gonality table gives

```plain text
Q-gonality(X_1(49)) ≤ 21
```

and does not make it a genus-one or genus-two problem.

Mapping to X_0(49) is useful, but X_0(49) alone cannot close the problem: rational cyclic 49-isogenies exist, whereas a rational generator of the kernel is stronger data. One must analyze the finite fibers/kernel characters over the rational points of X_0(49) or use an explicit intermediate quotient.

Thus

```plain text
"order 49 -> rank-zero curve 49a1"
```

is an incomplete proof unless it includes this fiber analysis.

N49 may still be a finite bespoke computation, but it is not currently discharged by the stated sentence.

# 7. The prime tail p≥17

This is the unique genuinely uniform and research-scale block.

A rational point of order p gives a noncuspidal rational point on X_1(p). Ruling these out uniformly for all p≥17 is the core modular-curve theorem in Mazur's work.

There are alternative packages, but none is elementary:

## 7.1 Mazur's Eisenstein-ideal route

This is the classical proof. It uses the Eisenstein quotient/ideal, Néron models, Hecke operators, cuspidal divisors, and formal immersion/reduction arguments.

## 7.2 Winding quotient / Kamienny–Merel–Parent route

This can be viewed as a modern replacement. It uses modular symbols, a finite-rank quotient of a modular Jacobian, formal immersion at a cusp, and deep nonvanishing/input on modular forms. It is not a fixed-curve computation.

## 7.3 Make it finite using an effective uniform bound

Parent gives an explicit bound depending on the field degree. Specializing to Q makes the set of primes finite in principle. But:

```plain text
- the effective-bound theorem itself is deep modular-curve machinery;
- the resulting finite list is large;
- checking thousands of high-genus X_1(p) curves is not a practical replacement.
```

Oesterlé/Kamienny-style sharper prime bounds specialized to degree one essentially prove the desired prime theorem directly; formalizing their proof is still the same research-scale modular-symbol/formal-immersion project.

## 7.4 Use Mazur's rational-isogeny theorem first

A rational p-torsion point gives a rational p-isogeny, so the rational-isogeny classification reduces the tail to finitely many exceptional prime degrees. But the rational-isogeny theorem is itself another major Mazur theorem using comparable Eisenstein-ideal machinery. It repackages the mountain; it does not remove it.

Conclusion: there is no known elementary finite fixed-curve replacement for the tail. A fully axiom-free proof must formalize one of these uniform modular-curve arguments.

Recommended isolated interface:

```javascript
theorem no_rational_prime_torsion_ge_17
    (E : WeierstrassCurve ℚ) [E.IsElliptic]
    {p : ℕ} (hp : Nat.Prime p) (h17 : 17 ≤ p) :
    ¬ HasRationalPointOfOrder E p := by
  -- Eisenstein/winding quotient and formal immersion.
  sorry
```

Keep this theorem separate from all fixed-order arithmetic.

# 8. Layer 3: Mordell–Weil finite generation is avoidable

For the cardinality theorem, mordell_weil_fg should be removed from the dependency graph.

Use instead:

```plain text
two good reductions -> rational torsion finite;
finite abelian structure -> two invariant factors;
E[n](Qbar) has rank two;
Weil pairing -> first invariant factor ≤2;
fixed obstruction list + two isogeny bridges -> cardinality ≤16.
```

No free part of E(Q) is ever inspected. No rank of the arbitrary input elliptic curve is needed.

Mordell–Weil finite generation remains a useful independent theorem, but it is not logically necessary for Mazur_statement as currently formulated.
