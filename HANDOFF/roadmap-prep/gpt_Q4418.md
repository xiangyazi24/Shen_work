# Q4418: endgame chain after chem-div hH2

## Short verdict

Supplying

```javascript
hH2    : ∀ s, 0 ≤ s → IntervalWeakH2Neumann (coupledChemDivSourceLift p u s)
hdecay : ∀ s, 0 ≤ s → ∀ k, 1 ≤ k →
  |cosineCoeffs (coupledChemDivSourceLift p u s) k|
    ≤ Cchem / ((k : ℝ) * Real.pi)^2
hzero  : ∀ s, 0 ≤ s →
  |cosineCoeffs (coupledChemDivSourceLift p u s) 0| ≤ Cchem
```

is not the complete HSpectral endgame. It closes only the spatial H²_N / quadratic-decay part of the chem-div source coefficient producer.

The remaining carried hypotheses fall into four blocks:

1. chem-div time-regularity and factor-joint-C² inputs;

1. logistic-source time-C¹ and Fourier data;

1. total B-form source assembly / source-split rewrites;

1. the outer HSpectral leaves: restart representation, Fourier data, resolver witness.

So: hH2 is the hardest-looking spatial leaf, but it is not the only remaining obligation.

## 1. Immediate chem-div producer surface

The relevant producer is:

```javascript
coupledChemDivSource_timeC1_of_factorJointC2Inputs
```

in IntervalChemDivFluxJointC2Producer.lean. Its effective input list is:

```javascript
(Cchem : ℝ) (hCchem : 0 ≤ Cchem)
(hH2 : ∀ s, 0 ≤ s → IntervalWeakH2Neumann (coupledChemDivSourceLift p u s))
(hdecay : ∀ s, 0 ≤ s → ∀ k : ℕ, 1 ≤ k →
  |cosineCoeffs (coupledChemDivSourceLift p u s) k|
    ≤ Cchem / ((k : ℝ) * Real.pi) ^ 2)
(hzero : ∀ s, 0 ≤ s →
  |cosineCoeffs (coupledChemDivSourceLift p u s) 0| ≤ Cchem)
(H : CoupledChemDivFluxFactorJointC2Inputs p u)
(hadotcont : ∀ n, Continuous (fun s => coupledChemDivAdot p u s n))
(MchemDot : ℝ)
(hMdot : ∀ s, 0 ≤ s → ∀ n, |coupledChemDivAdot p u s n| ≤ MchemDot)
```

and the output is:

```javascript
DuhamelSourceTimeC1 (coupledChemDivSourceCoeffs p u)
```

### What hH2 + hdecay + hzero do

They discharge only the coefficient envelope side:

```javascript
|source_k(s)| ≤ Cchem / (kπ)^2
```

plus zero-mode control. hdecay can be obtained from hH2 by the existing weak-H² decay lemma, and hzero is usually a separate easy L¹ / zero-mode bound.

### What they do not discharge

They do not supply:

```javascript
H : CoupledChemDivFluxFactorJointC2Inputs p u
hadotcont : ∀ n, Continuous (fun s => coupledChemDivAdot p u s n)
MchemDot : ℝ
hMdot : ∀ s, 0 ≤ s → ∀ n, |coupledChemDivAdot p u s n| ≤ MchemDot
```

These are still real obligations.

## 2. Is H : CoupledChemDivFluxFactorJointC2Inputs produced or carried?

It is carried, unless you invoke the FAC lane.

There is a producer:

```javascript
coupledChemDivFluxFactorJointC2Inputs_of_FACInputs :
  CoupledChemDivFluxFactorFACInputs p u →
    CoupledChemDivFluxFactorJointC2Inputs p u
```

So the actual remaining question becomes whether you can produce:

```javascript
CoupledChemDivFluxFactorFACInputs p u
```

That structure carries a resolver package:

```javascript
∃ U : ℝ,
  ResolverHasSpectralAgreementC2Coeff U (coupledChemicalConcentration p u) ∧
  ∀ τ : ℝ, ∃ δ : ℝ, FACLocalSlabInputs p u U τ δ
```

and FACLocalSlabInputs includes the following local slab facts:

```javascript
0 < δ
∀ s, s ∈ Metric.ball τ δ → 0 < s ∧ s < U
∀ᶠ s in 𝓝 τ, ContinuousOn (coupledChemDivSourceLift p u s) (Icc 0 1)
∀ s, Continuous (u s)
∀ s x, 0 ≤ u s x
∀ x ∈ Ioo 0 1, ∀ s ∈ Metric.ball τ δ,
  ContDiffAt ℝ 2 (fun q => intervalDomainLift (u q.1) q.2) (s,x)
∀ x ∈ Ioo 0 1, ∀ s ∈ Metric.ball τ δ,
  (fun y => coupledChemDivFluxTimeDerivativeLift p u s y)
    =ᶠ[𝓝 x]
  (fun y => fderiv ℝ (Function.uncurry (coupledChemDivFluxLift p u)) (s,y) (1,0))
ContinuousOn
  (Function.uncurry (coupledChemDivTimeDerivativeLift p u))
  (Icc (τ - δ) (τ + δ) ×ˢ Icc 0 1)
```

The FAC producer then derives the resolver value/gradient joint C² fields and the 1+v>0 denominator floor, but the slice u joint-C², time-partial bridge, and chem-div time derivative continuity remain in the FAC package.

So H is not automatic from hH2; it is automatic only after the FAC package is built.

## 3. hadotcont and MchemDot

These are still carried explicitly by the chem-div source-time-C¹ producer.

They are:

```javascript
hadotcont : ∀ n, Continuous (fun s => coupledChemDivAdot p u s n)
MchemDot : ℝ
hMdot : ∀ s, 0 ≤ s → ∀ n, |coupledChemDivAdot p u s n| ≤ MchemDot
```

The factor-joint-C²/FAC inputs should make these provable by a coefficient-level dominated differentiation argument, but the producer does not currently synthesize them from H. They remain leaf obligations unless a wrapper is added.

Important distinction:

The raw lower-level producer

```javascript
coupledChemDivSource_duhamelSourceTimeC1
```

also carries an explicit coefficient derivative identity:

```javascript
hderiv : ∀ s n,
  HasDerivAt
    (fun r => cosineCoeffs (coupledChemDivSourceLift p u r) n)
    (adot s n) s
```

The factor-joint-C² route is designed to discharge this derivative identity through the outer-commute machinery. But it still leaves hadotcont and hMdot as explicit inputs.

## 4. Logistic-source side

Yes, the reaction/logistic branch has its own parallel obligations.

To assemble total B-form source time-C¹, you need:

```javascript
hlog  : DuhamelSourceTimeC1 (coupledLogisticSourceCoeffs p u)
hchem : DuhamelSourceTimeC1 (coupledChemDivSourceCoeffs p u)
```

and then the existing combiner gives:

```javascript
coupledChemicalSource_duhamelSourceTimeC1 hlog hchem hsplit
```

The representation-based logistic producer exists:

```javascript
logisticSource_duhamelSourceTimeC1_of_representation
```

but it carries its own data:

```javascript
bc : ℝ → ℕ → ℝ
hbsum : ∀ σ, Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|)
hagree : ∀ σ,
  Set.EqOn (intervalDomainLift (w σ))
    (fun x => ∑' n, bc σ n * cosineMode n x) (Set.Icc 0 1)
hpos : ∀ σ, ∀ x ∈ Set.Icc 0 1, 0 < intervalDomainLift (w σ) x
hC : 0 ≤ C
hdecay : ∀ σ, 0 ≤ σ → ∀ k, 1 ≤ k →
  |cosineCoeffs (logisticSourceFun p.a p.b p.α (intervalDomainLift (w σ))) k|
    ≤ C / ((k : ℝ) * Real.pi)^2
ha0 : ∀ σ, 0 ≤ σ →
  |cosineCoeffs (logisticSourceFun p.a p.b p.α (intervalDomainLift (w σ))) 0| ≤ C
hderiv : ∀ σ n, HasDerivAt ...
hadotcont : ∀ n, Continuous (fun σ => adot σ n)
hMdot : ∀ σ, 0 ≤ σ → ∀ n, |adot σ n| ≤ Mdot
```

The weak-H² certificate for the logistic source is internally built from representation plus positivity, but the decay, zero-mode, and time-derivative fields still need to be supplied or wrapped.

So the logistic side is not free just because the chem-div hH2 is solved.

## 5. Total source / B-form source rewrite

The chem-div and logistic sources combine by:

```javascript
coupledChemicalSource_duhamelSourceTimeC1
```

with source split:

```javascript
coupledChemicalSourceCoeffs p u =
  fun s n => -(p.χ₀ * coupledChemDivSourceCoeffs p u s n)
    + coupledLogisticSourceCoeffs p u s n
```

But the HSpectral leaf wants:

```javascript
hsrcB : DuhamelSourceTimeC1 (bFormSourceCoeffs p S.u)
```

In hasBFormSpectralPdeAgreement_of_leaves, the B-form source is used definitionally through:

```javascript
bFormSourceCoeffs p S.u t₀ n =
  coupledLogisticSourceCoeffs p S.u t₀ n
    - p.χ₀ * coupledChemDivSourceCoeffs p S.u t₀ n
```

So there must be either a definitional simpa/rfl bridge or an explicit congruence lemma moving the total coupled source result into the exact bFormSourceCoeffs spelling.

This is usually small, but it is another required wiring step.

## 6. HSpectral leaf surface after hsrcB

Even after you have:

```javascript
hsrcB : DuhamelSourceTimeC1 (bFormSourceCoeffs p S.u)
```

HSpectral is not complete. The generic producer consumes:

```javascript
BFormMildSpectralLeaves p S
```

whose fields are:

```javascript
hsrcB : DuhamelSourceTimeC1 (bFormSourceCoeffs p S.u)
hB_restart : RestartRepresentation S
hlogData : ∀ t, 0 < t → t < S.T → LogisticCosineFourierData p S.u t
hchemData : ∀ t, 0 < t → t < S.T →
  ChemDivCosineFourierData p (S.u t)
    (coupledChemicalConcentration p S.u t)
hResolverWitness : ResolverSourceWitnessFromMild p S
```

Then the chain is:

```javascript
BFormMildSpectralLeaves p S
  → spectralFrontier_of_leaves
  → bootstrapData_of_positiveTime_frontier
  → BFormMildSpectralBootstrapData p S
```

### Meaning of the remaining HSpectral leaves

- hB_restart is the restart cosine representation of S.u near every interior time. It is independent of chem-div hH2.

- hlogData is the pointwise Fourier convergence / representative data for the logistic source. It is related to the logistic weak-H²/decay work, but it is a separate field.

- hchemData is the pointwise Fourier convergence / representative data for the chem-div source. The chem-div hH2 can produce the summability side, but you still need the continuous representative and agreement with the physical source on [0,1].

- hResolverWitness is the resolver-source witness. It is separate from chem-div hH2; it normally comes from the power-source / resolver-source side.

## 8. What is genuinely next after hH2?

The next highest-risk items are not the scalar Cchem or hzero; those are small.

The real remaining risks are:

1. CoupledChemDivFluxFactorFACInputs p u, especially the joint C² of u and the time-partial bridge for the flux.

1. The coefficient time-derivative continuity/bound:

```javascript
hadotcont
hMdot
```

1. The logistic-source mirror obligations.

1. hB_restart, if it is not already available from the mild/Duhamel representation layer for the same S.

1. hchemData and hlogData, because HSpectral requires pointwise Fourier convergence data, not only coefficient envelopes.

## Bottom line

hH2 + hdecay + hzero gives the spatial quadratic coefficient control for the chem-div source, and that is a major crux. But the path to HSpectral still needs the time-C¹ coefficient controls, the factor-joint-C² or FAC package, the logistic source producer, the total-source rewrite into bFormSourceCoeffs, and the remaining BFormMildSpectralLeaves fields. The complete endgame should therefore track hH2 as one branch of hsrcB, not as the whole HSpectral leaf.
