# 1. The local 3-adic formal-group inequality

Let

```plain text
K = L_pi,
O = O_K,
m = (pi),
ord_pi(pi)=1.
```

Because L/Q is totally ramified of degree 3 at 3,

```plain text

```

Let F(X,Y) be the one-dimensional formal group of the good integral Weierstrass model of E0 at pi, with local parameter

```plain text
z=-x/y.
```

For a fixed curve, do not build a general elliptic formal-group library. Prove directly that the multiplication-by-three series has the shape

```plain text
[3]_F(T) = 3*T + T^2*A(T),
```

where

```plain text
A(T) in O[[T]].
```

Equivalently, for every z in m, evaluation converges and

```plain text
[3]_F(z)=3*z+z^2*A(z),
ord_pi(A(z)) >= 0.
```

## Exact valuation inequality

Use the zero-aware valuation

```plain text
ord_pi_top : K -> WithTop Z,
ord_pi_top(0)=top.
```

Then the precise reusable inequality is

```plain text
ord_pi_top ([3]_F(z))
  >= min (ord_pi_top(z)+3) (2*ord_pi_top(z)).
```

For a nonzero formal parameter with

```plain text
r=ord_pi(z)>=1,
```

this becomes

```plain text
ord_pi([3]_F(z))
  >= min (r+3) (2*r)
   = r + min 3 r
  >= r+1
  > r.
```

This is the correct inequality accounting for ramification index e=3.

With the rationally normalized valuation v_3(3)=1, one instead has

```plain text
v_3([3]_F(z))
  >= min (v_3(z)+1) (2*v_3(z)),
```

and because v_3(z)>=1/3, the guaranteed increase is at least 1/3. For Lean, the uniformizer-normalized integer valuation is much easier.

## Compile-shaped fixed-series statement

```javascript

/-- Uniformizer-normalized order, with zero sent to top. -/

/-- The fixed E0 multiplication-by-three parameter function. -/

    (0 : WithTop Z) <= ordPi (tail z)
  ord_three : ordPi (3 : K) = 3
  ord_mul : forall x y, ordPi (x*y) = ordPi x + ordPi y
  ord_add_lower : forall x y,
    min (ordPi x) (ordPi y) <= ordPi (x+y)
```

The actual implementation should use the repository's fixed valuation API rather than storing the elementary valuation laws as fields; the structure above records the mathematical interface.

The resulting theorem is:

```javascript
theorem tripleParam_ord_ge
    (cert : TripleSeriesCertificate ordPi tripleParam)
    {z : K}
    (hz : (1 : WithTop Z) <= ordPi z) :
    min (ordPi z + 3) (2 * ordPi z)
      <= ordPi (tripleParam z) := by
  rw [cert.formula]
  apply le_trans ?_ (cert.ord_add_lower (3*z) (z^2*cert.tail z))
  -- `ord(3*z)=ord(z)+3`.
  -- `ord(z^2*tail z)>=2*ord(z)`.
  simp [cert.ord_mul, cert.ord_three, cert.tail_integral hz]
```

The orientation may be written in the more readable form

```javascript
ordPi (tripleParam z)
  >= min (ordPi z + 3) (2 * ordPi z).
```

Corollary:

```javascript
theorem tripleParam_strict_growth
    (cert : TripleSeriesCertificate ordPi tripleParam)
    {z : K}

    (hz : (1 : WithTop Z) <= ordPi z) :

  have h := tripleParam_ord_ge ordPi tripleParam cert hz

  -- `r+1 <= min (r+3) (2*r)`.
```

# 2. The shared filtration theorem

The cleanest reusable layer does not mention valuations. It only records that multiplication by m raises a separated filtration.

```javascript
section RaisingFiltration

variable {K : Type*} [AddCommGroup K]

structure RaisingFiltration (m : N) where
  level : N -> K -> Prop
  level_zero : forall x, level 0 x
  raise : forall n x,
    level n x -> level (n+1) (m • x)
  separated : forall x,
    (forall n, level n x) -> x = 0
```

For the formal kernel, instantiate

```plain text
level(n,P) :<=> n <= ord_pi_top(z(P)).
```

Here z(O)=0, so ord_pi_top(z(O))=top. Every formal-kernel point is in level 0; every nonzero formal-kernel point is in level 1; the valuation inequality proves raise; and unbounded valuation implies z(P)=0, hence P=O.

## Iterated raising

```javascript
theorem RaisingFiltration.pow_smul_mem
    {m : N}
    (F : RaisingFiltration (K := K) m) :
    forall n x, F.level n ((m^n) • x) := by
  intro n
  induction n with
  | zero =>
      intro x
      simpa using F.level_zero x
  | succ n ih =>
      intro x
      have h := F.raise n ((m^n) • x) (ih x)
      simpa [pow_succ, mul_nsmul, N.mul_comm, N.mul_left_comm,
        N.mul_assoc] using h
```

## Infinite divisibility is impossible

```javascript
def InfinitelyDivisible (m : N) (x : K) : Prop :=
  forall n : N, exists q : K, (m^n) • q = x

theorem RaisingFiltration.eq_zero_of_infinitelyDivisible
    {m : N}
    (F : RaisingFiltration (K := K) m)
    {x : K}
    (hx : InfinitelyDivisible m x) :
    x = 0 := by
  apply F.separated
  intro n
  obtain ⟨q, hq⟩ := hx n
  rw [← hq]
  exact F.pow_smul_mem n q
```

This theorem is the exact shared filtration core for the 2-adic and 3-adic arguments.

# 4. Iterating weak descent

This is pure additive group theory and should be shared with N15/N21.

```javascript
section WeakDescent

variable {G : Type*} [AddCommGroup G]

lemma weakDescent_iterate
    (m : N)
    (H : AddSubgroup G)
    (hweak : forall x : G,
      exists h : H, exists y : G,
        x = (h : G) + m • y) :
    forall n : N, forall x : G,
      exists h : H, exists y : G,
        x = (h : G) + (m^n) • y := by
  intro n
  induction n with
  | zero =>
      intro x
      exact ⟨0, x, by simp⟩
  | succ n ih =>
      intro x
      obtain ⟨h, y, hxy⟩ := ih x
      obtain ⟨h', y', hy⟩ := hweak y
      refine ⟨h + (m^n) • h', y', ?_⟩
      rw [hxy, hy]
      -- The accumulated term lies in H because H is a subgroup.
      simp [nsmul_add, pow_succ, mul_nsmul,
        add_assoc, add_left_comm, add_comm]
```

The accumulated element is

```plain text
h_0 + m*h_1 + ... + m^(n-1)*h_(n-1),
```

which lies in H; no extra assumption is hidden here.

# 5. The general reduction-plus-separatedness annihilator

The reusable theorem is more general than the concrete number 21.

```javascript
theorem annihilator_of_weakDescent_reduction
    {G R : Type*}
    [AddCommGroup G] [AddCommGroup R]
    (m hExp rExp : N)
    (H : AddSubgroup G)
    (red : G ->+ R)
    (hweak : forall x : G,
      exists h : H, exists y : G,
        x = (h : G) + m • y)
    (hHExp : forall h : H, hExp • (h : G) = 0)
    (hRExp : forall r : R, rExp • r = 0)
    (hsep : forall z : red.ker,
      InfinitelyDivisibleIn m red.ker z -> z = 0) :
    forall x : G, (hExp*rExp) • x = 0 := by
  intro x
  let N := hExp*rExp

  have hNker (y : G) : red (N • y) = 0 := by
    rw [map_nsmul]
    calc
      N • red y = hExp • (rExp • red y) := by
        simp [N, mul_nsmul]
      _ = 0 := by simp [hRExp]

  let z : red.ker := ⟨N • x, hNker x⟩

  have hzdiv : InfinitelyDivisibleIn m red.ker z := by
    intro n
    obtain ⟨h, y, hxy⟩ := weakDescent_iterate m H hweak n x
    let q : red.ker := ⟨N • y, hNker y⟩
    refine ⟨q, Subtype.ext ?_⟩

    have hNh : N • (h : G) = 0 := by
      calc
        N • (h : G) = rExp • (hExp • (h : G)) := by
          simp [N, mul_nsmul, N.mul_comm]
        _ = 0 := by simp [hHExp]

    have hxN := congrArg (fun w : G => N • w) hxy
    -- `N • x = (m^n) • (N • y)` after killing the H term.
    simpa [z, q, nsmul_add, hNh, mul_nsmul,
      N.mul_comm, N.mul_left_comm, N.mul_assoc] using hxN.symm

  have hz : z = 0 := hsep z hzdiv
  exact by
    simpa [z] using congrArg Subtype.val hz
```

The proof uses the product hExp*rExp; no coprimality or divisibility condition is needed.

# 7. Why the proof works

For each n, iterated weak descent gives

```plain text
x = t_n + 3^n*y_n,
t_n in H.
```

Multiplying by 21 gives

```plain text
21*x = 21*t_n + 3^n*(21*y_n)
     = 3^n*(21*y_n),
```

because 3*t_n=0.

Moreover,

```plain text
red(21*y_n)=3*red(7*y_n)=0,
```

because the reduction group has exponent 7. Therefore every divisor 21*y_n lies in the formal kernel. Thus 21*x is infinitely 3-divisible inside the formal kernel, and formal separatedness implies

```plain text
21*x=0.
```

This is the entire argument.

# 10. The push-pull annihilator

Use either a product formulation or separate maps.

## Product formulation

Let

```javascript
Phi : A ->+ B,
Psi : B ->+ A,
```

and assume

```plain text
Psi(Phi x)=d•x,
N•b=0 for every b in B.
```

Then N*d kills A.

```javascript
theorem annihilator_of_pushPull
    {A B : Type*}
    [AddCommGroup A] [AddCommGroup B]
    (Phi : A ->+ B)
    (Psi : B ->+ A)
    (d N : N)
    (hcomp : forall x : A,
      Psi (Phi x) = d • x)
    (hB : forall y : B, N • y = 0) :
    forall x : A, (N*d) • x = 0 := by
  intro x
  calc
    (N*d) • x = N • (d • x) := by simp [mul_nsmul]
    _ = N • Psi (Phi x) := by rw [hcomp]
    _ = Psi (N • Phi x) := by simp
    _ = 0 := by simp [hB]
```

## Product exponent

Take

```plain text
B=E0(L) x Ehat0(L).
```

Since 21 divides 63, the product is killed by 63.

# 11. Rank-zero conclusion without finite generation

The best final theorem is the bounded exponent statement itself:

```javascript
theorem J1_18_Q_boundedExponent :
  forall D : J1_18_Q, 126 • D = 0
```

This is strictly stronger than rank zero.

To avoid depending on whichever rank API Mathlib later adopts, define the project-level predicate

```javascript
def HasBoundedExponent
    (G : Type*) [AddCommGroup G] : Prop :=
  exists N : N, 0 < N ∧ forall x : G, N • x = 0
```

Then:

```javascript
theorem Pic0C_Q_hasBoundedExponent :
    HasBoundedExponent Pic0C_Q :=
  ⟨126, by norm_num, Pic0C_Q_exponent_oneTwentySix⟩
```

Any standard Mordell-Weil rank notion is zero for a bounded-exponent group. A completely elementary certificate is that no copy of Z embeds:

```javascript
theorem not_injective_int_hom_of_boundedExponent
    {G : Type*} [AddCommGroup G]
    {N : N}
    (hN : 0 < N)
    (hkill : forall x : G, N • x = 0)
    (f : Z ->+ G) :
    not Function.Injective f := by
  intro hinj
  have hf : f (N : Z) = f 0 := by
    simpa using hkill (f 1)
  have : (N : Z) = 0 := hinj hf
  omega
```

Thus there is no free rank-one subgroup, and the rationalized group is zero. No finite-generation premise is required.
