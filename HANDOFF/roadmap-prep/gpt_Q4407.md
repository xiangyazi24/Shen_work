# 5. Why Theorem_1_1 → Proposition_1_2 is not pure wiring

There are three separate interface gaps.

## Gap A: eventual boundedness

Proposition_1_2 asks for:

```javascript
IsPaper2Bounded intervalDomain u
```

which is an atTop assertion. Theorem_1_1 exposes only a finite-horizon estimate. The repository explicitly records this mismatch and proves the generic non-implication theorem.

For intervalDomain, Step 0 therefore needs a theorem such as:

```javascript
theorem paper2_chiNonpos_eventualSupBound
   (p : CM2Params) :
   p.χ₀ ≤ 0 → 1 ≤ p.m →
     ∀ u₀, PaperPositiveInitialDatum intervalDomain u₀ →
       ∀ u v,
         IsPaper2GlobalClassicalSolution intervalDomain p u v →
         InitialTrace intervalDomain u₀ u →
           IsPaper2Bounded intervalDomain u
```

or the stronger PositiveInitialDatum version matching the existing interval frontier.

## Gap B: parameter-branch coverage

Proposition_1_2 has no explicit a,b branch hypothesis. It quantifies over the fixed p under only

```javascript
p.χ₀ ≤ 0
1 ≤ p.m.
```

Theorem_1_1, however, provides conclusions only under either

```plain text
0 < a and 0 < b
```

or

```plain text
a = 0 and b = 0.
```

The mixed regimes a=0<b and b=0<a are not covered by the two fields of Theorem_1_1. A full bridge to the current unrestricted Proposition_1_2 must cover those regimes separately, or the Paper 3 target must be intentionally restricted to the strict-logistic/minimal cases.

## Gap C: initial-data class of the stronger residual

Theorem_1_1 quantifies over PaperPositiveInitialDatum, while NegativeSensitivityGlobalEventualBound quantifies over the weaker PositiveInitialDatum. Therefore Theorem_1_1 cannot directly produce that stronger residual merely by unfolding definitions.

There are two clean choices:

1. prove the stronger interval theorem for every PositiveInitialDatum; then reuse the existing residual/assembly unchanged;

1. bypass the stronger residual and prove Proposition_1_2 intervalDomain p directly for PaperPositiveInitialDatum.

Do not silently use PositiveInitialDatum → PaperPositiveInitialDatum; that implication is false in general. Interior-positive data may have zero boundary infimum.
