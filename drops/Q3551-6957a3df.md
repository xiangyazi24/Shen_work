# R2 zero-start source audit A

Audited commit `ca5594a4` on `xiangyazi24/Shen_work`. Target package:

```lean
H1ZeroStartClosedPrimitiveC1SignBefore u v T
```

It asks for every zero-start closed slab `Set.Icc 0 b ×ˢ Set.Icc 0 1`, `0 ≤ b`, `b < T`:

1. joint continuity of `intervalDomainLift (u t) x`,
2. joint continuity of `intervalDomainLift (v t) x`,
3. joint continuity of `deriv (intervalDomainLift (u t)) x`,
4. joint continuity of `deriv (intervalDomainLift (v t)) x`,
5. `0 < intervalDomainLift (u t) x`,
6. `0 ≤ intervalDomainLift (v t) x`.

## Verdict

There is **no general existing producer** of `H1ZeroStartClosedPrimitiveC1SignBefore u v T` from the construction-level or global data currently exposed in the repo.

There is a **constant-solution special case** that is source-producible from existing constant-function lemmas in `ShenWork/PDE/IntervalDomainExistence.lean`, but I did not find an existing theorem with the exact target conclusion. A small bridge theorem can be added outside the active sqrt/reaction-bound/sigma-ladder files.

The strongest obstruction for the B-form/conjugate Picard route is not merely “missing theorem”: the raw B-form Picard output is definitionally zero at `t = 0`. In `ShenWork/Paper2/IntervalConjugatePicard.lean`,

```lean
def conjugatePicardLimit (p : CM2Params) (u₀ : intervalDomainPoint → ℝ) (T : ℝ)
    (t : ℝ) (x : intervalDomainPoint) : ℝ :=
  if 0 < t ∧ t ≤ T then
    atTop.limUnder (fun n => conjugatePicardIter p u₀ n t x)
  else 0
```

So for the raw local output

```lean
u := conjugatePicardLimit p u₀ DB.T
```

the target field `u_pos0` fails at `t = 0`, because `u 0 x = 0`. Any zero-start producer for the actual positive initial datum must first use a reinitialized trajectory whose `t = 0` slice is the datum, and then prove closed-slab continuity of the value and first spatial derivative fields through that reinitialization.

## Why classical/global solution APIs do not suffice

`ShenWork/PDE/IntervalDomain.lean`, `intervalDomainClassicalRegularity`, gives the relevant solution-field continuity only on

```lean
Set.Ioo (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1
```

and the closed-spatial `C²`/endpoint Neumann fields only for `t ∈ Set.Ioo 0 T`. Its “closed-slab” wording is closed in **space**, not closed at time zero.

`ShenWork/Paper2/Statements.lean`, `IsPaper2ClassicalSolution`, likewise exposes positivity/nonnegativity only for positive times:

```lean
(∀ t x, 0 < t → t < T → 0 < u t x)
(∀ t x, 0 < t → t < T → 0 ≤ v t x)
```

The accessors `IsPaper2ClassicalSolution.u_pos'`, `.u_pos`, and `.v_nonneg` all require `0 < t`. `IsPaper2GlobalClassicalSolution.classical` only supplies such strict-positive-time classical solutions for each positive horizon; its `.u_pos`, `.pde_u`, `.pde_v`, and `.neumann` accessors also require `0 < t`.

Therefore neither `IsPaper2ClassicalSolution` nor `IsPaper2GlobalClassicalSolution` can honestly produce the zero-start package, whose slabs include `t = 0`.

## B-form/conjugate Picard construction data

### `ConjugateMildExistenceData` / `conjugateMildSolutionData_of_data`

`ShenWork/Paper2/IntervalConjugatePicard.lean` has construction-level positivity and continuity data, but all of it is positive-time:

- `ConjugateMildExistenceData.hmapsTo_pos` gives positivity for `0 < t`, `t ≤ T` after applying the Picard map.
- `conjugateMildSolutionData_of_data` exposes `hpos`, `hnonneg`, `hcont`, and `hmeas`; its `hpos` and `hnonneg` are for `0 < t`, `t ≤ T`, and `hcont` is `HasContinuousSlices`, i.e. per-time spatial continuity for `0 < t`, `t ≤ T`.
- It does not expose closed-zero-slab joint continuity in `(t,x)` and does not expose first spatial derivative continuity through `t = 0`.

Most importantly, the raw `conjugatePicardLimit` is zero at `t = 0`, so no bridge from this raw trajectory can prove target `u_pos0`.

```lean

  ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
    D.supNorm (fun x => u t x - u₀ x) < ε
```

It does not assert `u 0 = u₀`, does not assert continuity at `t = 0`, and gives no first-spatial-derivative trace.

## Weakest honest source record/theorem needed

The weakest honest source record is essentially the target record itself. A weaker decomposition through `InitialTrace` is not enough; the target needs pointwise closed-slab continuity of both values and first spatial derivatives and actual signs at `t = 0`.

If the construction route wants a more construction-facing record, it should explicitly include the same six obligations, preferably after defining a reinitialized trajectory whose zero slice is not the raw `conjugatePicardLimit` zero value:

```lean
import ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents

open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2

closed-slab value, first-spatial-derivative, and sign facts needed at `t = 0`. -/
structure H1ZeroStartPrimitiveC1TraceSource
    (u v : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  u_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ContinuousOn
      (Function.uncurry (fun t x => intervalDomainLift (u t) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)
  v_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ContinuousOn
      (Function.uncurry (fun t x => intervalDomainLift (v t) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)
  ux_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ContinuousOn
      (Function.uncurry
        (fun t x => deriv (intervalDomainLift (u t)) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)
  vx_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ContinuousOn
      (Function.uncurry
        (fun t x => deriv (intervalDomainLift (v t)) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)
  u_pos_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ∀ z ∈ Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1,
      0 < Function.uncurry
        (fun t x => intervalDomainLift (u t) x) z
  v_nonneg_closed : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ∀ z ∈ Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1,

        (fun t x => intervalDomainLift (v t) x) z

theorem H1ZeroStartClosedPrimitiveC1SignBefore_of_traceSource
```

## Constant-solution special case

The constant-in-time/space solutions are the only currently source-visible special case where the target package is honestly bridgeable.

Relevant existing names in `ShenWork/PDE/IntervalDomainExistence.lean`:

- `ellipticV`, `ellipticV_pos`;
- `intervalDomainLift_const`;
- `intervalDomainLift_const_deriv_zero`;
- `intervalDomainLift_const_deriv_endpoint_zero`;
- `intervalDomainLift_const_contDiffOn_Icc`;
- `constantInTime_classicalRegularity`;
- `equilibrium_isPaper2ClassicalSolution` for `0 < p.a`, `0 < p.b`;
- `zeroReaction_isPaper2ClassicalSolution` for `p.a = 0`, `p.b = 0`;
- `constantSolution_globalExistence` and `constantSolution_initialTrace`.

Smallest bridge skeleton for the constant package:

```lean
import ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents
import ShenWork.PDE.IntervalDomainExistence

open MeasureTheory Set
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents
open ShenWork.IntervalDomainExistence

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents

private lemma intervalDomainLift_const_deriv_zero_Icc (c : ℝ) :
    Set.EqOn
      (fun x : ℝ => deriv (intervalDomainLift
        (fun _ : intervalDomainPoint => c)) x)
      (fun _ : ℝ => (0 : ℝ))
      (Set.Icc (0 : ℝ) 1) := by
  intro x hx
  by_cases h0 : x = 0
  · subst x
    exact (intervalDomainLift_const_deriv_endpoint_zero c).1
  by_cases h1 : x = 1
  · subst x
    exact (intervalDomainLift_const_deriv_endpoint_zero c).2
  have hxIoo : x ∈ Set.Ioo (0 : ℝ) 1 :=
    ⟨lt_of_le_of_ne hx.1 (Ne.symm h0), lt_of_le_of_ne hx.2 h1⟩
  exact intervalDomainLift_const_deriv_zero c hxIoo

private theorem const_lift_closed_zero_slab_continuousOn
    (c b : ℝ) :
    ContinuousOn
      (Function.uncurry
        (fun (_t : ℝ) (x : ℝ) =>
          intervalDomainLift (fun _ : intervalDomainPoint => c) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) := by
  refine (continuousOn_const :
    ContinuousOn (fun _ : ℝ × ℝ => c)
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)).congr ?_
  intro z hz
  rcases z with ⟨t, x⟩
  simp [Function.uncurry, intervalDomainLift_const, hz.2]

private theorem const_lift_deriv_closed_zero_slab_continuousOn
    (c b : ℝ) :
    ContinuousOn
      (Function.uncurry
        (fun (_t : ℝ) (x : ℝ) =>
          deriv (intervalDomainLift (fun _ : intervalDomainPoint => c)) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) := by
  refine (continuousOn_const :
    ContinuousOn (fun _ : ℝ × ℝ => (0 : ℝ))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1)).congr ?_
  intro z hz
  rcases z with ⟨t, x⟩
  exact (intervalDomainLift_const_deriv_zero_Icc c hx).symm

/-- Constant-in-time/space positive solution data produce the zero-start
primitive C¹/sign source package.  This is a special-case producer, not a general
B-form/construction producer. -/
theorem H1ZeroStartClosedPrimitiveC1SignBefore_const
    (p : CM2Params) {c T : ℝ} (hc : 0 < c) :
    H1ZeroStartClosedPrimitiveC1SignBefore
      (fun _ (_ : intervalDomainPoint) => c)
      (fun _ (_ : intervalDomainPoint) => ellipticV p c)
      T where
  u_cont0 := by
    intro b _hb _hbT
    simpa using const_lift_closed_zero_slab_continuousOn c b
  v_cont0 := by
    intro b _hb _hbT
    simpa using const_lift_closed_zero_slab_continuousOn (ellipticV p c) b
  ux_cont0 := by
    intro b _hb _hbT
    simpa using const_lift_deriv_closed_zero_slab_continuousOn c b
  vx_cont0 := by
    intro b _hb _hbT
    simpa using const_lift_deriv_closed_zero_slab_continuousOn (ellipticV p c) b
  u_pos0 := by
    intro b _hb _hbT z hz
    rcases z with ⟨t, x⟩
    simpa [Function.uncurry, intervalDomainLift_const, hz.2] using hc
  v_nonneg0 := by
    intro b _hb _hbT z hz
    rcases z with ⟨t, x⟩
    exact by
      simpa [Function.uncurry, intervalDomainLift_const, hz.2] using
        (ellipticV_pos p hc).le

end ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents
```

This special theorem covers the existing equilibrium cases by choosing:

```lean
c = (p.a / p.b) ^ (1 / p.α)
```

under `0 < p.a`, `0 < p.b`, and the zero-reaction constant case by choosing any `c > 0` under `p.a = 0`, `p.b = 0`. It does not address the general B-form/Picard construction.
