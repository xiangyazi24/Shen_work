# Q3432: restricted local H¹ seed from scalar DI + endpoint data

## Verdict

The restricted local H¹ seed

```lean
∃ Ylocal : ℝ,
  ∀ τ, τ ∈ Set.Ioc (0 : ℝ) 1 → τ < T →
    H1energy u τ ≤ Ylocal
```

is **honestly derivable from the scalar H¹ DI data**, but **not by compactness/continuity alone in the general `0 < T` case**.

For the full route, the smallest honest next Lean task is a **pure scalar Gronwall/FTC lemma**. No PDE assumption and no bounded-before input should be introduced.

A useful nuance: once `hDI : H1ScalarDIOnBefore u T A B` is already available, `H1InitialEndpointData u₀ u T` is not logically needed for existence of *some* local bound, because `hDI.hcont` already includes continuity of `H1energy u` on `[0,b]` for every `b < T`. The endpoint bundle is still useful to identify the initial value as `H1InitialEnergy u₀` via `hinit.compatible`, and it is used upstream to construct `hDI` in the current H¹ wiring.

## Source-grounded API facts

### `H1ScalarDIOnBefore` has exactly the scalar data needed

In `ShenWork/Paper2/IntervalChiNegH1AverageWiring.lean`, the scalar DI record is:

```lean
structure H1ScalarDIOnBefore
    (u : ℝ → intervalDomainPoint → ℝ) (T A B : ℝ) : Prop where
  hA : 0 ≤ A
  hB : 0 ≤ B
  hcont : ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
    ContinuousOn (H1energy u) (Set.Icc a b)
  hderivInt : ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
    IntervalIntegrable (fun r => deriv (H1energy u) r) volume a b
  hhasDerivRight : ∀ {a b r : ℝ}, 0 ≤ a → a ≤ b → b < T →
    r ∈ Set.Ioo a b →
      HasDerivWithinAt (H1energy u)
        (deriv (H1energy u) r) (Set.Ioi r) r
  hDI : ∀ r, 0 < r → r < T →
    deriv (H1energy u) r ≤ A * H1energy u r + B
```

These fields are enough for a scalar FTC/Gronwall argument on each interval `[0,τ]` with `τ < T`.

### Endpoint data supplies the initial value identification, not the scalar DI

In `IntervalChiNegH1InitialContinuity.lean`:

```lean
structure H1InitialEndpointData
    (u₀ : intervalDomainPoint → ℝ)
    (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  tendsto : H1InitialTraceEnergyTendsto u₀ u T
  compatible : H1InitialEnergyCompatibleAtZero u₀ u
```

where

```lean
H1InitialEnergyCompatibleAtZero u₀ u :=
  H1energy u 0 = H1InitialEnergy u₀
```

The local seed proof needs the starting value `H1energy u 0`. If the desired bound is expressed in terms of the prescribed datum, `hinit.compatible` rewrites it to `H1InitialEnergy u₀`. The `tendsto` field is not needed once `H1ScalarDIOnBefore` is already available.

## Why compactness alone is insufficient in general

If `1 < T`, then compactness/continuity is enough: `hDI.hcont` gives `ContinuousOn (H1energy u) (Set.Icc 0 1)`, and `Set.Icc 0 1` is compact.

But the route is for arbitrary `0 < T`. If `T ≤ 1`, the local set is essentially `(0,T)`, and `hDI.hcont` only gives continuity on each `[0,b]` with `b < T`; it does not give compactness on `[0,T]`. Without the differential inequality, a function can be continuous on every `[0,b<T]`, continuous at `0`, and still blow up as `t → T-`. Therefore the general theorem needs to use the scalar DI.

## Cheap compactness theorem, only for `1 < T`

This is already possible and does not need `H1InitialEndpointData`, but it does **not** solve the actual local-start frontier for small horizons.

```lean

open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy

namespace ShenWork.Paper2.IntervalChiNegH1AverageWiring

/-- Cheap large-horizon local seed: if `1 < T`, compactness of `[0,1]` and
`hDI.hcont` already bound the local H¹ energy.  This is not enough for general
`0 < T`. -/
theorem exists_H1_localSeed_of_scalarDI_compact_of_one_lt_T
    {u : ℝ → intervalDomainPoint → ℝ} {T A B : ℝ}
    (hT1 : (1 : ℝ) < T)
    (hDI : H1ScalarDIOnBefore u T A B) :
    ∃ Ylocal : ℝ,
      ∀ τ, τ ∈ Set.Ioc (0 : ℝ) 1 → τ < T →
        H1energy u τ ≤ Ylocal := by
  have hcont : ContinuousOn (H1energy u) (Set.Icc (0 : ℝ) 1) :=
    hDI.hcont (a := 0) (b := 1) (by norm_num) (by norm_num) hT1
  obtain ⟨Ylocal, hYlocal⟩ :=
    (isCompact_Icc : IsCompact (Set.Icc (0 : ℝ) 1)).exists_bound_of_continuousOn hcont
  refine ⟨Ylocal, ?_⟩
  intro τ hτ _hτT
  have hτIcc : τ ∈ Set.Icc (0 : ℝ) 1 := ⟨le_of_lt hτ.1, hτ.2⟩
  have hnorm := hYlocal τ hτIcc
  exact le_trans (le_abs_self (H1energy u τ)) (by
    simpa [Real.norm_eq_abs] using hnorm)

end ShenWork.Paper2.IntervalChiNegH1AverageWiring
```

The only possible local API adjustment is the exact shape returned by `IsCompact.exists_bound_of_continuousOn`, but this is the same compact-bound API already used elsewhere in the repo.

## Fields feeding the scalar proof

For the general theorem, the exact existing fields are:

* `hDI.hA : 0 ≤ A`
* `hDI.hB : 0 ≤ B`
* `hDI.hcont` on `[0,τ]`
* `hDI.hderivInt` on `[0,τ]`
* `hDI.hhasDerivRight` on `(0,τ)`
* `hDI.hDI : deriv (H1energy u) r ≤ A * H1energy u r + B`
* `H1energy_nonneg` for nonnegativity of the scalar unknown if the selected Gronwall API needs it
* optionally `hinit.compatible` to rewrite `H1energy u 0` to `H1InitialEnergy u₀`

`hinit.tendsto` is not used once `H1ScalarDIOnBefore` has already been produced.
