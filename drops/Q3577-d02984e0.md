## 1. The proposed weak gluing is not mathematically valid

The gluing route is false if the assumptions are only:

- joint continuity on the strict positive-time slab, e.g. `(0,T] × [0,1]` or `(0,T) × [0,1]`, and
- for every fixed `x`, a right limit as `t → 0+`.

Pointwise-in-`x` right limits do not imply joint continuity on the zero face. They also do not imply the `ContinuousOn` fields demanded by the current zero-start source records.

A standard counterexample on the closed spatial interval is

```text
f(t,x) = t*x / (t^2 + x^2)       for t > 0,
f(0,x) = 0.
```

For each fixed `x`, `f(t,x) → 0` as `t → 0+`; the zero-face trace is even the continuous function `0`. But along the curve `x = t`, `f(t,t) = 1/2`, so the extension is not continuous at `(0,0)`. Thus pointwise zero-face traces are too weak even with a continuous zero-face value.

The current source records require a stronger field. In `IntervalChiNegH1ZeroStartComponents.lean`, `H1ZeroStartClosedPrimitiveC1SignBefore` requires, for every `b` with `0 ≤ b` and `b < T`, closed-zero-slab continuity on

```lean
Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1
```

for `u`, `v`, `u_x`, and `v_x`, plus positivity/nonnegativity on that same closed slab. In `IntervalChiNegH1ZeroStartInitializedPrimitive.lean`, `H1ZeroStartInitializedPrimitiveC1SignSource` repeats those same closed-zero-slab fields and adds the source-facing guards

```lean

v_zero : v (0 : ℝ) = v₀
```

The existing theorem

```lean

```

is only a projection from the initialized source record to the p-free closed primitive record. It does not produce any zero-face continuity.

`P3MoserDxJointContinuity.lean` is also not a producer for the zero-start closed-slab fields. Its positive-time continuity theorems live on sets of the form

```lean
Set.Ioo (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1
```

The phrase “closed-slab” in that file is spatially closed, not closed at `t = 0`. Its file comment is consistent with this: the zero-time closed-slab primitive theorem remains an analytic frontier, not an assumption already discharged there.

Therefore a theorem named like

```lean

```

would be unsound if `zeroFace` means only pointwise limits at each `x`. To make such a gluing lemma true, the zero-face hypothesis must be genuinely joint, for example `ContinuousWithinAt` at every `(0,x)` relative to the closed slab, or a uniform/local-uniform-in-`x` convergence statement to a continuous zero-face trace. In Lean, the least ambiguous and safest field is the direct `ContinuousOn` field on the closed slab.

## 3. If a zero-face trace record is introduced, it must not be pointwise

A pointwise trace record like

```lean
∀ x ∈ Set.Icc (0 : ℝ) 1,
  Filter.Tendsto (fun t => F (t, x)) (𝓝[>] (0 : ℝ)) (𝓝 (F (0, x)))
```

is too weak and should not be used to prove a closed-slab `ContinuousOn` field.

    {F : ℝ × ℝ → ℝ} {b : ℝ}

      0 < z.1 →

  intro z hz
  rcases z with ⟨t, x⟩
  by_cases ht : t = 0
  · subst t
    exact hzero x hz.2
  · have ht_ne : (0 : ℝ) ≠ t := by
      intro h0t
      exact ht h0t.symm
    have ht_pos : 0 < t := lt_of_le_of_ne hz.1.1 ht_ne

More importantly, the zero-face side still needs joint relative continuity, or a separate uniform/local-uniform convergence theorem strong enough to imply it.

## 4. False or too-weak assumptions to flag

- “Strict-time joint continuity plus pointwise `t → 0+` limits” is false as a closed-slab continuity producer.
- “Zero-face trace” is ambiguous. If it means pointwise limits, it is too weak. If it means `ContinuousWithinAt` at each `(0,x)` relative to the closed slab, it is strong enough but is essentially the zero-face part of closed-slab continuity.
- Existing P3 Moser derivative continuity is time-open at zero. It cannot fill `ux_cont0` or `vx_cont0` for `Set.Icc 0 b ×ˢ Set.Icc 0 1` without additional zero-time work.
- `u_pos0` and `v_nonneg0` in the closed primitive records are closed-slab sign fields. Strict-positive-time sign information alone does not automatically give the sign at `t = 0`; the initialized zero slices must carry it.
- Continuity of `u` and `v` at the zero face does not imply continuity of the derivative fields `u_x` and `v_x` at the zero face. Those are separate fields in the current records and must be produced separately.

Bottom line: the honest next step is not a downstream production route. It is either the small packaging theorem above, or an explicitly named frontier record with joint zero-face/closed-slab fields and no producer. The actual analytic frontier remains proving the direct closed-zero-slab continuity/sign package for the general zero-start initialized primitive construction.
