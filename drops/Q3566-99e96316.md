## Verdict

The repo can already support a **non-circular constant special-case producer** for the new source-facing record

```lean
H1ZeroStartLiteralUPDETraceWithChemRepBefore p u v
```

for the constant equilibrium / zero-reaction constant trajectories. The theorem is not currently present as a named producer, but all ingredients are already available.

The repo **cannot** currently prove a general B-form/Picard construction producer for this record. The missing general data are exactly the two literal initial fields in `H1ZeroStartLiteralUPDETraceWithChemRepBefore`:

1. `pde0`: the pointwise `u`-equation at `t = 0` on the spatial interior;
2. `chem0`: the initial-time equality between the literal lifted `intervalDomain.chemotaxisDiv` and the physical product-rule representative `liftChemotaxisDivPhysicalRep`.

The existing P3/Moser/FTC, B-form, and mixed chem-div layers supply scalar integrability, coefficient identities, or chem-div time-derivative regularity. They do not supply this literal pointwise initial `u_xx = u_t + χ chem - reaction` trace.

## What the new interface does

`IntervalChiNegH1ZeroStartInitialTrace.lean` introduces:

```lean
structure H1ZeroStartLiteralUPDETraceWithChemRepBefore
    (p : CM2Params) (u v : ℝ → intervalDomainPoint → ℝ) : Prop where
  pde0 : ∀ X : intervalDomainPoint, X.1 ∈ Set.Ioo (0 : ℝ) 1 →
    intervalDomain.timeDeriv u (0 : ℝ) X =
      intervalDomain.laplacian (u (0 : ℝ)) X -
        p.χ₀ * intervalDomain.chemotaxisDiv p (u (0 : ℝ)) (v (0 : ℝ)) X +
        u (0 : ℝ) X * (p.a - p.b * (u (0 : ℝ) X) ^ p.α)
  chem0 : ∀ X : intervalDomainPoint, X.1 ∈ Set.Ioo (0 : ℝ) 1 →
    intervalDomainLift
        (fun Y : intervalDomainPoint =>
          intervalDomain.chemotaxisDiv p (u (0 : ℝ)) (v (0 : ℝ)) Y) X.1 =
      liftChemotaxisDivPhysicalRep p u v (0 : ℝ) X.1
```

and proves the exact non-circular reducer:

```lean
theorem eq0Interior_of_initialLiteralUPDETrace_withChemRep
    {p : CM2Params} {u v : ℝ → intervalDomainPoint → ℝ}
    (h : H1ZeroStartLiteralUPDETraceWithChemRepBefore p u v) :
    Set.EqOn
      (fun x => liftDeriv2 u (0 : ℝ) x)
      (fun x =>
        liftDeriv2PhysicalRHSWithChemRep p u
          (liftChemotaxisDivPhysicalRep p u v) (0 : ℝ) x)
      (Set.Ioo (0 : ℝ) 1)
```

That theorem is the right bridge. It uses only the literal source-facing initial trace and does not consume H¹ boundedness, sqrt bounds, or zero-start physical RHS outputs.

## Constant special case

`IntervalChiNegH1ZeroStartConstant.lean` already proves:

```lean
theorem H1ZeroStartClosedPrimitiveC1SignBefore_const
    (p : CM2Params) {c T : ℝ} (hc : 0 < c) :
    H1ZeroStartClosedPrimitiveC1SignBefore
      (fun _ (_ : intervalDomainPoint) => c)
      (fun _ (_ : intervalDomainPoint) => ellipticV p c)
      T
```

This covers the primitive C¹/sign half for positive constants. It does **not** by itself prove the initial PDE trace: the PDE trace for a constant `u ≡ c` additionally needs

```lean
c * (p.a - p.b * c ^ p.α) = 0
```

For the two constant solution families already in `IntervalDomainExistence.lean`, this condition is exactly available:

* equilibrium case: `equilibrium_reaction_zero p ha hb` for `c = (p.a / p.b) ^ (1 / p.α)` when `0 < p.a` and `0 < p.b`;
* zero-reaction case: `p.a = 0`, `p.b = 0`, any positive constant `c`.

The following is the smallest constant trace producer. It is source-facing and non-circular.

```lean

/-- Constant-in-time/space trajectories satisfy the literal initial `u`-PDE trace
provided the constant is a reaction equilibrium.  This is the source-facing trace
half only; positivity is deliberately not needed here. -/
theorem H1ZeroStartLiteralUPDETraceWithChemRepBefore_const_of_reaction_zero
    (p : CM2Params) {c : ℝ}
    (hreact : c * (p.a - p.b * c ^ p.α) = 0) :
    H1ZeroStartLiteralUPDETraceWithChemRepBefore p
      (fun _ (_ : intervalDomainPoint) => c)
      (fun _ (_ : intervalDomainPoint) => ellipticV p c) := by
  refine ⟨?_, ?_⟩
  · intro X hX
    have hXin : X ∈ intervalDomain.inside := by
      change X.1 ∈ Set.Ioo (0 : ℝ) 1
      exact hX
    change deriv (fun _s : ℝ => c) (0 : ℝ) =
      intervalDomainLaplacian (fun _ : intervalDomainPoint => c) X -
        p.χ₀ * intervalDomainChemotaxisDiv p
          (fun _ : intervalDomainPoint => c)
          (fun _ : intervalDomainPoint => ellipticV p c) X +
        c * (p.a - p.b * c ^ p.α)
    rw [deriv_const,
      intervalDomainLaplacian_const_zero c hXin,
      intervalDomainChemotaxisDiv_const_zero p c (ellipticV p c) hXin,
      hreact]
    ring
  · intro X hX
    have hxIcc : X.1 ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hX
    have hXin : X ∈ intervalDomain.inside := by
      change X.1 ∈ Set.Ioo (0 : ℝ) 1
      exact hX
    have hleft :
        intervalDomainLift
            (fun Y : intervalDomainPoint =>
              intervalDomain.chemotaxisDiv p
                (fun _ : intervalDomainPoint => c)
                (fun _ : intervalDomainPoint => ellipticV p c) Y) X.1 = 0 := by
      have hchem := intervalDomainChemotaxisDiv_const_zero p c (ellipticV p c) hXin
      simpa [intervalDomain, intervalDomainLift, hxIcc] using hchem

        deriv (intervalDomainLift (fun _ : intervalDomainPoint => c)) X.1 = 0 :=
      intervalDomainLift_const_deriv_zero c hX
    have hdv :
        deriv (intervalDomainLift
          (fun _ : intervalDomainPoint => ellipticV p c)) X.1 = 0 :=
      intervalDomainLift_const_deriv_zero (ellipticV p c) hX
    have hu :
        intervalDomainLift (fun _ : intervalDomainPoint => c) X.1 = c := by
      simp [intervalDomainLift_const, hxIcc]
    have hv :
        intervalDomainLift
          (fun _ : intervalDomainPoint => ellipticV p c) X.1 = ellipticV p c := by
      simp [intervalDomainLift_const, hxIcc]
    have hrel : p.μ * ellipticV p c - p.ν * c ^ p.γ = 0 := by
      unfold ellipticV
      field_simp [ne_of_gt p.hμ]
      ring

          (fun _ (_ : intervalDomainPoint) => c)
          (fun _ (_ : intervalDomainPoint) => ellipticV p c)

      simp [hdu, hdv, hu, hv, hrel]
    exact hleft.trans hright.symm
```

The first theorem proves the new literal trace interface. The second theorem is often the more useful route-facing theorem because it also fills `time_cont0`. The equilibrium and zero-reaction corollaries match the existing constant solution families in `IntervalDomainExistence.lean`.

This is worth adding: it is small, non-circular, and pairs naturally with the existing

## General B-form / Picard construction status

No general producer is present.

The relevant source facts are still not the literal initial trace:

* `Statements.lean` stores the `u` PDE only under `0 < t` and `t < T`.
* `IntervalChiNegH1LiftDeriv2Transfer.lean` gives `liftDeriv2_eq_liftDeriv2PhysicalRHS_interior_of_classicalSolution`, but it requires `t ∈ Ioo 0 T`.
* `IntervalChiNegH1ChemDivRepresentative.lean` gives `lift_chemotaxisDiv_eq_liftChemotaxisDivPhysicalRep_interior`, but it also requires `t ∈ Ioo 0 T`.
* `IntervalDomainExistence.lean` has `IsMildSolutionData`, but its pointwise `u` PDE field is also strict-positive-time only. The Duhamel fixed-point field is not a pointwise initial `u_xx` trace, and the construction notes still list regularity bootstrap as a genuine PDE gap.
* `P3MoserInitialTimeRegularity.lean` and `P3MoserPDECombinedInitialProducer.lean` are scalar initial-window/integrability routes; they deliberately do not constrain endpoint values at `t = 0` in the pointwise way needed here.
* B-form divergence-mode identities such as `cosineCoeffs_deriv_eq_sqrtLambda_sineCoeff` and the Duhamel coefficient identities prove spectral/IBP coefficient relations for spatial divergence, not the pointwise initial `u` PDE trace.

For a general construction, the missing source-facing fields are exactly the two fields already isolated in `H1ZeroStartLiteralUPDETraceWithChemRepBefore`:

```lean
pde0 : ∀ X : intervalDomainPoint, X.1 ∈ Set.Ioo (0 : ℝ) 1 →
  intervalDomain.timeDeriv u 0 X =
    intervalDomain.laplacian (u 0) X -
      p.χ₀ * intervalDomain.chemotaxisDiv p (u 0) (v 0) X +
      u 0 X * (p.a - p.b * (u 0 X) ^ p.α)

chem0 : ∀ X : intervalDomainPoint, X.1 ∈ Set.Ioo (0 : ℝ) 1 →
  intervalDomainLift
      (fun Y : intervalDomainPoint =>
        intervalDomain.chemotaxisDiv p (u 0) (v 0) Y) X.1 =
    liftChemotaxisDivPhysicalRep p u v 0 X.1
```

If the construction uses `v = coupledChemicalConcentration p u`, an additional identification is needed before the H¹ target for arbitrary `v` can use it: the stored `v 0` and the coupled concentration at `0` must agree with enough spatial derivative/elliptic trace information to rewrite `liftChemotaxisDivPhysicalRep p u v 0`.

## False/circular routes to avoid

Do not use any of the following to claim the general initial trace:

* H¹ bounded-before, sqrt-bound, or physical RHS zero-start outputs;
* `H1ZeroStartPhysicalPDEInitialCompatibilityBefore` itself;
* positive-time classical PDE equality at `t = 0`;
* scalar initial-window integrability or Moser/FTC data;
* chem-div time-derivative representatives (`coupledChemDivTimeDerivativeLift`) in place of the `u`-equation trace;
* reanchoring via `intervalDomainWithInitialSlice` without a separate proof that the new zero slice satisfies the pointwise PDE trace.

## Recommendation

Add the constant special-case producer. It is a clean, non-circular source theorem and validates the new interface against an existing honest source class. For the general B-form/Picard construction, keep `H1ZeroStartLiteralUPDETraceWithChemRepBefore` as the minimal source-facing carry until the construction layer proves the literal initial `u` PDE trace and the initial chem-representative seam.
