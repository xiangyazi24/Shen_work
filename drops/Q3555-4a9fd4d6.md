# Q3555 audit: remaining taxis/uvxx physical H┬╣ sqrt estimates

## Verdict

There is **no complete non-circular producer currently visible** for the two remaining concrete physical sqrt-bound estimates from `IsPaper2ClassicalSolution` alone:

```lean

  VŌéü * (H1lapL2Norm u Žä * H1gradL2Norm u Žä)

|H1PhysicalUvxxX p u v Žä| Ōēż
  M * (VŌéé * H1lapL2Norm u Žä)
```

The existing source APIs give the right local/pointwise ingredients, but the constants required by `H1PhysicalRHSSqrtBoundsBefore` are **fixed before `T`**. Current resolver and classical compactness lemmas mostly give **per-time existential bounds**, or strict-slab continuity, not one uniform `VŌéü`, `VŌéé`, `M` for all `Žä Ōłł (0,T)`.

So the correct next step is not to claim the estimates from `hsol` alone. Add a small source-facing residual for the needed fixed-before-`T` chemotaxis factor bounds, then prove a reducer into the existing `H1PhysicalRHSL2SqrtBoundDataBefore`/`H1PhysicalRHSSqrtBoundsBefore` pipeline. This mirrors the reaction reducer and is non-circular.

## 1. What existing data can prove the taxis estimate?

The taxis estimate is mathematically straightforward if we have a fixed-before-`T` bound on the chemical gradient quotient:

```lean
ŌłĆ Žä Ōłł Set.Ioo (0 : ŌäØ) T, ŌłĆ x Ōłł Set.Icc (0 : ŌäØ) 1,
  |deriv (intervalDomainLift (v Žä)) x| /
      (1 + intervalDomainLift (v Žä) x) ^ p.╬▓ Ōēż VŌéü
```

or equivalently a simpler resolver-gradient bound plus `v Ōēź 0`:

```lean
ŌłĆ Žä Ōłł Set.Ioo (0 : ŌäØ) T, ŌłĆ x Ōłł Set.Icc (0 : ŌäØ) 1,
  |resolverGradReal p (u Žä) x| Ōēż VŌéü
```

Then, since `v Ōēź 0` and `╬▓ Ōēź 0`, the denominator is at least one, so

```lean
|H1PhysicalChemTaxisPart p u v Žä x| Ōēż
  VŌéü * |deriv (intervalDomainLift (u Žä)) x|.
```

From this point the proof is pure Cauchy-Schwarz:

```lean
|H1PhysicalTaxisX p u v Žä|
  Ōēż Ōł½ |u_xx * taxisPart|
  Ōēż sqrt (Ōł½ u_xx^2) * sqrt (Ōł½ taxisPart^2)
  Ōēż H1lapL2Norm u Žä * (VŌéü * H1gradL2Norm u Žä)
```

This is non-circular. It uses only fixed-time spatial Cauchy and a source-side `v_x`/resolver bound. It does **not** use bounded-before or H┬╣ route data.

But the repo currently only appears to provide `resolverGradReal_bounded` per time. That is not enough to fill the global `VŌéü` field.

## 2. What existing data can prove the uvxx estimate?

The uvxx estimate is not just a `v_xx` estimate. The actual factor is

```lean
H1PhysicalChemUvxxPart =
  u * ((╬╝v - ╬Įu^╬│)/(1+v)^╬▓ - ╬▓ * v_x^2/(1+v)^(╬▓+1)).
```

Therefore a bound on `v_xx = ╬╝v - ╬Įu^╬│` alone is insufficient unless the `v_x^2` denominator term is also controlled.

The clean source-side bound is:

```lean
|H1PhysicalChemUvxxPart p u v Žä x| Ōēż M * VŌéé
```

This can be produced from:

```lean
|u(Žä,x)| Ōēż M
|((╬╝v - ╬Įu^╬│)/(1+v)^╬▓ - ╬▓*v_x^2/(1+v)^(╬▓+1))| Ōēż VŌéé
```

or from cruder pieces such as:

```lean
|u| Ōēż M
|(╬╝v - ╬Įu^╬│)/(1+v)^╬▓| Ōēż VŌééa
|v_x| Ōēż VŌéü
VŌéé = VŌééa + p.╬▓ * VŌéü^2
```

using `v Ōēź 0` to suppress denominators.

Again, existing APIs like `lift_u_bounded`, `resolverGradReal_bounded`, and `resolverGrad2Real_bounded` are per-time. They are not a fixed-before-`T` `M`/`VŌéé` producer.

### Is a `v_xx` bound enough?

Only if ŌĆ£`v_xx bound`ŌĆØ means the full bracket bound including the denominator derivative term. A bound just on

```lean
|p.╬╝ * v - p.╬Į * u^╬│| Ōēż VŌéé
```

does not control

```lean
p.╬▓ * u * v_x^2 / (1+v)^(╬▓+1)
```

So a record named as a plain `v_xx` bound would be misleading. Carry a `uvxx_core_le` or direct `uvxx_part_le` bound.

## 5. Is this non-circular?

Yes, if the residual is stated as pointwise source-factor bounds. It does not consume:

```lean
H1PhysicalRHSStrictInitialRouteBefore
H1IdentityRHSIntegrableBefore
H1IdentityRHSInitialWindowIntegrableBefore
H1EnergyDerivativeInitialWindowIntegrableBefore
IsPaper2BoundedBefore
```

It also does not treat zero-window Young majorants as sqrt bounds.

The suspicious/circular move would be to produce `M`, `VŌéü`, or `VŌéé` from a bounded-before theorem or from the H┬╣ route itself. Another suspicious move is to use per-time `Ōłā G` bounds to define a single before-`T` constant without an explicit compactness/uniformization theorem. That is exactly where bounded-before usually sneaks in.
