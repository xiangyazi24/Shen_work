## 4. Exact base-window scale

Put

```text
K := Cg · |χ₀| · Γ_M,
Cg := heatGradientLinftyLinftyConstant.
```

For `[a0,2a0,4a0]`,

```text
truncWindowB Γ_M χ₀ a0 (4a0)
  = Cg · 2√(4a0-a0) · |χ₀| · Γ_M
  = 2K√(3a0).
```

Thus, when `K > 0`, the exact admissible range is

```text
0 < a0 < 1 / (12 K²)
```

because

```text
2K√(3a0) < 1  ⇔  12K²a0 < 1.
```

If `K = 0`, both contraction coefficients are identically zero and every `a0 > 0` works, subject only to the horizon condition `4*a0 ≤ DT.T`.

There is therefore **no smallest positive `a0`**. The number

```text
1 / (12 (Cg·|χ₀|·Γ_M)²)
```

is the strict upper threshold (the supremum of admissible scales when the product is nonzero), not a minimum.

The left coefficient is

```text
truncLeftB Γ_M χ₀ (2a0) = 2K√(2a0).
```

Its standalone threshold would be

```text
a0 < 1 / (8 K²).
```

Since

```text
√(2a0) ≤ √(3a0),
```

the base-window condition is stronger:

```text
truncLeftB Γ_M χ₀ (2a0)
  ≤ truncWindowB Γ_M χ₀ a0 (4a0)
  < 1.
```

Hence one local choice of `a0` gives both inequalities. This is purely “`a0` small enough”; it imposes no global smallness on `χ₀`, `Γ_M`, or their product. Large fixed constants merely force a smaller first window.

The committed equal-step-chain constructor expresses the same fact without division or a `K = 0` case split:

```lean
theorem exists_equalStepGradientWindowChain
    {t B_F χ : ℝ} (ht : 0 < t) (hBF : 0 ≤ B_F) :
    Nonempty (EqualStepGradientWindowChain t B_F χ)
```

It sets

```text
q = Cg · 2(|χ|B_F),
h = t/(N+4),
```

with `N > 3tq²`, and proves

```text
q √(3h) < 1.
```

For the chain, simply take `a0 := C.h`; then `C.a 0 = a0`, `C.lo 0 = 2a0`, and `C.hi 0 = 4a0`.
