# 3. Terminal strong maximum: shortest non-circular implementation

## 3.1 Do not force classical regularity through the limit

Passing `C²`, time derivatives, the pointwise PDE, and ordinary endpoint derivatives through only uniform convergence would require a much stronger compactness theorem. None is necessary.

Instead:

1. Use the limit only to obtain a fixed nonzero nonnegative initial seed.
2. Apply the linear drift square-heat comparison to each original classical translated solution, with coefficient bounds uniform in the translate index.
3. Use the kernel floor at terminal heat age `1`.
4. Pass the resulting positive terminal lower bound through uniform convergence.

This is the formal analogue of the terminal strong maximum principle, but it is quantitative and avoids limit-level regularity.

## 3.2 Fixed seed from the nonzero limit

From positive mass, choose `x₀` with `0 < W(0,x₀)`. By continuity, choose a nonnegative continuous bump `f`, supported in a neighborhood on which `W(0,·)` has a positive margin, such that

```text
f is positive somewhere,
f² ≤ W(0,·)/2.
```

Uniform convergence of the initial translated slices then gives, eventually,

```text
f² ≤ Wₙ(0,·).
```

A suitable reusable statement is:

```lean
theorem squareHeatSeed_eventually_below_of_uniform_nonnegative_nonzero_limit
    {wn : ℕ → C(intervalDomainPoint, ℝ)}
    {w : C(intervalDomainPoint, ℝ)}
    (hconv : TendstoUniformly (fun n x => wn n x) w atTop)
    (hnonneg : ∀ n x, 0 ≤ wn n x)
    (hpos : ∃ x, 0 < w x) :
    ∃ f : ℝ → ℝ,
      Continuous f ∧
      (∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ f y) ∧
      (∃ y ∈ Set.Icc (0 : ℝ) 1, 0 < f y) ∧
      ∀ᶠ n in atTop,
        ∀ y ∈ Set.Icc (0 : ℝ) 1,
          f y ^ 2 ≤ intervalDomainConstExtend (wn n) y
```

Using a bump rather than `sqrt w` gives a strict margin on its support and avoids any problem at zeros of `w`.

## 3.3 Endpoint-safe faithful linear drift form

On an actual positive classical translate define, on the physical interval,

```text
q = Vx (1+V)^(-β),
B = -χ₀ m U^(m-1) q,
C = -χ₀ U^(m-1)
      [-β(1+V)^(-β-1)Vx² + (1+V)^(-β)Vxx].
```

Then on `(0,1)` and in the minimal branch,

```text
U_t = U_xx + B U_x + C U.
```

The common ceiling gives

```text
|B| ≤ A(p,M),
-C ≤ D(p,M),
```

with `A,D` displayed above. Choose

```text
Mbar = A(p,M)^2 / 2 + D(p,M).
```

## 3.4 Correct the zero-time barrier instead of using the known-vacuous package

For the literal translate proof, the cleanest small addition is a closed-at-zero barrier:

```lean
def closedSquareHeatBarrier
    (M : ℝ) (f : ℝ → ℝ) (t x : ℝ) : ℝ :=
  if t = 0 then f x ^ 2 else squareHeatBarrier M f t x
```

and a direct comparison theorem:

```lean
theorem closed_squareHeatBarrier_le_of_neumann_linear_drift
    {T A D M : ℝ} {u₀ f : ℝ → ℝ} {B C u : ℝ → ℝ → ℝ}
    (hT : 0 < T)
    (hf : Continuous f)
    (hcoeff : NeumannLinearDriftCoefficientsRegular T B C)
    (hsuper : IsClassicalNeumannLinearDriftSuperSolution T B C u)
    (hinit_u : ∀ x ∈ Set.Icc (0 : ℝ) 1, u 0 x = u₀ x)
    (hM : A ^ 2 / 2 + D ≤ M)
    (hB : ∀ t x, 0 < t → t < T → x ∈ Set.Ioo (0 : ℝ) 1 →
      |B t x| ≤ A)
    (hC : ∀ t x, 0 < t → t < T → x ∈ Set.Ioo (0 : ℝ) 1 →
      -C t x ≤ D)
    (hseed : SquareHeatSeed u₀ f) :
    ∀ t x, 0 ≤ t → t < T → x ∈ Set.Icc (0 : ℝ) 1 →
      closedSquareHeatBarrier M f t x ≤ u t x
```

For `t>0`, all derivative and residual calculations are the already-proved square-heat calculations. At `t=0`, the initial comparison is exactly `hseed.square_le_initial`. Closed-strip continuity at zero follows from `intervalFullSemigroup_tendstoUniformlyOn`. This directly repairs the obstruction recorded in `IntervalBFormSquareHeatSubsolutionRegularAssemblyStall.lean` without changing the global semigroup definition.

Apply this theorem to every sufficiently large translate with the fixed seed selected from `W(0)`. At terminal time `1`,

```text
Wₙ(1,x)
  ≥ exp(-Mbar) · (S_N(1) f(x))²
  ≥ exp(-Mbar) ·
      (unitWindowHeatKernelFloor · ∫ f)²
  =: δ > 0
```

for every `x ∈ [0,1]`, including both endpoints. Uniform terminal convergence gives `W(1,x) ≥ δ`, contradicting the terminal zero obtained from the bad minima.

This is the terminal strong maximum principle in the exact form needed by Part 1.

# 4. Faster headline-closing route: compactness is not actually necessary

The latest repository makes a stronger quantitative route possible.

For every large target time `t`, use the conserved positive mass and common ceiling on a prior positive slice to construct a slice-dependent square-root seed. Use the positive-time T0 restart barrier already implemented in

```text
IntervalBFormSquareHeatT0Restart.lean
IntervalBFormSquareHeatT0RestartDerivativeData.lean
```

rather than the unshifted barrier.

A convenient choice on a slice `s` is a scaled continuous seed

```text
f_s = (1/2) sqrt(u(s,·)).
```

The mass and ceiling imply a uniform integral lower bound

```text
∫ f_s ≥ mass / (2 sqrt M).
```

For each slice separately, strict positivity and compactness give a positive slice minimum. The fixed-function theorem `intervalFullSemigroup_tendstoUniformlyOn` allows choosing a small heat age `τ_s ∈ (0,1]` for which

```text
squareHeatBarrier Mbar f_s τ_s ≤ u(s,·).
```

No uniform lower bound on the slice is used: `τ_s` may shrink. Start the existing T0 barrier at heat age `τ_s`, propagate for one physical time unit, and note that the terminal heat age lies in `[1,2]`. The kernel floor and the mass lower bound then give one `δ>0` independent of `s` and `t`.

This route still requires the faithful `intervalDomainM_minimal_constExtend_superSolution` producer above, but it deletes all time-translate compactness and nonlinear mild-limit work. Therefore:

- **If the priority is the fastest unconditional corrected headline, implement the faithful linear-drift/super-solution producer and use the existing T0 restart barrier.**
- **If the priority is a formalization closely mirroring Section 4.1, implement the one-unit translate route.**

The premise that the minimal branch *must* construct an entire limit is therefore too strong. Such a limit is sufficient, but not necessary in this concrete repository.

# 5. Circularity audit

The following dependencies are sound.

- `intervalDomainM_globalBounded_eventual_holder` is a real analytic producer. It may use actual positive local restart floors internally, but its exported constants depend only on the tail ceiling and parameters.
- `chemFluxMLifted_abs_le_of_pos_slice` may be applied to each actual positive translate. Its exported sup constant is floor-free.
- Mass constancy is derived from the faithful mass ODE at positive times. It does not use `HasInitialMass` or the stored zero slice.
- The translate limit is only assigned nonnegativity and mass. It is not assumed to have a positive floor.
- The drift/reaction bounds use `m ≥ 1` so that `U^(m-1) ≤ M^(m-1)`. No negative power of `U` occurs.
- The terminal barrier is applied to classical approximants with a common ceiling. It does not require passing `C²` or PDE data to the limit.
- The `v` conclusion is obtained only after the `u` floor, through the already-proved faithful elliptic lower comparison.

The following routes are circular or invalid and must not be used.

1. `CompactnessData`, `Lemma_3_2.of_timeTranslateCompactnessRaw`, or any assumed raw compactness field.
2. `UniformPersistencePart1Raw`, `IntervalDomainLogisticPersistenceInputs.part1ULower`, or a projection from a persistence package.
3. Sectorial stability packages; Section 4.1 does not need them.
4. Positive-time derivative/Hölder estimates whose constant contains the per-restart floor `D.c`.
5. `chemFluxMLifted_diff_bound_of_pos_slice` as the limit-passage theorem when the limit may vanish. Use uniform continuity at zero instead.
6. Packaging the limit as `ConjugateMildSolutionDataM`; that structure requires the missing positive floor.
7. The current unshifted `SquareHeatSubsolutionCalculus` with a nonzero seed. The repository formally proves that interface is inconsistent with the semigroup's time-zero definition.
8. Ordinary endpoint differentiability of `intervalDomainLift`. It is a zero extension and is generally discontinuous at a positive endpoint value.

# 6. Endpoint audit

- The translated relative time `0` is an actual positive absolute time `aₙ`; it is not the unconstrained stored slice `u 0`.
- Product compactness includes the closed spatial endpoints.
- The mild flux has the proved endpoint identities `chemFluxMLifted_endpoint_zero` and `chemFluxMLifted_endpoint_one`.
- Never pass the literal endpoint value of `intervalDomainChemotaxisDivM`. On the interior use `intervalDomainMChemotaxisDiv_eq_physicalRep_interior`; for closed bounds use `classicalChemDivMPhysicalRep_continuousOn_Icc`.
- Use `intervalDomainConstExtend`, not `intervalDomainLift`, when building the real-valued comparison field. Add the two Neumann endpoint derivative lemmas described above.
- Choose the classical horizon beyond the terminal translated time, for example `aₙ + 2`; then the PDE is used only at strict interior times.
- The heat-kernel floor is stated for `x,y ∈ Icc 0 1`, so a terminal minimizer at `0` or `1` causes no special case.
