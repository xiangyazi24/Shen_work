theorem resolver_lift_deriv_eq_resolverGrad_of_sourceDecay
    {p : CM2Params} {u : intervalDomainPoint → ℝ}
    (hdecay : SourceCoeffQuadraticDecay p u)
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    deriv (intervalDomainLift (intervalNeumannResolverR p u)) x =
      resolverGradReal p u x := by
  classical
  set S : ℝ → ℝ := fun z =>
    ∑' k : ℕ, (intervalNeumannResolverCoeff p u k).re *
      Real.cos ((k : ℝ) * Real.pi * z) with hS
  have hxIcc : x ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hx
  have hSderiv :
      HasDerivAt S (intervalNeumannResolverRGrad p u ⟨x, hxIcc⟩) x := by
    rw [hS]
    exact solution_resolver_grad_hasDerivAt_of_sourceDecay hdecay hxIcc
  have hEq : ∀ y ∈ Set.Ioo (0 : ℝ) 1,
      intervalDomainLift (intervalNeumannResolverR p u) y = S y := by
    intro y hy
    have hyIcc : y ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hy
    simp only [intervalDomainLift, hyIcc, dif_pos]
    rw [resolverR_apply_eq, hS]
  have hloc : intervalDomainLift (intervalNeumannResolverR p u) =ᶠ[𝓝 x] S := by
    refine Filter.eventuallyEq_of_mem ?_ hEq
    exact IsOpen.mem_nhds isOpen_Ioo hx
  rw [hloc.deriv_eq, hSderiv.deriv, resolverGradReal_eq p u ⟨x, hxIcc⟩]

end ShenWork.IntervalCoupledRegularityBootstrap

## Why the closed-`Icc` theorem is not the next target

The tempting theorem statement is:

```lean
theorem resolver_lift_deriv_eq_resolverGrad_Icc_of_sourceDecay
    {p : CM2Params} {u : intervalDomainPoint → ℝ}
    (hdecay : SourceCoeffQuadraticDecay p u)
    {x : ℝ} (hx : x ∈ Set.Icc (0 : ℝ) 1) :
    deriv (intervalDomainLift (intervalNeumannResolverR p u)) x =
      resolverGradReal p u x := by
  -- endpoint cases currently need a missing source-side theorem
```

The interior case is exactly the theorem above.  The endpoint cases would require:

```lean
deriv (intervalDomainLift (intervalNeumannResolverR p u)) 0 = 0
```

and

```lean
deriv (intervalDomainLift (intervalNeumannResolverR p u)) 1 = 0
```

because `resolverGradReal p u 0 = 0` and `resolverGradReal p u 1 = 0` are already available on the solution-side utility path as `resolverGradReal_zero` and `resolverGradReal_one`.

But the source-side resolver boundary theorem currently available is:

```lean
theorem resolver_normalDeriv_zero_of_sourceDecay
    {p : CM2Params} {u : intervalDomainPoint → ℝ}
    (hdecay : SourceCoeffQuadraticDecay p u)
    {x : intervalDomainPoint} (hx : x ∈ intervalDomain.boundary) :
    intervalDomain.normalDeriv (intervalNeumannResolverR p u) x = 0
```

That statement unfolds to `derivWithin` at the endpoints, not unrestricted `deriv`:

```lean
if x.1 = 0 then derivWithin (intervalDomainLift f) (Set.Ici 0) 0
else if x.1 = 1 then derivWithin (intervalDomainLift f) (Set.Iic 1) 1
else deriv (intervalDomainLift f) x.1
```

So it proves the right-hand derivative at `0` and the left-hand derivative at `1`.  The desired closed-`Icc` theorem asks for the ordinary two-sided derivative of the zero extension.  Since `intervalDomainLift` is definitionally

```lean
fun x => if hx : x ∈ Set.Icc (0 : ℝ) 1 then f ⟨x, hx⟩ else 0
```

local equality with the cosine value series is available only on the open interval.  At `0` and `1`, the outside side of the zero extension is identically `0`, while the inside side is the resolver trace.  The existing one-sided theorem therefore does not rewrite the unrestricted endpoint `deriv`.

That is exactly why `solution_lift_v_deriv_eq_resolverGrad_Icc` can close the endpoints for a solution `v`: it uses the solution regularity/Neumann endpoint fields giving actual endpoint values of `deriv (intervalDomainLift (v t))`.  The static resolver path has not yet exposed the analogous endpoint unrestricted-`deriv` fact.

## Next endpoint theorem needed

The direct missing theorem is:

```lean
theorem resolver_lift_deriv_endpoint_zero_of_sourceDecay
    {p : CM2Params} {u : intervalDomainPoint → ℝ}
    (hdecay : SourceCoeffQuadraticDecay p u) :
    deriv (intervalDomainLift (intervalNeumannResolverR p u)) 0 = 0 ∧
    deriv (intervalDomainLift (intervalNeumannResolverR p u)) 1 = 0 := by
  -- prove from the same endpoint `HasDerivWithinAt` series proof used in
  -- `resolver_normalDeriv_zero_of_sourceDecay`, plus a zero-extension endpoint
  -- lemma converting the one-sided zero derivative/traces into ordinary `deriv`.
```

A cleaner reusable helper would be a zero-extension endpoint lemma of this shape:

```lean
theorem intervalDomainLift_endpoint_deriv_zero_of_hasDerivWithin_zero
    {f : intervalDomainPoint → ℝ}
    (h0 : HasDerivWithinAt (intervalDomainLift f) 0 (Set.Ici (0 : ℝ)) 0)
    (h1 : HasDerivWithinAt (intervalDomainLift f) 0 (Set.Iic (1 : ℝ)) 1) :
    deriv (intervalDomainLift f) 0 = 0 ∧
    deriv (intervalDomainLift f) 1 = 0 := by
  -- split on endpoint trace values.  If the trace is nonzero, the zero extension
  -- is discontinuous from the outside and `deriv` is Mathlib's non-differentiable
  -- junk value `0`; if the trace is zero, combine the outside constant-zero side
  -- with the one-sided zero derivative.
```

After that endpoint theorem exists, the closed theorem becomes a routine three-case assembly:

```lean
theorem resolver_lift_deriv_eq_resolverGrad_Icc_of_sourceDecay
    {p : CM2Params} {u : intervalDomainPoint → ℝ}
    (hdecay : SourceCoeffQuadraticDecay p u)
    {x : ℝ} (hx : x ∈ Set.Icc (0 : ℝ) 1) :
    deriv (intervalDomainLift (intervalNeumannResolverR p u)) x =
      resolverGradReal p u x := by
  rcases eq_or_lt_of_le hx.1 with rfl | hx0
  · rw [(resolver_lift_deriv_endpoint_zero_of_sourceDecay hdecay).1,
        ShenWork.Paper2.resolverGradReal_zero]
  rcases eq_or_lt_of_le hx.2 with rfl | hx1
  · rw [(resolver_lift_deriv_endpoint_zero_of_sourceDecay hdecay).2,
        ShenWork.Paper2.resolverGradReal_one]
  · exact resolver_lift_deriv_eq_resolverGrad_of_sourceDecay hdecay ⟨hx0, hx1⟩
```

## Recommendation for `patchedChemical_vx`

Use the interior theorem above as the next public source-side bridge.  It is the smallest non-circular extraction from current `main` and matches the proof already trusted by `resolver_laplacian_eq_RLap_of_sourceDecay`.

Do not route the zero-face record through a closed-`Icc` unrestricted-`deriv` theorem until `resolver_lift_deriv_endpoint_zero_of_sourceDecay` (or the generic zero-extension endpoint helper plus resolver endpoint `HasDerivWithinAt` exposure) is proved.  Until then, endpoint data should continue to use `resolver_normalDeriv_zero_of_sourceDecay` for one-sided boundary normal derivatives, not unrestricted `deriv`.
