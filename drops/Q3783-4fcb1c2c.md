## Source certificates for actual Picard/classical data: next after the wiring theorem

For actual `GradientMildSolutionData`, the source certificates should be produced with a **windowed-source** theorem, because `D.hbound`, `D.hnonneg`, and `D.hcont` are only stated on `0 < s ≤ D.T`, while the generic integrability producers want global `∀ s y` bounds.

Use the same pattern already documented in `IntervalPicardG1All.lean`: define the sources to be zero outside the time window.

Suggested names:

```lean
def windowedChemFluxSource (p : CM2Params)
    (T : ℝ) (u : ℝ → intervalDomainPoint → ℝ) : ℝ → ℝ → ℝ :=
  fun s y => if 0 < s ∧ s ≤ T then chemFluxLifted p (u s) y else 0

def windowedLogisticSource (p : CM2Params)
    (T : ℝ) (u : ℝ → intervalDomainPoint → ℝ) : ℝ → ℝ → ℝ :=
  fun s y => if 0 < s ∧ s ≤ T then logisticLifted p (u s) y else 0
```

Then prove:

```lean
windowedLogisticSource_measurable_of_hasJointMeasurability
windowedLogisticSource_bound_of_ball
windowedChemFluxSource_measurable_of_hasJointMeasurability
windowedChemFluxSource_bound_of_ball
```

The logistic measurability proof should reuse the existing/public `logisticLifted_joint_measurable'` pattern from `IntervalMildPicardThreshold`. The chem-flux measurability proof should promote/copy the currently-private proof `chemFluxLifted_joint_measurable` from `IntervalMildPicardThreshold.lean`; it already proves the exact source measurability from `HasJointMeasurability`.

The chem-flux bound should promote/copy the currently-private `chemFluxLifted_bound_of_ball` from `IntervalMildPicardThreshold.lean`, with the uniform constant

* Use `Set.uIoc_of_le ht0.le` and `MeasureTheory.Ioo_ae_eq_Ioc` when you need strict `s < t`, but for **window equality** on `Ioc 0 t`, `0 < s ∧ s ≤ D.T` follows from `s ∈ Ioc 0 t` and `htT : t ≤ D.T`.
