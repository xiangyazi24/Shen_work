# Q3762 route selector: Task295 variable-seed restart producer

Audit basis: repository source at requested ref `04188701` unless otherwise noted. I treated the Task295 theorem as a reducer theorem, not as a producer, and checked the actual Lean objects it consumes.

## Executive verdict

`bform_strictPos_via_t0_restart_semigroup_variableSeed` is a useful and mostly faithful *consumer* shape: it has removed the fixed `paperPositiveFloor` constant seed and lets the seed depend on the target time. However, the full `hrestart` hypothesis is not yet produced by the committed floor-free B-form path. The remaining hard condition is the positive-restart initial comparison

```lean
∀ x ∈ Set.Icc (0 : ℝ) 1,
  squareHeatBarrier M f τ x ≤ bformConjugatePicardLift p DB τ x
```

for a nonzero `SquareHeatSeed`. This comparison is not a harmless technical side condition. Together with `SquareHeatSeed` and `0 < τ`, it forces the entire closed slice of `bformConjugatePicardLift p DB τ` to be strictly positive. Therefore it cannot be discharged from `negativePart_zero` / nonnegativity alone. A producer needs an independent positive slice floor, or it is circular.

The smallest next theorem should not be another headline alias. It should isolate the missing comparison field: first prove that restart initial comparison requires a positive slice, then prove the comparison from a supplied positive slice floor plus a semigroup bound on the chosen seed. If a non-circular Picard-side `hpos` is accepted as already-proved core input, a second small theorem can turn joint continuity plus that `hpos` into the needed slice floor.

## 1. Is the Task295 variable-seed input faithful/satisfiable?

### Exact source facts

* `SquareHeatSeed` is defined in `ShenWork/Paper2/IntervalBFormSquareHeatSubsolution.lean` with four fields: continuity of `f` on `[0,1]`, nonnegativity on `[0,1]`, positivity somewhere on `[0,1]`, and the initial square comparison `f y ^ 2 ≤ u₀ y` on `[0,1]`.
* `squareHeatBarrier_pos` in the same file proves that if `0 < t` and `f` has the first three `SquareHeatSeed` fields, then `squareHeatBarrier M f t x` is strictly positive for every real `x`.
* `bform_strictPos_via_t0_restart_semigroup_variableSeed` in `IntervalBFormSquareHeatT0RestartDerivativeData.lean` consumes a per-target package `∃ τ f K, ...` and then uses `square_heat_hbarrier_via_t0_restart` plus `squareHeatBarrier_pos` to push strict positivity to the target time.
* `squareHeatRestartDerivativeData_of_semigroup` in the same file honestly discharges the positive-time derivative/Neumann/heat-equation package from `0 < τ`, `Continuous f`, a uniform cosine-coefficient bound, and square summability of the coefficients. This part is not hiding endpoint `t=0` regularity.

### Assessment

The input shape is satisfiable in principle, but not yet produced by the current floor-free source. The `SquareHeatSeed` requirement itself is not impossible for boundary-vanishing positive data: one should choose a small bump supported in the open interval where `u₀` has a local positive lower bound. What I did **not** find in committed source is a theorem of the form

```lean
SquareHeatSeed (intervalDomainLift u₀) f
```

for such a local bump from `PositiveInitialDatum intervalDomain u₀`, together with the required cosine coefficient bound and square summability. The only explicit seed producer I found is the old constant `paperPositiveConstSeed`, which goes through `paperPositiveFloor` and therefore is exactly the route to avoid here.

The real hidden strength is the restart initial comparison at positive time. For any nonzero seed, the left side is strictly positive on the whole closed interval once `τ > 0`; hence the right side must have a strictly positive closed-slice lower bound. Nonnegativity of the Picard limit is insufficient.

A minimal obstruction theorem makes this precise:

```lean
import ShenWork.Paper2.IntervalBFormSquareHeatT0RestartDerivativeData

open Filter Topology Set

noncomputable section

namespace ShenWork.Paper2.BFormPositiveDatumNegPart

/-- A nonzero square-heat restart comparison already contains closed-slice
strict positivity of the target at the restart time.  This is the core reason
`hinitial` cannot be produced from nonnegativity alone. -/
theorem restart_initial_comparison_forces_positive_slice
    {u₀ f : ℝ → ℝ} {u : ℝ → ℝ → ℝ} {M τ : ℝ}
    (hτ : 0 < τ)
    (hseed : SquareHeatSeed u₀ f)
    (hinitial :
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        squareHeatBarrier M f τ x ≤ u τ x) :
    ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 < u τ x := by
  intro x hx
  exact lt_of_lt_of_le
    (squareHeatBarrier_pos (M := M) hτ
      hseed.continuousOn hseed.nonneg hseed.pos_somewhere x)
    (hinitial x hx)

end ShenWork.Paper2.BFormPositiveDatumNegPart
```

So the Task295 input is faithful as a reducer, but the full producer is still too strong unless we separately prove a positive slice floor by a non-circular route.

### 2B. Second theorem: produce the missing `hinitial` from an honest slice floor

This is the smallest constructive theorem that actually produces a field of `hrestart` without using the old fixed global seed. It turns a positive floor for `bformConjugatePicardLift p DB τ` and a bound on the positive-time heat trace of the chosen seed into the restart initial comparison.

```lean
import ShenWork.Paper2.IntervalBFormSquareHeatT0RestartDerivativeData

open Filter Topology Set
open ShenWork.IntervalDomain (intervalDomainPoint intervalDomainLift)
open ShenWork.IntervalConjugatePicard
  (ConjugateMildExistenceData conjugatePicardLimit)
open ShenWork.IntervalNeumannFullKernel
  (intervalFullSemigroupOperator)

noncomputable section

namespace ShenWork.Paper2.BFormPositiveDatumNegPart

/-- Produce the restart initial comparison from a positive slice floor and a
closed-interval bound on the heat trace of the chosen seed. -/
theorem squareHeatBarrier_restart_initial_le_of_slice_floor_and_semigroup_bound
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {DB : ConjugateMildExistenceData p u₀}
    {M τ c B : ℝ} {f : ℝ → ℝ}
    (_hc : 0 < c)
    (hfloor :
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        c ≤ bformConjugatePicardLift p DB τ x)
    (hSbound :
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |intervalFullSemigroupOperator τ f x| ≤ B)
    (hB_nonneg : 0 ≤ B)
    (hsmall : Real.exp (-M * τ) * B ^ 2 ≤ c) :
    ∀ x ∈ Set.Icc (0 : ℝ) 1,
      squareHeatBarrier M f τ x ≤ bformConjugatePicardLift p DB τ x := by
  intro x hx
  have hSabsB : |intervalFullSemigroupOperator τ f x| ≤ |B| := by
    simpa [abs_of_nonneg hB_nonneg] using hSbound x hx
  have hSsq : (intervalFullSemigroupOperator τ f x) ^ 2 ≤ B ^ 2 := by
    simpa using (sq_le_sq.mpr hSabsB)
  have hexp_nonneg : 0 ≤ Real.exp (-M * τ) :=
    le_of_lt (Real.exp_pos _)
  calc
    squareHeatBarrier M f τ x
        = Real.exp (-M * τ) *
            (intervalFullSemigroupOperator τ f x) ^ 2 := rfl
    _ ≤ Real.exp (-M * τ) * B ^ 2 :=
        mul_le_mul_of_nonneg_left hSsq hexp_nonneg
    _ ≤ c := hsmall
    _ ≤ bformConjugatePicardLift p DB τ x := hfloor x hx

end ShenWork.Paper2.BFormPositiveDatumNegPart
```

This theorem is not a wrapper around Task295. It proves exactly the nontrivial `hinitial` field once the true missing analytic input, a positive slice floor, is available.

### 2C. Optional producer of the slice floor, only if the Picard `hpos` field is accepted

There is an existing compact-min theorem, `conjugatePicardLimit_uniformPositive_on_compact`, but it consumes strict positivity as input. It is non-circular only if its `hpos` argument comes from an already accepted source such as `ConjugateMildExistenceData.hmapsTo_pos` via `conjugateMildSolutionData_of_data DB`, not from `bform_strictPos_closed`, `PositiveDatumBFormLocalComponentsSqRegular.strictPos`, or Task295 itself.

A clean low-level theorem would take the joint continuity and positivity explicitly:

```lean
import ShenWork.Paper2.IntervalBFormSquareHeatT0RestartDerivativeData
import ShenWork.Paper2.IntervalConjugatePicardInfThresholdDischarge

open Filter Topology Set
open ShenWork.IntervalDomain (intervalDomainLift intervalDomainPoint)
open ShenWork.IntervalConjugatePicard
  (ConjugateMildExistenceData conjugatePicardLimit)
open ShenWork.IntervalMildPicardThreshold (unitClip_of_mem)

noncomputable section

namespace ShenWork.Paper2.BFormPositiveDatumNegPart

/-- Convert non-circular pointwise positivity plus joint continuity into the
closed-slice floor needed by the restart initial comparison.  Do not feed this
with `hpos` obtained from `bform_strictPos_closed` or from Task295. -/
theorem bformConjugatePicardLift_slice_floor_of_hjoint_hpos
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {DB : ConjugateMildExistenceData p u₀} {τ : ℝ}
    (hτ0 : 0 < τ) (hτT : τ < DB.T)
    (hjoint : ContinuousOn
      (Function.uncurry
        (fun (σ : ℝ) (x : ℝ) =>
          intervalDomainLift (conjugatePicardLimit p u₀ DB.T σ) x))
      (Set.Ioo (0 : ℝ) DB.T ×ˢ Set.Icc (0 : ℝ) 1))
    (hpos : ∀ σ x, 0 < σ → σ < DB.T →
      0 < conjugatePicardLimit p u₀ DB.T σ x) :
    ∃ c : ℝ, 0 < c ∧
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        c ≤ bformConjugatePicardLift p DB τ x := by
  obtain ⟨c, hc, hc_le⟩ :=
    ShenWork.IntervalConjugatePicard.conjugatePicardLimit_uniformPositive_on_compact
      (p := p) (u₀ := u₀) (T := DB.T) (s := τ) (t := τ)
      hτ0 le_rfl hτT hjoint hpos
  refine ⟨c, hc, ?_⟩
  intro x hx
  have hτ_mem : τ ∈ Set.Icc τ τ := Set.mem_Icc.mpr ⟨le_rfl, le_rfl⟩
  have h := hc_le τ hτ_mem x hx
  simpa [bformConjugatePicardLift, intervalDomainLift, unitClip_of_mem hx] using h

end ShenWork.Paper2.BFormPositiveDatumNegPart
```

For `PositiveDatumBFormLocalComponentsSqRegular`, the joint-continuity argument is already used in source as


inside its `uniformPositive_on_compact` definition. The safe refactor is to use this projection only for `hjoint`; do not use `K.strictPos`, because that currently calls `bform_strictPos_closed`.

## 3. Existing source facts to reuse before proving anything new

## 4. Circular routes to avoid

Do not use any of the following to produce the Task295 `hrestart` package:

* `bform_strictPos_closed`.
* `paperPositiveConstSeed`, `paperPositiveSeedValue`, or `squareHeatBarrier_paperPositiveConstSeed_initial_le`.
* `conjugatePicardLimit_ge_half_floor_of_PID`, `conjugatePicardLimit_pos_of_PID`, or `conjugatePicardLimit_hpost_of_PID` when the goal is the floor-free positive-datum path.
* `PositiveDatumBFormLocalComponentsSq.strictPos` or `PositiveDatumBFormLocalComponentsSqRegular.strictPos`; both route through `bform_strictPos_closed`.
* `PositiveDatumBFormLocalComponentsSqRegular.uniformPositive_on_compact` as currently written, because it uses `K.strictPos`.
* Any local-existence/frontier theorem whose `strictPos` field is already discharged by `bform_strictPos_closed` or by Task295.

`conjugatePicardLimit_uniformPositive_on_compact` is safe only as a compactness lemma. Its safety depends entirely on the source of its `hpos` argument.

## 5. Route decision

The right next move is not to add aliases around Task295. The producer is still too strong if the available inputs are only floor-free negative-part nonnegativity plus the linear strip data. The core obstruction/progress theorem is:

```lean
restart_initial_comparison_forces_positive_slice
```

and the smallest constructive missing-field theorem is:

```lean
squareHeatBarrier_restart_initial_le_of_slice_floor_and_semigroup_bound
```

After those land, there are two honest branches:

1. **If Picard `hmapsTo_pos` is accepted as a non-circular core positivity input**, prove `bformConjugatePicardLift_slice_floor_of_hjoint_hpos`, instantiate `hjoint` from the B-form regularity data, then choose a small local seed and use the initial-comparison theorem above.
2. **If the strict-negative headline must produce strict positivity without consuming any pre-existing strict positivity of the same Picard limit**, then Task295’s full `hrestart` remains too strong. In that case, first prove a strong-maximum-principle/local-slice-floor theorem from `negativePart_zero` plus the classical B-form PDE/Neumann data. Only after that floor exists can the per-target square-heat seed be chosen and compared at the restart time.

In either branch, the next non-wrapper seed theorem still needed is a local-bump producer for `SquareHeatSeed (intervalDomainLift u₀) f` from `PositiveInitialDatum intervalDomain u₀`, together with the coefficient bound and coefficient-square summability required by `squareHeatRestartDerivativeData_of_semigroup`. The committed source currently has those coefficient lemmas only for the old constant paper-positive seed, not for a floor-free local bump.
