# Q3841 — cheapest Lean routes for weak testing and square-heat barrier comparison

## Executive answer

For the two irreducible PDE lemmas, the cheapest Lean route is:

1. **Weak PDE testing:** use **(B) the mild identity directly via Fubini/semigroup duality**, not coefficient-level restart differentiation.  More precisely, prove the repo’s already-isolated missing theorem
   ```lean
   TruncatedMildSemigroupWeakAfterBNDualityOn
   ```
   for the negative-part energy test class, then feed the existing Cron2 energy/Gronwall machinery.

2. **Barrier comparison:** use **(A) a time-shift/restart version of the existing square-heat barrier/comparison route**, not a fresh B-form comparison from scratch and not raw Green-kernel positivity plus Duhamel bounds.  The repo already has the square-heat seed, squared barrier calculus, and a regular Neumann comparison interface.  Generalize that API from `t=0` to a restart time `s>0`.

The important point is that both tasks already have the right partial infrastructure in the repo.  Do **not** replace them with new opaque frontiers.

## 1. Weak PDE testing

### Route choice

Best route: **B — use the mild identity directly via Fubini**, with one refinement:

* first prove the weak PDE for a fixed admissible test class `Test`;
* then derive the negative-part energy inequality from that weak PDE using the existing Cron2 negative-part package;
* do not try to differentiate the coefficient restart identity term-by-term just to get weak testing.

The repo already says this.  In `ShenWork/Paper2/IntervalBFormCron2MildToWeak.lean`, the module comment states that B_N duality is done and that the only missing analytic input is the Neumann semigroup weak Duhamel identity turning the tested mild formula into the local weak PDE.  The precise missing interface is:

```lean
def TruncatedMildSemigroupWeakAfterBNDualityOn
    (p : CM2Params) (T : ℝ) (u0 : intervalDomainPoint -> ℝ)
    (u : ℝ -> intervalDomainPoint -> ℝ) (Test : (ℝ -> ℝ) -> Prop) : Prop :=
  TruncatedConjugateMildSolution p T u0 u ->
    ∀ t, 0 < t -> t <= T -> ∀ φ : ℝ -> ℝ, Test φ ->
      (∀ s, 0 < s -> s < t -> TruncatedBNDualityForTestAt p u t s φ) ->
        (∫ x, intervalDomainLift
              (fun z : intervalDomainPoint => intervalDomain.timeDeriv u t z) x * φ x
            ∂ intervalMeasure 1)
          + (∫ x, deriv (intervalDomainLift (u t)) x * deriv φ x
            ∂ intervalMeasure 1)
          =
        p.chi0 *
            (∫ x, truncatedChemFluxLifted p (u t) x * deriv φ x
              ∂ intervalMeasure 1)
          + (∫ x, truncatedLogisticLifted p (u t) x * φ x
              ∂ intervalMeasure 1)
```

This is exactly the right target.  The chemotaxis duality is not the bottleneck anymore; `truncated_bN_duality_regular_at` already turns the B-form kernel into the weak `Q * φ_x` term when integrability and the semigroup derivative identity are supplied (`IntervalBFormCron2MildToWeak.lean:54-95`).

### Why not route A?

Route A, “differentiate the coefficient-level restart identity term-by-term,” is useful for the **classical bootstrap**, e.g. `HasBFormSpectralPdeAgreement` and pointwise `hpde_u`.  It is too expensive and likely circular for negative-part nonnegativity:

* term-by-term coefficient differentiation needs source coefficient `TimeC1` and summable envelopes;
* those are exactly part of the mild-to-classical bootstrap;
* nonnegativity is needed earlier, before the full classical bootstrap is available.

So route A proves too much, too late.  It belongs to the positive-time spectral bootstrap, not the Cron2 negative-part energy route.

### Why not test with `u_minus` directly from the mild equation?

That is tempting but harder in Lean.  The test function

```lean
negativePartTest u t x = -negativePartLift (u t) x
```

is time-dependent and nonsmooth.  To use it directly in the mild formula you still need:

* chain rules for `u_-`;
* spatial derivative of the negative part;
* time derivative of the negative-part energy;
* initial vanishing of the negative-part energy;
* representative upgrade from zero `L2` negative part to pointwise nonnegativity.

Those are exactly the fields already isolated in `NegativePartEnergyCoreData`, whose module comment lists the remaining PDE/Sobolev obligations: weak formulation, chain rules for `-u_-`, positive-time energy differentiability, initial trace of the negative part, and representative upgrade.  File reference: `ShenWork/Paper2/IntervalBFormCron2NegativePartEnergy.lean:4-17` and the record at `:83-117` in the previous audit.

## 2. Barrier comparison

### Route choice

Best route: **A — time-shift the existing square-heat barrier route**, using the already-proved regular Neumann comparison interface.

The repo already has the right components:

* `SquareHeatSeed` and `squareHeatBarrier` in `IntervalBFormSquareHeatSubsolution.lean:23-34`.
* a weak-PID floor-free seed `positiveInitialDatumSqrtSeed_squareHeatSeed` in `IntervalBFormSquareHeatSubsolution.lean:36-83`.
* the abstract comparison output
  ```lean
  square_heat_hbarrier_of_neumann_linear_drift_square_heat_subsolution
  ```
  at `IntervalBFormSquareHeatSubsolution.lean:209-230`.
* the corrected regular comparison output
  ```lean
  square_heat_hbarrier_of_neumann_linear_drift_square_heat_subsolution_regular
  strict_pos_of_neumann_linear_drift_square_heat_subsolution_regular
  ```
  in `IntervalBFormSquareHeatSubsolutionRegular.lean:19-84`.
* the B-form specialization
  ```lean
  bform_square_heat_hbarrier_of_neumann_linear_drift_square_heat_subsolution_regular
  bform_strictPos_of_square_heat_subsolution_regular
  ```
  in `IntervalBFormSquareHeatSubsolutionRegular.lean:86-162`.

So the work is not to invent a new comparison principle.  The work is to **restart the existing comparison at time `s`**.

### Why not route B?

A fresh comparison for the full B-form mild equation is strictly harder.  You would need to build, from scratch, a comparison principle for a divergence-form nonlinear mild equation with chemotaxis flux.  The existing square-heat route has already reduced the problem to a **linear drift-reaction comparison** for coefficients `B` and `C`, which is exactly the right PDE shape after expanding the positive classical trajectory on an interval.

### Why not route C?

Green kernel positivity plus Duhamel bounds can prove small-time positivity of the form

```text
u(t) >= S(t)u0 - small_error
```

or positivity after shrinking `T`, but it is not the cleanest way to get the reusable lower barrier

```text
u(t) >= exp(-C(t-s)) * (S(t-s) seed)^2.
```

That lower barrier is a comparison theorem, not just a perturbative estimate.  Raw Duhamel bounds are useful for choosing small horizons, but they are weaker and more brittle than the square-heat comparison route.

### Recommended time-shift theorem

The theorem should be stated for a restarted trajectory

```lean
uShift τ x = u (s + τ) x
```

and initial datum

```lean
u_s x = u s x.
```

Then reuse the existing `square_heat_hbarrier_of_neumann_linear_drift_square_heat_subsolution_regular` with shifted coefficients.

it is only the existing regular comparison theorem applied to shifted data. -/
theorem square_heat_hbarrier_restart_regular
    {T s A D M : ℝ} {seed : ℝ -> ℝ}
    {B C u : ℝ -> ℝ -> ℝ}
    (hsT : s < T)
    (hW : 0 < T - s)
    (hcoeff :
      NeumannLinearDriftCoefficientsRegular (T - s)
        (timeShift2 s B) (timeShift2 s C))
    (hsuper :
      IsClassicalNeumannLinearDriftSuperSolution (T - s)
        (timeShift2 s B) (timeShift2 s C) (timeShift2 s u))
    (hu_initial :
      ∀ x ∈ Set.Icc (0 : ℝ) 1, timeShift2 s u 0 x = u s x)
    (hcompare :
      NeumannLinearDriftComparisonRegular (T - s)
        (timeShift2 s B) (timeShift2 s C) (u s) (timeShift2 s u))
    (hbarrier_reg :
      NeumannLinearDriftSubSolutionRegularity (T - s)
        (timeShift2 s B) (timeShift2 s C) (squareHeatBarrier M seed))
    (hcalc :
      SquareHeatSubsolutionCalculus (T - s) M seed
        (timeShift2 s B) (timeShift2 s C))
    (hM : A ^ 2 / 2 + D <= M)
    (hB_bound :
      ∀ τ x, 0 < τ -> τ < T - s -> x ∈ Set.Ioo (0 : ℝ) 1 ->

  have hτ0 : 0 < t - s := sub_pos.mpr hst

      (T := T - s) (A := A) (D := D) (M := M)
      (u0 := u s) (f := seed)
      (B := timeShift2 s B) (C := timeShift2 s C)
      (u := timeShift2 s u)
      hW hcoeff hsuper hu_initial hcompare hbarrier_reg hcalc hM
      hB_bound hC_neg_bound hseed
      (t - s) x hτ0 hτT hx
  simpa [timeShift2, add_comm, add_sub_cancel_right] using hbar

end ShenWork.Paper2.BFormPositiveDatumNegPart

This is the theorem shape I would implement first.  It is pure API glue if all shifted coefficient/regularity facts are available.

### The seed at restart time

At restart time `s > 0`, if you already have `u s x > 0` and continuity, the seed can be either:

1. a square-root seed:
   ```lean
   seed x = sqrt (u s x)
   ```
   giving `seed^2 <= u s`; or
2. a smaller smooth bump/clip seed if global smoothness of `sqrt (u s)` is awkward.

The repo already has a floor-free square-root seed for original weak PID:

## Overall dependency order

### Weak testing route

1. Prove fixed-test heat semigroup weak identity.
2. Prove fixed-test value-Duhamel weak identity.
3. Prove fixed-test conjugate/B-form Duhamel weak identity after `TruncatedBNDualityForTestAt`.
4. Assemble `TruncatedMildSemigroupWeakAfterBNDualityOn` for the chosen test class.
5. Feed `truncatedWeakLocalPDEOn_of_regularData`.
6. Feed the existing negative-part energy core / Gronwall route.

### Barrier route

1. Prove restart-slice `SquareHeatSeed`.
2. Prove shifted coefficient regularity/bounds for `B(t,x)` and `C(t,x)` on `[s,T]`.
3. Prove shifted `NeumannLinearDriftSubSolutionRegularity` and `SquareHeatSubsolutionCalculus` by reusing existing square-heat calculus.
4. Apply `square_heat_hbarrier_restart_regular`.
5. Get strict positivity from `squareHeatBarrier_pos` plus the lower barrier.

---

## Final recommendation

For **weak testing**, implement route **B**: mild identity + Fubini + semigroup weak identity, using the existing `TruncatedMildSemigroupWeakAfterBNDualityOn` stall point.  Route A is for the classical spectral bootstrap, not for negative-part energy.

For **barrier comparison**, implement route **A**: time-shift the existing square-heat regular comparison theorem.  Route B duplicates already-built comparison infrastructure.  Route C is useful for small-time estimates but is not the clean reusable lower-barrier theorem you need.
