## The key closed-`Icc` equality

I would add this either to `IntervalDomainL2StaticVDifference.lean` if import order permits, or locally in a new producer file that imports both `IntervalChiNegH1PhysicalResolverSupProducer` and `IntervalDuhamelIntegrability`.  Placing it in a new producer file is lower-risk for import cycles.

```lean
private theorem continuous_eqOn_Ioo_extend_to_Icc
    {f g : ℝ → ℝ}
    (hf : Continuous f) (hg : Continuous g)
    (heq : Set.EqOn f g (Set.Ioo (0 : ℝ) 1)) :
    Set.EqOn f g (Set.Icc (0 : ℝ) 1) := by
  intro x hx
  have hclosed : IsClosed {y : ℝ | f y = g y} := isClosed_eq hf hg
  have hsub : Set.Ioo (0 : ℝ) 1 ⊆ {y : ℝ | f y = g y} := by
    intro y hy
    exact heq hy
  have hclosure_subset : closure (Set.Ioo (0 : ℝ) 1) ⊆ {y : ℝ | f y = g y} :=
    closure_minimal hsub hclosed
  have hxclosure : x ∈ closure (Set.Ioo (0 : ℝ) 1) := by
    -- Mathlib has this as `closure_Ioo`; exact namespace may be `Set.closure_Ioo`.
    simpa [Set.closure_Ioo (a := (0 : ℝ)) (b := 1) (by norm_num)] using hx
  exact hclosure_subset hxclosure

/-- Closed-interval version of the static elliptic characterization `v = R(u)`.
The existing theorem gives equality on `Ioo`; this theorem closes it to `Icc` by
continuity of both sides. -/
theorem solution_v_eq_resolver_pointwise_Icc
    {p : CM2Params} {T : ℝ}
    {u v : ℝ → intervalDomainPoint → ℝ}
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u v)
    {t : ℝ} (ht : t ∈ Set.Ioo (0 : ℝ) T)
    {x : ℝ} (hx : x ∈ Set.Icc (0 : ℝ) 1) :
    intervalNeumannResolverR p (u t) ⟨x, hx⟩ =
      intervalDomainLift (v t) x := by
  classical
  let Rval : ℝ → ℝ := fun y => ∑' k : ℕ,
    (intervalNeumannResolverCoeff p (u t) k).re * unitIntervalCosineMode k y
  let Vrepr : ℝ → ℝ := liftRepr (v t)

  have hreg : intervalDomainClassicalRegularity T u v := hsol.regularity
  have huC2 : ContDiffOn ℝ 2 (intervalDomainLift (u t)) (Set.Icc (0 : ℝ) 1) :=
    (hreg.2.2.2.2.1 t ht).1.1
  have hvC2 : ContDiffOn ℝ 2 (intervalDomainLift (v t)) (Set.Icc (0 : ℝ) 1) :=
    (hreg.2.2.2.2.1 t ht).2.1
  have hRcont : Continuous Rval := by
    simpa [Rval] using
      resolverValueReal_continuous_of_continuousOn
        (p := p) (w := u t) huC2.continuousOn
  have hVcont : Continuous Vrepr := by
    simpa [Vrepr] using liftRepr_continuous (w := v t) hvC2.continuousOn

  have heqIoo : Set.EqOn Rval Vrepr (Set.Ioo (0 : ℝ) 1) := by
    intro y hy
    have hresolver :=
      solution_v_eq_resolver_pointwise_unconditional
        (p := p) (T := T) (u := u) (v := v) hsol ht hy
    have hrepr : Vrepr y = intervalDomainLift (v t) y := by
      simpa [Vrepr] using
        liftRepr_eq_on_Icc (w := v t) (Set.Ioo_subset_Icc_self hy)
    -- `intervalNeumannResolverR` is definitionally the same cosine tsum as `Rval`.
    simpa [Rval, Vrepr, intervalNeumannResolverR, hrepr] using hresolver

  have hclosed : Rval x = Vrepr x :=
    continuous_eqOn_Ioo_extend_to_Icc hRcont hVcont heqIoo hx
  have hRdef : intervalNeumannResolverR p (u t) ⟨x, hx⟩ = Rval x := by
    simp [Rval, intervalNeumannResolverR]
  have hVrepr : Vrepr x = intervalDomainLift (v t) x := by
    simpa [Vrepr] using liftRepr_eq_on_Icc (w := v t) hx
  calc
    intervalNeumannResolverR p (u t) ⟨x, hx⟩ = Rval x := hRdef
    _ = Vrepr x := hclosed
    _ = intervalDomainLift (v t) x := hVrepr

end ShenWork.Paper2.IntervalChiNegH1PhysicalResolverSupProducer
```

```lean
    (p : CM2Params) {M : ℝ} (hM : 0 ≤ M) :
    0 ≤ H1PhysicalChemResolverValueCap p M := by

        (mul_nonneg p.hν.le (Real.rpow_nonneg hM p.γ)))

      hG := H1PhysicalChemResolverGradCap_nonneg p hM
      u_nonneg_le := ?_

      intervalDomainLift_le_supNorm_of_classical
```

## Why the `v_abs_le` route is legitimate

The subtle point is real.  `resolverValue_sup_le_of_ub` bounds

```lean
|intervalNeumannResolverR p (u τ) ⟨x,hx⟩|
```

on every closed-domain point `⟨x,hx⟩`, but the existing elliptic identification only rewrites it to `intervalDomainLift (v τ) x` for `x ∈ Ioo 0 1`.

The clean proof is exactly the closure argument suggested in the prompt, with one detail: use `liftRepr (v τ)` for the `v` side, not the raw zero-extension as a global function.  `intervalDomainLift (v τ)` is only known continuous on `Icc 0 1`; as a zero-extension it need not be globally continuous at endpoints.  The existing `liftRepr` was designed for this: it is globally continuous when the lift is continuous on `Icc`, and it agrees with the lift on `Icc`.

The resolver side is continuous by `resolverValueReal_continuous_of_continuousOn`, applied to the classical `u` slice continuity.  The two global continuous functions agree on `Ioo` by `solution_v_eq_resolver_pointwise_unconditional`, hence agree on `closure (Ioo 0 1) = Icc 0 1`.  Then `resolverValue_sup_le_of_ub` rewrites directly to the desired `v_abs_le` field.

## Hidden circularity audit

No hidden circularity in the proposed theorem itself:

* `intervalDomainLift_le_supNorm_of_classical` uses classical regularity plus the concrete interval sup norm, not H¹ data.
* `resolverValue_sup_le_of_ub` and `resolverGrad_sup_le_of_ub` are quantitative elliptic resolver estimates from `IntervalDomainResolverSupQuantitative.lean`; they do not use the physical H¹ sqrt route.
* `solution_v_eq_resolver_pointwise_unconditional` is the static elliptic characterization of `v`, not a bounded-before or H¹ consequence.
* `H1PhysicalChemResolverSupBefore_of_valueGradSup` is an algebraic denominator/core reducer already landed in the chem sqrt file.

The only caveat is provenance of `hbounded`.  If `hbounded` is imported from the negative-sensitivity maximum-principle/Lemma 3.1 branch, this is non-circular.  If one tries to use the physical H¹ sqrt theorem being proved to first obtain `IsPaper2BoundedBefore`, then feeds that back into this theorem, that would be circular.  The route should be recorded as a bounded-before-source reducer, not as a classical-solution-alone producer.

## Practical recommendation

Land the new `solution_v_eq_resolver_pointwise_Icc` first.  It is independently useful and removes the only endpoint friction in the current physical H¹ source chain.

Then land `H1PhysicalChemValueGradSupBefore_of_supNormBound` and the existential `..._of_boundedBefore` wrapper.  That replaces the current exact-`uvxx_core` residual with a cleaner finite-horizon boundedness residual and explicit constants `Fv(M), Fg(M)`.
