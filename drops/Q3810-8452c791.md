# Q3810 — coefficient-gain accounting for the chemotaxis branch

## Verdict

Your zero-gain computation is the right warning sign, but it is not the correct accounting once the elliptic smoothing of `v` is included and the product is estimated in a weighted ell-one algebra.

If the input `u` coefficients have decay `k^{-m}`, the correct high-mode accounting is:

```text
u                         : k^{-m}
u^gamma                   : k^{-m}
v = R(u^gamma)             : k^{-m-2}      elliptic gain +2
v_x                       : k^{-m-1}      one derivative loses 1
W = u^m * (1+v)^(-beta)    : k^{-m}        zeroth-order multiplier
flux F = W * v_x           : k^{-m}        product takes the weaker regularity
chemDiv = d_x F            : k^{-m+1}      divergence loses 1
Duhamel/lambda gain         : k^{-m-1}      heat gain +2
```

So the chemotaxis branch gives **one net power of coefficient decay per pass**:

```text
k^{-m}  ->  k^{-(m+1)}.
```

The logistic branch is better:

```text
u -> logistic(u) : k^{-m}
Duhamel gain      : k^{-m-2}
```

so the bottleneck is the chemotaxis branch, with net `+1` per pass.

## Where the zero-gain computation went wrong

The problematic line is:

```text
flux sine coefficients decay as k^{-m+1}
```

That would be true if `v_x` had only the same roughness as an ordinary derivative of `u`, or if the product estimate lost a full power. But the elliptic equation gives two powers to `v`; after differentiating once, `v_x` is still one power smoother than `u`.

The product is

```text
F = W * v_x,
W = u^m * (1+v)^(-beta).
```

`W` has the same decay class as `u`, namely `k^{-m}`. `v_x` has decay `k^{-m-1}`. In the weighted ell-one algebra, multiplication has no extra full-power loss: the product lies in the weaker class, `k^{-m}`, not `k^{-m+1}`.

Then the exterior derivative changes `k^{-m}` to `k^{-m+1}`, and the heat Duhamel factor `1/lambda_k` changes it to `k^{-m-1}`.

## Why the elliptic gain is essential

Without elliptic smoothing, the count would indeed stall. If `v_x` had only decay `k^{-m+1}`, then the flux would have decay `k^{-m+1}`, the divergence would be `k^{-m+2}`, and the Duhamel gain would return `k^{-m}`. That is your zero-gain scenario.

The reason it does not happen is exactly:

```text
v_hat_k = nu * (u^gamma)_hat_k / (mu + lambda_k),
```

so `v` gains two powers and `v_x` still gains one net power over `u`.

## Lean-relevant rule of thumb

Use a weighted ell-one algebra statement, not a pointwise convolution estimate that burns a power. The lemma shape should say:

```text
A_m is an algebra for m > 1,
A_m * A_{m+1} -> A_m,
resolver maps A_m -> A_{m+2},
d_x maps A_{m+2} -> A_{m+1},
d_x maps A_m -> A_{m-1},
Duhamel maps A_{m-1} -> A_{m+1}.
```

This is the correct per-pass ladder:

```text
A_m --u^gamma--> A_m
    --resolver--> A_{m+2}
    --gradient--> A_{m+1}

A_m --weight W--> A_m
A_m * A_{m+1} --> A_m        flux
A_m --div--> A_{m-1}          chemDiv source
A_{m-1} --Duhamel--> A_{m+1}  new u
```

Low modes and the zeroth chemDiv mode should be handled separately. The zeroth mode vanishes by boundary flux cancellation under Neumann conditions, so the gain discussion is only about high modes.

## Practical conclusion

The ell-one ladder is viable, but only with the elliptic gain and a sharp weighted-algebra product lemma. The chemotaxis branch improves by one coefficient power per pass; the logistic branch improves by two. The whole ladder is therefore limited to one net power per pass by the chemotaxis term, not zero.
