# Q3418 audit: chemDiv physical representative frontier

## Verdict

The next non-circular producer should **not** target closed-slab continuity of the lifted literal

```lean
Function.uncurry
  (fun t x =>
    intervalDomainLift
      (fun X : intervalDomainPoint =>
        intervalDomain.chemotaxisDiv p (u t) (v t) X) x)
```

That was the Q3415 endpoint trap. For the new L1 bridge, target instead a **continuous physical representative** on the closed spatial slab and an **interior EqOn** identifying the literal chemDiv with that representative on `x ∈ Ioo 0 1`. Endpoint pointwise equality is unnecessary and should not be demanded.

The intended representative is the product-rule derivative of

```lean
u * v_x / (1 + v)^β
```

with the elliptic equation

```lean
0 = v_xx - p.μ * v + p.ν * u^p.γ
```

used only on the open spatial interior, where `hsol.pde_v` is legitimate. Thus `v_xx` is represented by

```lean
p.μ * v - p.ν * u^p.γ
```

and the representative needs only joint continuity of `u`, `v`, `u_x`, and `v_x`, plus positivity of `u` and `1+v`. It does **not** need joint continuity of literal `intervalDomain.chemotaxisDiv`, and it does **not** need H1 or `IsPaper2BoundedBefore` outputs.

There are two layers.

This is definitional; I did not find a named general theorem, but the coupled-specialized theorem already exists as `coupledChemDivSourceLift_eq_deriv_fluxLift_interior`. The direct hsol-independent version should be:

## Endpoint and circularity warnings

* Do **not** ask for physical equality at `x = 0` or `x = 1`. `hsol.pde_v` is only available on `intervalDomain.inside`, i.e. `x.1 ∈ Ioo 0 1`. The ordinary endpoint values of `deriv` for zero-extended lifts are not the physical one-sided second derivatives needed by the PDE.

* The theorem `intervalDomain_lift_chemDiv_eq_deriv_intervalFlux_on_Icc` can unfold at endpoints, because it only says the literal chemDiv is the ordinary derivative of the literal flux primitive. That is not the same as saying the physical product-rule representative agrees at endpoints.

* Do **not** divide the parabolic PDE by `p.χ₀` to define chemDiv. `CM2Params` does not give `p.χ₀ ≠ 0`, and the parabolic PDE is only interior anyway.

* Do **not** route this through H1 energy, `IsPaper2BoundedBefore`, `TrajectoryHSigmaEnvelope`, or `DuhamelSourceTimeC1` for the chemDiv source. The representative frontier is a classical/local analytic input: joint spatial derivative continuity for `u_x` and `v_x`, plus fixed-time interior calculus and the elliptic equation for `v`.

* The existing coupled FAC route contains useful product/quotient and resolver joint-C² machinery, but some packages still mention literal `ContinuousOn (coupledChemDivSourceLift p u s) (Icc 0 1)`. For the L1 bridge, prefer the representative package above and an interior EqOn/a.e. transfer; otherwise the Q3415 endpoint obstruction reappears under a different name.
