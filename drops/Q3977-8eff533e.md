# Q3977 (cron2): differentiability of the lifted Picard iterate on `(0,1)`

The key point is that the semigroup lemma **does** apply to the `intervalDomainLift` wrapping, but not by a global rewrite.  At an interior point `z ∈ Set.Ioo 0 1`, the set `Set.Ioo 0 1` is a neighborhood of `z`, and on that neighborhood `intervalDomainLift` unfolds through the `dif_pos` branch.  Thus, for the base Picard iterate,

```lean
intervalDomainLift (truncatedConjugatePicardIter p u₀ 0 t)
  =ᶠ[𝓝 z]
fun x => intervalFullSemigroupOperator t (intervalDomainLift u₀) x
```

Then `intervalFullSemigroupOperator_hasDerivAt_fst` gives `HasDerivAt` for the right-hand side, and differentiability transfers across the eventual equality.

This is exactly the same semigroup route used in `ChemMildHolder.neumannHeat_lipschitz_of_grad`: the theorem is called, then only `.differentiableAt` is consumed.  The missing ingredient in the `hda` hole is therefore not a new semigroup theorem; it is the local “lift unwraps on the interior” bridge.

## Base case `n = 0`

A reusable helper for the base case should look like this.  The final congruence line may need the exact Mathlib spelling available in this repo (`congr_of_eventuallyEq`, or the equivalent `EventuallyEq.differentiableAt_iff` form), but the proof shape is the important part.

```lean
private lemma truncatedPicardIter_zero_lift_eventually_eq_semigroup
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {t z : ℝ} (hz : z ∈ Set.Ioo (0 : ℝ) 1) :
    intervalDomainLift (truncatedConjugatePicardIter p u₀ 0 t)
      =ᶠ[𝓝 z]
        fun x : ℝ => intervalFullSemigroupOperator t (intervalDomainLift u₀) x := by
  filter_upwards [isOpen_Ioo.mem_nhds hz] with x hx
  have hxIcc : x ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hx
  -- On `hxIcc`, the lift takes the `dif_pos` branch, and the `0` Picard
  -- iterate unfolds to the full Neumann semigroup applied to `intervalDomainLift u₀`.
  simp [intervalDomainLift, truncatedConjugatePicardIter, hxIcc]

private lemma truncatedPicardIter_zero_lift_differentiableAt
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {t z Cf : ℝ} (ht : 0 < t)
    (hu₀_meas : AEStronglyMeasurable (intervalDomainLift u₀) (intervalMeasure 1))
    (hu₀_bound : ∀ y : ℝ, |intervalDomainLift u₀ y| ≤ Cf)
    (hz : z ∈ Set.Ioo (0 : ℝ) 1) :
    DifferentiableAt ℝ
      (intervalDomainLift (truncatedConjugatePicardIter p u₀ 0 t)) z := by
  let g : ℝ → ℝ :=
    fun x : ℝ => intervalFullSemigroupOperator t (intervalDomainLift u₀) x
  have hg : DifferentiableAt ℝ g z := by
    exact (intervalFullSemigroupOperator_hasDerivAt_fst
      (t := t) ht (f := intervalDomainLift u₀)
      hu₀_meas (Cf := Cf) hu₀_bound z).differentiableAt
  have hloc :
      intervalDomainLift (truncatedConjugatePicardIter p u₀ 0 t)
        =ᶠ[𝓝 z] g := by
    simpa [g] using
      truncatedPicardIter_zero_lift_eventually_eq_semigroup
        (p := p) (u₀ := u₀) (t := t) hz
  -- One of these equivalent Mathlib spellings should close the goal, depending
  -- on imports/name resolution in the branch:
  --   exact hg.congr_of_eventuallyEq hloc.symm
  --   exact (Filter.EventuallyEq.differentiableAt_iff hloc).2 hg
  --   exact (DifferentiableAt.congr_of_eventuallyEq hg hloc.symm)
  exact hg.congr_of_eventuallyEq hloc.symm

end ShenWork.Paper2.TruncatedPositiveTimeBootstrap
```

Inside the existing `DT` proof, the `Cf` needed above can be `DT.M`.  If `DT.hbase_ball` is stated on interval points, convert it to a bound for the lifted real function by splitting on membership in `[0,1]`:

```lean
have hu₀_bound_lift : ∀ y : ℝ, |intervalDomainLift u₀ y| ≤ DT.M := by
  intro y
  by_cases hy : y ∈ Set.Icc (0 : ℝ) 1
  · simpa [intervalDomainLift, hy] using DT.hbase_ball ⟨y, hy⟩
  · simp [intervalDomainLift, hy, le_of_lt DT.hM]
```

Then the `n = 0` branch of a strengthened iterate differentiability theorem is just:

```lean
exact truncatedPicardIter_zero_lift_differentiableAt
  (p := p) (u₀ := u₀) (t := t) (Cf := DT.M)
  ht DT.hbase_meas hu₀_bound_lift hz
```

## How to handle `n + 1`

Do not try to solve the current local `hda` hole by a one-off induction only at the fixed time `t`.  The successor term contains a time integral over previous slices, so the induction hypothesis should be strengthened to a positive-time slice theorem:

```lean
theorem truncatedConjugatePicardIter_lift_differentiableAt_positive_time
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (DT : TruncatedConjugateMildExistenceData p u₀) :
    ∀ n : ℕ, ∀ τ : ℝ, 0 < τ → τ ≤ DT.T →
      ∀ z ∈ Set.Ioo (0 : ℝ) 1,
        DifferentiableAt ℝ
          (intervalDomainLift (truncatedConjugatePicardIter p u₀ n τ)) z := by
  intro n
  induction n with
  | zero =>
      intro τ hτ hτT z hz
      -- Base case: local unwrap + `intervalFullSemigroupOperator_hasDerivAt_fst`.
      have hu₀_bound_lift : ∀ y : ℝ, |intervalDomainLift u₀ y| ≤ DT.M := by
        intro y
        by_cases hy : y ∈ Set.Icc (0 : ℝ) 1
        · simpa [intervalDomainLift, hy] using DT.hbase_ball ⟨y, hy⟩
        · simp [intervalDomainLift, hy, le_of_lt DT.hM]
      exact truncatedPicardIter_zero_lift_differentiableAt
        (p := p) (u₀ := u₀) (t := τ) (Cf := DT.M)
        hτ DT.hbase_meas hu₀_bound_lift hz
  | succ n ih =>
      intro τ hτ hτT z hz
      -- Unfold:
      --   U_{n+1}(τ) = S(τ)u₀
      --       + (-χ₀) * ∫₀^τ B(τ-s)(truncatedChemFluxLifted p (U_n s)) ds
      --       + ∫₀^τ S(τ-s)(truncatedLogisticLifted p (U_n s)) ds.
      -- Prove differentiability term-by-term.

      -- 1. `S(τ)u₀`: same base semigroup argument as above.
      -- 2. logistic Duhamel: use

      --    with `F s y = truncatedLogisticLifted p (truncatedConjugatePicardIter p u₀ n s) y`.
      --    Boundedness comes from `truncatedConjugatePicardIter_ball` plus
      --    `truncatedLogisticLocal_abs_le_of_abs_le'`.

      --    full-semigroup Duhamel term with source

      --    Then apply the same full-kernel Duhamel differentiability lemma.

      -- The source bound for the IBP source is exactly the one already used in
```

The successor case should be developed as a named lemma, not inline at the MVT site.  The existing file has already set up the correct analytic ingredients:

* `truncatedConjugatePicardIter_ball` gives the uniform ball bound for every iterate slice.
* The logistic source bound is already coded locally as `truncatedLogisticLocal_abs_le_of_abs_le'`.
* The flux derivative bound is already coded locally as `truncatedChemFluxLifted_deriv_abs_le_of_ball_grad`.
* The full-kernel Duhamel differentiability theorem already exists as `intervalFullCoupledDuhamel_grad_integral_hasDerivAt` in the full-kernel Leibniz infrastructure.

So the right end-state for the local hole at lines 285–300 is very small:

```lean
set f := intervalDomainLift (truncatedConjugatePicardIter p u₀ n t) with hf_def

have hda : ∀ z ∈ Set.Ioo (0 : ℝ) 1, DifferentiableAt ℝ f z := by
  intro z hz
  subst f
  exact truncatedConjugatePicardIter_lift_differentiableAt_positive_time
    (p := p) (u₀ := u₀) DT n t ht htT z hz
```
