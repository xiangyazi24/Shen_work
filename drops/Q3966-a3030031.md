# Q3966: B-form Picard gradient window after IBP

## Short conclusion

After the IBP rewrite

```text
B_N(t) Q = S(t) Q'
```

the source in the heat-gradient estimate is

```text
L_n - ŽćŌéĆ Q'_n,
```

not `L_n - ŽćŌéĆ Q_n`. Therefore the chemotaxis source envelope must bound the **spatial derivative of the flux**. In that after-IBP equation, `B_F = 0` is not the correct generic choice. The flux derivative contains the iterate gradient, so the correct window has a genuine feedback coefficient

```lean
B_F = ╬ō_M
```

where `╬ō_M` is the uniform bound for the resolver gradient `|R_x(U)|` on the `M`-ball.

With the usual notation

```text
u = lift (U_n s)
v = R(U_n s)
g = v_x = resolverGrad(U_n s)
h = v_xx = (resolverGrad(U_n s))'
D = (1 + v)^╬▓
Q = u_+ * g / D,
```

and assuming

```text
|u| Ōēż M,
|u_x| Ōēż G,
0 Ōēż v,
|g| Ōēż ╬ō_M,
|h| Ōēż H_M,
╬▓ Ōēź 0,
```

the product-rule bound is

```text
|Q'| Ōēż ╬ō_M * G + M * H_M + ╬▓ * M * ╬ō_M^2.
```

Thus the after-IBP source constants should be

```lean
A_F = M * H_M + p.╬▓ * M * ╬ō_M ^ 2
B_F = ╬ō_M
```

and the full heat-source envelope is

```lean
|L_n - p.ŽćŌéĆ * Q'_n|
  Ōēż A_L + |p.ŽćŌéĆ| * (A_F + B_F * G).
```

So: `B_F = 0` is correct only for a different, non-after-IBP flux envelope that bounds `Q_n` itself. It is not correct for the after-IBP source `Q'_n`.

## 1. Product-rule calculation

For

```text
Q = u_+ * g * (1 + v)^(-╬▓),
```

where `u_+ = positivePart u`, formally

```text
Q'
  = (u_+)' * g * (1 + v)^(-╬▓)
    + u_+ * g' * (1 + v)^(-╬▓)
    - ╬▓ * u_+ * g * v' * (1 + v)^(-╬▓-1).
```

Since `v' = g` and `g' = h = v_xx`, this is

```text
Q'
  = (u_+)' * g / (1 + v)^╬▓
    + u_+ * h / (1 + v)^╬▓
    - ╬▓ * u_+ * g^2 / (1 + v)^(╬▓ + 1).
```

If `0 Ōēż v` and `0 Ōēż ╬▓`, then

```text
(1 + v)^(-╬▓)     Ōēż 1,
(1 + v)^(-╬▓ - 1) Ōēż 1.
```

Also

```text
0 Ōēż u_+ Ōēż M,
|(u_+)'| Ōēż |u'| Ōēż G
```

at every point where the classical chain rule applies, or a.e. in the weak/Lipschitz chain-rule sense. Hence

```text
|Q'|
  Ōēż |(u_+)'| * |g| + u_+ * |h| + ╬▓ * u_+ * |g|^2
  Ōēż G * ╬ō_M + M * H_M + ╬▓ * M * ╬ō_M^2.
```

This is affine in the Picard-gradient bound `G`:

```text
|Q'| Ōēż A_F + B_F * G
```

with

```lean
A_F = M * H_M + p.╬▓ * M * ╬ō_M ^ 2
B_F = ╬ō_M
```

## 2. What are `╬ō_M` and `H_M`?

You need two resolver envelopes:

```text
╬ō_M Ōēź sup |R_x(u)|,
H_M Ōēź sup |R_xx(u)|,
```

uniformly for all slices with `|u| Ōēż M`.

In the current repository, the quantitative gradient envelope has the spectral shape

```lean
╬ō_M =
  Real.sqrt (Ōłæ' k : ŌäĢ,
    (ShenWork.PDE.intervalNeumannResolverGradWeight p k) ^ 2)
      * (2 * sourceAmp_M)
```

where `sourceAmp_M` is the sup-envelope for the elliptic resolver source. In the current checked files that source is `p.╬Į * u ^ p.╬│`, so

```lean
sourceAmp_M = p.╬Į * M ^ p.╬│
```

If your cron1 branch uses the normalized truncated source `(u_+)^p.m`, then use instead

```lean
sourceAmp_M = M ^ p.m.
```

For the second derivative, use the elliptic equation. If the resolver solves

```text
-R_xx + ╬╝ R = Žü(u),
```

then

```text
R_xx = ╬╝ R - Žü(u),
```

so an `M`-only second-derivative envelope is

```text
H_M = ╬╝ * V_M + S_M,
```

where

```text
V_M Ōēź sup |R(u)|,
S_M Ōēź sup |Žü(u)|.
```

For the current repository source `Žü(u) = p.╬Į * u^p.╬│`, take

```text
S_M = p.╬Į * M ^ p.╬│.
```

For a normalized source `Žü(u) = (u_+)^p.m`, take

```text
S_M = M ^ p.m.
```

If the resolver has `╬╝ = 1` in the local branch, this reduces to

```text
H_M = V_M + S_M.
```

## 3. Correct source constants in the gradient window

The logistic envelope remains

```lean
A_L = M * (p.a + p.b * M ^ p.╬▒)
```

assuming `|u| Ōēż M` and `0 Ōēż M`.

For the after-IBP chemotaxis source, use

```lean
A_F = M * H_M + p.╬▓ * M * ╬ō_M ^ 2
B_F = ╬ō_M
```

so the source bound is

```lean
|L_n(s,y) - p.ŽćŌéĆ * Q'_n(s,y)|
  Ōēż A_L + |p.ŽćŌéĆ| * (A_F + B_F * G).
```

In the notation of `IntervalTruncatedGradientWindow`, this means the recurrence is

```lean
G_{n+1}
  Ōēż Cg * M / sqrt (lo - a)
      + Cg * (2 * sqrt (hi - a)) *
          (A_L + |p.ŽćŌéĆ| * A_F)
      + Cg * (2 * sqrt (hi - a)) * |p.ŽćŌéĆ| * B_F * G.
```

Equivalently,

```lean
G_{n+1}
  Ōēż truncWindowAffine M A_L A_F B_F p.ŽćŌéĆ a lo hi G.
```

The contraction coefficient is

```lean
truncWindowB B_F p.ŽćŌéĆ a hi
  = Cg * (2 * sqrt (hi - a)) * |p.ŽćŌéĆ| * B_F
  = Cg * (2 * sqrt (hi - a)) * |p.ŽćŌéĆ| * ╬ō_M.
```

This can be made `< 1` by choosing the window short enough. If `Cg * |p.ŽćŌéĆ| * ╬ō_M > 0`, it suffices to impose

```text
hi - a < (1 / (2 * Cg * |p.ŽćŌéĆ| * ╬ō_M))^2.
```

If `╬ō_M = 0` or `ŽćŌéĆ = 0`, the feedback coefficient is already zero.

A Lean-shaped closure pattern is:

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow

open ShenWork.Paper2.TruncatedGradientWindow

noncomputable section

/-- The gradient window must use the fixed point of the affine recurrence. -/
def afterIBPWindowG
    (M A_L A_F B_F Žć a lo hi : ŌäØ) : ŌäØ :=
  truncWindowFixedG M A_L A_F B_F Žć a lo hi

    {M A_L A_F B_F Žć a lo hi : ŌäØ}
    (hB : truncWindowB B_F Žć a hi < 1) :
    truncWindowAffine M A_L A_F B_F Žć a lo hi
        (afterIBPWindowG M A_L A_F B_F Žć a lo hi)
      Ōēż afterIBPWindowG M A_L A_F B_F Žć a lo hi := by
  unfold afterIBPWindowG truncWindowFixedG truncWindowAffine
  exact affine_fixed_closes
    (A := truncWindowA M A_L A_F Žć a lo hi)
    (B := truncWindowB B_F Žć a hi)
    hB

```

So if `B_F > 0`, do not keep the old raw choice

```lean
G = truncWindowA M A_L A_F p.ŽćŌéĆ a lo hi.
```

Use instead

```lean
G = truncWindowFixedG M A_L A_F B_F p.ŽćŌéĆ a lo hi.
```

## 4. A subtle formalization issue: `positivePart` is not C┬╣

The formula

```text
(u_+)' = 1_{u > 0} u'
```

is not a globally classical pointwise identity for arbitrary differentiable `u`: `positivePart` has a kink at `0`. Therefore a pointwise theorem of the shape

```lean
ŌłĆ y, |deriv (truncatedChemFluxLifted p (U n s)) y| Ōēż A_F + B_F * G
```

may be too strong unless you also prove that the iterate is strictly away from the kink, or you replace `positivePart` by a smooth truncation.

The robust analytic statement is an a.e./weak-derivative bound:

```text
|(u_+)'| Ōēż |u'|   a.e.
```

and then the same estimate for `Q'` holds a.e. This is often enough for Sobolev/weak IBP, but it is not the same as a classical `ŌłĆ y, deriv ...` bound. For the current Lean wiring, this matters because `gradDuhamel_shifted_sup_bound` is phrased with pointwise `ŌłĆ y` source bounds. If the source is literally `Q'`, the formalization may need either:

1. a smooth positive truncation,
2. a strict-positivity/non-crossing result for the iterates on the window, or
3. a new heat-gradient estimate consuming an a.e. `LŌł×` bound on the weak derivative.

This is another reason the after-IBP route is heavier than it first looks.

## 5. Simpler approach that avoids tracking `Q'_n`

There is a simpler approach, but it is not the same as the after-IBP heat-source equation.

If you keep the chemotaxis term in the B-form/conjugate-kernel formulation, the natural bounded object is the flux itself:

```text
Q = u_+ * R_x / (1 + R)^╬▓,
```

and this has an `M`-only bound

```text
|Q| Ōēż M * ╬ō_M.
```

That gives a `B_F = 0` envelope for `Q`, not for `Q'`:

```lean
A_F_for_Q = M * ╬ō_M
B_F_for_Q = 0
```

This is mathematically legitimate only for estimates whose input is the flux `Q` itself. It does **not** justify the after-IBP source bound for `L - ŽćŌéĆ Q'`.

For a Picard gradient window, the clean choices are therefore:

* **After-IBP route:** use `Q'`; then `B_F = ╬ō_M > 0` and close by a small-window affine fixed point.
* **Non-IBP / flux route:** use estimates formulated directly for the conjugate-kernel flux leg; then the source bound can use `|Q| Ōēż M ╬ō_M` with `B_F = 0`, but the needed kernel estimate must be a genuine B-kernel estimate for the gradient leg, not the ordinary heat-gradient estimate applied to `L - ŽćŌéĆ Q'`.
* **Regularity-black-box route:** avoid proving a pointwise `Q'` bound inside the Picard window; first construct the mild fixed point using `M`-ball estimates, then obtain positive-time C┬╣ regularity by a separate parabolic regularity theorem. This is often the Lean-friendly route because it isolates the nonsmooth `positivePart` chain rule and the resolver second-derivative estimate.

The key correction is: once the term has been rewritten as `S(t-s)(Q'_n)`, the feedback through `u_x` is unavoidable. In that equation, `B_F = 0` is false except in degenerate cases such as `╬ō_M = 0`.
