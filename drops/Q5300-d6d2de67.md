## 1.1 A weighted version adapted to the population variable

Fix `eta > 0`. Let `phi : R -> R` be a nonzero continuous compactly supported function with

```text
support phi subset [0,1].
```

For integers `n >= 1`, define the physical perturbation

```text
d_n(x) := exp(-eta*n) * phi(x-n).
```

Then

```text
||d_n||_infinity <= exp(-eta*n) ||phi||_infinity -> 0.
```

Thus these perturbations tend to zero in the unweighted BUC norm. They can be added to a bounded nonnegative profile on sufficiently far-right intervals while preserving a fixed physical box.

But the weighted perturbation is

```text
W_n(x)
  := exp(eta*x) * d_n(x)
   = exp(eta*(x-n)) * phi(x-n).
```

If

```text
psi(y) := exp(eta*y) * phi(y),
```

then

```text
W_n(x) = psi(x-n),
```

so

```text
integral_R W_n(x)^2 dx = integral_R psi(x)^2 dx > 0
```

for every `n`. The weighted mass has simply translated to the right.

If one integrable `g` dominated every `W_n^2`, then the supports `[n,n+1]` are disjoint and, for every `N`,

```text
integral_R g
  >= sum_{n=1}^N integral_[n,n+1] W_n^2
  = N * integral_R psi^2,
```

which is impossible as `N -> infinity`.

This example can be embedded into a BUC-continuous time path: choose times `t_n -> 0`, place the `n`th bump at time `t_n`, and interpolate continuously between consecutive bumps. By choosing `t_n` in terms of `exp(-eta*n)`, one can also respect a prescribed positive Hölder modulus in the unweighted BUC norm. Therefore a BUC time modulus does not, by itself, solve the weighted right-tail problem.

This is an interface counterexample, not a claim that the constructed path solves the chemotaxis PDE. The PDE restart rules it out, but the existing proof does so in `WholeLineRealL2` through semigroup continuity—not by producing a cheap pointwise envelope.

# 2. What each permitted input actually controls

## 2.1 Physical box: controls only the left tail

Assume

```text
0 <= coMovingPath c u q x <= M,
|U x| <= MU,
eta > 0.
```

Then

```text
|coMovingPath c u q x - U x| <= M + MU.
```

For `x <= 0`, this gives the uniform density bound

```text
W(q,x)^2
  <= (M+MU)^2 * exp(2*eta*x),
```

and the right-hand side is integrable on `Iic 0`.

For `x -> +infinity`, however, `exp(2*eta*x)` grows. The physical box supplies no useful right-tail estimate.

## 2.2 Per-slice `hclose`: finite tail at each time, not a uniform tail

For each fixed `q`,

```text
hclose q : Integrable (fun x => W(q,x)^2).
```

This proves that the tail of that one slice tends to zero. It does not permit choosing the cutoff uniformly in nearby times.

The quantifiers differ:

```text
per-slice hclose:
  forall q, forall eps > 0, exists N(q,eps), tail(q,N) < eps;

needed no-escape:
  forall eps > 0, exists N(eps), forall q near q0, tail(q,N) < eps.
```

Interchanging these quantifiers is exactly the right-escape error.

## 2.3 Uniform exact-weight `L²` bounds: still not enough

A bound

```text
forall q in J, integral W(q,x)^2 dx <= C
```

does not imply either

```text
integral (sup_q W(q,x)^2) dx < infinity
```

or uniform tail tightness. The escaping-bump family has constant norm.

In particular, the tempting definition

```text
g(x) := sup q in J, W(q,x)^2
```

does not help. Even when the pointwise supremum is measurable, its integrability does not follow from

```text
sup_q integral W(q)^2 < infinity.
```

## 2.4 Weighted resolver bounds preserve `L²`; they do not create a spatial envelope

The committed resolver estimates are bounded-operator estimates at the same exponential weight. They control norms such as

```text
||weightedSignalDifference||_2 <= C ||W||_2
```

and analogous first-derivative signal terms. A bounded operator on `L²` does not turn a uniformly bounded family into a pointwise-dominated family. Translation remains possible.

## 2.5 Positive-time heat smoothing gives an `L∞` bound, but a constant is not integrable

For a fixed positive lag, an `L² -> L∞` heat estimate gives

```text
|S(t)f(x)| <= C(t) ||f||_2.
```

The right-hand side is independent of `x`, hence is not an `L²` spatial envelope on `R`.

For the homogeneous term on an interior window, one can retain the Gaussian convolution and dominate all lags bounded away from zero by a fixed broader Gaussian. The Duhamel term is different: its lag `t-q` approaches zero at the terminal face. A uniform `L²` forcing bound alone does not produce a fixed pointwise Gaussian envelope for those near-zero-lag contributions.

To manufacture an envelope from the mild formula one would need a new positive-kernel Volterra majorant or a heat maximal-function theorem. Neither is part of the present repository route, and it would be substantially more Lean work than the already-proved Hilbert continuity theorem.

## 2.6 `hWx2` would not repair the basic translation issue by itself

Even a uniformly bounded family in exact-weight `H¹(R)` may translate to the right. Uniform `H¹` boundedness alone is not a spatial no-escape theorem. Therefore prohibiting `hWx2` here is appropriate, but allowing it would not automatically yield the requested `g` either.

# 5. A clean non-pointwise no-escape producer from a stronger weight

Although it is not available from the paper's exact-weight datum, the mathematically cleanest extra hypothesis is a uniform stronger-weight `H⁰` estimate.

Assume there are `sigma > 0` and `C >= 0` such that

```text
forall q in J,
  integral_R exp(2*(eta+sigma)*x) * |u_q(x)-U(x)|^2 dx <= C.
```

Then for `x >= N`,

```text
exp(2*eta*x) |u_q-U|^2
  = exp(-2*sigma*x)
      * exp(2*(eta+sigma)*x) |u_q-U|^2
  <= exp(-2*sigma*N)
      * exp(2*(eta+sigma)*x) |u_q-U|^2.
```

Hence the right tail satisfies

```text
integral_{x>=N} W(q,x)^2 dx
  <= exp(-2*sigma*N) * C,
```

uniformly in `q`.

The left tail follows from the physical box:

```text
integral_{x<=-N} W(q,x)^2 dx
  <= (M+MU)^2 * integral_{x<=-N} exp(2*eta*x) dx
  = (M+MU)^2 * exp(-2*eta*N)/(2*eta).
```

Together these estimates prove uniform square tightness without `hWx2`.

This is weaker and more natural than a pointwise envelope. But it requires initial/propagated control at `eta+sigma`; the current endpoint-weight campaign deliberately does not assume such a reserve.

# 6. The explicit pointwise envelope, conditional on a genuine right-tail theorem

To answer the envelope question literally, add the following uniform right-tail estimate on a closed positive window `J`:

```text
exists sigma Cplus,
  0 < sigma and 0 <= Cplus and
  forall q in J, forall x >= 0,
    |coMovingPath c u q x - U x|
      <= Cplus * exp(-(eta+sigma)*x).
```

Also assume

```text
eta > 0,
forall q in J, forall x, |coMovingPath c u q x| <= M,
forall x, |U x| <= MU.
```

Set

```text
Cminus := M + MU.
```

The correct density envelope is

```text
g(x) :=
  if x <= 0 then
    Cminus^2 * exp((2*eta)*x)
  else
    Cplus^2 * exp((-2*sigma)*x).
```

Equivalently, in a form convenient for integration:

```lean
let g : R -> R := fun x =>
  (Set.Iic (0 : R)).indicator
      (fun x => Cminus ^ 2 * Real.exp ((2 * eta) * x)) x
  +
  (Set.Ioi (0 : R)).indicator
      (fun x => Cplus ^ 2 * Real.exp ((-2 * sigma) * x)) x
```

Then, for every `q in J`:

```text
W(q,x)^2 <= g(x).
```

### Left half-line

For `x <= 0`:

```text
W(q,x)^2
  = exp(2*eta*x) * |u_q-U|^2
  <= Cminus^2 * exp(2*eta*x).
```

### Right half-line

For `x > 0`:

```text
W(q,x)^2
  <= exp(2*eta*x)
       * Cplus^2 * exp(-2*(eta+sigma)*x)
  = Cplus^2 * exp(-2*sigma*x).
```

# 8. A single envelope implies tightness, but tightness is the better API

Suppose one has

```text
forall q near q0, W(q,x)^2 <= g(x)
```

with `Integrable g`.

Then the tails of `g` vanish along `Icc (-n) n`, and

```text
wholeLineSquareTail (W q) n
  <= integral_{outside [-n,n]} g.
```

This produces `WholeLineSquareTightAt W (nhdsWithin q0 J)`.

But the converse does not hold: a uniformly tight family need not admit one pointwise integrable majorant. Therefore the generic theorem

```lean
tendsto_integral_sub_sq_zero_of_local_and_tight
```

captures the minimal analytic condition more faithfully than DCT.

For the exact-weight endpoint, the recommended hierarchy is:

```text
best current route:
  full-candidate Hilbert continuity;

alternative scalar route:
  local BUC convergence + WholeLineSquareTightAt;

strongest and usually unnecessary route:
  one pointwise integrable envelope + DCT.
```

# 9. False shortcuts to reject

1. **`g = sup_q density(q,.)` without an integrability proof.**
   Compactness of the time interval and pointwise continuity only make the supremum finite pointwise; they do not make it integrable.

2. **Uniform exact-weight energy implies uniform tails.**
   False by translation.

3. **BUC time continuity prevents right escape.**
   False because the physical bump amplitude can shrink exactly as the exponential weight grows.

4. **The physical box controls both tails.**
   It controls the left tail only; on the right the weight grows.

5. **Weighted resolver estimates create decay.**
   Same-weight `L²` boundedness is not a pointwise tail estimate.

6. **A positive-time `L² -> L∞` estimate gives an integrable envelope.**
   A constant-in-space bound is not integrable on `R`.

7. **Per-slice `hclose` permits choosing a uniform cutoff.**
   This is an invalid quantifier exchange.

8. **`hWx2` automatically supplies the envelope.**
   Exact-weight `H¹` boundedness still allows translations unless accompanied by tightness or a stronger tail estimate.

9. **Use a weighted-population time derivative to prove `hZcont`.**
   This is the circular route the generator/energy architecture was designed to avoid.

10. **Ignore totalization.**
    `wholeLineRealL2Total` is zero outside the measurable square-integrable branch. Always retain the measurable and square-integrable slice facts when identifying its representative.
