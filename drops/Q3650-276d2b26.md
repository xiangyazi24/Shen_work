## Verdict

Do **not** duplicate the common-fold-noise interface.  It is already the right consumer-facing shape.

The next faithful producer should be a real-space folded Gaussian integral identity, not a spectral theorem and not a direct optimal-transport coupling.  The smallest honest intermediate is the **per-cell folded-integral identity** for the map

```lean

```

This reduces the hard pushforward theorem to period-2 tiling plus two elementary fold-on-cell algebra lemmas.

## Why not direct kernel coupling?

The direct kernel route can define the two marginal measures using the already-proved facts:

```lean

intervalNeumannFullKernel_integrable
intervalNeumannFullKernel_intervalMeasure_integral_eq_one
```

from `IntervalFullKernelSupBound.lean` / `IntervalFullKernelMass.lean`.  But after that, one still needs a coupling of the two densities supported in

```lean
{z : ℝ × ℝ | |z.1 - z.2| ≤ |x.1 - y.1|}
```

The repo does not contain a monotone/quantile/Strassen coupling theorem.  A product coupling is false for the support condition.  So direct kernel coupling is much larger than the folded common-noise route.

## Can a spectral route avoid measure pushforward?

Not for the current `NeumannHeatCommonFoldNoiseFor` interface.

The spectral files are useful but produce scalar equality, not a probability law.  In particular:

- `IntervalFullKernelSpectralClean.lean` proves `intervalFullSemigroupOperator_eq_cosineHeatValue_clean` on `Ioo` for continuous `f`, and `intervalFullSemigroupOperator_eq_cosineHeatValue_Icc` under a coefficient bound.
- `IntervalSemigroupApproxIdentity.lean` and `IntervalCosineInversion.lean` provide spectral approximate-identity and pointwise inversion infrastructure under `ℓ¹` Fourier/cosine summability.

Those facts can show that two scalar formulas agree, but they do not supply a common measure `ν` such that both semigroup values are integrals against the same folded noise.  They also tend to require global `Continuous f` or cosine-coefficient hypotheses, while the eventual application is `f = intervalDomainLift u₀`, which is naturally only an interval function; it may fail to be globally continuous at the endpoints.

So the current common-fold-noise interface is **too strong for spectral facts alone**, but it is exactly right for the reflected-Brownian/folded-Gaussian route because it contains the common noise law needed for contraction.

## Step 2: define the folded heat integral

```lean
/-- The real-line folded Gaussian representation candidate for the interval
Neumann semigroup. -/
noncomputable def foldedHeatIntegral
    (t : ℝ) (x : intervalDomainPoint) (f : ℝ → ℝ) : ℝ :=
  ∫ z : ℝ,
    heatKernel t z * f (addCircleTwoFoldTranslatePoint x z).1
```

For the eventual application to `intervalDomainLift u₀`, do not require global continuity of `f`.  A good working hypothesis is:

```lean
hf_cont : ContinuousOn f (Set.Icc (0 : ℝ) 1)
```

`hf_bound` follows from `hf_cont` by compactness, but carrying it explicitly can make the first proof easier.  This is better than `Continuous f`, because `intervalDomainLift u₀` is not necessarily globally continuous outside `[0,1]`.

## Step 3: per-cell folded integral identity

This is the smallest analytic lemma that truly reduces the pushforward identity.  Use interval integrals on each period-2 cell; the endpoint convention is irrelevant for continuous integrands.

```lean
/-- On one period-2 cell for the noise variable, the folded integrand splits into
its two interval branches. -/
theorem foldedHeat_cell_integral_eq_two_branches
    {t : ℝ} (ht : 0 < t)
    {f : ℝ → ℝ}
    (hf_cont : ContinuousOn f (Set.Icc (0 : ℝ) 1))
    (x : intervalDomainPoint) (k : ℤ) :
    (∫ z in (-x.1 + 2 * (k : ℝ))..(-x.1 + 2 * (k : ℝ) + 2),
        heatKernel t z * f (addCircleTwoFoldTranslatePoint x z).1)
      =
      (∫ y in (0 : ℝ)..1,
        heatKernel t (y - x.1 + 2 * (k : ℝ)) * f y)
      +
      (∫ y in (0 : ℝ)..1,
        heatKernel t (-y - x.1 + 2 * ((k : ℝ) + 1)) * f y) := by
  -- Proof route:
  -- 1. split the interval at `-x.1 + 2*k + 1` using
  --    `intervalIntegral.integral_add_adjacent_intervals`;
  -- 2. first half: substitute `z = -x.1 + 2*k + y`, `y ∈ [0,1]`;
  -- 3. second half: substitute `z = -x.1 + 2*k + 2 - y`, `y ∈ [0,1]`;
  -- 4. use `addCircleTwoFoldPoint_coe_add_period_eq` and
  --    `addCircleTwoFoldPoint_neg_add_period_eq` to rewrite the folded point to `y`;
  -- 5. close the algebra by `ring`/`ring_nf`.
  -- Existing useful lemmas:
  -- * `intervalIntegral.integral_comp_add_left`
  -- * `intervalIntegral.integral_comp_sub_left`
  -- * `intervalIntegral.integral_add_adjacent_intervals`
  -- * `heatKernel` continuity from `HeatSemigroup.lean`
  admit
```

This lemma is much smaller than the full pushforward theorem and gives an immediate sanity check that the `AddCircle` fold has the correct branch behavior.

## Step 4: scalar folded representation of the semigroup

After Step 3, prove the scalar equality.

```lean
/-- The full Neumann semigroup equals the folded real-line Gaussian integral. -/
theorem intervalFullSemigroupOperator_eq_foldedHeatIntegral
    {t : ℝ} (ht : 0 < t)
    {f : ℝ → ℝ}
    (hf_cont : ContinuousOn f (Set.Icc (0 : ℝ) 1))
    (hf_bound : ∃ C, 0 ≤ C ∧ ∀ y ∈ Set.Icc (0 : ℝ) 1, |f y| ≤ C)
    (x : intervalDomainPoint) :
    intervalFullSemigroupOperator t f x.1 =
      foldedHeatIntegral t x f := by
  -- Proof route:
  -- A. Unfold `foldedHeatIntegral`.
  -- B. Use `ShenWork.integral_eq_tsum_integral_Ioc_offset` with offset `-x.1`
  --    on the integrable function
  --      `z ↦ heatKernel t z * f (addCircleTwoFoldTranslatePoint x z).1`.
  --    Integrability follows from `heatKernel_integrable ht` and `hf_bound`.
  -- C. Rewrite each cell integral by `foldedHeat_cell_integral_eq_two_branches`.
  -- D. Reindex the two resulting `ℤ` sums using `k ↦ -k` and `k ↦ -k-1`, and
  --    `heatKernel_neg`, to get
  --      `∑ k ∫ y, [heatKernel t (x-y+2k) + heatKernel t (x+y+2k)] * f y`.
  -- E. Use `intervalNeumannFullKernel` definition and the same summable-integral
  --    style as `IntervalFullKernelMass.lean` / `IntervalFullKernelInterchange.lean`
  --    to exchange `tsum` and the interval integral, yielding
  --      `∫ y, intervalNeumannFullKernel t x.1 y * f y ∂ intervalMeasure 1`.
  -- F. Fold back to `intervalFullSemigroupOperator` by definition.
  admit
```

This is the first theorem that actually says the folded common-noise model represents the existing `intervalFullSemigroupOperator`.  It still avoids constructing a `Measure.withDensity` probability law.

## Why Step 3 is the smallest honest next lemma

The full pushforward identity is too large as a first step: it combines fold geometry, interval splitting, full-line tiling, two integer reindexings, integrability, and `tsum`/integral interchange.  The per-cell identity isolates only the fold geometry plus elementary interval changes of variables.  Once it is proved, all remaining work is standard real-space kernel summation and can reuse the existing tiling and mass/interchange proof patterns.

## Interface assessment

`NeumannHeatCommonFoldNoiseFor` is the right strength for the contraction proof: it contains an actual common noise law, so the fold contraction lemma can be used pointwise under one integral.  It is stronger than spectral scalar equality, so spectral facts cannot produce it alone.  It is weaker and cleaner than the older `NeumannHeatContractiveCouplingFor`, because it avoids a pushforward measure on `ℝ × ℝ` and the extra `hdiff_sm` measurability obligation; the current file already has a direct consumer
