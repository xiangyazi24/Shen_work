## Why a plain successor theorem is not the first target

A theorem named something like

```lean
conjLogSourceTimeC1On_succ_of_conjLogSourceTimeC1On
```

would be underpowered for chi-nonzero.  The successor `conjugatePicardIter (n+1)` is generated by `intervalConjugateDuhamelMap`, whose source is

```lean
bFormSourceCoeffs p (conjugatePicardIter p u₀ n)
```

not merely the logistic source of level `n`.  The correct successor statement would have to consume a predecessor B-form package:

```lean
DuhamelSourceTimeC1On
  (bFormSourceCoeffs p (conjugatePicardIter p u₀ n)) 0 W
```

plus restart representation, positivity, closed-window bounds, and joint continuity data for the successor slice, then call the generic
`sourceTimeC1On_succ_of_sourceTimeC1On` with `w := conjugatePicardIter p u₀ (n+1)`.

So the eventual tower should be B-form-aware, not logistic-only.

## False or circular routes to avoid

Do not use any of the following to prove `hsrcBDirect`:

* `GradientMildSolutionData` for the conjugate fixed point unless an independently proved, non-`hgradB` bridge exists.  The known old route was rejected.
* `IsPaper2ClassicalSolution` or any theorem that already consumes `hpde_u`; source-time regularity is an input to the `hpde_u` provider, not a consequence to be backfilled from it.
* `BFormBankedInputs.hpde_u`, `intervalConjugateMildSolution_pde_u_*`, or `intervalConjugatePicardLimit_classicalRegularity_direct`.
* The gradient-specific convenience theorem
  `coupledChemDivSource_timeC1On_of_gradientSolution` for the conjugate limit.  Use the generic residual theorem inside `IntervalChemDivWinDischarge` instead.
* Endpoint continuity of `intervalDomainConstExtend` for the chem-div source.  The endpoint-safe chem-div package is `ChemDivCosineFourierDataIoo`; do not revive the false closed-endpoint const-extension continuity assumption.
* Any `hgradB` or B-kernel conversion from a gradient mild solution.
