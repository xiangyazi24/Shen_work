The `hsrc0` producer uses the iterate-side bootstrap in `IntervalDomainHresWiring.duhamelSourceBddOn_of_core`: datum-side source coefficient bounds, n-uniform iterate source envelopes, iterate-to-limit coefficient convergence, and the tower coefficient-time-continuity input. It does not use `Hu`, the Hu finite-cover envelope, or the downstream no-K1 wrapper.

The Hu coefficient envelope is independently produced from `HasTimeNeighborhoodSpectralAgreement` by `huRestartCoeff_henv_of_timeNeighborhood`. Thus the proposed bridge cannot create the old `hsrc0 ← representation ← hsrc0` loop.

## What not to do next

### Do not try to remove the PPID power-source K1 fields by copying the chi-zero no-K1 route

The PPID no-envelope input package still has:

```lean
adotPow
hderivPow
hadotPowCont
hMdotPow
```

The no-K1 derivation in `IntervalResolverSourceWindowEnvelopeOnlyNoJointNoK1Inputs.lean` is chi-zero-specific. It consumes:

```lean

hsrc0 : DuhamelSourceBddOn (patchedSource p u₀ D.u) D.T
```

and derives power-source K1 from the logistic patched-source package and the chi-zero mild/fixed-point shape. For `χ₀ < 0`, the `u` equation contains the chemotaxis term, so the same logistic-only patched source is not the full source of the mild map. A direct generalization would be mathematically dishonest.

### Do not claim strict-negative unconditionality from the current EWA fixed-point/floor milestone

`chiNegStrong_EWA_fixedPoint_of_floor` produces a fixed point, but it still carries framework-wide analytic and contraction hypotheses. More importantly, `ChiNegStrongFaithfulRealizationFrontier` asks for a reduced classical core for every `δ > 0`, while the construction interfaces such as `ChiNegDatumUniformConstructionStrong` are existential in the lifespan. The direction used by `chiNeg_strongDatumUniform_of_faithfulFrontier` is:

```text
∀ δ > 0 faithful realization frontier
  → choose one δ
  → strong datum-uniform construction
```

The reverse direction is unavailable without a real horizon-restriction/small-time realization theorem for the reduced core, and the named `realSlice_reducedCore` / `realizes_evalST_discharged` atoms are still not all produced.
