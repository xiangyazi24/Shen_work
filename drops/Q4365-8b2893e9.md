# Definitive verdict

The package `(a)`–`(f)` is **not yet sufficient** to implement the current

```lean
SourceFromSolutionEnvelopePass S :=
  ∀ {m}, 1 ≤ m →
    WindowCoefficientEnvelope m ... →
    WindowSourceEnvelope (m - 1) ...
```

as stated.

There are two different issues, and they should not be conflated:

1. **For the finite ladder actually needed to reach source order 2**, the immediate analytic gap is **one missing spatial derivative of the chemotactic flux**.  A pointwise `C²` bound on `Q` closes only the first source pass, not the final pass-2.
2. **For the universally quantified interface for every `m ≥ 1`**, fixed `C²` estimates can never suffice.  One would need a genuine general-order nonlinear coefficient/Sobolev algebra theorem, or the interface should be reduced to the two values actually used, `m = 2` and `m = 3`.

So the single most likely missing hard lemma for the intended finite route is:

```lean
-- proposed semantic name
chemDivSource_quadraticDecay_of_flux_C3_window
```

namely a uniform-window theorem saying that if the flux `Q_s` has an integrable third derivative, `Q_s(0)=Q_s(1)=0`, and `Q_s''(0)=Q_s''(1)=0`, then

```lean
|(k * π) * sineCoeff Q_s k| ≤ C / (k * π)^2.
```

Equivalently, prove directly that the divergence source `∂ₓQ_s` is a uniformly bounded weak-`H²_N` / closed-`C²` Neumann family and apply the already existing quadratic cosine-coefficient decay theorem.

# 1. The product/convolution question

## Fixed-order direct route: no convolution lemma is needed

If one already has a pointwise derivative tower for

```text
Q = u · Rₓ · (1 + R)^(-β),
```

then the product has already been handled by ordinary product and chain rules.  To deduce a **fixed** coefficient decay order from a bound on a derivative of `Q`, one can integrate by parts directly.  There is no need to expand the cosine coefficients of `u · g` as a convolution.

Thus the following implication is legitimate without a Fourier-product theorem:

```text
uniform C²/W^{2,1} bound on Q
+ the required endpoint identities
→ a fixed O(k⁻²) estimate for the relevant coefficient of Q.
```

The fact that `Q` is a product does not matter once a derivative formula and a uniform derivative bound for the product itself have been proved.

## But `C²(Q)` does not preserve arbitrary input order `m`

A standard `C²` coefficient-decay lemma gives one fixed exponent, namely order 2.  It does **not** prove

```text
solution coefficients O(k⁻ᵐ)
→ Q coefficients O(k⁻ᵐ)
```

for arbitrary `m`.

For `m > 2`, either:

- prove higher pointwise derivative bounds on `Q` up to the corresponding order and iterate IBP; or
- prove a weighted Fourier/Wiener/Sobolev algebra theorem showing that multiplication and the Nemytskii maps preserve the weighted coefficient class.

That general-order result is the missing ingredient if the current `∀ m` type is retained.  A suitable semantic statement would be something like

```lean
windowFluxEnvelope_of_windowSolutionEnvelope
    (hu : WindowCoefficientEnvelope m c T' uCoeff) :
    WindowSineCoefficientEnvelope m c T' fluxCoeff
```

with the resolver multiplier and weighted product/composition estimates built into its proof.

The repository note in `IntervalMildPositiveTimeRegularityV6.lean` already acknowledges that the abstract general-order `MemHSigma`/product route was not plugged into the pointwise `WindowSourceEnvelope` target.  Fixed C² estimates do not repair that mismatch for all `m`.

# 2. The crucial divergence arithmetic

Let `a_k := kπ`, suppressing normalization factors in the sine coefficient.  With

```text
Q(0)=Q(1)=0,
```

two integrations by parts give

```text
∫₀¹ Q(x) sin(a_k x) dx
  = -a_k⁻² ∫₀¹ Q''(x) sin(a_k x) dx.
```

Therefore a uniform `L¹` or sup bound on `Q''` gives

```text
sineCoeff(Q,k) = O(a_k⁻²).
```

But the divergence coefficient is

```text
cosineCoeff(∂ₓQ,k) = a_k · sineCoeff(Q,k),
```

so this yields only

```text
cosineCoeff(∂ₓQ,k) = O(a_k⁻¹).
```

This is exactly source order 1, not source order 2.

To obtain source order 2, one more IBP is needed.  If

```text
Q''(0)=Q''(1)=0
```

and `Q''' ∈ L¹`, then

```text
∫₀¹ Q''(x) sin(a_k x) dx
  = a_k⁻¹ ∫₀¹ Q'''(x) cos(a_k x) dx,
```

hence

```text
sineCoeff(Q,k) = O(a_k⁻³)
```

and finally

```text
cosineCoeff(∂ₓQ,k) = O(a_k⁻²).
```

Equivalently, set

```text
Schem := ∂ₓQ.
```

Then the final estimate is exactly the ordinary quadratic cosine decay of `Schem`, requiring

```text
Schem ∈ C²/W^{2,1},
Schem'(0)=Schem'(1)=0.
```

In terms of `Q`, these are

```text
Q ∈ C³/W^{3,1},
Q''(0)=Q''(1)=0.
```

The currently listed flux result `(d)`, `Q` pointwise `C²`, is therefore **one derivative short**.

## Why the endpoint condition `Q''=0` is plausible

For

```text
Q = u · Rₓ · (1+R)^(-β),
```

Neumann parity gives, at `x=0,1`,

```text
uₓ = 0,
Rₓ = 0,
Rₓₓₓ = 0.
```

Expanding `Q''`, every endpoint term contains one of these vanishing odd derivatives, so `Q''(0)=Q''(1)=0` should be a routine algebraic consequence once the derivative tower is available.

The nontrivial new bound is `Q'''`.  It introduces `R''''`; that can be replaced using the elliptic equation, schematically

```text
R'''' = μ R'' - (ν u^γ)'',
```

so the already proved C² bound for `u^γ` is exactly the right input.  This is why the missing third-flux-derivative lemma is finite and concrete rather than a new infinite regularity theory.

# 3. Correct order arithmetic of the ladder

The source pass itself has the sharp arithmetic

```text
solution order m
→ resolver order m+2
→ resolver gradient order m+1
→ flux order m
→ divergence-source order m-1.
```

Thus the target

```lean
WCE m → WSE (m - 1)
```

is correct.

The heat/Duhamel pass then gains two orders:

```text
WSE r → WCE (r+2).
```

Consequently a complete heat-plus-source cycle gains one source order:

```text
WSE r
→ WCE (r+2)
→ WSE (r+1).
```

The finite ladder needed for a pass-2 source envelope is therefore

```text
WSE 0
  -- heat +2 --> WCE 2
  -- source -1 --> WSE 1
  -- heat +2 --> WCE 3
  -- source -1 --> WSE 2.
```

Only the source passes at `m=2` and `m=3` are needed.

- At `m=2`, the target is `WSE 1`.  `Q ∈ C²` is enough for the chem-divergence term because it gives `O(k⁻¹)` after the divergence multiplier.
- At `m=3`, the target is `WSE 2`.  `Q ∈ C²` is no longer enough; the `Q'''` / `Q''`-endpoint lemma above is needed.

This gives a clean implementation choice:

```lean
structure SourceFromSolutionEnvelopePassLow ... where
  pass2 : WCE 2 ... → WSE 1 ...
  pass3 : WCE 3 ... → WSE 2 ...
```

or equivalently restrict the current quantifier by `m ≤ 3`.  This avoids building an unused all-orders convolution algebra.

# 4. A separate seed-interface check

The displayed finite ladder begins with `WSE 0`, not merely `WCE 0`.

obtained from the uniform sup bounds for the logistic and chem-divergence source coefficients.  This should be routine and may already live in another ladder module, but it must be explicitly present in the final chain.  `WCE 0` alone does not seed the displayed `WSE 0 → WCE 2` iteration.

This is a wiring/seed issue, not the main nonlinear missing lemma.

# 5. The summability endpoint

Once a genuine `WindowSourceEnvelope 2` has been built, its envelope is dominated by `C/k²`, hence is `ℓ¹`.  This is exactly strong enough for the eigenvalue-weighted restart-coefficient estimate:

```text
Σ λ_k |restartCoeff_k| < ∞.
```

For the Duhamel term, the key scalar estimate is

```text
λ_k ∫₀ᵗ exp(-λ_k(t-s)) |source_k(s)| ds
≤ envelope(k),
```

because

```text
∫₀ᵗ λ_k exp(-λ_k(t-s)) ds ≤ 1.
```

Thus summing is reduced to `Summable envelope`, supplied by the pass-2 decay.  The homogeneous restart term is handled by heat exponential summability.

This adapter is analytically light, but it still needs a concrete theorem.  At the inspected commit:

- the name `restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope` appears in the design comment in `IntervalMildPositiveTimeRegularityV6.lean`;
- the visible proved theorem `restartSliceCoeff_eigenvalueSummable` instead consumes a full `DuhamelSourceTimeC1` package;
- `IntervalBFormRestartCoeffSummability.localRestartCoeff_abs_summable_of_sourceC1On` proves only **unweighted** absolute summability.

So the pass-2-envelope-to-weighted-`ℓ¹` adapter should be checked as a real declaration, not assumed from the comment.  If absent, it is a routine but load-bearing final wiring lemma.

# Final answer

- **No**, a product/convolution lemma is not needed for the intended fixed-order direct-IBP proof once a pointwise derivative bound for the whole product `Q` is available.
- **Yes**, fixed `C²(Q)` is insufficient for the final source pass: after multiplication by the divergence factor `kπ`, it yields only source order 1.
- The sharp full-cycle arithmetic is `WSE r → WCE (r+2) → WSE (r+1)`; reaching `WSE 2` uses precisely the `m=2` and `m=3` source passes.
- The single most likely missing hard lemma is the **uniform `Q'''` / `Q''`-endpoint theorem that makes `∂ₓQ` a C²-Neumann (weak-H²) source and hence gives quadratic chem-divergence coefficient decay**.
- If the current `∀ m` type is retained, there is an additional larger mismatch: one needs an all-orders weighted nonlinear product/composition theorem.  The finite ladder should instead expose only the two required passes unless that general theory is genuinely desired.
- Finally verify two routine adapters: a genuine `WSE 0` seed and a real pass-2-envelope-to-eigenvalue-weighted-restart-summability theorem.
