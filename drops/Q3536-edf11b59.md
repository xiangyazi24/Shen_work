## Endpoint and zero-time traps

### 1. The `liftDeriv2` endpoint trap

Do **not** try to prove:

```lean
ContinuousOn (Function.uncurry (fun t x => liftDeriv2 u t x))
  (Set.Icc (0 : ℝ) δ ×ˢ Set.Icc (0 : ℝ) 1)
```

from zero-start primitive data. The primitive package only gives equality of `liftDeriv2` with the continuous physical representative on `[0,b] × Ioo 0 1`, not at both endpoints. This is deliberate.

The safe path is:

```lean
H1ZeroStartPhysicalPrimitiveDataBefore
  → H1LiftDeriv2ZeroSlabRepresentativeBefore
  → lapL2sq_continuousOn_before_of_zeroSlabRepresentativeBefore
  → IntervalIntegrable (lapL2sq u) volume 0 δ
```

The lap reducer already uses interval-integral a.e. congruence to ignore endpoint mismatch. This is exactly why it exists.

### 2. The interval-integral endpoint convention

The code repeatedly works over `∫ x in 0..1`, with a.e. congruence over the spatial interval. In the existing proofs, the endpoint `x = 1` is removed by `Real.volume_singleton`, and `x = 0` is already outside the `Ioc` side used by the interval-integral bridge. So endpoint equality should never be demanded for `liftDeriv2`.

### 3. The `t = 0` trap

Strict-positive-time classical continuity is not enough for time integrability on `(0,δ)` if the proof needs a bounded/continuous extension to `[0,δ]`. For the non-lap square profiles, use the primitive zero-start closed-slab continuities. For `lapL2sq`, use the zero-start representative family. Do not use strict continuity plus an informal “bounded near zero” argument unless the boundedness is produced from an actual zero-start closed-slab object.

### 4. Real powers and denominator positivity

The non-lap continuity lemmas need the positivity/nonnegativity fields from `H1ZeroStartPhysicalPrimitiveDataBefore`:

* `u_pos0` is needed for `u^γ` and `u^α` when exponents are real.
* `v_nonneg0` gives `0 < 1 + v`, so denominator powers are nonzero and continuous.

Dropping these fields would make the continuity lemmas false or unprovable for general real exponents.
