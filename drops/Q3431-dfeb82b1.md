# Q3431: H¹ endpoint lap-continuity audit

## Verdict

The zero-endpoint lap continuity input

```lean
hLap0 : ∀ {b : ℝ}, 0 ≤ b → b < T →
  ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc (0 : ℝ) b)
```

is **not honestly derivable from the currently committed APIs** at HEAD `11adfb6d2859fefaa9059d54e538a2a394c69d25`.

It is a genuine additional **H² / lap-trace endpoint frontier**. It is not implied by `H1InitialEndpointData`, `H1UxxL1ContBefore`, the classical-solution record, or `H1ScalarRegularityBefore`.

The strict positive-time side is now landed, but the time-zero endpoint remains separate.

### 4. `H1UxxL1ContBefore` is positive-time and L¹, not an endpoint squared-lap trace

`H1UxxL1ContBefore` is consumed in `H1energy_continuousOn_before_of_uxxL1Cont`. The proof uses it only at `x > 0` in time; the zero case is handled by the separate `hcont0 : ContinuousWithinAt (H1energy u) (Set.Ici 0) 0`.

So `H1UxxL1ContBefore` gives local L¹ continuity of `u_xx` at positive times, sufficient for H¹-energy differentiability/continuity. It does not give:

```lean
Tendsto (lapL2sq u) (𝓝[Set.Ioc 0 T] 0) (𝓝 (lapL2sq u 0))
```

and it also does not by itself give continuity of `∫ (u_xx)^2`, even away from zero, without the stronger representative/joint-continuity route that Task 68 just supplied.

## Minimal derivable theorem if explicit endpoint input is supplied

Once an explicit endpoint-lap continuity input is carried, the remaining combination with the strict theorem is pure order/case-splitting. This is the smallest useful Lean theorem.

```lean
import ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity

open MeasureTheory Set Filter
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1EnergyIdentity
open ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity

/-- Explicit endpoint-lap continuity frontier.  This is H²/lap-trace data, not
H¹-energy initial data. -/
structure H1LapComponentEndpointContinuousBefore
    (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  lap_cont0 : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc (0 : ℝ) b)

/-- Endpoint-lap continuity plus strict positive-time lap continuity gives the
full `0 ≤ a` lap-continuity field expected by
`H1IdentityRHSComponentsContinuousBefore`. -/
theorem lapL2sq_continuousOn_before_of_endpoint_and_strict
    {u : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (h0 : H1LapComponentEndpointContinuousBefore u T)
    (hstrict : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
      ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc a b)) :
    ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
      ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc a b) := by
  intro a b ha hab hbT
  rcases eq_or_lt_of_le ha with hzero | hpos
  · subst hzero
    exact h0.lap_cont0 hab hbT
  · exact hstrict hpos hab hbT

end ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity
```

Minor note: the `subst hzero` line may need the usual orientation adjustment depending on the exact output of `eq_or_lt_of_le`; the proof is otherwise just the `a = 0` / `0 < a` split.

## Why not derive it from existing endpoint data?

The tempting but invalid route would be:

```text
H1InitialEndpointData
+ H1UxxL1ContBefore
+ strict-window lap continuity
--------------------------------
ContinuousOn lapL2sq on [0,b]
```

This fails at the first line: `H1InitialEndpointData` gives convergence of `∫ u_x²`, not of `∫ u_xx²`. No theorem in the inspected files upgrades H¹ energy convergence to H²/laplacian trace convergence. `H1UxxL1ContBefore` is only a positive-time L¹ continuity statement and has no time-zero limit. `IsPaper2ClassicalSolution` similarly supplies positive-time regularity, not a t=0 `u_xx` trace.

So the honest next frontier is not another wrapper around `H1InitialEndpointData`; it is explicit endpoint `lapL2sq` continuity / lap trace data.

## Final recommendation

Keep `H1InitialEndpointData` unchanged. Add a separate frontier in `IntervalChiNegH1LapComponentContinuity.lean`:

```lean
H1LapComponentEndpointContinuousBefore u T
```

with field:

```lean
lap_cont0 : ∀ {b : ℝ}, 0 ≤ b → b < T →
  ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc (0 : ℝ) b)
```

Then add a bridge helper combining `lap_cont0` with the landed strict-window theorem to fill the `lap_cont` field of `H1IdentityRHSComponentsContinuousBefore`.

This keeps the source of H²/lap endpoint regularity honest and prevents accidental claims that H¹ initial energy convergence implies squared-laplacian endpoint continuity.
