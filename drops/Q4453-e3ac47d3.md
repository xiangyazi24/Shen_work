# The interface trap: global vs window-local `Mdot`

The most important Lean-level issue is the time domain.

The current field in `CoupledChemDivTimeC1Fields` is:

```lean
MchemDot : ‚ĄĚ
hMdot : ‚ąÄ s, 0 ‚Č§ s ‚Üí ‚ąÄ n,
  |coupledChemDivAdot p u s n| ‚Č§ MchemDot
```

That is a **global half-line bound**. Positive-time regularity of a mild solution typically gives only:

```lean
‚ąÄ c T', 0 < c ‚Üí T' < S.T ‚Üí ‚ąÉ Mdot,
  ‚ąÄ s ‚ąą Icc c T', ‚ąÄ n, |coupledChemDivAdot p S.u s n| ‚Č§ Mdot.
```

This is exactly the shape of a window-local statement, not the global field. If the raw source is not clamped/extended and the initial datum is only continuous, `‚ąā‚āúS` may have a singularity as `s ‚Üď 0`, so a global `Mdot` is not automatic and may be false.

So the cleanest implementation choice is:

1. Prefer a window-local source-time-C¬Ļ path using `DuhamelSourceTimeC1On` where possible.
2. If the final consumer is still the old global `DuhamelSourceTimeC1`, then either:
   - prove that the actual source family used there is clamped/shifted away from `0`, so the global bound is only for a smooth localized family; or
   - strengthen the source family definition/constructor so the bound is requested only on the integration window.

# Bottom line

* **Q1:** Yes, under `hchain`, `coupledChemDivAdot` is exactly the time derivative of the `n`-th cosine coefficient of the chem-div source. Definitionally it is the coefficient of the explicit full `‚ąā‚āú(‚ąā‚āďQ)` chain-rule field, not just the resolver time derivative.

* **Q2:** Yes, if `‚ąā‚āúS` has weak-H¬≤/O(k‚ĀĽ¬≤) uniformly on the relevant time window, then `MchemDot` follows. But weak-H¬≤ is overkill; a uniform `L‚ąě`/`L¬Ļ` bound on `coupledChemDivTimeDerivativeLift` already proves the uniform-in-mode bound.

* **Q3:** No circularity if `‚ąā‚āúS` is built from `L = MildSolutionSliceHasDerivAtTime` and positive-time spatial regularity, with `L` obtained coefficientwise from the already-built spatial/source envelope ladder. The single most likely remaining hole is **not** the identity of `Adot`; that part is correctly wired. The likely hole is the **global-vs-window-local `Mdot` mismatch**. The safe proof gives `Mdot` on positive compact windows. If the final route still demands the global `‚ąÄ s ‚Č• 0` field, the interface should be shifted/clamped or replaced by `DuhamelSourceTimeC1On` rather than pretending a positive-time bound controls the initial singularity.
