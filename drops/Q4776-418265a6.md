# Executive verdict

Use a **sign split and Mathlib's scalar fencing lemmas**. The shortest robust proof on the global domain `ℝ` is:

```text
positive side:
  an ε-raised constant barrier R + ε
  + image_le_of_deriv_right_lt_deriv_boundary
  + w(x) → 0 at atBot

negative side:
  assume w x₀ < -R
  + a strict constant fence makes {w ≤ w x₀} forward invariant
  + an affine lower comparison makes y := -w reach a large threshold C
  + another constant fence keeps y ≥ C
  + v := 1 / y has v' ≤ -1/2
  + an affine upper comparison makes v negative in finite time
  + contradiction, since v remains positive.
```

This proves the exact closed estimate

```lean
|w x| ≤ (A + Real.sqrt (A ^ 2 + 4 * B)) / 2
```

for every `x`, including the degenerate case `A = B = 0`. No Dini derivative, first-crossing supremum, ODE existence/uniqueness package, compactness argument, axiom, or `sorry` is needed.

The negative argument uses the global domain exactly once: after constructing a finite future time `t₂`, `w t₂` must still exist.

# Recommended theorem interface

The clean interface is the absolute-value bounded version:

```lean
noncomputable def riccatiRadius (A B : ℝ) : ℝ :=
  (A + Real.sqrt (A ^ 2 + 4 * B)) / 2

/-- Global scalar Riccati bound. -/
theorem abs_le_riccatiRadius
    {w a b : ℝ → ℝ} {A B : ℝ}
    (hA : 0 ≤ A) (hB : 0 ≤ B)
    (ha : ∀ x, |a x| ≤ A)
    (hb : ∀ x, |b x| ≤ B)
    (hw : Differentiable ℝ w)
    (hw0 : Tendsto w atBot (𝓝 0))
    (hODE : ∀ x,
      deriv w x + w x ^ 2 + a x * w x + b x = 0) :
    ∀ x, |w x| ≤ riccatiRadius A B
```

If the actual hypotheses are

```lean
0 ≤ a x ∧ a x ≤ A
0 ≤ b x ∧ b x ≤ B,
```

convert them with `abs_of_nonneg`; the proof below then applies unchanged. In fact, the argument only consumes `-A ≤ a x ≤ A` and `-B ≤ b x`, but retaining the stated absolute bounds is the least surprising API.

No continuity or differentiability of `a` or `b` is required.

# Three small calculus wrappers

These wrappers isolate all API-sensitive calculus. They are direct applications of Mathlib's fencing theorems, not custom Dini arguments.

```lean
open Filter Set
open scoped Topology

private lemma fence_const
    {f : ℝ → ℝ} (hf : Differentiable ℝ f)
    {s t c : ℝ} (hst : s ≤ t) (hsc : f s ≤ c)
    (hlevel : ∀ x ∈ Ico s t, f x = c → deriv f x < 0) :
    f t ≤ c := by
  exact image_le_of_deriv_right_lt_deriv_boundary
    (f := f) (f' := deriv f) (a := s) (b := t)
    (B := fun _ => c) (B' := fun _ => 0)
    hf.continuous.continuousOn
    (fun x _ => (hf x).hasDerivAt.hasDerivWithinAt)
    hsc
    (fun x => hasDerivAt_const x c)
    hlevel
    (show t ∈ Icc s t from ⟨hst, le_rfl⟩)

private lemma affine_le_of_deriv_ge
    {f f' : ℝ → ℝ} (hf : ∀ x, HasDerivAt f (f' x) x)
    {s t q : ℝ} (hst : s ≤ t)
    (hq : ∀ x ∈ Ico s t, q ≤ f' x) :
    f s + q * (t - s) ≤ f t := by
  let line : ℝ → ℝ := fun x => f s + q * (x - s)
  have hline : ∀ x, HasDerivAt line q x := by
    intro x
    simpa [line] using
      (((hasDerivAt_id x).sub_const s).const_mul q).const_add (f s)
  have H := image_le_of_deriv_right_le_deriv_boundary
    (f := line) (f' := fun _ => q) (a := s) (b := t)
    (B := f) (B' := f')
    (fun x _ => (hline x).continuousAt.continuousWithinAt)
    (fun x _ => (hline x).hasDerivWithinAt)
    (by simp [line])
    (fun x _ => (hf x).continuousAt.continuousWithinAt)
    (fun x _ => (hf x).hasDerivWithinAt)
    hq
  simpa [line] using H (show t ∈ Icc s t from ⟨hst, le_rfl⟩)

private lemma le_affine_of_deriv_le
    {f f' : ℝ → ℝ} {s t q : ℝ} (hst : s ≤ t)
    (hf : ∀ x ∈ Icc s t, HasDerivAt f (f' x) x)
    (hq : ∀ x ∈ Ico s t, f' x ≤ q) :
    f t ≤ f s + q * (t - s) := by
  let line : ℝ → ℝ := fun x => f s + q * (x - s)
  have hline : ∀ x, HasDerivAt line q x := by
    intro x
    simpa [line] using
      (((hasDerivAt_id x).sub_const s).const_mul q).const_add (f s)
  have H := image_le_of_deriv_right_le_deriv_boundary
    (f := f) (f' := f') (a := s) (b := t)
    (B := line) (B' := fun _ => q)
    (fun x hx => (hf x hx).continuousAt.continuousWithinAt)
    (fun x hx =>
      (hf x (Ico_subset_Icc_self hx)).hasDerivWithinAt)
    (by simp [line])
    (fun x _ => (hline x).continuousAt.continuousWithinAt)
    (fun x _ => (hline x).hasDerivWithinAt)
    hq
  simpa [line] using H (show t ∈ Icc s t from ⟨hst, le_rfl⟩)
```

The first wrapper is the positive and negative constant barrier. The second gives linear growth from a derivative lower bound. The third is used on the reciprocal during finite-time blowdown.

# Radius algebra

Put

```text
R = (A + sqrt(A² + 4B)) / 2,
q(z) = z² - A z - B.
```

The four useful facts are

```text
0 ≤ R,
A ≤ R,
R² = A R + B,
R < z  →  0 < q(z).
```

A compact Mathlib implementation is:

```lean
noncomputable def riccatiRadius (A B : ℝ) : ℝ :=
  (A + Real.sqrt (A ^ 2 + 4 * B)) / 2

lemma riccatiRadius_root {A B : ℝ} (hB : 0 ≤ B) :
    riccatiRadius A B ^ 2 = A * riccatiRadius A B + B := by
  have hD : 0 ≤ A ^ 2 + 4 * B := by
    nlinarith [sq_nonneg A]
  have hs := Real.sq_sqrt hD
  dsimp [riccatiRadius]
  nlinarith

lemma riccatiRadius_ge_A {A B : ℝ} (hB : 0 ≤ B) :
    A ≤ riccatiRadius A B := by
  have hs : A ≤ Real.sqrt (A ^ 2 + 4 * B) :=
    Real.le_sqrt_of_sq_le (by nlinarith)
  dsimp [riccatiRadius]
  linarith

lemma riccatiRadius_nonneg {A B : ℝ} (hA : 0 ≤ A) (hB : 0 ≤ B) :
    0 ≤ riccatiRadius A B :=
  hA.trans (riccatiRadius_ge_A hB)

lemma quadratic_pos_of_radius_lt
    {A B z : ℝ} (hA : 0 ≤ A) (hB : 0 ≤ B)
    (hz : riccatiRadius A B < z) :
    0 < z ^ 2 - A * z - B := by
  have hroot := riccatiRadius_root (A := A) hB
  have hRA := riccatiRadius_ge_A (A := A) hB
  have hfactor :
      z ^ 2 - A * z - B =
        (z - riccatiRadius A B) *
          (z + riccatiRadius A B - A) := by
    nlinarith
  rw [hfactor]
  exact mul_pos (sub_pos.mpr hz) (by nlinarith)
```

The factorization

```text
q(z) = (z - R)(z + R - A)
```

is preferable to repeatedly expanding the square root.

# Boundary derivative lemmas

Prove two tiny arithmetic lemmas once.

For a positive level `m > R`, if `w x = m`, then

```text
w' x = -m² - a(x)m - b(x)
     ≤ -m² + A m + B
     = -q(m) < 0.
```

Use the lower bounds extracted by

```lean
(abs_le.mp (ha x)).1   -- -A ≤ a x
(abs_le.mp (hb x)).1   -- -B ≤ b x
```

and, because `m ≥ 0`, feed

```lean
mul_nonneg (by linarith [(abs_le.mp (ha x)).1]) hm_nonneg
```

for `(a x + A) * m` to `nlinarith`.

For a negative level `-m`, `m > R`, if `w x = -m`, then

```text
w' x = -m² + a(x)m - b(x)
     ≤ -m² + A m + B
     = -q(m) < 0.
```

This time use

```lean
(abs_le.mp (ha x)).2   -- a x ≤ A
(abs_le.mp (hb x)).1   -- -B ≤ b x
mul_nonneg (by linarith [(abs_le.mp (ha x)).2]) hm_nonneg
```

for `(A - a x) * m`. After rewriting `hODE x` by the level equality, `ring_nf` followed by `nlinarith` closes both lemmas.

These are the only places where the coefficient bounds enter the constant fences.

# Positive barrier: use `R + ε`, not `R`

Do not try to obtain `w s ≤ R` directly from `w → 0` at `atBot`. That fails as a proof pattern when `R = 0`. Instead prove

```text
w x ≤ R + ε
```

for every `ε > 0`, then use `_root_.le_of_forall_pos_le_add`.

For fixed `x` and `ε > 0`, let `M = R + ε`. Since `0 < M`, the limit gives an eventual one-sided bound:

```lean
have hEv : ∀ᶠ z in atBot, w z < M := by
  simpa only [mem_Iio] using
    hw0.eventually (Iio_mem_nhds (by linarith [hR_nonneg, hε]))

obtain ⟨c, hc⟩ := hEv.exists_forall_of_atBot
let s := min c x
have hsx : s ≤ x := min_le_right c x
have hws : w s ≤ M :=
  (hc s (min_le_left c x)).le
```

At every boundary point with `w z = M`, the positive-level derivative lemma gives `deriv w z < 0`, because `R < M`. Therefore

```lean
fence_const hw hsx hws
```

with constant `M` yields `w x ≤ M`. Finally:

```lean
apply _root_.le_of_forall_pos_le_add
intro ε hε
-- perform the M = R + ε fence
```

This proves the exact upper bound `w x ≤ R`, with no separate case split for `R = 0`.

# Negative side: finite-time blowdown

The lower estimate is logically independent of `hw0`: global existence and the differential equation alone imply `-R ≤ w x`.

Assume for contradiction that

```text
w x₀ < -R.
```

Define

```text
y(t)  = -w(t),
y₀    = y(x₀),
dy(t) = -deriv w t,
q₀    = y₀² - A y₀ - B.
```

Then `R < y₀`, hence `0 < q₀` by `quadratic_pos_of_radius_lt`. The derivative is represented without rewriting `deriv y`:

```lean
have hy : ∀ t, HasDerivAt y (dy t) t := by
  intro t
  simpa [y, dy] using (hw t).hasDerivAt.neg

have hy_formula : ∀ t,
    dy t = y t ^ 2 - a t * y t + b t := by
  intro t
  dsimp [y, dy]
  nlinarith [hODE t]
```

## 1. The bad negative region is forward invariant

Apply `fence_const` to `w` with the constant `w x₀ = -y₀`. At a boundary point `w z = -y₀`, the negative-level derivative lemma gives `deriv w z < 0`. Thus for every `t ≥ x₀`,

```text
w t ≤ w x₀,
y t ≥ y₀.
```

No first-exit time is needed.

## 2. `y` has a fixed positive derivative lower bound

From the coefficient bounds and `y t ≥ y₀ > R ≥ A ≥ 0`,

```text
dy(t) ≥ y(t)² - A y(t) - B.
```

Also

```text
q(y(t)) - q(y₀)
  = (y(t) - y₀)(y(t) + y₀ - A)
  ≥ 0.
```

Hence

```text
dy(t) ≥ q₀ > 0
```

for all `t ≥ x₀`. This is a good `nlinarith` block after supplying the two nonnegative products

```lean
(A - a t) * y t
(y t - y₀) * (y t + y₀ - A).
```

Use `affine_le_of_deriv_ge` to obtain the global-in-time linear estimate

```text
y₀ + q₀ (t - x₀) ≤ y(t)      for t ≥ x₀.
```

## 3. Reach a simple large threshold

Use the deliberately oversized threshold

```text
C = 4 * (A + B + R + 1).
```

This choice is much easier in Lean than a nest of `max` and square roots. From nonnegativity,

```text
R < C,
4A ≤ C,
4B ≤ C,
4 ≤ C.
```

Set

```text
t₁ = x₀ + (C + 1) / q₀.
```

Since `q₀ > 0`, `x₀ < t₁`. The affine lower estimate and

```lean
q₀ * (t₁ - x₀) = C + 1
```

show `C < y t₁`, hence `w t₁ ≤ -C`.

Apply `fence_const` again, now with constant `-C`. Since `R < C`, the negative-level derivative lemma shows that for every `t ≥ t₁`,

```text
w t ≤ -C,
y t ≥ C.
```

## 4. Above `C`, the quadratic term dominates

For `t ≥ t₁`, the inequalities `y t ≥ C` imply

```text
4A ≤ y t,
4B ≤ y t,
1 ≤ y t.
```

Consequently

```text
4 A y(t) ≤ y(t)²,
4 B ≤ y(t)²,
```

and therefore

```text
dy(t)
  ≥ y(t)² - A y(t) - B
  ≥ y(t)² / 2.
```

A stable Lean proof is to provide `nlinarith` with

```lean
mul_le_mul_of_nonneg_right h4A (show 0 ≤ y t by positivity)
mul_nonneg (show 0 ≤ y t by positivity) (show 0 ≤ y t - 1 by linarith)
```

plus `hy_formula t` and the coefficient-bound product `(A - a t) * y t ≥ 0`.

## 5. Reciprocal decreases linearly to a contradiction

Define

```text
v(t)  = (y(t))⁻¹,
dv(t) = -dy(t) / y(t)².
```

On `[t₁, ∞)`, `y(t) ≥ C > 0`, so `HasDerivAt.inv` gives exactly

```lean
have hv : HasDerivAt v (dv t) t := by
  simpa [v, dv] using (hy t).inv (ne_of_gt hy_pos)
```

and the quadratic lower bound gives

```text
dv(t) ≤ -1/2.
```

The denominator clearing is native:

```lean
have hy_sq_pos : 0 < y t ^ 2 := pow_pos hy_pos 2
rw [dv, div_le_iff₀ hy_sq_pos]
nlinarith [hdy_quadratic]
```

Now set

```text
t₂ = t₁ + 2 * v(t₁) + 1.
```

Since `v(t₁) > 0`, `t₁ < t₂`. Apply `le_affine_of_deriv_le` on `[t₁,t₂]` with `q = -1/2`:

```text
v(t₂) ≤ v(t₁) - (t₂ - t₁)/2 = -1/2.
```

But the second constant fence still gives `y(t₂) ≥ C > 0`, so

```text
v(t₂) = 1 / y(t₂) > 0,
```

which is the desired contradiction.

This is the finite-time blowdown argument in a form that uses only `MeanValue` fencing and `HasDerivAt.inv`.

# Recommended statement decomposition

Implement in this order:

```text
1. riccatiRadius_root
2. riccatiRadius_ge_A
3. riccatiRadius_nonneg
4. quadratic_pos_of_radius_lt

5. fence_const
6. affine_le_of_deriv_ge
7. le_affine_of_deriv_le

8. deriv_lt_zero_of_eq_pos_level
9. deriv_lt_zero_of_eq_neg_level

10. riccati_upper_bound
      assumptions: hw, hw0, hODE, coefficient bounds
      conclusion:  ∀ x, w x ≤ R

11. riccati_lower_bound_global
      assumptions: hw, hODE, coefficient bounds
      conclusion:  ∀ x, -R ≤ w x

12. abs_le_riccatiRadius
      exact (abs_le).2
        ⟨riccati_lower_bound_global ..., riccati_upper_bound ...⟩
```

The split in items 10 and 11 is worth preserving: it documents that the limit at `atBot` is used only for the upper barrier, while the lower bound is a pure global-existence obstruction.

# Avoid these proof designs

Do not use a direct barrier at `R`; it mishandles `R = 0` at the initialization step. Use `R + ε` and `le_of_forall_pos_le_add`.

Do not define a first crossing as an `sInf` or `IsLeast`. `image_le_of_deriv_right_lt_deriv_boundary` already packages the needed first-contact argument.

Do not introduce upper Dini derivatives. The public fencing theorem accepts `HasDerivWithinAt`, and global differentiability supplies it through

```lean
(hw x).hasDerivAt.hasDerivWithinAt.
```

Do not apply norm Grönwall directly to `w`; the estimate is fundamentally sign-sensitive. Separating the two signs and applying the scalar fences is both shorter and sharper.

Do not add continuity assumptions on `a` or `b`; they are unnecessary.

# Final proof shape

The final assembly is exactly

```text
atBot limit + ε-positive fence  →  w ≤ R,
global-domain blowdown          →  -R ≤ w,
abs_le                          →  |w| ≤ R.
```

This keeps the exact requested radius, handles all nonnegative `A,B`, and is compatible with the Mathlib 4.29.1 theorem surface.
