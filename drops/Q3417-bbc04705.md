# Q3417: H1 route audit — interior equality plus endpoint-null transfer

## Executive answer

Yes. The H1 transfer can be revised so that the downstream consumer is `H1UxxL1ContBefore` directly, without first constructing `H1LiftDeriv2JointContinuousBefore` for the literal `liftDeriv2` on the closed spatial slab.

The key change is architectural:

* keep the continuous physical representative, e.g. your new `liftDeriv2PhysicalRHS p u v`, continuous on `Set.Icc a b ×ˢ Set.Icc 0 1`;
* prove equality with `liftDeriv2 u` only on `Set.Icc a b ×ˢ Set.Ioo 0 1`;
* use the existing representative L1-continuity theorem on the representative itself;
* transfer the resulting interval integral back to `liftDeriv2` by an a.e. congruence lemma, since the interval integral over `0..1` ignores endpoint changes.

This avoids the endpoint-sensitive ordinary-derivative problem for zero extensions at `x = 0, 1`. It should **not** try to manufacture `H1LiftDeriv2JointContinuousBefore u T`, because that structure literally requires closed-slab `ContinuousOn` of `Function.uncurry (fun t x => liftDeriv2 u t x)`.

## Are endpoints genuinely ignorable?

Mathematically, yes, in the revised route.

For the ordered interval integral `∫ x in (0 : ℝ)..1, ...`, Mathlib's interval integral is over the interval-integral measure associated to `Ioc 0 1` when `0 ≤ 1`. The point `0` is not included in `Ioc 0 1`; the point `1` is included, but `{1}` has volume zero. If one instead uses the closed interval measure `volume.restrict (Set.Icc 0 1)`, the complement of `Ioo 0 1` inside `Icc 0 1` is `{0,1}`, also null.

So equality on `Ioo 0 1` is enough for equality of the spatial interval integrals, provided the equality is converted to an a.e. equality under the interval-integral measure.

However, the **current** proof path does not exploit this. The current path first uses closed-slab pointwise equality to prove literal closed-slab continuity of `liftDeriv2` by `ContinuousOn.congr`. That is where endpoint equality is genuinely required today. The theorem `l1_time_continuity_at_of_jointContinuousOn_slab` itself is not the problem; it can be run on the representative `F`. The problem is only the existing wrapper that insists on making the literal `liftDeriv2` continuous before integration.

## Hidden circularity warnings

1. **Do not prove the new `H1UxxL1ContBefore` theorem using `H1ScalarRegularityBefore`.**  `H1ScalarRegularityBefore_of_uxxL1Cont_and_hderivInt` and `H1ScalarRegularityBefore_of_uxxL1Cont_and_identityRHSIntegrable` already consume `H1UxxL1ContBefore`. Using scalar regularity to get `u_xx` L1 continuity would be circular.

2. **Do not prove the new `H1UxxL1ContBefore` theorem using `H1EnergyIdentity_of_classicalSolution_and_H1UxxL1ContBefore`.**  That theorem consumes `H1UxxL1ContBefore`; it cannot help produce it.

3. **Do not route through `H1LiftDeriv2JointContinuousBefore`.**  That structure requires literal closed-slab continuity of `liftDeriv2`, which is exactly what the endpoint problem obstructs. The new theorem should produce `H1UxxL1ContBefore` directly.

4. **It is safe to use `IsPaper2ClassicalSolution` spatial `C²` regularity only for slice integrability.**  The wrapper above uses `intervalIntegrable_deriv_deriv_of_contDiffOn_two` from the `ContDiffOn ℝ 2` closed-slice regularity field. That is upstream of the H1 scalar route and is already used in `IntervalChiNegH1EnergyIdentity.lean`, so it does not create an H1 circular dependency.

5. **If RHS component continuity still depends on `lapL2sq` continuity, add the optional representative lemma above.**  Otherwise replacing the transfer with direct `H1UxxL1ContBefore` may close the L1 frontier but leave the `H1IdentityRHSComponentsContinuousBefore.lap_cont` field unsupported.

## Recommended edit summary

Replace the closed-endpoint transfer in the physical-RHS route with:

```lean
H1UxxL1ContBefore_of_physicalRHS_components_Ioo
```

whose inputs are:

1. `ContinuousOn (Function.uncurry (liftDeriv2PhysicalRHS p u v)) (Set.Icc a b ×ˢ Set.Icc 0 1)` for every strict time slab;
2. `EqOn liftDeriv2 (liftDeriv2PhysicalRHS p u v)` only on `Set.Icc a b ×ˢ Set.Ioo 0 1`;
3. literal `liftDeriv2` slice `IntervalIntegrable`, discharged from `IsPaper2ClassicalSolution` closed-slice `C²` regularity.

This is the minimal non-circular Lean route. The endpoint values of `liftDeriv2` and of the literal chemotaxis divergence no longer need to agree pointwise, because the only post-equality consumer is the spatial interval integral, where endpoint changes are null.
