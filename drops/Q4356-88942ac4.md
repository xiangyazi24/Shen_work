# 1. Sharp multiplier bound

Set

```text
r(t) := (1+t)/(μ+νt),    t ≥ 0.
```

Its derivative is

```text
r'(t)
  = [(μ+νt) - ν(1+t)]/(μ+νt)²
  = (μ-ν)/(μ+νt)².
```

Since the denominator is strictly positive, the sign is constant:

- if `μ < ν`, then `r` is strictly decreasing;
- if `μ = ν`, then `r` is constant;
- if `μ > ν`, then `r` is strictly increasing.

The two endpoint values are

```text
r(0) = 1/μ,
lim_{t→∞} r(t) = 1/ν.
```

Thus

```text
sup_{t≥0} r(t) = max (1/μ) (1/ν).
```

Because `λ₀=0` and `λₖ→∞`, restricting `t` to the discrete set `{λₖ | k∈ℕ}` gives the same supremum.

The attainment statement is exactly:

```text
μ < ν :  sup = 1/μ, attained at k=0;
μ = ν :  r(λₖ)=1/μ=1/ν for every k;
μ > ν :  sup = 1/ν, not attained by any finite k, but r(λₖ)→1/ν.
```

For the last case, the strict finite-mode inequality is visible from

```text
r(t) - 1/ν = (ν-μ) / [ν(μ+νt)] < 0
```

when `μ>ν`, while the right-hand side tends to zero.

## Lean-friendly proof of the uniform bound

For the Sobolev bootstrap, formalizing the actual `sSup` is unnecessary. The load-bearing theorem is the pointwise upper bound. A direct two-case algebra proof is cleaner than differentiating:

```lean
namespace ShenWork.Paper2.HSigmaScale

/-- Two-parameter version for `(μ - ν∂ₓₓ)⁻¹`. -/
theorem elliptic_multiplier_two_param_le
    {μ ν : ℝ} (hμ : 0 < μ) (hν : 0 < ν) (k : ℕ) :
    (1 + lam k) / (μ + ν * lam k) ≤ max (1 / μ) (1 / ν) := by
  have hλ : 0 ≤ lam k := lam_nonneg k
  have hden : 0 < μ + ν * lam k :=
    add_pos_of_pos_of_nonneg hμ (mul_nonneg hν.le hλ)
  rcases le_total μ ν with hμν | hνμ
  · -- μ ≤ ν: compare with 1/μ.
    have hgap : 0 ≤ (ν - μ) * lam k :=
      mul_nonneg (sub_nonneg.mpr hμν) hλ
    have hq : (1 + lam k) / (μ + ν * lam k) ≤ 1 / μ := by
      rw [div_le_div_iff₀ hden hμ]
      nlinarith
    exact hq.trans (le_max_left _ _)
  · -- ν ≤ μ: compare with 1/ν.
    have hq : (1 + lam k) / (μ + ν * lam k) ≤ 1 / ν := by
      rw [div_le_div_iff₀ hden hν]
      nlinarith
    exact hq.trans (le_max_right _ _)

/-- Squared multiplier used in the `H^σ` energy. -/
theorem elliptic_multiplier_two_param_sq_le
    {μ ν : ℝ} (hμ : 0 < μ) (hν : 0 < ν) (k : ℕ) :
    (1 + lam k) ^ 2 / (μ + ν * lam k) ^ 2 ≤
      (max (1 / μ) (1 / ν)) ^ 2 := by
  have hλ : 0 ≤ lam k := lam_nonneg k
  have hden : 0 < μ + ν * lam k :=
    add_pos_of_pos_of_nonneg hμ (mul_nonneg hν.le hλ)
  have hratio_nonneg :
      0 ≤ (1 + lam k) / (μ + ν * lam k) :=
    div_nonneg (by linarith) hden.le
  have hsq := pow_le_pow_left₀ hratio_nonneg
    (elliptic_multiplier_two_param_le hμ hν k) 2
  simpa [div_pow] using hsq

end ShenWork.Paper2.HSigmaScale
```

This is the exact generalization of the proof style already used by the repository's one-parameter `elliptic_multiplier_le`.

## Lean-friendly monotonicity

The calculus proof above is immediate, but cross multiplication is usually easier in Lean. For `0≤s≤t`, both denominators are positive, and

```text
(1+s)(μ+νt) ≤ (1+t)(μ+νs)
```

is equivalent to

```text
0 ≤ (μ-ν)(t-s).
```

Thus when `ν≤μ`, use

```lean
rw [div_le_div_iff₀ hsden htden]
have hp : 0 ≤ (μ - ν) * (t - s) :=
  mul_nonneg (sub_nonneg.mpr hνμ) (sub_nonneg.mpr hst)
nlinarith
```

to prove `MonotoneOn r (Set.Ici 0)`. Reversing the quotient inequality and using

```text
0 ≤ (ν-μ)(t-s)
```

proves `AntitoneOn r (Set.Ici 0)` when `μ≤ν`.

# 2. The exact `+2` Sobolev gain

Let

```text
R̂ₖ = f̂ₖ/(μ+νλₖ).
```

For any real Sobolev exponent `σ`, since `1+λₖ>0`,

```text
(1+λₖ)^(σ+2) |R̂ₖ|²
 = (1+λₖ)^σ |f̂ₖ|²
     * [(1+λₖ)/(μ+νλₖ)]²
 ≤ C(μ,ν)² (1+λₖ)^σ |f̂ₖ|².
```

Summing gives

```text
‖R‖²_{H^(σ+2)} ≤ C(μ,ν)² ‖f‖²_{H^σ}.
```

This is exactly the coefficient-level `+2` gain needed by `SourceFromSolutionEnvelopePass`; it is a single bounded multiplier estimate, not a circular regularity argument.

If the equation is written with the full source

```text
f = ν u^γ,
```

then the formula above is already in terms of the coefficients of `f`. If instead the right-hand side is measured using the coefficients of `u^γ`, then

```text
R̂ₖ = ν (u^γ)^hatₖ/(μ+νλₖ),
```

so the corresponding sharp multiplier constant is

```text
ν C(μ,ν) = max (ν/μ) 1.
```

Keep these two conventions separate when assigning constants.

# 3. Exact Neumann diagonalization and absence of cross terms

For

```text
eₖ(x) = cos(kπx),
λₖ = (kπ)²,
```

one has

```text
eₖ'(0)=eₖ'(1)=0,
-∂ₓₓ eₖ = λₖ eₖ.
```

Therefore

```text
(μ-ν∂ₓₓ)eₖ = (μ+νλₖ)eₖ.
```

The Neumann cosine functions are mutually orthogonal in `L²(0,1)`. Hence applying the operator to a cosine expansion changes each coefficient independently; no mode mixing and no cross terms occur.

Equivalently, for a weak or classical solution, test the equation against `eₖ`. Two spatial integrations by parts give

```text
∫₀¹ (-νRₓₓ)eₖ dx = νλₖ ∫₀¹ R eₖ dx.
```

The first boundary term vanishes because `Rₓ(0)=Rₓ(1)=0`; the other boundary term contains `sin(kπx)` and vanishes at both endpoints. Thus

```text
(μ+νλₖ) R̂ₖ = f̂ₖ.
```

Since `μ+νλₖ>0`, division is legitimate and yields

```text
R̂ₖ = f̂ₖ/(μ+νλₖ)
```

exactly.

The conclusion is independent of whether one uses raw coefficients `∫f cos(kπx)` or a normalized cosine coefficient convention: the same normalization occurs on both sides, so it cancels. In particular, the factor `2` often used in reconstructing the positive cosine modes does not alter the diagonal denominator.

The assumptions `μ>0` and `ν>0` are essential for this clean statement. If `μ=0`, the Neumann zero mode is not invertible without a mean-zero compatibility condition and a normalization of `R`. If `ν=0`, there is no two-order elliptic smoothing.
