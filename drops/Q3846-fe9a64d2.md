## 1. Does W1–W10 survive?

### What survives

Only these pieces survive as reusable side lemmas:

* **W1** one-mode gradient IBP is still useful and true.
* **W2/W3** gradient pairing via cosine series is still useful if later pursuing a smooth-test weak formulation.
* Some scalar Duhamel coefficient lemmas from **W6/W7** may be useful if proving endpoint identities coefficientwise.

### What does not survive as a dispatch target

The target

```lean
TruncatedMildSemigroupWeakAfterBNDualityOn p T u₀ u Test
```

should no longer be a priority target unless `Test` is explicitly a sufficiently regular class and the boundary terms are built into the interface.  It is unnecessary for the negative-part argument, and the all-test bridge back to

```lean
TruncatedWeakLocalPDE p T u
```

must not be used.

The correct route already exists in the tree structurally:

```lean
NegativePartMildSemigroupWeakAfterFluxTestDuality
NegativePartMildToWeakRegularData
NegativePartEnergyEstimateRegularData
NegativePartEnergyCoreRegularData
```

The key identity is only at the test

```lean
negativePartTest u t = fun x => - negativePartLift (u t) x
```

not at arbitrary `φ`.

### Why the mild route should be demoted

The old W-route differentiates a mild expression involving truncated nonlinearities.  That is hazardous because the map `r ↦ positivePart r` is not classically differentiable at zero, so trying to differentiate `u_+` through zero crossings is the wrong interface.  The coefficient/energy route only needs the negative-part test and weak/Sobolev chain rules for `u_-`, exactly the object used by `NegativePartEnergyCoreRegularData`.
