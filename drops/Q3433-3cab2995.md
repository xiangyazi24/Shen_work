# Q3433: Task70 local H¹ seed after Q3432

## Verdict

`le_gronwallBound_of_liminf_deriv_right_le` is **not usable directly on `[0, τ]`** from the current `H1ScalarDIOnBefore` API, because Mathlib's Gronwall theorem wants the right-slope/derivative hypothesis on all `x ∈ Set.Ico a b`, including the left endpoint `x = a`. With `a = 0`, this would require the slope hypothesis and DI at `x = 0`, while the current record only gives:

```lean
hhasDerivRight : ∀ {a b r : ℝ}, 0 ≤ a → a ≤ b → b < T →
  r ∈ Set.Ioo a b →
    HasDerivWithinAt (H1energy u)
      (deriv (H1energy u) r) (Set.Ioi r) r

hDI : ∀ r, 0 < r → r < T →
  deriv (H1energy u) r ≤ A * H1energy u r + B
```

However, an **open-left limiting Gronwall is not necessary**. The Lean-feasible route is simpler:

1. Choose a fixed positive `δ < min T 1`, for example
   ```lean
   δ := min (T / 2) (1 / 2)
   ```
2. Use compactness and `hDI.hcont` to bound `H1energy u` on `[0, δ]`.
3. For `τ > δ`, apply Mathlib Gronwall on `[δ, τ]`. This is now possible because `δ > 0`; for the left endpoint `x = δ`, obtain `hhasDerivRight` by widening the source window to `(δ/2, τ)`, so `δ ∈ Ioo (δ/2) τ`.
4. Monotonize the Gronwall bound to time-length `≤ 1` using `gronwallBound_mono`.

So the smallest honest theorem is a scalar theorem in `IntervalChiNegH1AverageWiring.lean`, **not** importing `InitialContinuity`, and not using any PDE data.

## Why direct Gronwall from zero fails

Mathlib's theorem is used in the repo's `ShenWork/Analysis/UniformGronwall.lean` in the form:

```lean
le_gronwallBound_of_liminf_deriv_right_le
  (f := y) (f' := dy) (δ := y s) (K := α) (ε := β)
  (a := s) (b := t) hcont_st hlim_st le_rfl hbound_st
```

The hypothesis `hlim_st` has shape:

```lean
∀ x ∈ Set.Ico s t, ∀ R, dy x < R →
  ∃ᶠ z in 𝓝[>] x, (z - x)⁻¹ * (y z - y x) < R
```

If `s = 0`, this includes `x = 0`. The current `H1ScalarDIOnBefore` cannot supply either `hhasDerivRight` or `hDI` at `0`, since both are strictly positive-time inputs. Therefore a direct `[0, τ]` application is not feasible without strengthening the record.

## Better route: compact seed on `[0,δ]` + Gronwall from `δ`

This route avoids a limiting argument. It also avoids importing `IntervalChiNegH1InitialContinuity` into `AverageWiring`, preventing an import cycle.

## Helper 2: Gronwall from a positive start time

This is the core reason no open-left limiting lemma is needed. The left endpoint is `δ > 0`; to get the derivative hypothesis at `x = δ`, call `hDI.hhasDerivRight` on the wider window `(δ/2, τ)`.

```lean
import ShenWork.Paper2.IntervalChiNegH1AverageWiring
import Mathlib.Analysis.ODE.Gronwall

open MeasureTheory Set Filter
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1AverageWiring

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1AverageWiring

/-- Scalar Gronwall bound for `H1energy` starting from a strictly positive time.
This avoids the missing derivative/DI hypothesis at `0`. -/
theorem H1energy_le_gronwallBound_from_pos_start
    {u : ℝ → intervalDomainPoint → ℝ} {T A B δ τ : ℝ}
    (hDI : H1ScalarDIOnBefore u T A B)
    (hδ0 : 0 < δ) (hδτ : δ ≤ τ) (hτT : τ < T) :
    H1energy u τ ≤ gronwallBound (H1energy u δ) A B (τ - δ) := by
  by_cases hτ_eq : τ = δ
  · subst τ
    -- Should reduce to `y δ ≤ gronwallBound (y δ) A B 0`.
    -- If no simp lemma fires, unfold `gronwallBound`; or use a small local lemma.
    -- Expected simp names: `gronwallBound`, `gronwallBound_K0`, `Real.exp_zero`.
    sorry
  · have hδτ_lt : δ < τ := lt_of_le_of_ne hδτ (Ne.symm hτ_eq)
    have hcont : ContinuousOn (H1energy u) (Set.Icc δ τ) :=
      hDI.hcont hδ0.le hδτ hτT
    have hlim :
        ∀ x ∈ Set.Ico δ τ, ∀ R,
          A * H1energy u x + B < R →
            ∃ᶠ z in 𝓝[>] x,
              (z - x)⁻¹ * (H1energy u z - H1energy u x) < R := by
      intro x hx R hR
      have hx_pos : 0 < x := lt_of_lt_of_le hδ0 hx.1
      have hxT : x < T := lt_of_lt_of_le hx.2 hτT
      have hderiv_lt : deriv (H1energy u) x < R :=
        lt_of_le_of_lt (hDI.hDI x hx_pos hxT) hR
      -- Widen the slab so the left endpoint `x = δ` is interior.
      let a : ℝ := δ / 2
      have ha0 : 0 ≤ a := by positivity
      have hab : a ≤ τ := by
        dsimp [a]
        linarith [hδ0, hδτ]
      have hxIoo : x ∈ Set.Ioo a τ := by
        constructor
        · dsimp [a]
          linarith [hδ0, hx.1]
        · exact hx.2
      have hder := hDI.hhasDerivRight (a := a) (b := τ) ha0 hab hτT hxIoo
      exact eventually_right_slope_lt_of_hasDerivWithinAt_lt hder hderiv_lt
    have hbound : ∀ x ∈ Set.Ico δ τ,
        A * H1energy u x + B ≤ A * H1energy u x + B := by
      intro x hx
      exact le_rfl
    have hG :=
      le_gronwallBound_of_liminf_deriv_right_le
        (f := H1energy u)
        (f' := fun x => A * H1energy u x + B)
        (δ := H1energy u δ) (K := A) (ε := B)
        (a := δ) (b := τ)
        hcont hlim le_rfl hbound
    exact hG τ ⟨hδτ, le_rfl⟩

end ShenWork.Paper2.IntervalChiNegH1AverageWiring
```

Likely troublesome point: the final membership expected by `hG` may be `τ ∈ Set.Icc δ τ`, not exactly the displayed pair, depending on the theorem return shape. In `UniformGronwall.lean`, the corresponding final line is:


## Final theorem: local seed from scalar DI

This is the theorem that should close the Task70 frontier in `AverageWiring`.

```lean
import ShenWork.Paper2.IntervalChiNegH1AverageWiring
import Mathlib.Analysis.ODE.Gronwall

open MeasureTheory Set Filter
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1AverageWiring

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1AverageWiring

/-- Restricted local H¹ seed from the scalar H¹ differential inequality.

No PDE input and no bounded-before input are used.  The proof uses compactness on
`[0,δ]` and Mathlib Gronwall on `[δ,τ]`, with `δ = min (T/2) (1/2)`. -/
theorem exists_H1_localSeed_of_scalarDI_before
    {u : ℝ → intervalDomainPoint → ℝ} {T A B : ℝ}
    (hT : 0 < T)
    (hDI : H1ScalarDIOnBefore u T A B) :
    ∃ Ylocal : ℝ,
      ∀ τ, τ ∈ Set.Ioc (0 : ℝ) 1 → τ < T →
        H1energy u τ ≤ Ylocal := by
  let δ : ℝ := min (T / 2) (1 / 2)
  have hδ0 : 0 < δ := by
    dsimp [δ]
    exact lt_min (half_pos hT) (by norm_num)
  have hδT : δ < T := by
    dsimp [δ]
    have hle : min (T / 2) (1 / 2) ≤ T / 2 := min_le_left _ _
    linarith [hT]
  have hδle1 : δ ≤ 1 := by
    dsimp [δ]
    have hle : min (T / 2) (1 / 2) ≤ 1 / 2 := min_le_right _ _
    linarith

  -- Compact bound on `[0,δ]`.
  have hcont0δ : ContinuousOn (H1energy u) (Set.Icc (0 : ℝ) δ) :=
    hDI.hcont (a := 0) (b := δ) (by norm_num) hδ0.le hδT
  obtain ⟨Y0, hY0⟩ :=
    (isCompact_Icc : IsCompact (Set.Icc (0 : ℝ) δ)).exists_bound_of_continuousOn hcont0δ

  let YG : ℝ := gronwallBound (H1energy u δ) A B 1
  refine ⟨max Y0 YG, ?_⟩
  intro τ hτ hτT
  by_cases hτδ : τ ≤ δ
  · have hτIcc : τ ∈ Set.Icc (0 : ℝ) δ := ⟨le_of_lt hτ.1, hτδ⟩
    have hnorm := hY0 τ hτIcc
    exact le_trans (le_abs_self (H1energy u τ))
      (le_trans (by simpa [Real.norm_eq_abs] using hnorm) (le_max_left _ _))
  · have hδτ : δ ≤ τ := le_of_lt (lt_of_not_ge hτδ)
    have hGτ :
        H1energy u τ ≤ gronwallBound (H1energy u δ) A B (τ - δ) :=
      H1energy_le_gronwallBound_from_pos_start hDI hδ0 hδτ hτT
    have hlen : τ - δ ≤ 1 := by
      linarith [hτ.2]
    have hGmono :
        gronwallBound (H1energy u δ) A B (τ - δ) ≤
          gronwallBound (H1energy u δ) A B 1 := by
      exact (gronwallBound_mono (H1energy_nonneg u δ) hDI.hB hDI.hA) hlen
    exact le_trans hGτ (le_trans hGmono (le_max_right _ _))

end ShenWork.Paper2.IntervalChiNegH1AverageWiring
```

Expected minor issues:

* `exists_bound_of_continuousOn` may produce a norm-bound with a different binder order. Follow usages in the repo such as compact slab bounds.
* The result of `gronwallBound_mono` in the local Mathlib may require an explicit proof `0 ≤ τ - δ`; this follows from `hδτ`.
* `lt_of_not_ge hτδ` gives `δ < τ`; use `.le` if a non-strict inequality is needed.
* The theorem `gronwallBound_mono` is used in `ShenWork/Analysis/UniformGronwall.lean` as:
  ```lean
  (gronwallBound_mono hδ hε hK) hx
  ```
  with arguments corresponding to nonnegativity of the initial value, epsilon/B, and K/A.

## Answer to the route question

* Direct `le_gronwallBound_of_liminf_deriv_right_le` on `[0,τ]`: **no**, not with the current API.
* Open-left Gronwall by applying on `[ε,τ]` and passing `ε → 0`: **possible but unnecessarily hard**.
* Simpler compact/FTC/Mathlib route: **yes** — compact bound on `[0,δ]`, then direct Mathlib Gronwall on `[δ,τ]` for `δ > 0`. This is the recommended Lean-feasible route.
