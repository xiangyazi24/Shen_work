# Q3775 route selector: remove false endpoint continuity from the mixed gradient-source bridge

I inspected the current `main` source around `ShenWork/Paper2/IntervalGradientSourceBridgeOpen.lean`. The suspicion is right: the remaining input

```lean
hdiv_cont : ContinuousOn (coupledChemDivSourceLift p u s) (Set.Icc 0 1)
```

in `gradient_source_bridge_slice_open_of_classical_ball` is stronger than what the gradient-source bridge should need. `coupledChemDivSourceLift` is defined as an `intervalDomainLift` of the subtype chemotaxis divergence source, so it is a zero extension off `[0,1]`. Its endpoint values are subtype values, while outside values are zero. There is no reason the exterior zero extension should be continuous at `0` or `1`, and the Ktilde integral/sine coefficient side should not see endpoint values anyway.

The smallest honest fix is not to manufacture endpoint continuity. It is to add an endpoint-insensitive Ktilde/sine theorem and a new slice bridge that assumes:

* `IntervalIntegrable (coupledChemDivSourceLift p u s) volume 0 1`, for Task294 open IBP;
* a continuous closed representative `divRep` on `[0,1]`;
* equality of the actual zero-extended source and `divRep` only on `(0,1)`.

This keeps endpoint data out of the analytic statement.

## 1. Endpoint-insensitive sine coefficients and Ktilde integrals

For sine coefficients, equality on `Ioo 0 1` is enough even pointwise, because the sine factor vanishes at both endpoints. No a.e. theorem is needed for this one.

```lean
import ShenWork.Paper2.IntervalGradientSourceBridgeOpen

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.Paper2.IntervalGradientSourceBridgeOpen

open ShenWork.IntervalNeumannFullKernel
open ShenWork.Paper2.IntervalDivergenceModeIdentity

/-- Sine coefficients do not see endpoint values.  Equality on `(0,1)` is
sufficient because `sin (nπx)` vanishes at `x = 0,1`. -/
theorem sineCoeffs_eqOn_Ioo {f g : ℝ → ℝ}
    (h : Set.EqOn f g (Set.Ioo (0 : ℝ) 1)) :
    sineCoeffs f = sineCoeffs g := by
  funext n
  simp only [sineCoeffs]
  split_ifs with hn
  · rfl
  · congr 1
    apply intervalIntegral.integral_congr
    intro y hy
    rw [Set.uIcc_of_le (by norm_num : (0 : ℝ) ≤ 1)] at hy
    change Real.sin ((n : ℝ) * Real.pi * y) * f y =
      Real.sin ((n : ℝ) * Real.pi * y) * g y
    rcases lt_or_eq_of_le hy.1 with hy0 | hy_eq0
    · rcases lt_or_eq_of_le hy.2 with hy1 | hy_eq1
      · rw [h ⟨hy0, hy1⟩]
      · subst hy_eq1
        simp [Real.sin_nat_mul_pi]
    · subst hy_eq0
      simp

/-- Equality on `(0,1)` gives a.e. equality on the interval-integral domain.
This is the endpoint-erasing lemma for Ktilde integrals.  If the local mathlib
signature of `intervalIntegral.integral_congr_ae` expects `Set.uIoc 0 1`, use
this lemma directly; if it expects `Set.Icc 0 1`, use the same proof with both
`0` and `1` singleton filters. -/
private theorem ae_eqOn_Ioo_restrict_uIoc {f g : ℝ → ℝ}
    (h : Set.EqOn f g (Set.Ioo (0 : ℝ) 1)) :
    f =ᵐ[volume.restrict (Set.uIoc (0 : ℝ) 1)] g := by
  rw [Set.uIoc_of_le (by norm_num : (0 : ℝ) ≤ 1)]
  refine (ae_restrict_iff' measurableSet_Ioc).2 ?_
  have hne1 : ∀ᵐ y : ℝ ∂volume, y ≠ 1 := by
    rw [ae_iff]
    simp only [not_not, Set.setOf_eq_eq_singleton, Real.volume_singleton]
  filter_upwards [hne1] with y hy1 hy
  exact h ⟨hy.1, lt_of_le_of_ne hy.2 hy1⟩

/-- Ktilde source integrals do not see endpoint values. -/
theorem ktilde_source_integral_congr_Ioo
    {t x : ℝ} {f g : ℝ → ℝ}
    (h : Set.EqOn f g (Set.Ioo (0 : ℝ) 1)) :
    (∫ y in (0 : ℝ)..1,
        f y * intervalNeumannConjugateKernel t x y)
      =
    ∫ y in (0 : ℝ)..1,
        g y * intervalNeumannConjugateKernel t x y := by
  apply intervalIntegral.integral_congr_ae
  exact (ae_eqOn_Ioo_restrict_uIoc h).mono (fun y hy => by rw [hy])

end ShenWork.Paper2.IntervalGradientSourceBridgeOpen
```

If the local `intervalIntegral.integral_congr_ae` lemma wants a different interval-set spelling, the proof should be changed only at the helper. The repository pattern with `ae_restrict_iff'` and `Real.volume_singleton` is already present, so this is not a new analytic assumption.

## 2. Ktilde value theorem from a continuous representative

This is the main replacement for `ktilde_source_integral_eq_sineHeatValue_open_of_continuousOn` in the chem-div path. It keeps the actual source `f` in both the Ktilde integral and `sineCoeffs f`, but it uses a closed continuous representative `g` only to invoke the existing continuous-on theorem.

```lean
import ShenWork.Paper2.IntervalGradientSourceBridgeOpenRepresentatives

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.Paper2.IntervalGradientSourceBridgeOpen

open ShenWork.IntervalNeumannFullKernel
open ShenWork.Paper2.IntervalDivergenceModeIdentity

/-- Ktilde-to-sine value theorem for a possibly endpoint-discontinuous source,
provided it agrees on `(0,1)` with a continuous closed representative. -/
theorem ktilde_source_integral_eq_sineHeatValue_open_of_Ioo_eq_continuousOn
    {t : ℝ} (ht : 0 < t)
    {f g : ℝ → ℝ}
    (hg : ContinuousOn g (Set.Icc (0 : ℝ) 1))
    (hfg : Set.EqOn f g (Set.Ioo (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    -(∫ y in (0 : ℝ)..1,
        f y * intervalNeumannConjugateKernel t x y)
      = unitIntervalSineHeatValue t (sineCoeffs f) x := by
  have hint :
      (∫ y in (0 : ℝ)..1,
          f y * intervalNeumannConjugateKernel t x y)
        =
      ∫ y in (0 : ℝ)..1,
          g y * intervalNeumannConjugateKernel t x y :=
    ktilde_source_integral_congr_Ioo (t := t) (x := x) hfg
  have hsine : sineCoeffs f = sineCoeffs g := sineCoeffs_eqOn_Ioo hfg
  rw [hint, hsine]
  exact ktilde_source_integral_eq_sineHeatValue_open_of_continuousOn
    (t := t) ht (f := g) hg hx

end ShenWork.Paper2.IntervalGradientSourceBridgeOpen
```

This theorem is the key non-wrapper progress theorem. It proves the same value formula as before, but its assumptions match what the zero-extension can honestly provide.

## 3. New slice bridge with integrability plus representative, not endpoint continuity

This is the direct replacement for the current `gradient_source_bridge_slice_open_continuousOn` in the chem-div path. It keeps Task294’s only source-side analytic need, `IntervalIntegrable`, and replaces `hdiv_cont` by a continuous representative.

## 4. New classical-ball discharger

This is the Task301/302-facing theorem. It discharges the same easy inputs as the existing `gradient_source_bridge_slice_open_of_classical_ball`, but leaves the correct frontier: integrability plus a continuous interior/closed representative for the chem-div source.

    {M r x s : ℝ} (hr : 0 < r) (hx : x ∈ Set.Ioo (0 : ℝ) 1)

    (hu_nonneg : ∀ y : intervalDomainPoint, 0 ≤ u s y)
    (hM : 0 < M)

    (hdiv_int : IntervalIntegrable (coupledChemDivSourceLift p u s) volume (0 : ℝ) 1)

        deriv (fun z : ℝ =>
          intervalFullSemigroupOperator r (chemFluxLifted p (u s)) z) x

        unitIntervalSineHeatValue r

## Optional helper: derive integrability from the representative

If the representative theorem is strong enough, `hdiv_int` can also be produced locally. I would not bake this into the first replacement theorem unless the local `IntervalIntegrable` congruence names are already convenient. A separate helper is cleaner:

```lean
import ShenWork.Paper2.IntervalGradientSourceBridgeOpenRepresentatives

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.Paper2.IntervalGradientSourceBridgeOpen

/-- Optional: a source equal a.e. to a continuous closed representative is interval
integrable.  Use this only if the local congruence lemma names are readily
available; otherwise keep `hdiv_int` explicit in the bridge. -/
theorem intervalIntegrable_of_Ioo_eq_continuousOn
    {f g : ℝ → ℝ}
    (hg : ContinuousOn g (Set.Icc (0 : ℝ) 1))
    (hfg : Set.EqOn f g (Set.Ioo (0 : ℝ) 1)) :
    IntervalIntegrable f volume (0 : ℝ) 1 := by
  -- Proof sketch:
  -- 1. `hg` gives `IntervalIntegrable g volume 0 1` after rewriting `uIcc`.
  -- 2. `ae_eqOn_Ioo_restrict_uIoc hfg` gives `f =ᵐ g` on the interval domain.
  -- 3. Apply the local `IntervalIntegrable.congr` / `congr_ae` lemma.
  have hg_int : IntervalIntegrable g volume (0 : ℝ) 1 := by
    exact (by
      rwa [Set.uIcc_of_le (by norm_num : (0 : ℝ) ≤ 1)] :
        ContinuousOn g (Set.uIcc (0 : ℝ) 1)).intervalIntegrable
  -- Finish with the repo/mathlib congruence lemma used locally for
  -- `IntervalIntegrable`; keep this as a small separate patch if names differ.
  exact hg_int.congr_ae (ae_eqOn_Ioo_restrict_uIoc hfg).symm

end ShenWork.Paper2.IntervalGradientSourceBridgeOpen
```

If `IntervalIntegrable.congr_ae` is not the exact method name in this checkout, do not block the main route on it. The main bridge theorem can and should take `hdiv_int` explicitly.

## Route decision

The correct smallest next theorem is:

```lean
ktilde_source_integral_eq_sineHeatValue_open_of_Ioo_eq_continuousOn
```

supported by:

```lean
sineCoeffs_eqOn_Ioo
ktilde_source_integral_congr_Ioo
```

Then replace the Task302 final theorem target with:

```lean
gradient_source_bridge_slice_open_of_classical_ball_representative
```

rather than trying to prove `ContinuousOn (coupledChemDivSourceLift p u s) (Set.Icc 0 1)`.

The new remaining frontier should be a genuine source-regularity statement of the form:

```lean
∃ divRep : ℝ → ℝ,
  ContinuousOn divRep (Set.Icc (0 : ℝ) 1) ∧
  Set.EqOn (coupledChemDivSourceLift p u s) divRep (Set.Ioo (0 : ℝ) 1)
```

plus `IntervalIntegrable (coupledChemDivSourceLift p u s) volume 0 1` if not derived from the representative. This is satisfiable from interior classical regularity and an endpoint trace/extension choice; it does not ask the zero extension itself to be continuous at the endpoints.

## Circular / false routes to avoid

Do not try to discharge `hdiv_cont` by proving continuity of the zero extension. That would force endpoint compatibility with the exterior zero values and is not needed for the integral/sine identity.

Do not change `Task294` open IBP to require endpoint continuity of `Q'`. Its correct source-side input is `IntervalIntegrable Q'`; the derivative/IBP theorem already avoids endpoint derivative requirements.

Do not hide this as a headline alias around `gradient_source_bridge_slice_open_continuousOn`. The useful theorem has a strictly different input shape: actual source integrability plus a continuous representative equal only on `(0,1)`.
