The load-bearing residual is the uniform lower floor hidden inside `DatumWienerData`.

`DatumWienerData` currently requires, for each bound `M > 0`, a uniform positive number `fm > 0` such that every paper-positive datum `u₀p` with `|u₀p| ≤ M` admits a Wiener lifting `W` with `fm ≤ W.floor`.

That is not a consequence of `PaperPositiveInitialDatum`. In `ShenWork/Paper2/Statements.lean`, `PaperPositiveInitialDatum` only gives a datum-specific closed-domain floor:

`∃ η > 0, ∀ x, η ≤ u₀ x`.

It does not give a common `η` for all PPID data bounded by the same `M`. Analytically, a bounded family of strictly positive closed-domain functions can have infima tending to `0`. Therefore `DatumWienerData` is a real restriction, not just a packaging inconvenience.

This is exactly why the current existing chain closes the strict branch only modulo `DatumWienerData`, not from arbitrary PPID data.
