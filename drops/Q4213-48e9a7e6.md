# Q4213: final assembly of per-`t` local Lipschitz theorem

## Verdict

Yes: the final theorem should simply package the last short-window gradient bound into the existential witness.

The caller is shielded from the new formula for `G`. If the theorem conclusion remains

```lean
∃ G : ℝ, 0 ≤ G ∧
  ∀ x ∈ Set.Icc (0 : ℝ) 1, ∀ y ∈ Set.Icc (0 : ℝ) 1,
    |intervalDomainLift (truncatedPicardLimit ... t) x -
      intervalDomainLift (truncatedPicardLimit ... t) y| ≤ G * |x - y|
```

then downstream callers cannot depend on whether `G` came from the old `[t/4,t/2,t]` window or from a chained short-window construction, unless they were using a stronger theorem exposing a concrete formula for `G`. With an existential result, the caller just receives some nonnegative Lipschitz constant.

## Target assembly pattern

Assume the chained short-window construction has already produced the last-window package:

```lean
-- Last short window containing t.
loN hiN Gbar : ℝ

hGbar_nonneg : 0 ≤ Gbar
ht_lo : loN ≤ t
ht_hi : t ≤ hiN

-- Uniform iterate gradient bound on the last window.
hlast : ∀ n : ℕ, IterGradOnWindow U loN hiN n Gbar
```

Then the final theorem proof should look like this:

```lean
theorem truncatedPicardLimit_lipschitzOn_positive_time
    (DT : TruncatedConjugateMildExistenceData p u₀)
    {t : ℝ} (ht : 0 < t) (htT : t ≤ DT.T) :
    ∃ G : ℝ, 0 ≤ G ∧
      ∀ x ∈ Set.Icc (0 : ℝ) 1, ∀ y ∈ Set.Icc (0 : ℝ) 1,
        |intervalDomainLift (truncatedConjugatePicardLimit p u₀ t) x -
          intervalDomainLift (truncatedConjugatePicardLimit p u₀ t) y| ≤
            G * |x - y| := by
  classical

  -- Your internal short-window chain.  The exact theorem name is whatever you
  -- use for the construction from `ht` and `htT`.
  obtain ⟨loN, hiN, Gbar, hGbar_nonneg, ht_lo, ht_hi, hlast⟩ :=
    truncatedShortWindowChain_final
      (DT := DT) (t := t) ht htT

  refine ⟨Gbar, hGbar_nonneg, ?_⟩

  -- Instantiate the limit-passage lemma at τ = t on the final window.
  have hLip_t :
      ∀ x ∈ Set.Icc (0 : ℝ) 1, ∀ y ∈ Set.Icc (0 : ℝ) 1,
        |intervalDomainLift (truncatedConjugatePicardLimit p u₀ t) x -
          intervalDomainLift (truncatedConjugatePicardLimit p u₀ t) y| ≤
            Gbar * |x - y| := by
    exact
      truncatedPicardLimit_lipschitzOn_of_window_grad
        (DT := DT)
        (U := U)
        (lo := loN) (hi := hiN)
        (τ := t) (G := Gbar)
        hlast ht_lo ht_hi

  exact hLip_t
```

If the limit-passage lemma has the arguments in a different order, use named arguments for the stable parameters and then let Lean show the remaining holes. The key instantiation is always:

```lean
(τ := t) (lo := loN) (hi := hiN) (G := Gbar)
```

with:

```lean
hlast : ∀ n, IterGradOnWindow U loN hiN n Gbar
ht_lo : loN ≤ t
ht_hi : t ≤ hiN
```

## If the chain gives a last-window constant `G_N`, not already `Gbar`

If the finite chain gives the last window with its own constant:

```lean
G_N : ℝ
hG_N_nonneg : 0 ≤ G_N
hlast_raw : ∀ n, IterGradOnWindow U loN hiN n G_N
hG_N_le_Gbar : G_N ≤ Gbar
```

then first monotone-upgrade the final window:

```lean
have hlast : ∀ n : ℕ, IterGradOnWindow U loN hiN n Gbar := by
  intro n
  exact IterGradOnWindow.mono hG_N_le_Gbar (hlast_raw n)
```

Then use the same existential witness:

```lean
refine ⟨Gbar, hGbar_nonneg, ?_⟩
```

This is usually the cleanest if `Gbar` is the finite max of all window constants.

## If `Gbar` is a finite max

A convenient shape is:

```lean
def Gbar : ℝ := (Finset.range (N + 1)).sup Gwin
```

or, more Lean-friendly for reals, use a fold/max with a fallback lower bound:

```lean
def Gbar : ℝ :=
  (Finset.range (N + 1)).sup' ⟨0, by simp⟩ (fun k => Gwin k)
```

Then prove once:

```lean
have hGbar_nonneg : 0 ≤ Gbar := by
  -- from `0 ≤ Gwin 0` or by defining `Gbar := max 0 (...)`
  ...

have hGwin_le_Gbar : ∀ k ≤ N, Gwin k ≤ Gbar := by
  intro k hk
  -- `Finset.le_sup'` / `Finset.le_sup` depending on the chosen definition
  ...
```

For this theorem, however, you only need the last-window consequence:

```lean
have hGlast_le_Gbar : Gwin N ≤ Gbar := hGwin_le_Gbar N (le_rfl)
```

and then:

```lean
have hlast : ∀ n, IterGradOnWindow U loN hiN n Gbar := by
  intro n
  exact IterGradOnWindow.mono hGlast_le_Gbar (hlast_raw n)
```

## Why the caller is agnostic to the new `G(t)`

The downstream caller typically does:

```lean
obtain ⟨G, hG_nonneg, hLip⟩ :=
  truncatedPicardLimit_lipschitzOn_positive_time (DT := DT) (t := t) ht htT
```

At that point, `G` is just a local witness. The caller can use:

```lean
hG_nonneg : 0 ≤ G
hLip : ∀ x ∈ Set.Icc (0 : ℝ) 1, ∀ y ∈ Set.Icc (0 : ℝ) 1,
  |... x - ... y| ≤ G * |x - y|
```

It cannot see the old formula or the new formula unless the theorem exposes it in the conclusion. Therefore a short-window/chained `G(t)`, even one that grows as `t ↓ 0`, does not change the caller’s typechecking as long as the caller only needs the existential Lipschitz package.

The only exception is if some downstream proof relied on a numeric side condition involving the old `G`, for example:

```lean
G ≤ oldExplicitConstant t
```

or unfolded the old theorem to recover a formula. With the stated existential theorem, that should not happen. The source-coefficient caller should remain valid because it only needs a finite local Lipschitz constant at the fixed positive time.
