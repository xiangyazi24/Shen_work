# Q3843 (cron1): CROSS-POLLINATION R2

## Executive answer

**(1) Semigroup weak identity.**  Yes: the identity

\[
  \frac{d}{dt}\langle S(t)f,\phi\rangle + \langle \partial_x S(t)f,\partial_x\phi\rangle = 0
\]

is reachable from the existing cosine-spectral infrastructure, but the Lean-friendly route should be phrased as **termwise time differentiation + one-mode gradient IBP + spatial integral/`tsum` interchange**, not as a black-box Parseval theorem.

The repository already contains almost all of the algebra needed:

* `ShenWork.PDE.CosineSpectrum` defines `cosineMode`, `cosineEigenvalue`, proves orthogonality, `cosineMode_deriv`, `cosineMode_second_deriv`, `cosineEigenfunction`, and the Neumann endpoint identities `cosineMode_neumann_left/right`.
* `ShenWork.PDE.IntervalSolutionCoeffDeriv` proves the twice-IBP coefficient identity `intervalCosineLaplacianCoeff_eq`.
* `ShenWork.Paper2.IntervalBFormCron2MildToWeakSpectral` already defines the tested pairings
  `cosineTestCoeff`, `spectralTestPairing`, `spectralTestLaplacianPairing`, and proves
  `spectralTestPairing_hasDerivAt` and `intervalFullSemigroupOperator_spectralTestedWeak_hasDerivAt`.

The current theorem in `IntervalBFormCron2MildToWeakSpectral` is the nearby **`φ''` formulation**:

```text
HasDerivAt (fun s => ∫ S(s)g * φ) (∫ S(t)g * φ'') t
```

under a smooth Neumann-test hypothesis on `φ` (`φ'(0)=φ'(1)=0`) and explicit series-interchange hypotheses.  For the exact weak-gradient identity in the question, add the one-IBP gradient coefficient lemma

\[
  \int_0^1 e_n'(x)\phi'(x)\,dx = \lambda_n \int_0^1 e_n(x)\phi(x)\,dx,
\]

where `e_n = cosineMode n` and `λ_n = unitIntervalCosineEigenvalue n`.  This uses the Neumann endpoint cancellation of `e_n'`, so it does **not** need `φ'(0)=φ'(1)=0`; it needs the usual `C^1`/integrability hypotheses on `φ` and `φ'`.

The important normalization warning: repo `cosineCoeffs` are the normalized Neumann coefficients, with zeroth mode unscaled and positive modes carrying the factor `2`.  Therefore the most robust proof is not to invoke a generic Parseval statement.  Instead prove the explicit coefficient identity for each mode and then use `tsum_congr` after the value/gradient interchanges.

The repo has the cosine eigenfunction calculus and a tested spectral weak theorem; the missing reusable lemma for the exact weak-gradient form is the gradient pairing interchange plus the one-mode identity `∫ e_n' φ' = λ_n ∫ e_n φ`.

For Route B, I would split the three lemmas this way:

1. `heatSemigroup_weak_identity`: homogeneous identity above, using `spectralTestPairing_hasDerivAt` plus `semigroup_gradient_pairing_series`.
2. `valueDuhamel_weak_identity`: for a source `F`, prove
   \[
     \frac d{dt}\left\langle\int_0^t S(t-s)F(s)ds,\phi\right\rangle
       + \left\langle\partial_x\int_0^t S(t-s)F(s)ds,\partial_x\phi\right\rangle
       = \langle F(t),\phi\rangle.
   \]
   In spectral form this is a per-mode Leibniz rule for
   `∫ s in 0..t, exp (-(t-s)*λ_n) * Fhat s n`, so the endpoint term is `Fhat t n`; this avoids needing a pointwise `S(0)=id` theorem at function level.
3. `conjugateDuhamel_weak_identity_after_BN_duality`: use the existing lagwise `TruncatedBNDualityForTestAt` to rewrite the conjugate chemotaxis leg into the adjoint semigroup-gradient form before applying the same Duhamel endpoint argument.  The sign should be left to the already-stated mild formula plus `TruncatedBNDualityForTestAt`; after endpoint extraction it should match the final
   `p.χ₀ * ∫ truncatedChemFluxLifted p (u t) * deriv φ` term.

This would close the named frontier `TruncatedMildSemigroupWeakAfterBNDualityOn` without reopening the B_N duality: the file `IntervalBFormCron2MildToWeak.lean` correctly isolates the remaining problem as semigroup/Duhamel weak differentiation after lagwise duality.

## (2) Time shift for Route A and the comparison API

The existing comparison theorem **does handle the time-shifted problem directly**, but it should be instantiated with the shorter horizon

\[
  T' = T - s,
\]

not with the original horizon `T`.

The coefficients after shifting are functions on all real times anyway:

```lean

  fun τ x => F (s + τ) x
```

Do **not** instantiate the comparison theorem over the original length `T` after shifting unless you actually have coefficient and solution hypotheses on the larger original interval `[s, s+T]`.  The current API only asks that `Bshift` and `Cshift` be regular on the strip `[0,T'] × [0,1]`; choosing `T' = T-s` is exactly what maps that strip back into the known original strip `[0,T] × [0,1]`.

For the square-heat barrier route, the same rule applies: any shifted barrier/comparison call should use `T-s`.  The comparison theorem itself needs no change; the work is in producing the shifted coefficient, subsolution/supersolution, bounds, and initial-seed data at `τ=0`, i.e. original time `s`.
