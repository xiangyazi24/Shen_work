# Q3909: can the truncated conjugate limit be lifted to `Traj` and compared by uniqueness?

## Short verdict

Yes, but **not from “mild equation + ball bound” alone**.  The `Traj` route needs two extra structural ingredients that are not part of `IntervalConjugateMildSolution`:

1. a **jointly continuous closed-box lift**

```lean
Continuous (fun z : ↥(Set.Icc (0 : ℝ) t) × intervalDomainPoint => w z.1.1 z.2)
```

for the candidate `w`, and

2. a **time-zero seam** for the `Traj` fixed-point equation, because `Traj t` is over `Icc 0 t`, while `IntervalConjugateMildSolution` only states the Duhamel identity for `0 < t`.

The ball bound `|w| ≤ M` is not a lifting requirement.  It is relevant only upstream for contraction estimates / invariance / the older pointwise uniqueness theorem, not for `fixedPoint_unique_traj` once both `Traj` fixed-point hypotheses are available.

## (a) What is `Traj t`?

In `IntervalChiNegTrajBanach.lean`, it is exactly:

```lean
/-- The trajectory BCF space: continuous maps on the compact product
`[0,t] × Ω̄`, sup-metrised.  A Mathlib `CompleteSpace`. -/
abbrev Traj (t : ℝ) : Type := C(↥(Set.Icc (0 : ℝ) t) × intervalDomainPoint, ℝ)
```

So `Traj t` is a `ContinuousMap` on the compact closed time box `Set.Icc 0 t`, not on `Ioc 0 t`.

The bridge from `Traj` back to landed functions is:

```lean
def trajFun {t : ℝ} (U : Traj t) : ℝ → intervalDomainPoint → ℝ :=
  fun s x => if hs : s ∈ Set.Icc (0 : ℝ) t then U (⟨s, hs⟩, x) else 0
```

Thus inside `[0,t]`, `trajFun U s x` is the BCF value; outside it is forced to `0`.

## (b) Does lifting require anything beyond continuity + mild equation + ball bound?

To **construct** a `Traj t` element from a candidate `w : ℝ → intervalDomainPoint → ℝ`, you need closed-box joint continuity:

```lean
def trajLift {t : ℝ} (w : ℝ → intervalDomainPoint → ℝ)
    (hwcont : Continuous (fun z : ↥(Set.Icc (0 : ℝ) t) × intervalDomainPoint =>
      w z.1.1 z.2)) : Traj t :=
  ⟨fun z => w z.1.1 z.2, hwcont⟩
```

A proof of `HasContinuousSlices` is not enough: it gives spatial continuity for each positive time, not joint continuity in `(time,space)` on the closed box.  The file headers explicitly mark this as the G1 issue: BCF-valued / full-box Duhamel continuity is a real missing analytic bridge, especially at the singular Duhamel endpoint.

The mild equation is not needed to form the `Traj`; it is needed to prove that the `Traj` lift is a fixed point of the same `Phi`.

The ball bound is not needed for either the raw lift or `fixedPoint_unique_traj`.  It may be needed to prove the contraction self-map or to use pointwise uniqueness, but `fixedPoint_unique_traj` itself only consumes:

```lean
hPhi : ContractingWith q Phi
hU   : Function.IsFixedPt Phi U
hV   : Function.IsFixedPt Phi V
```

## (d) What about the `t = 0` boundary?

This is the main trap.  `Traj` uses `Icc 0 t`, not `Ioc 0 t`.  So a `Traj` fixed-point proof is an equality of continuous maps on the **closed** box and must address `s = 0`.

By contrast:

```lean
IntervalConjugateMildSolution p T u₀ u
```

is defined as:

```lean
∀ t, 0 < t → t ≤ T → ∀ x,
  u t x = intervalConjugateDuhamelMap p u₀ u t x
```

It says nothing at `t = 0`.

This is why `IntervalChiNegTrajBridges.lean` states the uniqueness identity only for

```lean
hs : s ∈ Set.Ioc (0 : ℝ) t
```

in `conjugatePicardLimit_trajLift_isFixedPt`.  The header explicitly calls the `s=0` endpoint a separate `τ=0` convention.

## Consequence for truncated vs canonical equality

The proposed uniqueness proof can work in the `Traj` model **after** you prove/provide:

1. `truncatedConjugatePicardLimit` has a closed-box joint-continuous lift to `Traj T`;
2. the same for the canonical candidate, or a preexisting `Vstar : Traj T` fixed point;
3. both are fixed points of the same `Phi`; for the truncated candidate, this needs the positive-time mild equation **plus** the zero-time seam;
4. the same `ContractingWith q Phi` hypothesis used by `fixedPoint_unique_traj`.

Then `fixedPoint_unique_traj` gives equality of the two `Traj` elements.  Evaluating on `s ∈ Icc 0 T` gives closed-box equality.  If you only have the positive-time mild equation, you get at most the `Ioc 0 T` equality pattern already used by `conjugatePicardLimit_trajLift_isFixedPt`; `s=0` must be handled separately by the definitions/conventions of the two limits.

## Practical recommendation

The smallest real next lemma is not “lift from mild solution” directly.  Prove first:

```lean
intervalConjugateDuhamelMap_congr_on_Icc
```

Then prove:

```lean
trajLift_isFixedPt_of_intervalConjugateMildSolution
```

with explicit hypotheses `hwcont` and `hzero`.  That will let you reuse the existing `fixedPoint_unique_traj` for truncated-vs-canonical equality without pretending that ball boundedness or positive-time mildness solves the closed-box `t=0` seam.

There is also an older pointwise uniqueness theorem:

```lean
ShenWork.IntervalConjugatePicard.intervalConjugateMildSolution_unique_of_data
```

It may be simpler for positive-time equality if you have its exact data hypotheses (`hbound`, `hnonneg`, `HasContinuousSlices`, `HasJointMeasurability`).  But for the specific `Traj`/Banach route, the missing pieces are the closed-box lift, Duhamel congruence, and zero seam described above.
