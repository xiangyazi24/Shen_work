## Recommended theorem: return `DifferentiatedMildSliceDiffOn`

**Placement:** preferably `ChemMildC1etaUncond.lean`; alternatively a new bridge module importing both source and C1/η layers.

**Why this return type:** It is more reusable than immediately returning the final triple. The package records the still-honest bridge assumptions and lets existing consumers (`differentiatedMildSliceDiffOn_*` and `chemMild_C1eta_slice_diffOn`) be used without duplicating proof logic. It also prevents the theorem from looking more unconditional than it is.

## Hidden gaps / overclaim risks

1. **Cutoff source vs real source.** The theorem should use
   ```lean
   Q := chemFluxCthetaCutoffSource p D.u D.T
   ```
   in both `hleg_int` and `w_split`. If the actual mild representation is written with the uncut `chemFluxLifted p (D.u s)`, a separate cutoff-agreement theorem over `0 < s < t ≤ D.T` is needed. Do not silently rewrite this in the C1/η theorem.

2. **Representation is still an assumption.** `w_split` is the differentiated mild representation on `[0,1]`. Task188 only produces `ChemLegData`; it does not prove the differentiated mild equation or the spatial Leibniz interchange for the initial/react legs. Keep `w_split` carried.

3. **Leg integrability is still an assumption.** `differentiatedMildSliceDiffOn_of_brick4_chem` needs
   ```lean
   hleg_int : ∀ x, IntervalIntegrable (... unitIntervalCosineHeatSecondValue ...)
   ```
   Task188 does not provide it. Do not claim the new wiring theorem closes that.

4. **Init/react legs are still external.** `init_diff`, `react_diff`, `init_holder`, and `react_holder` should remain inputs. They are separate from chemFlux source regularity.

5. **Exponent bookkeeping.** Task188 gives `θ < 1/2`. The C1/η constructor needs `η < θ` and `η < 1`. Taking `hη0 : 0 < η` and `hηθ : η < θ` is enough; derive `η < 1` from `hηθ` and `hθlt` by `linarith`.

6. **Do not collapse to a false unconditional theorem.** The theorem should not be named as if it proves full mild `C^{1+η}` from `GradientMildSolutionData` alone. It only discharges the chem-data input; representation and non-chem leg inputs remain.

## Recommendation

Implement the `DifferentiatedMildSliceDiffOn` theorem first. It is the shortest honest next theorem and is structurally aligned with `ChemMildC1etaUncond.lean`. Then add the direct triple wrapper only if the next headline module wants the final `DifferentiableOn ∧ Holder ∧ Summable` statement immediately.
