# Q3792 audit: gradient mild vs Neumann divergence-source mild formulation

## 1. Mathematical verdict

The current weak gradient mild map in `IntervalGradientDuhamelMap` is **not** the Neumann/no-flux mild formulation of

```text
u_t = u_xx - Ōá ‚ąāx Q + logistic
```

on `[0,1]`.

The map currently uses the chemotaxis Duhamel leg

```lean
‚ąę s in (0 : ‚ĄĚ)..t,
  deriv (fun z => intervalFullSemigroupOperator (t - s) (chemFluxLifted p (u s)) z) x
```

that is, `‚ąā‚āď S_N(ŌĄ) Q`.  For the Neumann heat kernel `K_N`, this is

```text
‚ąę‚āÄ¬Ļ ‚ąā‚āď K_N(ŌĄ,x,y) Q(y) dy.
```

The Neumann variation-of-constants formula for the divergence source `Q_x` is instead

```text
S_N(ŌĄ)(Q_x)(x) = ‚ąę‚āÄ¬Ļ K_N(ŌĄ,x,y) Q_y(y) dy.
```

When `Q(0)=Q(1)=0`, integration by parts gives

```text
S_N(ŌĄ)(Q_x)(x)
  = -‚ąę‚āÄ¬Ļ ‚ąāŠĶß K_N(ŌĄ,x,y) Q(y) dy.
```

This is exactly the B/conjugate operator in `IntervalConjugateDuhamelMap`:

```lean
intervalConjugateKernelOperator ŌĄ Q x
  = -‚ąę y,
      deriv (fun y' => intervalNeumannFullKernel ŌĄ x y') y * Q y
      ‚ąā intervalMeasure 1
```

So the correct Neumann divergence-source operator is `intervalConjugateKernelOperator`, not `deriv_x (intervalFullSemigroupOperator ... Q)`.

The key Neumann-kernel identity is **not** `‚ąā‚āď K_N = -‚ąāŠĶß K_N`.  That identity is true for the whole-line heat kernel, but false for the reflected Neumann interval kernel.  The committed identity is instead the `KŐÉ` identity:

```lean
ShenWork.IntervalNeumannFullKernel.deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_source_integral_open
```

which proves

```lean
deriv (fun z => intervalFullSemigroupOperator ŌĄ Q z) x
  = -(‚ąę y in (0 : ‚ĄĚ)..1,
        Q' y * intervalNeumannConjugateKernel ŌĄ x y)
```

under open/right derivative hypotheses.  Here `intervalNeumannConjugateKernel` is the signed primitive kernel `KŐÉ`, satisfying

```text
‚ąāŠĶß KŐÉ = ‚ąā‚āď K_N,
```

not the B-kernel operator `-‚ąāŠĶßK_N`.

Spectrally, the mismatch is also clear:

```text
‚ąā‚āď S_N(ŌĄ) Q  -> sine/Dirichlet heat expression,
S_N(ŌĄ) Q_x   -> cosine/Neumann heat expression,
B_N(ŌĄ) Q     -> cosine/Neumann heat expression.
```

For Neumann/no-flux PDE closure, the chemotaxis source must go through the cosine/B-form route.  The gradient map‚Äôs chemotaxis leg produces a sine/Dirichlet component.  It can satisfy an interior heat equation with endpoint source `Q_x`, but it is not the Neumann semigroup realization of the divergence source and does not give the global cosine representation required by `hpde_u_of_generalChi_sourceSpectralData`.

The endpoint obstruction in Q3790 is a symptom, not the only problem: at `x=0,1`, `‚ąā‚āďS_N(ŌĄ)Q` is forced to vanish by `intervalFullSemigroupOperator_deriv_at_zero_eq_zero` / `_at_one_eq_zero`, while `intervalConjugateKernelOperator ŌĄ Q` is not generically endpoint-zero.

## 2. Lean strategy

Do not try to prove

```lean

```

from actual gradient mild data.  Its explicit `hgradB` input is the false identity

```text

```

The Lean route should use `IntervalConjugateDuhamelMap` as the actual mild formulation for the Neumann divergence-source PDE.  Concretely:

1. Keep `GradientMildSolutionData` quarantined as the fixed point for the old gradient map.  It is useful for estimates that only require the map‚Äôs bounds, but it should not be the final PDE-producing object for the Neumann/no-flux theorem.
2. Route the final PDE proof through `IntervalConjugateDuhamelMap.IntervalConjugateMildSolution`, or introduce a parallel `ConjugateMildSolutionData` record later if the Picard/existence pipeline needs packaged bounds and positivity.
3. Use Task312 to get the B-form global cosine representation for a conjugate/B-form fixed point.
4. Use Task311‚Äôs `IntervalDomainPdeUGeneralChi.Hsource_of_bForm_global_generalChi` / `hpde_u_of_bForm_global_generalChi` to produce `hpde_u`.

The smallest Lean-safe next theorem is therefore not another gradient-to-B bridge.  It is a direct `hpde_u` producer for the conjugate Picard/fixed-point route, composing Task312 and Task311.

## 3. Can the gradient map be salvaged?

Not for the Neumann/no-flux B-form route.

The only true identity replacing the false `hgradB` is the `KŐÉ`/sine identity:

```lean
deriv (fun z => intervalFullSemigroupOperator ŌĄ Q z) x
  = -(‚ąę y in (0 : ‚ĄĚ)..1,
        Q' y * intervalNeumannConjugateKernel ŌĄ x y)
```

already represented by

```lean
ShenWork.IntervalNeumannFullKernel.deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_source_integral_open
```

and by the regularity/representative integrated bridge in

```lean
ShenWork.Paper2.IntervalGradientSourceBridgeRegularityRepresentative
```

via

```lean
gradientMildDuhamelTerms_eq_integral_mixedSpectralSource_of_gradientMildSolutionData_and_regular
```

This avoids `hgradB`, but it lands in a mixed sine/cosine source representation, not in the cosine B-form source representation consumed by

```lean
ShenWork.IntervalDomainPdeUGeneralChi.hpde_u_of_generalChi_sourceSpectralData
```

To salvage only the **interior PDE identity** for the old gradient map, one would need a separate direct time-differentiation theorem for the gradient Duhamel leg, roughly:

```lean
‚ąāt (‚ąę‚āÄŠĶó ‚ąā‚āďS_N(t-s) Q(s) ds)
  = ‚ąāxx (‚ąę‚āÄŠĶó ‚ąā‚āďS_N(t-s) Q(s) ds) + Q_x(t,x)
```

on `x ‚ąą Ioo 0 1`, with substantial endpoint/singularity and source-regularity hypotheses.  That theorem would not prove the Neumann/no-flux mild formulation, would not produce a B-form global cosine representation, and would not feed Task311.  It is therefore not the shortest route to the paper theorem.
