# Q3976 (cron2): closing the `hleft` gradient-window hypothesis

## Executive answer

For the current proof architecture, I would **not** try to close

```lean
hleft : ∀ n : ℕ, IterGradOnWindow U a lo n Gw
```

from `truncatedConjugatePicardIter_geometric`.  That geometric estimate is an
`L∞` Picard-difference/convergence estimate.  It does not control spatial
derivatives of the iterates, and differentiating the Picard map is exactly where
the singular Volterra kernel and the gradient feedback appear.

The clean way to close `hleft` is to make the left Volterra profile theorem have
a real statement, then use the already-proved algebraic comparison

```lean
truncLeftProfile_le_Gw
```

together with monotonicity of

```lean
P τ = truncLeftProfile M A_L A_F B_F χ lo τ = C / sqrt τ + D
```

on `[a, lo]`.  In other words, the actual `hleft` proof should be only the
following chain:

```text
iterate gradient at τ ≤ P(τ) ≤ P(a) ≤ Gw.
```

The only remaining analytic frontier is therefore the theorem

```lean
∀ n, ∀ τ, 0 < τ → τ ≤ lo → ∀ x,
  |deriv (intervalDomainLift (U n τ)) x| ≤
    truncLeftProfile Mw A_L A_F B_F p.χ₀ lo τ
```

not the final `hleft` glue.

## Why the direct geometric-series shortcut is not enough

There are three possible “geometric” arguments, and none closes the current goal
under the present hypotheses.

### 1. Picard geometric convergence is in the wrong norm

`truncatedConjugatePicardIter_geometric` controls differences of iterates in the
Picard ball norm.  The `hleft` field asks for

```lean
|deriv (intervalDomainLift (U n t')) x| ≤ Gw
```

uniformly in `n`.  Passing from an `L∞` geometric estimate to a derivative
estimate would require differentiating the Picard operator and bounding the
source derivative.  That is exactly the Volterra-gradient estimate.

### 2. A uniform bound on `[a, lo]` alone cannot see the Duhamel integral over `[0, a]`

For `t' ∈ [a, lo]`, the Duhamel term integrates over `0 ≤ s ≤ t'`.  The part
`s < a` still occurs.  The source bound after IBP contains the flux derivative,
which contains `|∂x U_n(s)|`.  Thus a bound only on `[a, lo]` is not enough.
You either need a left profile down to `0+`, or you need to recurse to earlier
windows forever.

### 3. A weighted-gradient geometric proof is possible, but stronger than the existing assumptions

One can define

```text
E_n = sup_{0 < s ≤ lo} sqrt s · ||∂x U_n(s)||∞.
```

Then the singular source estimate gives roughly

```text
E_{n+1} ≤ C + 2 K L0 lo + K · beta · κ · sqrt(lo) · E_n,
```

where `K = Cg`, `beta = |χ₀| B_F`, and `κ = 2√3`.

So this would require

```text
K · beta · κ · sqrt(lo) < 1.
```

In the current notation, with `a = t/4`, `lo = t/2`, `hi = t`, the existing
window smallness is

```text
truncWindowB B_F χ a hi
  = K · 2√(hi - a) · |χ| · B_F
  = K · 2√(3t/4) · beta < 1.
```

But the weighted-gradient coefficient would be

```text
K · 2√3 · sqrt(t/2) · beta
  = √2 · K · 2√(3t/4) · beta,
```

which is stronger by a factor `√2`.  It also gives an envelope of the form
`E / sqrt a`; you would then need a new comparison theorem showing
`E / sqrt a ≤ Gw`.  So it is not a simpler replacement for the existing
`truncLeftProfile_le_Gw` path.

The chosen profile

```text
P(t') = C / sqrt(t') + D
```

is better because its feedback coefficient is

```text
truncLeftB B_F χ lo = K · 2√lo · |χ| · B_F,
```

and for `lo = t/2`, `hi - a = 3t/4`, we have

```text
2√lo ≤ 2√(hi-a).
```

Thus the already-assumed `truncWindowB < 1` implies the left-profile
contraction.  This is exactly why the left profile is the right abstraction.

## Minimal Lean bridge

The following is the small layer I would add to
`IntervalTruncatedLeftProfile.lean`.  It does not prove the analytic Duhamel
statement; instead, it gives that statement the right interface and makes the
`hleft` closure algebraic.

```lean
import ShenWork.Paper2.IntervalTruncatedLeftProfile

open MeasureTheory Set
open scoped BigOperators Topology Real

noncomputable section

namespace ShenWork.Paper2.TruncatedGradientWindow

open ShenWork.IntervalDomain (intervalDomainLift intervalDomainPoint)
open ShenWork.HeatKernelGradientEstimates
  (heatGradientLinftyLinftyConstant heatGradientLinftyLinftyConstant_nonneg)

/-- The left singular Volterra profile bound, valid on `(0, lo]`. -/
abbrev IterGradLeftProfile
    (U : ℕ → ℝ → intervalDomainPoint → ℝ)
    (M A_L A_F B_F χ lo : ℝ) (n : ℕ) : Prop :=
  ∀ τ, 0 < τ → τ ≤ lo → ∀ x : ℝ,
    |deriv (intervalDomainLift (U n τ)) x| ≤
      truncLeftProfile M A_L A_F B_F χ lo τ

/-- `P(τ) = C / sqrt τ + D` is decreasing in `τ`, provided `C = Cg*M ≥ 0`. -/
theorem truncLeftProfile_anti_mono_time
    {M A_L A_F B_F χ lo a τ : ℝ}
    (hM : 0 ≤ M) (ha : 0 < a) (haτ : a ≤ τ) :
    truncLeftProfile M A_L A_F B_F χ lo τ
      ≤ truncLeftProfile M A_L A_F B_F χ lo a := by
  have hC_nonneg : 0 ≤ truncLeftSingularC M := by
    unfold truncLeftSingularC
    exact mul_nonneg heatGradientLinftyLinftyConstant_nonneg hM
  have hτ : 0 < τ := lt_of_lt_of_le ha haτ
  have hsqrta_pos : 0 < Real.sqrt a := Real.sqrt_pos_of_pos ha
  have hsqrta_le_sqrtτ : Real.sqrt a ≤ Real.sqrt τ := Real.sqrt_le_sqrt haτ
  have hinv : (Real.sqrt τ)⁻¹ ≤ (Real.sqrt a)⁻¹ := by
    -- In some Mathlib snapshots this lemma is named `inv_le_inv_of_le₀`.
    exact inv_le_inv_of_le hsqrta_pos hsqrta_le_sqrtτ
  have hsing :
      truncLeftSingularC M / Real.sqrt τ
        ≤ truncLeftSingularC M / Real.sqrt a := by
    rw [div_eq_mul_inv, div_eq_mul_inv]
    exact mul_le_mul_of_nonneg_left hinv hC_nonneg
  unfold truncLeftProfile
  exact add_le_add_right hsing _

/-- Convert the left profile into the `IterGradOnWindow` field needed by
`TruncatedGradientWindowWiring`. -/
theorem IterGradOnWindow.of_left_profile
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {M A_L A_F B_F χ a lo G : ℝ}
    (hM : 0 ≤ M) (ha : 0 < a)
    (hprofile : ∀ n, IterGradLeftProfile U M A_L A_F B_F χ lo n)
    (hPaG : truncLeftProfile M A_L A_F B_F χ lo a ≤ G) :
    ∀ n : ℕ, IterGradOnWindow U a lo n G := by
  intro n τ haτ hτlo x
  have hτpos : 0 < τ := lt_of_lt_of_le ha haτ
  have hPτ :
      |deriv (intervalDomainLift (U n τ)) x| ≤
        truncLeftProfile M A_L A_F B_F χ lo τ :=
    hprofile n τ hτpos hτlo x
  have hmono :
      truncLeftProfile M A_L A_F B_F χ lo τ
        ≤ truncLeftProfile M A_L A_F B_F χ lo a :=
    truncLeftProfile_anti_mono_time hM ha haτ
  exact hPτ.trans (hmono.trans hPaG)

/-- Source estimate with the left profile in place of a constant gradient bound.
This is the profile analogue of `abs_logistic_sub_chi_flux_le`. -/
theorem abs_logistic_sub_chi_flux_le_profile
    {L Fp A_L A_F B_F χ P : ℝ}
    (hL : |L| ≤ A_L)
    (hF : |Fp| ≤ A_F + B_F * P) :
    |L - χ * Fp| ≤
      truncLeftSourceConst A_L A_F χ + truncLeftBeta B_F χ * P := by
  unfold truncLeftSourceConst truncLeftBeta
  calc
    |L - χ * Fp| ≤ |L| + |χ * Fp| := by
      simpa using (abs_sub_le L 0 (χ * Fp))
    _ = |L| + |χ| * |Fp| := by
      rw [abs_mul]
    _ ≤ A_L + |χ| * (A_F + B_F * P) := by
      exact add_le_add hL (mul_le_mul_of_nonneg_left hF (abs_nonneg χ))
    _ = A_L + |χ| * A_F + (|χ| * B_F) * P := by
      ring

/-- Abstract left-profile wiring.  The two analytic fields are `hbase` and
`hkernel_step`; the induction itself is completely formal. -/
structure TruncatedLeftProfileWiring
    (U : ℕ → ℝ → intervalDomainPoint → ℝ)
    (Src : ℕ → ℝ → ℝ → ℝ)
    (M A_L A_F B_F χ lo : ℝ) : Prop where
  hbase : IterGradLeftProfile U M A_L A_F B_F χ lo 0
  hsource_of_profile : ∀ n,
    IterGradLeftProfile U M A_L A_F B_F χ lo n →
      ∀ s, 0 < s → s ≤ lo → ∀ y : ℝ,
        |Src n s y| ≤
          truncLeftSourceConst A_L A_F χ
            + truncLeftBeta B_F χ *
                truncLeftProfile M A_L A_F B_F χ lo s
  hkernel_step : ∀ n,
    (∀ s, 0 < s → s ≤ lo → ∀ y : ℝ,
      |Src n s y| ≤
        truncLeftSourceConst A_L A_F χ
          + truncLeftBeta B_F χ *
              truncLeftProfile M A_L A_F B_F χ lo s) →
      IterGradLeftProfile U M A_L A_F B_F χ lo (n + 1)

/-- The real replacement for the placeholder `truncLeftProfile_all`. -/
theorem truncLeftProfile_all_of_wiring
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {Src : ℕ → ℝ → ℝ → ℝ}
    {M A_L A_F B_F χ lo : ℝ}
    (W : TruncatedLeftProfileWiring U Src M A_L A_F B_F χ lo) :
    ∀ n : ℕ, IterGradLeftProfile U M A_L A_F B_F χ lo n := by
  intro n
  induction n with
  | zero =>
      exact W.hbase
  | succ n IH =>
      exact W.hkernel_step n (W.hsource_of_profile n IH)

end ShenWork.Paper2.TruncatedGradientWindow
```

The proof of `truncLeftProfile_anti_mono_time` is intentionally elementary.  If
your local Mathlib names the inverse lemma differently, the only line that may
need adjustment is

```lean
exact inv_le_inv_of_le hsqrta_pos hsqrta_le_sqrtτ
```

which can be replaced by the local spelling, usually one of:

```lean
exact inv_le_inv_of_le₀ hsqrta_pos hsqrta_le_sqrtτ
```

or a two-line `gcongr` proof.

## How this closes the existing `hleft` field

In `IntervalTruncatedPositiveTimeBootstrap.lean`, import the left-profile file:

```lean
import ShenWork.Paper2.IntervalTruncatedLeftProfile
```

Then replace

```lean
hleft := by sorry
```

inside the construction of

```lean
W : TruncatedGradientWindowWiring p U Src Mw A_L A_F B_F a lo hi Gw
```

by the following shape.

```lean
hleft := by
  -- This is the only genuinely analytic input left for the left interval.
  -- It is Theorems 2--4 of `IntervalTruncatedLeftProfile.lean` with real
  -- statements instead of `True := trivial`.
  have hleft_profile :
      ∀ n : ℕ,
        IterGradLeftProfile U Mw A_L A_F B_F p.χ₀ lo n := by
    /-
    Prove this by `truncLeftProfile_all_of_wiring`.

    The fields of the wiring are:

    * `hbase`:
        `U 0 τ = S(τ) u₀`, so
        `|∂x U 0 τ| ≤ Cg * Mw / sqrt τ ≤ P(τ)`.

    * `hsource_of_profile`:
        from the ball bound and the profile-gradient IH,
        use the Q3974 product-rule flux estimate with
        `G := truncLeftProfile Mw A_L A_F B_F p.χ₀ lo s` to get
        `|flux'| ≤ A_F + B_F * P(s)`, then use
        `abs_logistic_sub_chi_flux_le_profile`.

    * `hkernel_step`:
        use the iterate representation and the singular-source Duhamel estimate
        for
        `|Src n s y| ≤ L0 + beta * (C / sqrt s + D)`.
        The kernel integrals give
        `K * 2√τ * L0 + K * beta * C * κ + K * beta * D * 2√τ`;
        since `τ ≤ lo`, this is bounded by the definition of `D`.
    -/
    sorry

  have hPa_le_Gw :
      truncLeftProfile Mw A_L A_F B_F p.χ₀ lo a ≤ Gw := by
    dsimp only [a, lo, hi, Gw]
    -- Here `a = t/4`, `lo = t/2 = 2*a`, and `hi = t = 4*a`.
    convert
      (truncLeftProfile_le_Gw
        (M := Mw) (A_L := A_L) (A_F := A_F) (B_F := B_F)
        (chi := p.χ₀) (a := t / 4) (lo := t / 2) (hi := t)
        (hM := le_of_lt DT.hM)
        (hAL := hAL_nn)
        (hAF := hAF_nn)
        (hBF := hΓ_M_nn)
        (ha := by linarith)
        (hlo := by ring)
        (hhi := by ring)
        (hcontr := hBcontr)) using 1
    · rfl
    · ring
    · ring
    · rfl

  exact
    IterGradOnWindow.of_left_profile
      (U := U) (M := Mw) (A_L := A_L) (A_F := A_F)
      (B_F := B_F) (χ := p.χ₀) (a := a) (lo := lo) (G := Gw)
      (le_of_lt DT.hM)
      (by dsimp [a]; linarith)
      hleft_profile
      hPa_le_Gw
```

Depending on how `convert` simplifies the local `let`s, the four small `rfl` /`ring` subgoals after `truncLeftProfile_le_Gw` may differ.  The important
mathematical content is exactly this:

```lean
truncLeftProfile_le_Gw
  (ha := by linarith)
  (hlo := by ring)   -- t/2 = 2 * (t/4)
  (hhi := by ring)   -- t   = 4 * (t/4)
  (hcontr := hBcontr)
```

followed by `IterGradOnWindow.of_left_profile`.

For the current local proof, you do not actually need to expose every constant
as arguments.  It is often easier to state this theorem in the same local style
as the `let`s in `truncatedPicardLimit_lipschitzOn_positive_time`, so the
instantiation is mostly `rfl`.

## Bottom line

The right minimal change is:

1. Add the `IterGradLeftProfile` abstraction and the no-sorry algebraic glue
   `IterGradOnWindow.of_left_profile`.
2. Replace the `True := trivial` left-profile placeholders by a real analytic
   provider proving `∀ n, IterGradLeftProfile ... n`.
3. Close `hleft` by
   `profile bound → monotonicity on [a,lo] → truncLeftProfile_le_Gw`.

This avoids duplicating the gradient bootstrap in a stronger weighted norm and
keeps all derivative-under-integral and Duhamel representation work localized in
one left-profile theorem.
