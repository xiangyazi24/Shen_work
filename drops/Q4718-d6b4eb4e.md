# Executive verdict

There are three separate issues here, and all three admit a much cheaper treatment than a `decide` proof over `243^3` triples.

1. **For the ring laws**, the cleanest mathematical type is the actual ideal quotient
   ```lean
   O_L ⧸ (Ideal.span ({pi ^ 5} : Set O_L))
   ```
   or equivalently `O_L ⧸ (Ideal.span ({pi} : Set O_L)) ^ 5`. It gets `CommRing` definitionally from `Ideal.Quotient.commRing`. If the explicit coordinate carrier must be retained, use either `Function.Surjective.commRing` from a raw surjective reduction map, or transport the quotient structure across a proved equivalence with `Equiv.commRing`. There is no need to enumerate associativity or distributivity triples.

2. **For cubes**, ordinary Hensel lifting does not apply. The derivative of `X^3-u` is zero in the residue field, and in fact most units of `R5` are not cubes. Using the stated relations, one gets the exact formula
   ```text
   (a + b*pi + c*pi^2)^3 = a^3 + 3*b^3 + 3*a^2*b*pi.
   ```
   Consequently the unit cubes are exactly
   ```text
   epsilon + t*pi^3 + t*pi^4
   = epsilon + 3*t*(1+pi),
   epsilon in {+1,-1}, t in F_3.
   ```
   There are only six unit cubes among 162 units. Thus `R5^×/(R5^×)^3` has order 27.

3. **For the homogeneous equation**, a truncated `pi`-adic order removes nearly all cases. It cannot resolve branches where two terms have equal valuation and cancel, so a tiny residue-field or associated-graded calculation remains. The complete valuation split is given below. In particular, if `D` is a unit, divide by `D^6` and reduce to the two-variable equation
   ```text
   y * (y - 3*x + 2) = x^3,
   x = U/D^2, y = W/D^3.
   ```
   That reduction alone changes a three-variable search into a two-variable one; the valuation split makes the remaining checks much smaller still.

I audited the exact APIs in Mathlib `v4.31.0`. I did not find `reduceOLHom` in the connected `Shen_work` repository, so the code below uses the names from the question and may need namespace adjustments.

# 1. Obtaining `CommRing` without enumerating ring laws

## 1.1 Preferred mathematical definition: an ideal quotient

Use the canonical quotient type.

```lean
import Mathlib.RingTheory.Ideal.Quotient.Operations
import Mathlib.RingTheory.Ideal.Quotient.PowTransition

variable (O_L : Type*) [CommRing O_L]
variable (pi : O_L)

def P : Ideal O_L := Ideal.span ({pi} : Set O_L)

def I5 : Ideal O_L := P O_L pi ^ 5

abbrev R5Q : Type _ := O_L ⧸ I5 O_L pi

-- No proof and no finite search:
example : CommRing (R5Q O_L pi) := inferInstance

def reduceQ : O_L →+* R5Q O_L pi :=
  Ideal.Quotient.mk (I5 O_L pi)

example : Function.Surjective (reduceQ O_L pi) :=
  Ideal.Quotient.mk_surjective
```

The exact relevant declarations are:

```text

Ideal.Quotient.mk_eq_mk_iff_sub_mem
```

and prove `I5' = P O_L pi ^ 5` by `Ideal.span_singleton_pow`. Defining the quotient by `I5'` avoids even that equality in the initial construction.

## 1.2 Keeping the current explicit coordinate type: surjective transfer

Mathlib has exactly the transfer constructor needed:

```text
Function.Surjective.commRing
```

in `Mathlib/Algebra/Ring/InjSurj.lean`. Its arguments are a surjective raw function and proofs that it preserves the primitive operations. It then proves all ring laws by lifting elements to the source ring, not by enumerating the target.

Schematically:

```lean
import Mathlib.Algebra.Ring.InjSurj

-- `reduceOL` should be a raw function here, not something whose definition
-- already depends on the CommRing instance being constructed.
def reduceOL : O_L → R5 := ...

theorem reduceOL_surjective : Function.Surjective reduceOL := ...
theorem reduceOL_zero : reduceOL 0 = 0 := ...
theorem reduceOL_one : reduceOL 1 = 1 := ...
theorem reduceOL_add (x y) : reduceOL (x + y) = reduceOL x + reduceOL y := ...
theorem reduceOL_mul (x y) : reduceOL (x * y) = reduceOL x * reduceOL y := ...
theorem reduceOL_neg (x) : reduceOL (-x) = -reduceOL x := ...
theorem reduceOL_sub (x y) : reduceOL (x - y) = reduceOL x - reduceOL y := ...
theorem reduceOL_nsmul (n : Nat) (x) : reduceOL (n • x) = n • reduceOL x := ...
theorem reduceOL_zsmul (n : Int) (x) : reduceOL (n • x) = n • reduceOL x := ...
theorem reduceOL_pow (x) (n : Nat) : reduceOL (x ^ n) = reduceOL x ^ n := ...
theorem reduceOL_natCast (n : Nat) : reduceOL n = n := ...
theorem reduceOL_intCast (n : Int) : reduceOL n = n := ...

instance : CommRing R5 :=
  Function.Surjective.commRing reduceOL reduceOL_surjective
    reduceOL_zero reduceOL_one reduceOL_add reduceOL_mul
    reduceOL_neg reduceOL_sub reduceOL_nsmul reduceOL_zsmul
    reduceOL_pow reduceOL_natCast reduceOL_intCast
```

This is usually the **smallest edit** to an existing coordinate implementation. The only nontrivial algebra is proving `reduceOL_add` and `reduceOL_mul`; these are binary normalization statements. They should be proved by coefficient extensionality and `simp`/`ring_nf` in `ZMod 9` and `ZMod 3`. Even a temporary finite proof of these two binary lemmas is vastly cheaper than checking all ternary ring laws, but an algebraic proof is preferable.

### Important circularity warning

A declaration

```lean
reduceOLHom : O_L →+* R5
```

already requires at least `NonAssocSemiring R5`; in particular, it cannot by itself bootstrap all the laws needed to define its own target structure. There are two legitimate situations:

- If `reduceOLHom` was constructed using an **independent weaker target structure**, and only associativity/commutativity remain, its `map_*` lemmas may be used in the transfer.
- If `reduceOLHom` was constructed only after installing the current `CommRing R5` proved by `decide`, using it to define that same `CommRing` is circular. Extract its underlying raw function and reprove the operation formulas without the final instance.

There is no magic theorem of the form “a surjective `RingHom` creates the target ring structure” that avoids this dependency. The theorem is `Function.Surjective.commRing`, and it consumes a function plus preservation equations.

## 1.3 Transport across an explicit equivalence

If changing the coordinate operations to transported operations is acceptable, prove a plain equivalence

```lean
def coordEquiv : R5 ≃ R5Q O_L pi := ...
```

and use the API in `Mathlib/Algebra/Ring/TransferInstance.lean`:

```lean
noncomputable instance : CommRing R5 := coordEquiv.commRing

def coordRingEquiv : R5 ≃+* R5Q O_L pi :=
  coordEquiv.ringEquiv
```

This route defines all operations on `R5` by transport. It is mathematically clean but may give less convenient reduction behavior than the hand-written coordinate formulas. Do not install this instance alongside conflicting manual `Add`/`Mul` instances.

## 1.4 Identifying the coordinate ring with the quotient

Once both sides are rings, use the first isomorphism theorem. If


then:

```lean
noncomputable def quotientEquivR5 :

```


to avoid classical choice and obtain a computable inverse.

# 2. Cubes in residue characteristic 3

## 2.1 Ordinary Hensel is inapplicable

For

```text
f(X) = X^3 - u,
```

we have `f'(X)=3X^2`. Modulo the maximal ideal above 3, this derivative is zero. Mathlib's declarations

```text
HenselianRing.is_henselian
HenselianLocalRing.is_henselian
```

in `Mathlib/RingTheory/Henselian.lean` explicitly require a **simple** root: the derivative must map to a unit, or be nonzero in the residue field. They cannot be applied here.

The fact that every element of `F_3` satisfies `x^3=x` says only that every residue unit has a residue cube root. It gives no automatic lift through the wild principal-unit filtration.

A concrete counterexample to “every unit is a cube” is `1+pi`: it is a unit, but the exact cube calculation below shows that every unit cube is congruent to `+1` or `-1` modulo `pi^3`, whereas `1+pi` is not.

## 2.2 Simplify the quotient relations first

Inside `R5`, the supplied identities imply

```text
pi^5 = 0,
3*pi^2 = 0,
pi^3 = 3,
pi^4 = 3*pi,
9 = 0.
```

Indeed:

```text
3*pi^2 = pi*pi^4 = pi^5 = 0,
pi^3 = 3 - 3*pi^2 = 3,
9 = (pi^3)^2 = pi^6 = 0.
```

These should be named once and added to the simplifier locally. They make all later polynomial identities small.

Illustrative Lean shapes:

```lean
lemma three_mul_pi_sq : (3 : R5) * pi ^ 2 = 0 := by
  calc
    (3 : R5) * pi ^ 2 = pi * (3 * pi) := by ring
    _ = pi * pi ^ 4 := by rw [pi_pow_four]
    _ = pi ^ 5 := by ring
    _ = 0 := pi_pow_five

lemma pi_cube_eq_three : pi ^ 3 = (3 : R5) := by
  rw [pi_cube_relation, three_mul_pi_sq, sub_zero]

lemma nine_eq_zero : (9 : R5) = 0 := by
  calc
    (9 : R5) = 3 * 3 := by norm_num
    _ = pi ^ 3 * pi ^ 3 := by rw [pi_cube_eq_three]
    _ = pi ^ 6 := by ring
    _ = pi * pi ^ 5 := by ring
    _ = 0 := by rw [pi_pow_five, mul_zero]
```

## 2.3 Exact cube formula

Write a normal form as

```text
x = a + b*pi + c*pi^2,
a,b in Z/9Z, c in Z/3Z.
```

The binomial expansion and the relations above give

```text
x^3 = a^3 + 3*b^3 + 3*a^2*b*pi.
```

All terms involving `c` vanish. For example, `3*a^2*c*pi^2=0`, terms containing `pi^5` vanish, and `6abc*pi^3` is a multiple of 18 and hence zero.

For the coordinate structure, prove one theorem of the following shape:

```lean
theorem cube_coordinates (x : R5) :
    x ^ 3 =
      mk (x.a ^ 3 + 3 * x.b ^ 3)
         (3 * x.a ^ 2 * x.b)
         0 := by
  ext <;> simp [R5.mul_def, pow_succ, pi_cube_eq_three,
    pi_pow_four, pi_pow_five, three_mul_pi_sq, nine_eq_zero] <;>
    ring_nf
```

The exact field names will differ, but the proof should be coefficientwise and should not enumerate elements.

## 2.4 Exact classification of unit cubes

An element is a unit exactly when its constant coefficient has nonzero image in `ZMod 3`. For a unit `a`:

```text
a^2 = 1 mod 3,
a^3 = +1 or -1 mod 9,
b^3 = b mod 3.
```

Hence every unit cube has the form

```text
epsilon + 3*t + 3*t*pi
  = epsilon + t*pi^3 + t*pi^4,
```

where `epsilon` is `+1` or `-1` and `t` is in `F_3`. Conversely, for any lift `t_tilde` of `t`,

```text
(epsilon + t_tilde*pi)^3
  = epsilon + t*pi^3 + t*pi^4.
```

Therefore:

```text
(R5^×)^3
  = { epsilon + t*pi^3 + t*pi^4
      | epsilon in {+1,-1}, t in F_3 }.
```

Consequences:

```text
|R5^×| = (3-1)*3^4 = 162,
|(R5^×)^3| = 2*3 = 6,
|R5^×/(R5^×)^3| = 27.
```

A useful intrinsic formulation is:

- a unit cube is congruent to `+1` or `-1` modulo `pi^3`;
- in the two-dimensional layer `pi^3 R5 / pi^5 R5`, its `pi^3` and `pi^4` coefficients are equal.

Congruence modulo `pi` is thus far too weak.

## 2.5 The correct wild lifting picture

Let `U_i=1+pi^i O` in the completed local DVR and let `e=v_pi(3)=3`. From

```text
(1+y)^3 = 1 + 3y + 3y^2 + y^3
```

one always has

```text
U_i^3 ⊆ U_min(i+e,3i).
```

Here:

```text
U_1^3 ⊆ U_3,
U_2^3 ⊆ U_5.
```

For `i > e/(3-1)=3/2`, so in particular for `i >= 2`, the convergent `3`-adic logarithm and exponential identify `U_i` with `pi^i O`; cubing corresponds to multiplication by 3. Thus in the complete DVR the deep-unit map is

```text
U_i^3 = U_(i+3),  i >= 2.
```

In the truncated quotient `R5`, `U_5=1`, so **every element of `U_2` cubes to 1**. The exceptional map on `U_1/U_2` is exactly the three-element diagonal image computed above.

A standard strong Hensel criterion over a complete DVR can handle multiple residue roots if an approximate root satisfies an inequality such as

```text
v(f(a)) > 2*v(f'(a)).
```

For a unit `a` here, `v(f'(a))=v(3a^2)=3`, so this asks for accuracy beyond valuation 6. Data modulo `pi^5` is insufficient. Moreover, Mathlib `v4.31.0`'s `HenselianRing` API packages the simple-root form, not this valuation-form multiple-root criterion. For the finite quotient, the explicit cube formula is both shorter and stronger.

## 2.6 Lean lemma DAG for cubes

A low-cost dependency order is:

1. `three_mul_pi_sq`, `pi_cube_eq_three`, `nine_eq_zero`.
2. A normal-form extensionality theorem for `R5`.
3. `cube_coordinates`.
4. A residue map
   ```lean
   residue : R5 →+* ZMod 3
   ```
   using
   ```lean
   ZMod.castHom (by norm_num : 3 ∣ 9) (ZMod 3)
   ```
   on the constant coefficient.
5. `isUnit_iff_residue_ne_zero`; prove it either by an explicit inverse or, after installing `IsLocalRing R5`, by
   ```text
   IsLocalRing.notMem_maximalIdeal
   IsLocalRing.mem_maximalIdeal
   ```
6. `unit_cube_normal_form` from `cube_coordinates`.
7. `unit_is_cube_iff` giving the six-element description.

If a finite split is convenient, use `fin_cases` only on `ZMod 3` or on the six units of `ZMod 9`. That is negligible and does not scale as `243^3`.

# 3. Valuation analysis of the homogeneous equation

Let

```text
F = W - 3*U*D + 2*D^3
```

and define the truncated order

```text
nu(0)=5,
nu(x)=min(v_pi(x_lift),5) for x != 0.
```

Write

```text
a = nu(U), b = nu(D), c = nu(W), d = nu(F).
```

Use saturated addition

```text
r ⊕ s = min(5,r+s).
```

Then

```text
nu(3UD) = min(5,3+a+b),
nu(2D^3) = min(5,3b),
nu(U^3) = min(5,3a),
min(5,c+d) = min(5,3a).
```

Also

```text
d >= min(c, min(5,3+a+b), min(5,3b)).
```

If the minimum on the right is attained by exactly one term and is less than 5, equality holds. If two or three terms have the same minimum, valuation alone cannot detect cancellation; one must pass to the corresponding associated-graded layer.

## 3.1 Complete valuation split

### Case A: `a=0` — `U` is a unit

The right side is a unit, so both factors on the left are units:

```text
c=0, d=0.
```

Thus `W` is a unit. Reducing modulo `pi` gives

```text
Wbar * (Wbar + 2*Dbar^3) = Ubar.
```

Since Frobenius is the identity on `F_3`:

- If `b>=1`, then `Dbar=0`, so `Ubar=Wbar^2=1`.
- If `b=0`, all three variables are units and necessarily
  ```text
  Wbar = -Dbar,
  Ubar = -1.
  ```

These are very cheap necessary conditions for the unit branches.

### Case B: `a=1`

Now `nu(U^3)=3`, so

```text
c+d=3.
```

The other term orders are

```text
nu(3UD)=min(5,4+b),
nu(2D^3)=min(5,3b).
```

This forces `b=0`; `D` must be a unit. There are only two valuation patterns:

```text
(c,d)=(3,0),
or
(c,d)=(0,3).
```

- In `(3,0)`, `W` has exact order 3 and `2D^3` supplies the unit part of `F`.
- In `(0,3)`, `W` is a unit and `W+2D^3` must cancel through order 2; the term `-3UD` then contributes at order 4.

These two branches require a residue/graded check. For example, in the second branch the first cancellation condition is

```text
Wbar = Dbar
```

because `Wbar+2Dbar=0` in `F_3`.

No solution with `a=1` and `b>=1` is possible.

### Case C: `a>=2`

Then

```text
U^3=0,
3UD=0
```

in `R5`, and the equation becomes exactly

```text
W*(W+2D^3)=0.
```

The solutions split as follows.

- If `b>=1`, then the equation holds only if
  ```text
  c>=3.
  ```
  For `b>=2`, `D^3=0` and this is simply `W^2=0`. For `b=1`, both `W` and `2D^3` have order at least 3, so their product with `W` vanishes exactly when `c>=3`.

- If `b=0`, then `D` is a unit. The exact alternatives are
  ```text
  W=0,
  or
  W is a unit and W=-2D^3.
  ```
  A nonzero nonunit `W` cannot work because `W+2D^3` is then a unit.

This is already a substantial reduction from 243-cubed possibilities.

## 3.2 Normalize the unit-`D` branch

The equation is weighted homogeneous with

```text
wt(D)=1, wt(U)=2, wt(W)=3.
```

If `D` is a unit, put

```text
x=U/D^2,
y=W/D^3.
```

Dividing by `D^6` gives

```text
y*(y-3*x+2)=x^3.
```

This should be a named equivalence theorem in Lean. It removes one variable and is especially effective in the `a=0` and `a=1` cases, where the valuation analysis often forces `D` to be a unit.

## 3.3 Why valuation alone is not a complete decision procedure

The rules

```text
nu(x+y)=min(nu(x),nu(y))
```

hold only when the two orders are different. Equal-order terms can cancel. The unresolved branches above are precisely equal-order branches. Finish them by mapping to successive quotients

```text
(pi^n)/(pi^(n+1)) ≃ F_3.
```

Mathlib has a useful principal-ideal layer equivalence:

```text
Ideal.quotEquivPowQuotPowSucc
Ideal.quotEquivPowQuotPowSuccEquiv
```

in `Mathlib/RingTheory/Ideal/IsPrincipalPowQuotient.lean`. This is a principled way to turn every cancellation question into one or two `F_3` coefficient equations. Coordinate projection is likely simpler in this concrete file, but the theorem is available.

# 4. A Lean-friendly truncated valuation

Do not try to instantiate Mathlib's DVR valuation directly on `R5`: the quotient has zero divisors and is not a domain. The DVR API applies to the local source ring, not to `R5`.

## 4.1 Source DVR API

If `O_L` is treated as a Dedekind domain and `P=(pi)` is a nonzero prime, localize at `P`:

```lean
abbrev A := Localization.AtPrime P
```

The exact theorem

```text
IsLocalization.AtPrime.isDiscreteValuationRing_of_dedekind_domain
```

in `Mathlib/RingTheory/DedekindDomain/Dvr.lean` supplies the DVR structure. Then use:

```text
IsDiscreteValuationRing.addVal
IsDiscreteValuationRing.addVal_zero
IsDiscreteValuationRing.addVal_one
IsDiscreteValuationRing.addVal_uniformizer
IsDiscreteValuationRing.addVal_mul
IsDiscreteValuationRing.addVal_pow
IsDiscreteValuationRing.addVal_add
IsDiscreteValuationRing.addVal_eq_top_iff
IsDiscreteValuationRing.addVal_eq_zero_iff
```

This is useful for general local-number-theory lemmas. Defining the valuation of a residue class from arbitrary lifts still requires a well-defined truncation argument.

## 4.2 Cheapest concrete definition on the coordinate ring

For the normal form `x=(a,b,c)` with `a,b : ZMod 9`, `c : ZMod 3`, the filtration is:

```text
nu(x)>=1  iff a mod 3 = 0;
nu(x)>=2  iff a mod 3 = 0 and b mod 3 = 0;
nu(x)>=3  iff a mod 3 = 0, b mod 3 = 0, and c=0;
nu(x)>=4  iff a=0, b mod 3 = 0, and c=0;
nu(x)>=5  iff x=0.
```

These follow from

```text
pi R5,
pi^2 R5,
pi^3 R5=3R5,
pi^4 R5=3pi R5,
pi^5 R5=0.
```

A direct executable definition is therefore:

```lean
def piOrd (x : R5) : Nat :=
  if residue9to3 x.a ≠ 0 then 0
  else if residue9to3 x.b ≠ 0 then 1
  else if x.c ≠ 0 then 2
  else if x.a ≠ 0 then 3
  else if x.b ≠ 0 then 4
  else 5
```

Use `Nat`, not `Fin 6`, for the first version; saturated addition and `omega` are easier. Prove:

```lean
piOrd_le_five
piOrd_eq_five_iff
piOrd_eq_zero_iff_isUnit
piOrd_pi
piOrd_two
piOrd_three
piOrd_mul
piOrd_add_ge_min
piOrd_add_eq_min_of_ne
```

where multiplication is stated as

```lean
piOrd (x*y) = min 5 (piOrd x + piOrd y).
```

The clean proof engine is the factorization theorem

```lean
theorem zero_or_unit_mul_pi_pow (x : R5) :
  x=0 ∨ ∃ k<5, ∃ u, IsUnit u ∧ x=pi^k*u
```

proved by the five coordinate cases. It makes the multiplication and unequal-order addition lemmas almost formal.

## 4.3 Quotient-filtration alternative

On the canonical quotient `O_L/P^5`, define “order at least `n`” without choosing lifts:

```lean
def PiAtLeast (n : Nat) (hn : n ≤ 5) (x : O_L ⧸ P^5) : Prop :=
  Ideal.Quotient.factorPow P hn x = 0
```

This says exactly that `x` vanishes modulo `P^n`. The useful APIs are:

```text
Ideal.Quotient.factorPow
Ideal.Quotient.eq_zero_iff_mem
Ideal.pow_add
Ideal.pow_mem_pow
Ideal.mul_mem_mul
Ideal.span_singleton_pow
Ideal.mem_span_singleton
```

This route is more invariant but takes more setup than the six-case coordinate order.

# 5. Recommended implementation order

A practical zero-large-enumeration plan is:

1. **Replace the ring-law instance.**
   - Fastest patch: raw `reduceOL` plus `Function.Surjective.commRing`.
   - Best long-term design: canonical quotient `R5Q` plus a ring equivalence to coordinates.

2. **Prove the four simplification relations**
   ```text
   pi^5=0, 3pi^2=0, pi^3=3, 9=0.
   ```

3. **Prove one coefficientwise cube formula.**
   Derive the six unit cubes and the 27 cube classes. Replace any 243-element cube search with the theorem `unit_is_cube_iff`.

4. **Define `piOrd` by the six coordinate branches.**
   Prove the saturated product law and unequal-order sum law.

5. **Prove the master valuation theorem for the equation**, then the three `a=0`, `a=1`, `a>=2` case lemmas.

6. **Normalize when `D` is a unit** to `y(y-3x+2)=x^3`.

7. **Resolve cancellation branches in `F_3` layers.**
   Tiny `fin_cases` proofs over `ZMod 3` are appropriate here. They are transparent and essentially free.

# 6. False shortcuts to avoid

- **Do not claim every unit is a cube because every residue unit is a cube.** The derivative vanishes and there are only six unit cubes in `R5`.
- **Do not use `HenselianLocalRing.is_henselian` for `X^3-u`.** Its simple-root hypothesis fails.
- **Do not use a `RingHom` to create the very target ring structure on which that `RingHom` depends.** Use a raw function, a canonical quotient, or a transported equivalence.
- **Do not apply `IsDiscreteValuationRing.addVal` to `R5`.** It is a non-domain Artinian local ring. Use the source DVR or a truncated filtration order.
- **Do not expect valuation alone to decide equal-order cancellation.** Pass to `P^n/P^(n+1)` or coordinate residues.
- **Do not enumerate ternary laws.** At worst, verify binary reducer compatibility and transfer the laws; the preferred proof is symbolic.
