## Answer to the strategic question

Do **not** try to produce the weak `BFormPositiveLocalFrontier p` merely to close `Theorem_1_1`. That frontier is a stronger PID local-existence surface than the paper headline needs.

Do continue on the PPID-typed route, but the clean target is not `ChiNegDatumUniformConstructionPPID p` either. The shortest committed route bypasses that interface and targets:

```lean
ConeQuantBridge.PicardLimitRestartFrontier p
```

or whatever currently feeds that frontier.

So the honest shortest next proof target is:

```lean
ConeQuantBridge.PicardLimitRestartFrontier p
```

If you want a stricter route-facing split where the zero branch stays unconditional and the Picard-limit frontier is only required on the negative branch, add the small wrapper below. Otherwise no wrapper is needed; `theorem_1_1_intervalDomain_of_ppid_picardLimitFrontier_chiNonpos` already gives the mathematical reduction.

## B-form frontier audit

### Existing weak-positive producer

There is a safe weak-positive producer:

```lean
theorem BFormPositiveDatumLocal.bFormPositiveLocalFrontier_of_localHyp
    {p : CM2Params}
    (hBForm : PositiveDatumBFormLocalHyp p) :
    BFormPositiveLocalFrontier p
```

where:

```lean
def PositiveDatumBFormLocalHyp (p : CM2Params) : Prop :=
  ∀ u₀ : intervalDomainPoint → ℝ,
    PositiveInitialDatum intervalDomain u₀ →
      Nonempty (PositiveDatumBFormLocalComponents p u₀)
```

This is safe: it does not convert `PositiveInitialDatum` to `PaperPositiveInitialDatum`. But it is still a real weak-positive analytic/plumbing input, not a discharged producer.

### Direct/noHvpos/squared/regular/deepest B-form constructors

These are paper-positive/closed-floor routes.

`BFormBankedInputs` has:

```lean
huPaper : PaperPositiveInitialDatum intervalDomain u₀
hsmall : ... ≤ paperPositiveFloor huPaper / 2
```

`BFormDirectClassicalNoHvpos.lean` defines:

```lean
def BFormPaperLocalFrontierNoHvpos (p : CM2Params) : Prop :=
  ∀ u₀ : intervalDomainPoint → ℝ,
    PaperPositiveInitialDatum intervalDomain u₀ →
      ∃ DB : ConjugateMildExistenceData p u₀,
        Nonempty (BFormDirectFrontierNoHvpos p DB)
```

`PositiveDatumBFormLocalComponentsSq`, `...SqRegular`, and `...SqDeepest` all rely on `PaperPositiveInitialDatum` or on `BFormBankedInputs`, hence on `paperPositiveFloor`. They produce paper-positive frontiers such as:

```lean
BFormPaperPositiveLocalFrontier p
```

not the weak-positive:

```lean
BFormPositiveLocalFrontier p
```

Therefore they cannot safely be adapted to the weak frontier by a wrapper. A real weak-positive construction would need the negative-part route for boundary-vanishing data, not the closed-floor squared-barrier route.

## Tempting but invalid bridges to avoid

Avoid:

```lean
PositiveInitialDatum intervalDomain u₀ -> PaperPositiveInitialDatum intervalDomain u₀
```

This is false: `PositiveInitialDatum` only requires positivity on the open interior and can allow zero at the boundary.

Avoid using all-PPID common-floor/Wiener assumptions. The repo already proves obstruction theorems:

```lean
ShenWork.EWA.not_datumWienerData
ShenWork.EWA.not_uniformFlooredC3NeumannData
```

Avoid trying to fill `CoupledFluxClassicalLocalExistenceResidual p` from paper-positive local existence alone. That residual quantifies over all weak-positive bounded data, so PPID-only factories are insufficient.

Avoid treating `ChiNegDatumUniformConstructionPPID p` as the only PPID route. It is a valid interface, but `IntervalDomainPPIDPicardLimitFrontier.lean` already provides a shorter route from `PicardLimitRestartFrontier p` directly to `Theorem_1_1`.
