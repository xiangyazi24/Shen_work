# Q3968: non-circular M-ball bound for the resolver second derivative

## Verdict

The proposed identity route is mathematically correct and is the right way to avoid the circularity:

```text
-R'' + μR = ρ(u)  ⇒  R'' = μR - ρ(u)
```

so, under the weak ball hypotheses,

```text
|R''(x)| ≤ μ |R(x)| + |ρ(u(x))|
        ≤ μ V_M + ν M^γ.
```

The important Lean distinction is that the repo currently has two different layers:

1. **Weak value/gradient layer** in `IntervalResolverWeakBounds.lean`: from continuity plus `0 ≤ lift u ≤ M`, it proves value and gradient bounds without classical solution regularity.
2. **Spectral Laplacian / second derivative layer** in `IntervalResolverLaplacianBridge.lean` and `IntervalDomainL2UEnergyCombine.lean`: its strong identity and boundedness currently require `SourceCoeffQuadraticDecay p u`.

Therefore, the non-circular target should first be stated for a new **physical Laplacian** expression

```lean
p.μ * intervalNeumannResolverR p u y - ρ(u)(y)
```

rather than directly for the existing termwise second-derivative series `resolverGrad2Real` / `intervalNeumannResolverRLap`. Then add a separate Green/ODE regularity theorem identifying this physical Laplacian with the classical second derivative when the source is continuous.

## Direct answers

### 1. Is the approach correct?

Yes. For a classical Neumann solution of `-R'' + μR = ρ(u)`, the identity `R'' = μR - ρ(u)` gives exactly

```text
H_M = μ V_M + ν M^γ.
```

This uses no bound on `u'`. It uses only the value bound for `R` and the pointwise source bound.

One repo-specific detail: the current spectral resolver source coefficients are built from

```lean
p.ν * intervalDomainLift u x ^ p.γ
```

whereas your cron2/truncated formulation writes `ρ(u) = ν * (positivePart u)^γ`. Under the weak nonnegative ball hypothesis `0 ≤ intervalDomainLift u x`, the two are equal by `positivePart_eq_self_of_nonneg`. If you want a resolver that accepts sign-changing inputs, introduce a separate truncated resolver source; otherwise keep the existing nonnegative ball hypotheses.

### 2. Do you need uniform convergence of the `R''` cosine series?

Not for the physical identity route.

You only need uniform or absolute convergence of the `R''` series if you try to justify the identity by termwise differentiating

```text
R(y) = Σ c_k cos(kπy).
```

That is exactly the route that becomes circular: with only continuous bounded source, source coefficients are generally `ℓ²`, not `ℓ¹`; multiplying by `(kπ)^2/(μ+(kπ)^2)` does not improve summability. The repo’s `intervalNeumannResolverRLap_elliptic_identity` therefore assumes `SourceCoeffQuadraticDecay p u`, because that gives enough `ℓ¹` summability for the Laplacian-mode series.

For the non-circular weak route, prove the ODE/Green theorem directly: continuous source implies the Green convolution resolver is `C²` and satisfies the ODE pointwise. That proof should not mention the `R''` Fourier series.

### 3. Is there a non-circular way?

Yes. Add a weak physical-Laplacian layer:

```lean
def resolverLapPhysical (p : CM2Params) (u : intervalDomainPoint → ℝ)
    (y : intervalDomainPoint) : ℝ :=
  p.μ * intervalNeumannResolverR p u y -
    p.ν * (positivePart (intervalDomainLift u y.1)) ^ p.γ
```

Then prove

```lean
|resolverLapPhysical p u y| ≤ μ * V_M + ν * M^γ
```

from `resolverValue_sup_le_of_bounded` and `0 ≤ lift u ≤ M`.

Only after that add a bridge theorem, proved by Green/ODE regularity:

```lean
HasDerivAt (fun x => resolverGradReal p u x)
  (resolverLapPhysical p u ⟨x, hx⟩) x
```

or equivalently

```lean
deriv (fun x => resolverGradReal p u x) x = resolverLapPhysical p u ⟨x, hx⟩.
```

This bridge uses continuity of the source, not `u'`, not BV, and not `SourceCoeffQuadraticDecay`.

### 4. Does the repo already have C² regularity from continuity of `u`?

No, not as a committed theorem. The audit file `ShenWork/Paper2/IntervalEllipticResolverC2Audit.lean` explicitly records that the available spatial `C²` theorem is `IntervalResolverSpatialC2.resolverR_contDiff_two`, and that it requires `SourceCoeffQuadraticDecay p u`. The weak file `IntervalResolverWeakBounds.lean` gives source `ℓ²`, absolute convergence of value/gradient series, and weak value/gradient sup bounds, but not the weak-source `C²` Green bridge.

## Immediate Lean formalization: weak physical Laplacian bound

I would add something like the following in a new file, for example
`ShenWork/Paper2/IntervalResolverWeakLapBound.lean`, importing `IntervalResolverWeakBounds`. The theorem is intentionally phrased for `resolverLapPhysical`, not for the spectral `resolverGrad2Real`.

```lean
import ShenWork.Paper2.IntervalResolverWeakBounds
import ShenWork.Paper2.Statements
import ShenWork.PDE.IntervalNeumannEllipticResolverR

open MeasureTheory
open ShenWork.IntervalDomain
open ShenWork.PDE
open ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

/-- Truncated pointwise elliptic source used by the cron2 route.
On a nonnegative ball this agrees with the repo's current untruncated source. -/
def resolverPositiveSourceLifted (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun x => p.ν * (positivePart (intervalDomainLift u x)) ^ p.γ

/-- Physical second derivative / Laplacian expression from the elliptic equation.
This is the non-circular object to bound from the M-ball. -/
def resolverLapPhysical (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : intervalDomainPoint → ℝ :=
  fun y => p.μ * intervalNeumannResolverR p u y -
    resolverPositiveSourceLifted p u y.1

/-- Existing weak resolver value bound packaged as a constant. -/
def resolverWeakValueBound (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ, (intervalNeumannResolverWeight p k) ^ 2) *
    (2 * (p.ν * M ^ p.γ))

/-- M-ball bound for the physical second derivative. -/
def resolverWeakLapBound (p : CM2Params) (M : ℝ) : ℝ :=
  p.μ * resolverWeakValueBound p M + p.ν * M ^ p.γ

/-- Pointwise bound for the truncated source from `0 ≤ u ≤ M`. -/
theorem resolverPositiveSourceLifted_abs_le_of_bounded
    (p : CM2Params) {u : intervalDomainPoint → ℝ} {M x : ℝ}
    (hlb : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u y)
    (hub : ∀ y ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift u y ≤ M)
    (hx : x ∈ Set.Icc (0 : ℝ) 1) :
    |resolverPositiveSourceLifted p u x| ≤ p.ν * M ^ p.γ := by
  have hux_nonneg : 0 ≤ intervalDomainLift u x := hlb x hx
  have hux_le : intervalDomainLift u x ≤ M := hub x hx
  have hpp_eq : positivePart (intervalDomainLift u x) = intervalDomainLift u x :=
    positivePart_eq_self_of_nonneg hux_nonneg
  have hpp_nonneg : 0 ≤ positivePart (intervalDomainLift u x) := positivePart_nonneg _
  have hpp_le : positivePart (intervalDomainLift u x) ≤ M := by
    simpa [hpp_eq] using hux_le
  have hpow_le :
      (positivePart (intervalDomainLift u x)) ^ p.γ ≤ M ^ p.γ := by
    exact Real.rpow_le_rpow hpp_nonneg hpp_le p.hγ.le
  have hsrc_nonneg :
      0 ≤ p.ν * (positivePart (intervalDomainLift u x)) ^ p.γ :=
    mul_nonneg p.hν.le (Real.rpow_nonneg hpp_nonneg _)
  calc
    |resolverPositiveSourceLifted p u x|
        = p.ν * (positivePart (intervalDomainLift u x)) ^ p.γ := by
          simp [resolverPositiveSourceLifted, abs_of_nonneg hsrc_nonneg]
    _ ≤ p.ν * M ^ p.γ :=
          mul_le_mul_of_nonneg_left hpow_le p.hν.le

/-- Non-circular M-ball bound for the physical elliptic second derivative. -/
theorem resolverLapPhysical_abs_le_of_bounded
    (p : CM2Params) {u : intervalDomainPoint → ℝ} {M : ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hlb : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u y)
    (hub : ∀ y ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift u y ≤ M)
    (y : intervalDomainPoint) :
    |resolverLapPhysical p u y| ≤ resolverWeakLapBound p M := by
  have hR : |intervalNeumannResolverR p u y| ≤ resolverWeakValueBound p M := by
    simpa [resolverWeakValueBound] using
      resolverValue_sup_le_of_bounded p hUcont hlb hub y
  have hsrc : |resolverPositiveSourceLifted p u y.1| ≤ p.ν * M ^ p.γ :=
    resolverPositiveSourceLifted_abs_le_of_bounded p hlb hub y.2
  calc
    |resolverLapPhysical p u y|
        = |p.μ * intervalNeumannResolverR p u y -
            resolverPositiveSourceLifted p u y.1| := by
          rfl
    _ ≤ |p.μ * intervalNeumannResolverR p u y| +
          |resolverPositiveSourceLifted p u y.1| := abs_sub _ _
    _ = p.μ * |intervalNeumannResolverR p u y| +
          |resolverPositiveSourceLifted p u y.1| := by
          rw [abs_mul, abs_of_pos p.hμ]
    _ ≤ p.μ * resolverWeakValueBound p M + p.ν * M ^ p.γ := by
          exact add_le_add
            (mul_le_mul_of_nonneg_left hR p.hμ.le)
            hsrc
    _ = resolverWeakLapBound p M := by
          rfl

/-- Packaged existential form if downstream code prefers `∃ H`. -/
theorem exists_resolverLapPhysical_bound_of_bounded
    (p : CM2Params) {u : intervalDomainPoint → ℝ} {M : ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hlb : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u y)
    (hub : ∀ y ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift u y ≤ M) :
    ∃ H : ℝ, 0 ≤ H ∧ ∀ y : intervalDomainPoint,
      |resolverLapPhysical p u y| ≤ H := by
  refine ⟨resolverWeakLapBound p M, ?_, ?_⟩
  · have h0 : (0 : ℝ) ∈ Set.Icc (0 : ℝ) 1 := by norm_num
    have hMnn : 0 ≤ M := (hlb 0 h0).trans (hub 0 h0)
    have hMpow : 0 ≤ M ^ p.γ := Real.rpow_nonneg hMnn _
    have hsrcB : 0 ≤ p.ν * M ^ p.γ := mul_nonneg p.hν.le hMpow
    have hV : 0 ≤ resolverWeakValueBound p M := by
      unfold resolverWeakValueBound
      exact mul_nonneg (Real.sqrt_nonneg _)
        (mul_nonneg (by norm_num) hsrcB)
    unfold resolverWeakLapBound
    exact add_nonneg (mul_nonneg p.hμ.le hV) hsrcB
  · intro y
    exact resolverLapPhysical_abs_le_of_bounded p hUcont hlb hub y

end ShenWork.IntervalResolverWeakBounds
```

## New analytic bridge needed for actual `R''`

The previous theorem bounds the RHS of the elliptic equation. To use it as a bound on the derivative of `resolverGradReal`, add a Green/ODE regularity theorem. This is the genuine missing infrastructure.

```lean
import ShenWork.Paper2.IntervalResolverWeakLapBound

open ShenWork.IntervalDomain
open ShenWork.PDE
open ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

/-- Missing Green/ODE bridge.

This should be proved from the one-dimensional Neumann Green kernel or from a
weak-to-classical ODE theorem for continuous right-hand side. It should not be
proved by termwise differentiating the `R''` Fourier series. -/
theorem resolverGradReal_hasDerivAt_physicalLap_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    HasDerivAt (fun z : ℝ => resolverGradReal p u z)
      (resolverLapPhysical p u ⟨x, Set.Ioo_subset_Icc_self hx⟩) x := by
  -- Prove by Green kernel / ODE regularity:
  -- 1. Show `ρ(u)` is continuous on `[0,1]`.
  -- 2. Show the concrete Neumann resolver equals the Green convolution of `ρ(u)`.
  -- 3. Differentiate the Green representation, or invoke the 1D weak-to-classical theorem.
  -- 4. Conclude `R'' x = μR x - ρ(u x)` for `x ∈ (0,1)`.
  sorry

/-- Once the Green/ODE bridge exists, the physical M-ball bound becomes a bound on
`deriv resolverGradReal`. -/
theorem deriv_resolverGradReal_abs_le_of_bounded
    (p : CM2Params) {u : intervalDomainPoint → ℝ} {M : ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hlb : ∀ y ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u y)
    (hub : ∀ y ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift u y ≤ M)
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    |deriv (fun z : ℝ => resolverGradReal p u z) x| ≤ resolverWeakLapBound p M := by
  have hder := resolverGradReal_hasDerivAt_physicalLap_of_continuousOn p hUcont hx
  rw [hder.deriv]
  exact resolverLapPhysical_abs_le_of_bounded p hUcont hlb hub
    ⟨x, Set.Ioo_subset_Icc_self hx⟩

end ShenWork.IntervalResolverWeakBounds
```

Endpoint versions can be added later if needed, but for most chemotaxis flux differentiation arguments on `(0,1)`, the open-interval statement is the useful one. Bounds on the closed interval can be obtained either by continuity after the Green theorem or by using `resolverLapPhysical_abs_le_of_bounded` directly at subtype points.

## Why this avoids circularity

The proof of `resolverLapPhysical_abs_le_of_bounded` consumes only:

```lean
ContinuousOn (intervalDomainLift u) (Icc 0 1)
∀ x ∈ Icc 0 1, 0 ≤ intervalDomainLift u x
∀ x ∈ Icc 0 1, intervalDomainLift u x ≤ M
resolverValue_sup_le_of_bounded
```

It does not consume:

```lean
truncatedPicardLimit_gradient_bound_positive_time
resolverGrad2Real_bounded_of_sourceDecay
SourceCoeffQuadraticDecay p u
source BV / source ℓ¹
```

The Green/ODE bridge also does not require `u'`; it uses only continuity of `ρ(u)`. Thus the second-derivative bound becomes available before the parabolic gradient bound `G`, and the gradient-bound proof can safely use it without a loop.

Use these names to keep the layers clear:
