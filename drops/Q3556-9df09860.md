## Verdict

The mixed-representative files are **not a usable producer** for the full target package.

They give a **partial non-circular bridge** only for the first field `time_cont0`, and only if one keeps the raw witness data `ChemDivMixedReprWitnessData` (or its construction inputs) rather than only the capstone `ChemDivMixedTimeDerivClosedRepr`. This bridge is overpowered and construction-specific: it proves closed-slab continuity of `slopeSlice u`, which is definitionally `liftTimeDeriv u`, from spectral iterate representatives.

They do **not** produce `eq0Interior`. Using them to claim the initial `liftDeriv2 u 0 = liftDeriv2PhysicalRHSWithChemRep ... 0` PDE trace would be a false route. The mixed layer is about the time derivative of the chemotaxis-divergence source, not the initial-time `u`-equation/laplacian trace.

## Source-grounded audit

### 1. What the target bridge at `9c200158` already does

`IntervalChiNegH1ZeroStartComponents.lean` defines the target package and the bridge:


Its proof copies `hinit.time_cont0`, then splits `eqInterior0` on `t = 0`. At `t = 0` it uses `hinit.eq0Interior`; at `0 < t` it uses:

So positive time is already closed. The only remaining question is whether the mixed-repr layer can provide `hinit.time_cont0` or `hinit.eq0Interior`.

### 2. `IntervalChemDivTimeDerivClosed.lean`: wrong function for the target

This file defines:


and:

This is closed-slab continuity of `coupledChemDivTimeDerivativeLift p u`, i.e. the spatial derivative of the flux time-derivative. It is not continuity of

```lean

```

and it says nothing about `liftDeriv2 u 0`. Therefore using only `ChemDivMixedTimeDerivClosedRepr` or `chemDivMixedTimeDeriv_jointContinuousOn_closed` for `H1ZeroStartPhysicalPDEInitialCompatibilityBefore` is a false route.

### 4. `IntervalChemDivMixedReprWitness.lean`: partial bridge for `time_cont0`

This file is the only place where a partial bridge exists. It defines:

```lean

    (p : CM2Params) (u : ℝ → intervalDomainPoint → ℝ) (τ δ : ℝ)
```

with continuous representatives, including:

```lean

Utc_eq : ∀ t ∈ Icc (τ - δ) (τ + δ), ∀ x ∈ Icc (0 : ℝ) 1,
  Utc (t, x) = ShenWork.Paper2.PicardLimitK1.slopeSlice u t x
```

`PicardLimitK1.slopeSlice` is defined in `IntervalPicardLimitK1.lean` as:


which is the same expression as `liftTimeDeriv u σ x` in `IntervalChiNegH1EnergyIdentity.lean`.

Therefore, if for each `0 ≤ b < T` one has a witness whose time slab covers `[0,b]`, then one can prove `time_cont0` by restricting `cont_Utc` and rewriting by `Utc_eq`.

### 5. No bridge for `eq0Interior`

None of the mixed-repr files proves or even states the required initial-time `u`-equation trace:

```lean
liftDeriv2 u 0 x =
  liftDeriv2PhysicalRHSWithChemRep p u
    (liftChemotaxisDivPhysicalRep p u v) 0 x
```

The mixed files are about:

* `coupledChemDivTimeDerivativeLift p u`, the time derivative of the chemotaxis-divergence source;
* the coupled chemical concentration `coupledChemicalConcentration p u`, not an arbitrary `v` unless an external identification is supplied;
* spectral representatives for `U`, `∂ₜU`, `∂ₓ∂ₜU`, `v`, `∂ₜv`, and their spatial derivatives.

They do not give the initial PDE relation

```lean
u_xx(0) = u_t(0) + χ₀ * chemDiv(0) - reaction(0)
```

and they do not relate an arbitrary `v` in the H¹ physical route to `coupledChemicalConcentration p u` at `t = 0`.

Thus, for `eq0Interior`, the mixed-repr route is not merely incomplete; it is targeting a different analytic object.

## Smallest honest package connecting mixed-repr to the target

A useful package could extract **only `time_cont0`** from raw mixed witness data. It should be stated as partial, not as a producer of the full PDE compatibility record.

A safe shape is:

```lean

open ShenWork.IntervalDomain

/-- Raw mixed witnesses covering every zero-start slab.  This is stronger than
`time_cont0`; it is useful only if the construction already has the spectral
witnesses. -/
structure H1ZeroStartTimeDerivMixedWitnessBefore
    (p : CM2Params) (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  witness : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ∃ τ δ : ℝ,
      Set.Icc (0 : ℝ) b ⊆ Set.Icc (τ - δ) (τ + δ) ∧
      ChemDivMixedReprWitnessData p u τ δ

/-- The raw mixed witness package implies the `time_cont0` half of the H¹
zero-start PDE compatibility record, by the `Utc` representative. -/
theorem time_cont0_of_mixedWitnessBefore
    {p : CM2Params} {u : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (H : H1ZeroStartTimeDerivMixedWitnessBefore p u T) :
    ∀ {b : ℝ}, 0 ≤ b → b < T →
      ContinuousOn
        (Function.uncurry (fun t x => liftTimeDeriv u t x))
        (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) := by
  intro b hb0 hbT
  rcases H.witness (b := b) hb0 hbT with ⟨τ, δ, hcover, W⟩
  refine W.cont_Utc.continuousOn.congr ?_
  intro z hz
  rcases z with ⟨t, x⟩
  rcases hz with ⟨ht, hx⟩
  have ht' : t ∈ Set.Icc (τ - δ) (τ + δ) := hcover ht
  simpa [Function.uncurry, liftTimeDeriv, ShenWork.Paper2.PicardLimitK1.slopeSlice]
    using W.Utc_eq t ht' x hx

end ShenWork.IntervalChemDivMixedReprWitness
```

This bridge is non-circular with respect to H¹ bounded-before and sqrt bounds, but it is not better than the current target record for the H¹ route:

* it only gives `time_cont0`, not `eq0Interior`;
* it is much stronger than `time_cont0` because it requires the full mixed chem-div spectral witness bundle;
* it is construction-specific and tied to `coupledChemicalConcentration p u`, while the H¹ target record is stated for arbitrary `v`.

If one insists on a full mixed-facing package, the honest version would still need to carry the missing PDE trace explicitly:

## False/circular routes to flag

* **False route:** using `ChemDivMixedTimeDerivClosedRepr` as `time_cont0`. It proves continuity of `coupledChemDivTimeDerivativeLift`, not `liftTimeDeriv u`.
* **False route:** using `chemDivMixedTimeDeriv_jointContinuousOn_closed` to claim the initial PDE trace. It never mentions `liftDeriv2 u 0`.
* **False route:** using `chemDivMixedClosedRepr_of_iterateGradSummable` as a full seam producer. Its conclusion is only the chem-div mixed derivative representative.
* **Potential type/semantic mismatch:** the mixed layer uses `coupledChemicalConcentration p u`; the H¹ target is for an arbitrary `v`. Any bridge involving the chem representative would need an explicit equality identifying `v` with that coupled concentration, especially at `t = 0`.
* **Circular route:** any use of H¹ bounded-before, `H1SupBoundSqrtRHSIntegrableBefore`, sqrt-bound outputs, or zero-start physical RHS outputs to prove this initial compatibility. None of those should be used.

## Final recommendation

Do not replace `H1ZeroStartPhysicalPDEInitialCompatibilityBefore` with a mixed-repr package.

The mixed-repr layer can provide a partial non-circular bridge for `time_cont0` only if raw `ChemDivMixedReprWitnessData` is available on slabs covering `[0,b]`. It cannot provide `eq0Interior`, which remains the real initial-time PDE compatibility obligation.

The smallest honest source-facing package for the full target remains the current record:

```lean
H1ZeroStartPhysicalPDEInitialCompatibilityBefore p u v T
```

A mixed-facing auxiliary is worthwhile only as a convenience theorem for the first field, with the name and statement making clear that it proves `time_cont0` and not the zero-time PDE trace.
