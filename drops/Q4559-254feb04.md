# Q4559 — Paper 2 Lᵖ mountain, exact current gap for the interval `m = 1` critical branch

## Executive verdict

The current repo is **much greener** than the old “all finite-`p` energy identity is open” picture.

For the interval-domain Lᵖ energy layer, the following are already proved and reusable:

- time Leibniz / differentiation under the interval integral;
- PDE-under-integral for the weighted test;
- Neumann spatial IBP;
- diffusion coercivity into the weighted-gradient dissipation;
- chemotaxis integration-by-parts cross-control;
- logistic upper bound;
- lower-order comparison;
- the assembler from `CrossDiffusionBootstrapEstimate` + bootstrap seed to `LpBootstrapEnergyInequality`.

The current **m = 1 critical branch** gap is not those atoms. It is the branch-level producer demanded by `IntervalDomainTheorem12.lean`:

```lean
hcriticalBootstrap :
  1 ≤ p.β → p.χ₀ < chiBeta p →
  ∀ u₀, PositiveInitialDatum intervalDomain u₀ →
  ∀ T > 0, ∀ u v,
    IsPaper2ClassicalSolution intervalDomain p T u v →
```

and then the long-time boundedness producer:

```lean

```

The generic cross-diffusion half of `hcriticalBootstrap` is already available as `rho = 2 * p.γ`, but the compatible initial/seed `LpPowerBoundedBefore` is not automatically produced in the critical theorem API. If the intended paper proof uses the sharper `m = 1`, `χ₀ < chiBeta p` absorption to get a low-order seed or an effectively smaller remainder exponent, that sharper branch-specific bootstrap producer is still the real mathematical gap.

### Still not assembled at branch level

To inhabit `hcriticalBootstrap` for `m = 1`, the remaining branch-level work is:

1. choose a `rho > 0`;
2. prove `CrossDiffusionBootstrapEstimate intervalDomain p T rho u v`;
3. prove a compatible seed `∃ p0 > max 1 (rho * N / 2), LpPowerBoundedBefore intervalDomain p0 T u`.

Item 2 is already available with `rho = 2γ`.

Item 3 is the hard part if using the generic `rho = 2γ`. Choosing `p0 = 2` only works if:

```lean

```

The current `chiBeta` assumption alone does not obviously imply that. In fact `IntervalDomainAPrioriGlobal.lean` explicitly records a stronger boundedness hypothesis including `p.γ * (p.N : ℝ) < 2`, showing the repo treats this as an additional exponent restriction, not a consequence of `χ₀ < chiBeta p`.

Therefore, for the paper-faithful `m = 1`, `χ₀ < chiBeta p` branch, the true missing theorem is a **critical branch bootstrap producer**, not another restatement of the time/IBP energy identity.

## 3. Does the m = 1 cross term need an elliptic `∇v` bound?

Yes, in the current repo route the cross-term estimate still uses an elliptic resolver gradient bound, but it is already internalized.

The theorem:

```lean
intervalDomain_crossDiffusionEnergyTerm_young_sourceL2
```

sets

```lean
G := Real.sqrt (∑' k, (intervalNeumannResolverGradWeight p k)^2) *
       (2 * Real.sqrt Ssrc)
```

where

```lean
Ssrc := ∫₀¹ ‖p.ν * u^γ‖².
```

Then pointwise Young gives:

```lean
crossDiffusionEnergyTerm ≤ eps * weightedGradient + (G^2/(4 eps)) * ∫ u^p.
```

The product `Ssrc * ∫u^p` is then bounded by the higher moment:

```lean
p.ν^2 * ∫ u^(p + 2γ)
```

via:

```lean
intervalDomain_sourceL2_mul_power_le
```

and that produces the final generic estimate with `rho = 2γ`.

The denominator is handled by `v ≥ 0`, so `1 + v ≥ 1`; the code uses positivity of `(1 + lv y)^β` and the denominator form in the pointwise Young inequality. There is no separate external `‖v_x‖∞` hypothesis in the final theorem.

There is also a quantitative resolver sup/gradient file:

```lean
IntervalDomainResolverSupQuantitative.lean
```

with explicit bounds of the form

```lean
|R[u]|   ≤ Fv(M)
|R_x[u]| ≤ Fg(M)
```

assuming a prior uniform pointwise bound `0 ≤ u ≤ M`. That is useful after a sup bound is already known, but it is **not** the right non-circular tool for the Lᵖ bootstrap seed, because the L∞/boundedness conclusion is what Theorem 1.2 is trying to reach.

So the current non-circular Lᵖ path is the source-L² resolver gradient estimate above, not a preexisting sup bound on `v_x`.

## 4. Single most load-bearing open lemma to attack first

The best first attack target is not a new generic energy-identity lemma. It is the critical-branch bootstrap producer.

Paste-ready target shape:

```lean

    ∀ T > 0, ∀ u v : ℝ → intervalDomain.Point → ℝ,

        ∃ rho > 0,
          CrossDiffusionBootstrapEstimate intervalDomain p T rho u v ∧
            ∃ p0 > max 1 (rho * (p.N : ℝ) / 2),
              LpPowerBoundedBefore intervalDomain p0 T u := by
  -- choose branch-specific rho and p0
  -- supply CrossDiffusionBootstrapEstimate
  -- supply LpPowerBoundedBefore seed
```

There are two possible proof strategies.

### Strategy A: use the already-proved generic cross estimate

Set:

```lean
rho := 2 * p.γ
```

and use:

```lean
intervalDomain_crossDiffusionBootstrapEstimate_of_classical hsol
```

Then the entire problem becomes the seed:

```lean
∃ p0 > max 1 (p.γ * (p.N : ℝ)),
  LpPowerBoundedBefore intervalDomain p0 T u
```

This is only feasible if another estimate gives an initial high-`p0` bound. The existing L²-seed route only gives `p0 = 2`, which requires `p.γ * N < 2`. That condition is recorded in `IntervalDomainBoundednessHyp`; it is not implied by `χ₀ < chiBeta p` in the theorem statement.

So Strategy A is quick only under an additional exponent guard.

### Strategy B: prove the sharper critical smallness estimate

This is the paper-faithful route if Theorem 1.2 is supposed to close under only:

```lean
p.m = 1
1 ≤ p.β
p.χ₀ < chiBeta p
```

The new lemma must use the `χ₀ < chiBeta p` smallness to produce a sharper finite-`p` branch estimate, effectively reducing the cross-diffusion remainder so that a low seed such as `p0 = 2` is enough. This is the actual “weighted finite-p critical estimate” from Q4409.

A useful internal target is:

```lean
theorem intervalDomain_critical_LpPowerBoundedBefore_seed
    (p : CM2Params) (hm_eq : p.m = 1)
    (hβ : 1 ≤ p.β) (hχ : p.χ₀ < chiBeta p)
    {u₀ : intervalDomain.Point → ℝ}
    (hu₀ : PositiveInitialDatum intervalDomain u₀)
    {T : ℝ} (hT : 0 < T)
    {u v : ℝ → intervalDomain.Point → ℝ}
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u v)
    (htrace : InitialTrace intervalDomain u₀ u) :
    ∃ rho > 0, ∃ p0 > max 1 (rho * (p.N : ℝ) / 2),
      CrossDiffusionBootstrapEstimate intervalDomain p T rho u v ∧
      LpPowerBoundedBefore intervalDomain p0 T u := by
  -- use sharp m=1/χ<χβ estimate to pick a compatible rho and p0
```

Proof skeleton for Strategy B:

1. Start from the already-proved energy identity atoms:
   - `intervalDomain_lp_energy_hLpTime_frontier`
   - `intervalDomain_lp_energy_hPDEIntegral_of_regularity`
   - `intervalDomain_lp_energy_hIBP_of_regularity`
   - `intervalDomain_lp_energy_hDiffusionCoercive_of_regularity`
   - `intervalDomain_lp_energy_hCrossControl_of_regularity`
2. Replace the generic `rho = 2γ` cross estimate by the sharper critical estimate that uses `m = 1`, `β ≥ 1`, and `χ₀ < chiBeta p`.
3. Obtain a closed differential inequality for a low exponent, preferably `p0 = 2`:
   ```lean
   deriv (fun τ => intervalDomain.integral (fun x => (u τ x)^2)) t
     ≤ C * intervalDomain.integral (fun x => (u t x)^2) + C'
   ```
   or the stronger dissipative variant.
4. Use initial trace + Gronwall to get:
   ```lean
   LpPowerBoundedBefore intervalDomain 2 T u
   ```
5. Choose `rho` so that:
   ```lean
   max 1 (rho * (p.N : ℝ) / 2) < 2
   ```
6. Package the result as `hcriticalBootstrap`.

### Why this is the right first lemma

Once `hcriticalBootstrap` is in hand, the already-proved `boundedBefore_of_corollary21_and_proposition25` turns Corollary 2.1 + Proposition 2.5 into finite-horizon boundedness. Then `hglobalExtension` yields global existence, and the remaining branch-level task is the long-time/eventual sup bound.

By contrast, rewriting the Lᵖ time/IBP identity would duplicate already-proved code. The current code has the identity layer; it lacks the critical-branch seed/global bound layer.
