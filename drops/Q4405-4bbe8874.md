# Q4405 — adversarial check: can the ladder avoid the C³ flux bound?

## Verdict

**No C²-only route reaches the stated endpoint, if the endpoint is truly `WindowSourceEnvelope 2` / pass-2 eigenvalue-weighted summability.**

The earlier conclusion survives the adversarial check: **one C³ flux bound is genuinely needed at the final source pass**. The committed C² flux bound is enough to get the first source pass, but it only produces `WindowSourceEnvelope 1`. It does not produce `WindowSourceEnvelope 2`.

The relevant repo note in `IntervalMildPositiveTimeRegularityV6.lean` states the intended source pass as

```lean
WCE m → WSE (m - 1)
```

with the loss coming from the chemotaxis divergence `∂ₓQ`, and explicitly says the finite induction must reach `WindowSourceEnvelope 2`, whence `restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope` gives the representation `R`. It also identifies the direct IBP route as the coefficient-level path: source coefficient decay comes from integration by parts and pointwise derivative bounds. That is exactly where the C³ requirement enters.

## 1. The finite ladder endpoint forces a final `WCE3 → WSE2` source pass

The clean sharp ladder is:

```text
WSE0  --heat +2-->  WCE2
WCE2  --source -1--> WSE1
WSE1  --heat +2-->  WCE3
WCE3  --source -1--> WSE2
```

So the shortest route to `WSE2` already contains a final source pass from `m = 3`:

```text
WCE3 → WSE2.
```

That is the pass which needs enough regularity of the total source `S` to prove cosine coefficient decay

```text
|S_k| ≤ C / k².
```

For the chemotaxis divergence piece,

```text
S_chem = ∂ₓ Q,
```

so getting `S_chem` coefficient decay `O(k⁻²)` requires either:

```text
S_chem ∈ C² with the p = 2 Neumann IBP boundary condition,
```

or equivalently, via the divergence/sine route,

```text
sineCoeff(Q,k) = O(k⁻³), because cosCoeff(∂ₓQ,k) = kπ · sineCoeff(Q,k).
```

Both formulations require a bound on the third spatial derivative of `Q`:

```text
∂ₓ² S_chem = ∂ₓ³ Q.
```

Thus the last source pass needs `Q ∈ C³` plus the relevant endpoint compatibility, not merely `Q ∈ C²`.

## 2. Why C² flux only reaches `WSE1`

With only `Q ∈ C²` and `Q(0)=Q(1)=0`, the divergence identity gives

```text
cosCoeff(∂ₓQ,k) = kπ · sineCoeff(Q,k).
```

A C² bound on `Q`, with the endpoint `Q=0`, gives at best

```text
|sineCoeff(Q,k)| ≤ C / k².
```

Multiplying by `kπ` loses one power:

```text
|cosCoeff(∂ₓQ,k)| ≤ C / k.
```

That is exactly `WSE1`, not `WSE2`.

Equivalently, if one applies cosine IBP directly to

```text
S_chem = ∂ₓQ,
```

then C² flux only makes `S_chem ∈ C¹`. One integration by parts can give `O(k⁻¹)` decay, but two integrations by parts require `S_chem''`, i.e. `Q'''`.

So the C² flux bound is sufficient for:

```text
WCE2 → WSE1.
```

It is not sufficient for:

```text
WCE3 → WSE2.
```

## 3. Alternative pass ordering cannot avoid the final loss

Changing the order of heat/source passes does not help because the only operation that creates a source envelope is still the source-from-solution pass:

```text
WCE m → WSE (m - 1).
```

To get `WSE2`, some source pass must be instantiated with

```text
m - 1 = 2,
```

hence

```text
m = 3.
```

That means the pass must consume enough solution/flux regularity corresponding to order 3. For the divergence piece, that is the C³ flux estimate.

Heat passes can raise the solution envelope, but they do not directly improve the already-formed source coefficient envelope. The heat pass can produce `WCE3` from `WSE1`; it cannot by itself turn `WSE1` into `WSE2`. The only way to improve the source envelope from 1 to 2 is another source pass, and that pass is precisely where C³ appears.

## 4. Going through pass-4 is worse, not better

A pass-4/eigenvalue-weighted route does not reduce the flux regularity requirement.

If the endpoint were something like:

```text
WindowSourceEnvelope 4
```

then the source decay requirement would be stronger:

```text
|source_k| ≤ C / k⁴.
```

For `source = ∂ₓQ`, that would require roughly

```text
sineCoeff(Q,k) = O(k⁻⁵),
```

or direct source IBP of order 4, hence derivatives of `Q` up to order 5, modulo endpoint towers.

So pass-4 is not a C² workaround. It demands more regularity, not less.

Likewise, trying to reach a high solution coefficient envelope such as `WCE4` before using the final representation does not remove the need for source improvement: the parabolic gain that creates high `WCE` still depends on source envelopes. To get `WCE4` by heat gain you already need a sufficiently strong prior source envelope; it does not appear from C² flux alone.

## 5. The elliptic `+2` gain cannot offset the divergence loss at the source endpoint

The elliptic resolver gain helps inside the nonlinear flux estimate:

```text
u regularity → R regularity → g = R_x(1+R)^(-β) regularity → Q = u g regularity.
```

But after `Q` is formed, the PDE source contains

```text
∂ₓQ.
```

That derivative is an unavoidable one-order loss in the source coefficient envelope. The elliptic `+2` can help prove `Q ∈ C³` from the solution regularity; it does not let a merely C² `Q` produce `O(k⁻²)` coefficients for `∂ₓQ`.

The algebraic obstruction is simple:

```text
Q ∈ C²  ⇒  sineCoeff(Q,k) = O(k⁻²)
        ⇒  cosCoeff(∂ₓQ,k) = kπ·O(k⁻²) = O(k⁻¹).
```

No rearrangement of heat/elliptic gains changes that coefficient identity.

## 6. Precise failure of every C²-only attempt

A C²-only attempt can do this:

```text
WSE0 → WCE2 → WSE1 → WCE3
```

and then it gets stuck.

At `WCE3`, to reach the required `WSE2`, the source pass must prove

```text
|bFormSourceCoeffs p S.u t k| ≤ C / k².
```

For the chem-div part this contains

```text
|cosCoeff(∂ₓQ,k)| ≤ C / k².
```

But C² control of `Q` gives only `C/k`. The missing estimate is exactly one more spatial derivative of `Q`.

So the failing window/pass is:

```text
final source pass: WCE3 → WSE2.
```

The missing leaf is exactly:

```text
Q ∈ C³, with the endpoint compatibility needed for the third sine/direct second-source IBP step.
```

## Bottom line

There is no genuine C²-only route to the pass-2 endpoint under the current ladder semantics.

The minimal sharp route is:

```text
WSE0 → WCE2 → WSE1 → WCE3 → WSE2 → pass-2 summability → R.
```

The final arrow

```text
WCE3 → WSE2
```

requires `source = ∂ₓQ` to have two powers of coefficient decay, hence requires C³ control of the flux `Q`. Going through pass-4 or trying to exploit elliptic gain earlier only increases or relocates the regularity burden. It does not eliminate it.
