## 1. Endpoint compatibility

The exact closed-interval endpoint compatibility is

```lean
g 0 = 0 ∧ g 1 = 0
```

For the intended application `g = u₀'`, this is exactly the Neumann compatibility of the initial datum:

```text
u₀'(0) = 0,   u₀'(1) = 0.
```

It is mathematically necessary.  At `x = 0` and `x = 1`, the full Dirichlet heat kernel vanishes, so the left side at the endpoint is always `0` for every `t > 0`.  Therefore uniform convergence on `Set.Icc 0 1` can only converge to `g` if `g 0 = 0` and `g 1 = 0`.

It is also mathematically sufficient when combined with continuity of `g` on `[0,1]` and the usual Dirichlet heat-kernel approximate identity: continuous zero-boundary data are uniformly approximated by the Dirichlet heat semigroup on the closed interval.

So the right theorem should be morally:

```lean
ContinuousOn g (Set.Icc (0 : ℝ) 1) →
g 0 = 0 → g 1 = 0 →
InitialLegConjugateDerivativeApprox g
```

but the current source tree does not yet have enough full Dirichlet-kernel facts to prove this directly.

## 3. Smallest useful compile-now reducer theorem

Do **not** add a theorem that simply renames `InitialLegConjugateDerivativeApprox`.  A useful next reducer is to split the target into two natural analytic estimates:

1. an **oscillation estimate** for the kernel applied to `g y - g x`, and
2. a **mass-defect estimate** weighted by `g x`.

This is the standard approximate-identity decomposition:

```text
-∫ g(y) K̃(t,x,y) dy - g(x)
  = -∫ (g(y) - g(x)) K̃(t,x,y) dy
    + g(x) * ( -∫ K̃(t,x,y) dy - 1 ).
```

This reducer is genuinely useful because it identifies exactly what the future Dirichlet-kernel work must prove, without hiding the target unchanged.

def InitialLegConjugateOscillationControl (g : ℝ → ℝ) : Prop :=
  ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →

/-- Mass-defect part of the conjugate-kernel approximate identity, weighted by
`g x`.  This is where the endpoint compatibility `g 0 = g 1 = 0` matters: the
Dirichlet mass does not converge to `1` at the endpoints, but the weighted defect
can still vanish uniformly for zero-boundary continuous `g`. -/
def InitialLegConjugateMassDefectControl (g : ℝ → ℝ) : Prop :=
  ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
    ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |g x * (-(∫ y in (0 : ℝ)..1,
          intervalNeumannConjugateKernel t x y) - 1)| < ε

This theorem is only algebra plus interval-integral linearity.  It does not
assert positivity, mass convergence, or tail concentration.  Those remain the
true analytic Dirichlet-kernel obligations. -/
theorem InitialLegConjugateDerivativeApprox_of_splitControls
    {g : ℝ → ℝ}
    (hg : ContinuousOn g (Set.Icc (0 : ℝ) 1))
    (hosc : InitialLegConjugateOscillationControl g)
    (hmass : InitialLegConjugateMassDefectControl g) :
    InitialLegConjugateDerivativeApprox g := by
  intro ε hε
  have hε2 : 0 < ε / 2 := by linarith
  rcases hosc (ε / 2) hε2 with ⟨δo, hδo, ho⟩
  rcases hmass (ε / 2) hε2 with ⟨δm, hδm, hm⟩
  refine ⟨min δo δm, lt_min hδo hδm, ?_⟩
  intro t ht htδ x hx
  have hto : t < δo := lt_of_lt_of_le htδ (min_le_left _ _)
  have htm : t < δm := lt_of_lt_of_le htδ (min_le_right _ _)
  have ho' := ho t ht hto x hx
  have hm' := hm t ht htm x hx
  have h01 : (0 : ℝ) ≤ 1 := by norm_num
  let K : ℝ → ℝ := fun y => intervalNeumannConjugateKernel t x y
  have hg_u : ContinuousOn g (Set.uIcc (0 : ℝ) 1) := by
    simpa [Set.uIcc_of_le h01] using hg
  have hK_u : ContinuousOn K (Set.uIcc (0 : ℝ) 1) := by
    simpa [K, Set.uIcc_of_le h01] using continuousOn_conjugateKernel_snd ht x
  have hosc_int : IntervalIntegrable
      (fun y : ℝ => (g y - g x) * K y) volume 0 1 :=
    ((hg_u.sub continuousOn_const).mul hK_u).intervalIntegrable
  have hconst_int : IntervalIntegrable
      (fun y : ℝ => g x * K y) volume 0 1 :=
    (continuousOn_const.mul hK_u).intervalIntegrable

          g x * (-(∫ y in (0 : ℝ)..1, K y) - 1) := by

        (fun y : ℝ => g y * K y) =
          fun y : ℝ => (g y - g x) * K y + g x * K y := by
      funext y
      ring
    rw [hfun]
    rw [intervalIntegral.integral_add hosc_int hconst_int]
    rw [intervalIntegral.integral_const_mul]
    ring
  simpa [K, hsplit] using
    (lt_of_le_of_lt (abs_add_le _ _)
      (by simpa [K] using add_lt_add ho' hm'))
