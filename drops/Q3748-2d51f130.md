## Why the Hu-only statement is not too strong

`Hu` gives local restart Duhamel representations through:

```lean
HasTimeNeighborhoodSpectralAgreement.exists_data
```

and those representations are already strong enough to produce:

```lean
jointSolutionClosed_u_of_spectralAgreement
jointTimeDerivClosed_u_of_spectralAgreement
```

on the closed spatial slab. Together with `D.hpos`, that is exactly what the power-source K1 chain rule needs:

```lean
d/dr [ν · u(r,x)^γ] = ν · γ · u(r,x)^(γ-1) · ∂r u(r,x)
```

The old chi-zero theorem `powerK1_quadruple_of_subtypeCont` is not needed for the mathematics; it only packages a different proof path that reconstructs the same kind of time-neighborhood regularity from `hsrc0` and `χ₀ = 0` ledger data.

## Likely blocker goals

1. **Closed-slab derivative continuity already exists, use it.** Do not use only `mildSolution_timeDeriv_jointContinuousOn`, whose target is `Ioo 0 T ×ˢ Ioo 0 1`. Use the exported closed version:
   ```lean
   RegularityFrontierAssembly.jointTimeDerivClosed_u_of_spectralAgreement
   ```

2. **`HasDerivAt` must target the intrinsic slope.** `intervalDomainLift_hasDerivAt_time` gives a restart derivative series. Convert it to `PicardLimitK1.slopeSlice` using `hraw.deriv.symm`.

3. **Endpoint positivity for `rpow_const`.** For `x ∈ Icc 0 1`, positivity comes from `D.hpos` after rewriting `intervalDomainLift` with `dif_pos hx`. Do not restrict to `Ioo 0 1` for continuity of the chain-rule integrand; the generic K1 lemmas require closed `Icc` continuity.

4. **Definitional equality between `EWA.adotPow` and `ResolverPowerK1.adotPowOf`.** Prefer `EWA.adotPow` in the new producer because the generic lemmas conclude with that exact coefficient family. Downstream does not care about the name of `adotPow`. If consistency with `ResolverPowerK1.adotPowOf` is desired, add a tiny `rfl` lemma later.

5. **`hMdotPow` has no `a ≤ b` premise.** If a compact-window bound theorem expects a nonempty interval, split on `a ≤ b`. Empty `Icc` is trivial.

6. **Avoid the EWA `realSlice` whnf wall by staying in `D.u`.** The target is canonical Picard/PPID, not the EWA realSlice route. The imported EWA lemmas are generic over an opaque `v`, so using them with `v := D.u` should be safe.
