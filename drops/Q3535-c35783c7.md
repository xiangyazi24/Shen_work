# Q3535 audit: remaining zero-window square-time integrability fields

Audited target: pushed main `e55e3d56` (Task 102). This is an audit only. No unconditional physical H1 bounded-before theorem is implied.

## Short answer

`H1PhysicalRHSComponentSquareSpatialYoungDataBefore_of_classical_squareTimeIntegrable` is correctly shaped: classical APIs now supply RHS measurability, abs-product measurability, and a.e. spatial square-integrability, but the four zero-window **time** integrability fields remain honest analytic inputs.

From current pushed APIs:

1. `IntervalIntegrable (lapL2sq u) volume 0 δ` can be reduced to existing zero-slab/endpoint-lap frontiers, but **not** to ordinary strict-positive-time classical continuity alone.
2. `IntervalIntegrable (H1PhysicalReactPartSq p u) volume 0 δ` can be reduced by a small generic square-profile lemma if one assumes the existing zero-start reaction continuity field from `H1ZeroStartPhysicalRHSDataBefore`.
3. `IntervalIntegrable (H1PhysicalTaxisPartSq p u v) volume 0 δ` and `IntervalIntegrable (H1PhysicalUvxxPartSq p u v) volume 0 δ` are **not** discharged by any current exact API I found. They need explicit zero-start continuity/integrability of the individual physical non-lap factors, not merely strict-positive-time continuity and not merely continuity of the summed chemotaxis representative.

## 2. Strict-positive-time classical continuity is not enough

Confirmed from code.

`IntervalChiNegH1PhysicalClassicalContinuity.lean` proves strict-slab continuity for the physical pieces with hypotheses `0 < a`, `a ≤ b`, `b < T`. Those theorems are excellent for a.e. spatial square-integrability at almost every `r > 0` and for RHS measurability on `Ioc 0 δ`, but they do **not** imply near-zero time integrability of the scalar square profiles.

The new private bridge in the same file,

```lean
continuousOn_Ioc_of_continuousOn_strictWindows
```

is enough for `AEStronglyMeasurable` on `volume.restrict (Set.Ioc 0 δ)`, but `ContinuousOn f (Set.Ioc 0 δ)` alone is not an `IntervalIntegrable f volume 0 δ` theorem. A function may be continuous on `(0, δ]` and still blow up non-integrably at zero. This is exactly why Task 102 correctly keeps the four time-integrability fields explicit.

## 3. What zero-slab package exists, and what it implies

### Existing zero-slab package for `lapL2sq`

`IntervalChiNegH1LapComponentContinuity.lean` contains a real zero-start route for the lap component:

- `H1LiftDeriv2ZeroSlabRepresentativeBefore`
- `H1LapComponentEndpointContinuousBefore_of_zeroSlabRepresentativeBefore`
- `lapL2sq_continuousOn_before_of_zeroSlabRepresentativeBefore`

`IntervalChiNegH1ZeroSlabPhysicalRHS.lean` supplies physical wrappers:

- `H1ZeroStartPhysicalRHSDataBefore`
- `H1LiftDeriv2ZeroSlabRepBefore_of_zeroStartPhysicalRHSData`
- `H1LiftDeriv2ZeroSlabRepBefore_of_zeroStartPhysicalRHS_lift_positive`

So, if one supplies the zero-start physical RHS data, the lap time-integrability field can be obtained non-circularly through the lap zero-slab representative path. The missing source assumptions are exactly the fields of `H1ZeroStartPhysicalRHSDataBefore`: zero-start time derivative continuity, zero-start chem-representative continuity, zero-start reaction continuity, and zero-start interior equality to `liftDeriv2`.

### No corresponding zero-slab package for taxis/uvxx square profiles

There is no analogous current record for the individual non-lap physical factors:

```lean
ContinuousOn (Function.uncurry (H1PhysicalChemTaxisPart p u v))
  (Set.Icc 0 b ×ˢ Set.Icc 0 1)

ContinuousOn (Function.uncurry (H1PhysicalChemUvxxPart p u v))
  (Set.Icc 0 b ×ˢ Set.Icc 0 1)
```

The exact missing source assumptions can be stated either at the part level, as above, or more primitively as zero-start continuity of the ingredients used in those parts:

- `intervalDomainLift (u t) x`
- `intervalDomainLift (v t) x`
- `deriv (intervalDomainLift (u t)) x`
- `deriv (intervalDomainLift (v t)) x`
- the reaction representative `p.μ * intervalDomainLift (v t) x - p.ν * (intervalDomainLift (u t) x) ^ p.γ`
- positivity of `1 + intervalDomainLift (v t) x` on the zero slab, to justify the `rpow` denominator continuity.

The part-level assumptions are the smaller honest API for this route.
