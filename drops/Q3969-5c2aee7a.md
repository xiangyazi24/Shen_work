# Q3969 (cron2): `hleft` for `TruncatedGradientWindowWiring` when `B_F != 0`

## Verdict

Yes: prove `hleft` by a separate induction on the Picard level, but do **not** try to use only the bare semigroup estimate

```text
|d_x U_n(s)| <= Cg * M / sqrt s.
```

That estimate is only the singular part.  Once `B_F = Gamma_M > 0`, the Duhamel contribution feeds the previous gradient back into the source, so the invariant profile should be

```text
|d_x U_n(t')| <= C / sqrt(t') + D                 (0 < t' <= lo),
```

with `C = Cg * M` and `D` chosen to absorb the Volterra self-coupling.  This gives an all-`n` left bound on `[a, lo]` by evaluating the profile at `a`, and for the cron2 split

```text
a = t/4,   lo = t/2,   hi = t,
```

that value is absorbed by the existing window fixed point `Gw`.

For Lean, the cleanest route is to keep `TruncatedGradientWindowWiring` unchanged and add a provider theorem for its `hleft` field.  The provider theorem proves the stronger `(C / sqrt t') + D` bound on `(0, lo]`, then specializes it to `[a, lo]`.

## Constants

Write

```text
K     := Cg = heatGradientLinftyLinftyConstant
beta  := |chi_0| * Gamma_M
L0    := A_L + |chi_0| * A_F
C     := K * M
bL    := 2 * K * beta * sqrt(lo)
bW    := 2 * K * beta * sqrt(hi - a) = truncWindowB Gamma_M chi_0 a hi
```

For any Volterra constant `kappa` satisfying

```text
Ōł½_0^r (r-s)^(-1/2) * s^(-1/2) ds <= kappa
```

for `0 < r <= lo`, define

```text
D_kappa := (K * beta * C * kappa + 2 * K * sqrt(lo) * L0) / (1 - bL).
```

The exact analytic value is `kappa = pi`, so the sharp version is

```text
D_pi =
  (K * beta * C * pi + 2 * K * sqrt(lo) * L0)
  / (1 - 2 * K * beta * sqrt(lo)).
```

This is the explicit `D` for question 3.

For Lean formalization, I would usually avoid proving the exact beta integral at first.  In the cron2 geometry it is enough, and much easier, to prove the elementary bound

```text
kappa := 2 * sqrt 3.
```

This is larger than `pi`, but it is still small enough to be absorbed by the existing `Gw` because `hi - a = 3a`.  So the Lean-first constant is

```text
D_left :=
  (K * beta * C * (2 * sqrt 3) + 2 * K * sqrt(lo) * L0)
  / (1 - 2 * K * beta * sqrt(lo)).
```

The exact `pi` version can replace it later without changing the induction shape.

## The left-profile induction

Let

```text
P_kappa(t') := C / sqrt(t') + D_kappa.
```

Prove by induction on `n`:

```text
LeftProfile n : forall t', 0 < t' -> t' <= lo -> forall x,
  |d_x U_n(t', x)| <= P_kappa(t').
```

The base case is the semigroup initial-data gradient estimate:

```text
|d_x U_0(t')| <= K * M / sqrt(t') = C / sqrt(t') <= P_kappa(t'),
```

using `0 <= D_kappa`.

For the step, use the mild formula and the inductive hypothesis in the source bound.  For `0 < t' <= lo`,

```text
|Src_n(s,y)|
  <= L0 + beta * |d_x U_n(s,y)|
  <= L0 + beta * (C / sqrt(s) + D_kappa)
```

for `0 < s <= t'`.  The endpoint `s = 0` should be ignored in Lean by proving the domination a.e. on the interval, since the singleton has measure zero.

Then the gradient-Duhamel estimate gives

```text
|d_x U_{n+1}(t')|
<= C / sqrt(t')
   + K * Ōł½_0^{t'} (t' - s)^(-1/2)
       * (L0 + beta * (C / sqrt(s) + D_kappa)) ds
<= C / sqrt(t')
   + K * beta * C * kappa
   + 2 * K * sqrt(t') * (L0 + beta * D_kappa)
<= C / sqrt(t')
   + K * beta * C * kappa
   + 2 * K * sqrt(lo) * (L0 + beta * D_kappa).
```

The definition of `D_kappa` is exactly the closure condition

```text
D_kappa
  = K * beta * C * kappa
    + 2 * K * sqrt(lo) * L0
    + 2 * K * sqrt(lo) * beta * D_kappa,
```

or equivalently

```text
D_kappa * (1 - bL)
  = K * beta * C * kappa + 2 * K * sqrt(lo) * L0.
```

Thus

```text
|d_x U_{n+1}(t')| <= C / sqrt(t') + D_kappa.
```

The only smallness needed for this induction is

```text
bL = 2 * K * beta * sqrt(lo) < 1.
```

In cron2 this follows from the window smallness `bW < 1`, since

```text
lo = t/2 <= 3t/4 = hi - a,
```

so `sqrt(lo) <= sqrt(hi-a)` and hence `bL <= bW`.

## From the left profile to `hleft`

For `t' in [a, lo]`, monotonicity of `1 / sqrt` gives

```text
C / sqrt(t') + D_kappa <= C / sqrt(a) + D_kappa.
```

Now compare `C / sqrt(a) + D_kappa` with the existing fixed point

```text
Gw = truncWindowA M A_L A_F chi_0 a lo hi / (1 - truncWindowB Gamma_M chi_0 a hi)
```

where, in cron2,

```text
truncWindowA = C / sqrt(lo-a) + 2 * K * sqrt(hi-a) * L0
             = C / sqrt(a)    + 2 * K * sqrt(hi-a) * L0.
```

The comparison is clean if `kappa <= 2 * sqrt((hi-a)/a)`.  For cron2, `(hi-a)/a = 3`, so the required inequality is

```text
kappa <= 2 * sqrt 3.
```

This holds for the exact choice `kappa = pi`, and it is equality for the Lean-friendly choice `kappa = 2 * sqrt 3`.

The two terms in `D_kappa` are absorbed separately.

First, the source-constant term:

```text
2 * K * sqrt(lo) * L0 / (1 - bL)
  <= 2 * K * sqrt(hi-a) * L0 / (1 - bW),
```

because `lo <= hi-a` and `bL <= bW < 1`.

Second, the singular self-coupling term:

```text
K * beta * C * kappa / (1 - bL)
  <= K * beta * C * (2 * sqrt((hi-a)/a)) / (1 - bW)
  =  bW * (C / sqrt(a)) / (1 - bW).
```

Therefore

```text
C / sqrt(a) + D_kappa
<= C / sqrt(a)
   + bW * (C / sqrt(a)) / (1 - bW)
   + 2 * K * sqrt(hi-a) * L0 / (1 - bW)
=  (C / sqrt(a) + 2 * K * sqrt(hi-a) * L0) / (1 - bW)
=  Gw.
```

This gives

```text
forall n, IterGradOnWindow U a lo n Gw.
```

That is exactly the missing `hleft` field.

## Why the bare `Cg*M/sqrt(s)` source bound is not enough

Using only

```text
|d_x U_n(s)| <= C / sqrt(s)
```

inside the source would produce the finite term

```text
K * beta * C * pi
```

in the next iterate.  That term is not of the form `C / sqrt(t')`; it is a bounded-in-time additive constant.  Therefore the invariant must include `+ D`.  The correct intuition is:

```text
singular semigroup part  ->  C / sqrt(t')
Volterra feedback        ->  bounded constant D
constant source part     ->  bounded constant D
```

## On eliminating `hleft`

A unified variable-envelope induction on `[a, hi]` is possible, but it is more invasive than necessary.

The natural variable envelope after restarting at `a` would look like

```text
E(r) = C_a / sqrt(r) + D_a,    r = t' - a.
```

That envelope is singular as `t' -> a+`, so it does not directly give the uniform `G` needed by the current affine source wiring.  To make this approach work, the framework would need to replace

```text
IterGradOnWindow U a hi n G
```

by a time-dependent profile bound and replace the affine constant source estimate by a weighted Volterra estimate.  That would be a larger rewrite of `TruncatedGradientWindowWiring`.

For the current architecture, the cleanest feasible Lean path is:

```text
keep TruncatedGradientWindowWiring unchanged;
prove hleft externally using the left Volterra profile;
feed that theorem into the existing hleft field.
```

This also preserves the existing right-window affine contraction proof unchanged.
