# Q4352 — chemDiv cosine coefficient bookkeeping

## Verdict

Yes: the divergence bookkeeping is exactly right, provided the flux is evaluated with the same endpoint trace as the Neumann resolver and the denominator is known nonzero.

For

```text
Q(x) = ρ₊(u)(x) · R_x(x) · (1 + R(x))^{-β},
```

if the elliptic resolver satisfies the Neumann endpoint condition

```text
R_x(0) = R_x(1) = 0,
```

and `1 + R(x) > 0`, then

```text
Q(0) = Q(1) = 0.
```

This is independent of the endpoint values of `ρ₊(u)`. The only needed endpoint facts are that `ρ₊(u)` is finite at the endpoints, the denominator is finite and nonzero, and `R_x` vanishes there.

Since in the intended `ρ₊` resolver route one has `R ≥ 0`, the denominator is harmless:

```text
1 + R ≥ 1 > 0.
```

So the endpoint cancellation is airtight.

## IBP identity

For the raw cosine coefficient

```text
cosCoeff(S, k) = ∫₀¹ S(x) cos(kπx) dx
```

and raw sine coefficient

```text
sineCoeff(Q, k) = ∫₀¹ Q(x) sin(kπx) dx,
```

we have, for sufficiently regular `Q`, e.g. `Q ∈ C¹([0,1])` or `Q` absolutely continuous with integrable derivative,

```text
∫₀¹ Q'(x) cos(kπx) dx
  = [Q(x) cos(kπx)]₀¹ + kπ ∫₀¹ Q(x) sin(kπx) dx.
```

The boundary bracket is

```text
Q(1) cos(kπ) - Q(0) cos(0)
= (-1)^k Q(1) - Q(0).
```

Since `Q(0)=Q(1)=0`, this vanishes for every `k`.

Therefore, for `k ≥ 1`,

```text
cosCoeff(Q', k) = kπ · sineCoeff(Q, k).
```

For `k = 0`, the identity reduces to

```text
∫₀¹ Q'(x) dx = Q(1) - Q(0) = 0,
```

and the right side `kπ · sineCoeff(Q,k)` is also zero. So the zero mode is harmless, but it is usually better to handle it separately in Lean because many coefficient conventions split `k=0` from `k>0`.

## Normalization caveat

Check the repository’s coefficient normalizations. If the Neumann cosine coefficient for `k ≥ 1` is normalized as

```text
2 ∫₀¹ f(x) cos(kπx) dx,
```

and the sine coefficient is also normalized with the matching factor

```text
2 ∫₀¹ Q(x) sin(kπx) dx,
```

then the same identity still reads

```text
cosCoeff(Q', k) = kπ · sineCoeff(Q, k).
```

If one side is raw and the other side is normalized, there will be an extra factor of `2`. This is the main Lean bookkeeping risk. Mathematically the raw identity is unambiguous.

## Decay consequence

The divergence costs exactly one power of `k`:

```text
|cosCoeff(∂ₓQ, k)| ≤ kπ · |sineCoeff(Q, k)|.
```

So if

```text
|sineCoeff(Q, k)| ≤ C / k^p,
```

then

```text
|cosCoeff(∂ₓQ, k)| ≤ Cπ / k^{p-1}.
```

This is exactly the expected `-1` loss for the chem-div branch:

```text
Q regularity/decay order p  ⇒  ∂ₓQ source decay order p-1.
```

In particular, if the source ladder needs `S_chem = ∂ₓQ` to have `O(k^{-2})` cosine coefficients, then the sine coefficients of `Q` must have `O(k^{-3})` decay, or one must otherwise obtain the same net bound by a direct source-level C² Neumann IBP.

## Lean-facing statement

A useful raw lemma shape is:

```lean
lemma cosineCoeff_deriv_eq_kpi_sineCoeff_of_flux_zero_boundary
    {Q : ℝ → ℝ} {k : ℕ}
    (hQ_ac_or_C1 : ...) -- enough for interval IBP
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0) :
    (∫ x in (0:ℝ)..1,
        deriv Q x * Real.cos ((k:ℝ) * Real.pi * x)) =
      ((k:ℝ) * Real.pi) *
        (∫ x in (0:ℝ)..1,
          Q x * Real.sin ((k:ℝ) * Real.pi * x)) := by
  -- interval IBP with d/dx cos(kπx) = -kπ sin(kπx)
  -- boundary term: Q 1 * cos(kπ) - Q 0 * cos 0 = 0
```

For `k = 0`, this gives zero automatically after simplification, but a separate lemma is often easier:

```lean
lemma cosineCoeff_deriv_zero_mode_of_flux_zero_boundary
    {Q : ℝ → ℝ}
    (hQ_ac_or_C1 : ...)
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0) :
    (∫ x in (0:ℝ)..1, deriv Q x) = 0 := by
  -- FTC: Q 1 - Q 0
```

## Flux endpoint lemma

The flux endpoint fact should be packaged separately:

```lean
lemma truncatedChemFlux_endpoint_zero_of_resolver_neumann
    {u R Rx : ℝ → ℝ} {β : ℝ}
    (hRx0 : Rx 0 = 0)
    (hRx1 : Rx 1 = 0)
    (hden0 : 1 + R 0 ≠ 0)
    (hden1 : 1 + R 1 ≠ 0) :
    positivePart (u 0) * Rx 0 / (1 + R 0)^β = 0 ∧
    positivePart (u 1) * Rx 1 / (1 + R 1)^β = 0 := by
  simp [hRx0, hRx1]
```

If the code writes multiplication by `(1+R)^(-β)` rather than division, the denominator hypotheses may only be needed for differentiability and simplification of `rpow`, not for endpoint zero itself; the factor `Rx = 0` kills the product.

## Subtleties to keep explicit

1. **Regularity for IBP:** `Q` must be at least absolutely continuous with integrable weak derivative, or classically `C¹` on `[0,1]`. For the current source ladder, the pointwise `C^p` route is more than enough.

2. **Endpoint traces:** the endpoint values must be the actual closed-interval traces. Do not use a zero-extension derivative at the endpoint unless the lemma is stated for the lifted function and the endpoint derivative facts are already adapted.

3. **Neumann resolver:** the only endpoint condition needed for `Q(0)=Q(1)=0` is `R_x(0)=R_x(1)=0`. No condition on `u` or `u₊` at the endpoints is needed for this boundary bracket.

4. **Coefficient normalization:** the raw identity has no hidden factor. Repository-specific normalized coefficients may introduce a factor unless the sine and cosine normalizations match.

## Bottom line

The chem-div coefficient identity is sound:

```text
cosCoeff(∂ₓQ,k) = kπ · sineCoeff(Q,k)
```

with no boundary term because `Q(0)=Q(1)=0`, and that follows directly from the Neumann resolver condition `R_x=0` at both endpoints. The divergence branch loses exactly one power of `k`, so `Q` sine decay `O(k^{-p})` gives source cosine decay `O(k^{-(p-1)})`. The only real Lean risks are regularity for the IBP lemma and matching the repository’s coefficient normalizations.
