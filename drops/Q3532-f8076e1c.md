# Git-drop help

This drop records a minimal help response for the `@GitHubQ####` workflow.

## Expected task format

```text
@GitHubQ#### (dm1): <task description>

IMPORTANT (git-drop): The FIRST LINE of the file MUST be exactly:
ANSWER Q#### <token>
write your COMPLETE response into `drops/Q####-<token>.md` on the `chatgpt-scratch` branch of `xiangyazi24/Shen_work` via the GitHub connector ...
```

## What to include in a good request

- Repository and branch or commit to inspect.
- Exact files, theorem names, or APIs to audit.
- Whether the task is an audit, proof-shape proposal, code patch, CI/debug task, or synthesis.
- The exact drop path and first-line token.
- Any strict constraints, such as no axioms, no new assumptions, no sandbox delivery, or no proof attempt.

## What the response will do

- Write a markdown answer to the requested `drops/...md` file.
- Create the file if absent or overwrite/update it if present.
- Commit the change to `chatgpt-scratch` using the GitHub connector.
- Report the resulting commit SHA.

## Notes

For Lean audit tasks, the safest request style is to ask for exact import lists, namespace/open declarations, theorem statement shapes, compile-shape hazards, and the specific remaining frontiers. Avoid task names that imply unconditional progress unless the theorem actually proves the unconditional result.
