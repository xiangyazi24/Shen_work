## 3. Carry-by-carry audit

### 3.1 Restricted local H1 seed

This is the most delicate carry.

The restricted shape is now faithful:

```text
forall tau, tau in Ioc 0 1 -> tau < T -> H1energy u tau <= Ylocal
```

It avoids the old over-strong demand for times `tau >= T` when `T < 1`. But it still controls all positive times arbitrarily close to zero. That is harmless if the initial datum has finite H1 energy and the solution has H1energy continuity/right-continuity at zero. It is not automatic from mere positivity.

A plausible existing-infrastructure chain exists only if one of the following is already available:

```text
InitialTrace + AdmissibleInitialDatum
  -> H1energy u 0 <= E0
  -> ContinuousWithinAt (H1energy u) (Set.Ici 0) 0
  -> local boundedness on [0, min 1 T)
  -> restricted local H1 seed
```

or:

```text
H1ScalarDIOnBefore u T A B
  + initial H1 seed H1energy u 0 <= E0
  + FTC/absolute-continuity on [0,s] for s < T
  -> Gronwall local bound on (0, min 1 T)
  -> restricted local H1 seed
```

What would *not* be enough:

- `H1ScalarDIOnBefore` alone, with no finite initial/local seed;
- pointwise finiteness of `H1energy u tau` for each `tau > 0`, with no uniform control as `tau -> 0+`;
- parabolic smoothing for positive time only, unless it includes an estimate uniform down to zero or the paper datum has enough initial regularity.

Therefore, if the search does not find an initial-H1 or zero-time continuity theorem, the restricted local seed should be recorded as a real PDE residual rather than discharged by naming.

### 3.3 `H1SupBoundDIDataBefore`

This is probably the best next target.

It looks like a data package for producing `H1IdentityRHSBoundBefore`, so it is likely a wiring artifact if the existing files already contain H1 identity/RHS estimates and the P3/L2/Moser infrastructure already provides the constants it needs.

The audit question is circularity. Inspect the fields/producers for `H1SupBoundDIDataBefore` and classify the required data:

Good/non-circular inputs:

- mass bound from `intervalDomain_Proposition_2_4`;
- L2 seed/frontier data;
- p=2 Lp seed or L2 half-energy data;
- already-proved v/sup/coefficient bounds that do not require the final `IsPaper2BoundedBefore`;

If the package only needs mass/L2/Lp data, there is a plausible existing chain. If it needs the final sup bound for `u`, then it is not a harmless wiring artifact; it is a circular carry and should be split or weakened.

## 7. Overclaim risks

1. **Calling the route unconditional too early.** The route still carries `H1SupBoundDIDataBefore`, scalar regularity fields, the restricted local H1 seed, and the L2 frontier.

2. **Treating local H1 boundedness near zero as automatic.** Uniform H1 control on `(0,min 1 T)` needs a finite initial/local seed or a smoothing estimate uniform down to zero. Pointwise positive-time regularity is not enough.

3. **Using final boundedness to prove `H1SupBoundDIDataBefore`.** If the package requires `IsPaper2BoundedBefore` or an equivalent sup bound for `u`, the argument is circular.

4. **Using pointwise-gradient producers in the wrong direction.** Theorems like `produce_pointwiseGradientBound_of_H1energy_bound` are downstream of an H1 bound. They should not be used to produce the H1 DI that is needed to get that bound.

5. **Reintroducing old residuals.** Avoid routing through `SubintervalAssemblyResidual` or `PointwiseUniformizationResidual` unless there is an independent reason. The current P3 bypass is meant to avoid those residuals.

6. **Hiding `H1UxxL1ContBefore` inside a package.** If this is not already produced from the classical solution record, it is a real PDE regularity carry and should remain explicit.

7. **Conflating the L2 frontier with H1.** `IntervalDomainL2SeedRegularityFrontier` has its own endpoint-continuity/right-derivative obligations. Keep it as a separate branch until both branches are stable.
