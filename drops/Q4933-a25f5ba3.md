## 1. Why `hbdry_left_M_of_chemDiv_limit` / right cannot be upgraded by a short wrapper

### What the existential `gchem` originally is

Inside `boundaryChemDivM_left_limit_factor`, the proof defines, modulo the zero Neumann terms,

```lean
r := p.μ * V0 - p.ν * U0 ^ p.γ
G := (1 + V0) ^ (-p.β) * r
g := U0 ^ (p.m - 1) * G
```

and proves a limit to `U0 * g`. Because `U0 > 0`, that product is exactly

```lean
U0 ^ p.m * (1 + V0) ^ (-p.β) *
  (p.μ * V0 - p.ν * U0 ^ p.γ).
```

The right theorem is identical with `0` replaced by `1`.

### What the theorem statement keeps

The public factor theorem retains only

```lean
∃ gchem,
  |gchem| ≤ M ^ (p.m - 1) * fluxCoeffConst ... ∧
  Tendsto boundaryChemDivMReal ... (nhds (Ue * gchem))
```

It loses the definitional identity

```lean
gchem = Ue ^ (p.m - 1) * (1 + Ve) ^ (-p.β) *
  (p.μ * Ve - p.ν * Ue ^ p.γ),
```

and therefore loses all of the information needed for the sharp argument:

* the sign of the elliptic reaction factor;
* the favorable subtraction `-p.ν * Ue^p.γ`;
* the endpoint value `Ve`;
* the direct route to `theta_linear_bound_public`;
* the exact identity between `Ue * gchem` and the physical `Ue^m` limit.

`hbdry_left_M_of_chemDiv_limit` and its right analogue then use only `|gchem|` and return the coarse statement

```lean
-generalMMinSlopeConst p M * min ≤ endpointTimeSlope.
```

That conclusion is too weak to imply

```lean
generalMLogisticRhs p min ≤ endpointTimeSlope.
```

In particular, the desired right-hand side contains the positive `p.a * min` term and the sharp signed theta loss; it is not bounded above by the old coarse negative Hamilton expression in general.

One could theoretically prove a second exact limit, use uniqueness of limits on the nontrivial one-sided filter to identify `Ue * gchem`, and cancel `Ue > 0`. That is longer than using the physical representative directly, and it still does not recover the signed balance from the already-finished coarse theorem. So this is not a useful wrapper route.

### Correct refactor point

The useful statement is already present inside each `hbdry_*_M_of_chemDiv_limit` proof. On the left it is the step currently called `hGt_ge`:

```lean
R 0 - p.χ₀ * (minv * gchem) ≤ Gt 0
```

and analogously on the right. Extract the proof prefix through that point into a new exact-balance lemma. Keep the current absolute-value/M-dependent algebra as a wrapper around the new lemma.

```lean
theorem generalM_endpoint_physicalChem_le
    {p : CM2Params} {U V : ℝ}
    (hβ : 1 ≤ p.β) (hU : 0 ≤ U) (hV : 0 ≤ V) :
    U ^ p.m * ((1 + V) ^ (-p.β) *
      (p.μ * V - p.ν * U ^ p.γ)) ≤
      U ^ p.m * (p.μ * Theta_beta (p.β - 1)) := by
  have hbase : 0 < 1 + V := by linarith
  have hden_nonneg : 0 ≤ (1 + V) ^ (-p.β) :=
    Real.rpow_nonneg hbase.le _
  have hUγ_nonneg : 0 ≤ U ^ p.γ := Real.rpow_nonneg hU _
  have hdrop :
      (1 + V) ^ (-p.β) * (p.μ * V - p.ν * U ^ p.γ) ≤
        (1 + V) ^ (-p.β) * (p.μ * V) := by
    apply mul_le_mul_of_nonneg_left _ hden_nonneg
    have hνU : 0 ≤ p.ν * U ^ p.γ :=
      mul_nonneg p.hν.le hUγ_nonneg
    linarith
  have hpow_eq :
      (1 + V) ^ (-p.β) * (p.μ * V) =
        p.μ * (V / (1 + V) ^ p.β) := by
    rw [Real.rpow_neg hbase.le]
    ring
  have hscalar :
      (1 + V) ^ (-p.β) * (p.μ * V - p.ν * U ^ p.γ) ≤
        p.μ * Theta_beta (p.β - 1) :=
    hdrop.trans (by
      rw [hpow_eq]
      exact mul_le_mul_of_nonneg_left
        (theta_linear_bound_public (p := p) hβ hV) p.hμ.le)
  exact mul_le_mul_of_nonneg_left hscalar
    (Real.rpow_nonneg hU _)
```

This helper needs neither `hm` nor strict positivity of `U`. The classical solution supplies `U > 0` later for the logistic power identity.

```lean
    ⟨le_rfl, zero_le_one⟩
  have hUpos : 0 < U := by
    dsimp [U]
    simpa [intervalDomainLift] using
      hsol.u_pos' (x := (⟨0, h0⟩ : intervalDomainPoint)) ht0 htT
  have hVnn : 0 ≤ V := by
    dsimp [V]
    simpa [intervalDomainLift] using
      hsol.v_nonneg (x := (⟨0, h0⟩ : intervalDomainPoint)) ht0 htT
  have hCLle : CL ≤ U ^ p.m *
      (p.μ * Theta_beta (p.β - 1)) := by
    simpa [CL] using
      generalM_endpoint_physicalChem_le
        (p := p) hβ hUpos.le hVnn
  have hchem :
      -p.χ₀ * (U ^ p.m *
        (p.μ * Theta_beta (p.β - 1))) ≤ -p.χ₀ * CL :=
    mul_le_mul_of_nonpos_left hCLle (by linarith : -p.χ₀ ≤ 0)
  have hpow : U * (p.b * U ^ p.α) =
      p.b * U ^ (1 + p.α) := by
    rw [Real.rpow_add_of_nonneg hUpos.le
      (by norm_num : 0 ≤ (1 : ℝ)) p.hα.le,
      Real.rpow_one]
    ring
  calc
    generalMLogisticRhs p U =
        U * (p.a - p.b * U ^ p.α) -
          p.χ₀ * (U ^ p.m *
            (p.μ * Theta_beta (p.β - 1))) := by
      simp only [generalMLogisticRhs, generalMChemLoss]
      rw [← hpow]
      ring
    _ ≤ U * (p.a - p.b * U ^ p.α) - p.χ₀ * CL := by
      linarith
    _ ≤ deriv (fun r => intervalDomainLift (u r) 0) t := hbal
```

At the final goal, `generalMLogisticRhs p U` is definitionally the desired expression. If Lean does not unfold the local `U` automatically at the theorem boundary, finish with `simpa [U]` around the `calc` result.

### Right theorem

The right proof is literally the same scalar proof with:

```text

h0 -> h1 := ⟨zero_le_one, le_rfl⟩

```

No product-rule or limit proof should appear in the right Paper3 theorem.

### No `hm` hypothesis

The final sharp endpoint theorem does not need `1 ≤ p.m`. The only uses of `m` are:

* the physical representative, already proved for real `m` at positive `U`.

`hm` is needed by the old global-`M` factor bound, not by the faithful theta endpoint estimate.

## 8. Lean elaboration and `rpow` pitfalls

### `^` has two exponent types in the same expressions

These are real powers:

```lean
U ^ p.m
U ^ p.γ
U ^ (1 + p.α)
(1 + V) ^ (-p.β)
```

but this is a natural power:

```lean
(deriv V x) ^ 2
```

Annotate endpoint literals as `(0 : ℝ)` and `(1 : ℝ)` where inference is not forced. Do not expect a theorem about `Real.rpow` to rewrite a `Nat` power or conversely.

### Rewrite negative powers only after proving the base is positive

Use the established pattern:

```lean
have hbase : 0 < 1 + V := by linarith [hV]
rw [Real.rpow_neg hbase.le]
```

For `Tendsto.rpow_const` or `ContinuousOn.rpow_const` with a negative exponent, pass the nonzero-base branch:

```lean
Or.inl (ne_of_gt hbase)
```

Do not try the exponent-nonnegative branch for `-p.β`.

### Separate `rpow` identities from ring normalization

`ring` treats real-power terms as atoms. It can reorder

```lean
U ^ p.m * reaction * weight
```

into

```lean
U ^ p.m * weight * reaction
```

but it cannot prove an identity such as `U^(1+α) = U * U^α`. First use

```lean
Real.rpow_add_of_nonneg
Real.rpow_one
```

or the existing

```lean
rpow_eq_rpow_sub_one_mul
```

then call `ring`.

### Prefer `rpow_eq_rpow_sub_one_mul` when extracting one factor of `U`

The existing lemma

```lean
rpow_eq_rpow_sub_one_mul hUpos
```

requires only `0 < U`; it does **not** require `1 ≤ m`. This is safer than proving nonnegativity of `m - 1` merely to use monotonicity/additivity.

### `nlinarith` needs sign facts for power atoms

Supply facts such as:

```lean
Real.rpow_nonneg hU _
Real.rpow_nonneg hbase.le _
p.hν.le
p.hμ.le
```

`nlinarith` will regard each real-power expression as an independent atom; it will not infer its sign or monotonicity.

### Multiplying the chem-div upper bound reverses the useful direction

From

```lean
CL ≤ U^m * μΘ
```

and `0 ≤ χ₀`, the needed inequality is

```lean
-χ₀ * (U^m * μΘ) ≤ -χ₀ * CL.
```

Use exactly:

```lean
mul_le_mul_of_nonpos_left hCLle (by linarith : -p.χ₀ ≤ 0)
```

Using `mul_le_mul_of_nonneg_left` here gives the wrong direction.

### Endpoint lift simplification is proof-sensitive

Use explicit membership proofs:

```lean
rw [intervalDomainLift, dif_pos h0]
rw [intervalDomainLift, dif_pos h1]
```

For interior points use `Set.Ioo_subset_Icc_self hy`. When two subtype terms have the same real value but different proof fields, close the equality with:

```lean
Subtype.ext rfl
```

### Keep `intervalDomain` and `intervalDomainM` explicit

Several files open both Paper2 and interval namespaces. If inference drifts to the legacy operator, use the same style as clean-3:

```lean
ShenWork.Paper2.IsPaper2ClassicalSolution.pde_u
  (D := intervalDomainM) ...
```

and likewise for `pde_v`.

### The literal endpoint divergence is not the physical endpoint value

Do not unfold `intervalDomainChemotaxisDivM` at `x = 0` or `x = 1`. The zero extension makes that two-sided endpoint derivative the wrong object. Always take the one-sided limit of `boundaryChemDivMReal` through `classicalChemDivMPhysicalRep`.

### `Tendsto.congr'` orientation

The known limit is first proved for the physical representative, while the desired limit is for `boundaryChemDivMReal`. If

```lean
heq : boundary =ᶠ physical
```

then use

```lean
Filter.Tendsto.congr' heq.symm hlim
```

as the existing file does.

### Right closure normalization

`boundary_min_deriv2_llimit_nonneg` is stated on
`Set.Ioo (1 - η) 1`. With `η := 1`, Lean may not normalize the predicate automatically in higher-order arguments. Introduce

```lean
have hII : Set.Ioo (1 - (1 : ℝ)) 1 = Set.Ioo (0 : ℝ) 1 := by
  rw [sub_self]
```

and rewrite each predicate before applying the theorem.

### Heartbeats

The current boundary Hamilton theorems use `set_option maxHeartbeats 2400000`. The exact balance cores contain the expensive filter/PDE/closure part, so retain a local heartbeat override there. The final Paper3 endpoint wrappers should be light.
