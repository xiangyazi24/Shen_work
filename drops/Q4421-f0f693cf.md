# Q4421: finish-line audit from chem-div `hH2` to χ<0 HSpectral / headline

## Verdict

`hH2 + hdecay + hzero + H + hadotcont` is **not quite sufficient by itself** for the old `coupledChemDivSource_timeC1_of_factorJointC2Inputs` route. That producer also needs the scalar time-derivative coefficient budget

```lean
(MchemDot : ℝ)
(hMdot : ∀ s, 0 ≤ s → ∀ n,
  |coupledChemDivAdot p u s n| ≤ MchemDot)
```

This is the most important remaining item to keep visible. The newer package `CoupledChemDivTimeC1Fields` makes the same point even more explicitly: `hH2`, `hdecay`, `hzero`, the local chain-rule field, `hadotcont`, and `MchemDot/hMdot` are all fields of the chem-div time-C¹ package. So the true finish line for the chem-div source is not only weak-H²; it is weak-H² **plus** a uniform-in-mode bound on the coefficient time derivative.

## 2. Is `hadotcont` produced or carried?

Partially produced, but be careful about the exact shape.

The file `ChemDivAdot.lean` produces the coefficient derivative legs from time-smoothness:

```lean
chemDivAdot_deriv_legs_of_smoothness
  (hchain : CoupledChemDivLocalChainRule p u)
  (hjointcont : ContinuousOn
    (Function.uncurry (coupledChemDivTimeDerivativeLift p u))
    (Set.Icc (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1))
```

This gives:

```lean
∀ s ∈ Set.Icc 0 T, ∀ n,
  HasDerivWithinAt ... (coupledChemDivAdot p u s n) (Set.Icc 0 T) s
```

and

```lean
∀ n, ContinuousOn (fun s => coupledChemDivAdot p u s n) (Set.Icc 0 T)
```

So the committed reduction gives **windowed/closed-interval continuity**, not automatically the global

```lean
∀ n, Continuous (fun s => coupledChemDivAdot p u s n)
```

required by the older `DuhamelSourceTimeC1` constructors.

There are two clean ways to close this mismatch:

1. Produce a global joint-continuity theorem for `coupledChemDivTimeDerivativeLift`, then use the same coefficient-continuity argument globally.
2. Better: add a windowed/within version of `DuhamelSourceTimeC1` or a wrapper that extends the windowed derivative field outside the active interval harmlessly.

If the current target is the old global `DuhamelSourceTimeC1`, then `hadotcont` is still **carried unless a global-continuity bridge is supplied**.

## 3. `MchemDot/hMdot` is not produced by `hadotcont`

This is the real remaining scalar budget.

`ChemDivAdot.lean` explicitly separates:

```lean
MchemDot : ℝ
hMdot : ∀ s, 0 ≤ s → ∀ n,
  |coupledChemDivAdot p u s n| ≤ MchemDot
```

from pointwise/per-mode time smoothness. Per-mode continuity on a compact time interval gives a bound depending on `n`; it does **not** give one single `MchemDot` uniform in all modes.

The file isolates the residual as a summable-envelope requirement:

```lean
chemDivAdot_Mdot_residual
  (env : ℕ → ℝ)
  (henvnn : ∀ n, 0 ≤ env n)
  (henvsum : Summable env)
  (henv : ∀ s ∈ Set.Icc 0 T, ∀ n,
    |coupledChemDivAdot p u s n| ≤ env n)
  : ∃ Mdot, ∀ s ∈ Set.Icc 0 T, ∀ n,
      |coupledChemDivAdot p u s n| ≤ Mdot
```

So the honest remaining obligation is:

```lean
∃ env : ℕ → ℝ,
  (∀ n, 0 ≤ env n) ∧ Summable env ∧
  ∀ s ∈ Set.Icc 0 T, ∀ n,
    |coupledChemDivAdot p u s n| ≤ env n
```

This is the time-derivative analogue of the source envelope. Without it, `DuhamelSourceTimeC1` cannot be closed in the current structure.



## 8. Complete ordered obligation list from chem-div H² to χ<0 headline

Here is the precise finish-line checklist.

### A. Chem-div source-time-C¹

Required to produce:

```lean
hchem : DuhamelSourceTimeC1 (coupledChemDivSourceCoeffs p u)
```

Inputs/status:

1. `Cchem`, `hCchem` — scalar budget. Usually produced from the quadratic-decay and zero-mode constants.
2. `hH2` — weak-H² witness for `coupledChemDivSourceLift`; in progress via parity / smooth representative.
3. `hdecay` — produced from `intervalWeakH2Neumann_cosineCoeff_quadratic_decay` once `hH2` exists.
4. `hzero` — scalar zero-mode bound; should be produced from boundedness or divergence zero-mode, but must be explicitly supplied.
5. `H : CoupledChemDivFluxFactorJointC2Inputs` — produced if FAC chain is closed; otherwise carries `CoupledChemDivFluxFactorFACInputs` local slab data.
6. `hchain : CoupledChemDivLocalChainRule p u` — needed in the newer `CoupledChemDivTimeC1Fields` route; produced from pointwise chain atoms if those are closed.
7. `hadotcont` — only windowed `ContinuousOn` is currently reduced in `ChemDivAdot.lean`; global `Continuous` still needs a bridge if using old `DuhamelSourceTimeC1`.
8. `MchemDot`, `hMdot` — **still carried unless you produce a summable envelope for `coupledChemDivAdot`**. This is the major remaining hidden budget.

### B. Logistic source-time-C¹

Required to produce:

```lean
hlog : DuhamelSourceTimeC1 (coupledLogisticSourceCoeffs p u)
```

Inputs/status:

1. Representation coefficients `bc`.
2. `hbsum : ∀ σ, Summable (fun n => λ_n * |bc σ n|)`.
3. `hagree :` slice equals the cosine series on `[0,1]`.
4. `hpos :` strict positive slice on `[0,1]`.
5. Logistic weak-H²/decay/zero-mode, provided by the representation adapter if the above are available.
6. Logistic coefficient derivative, `hadotcont`, and derivative bound. These are parallel time-C¹ obligations; they are not produced by the chem-div `hH2`.

### C. Total B-form source

Required to produce:

```lean
hsrcB : DuhamelSourceTimeC1 (bFormSourceCoeffs p S.u)
```

Inputs/status:

1. `hlog`.
2. `hchem`.
3. Source split, essentially definitional:

```lean
bFormSourceCoeffs = logistic - χ₀ * chemDiv
```

This step is produced by scalar multiplication and addition once the two component source-time-C¹ packages exist.

### D. HSpectral leaves

Required to build:

```lean
BFormMildSpectralLeaves p S
```

Inputs/status:

1. `hsrcB` — from C.
2. `hB_restart : RestartRepresentation S` — separate restart cosine representation leaf.
3. `hlogData` — logistic Fourier data for each interior time.
4. `hchemData` — chem-div Fourier data for each interior time.
5. `hResolverWitness` — resolver source witness.

Then:

```lean
bFormMildSpectralBootstrapData_of_leaves H
```

produces:

```lean
BFormMildSpectralBootstrapData p S
```

### E. Uniform HSpectral input for V6

The V6 assembly wants a generic/uniform producer, essentially:

```lean

  BFormMildSpectralBootstrapData p S
```

So the leaf construction in D must be generic in `S`, not specialized only to one truncated Picard limit, unless the final assembly route has been specialized accordingly.

### F. χ<0 final assembly

Then the final theorem application is pure assembly.

## Bottom line

After `hH2`, `hdecay`, `hzero`, and `H`, the chem-div source-time-C¹ is **not** fully closed unless the following are also supplied:

```lean
hadotcont : ∀ n, Continuous (fun s => coupledChemDivAdot p u s n)
MchemDot : ℝ
hMdot : ∀ s, 0 ≤ s → ∀ n,
  |coupledChemDivAdot p u s n| ≤ MchemDot
```

Of these, the most serious remaining one is `hMdot`. The repo already documents that per-mode smoothness does not imply a uniform-in-`n` bound; it requires a summable envelope for the coefficient time-derivative field.

After chem-div source-time-C¹, the finish line is:

```text
hchem + hlog + source split
  ⇒ hsrcB
  ⇒ BFormMildSpectralLeaves together with hB_restart, hlogData, hchemData, hResolverWitness
  ⇒ bFormMildSpectralBootstrapData_of_leaves
  ⇒ HSpectral
  ⇒ V6 assembly with energy + jensen + mapCertificate
  ⇒ χ<0 headline
```

So the exact remaining obligation set is larger than `hH2`: the chem-div `Mdot` envelope, the logistic source-time-C¹ package, restart representation, Fourier data leaves, resolver witness, and the final uniformization over arbitrary `ConjugateMildSolutionData` remain to be checked or produced.
