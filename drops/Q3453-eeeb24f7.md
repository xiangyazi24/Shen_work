# Q3453 route audit: strict-positive-time H¹ RHS route without zero-start lap continuity

## Verdict

The smallest honest route C target is **not** a new zero-start lap-continuity theorem and not a new `H1ScalarDIOnBeforeStrict` refactor. Keep the existing downstream `H1ScalarDIOnBefore` / bounded-before route, and add a narrow adapter that produces the existing

`H1IdentityRHSIntegrableBefore p u T taxisX uvxx reactX`

from:

1. pointwise H¹ identity on `Ioo 0 T`;
2. strict-positive-time component continuity, already packaged as
   `H1IdentityRHSComponentsContinuousStrictBefore`;
3. an explicit zero-start **integrability** input for the assembled RHS, not zero-start continuity of `lapL2sq`.

This feeds the current bounded-before route while avoiding the false upgrade from strict slabs to closed zero-start lap/H² continuity.

The current code already has most of the right pieces:

- `H1IdentityRHSComponentsContinuousStrictBefore` gives component continuity only on `Icc a b` with `0 < a`.
- `H1IdentityRHSIntegrableBefore` requires RHS interval-integrability for all `0 ≤ a`, including `a = 0`.
- `H1ScalarDIOnBefore` and `H1ScalarRegularityBefore` still expect closed-window continuity/integrability at the scalar energy level, but that is fine: H¹ energy continuity at zero is already handled by the initial endpoint route, and derivative integrability can come from RHS integrability rather than lap continuity.

So the missing atom is: **initial-window integrability of the assembled H¹ identity RHS**.

## What near-zero input is actually necessary?

Strict positive-time continuity only gives integrability on `[a,b]` for `0 < a`. It gives **no** integrability on `[0,b]`. A function can be continuous on every `[a,b]`, `0 < a`, and still fail to be integrable at zero, e.g. `1 / t`.

Therefore some genuine near-zero input is necessary. The minimal input for preserving the existing route is one of these:

1. Direct assembled-RHS initial-window integrability:

```lean
∀ {b : ℝ}, 0 ≤ b → b < T →
  IntervalIntegrable
    (H1IdentityRHSValue p u taxisX uvxx reactX) volume (0 : ℝ) b
```

2. A stronger componentwise version:

```lean
∀ {b : ℝ}, 0 ≤ b → b < T →
  IntervalIntegrable (fun τ => lapL2sq u τ) volume 0 b ∧
  IntervalIntegrable taxisX volume 0 b ∧
  IntervalIntegrable uvxx volume 0 b ∧
  IntervalIntegrable reactX volume 0 b
```

3. A majorant/cutoff version, such as an `L¹` majorant for the assembled RHS on a small zero-neighborhood plus strict continuity on the positive-time tail.

For the next Lean task, choose option 1. It is the smallest, most robust interface and avoids extra component bookkeeping. Option 2 is useful only if existing producers naturally give componentwise integrability. Option 3 is a later producer theorem for option 1, not the core interface.

## Why not introduce `H1ScalarDIOnBeforeStrict` now?

A strict scalar-DI record would force a wider refactor. The current scalar average theorem uses closed windows with nonnegative left endpoint:

- `H1_avg_of_scalarDI_before` integrates over `[τ - 1, τ]`; when `τ = 1`, this is `[0,1]`.
- `exists_H1_localSeed_of_scalarDI_before` uses H¹ energy continuity on `[0,T/2]` to get the local seed.

Those uses are not the problem. The problem is only how to produce derivative/RHS interval-integrability on zero-start windows without lap continuity. The direct adapter above solves that while keeping the existing downstream API unchanged.

## Circularity risks and false shortcuts

Do not prove or use any of these as part of route C:

1. `H1IdentityRHSComponentsContinuousBefore` from strict data alone. That would reassert the closed zero-start component continuity route.

2. `H1LapComponentZeroRightContinuous u`, `H1LiftDeriv2ZeroSlabRepresentativeBefore`, or `H1ZeroStartPhysicalRHSDataBefore` as a hidden intermediate. Those are exactly the route-B frontier, not route C.

3. `ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc 0 b)` from strict positive-time continuity. This is false without a zero-time H²/lap trace.

4. Initial H¹ energy convergence as a substitute for lap/RHS integrability. It gives scalar H¹ energy continuity at zero, not integrability of the second-derivative/lap component.

5. `IsPaper2BoundedBefore intervalDomain T u` or the final H¹ bound to prove the near-zero RHS integrability that is needed to obtain bounded-before. That would be circular.

6. `H1SupBoundDIDataBefore` as a producer of fixed-split RHS integrability. Its `point` field gives existential witnesses at each time for the DI estimate; it does not provide interval-integrability of a chosen fixed triple `taxisX uvxx reactX`.

7. A cutoff/limit proof without a uniform near-zero `L¹` bound or direct initial-window integrability statement. Strict-slab continuity alone does not control the `a → 0+` limit.
