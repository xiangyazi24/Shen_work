# Q3824 — patching is likely unnecessary if the uniform-core mild solution is bootstrapped on `(0,T)`

## Verdict

Yes, the new insight is essentially correct, with one important formulation caveat.

If the **same** uniform-core B-form mild solution can be shown to be:

1. strictly positive for every `0 < t < T`,
2. spatially smooth for every positive time,
3. time differentiable on every interior positive-time window,
4. satisfying the pointwise PDE and Neumann boundary conditions on `(0,T)`, and
5. having `InitialTrace intervalDomain u0 u`,

then no restart patching is needed. The solution is already the local classical solution required by

```lean
CoupledFluxClassicalLocalExistenceResidual p
```

The correct target is then a **single-trajectory positive-time bootstrap** theorem, not a heat-restart patch theorem.

The caveat: if `PPIDSpectralBootstrapFrontier` is formulated as a theorem about a **fresh PPID initial value problem** starting from `u epsilon`, it still does not automatically prove properties of the original weak-core mild trajectory after `epsilon` unless uniqueness/identification is supplied. But if it is formulated as an **interior-window spectral bootstrap for the same mild trajectory**, using the fact that `u epsilon` is PPID only as a local smoothing/floor input, then it is exactly the remaining hypothesis and `HeatRestartPatchFrontier` is unnecessary.

## Why the no-patch route is mathematically right

For weak positive datum `u0`, the B-form mild solution has the shape

```text
u(t) = S(t)u0 + chemotaxis Duhamel leg + logistic Duhamel leg.
```

For `t > 0`:

* `S(t)u0` is smooth in space even if `u0` is only bounded/continuous.
* The Duhamel legs are positive-time smoothing objects; their endpoint singularities are integrable in the already-used gradient/heat estimates.
* `S(t)u0 > 0` on `[0,1]` for `t > 0` if `u0` is nonnegative and positive somewhere, by the delivered heat-kernel positivity.
* If the Duhamel perturbation is uniformly `O(sqrt t)` or otherwise small compared with a heat lower barrier on a sufficiently short window, then `u(t) > 0` for `0 < t <= T` after shrinking `T`.

Thus the same mild solution should become positive and classical immediately after time zero, while still having initial trace `u0`. That is exactly the local-existence theorem we need. There is no logical need to splice a second PPID solution into it.

## What remains nontrivial

The statement “Duhamel integrals are smooth for `t > 0`” must be formalized in the precise spectral shape consumed by the existing bootstrap theorem. In this codebase, that means building:

```lean

hTimeNhd
hPdeAgreement
```

for the **same** mild trajectory from the uniform core.

The already-delivered PPID-free bootstrap theorem, as described, consumes:

```lean

```

plus four spectral fields and produces:

```lean

```

With `hResolverPos` now done, the remaining work is the spectral/time-source core. No seam/glue theorem is needed if those fields are produced for `S.u` on all positive interior times.

## If `PPIDSpectralBootstrapFrontier` remains, how it should be typed

The name `PPIDSpectralBootstrapFrontier` is acceptable only if it does **not** mean “start a new unrelated PPID IVP and use its solution.” It should mean something like:

```lean
/-- At every interior time, heat smoothing gives enough PPID-style local floor and
spectral source data to bootstrap the existing mild trajectory near that time. -/
def PPIDSpectralBootstrapFrontier
    (p : CM2Params) {u0 : intervalDomainPoint -> ℝ}
    (S : ConjugateMildSolutionData p u0) : Prop :=
  ∀ t0, 0 < t0 -> t0 < S.T ->
    ∃ offset, 0 < offset ∧ offset < t0 ∧
      PaperPositiveInitialDatum intervalDomain (S.u offset) ∧
      /* source witness / restart representation for S.u near t0 */
      True
```

The key is the last line: the witness must be for the **existing** `S.u` near `t0`, not merely for a newly constructed solution with initial datum `S.u offset`.

If the frontier only gives a new PPID classical solution from `S.u offset`, then it still needs uniqueness to identify it with `S.u` near `t0`. That would reintroduce an identification gap, even if not a patching gap.

## Answer to the question

Yes: if the uniform-core mild solution is bootstrapped directly on every positive interior time, patching is unnecessary. The mild solution itself is the local classical solution on `(0,T)` and has initial trace `u0`.

But be careful about the frontier formulation. The remaining hypothesis should not be a “fresh PPID restart” frontier unless it also identifies the restarted solution with the original mild trajectory. The right remaining hypothesis is an **interior positive-time spectral bootstrap frontier for the same trajectory**. In that formulation, `HeatRestartPatchFrontier` can be deleted.
