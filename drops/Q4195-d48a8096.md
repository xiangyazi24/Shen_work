# Q4195 — short-window gradient bootstrap and removal of `hcontr_grad`

## Executive answer

The fixed-point algebra is sound, and the constants in the existing window file have exactly the affine shape you want:

```lean
def truncWindowSourceCL (A_L A_F B_F χ G : ℝ) : ℝ :=
  A_L + |χ| * (A_F + B_F * G)

def truncWindowA (M A_L A_F χ a lo hi : ℝ) : ℝ :=
  heatGradientLinftyLinftyConstant / Real.sqrt (lo - a) * M
    + heatGradientLinftyLinftyConstant * (2 * Real.sqrt (hi - a))
        * (A_L + |χ| * A_F)

def truncWindowB (B_F χ a hi : ℝ) : ℝ :=
  heatGradientLinftyLinftyConstant * (2 * Real.sqrt (hi - a)) * |χ| * B_F

def truncWindowAffine
    (M A_L A_F B_F χ a lo hi G : ℝ) : ℝ :=
  truncWindowA M A_L A_F χ a lo hi
    + truncWindowB B_F χ a hi * G

def truncWindowFixedG (M A_L A_F B_F χ a lo hi : ℝ) : ℝ :=
  truncWindowA M A_L A_F χ a lo hi /
    (1 - truncWindowB B_F χ a hi)
```

and the fixed-point closure is already packaged as:

```lean
theorem affine_fixed_closes {A B : ℝ} (hB : B < 1) :
    A + B * (A / (1 - B)) ≤ A / (1 - B)
```

So the arithmetic part is correct:

```lean
truncWindowAffine M A_L A_F B_F χ a lo hi
  (truncWindowFixedG M A_L A_F B_F χ a lo hi)
≤ truncWindowFixedG M A_L A_F B_F χ a lo hi
```

under `truncWindowB B_F χ a hi < 1`.

However, the proposed **NO-left-profile induction is not sound as stated** if the successor theorem needs the source bound on all of `[a, hi]`. The induction hypothesis only gives the gradient bound on `[lo, hi]`. The source bound for `s ∈ [a, lo)` is not supplied.

This is exactly why the committed generic wiring has a left input:

```lean
structure TruncatedGradientWindowWiring ... where
  ...
  hleft : ∀ n : ℕ, IterGradOnWindow U a lo n G
  hbase : IterGradOnWindow U lo hi 0 G
  hsource_of_grad : ∀ n : ℕ,
    IterGradOnWindow U a hi n G →
      ∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
        |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G
  hkernel_step : ∀ n : ℕ,
    (∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
      |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G) →
      IterGradOnWindow U lo hi (n + 1)
        (truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G)
```

and the induction theorem uses `hleft` to glue:

```lean
have HsrcInterval : IterGradOnWindow U a hi n G :=
  IterGradOnWindow.glue_left (W.hleft n) IH
```

So: **the short-window strategy removes the need for one long-window contraction, but it does not remove the need to control the source on the whole restart integration interval.** That source control can come from a left profile, from a previous short window, or from a separately proved bound on `[a, lo]`. It cannot simply be dropped.

## 1. Is the induction sound?

### Algebra: yes

The fixed-point algebra is exactly the standard affine invariant-envelope argument.

Let:

```lean
A := truncWindowA M A_L A_F χ a lo hi
B := truncWindowB B_F χ a hi
Gfix := truncWindowFixedG M A_L A_F B_F χ a lo hi
```

Then:

```lean
Gfix = A / (1 - B)
```

and the existing lemma gives:

```lean
A + B * Gfix ≤ Gfix
```

provided:

```lean
hB : B < 1
```

In Lean:

```lean
have hclosed :
    truncWindowAffine M A_L A_F B_F χ a lo hi
      (truncWindowFixedG M A_L A_F B_F χ a lo hi)
      ≤ truncWindowFixedG M A_L A_F B_F χ a lo hi := by
  simpa [truncWindowAffine, truncWindowFixedG]
    using TruncatedGradientWindow.affine_fixed_closes
      (A := truncWindowA M A_L A_F χ a lo hi)
      (B := truncWindowB B_F χ a hi)
      hBcontr
```

Depending on simp unfolding, you may need a `change` or `dsimp`, but the proof is just `affine_fixed_closes`.

### Source closure: yes, but it needs the gradient on the source interval

The source estimate is the standard chemotaxis source-from-gradient estimate:

```lean
Src n s y = logistic(U n s) y - χ₀ * deriv (chemFlux(U n s)) y
```

with bounds:

```lean
|logistic(U n s) y| ≤ A_L
|deriv (chemFlux(U n s)) y| ≤ A_F + B_F * G
```

Then the existing arithmetic lemma is exactly:

```lean
theorem abs_logistic_sub_chi_flux_le
    {L Fp A_L A_F B_F χ G : ℝ}
    (hL : |L| ≤ A_L)
    (hF : |Fp| ≤ A_F + B_F * G) :
    |L - χ * Fp| ≤ truncWindowSourceCL A_L A_F B_F χ G
```

So the intended source closure is:

```lean
lemma source_of_window_grad
    (H : IterGradOnWindow U a hi n G) :
    ∀ s, a ≤ s → s ≤ hi → ∀ y,
      |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G := by
  intro s hsa hshi y
  have hgrad : ∀ x, |deriv (intervalDomainLift (U n s)) x| ≤ G :=
    (H s hsa hshi).1
  -- logistic bound from the Picard ball:
  have hL : |truncatedLogisticLifted p (U n s) y| ≤ A_L := ...
  -- flux derivative bound from product-rule/resolver estimates and hgrad:
  have hF : |deriv (truncatedChemFluxLifted p (U n s)) y| ≤ A_F + B_F * G := ...
  simpa [hSrc n s y] using
    abs_logistic_sub_chi_flux_le
      (L := truncatedLogisticLifted p (U n s) y)
      (Fp := deriv (truncatedChemFluxLifted p (U n s)) y)
      (χ := p.χ₀) hL hF
```

The file already contains the product-rule envelope that has the correct affine dependence on `G`:

```lean
|deriv (truncatedChemFluxLifted p w) y| ≤
  (M * H * Q + p.β * M * Γ ^ 2 * QDen) + Γ * G * Q
```

So `A_F` is the non-gradient part and `B_F * G` is the gradient-dependent part. This matches your intended `A_L + |χ| * (A_F + B_F * G)`.

### The gap in the proposed induction

Your proposed induction says:

```text
IH: IterGradOnWindow U lo hi n Gfix
```

Then it tries to prove the source bound on:

```text
s ∈ [a, hi]
```

But `IH` only gives the gradient bound for:

```text
s ∈ [lo, hi]
```

It says nothing for:

```text
s ∈ [a, lo)
```

The successor Duhamel estimate integrates from `a` to `τ`, and for `τ ∈ [lo, hi]` this includes the whole interval `[a, lo]`. Therefore the source estimate cannot be derived from `IH` alone.

The committed generic induction solves exactly this by requiring:

```lean
hleft : ∀ n, IterGradOnWindow U a lo n G
```

and then gluing:

```lean
IterGradOnWindow.glue_left (W.hleft n) IH
```

So the correct statement is:

```text
The induction is sound with a left-window/previous-window source control.
It is not sound with only the right-window IH unless STEP is weakened to require source control only on [lo,hi], which is not the current Duhamel restart estimate.
```

## 2. Does this remove `hcontr_grad` or only relocate it?

It removes the **global long-window** contraction hypothesis, but it replaces it by a **local short-window condition**:

```lean
truncWindowB B_F χ a hi < 1
```

Because:

```lean
truncWindowB B_F χ a hi
= Cg * 2 * sqrt(hi - a) * |χ| * B_F
```

this can always be forced by taking `hi - a` sufficiently small, assuming `B_F` is finite. That is a major improvement over a fixed `t/4, t/2, t` geometry, where the window length is not freely shrinkable.

But the constraint has not disappeared logically. It has become:

```text
choose a short enough restart interval around the target time.
```

### For a per-time local Lipschitz theorem: yes, short windows are enough

If the downstream use is local in time, for example:

```text
At a fixed t > 0, obtain a Lipschitz/source-coefficient bound at time t.
```

then choose:

```text
a < lo ≤ t ≤ hi
```

with `hi - a` small and `lo > a > 0`. The gradient bound on `[lo, hi]` gives the needed bound at `t` if `t ∈ [lo, hi]`.

This is enough for local positive-time regularity or for feeding a source-coefficient estimate at a fixed positive time.

### For a compact positive-time window: finite short-window chaining is enough

If the downstream theorem needs a bound on a compact interval:

```text
[t₀, T],  with 0 < t₀ < T,
```

then cover `[t₀, T]` by finitely many short windows. On each window choose `hi - a` small enough so the local contraction holds. The constants may change from window to window, but for a finite cover you can take the maximum.

This avoids the long-window contraction condition.

Lean shape:

```lean
-- Choose a partition t₀ = r₀ < r₁ < ... < rN = T
-- with each local restart span small enough:
hB_i : truncWindowB B_F χ a_i hi_i < 1

-- Prove local window gradient bound on each window.
H_i : ∀ n, IterGradOnWindow U lo_i hi_i n G_i

-- Combine finite windows by maxing constants or by finite-cover membership.
```

This is more bookkeeping than the old fixed `t/4,t/2,t` proof, but mathematically it is the right way to remove a global smallness/contraction hypothesis.

### For all of `(0,t]` with one flat constant: no

If the downstream theorem insists on one finite flat bound on the whole interval:

```text
∀ τ ∈ (0,t], ∀x, |∂x U(τ,x)| ≤ G
```

then the short-window strategy does not give that directly, and in general such a flat constant is not the natural object near `τ = 0`. The heat semigroup gradient has the singular behavior:

```text
‖∂x S(τ)u₀‖∞ ~ τ^{-1/2} ‖u₀‖∞.
```

So a uniform flat bound down to time zero is either false for rough initial data or requires a singular profile such as:

```text
C / sqrt(τ) + D.
```

That is exactly the role of the left-profile infrastructure.

Thus:

```text
local/compact positive-time theorem: short windows remove hcontr_grad;
whole (0,t] flat theorem: no, you need a singular left profile or an equivalent substitute.
```

## Limit passage

The uniform-in-`n` window gradient bound passes to a Lipschitz bound for the Picard limit on `[lo,hi]`, but the clean route is through difference quotients, not derivative convergence.

From:

```lean
IterGradOnWindow U lo hi n G
```

one should first derive, for each `τ ∈ [lo,hi]`:

```lean
∀ x y ∈ Icc 0 1,
  |intervalDomainLift (U n τ) x - intervalDomainLift (U n τ) y| ≤ G * |x - y|.
```

Use a one-dimensional mean-value/Lagrange or integral-of-derivative lemma from the derivative bound. Since `IterGradOnWindow` also gives differentiability on `Ioo 0 1`, endpoint handling should be done by continuity on `[0,1]` or by an `Icc` closure argument. Avoid relying on derivatives at endpoints.

Then pass to the limit using pointwise convergence of the Picard iterates:

```lean
truncatedConjugatePicardIter_pointwise_convergent
```

or the corresponding limit theorem in your namespace. For fixed `τ,x,y`, take limits on both sides:

```lean
|U∞ τ x - U∞ τ y|
  = lim_n |U n τ x - U n τ y|
  ≤ G * |x - y|.
```

This proves Lipschitz of the limit slice without proving derivative convergence.

If downstream code requires an actual `deriv` bound for the limit, that is stronger and needs extra regularity/compactness. For coefficient boundedness, Lipschitz or a.e. weak derivative is often the better target, but the existing file may still be organized around classical derivative bounds.

## Bottom line

The affine fixed-point mechanism is correct and already encoded.

The source closure is the standard chemotaxis estimate and matches the constants:

```text
truncWindowSourceCL = A_L + |χ| * (A_F + B_F * G)
truncWindowAffine  = truncWindowA + truncWindowB * G
```

But the proposed proof without a left profile is missing a bound for source times `s ∈ [a,lo)`. The fully window-based strategy removes the **long-window** contraction hypothesis only if you either:

```text
1. keep/prove a left bound on [a,lo], or
2. chain short windows so that the left bound comes from the previous window.
```

For downstream local-in-time or compact-positive-time use, this is enough and genuinely removes the old fixed-window `hcontr_grad`. For a single flat bound on all of `(0,t]`, it does not; near zero you still need a singular left profile or an equivalent substitute.
