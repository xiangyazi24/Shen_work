## 3. Identity and sqrt bounds

I did not find an exact small derivation of either physical identity or physical sqrt bounds from the current APIs.

For identity, the available constructor is:

```lean
H1PhysicalRHSIdentityBefore_of_parametric_and_substitution
```

but it still requires both the parametric derivative formula and the exact physical substitution equality. The Route-C identity API gives an existential scalar triple, not the concrete physical scalar triple. The minimal honest frontier remains the exact substitution into `H1IdentityRHSValue p u (H1PhysicalTaxisX p u v) (H1PhysicalUvxxX p u v) (H1PhysicalReactX p u)`.

For sqrt bounds, `H1PhysicalRHSSqrtBoundsBefore` is only a wrapper around `H1SqrtTermBoundsBefore` specialized to the physical scalar triple. A trivial constructor from an already supplied generic `H1SqrtTermBoundsBefore` is possible, but that would not prove the pointwise taxis/uvxx/react estimates. The honest frontier remains those three pointwise inequalities and the nonnegativity fields in `H1SqrtTermBoundsBefore`.

## 4. Status of `H1ZeroStartPhysicalPrimitiveDataBefore`

`H1ZeroStartPhysicalPrimitiveDataBefore` is not currently derivable from `IsPaper2ClassicalSolution` plus ordinary initial/positivity APIs. It is a genuine zero-start closed-time regularity package in `IntervalChiNegH1ZeroStartComponents.lean`.

The exact fields are:

```lean
u_cont0
v_cont0
ux_cont0
vx_cont0
u_pos0
v_nonneg0
time_cont0
eqInterior0
```

Strict classical regularity supplies positive-time slabs, not closed-time slabs including `t = 0`. The hard blockers are especially `ux_cont0`, `vx_cont0`, `time_cont0`, and `eqInterior0` on `Set.Icc 0 b ×ˢ Set.Ioo 0 1`. Positivity at `t = 0` might follow from strong initial-slice data, but the full primitive package requires much more than positivity.

Suggested location: new `ShenWork/Paper2/IntervalChiNegH1PhysicalExit.lean`, or append to `IntervalChiNegH1PhysicalYoungSpatial.lean` if avoiding a new file.
