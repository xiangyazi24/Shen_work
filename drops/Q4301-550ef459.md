# Q4301 — elementary cosine-coefficient decay from pointwise derivative bounds

## Executive answer

Yes: for the `WindowSourceEnvelope` leaf, the clean Lean-feasible route is exactly the elementary one:

```text
closed-interval C^p regularity
+ endpoint Neumann-tower boundary cancellations
+ repeated interval integration by parts
+ uniform sup bound on S^(p)
⇒ |cosineCoeff(S,k)| ≤ C / (kπ)^p
⇒ one summable envelope E_k, uniform over r ∈ [c,T'].
```

This avoids abstract Sobolev/Moser infrastructure entirely. The existing repository already has the important p=2 version in `ShenWork/PDE/IntervalCosineCoeffDecay.lean`: `cosineCoeff_decay`, with the corresponding `fourierCoeff_reflCircle_summable` consequence. The p=2 route is enough for a plain ℓ¹ `WindowSourceEnvelope`. If a later weighted ladder requires higher decay, the same IBP pattern works, but the endpoint compatibility tower becomes a real additional obligation.

The main warning is not the IBP. The main warning is the nonlinear derivative bounds: for real exponents α, γ, high derivatives of `u^α` or `u^γ` at zeros are not automatic. For p=2, some exponents are still manageable; for higher p, either require integer exponents, sufficiently large real exponents, or a positive lower floor on the compact window.

## 1. Elementary IBP lemma

Let

```text
κ_k = k * π,   k ≥ 1,
ĉ_k(S) = ∫_0^1 S(x) cos(κ_k x) dx.
```

For p=2, the standard Neumann IBP is:

```text
∫ S cos(κx)
  = - κ^{-2} ∫ S'' cos(κx),
```

provided

```text
S'(0) = 0,
S'(1) = 0.
```

Therefore

```text
|ĉ_k(S)| ≤ κ^{-2} ∫_0^1 |S''(x)| dx
         ≤ κ^{-2} ‖S''‖_∞.
```

This is exactly the repo’s p=2 lemma shape:

```lean
cosineCoeff_decay
```

in

```text
ShenWork/PDE/IntervalCosineCoeffDecay.lean
```

It assumes closed-interval C² regularity, endpoint Neumann values for `deriv f`, and a uniform bound on the Laplacian coefficient.

### General p

For p integrations by parts:

```text
|ĉ_k(S)| ≤ (kπ)^(-p) ∫_0^1 |S^(p)(x)| dx
         ≤ (kπ)^(-p) ‖S^(p)‖_∞,
```

provided all boundary terms vanish.

The precise boundary tower is:

```text
S^(j)(0) = S^(j)(1) = 0  for every odd j with 1 ≤ j < p.
```

Why only odd j? The cosine-to-sine integrations have sine endpoint factors and vanish automatically. The sine-to-cosine integrations produce endpoint terms involving odd derivatives.

Examples:

```text
p = 1: no boundary condition needed, since sin(0)=sin(kπ)=0.
p = 2: need S'(0)=S'(1)=0.
p = 3: still only need S'(0)=S'(1)=0.
p = 4: need S'(0)=S'(1)=0 and S'''(0)=S'''(1)=0.
```

For a summable envelope, any p > 1 suffices:

```text
∑_{k≥1} 1 / k^p < ∞.
```

So p=2 is the minimal practical target for ℓ¹.

## 2. Lean envelope shape

For a source family `S r x`, with `r ∈ [c,T']`, prove:

```lean
hS0 : ∀ r ∈ Set.Icc c T', |coeff S r 0| ≤ C0
hSdecay : ∀ r ∈ Set.Icc c T', ∀ k : ℕ, 1 ≤ k →
  |coeff S r k| ≤ C2 / (((k : ℝ) * Real.pi) ^ 2)
```

Then define the single envelope:

```lean
def E (k : ℕ) : ℝ :=
  match k with
  | 0 => C0
  | k + 1 => C2 / ((((k + 1 : ℕ) : ℝ) * Real.pi) ^ 2)
```

or, if the coefficient normalization has a factor `2` for nonzero Neumann cosine modes, absorb it into `C2`:

```lean
|unitIntervalNeumannCosineCoeff S k| ≤ 2 * C2 / ((kπ)^2)
```

Then:

```lean
have hE_summable : Summable E := by
  -- finite head plus comparison with C2 / (π^2 * (k+1)^2)
  -- use Mathlib p-series summability or the repo’s existing summability wrappers.
```

The `WindowSourceEnvelope` constructor should then be supplied with:

```lean
⟨E, hE_nonneg, hE_summable, by
  intro r hr k
  cases k with
  | zero => exact hS0 r hr
  | succ k =>
      have hk : 1 ≤ k + 1 := Nat.succ_le_succ (Nat.zero_le k)
      exact hSdecay r hr (k + 1) hk
⟩
```

The important feature is that `E` does not mention `r`. All constants must be taken as sup bounds over the compact window `[c,T']`.

## 3. Logistic branch

Let

```text
L(u) = u * (a - b * u^α)
```

after nonnegativity has removed the truncation. For p=2, use:

```text
(L(u))'  = L'(u) u'
(L(u))'' = L''(u) (u')^2 + L'(u) u''.
```

The endpoint boundary condition needed for p=2 is:

```text
(L(u))'(0) = 0,
(L(u))'(1) = 0.
```

This follows from the Neumann condition:

```text
u_x(0) = u_x(1) = 0.
```

The uniform second-derivative bound is:

```text
‖(L(u))''‖∞
  ≤ sup|L''(u)| * ‖u_x‖∞^2 + sup|L'(u)| * ‖u_xx‖∞.
```

For `0 ≤ u ≤ M`, a usable p=2 constant is:

```text
C_log,2
  = (b * α * (α + 1) * M^(α - 1)) * U1^2
    + (a + b * (α + 1) * M^α) * U2,
```

where

```text
U1 = sup_{r∈[c,T']} ‖u_x(r)‖∞,
U2 = sup_{r∈[c,T']} ‖u_xx(r)‖∞.
```

Caveat for real α: the displayed formula uses `M^(α-1)`. For p=2, this is harmless for α ≥ 1 at the level of boundedness, but formal differentiability of `r ↦ r^α` at zero may still require careful Mathlib facts. For higher p, derivatives contain powers like `u^(α+1-m)`, so if α is not an integer and zeros are allowed, a positive lower floor or exponent restrictions are needed.

The resulting coefficient envelope is:

```text
|s_k^log(r)| ≤ C_log,2 / (kπ)^2,    k ≥ 1.
```

For k=0, use the simple L∞/L¹ bound:

```text
|s_0^log(r)| ≤ ‖L(u(r))‖∞ ≤ M * (a + b * M^α).
```

## 4. Chemotaxis-divergence branch

The chemotaxis flux is

```text
Q = u * R_x / (1 + R)^β.
```

The source branch is the divergence:

```text
S_chem = ∂_x Q.
```

The B-form coefficient often appears as

```text
s_k^chem = kπ * sineInner(Q,k),
```

which is equivalent to the cosine coefficient of `∂_x Q`, using `Q(0)=Q(1)=0`.

For an ℓ¹ envelope, the cleanest p=2 route is to apply the p=2 cosine lemma to

```text
S_chem = Q_x.
```

So we need:

```text
S_chem ∈ C²([0,1]),
(S_chem)'(0) = (S_chem)'(1) = 0,
‖(S_chem)''‖∞ = ‖Q'''‖∞ ≤ C_chem,2.
```

Then:

```text
|s_k^chem(r)| ≤ C_chem,2 / (kπ)^2,    k ≥ 1.
```

### Boundary cancellation for the flux branch

Since Neumann gives

```text
R_x(0)=R_x(1)=0,
```

we have

```text
Q(0)=Q(1)=0.
```

For the p=2 cosine decay of `S_chem = Q_x`, we need:

```text
Q''(0)=Q''(1)=0.
```

This is not automatic from `R_x=0` alone, but it follows under the usual compatibility conditions:

```text
u_x(0)=u_x(1)=0,
R_x(0)=R_x(1)=0,
(u^γ)_x(0)=(u^γ)_x(1)=0,
```

because differentiating

```text
μ R - ν R_xx = ν u^γ
```

gives

```text
μ R_x - ν R_xxx = ν (u^γ)_x.
```

At the boundary, `R_x=0` and `(u^γ)_x=0`, hence

```text
R_xxx(0)=R_xxx(1)=0.
```

Expanding

```text
Q = u * R_x * h(R),    h(R) = (1+R)^(-β),
```

all terms in `Q''` at the boundary contain either `u_x`, `R_x`, or `R_xxx`; hence they vanish under those conditions.

This boundary check is a real load-bearing leaf. It is the chemotaxis analogue of `S'(0)=S'(1)=0` for the logistic branch.

### Uniform bound on Q'''

Let

```text
Uj = sup_{r∈[c,T']} ‖∂_x^j u(r)‖∞,
Rj = sup_{r∈[c,T']} ‖∂_x^j R(r)‖∞.
```

For p=2, the required chem bound is a polynomial expression:

```text
‖Q'''‖∞ ≤ C_chem,2(M, U1,U2,U3, R0,R1,R2,R3,R4, β).
```

Because `R ≥ 0`,

```text
1 + R ≥ 1,
```

so every denominator derivative of `h(R)=(1+R)^(-β)` is bounded by a polynomial in `β` and the derivatives of `R`, without any smallness denominator loss.

A schematic bound is:

```text
C_chem,2 = C(β) * P(U0,U1,U2,U3,R1,R2,R3,R4),
```

where `U0 ≤ M` and `P` is the finite polynomial produced by repeated product and chain rules.

## 5. Elliptic pointwise bounds for R

The interval elliptic resolver gives two derivatives. In pointwise form:

```text
μ R - ν R_xx = ν u^γ.
```

So, recursively:

```text
R_xx = (μ/ν) R - u^γ,
R^(m+2) = (μ/ν) R^(m) - ∂_x^m(u^γ).
```

Thus to bound `R^(p+2)` in sup norm, it is enough to bound:

```text
R, R_x, ..., R^(p)   and   ∂_x^m(u^γ), m≤p.
```

The zeroth and first resolver bounds can come from the already established resolver series/gradient envelopes. Higher derivatives can be obtained either by differentiating the ODE recursively or through cosine multipliers.

For p=2 on `S_chem=Q_x`, we need `Q'''`, hence `R` up to order 4. This requires `u^γ` up to order 2.

Again, for real γ, differentiating `u^γ` at zeros is the main caveat. For p=2, `(u^γ)''` contains terms like

```text
γ u^(γ-1) u_xx + γ(γ-1) u^(γ-2) (u_x)^2.
```

If `1 ≤ γ < 2` and zeros are present, this is not classically bounded without extra structure. The clean classical route is one of:

```text
1. strict positive lower floor on [c,T'];
2. integer γ;
3. γ large enough for the derivative order being used;
4. a weaker nonsmooth/AC argument instead of pointwise C^p IBP.
```

For this elementary C^p coefficient route, one should explicitly carry whichever of these hypotheses is intended.

## 6. Combined source envelope

Let the source be

```text
S = S_log - χ₀ * S_chem.
```

Assume the uniform branch bounds on `[c,T']`:

```text
|s_k^log(r)|  ≤ C_log,2  / (kπ)^2,
|s_k^chem(r)| ≤ C_chem,2 / (kπ)^2,
```

for all `r ∈ [c,T']`, `k ≥ 1`, and zeroth-mode bounds:

```text
|s_0^log(r)| ≤ C_log,0,
|s_0^chem(r)| ≤ C_chem,0.
```

Then define:

```text
C0   = C_log,0 + |χ₀| * C_chem,0,
Cdec = C_log,2 + |χ₀| * C_chem,2.
```

The single envelope is:

```lean
E 0       = C0
E (k + 1) = Cdec / ((((k + 1 : ℕ) : ℝ) * Real.pi) ^ 2)
```

or with an additional factor if the local coefficient normalization uses one.

## 7. Is this a complete substitute for Sobolev/Moser?

For the `WindowSourceEnvelope` leaf: yes.

You do not need abstract `H^r`, Moser composition, or Sobolev algebra estimates if you can provide:

```text
1. pointwise C² bounds for the total source S on [c,T']×[0,1];
2. the first Neumann boundary condition S_x(0)=S_x(1)=0;
3. uniformity of those bounds in r∈[c,T'].
```

For the chemotaxis divergence branch, that means proving `Q'''` is uniformly bounded and `Q''` vanishes at endpoints.

For higher weighted ladders that require more than ℓ¹, use the same route with p=4, p=6, etc., but then the boundary tower and nonlinear derivative bounds become much heavier. In particular, p=4 requires:

```text
S'(0)=S'(1)=0,
S'''(0)=S'''(1)=0,
‖S^(4)‖∞ ≤ C.
```

For `S=Q_x`, that means endpoint vanishing of `Q''` and `Q''''`, plus a bound on `Q^(5)`. That is far more expensive and should not be introduced unless the ladder truly needs it.

## Bottom line

Use the repo’s existing p=2 coefficient-decay lemma as the main engine. Package the source envelope as:

```text
E_0 = zeroth-mode bound,
E_k = C_total / (kπ)^2  for k≥1.
```

The formalization burden is then concentrated in finite, pointwise calculus lemmas:

```text
logistic source: C² bound + first Neumann boundary;
chemDiv source: Q''' bound + Q'' endpoint cancellation;
resolver: R-derivative sup bounds from the elliptic ODE;
real exponents: handle zeros by a floor/integer/exponent hypothesis.
```

That is the Lean-feasible substitute for the abstract Sobolev/Moser stack.
