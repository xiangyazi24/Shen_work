Minimal content of `mapCertificate` should be the usual Picard-map package:


The ρ₊ resolver change removes the **denominator smallness** obstruction, because the resolver source is nonnegative, hence `R ≥ 0`, hence `1 + R ≥ 1`. So: `mapCertificate` is dischargeable unconditionally in the local-existence sense, provided the truncated flux/logistic source has uniform ball Lipschitz bounds after the ρ₊ fix. But it remains a separate producer. The final assembly is not done until this certificate is built.

## 5. Hidden parameter constraints

The headline inputs listed in the prompt are:

```lean
hχ : p.χ₀ < 0
ha : 0 < p.a
hb : 0 < p.b
hα : 1 ≤ p.α
hγ : 1 ≤ p.γ
```

plus the positivity assumptions already inside `CM2Params`, such as positive `μ`, `ν`, and nonnegative/positive `β` as encoded by the record.

There are two important hidden-constraint risks:

### A. Real powers at zero

For map contraction and bounded continuous mild existence, `positivePart u ^ α` and `positivePart u ^ γ` are fine under `α,γ ≥ 1`: on `[0,M]`, the maps are continuous and locally Lipschitz with bounded derivative when the exponent is at least one.

For high-order classical/spectral bootstrapping, noninteger powers at zeros are more delicate. If the source ladder differentiates `u^γ` or `u^α` to high order using pointwise derivative formulas, then either:

```lean
u(t,x) > 0 on the positive-time window
```

is needed as a δ-floor, or the exponents must be restricted to integers / sufficiently high exponents for the derivative order being used.

So if the HSpectral source ladder is based on high-order pointwise derivatives and noninteger `α,γ`, strict positivity is not just a Jensen convenience; it is what makes the real-power derivatives classical at zero. If the source ladder avoids differentiating through zeros via weak/Sobolev or only low-order estimates, then `α,γ ≥ 1` may suffice. This is the main parameter caveat.

### B. Denominator positivity

The denominator no longer creates a smallness condition after the ρ₊ resolver fix:

```lean
(μ - ν ∂ₓₓ) R = ν u₊^γ ≥ 0
```

with Neumann resolvent positivity gives:

```lean
R ≥ 0
1 + R ≥ 1
```

Thus no hypothesis like `sqrt(sum weights) * 2νM^γ < 1` should remain. If such a hypothesis is still present in `mapCertificate`, flux bounds, or HSpectral, it is stale signed-source infrastructure and should be replaced by the ρ₊ resolver positivity lemma.
