## A vs B

### A: derive `bc/hagree` from `HasTimeNeighborhoodSpectralAgreement`

This is the best next direction, but only in the modest form: derive `bc/hagree` from `Hu`, while keeping `henv` as an input for the chosen coefficients. Do not try to prove the compact-window `henv` finite-cover theorem in the same task.

Reason: every PPID source frontier already carries

```lean
Hu : HasTimeNeighborhoodSpectralAgreement D.T D.u
```

For every interior `σ`, `Hu.exists_data σ hσ hσT` gives local restart data and eventual agreement near `σ`. Define a canonical chosen coefficient sequence

```lean
huRestartCoeff Hu σ n :=
  localRestartCoeff a₀ a (σ - offset) n
```

where `a₀`, `a`, and `offset` are chosen from `Hu.exists_data σ ...`. Then prove:

```lean
huRestartCoeff_agree :
  ∀ σ, 0 < σ → σ < T →
    Set.EqOn (intervalDomainLift (u σ))
      (fun x => ∑' n, huRestartCoeff Hu σ n * cosineMode n x)
      (Set.Icc 0 1)
```

by extracting the eventual agreement at `s = σ`. Also prove the sanity lemma

```lean
huRestartCoeff_hbsum :
  ∀ σ, 0 < σ → σ < T →
    Summable (fun n => unitIntervalCosineEigenvalue n * |huRestartCoeff Hu σ n|)
```

using `IntervalMildRegularityBootstrap.restartDuhamelCoeff_eigenvalue_summable`.

This is non-circular and general in `χ₀`; it uses only the u-side restart agreement already carried by the source surface.

### B: package existing power-source K1 machinery

This is not the best next general PPID task. `IntervalResolverPowerK1.powerK1_quadruple_of_subtypeCont` has the right K1-shaped conclusion, but its assumptions are the χ₀=0 weak-ledger assumptions: `hχ0 : p.χ₀ = 0`, `hsrc0`, `hfix`, `hLc_ce`, plus representation and spatial bounds. Some representation/spatial pieces can come from the current envelope surfaces, but `hχ0` and the weak-ledger data are not part of the general `χ≤0/χ<0` PPID source surface.

So B is safe only as a later explicitly χ₀-only adapter. It should not be presented as a non-χ₀ PPID producer. It becomes circular if `hsrc0`, local restart data, or spatial bounds are obtained from downstream `GradientMildClassicalFrontierCoreData`, `IsPaper2ClassicalSolution`, or `HasResolverDirectSpectralData`.
