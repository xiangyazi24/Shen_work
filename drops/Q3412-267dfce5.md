## B. Strict-slab continuity of lifted `intervalDomain.chemotaxisDiv p (u t) (v t)`

### Concrete definition and endpoint warning

```lean

    (u v : intervalDomainPoint → ℝ) (x : intervalDomainPoint) : ℝ :=
```

`intervalDomain.chemotaxisDiv` is instantiated as `intervalDomainChemotaxisDiv` in the `intervalDomain : BoundedDomainData` record.

### What it gives immediately

It should give a wrapper on the **open spatial interior**:

```lean
Set.EqOn
  (Function.uncurry (fun t x => liftDeriv2 u t x))
  (Function.uncurry
    (fun t x =>
      liftTimeDeriv u t x
        + params.χ₀ * intervalDomainLift (intervalDomain.chemotaxisDiv params (u t) (v t)) x
        - intervalDomainLift (u t) x *
            (params.a - params.b * (intervalDomainLift (u t) x) ^ params.α)))
  (Set.Icc a b ×ˢ Set.Ioo (0 : ℝ) 1)
```

The proof is a direct `simp [intervalDomain, intervalDomainLaplacian, intervalDomainLift, liftDeriv2, liftTimeDeriv]` wrapper around `intervalDomain_laplacian_u_eq_time_chem_logistic`, with `t ∈ Icc a b` converted to `0 < t ∧ t < T` and `x ∈ Ioo 0 1` converted to an `intervalDomain.Point` with `x ∈ intervalDomain.inside`.

### Why this is not enough for the Task59 transfer theorem

The transfer theorem demands `EqOn` on:

```lean
Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1
```

including `x = 0` and `x = 1`.

The existing pointwise PDE theorem only applies to:

```lean
x ∈ intervalDomain.inside
```

which is the open spatial interval `Ioo 0 1`.

This is a real endpoint problem, not just a missing `mono` wrapper, because both sides contain endpoint-sensitive ordinary derivatives of zero-extended functions:

- `liftDeriv2 u t x` is `deriv (fun y => deriv (intervalDomainLift (u t)) y) x`.
- `intervalDomainChemotaxisDiv` is `deriv` of the zero-extended flux quotient at `x.1`.

The repo already reflects this caution elsewhere:

- `IntervalChiNegH1LiftDeriv2Transfer.lean` says “Endpoint equality remains an explicit hypothesis.”
- `IntervalDomainL2PDEIntegral.lean` proves `intervalDomainLift_chemDiv_intervalIntegrable_of_regularity` by a.e. equality to a closed derivative-within representative, avoiding endpoint pointwise equality.

### Status for D

**Only an interior EqOn is available from existing declarations. The full closed-spatial-slab EqOn needed by Task59 is not available as-is and has endpoint problems.**

A safe next theorem would be the interior version first:

- `liftDeriv2_eq_physicalRHS_on_strictSlab_interior`

Then either:

1. adjust the downstream transfer to consume an interior EqOn plus an a.e./L¹ route, or
2. prove explicit endpoint identities for both `liftDeriv2` and `intervalDomainChemotaxisDiv` against the chosen closed representative.

I would not try to silently extend the interior EqOn to `Icc 0 1` by density unless the endpoint ordinary-derivative identities are proved. Doing so risks exactly the zero-extension endpoint bug that Task59’s header is trying to isolate.
