## What is still missing

### Missing v patch

There is no audited definition like:

```lean

  else mildChemicalConcentration p u t
```

nor branch lemmas:

Without this, `v_zero`, `v_zero_nonneg`, `v_zeroFace`, and `vx_zeroFace` do not even have a canonical source object.

### Missing v value zero-face

Even after defining `patchedChemical`, proving

```lean
v_zeroFace : ContinuousWithinAt
  (Function.uncurry (fun t x => intervalDomainLift (patchedChemical p u₀ u t) x))
  (Set.Icc 0 b ×ˢ Set.Icc 0 1) (0,x)
```

requires continuity of the elliptic resolver with respect to the source slice as `u(t) → u₀`, uniformly enough in `x`.  Existing resolver direct spectral theorems such as

```lean
resolver_direct_jointSolutionClosed
```

work only on `Ioo 0 T ×ˢ Icc 0 1` from `HasResolverDirectSpectralData`; they do not address the zero face.

### Missing derivative zero-face fields

The hardest fields remain:

```lean
ux_zeroFace
vx_zeroFace
```

Current `P3MoserDxJointContinuity` proves strict-time continuity only.  `InitialTrace` and `patchedSlice_timeContinuousAt_zero` are value/sup-norm statements, not derivative trace statements.  They cannot imply convergence of

```lean
deriv (intervalDomainLift (u t)) x
```

to the initial derivative.  For v, the resolver-gradient trace would require a uniform differentiable resolver-continuity theorem, not just static resolver nonnegativity.

## Circularities and false weak assumptions to avoid

Do not use these to produce the source trace fields:

1. `H1ZeroStartPhysicalPrimitiveDataBefore`: already contains the fields being sought.
2. `H1ZeroStartPhysicalRHSDataBefore` or `H1LiftDeriv2ZeroSlabRepresentativeBefore`: downstream of primitive continuity.
3. sqrt/Young/bounded-before physical outputs: downstream and out of scope.
4. `InitialTrace` alone: value sup-norm right approach only; no pointwise zero slice unless patched, and no derivative trace.
5. strict-time dx theorems alone: no information at `t=0`.
6. pointwise right limits in t: Task134 correctly requires joint `ContinuousWithinAt` on the zero face.

## Partial record worth adding

Add a partial value/sign record to avoid repeatedly threading the easier fields while not claiming C1 closure:

```lean
structure H1ZeroStartPrimitiveValueZeroFaceTrace
    (u v : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  u_zeroFace : ∀ {b : ℝ}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,
    ContinuousWithinAt
      (Function.uncurry (fun t x => intervalDomainLift (u t) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0,x)
  v_zeroFace : ∀ {b : ℝ}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,
    ContinuousWithinAt
      (Function.uncurry (fun t x => intervalDomainLift (v t) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0,x)
  u_zero_pos : ∀ x : intervalDomainPoint, 0 < u 0 x
  v_zero_nonneg : ∀ x : intervalDomainPoint, 0 ≤ v 0 x
```

Then add a simple upgrader that combines this partial record with derivative zero-face fields:

```lean
structure H1ZeroStartPrimitiveDerivativeZeroFaceTrace
    (u v : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  ux_zeroFace : ∀ {b : ℝ}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,

      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0,x)
  vx_zeroFace : ∀ {b : ℝ}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,
    ContinuousWithinAt
      (Function.uncurry (fun t x => deriv (intervalDomainLift (v t)) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0,x)

theorem H1ZeroStartPrimitiveC1ZeroFaceTrace_of_value_derivative
    {u v : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (hv : H1ZeroStartPrimitiveValueZeroFaceTrace u v T)
    (hd : H1ZeroStartPrimitiveDerivativeZeroFaceTrace u v T) :
    H1ZeroStartPrimitiveC1ZeroFaceTrace u v T :=
  { u_zeroFace := hv.u_zeroFace
    v_zeroFace := hv.v_zeroFace
    ux_zeroFace := hd.ux_zeroFace
    vx_zeroFace := hd.vx_zeroFace
```

This is worth coding because it cleanly separates the tractable value/sign front from the derivative frontier.
