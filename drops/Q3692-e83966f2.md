## Why a floor-robust fixed-point theorem is not the next task

A floor-robust theorem would need to replace the following current proof pattern:

```text
u₀ ≥ δ > 0
ρ := δ / 2
ball := closedBall (heatEWA u₀E) ρ
ball-floor := uniformFloor_on_ball hheat : UniformFloor u (δ - ρ) = UniformFloor u (δ / 2)
self-map target: perturbation ≤ ρ = δ / 2
constants: CleanFPConst(..., δ), all using floor / 2
```

This occurs concretely in `picardEWA_clean_fixedPoint_evenReal_prescribedT`:

* hypotheses include `(hfloor : ∀ y, δ ≤ u₀ y)`;
* the conclusion places `u_star` in the ball of radius `δ / 2`;
* the contraction/self-map assumptions are stated with `CleanFPConst.* p normBound δ`;
* the self-map smallness target is `≤ δ / 2`.

The lower estimates ultimately use:

* `SourceFixedPointClean.vdEWA_gDeriv_norm_le`
* `SourceFixedPointClean.chemFluxEWA_norm_le`
* `SourceFixedPointClean.growthEWA_norm_le`
* `FluxLipschitzGraded.chemFluxEWA_lipschitz`
* `FluxLipschitzGraded.growthEWA_lipschitz`
* `FnegLipschitz.negNormConst`
* `FnegLipschitz.negLipConst`

The last two constants contain explicit `(1 / δ)^s`, `(1 / δ)^(s+1)`, etc. The current estimates therefore become worse as the floor goes to zero. Since `PaperPositiveInitialDatum` gives only a datum-specific floor, there is no existing path to a uniform `δ(M)` over all bounded PPID data.

A genuinely floor-robust route would need a new analytic strategy, for example a time-positive smoothing floor or a different ball that does not require uniform positivity down to `t = 0`. Nothing in the current clean prescribed-time fixed-point chain supports that.
