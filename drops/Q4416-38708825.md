# 5. Why `Theorem_1_1 intervalDomain p` alone is not the producer

The Paper 2 theorem has the shape:

```lean
def Theorem_1_1 (D : BoundedDomainData) (p : CM2Params) : Prop :=
  p.χ₀ ≤ 0 →
    (0 < p.a → 0 < p.b →
      ∀ u₀, PaperPositiveInitialDatum D u₀ →
        ∃ Tmax > 0, ∃ u v,
          IsPaper2ClassicalSolution D p Tmax u v ∧
          InitialTrace D u₀ u ∧
          (∀ t, 0 < t → t < Tmax →
            D.supNorm (u t) ≤
              max (D.supNorm u₀) ((p.a / p.b) ^ (1 / p.α))) ∧
          (1 ≤ p.m → IsPaper2GlobalClassicalSolution D p u v)) ∧
    (p.a = 0 → p.b = 0 →
      ∀ u₀, PaperPositiveInitialDatum D u₀ →
        ∃ Tmax > 0, ∃ u v,
          IsPaper2ClassicalSolution D p Tmax u v ∧
          InitialTrace D u₀ u ∧
          (∀ t, 0 < t → t < Tmax →
            D.supNorm (u t) ≤ D.supNorm u₀) ∧
          (1 ≤ p.m → IsPaper2GlobalClassicalSolution D p u v))
```

From this one can immediately project, in either covered parameter branch:

```text
global classical solution
initial trace
```

when `1 ≤ p.m`.

But three gaps remain before obtaining the full current P3.1 target.

## Gap A — eventual boundedness of the same global orbit

The displayed Paper 2 bound is only for

```lean
0 < t ∧ t < Tmax
```

on one selected finite interval. `Proposition_1_2` requires

```lean
∃ M, ∀ᶠ t in atTop, intervalDomain.supNorm (u t) ≤ M
```

for the globally classical witness.

Global classicality by itself does not imply a uniform eventual bound. Re-running local existence at later times is also insufficient unless uniqueness/restart compatibility identifies all restarted solutions with the same `u`.

The repository explicitly records the generic obstruction:

```lean
theorem not_paper2_theorem_1_1_implies_paper3_proposition_1_2 :
  ¬ (∀ D : BoundedDomainData, ∀ p : CM2Params,
      Paper2.Theorem_1_1 D p → Proposition_1_2 D p)
```

For `intervalDomain`, this does not rule out a stronger concrete theorem, but it proves that the bridge is not mere unfolding of `Theorem_1_1`.

The missing interval theorem should have essentially the `eventualSupBound` field above.

## Gap B — initial-datum quantifier mismatch

`Theorem_1_1` consumes:

```lean
PaperPositiveInitialDatum intervalDomain u₀
```

while the existing monolithic producer `NegativeSensitivityGlobalEventualBound` and the interval frontier structure quantify over the weaker:

```lean
PositiveInitialDatum intervalDomain u₀
```

A theorem for paper-positive data does not automatically give a theorem for every merely interior-positive datum.

There are two sound options:

1. prove the stronger Paper 2 API for `PositiveInitialDatum`, preserving the existing producer type; or
2. bypass `NegativeSensitivityGlobalEventualBound` and prove `Proposition_1_2 intervalDomain p` directly for `PaperPositiveInitialDatum`.

Do not use the false converse of `PaperPositiveInitialDatum.toPositive`.

## Gap C — parameter-branch coverage

`Theorem_1_1` exposes two branches:

```text
0 < a and 0 < b
```

or

```text
a = 0 and b = 0.
```

`Proposition_1_2`, as currently stated, assumes only:

```text
χ₀ ≤ 0
1 ≤ m
```

and does not add an `a,b` branch hypothesis. Thus deriving the full current proposition also requires coverage of any remaining nonnegative mixed regimes allowed by `CM2Params`, or an intentional restriction/restatement of the Paper 3 target.

This is independent of the eventual-bound gap.
