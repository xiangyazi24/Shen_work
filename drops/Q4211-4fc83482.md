## Recommendation

Use **(B) as the public API**, but implement it from the two named analytic atoms in (A).

Concretely:

1. Extract the long `hsource_of_grad` proof into one standalone lemma.
2. Reuse the already named successor atom
   `truncatedConjugatePicardIter_succ_window_gradient` as `hkernel_step`.
3. Make one short per-window producer that assembles a
   `TruncatedGradientWindowWiring` and immediately applies
   `truncatedGradientWindow_all`.
4. Chain those producers in the **time-window index**. Each producer still performs
   the existing inner induction in the Picard index `n`.

There is one important correction to the proposed signature: for an arbitrary
common bound `G`, the hypothesis

```lean
hshort : truncWindowB B_F p.χ₀ a hi < 1
```

alone does **not** imply

```lean
truncWindowAffine DT.M A_L A_F B_F p.χ₀ a lo hi G ≤ G.
```

It only says that the affine map has a finite fixed point. The producer should
therefore also take either

```lean
hGdom : truncWindowFixedG DT.M A_L A_F B_F p.χ₀ a lo hi ≤ G
```

or the equivalent `hclosed` inequality. `hGdom` is preferable for chaining: use
one uniform `G` dominating the fixed point of every window. On an equal mesh,
all the window fixed points are definitionally/algebraically equal, so
`hGdom` is just equality followed by `le_rfl`.

## 1. Hide all window-independent constants behind definitions

The source and flux constants should not be recomputed inside each window.
Define them once from `p` and `DT.M`. In the notation of the question:

```lean
namespace ShenWork.Paper2.TruncatedPositiveTimeBootstrap

open ShenWork.Paper2.TruncatedGradientWindow
open ShenWork.Paper2.BFormPositiveDatumNegPart

abbrev TruncPicardU
    (p : CM2Params) (u₀ : intervalDomainPoint → ℝ) :
    ℕ → ℝ → intervalDomainPoint → ℝ :=
  fun n s => truncatedConjugatePicardIter p u₀ n s

def truncPicardGradSource
    (p : CM2Params) (u₀ : intervalDomainPoint → ℝ) :
    ℕ → ℝ → ℝ → ℝ :=
  fun n s y =>
    truncatedLogisticLifted p (TruncPicardU p u₀ n s) y
      - p.χ₀ * deriv
          (truncatedChemFluxLifted p (TruncPicardU p u₀ n s)) y

def truncGradAL (p : CM2Params) (M : ℝ) : ℝ :=
  M * (p.a + p.b * M ^ p.α)

def truncGradGamma (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ,
      (ShenWork.PDE.intervalNeumannResolverGradWeight p k) ^ 2)
    * (2 * (p.ν * M ^ p.γ))

def truncGradValue (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ,
      (ShenWork.PDE.intervalNeumannResolverWeight p k) ^ 2)
    * (2 * (p.ν * M ^ p.γ))

def truncGradH (p : CM2Params) (M : ℝ) : ℝ :=
  p.μ * truncGradValue p M + p.ν * M ^ p.γ

def truncGradAF (p : CM2Params) (M : ℝ) : ℝ :=
  M * truncGradH p M
    + p.β * M * (truncGradGamma p M) ^ 2

def truncGradBF (p : CM2Params) (M : ℝ) : ℝ :=
  truncGradGamma p M

def truncPicardWindowG
    (p : CM2Params) (DT : TruncatedConjugateMildExistenceData p u₀)
    (a lo hi : ℝ) : ℝ :=
  truncWindowFixedG DT.M
    (truncGradAL p DT.M)
    (truncGradAF p DT.M)
    (truncGradBF p DT.M)
    p.χ₀ a lo hi
```

If the committed analytic proof still contains denominator factors such as
`Q_M` and `QDen_M`, put those factors inside `truncGradAF` and `truncGradBF`.
The producer and the chaining theorem remain unchanged. The key invariant is
that these definitions depend only on `p` and `DT.M`, never on the window index.

## 3. Two small affine helper lemmas

The first packages nonnegativity of the fixed point. The second is what allows a
single larger `G` to be used across many windows.

```lean
theorem affine_closes_of_fixed_le
    {A B G : ℝ}
    (hB_nonneg : 0 ≤ B)
    (hB_lt_one : B < 1)
    (hfix : A / (1 - B) ≤ G) :
    A + B * G ≤ G := by
  have hden : 0 < 1 - B := sub_pos.mpr hB_lt_one
  have hA : A ≤ G * (1 - B) :=
    (div_le_iff₀ hden).mp hfix
  nlinarith

theorem truncWindowFixedG_nonneg
    {M A_L A_F B_F chi a lo hi : ℝ}
    (hM : 0 ≤ M)
    (hAL : 0 ≤ A_L)
    (hAF : 0 ≤ A_F)
    (hBF : 0 ≤ B_F)
    (ha_lt_lo : a < lo)
    (hB_lt_one : truncWindowB B_F chi a hi < 1) :
    0 ≤ truncWindowFixedG M A_L A_F B_F chi a lo hi := by
  unfold truncWindowFixedG
  apply div_nonneg
  · unfold truncWindowA
    exact add_nonneg
      (mul_nonneg
        (div_nonneg
          ShenWork.HeatKernelGradientEstimates.heatGradientLinftyLinftyConstant_nonneg
          (Real.sqrt_nonneg _)) hM)
      (mul_nonneg
        (mul_nonneg
          ShenWork.HeatKernelGradientEstimates.heatGradientLinftyLinftyConstant_nonneg
          (mul_nonneg (by norm_num) (Real.sqrt_nonneg _)))
        (add_nonneg hAL (mul_nonneg (abs_nonneg chi) hAF)))
  · linarith
```

The `ha_lt_lo` argument is mathematically useful for the strictly positive
square-root denominator, although the displayed nonnegativity proof only needs
its nonnegativity.

## 4. Exact per-window producer signature

This is the recommended public theorem. It takes a common chain bound `G`, a
short-window condition, domination of the local fixed point by `G`, and the
left-window result from the preceding chain step.

```lean
theorem truncatedPicardIter_iterGrad_window
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (DT : TruncatedConjugateMildExistenceData p u₀)
    (hresolver_small :
      Real.sqrt (∑' k : ℕ,
        (ShenWork.PDE.intervalNeumannResolverWeight p k) ^ 2)
          * (2 * (p.ν * DT.M ^ p.γ)) < 1)
    {a lo hi G : ℝ}
    (ha_pos : 0 < a)
    (ha_lt_lo : a < lo)
    (hlo_le_hi : lo ≤ hi)
    (hhiT : hi ≤ DT.T)
    (hshort :
      truncWindowB (truncGradBF p DT.M) p.χ₀ a hi < 1)
    (hGdom :
      truncPicardWindowG p DT a lo hi ≤ G)
    (hleft :
      ∀ n : ℕ,
        IterGradOnWindow (TruncPicardU p u₀) a lo n G) :
    ∀ n : ℕ,
      IterGradOnWindow (TruncPicardU p u₀) lo hi n G := by
  let AL : ℝ := truncGradAL p DT.M
  let AF : ℝ := truncGradAF p DT.M
  let BF : ℝ := truncGradBF p DT.M
  let U := TruncPicardU p u₀
  let Src := truncPicardGradSource p u₀

  have hAL : 0 ≤ AL := by
    -- one DT.M-based proof, independent of `a`, `lo`, and `hi`
    ...
  have hAF : 0 ≤ AF := by
    ...
  have hBF : 0 ≤ BF := by
    ...
  have hM : 0 ≤ DT.M := DT.hM.le

  have hGwin_nonneg : 0 ≤ truncPicardWindowG p DT a lo hi := by
    simpa [truncPicardWindowG, AL, AF, BF] using
      truncWindowFixedG_nonneg hM hAL hAF hBF ha_lt_lo hshort
  have hG : 0 ≤ G := hGwin_nonneg.trans hGdom

  have hB_nonneg : 0 ≤ truncWindowB BF p.χ₀ a hi := by
    unfold truncWindowB
    positivity
  have hclosed :
      truncWindowAffine DT.M AL AF BF p.χ₀ a lo hi G ≤ G := by
    apply affine_closes_of_fixed_le hB_nonneg hshort
    simpa [truncPicardWindowG, AL, AF, BF] using hGdom

  let W : TruncatedGradientWindowWiring
      p U Src DT.M AL AF BF a lo hi G :=
    { hM_nonneg := hM
      hAL_nonneg := hAL
      hAF_nonneg := hAF
      hBF_nonneg := hBF
      hG_nonneg := hG
      ha_lt_lo := ha_lt_lo
      hlo_le_hi := hlo_le_hi
      hclosed := hclosed
      hleft := hleft
      hbase := by
        have hbase_local :=
          truncatedConjugatePicardIter_zero_window_gradient
            (p := p) (u₀ := u₀) DT U (by intro n s; rfl)
            hAL hAF hBF ha_pos ha_lt_lo hlo_le_hi hhiT hshort
        exact IterGradOnWindow.mono hGdom
          (by simpa [U, AL, AF, BF, truncPicardWindowG] using hbase_local)
      hsource_of_grad := by
        intro n hgrad
        simpa [U, Src, AL, AF, BF] using
          truncPicardGradSource_abs_le_of_iterGrad
            DT hresolver_small ha_pos hhiT n hgrad
      hkernel_step := by
        intro n hsrc
        simpa [U, Src, AL, AF, BF] using
          truncatedConjugatePicardIter_succ_window_gradient
            (p := p) (u₀ := u₀) DT U (by intro n s; rfl)
            Src (by intro n s y; rfl)
            hAL hAF hBF hG
            ha_pos ha_lt_lo hlo_le_hi hhiT n hsrc }

  exact truncatedGradientWindow_all W
```

This is the right abstraction boundary. The final Lipschitz theorem no longer
knows how the source estimate or Duhamel gradient estimate is proved.

A convenience wrapper matching the literal “fixed point for this window” API is
then immediate:

```lean

```

## 5. Generic finite chaining induction

Keep the combinatorial induction independent of the PDE:

```lean
theorem IterGradOnWindow.chain_all
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {G : ℝ}
    (x : ℕ → ℝ) (N : ℕ)
    (hzero :
      ∀ n, IterGradOnWindow U (x 0) (x 1) n G)
    (hstep :
      ∀ k, k < N →
        (∀ n, IterGradOnWindow U (x k) (x (k + 1)) n G) →
        ∀ n, IterGradOnWindow U (x (k + 1)) (x (k + 2)) n G) :
    ∀ k, k ≤ N →
      ∀ n, IterGradOnWindow U (x k) (x (k + 1)) n G := by
  intro k hk
  induction k with
  | zero =>
      simpa using hzero
  | succ k ih =>
      have hkN : k < N := Nat.lt_of_succ_le hk
      have hk_le_N : k ≤ N :=
        Nat.le_trans (Nat.le_succ k) hk
      simpa [Nat.succ_eq_add_one, Nat.add_assoc] using
        hstep k hkN (ih hk_le_N)

theorem IterGradOnWindow.chain_last
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {G : ℝ}
    (x : ℕ → ℝ) (N : ℕ)
    (hzero :
      ∀ n, IterGradOnWindow U (x 0) (x 1) n G)
    (hstep :
      ∀ k, k < N →
        (∀ n, IterGradOnWindow U (x k) (x (k + 1)) n G) →
        ∀ n, IterGradOnWindow U (x (k + 1)) (x (k + 2)) n G) :
    ∀ n, IterGradOnWindow U (x N) (x (N + 1)) n G := by
  exact IterGradOnWindow.chain_all x N hzero hstep N le_rfl
```

The theorem `chain_all` records the bound on every covering subwindow;
`chain_last` gives the neighborhood of the target time.

## 6. PDE-specific chain, with the base supplied by the near-zero profile

This is the exact theorem that connects the near-zero profile to the per-window
producer:

```lean
theorem truncatedPicardIter_iterGrad_chain
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (DT : TruncatedConjugateMildExistenceData p u₀)
    (hresolver_small :
      Real.sqrt (∑' k : ℕ,
        (ShenWork.PDE.intervalNeumannResolverWeight p k) ^ 2)
          * (2 * (p.ν * DT.M ^ p.γ)) < 1)
    (x : ℕ → ℝ) (N : ℕ) {G : ℝ}
    (hx0_pos : 0 < x 0)
    (hnear0 :
      ∀ n,
        IterGradLeftProfile
          (TruncPicardU p u₀)
          DT.M
          (truncGradAL p DT.M)
          (truncGradAF p DT.M)
          (truncGradBF p DT.M)
          p.χ₀ (x 1) n)
    (hprofile_le_G :
      truncLeftProfile
        DT.M
        (truncGradAL p DT.M)
        (truncGradAF p DT.M)
        (truncGradBF p DT.M)
        p.χ₀ (x 1) (x 0) ≤ G)
    (hgeom : ∀ k, k < N →
      0 < x k ∧
      x k < x (k + 1) ∧
      x (k + 1) ≤ x (k + 2) ∧
      x (k + 2) ≤ DT.T)
    (hshort : ∀ k, k < N →
      truncWindowB
        (truncGradBF p DT.M) p.χ₀ (x k) (x (k + 2)) < 1)
    (hGdom : ∀ k, k < N →
      truncPicardWindowG p DT (x k) (x (k + 1)) (x (k + 2)) ≤ G) :
    ∀ n,
      IterGradOnWindow
        (TruncPicardU p u₀) (x N) (x (N + 1)) n G := by
  have hbase :
      ∀ n,
        IterGradOnWindow
          (TruncPicardU p u₀) (x 0) (x 1) n G :=
    IterGradOnWindow.of_left_profile
      DT.hM.le hx0_pos hnear0 hprofile_le_G

  apply IterGradOnWindow.chain_last x N hbase
  intro k hk hleft
  rcases hgeom k hk with ⟨hk_pos, hk_lt, hk1_le, hk2T⟩
  exact truncatedPicardIter_iterGrad_window
    DT hresolver_small
    hk_pos hk_lt hk1_le hk2T
    (hshort k hk) (hGdom k hk) hleft
```

This theorem contains no source algebra and no Duhamel analysis. Its induction is
entirely structural.

## 7. Equal-mesh specialization covering `(ε,t]`

Choose `N ≥ 1` and

```text
d = (t - ε) / N,
x k = ε - d + k d.
```

Assume

```text
0 < d,
d < ε,
2d < δ*,
t ≤ DT.T.
```

Then

```text
x 0     = ε - d > 0,
x 1     = ε,
x (N+1) = t,
x (k+1) - x k = d,
x (k+2) - x k = 2d.
```

Define the one uniform chain bound

```lean
let G : ℝ :=
  truncWindowFixedG
    DT.M
    (truncGradAL p DT.M)
    (truncGradAF p DT.M)
    (truncGradBF p DT.M)
    p.χ₀ 0 d (2 * d)
```

For every `k < N`, unfolding `x`, `truncWindowA`, `truncWindowB`, and
`truncWindowFixedG` gives

```lean
have hG_window (k : ℕ) :
    truncPicardWindowG p DT (x k) (x (k + 1)) (x (k + 2)) = G := by
  simp [x, G, truncPicardWindowG,
    truncWindowFixedG, truncWindowA, truncWindowB]
  ring
```

and similarly the single shortness inequality at span `2d` gives all window
shortness inequalities:

```lean
have hshort_window (k : ℕ) :
    truncWindowB (truncGradBF p DT.M) p.χ₀ (x k) (x (k + 2)) < 1 := by
  simpa [x, truncWindowB] using hshort_two_d
```

Thus `hGdom k hk` is simply

```lean
rw [hG_window k]
```

followed by `exact le_rfl`.

The near-zero profile is used only once, on `(0,ε]`:

```lean
have hbase : ∀ n,
    IterGradOnWindow (TruncPicardU p u₀) (ε - d) ε n G := by
  simpa [x] using
    IterGradOnWindow.of_left_profile
      DT.hM.le
      (by linarith : 0 < ε - d)
      hnear0_at_epsilon
      hprofile_at_epsilon_minus_d_le_G
```

Applying `truncatedPicardIter_iterGrad_chain` yields

```lean
∀ n,
  IterGradOnWindow (TruncPicardU p u₀) (t - d) t n G.
```

That is the required uniform iterate gradient bound on a left neighborhood of
`t`. Moreover, `IterGradOnWindow.chain_all` gives all subwindows

```text
[ε, ε+d], [ε+d, ε+2d], ..., [t-d,t],
```

so their union covers `[ε,t]`.

## Bottom line

- Do not leave the long analytic fields inline in the final positive-time theorem.
- Extract the source field as one named lemma; keep the existing named kernel atom.
- Expose one per-window producer.
- Add `hGdom` (or equivalently `hclosed`); `hshort` alone is insufficient for an
  arbitrary chain-wide `G`.
- Use a generic `chain_all` induction in the window index.
- On an equal mesh, all constants and all local fixed points are identical because
  `A_L`, `A_F`, and `B_F` depend only on `DT.M`, while the window formulas depend
  only on the differences `lo-a` and `hi-a`.
