## Why `SourceIBP` is the right theorem, not `InitialIBP`

`IntervalFullKernelInitialIBP.lean` already defines the conjugate kernel and proves the left endpoint cancellation:

```lean
noncomputable def intervalNeumannConjugateKernel (t x y : ℝ) : ℝ

theorem conjugateKernel_at_zero {t : ℝ} (x : ℝ) :
  intervalNeumannConjugateKernel t x 0 = 0

theorem hasDerivAt_conjugateKernel_snd {t : ℝ} (ht : 0 < t) (x y : ℝ) :
  HasDerivAt (fun y : ℝ => intervalNeumannConjugateKernel t x y)
    ((∑' k : ℤ, deriv (fun u : ℝ => heatKernel t u) (x - y + 2 * (k : ℝ)))
      + (∑' k : ℤ, deriv (fun u : ℝ => heatKernel t u) (x + y + 2 * (k : ℝ)))) y
```

It also proves the older bound:

```lean
theorem intervalFullCoupledDuhamel_grad_initial_bound ... :
  |deriv (fun z : ℝ => intervalFullSemigroupOperator t u₀ z) x| ≤ G_init
```

but that theorem is not the next C¹-approach equality. It is a bound, and it carries the special value hypothesis `hu₀_one : u₀ 1 = 0` to kill the right boundary term.

`IntervalFullKernelSourceIBP.lean` adds the missing right endpoint cancellation:

```lean
theorem conjugateKernel_at_one {t : ℝ} (ht : 0 < t) (x : ℝ) :
  intervalNeumannConjugateKernel t x 1 = 0
```

Then it proves the public equality above. This is stronger and cleaner for the C¹ initial-leg route: no endpoint value condition on `Q` is required, because both conjugate-kernel boundary values vanish.

## Endpoint hypotheses and the raw lift caveat

The theorem has no boundary condition such as `Q 0 = 0` or `Q 1 = 0`. However, it does require

```lean
hQ_deriv : ∀ y ∈ Set.uIcc (0 : ℝ) 1, HasDerivAt Q (Q' y) y
```

This is a **global** `HasDerivAt`, including at `y = 0` and `y = 1`, not a one-sided derivative. Therefore, using `Q := intervalDomainLift u₀` directly is usually too strong for positive interval data, because `intervalDomainLift u₀` is the zero extension outside `[0,1]` and may jump at the endpoints.

The safe source-side pattern is:

1. choose a globally C¹ representative `Q : ℝ → ℝ` agreeing with `intervalDomainLift u₀` on `[0,1]`;
2. apply the SourceIBP theorem to `Q` and `Q'`;
3. use semigroup congruence on `[0,1]` to replace `S_N(t) Q` by `S_N(t) (intervalDomainLift u₀)`.

A small compile-now wrapper for this pattern is:

```lean
import ShenWork.PDE.IntervalFullKernelSourceIBP

open MeasureTheory Filter
open scoped Topology

open ShenWork.IntervalDomain
open ShenWork.IntervalNeumannFullKernel

namespace ShenWork.IntervalNeumannFullKernel

noncomputable section

/-- The full Neumann semigroup reads the source only on `[0,1]`. -/
theorem intervalFullSemigroupOperator_congr_on_Icc
    {f g : ℝ → ℝ}
    (hfg : ∀ y ∈ Set.Icc (0 : ℝ) 1, f y = g y)
    (t x : ℝ) :
    intervalFullSemigroupOperator t f x =
      intervalFullSemigroupOperator t g x := by
  unfold intervalFullSemigroupOperator
  apply MeasureTheory.integral_congr_ae
  have hmem : ∀ᵐ y ∂(intervalMeasure 1), y ∈ Set.Icc (0 : ℝ) 1 := by
    simp only [intervalMeasure, intervalSet]
    exact (MeasureTheory.ae_restrict_iff' measurableSet_Icc).mpr
      (Filter.Eventually.of_forall fun y hy => hy)
  filter_upwards [hmem] with y hy
  rw [hfg y hy]

/-- SourceIBP applied through an arbitrary global C¹ representative `Q` agreeing
with the desired source `f` on `[0,1]`.  This is often the right way to use the
IBP theorem for `intervalDomainLift u₀`, whose hard zero extension need not be
globally differentiable at the endpoints. -/
theorem deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_integral_of_Icc_repr
    {t : ℝ} (ht : 0 < t)
    {f Q Q' : ℝ → ℝ}
    (hfQ : ∀ y ∈ Set.Icc (0 : ℝ) 1, f y = Q y)
    (hQ_meas : AEStronglyMeasurable Q (intervalMeasure 1))
    {CQ : ℝ} (hQ_bound : ∀ y, |Q y| ≤ CQ)
    (hQ_deriv : ∀ y ∈ Set.uIcc (0 : ℝ) 1, HasDerivAt Q (Q' y) y)
    (hQ'_int : IntervalIntegrable Q' MeasureTheory.volume 0 1)
    (x : ℝ) :
    deriv (fun z : ℝ => intervalFullSemigroupOperator t f z) x =
      -(∫ y in (0 : ℝ)..1, Q' y * intervalNeumannConjugateKernel t x y) := by
  have hfun :
      (fun z : ℝ => intervalFullSemigroupOperator t f z) =
        fun z : ℝ => intervalFullSemigroupOperator t Q z := by
    funext z
    exact intervalFullSemigroupOperator_congr_on_Icc hfQ t z
  rw [hfun]
  exact deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_source_integral
    ht hQ_meas hQ_bound hQ_deriv hQ'_int x

#print axioms intervalFullSemigroupOperator_congr_on_Icc
#print axioms deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_integral_of_Icc_repr

end

end ShenWork.IntervalNeumannFullKernel
```

This wrapper is not necessary to use the theorem, but it is the smallest safe source-side patch if the team wants a convenience theorem for initial-leg work.
