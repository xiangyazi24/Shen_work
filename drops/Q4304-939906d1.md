# Q4304 — pointwise derivative bounds for the chemotaxis flux

## Verdict

Yes: the flux derivative ladder can be done entirely in a pointwise `C^r` / sup-norm style, without Sobolev/Moser infrastructure. The clean route is:

```text
u ∈ C^r,  δ ≤ u ≤ M
  ⇒ u^γ ∈ C^r                      -- Faà di Bruno with δ-floor
  ⇒ R[u] ∈ C^{r+2}                 -- 1D elliptic +2 gain
  ⇒ g = R_x · (1 + R)^(-β) ∈ C^{r+1}
  ⇒ Q = u · g ∈ C^r
```

For the source coefficient ladder, remember that the chemotaxis source is usually `∂ₓQ`. Thus if the source needs `p` spatial derivatives for an IBP decay lemma, the flux needs `p+1` derivatives. Equivalently, `Q ∈ C^r` gives `∂ₓQ ∈ C^{r-1}`, so the chemotaxis source coefficient decay coming from the flux part is one derivative shallower than the flux regularity.

The best formalization pattern is to define explicit finite combinatorial constants and prove one reusable lemma for each operation:

```lean
powerDerivBound
resolverDerivBound
invDenomCompDerivBound
gDerivBound
fluxDerivBound
```

The constants can be ugly; they only need to be finite, nonnegative, monotone in the input derivative bounds, and uniform on the positive-time window.

## Notation for derivative bounds

Work on one compact positive-time window and fix a spatial slice. Let

```text
B_j  := sup_x |∂ₓ^j u(x)|,      j = 0..r
A_j  := sup_x |∂ₓ^j R(x)|,      j = 0..r+2
```

with

```text
0 < δ ≤ u(x) ≤ M,
R(x) ≥ 0,
β ≥ 0,
γ ≥ 1.
```

The lower floor `δ` is only needed for noninteger/real-power derivatives of `u^γ` when the derivative order can exceed `γ`. It is not needed for the denominator because `R ≥ 0` gives `1 + R ≥ 1`.

## 1. Leibniz bound for `Q = u · g`

For every `r ≥ 0`,

```text
∂ₓ^r Q = ∑_{j=0}^r binom(r,j) (∂ₓ^j u) (∂ₓ^{r-j} g).
```

Therefore, if

```text
G_m := sup_x |∂ₓ^m g(x)|,
```

then

```text
‖Q^{(r)}‖∞ ≤ ∑_{j=0}^r binom(r,j) B_j · G_{r-j}.
```

Lean shape:

```lean
def fluxDerivConst (r : ℕ) (B G : ℕ → ℝ) : ℝ :=
  ∑ j in Finset.range (r + 1),
    (Nat.choose r j : ℝ) * B j * G (r - j)
```

Target lemma:

```lean
lemma flux_deriv_abs_le
    {u g : ℝ → ℝ} {r : ℕ} {B G : ℕ → ℝ}
    (hu : ∀ j ≤ r, ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv j u x| ≤ B j)
    (hg : ∀ j ≤ r, ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv j g x| ≤ G j)
    (hLeibniz :
      ∀ x ∈ Icc (0:ℝ) 1,
        iterDeriv r (fun x => u x * g x) x =
          ∑ j in Finset.range (r + 1),
            (Nat.choose r j : ℝ) * iterDeriv j u x * iterDeriv (r-j) g x) :
    ∀ x ∈ Icc (0:ℝ) 1,
      |iterDeriv r (fun x => u x * g x) x| ≤ fluxDerivConst r B G := by
  -- `abs_sum_le_sum_abs`, `abs_mul`, `hu`, `hg`, nonnegativity of constants.
```

If the repo does not have a convenient `iterDeriv` API, state this per order for the needed concrete orders first. For the coefficient ladder, concrete `r = 1,2,3,4` often suffices and is much faster than building a full generic calculus stack.

## 2. Bound for `g = R_x · h(R)`, `h(z)=(1+z)^(-β)`

Let

```text
H_m := sup_x |∂ₓ^m (h ∘ R)(x)|.
```

Then Leibniz gives

```text
g^{(m)} = ∑_{j=0}^m binom(m,j) R^{(j+1)} · (h∘R)^{(m-j)}.
```

So

```text
G_m ≤ ∑_{j=0}^m binom(m,j) A_{j+1} · H_{m-j}.
```

Lean constant:

```lean
def gDerivConst (m : ℕ) (A H : ℕ → ℝ) : ℝ :=
  ∑ j in Finset.range (m + 1),
    (Nat.choose m j : ℝ) * A (j + 1) * H (m - j)
```

## 3. Faà di Bruno bound for `(1+R)^(-β)`

For

```text
h(z) = (1+z)^(-β),
```

its `q`th derivative is

```text
h^{(q)}(z) = (-1)^q · β(β+1)...(β+q-1) · (1+z)^(-β-q).
```

Because `R ≥ 0`, we have `1+R ≥ 1`, hence

```text
|h^{(q)}(R)| ≤ β^{↑q},
```

where

```text
β^{↑q} := ∏_{i=0}^{q-1} |β + i|.
```

Now Faà di Bruno gives, for `m ≥ 1`,

```text
∂ₓ^m (h∘R)
 = ∑ over κ₁,...,κ_m with ∑ j κ_j = q and ∑ j·κ_j = m
     m! / (∏ κ_j! · ∏ (j!)^{κ_j})
     · h^{(q)}(R)
     · ∏_{j=1}^m (R^{(j)})^{κ_j}.
```

Thus define

```text
H_0 := 1
```

because `|(1+R)^(-β)| ≤ 1`, and for `m ≥ 1`,

```text
H_m := ∑_{κ : partition data of m}
         coeff(m,κ) · risingAbs(β, q(κ)) · ∏_{j=1}^m A_j^{κ_j}.
```

A Lean-friendly version avoids quotienting by partitions by summing over all functions `κ : Fin m → ℕ` and filtering by the two constraints. For `m = 0`, use `1`.

```lean
def risingAbs (β : ℝ) : ℕ → ℝ
  | 0 => 1
  | q+1 => risingAbs β q * |β + q|

-- Schematic; implementation can use `Fin m` or `Fin (m+1)` carefully.
def fdbCoeff (m : ℕ) (κ : Fin m → ℕ) : ℝ :=
  (Nat.factorial m : ℝ) /
    ((∏ j : Fin m, (Nat.factorial (κ j) : ℝ)) *
     (∏ j : Fin m, ((Nat.factorial (j.val + 1) : ℝ) ^ κ j)))

def fdbWeight (m : ℕ) (κ : Fin m → ℕ) : Prop :=
  (∑ j : Fin m, (j.val + 1) * κ j) = m

def fdbOrder (m : ℕ) (κ : Fin m → ℕ) : ℕ :=
  ∑ j : Fin m, κ j

def denomCompConst (β : ℝ) (A : ℕ → ℝ) : ℕ → ℝ
  | 0 => 1
  | m+1 =>
      ∑ κ in (Finset.univ.filter (fun κ : Fin (m+1) → ℕ =>
        fdbWeight (m+1) κ)),
        fdbCoeff (m+1) κ * risingAbs β (fdbOrder (m+1) κ) *
          ∏ j : Fin (m+1), A (j.val + 1) ^ κ j
```

The exact encoding of `κ` can be adjusted to what is easiest in Lean. The important point is that this is a finite sum, so no analytic summability is involved.

Then prove:

```lean
lemma denom_comp_deriv_abs_le
    {R : ℝ → ℝ} {β : ℝ} {A : ℕ → ℝ} {m : ℕ}
    (hRnonneg : ∀ x ∈ Icc (0:ℝ) 1, 0 ≤ R x)
    (hA : ∀ j, 1 ≤ j → j ≤ m →
      ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv j R x| ≤ A j)
    -- plus chain rule/Faà di Bruno identity for this m
    : ∀ x ∈ Icc (0:ℝ) 1,
        |iterDeriv m (fun x => (1 + R x) ^ (-β)) x| ≤
          denomCompConst β A m
```

For the first implementation pass, it may be easier to prove explicit versions for `m = 0,1,2,3,4` by repeated `ring_nf`, product rule, and chain rule. The generic finite-sum version is conceptually clean but slower to formalize.

## 4. Real-power derivative bound for `u^γ` with a `δ` floor

For `F = u^γ`, Faà di Bruno gives

```text
F^{(p)} = ∑_{κ} coeff(p,κ) · (γ)_{q}^{falling} · u^{γ-q} · ∏_{j=1}^p (u^{(j)})^{κ_j}.
```

Here

```text
(γ)_q^{falling} = γ(γ-1)...(γ-q+1).
```

With

```text
δ ≤ u ≤ M,
```

we bound

```text
|u^{γ-q}| ≤ max(δ^{γ-q}, M^{γ-q}).
```

This handles negative powers when `q > γ`.

Define:

```lean
def fallingAbs (γ : ℝ) : ℕ → ℝ
  | 0 => 1
  | q+1 => fallingAbs γ q * |γ - q|

def floorPowerBound (δ M γ : ℝ) (q : ℕ) : ℝ :=
  max (δ ^ (γ - q)) (M ^ (γ - q))

-- same finite Faà di Bruno index as above
def powerCompConst (γ δ M : ℝ) (B : ℕ → ℝ) : ℕ → ℝ
  | 0 => M ^ γ
  | p+1 =>
      ∑ κ in (Finset.univ.filter (fun κ : Fin (p+1) → ℕ =>
        fdbWeight (p+1) κ)),
        fdbCoeff (p+1) κ * fallingAbs γ (fdbOrder (p+1) κ) *
          floorPowerBound δ M γ (fdbOrder (p+1) κ) *
          ∏ j : Fin (p+1), B (j.val + 1) ^ κ j
```

Target:

```lean
lemma power_comp_deriv_abs_le
    {u : ℝ → ℝ} {γ δ M : ℝ} {B : ℕ → ℝ} {p : ℕ}
    (hδ : 0 < δ)
    (hfloor : ∀ x ∈ Icc (0:ℝ) 1, δ ≤ u x)
    (hceil : ∀ x ∈ Icc (0:ℝ) 1, u x ≤ M)
    (hB : ∀ j, 1 ≤ j → j ≤ p →
      ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv j u x| ≤ B j)
    -- plus Faà di Bruno identity
    : ∀ x ∈ Icc (0:ℝ) 1,
        |iterDeriv p (fun x => (u x) ^ γ) x| ≤
          powerCompConst γ δ M B p
```

For `p=0`, the bound is simply `|u^γ| ≤ M^γ` when `γ ≥ 0`, `0 ≤ u ≤ M`. If using the δ-floor uniformly, `max (δ^γ) (M^γ)` is also acceptable.

## 5. Elliptic derivative bounds for `R`

The cleanest pointwise route is a hybrid:

1. Use existing Green/resolvent kernel estimates for the low derivatives:

```text
A_0 ≥ ‖R‖∞,
A_1 ≥ ‖R_x‖∞.
```

2. Use the ODE recursively for higher derivatives.

The equation is

```text
μ R - ν R'' = ν F,
F = u^γ.
```

So

```text
R'' = (μ/ν) R - F.
```

For `m ≥ 2`, differentiating `m-2` times gives

```text
R^{(m)} = (μ/ν) R^{(m-2)} - F^{(m-2)}.
```

Thus, if

```text
F_p := ‖F^{(p)}‖∞,
```

then

```text
A_m ≤ (μ/ν) A_{m-2} + F_{m-2},       m ≥ 2.
```

Define:

```lean
def resolverDerivConst (μ ν : ℝ) (A0 A1 : ℝ) (F : ℕ → ℝ) : ℕ → ℝ
  | 0 => A0
  | 1 => A1
  | m+2 => (μ / ν) * resolverDerivConst μ ν A0 A1 F m + F m
```

Assuming `μ,ν > 0`, all constants are nonnegative if `A0,A1,F_m ≥ 0`.

For the low bounds, typical choices are:

```text
A_0 = (ν/μ) · F_0
A_1 = C_res_grad · F_0
```

where `C_res_grad` is the already-proved interval resolver gradient kernel bound. If the repo already has stronger cosine-multiplier bounds for `R_x`, use those.

The pointwise elliptic lemma shape:

```lean
lemma resolver_deriv_abs_le
    {R F : ℝ → ℝ} {m : ℕ}
    (hODE : ∀ x ∈ Icc (0:ℝ) 1,
      μ * R x - ν * iterDeriv 2 R x = ν * F x)
    (hODE_derivs : ∀ q ≤ m-2, ... derivative of ODE ...)
    (hA0 : ∀ x ∈ Icc (0:ℝ) 1, |R x| ≤ A0)
    (hA1 : ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv 1 R x| ≤ A1)
    (hF : ∀ q ≤ m-2, ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv q F x| ≤ Fconst q) :
    ∀ x ∈ Icc (0:ℝ) 1,
      |iterDeriv m R x| ≤ resolverDerivConst μ ν A0 A1 Fconst m
```

Again, concrete-order lemmas are often simpler than generic `iterDeriv` calculus.

## 6. Consolidated constants for `g` and `Q`

Given:

```text
F_p = powerCompConst γ δ M B p
A_m = resolverDerivConst μ ν A0 A1 F m
H_m = denomCompConst β A m
G_m = ∑_{j=0}^m binom(m,j) A_{j+1} H_{m-j}
Q_r = ∑_{j=0}^r binom(r,j) B_j G_{r-j}
```

we get:

```text
‖Q^{(r)}‖∞ ≤ Q_r.
```

Expanded:

```lean
def chemGConst (m : ℕ) (A H : ℕ → ℝ) : ℝ :=
  ∑ j in Finset.range (m + 1),
    (Nat.choose m j : ℝ) * A (j + 1) * H (m - j)

def chemFluxConst (r : ℕ) (B G : ℕ → ℝ) : ℝ :=
  ∑ j in Finset.range (r + 1),
    (Nat.choose r j : ℝ) * B j * G (r - j)
```

Then:

```lean
lemma chemFlux_deriv_abs_le_Cr
    {u R : ℝ → ℝ} {r : ℕ}
    (hu_bounds : ... |u^{(j)}| ≤ B j ...)
    (hR_bounds : ... |R^{(j)}| ≤ A j ...)
    (hR_nonneg : ∀ x ∈ Icc (0:ℝ) 1, 0 ≤ R x)
    -- derivative identities for h(R), g, and Q
    : ∀ x ∈ Icc (0:ℝ) 1,
        |iterDeriv r
          (fun x => u x * (iterDeriv 1 R x * (1 + R x) ^ (-β))) x|
          ≤ chemFluxConst r B (fun m => chemGConst m A (denomCompConst β A))
```

In practice, you will likely write this as a local theorem parameterized by already-proved derivative-bound packages:

```lean
structure CkSupBounds (f : ℝ → ℝ) (r : ℕ) where
  C : ℕ → ℝ
  nonneg : ∀ j ≤ r, 0 ≤ C j
  bound : ∀ j ≤ r, ∀ x ∈ Icc (0:ℝ) 1, |iterDeriv j f x| ≤ C j
```

Then the product/composition lemmas consume and produce `CkSupBounds` packages.

## 7. How this feeds IBP coefficient decay

For the source

```text
S = logistic(u) - χ ∂ₓQ,
```

if you need `S^{(p)}` bounded, then you need:

```text
logistic(u) ∈ C^p,
Q ∈ C^{p+1}.
```

The flux constant for `Q^{(p+1)}` is the chemotaxis part of the source derivative bound.

A typical source derivative constant is:

```text
Source_p ≤ Logistic_p + |χ| · Q_{p+1}.
```

Then the elementary cosine IBP lemma gives, depending on boundary conditions:

```text
|cosCoeff(S,k)| ≤ Source_p / (kπ)^p.
```

For the existing repo `C²` Neumann quadratic lemma, `p=2` and the required boundary condition is `S'(0)=S'(1)=0`; the constant is a bound on `S''`.

For higher-order decay, after `p` integrations by parts, the boundary conditions are:

```text
S^{(1)}(0)=S^{(1)}(1)=0,
S^{(3)}(0)=S^{(3)}(1)=0,
...
S^{(p-1)}(0)=S^{(p-1)}(1)=0    when p is even,
```

and for odd `p`, the boundary tower alternates accordingly, but cosine coefficients usually benefit most cleanly from even `p=2m`. For `p=2m`, require all odd derivatives up to `2m-1` vanish at the endpoints. This is the Neumann tower.

## 8. Practical Lean recommendation

Do not start with the full generic Faà di Bruno theorem unless absolutely necessary.

For the immediate source ladder, implement concrete-order bound packages:

```text
C0, C1, C2, C3, C4, maybe C5
```

and prove product/chain/quotient formulas explicitly for those orders. This is much more Lean-feasible than a fully generic partition-indexed Faà di Bruno development.

The generic constants above are still useful as the mathematical specification. But the coding path should be:

1. Define a `CkSupBounds4` or `CkSupBounds5` structure.
2. Prove product bounds up to that order.
3. Prove `rpow_pos_floor_deriv_bounds` up to that order using `δ ≤ u`.
4. Prove resolver bounds using the ODE recursion.
5. Prove denominator bounds for `(1+R)^(-β)` using `R ≥ 0`.
6. Combine into `chemFlux_deriv_abs_le_orderN`.
7. Feed the resulting `Source_p` into the existing `IntervalCosineCoeffDecay.cosineCoeff_decay` / higher-order IBP lemma.

This route is a complete substitute for Sobolev/Moser for the coefficient ladder: all estimates are finite sums, pointwise derivative rules, compact-window sup bounds, and elementary interval IBP.
