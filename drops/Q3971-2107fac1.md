# Q3971: continuity of the physical Laplacian at interior points

## Short answer

The right Lean cut is to prove continuity of the two plain real-coordinate functions separately:

```lean
Rplain z   = ∑' k, (intervalNeumannResolverCoeff p u k).re * cos(kπz)
Splain z   = p.ν * (positivePart (intervalDomainLift u z)) ^ p.γ
LapPlain z = p.μ * Rplain z - Splain z
```

Then show that, near an interior point `x ∈ Ioo 0 1`, your piecewise function

```lean
resolverLapPhysicalReal p u z
```

is eventually equal to `LapPlain z`. The outside branch is irrelevant because `Ioo 0 1 ∈ 𝓝 x`.

The most useful Mathlib lemma is:

```lean
continuous_tsum
```

from

```lean
import Mathlib.Analysis.Normed.Group.FunctionSeries
```

It has the shape:

```lean
theorem continuous_tsum
    (hf : ∀ i, Continuous (f i))
    (hu : Summable u)
    (hfu : ∀ n x, ‖f n x‖ ≤ u n) :
    Continuous fun x => ∑' n, f n x
```

For the resolver value, take

```lean
f k z = (intervalNeumannResolverCoeff p u k).re * unitIntervalCosineMode k z
u k   = |(intervalNeumannResolverCoeff p u k).re|.
```

The bound is immediate from `|cos| ≤ 1`. The summability of `u` follows from the repo theorem

```lean
resolver_cosineSeries_summable_of_sourceL2
```

applied at `x = 0`, because `cos(kπ * 0) = 1`. That theorem already packages the Cauchy-Schwarz/AM-GM argument using source `ℓ²` and the resolvent weight `ℓ²`; you do **not** need to reproach `∑ |a_k|`.

## Important correction to the analytic route

Do **not** use

```text
|c_k| ≤ |a_k| / μ
```

together with `a ∈ ℓ²`: that only gives `c ∈ ℓ²`, not `c ∈ ℓ¹`. The uniform convergence majorant should be obtained from

```text
c_k = a_k / (μ + λ_k),     a ∈ ℓ²,     (1 / (μ + λ_k)) ∈ ℓ²,
```

so

```text
∑ |c_k| ≤ ‖a‖₂ * ‖1/(μ+λ_k)‖₂ < ∞.
```

In this repo, that argument is already embodied by `resolver_cosineSeries_summable_of_sourceL2`.

## Suggested Lean skeleton

This is the file-level proof shape I would add near `IntervalResolverWeakLapBound.lean`. The local theorem names may need tiny namespace adjustments, but the hypotheses and Mathlib lemmas are the right ones.

```lean
import ShenWork.Paper2.IntervalResolverWeakBounds
import ShenWork.Paper2.IntervalResolverWeakLapBound
import Mathlib.Analysis.Normed.Group.FunctionSeries

open MeasureTheory intervalIntegral
open ShenWork.IntervalDomain ShenWork.PDE ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open ShenWork.HeatKernelGradientEstimates
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

/-- Plain real-coordinate resolver value series.  This avoids dependent interval-point
bookkeeping while proving continuity. -/
def resolverValueSeriesReal (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun z => ∑' k : ℕ,
    (intervalNeumannResolverCoeff p u k).re * unitIntervalCosineMode k z

/-- Plain real-coordinate physical source. -/
def resolverPhysicalSourceReal (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun z => p.ν * (positivePart (intervalDomainLift u z)) ^ p.γ

/-- Plain physical Laplacian, without the outside-`[0,1]` branch. -/
def resolverLapPhysicalPlain (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun z => p.μ * resolverValueSeriesReal p u z - resolverPhysicalSourceReal p u z

/-- If this is not already in the file, define the real-coordinate piecewise version.
At interior points it is locally equal to `resolverLapPhysicalPlain`. -/
def resolverLapPhysicalReal (p : CM2Params)
    (u : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun z => if hz : z ∈ Set.Icc (0 : ℝ) 1 then
    resolverLapPhysical p u ⟨z, hz⟩
  else
    0

/-- Source continuity on `[0,1]`.  The key Mathlib lemmas are:
`ContinuousOn.max`, `ContinuousOn.rpow_const`, and `.mul`. -/
theorem resolverPhysicalSourceReal_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1)) :
    ContinuousOn (resolverPhysicalSourceReal p u) (Set.Icc (0 : ℝ) 1) := by
  -- `positivePart r = max r 0`.
  have hpp :
      ContinuousOn (fun z : ℝ => positivePart (intervalDomainLift u z))
        (Set.Icc (0 : ℝ) 1) := by
    simpa [positivePart] using
      hUcont.max (continuousOn_const : ContinuousOn (fun _ : ℝ => (0 : ℝ)) (Set.Icc 0 1))

  -- `rpow_const` is convenient here.  The side condition is discharged by `γ ≥ 0`.
  have hpow :
      ContinuousOn
        (fun z : ℝ => (positivePart (intervalDomainLift u z)) ^ p.γ)
        (Set.Icc (0 : ℝ) 1) := by
    exact hpp.rpow_const (fun z hz => Or.inr p.hγ.le)

  simpa [resolverPhysicalSourceReal] using continuousOn_const.mul hpow

/-- Source continuity at an interior point. -/
theorem resolverPhysicalSourceReal_continuousAt_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    ContinuousAt (resolverPhysicalSourceReal p u) x := by
  have hsrc_on := resolverPhysicalSourceReal_continuousOn p hUcont
  have hxIcc : x ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hx
  have hIcc_nhds : Set.Icc (0 : ℝ) 1 ∈ 𝓝 x := by
    exact mem_of_superset (IsOpen.mem_nhds isOpen_Ioo hx) Set.Ioo_subset_Icc_self
  have hcw := hsrc_on.continuousWithinAt hxIcc
  -- Turn continuity-within `[0,1]` into ordinary continuity because `[0,1]`
  -- is a neighborhood of an interior point.
  simpa [ContinuousAt, nhdsWithin_eq_nhds.2 hIcc_nhds] using hcw

/-- Absolute summability of the resolved value coefficients.

This is the uniform-convergence input.  The repo theorem
`resolver_cosineSeries_summable_of_sourceL2` already proves absolute summability
of the value series for each `x`; applying it at `0` gives `∑ |c_k| < ∞` because
`cos(kπ * 0) = 1`. -/
theorem resolverCoeff_re_abs_summable_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1)) :
    Summable fun k : ℕ => |(intervalNeumannResolverCoeff p u k).re| := by
  have hsrcL2 :
      Summable fun k : ℕ => ((intervalNeumannResolverSourceCoeff p u k).re) ^ 2 := by
    -- `resolverSourceCoeff_re_sq_summable_of_continuousOn` is stated as a
    -- difference from the zero source; simplify the zero source away.
    simpa [ShenWork.Paper2.intervalNeumannResolverSourceCoeff_zero] using
      resolverSourceCoeff_re_sq_summable_of_continuousOn p hUcont

  have hseries0 :
      Summable fun k : ℕ =>
        (intervalNeumannResolverCoeff p u k).re * unitIntervalCosineMode k (0 : ℝ) :=
    resolver_cosineSeries_summable_of_sourceL2 p hsrcL2 0

  -- At `0`, `unitIntervalCosineMode k 0 = 1`, so the norm of the summand is
  -- just `|(intervalNeumannResolverCoeff p u k).re|`.
  simpa [unitIntervalCosineMode, Real.norm_eq_abs] using hseries0.norm

/-- Continuity of the resolver value series by the Weierstrass M-test, using
`continuous_tsum`. -/
theorem resolverValueSeriesReal_continuous_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1)) :
    Continuous (resolverValueSeriesReal p u) := by
  classical
  let c : ℕ → ℝ := fun k => (intervalNeumannResolverCoeff p u k).re
  let f : ℕ → ℝ → ℝ := fun k z => c k * unitIntervalCosineMode k z
  let M : ℕ → ℝ := fun k => |c k|

  have hM : Summable M := by
    simpa [M, c] using resolverCoeff_re_abs_summable_of_continuousOn p hUcont

  have hf : ∀ k : ℕ, Continuous (f k) := by
    intro k
    -- `unitIntervalCosineMode k z = cos(kπz)`.
    -- `fun_prop` usually closes this after unfolding the cosine mode.
    dsimp [f, c]
    unfold unitIntervalCosineMode
    fun_prop

  have hbound : ∀ k z, ‖f k z‖ ≤ M k := by
    intro k z
    dsimp [f, M, c]
    rw [Real.norm_eq_abs, abs_mul]
    have hcos : |unitIntervalCosineMode k z| ≤ 1 := by
      unfold unitIntervalCosineMode
      exact Real.abs_cos_le_one _
    calc
      |(intervalNeumannResolverCoeff p u k).re| * |unitIntervalCosineMode k z|
          ≤ |(intervalNeumannResolverCoeff p u k).re| * 1 := by
            exact mul_le_mul_of_nonneg_left hcos (abs_nonneg _)
      _ = |(intervalNeumannResolverCoeff p u k).re| := by ring

  simpa [resolverValueSeriesReal, f, c] using continuous_tsum hf hM hbound

/-- The plain physical Laplacian is continuous at interior points. -/
theorem resolverLapPhysicalPlain_continuousAt_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    ContinuousAt (resolverLapPhysicalPlain p u) x := by
  have hR : ContinuousAt (resolverValueSeriesReal p u) x :=
    (resolverValueSeriesReal_continuous_of_continuousOn p hUcont).continuousAt
  have hS : ContinuousAt (resolverPhysicalSourceReal p u) x :=
    resolverPhysicalSourceReal_continuousAt_of_continuousOn p hUcont hx
  simpa [resolverLapPhysicalPlain] using (hR.const_mul p.μ).sub hS

/-- Main continuity theorem: the piecewise physical Laplacian is continuous at every
interior point. -/
theorem resolverLapPhysicalReal_continuousAt_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    ContinuousAt (resolverLapPhysicalReal p u) x := by
  have hplain : ContinuousAt (resolverLapPhysicalPlain p u) x :=
    resolverLapPhysicalPlain_continuousAt_of_continuousOn p hUcont hx

  have hIoo_nhds : Set.Ioo (0 : ℝ) 1 ∈ 𝓝 x :=
    IsOpen.mem_nhds isOpen_Ioo hx

  have hlocal :
      resolverLapPhysicalReal p u =ᶠ[𝓝 x] resolverLapPhysicalPlain p u := by
    filter_upwards [hIoo_nhds] with z hz
    have hzIcc : z ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hz
    -- This is the dependent-point bookkeeping step.  The important facts are:
    -- * the `if` branch chooses `hzIcc`,
    -- * `intervalNeumannResolverR` is definitionally the same value series,
    -- * `resolverPositiveSourceLifted` is the same positive-part source.
    simp [resolverLapPhysicalReal, resolverLapPhysicalPlain, resolverLapPhysical,
      resolverValueSeriesReal, resolverPhysicalSourceReal, resolverPositiveSourceLifted,
      intervalNeumannResolverR, unitIntervalCosineMode, hzIcc]

  -- Depending on the local Mathlib version, one of these works:
  --   exact hplain.congr_of_eventuallyEq hlocal.symm
  --   exact hplain.congr hlocal.symm
  --   rw [ContinuousAt] at hplain ⊢; exact hplain.congr' hlocal.symm
  exact hplain.congr_of_eventuallyEq hlocal.symm

end ShenWork.IntervalResolverWeakBounds
```

## If `congr_of_eventuallyEq` has a different name

The last line is purely API-sensitive. If Lean does not find

```lean
ContinuousAt.congr_of_eventuallyEq
```

replace it with the corresponding `Tendsto` congruence proof:

```lean
  rw [ContinuousAt] at hplain ⊢
  exact hplain.congr' hlocal.symm
```

or use `filter_mono`/`Tendsto.congr` explicitly. The mathematical point is simply that two functions equal on the neighborhood `Ioo 0 1` have the same continuity behavior at `x`.

## Checklist of hypotheses discharged

For the source term:

```lean
hUcont.max continuousOn_const
hpp.rpow_const (fun z hz => Or.inr p.hγ.le)
continuousOn_const.mul hpow
```

For the resolver value:

```lean
resolverSourceCoeff_re_sq_summable_of_continuousOn p hUcont
resolver_cosineSeries_summable_of_sourceL2 p hsrcL2 0
hseries0.norm
continuous_tsum hf hM hbound
```

For the piecewise physical Laplacian:

```lean
IsOpen.mem_nhds isOpen_Ioo hx
filter_upwards [hIoo_nhds]
simp [resolverLapPhysicalReal, resolverLapPhysical, resolverLapPhysicalPlain]
```

This proves exactly the continuity you need without invoking any second-derivative series and without requiring `∑ |c_k| (kπ)^2 < ∞`.
