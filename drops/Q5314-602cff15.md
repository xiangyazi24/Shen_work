# Paper 1 Henry local-to-global capstone: the two remaining adapters

Audit target: `xiangyazi24/Shen_work`, `main` at the current weighted-regularity assembly (`5932ed3f1bab5a436a96327f994b6187e4bed44f`).

## Executive answer

There are two small adapters, but neither is an FTC theorem and neither requires a measurable or continuous choice of the local window endpoints.

1. **Local-window elimination plus arbitrary-start scalar Grönwall.** For each physical time `t` on the tail, use any witness window
   `L < t < R < T`, obtain an ordinary derivative certificate for the *same globally defined energy*
   
   ```lean
   E t := paper5WeightedEnergy eta c u U t,
   ```
   
   and the sharp inequality at `t`. Eliminate the window witnesses immediately. This gives pointwise-in-time `HasDerivAt E (deriv E t) t`, hence the right-derivative field required by the repository's scalar Grönwall argument. Then use an arbitrary-left-endpoint variant of the existing scalar energy lemma, starting at a fixed positive `t0`.

2. **Positive-time integrable seed.** Select one deterministic `t0 > 0` after the eventual common ceiling `M` has become valid and after any preferred-chart threshold. Use the already committed exact-weight `H⁰` propagation from the original `WeightedL2InitialCloseness` to prove actual `Integrable` of the weighted density at `t0`. This is the meaningful Lean notion of `E(t0) < ∞`; merely having the totalized integral as a real number is not enough.

The key correction to the wording “per-window dissipation theorem” is that

```lean
paper5WeightedEnergy_deriv_le_common_of_exactGeneratorWindow_local
```

currently exports only an inequality involving `deriv`. Since `deriv` is totalized, that inequality by itself is not a differentiability certificate. The clean capstone should first add a paired-output wrapper exporting both `HasDerivAt` and the inequality from the same local core.

Once these two adapters are present, the remaining wiring is direct:

```text
initial weighted L² closeness
  -> exact-weight H⁰ propagation
  -> choose t0 after the eventual common M ceiling
  -> for every t >= t0, instantiate one local Henry window
  -> HasDerivAt E (deriv E t) t + deriv E t <= 2 q_M(eta) E t
  -> arbitrary-start scalar Grönwall
  -> eventual exponential decay of E
  -> CoMovingWeightedL2Convergence.
```

No `W_t`, no spatial derivative of the forcing, no second derivative of the trajectory whose continuity is being proved, no global-in-time choice of `X` or `F`, and no sharp inequality is used to create the finite seed.

# Adapter A: eliminate the local windows and make a tail scalar frontier

## 2. First add a paired local theorem

The safest local interface is not an inequality involving raw `deriv`; it is a pair.

A suitable theorem shape is:

```lean
/-- One exact-generator window certifies the ordinary derivative of the
physical global energy and the sharp inequality at its target time. -/
theorem
    paper5WeightedEnergy_hasDerivAt_and_deriv_le_common_of_exactGeneratorWindow_local
    (p : CMParams)
    {M T eta c L R t : ℝ}
    {u v : ℝ -> ℝ -> ℝ} {U V : ℝ -> ℝ}
    {X A F : ℝ -> WholeLineRealL2}
    -- exactly the hypotheses of
    -- paper5WeightedEnergy_deriv_le_common_of_exactGeneratorWindow_local
    ... :
    let E := paper5WeightedEnergy eta c u U
    let qM := paper531Quadratic c
      (paper531CommonA p M) (paper531CommonB p M) eta
    HasDerivAt E (deriv E t) t /\
      deriv E t <= 2 * qM * E t
```

Implementation should call the core producer once:

```lean
  dsimp only
  obtain ⟨hhalf, hdiff, hgrad⟩ :=
    paper5WeightedEnergy_coreIntegrability_of_exactGeneratorWindow_local ...

  have hEraw := paper5WeightedEnergy_hasDerivAt_of_half hhalf
  have hE : HasDerivAt
      (paper5WeightedEnergy eta c u U)
      (deriv (paper5WeightedEnergy eta c u U) t) t :=
    hEraw.differentiableAt.hasDerivAt

  have hineq :=
    paper5WeightedEnergy_deriv_le_common_of_coreIntegrability
      ... hhalf hdiff hgrad hrem_int

  exact ⟨hE, hineq⟩
```

This avoids running the large core construction twice and prevents a future caller from accidentally treating a `deriv` inequality as proof of differentiability.

## 3. Quantifier elimination: no endpoint functions are needed

Let

```lean
E : ℝ -> ℝ := paper5WeightedEnergy eta c u U
qM : ℝ := paper531Quadratic c
  (paper531CommonA p M) (paper531CommonB p M) eta
```

Suppose the capstone has a local producer of the logical form

```lean
hwindow : forall t, t0 <= t ->
  exists L R T X A F,
    0 < L /\ L < t /\ t < R /\ R < T /\
    LocalHenryWindowData p M eta c u v U V X A F L R T t
```

where `LocalHenryWindowData` abbreviates the hypotheses needed by the paired local theorem.

The global tail facts are obtained by ordinary existential elimination:

```lean
have hpoint : forall t, t0 <= t ->
    HasDerivAt E (deriv E t) t /\
      deriv E t <= 2 * qM * E t := by
  intro t ht
  rcases hwindow t ht with ⟨L, R, T, X, A, F, hdata⟩
  exact
    paper5WeightedEnergy_hasDerivAt_and_deriv_le_common_of_exactGeneratorWindow_local
      ... hdata
```

There is no reason to define functions

```lean
a : ℝ -> ℝ
r : ℝ -> ℝ
X : ℝ -> (ℝ -> WholeLineRealL2)
F : ℝ -> (ℝ -> WholeLineRealL2).
```

The witnesses disappear before the conclusion is formed. In particular:

- `a(t)` and `r(t)` need not vary continuously;
- they need not be measurable;
- no measurable-selection theorem is needed;
- the auxiliary Hilbert trajectories from two overlapping windows need not be identified globally;
- both windows certify the derivative of the same scalar function `E`, so derivative uniqueness handles any overlap automatically.

If desired, the overlap fact is literally

```lean
(hE1 : HasDerivAt E d1 t).unique (hE2 : HasDerivAt E d2 t) : d1 = d2.
```

Normalizing every local certificate to `deriv E t` makes even that equality invisible downstream.

## 4. Produce the exact scalar-frontier fields

From `hpoint`, the three fields needed on `[t0,∞)` are elementary:

```lean
have hcontTail : forall T, t0 <= T ->
    ContinuousOn E (Set.Icc t0 T) := by
  intro T hT t ht
  exact (hpoint t ht.1).1.continuousAt.continuousWithinAt

have hrightTail : forall T, t0 <= T -> forall t in Set.Ico t0 T,
    HasDerivWithinAt E (deriv E t) (Set.Ici t) t := by
  intro T hT t ht
  exact (hpoint t ht.1).1.hasDerivWithinAt

have hdissTail : forall t, t0 <= t ->
    deriv E t <= 2 * qM * E t := by
  intro t ht
  exact (hpoint t ht).2
```

So the window-elimination theorem can directly return:

```lean
/-- Pointwise local Henry windows give one scalar energy frontier on a tail. -/
theorem paper5WeightedEnergy_tail_frontier_of_local_windows
    ...
    (hwindow : forall t, t0 <= t -> exists ..., LocalHenryWindowData ... t) :
    (forall T, t0 <= T -> ContinuousOn E (Set.Icc t0 T)) /\
    (forall T, t0 <= T -> forall t in Set.Ico t0 T,
      HasDerivWithinAt E (deriv E t) (Set.Ici t) t) /\
    (forall t, t0 <= t -> deriv E t <= 2 * qM * E t)
```

This is a proposition-level assembly theorem, not a new PDE theorem.

## 5. Do not weaken to an a.e. derivative unless another consumer requires it

The repository's scalar Grönwall proof does not use FTC. `scalarEnergy_crude_exponential_bound` invokes

```lean
le_gronwallBound_of_liminf_deriv_right_le
```

and turns each right derivative certificate into the required Dini-slope estimate with

```lean
(hderiv s hs).liminf_right_slope_le.
```

Consequently, the pointwise `HasDerivWithinAt` frontier above is already the best and cheapest interface.

An a.e. statement could be derived afterward without choosing measurable windows:

```lean
have hae : forallᵐ t ∂(volume.restrict (Set.Ioi t0)),
    HasDerivAt E (deriv E t) t /\
      deriv E t <= 2 * qM * E t := by
  filter_upwards [ae_restrict_mem measurableSet_Ioi] with t ht
  exact hpoint t ht.le
```

But an FTC route would then additionally need measurability/integrability of `fun t => deriv E t`, absolute continuity of `E`, and an interval-integral identity. None of those are needed by the existing Dini-right Grönwall architecture. Building them would enlarge, not reduce, the proof.

## 6. Add an arbitrary-start scalar lemma

The existing

```lean
scalarEnergy_crude_exponential_bound
```

is based at zero. The tail version should be:

```lean
/-- Scalar Grönwall from an arbitrary left endpoint. -/
theorem scalarEnergy_exponential_bound_from
    {E : ℝ -> ℝ} {C t0 t : ℝ}
    (ht : t0 <= t)
    (hcont : ContinuousOn E (Set.Icc t0 t))
    (hderiv : forall s in Set.Ico t0 t,
      HasDerivWithinAt E (deriv E s) (Set.Ici s) s)
    (hgrowth : forall s in Set.Ico t0 t,
      deriv E s <= C * E s) :
    E t <= E t0 * Real.exp (C * (t - t0))
```

The shortest proof is a direct copy of the existing zero-based proof with

```lean
(a := t0) (b := t)
```

in `le_gronwallBound_of_liminf_deriv_right_le`, followed by

```lean
rw [gronwallBound_epsilon0]
```

and ring normalization. This avoids all composition bookkeeping.

A shifted-function proof with `G s = E (t0 + s)` is possible, but then one must transport `ContinuousOn`, compose derivatives with `s ↦ t0+s`, and normalize `deriv G`. It is strictly more tactic work.

## 7. Tail-start version of the final convergence consumer

Add a sibling of the existing zero-based theorem:

```lean
/-- The paper-(5.31) energy inequality only after a positive tail time. -/
theorem CoMovingWeightedL2Convergence.of_paper531_energy_inequality_from
    {eta c A B t0 : ℝ}
    {u : ℝ -> ℝ -> ℝ} {U : ℝ -> ℝ} {E : ℝ -> ℝ}
    (hq : paper531Quadratic c A B eta < 0)
    (hcontrol : forallᶠ t in atTop,
      coMovingWeightedL2Energy eta c u U t <= E t)
    (henergy_int : forallᶠ t in atTop, Integrable (fun z : ℝ =>
      Real.exp (2 * eta * z) * |u t (z + c * t) - U z| ^ 2))
    (hcont : forall T, t0 <= T -> ContinuousOn E (Set.Icc t0 T))
    (hderiv : forall T, t0 <= T -> forall t in Set.Ico t0 T,
      HasDerivWithinAt E (deriv E t) (Set.Ici t) t)
    (hdiss : forall t, t0 <= t ->
      deriv E t <= 2 * paper531Quadratic c A B eta * E t) :
    CoMovingWeightedL2Convergence eta c u U
```

Let

```lean
lam := -2 * paper531Quadratic c A B eta.
```

Then `0 < lam`. The arbitrary-start scalar lemma gives, for `t >= t0`,

```lean
E t <= E t0 * Real.exp (-lam * (t - t0)).
```

Rewrite the right side as

```lean
(E t0 * Real.exp (lam * t0)) * Real.exp (-lam * t)
```

using `Real.exp_add` and ring arithmetic. Turn this into an eventual statement with

```lean
eventually_atTop.2 ⟨t0, ...⟩.
```

Combine it with `hcontrol` using `filter_upwards`, then call the already committed

```lean
CoMovingWeightedL2Convergence.of_eventual_exponential_decay.
```

This is the clean arbitrary-start generalization of `of_paper531_energy_inequality`. The current zero-based theorem may subsequently be made the specialization `t0 = 0`.

### Why not literally time-shift and invoke the existing theorem?

One can define

```lean
Eshift s := E (t0 + s)
uShift s x := u (t0 + s) (x + c * t0)
```

for which

```lean
coMovingWeightedL2Energy eta c uShift U s =
  coMovingWeightedL2Energy eta c u U (t0 + s).
```

However, after applying the zero-based theorem, one must still transfer eventual integrability and convergence from the shifted orbit back to the original orbit using cofinality of `s ↦ t0+s`. That adds an extra time-shift convergence lemma. The direct arbitrary-start scalar theorem above is shorter and keeps the promised count at two adapters.

# Adapter B: select and certify one positive-time finite-energy seed

## 8. “Finite energy” must mean `Integrable`

In Lean, the Bochner/Lebesgue integral is totalized. The term

```lean
paper5WeightedEnergy eta c u U t0 : ℝ
```

exists even if its density is not integrable. Therefore the seed theorem must not merely conclude

```lean
exists C : ℝ, E t0 <= C.
```

It must include

```lean
Integrable (fun z : ℝ =>
  Real.exp (2 * eta * z) * |u t0 (z + c * t0) - U z| ^ 2).
```

Once this is known, the identities

```lean
paper5WeightedEnergy_eq_coMovingWeightedL2Energy
paper5WeightedEnergy_eq_weightedPopulationIntegral
```

identify `E t0` with the actual squared weighted `L²` norm, and

```lean
integral_nonneg
```

gives `0 <= E t0`.

## 9. Choose `t0` only after all tail thresholds are known

For the sharp coefficient to be one fixed

```lean
qM = paper531Quadratic c
  (paper531CommonA p M) (paper531CommonB p M) eta,
```

the common profile bound `M` must hold for every time on the tail. The correct order is:

1. Use `exists_common_bound_gt_MChi_of_weight_window` to choose `M > MChi p` with `qM < 0`.
2. Use the eventual Cauchy ceiling to obtain a threshold `TM` such that the physical solution lies in `[0,M]` for every `t >= TM`.
3. Include any chart/interior threshold, for example `wholeLineCauchyGlobalStep p u0`, and a positive numerical threshold.
4. Set one deterministic start, schematically

   ```lean
   t0 := max 1 (max TM (wholeLineCauchyGlobalStep p u0)).
   ```

5. Obtain exact-weight `H⁰` integrability at this `t0` from the committed H⁰ propagation.

If the H⁰ producer is only eventual rather than all-positive-time, intersect its eventual predicate with the eventual `M` ceiling and choose `t0` from the intersection. This uses only

```lean
Eventually.and
eventually_atTop.1
eventually_atTop.2
```

or `filter_upwards`; there is still no measurable selection.

## 11. The left endpoint subtlety

The repository scalar Grönwall interface uses `Set.Ico t0 T`, which contains `t0`. Thus the clean frontier should provide the local derivative certificate at `t = t0` as well as at later times.

This is not an analytic obstacle: because `t0 > 0`, choose a local window

```lean
L < t0 < R
```

exactly as for every other target time. Therefore formulate the local-window quantifier as

```lean
forall t, t0 <= t -> ...
```

rather than only `t0 < t`.

If an upstream interface is stubbornly strict and only gives derivatives for `t0 < t`, there are two alternatives:

1. **Recommended:** replace `t0` by any `t1 > t0` and use the H⁰ propagation again at `t1`.
2. Prove an open-left Grönwall lemma by applying the estimate from `t0 + eps` and sending `eps -> 0`; this requires

   ```lean
   Tendsto E (nhdsWithin t0 (Set.Ioi t0)) (nhds (E t0))
   ```

   plus continuity of `Real.exp` and limit algebra.

The first option is far cheaper. There is no reason to create the second adapter merely to preserve a particular numerical label for the start time.

# 13. Minimal dependency order and noncircularity audit

```text
WeightedL2InitialCloseness at time 0
  |
  v
canonical cap/H0 propagation on each restart segment
  |
  +--> choose one t0 after the eventual common-M threshold
  |      and certify Integrable(weighted density at t0)
  |
  v
on each target t >= t0:
  value candidate W continuity
  gradient candidate X continuity
  physical forcing F continuity/Holder
  exact generator restart
  |
  v
coreIntegrability at t
  |
  +--> HasDerivAt E (deriv E t) t
  +--> deriv E t <= 2 qM E t
  |
  v

  |
  v

  |
  v
arbitrary-start Grönwall, qM < 0
  |
  v

```

There is no cycle:

- The `H⁰` seed is produced before and independently of the sharp inequality.
- `X` is obtained by differentiating the value mild restart through the heat kernel; no `W_t` is used.
- `F` is assembled from `W`, `X`, resolver fields, and coefficient moduli; no derivative of `F` is used.
- The local energy derivative is obtained only after `W`, `X`, and `F` are realized.
- Grönwall is invoked only after the pointwise derivative inequality has been established on the whole tail.
