# Paper 1 Henry local-to-global capstone: the two remaining adapters

Audited against `xiangyazi24/Shen_work` `main` at the current weighted-regularity assembly (`5932ed3f1bab5a436a96327f994b6187e4bed44f`).

## Verdict

There are exactly two small adapters left at this layer:

1. **Local-window elimination plus an arbitrary-start scalar Grönwall wrapper.** For each target time on the tail, use any local Henry window `L < t < R < T`, obtain both an ordinary derivative certificate and the sharp inequality for the same globally defined scalar energy, then immediately eliminate the window witnesses. No function `t ↦ L(t)` or `t ↦ R(t)` is needed, so there is no continuity or measurable-selection problem.

2. **A positive-time integrable seed selector.** Choose one deterministic `t₀ > 0` after the eventual common bound `M` and any global-chart threshold. Use the already committed exact-weight `H⁰` propagation from `WeightedL2InitialCloseness` to prove actual `Integrable` of the weighted density at `t₀`. In Lean this integrability statement—not merely the totalized real-valued integral—is what means that `E(t₀)` is a genuine finite weighted energy.

The current local theorem

```lean
paper5WeightedEnergy_deriv_le_common_of_exactGeneratorWindow_local
```

exports only an inequality involving `deriv`. Because `deriv` is totalized, that inequality alone is not a differentiability certificate. The first tiny local wrapper should therefore export `HasDerivAt` and the inequality together from the same call to the core-integrability producer.

The final route is:

```text
initial weighted L² closeness
  → exact-weight H⁰ propagation
  → choose t₀ after the eventual common-M threshold
  → instantiate one local Henry window for every t ≥ t₀
  → HasDerivAt E (deriv E t) t
      and deriv E t ≤ 2 q_M(η) E t
  → arbitrary-start scalar Grönwall
  → eventual exponential decay
  → CoMovingWeightedL2Convergence.
```

No `W_t`, no spatial derivative of the forcing, no second derivative of the trajectory whose continuity is being established, and no sharp-energy inequality is used to construct the seed.

# Adapter A: eliminate local windows and build one tail energy frontier

## 2. Export a paired local result

Add a small wrapper with the following shape:

```lean
/-- One exact-generator window certifies both differentiability of the
physical energy and the common-bound sharp inequality at its target time. -/
theorem
    paper5WeightedEnergy_hasDerivAt_and_deriv_le_common_of_exactGeneratorWindow_local
    (p : CMParams)
    {M T eta c L R t : ℝ}
    {u v : ℝ → ℝ → ℝ} {U V : ℝ → ℝ}
    {X A F : ℝ → WholeLineRealL2}
    -- exactly the hypotheses of the current local dissipation theorem
    ... :
    let E := paper5WeightedEnergy eta c u U
    let qM := paper531Quadratic c
      (paper531CommonA p M) (paper531CommonB p M) eta
    HasDerivAt E (deriv E t) t ∧
      deriv E t ≤ 2 * qM * E t
```

Its implementation should call the core only once:

```lean
  dsimp only
  obtain ⟨hhalf, hdiff, hgrad⟩ :=
    paper5WeightedEnergy_coreIntegrability_of_exactGeneratorWindow_local ...

  have hEraw := paper5WeightedEnergy_hasDerivAt_of_half hhalf
  have hE : HasDerivAt
      (paper5WeightedEnergy eta c u U)
      (deriv (paper5WeightedEnergy eta c u U) t) t :=
    hEraw.differentiableAt.hasDerivAt

  have hineq :=
    paper5WeightedEnergy_deriv_le_common_of_coreIntegrability
      ... hhalf hdiff hgrad hrem_int

  exact ⟨hE, hineq⟩
```

This avoids recomputing the large local core and prevents callers from mistaking a statement about totalized `deriv` for a differentiability proof.

## 3. Eliminate windows pointwise; do not choose endpoint functions

Define once and for all

```lean
let E : ℝ → ℝ := paper5WeightedEnergy eta c u U
let qM : ℝ := paper531Quadratic c
  (paper531CommonA p M) (paper531CommonB p M) eta
```

Assume the per-target producer has the logical form

```lean
hwindow : ∀ t, t₀ ≤ t →
  ∃ L R T X A F,
    0 < L ∧ L < t ∧ t < R ∧ R < T ∧
    LocalHenryWindowData p M eta c u v U V X A F L R T t
```

Then form the global pointwise fact by plain existential elimination:

```lean
have hpoint : ∀ t, t₀ ≤ t →
    HasDerivAt E (deriv E t) t ∧
      deriv E t ≤ 2 * qM * E t := by
  intro t ht
  rcases hwindow t ht with ⟨L, R, T, X, A, F, hdata⟩
  exact
    paper5WeightedEnergy_hasDerivAt_and_deriv_le_common_of_exactGeneratorWindow_local
      ... hdata
```

No functions

```lean
L R : ℝ → ℝ
X F : ℝ → (ℝ → WholeLineRealL2)
```

need to be defined. The witnesses disappear before the conclusion is formed. Consequently:

- `L(t)` and `R(t)` do not need to be continuous;
- they do not need to be measurable;
- no measurable-selection theorem is involved;
- local `X` and `F` trajectories from overlapping windows do not need to be glued;
- all windows certify the derivative of the same scalar function `E`.

If two windows return coefficients `d₁` and `d₂`, consistency is automatic from

```lean
hE₁.unique hE₂
```

for `hE₁ : HasDerivAt E d₁ t` and `hE₂ : HasDerivAt E d₂ t`. Normalizing every certificate to `deriv E t` removes even this bookkeeping.

This is proposition-level assembly, not a new PDE estimate.

## 5. An a.e. derivative is optional and strictly weaker than what is available

If another theorem later wants an a.e. statement, it follows without selecting measurable windows:

```lean
have hae :
    ∀ᵐ t ∂(volume.restrict (Set.Ioi t₀)),
      HasDerivAt E (deriv E t) t ∧
        deriv E t ≤ 2 * qM * E t := by
  filter_upwards [ae_restrict_mem measurableSet_Ioi] with t ht
  exact hpoint t ht.le
```

But this should not be the main route. The repository's scalar proof uses the Dini-right interface

```lean
le_gronwallBound_of_liminf_deriv_right_le
```

and converts the right derivative using

```lean
(hderiv s hs).liminf_right_slope_le.
```

An FTC route would additionally require measurability and interval integrability of `fun t => deriv E t`, absolute continuity of `E`, and an integral identity. None of those obligations is needed for the committed Grönwall architecture.

## 6. Add arbitrary-start scalar Grönwall

Generalize the existing `scalarEnergy_crude_exponential_bound` to:

```lean
/-- Scalar Grönwall from an arbitrary left endpoint. -/
theorem scalarEnergy_exponential_bound_from
    {E : ℝ → ℝ} {C t₀ t : ℝ}
    (ht : t₀ ≤ t)
    (hcont : ContinuousOn E (Set.Icc t₀ t))
    (hderiv : ∀ s ∈ Set.Ico t₀ t,
      HasDerivWithinAt E (deriv E s) (Set.Ici s) s)
    (hgrowth : ∀ s ∈ Set.Ico t₀ t,
      deriv E s ≤ C * E s) :
    E t ≤ E t₀ * Real.exp (C * (t - t₀))
```

The shortest proof is the existing proof with

```lean
(a := t₀) (b := t)
```

in `le_gronwallBound_of_liminf_deriv_right_le`, followed by

```lean
rw [gronwallBound_ε0]
```

and elementary algebra. No FTC and no time translation of the solution are needed.

## 7. Add the tail-start final consumer

A sibling of the current zero-based theorem should have shape:

```lean
/-- Paper-(5.31) energy decay beginning at an arbitrary tail time. -/
theorem CoMovingWeightedL2Convergence.of_paper531_energy_inequality_from
    {eta c A B t₀ : ℝ}
    {u : ℝ → ℝ → ℝ} {U : ℝ → ℝ} {E : ℝ → ℝ}
    (hq : paper531Quadratic c A B eta < 0)
    (hcontrol : ∀ᶠ t in atTop,
      coMovingWeightedL2Energy eta c u U t ≤ E t)
    (henergy_int : ∀ᶠ t in atTop, Integrable (fun z : ℝ =>
      Real.exp (2 * eta * z) * |u t (z + c * t) - U z| ^ 2))
    (hcont : ∀ T, t₀ ≤ T → ContinuousOn E (Set.Icc t₀ T))
    (hderiv : ∀ T, t₀ ≤ T → ∀ t ∈ Set.Ico t₀ T,
      HasDerivWithinAt E (deriv E t) (Set.Ici t) t)
    (hdiss : ∀ t, t₀ ≤ t →
      deriv E t ≤ 2 * paper531Quadratic c A B eta * E t) :
    CoMovingWeightedL2Convergence eta c u U
```

Set

```lean
let lam := -2 * paper531Quadratic c A B eta
```

so `0 < lam`. The arbitrary-start scalar lemma gives, for `t₀ ≤ t`,

```lean
E t ≤ E t₀ * Real.exp (-lam * (t - t₀)).
```

Rewrite this as

```lean
E t ≤ (E t₀ * Real.exp (lam * t₀)) * Real.exp (-lam * t)
```

using `Real.exp_add`. Package it with

```lean
eventually_atTop.2 ⟨t₀, ...⟩
```

combine it with `hcontrol` using `filter_upwards`, and finish with the already committed

```lean
CoMovingWeightedL2Convergence.of_eventual_exponential_decay.
```

The present zero-based `of_paper531_energy_inequality` can then be retained as the specialization `t₀ = 0`.

A literal time-shift proof is possible with

```lean
Eshift s := E (t₀ + s)
uShift s x := u (t₀ + s) (x + c * t₀),
```

but transferring the resulting convergence back from `uShift` to `u` adds a cofinal time-translation lemma. The arbitrary-start scalar statement is shorter and preserves the promised two-adapter endgame.

# Adapter B: choose and certify one positive-time finite-energy seed

## 8. The seed must export `Integrable`

Because the integral is totalized, the expression

```lean
paper5WeightedEnergy eta c u U t₀ : ℝ
```

exists even when the density is not integrable. Therefore the seed theorem must include

```lean
Integrable (fun z : ℝ =>
  Real.exp (2 * eta * z) * |u t₀ (z + c * t₀) - U z| ^ 2).
```

Then

```lean
paper5WeightedEnergy_eq_coMovingWeightedL2Energy
```

identifies the scalar seed with the physical energy, and `integral_nonneg` gives its nonnegativity.

## 9. Select `t₀` after all tail thresholds

The correct order is:

1. Choose `M > MChi p` with

   ```lean
   qM := paper531Quadratic c
     (paper531CommonA p M) (paper531CommonB p M) eta < 0
   ```

   using `exists_common_bound_gt_MChi_of_weight_window`.

2. Obtain a time `T_M` after which the whole solution lies in `[0,M]`.

3. Include any positive/chart threshold, for example `wholeLineCauchyGlobalStep p u₀`.

4. Choose deterministically, schematically,

   ```lean
   t₀ := max 1 (max T_M (wholeLineCauchyGlobalStep p u₀)).
   ```

5. Apply exact-weight `H⁰` propagation at this `t₀`.

If the `H⁰` statement is itself only eventual, intersect it with the eventual common-bound statement before choosing `t₀`. The only filter facts needed are `Eventually.and`, `eventually_atTop.1`, `eventually_atTop.2`, and `filter_upwards`.

## 10. Exact seed-selector shape

## 11. Left-endpoint subtlety

`Set.Ico t₀ T` contains `t₀`, so the clean scalar frontier should include the derivative certificate at `t = t₀`.

This causes no analytic difficulty: because `t₀ > 0`, instantiate a local window

```lean
L < t₀ < R
```

at the seed time itself. Therefore the local-window quantifier should be

```lean
∀ t, t₀ ≤ t → ...
```

not merely `t₀ < t`.

If an upstream theorem is stubbornly strict, the cheapest solution is to pick a new `t₁ > t₀`, use `H⁰` propagation again at `t₁`, and start there. An open-left Grönwall argument is possible but would require

```lean
Tendsto E (nhdsWithin t₀ (Set.Ioi t₀)) (nhds (E t₀))
```

and an `ε ↓ 0` limit. There is no reason to pay for that extra adapter when every positive time already admits a local Henry window.

## 13. Noncircular dependency order

```text
WeightedL2InitialCloseness at t = 0
  ↓
canonical cap/H⁰ propagation through restart data and segments
  ↓
choose t₀ after the eventual common-M threshold
  ↓
for each t ≥ t₀:
  value candidate W
  gradient candidate X
  physical forcing F
  exact generator restart
  ↓
coreIntegrability at t
  ├─ HasDerivAt E (deriv E t) t
  └─ deriv E t ≤ 2 qM E t
  ↓
eliminate local window witnesses
  ↓
one tail scalar frontier for the same E
  ↓
arbitrary-start Grönwall with qM < 0
  ↓
CoMovingWeightedL2Convergence
```

There is no cycle:

- `H⁰` is obtained before the sharp inequality;
- `X` comes from differentiating the value mild restart through the heat kernel, not from `W_t`;
- `F` is assembled from `W`, `X`, resolver fields, and coefficient moduli, without differentiating `F`;
- the local energy derivative is used only after those trajectories are realized;
- Grönwall is invoked only after the pointwise inequality has been established on the whole tail.

## Bottom line

Adapter A is a **paired local derivative/dissipation export, pointwise window elimination, and arbitrary-start scalar Grönwall wrapper**.
