# 1. The standard models of `X_1(13)`

## 1.1 The sextic model

Use the smooth weighted-projective curve

\[
C:\quad Y^2=F_{13}(X,Z),
\]

where

\[
\begin{aligned}
F_{13}(X,Z)={}&X^6+4X^5Z+6X^4Z^2+2X^3Z^3\\
 &+X^2Z^4+2XZ^5+Z^6.
\end{aligned}
\]

Weighted coordinates have weights `(1,3,1)`:

\[
[X:Y:Z]=[tX:t^3Y:tZ].
\]

The affine chart `Z=1` is

\[
y^2=x^6+4x^5+6x^4+2x^3+x^2+2x+1.
\]

A Lean-facing definition is:

```lean
def f13 {R : Type*} [CommRing R] (x : R) : R :=
  x^6 + 4*x^5 + 6*x^4 + 2*x^3 + x^2 + 2*x + 1

def f13Hom {R : Type*} [CommRing R] (X Z : R) : R :=
  X^6 + 4*X^5*Z + 6*X^4*Z^2 + 2*X^3*Z^3
    + X^2*Z^4 + 2*X*Z^5 + Z^6
```

The discriminant of this displayed sextic equation is

\[
2^{12}13^2.
\]

Consequently, do **not** use reduction at `2` as good reduction of this rational model. The LMFDB minimal generalized model has absolute minimal discriminant `169`, which is the third component of the label `169.a.169.1`.

## 1.2 The LMFDB optimized model

The LMFDB curve data for `169.a.169.1` records the pair

```text
[[0,1,1],[1,0,1,1]]
```

in the convention `v^2 + h(u)v = f(u)`. Thus

\[
\boxed{v^2+(u^3+u^2+1)v=u^2+u.}
\]

The explicit conversion to the sextic above is

\[
\boxed{
 x=-u-1,\qquad
 y=2v+u^3+u^2+1.
}
\]

Indeed, from the optimized equation,

\[
(2v+u^3+u^2+1)^2
=(u^3+u^2+1)^2+4(u^2+u),
\]

and the right side is `f13(-u-1)`.

The inverse affine conversion is

\[
\boxed{
 u=-x-1,\qquad
 v=\frac{y-(u^3+u^2+1)}2.
}
\]

These are only `ring` identities after clearing the factor `2`.

## 1.3 The second commonly used sextic

The model used in several Magma verification scripts is

\[
C^-:\quad y^2=x^6-2x^5+x^4-2x^3+6x^2-4x+1.

It is not a different modular curve. It is obtained by the translation

\[
X=x+1,\qquad
f_{13}(x)=f^-(x+1).

Equivalently, completing the square in the optimized model and setting `X=-u` gives `C^-`.

For the final Lean development, freeze one model. The positive-coefficient sextic is convenient because the four affine rational cusps have `x=0,-1`; the LMFDB generalized model is convenient for the Tate-normal-form bridge.

# 2. Rational automorphisms and the natural prime `3`

## 2.1 An order-six automorphism

Define

\[
\rho[X:Y:Z]=[-Z:Y:X+Z].
\]

On the affine chart where `x+1` is nonzero,

\[
\rho(x,y)=\left(-\frac1{x+1},\frac{y}{(x+1)^3}\right).
\]

The curve-preservation certificate is the homogeneous polynomial identity

\[
F_{13}(-Z,X+Z)=F_{13}(X,Z),
\]

which closes by `ring`.

The matrix on `(X,Z)` is

\[
\begin{pmatrix}0&-1\\1&1\end{pmatrix}.
\]

Its cube is `-I`. Since `Y` is unchanged, weighted-projectively

\[
\rho^3[X:Y:Z]=[X:-Y:Z],
\]

so `rho^3` is the hyperelliptic involution. Hence `rho` has order six.

## 2.2 The order-three automorphism

Put

\[
\sigma=\rho^2.
\]

Then

\[
\boxed{
\sigma[X:Y:Z]=[-X-Z:Y:X]
}
\]

and affinely

\[
\boxed{
\sigma(x,y)=\left(-\frac{x+1}{x},\frac{y}{x^3}\right).
}
\]

The homogeneous formula is total, including `x=0` and the points at infinity. Its matrix satisfies `sigma^3=1` literally.

There are four geometric fixed points, over the two roots of `x^2+x+1`; Riemann--Hurwitz gives

\[
g(C/\langle\sigma\rangle)=0.
\]

Equivalently, one can construct the rational quotient directly. Therefore, on `Pic^0(C)`,

\[
1+\sigma_*+\sigma_*^2=0.
\]

This follows from

\[
q^*q_*=1+\sigma_*+\sigma_*^2
\]

and `Pic^0(P^1)=0`.

## 2.3 The explicit `3`-isogeny endomorphism

Define

\[
\boxed{\lambda=1-\sigma_*\in\operatorname{End}_{\mathbf Q}(J).}
\]

Using `1+sigma+sigma^2=0`,

\[
\begin{aligned}
\lambda^2
  &=(1-\sigma_*)^2\\
  &=1-2\sigma_*+\sigma_*^2\\
  &=-3\sigma_*.
\end{aligned}
\]

Thus

\[
\boxed{\lambda^2=-[3]\circ\sigma_*.}
\]

In characteristic zero `lambda` has degree `9`; it is a `(3,3)`-isogeny, not a cyclic degree-three isogeny.

This identity is the key separatedness bridge:

```text
D is infinitely lambda-divisible
  -> for every n, D is divisible by lambda^(2n)
  -> for every n, D is divisible by 3^n
```

because `sigma_*` is an automorphism.

So the intrinsic modular descent prime is **3**.

A Lean-facing algebra core is:

```lean
-- after constructing the action on the concrete Picard/Mumford group
noncomputable def sigmaStar : J13Q →+ J13Q := ...

noncomputable def lambda13 : J13Q →+ J13Q :=
  AddMonoidHom.id J13Q - sigmaStar

theorem sigma_trace_zero :
    AddMonoidHom.id J13Q + sigmaStar + sigmaStar.comp sigmaStar = 0 := ...

theorem lambda13_sq (D : J13Q) :
    lambda13 (lambda13 D) = -(3 • sigmaStar D) := by
  -- pure additive algebra after sigma_trace_zero
  ...
```

The difficult theorem here is `sigma_trace_zero`, because it requires quotient push-pull or an equivalent principal-divisor proof. The final `lambda13_sq` identity is routine.

More explicitly:

The Magma torsion verification for the shifted sextic `C^-` returns exactly `Z/19` over `Q`. Since the two sextics differ by `x ↦ x+1`, this is the same Jacobian.

## 3.2 An explicit rational generator

Let

\[
A=\infty_-=(1,-1,0),\qquad B=(0,-1).
\]

Set

\[
\boxed{G=[B-A].}
\]

The subgroup generated by rational cusp differences has order `19`. Since the Abel--Jacobi map is injective and `A != B`, `G` is nonzero. Every nonzero element of a group of prime order `19` is a generator. Thus

\[
\operatorname{ord}(G)=19.
\]

For a self-contained fixed-curve Lean build, do not import the statement “the cuspidal subgroup has order 19” as a database axiom. Prove

```lean
theorem nineteen_nsmul_G : 19 • G = 0 := ...
theorem G_ne_zero : G ≠ 0 := ...
theorem order_G : addOrderOf G = 19 := ...
```

by a generated Cantor/Mumford multiplication certificate. Since `19` is prime, the last theorem needs only the first two.

```text

t(X,Y) = (-X,-Y) = hyperelliptic o s.
```

Let `s_*` and `t_*` denote their actions on `Pic^0`. Since the hyperelliptic involution acts as `-1`,

\[
t_*=-s_*.

For pushforward and pullback,

\[
q_+^*q_{+*}=1+s_*,\qquad
q_-^*q_{-*}=1+t_*.

Consequently, if

\[
\Phi=(q_{+*},q_{-*}),\qquad
\Psi=q_+^*+q_-^*,

then

\[
\boxed{\Psi\circ\Phi=[2]\quad\text{on }J(K).}
\]

There is no ramification correction term: at a fixed point, both sides of `q^*q_*P=P+s(P)` have multiplicity two.

# 5. MW-finite-generation-free annihilation

## 5.1 Exact descent output required

A Selmer dimension statement is not enough. The elliptic descent block must export the witness-form theorem

\[
\forall P\in E_i(K),\quad
\exists h\in H_i,\ \exists Q\in E_i(K),
\qquad P=h+[3]Q,
\]

where

\[
H_i=\langle U_i\rangle\cong\mathbf Z/19.
\]

In Lean-shaped form:

```lean
theorem weak_three_descent_E2 (P : E2Point) :
    ∃ h : AddSubgroup.zmultiples U2, ∃ Q : E2Point,
      P = (h : E2Point) + 3 • Q

theorem weak_three_descent_E3 (P : E3Point) :
    ∃ h : AddSubgroup.zmultiples U3, ∃ Q : E3Point,
      P = (h : E3Point) + 3 • Q
```

Because `3` is invertible modulo `19`, every `h in H_i` is itself a triple:

\[
h=[3]([13]h),
\]

since `3*13=39=1 mod 19`. Therefore the weak statement sharpens to

\[
\boxed{E_i(K)=3E_i(K).}
\]

This implication is finite group algebra and uses no finite generation.

The actual descent calculation is a finite `3`-isogeny descent over the class-number-one totally real field `K`. Since the kernels have no nonzero `K`-rational point, the two Kummer targets must be typed correctly; they are not both naively `K^*/K^{*3}`. The safest public endpoint remains the weak divisibility theorem above.

## 5.2 The prime above `3`

Modulo `3`, the defining polynomial of `K` factors as

\[
\boxed{
T^6+2T^5+T^4+T^3+2
=(T^3+2T+1)(T^3+2T^2+2T+2).
}
\]

Both cubic factors are irreducible over `F_3`. Choose, for example,

\[
\mathfrak p_3=(3,a^3+2a+1).
\]

Then

\[
\mathcal O_K/\mathfrak p_3\cong\mathbf F_{27}.
\]

The conductor-one elliptic models have good reduction at this prime. The finite point counts are

\[
\boxed{
\#\widetilde E_+(\mathbf F_{27})
=
\#\widetilde E_-(\mathbf F_{27})
=38.
}
\]

One way to cross-check this is:

- Kamienny--Newman compute `#J(F_27)=4*19^2=1444`;
- `J` is isogenous to `E_+ x E_-` over `K`;
- `E_+` and `E_-` are 3-isogenous, hence have equal finite-field cardinality;
- therefore each cardinality is `sqrt(1444)=38`.

For Lean, direct finite enumeration over

```text
F27 = F3[T]/(T^3+2T+1)
```

is preferable:

```lean
theorem card_E2_F27 : Fintype.card E2F27 = 38 := by native_decide
theorem card_E3_F27 : Fintype.card E3F27 = 38 := by native_decide
```

Only the exponent bound `38 • R = 0` for every reduced point is needed.

## 5.3 Formal-kernel separatedness

Let `K_3=K_{p3}`. It is unramified over `Q_3`, so the normalized valuation satisfies

\[
v_{\mathfrak p_3}(3)=1.
\]

For the formal parameter `z=-x/y`, multiplication by three has the robust form

\[
[3](z)=3z+z^2A(z),\qquad A(z)\in\mathcal O_{K_3}[[z]].
\]

If `r=v(z)>=1`, then

\[
v(3z)=r+1,
\qquad
v(z^2A(z))\ge2r\ge r+1,
\]

so

\[
\boxed{v([3]z)\ge v(z)+1.}
\]

Use an extended valuation with `v(0)=infinity`, or state the finite-valued lemma with the extra hypothesis `[3]P != 0`. Iteration proves that the formal reduction kernel contains no nonzero infinitely 3-divisible point.

This is strictly easier than the ramified `3`-adic calculation in the `N=18` route because here `v(3)=1`.

## 5.4 Killing each elliptic factor by `38`

Let `P in E_i(K)`. Since `E_i(K)=3E_i(K)`, for every `n` there is `Q_n` with

\[
P=[3^n]Q_n.
\]

Because the reduced group has order `38`,

\[
[38]P
\]

lies in the formal kernel. Moreover,

\[
[38]P=[3^n]([38]Q_n),
\]

and every `[38]Q_n` also lies in the formal kernel. Thus `[38]P` is infinitely 3-divisible in the formal kernel. Separatedness gives

\[
\boxed{[38]E_i(K)=0.}
\]

No Mordell--Weil theorem is used.

## 5.5 Killing `J(Q)` by `76`

Let `D in J(Q)` and write `D_K` for its base change. From the push-pull identity,

\[
[2]D_K=\Psi(\Phi(D_K)).
\]

The product target is killed by `38`, so

\[
\begin{aligned}
[76]D_K
 &= [38]([2]D_K)\\
 &= [38]\Psi(\Phi(D_K))\\
 &= \Psi([38]\Phi(D_K))\\
 &=0.
\end{aligned}
\]

Injectivity of scalar extension on the concrete canonical Mumford representatives gives

\[
\boxed{[76]J(\mathbf Q)=0.}
\]

A Lean capstone should be exactly:

```lean
theorem j13_annihilated_76 (D : J13Q) : 76 • D = 0 := by
  apply baseChangePic_injective
  have hPhi : 38 • Phi (baseChangePic D) = 0 := by
    ext <;> simp [E2_annihilated_38, E3_annihilated_38]
  have hPsi := congrArg Psi hPhi
  simpa [map_nsmul, psi_phi, mul_nsmul] using hPsi
```

The exact simplification order of `38*2` is routine.

## 5.6 Direct `lambda`-descent alternative

Instead of descending separately on the elliptic factors, one may compute the `lambda=1-sigma_*` descent directly on `J(Q)`.

The ideal endpoint is

\[
\boxed{J(\mathbf Q)=\lambda J(\mathbf Q).}
\]

Even a weaker statement

\[
J(\mathbf Q)=C+\lambda J(\mathbf Q)
\]

is enough, because `lambda` acts invertibly on the order-19 cusp group. With the generator convention below, `sigma_*` acts as multiplication by `11`, so `lambda` acts as multiplication by `1-11=9`, which is a unit modulo `19`.

Since `lambda^2=-3sigma_*`, `lambda`-surjectivity makes every point infinitely 3-divisible. The same elliptic splitting and local separatedness then give `[76]J(Q)=0`.

This route is mathematically the most intrinsic and is closest in spirit to the original Mazur--Tate flat-cohomology descent. It is probably not the shortest Lean route unless the project already has isogeny descent for a two-dimensional Picard group. With the full elliptic API available, the factor-by-factor 3-isogeny descent is more reusable.

## 5.7 The earlier `2`-descent route

The Q4478 proposal of a fixed hyperelliptic `2`-descent is mathematically viable. Its Kummer map is the usual genus-two map into squareclasses of

\[
\mathbf Q[\theta]/(f_{13}(\theta)).
\]

A finite certificate could prove `J(Q)=2J(Q)` and then use a prime above `2` after base change to `K`, where the elliptic factors have everywhere good reduction.

However:

- it introduces a sextic étale-algebra squareclass calculation;
- the needed reduction exponent at the prime over `2` must still be computed;
- the assertion in Q4478 that there is “no useful small isogeny” is false: the ECNF elliptic factors are 3-isogenous;
- reduction at `2` on the original rational sextic is not good reduction.

Therefore prime `3` is the recommended route.

# 6. From an annihilator to the exact finite group

## 6.1 Bounded exponent is not finiteness in abstract Lean group theory

The implication

```text
[76]J(Q)=0  ->  J(Q) is finite
```

is false for an arbitrary additive group. An infinite direct sum of copies of `Z/2` has exponent two.

Classically, an abelian variety has finite `N`-torsion, but if the project is avoiding a general Jacobian/abelian-variety API, that theorem is not available for free. Add an explicit finite-capture theorem.

## 6.2 Good reduction at `5`

The curve has good reduction at `5`. The exact finite data are

\[
\boxed{\#J(\mathbf F_5)=19.}
\]

A convenient finite certificate is

```text
#C(F5)  = 6
#C(F25) = 12
P_5(T)  = 1 - 7*T^2 + 25*T^4
#J(F5)  = P_5(1) = 19.
```

Kamienny--Newman also report `#J(F25)=19^2`, consistent with

\[
P_5(1)P_5(-1)=19^2.
\]

All of these are small `native_decide` enumerations once `F25` is represented as a quadratic extension of `ZMod 5`.

Since `gcd(76,5)=1`, good-reduction formal-group theory gives

\[
\operatorname{red}_5:
J(\mathbf Q)[76]\hookrightarrow J(\mathbf F_5).
\]

Thus

\[
\#J(\mathbf Q)\le19.
\]

The explicit cusp class `G` has order `19`, so equality holds:

\[
\boxed{J(\mathbf Q)=\langle G\rangle\cong\mathbf Z/19.}
\]

Lean-facing endpoints:

```lean
theorem card_J13_F5 : Fintype.card J13F5 = 19 := by
  native_decide

noncomputable def red5 : J13Q →+ J13F5 := ...

theorem red5_injective_on_76_torsion :
    Set.InjOn red5 {D | 76 • D = 0} := ...

noncomputable def j13EquivZMod19 : J13Q ≃+ ZMod 19 := ...
```

The hard part is not the nineteen-element computation. It is constructing reduction from locally integral representatives, rather than reducing arbitrary rational coefficients whose denominators may be divisible by `5`.

# 7. The six cusp classes explicitly

## 7.1 Names and the cusp cycle

Use

\[
\begin{array}{c|c}
\text{name}&\text{point}\\ \hline
A&\infty_-=(1,-1,0)\\
B&(0,-1)\\
C&(-1,-1)\\
D&\infty_+=(1,1,0)\\
E&(0,1)\\
F&(-1,1).
\end{array}
\]

The order-six automorphism `rho` cycles them as

\[
\boxed{A\to B\to C\to D\to E\to F\to A.}
\]

This follows immediately from

\[
\rho[X:Y:Z]=[-Z:Y:X+Z]
\]

with weighted-projective rescaling at infinity.

## 7.2 A useful modular-unit-style divisor identity

Put

\[
p(x)=x^3+2x^2+x-1.
\]

Then

\[
\boxed{f_{13}(x)-p(x)^2=4x(x+1).}
\]

The function `y-p(x)` has divisor

\[
\boxed{
\operatorname{div}(y-p(x))=B+C+D-3A.
}
\]

Similarly,

\[
\operatorname{div}(x)=B+E-A-D,
\]

and

\[
\operatorname{div}(x+1)=C+F-A-D.
\]

These three identities are extremely useful Lean certificates: the polynomial identity is `ring`, and the valuation orders at the six explicit points are local one-line computations.

## 7.3 Cusp codes in `Z/19`

Set

\[
G=[B-A].
\]

The automorphism `rho_*` acts on the cyclic cusp group by multiplication by `12`:

\[
\boxed{\rho_*G=12G.}
\]

One conceptual derivation is as follows. Write `rho_*G=rG`. Since `rho^3` is hyperelliptic, `r^3=-1 mod 19`. The divisor of `y-p(x)` gives

\[
G+(1+r)G+(1+r+r^2)G=0.
\]

Among the two primitive sixth roots `r=8,12 mod 19`, this relation selects `r=12`.

The Abel--Jacobi images relative to `A` are therefore the partial sums of

\[
1,12,11,18,7,8.
\]

Explicitly:

\[
\boxed{
\begin{array}{c|c}
P&[P-A]\text{ in units of }G\\ \hline
A=\infty_-&0\\
B=(0,-1)&1\\
C=(-1,-1)&13\\
D=\infty_+&5\\
E=(0,1)&4\\
F=(-1,1)&11.
\end{array}}
\]

Thus the cusp-code set is

\[
\boxed{\{0,1,4,5,11,13\}\subset\mathbf Z/19.}
\]

These numbers are tied to the chosen generator `G=[(0,-1)-infinityMinus]`. Changing the generator multiplies all six codes by a unit modulo `19`.

# 8. From `J(Q)=Z/19` to `C(Q)=cusps`

## 8.1 Abel--Jacobi injectivity

Choose `A=infinityMinus` and define

\[
\iota_A:C(\mathbf Q)\to J(\mathbf Q),
\qquad
P\mapsto[P-A].
\]

This map is injective. If `[P-A]=[Q-A]`, then `P-Q` is principal. For `P != Q`, a function with divisor `P-Q` would have a single simple pole and hence induce a degree-one morphism `C -> P^1`, forcing `C` to have genus zero. Contradiction.

In the concrete balanced-Mumford implementation, injectivity can instead be proved by uniqueness of canonical representatives; this is usually shorter in Lean.

## 8.2 Finiteness alone is not enough

Even after proving `J(Q)` has 19 elements, injectivity gives only

\[
\#C(\mathbf Q)\le19.
\]

It does not show that the six known points are all the rational points. One must compute the Abel--Jacobi image.

## 8.3 The nineteen-entry point-class table

Construct the unique balanced reduced Mumford representative of

\[
[k]G,\qquad 0\le k<19.
\]

For an even-degree genus-two model, the representation must retain the infinity-orientation counter. A class is of the form `[P-A]` for a rational point `P` exactly when its canonical representative has point degree at most one, including the two infinity-counter cases.

The finite endpoint should be

\[
\boxed{
(\exists P\in C(\mathbf Q),\ [P-A]=kG)
\quad\Longleftrightarrow\quad
k\in\{0,1,4,5,11,13\}.
}
\]

Lean shape:

```lean
def cuspCodes13 : Finset (ZMod 19) :=
  {0, 1, 4, 5, 11, 13}

theorem multiple_is_point_class_iff (k : ZMod 19) :
    (∃ P : C13PointQ, abelJacobi P = k.val • G) ↔
      k ∈ cuspCodes13 := by
  finite_cases k
  -- normalize the stored balanced-Mumford representative of k • G
  -- six point-class branches; thirteen degree-two branches
  ...
```

Then

```lean
theorem C13_rational_points_are_cusps (P : C13PointQ) :
    P = infinityMinus ∨ P = infinityPlus ∨
    P = cusp0Minus ∨ P = cusp0Plus ∨
    P = cuspNeg1Minus ∨ P = cuspNeg1Plus := by
  let k : ZMod 19 := j13EquivZMod19 (abelJacobi P)
  have hk := (multiple_is_point_class_iff k).mp ⟨P, ...⟩
  fin_cases hk <;> ...
```

This is a genuinely finite algebraic computation. It is not Chabauty.

# 10. Tate-normal-form bridge to order `13`

For the modular interpretation, the optimized model

# 11. Dependency-ordered Lean plan

## Block A -- curve, points, and automorphisms

1. Define `f13`, `f13Hom`, the smooth projective point type, and the six cusp constructors.

# 12. Audit of Q4478

The following parts of Q4478 are correct and should be retained:

- the sextic polynomial `x^6+4x^5+6x^4+2x^3+x^2+2x+1`;
- the six rational cusp coordinates;
- the field `K=Q(zeta_13)^+` and its degree-six polynomial;
- the even-sextic transformation and the `c0,c2,c4,c6` formulas;
- the two quotient maps and the push-pull strategy;
- the need for a finite nineteen-class Abel--Jacobi image table.

The following points should be corrected:

1. **Use prime 3 as the preferred descent prime.** The order-three automorphism gives `lambda=1-sigma`, and the elliptic factors are explicitly 3-isogenous.
2. **There is a useful small isogeny.** ECNF curves `1.1-a2` and `1.1-a3` are linked by degree `3`.
3. **Do not claim good reduction at 2 for the rational sextic/Jacobian.** Its displayed discriminant is divisible by `2`. Use good reduction at `5` for the final finite capture.
4. **Do not infer finiteness merely from an annihilator in an abstract group.** Add reduction injectivity and `#J(F5)=19`.
5. **Do not identify quotient curves by labels alone.** Store explicit variable changes and point-level additive equivalences.
6. **The weak descent output must contain witnesses.** “Rank bound zero” or a Selmer dimension is not the theorem consumed by formal separatedness.
