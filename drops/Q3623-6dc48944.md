## 1. Recommended next theorem

### Base theorem: integral-of-gradient Duhamel vanishes

Add this to `ShenWork/PDE/IntervalGradDuhamelBound.lean`, after `gradDuhamel_sup_bound`.

```lean
/-- Zero-time vanishing of the full-kernel gradient-Duhamel integral for a
uniformly bounded source family.  This is the `T := t` specialization of
`gradDuhamel_sup_bound`, with the `√t` factor made into an epsilon-delta limit. -/
theorem gradDuhamel_tendsto_zero_of_bounded
    {q : ℝ → ℝ → ℝ} {Cq : ℝ}
    (hCq : 0 ≤ Cq)
    (hq_int : ∀ s, Integrable (q s) (intervalMeasure 1))
    (hq_sup : ∀ s y, |q s y| ≤ Cq)
    (hg_int : ∀ {t x : ℝ}, 0 < t →
      IntervalIntegrable
        (fun s : ℝ => deriv
          (fun z : ℝ => intervalFullSemigroupOperator (t - s) (q s) z) x)
        volume 0 t) :
    ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ → ∀ x : ℝ,
      |∫ s in (0 : ℝ)..t,
          deriv (fun z : ℝ => intervalFullSemigroupOperator (t - s) (q s) z) x| < ε := by
  intro ε hε
  let A : ℝ := heatGradientLinftyLinftyConstant * 2 * Cq
  have hA_nonneg : 0 ≤ A := by
    dsimp [A]
    positivity
  let δ : ℝ := (ε / (A + 1)) ^ 2
  have hδ : 0 < δ := by
    dsimp [δ]
    positivity
  refine ⟨δ, hδ, ?_⟩
  intro t ht htδ x
  have hbound := gradDuhamel_sup_bound
    (t := t) (T := t) ht (le_rfl)
    (q := q) hq_int hCq hq_sup x (hg_int (t := t) (x := x) ht)
  have hsqrt_bound : Real.sqrt t < ε / (A + 1) := by
    dsimp [δ] at htδ
    rw [← Real.sqrt_sq (show 0 ≤ ε / (A + 1) by positivity)]
    exact Real.sqrt_lt_sqrt ht.le htδ
  have htail : heatGradientLinftyLinftyConstant * (2 * Real.sqrt t) * Cq < ε := by
    have hA_lt : A * Real.sqrt t < ε := by
      calc
        A * Real.sqrt t < A * (ε / (A + 1)) :=
          mul_lt_mul_of_nonneg_left hsqrt_bound hA_nonneg
        _ ≤ (A + 1) * (ε / (A + 1)) := by
          refine mul_le_mul_of_nonneg_right ?_ (by positivity)
          linarith
        _ = ε := by
          field_simp [show A + 1 ≠ 0 by positivity]
    simpa [A, mul_comm, mul_left_comm, mul_assoc] using hA_lt
  exact lt_of_le_of_lt hbound htail
```

## 2. Does the repo already have enough for the logistic derivative leg?

Yes, modulo standard regularity hypotheses that are already explicit in the estimate style:

1. The source family is uniformly bounded:
   ```lean
   ∀ s y, |q s y| ≤ Cq
   ```
2. Each source slice is integrable against `intervalMeasure 1`:
   ```lean
   ∀ s, Integrable (q s) (intervalMeasure 1)
   ```
3. The gradient integrand is interval-integrable in time:
   ```lean
   IntervalIntegrable (fun s => deriv (fun z => S_full(t-s)(q s) z) x) volume 0 t
   ```
4. If the consumer is written as `deriv (fun x => ∫ S_full...)`, supply the Leibniz identity rewriting it to the integral of `deriv S_full`.

For the actual logistic term, take:

```lean
q s := logisticLifted p (D.u s)
```

The source bound exists in the broader Picard/cone code via the logistic source bounds, e.g. usages of `intervalLogisticSource_lift_abs_bound` and `logisticLifted`.  The theorem above should stay generic; a later bridge can instantiate it with whichever ball data the current construction carries.

This is faithful to the full-kernel mild map because `IntervalGradientDuhamelMap.lean` uses `intervalFullSemigroupOperator`, not the helper operator.

## 3. Is the repo enough for the chemotaxis differentiated leg?

Not from boundedness alone.

The chemotaxis derivative leg, after differentiating the gradient mild map once more in space, has the shape:

```lean
∫ s in (0 : ℝ)..t,
  deriv (fun z => deriv
    (fun w => intervalFullSemigroupOperator (t - s) (chemFluxLifted p (D.u s)) w) z) x
```

A bounded-source estimate for `∂ₓₓ S_full(σ)` has a nonintegrable `σ⁻¹` singularity.  The repository explicitly documents this in `IntervalFullKernelSecondDerivCtheta.lean:10-16`.  The correct estimate is the Hölder cancellation theorem:

```lean
theorem neumannHeatSecondDeriv_Ctheta_to_Linfty {t θ : ℝ} (ht : 0 < t)
    (hθ0 : 0 < θ) (hθ1 : θ < 1) {h : ℝ → ℝ}
    (hh_meas : AEStronglyMeasurable h (intervalMeasure 1))
    {Ch : ℝ} (hh : ∀ y, |h y| ≤ Ch) {Hh : ℝ} (hHh_nn : 0 ≤ Hh)
    (hHh : ∀ a b, a ∈ Set.Icc (0:ℝ) 1 → b ∈ Set.Icc (0:ℝ) 1 →
      |h a - h b| ≤ Hh * |a - b| ^ θ)
    {x : ℝ} (hx : x ∈ Set.Icc (0:ℝ) 1) :
    |deriv (fun z : ℝ => deriv
      (fun w : ℝ => intervalFullSemigroupOperator t h w) z) x|
      ≤ weightedHeatHessConst θ * t ^ (-1 + θ / 2 : ℝ) * Hh
```

So the next chemotaxis theorem must assume a **uniform Hölder modulus** for `chemFluxLifted p (D.u s)` on the small time interval.  Existing boundedness hypotheses are not enough.

Recommended future theorem, but not the smallest next step:

```lean
-- Target file: probably a new PDE file, e.g.

theorem secondDerivDuhamel_tendsto_zero_of_uniform_holder
    {q : ℝ → ℝ → ℝ} {θ Hq Cq : ℝ}
    (hθ0 : 0 < θ) (hθ1 : θ < 1)
    (hHq : 0 ≤ Hq)
    (hq_meas : ∀ s, AEStronglyMeasurable (q s) (intervalMeasure 1))
    (hq_bound : ∀ s y, |q s y| ≤ Cq)
    (hq_holder : ∀ s a b,
      a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
        |q s a - q s b| ≤ Hq * |a - b| ^ θ)
    (h2_int : ∀ {t x : ℝ}, 0 < t →
      IntervalIntegrable
        (fun s : ℝ => deriv (fun z : ℝ => deriv
          (fun w : ℝ => intervalFullSemigroupOperator (t - s) (q s) w) z) x)
        volume 0 t) :
    ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ → ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |∫ s in (0 : ℝ)..t,
        deriv (fun z : ℝ => deriv
          (fun w : ℝ => intervalFullSemigroupOperator (t - s) (q s) w) z) x| < ε
```

Proof route:

* For `s < t`, apply `neumannHeatSecondDeriv_Ctheta_to_Linfty` to `σ = t - s`.
* Dominate by:
  ```lean
  weightedHeatHessConst θ * Hq * (t - s) ^ (-1 + θ / 2)
  ```
* Integrate:
  ```lean
  ∫₀ᵗ (t - s)^(-1 + θ/2) ds = t^(θ/2) / (θ/2)
  ```
  or an equivalent `2/θ * t^(θ/2)` formula.
* Choose `δ` so that the resulting constant times `t^(θ/2)` is `< ε`.

This theorem would close the chemotaxis differentiated leg **only after** a downstream proof supplies uniform Hölder data for

```lean
chemFluxLifted p (D.u s)
```

near zero.  That uniform Hölder input is itself nontrivial and should not be smuggled in from `PatchedSliceDerivUniformApproachAtZero` or `ux_zeroFace`.

## 4. Faithfulness/circularity checks

### Do not consume the target residual

Do not use:

```lean

```

or the derived zero-face theorem:

```lean

```

to prove any part of the Duhamel vanishing.  Those are downstream consumers.

### Do not mix helper and full semigroups

The mild map is full-kernel.  The theorem above must use:

```lean

```

Do not use:

```lean

```

unless a bridge theorem is explicitly proved.  The older `IntervalCoupledBallEstimates.lean` stack still uses the helper operator in some hmap/hint estimates, so it is not the right target for this initial-derivative trace frontier.

### Do not claim boundedness closes the chemotaxis differentiated leg

Boundedness closes the logistic derivative leg through the `∂ₓS_full` estimate.  It does not close the chemotaxis differentiated leg through `∂ₓₓS_full`; the latter requires Hölder cancellation.
