# Q3983 — source cosine coefficient absolute summability

## Bottom line

The requested theorem is **not provable** from the stated hypotheses, and it should
not be added as a theorem:

```lean
theorem resolverSourceCoeff_re_abs_summable_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x) :
    Summable fun k : ℕ =>
      |(intervalNeumannResolverSourceCoeff p u k).re| := by
  -- impossible from continuity alone
  sorry
```

The obstruction is mathematical, not Lean-specific.  Continuity of the source
function on `[0,1]` gives Bessel/Parseval-type `ℓ²` control of its cosine
coefficients, and this is exactly what the repo already proves in
`resolverSourceCoeff_re_sq_summable_of_continuousOn`.  But `ℓ²` does not imply
`ℓ¹`, and continuous functions can have Fourier/cosine coefficients that are not
absolutely summable.  The nonnegativity hypothesis only identifies
`positivePart u` with `u`; it gives no extra smoothness or coefficient decay.

The key asymmetry is:

* resolver coefficients `c_k` are absolutely summable because
  `c_k = a_k / (μ + (kπ)^2)` has a two-derivative elliptic gain;
* source coefficients `a_k` have no such gain.  In fact
  `a_k = (μ + (kπ)^2)c_k`, which loses exactly the smoothing.

So the theorem as stated would assert that every continuous source has an
absolutely convergent cosine series.  That is false.

## What is actually needed for the weak ODE bridge

For the purely spectral proof of

```lean
resolverGradReal p u b - resolverGradReal p u a
  = ∫ t in a..b, resolverLapPhysicalReal p u t
```

you do **not** need `∑ |a_k| < ∞`.  The source part only needs

```lean
Summable fun k : ℕ =>
  |(intervalNeumannResolverSourceCoeff p u k).re *
    ∫ x in a..b, unitIntervalCosineMode k x|
```

This follows from Cauchy--Schwarz:

```text
(a_k) ∈ ℓ²                         -- already proved by resolverSourceCoeff_re_sq_summable_of_continuousOn
(J_k) ∈ ℓ², J_k = ∫_a^b cos(kπx) dx -- explicit: J_0 = b-a, J_k = O(1/k)
⇒ (a_k J_k) ∈ ℓ¹.
```

That is the correct replacement lemma.  It is strictly weaker than source
coefficient absolute summability and is exactly suited to the interval integral.

Add this instead of the impossible theorem:

```lean
import ShenWork.Paper2.IntervalResolverWeakBounds

/-- The interval cosine primitive coefficients are square-summable.
This is the correct replacement for trying to prove `∑ |a_k| < ∞`.
For `k > 0`,
`∫_a^b cos(kπx) dx = (sin(kπb)-sin(kπa))/(kπ)`, hence the square is `O(1/k²)`.
-/
theorem interval_cosine_integral_sq_summable (a b : ℝ) :
    Summable fun k : ℕ =>
      (∫ x in a..b, unitIntervalCosineMode k x) ^ 2 := by
  classical
  -- Proof sketch:
  -- split off `k = 0`; for `k+1`, use the explicit primitive formula and
  -- `|sin _ - sin _| ≤ 2`, so
  --   |J_{k+1}|² ≤ 4 / (((k+1)π)²).
  -- Then compare with `Real.summable_one_div_nat_add_rpow` at exponent 2.
  sorry

/-- Source coefficients paired with an interval cosine test are absolutely
summable from the existing source `ℓ²` theorem and the square-summability of the
interval test coefficients.
-/
theorem resolverSourceCoeff_re_mul_intervalCosine_integral_abs_summable
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {a b : ℝ} :
    Summable fun k : ℕ =>
      |(intervalNeumannResolverSourceCoeff p u k).re *
        ∫ x in a..b, unitIntervalCosineMode k x| := by
  classical
  have ha2 : Summable fun k : ℕ =>
      ((intervalNeumannResolverSourceCoeff p u k).re) ^ 2 := by
    simpa [ShenWork.Paper2.intervalNeumannResolverSourceCoeff_zero] using
      resolverSourceCoeff_re_sq_summable_of_continuousOn p hUcont
  have hJ2 : Summable fun k : ℕ =>
      (∫ x in a..b, unitIntervalCosineMode k x) ^ 2 :=
    interval_cosine_integral_sq_summable a b

  -- Cauchy--Schwarz / AM-GM majorization:
  -- `|a_k J_k| ≤ (a_k^2 + J_k^2)/2`.
  refine Summable.of_nonneg_of_le (fun k => abs_nonneg _) ?_
    ((ha2.add hJ2).div_const 2)
  intro k
  set ak : ℝ := (intervalNeumannResolverSourceCoeff p u k).re
  set Jk : ℝ := ∫ x in a..b, unitIntervalCosineMode k x
  calc
    |ak * Jk| = |ak| * |Jk| := abs_mul ak Jk
    _ ≤ (ak ^ 2 + Jk ^ 2) / 2 := by
      have h := two_mul_le_add_sq |ak| |Jk|
      rw [sq_abs, sq_abs] at h
      nlinarith [h]

end ShenWork.IntervalResolverWeakBounds
```

## Source integral theorem without false `ℓ¹`

For the spectral ODE bridge, prove the source integral by Hilbert-space pairing,
not by `integral_tsum_pairing` with an `ℓ¹` majorant:

```lean
/-- Correct source interval-integral identity.
No absolute summability of source coefficients is assumed.  The series on the
right is absolutely summable only after multiplication by the interval test
coefficient `J_k`.
-/
theorem resolverPhysicalSourceReal_integral_tsum_L2
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x)
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    (∫ x in a..b, resolverPhysicalSourceReal p u x)
      = ∑' k : ℕ,
          (intervalNeumannResolverSourceCoeff p u k).re *
            ∫ x in a..b, unitIntervalCosineMode k x := by
  classical
  -- Recommended proof route:
  -- 1. Let `ρ(x) = resolverPhysicalSourceReal p u x` on `[0,1]`.
  -- 2. Let `ψ = Set.Ioc a b` or the corresponding interval test.
  -- 3. Use cosine Parseval / L² pairing:
  --      ∫ ρ ψ = ∑ coeff(ρ,k) coeff(ψ,k).
  -- 4. Identify `coeff(ρ,k)` with `(intervalNeumannResolverSourceCoeff p u k).re`
  --    using `hUnonneg`.
  -- 5. Identify `coeff(ψ,k)` with `∫_a^b unitIntervalCosineMode k`.
  -- 6. Summability is discharged by
  --    `resolverSourceCoeff_re_mul_intervalCosine_integral_abs_summable`.
  sorry
```

## If you really want an `ℓ¹` theorem

You need a stronger regularity assumption on the source, e.g. a hypothesis that
its cosine coefficients have one extra summable decay, or a concrete Sobolev/BV
condition strong enough to imply `|a_k| = O(k^{-1-ε})`.  A usable abstract Lean
version is:

```lean
theorem resolverSourceCoeff_re_abs_summable_of_weighted
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hweighted : Summable fun k : ℕ =>
      ((1 : ℝ) + ((k : ℝ) * Real.pi) ^ 2) *
        |(intervalNeumannResolverSourceCoeff p u k).re|) :
    Summable fun k : ℕ =>
      |(intervalNeumannResolverSourceCoeff p u k).re| := by
  refine Summable.of_nonneg_of_le (fun k => abs_nonneg _) ?_ hweighted
  intro k
  have hge : 1 ≤ (1 : ℝ) + ((k : ℝ) * Real.pi) ^ 2 := by nlinarith [sq_nonneg ((k : ℝ) * Real.pi)]
  exact le_mul_of_one_le_left (abs_nonneg _) hge
```

But this is not derivable from `hUcont` and `hUnonneg` alone.

## Recommendation

Do **not** try to close Q3983 as stated.  Replace the use of
`resolverSourceCoeff_re_abs_summable_of_continuousOn` in the Q3981 spectral proof
with the paired summability lemma
`resolverSourceCoeff_re_mul_intervalCosine_integral_abs_summable` and an L²
Parseval/pairing source integral theorem.  This preserves the purely spectral
proof and avoids introducing a false API theorem.
