# Q3833 — nonnegativity from uniform core vs Cron2, and strong positivity in Lean

## Executive answer

There are two viable routes, but the lowest-risk Lean route is **not** a brand-new proof from raw Mathlib. It is to **refactor the existing Cron2/negative-part track so it is trajectory- or uniform-core-typed instead of `ConjugateMildExistenceData`/PPID-typed**.

A still cheaper route may exist if the `UniformConjugateMildExistenceCore` can provide the same cone-preservation fields used by the old Picard data, especially an `hmapsTo_nn`-style field. The existing Picard limit theorem already proves:

```lean
conjugatePicardIter nonnegative for all n
  -> conjugatePicardLimit nonnegative
```

via

```lean
theorem conjugatePicardLimit_nonneg
```

in `ShenWork/Paper2/IntervalConjugatePicard.lean:142-161`. So if the uniform core can prove nonnegative iterates, use that. But do **not** claim this from a naive “nonnegative kernel times nonnegative source” argument: the B-form chemotaxis source is not a nonnegative scalar source.

For strict positivity, do not expect Mathlib to supply a ready parabolic strong maximum principle. Use the repo’s heat-kernel positivity / lower-barrier route instead.

### But the naive kernel-positivity argument is not correct

The statement

> “the Duhamel integral of a nonnegative source with a nonnegative kernel is nonnegative”

is not the B-form chemotaxis map as written.

The logistic term is not globally nonnegative: for large `u`,

```text
u*(a - b*u^alpha)
```

can be negative. This is okay for invariance of the nonnegative cone because it is bounded below by `-C*u`, but it is not a nonnegative source.

The chemotaxis term is also not a nonnegative scalar source. The flux has a factor of `u`, so it vanishes where `u=0`, which is exactly what a maximum-principle/negative-part argument needs. But the flux itself contains `v_x`, and the divergence/B-kernel contribution is sign-changing in general.

Thus the valid direct arguments are:

1. a cone-preservation theorem for the actual Picard map, with lower-order terms controlled; or
2. the negative-part energy argument.

The invalid shortcut is treating the whole Duhamel integrand as a nonnegative scalar source.

## Is the negative-part proof doable in Lean without full Cron2?

Not from Mathlib alone in any meaningful sense.

Mathematically, the proof is exactly:

```text
test with u_minus,
chemotaxis cancellation from the leading u_+ factor,
reaction one-sided bound,
Gronwall,
initial negative part = 0.
```

But formalizing that for a mild solution requires the same hard ingredients Cron2 isolated:

* weak formulation from the mild Duhamel equation;
* B_N / integration-by-parts duality;
* chain rule for `u_minus`;
* energy differentiability;
* initial trace of the negative part;
* upgrade from zero L2 negative part to pointwise nonnegativity.

The current repo has already done large pieces of this. `IntervalBFormNegativePartCron2.lean` defines the truncated flux/map and the weak PDE interfaces; it also proves the final `negativePart_zero` from a `BFormCron2NegativePartHyp` certificate (`IntervalBFormNegativePartCron2.lean:23-185`). `IntervalBFormCron2NegativePartEnergy.lean` proves cancellation and Gronwall core, and explicitly lists the remaining weak/Sobolev obligations (`IntervalBFormCron2NegativePartEnergy.lean:18-35`).

So the answer is:

* If the solution is already classical, a shorter max-principle/negative-part proof is plausible.
* If the solution is only mild and nonnegativity is needed before classical bootstrap, “without Cron2” means re-proving the same weak/Sobolev machinery under another name.
* The best Lean engineering is to refactor Cron2 to uniform-core trajectory data, not discard it.

## Strict positivity for `t > 0`

I would not rely on Mathlib for a parabolic strong maximum principle. A search of current mathlib4 names for `parabolic strong maximum principle` and `strong maximum principle` did not reveal a usable parabolic SMP theorem; the hits are unrelated complex/convex/order files, not a heat-equation SMP.

Use the repo-specific heat-kernel/lower-barrier route instead. The key already-landed theorem is:

```lean
theorem intervalFullSemigroupOperator_pos_of_positiveInitialDatum
    (hu0 : PositiveInitialDatum intervalDomain u0)
    {t : ℝ} (ht : 0 < t) (x : ℝ) :
    0 < intervalFullSemigroupOperator t (intervalDomainLift u0) x
```

File: `ShenWork/Paper2/IntervalBFormNegPartStrictPosBarrier.lean:134-145`.

And the lower-barrier-to-strict-positivity theorem is:

```lean
theorem bform_strictPos_of_semigroup_lower_barrier
    ...
    (hbarrier :
      ∀ t x, 0 < t -> t < DB.T ->
        Real.exp (-C*t) * intervalFullSemigroupOperator t (intervalDomainLift u0) x.1
          <= conjugatePicardLimit p u0 DB.T t x) :
    ∀ t x, 0 < t -> t < DB.T ->
      0 < conjugatePicardLimit p u0 DB.T t x
```

File: `IntervalBFormNegPartStrictPosBarrier.lean:160-184` in the fetched chunk.

For uniform core, generalize this theorem from `DB.T` to an arbitrary `T,u`, then prove the lower barrier for the uniform-core trajectory. This is much easier than formalizing a general parabolic strong maximum principle.

## Recommended route

### First choice: prove cone preservation in the uniform Picard core

If feasible, add fields/lemmas:

```lean
hbase_nonneg_uniform
hmapsTo_nn_uniform
conjugatePicardIter_nonneg_of_uniformCore
conjugatePicardLimit_nonneg_of_uniformCore
```

Then build the missing `ConjugateMildSolutionData.hnonneg` using the existing `conjugatePicardLimit_nonneg`.

For strict positivity, do **not** ask this cone argument to prove `hpos`; use a heat lower barrier:

```lean
uniformCore_heat_lower_barrier
strictPos_of_uniformCore_heat_lower_barrier
```

### Second choice: generic Cron2 refactor

If cone preservation is not straightforward because the B-form map is sign-indefinite, refactor Cron2:

```lean
NegativePartEnergyCoreDataFor p T u
nonneg_of_negativePartEnergyCoreDataFor
negativePart_zero_of_uniformCore_energy
```

This reuses the already-proved cancellation/Gronwall lemmas and avoids PPID.

### Avoid

Avoid any theorem that states or secretly uses:

```lean
PositiveInitialDatum intervalDomain u0 -> PaperPositiveInitialDatum intervalDomain u0
```

Also avoid claiming a raw B-form Duhamel source is nonnegative. The structural truth is “zero is a subsolution” / “negative part energy cannot grow,” not “all Duhamel source terms are nonnegative.”

## Final answers

1. **Can Cron2 be adapted to uniform core?** Yes. Refactor the energy package to be `T,u`-parametric or uniform-core-parametric. The proof content is already mostly independent of PPID.

2. **Can nonnegativity be proved more directly?** Maybe, if the uniform core can prove Picard cone preservation (`hmapsTo_nn`). The existing theorem `conjugatePicardLimit_nonneg` then gives the limit. But the naive nonnegative-source/kernel argument is not correct for the B-form chemotaxis map.

3. **Can the negative-part proof be done without full Cron2?** For a merely mild solution, not really. You either use/refactor Cron2 or rebuild its weak formulation, chain rules, and energy differentiability. For an already classical solution, a shorter maximum-principle proof is plausible.

4. **Does Mathlib have a parabolic strong maximum principle?** I would treat the answer as no for this project. Use the repo’s Neumann heat-kernel positivity and lower-barrier lemmas instead.
