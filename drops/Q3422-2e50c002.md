## Executive answer

I did **not** find an existing named theorem that already proves the full target

```lean
intervalDomainLift
  (fun X => intervalDomain.chemotaxisDiv p (u t) (v t) X) x
  = intervalDomainChemDivPhysicalRep p u v t x
```

on `t ∈ Set.Ioo 0 T`, `x ∈ Set.Ioo 0 1`.

What already exists is enough to make this a small local calculus theorem, not a new H1 or bounded-before frontier:

1. The repo already uses the definitional fact that the lifted chemotaxis divergence is `deriv (intervalFlux p (u t) (v t))` on interior spatial points.
2. The repo already has the exact `v_xx = μV - νU^γ` lifted interior theorem.
3. The missing piece is a named product/quotient/rpow derivative theorem for `intervalFlux`, followed by the existing `v_xx` substitution.

The clean route is:

```text
lift chemDiv
= deriv intervalFlux                                  -- definitional, interior branch
= Ux*Vx/(1+V)^β + U*Vxx/(1+V)^β - β*U*Vx^2/(1+V)^(β+1) -- product/quotient/rpow
= Ux*Vx/(1+V)^β + U*(μV-νU^γ)/(1+V)^β - β*U*Vx^2/(1+V)^(β+1)
```

No endpoint equality is involved; no division by `χ₀`; no H1 or bounded-before output is needed.

## Tactic/API risks

### 1. `rpow` derivative side condition

Use `HasDerivAt.rpow_const` with the side condition

```lean
Or.inl (ne_of_gt hbase)
```

where `hbase : 0 < 1 + intervalDomainLift (v t) x`. Do not use a nonnegativity-only side condition; for the derivative theorem, nonzero positive base is the stable route.

### 2. Derivative value order from `rpow_const`

The derivative produced by the API is likely ordered as

```lean
Vx * params.β * B ^ (params.β - 1)
```

not necessarily

```lean
params.β * Vx * B ^ (params.β - 1)
```

Keep the algebra lemma tolerant with `mul_comm`, `mul_left_comm`, and `mul_assoc`, or bind the value exactly as produced and normalize at the end.

### 3. `Real.rpow_*` algebra is not `ring`

The step

```lean
B ^ (β - 1) / B ^ (2 * β) = 1 / B ^ (β + 1)
```

will not be found by `ring` or `ring_nf`. Copy the proof pattern from the private `chemDiv_flux_deriv_algebra`:

```lean
Real.rpow_mul
Real.rpow_sub
Real.rpow_neg
field_simp
ring_nf
```

with `hB : 0 < B` and nonzero proofs from `Real.rpow_pos_of_pos`.

### 4. `ContDiffOn` to `HasDerivAt` on `Ioo`

Use `isOpen_Ioo.mem_nhds hx` / `IsOpen.mem_nhds isOpen_Ioo hx`. For example:

```lean
((hC2u.differentiableOn (by norm_num)).differentiableAt
  (isOpen_Ioo.mem_nhds hx)).hasDerivAt
```

For `deriv V`, first derive the `C¹` theorem on `Ioo`:

```lean
have hDV_C1 : ContDiffOn ℝ 1 (deriv (intervalDomainLift (v t))) (Set.Ioo (0 : ℝ) 1) :=
  hC2v.deriv_of_isOpen isOpen_Ioo (by norm_num)
```

### 5. Do not use closed endpoint facts

The theorem is interior-only. There is no need for `derivWithin`, `flux_endpoint_zero`, `intervalFluxByParts`, or endpoint equality. Endpoint facts are for integration by parts or closed-slab continuity, not for this EqOn proof.

### 6. Do not divide by `χ₀`

This theorem is about `chemotaxisDiv` itself. The `u` PDE contains `χ₀ * chemDiv`, but using that PDE would introduce an unnecessary and forbidden division-by-`χ₀` path. Use the definition of `chemotaxisDiv` plus the `v` elliptic equation.

### 7. Avoid H1 and bounded-before imports

The theorem only needs:

```lean

```

or a direct lower-level import path exposing `intervalFlux`, `intervalDomain_v_xx_eq_reaction_lift`, and `IsPaper2ClassicalSolution`. It should not import `IntervalChiNegH1...` files, bounded-before files, or scalar H1 route files.

## Quotient-form versus final `(β+1)` denominator form

The final representative with denominator `(1+V)^(β+1)` is mathematically correct and is the right closed-slab continuity target. But it is **not** the easiest first target for `HasDerivAt.div`.

The easiest Lean route is to use an intermediate quotient-rule representative first:

```lean
(((Ux * Vx + U * Vxx) * B ^ β - U * Vx * (Vx * β * B ^ (β - 1))) /
  (B ^ β) ^ 2)
```

Then normalize it by a separate algebra lemma to

```lean
Ux * Vx / B ^ β + U * Vxx / B ^ β - β * U * Vx ^ 2 / B ^ (β + 1)
```

Then rewrite

```lean
Vxx = μ * V - ν * U ^ γ
```

using `intervalDomain_v_xx_eq_reaction_lift`.

So: keep the final physical representative in the user-facing formula, but prove through a quotient-form/raw-quotient intermediate. This isolates calculus from real-power algebra and makes the proof much more robust.
