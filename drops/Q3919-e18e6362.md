# Q3919: chem-flux differentiability circularity in the truncated Stampacchia route

## Bottom line

Yes: the circularity analysis is correct.  A field requiring global classical differentiability of

```lean
truncatedChemFluxLifted p (u t)
```

before proving `u ≥ 0` is too strong.  At a simple zero of `u(t, ·)`, the factor `positivePart (u(t,y))` creates a kink.  So fields like

```lean
chem_hasDerivAt : ∀ x ∈ Set.Icc (0 : ℝ) 1,
  HasDerivAt (truncatedChemFluxLifted p (u t))
    (deriv (truncatedChemFluxLifted p (u t)) x) x

chem_deriv_contOn :
  ContinuousOn (fun x => deriv (truncatedChemFluxLifted p (u t)) x)
    (Set.Icc (0 : ℝ) 1)
```

should not be part of the pre-Stampacchia spectral data.  They are valid only after truncation is inactive, or under a special structural condition forcing the kink coefficient to vanish at every sign-changing zero.

## Q1. Is the circularity real?

Yes.  Locally write

```text
F(y) = positivePart(w(y)) * G(y)
```

where `G(y)` abbreviates the resolver-gradient/denominator factor.  Suppose

```text
w(y₀) = 0,
w'(y₀) ≠ 0,
G(y₀) ≠ 0.
```

Then on one side of `y₀`, `positivePart(w)=0`; on the other side, `positivePart(w)=w`.  Thus the one-sided derivatives are

```text
0
w'(y₀) * G(y₀)
```

up to the side determined by the sign of `w'`.  These do not agree unless `G(y₀)=0`.  Therefore `positivePart(w)*G` is not classically differentiable at a generic simple zero.

So if `TruncatedPositiveTimeSpectralData` is consumed to prove `u ≥ 0`, it cannot require global differentiability of `truncatedChemFluxLifted p (u t)`; that assumes away the possible sign-changing zeros that Stampacchia is supposed to eliminate.

## Q2. Is the off-countable fix sound?

It is sound, but the reason is **not** Lipschitzness alone.  The map `positivePart` is Lipschitz, so `positivePart ∘ w` is a.e. differentiable if `w` has enough Sobolev/absolutely-continuous regularity.  But an a.e. exceptional set need not be countable.

To get an **off-countable** statement, you need a stronger zero-set theorem.  A typical sufficient route in this 1D positive-time setting is:

1. the pre-truncation/restart/cosine representation gives real-analytic spatial slices on `(0,1)`;
2. a nonzero real-analytic function has isolated zeros;
3. isolated subsets of an interval are countable;
4. if the analytic function is identically zero on a component, then `positivePart(w)=0` locally there, so there is no kink.

Thus the non-differentiability set of `positivePart(w)*G` is contained in the set of non-flat/sign-changing zeros of `w`, and that set is countable under analyticity.  Without analyticity or an explicit countability hypothesis for the zero set, the off-countable claim is not justified.

### What proves countability?

A useful atom is not “Lipschitz implies countable.”  That is false.  The atom should be analytic/zero-set based:

```lean
/-- Analytic zero-set route for the kink set.  If the represented slice is analytic,
then the non-flat zeroes where `positivePart` can fail to be differentiable are
countable. -/
-- theorem countable_kinkSet_positivePart_mul_of_analytic
--     {w G : ℝ → ℝ}
--     (hw_an : AnalyticOn ℝ w (Set.Ioo (0 : ℝ) 1))
--     (hG_diff : ∀ x ∈ Set.Ioo (0 : ℝ) 1, DifferentiableAt ℝ G x) :
--     {x | x ∈ Set.Ioo (0 : ℝ) 1 ∧
--       ¬ DifferentiableAt ℝ (fun y => positivePart (w y) * G y) x}.Countable
```

If the repository does not already have analytic machinery available, carry the countable bad set as an explicit field rather than pretending it follows from `C²` or Lipschitz regularity.

## Q3. Does swapped `cosine_parseval_pairing` work?

Conceptually yes, provided the theorem’s hypotheses are genuinely asymmetric in the way you describe:

```text
continuous test function φ  as the “regular/continuous” role,
bounded measurable g'      as the “L² / measurable” role.
```

The integral is symmetric by commutativity of multiplication:

```text
∫ g'(x) * φ(x) dx = ∫ φ(x) * g'(x) dx.
```

The coefficient sum is also symmetric termwise:

```text
a_n(φ) * a_n(g') = a_n(g') * a_n(φ).
```

So role-swapping gives the same scalar pairing after rewriting by `mul_comm` / `Finset.sum_congr` / `tsum_congr` as appropriate.

The exact Lean feasibility depends on the actual statement of `cosine_parseval_pairing`.  If it requires continuity of both functions, it will not help.  If it allows the second role to be bounded measurable / `L²`, then the swap is the right move.  The necessary downstream hypotheses for `g' := deriv (truncatedChemFluxLifted p (u t))` are:

```lean
AEStronglyMeasurable g' (intervalMeasure 1)
∃ C, ∀ x ∈ Set.Icc (0 : ℝ) 1, |g' x| ≤ C
```

which imply `g' ∈ L²` on the finite interval.

## Q4. Is there a simpler fix?

Yes: avoid differentiating the truncated chem flux at all in the Stampacchia step.

For the weak energy argument, the chemotaxis term should remain in divergence/weak form:

```text
χ₀ ∫ Q_T(u) * ∂x φ
```

with the test `φ = -u⁻` or an approximation of it.  You need:

```text
Q_T(u) * ∂x(u⁻) = 0  a.e.
```

because the leading factor in `Q_T` is `positivePart(u)` or a power of it.  This cancellation uses the Lipschitz chain rule for `negativePart`, not differentiability of `Q_T(u)`.

The repository already has the scalar pointwise core in the visible file:

```lean
lemma truncatedChemotacticPower_mul_deriv_negativePart_eq_zero
    (p : CM2Params) {u : ℝ → ℝ} {x : ℝ}
    (hu : ContinuousAt u x) :
    truncatedChemotacticPower p (u x)
        * deriv (fun y : ℝ => negativePart (u y)) x = 0
```

That is much closer to the Stampacchia argument than proving `chem_hasDerivAt` for the flux.  The cleanest design is:

1. Keep the truncated mild-to-weak PDE in divergence form.
2. Test with `-u⁻` using the weak formulation.
3. Use the support cancellation of `positivePart(u)` against `∂x u⁻`.
4. Prove Gronwall for `∫ |u⁻|²`.
5. Only after `u ≥ 0`, rewrite `positivePart u = u`; at that point the classical chem-flux differentiability fields become valid for the untruncated/nonnegative solution if needed.

This avoids the circularity completely.  It also avoids needing a countable-zero-set theorem for the energy argument.  The off-countable spectral repair is sound if you need a spectral integration-by-parts route, but the weak-form Stampacchia route is simpler and more faithful.

## Direct answers

1. **Yes.**  `positivePart(w) * G` is not differentiable at a generic simple zero of `w` unless `G` vanishes there.  Requiring global `chem_hasDerivAt` before proving `u ≥ 0` is circular.

2. **Yes, with the analytic-zero-set caveat.**  Off-countable differentiability is sound if positive-time spectral/heat regularity gives analyticity, or if a countable kink set is carried/proved explicitly.  It does not follow from Lipschitzness alone.

3. **Yes, if the theorem’s hypotheses allow bounded measurable data in the swapped role.**  After swapping, commute factors in the integral and coefficient products.  Add measurability/integrability of `deriv g` explicitly.

4. **Simpler fix:** do not differentiate the truncated chem flux in the Stampacchia step.  Use the weak divergence formulation and the cancellation `positivePart(u) * ∂x u⁻ = 0` a.e.; recover classical chem-flux differentiability only after the nonnegativity theorem makes the truncation inactive.
