This isolates all analytic differentiability work.  The witnesses are:
* `dpos`: derivative of the positive part of `w`, bounded by `|w'|`;

## Important correction to the current theorem shape

The existing local source bound calls

```lean
∀ y, |Src n s y| ≤ ...
```

but the product-rule bridge for the flux derivative is only naturally true on `Ioo 0 1`.  This is not a mathematical problem for the coefficient/IBP argument, because endpoints are measure zero and all interval integrals are insensitive to changing the derivative there.  It is, however, a Lean API issue.

So the most robust implementation path is:

1. Prove the product-rule decomposition on `y ∈ Ioo 0 1`.
2. Prove the resulting source envelope on `Ioo 0 1`.
3. Add a measurable/a.e. wrapper lemma converting the interior pointwise envelope into the integral/coefficient estimate needed by the IBP step.

Trying to prove the displayed decomposition for arbitrary `y : ℝ` is too strong because:

* `intervalDomainLift` changes definition at `0` and `1`;
* `positivePart` has a kink at `0`;
* the Hessian identity is only available at interior points.

If the current surrounding helper absolutely requires `∀ y : ℝ`, then the right fix is not to force the product-rule lemma globally, but to weaken that helper to an a.e. or interior statement and use interval-integral congruence to ignore endpoints.
