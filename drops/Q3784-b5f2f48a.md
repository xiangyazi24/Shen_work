## Executive verdict

The remaining

```lean

  (coupledChemicalConcentration p D.u)
```

in


is too strong for the non-circular route. It appears only because the per-slice physical gradient-source bridge currently uses the full `IsPaper2ClassicalSolution` package to get flux differentiability and the physical chem-div representative. For the actual mild-solution closure, using full `hclassical` here is circular: the main missing field of `IsPaper2ClassicalSolution` is exactly `hpde_u`, and the integrated mixed-source bridge is intended to help prove that field.

The next smallest honest core theorem is therefore **not** another integrated wrapper. It is a **regularity-only version of the per-slice gradient-source bridge**: replace the full classical-solution assumption by the non-circular regularity data already produced by `IntervalMildToClassical`, namely an `intervalDomainClassicalRegularity` package, together with the positivity/nonnegativity already in `GradientMildSolutionData`.

```lean
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀)
    (hreg : intervalDomainClassicalRegularity D.T D.u
      (coupledChemicalConcentration p D.u))
    {s : ℝ} (hs : s ∈ Set.Ioo (0 : ℝ) D.T) :

      HasDerivWithinAt (chemFluxLifted p (D.u s))
        (coupledChemDivSourceLift p D.u s y) (Set.Ioi y) y := by
  intro y hy
  have hyIcc : y ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hy
  let Y : intervalDomainPoint := ⟨y, hyIcc⟩
  have hv_nonneg : ∀ t x, 0 < t → t < D.T →
      0 ≤ coupledChemicalConcentration p D.u t x := by

      mildChemical_nonneg p D.hnonneg D.hcont ht0 (le_of_lt htT) x
  have hflux :
      HasDerivAt
        (fun y' : ℝ =>
          intervalDomainLift (D.u s) y' *
            deriv (intervalDomainLift (coupledChemicalConcentration p D.u s)) y' /
            (1 + intervalDomainLift (coupledChemicalConcentration p D.u s) y') ^ p.β)

      (nhdsWithin (0 : ℝ) (Set.Ioi 0)) (nhds 0) :=
    (hreg.2.2.2.1 s hs).1.1
  have hN1 : Filter.Tendsto (deriv (intervalDomainLift (D.u s)))

      p D hs.1 hs.2.le hC2 hN0 hN1

            (1 + intervalDomainLift (coupledChemicalConcentration p D.u s) y') ^ p.β) := by
```

## Hidden circularity / vacuity risks

1. **Do not supply `hclassical` via `isPaper2ClassicalSolution_of_gradientMildSolutionData_*` while proving `hpde_u`.** Those theorems consume `GradientMildClassicalCoreData` or `GradientMildClassicalFrontierCoreData`, and both include `hpde_u`.

2. **Do not use `mildSolution_parabolicPDE` as a producer.** It only projects `hclassical.pde_u`.

3. **Do not replace `hclassical` with `RegularityBootstrap`.** The `RegularityBootstrap` constructors in this lane either consume `hclassical` directly or consume the core data containing `hpde_u`.

4. **Do not reintroduce endpoint continuity of zero extensions.** The Task303--308 endpoint-insensitive path is correct. The physical representative should only be equal to `coupledChemDivSourceLift` on `Ioo 0 1`.

5. **Avoid BForm detours.** The hclassical bottleneck here is a gradient-mild source-bridge issue, and the existing `GradientMildClassicalRegularityFrontierData` / restart-cosine packages are the right inputs.
