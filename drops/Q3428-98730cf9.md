# Q3428: H¹ initial endpoint audit

## Verdict

`H1InitialTraceEnergyTendsto` is still a genuine additional H¹ trace frontier in the current APIs. Task64/65 eliminated the explicit `hUxxL1` carry when `hsol : IsPaper2ClassicalSolution intervalDomain p T u v` is present, via

```lean
ShenWork.Paper2.IntervalChiNegH1ChemDivRepresentative.H1UxxL1ContBefore_of_classical_liftChemotaxisDivPhysicalRep
```

but it does **not** supply the right-continuity-at-zero input for `H1energy`.

The current repo already treats this as a separate endpoint seam. `IntervalChiNegH1InitialContinuity.lean` defines:

```lean
def H1InitialTraceEnergyTendsto
    (u₀ : intervalDomainPoint → ℝ)
    (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop :=
  Tendsto (H1energy u) (𝓝[Set.Ioc (0 : ℝ) T] (0 : ℝ))
    (𝓝 (H1InitialEnergy u₀))

def H1InitialEnergyCompatibleAtZero
    (u₀ : intervalDomainPoint → ℝ)
    (u : ℝ → intervalDomainPoint → ℝ) : Prop :=
  H1energy u 0 = H1InitialEnergy u₀
```

and proves only the topological endpoint glue:

```lean
theorem H1energy_continuousWithinAt_zero_of_initialTraceEnergy
```

from those two explicit endpoint assumptions.

## Why it is not derivable from the currently landed data

### `InitialTrace` is only a sup-norm trace

`Statements.lean` defines:

```lean
def InitialTrace
    (D : BoundedDomainData) (u₀ : D.Point → ℝ) (u : ℝ → D.Point → ℝ) : Prop :=
  ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
    D.supNorm (fun x => u t x - u₀ x) < ε
```

This gives no convergence of spatial derivatives. Since `H1energy` is the squared L² norm of `deriv (intervalDomainLift (u t))`, sup-norm convergence of values is too weak.

### `PaperPositiveInitialDatum` has no H¹ regularity content

For `intervalDomain`, the field `initialAdmissible` is:

```lean
initialAdmissible := fun u₀ =>
  BddAbove (Set.range fun x => |u₀ x|) ∧ Continuous u₀
```

and `PaperPositiveInitialDatum` adds only a positive floor. This is enough for positivity and value-level mild data, but not for convergence of `deriv (intervalDomainLift (u t))` to `deriv (intervalDomainLift u₀)` in L², nor even for classical differentiability of `u₀` as a theorem-facing assumption.

### Local classical regularity is positive-time only

`IsPaper2ClassicalSolution` gives spatial `C²` and joint continuity on `(0,T) × [0,1]`, but not continuity up to `t = 0` of `u_x`, `u_xx`, or `H1energy`. The H¹ transfer theorem now gives `H1UxxL1ContBefore u T`, hence H¹ energy continuity on closed windows **after** zero, but the initial endpoint still enters through `hcont0` or the split initial endpoint data.

### Mild / HSigma / cosine data are not currently an H¹ trace theorem

The closest relevant files are useful but not enough:

* `IntervalBFormInitialTrace.lean` proves `conjugatePicardLimit_initialTrace_of_conjugate_data`, but that theorem produces the same value-level `InitialTrace`; it is still sup-norm in values, not H¹ energy convergence.
* `IntervalMildPosTimeHSigma.lean` proves positive-time `MemHSigma 0`, and a closest-form positive-time `MemHSigma 1` theorem:
  ```lean
  theorem u_posTime_memHSigma_one_of_mild ...
  ```
  but it requires an external per-slice `UniformBootstrapStep`, and it only asserts positive-time membership, not convergence as `t ↓ 0` to the initial H¹ energy.
* `IntervalChiNegGradSummable.lean` proves fixed-positive-time eigenvalue-weighted summability from a mild decomposition and carried source packages, but again there is no theorem turning that into `Tendsto (H1energy u)` at zero.
* `IntervalChiNegH1EnergyDeriv.lean` explicitly carries the Parseval/summability bridge for H¹ spectral energy; it does not land an initial trace convergence theorem.

So the missing theorem is not a search miss: current landed data provide value trace and positive-time regularity, but not an endpoint H¹ trace.

## Clean next seam

The cleanest seam is to keep the H¹ endpoint data explicit and bundle it to reduce wrapper arity. Add this small record in `IntervalChiNegH1InitialContinuity.lean`:

```lean
import ShenWork.Paper2.IntervalChiNegH1InitialContinuity

open MeasureTheory Set Filter
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1InitialContinuity

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1InitialContinuity

/-- Bundled H¹ endpoint data: deleted-right convergence to the prescribed initial
H¹ energy plus compatibility of the stored zero slice. -/
structure H1InitialEndpointData
    (u₀ : intervalDomainPoint → ℝ)
    (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  tendsto : H1InitialTraceEnergyTendsto u₀ u T
  compatible : H1InitialEnergyCompatibleAtZero u₀ u

end ShenWork.Paper2.IntervalChiNegH1InitialContinuity
```

Then add thin wrappers in `IntervalChiNegH1InitialWiring.lean` and `IntervalChiNegH1BridgeInitialWiring.lean` that take

```lean
(hinit : H1InitialEndpointData u₀ u T)
```

and pass `hinit.tendsto` / `hinit.compatible` to the existing `*_initialTraceEnergy` theorems. This does not invent an analytic proof; it names the seam exactly where the current route consumes it.

A stronger future producer could prove this record from a genuine H¹ trace theorem, for example a mild/cosine theorem asserting convergence of the gradient energy (or of cosine H¹ energy plus Parseval) to the initial gradient energy. That theorem is not currently landed.

## Zero-slice compatibility

A simple compatibility theorem from equality of the stored zero slice is valid and useful. It does not prove the deleted-right trace, but it discharges the second half of the split endpoint data for constructors that set `u 0 = u₀`.

This should typecheck as a small addition to `IntervalChiNegH1InitialContinuity.lean`:

```lean
import ShenWork.Paper2.IntervalChiNegH1InitialContinuity

open MeasureTheory Set Filter
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1InitialContinuity

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1InitialContinuity

/-- If the stored zero slice is definitionally/equationally the prescribed datum,
then the H¹ initial energy compatibility field follows. -/
theorem H1InitialEnergyCompatibleAtZero_of_zeroSlice
    {u₀ : intervalDomainPoint → ℝ}
    {u : ℝ → intervalDomainPoint → ℝ}
    (h0 : u 0 = u₀) :
    H1InitialEnergyCompatibleAtZero u₀ u := by
  simp [H1InitialEnergyCompatibleAtZero, H1InitialEnergy,
    ShenWork.Paper2.IntervalChiNegH1Energy.H1energy, h0]

/-- Pointwise zero-slice equality version. -/
theorem H1InitialEnergyCompatibleAtZero_of_zeroSlice_pointwise
    {u₀ : intervalDomainPoint → ℝ}
    {u : ℝ → intervalDomainPoint → ℝ}
    (h0 : ∀ x : intervalDomainPoint, u 0 x = u₀ x) :
    H1InitialEnergyCompatibleAtZero u₀ u := by
  exact H1InitialEnergyCompatibleAtZero_of_zeroSlice
    (u₀ := u₀) (u := u) (funext h0)

end ShenWork.Paper2.IntervalChiNegH1InitialContinuity
```

This is worth adding because `InitialTrace` does not mention `u 0`; any construction that actually defines the zero slice to be `u₀` can close `hcompat` immediately.

## Final recommendation

Do not try to derive `H1InitialTraceEnergyTendsto` from `InitialTrace`, `PaperPositiveInitialDatum`, or positive-time classical regularity. The repo currently lacks the needed derivative-level initial trace theorem. Keep it as a named H¹ endpoint frontier, optionally bundled with zero-slice compatibility as `H1InitialEndpointData`, and add the zero-slice compatibility helper above.
