# Q3973 (cron1): what `hkernel_step` has to prove

## Bottom line

`hkernel_step` is **not** a Duhamel-only obligation. In the current wiring it must prove the full successor-iterate gradient bound on the right window:

```lean
hkernel_step : ∀ n : ℕ,
  (∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
    |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G) →
  IterGradOnWindow U lo hi (n + 1)
    (truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G)
```

`hbase` is only the `n = 0` base case for `truncatedGradientWindow_all`. It is not automatically combined with the successor step. In `truncatedGradientWindow_succ`, the proof does exactly this:

```lean
have HsrcInterval : IterGradOnWindow U a hi n G :=
  IterGradOnWindow.glue_left (W.hleft n) IH
have Hsrc := W.hsource_of_grad n HsrcInterval
have Hraw := W.hkernel_step n Hsrc
exact IterGradOnWindow.mono W.hclosed Hraw
```

So `hleft` is used to extend the **previous iterate** gradient hypothesis from `[lo, hi]` to `[a, hi]`, so that `hsource_of_grad` can bound `Src n` on `[a, hi]`. It does **not** separately bound the early `[0, a]` part of the successor Duhamel integral.

## The decomposition should be a restarted decomposition, not the raw `0..t` one

If you unfold the Picard iterate from time zero, you get morally

```text
U(n+1,t) = S(t)u₀ + ∫₀ᵗ S(t-s) Src_n(s) ds
```

after the conjugate-kernel IBP has converted the B-form flux term into a heat-semigroup source. But the `hkernel_step` assumption only bounds `Src n s` for `s ∈ [a, hi]`. Therefore, this raw `0..t` formula is not enough for the field as stated: it leaves the integral over `[0, a]` uncovered.

The intended window estimate is the **restart-at-`a`** estimate:

```text
U(n+1,t)
  = S(t-a)(U(n+1,a))
    + ∫ₐᵗ S(t-s) Src_n(s) ds,
      for a ≤ t.
```

This is exactly why `truncWindowAffine` has the term

```text
Cg * M / sqrt (lo - a)
```

rather than only a term based on `sqrt lo` or `sqrt t`. The first term is the heat-gradient bound for the restarted homogeneous piece `S(t-a)(U(n+1,a))`, using the Picard ball bound `|U(n+1,a)| ≤ M`. Since `t ∈ [lo, hi]`, one has `t-a ≥ lo-a`, so

```text
Cg * M / sqrt (t-a) ≤ Cg * M / sqrt (lo-a).
```

The second term is the shifted Duhamel gradient bound on `[a,t]`:

```text
|∫ₐᵗ ∂ₓ S(t-s) Src_n(s) ds|
  ≤ Cg * (2 * sqrt (hi-a)) * SourceCL.
```

Thus `hkernel_step` proves **both** pieces:

1. restarted base/homogeneous piece `S(t-a)(U(n+1,a))`, and
2. shifted Duhamel piece over `[a,t]`.

The early history `[0,a]` is not ignored; it is absorbed into the restart datum `U(n+1,a)`.

## Consequence for the current attempted proof

The proof should not be organized as

```text
base = S(t)u₀
Duhamel = ∫ₐᵗ ...
```

because that drops the interval `[0,a]`. And it should not be organized as

```text
base = S(t)u₀
Duhamel = ∫₀ᵗ ...
```

unless the `hkernel_step` assumption is strengthened to bound the source on all of `[0, hi]`. That would also change the natural Duhamel constant from `2√(hi-a)` to `2√hi`, defeating the purpose of the shifted window.

## Strengthen/localize the shifted Duhamel lemma

There is another small but important mismatch. The current shifted lemma is stated with a global source bound:

```lean
(hq_sup : ∀ s y, |q s y| ≤ Cq)
```

But `hkernel_step` receives only

```lean
∀ s, a ≤ s → s ≤ hi → ∀ y, |Src n s y| ≤ Cq
```

The proof of `gradDuhamel_shifted_sup_bound` only uses the source bound for `s ∈ (a,t)`, and `t ≤ hi`. So the clean fix is to add a localized wrapper/copy:

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow

  (heatGradientLinftyLinftyConstant heatGradientLinftyLinftyConstant_nonneg)

Only the source bound on the integration window is needed. -/
theorem gradDuhamel_shifted_sup_bound_on
    {a t T : ℝ} (ha : 0 ≤ a) (hat : a < t) (htT : t ≤ T)
    {q : ℝ → ℝ → ℝ}
    (hq_int : ∀ s, Integrable (q s) (intervalMeasure 1))
    {Cq : ℝ} (hCq : 0 ≤ Cq)
    (hq_sup : ∀ s, a ≤ s → s ≤ T → ∀ y : ℝ, |q s y| ≤ Cq)
    (x : ℝ)
    (hg_int : IntervalIntegrable
      (fun s : ℝ => deriv
        (fun z : ℝ => intervalFullSemigroupOperator (t - s) (q s) z) x)
      volume a t) :
    |∫ s in a..t, deriv
        (fun z : ℝ => intervalFullSemigroupOperator (t - s) (q s) z) x|
      ≤ heatGradientLinftyLinftyConstant * (2 * Real.sqrt (T - a)) * Cq := by
  -- Copy the proof of `gradDuhamel_shifted_sup_bound`.
  -- The only changed line is the per-slice source bound:
  --
  --   have hsrc_s : ∀ y, |q s y| ≤ Cq :=
  --     hq_sup s hs.1.le (le_trans hs.2.le htT)
  --
  -- and then pass `hsrc_s` to
  -- `intervalFullCoupledDuhamel_grad_integrand_pointwise_bound`.
```

Equivalently, keep the old theorem and apply it to a time-cutoff source that equals `q` on `[a,T]` and is zero outside. But the localized lemma is cleaner and matches `hkernel_step` exactly.

## The actual `hkernel_step` proof shape

Once the restart/IBP identity and the derivative-under-time-integral identity are packaged, the field proof should look like this.

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow
import ShenWork.Paper2.IntervalBFormCron2TruncatedPicard
import ShenWork.Paper2.IntervalConjugateKernelIBP
import ShenWork.PDE.IntervalFullKernelGradientLinfty
import ShenWork.PDE.IntervalGradDuhamelBound

open MeasureTheory Set
open scoped BigOperators Topology Real

noncomputable section

namespace ShenWork.Paper2.TruncatedPositiveTimeBootstrap

open ShenWork.IntervalDomain
  (intervalDomainLift intervalDomainPoint intervalMeasure)
open ShenWork.IntervalNeumannFullKernel
  (intervalFullSemigroupOperator
   intervalFullSemigroupOperator_deriv_Linfty_pointwise_sqrt_t)
open ShenWork.HeatKernelGradientEstimates
  (heatGradientLinftyLinftyConstant heatGradientLinftyLinftyConstant_nonneg)
open ShenWork.Paper2.TruncatedGradientWindow

-- This is the shape to use inside the record literal:
--
--   hkernel_step := by
--     intro n hsrc
--     intro t htlo hthi x
--     ...
--
-- The proof below is schematic in the places where the concrete Picard restart
-- and Leibniz lemmas must be supplied.

example
    {p : CM2Params}
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {Src : ℕ → ℝ → ℝ → ℝ}
    {M A_L A_F B_F a lo hi G : ℝ}
    (hM_nonneg : 0 ≤ M)
    (hAL_nonneg : 0 ≤ A_L)
    (hAF_nonneg : 0 ≤ A_F)
    (hBF_nonneg : 0 ≤ B_F)
    (hG_nonneg : 0 ≤ G)
    (ha_nonneg : 0 ≤ a)
    (ha_lt_lo : a < lo)
    (hlo_le_hi : lo ≤ hi)
    -- concrete Picard ball at restart time; for the real call site this comes
    -- from `truncatedConjugatePicardIter_ball` and `intervalDomainLift`.
    (hball_restart : ∀ n t, 0 ≤ t → t ≤ hi →
      ∀ y : ℝ, |intervalDomainLift (U n t) y| ≤ M)
    -- measurability of restart slices, also from continuity/ball infrastructure.
    (hmeas_restart : ∀ n t,
      AEStronglyMeasurable (intervalDomainLift (U n t)) (intervalMeasure 1))
    -- per-slice integrability of source.
    (hSrc_int : ∀ n s, Integrable (Src n s) (intervalMeasure 1))
    -- derivative-under-integral / restart representation for the gradient.
    -- This should be proved from the concrete Picard map plus IBP.
    (hrestart_grad : ∀ n t x, lo ≤ t → t ≤ hi →
      deriv (intervalDomainLift (U (n + 1) t)) x =
        deriv
          (fun z : ℝ => intervalFullSemigroupOperator (t - a)
            (intervalDomainLift (U (n + 1) a)) z) x
        + ∫ s in a..t,
            deriv
              (fun z : ℝ => intervalFullSemigroupOperator (t - s) (Src n s) z) x)
    -- integrability of the gradient integrand in time.
    (hgrad_int : ∀ n t x, lo ≤ t → t ≤ hi →
      IntervalIntegrable
        (fun s : ℝ => deriv
          (fun z : ℝ => intervalFullSemigroupOperator (t - s) (Src n s) z) x)
        volume a t) :
    ∀ n : ℕ,
      (∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
        |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G) →
      IterGradOnWindow U lo hi (n + 1)
        (truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G) := by
  intro n hsrc t htlo hthi x

  let Cg : ℝ := heatGradientLinftyLinftyConstant
  let Cq : ℝ := truncWindowSourceCL A_L A_F B_F p.χ₀ G

  have hCq_nonneg : 0 ≤ Cq := by
    dsimp [Cq, truncWindowSourceCL]
    exact add_nonneg hAL_nonneg
      (mul_nonneg (abs_nonneg p.χ₀)
        (add_nonneg hAF_nonneg (mul_nonneg hBF_nonneg hG_nonneg)))

  have hat : a < t := lt_of_lt_of_le ha_lt_lo htlo
  have hta_pos : 0 < t - a := sub_pos.mpr hat
  have hlo_a_pos : 0 < lo - a := sub_pos.mpr ha_lt_lo
  have hta_ge_hloa : lo - a ≤ t - a := by linarith

  -- Base/restart term.
  have hbase_raw :
      |deriv
        (fun z : ℝ => intervalFullSemigroupOperator (t - a)
          (intervalDomainLift (U (n + 1) a)) z) x|
        ≤ Cg * (t - a) ^ (-(1 / 2) : ℝ) * M := by
    simpa [Cg] using
      intervalFullSemigroupOperator_deriv_Linfty_pointwise_sqrt_t
        (t := t - a) hta_pos
        (f := intervalDomainLift (U (n + 1) a))
        (hf_meas := hmeas_restart (n + 1) a)
        (Cf := M)
        (hf := hball_restart (n + 1) a ha_nonneg (le_trans (le_of_lt ha_lt_lo) hlo_le_hi))
        x

  have hbase_bound :
      |deriv
        (fun z : ℝ => intervalFullSemigroupOperator (t - a)
          (intervalDomainLift (U (n + 1) a)) z) x|
        ≤ Cg / Real.sqrt (lo - a) * M := by
    -- Convert `(t-a)^(-1/2)` to `1 / sqrt (t-a)` and use
    -- `lo-a ≤ t-a`, positivity, `Real.sqrt_le_sqrt`, and multiplication by
    -- the nonnegative constants `Cg` and `M`.
    -- In code I would usually factor this into a small monotonicity lemma:
    --
    --   heatGradient_sqrt_mono
    --     (hCg : 0 ≤ Cg) (hM : 0 ≤ M)
    --     (hlo_a_pos : 0 < lo-a) (hta_ge_hloa : lo-a ≤ t-a) :
    --       Cg * (t-a)^(-1/2) * M ≤ Cg / sqrt(lo-a) * M
    --
    exact hbase_raw.trans (by
      -- arithmetic-only subgoal
      sorry)

  -- Duhamel term on `[a,t]`.
  have hduh_bound :
      |∫ s in a..t,
          deriv
            (fun z : ℝ => intervalFullSemigroupOperator (t - s) (Src n s) z) x|
        ≤ Cg * (2 * Real.sqrt (hi - a)) * Cq := by
    simpa [Cg, Cq] using
      gradDuhamel_shifted_sup_bound_on
        (a := a) (t := t) (T := hi)
        ha_nonneg hat hthi
        (q := Src n)
        (hSrc_int n)
        hCq_nonneg
        (fun s has hshi y => hsrc s has hshi y)
        x
        (hgrad_int n t x htlo hthi)

  -- Combine the two estimates using the gradient representation.
  calc
    |deriv (intervalDomainLift (U (n + 1) t)) x|
        = |deriv
            (fun z : ℝ => intervalFullSemigroupOperator (t - a)
              (intervalDomainLift (U (n + 1) a)) z) x
          + ∫ s in a..t,
              deriv
                (fun z : ℝ => intervalFullSemigroupOperator (t - s) (Src n s) z) x| := by
          rw [hrestart_grad n t x htlo hthi]
    _ ≤ |deriv
            (fun z : ℝ => intervalFullSemigroupOperator (t - a)
              (intervalDomainLift (U (n + 1) a)) z) x|
        + |∫ s in a..t,
              deriv
                (fun z : ℝ => intervalFullSemigroupOperator (t - s) (Src n s) z) x| :=
          abs_add _ _
    _ ≤ Cg / Real.sqrt (lo - a) * M
        + Cg * (2 * Real.sqrt (hi - a)) * Cq :=
          add_le_add hbase_bound hduh_bound
    _ = truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G := by
          dsimp [Cg, Cq]
          unfold truncWindowAffine truncWindowA truncWindowB truncWindowSourceCL
          ring

end ShenWork.Paper2.TruncatedPositiveTimeBootstrap
```

The two `sorry`s in the schematic proof above are not analytic mysteries:

* the first one is just real arithmetic/monotonicity of `r ↦ 1 / sqrt r` on positive reals;
* the second one marks where a small reusable arithmetic lemma should be inserted.

The genuinely necessary nontrivial infrastructure is not that arithmetic; it is the concrete `hrestart_grad` bridge:

```lean
deriv (intervalDomainLift (U (n+1) t)) x
  = deriv (fun z => S(t-a)(intervalDomainLift (U (n+1) a)) z) x
    + ∫ s in a..t, deriv (fun z => S(t-s)(Src n s) z) x
```

That bridge combines all of:

1. unfolding `truncatedConjugatePicardIter` and `truncatedConjugateDuhamelMap`,
2. applying `intervalConjugateKernelOperator_eq_semigroup_deriv` to the flux term,
3. using the endpoint flux vanishings `Q(0)=Q(1)=0`,
4. splitting the time integral at `a`,
5. using semigroup composition to restart at `a`, and
6. differentiating the shifted Duhamel integral under the time integral.

## Answer to the two direct questions

### 1. Does `hkernel_step` prove both pieces or only Duhamel?

It proves both pieces. The field conclusion is a bound for the full `U (n+1)` slice on `[lo, hi]`. The homogeneous term is not supplied by `hbase`; `hbase` is only the induction base case for `n = 0`. In the right proof, the homogeneous term is the restarted term `S(t-a)(U(n+1,a))`, bounded by `Cg*M/sqrt(lo-a)`.

### 2. What is the correct Lean 4 proof structure?

Use this order:

1. Define `Src` as the post-IBP source:

   ```lean
   Src n s y = truncatedLogisticLifted p (U n s) y
     - p.χ₀ * deriv (truncatedChemFluxLifted p (U n s)) y
   ```

2. Prove or import an iterate restart-after-IBP lemma for `U (n+1)`:

   ```text
   U(n+1,t) = S(t-a)(U(n+1,a)) + ∫ₐᵗ S(t-s)(Src n s) ds.
   ```

3. Prove or import the derivative version of that lemma, i.e. the `hrestart_grad` equation above.

4. Bound the restarted base term with

   ```lean
   intervalFullSemigroupOperator_deriv_Linfty_pointwise_sqrt_t
   ```

   using `Cf = M` and the Picard ball bound at time `a`.

5. Bound the shifted Duhamel term with a localized version of

   ```lean
   gradDuhamel_shifted_sup_bound
   ```

   using `Cq = truncWindowSourceCL A_L A_F B_F p.χ₀ G` and the supplied `hsrc`.

6. Combine by `abs_add`, `add_le_add`, and unfold `truncWindowAffine`, `truncWindowA`, `truncWindowB`, and `truncWindowSourceCL`; finish the algebra by `ring` or `ring_nf`.

The main conceptual correction is: **do not leave the `[0,a]` part dangling**. Either restart the whole successor iterate at `a`, or change the wiring. With the existing `truncWindowAffine` and source hypothesis, restart at `a` is the intended solution.
