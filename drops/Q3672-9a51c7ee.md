# Audit: C1η/Wiener feed versus the restart/frontier residuals

## Verdict

No. The current no-`hplan` CommonFold C1η wrapper does **not** discharge any field of `GradientMildHalfStepRestartData`, `GradientMildClassicalFrontierCoreData`, or `PicardLimitRestartFrontier` directly.

The proved C1η/Wiener theorem is a downstream slice-regularity consequence. Its strongest canonical true-lift form is:

```lean
theorem gradientMild_trueLift_coeffs_summable_of_phase1CutoffRep_initialHolder_intrinsic
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (Dsol : GradientMildSolutionData p u₀)
    (H : HasRestartCosineRepresentations Dsol.T Dsol.u)
    {t θ η H₀ : ℝ}
    (hη0 : 0 < η) (hθη : η < θ) (hθlt : θ < (1 / 2 : ℝ))
    (hH₀_nonneg : 0 ≤ H₀)
    (hholder : InitialDatumHolder u₀ θ H₀)
    (ht : 0 < t) (htT : t < Dsol.T) :
    Summable (fun n : ℕ => |cosineCoeffs (intervalDomainLift (Dsol.u t)) n|)
```

This theorem has removed the older explicit heat-coupling input and the initial-data measurability/bound inputs, but it still **takes**

```lean
H : HasRestartCosineRepresentations Dsol.T Dsol.u
```

as an input. Internally the route needs this `H` to obtain endpoint no-flux for `derivWithin` through

```lean
gradientMild_derivWithin_endpoint_zero_of_restartCosineRepresentations
```

and then feeds the resulting endpoint package into

```lean
holderCosineCoeff_summable_diffOn
```

via `chemMild_C1eta_slice_diffOn`. So the dependency direction is:

```text
GradientMildHalfStepRestartData
  → HasRestartCosineRepresentations
  → endpoint derivWithin no-flux / closed C²-Neumann package
  → C1η slice theorem
  → Summable (fun n => |cosineCoeffs (intervalDomainLift (D.u t)) n|)
```

not the reverse.

## Why it does not fill `GradientMildHalfStepRestartData`

The record is:

```lean
structure GradientMildHalfStepRestartData
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀) where
  a : ℝ → ℝ → ℕ → ℝ
  src : ∀ t, 0 < t → t < D.T → DuhamelSourceTimeC1 (a t)
  hagree : ∀ t, 0 < t → t < D.T →
    Set.EqOn (intervalDomainLift (D.u t))
      (fun x : ℝ =>
        ∑' n : ℕ,
          restartDuhamelCoeff (gradientMildHalfStepInitialCoeff D t)
            (a t) (t / 2) n * cosineMode n x)
      (Set.Icc (0 : ℝ) 1)
```

The C1η/Wiener feed provides neither:

1. a coefficient source family `a : ℝ → ℝ → ℕ → ℝ`,
2. a `DuhamelSourceTimeC1 (a t)` proof, nor
3. the restarted half-step spectral agreement `hagree`.

It only proves unweighted Wiener summability of the **already-realized slice**. The restart record needs a source-time coefficient package and an exact half-step Duhamel cosine-series realization. The generic restart engine then proves stronger spatial consequences from `restartDuhamelCoeff_eigenvalue_summable`, i.e. eigenvalue-weighted summability of the restarted coefficient sequence, not just the unweighted `Σ |cosineCoeffs (u(t)) n|` theorem supplied by the C1η route.

The closest existing bridge to `GradientMildHalfStepRestartData` is not the C1η wrapper; it is the logistic/Picard-limit source route in `IntervalPicardLimitLogisticSource.lean`:

```lean
def gradientMildHalfStepRestartData_of_iterateConvergence
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀)
    (I : PicardIterateConvergenceData D) :
    GradientMildHalfStepRestartData D
```

That file correctly identifies the real missing package: `PicardIterateConvergenceData`, especially the globally C² profile, source coefficient convergence/time-derivative data, decay, and `hSpectralAgree` field.

## Why it does not fill `GradientMildClassicalFrontierCoreData`

The reduced frontier core is:

```lean
structure GradientMildClassicalFrontierCoreData
    (p : CM2Params) {u0 : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u0) : Prop where
  hpde_u :
    ∀ t x, 0 < t → t < D.T → x ∈ intervalDomain.inside →
      intervalDomain.timeDeriv D.u t x =
        intervalDomain.laplacian (D.u t) x
          - p.χ₀ * intervalDomain.chemotaxisDiv p (D.u t)
              (mildChemicalConcentration p D.u t) x
          + D.u t x * (p.a - p.b * (D.u t x) ^ p.α)
  hregularityFrontier :
    GradientMildClassicalRegularityFrontierData p D
```

The C1η theorem gives neither the parabolic PDE field `hpde_u` nor the frontier record `hregularityFrontier`. The latter is also not a Wiener-L1 record. Its fields are the remaining time/v/joint regularity data after restart-cosine regularity has supplied the `u` spatial C² and Neumann pieces:

```lean
vSpatialInterior
timeSlices
jointTimeDerivInterior
vNeumannLimits
vClosedSpatial
jointTimeDerivClosed
jointSolutionClosed
```

A per-time theorem

```lean
Summable (fun n => |cosineCoeffs (intervalDomainLift (D.u t)) n|)
```

does not match any of these fields. It also does not give the resolver-side `v` regularity or the time-derivative/joint-continuity data.

## Why it does not fill `PicardLimitRestartFrontier`

The unified headline residual is:

```lean
def PicardLimitRestartFrontier (p : CM2Params) : Prop :=
  ∀ (u₀ : intervalDomainPoint → ℝ),
    PositiveInitialDatum intervalDomain u₀ →
  ∀ (D : GradientMildSolutionData p u₀),
    D.u = picardLimit p u₀ D.T →
    ∃ _R : GradientMildHalfStepRestartData D,
      GradientMildClassicalFrontierCoreData p D
```

So to close `PicardLimitRestartFrontier` one must produce both:

```lean
R : GradientMildHalfStepRestartData D
C : GradientMildClassicalFrontierCoreData p D
```

for every Picard-limit-valued mild solution. The C1η summability theorem is neither of these, and in its current canonical form it even assumes the restart representation package that `R` would produce.

## The missing bridge

To connect the current analysis to the headline residuals, the missing bridge is not another C1η wrapper. It is a restart/source-realization bridge of the following kind:

```lean
theorem picardLimit_halfStepRestartData_of_iterateConvergence
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀)
    (hD : D.u = picardLimit p u₀ D.T)
    (I : PicardIterateConvergenceData D) :
    GradientMildHalfStepRestartData D :=
  gradientMildHalfStepRestartData_of_iterateConvergence D I
```

The real work is therefore to prove or reduce `PicardIterateConvergenceData D` for canonical Picard-limit data. In the existing API this data contains exactly the hard restart fields:

```lean
profile
hC2
hpos
hN0
hN1
restartData
hdecay
ha0_bound
hMdot
hSpectralAgree
```

Among these, the smallest route-facing subtask is usually the algebraic realization field:

```lean
hSpectralAgree : ∀ t, 0 < t → t < D.T →
  Set.EqOn (intervalDomainLift (D.u t))
    (fun x => ∑' n,
      restartDuhamelCoeff
        (gradientMildHalfStepInitialCoeff D t)
        (fun σ n => cosineCoeffs
          (logisticSourceFun p.a p.b p.α (profile t σ)) n)
        (t / 2) n * cosineMode n x)
    (Set.Icc 0 1)
```

That is the exact equality needed by `GradientMildHalfStepRestartData.hagree`. It is not implied by Wiener L1 summability of `cosineCoeffs (intervalDomainLift (D.u t))`; it identifies the mild slice with a **particular restarted Duhamel coefficient formula**.
