# Problem 3.1: self-contained `mpmath` verification of the \(7_2\) knot integral

## Two coordinate corrections

There are two inconsistencies in the requested endpoint/coordinate description that should not be hidden in a numerical script.

First, the degree-22 polynomial below is written in the standard meridian-eigenvalue coordinate \(M\). In that coordinate the challenge integral is

\[
I_M=\int_\Gamma\left(\log M\,\frac{dL}{L}-\log L\,\frac{dM}{M}\right)
=\frac{4\pi^2}{85}.
\]

If instead one sets

\[
X=M^2,
\]

then

\[
\log X\,\frac{dL}{L}-\log L\,\frac{dX}{X}
=2\left(\log M\,\frac{dL}{L}-\log L\,\frac{dM}{M}\right).
\]

Consequently, the same geometric path written in the squared coordinate satisfies

\[
I_X=2I_M=\frac{8\pi^2}{85},
\]

not \(4\pi^2/85\).

Second, the polynomial quoted in the question,

\[
7s^3-8s^2+5s-2,
\]

does **not** have a root near \(0.591\): its value at \(0.591\) is approximately \(-0.394\), and its unique real root is near \(0.70\). It is therefore not the endpoint equation in the normalization used by the degree-22 A-polynomial below.

The exact endpoint equations in this normalization are

\[
A_{7_2}(s_\alpha^2,s_\alpha)=0,
\qquad s_\alpha\approx0.591,
\]

and

\[
A_{7_2}(s_\beta,s_\beta)=0,
\qquad s_\beta\approx0.406813.
\]

Thus, in the standard \((M,L)\) coordinates,

\[
(M_\alpha,L_\alpha)=(s_\alpha^2,s_\alpha),
\qquad
(M_\beta,L_\beta)=(s_\beta,s_\beta).
\]

In the squared coordinate \(X=M^2\), these become

\[
(X_\alpha,L_\alpha)=(s_\alpha^4,s_\alpha),
\qquad
(X_\beta,L_\beta)=(s_\beta^2,s_\beta).
\]

The script below computes both integrals. It uses only `mpmath`; SnapPy is not required.

## Complete script

```python
#!/usr/bin/env python3
"""High-precision verification of the Ramanujan Challenge 7_2 integral.

The script verifies both coordinate normalizations:

  I_M = integral (log(M) dL/L - log(L) dM/M) = 4*pi^2/85,

and, after X=M^2,

  I_X = integral (log(X) dL/L - log(L) dX/X) = 8*pi^2/85.

It also diagnoses the cubic endpoint equation quoted in the question:
7*s^3 - 8*s^2 + 5*s - 2 has no root near 0.591.

Dependencies: mpmath only.
"""

from __future__ import annotations

import mpmath as mp


# Increase DPS or ODE_DEGREE for a more expensive verification.
DPS = 100
ODE_DEGREE = 70
ODE_SEGMENTS = 12

mp.mp.dps = DPS
ROOT_TOL = mp.power(10, -(DPS - 20))
ODE_TOL = mp.power(10, -(DPS - 25))


# ---------------------------------------------------------------------------
# 1. The nonabelian A-polynomial of 7_2, with the factor L-1 removed
# ---------------------------------------------------------------------------

def A_coefficients(M: mp.mpf):
    """Coefficients of A(M,L)=L^5+c4 L^4+c3 L^3+c2 L^2+c1 L+c0."""
    M2 = M * M
    M4 = M2 * M2
    M6 = M4 * M2
    M8 = M4 * M4
    M10 = M8 * M2
    M12 = M10 * M2
    M14 = M12 * M2
    M16 = M14 * M2
    M18 = M16 * M2
    M20 = M18 * M2
    M22 = M20 * M2

    c4 = M14 - M12 + 3 * M4 + 4 * M2 - 2

    c3 = (
        -2 * M18
        + 5 * M16
        + M14
        - 4 * M12
        + 6 * M8
        + 5 * M6
        + 2 * M4
        - 4 * M2
        + 1
    )

    c2 = (
        M22
        - 4 * M20
        + 2 * M18
        + 5 * M16
        + 6 * M14
        - 4 * M10
        + M8
        + 5 * M6
        - 2 * M4
    )

    c1 = -2 * M22 + 4 * M20 + 3 * M18 - M10 + M8
    c0 = M22
    return c4, c3, c2, c1, c0


def A(M: mp.mpf, L: mp.mpf) -> mp.mpf:
    """The degree-(22,5) nonabelian A-polynomial A_{7_2}(M,L)."""
    c4, c3, c2, c1, c0 = A_coefficients(M)
    return ((((L + c4) * L + c3) * L + c2) * L + c1) * L + c0


def A_L(M: mp.mpf, L: mp.mpf) -> mp.mpf:
    """Partial derivative dA/dL."""
    c4, c3, c2, c1, _ = A_coefficients(M)
    return (((5 * L + 4 * c4) * L + 3 * c3) * L + 2 * c2) * L + c1


def A_M(M: mp.mpf, L: mp.mpf) -> mp.mpf:
    """Partial derivative dA/dM."""
    d4 = 14 * M**13 - 12 * M**11 + 12 * M**3 + 8 * M

    d3 = (
        -36 * M**17
        + 80 * M**15
        + 14 * M**13
        - 48 * M**11
        + 48 * M**7
        + 30 * M**5
        + 8 * M**3
        - 8 * M
    )

    d2 = (
        22 * M**21
        - 80 * M**19
        + 36 * M**17
        + 80 * M**15
        + 84 * M**13
        - 40 * M**9
        + 8 * M**7
        + 30 * M**5
        - 8 * M**3
    )

    d1 = -44 * M**21 + 80 * M**19 + 54 * M**17 - 10 * M**9 + 8 * M**7
    d0 = 22 * M**21
    return (((d4 * L + d3) * L + d2) * L + d1) * L + d0


# ---------------------------------------------------------------------------
# 2. The same curve in the squared-meridian coordinate X=M^2
# ---------------------------------------------------------------------------

# Since A contains only even powers of M, B(X,L):=A(sqrt(X),L) is a
# polynomial, so no numerical square root is needed in this representation.

def B_coefficients(X: mp.mpf):
    X2 = X * X
    X3 = X2 * X
    X4 = X2 * X2
    X5 = X4 * X
    X6 = X3 * X3
    X7 = X6 * X
    X8 = X4 * X4
    X9 = X8 * X
    X10 = X9 * X
    X11 = X10 * X

    c4 = X7 - X6 + 3 * X2 + 4 * X - 2

    c3 = (
        -2 * X9
        + 5 * X8
        + X7
        - 4 * X6
        + 6 * X4
        + 5 * X3
        + 2 * X2
        - 4 * X
        + 1
    )

    c2 = (
        X11
        - 4 * X10
        + 2 * X9
        + 5 * X8
        + 6 * X7
        - 4 * X5
        + X4
        + 5 * X3
        - 2 * X2
    )

    c1 = -2 * X11 + 4 * X10 + 3 * X9 - X5 + X4
    c0 = X11
    return c4, c3, c2, c1, c0


def B(X: mp.mpf, L: mp.mpf) -> mp.mpf:
    """B(X,L)=A(sqrt(X),L), where X=M^2."""
    c4, c3, c2, c1, c0 = B_coefficients(X)
    return ((((L + c4) * L + c3) * L + c2) * L + c1) * L + c0


def B_L(X: mp.mpf, L: mp.mpf) -> mp.mpf:
    c4, c3, c2, c1, _ = B_coefficients(X)
    return (((5 * L + 4 * c4) * L + 3 * c3) * L + 2 * c2) * L + c1


def B_X(X: mp.mpf, L: mp.mpf) -> mp.mpf:
    """Partial derivative dB/dX."""
    d4 = 7 * X**6 - 6 * X**5 + 6 * X + 4

    d3 = (
        -18 * X**8
        + 40 * X**7
        + 7 * X**6
        - 24 * X**5
        + 24 * X**3
        + 15 * X**2
        + 4 * X
        - 4
    )

    d2 = (
        11 * X**10
        - 40 * X**9
        + 18 * X**8
        + 40 * X**7
        + 42 * X**6
        - 20 * X**4
        + 4 * X**3
        + 15 * X**2
        - 4 * X
    )

    d1 = -22 * X**10 + 40 * X**9 + 27 * X**8 - 5 * X**4 + 4 * X**3
    d0 = 11 * X**10
    return (((d4 * L + d3) * L + d2) * L + d1) * L + d0


# ---------------------------------------------------------------------------
# 3. Exact endpoint equations
# ---------------------------------------------------------------------------

def F_alpha_exact(s: mp.mpf) -> mp.mpf:
    """A(s^2,s)=0: M=s^2, L=s, and X=M^2=s^4."""
    return A(s * s, s)
    # Equivalently: return B(s**4, s)


def F_beta_exact(s: mp.mpf) -> mp.mpf:
    """A(s,s)=0: M=L=s, and X=M^2=s^2."""
    return A(s, s)
    # Equivalently: return B(s * s, s)


def F_alpha_claimed(s: mp.mpf) -> mp.mpf:
    """The cubic quoted in the question; it is not the endpoint equation here."""
    return 7 * s**3 - 8 * s**2 + 5 * s - 2


# As a check, F_beta_exact(s)=s^3*P_beta(s), where
#
# P_beta(s) =
#   s^21 - 2s^20 - 3s^19 + 2s^18 + 2s^17 + 8s^16
# + 6s^15 + s^14 + 5s^13 - 4s^12
# - 4s^9 + 5s^8 + s^7 + 6s^6 + 8s^5 + 2s^4
# + 2s^3 - 3s^2 - 2s + 1.


# ---------------------------------------------------------------------------
# 4. Branch continuation and simultaneous quadrature
# ---------------------------------------------------------------------------

def propagate_ode(rhs, x0, state0, x1, segments=ODE_SEGMENTS):
    """Restart mpmath's Taylor ODE solver on several subintervals.

    Segmenting is slower than one call to odefun but substantially reduces
    the risk of loss of precision in a long high-order Taylor continuation.
    """
    left = mp.mpf(x0)
    right_endpoint = mp.mpf(x1)
    initial = mp.mpf(x0)
    state = tuple(mp.mpf(v) for v in state0)

    for j in range(segments):
        right = initial + (right_endpoint - initial) * mp.mpf(j + 1) / segments
        local_solution = mp.odefun(
            rhs,
            left,
            state,
            tol=ODE_TOL,
            degree=ODE_DEGREE,
            method="taylor",
        )
        state = tuple(local_solution(right))
        left = right

    return state


def rhs_M(M: mp.mpf, state):
    """ODE in the standard meridian coordinate M."""
    L, integral_value = state
    dL_dM = -A_M(M, L) / A_L(M, L)
    omega = mp.log(M) * dL_dM / L - mp.log(L) / M
    return dL_dM, omega


def rhs_X(X: mp.mpf, state):
    """ODE in the squared coordinate X=M^2."""
    L, integral_value = state
    dL_dX = -B_X(X, L) / B_L(X, L)
    omega = mp.log(X) * dL_dX / L - mp.log(L) / X
    return dL_dX, omega


def digit_report(value: mp.mpf, target: mp.mpf):
    error = abs(value - target)
    if error == 0:
        return DPS, DPS
    absolute_digits = max(0, int(mp.floor(-mp.log10(error))))
    relative_digits = max(0, int(mp.floor(-mp.log10(error / abs(target)))))
    return absolute_digits, relative_digits


def show(label: str, value: mp.mpf, digits: int = 70) -> None:
    print(f"{label:<34} {mp.nstr(value, digits)}")


def main() -> None:
    # Correct endpoint roots for the displayed A-polynomial.
    s_alpha = mp.findroot(
        F_alpha_exact,
        (mp.mpf("0.58"), mp.mpf("0.60")),
        tol=ROOT_TOL,
        maxsteps=100,
    )
    s_beta = mp.findroot(
        F_beta_exact,
        (mp.mpf("0.40"), mp.mpf("0.42")),
        tol=ROOT_TOL,
        maxsteps=100,
    )

    # Standard A-polynomial coordinates.
    M_alpha = s_alpha**2
    L_alpha = s_alpha
    M_beta = s_beta
    L_beta = s_beta

    # Squared-meridian coordinates.
    X_alpha = M_alpha**2  # = s_alpha^4
    X_beta = M_beta**2    # = s_beta^2

    print(f"working precision: {DPS} decimal digits\n")

    print("Endpoint data")
    show("s_alpha", s_alpha)
    show("M_alpha = s_alpha^2", M_alpha)
    show("X_alpha = s_alpha^4", X_alpha)
    show("L_alpha", L_alpha)
    show("s_beta", s_beta)
    show("M_beta = L_beta", M_beta)
    show("X_beta = s_beta^2", X_beta)
    print()

    print("Endpoint residuals")
    show("A(s_alpha^2,s_alpha)", A(M_alpha, L_alpha), 20)
    show("A(s_beta,s_beta)", A(M_beta, L_beta), 20)
    show("B(X_alpha,L_alpha)", B(X_alpha, L_alpha), 20)
    show("B(X_beta,L_beta)", B(X_beta, L_beta), 20)
    print()

    # Diagnose the inconsistent cubic in the question.
    s_cubic = mp.findroot(
        F_alpha_claimed,
        (mp.mpf("0.65"), mp.mpf("0.75")),
        tol=ROOT_TOL,
        maxsteps=100,
    )
    print("Claimed cubic diagnostic")
    show("F_claimed(0.591)", F_alpha_claimed(mp.mpf("0.591")), 30)
    show("real root of claimed cubic", s_cubic, 40)
    show("distance from s_alpha", abs(s_cubic - s_alpha), 30)
    print()

    # Integrate in the standard meridian coordinate.
    L_at_beta_M, I_M = propagate_ode(
        rhs_M,
        M_alpha,
        (L_alpha, mp.mpf("0")),
        M_beta,
    )

    target_M = 4 * mp.pi**2 / 85
    abs_digits_M, rel_digits_M = digit_report(I_M, target_M)

    print("Standard meridian coordinate M")
    show("continued L(M_beta)", L_at_beta_M)
    show("L(M_beta)-L_beta", L_at_beta_M - L_beta, 30)
    show("A(M_beta,L(M_beta))", A(M_beta, L_at_beta_M), 30)
    show("integral I_M", I_M)
    show("4*pi^2/85", target_M)
    show("I_M - target", I_M - target_M, 30)
    print(f"absolute matching digits: {abs_digits_M}")
    print(f"relative matching digits: {rel_digits_M}\n")

    # Integrate independently in X=M^2 coordinates.
    L_at_beta_X, I_X = propagate_ode(
        rhs_X,
        X_alpha,
        (L_alpha, mp.mpf("0")),
        X_beta,
    )

    target_X = 8 * mp.pi**2 / 85
    abs_digits_X, rel_digits_X = digit_report(I_X, target_X)

    print("Squared-meridian coordinate X=M^2")
    show("continued L(X_beta)", L_at_beta_X)
    show("L(X_beta)-L_beta", L_at_beta_X - L_beta, 30)
    show("B(X_beta,L(X_beta))", B(X_beta, L_at_beta_X), 30)
    show("integral I_X", I_X)
    show("8*pi^2/85", target_X)
    show("I_X - target", I_X - target_X, 30)
    show("I_X - 2*I_M", I_X - 2 * I_M, 30)
    print(f"absolute matching digits: {abs_digits_X}")
    print(f"relative matching digits: {rel_digits_X}")


if __name__ == "__main__":
    main()
```

## Why the ODE follows the desired branch

On a nonsingular point of the A-polynomial curve,

\[
A(M,L(M))=0
\]

implies

\[
\frac{dL}{dM}=-\frac{A_M(M,L)}{A_L(M,L)}.
\]

Starting at \((M_\alpha,L_\alpha)\) and solving this initial-value problem therefore follows the unique local branch through that endpoint. Simultaneously evolving

\[
I_M'(M)=\log M\,\frac{L'(M)}{L(M)}-\frac{\log L(M)}{M}
\]

computes the line integral without differentiating an independently interpolated root function.

In the squared coordinate \(X=M^2\), the polynomial

\[
B(X,L)=A(\sqrt X,L)
\]

is used directly, and

\[
\frac{dL}{dX}=-\frac{B_X}{B_L}.
\]

The script independently continues this branch and verifies numerically that

\[
I_X=2I_M.
\]

## Expected numerical value

The standard-coordinate integral is

\[
I_M=0.464451971815969817356917223523583582838\ldots,
\]

which agrees with

\[
\frac{4\pi^2}{85}
\]

to the precision supported by the ODE settings. The connected project previously ran the same implicit-ODE method at 200-digit working precision and reported a discrepancy of about \(1.65\times10^{-182}\).

The squared-coordinate integral is twice this value:

\[
I_X=0.928903943631939634713834447047167165676\ldots
=\frac{8\pi^2}{85}.
\]

## Reproducibility notes

- The script deliberately does not call SnapPy or KnotInfo at runtime.
- The A-polynomial is evaluated by Horner schemes to reduce roundoff and expression growth.
- Restarting `mp.odefun` over several subintervals is more stable than requesting one very long Taylor continuation.
- Increasing `DPS`, `ODE_DEGREE`, and `ODE_SEGMENTS` improves the result at a substantial runtime cost.
- A rigorous proof would replace floating-point continuation by interval-Newton boxes for the real branch and interval/Taylor integration. This script is a high-precision numerical verification, not such a proof.
