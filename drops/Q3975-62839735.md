# Q3975 (cron1): integrated weak ODE for the Neumann resolver

## Bottom line

The non-circular route is spectral, not FTC.  In particular, `resolverGradReal_sub_eq_integral_lapPhysicalReal` must not call either

```lean
resolverGradReal_hasDerivAt_physicalLap_of_continuousOn
resolverGradReal_hasDerivAt_physicalLap_of_continuousOn_via_FTC
```

because the second theorem obtains the derivative from this integrated identity, and the first theorem is the imported placeholder that this file is trying to replace.

However, the exact theorem as currently stated is not closeable from the listed spectral lemmas alone.  There are two concrete blockers in the current API.

1. **Source mismatch.**  The coefficient equation

```lean
intervalNeumannResolverCoeff_elliptic
```

is for `intervalNeumannResolverSourceCoeff`, whose source is

```lean
fun x : ℝ => ((p.ν * intervalDomainLift u x ^ p.γ : ℝ) : ℂ)
```

but `resolverLapPhysicalReal` unfolds through `resolverLapPhysical` to

```lean
p.μ * intervalNeumannResolverR p u y -
  p.ν * (positivePart (intervalDomainLift u y.1)) ^ p.γ
```

Under only

```lean
hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1)
```

there is no equality between these two sources.  The theorem needs either a nonnegativity hypothesis

```lean
(hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x)
```

or the resolver coefficients must be redefined to use the same `positivePart` source as `resolverLapPhysical`.

2. **Missing L² pairing lemma.**  Continuity gives only the existing Bessel input

```lean
resolverSourceCoeff_re_sq_summable_of_continuousOn
```

and then the resolver value/gradient summability lemmas

```lean
resolver_cosineSeries_summable_of_sourceL2
resolver_sineSeries_summable_of_sourceL2
```

These are not enough to justify

```lean
∫ x in a..b, ρ x = ∑' k, ρ̂ k * ∫ x in a..b, cos (kπx)
```

by `integral_tsum`, because the source coefficients are only `ℓ²`, not `ℓ¹`.  Pointwise cosine inversion from `IntervalCosineInversion.lean` is also too strong: `intervalCosine_hasSum_pointwise` requires `ℓ¹` Fourier coefficients of the even reflection.  For the weak theorem, the correct missing bridge is an `L²` Parseval/pairing theorem against the interval indicator `1_[a,b]`.

So the minimal non-circular proof is:

* align the source (`positivePart` everywhere, or add `hUnonneg`),
* add one cosine `L²` pairing lemma,
* then prove both sides equal the same spectral scalar series.

## The missing Parseval/pairing lemma

Put this in `CosineParsevalBridge.lean` or locally in `IntervalResolverWeakODEBridge.lean`.  The statement below uses the repo normalization: positive Neumann coefficients already include the factor `2`, so the reconstruction pairing has no extra `1/2` in front of positive modes.

```lean

/-- L² cosine pairing against the interval indicator.

This is the missing weak substitute for pointwise source inversion.  It follows by
applying the cosine totality/Parseval bridge to the continuous source and the
`L²` indicator `1_[a,b]`, or equivalently by approximating the indicator in `L²`
with finite cosine polynomials.

The important normalization point is that `unitIntervalNeumannCosineCoeff` is the
coefficient used in the reconstruction
`f = ∑' k, coeff k * unitIntervalCosineMode k`; hence the pairing has exactly the
raw mode integral on the right. -/
theorem unitIntervalNeumannCosineCoeff_integral_indicator_pair
    {f : ℝ → ℝ}
    (hf_cont : ContinuousOn f (Set.Icc (0 : ℝ) 1))
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    (∫ x in a..b, f x)
      = ∑' k : ℕ,
          unitIntervalNeumannCosineCoeff (fun x : ℝ => ((f x : ℝ) : ℂ)) k *
            (∫ x in a..b, unitIntervalCosineMode k x) := by
  -- Proof sketch:
  -- 1. Let `χab` be the oriented interval indicator on `[0,1]`, so
  --      ∫ a..b f = ∫ 0..1 f x * χab x.
  -- 2. Show `χab ∈ L²` and compute/identify its raw cosine coefficients as
  --      ∫ a..b unitIntervalCosineMode k x`.
  -- 3. Use the cosine Parseval polarization identity, or prove it from
  --    `unitIntervalCosine_nat_total_ae_zero`, to get the L² pairing formula.
  -- 4. The continuity of `f` supplies the `L²` side through
  --    `unitIntervalNeumannCosineCoeff_l2_bound`; no `ℓ¹` source summability is
  --    needed.
  --
  -- This is the single infrastructure lemma missing from the current repo.
  admit
```

Do **not** replace this lemma by `intervalCosine_hasSum_pointwise`; that would silently strengthen the source regularity from continuity to `ℓ¹` Fourier coefficients.

## Resolver-specific source pairing

Once the source is aligned, add a small wrapper in `IntervalResolverWeakODEBridge.lean`.  If the resolver coefficients are left as `intervalDomainLift u ^ p.γ`, this wrapper needs `hUnonneg`.  If the coefficients are changed to `positivePart`, remove `hUnonneg`.

    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x)
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    (∫ x in a..b, resolverPhysicalSourceReal p u x)
      = ∑' k : ℕ,
          (intervalNeumannResolverSourceCoeff p u k).re *
            (∫ x in a..b, unitIntervalCosineMode k x) := by
  have hsrc_cont : ContinuousOn (resolverPhysicalSourceReal p u) (Set.Icc (0 : ℝ) 1) := by
    -- This is already available locally in `IntervalResolverWeakODEBridge.lean`:
    -- `resolverPhysicalSourceReal_continuousOn p hUcont`.

  -- Under `hUnonneg`, positivePart drops out and its cosine coefficient is exactly

  simp only [resolverPhysicalSourceReal, resolverPhysicalSourceReal_continuousOn,
    intervalNeumannResolverSourceCoeff]
  -- The remaining proof is coefficient congruence over the raw integral:
  -- `positivePart (intervalDomainLift u x) = intervalDomainLift u x` on `[0,1]`.
  -- Use `positivePart_eq_self_of_nonneg (hUnonneg x hx)` inside
  -- `intervalIntegral.integral_congr` for the raw coefficient integral.

## Final spectral assembly

The following is the proof structure I would put in `IntervalResolverWeakODEBridge.lean` after the two helper lemmas above are available.  I write it with `hUnonneg`; if you instead change `intervalNeumannResolverSourceCoeff` to use `positivePart`, delete that hypothesis and the corresponding wrapper argument.

```lean
import ShenWork.Paper2.IntervalResolverWeakODEBridge
import Mathlib.MeasureTheory.Integral.IntervalIntegral.IntegrationByParts

open MeasureTheory intervalIntegral
open ShenWork.IntervalDomain
open ShenWork.PDE
open ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open ShenWork.HeatKernelGradientEstimates
open ShenWork.Paper3 (unitIntervalNeumannSpectrum)
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

private def modeInt (a b : ℝ) (k : ℕ) : ℝ :=
  ∫ x in a..b, unitIntervalCosineMode k x

private def srcCoeffR (p : CM2Params) (u : intervalDomainPoint → ℝ) (k : ℕ) : ℝ :=
  (intervalNeumannResolverSourceCoeff p u k).re

private def resCoeffR (p : CM2Params) (u : intervalDomainPoint → ℝ) (k : ℕ) : ℝ :=
  (intervalNeumannResolverCoeff p u k).re

private lemma resolver_lap_coeff_real
    (p : CM2Params) (u : intervalDomainPoint → ℝ) (k : ℕ) :
    p.μ * resCoeffR p u k - srcCoeffR p u k
      = -(((k : ℝ) * Real.pi) ^ 2) * resCoeffR p u k := by
  have hellRe :
      (p.μ + unitIntervalNeumannSpectrum.eigenvalue k) *
          (intervalNeumannResolverCoeff p u k).re
        = (intervalNeumannResolverSourceCoeff p u k).re := by
    have hcast :
        ((p.μ : ℂ) + (unitIntervalNeumannSpectrum.eigenvalue k : ℂ)) =
          (((p.μ + unitIntervalNeumannSpectrum.eigenvalue k : ℝ)) : ℂ) := by
      push_cast
      ring
    have h := congrArg Complex.re (intervalNeumannResolverCoeff_elliptic p u k)
    rw [hcast, Complex.re_ofReal_mul] at h
    exact h
  have hlam : unitIntervalNeumannSpectrum.eigenvalue k = ((k : ℝ) * Real.pi) ^ 2 := by
    show (k : ℝ) ^ 2 * Real.pi ^ 2 = _
    ring
  unfold resCoeffR srcCoeffR
  rw [hlam] at hellRe
  linarith

private lemma mode_grad_jump_eq_neg_lap_modeInt (a b : ℝ) (k : ℕ) :
    ((-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * b)) -
      (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * a)))
      = -(((k : ℝ) * Real.pi) ^ 2) * modeInt a b k := by
  by_cases hk : k = 0
  · subst k
    simp [modeInt, unitIntervalCosineMode]
  · set K : ℝ := (k : ℝ) * Real.pi with hKdef
    have hKpos : 0 < K := by
      rw [hKdef]
      exact mul_pos (Nat.cast_pos.mpr (Nat.pos_of_ne_zero hk)) Real.pi_pos
    have hKne : K ≠ 0 := ne_of_gt hKpos
    have hprimitive :
        (∫ x in a..b, Real.cos (K * x))
          = Real.sin (K * b) / K - Real.sin (K * a) / K := by
      have hderiv : ∀ x : ℝ,
          HasDerivAt (fun y : ℝ => Real.sin (K * y) / K) (Real.cos (K * x)) x := by
        intro x
        have h1 : HasDerivAt (fun y : ℝ => Real.sin (K * y))
            (K * Real.cos (K * x)) x := by
          have hlin : HasDerivAt (fun y : ℝ => K * y) K x := by
            simpa using (hasDerivAt_id x).const_mul K
          simpa [mul_comm, mul_left_comm, mul_assoc] using
            (Real.hasDerivAt_sin (K * x)).comp x hlin
        have h2 := h1.div_const K
        simpa [hKne, div_eq_mul_inv, mul_comm, mul_left_comm, mul_assoc] using h2
      exact intervalIntegral.integral_eq_sub_of_hasDerivAt hderiv
    unfold modeInt unitIntervalCosineMode
    rw [show (fun x : ℝ => Real.cos ((k : ℝ) * Real.pi * x)) =
        fun x : ℝ => Real.cos (K * x) by
          funext x
          rw [hKdef]]
    rw [hprimitive]
    field_simp [hKne]
    ring

/-- Corrected non-circular spectral theorem.  The currently committed theorem has
no `hUnonneg`; with the present coefficient definition that hypothesis is needed
unless `intervalNeumannResolverSourceCoeff` is changed to use `positivePart`. -/
theorem resolverGradReal_sub_eq_integral_lapPhysicalReal_spectral
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x)
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    resolverGradReal p u b - resolverGradReal p u a
      = ∫ t in a..b, resolverLapPhysicalReal p u t := by
  classical

  let A : ℕ → ℝ := srcCoeffR p u
  let C : ℕ → ℝ := resCoeffR p u
  let J : ℕ → ℝ := modeInt a b

  -- Source coefficients are ℓ² from continuity.
  have hA2 : Summable fun k : ℕ => (A k) ^ 2 := by
    have h := resolverSourceCoeff_re_sq_summable_of_continuousOn p hUcont
    simpa [A, srcCoeffR, ShenWork.Paper2.intervalNeumannResolverSourceCoeff_zero]
      using h

  -- Gradient sine series at both endpoints are summable.
  have hGb : Summable fun k : ℕ =>
      C k * (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * b)) := by
    simpa [C, resCoeffR] using resolver_sineSeries_summable_of_sourceL2 p hA2 b
  have hGa : Summable fun k : ℕ =>
      C k * (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * a)) := by
    simpa [C, resCoeffR] using resolver_sineSeries_summable_of_sourceL2 p hA2 a

  -- Left side: difference of the sine-gradient series.
  have hgrad_series :
      resolverGradReal p u b - resolverGradReal p u a
        = ∑' k : ℕ, (-(((k : ℝ) * Real.pi) ^ 2) * C k) * J k := by
    unfold resolverGradReal
    rw [hGb.tsum_sub hGa]
    apply tsum_congr
    intro k
    have hmode := mode_grad_jump_eq_neg_lap_modeInt a b k
    dsimp [C, J, resCoeffR]
    calc
      (intervalNeumannResolverCoeff p u k).re *
          (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * b)) -
        (intervalNeumannResolverCoeff p u k).re *
          (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * a))
          = (intervalNeumannResolverCoeff p u k).re *
              (((-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * b)) -
                (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * a)))) := by
              ring
      _ = (intervalNeumannResolverCoeff p u k).re *
            (-(((k : ℝ) * Real.pi) ^ 2) * modeInt a b k) := by
              rw [hmode]
      _ = (-(((k : ℝ) * Real.pi) ^ 2) * (intervalNeumannResolverCoeff p u k).re) *
            modeInt a b k := by
              ring

  -- Right side: integrate `μR - ρ` spectrally.
  -- This uses two non-FTC helpers:
  --   1. integral of the resolver value series (from `resolverCoeff_re_abs_summable_of_continuousOn`),
  --   2. integral of the source by the L² pairing lemma above.
  have hR_int :
      (∫ x in a..b, resolverValueSeriesReal p u x)
        = ∑' k : ℕ, C k * J k := by
    -- Prove this with `intervalIntegral.integral_tsum` and the uniform majorant
    -- `|(intervalNeumannResolverCoeff p u k).re|`, supplied by
    -- `resolverCoeff_re_abs_summable_of_continuousOn`.
    admit

  have hsrc_int :
      (∫ x in a..b, resolverPhysicalSourceReal p u x)
        = ∑' k : ℕ, A k * J k := by
    simpa [A, J, srcCoeffR] using
      resolver_source_integral_eq_tsum_coeff_modeInt p hUcont hUnonneg ha hb

  have hlap_series :
      (∫ x in a..b, resolverLapPhysicalReal p u x)
        = ∑' k : ℕ, (p.μ * C k - A k) * J k := by
    -- First replace `resolverLapPhysicalReal` by `resolverLapPhysicalPlain` on
    -- `Set.uIcc a b`, using `ha` and `hb`; then unfold
    -- `resolverLapPhysicalPlain = μ * resolverValueSeriesReal - resolverPhysicalSourceReal`.
    -- After `integral_sub`, `integral_const_mul`, `hR_int`, and `hsrc_int`, use
    -- `tsum_sub` and `tsum_mul_left`.
    admit

  rw [hgrad_series, hlap_series]
  apply tsum_congr
  intro k
  dsimp [A, C, J]
  rw [resolver_lap_coeff_real p u k]

end ShenWork.IntervalResolverWeakBounds
```

## Why this avoids the circularity

The final assembly uses only:

* `resolverSourceCoeff_re_sq_summable_of_continuousOn`,
* `resolver_sineSeries_summable_of_sourceL2`,
* `resolverCoeff_re_abs_summable_of_continuousOn`,
* `intervalNeumannResolverCoeff_elliptic`,
* the elementary integral of one cosine mode,
* the new `L²` cosine pairing lemma.

It does **not** use either derivative theorem for `resolverGradReal`, so `resolverGradReal_hasDerivAt_physicalLap_of_continuousOn_via_FTC` can safely depend on the integrated identity afterward.
