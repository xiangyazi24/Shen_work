# Q3711 audit: PPID producer frontier after Task255

Repo point audited: origin/main commit
`1ec0e562aae10a5e3de62736a8f8ac7114ef84a9` (`Task 255: lower PPID resolver source inputs`).

New Task255 files checked:

* `ShenWork/Paper2/IntervalResolverSourceWindowInputs.lean`
* `ShenWork/Paper2/IntervalDomainPPIDResolverSourceInputsFrontier.lean`

The short conclusion is:

* Task255 is a good lowering step: it replaces the awkward per-`t₀` existential
  `ResolverSourceWindowData` with the primitive structure
  `ResolverSourceWindowInput.ResolverSourceWindowInputs`.
* For arbitrary PPID / `χ₀ ≤ 0` Picard-limit data
  `D : GradientMildSolutionData p u₀` with `D.u = picardLimit p u₀ D.T`, only the
  direct positivity and upper-bound fields are already non-circularly available
  from `D` itself.
* The hard fields remain the cosine representation/eigenvalue control and,
  especially, the power-source time-`C¹` triple
  `hderivPow`/`hadotPowCont`/`hMdotPow` and the compact positive lower bound
  `hlower`.
* `hlower` is not analytically hard once joint continuity of
  `(σ,x) ↦ intervalDomainLift (D.u σ) x` on compact slabs is available.  There is
  no non-χ0 producer for that joint continuity in the current PPID primitive route,
  but adding a small `JointInputs` reduction is an honest, likely-compiling next
  step.

## 1. Field-by-field audit of `ResolverSourceWindowInputs`

Current structure in `IntervalResolverSourceWindowInputs.lean`:

```lean

structure ResolverSourceWindowInputs
    (p : CM2Params) {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀) where
  bc : ℝ → ℕ → ℝ

    ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 < intervalDomainLift (D.u σ) x

    ∀ x ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift (D.u σ) x ≤ Msup
  hlower : ∀ a b, 0 < a → b < D.T → a ≤ b →
    ∃ m : ℝ, 0 < m ∧
      ∀ σ ∈ Set.Icc a b, ∀ x ∈ Set.Icc (0 : ℝ) 1,
        m ≤ intervalDomainLift (D.u σ) x

    ∃ G2, ∀ σ ∈ Set.Icc a b, ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |deriv (deriv (intervalDomainLift (D.u σ))) x| ≤ G2
  adotPow : ℝ → ℕ → ℝ
  hderivPow : ∀ σ, 0 < σ → σ < D.T → ∀ n,
```

### `hpos`

**Already available, non-χ0, non-circular.**

Source: `ShenWork/Paper2/IntervalMildPicard.lean`, structure
`GradientMildSolutionData` has field

```lean
hpos : ∀ t, 0 < t → t ≤ T → ∀ x, 0 < u t x
```

For the lifted closed interval form required by `ResolverSourceWindowInputs`:

```lean
fun σ hσ hσT x hx => by
  simp only [intervalDomainLift, dif_pos hx]
  exact D.hpos σ hσ (le_of_lt hσT) ⟨x, hx⟩
```

No `χ₀ = 0`, no `hpde_u`, no `Hu`, no classical solution input.

### `Msup` and `hub`

**Already available, non-χ0, non-circular.**

Source: `GradientMildSolutionData.hbound` in
`ShenWork/Paper2/IntervalMildPicard.lean`:

```lean
hbound : ∀ t, 0 < t → t ≤ T → ∀ x, |u t x| ≤ M
```

Use `Msup := D.M` and drop absolute value by `le_abs_self`:

```lean
fun σ hσ hσT x hx => by
```

Again, no `χ₀ = 0` and no downstream object.

## 3. Smallest honest next reduction

Yes.  Add a small non-circular wrapper that removes three non-frontier fields from
`ResolverSourceWindowInputs`:

* `hpos` comes from `D.hpos`;
* `Msup`/`hub` comes from `D.M`/`D.hbound`;
* `hlower` comes from **joint continuity of the lifted solution** plus `D.hpos`.

This is not merely documentation.  It turns the compact lower-bound field from an
opaque residual into the exact analytic input it needs: joint continuity on
`(0,D.T) × [0,1]`.  It uses no downstream object:

* no `GradientMildClassicalFrontierCoreData`;
* no `IsPaper2ClassicalSolution`;
* no `hpde_u`;
* no `HasTimeNeighborhoodSpectralAgreement D.T D.u`.

It also avoids route names touching `BFormDirectClassicalNoHvpos` or related dm
work.

Main compactness helper:

```lean
/-- Joint continuity plus pointwise positivity gives the compact-window positive
lower bound required by `ResolverSourceWindowInputs`. -/
theorem lift_window_uniformPositive_of_jointContinuous
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {D : GradientMildSolutionData p u₀}
    (hliftCont :
      ContinuousOn
        (Function.uncurry
          (fun (σ : ℝ) (x : ℝ) => intervalDomainLift (D.u σ) x))
        (Set.Ioo (0 : ℝ) D.T ×ˢ Set.Icc (0 : ℝ) 1)) :
    ∀ a b, 0 < a → b < D.T → a ≤ b →
      ∃ m : ℝ, 0 < m ∧
        ∀ σ ∈ Set.Icc a b, ∀ x ∈ Set.Icc (0 : ℝ) 1,
          m ≤ intervalDomainLift (D.u σ) x
```

Proof route for the helper:

1. Fix `a b`, `ha : 0 < a`, `hb : b < D.T`, `hab : a ≤ b`.
2. Let `K := Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1`.
3. `K` is compact by `isCompact_Icc.prod isCompact_Icc` and nonempty by
   `(a,0)`.
4. `K ⊆ Set.Ioo 0 D.T ×ˢ Set.Icc 0 1`, using `ha`, `hb`, and membership in
   `Icc a b`.
5. Restrict `hliftCont` to `K`.
6. Use `IsCompact.exists_isMinOn` to get a minimizer `(σ₀,x₀)`.
7. Positivity of the minimum follows from `D.hpos σ₀ ... ⟨x₀,hx₀⟩` and
   `intervalDomainLift` agreement on `[0,1]`.
8. The lower-bound property follows from `isMinOn_iff.mp`.

## 5. What remains after that wrapper

After `ResolverSourceWindowJointInputs`, the remaining honest residuals are:

1. **A faithful non-χ0 representation/envelope for `D.u`.**
   For B-form/conjugate data, the natural coefficients are

   ```lean
   bc σ n := localRestartCoeff (cosineCoeffs (intervalDomainLift u₀))
      (bFormSourceCoeffs p u) σ n
   ```

   but the current PPID surface is stated for arbitrary standard
   `D.u = picardLimit p u₀ D.T`, not directly for `conjugatePicardLimit`.

2. **K2 from the representation envelope.**
   A later wrapper should replace `hG1/hG2` by a per-compact summable envelope

   ```lean
   ∀ a b, 0 < a → b < D.T → a ≤ b →
     ∃ E : ℕ → ℝ,
       Summable E ∧ (∀ n, 0 ≤ E n) ∧
       ∀ σ ∈ Set.Icc a b, ∀ n,
         unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n
   ```

   then use `CompactSliceGradientBounds.grad_series_abs_le` and
   `grad2_series_abs_le` to derive `hG1/hG2`.  This is a good second reduction,
   but it is larger than the joint-continuity wrapper.

3. **Power-source K1 for non-χ0.**
   The real analytic theorem still needed is a non-χ0 analogue of

   ```lean
   ResolverPowerK1.powerK1_quadruple_of_subtypeCont
   ```

   with no `hχ0 : p.χ₀ = 0`, probably driven by a B-form/conjugate local restart
   package or by a direct joint time-regularity theorem for the lifted solution.
   It should target exactly:

   ```lean
   (∀ σ, 0 < σ → σ < D.T → ∀ k,
      HasDerivAt
        (fun r => cosineCoeffs
          (fun x => p.ν * intervalDomainLift (D.u r) x ^ p.γ) k)
        (adotPow σ k) σ)
   ∧ (∀ k, ContinuousOn (fun σ => adotPow σ k) (Set.Ioo 0 D.T))
   ∧ (∀ a b, 0 < a → b < D.T →
        ∃ Mdot, ∀ σ ∈ Set.Icc a b, ∀ k,
          |adotPow σ k| ≤ Mdot)
   ```

This last item is the true non-χ0 resolver-source analytic frontier.  The joint
continuity wrapper is worth adding first because it removes a compactness artifact
(`hlower`) from that frontier and makes the remaining missing input sharper.
