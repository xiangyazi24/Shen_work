## Mismatch audit

### `Continuous u₀` vs `Continuous (intervalDomainLift u₀)`

No mismatch for this theorem.  `LimitRegularityInputs.hu₀_cont` has type:

```lean
Continuous u₀
```

and `PdeUWiring.hasSpectralPdeAgreement_of_localized_data` expects exactly:

```lean
hu₀_cont : Continuous u₀
```

Do not insert a lift-continuity bridge here.

### `DuhamelSourceBddOn` vs `DuhamelSourceL1ContOn`

No mismatch for this theorem.  `LimitRegularityInputs.hsrc0` has type:

```lean
ShenWork.IntervalPicardLimitRestartBdd.DuhamelSourceBddOn
  (ShenWork.IntervalPicardLimitBddProducer.patchedSource p u₀ D.u) D.T
```

and `PdeUWiring.hasSpectralPdeAgreement_of_localized_data` expects exactly the same `DuhamelSourceBddOn patchedSource` type.  Do not try to convert to `DuhamelSourceL1ContOn` in Task222; that belongs to older weak-restart APIs, not this PDE-U producer.

### `intervalLogisticSource` vs `logisticLifted` / `constExtend`

This is the only bridge needed.  The ledger has subtype continuity:

```lean
I.hLc : ∀ t, 0 < t → t < D.T →
  ∀ s, 0 < s → s ≤ t → Continuous (intervalLogisticSource p (D.u s))
```

The PDE-U wiring theorem expects const-extended continuity:

```lean
Continuous (intervalDomainConstExtend (intervalLogisticSource p (D.u s)))
```

Use exactly the existing bridge:

```lean
constExtend_continuous (I.hLc t ht htT s hs hst)
```

Inside `PdeUWiring`, this continuous surrogate is then related to `logisticLifted` and `logisticSourceFun` using the local `[0,1]` agreement lemmas.  No extra source-congruence theorem is needed in `IntervalDomainMildLocalChi0.lean`.
