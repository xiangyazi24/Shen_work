# Time differentiation of the Neumann cosine synthesis

## Verdict

Yes. The correct formalizable argument is:

1. work on an open time interval `J = Set.Ioo a b` around the evaluation time;
2. prove a **single summable per-mode envelope** for the differentiated summands, uniform for every `t ∈ J`;
3. invoke Mathlib's

```lean
hasDerivAt_tsum_of_isPreconnected
```

rather than trying to use pointwise summability of the derivative series;
4. transfer the resulting `HasDerivAt` from the cosine synthesis to the physical mild solution through the local realization identity.

For fixed `x`, set

\[
f_k(t)=c_k(t)\cos(k\pi x),
\qquad
f_k'(t)=c_k'(t)\cos(k\pi x).
\]

If, on `J`, there are summable nonnegative envelopes `A_k,E_k` such that

\[
\lambda_k |c_k(t)|\le A_k,
\qquad
|s_k(t)|\le E_k,
\]

and

\[
c_k'(t)=-\nu\lambda_kc_k(t)+s_k(t),
\]

then

\[
|f_k'(t)|
\le |c_k'(t)|
\le |\nu|A_k+E_k=:M_k.
\]

Since `Summable M`, the Weierstrass M-test gives uniform convergence of the derivative series on `J`. This is exactly the hypothesis packaged by `hasDerivAt_tsum_of_isPreconnected`.

Two cautions are load-bearing:

- Pointwise `Summable (fun k => |c'_k t|)` is not enough.
- Even a bound of the form
  \[
  \sup_{t\in[a,b]}\sum_k\lambda_k|c_k(t)|<\infty
  \]
  is not enough for the M-test. What is needed is a summable sequence `A k` satisfying the modewise bound uniformly in time—equivalently, a controlled form of
  \[
  \sum_k\sup_{t\in[a,b]}\lambda_k|c_k(t)|<\infty.
  \]
  The Duhamel representation provides such an explicit sequence, so no actual `sSup` needs to be formalized.

The current `Shen_work` repository already contains this route, including the public capstone theorem

```lean
ShenWork.EWA.fullSourceCoeffSeries_hasDerivAt
```

and its transfer to the realized solution

```lean
ShenWork.EWA.realSlice_hasDerivAt_time
ShenWork.EWA.realSlice_timeDeriv_of_atoms
ShenWork.EWA.realSlice_diffU_of_atoms
```

in `ShenWork/Wiener/EWA/SourceTimeDerivDischarge.lean`.

# 1. The exact abstract window lemma

Write

```lean
open Set Filter Topology
open ShenWork.CosineSpectrum (cosineMode)
```

and use the repository eigenvalue

```lean
unitIntervalCosineEigenvalue k = ((k : ℝ) * Real.pi) ^ 2.
```

A reusable theorem has the following shape. The theorem names used in its proof are exact for the Mathlib revision pinned by the repository (`Lean 4.29.1`, Mathlib commit `5e932f97dd25535344f80f9dd8da3aab83df0fe6`).

```lean
theorem cosineSeries_hasDerivAt_time_of_window
    {c cdot src : ℝ → ℕ → ℝ}
    {ν a b t₀ x : ℝ}
    (ht₀ : t₀ ∈ Set.Ioo a b)
    -- Genuine per-mode derivative data; do not use an equality involving
    -- the junk-valued `deriv` as a substitute for this.
    (hmode : ∀ k t, t ∈ Set.Ioo a b →
      HasDerivAt (fun r => c r k) (cdot t k) t)
    -- The spectral ODE.
    (hODE : ∀ t, t ∈ Set.Ioo a b → ∀ k,
      cdot t k =
        -ν * unitIntervalCosineEigenvalue k * c t k + src t k)
    -- Explicit time-uniform, modewise envelopes.
    {A E : ℕ → ℝ}
    (hAsum : Summable A)
    (hEsum : Summable E)
    (hcA : ∀ t, t ∈ Set.Ioo a b → ∀ k,
      unitIntervalCosineEigenvalue k * |c t k| ≤ A k)
    (hsE : ∀ t, t ∈ Set.Ioo a b → ∀ k,
      |src t k| ≤ E k)
    -- Convergence of the value series at one point of the connected interval.
    (hval : Summable (fun k => c t₀ k * cosineMode k x)) :
    HasDerivAt
      (fun t => ∑' k, c t k * cosineMode k x)
      (∑' k, cdot t₀ k * cosineMode k x) t₀ := by
  let M : ℕ → ℝ := fun k => |ν| * A k + E k

  have hMsum : Summable M :=
    (hAsum.mul_left |ν|).add hEsum

  have hterm : ∀ k t, t ∈ Set.Ioo a b →
      HasDerivAt
        (fun r => c r k * cosineMode k x)
        (cdot t k * cosineMode k x) t := by
    intro k t ht
    exact (hmode k t ht).mul_const (cosineMode k x)

  have hbound : ∀ k t, t ∈ Set.Ioo a b →
      ‖cdot t k * cosineMode k x‖ ≤ M k := by
    intro k t ht
    have hcos : |cosineMode k x| ≤ 1 := by
      simp only [cosineMode]
      exact Real.abs_cos_le_one _
    have hlam : 0 ≤ unitIntervalCosineEigenvalue k := by
      unfold unitIntervalCosineEigenvalue
      positivity
    rw [Real.norm_eq_abs, abs_mul]
    calc
      |cdot t k| * |cosineMode k x|
          ≤ |cdot t k| :=
        mul_le_of_le_one_right (abs_nonneg _) hcos
      _ = |-ν * unitIntervalCosineEigenvalue k * c t k + src t k| := by
        rw [hODE t ht k]
      _ ≤ |-ν * unitIntervalCosineEigenvalue k * c t k| + |src t k| :=
        abs_add_le _ _
      _ = |ν| *
            (unitIntervalCosineEigenvalue k * |c t k|) + |src t k| := by
        rw [abs_mul, abs_mul, abs_neg, abs_of_nonneg hlam]
        ring
      _ ≤ |ν| * A k + E k :=
        add_le_add
          (mul_le_mul_of_nonneg_left (hcA t ht k) (abs_nonneg ν))
          (hsE t ht k)
      _ = M k := rfl

  exact hasDerivAt_tsum_of_isPreconnected
    hMsum
    isOpen_Ioo
    isPreconnected_Ioo
    hterm
    hbound
    ht₀
    hval
    ht₀
```

This theorem proves the desired identity at every `t₀ ∈ Ioo a b`. The spatial point `x` can lie in `Icc 0 1`; in fact the proof works for every real `x`, since the only spatial input is `|cos(kπx)| ≤ 1`.

## Why `hval` is required

The classical theorem needs convergence of the original series at one point in the connected time interval. In this application, `hval` is normally already available from the spectral reconstruction. It can also be obtained from either:

```lean
Summable (fun k => |c t₀ k|)
```

or from eigenvalue-weighted summability via the repository helper

```lean
ShenWork.IntervalDuhamelClosedC2
  .cosineCoeff_summable_of_eigenvalue_summable
```

which turns

```lean
Summable (fun k => unitIntervalCosineEigenvalue k * |c t₀ k|)
```

into both the first-frequency-weighted and unweighted absolute summability statements. The zero mode causes no problem because it is only one finite term.

# 2. Exact Mathlib M-test and differentiation lemmas

## 2.1 M-test

The explicit Weierstrass statement is

```lean
tendstoUniformlyOn_tsum
```

for a general index type and finite-set partial sums. Its natural-number version is

```lean
tendstoUniformlyOn_tsum_nat
```

with the shape

```lean
theorem tendstoUniformlyOn_tsum_nat
    {f : ℕ → β → F} {u : ℕ → ℝ}
    (hu : Summable u) {s : Set β}
    (hfu : ∀ n x, x ∈ s → ‖f n x‖ ≤ u n) :
    TendstoUniformlyOn
      (fun N => fun x => ∑ n ∈ Finset.range N, f n x)
      (fun x => ∑' n, f n x)
      atTop s
```

Thus the derivative majorant above gives directly

```lean
have hderiv_uniform :
    TendstoUniformlyOn
      (fun N t => ∑ k ∈ Finset.range N,
        cdot t k * cosineMode k x)
      (fun t => ∑' k, cdot t k * cosineMode k x)
      atTop (Set.Ioo a b) :=
  tendstoUniformlyOn_tsum_nat hMsum hbound
```

This is the precise locally uniform convergence needed for termwise differentiation.

## 2.2 Limit of derivatives

The lower-level Mathlib theorem is

```lean
hasDerivAt_of_tendstoUniformlyOn
```

and there are related forms

```lean
hasDerivAt_of_tendstoLocallyUniformlyOn
hasDerivAt_of_tendstoUniformly
```

However, for a series, the preferred theorem is

```lean
hasDerivAt_tsum_of_isPreconnected
```

from `Mathlib.Analysis.Calculus.SmoothSeries`. Its exact signature is:

```lean
theorem hasDerivAt_tsum_of_isPreconnected
    (hu : Summable u)
    (ht : IsOpen t)
    (h't : IsPreconnected t)
    (hg : ∀ n y, y ∈ t → HasDerivAt (g n) (g' n y) y)
    (hg' : ∀ n y, y ∈ t → ‖g' n y‖ ≤ u n)
    (hy₀ : y₀ ∈ t)
    (hg0 : Summable fun n => g n y₀)
    (hy : y ∈ t) :
    HasDerivAt
      (fun z => ∑' n, g n z)
      (∑' n, g' n y) y
```

Internally, this theorem uses `tendstoUniformlyOn_tsum` for the derivative series and the uniform-limit derivative theorem. It also proves convergence of the value series throughout the preconnected open set from convergence at the chosen base point.

Other useful names are:

```lean
hasDerivAt_tsum              -- global-in-time majorant
summable_of_summable_hasDerivAt_of_isPreconnected
deriv_tsum_apply
deriv_tsum
continuousOn_tsum
continuous_tsum
```

For a local positive-time window, `hasDerivAt_tsum_of_isPreconnected` is the right one. `deriv_tsum_apply` is less convenient because its hypotheses are global in the differentiation variable and because `HasDerivAt` avoids the junk-value behavior of `deriv`.

## 2.3 Pointwise derivative summability becomes a consequence

Once the uniform majorant is available, for every `t ∈ Ioo a b` one obtains

```lean
Summable (fun k => cdot t k * cosineMode k x)
```

by `Summable.of_norm_bounded hMsum`. Thus the assumed pointwise summability is consistent, but it is not the hypothesis that justifies interchanging the derivative and the infinite sum.

# 3. Obtaining the summable uniform envelope from Duhamel

Assume `0 < ν`, choose a restart time `r < a`, and suppose

\[
c_k(t)
=e^{-\nu\lambda_k(t-r)}c_k(r)
 +\int_r^t e^{-\nu\lambda_k(t-q)}s_k(q)\,dq.
\]

Assume a bounded restart coefficient and a summable source envelope:

\[
|c_k(r)|\le C_0,
\qquad
|s_k(q)|\le E_k
\quad (q\in[r,b]),
\qquad
\sum_kE_k<\infty.
\]

Put `δ := a-r > 0`. For `t ∈ [a,b]`,

\[
\begin{aligned}
\lambda_k|c_k(t)|
&\le
C_0\lambda_k e^{-\nu\lambda_k(t-r)}
 +E_k\lambda_k\int_r^t e^{-\nu\lambda_k(t-q)}\,dq\\
&\le
C_0\lambda_k e^{-\nu\delta\lambda_k}
 +\frac{E_k}{\nu}.
\end{aligned}
\]

For `k ≥ 1`, the tail estimate is the exact identity

\[
\lambda_k\int_r^t e^{-\nu\lambda_k(t-q)}\,dq
=
\frac{1-e^{-\nu\lambda_k(t-r)}}{\nu}
\le \frac1\nu.
\]

For `k=0`, the left side `λ₀|c₀(t)|` is zero, so the same displayed envelope may harmlessly be used.

Therefore one may take

\[
A_k
=C_0\lambda_k e^{-\nu\delta\lambda_k}
 +\frac{E_k}{\nu}.
\]

This is summable because:

- `∑ λ_k exp(-ν δ λ_k)` converges by heat smoothing;
- `∑ E_k` converges by hypothesis.

The differentiated-series envelope can then be chosen explicitly as

\[
M_k=\nu A_k+E_k
\le
\nu C_0\lambda_k e^{-\nu\delta\lambda_k}+2E_k,
\]

which is summable.

## Repository lemmas for this step

The homogeneous summability theorem is

```lean

  .unitIntervalCosineEigenvalue_mul_exp_summable
```

and the packaged bounded-restart version is

```lean

  .restartHomogeneousCoeff_eigenvalue_summable
```

For the Duhamel tail, the repository uses the parabolic-gain inequality

```lean
ShenWork.IntervalDuhamelRegularity.parabolicGain_le_one
```

and packages heat plus Duhamel weighted summability as

```lean
ShenWork.IntervalMildRegularityBootstrap
  .restartDuhamelCoeff_eigenvalue_summable
```

The current repository normalizes the parabolic diffusion coefficient in this spectral ODE to `1`. For an equation with coefficient `ν`, replace the mode rate by

```lean
ν * unitIntervalCosineEigenvalue k
```

or absorb `ν` into elapsed time. The heat summability lemma is then applied with positive time parameter `ν * δ`, and the Duhamel tail acquires the factor `1 / ν` above.

## Important restart caveat

Do not restart exactly at the left endpoint `r=a` if all that is known about `c_k(a)` is a uniform bound. At `t=a` the homogeneous factor is then `1`, so the majorant contains

\[
\lambda_k|c_k(a)|,
\]

which need not be summable.

Use one of the following:

- the original Duhamel formula from time `0`, exploiting `a>0`;
- a restart at an earlier time such as `r=a/2`;
- or an already-proved weighted summability statement for the restart slice `c(r)`.

Likewise, if the Duhamel integral begins at `r`, the source envelope must hold on `[r,b]`, not merely on `[a,b]`. Historical forcing can instead be absorbed into the restart datum at `r`.

# 5. Transfer from the cosine synthesis to the mild solution

Suppose the level-5 realization is available on the open time slab:

```lean
hrealize : ∀ t ∈ Set.Ioo a b, ∀ x ∈ Set.Icc (0 : ℝ) 1,
  intervalDomainLift (u t) x =
    ∑' k, c t k * cosineMode k x
```

Fix `t₀ ∈ Ioo a b` and `x : intervalDomainPoint`. First obtain the synthesis derivative from the theorem above. Then build eventual equality in time:

```lean
have hagree :
    (fun t => u t x)
      =ᶠ[𝓝 t₀]
    (fun t => ∑' k, c t k * cosineMode k x.1) := by
  refine Filter.eventuallyEq_of_mem
    (isOpen_Ioo.mem_nhds ht₀) ?_
  intro t ht
  have hxIcc : x.1 ∈ Set.Icc (0 : ℝ) 1 := x.2
  have hlift : intervalDomainLift (u t) x.1 = u t x := by
    simp [intervalDomainLift, x.2]
  rw [← hlift, hrealize t ht x.1 hxIcc]
```

Then transfer `HasDerivAt`:

```lean
have huHasDeriv := hseries.congr_of_eventuallyEq hagree
```

and extract the requested statements with

```lean
huHasDeriv.deriv
huHasDeriv.differentiableAt
```

This is exactly the pattern used by `realSlice_hasDerivAt_time`.

# 6. Minimal dependency checklist

For each `t₀ ∈ (0,T)` and fixed `x`:

1. **Per mode:** prove
   ```lean
   HasDerivAt (fun t => c t k) (cdot t₀ k) t₀
   ```
   and preferably the same statement throughout an open neighborhood.
2. **Open connected window:** choose `J = Ioo a b` with `t₀ ∈ J`.
3. **Summable uniform derivative envelope:** provide `M : ℕ → ℝ` with
   ```lean
   Summable M
   ∀ k t, t ∈ J → ‖cdot t k * cosineMode k x‖ ≤ M k
   ```
4. **Value convergence at one time:** prove
   ```lean
   Summable (fun k => c t₀ k * cosineMode k x)
   ```
5. **Apply:**
   ```lean
   hasDerivAt_tsum_of_isPreconnected
   ```
6. **Realization transfer:** use
   ```lean
   HasDerivAt.congr_of_eventuallyEq
   ```
7. **Extract:** use `.deriv` and `.differentiableAt`.

There is no spatial-endpoint obstruction in the time argument: the estimate `|cos(kπx)|≤1` is uniform for all `x`, including `x=0,1`. The only endpoint issue here is in **time**; the theorem as stated is for interior times. At a time-window endpoint one would use a within-derivative formulation or enlarge the window inside `(0,T)`.
