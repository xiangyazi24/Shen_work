# Q3592 audit: Task146 boundary-window persistence residual

## 1. Is Task146 sound and useful?

Yes.  Task146 is logically sound and useful.  It does create a parallel residual, but it is a **strictly better residual** than the current one and should become the preferred boundary-persistence input.

Current residual:

```lean
def BFormPositiveDatumLocal.BoundaryMinPersistenceBound (p : CM2Params) : Prop :=
  ∀ {u₀}, PositiveInitialDatum intervalDomain u₀ →
  ∀ {M}, 0 < M → (∀ x, |u₀ x| ≤ M) →
  ∀ {t₁ T} {u v},
    IsPaper2ClassicalSolution intervalDomain p T u v →
    InitialTrace intervalDomain u₀ u →
    ...
```

It asks for the boundary derivative estimate at all formal `t₁`, even though the persistence theorem only uses it after introducing:

```lean
ht₁ : 0 < t₁
```

This is exactly why `IntervalDomainMinPersistChiZero.lean` had to inline the proof: endpoint boundary lemmas need `0 < t₁` to derive `0 < s` from `s ∈ Set.Ico (t₁ / 2) T`.

The corrected residual:

```lean
def BoundaryMinPersistenceWindowBound (p : CM2Params) : Prop :=
  ∀ {u₀}, PositiveInitialDatum intervalDomain u₀ →
  ∀ {M}, 0 < M → (∀ x, |u₀ x| ≤ M) →
  ∀ {t₁ T} {u v}, 0 < t₁ →
    IsPaper2ClassicalSolution intervalDomain p T u v →
    InitialTrace intervalDomain u₀ u →
    ...
```

is weaker, better aligned with `MinPersistenceAtoms.minPersist_existsC_uniform`, and avoids making future boundary proofs solve impossible/generic-time obligations.  Keep the old residual for compatibility, but route new source-side hQuant producers through the windowed residual.

## 4. Pitfalls to avoid

* **Do not route through the old `classicalMinPersistence_of_boundary`.**  It cannot pass `0 < t₁` into `hbdry`; call `MinPersistenceAtoms.minPersist_existsC_uniform` directly.

* **Use fully qualified names for min-persistence internals** inside `BFormPositiveDatumLocal`:

  ```lean
  ShenWork.MinPersistenceAtoms.pid_exists_bound
  ShenWork.MinPersistenceAtoms.hSupNorm_of_regime
  ShenWork.MinPersistenceAtoms.minPersist_existsC_uniform
  ```

  The file does not open `ShenWork.MinPersistenceAtoms`.

* **Keep `M` and `M'` straight.**  `BoundaryMinPersistenceWindowBound` should quantify over the datum bound `M`, but its coefficient must use:

  ```lean
  SupNormBridge.regimeBound p M
  ```

  In the `minPersist_existsC_uniform` call, instantiate:

  ```lean
  (M' := SupNormBridge.regimeBound p M)
  ```

  This makes the coefficient in the window bound definition unify with the lower-level theorem.

* **Use `hsol.T_pos` for hSupNorm.**  The `hSupNorm_of_regime` call needs `0 < T`; do not try to derive it from the `ClassicalMinPersistence` window hypotheses.

* **`intervalDomain.Point` vs `intervalDomainPoint`.**  Existing hQuant statements use `intervalDomain.Point → ℝ`; local residual definitions use `intervalDomainPoint → ℝ`.  In this repo those are definitionally aligned, but avoid changing statement shapes.  Mirror the existing theorem signatures exactly.

* **Negative-part namespace qualification.**  In `BFormPositiveDatumNegPart`, refer to the windowed residual as:

  ```lean
  ShenWork.Paper2.BFormPositiveDatumLocal.BoundaryMinPersistenceWindowBound p
  ```

  to avoid accidental lookup of a same-named local declaration if one is added later.

* **No squared-barrier substitution for hQuant.**  The `PositiveDatumBFormLocalHypSq*` packages are for `PaperPositiveInitialDatum`, not all `PositiveInitialDatum`; using them to prove hQuant would move assumptions sideways or require a false PID-to-PPID conversion.
