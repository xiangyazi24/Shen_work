# Definitive verdict

**Yes: in the repository's cosine-spectral Sobolev scale, the one-dimensional Neumann elliptic `+2` gain is an easy diagonal multiplier estimate, not a hard PDE-regularity theorem.**

For

```text
(μ - ν ∂ₓₓ) R = f,          μ > 0, ν > 0,
Rₓ(0) = Rₓ(1) = 0,
```

write

```text
λ_k := (kπ)²,
f(x) = Σ_k f̂_k cos(kπx),
R(x) = Σ_k R̂_k cos(kπx).
```

Since `-∂ₓₓ cos(kπx) = λ_k cos(kπx)`, the coefficient equation is

```text
(μ + ν λ_k) R̂_k = f̂_k,
R̂_k = f̂_k / (μ + ν λ_k).
```

Define the cosine-spectral energy

```text
E_s(a) := Σ_k (1 + λ_k)^s |a_k|².
```

Then, for every real `s`,

```text
E_{s+2}(R̂)
  ≤ M(μ,ν)² E_s(f̂),

M(μ,ν) := max (1/μ) (1/ν) = 1 / min μ ν.
```

Consequently, for the associated spectral norm,

```text
‖R‖_{H_N^{s+2}} ≤ M(μ,ν) ‖f‖_{H_N^s}.
```

The constant is independent of `s`. It is also the exact operator norm of this multiplier on the weighted coefficient space.

# 1. Exact multiplier estimate

Set

```text
q(λ) := (1 + λ) / (μ + ν λ),     λ ≥ 0.
```

Let

```text
M := max (1/μ) (1/ν).
```

Because

```text
M μ ≥ 1,
M ν ≥ 1,
```

we have, for every `λ ≥ 0`,

```text
1 + λ ≤ M μ + M ν λ = M (μ + ν λ).
```

The denominator is positive, so

```text
(1 + λ) / (μ + ν λ) ≤ M.
```

Therefore, mode by mode,

```text
(1 + λ_k)^(s+2) |R̂_k|²
  = (1 + λ_k)^s |f̂_k|²
      * ((1 + λ_k) / (μ + ν λ_k))²
  ≤ M² (1 + λ_k)^s |f̂_k|².
```

Summing gives

```text
Σ_k (1 + λ_k)^(s+2) |R̂_k|²
  ≤ M² Σ_k (1 + λ_k)^s |f̂_k|².
```

This is the whole elliptic estimate.

## Sharpness of the constant

The function

```text
q(λ) = (1 + λ)/(μ + νλ)
```

has derivative with the sign of `μ - ν`:

```text
q'(λ) = (μ - ν)/(μ + νλ)².
```

Hence:

- if `μ ≤ ν`, `q` is decreasing and its maximum is `q(0) = 1/μ`;
- if `μ ≥ ν`, `q` is increasing and its supremum is `lim_{λ→∞} q(λ) = 1/ν`.

Because the Neumann spectrum contains `λ_0 = 0` and is unbounded, the supremum over the discrete spectrum is exactly

```text
max (1/μ) (1/ν).
```

Thus the energy constant `M²` and norm constant `M` are optimal for the spectral norm.

# 4. An even shorter reuse route by rescaling

There is also a reduction to the existing `ν=1` theorem:

```text
(μ - ν∂ₓₓ)R = f
⇔ ((μ/ν) - ∂ₓₓ)R = f/ν.
```

At coefficient level,

```text
(f̂_k/ν) / (μ/ν + λ_k)
  = f̂_k / (μ + νλ_k).
```

Apply the existing theorem with

```text
μ' := μ/ν,
g_k := f̂_k/ν.
```

The resulting norm constant is

```text
(1/ν) * max 1 (ν/μ)
  = max (1/ν) (1/μ).
```

This route minimizes analytic code reuse but requires a few algebra lemmas about scalar multiplication of `MemHSigma`, `hSigmaEnergy`, and `max`. The direct two-parameter multiplier lemma is probably cleaner Lean.

# 5. How this plugs into the window/source ladder

The exact norm-level vehicle is the repository's

```lean
HSigmaScale.MemHSigma
HSigmaScale.hSigmaEnergy
```

not an abstract Mathlib Sobolev-space theorem.

For a time-dependent source coefficient family `fCoeff t k`, the same estimate is uniform in time because the multiplier contains no time variable:

```text
E_{r+2}(R̂(t)) ≤ M² E_r(f̂(t))
```

for every `t`. Therefore a window bound

```text
∀ t ∈ [c,T'], E_r(f̂(t)) ≤ B
```

immediately gives

```text
∀ t ∈ [c,T'], E_{r+2}(R̂(t)) ≤ M² B.
```

If the window record stores a summable per-mode envelope rather than only a total energy bound, use the pointwise estimate

```text
(1 + λ_k)^(r+2) |R̂_k(t)|²
  ≤ M² (1 + λ_k)^r |f̂_k(t)|².
```

Thus an input envelope `E_f(k)` is sent to

```text
E_R(k) := M² * E_f(k),
```

and summability is just

```lean
hEf.mul_left (M ^ 2)
```

plus `Summable.of_nonneg_of_le`.

Equivalently, at the unsquared coefficient level,

```text
1/(μ + νλ_k) ≤ M/(1 + λ_k),
```

so

```text
(1 + λ_k)^((r+2)/2) |R̂_k|
  ≤ M (1 + λ_k)^(r/2) |f̂_k|.
```

This is exactly “two powers of spatial frequency gained by dividing by the elliptic denominator.”

Therefore:

- `MemHSigma` is the clean coefficient-norm theorem;
- `WindowCoefficientEnvelope` is the correct outer wrapper when the ladder needs constants uniform on a positive-time window;
- the resolver pass is implemented by dividing every mode by `μ+νλ_k` and mapping the source envelope through the pointwise inequality above.

No elliptic PDE library is needed.

# 6. Neumann boundary conditions

## At the spectral-construction level: yes, Neumann is built in

Every mode satisfies

```text
d/dx cos(kπx) = -kπ sin(kπx),
```

and hence

```text
(d/dx cos(kπx))|_{x=0} = 0,
(d/dx cos(kπx))|_{x=1} = 0.
```

Therefore every finite cosine partial sum satisfies the Neumann condition. The cosine basis is the eigenbasis of the Neumann Laplacian, so choosing this basis selects the Neumann realization of the elliptic operator from the start.

There is also no zero-mode compatibility obstruction because `μ > 0`:

```text
R̂_0 = f̂_0 / μ.
```

The obstruction `f̂_0 = 0` arises only for the unshifted pure Neumann Laplacian `μ = 0`, which is not the present problem.

## Formal pointwise BC: distinguish the regularity level

The phrase “Neumann for free” is mathematically correct at the operator/coefficient level, but the Lean statement used to expose the BC depends on how much convergence has been proved.

When the coefficient regularity is strong enough to justify termwise differentiation, the current repository already has the necessary endpoint lemmas:


and the closed/interior bridges in

For example, once


is available, `cosineCoeffSeries_contDiff_two` gives a globally `C²` cosine representative and the endpoint derivatives vanish termwise.

Likewise, in the `MemHSigma` route, the landed embedding


converts


into the exact weighted-`ℓ¹` hypothesis consumed by the `C²` engine. Then the pointwise Neumann BC is completely automatic.

For only `H²` regularity (`σ = 2`), weighted `ℓ²` does **not** imply absolute convergence of the second-derivative series. The Neumann condition still holds in the Sobolev trace/operator-domain sense, because the solution is in the Neumann spectral domain, but exposing that as a pointwise `R'(0)=R'(1)=0` theorem would require either:

1. a Sobolev trace/domain bridge, or
2. the stronger coefficient summability already used later in the repository.

For the chemotaxis ladder, which climbs past `σ > 5/2` in order to obtain classical `C²`, route (2) is already landed. Thus there is no separate boundary-analysis burden in the intended application.

# 7. One necessary precision: spectral `H_N^r` versus unrestricted classical `H^r`

The multiplier theorem proves, exactly,

```text
H_N^r  →  H_N^(r+2),
```

where `H_N^r` is the Neumann cosine spectral scale defined by

```text
Σ_k (1+λ_k)^r |f̂_k|² < ∞.
```

This is precisely `HSigmaScale.MemHSigma r fCoeff` in the repository.

For low orders this coincides with the usual Sobolev scale in the expected way. At high orders, however, the spectral Neumann scale includes boundary compatibility conditions. It must not be silently identified with arbitrary classical `H^r(0,1)` data.

A concrete warning example is

```text
f(x) = x.
```

This function lies in ordinary `H²(0,1)`, but its cosine coefficients satisfy, on odd modes,

```text
|f̂_k| ≍ 1/k².
```

Hence

```text
Σ_k (1+λ_k)² |f̂_k|²
```

diverges: `f` is not in the Neumann spectral `H_N²` because it does not satisfy the corresponding boundary compatibility.

The classical one-dimensional theorem

```text
f ∈ ordinary H^r(0,1)  ⇒  R ∈ ordinary H^(r+2)(0,1)
```

is still true, but for arbitrary high-order `f` it is not obtained solely by asserting

```text
ordinary H^r norm ≍ Σ (1+λ_k)^r |f̂_k|²
```

without boundary terms or compatibility hypotheses. If that unrestricted classical theorem is needed as a standalone statement, one should use a direct one-dimensional ODE weak-derivative bootstrap (or prove the boundary-corrected cosine characterization).

For the present chemotaxis source ladder, this caveat does **not** undermine the proposed route:

- the ladder is already formulated in cosine coefficients / `MemHSigma`;
- the Neumann-compatible structure is propagated spectrally;
- the downstream goal is coefficient regularity and then `C²` via the existing `σ > 5/2` embedding.

So the correct formal target is the spectral multiplier theorem, not a general-purpose Mathlib abstract elliptic-regularity theorem.
