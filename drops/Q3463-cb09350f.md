## Existing chains checked conceptually

### Does classical regularity produce it?

No. The current classical H¹ route gives positive-time identity/regularity and strict-slab continuity. It does not give initial-window integrability of `deriv (H1energy u)` on `[0,b]`. That is exactly the missing near-zero absolute-continuity/integrability statement.

### Does initial H¹ trace produce it?

No. Initial H¹ trace and compatibility can give right-continuity of `H1energy u` at zero, but continuity of a scalar function does not imply integrability of its derivative. The missing statement is an absolute-continuity/FTC or near-zero L¹ estimate.

### Does P3/Moser initial-window machinery produce it?

No. The analogous P3 theorem has the shape:

```lean
intervalDomain_lpPDECombinedInitialWindowIntegrability_of_moserDerivativeInitial
```

but it consumes

```lean
IntegratedMoserEnergyDerivativeInitialWindowIntegrability intervalDomain u T p0
```

and concerns `integratedMoserEnergy`, not `H1energy`. It is a useful architecture model, not a theorem chain for H¹.

### Does the existing H¹ RHS-integrability route produce it?

No, and using it would be circular. The theorem

```lean
H1_derivInt_of_identityRHSIntegrableBefore
```

gets derivative integrability from already available RHS integrability. But the current route obtains RHS integrability from derivative initial-window integrability. Feeding it backward would close a loop with no new analytic input.

### Target 2: a.e. proxy implies existing majorant

```lean
/-- An a.e. integrable derivative proxy gives the existing majorant frontier.
This theorem is pure measure-theory wiring and should be axiom-clean. -/
theorem H1EnergyDerivativeInitialWindowMajorantBefore_of_aeProxy
    {u : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (h : H1EnergyDerivativeInitialWindowAEProxyBefore u T) :
    H1EnergyDerivativeInitialWindowMajorantBefore u T := by
  rcases h with ⟨g, hg_int, hderiv_eq⟩
  refine ⟨g, hg_int, ?_, ?_⟩
  · intro b hb0 hbT
    have hg_on : IntegrableOn g (Set.Ioc (0 : ℝ) b) volume :=
      (intervalIntegrable_iff_integrableOn_Ioc_of_le hb0).mp
        (hg_int hb0 hbT)
    have hg_meas : AEStronglyMeasurable g
        (volume.restrict (Set.Ioc (0 : ℝ) b)) := by
      simpa [IntegrableOn] using hg_on.aestronglyMeasurable
    -- `AEStronglyMeasurable.congr` with `(hderiv_eq hb0 hbT).symm`.
    exact hg_meas.congr ((hderiv_eq hb0 hbT).symm)
  · intro b hb0 hbT
    exact (hderiv_eq hb0 hbT).mono (fun r hr => by
      rw [hr]
      exact le_rfl)
```

If `hg_on.aestronglyMeasurable` needs a slightly different coercion, prove it through the restricted-measure form explicitly:

```lean
have hg_restrict : Integrable g (volume.restrict (Set.Ioc (0 : ℝ) b)) := by
  simpa [IntegrableOn] using hg_on
exact hg_restrict.aestronglyMeasurable.congr ((hderiv_eq hb0 hbT).symm)
```

## Optional source-facing FTC record

If you want the record to look like an actual integrated H¹ energy theorem rather than just an a.e. proxy, add a slightly richer frontier and an adapter down to `H1EnergyDerivativeInitialWindowAEProxyBefore`:

```lean
/-- Integrated initial-window H¹ energy identity frontier.

Only `g_intervalIntegrable` and `deriv_eq_ae` are needed for the current
near-zero bottleneck.  The `energy_eq` field is included to make the record match
an honest FTC/absolute-continuity theorem rather than a bare estimate. -/
structure H1EnergyInitialWindowFTCBefore
    (u : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where
  g : ℝ → ℝ
  g_intervalIntegrable : ∀ {b : ℝ}, 0 ≤ b → b < T →
    IntervalIntegrable g volume (0 : ℝ) b
  deriv_eq_ae : ∀ {b : ℝ}, 0 ≤ b → b < T →
    ∀ᵐ r ∂volume.restrict (Set.Ioc (0 : ℝ) b),
      deriv (H1energy u) r = g r
  energy_eq : ∀ {b : ℝ}, 0 ≤ b → b < T →
    H1energy u b - H1energy u 0 = ∫ r in (0 : ℝ)..b, g r

/-- Extract the a.e. proxy from the integrated H¹ energy FTC package. -/
theorem H1EnergyDerivativeInitialWindowAEProxyBefore_of_initialWindowFTC
    {u : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (h : H1EnergyInitialWindowFTCBefore u T) :
    H1EnergyDerivativeInitialWindowAEProxyBefore u T :=
  ⟨h.g, h.g_intervalIntegrable, h.deriv_eq_ae⟩
```

This is the best construction-facing frontier if the next analytic task is “prove an integrated H¹ energy identity near zero.” It makes the output of that future proof immediately usable by Task83.

## Why this is better than targeting the existing pointwise proxy directly

Current `H1EnergyDerivativeInitialWindowProxyBefore` requires pointwise equality:

```lean
∀ r, r ∈ Set.Ioc 0 b → deriv (H1energy u) r = g r
```

That is appropriate if the source is a pointwise classical identity on every `r > 0`. But an integrated/weak construction route often proves equality only a.e. on time intervals. For the current bottleneck, a.e. equality is enough, because the target is interval integrability. Adding the a.e. proxy avoids forcing future proofs to strengthen an a.e. identity to pointwise identity for no reason.

## What not to do

Do not try any of the following as route C producers:

1. Use `H1IdentityRHSIntegrableBefore` or `H1_derivInt_of_identityRHSIntegrableBefore` to produce `H1EnergyDerivativeInitialWindowIntegrableBefore`. That is circular in the Task82 route.

2. Use `H1IdentityRHSComponentsContinuousBefore`, `H1LapComponentZeroRightContinuous`, `H1ZeroStartPhysicalRHSDataBefore`, or `H1LiftDeriv2ZeroSlabRepresentativeBefore`. Those are route-B zero-start H²/lap assumptions.

3. Use `IsPaper2BoundedBefore`, `intervalDomain_boundedBefore...`, or the final H¹ bound to prove the majorant/proxy. That would use the target conclusion to prove an input.

4. Use `H1ScalarDIOnBefore` as the producer. It already contains `hderivInt` on zero-start windows, so using it to prove the bottleneck reverses the route.

5. Use strict positive-time continuity alone and pass `a → 0+` without a uniform integrable majorant. This is analytically false in general.

6. Treat P3/Moser derivative initial-window integrability as H¹ derivative integrability. The scalar energies differ.
