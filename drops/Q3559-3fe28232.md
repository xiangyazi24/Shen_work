## Why raw `conjugatePicardLimit` cannot be the target trajectory

`ShenWork/Paper2/IntervalConjugatePicard.lean` defines:

```lean
def conjugatePicardLimit (p : CM2Params) (u₀ : intervalDomainPoint → ℝ) (T : ℝ)
    (t : ℝ) (x : intervalDomainPoint) : ℝ :=
  if 0 < t ∧ t ≤ T then
    atTop.limUnder (fun n => conjugatePicardIter p u₀ n t x)
  else 0
```

At `t = 0`, the condition `0 < t ∧ t ≤ T` is false, so the raw local output has

```lean
conjugatePicardLimit p u₀ T 0 x = 0
```

Hence `u_pos0` fails for the raw trajectory.  `InitialTrace` cannot repair this: `InitialTrace intervalDomain u₀ u` is only a right-time sup-norm limit statement,

```lean
∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
  D.supNorm (fun x => u t x - u₀ x) < ε
```

and does not assert `u 0 = u₀`, `v 0 = ...`, or any first-spatial-derivative trace.
