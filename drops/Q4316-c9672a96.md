# Definitive verdict

**The proposed conclusion is only half correct.** The committed leaf

```lean
def MildSolutionSliceHasDerivAtTime (S : ConjugateMildSolutionData p u₀) : Prop :=
  ∃ udot : ℝ → intervalDomainPoint → ℝ,
    (∀ t, 0 < t → t < S.T → ∀ x,
        HasDerivAt (fun r => S.u r x) (udot t x) t) ∧
    (∀ t, 0 < t → t < S.T → Continuous (udot t)) ∧
    (∀ c T', 0 < c → T' < S.T → ∃ B ≥ 0,
        ∀ t ∈ Set.Icc c T', ∀ x, |udot t x| ≤ B)
```

does indeed contain the fixed-time **spatial** continuity

```lean
Continuous (udot t).
```

Therefore:

* for the logistic reaction, fixed-time spatial continuity of the source time-derivative is pure composition/product wiring;
* likewise for the power source `ν·u^γ`, the fixed-time field
  `ν·γ·u^(γ-1)·udot` is spatially continuous;
* the same datum is useful in the elliptic time-derivative source for `R_t`.

However, this does **not** by itself discharge the field named `hadotcont` in either `DuhamelSourceTimeC1` or `DuhamelSourceTimeC1On`. That field is literally continuity in the **time variable** of each coefficient derivative:

```lean
hadotcont : ∀ n,
  ContinuousOn (fun s => adot s n) (Set.Icc lo hi)
```

for the windowed structure, and analogously `Continuous` for the global structure. Fixed-time continuity of `x ↦ udot t x` is not the same statement.

So the exact answer is:

> The leaf subsumes the fixed-time spatial-continuity seam for the logistic and power-source derivative integrands. It does not, as a matter of type, subsume `hadotcont`. To obtain `hadotcont`, one still needs time continuity in an integral topology—most conveniently joint `(t,x)` continuity of the derivative integrand on a compact slab, or an equivalent pointwise-in-time continuity plus a uniform integrable dominator.

There is a further qualification for the chemotaxis-divergence branch: continuity of `udot t` alone does not give spatial continuity of the **divergence source derivative** `∂ₓ(Q_t)`, because the outer spatial derivative requires one more spatial derivative of the time-derivative fields. The B-form cosine↔sine route can avoid this extra derivative, but it still does not remove the time-continuity requirement for `hadotcont`.

# 2. Logistic branch: fixed-time spatial continuity is pure wiring

Let

```text
L(u) = u·(a - b·u^α).
```

At an interior time `t`, the pointwise derivative is

```text
L_t(t,x)
  = u_t(t,x) · (a - b(1+α)u(t,x)^α).
```

Equivalently, using the unreduced product-rule expression,

```text
L_t
  = u_t(a - b u^α)
      - b α u^α u_t.
```

The reduced expression is preferable in Lean because it minimizes `rpow` algebra.

Assume:

```text
0 < t,
t < S.T,
0 < S.u t x,
Continuous (S.u t),
Continuous (udot t).
```

Then:

1. `x ↦ (S.u t x)^p.α` is continuous, by `Continuous.rpow_const` and positivity/nonvanishing;
2. `x ↦ p.a - p.b * (1 + p.α) * (S.u t x)^p.α` is continuous;
3. multiplying by `x ↦ udot t x` gives continuity of `L_t(t,·)`.

A schematic subtype-level proof is:

```lean
let Ldot : intervalDomainPoint → ℝ := fun x =>
  udot t x *
    (p.a - p.b * (1 + p.α) * (S.u t x) ^ p.α)

have hu_cont : Continuous (S.u t) := S.hcont t ht htT.le
have hut_cont : Continuous (udot t) := hudot_cont t ht htT

have hpow_cont : Continuous (fun x => (S.u t x) ^ p.α) :=
  hu_cont.rpow_const (fun x => Or.inl (ne_of_gt (hpos t x)))

have hfactor_cont : Continuous (fun x =>
    p.a - p.b * (1 + p.α) * (S.u t x) ^ p.α) :=
  continuous_const.sub (continuous_const.mul hpow_cont)

have hLdot_cont : Continuous Ldot := hut_cont.mul hfactor_cont
```

For the real-coordinate lift, use the standard restriction/lift bridge to obtain

```lean
ContinuousOn
  (fun x : ℝ =>
    intervalDomainLift (udot t) x *
      (p.a - p.b * (1 + p.α) *
        (intervalDomainLift (S.u t) x) ^ p.α))
  (Set.Icc 0 1).
```

Thus the fixed-time spatial-continuity atom is already present in the leaf. No new analytic theorem is required for this logistic atom.

The repository has already implemented the stronger compact-slab version through

```lean
sourceDerivSlice_continuousOn_Icc_of_lt_horizon
adottOf_continuousOn_Icc_of_lt_horizon
```

in `IntervalDomainPositiveWindowK1OnEndpoint.lean`. Those proofs use joint continuity of the time derivative supplied by the resolver/spectral agreement route, then apply compact dominated convergence to the cosine coefficient.

# 3. Why this is not yet `hadotcont`

Define

```text
adotLog(t,k) = cosineCoeffs (L_t(t,·)) k.
```

The required field is

```lean
∀ k, ContinuousOn (fun t => adotLog t k) (Icc c T').
```

Knowing only

```text
∀ t, Continuous (L_t(t,·))
```

does not imply this. A family can have a continuous spatial slice for every `t` while varying discontinuously in `t`.

A sufficient and already-used Lean route is:

```lean
have hLdot_joint :
    ContinuousOn (Function.uncurry Ldot)
      (Set.Icc c T' ×ˢ Set.Icc (0 : ℝ) 1) := ...

have hadotcont : ∀ k,
    ContinuousOn (fun t => cosineCoeffs (Ldot t) k) (Set.Icc c T') :=
  fun k =>
    cosineCoeffs_continuousOn_of_jointContinuousOn_Icc k hLdot_joint
```

The theorem

```lean
cosineCoeffs_continuousOn_of_jointContinuousOn_Icc
```

uses compactness of the time-space slab to obtain one bounded dominator and then applies dominated continuity of the interval integral.

Alternatively, joint continuity can be replaced by the logically weaker pair:

```text
for a.e. x, t ↦ L_t(t,x) is continuous,
|L_t(t,x)| ≤ G(x) with G integrable uniformly in t.
```

But the committed leaf does not contain the first of these statements either. `HasDerivAt` of `u(·,x)` at every time does not imply continuity of its derivative as a function of time.

Therefore cq47's “extra seam” is not eliminated if cq47 meant the actual structure field `hadotcont`. It is eliminated only if cq47 meant the preliminary assertion “for each fixed `t`, the derivative integrand is spatially continuous.”

# 4. Power source and elliptic time derivative

For the elliptic source

```text
F_R(t,x) = ν·u(t,x)^γ,
```

the derivative is

```text
(F_R)_t(t,x)
  = ν·γ·u(t,x)^(γ-1)·u_t(t,x).
```

At fixed `t`, positivity of `u`, continuity of `u(t,·)`, and the leaf's

```lean
Continuous (udot t)
```

give spatial continuity immediately. This is exactly the integrand used by the committed power-source derivative definition such as `adotPow`.

By linearity of the elliptic resolver,

```text
R_t = (μ - ν_R ∂ₓₓ)⁻¹ [ν·γ·u^(γ-1)u_t].
```

Coefficientwise,

```text
(R_t)^hat_k
  = ((ν·γ·u^(γ-1)u_t)^hat_k) / (μ + ν_R λ_k).
```

Thus resolver spatial regularity gives continuity of `R_t` and the required resolver spatial derivatives, provided the corresponding coefficient summability/envelope has been established. Again, this fixed-time statement is coefficient/multiplier wiring.

But time continuity of the coefficients of `(F_R)_t`, and hence `hadotcont`, still needs time continuity of the derivative integrand. The leaf alone does not provide it.

# 5. Chemotaxis-divergence branch: one more distinction

Write

```text
Q = u · R_x · (1+R)^(-β),
S_chem = ∂ₓQ.
```

The time derivative of the flux is

```text
Q_t
  = u_t · R_x · (1+R)^(-β)
    + u · (R_t)_x · (1+R)^(-β)
    - β · u · R_x · R_t · (1+R)^(-β-1).
```

At a fixed time, continuity of

```text
u, u_t, R_x, R_t, (R_t)_x
```

implies continuity of `Q_t`. The leaf contributes the `u_t` continuity factor, while elliptic multiplier regularity supplies the `R_t` factors.

However,

```text
(S_chem)_t = ∂ₓQ_t.
```

To prove `x ↦ (S_chem)_t(t,x)` continuous by the strong pointwise route, one needs `Q_t(t,·) ∈ C¹`. The first term in `Q_t` contains `u_t`; differentiating it in `x` requires spatial regularity of `u_t`, not merely `Continuous (udot t)`. Likewise one needs higher spatial derivatives of `R_t`.

This is exactly reflected in the current definition

```lean
def coupledChemDivTimeDerivativeLift ... :=
  deriv (fun y =>
    slopeSlice u s y * deriv v y / (1+v y)^β
      + lift(u s) y * deriv vt y / (1+v y)^β
      - β * lift(u s) y * deriv v y * vt y / (1+v y)^(β+1))
```

and in `CoupledChemDivLocalChainRule`, whose continuity input is not merely

```text
Continuous (udot t),
```

but

```lean
ContinuousOn
  (Function.uncurry (coupledChemDivTimeDerivativeLift p u))
  (Icc (τ-δ) (τ+δ) ×ˢ Icc 0 1).
```

So, for the current **strong chemDiv derivative-field implementation**, the leaf's fixed-time `C⁰_x` statement is insufficient.

## B-form alternative

The outer derivative can be avoided at coefficient level. From the divergence identity,

```text
cosineCoeff(∂ₓQ, k) = (kπ)·sineCoeff(Q,k).
```

Differentiate this in time and define

```text
adotChem(t,k) = (kπ)·sineCoeff(Q_t(t,·),k).
```

This route requires continuity/integrability of `Q_t`, but not pointwise continuity of `∂ₓQ_t` and therefore not `u_{tx}` merely to define the coefficient derivative. It is the cleaner route if the goal is only source-coefficient time differentiation.

Nevertheless, to prove

```text
t ↦ adotChem(t,k)
```

continuous, one still needs time continuity of `Q_t` in `L¹`, or joint `(t,x)` continuity with a compact-slab bound. The fixed-time spatial continuity supplied by the leaf is again only one component.

# 6. Exact corrected dependency statement

The honest dependency graph is:

```text
MildSolutionSliceHasDerivAtTime
  ├─ pointwise HasDerivAt u
  │    └─ pointwise chain rule for logistic / power / flux
  ├─ Continuous (udot t)
  │    ├─ fixed-t spatial continuity of logistic derivative integrand
  │    ├─ fixed-t spatial continuity of power-source derivative integrand
  │    └─ fixed-t continuity of Q_t after elliptic R_t regularity
  └─ uniform |udot| bound
       └─ uniform coefficient derivative bound after source-factor bounds

ADDITIONAL for hadotcont:
  time continuity or joint (t,x) continuity of the derivative integrand
  └─ compact dominated-continuity theorem for cosine/sine coefficients

ADDITIONAL for strong chemDiv derivative field:
  enough spatial regularity to make ∂ₓQ_t continuous
  (or avoid this through the B-form sine-coefficient route).
```

# Final read

**No: `hadotcont` is not discharged from `MildSolutionSliceHasDerivAtTime` as currently typed.** The leaf does already contain the fixed-time spatial continuity that cq47 may have been worrying about, and for the logistic/power branches that atom is pure wiring. But the actual `hadotcont` field is time continuity of `adot`, and the chemDiv strong derivative field additionally needs spatial differentiability beyond `Continuous (udot t)`.

The best implementation strategy is:

1. retain `MildSolutionSliceHasDerivAtTime` for pointwise chain rules, fixed-time spatial continuity, and uniform bounds;
2. derive or carry the already-existing stronger spectral-agreement consequence
   `mildSolution_timeDeriv_jointContinuousOn` on compact positive windows;
3. use `cosineCoeffs_continuousOn_of_jointContinuousOn_Icc` to fill `hadotcont`;
4. for chemDiv, prefer the B-form formula
   `adotChem = sqrt(λ_k)·sineCoeff(Q_t,k)` if the consumer does not genuinely need the strong pointwise field `∂ₓQ_t`.

Thus the time half becomes wiring **once the stronger joint/time-continuity output of the spectral implementation is retained**, but not from the committed leaf proposition alone.
