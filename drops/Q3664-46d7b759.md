## Executive verdict

There is **not yet a public theorem with exactly this `derivWithin` endpoint package** for the concrete mild slice.

There **is** already enough restart-cosine / Neumann infrastructure to prove it for the concrete slice

```lean
w := intervalDomainLift (Dsol.u t)
```

provided we have

```lean
H : HasRestartCosineRepresentations Dsol.T Dsol.u
```

and `0 < t`, `t < Dsol.T`.  But the current public chain stops at:

```lean
ContDiffOn ℝ 2 (intervalDomainLift (Dsol.u t)) (Set.Icc 0 1)
Filter.Tendsto (deriv (intervalDomainLift (Dsol.u t))) (nhdsWithin 0 (Set.Ioi 0)) (nhds 0)
Filter.Tendsto (deriv (intervalDomainLift (Dsol.u t))) (nhdsWithin 1 (Set.Iio 1)) (nhds 0)
```

and at a separate `deriv` endpoint-zero package.  The missing small atom is the conversion from closed-interval `C²` plus one-sided Neumann limits to the exact `derivWithin ... (Set.Icc 0 1)` endpoint equalities.

Therefore: **a small wrapper is the right next step**, but it should live in the restart/regularity layer first, not directly hidden inside `ChemMildC1etaUncond.lean`.

Mismatch summary:

1. `gradientMild_closedC2_endpointDerivs_of_restartCosineRepresentations` gives endpoint `deriv`, not endpoint `derivWithin`.
2. `NeumannBoundarySlice` gives values of an externally supplied derivative representation `Dw`, not `derivWithin w (Icc 0 1)`.
3. `mildSolution_neumann_of_positive_time` is explicitly a junk `deriv` fact for the zero-extension, not the closed-interval one-sided derivative.
4. The real useful data for `derivWithin` are the one-sided `Tendsto (deriv ...)` facts from restart-cosine, but no public theorem currently converts them to the exact hNeumann pair.

## 3. Can a small wrapper discharge `hNeumann` now?

For the **concrete mild slice**

```lean
w := intervalDomainLift (Dsol.u t)
```

yes, a small wrapper can discharge it now from the restart-cosine package.

For an arbitrary representative `w`, no: one also needs a transport theorem showing that `w` and the concrete lift agree in a neighbourhood within `[0,1]` at the endpoints, or simply a hypothesis such as:

```lean
Set.EqOn w (intervalDomainLift (Dsol.u t)) (Set.Icc (0 : ℝ) 1)
```

together with a `derivWithin` equality transport.  That transport is also small, but it is a separate wrapper.  The source-side reusable atom should first be stated for the concrete lift.

```lean
import ShenWork.Paper2.IntervalMildRegularityBootstrap

open MeasureTheory Filter Topology
open ShenWork.IntervalDomain

namespace ShenWork.IntervalMildRegularityBootstrap

noncomputable section

/-- Restart-cosine regularity gives the exact closed-interval one-sided no-flux
package consumed by `holderCosineCoeff_summable_diffOn`. -/
theorem gradientMild_derivWithin_endpoint_zero_of_restartCosineRepresentations
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀)
    (H : HasRestartCosineRepresentations D.T D.u) :
    ∀ t, 0 < t → t < D.T →
      derivWithin (intervalDomainLift (D.u t)) (Set.Icc (0 : ℝ) 1) 0 = 0 ∧
      derivWithin (intervalDomainLift (D.u t)) (Set.Icc (0 : ℝ) 1) 1 = 0 := by
  intro t ht htT
  rcases gradientMild_closedC2_neumann_of_restartCosineRepresentations D H with
    ⟨hC2, hN0, hN1⟩
  let f : ℝ → ℝ := intervalDomainLift (D.u t)
  have hC2t : ContDiffOn ℝ 2 f (Set.Icc (0 : ℝ) 1) := hC2 t ht htT
  have hdiff : DifferentiableOn ℝ f (Set.Ioo (0 : ℝ) 1) :=
    (hC2t.differentiableOn (by norm_num)).mono Set.Ioo_subset_Icc_self
  have hcont0 : ContinuousWithinAt f (Set.Ioo (0 : ℝ) 1) 0 :=
    (hC2t.continuousOn 0 (by constructor <;> norm_num)).mono Set.Ioo_subset_Icc_self
  have hcont1 : ContinuousWithinAt f (Set.Ioo (0 : ℝ) 1) 1 :=
    (hC2t.continuousOn 1 (by constructor <;> norm_num)).mono Set.Ioo_subset_Icc_self
  have hmem0 : Set.Ioo (0 : ℝ) 1 ∈ nhdsWithin (0 : ℝ) (Set.Ioi 0) :=
    mem_nhdsWithin.mpr ⟨Set.Iio 1, isOpen_Iio, by norm_num,
      fun z hz => ⟨hz.2, hz.1⟩⟩
  have hmem1 : Set.Ioo (0 : ℝ) 1 ∈ nhdsWithin (1 : ℝ) (Set.Iio 1) :=
    mem_nhdsWithin.mpr ⟨Set.Ioi 0, isOpen_Ioi, by norm_num,
      fun z hz => ⟨hz.1, hz.2⟩⟩
  have hleftIci : HasDerivWithinAt f 0 (Set.Ici (0 : ℝ)) 0 :=
    hasDerivWithinAt_Ici_of_tendsto_deriv hdiff hcont0 hmem0 (hN0 t ht htT)
  have hrightIic : HasDerivWithinAt f 0 (Set.Iic (1 : ℝ)) 1 :=
    hasDerivWithinAt_Iic_of_tendsto_deriv hdiff hcont1 hmem1 (hN1 t ht htT)
  have hleftIcc : HasDerivWithinAt f 0 (Set.Icc (0 : ℝ) 1) 0 :=
    hleftIci.mono (by intro z hz; exact hz.1)
  have hrightIcc : HasDerivWithinAt f 0 (Set.Icc (0 : ℝ) 1) 1 :=
    hrightIic.mono (by intro z hz; exact hz.2)
  constructor
  · exact hleftIcc.derivWithin
      ((uniqueDiffOn_Icc (by norm_num : (0 : ℝ) < 1)) 0 (by constructor <;> norm_num))
  · exact hrightIcc.derivWithin
      ((uniqueDiffOn_Icc (by norm_num : (0 : ℝ) < 1)) 1 (by constructor <;> norm_num))

end

end ShenWork.IntervalMildRegularityBootstrap
```
