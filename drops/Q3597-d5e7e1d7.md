# Q3597 — C¹ approximate identity for the homogeneous initial leg

Repo inspected: `xiangyazi24/Shen_work`, current main with Task145 (`dc26e5f0837718e4febad5352d315bb0687003c3`).  Scope stayed source-side.

## Short answer

There is **not yet enough current machinery** to prove the actual homogeneous C¹ approximate identity

```lean
∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
  ∀ x ∈ Set.Icc (0 : ℝ) 1,
    |deriv (fun z => intervalFullSemigroupOperator t (intervalDomainLift u₀) z) x -
      deriv (intervalDomainLift u₀) x| < ε
```

from `ContinuousOn` of the initial derivative alone.

The repo has the **value** approximate identity and the **gradient bound** machinery, but it is missing the actual derivative-kernel/IBP identity or commutation theorem that rewrites

```lean
deriv (fun z => intervalFullSemigroupOperator t (intervalDomainLift u₀) z) x
```

as an operator applied to the initial derivative.

The smallest safe source-side theorem to commit next is a **conditional reducer**: if an explicit derivative-commutation/IBP input is supplied, and the derivative profile has the value approximate identity, then the C¹ initial approach follows. This is not the analytic producer, but it names exactly the missing input and lets later kernel work plug in cleanly.

## What exists now

### Value approximate identity

`ShenWork/PDE/IntervalSemigroupApproxIdentity.lean` proves a pointwise spectral approximate identity:

```lean
theorem intervalFullSemigroup_tendsto_id_at_zero
    (f : ℝ → ℝ) (hf : Continuous f) (x : ℝ) (hx : x ∈ Set.Ioo (0 : ℝ) 1)
    (hl1 : Summable (fun n => |cosineCoeffs f n|))
    (hrecon : HasSum (fun n => unitIntervalCosineMode n x * cosineCoeffs f n) (f x))
    (hkernel : ∀ t : ℝ, 0 < t → ∀ y,
      intervalNeumannFullKernel t x y =
        ∑' m : ℤ, Real.exp (-t * ((m : ℝ) * Real.pi) ^ 2) *
          (Real.cos ((m : ℝ) * Real.pi * x) * Real.cos ((m : ℝ) * Real.pi * y))) :
    Tendsto (fun t => intervalFullSemigroupOperator t f x)
      (𝓝[>] (0 : ℝ)) (𝓝 (f x))
```

`ShenWork/PDE/IntervalSemigroupUniform.lean` has the stronger uniform value form for globally continuous data:


## Why value approximate identity alone is not enough

The missing step is not the value limit for the derivative profile. The missing step is the identity/commutation that connects that value limit to the actual derivative of the semigroup leg.

A statement like

```lean
intervalFullSemigroupOperator t (fun x => deriv (intervalDomainLift u₀) x) x
  → deriv (intervalDomainLift u₀) x
```

does not by itself imply

```lean
deriv (fun z => intervalFullSemigroupOperator t (intervalDomainLift u₀) z) x
  → deriv (intervalDomainLift u₀) x
```

unless we also know a theorem identifying the left derivative field with the semigroup applied to the derivative, or with the correct conjugate/Dirichlet derivative-kernel operator applied to the derivative.

For the Neumann kernel, the derivative of the Neumann semigroup is naturally represented through the conjugate kernel in `IntervalFullKernelInitialIBP.lean`, not by a currently available public theorem saying it is the same as `intervalFullSemigroupOperator t` applied to `u₀'`.

## Small safe theorem to commit next

Recommended new file:

```text
ShenWork/PDE/IntervalSemigroupC1ApproxIdentity.lean
```

This file should add a purely source-side reducer. It does not claim the missing commutation theorem; it packages it as an explicit hypothesis.

```lean
import ShenWork.PDE.IntervalSemigroupUniform

open MeasureTheory Filter Topology
open scoped Topology

open ShenWork.IntervalDomain
open ShenWork.IntervalNeumannFullKernel

namespace ShenWork.IntervalSemigroupC1ApproxIdentity

noncomputable section

/-- The full Neumann semigroup only reads the source on `[0,1]`.  This small
congruence is useful when replacing an interval-only profile by a clipped global
continuous representative. -/
theorem intervalFullSemigroupOperator_congr_on_Icc
    {f g : ℝ → ℝ}
    (hfg : ∀ y ∈ Set.Icc (0 : ℝ) 1, f y = g y)
    (t x : ℝ) :
    intervalFullSemigroupOperator t f x =
      intervalFullSemigroupOperator t g x := by
  unfold intervalFullSemigroupOperator
  apply MeasureTheory.integral_congr_ae
  have hmem : ∀ᵐ y ∂(intervalMeasure 1), y ∈ Set.Icc (0 : ℝ) 1 := by
    simp only [intervalMeasure, intervalSet]
    exact (MeasureTheory.ae_restrict_iff' measurableSet_Icc).mpr
      (Filter.Eventually.of_forall fun y hy => hy)
  filter_upwards [hmem] with y hy
  rw [hfg y hy]

/-- Abstract value approximate identity on `[0,1]` for a candidate derivative
profile.  This can be supplied by `intervalFullSemigroup_tendstoUniformlyOn` for a
globally continuous representative, or by a later clipped-extension lemma from
`ContinuousOn` data. -/
def InitialLegDerivativeValueApprox (df : ℝ → ℝ) : Prop :=
  ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
    ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |intervalFullSemigroupOperator t df x - df x| < ε

/-- Explicit commutation/IBP hypothesis for the homogeneous initial leg.  This is
currently the missing analytic theorem: it should eventually be produced from a
public kernel IBP identity, not assumed silently. -/
def InitialLegDerivativeCommutes (f df : ℝ → ℝ) : Prop :=
  ∀ {t : ℝ}, 0 < t → ∀ x ∈ Set.Icc (0 : ℝ) 1,
    deriv (fun z : ℝ => intervalFullSemigroupOperator t f z) x =
      intervalFullSemigroupOperator t df x

/-- If the derivative field commutes with the homogeneous semigroup leg, and the
candidate derivative profile has the value approximate identity, then the
homogeneous C¹ initial approach follows. -/
theorem initialLegC1Approx_of_valueApprox_of_commute
    {f df : ℝ → ℝ}
    (happrox : InitialLegDerivativeValueApprox df)
    (hcomm : InitialLegDerivativeCommutes f df) :
    ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |deriv (fun z : ℝ => intervalFullSemigroupOperator t f z) x - df x| < ε := by
  intro ε hε
  rcases happrox ε hε with ⟨δ, hδpos, hδ⟩
  refine ⟨δ, hδpos, ?_⟩
  intro t ht htδ x hx
  simpa [hcomm ht x hx] using hδ t ht htδ x hx

/-- The existing uniform value approximate identity supplies
`InitialLegDerivativeValueApprox` for a globally continuous derivative
representative.  For merely `ContinuousOn` derivative data, use a clipped global
representative plus `intervalFullSemigroupOperator_congr_on_Icc`. -/
theorem derivativeValueApprox_of_continuous
    (df : ℝ → ℝ) (hdf : Continuous df) :
    InitialLegDerivativeValueApprox df := by
  intro ε hε
  have htend :=
    ShenWork.IntervalSemigroupUniform.intervalFullSemigroup_tendstoUniformlyOn df hdf
  rw [Metric.tendstoUniformlyOn_iff] at htend
  have hev := htend ε hε
  rw [Filter.eventually_iff, mem_nhdsGT_iff_exists_Ioo_subset] at hev
  rcases hev with ⟨δ, hδpos, hδ⟩
  refine ⟨δ, hδpos, ?_⟩
  intro t ht htδ x hx
  have hdist := hδ ⟨ht, htδ⟩ x hx
  simpa [Real.dist_eq, abs_sub_comm] using hdist

/-- Domain-facing conditional homogeneous C¹ initial approach.  This is the exact
shape needed for the Task145 `PatchedSliceDerivUniformApproachAtZero` homogeneous
leg, except that the real analytic input is isolated as `hcomm`. -/
theorem intervalFullSemigroup_initialLegC1Approx_of_global_deriv_continuous_of_commute
    {u₀ : intervalDomainPoint → ℝ}
    (hu₀x_cont : Continuous (fun x : ℝ => deriv (intervalDomainLift u₀) x))
    (hcomm : InitialLegDerivativeCommutes
      (intervalDomainLift u₀)
      (fun x : ℝ => deriv (intervalDomainLift u₀) x)) :
    ∀ ε > 0, ∃ δ > 0, ∀ t, 0 < t → t < δ →
      ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |deriv (fun z : ℝ =>
            intervalFullSemigroupOperator t (intervalDomainLift u₀) z) x -
          deriv (intervalDomainLift u₀) x| < ε :=
  initialLegC1Approx_of_valueApprox_of_commute
    (derivativeValueApprox_of_continuous
      (fun x : ℝ => deriv (intervalDomainLift u₀) x) hu₀x_cont)
    hcomm

#print axioms derivativeValueApprox_of_continuous
#print axioms intervalFullSemigroup_initialLegC1Approx_of_global_deriv_continuous_of_commute

end

end ShenWork.IntervalSemigroupC1ApproxIdentity
```

This is deliberately conditional.  It is still useful because it separates the easy metric/value-approximate-identity part from the true kernel theorem.

## What is still missing for the actual producer

The missing theorem is a **public kernel IBP / derivative-kernel identity** for general interval initial data, not only the bound currently in `IntervalFullKernelInitialIBP.lean`.

Those two future theorems are the real analytic content.  The existing `intervalFullCoupledDuhamel_grad_initial_bound` proves only a uniform bound

```lean
|deriv (fun z => intervalFullSemigroupOperator t u₀ z) x| ≤ G_init
```

and therefore cannot be used to prove convergence.

## Route selection

Commit the conditional reducer above first.  It is small, non-circular, and will not mislead downstream code: the commutation/IBP theorem remains explicit.  Then attack the real kernel work in two smaller steps:

1. add the missing conjugate-kernel endpoint cancellation/equality extraction from `IntervalFullKernelInitialIBP.lean`, and
2. prove the conjugate-kernel approximate identity for continuous initial derivatives.

Only after those are in place should one try to discharge `PatchedSliceDerivUniformApproachAtZero` for the B-form/Picard initial leg.
