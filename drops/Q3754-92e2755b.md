# Audit: strict-negative/general-χ `hpde_u` for `PicardLimitRestartFrontier`

## Verdict

There is **no existing non-wrapper producer path** in the committed repo that proves the `hpde_u` field for arbitrary

```lean
D : GradientMildSolutionData p u₀
hD : D.u = picardLimit p u₀ D.T
```

in the strict-negative/general-χ branch.

The missing theorem is a real core producer, not a surface theorem.  The reason is not the final algebra: the algebraic combination of spectral time derivative, Laplacian, logistic inversion, and chemotaxis-divergence inversion already exists.  The missing part is the **source representation data** connecting the Picard-limit restart source to the true total PDE source

```lean
logistic - p.χ₀ * chemDiv
```

for arbitrary positive initial data.

In particular, `HasTimeNeighborhoodSpectralAgreement D.T D.u` only says that, near each interior time, `D.u` is represented by *some* restart source family `a`.  It does **not** say that this `a` is the cosine coefficient family of

```lean
x ↦ u * (a - b*u^α) - p.χ₀ * chemotaxisDiv p u v
```

at the represented slice.  Without that source split, `hpde_u` cannot be proved from `Hu` alone.

## Why the existing producers do not close the PLF gap

### 1. Chi-zero producer is branch-specific

`IntervalDomainPdeUProducer.lean` is explicitly a χ₀ = 0 producer:

```lean
theorem mildSolution_pde_u_of_spectral
    (p : CM2Params) (hχ0 : p.χ₀ = 0) {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀)
    (Hpde : HasSpectralPdeAgreement p D.T D.u) :
    ∀ t x, 0 < t → t < D.T → x ∈ intervalDomain.inside →
      intervalDomain.timeDeriv D.u t x =
        intervalDomain.laplacian (D.u t) x
          - p.χ₀ * intervalDomain.chemotaxisDiv p (D.u t)
              (mildChemicalConcentration p D.u t) x
          + D.u t x * (p.a - p.b * (D.u t x) ^ p.α)
```

Its internal source agreement is only logistic.  The chemotaxis term is removed by `hχ0`; this cannot be used in strict-negative/general-χ.

### 2. General-χ algebra exists, but only after total-source spectral data

`IntervalBFormPdeUProducer.lean` exposes the useful general-χ spectral agreement shape:

```lean
structure HasBFormSpectralPdeAgreement
    (p : CM2Params) (T : ℝ) (u : ℝ → intervalDomainPoint → ℝ) : Prop where
  exists_data : ∀ t₀, 0 < t₀ → t₀ < T →
    ∀ {x : intervalDomainPoint}, x.1 ∈ Set.Ioo (0 : ℝ) 1 →
    ∃ (a₀ : ℕ → ℝ) (M : ℝ) (_ : 0 ≤ M) (_ : ∀ n, |a₀ n| ≤ M)
      (a : ℝ → ℕ → ℝ) (_ : DuhamelSourceTimeC1 a)
      (offset : ℝ) (_ : 0 < t₀ - offset)
      (_ : LogisticCosineFourierData p u t₀)
      (_ : ChemDivCosineFourierData p (u t₀)
        (coupledChemicalConcentration p u t₀)),
      (∀ᶠ s in 𝓝 t₀, ∀ y : intervalDomainPoint,
        u s y = ∑' n, localRestartCoeff a₀ a (s - offset) n
          * cosineMode n y.1) ∧
      (∀ n, a (t₀ - offset) n
        = coupledLogisticSourceCoeffs p u t₀ n
          - p.χ₀ * coupledChemDivSourceCoeffs p u t₀ n) ∧

        * |localRestartCoeff a₀ a (t₀ - offset) n|)
```

and proves:


The proof itself uses the right algebra: `bForm_timeDeriv_eq_of_local_restart`, `IntervalDomainPdeUChiZero.laplacian_eq_of_rep`, logistic Fourier convergence, chem-div Fourier convergence, and finally `IntervalConjugateDuhamelMap.hpde_u_core_general_chi`.

But this theorem is packaged for the B-form/conjugate route.  The argument `_hB` is unused in the body, so the algebra could be refactored to a generic theorem.  That refactor would still not prove the PLF field from `D.u = picardLimit`; it would merely expose the exact source-spectral data that must be produced.

### 4. The gradient mild map is weak divergence-form

`IntervalGradientDuhamelMap.lean` defines the actual gradient mild map as

```lean
S(t)u₀
  + (-p.χ₀) * ∫₀ᵗ ∂ₓ S(t-s) (chemFluxLifted p (u s)) ds
  + ∫₀ᵗ S(t-s) (logisticLifted p (u s)) ds
```

The chemotaxis contribution is a weak divergence-form Duhamel term with `∂ₓ` on the semigroup and only the flux `Q` as input.  The pointwise PDE field needs the divergence expression

```lean
intervalDomain.chemotaxisDiv p (u t) (mildChemicalConcentration p u t) x
```

So a proof from `D.hmild` alone must either differentiate/integrate by parts through the gradient Duhamel term, or produce a spectral total-source identity showing that the restart source coefficients are

```lean
coupledLogisticSourceCoeffs p u t n
  - p.χ₀ * coupledChemDivSourceCoeffs p u t n
```

No committed theorem currently performs that for arbitrary `GradientMildSolutionData` / `picardLimit`.

## What must be produced first to prove the PLF field from `D.u = picardLimit`

The missing producer is:

```lean
theorem generalChi_sourceSpectralPdeAgreement_of_picardLimit
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
```

That theorem is currently not derivable from the existing assumptions unless it is given the following representation data.

### Required representation data

For every interior `t₀` and interior point `x`, produce:

1. **Local restart representation of `D.u` near `t₀`:**

   ```lean
   ∃ a₀ M hM ha₀ a src offset hoff,
     (∀ᶠ s in 𝓝 t₀, ∀ y : intervalDomainPoint,
       D.u s y = ∑' n, localRestartCoeff a₀ a (s - offset) n * cosineMode n y.1)
   ```

   `Hu : HasTimeNeighborhoodSpectralAgreement D.T D.u` supplies a representation of this shape, but not the source identity below.

2. **True total-source coefficient split:**

   ```lean
   ∀ n, a (t₀ - offset) n =
     coupledLogisticSourceCoeffs p D.u t₀ n
       - p.χ₀ * coupledChemDivSourceCoeffs p D.u t₀ n
   ```

   This is the key missing field.  `GradientMildHalfStepLogisticSourceData D` supplies logistic restart information; it does not identify the chemotaxis-divergence part of the strict-negative source.  `Hu` alone also does not identify the source family.

3. **Eigenvalue-weighted summability of the restart coefficients:**

   ```lean
   Summable (fun n => unitIntervalCosineEigenvalue n
     * |localRestartCoeff a₀ a (t₀ - offset) n|)
   ```

   This is available if the chosen representation is the `Hu` representation, via the same mechanism as `huRestartCoeff_hbsum` / local restart coefficient summability.  It still has to be attached to the representation used in the source split.

4. **Logistic cosine inversion package:**

   ```lean
   LogisticCosineFourierData p D.u t₀
   ```

   This is needed to identify the logistic source cosine series with
   `D.u t₀ x * (p.a - p.b * (D.u t₀ x)^p.α)`.

5. **Chemotaxis-divergence cosine inversion package:**

   Prefer the interior version:

   ```lean
   ChemDivCosineFourierDataIoo p (D.u t₀)
     (coupledChemicalConcentration p D.u t₀)
   ```

   because the pointwise PDE is only on `intervalDomain.inside`, and the boundary behavior of the divergence expression is the hard part.  This package must give both summability and convergence of
   ```lean
   ∑' n, coupledChemDivSourceCoeffs p D.u t₀ n * cosineMode n x.1
   ```
   to `intervalDomain.chemotaxisDiv p (D.u t₀) (mildChemicalConcentration p D.u t₀) x` for interior `x`.

## Why this is impossible from the current PLF assumptions alone

The current thinnest PPID/non-chi0 source surface still carries `hpde_u` explicitly:

```lean
def PerDatumWindowHuNoEnvelopeSourceSpectralFrontier ... : Prop :=
  ∃ _S : GradientMildHalfStepLogisticSourceData D,
  ∃ Hu : HasTimeNeighborhoodSpectralAgreement D.T D.u,
  ∃ _H : ResolverSourceWindowInput.ResolverSourceWindowHuNoEnvelopeInputs p D Hu,
    hpde_u
```

The fields `S`, `Hu`, and the Hu/no-envelope power-source K1 package are not enough for `hpde_u` because:

* `S` is logistic-source data.
* `Hu` gives local restart representations but no semantic identification of the source family.
* `ResolverSourceWindowHuNoEnvelopeInputs` is resolver-source time-regularity for `ν·u^γ`; it feeds the v-side spectral data, not the u-equation source split.
* None of these fields proves the chemotaxis-divergence cosine inversion or the equality of restart source coefficients with `logistic - χ chemDiv`.

Thus a theorem of the form

```lean
theorem hpde_u_of_picardLimit
    (D : GradientMildSolutionData p u₀)
    (hD : D.u = picardLimit p u₀ D.T) :
    hpde_u_field
```

is not currently reachable.  It would need new analytic production of the total-source spectral agreement above.
