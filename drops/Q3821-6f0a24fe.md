# Q3821 — V2 heat-smoothing bridge after the PPID/weak-PID bug

## Bottom line

The negation of `FinalAssemblyResidualFrontier` is the right diagnosis: any final residual that asks for

```lean
PaperPositiveInitialDatum intervalDomain u0
```

inside a quantifier over only

```lean
PositiveInitialDatum intervalDomain u0
```

is unsatisfiable. The repair has to keep the weak datum at time `0`, then manufacture a **new PPID datum at positive time** by heat smoothing / positivity, and only then invoke the PPID spectral tower.

The V2 route is viable, but the two remaining gaps are real:

1. `UniformConjugateMildExistenceCore` must supply enough positivity/nonnegativity to build or feed `ConjugateMildSolutionData`, or a bridge must build `ConjugateMildSolutionData` from the uniform core.
2. A fresh PPID restart does **not** by itself prove a single `IsPaper2ClassicalSolution` starting from the original weak datum unless the first segment and the seam are handled explicitly.

## (1) Do B-form Picard iterates preserve nonnegativity from weak `u0 >= 0`?

Not by the simple “nonnegative source plus nonnegative kernel” argument.

The logistic part is fine at the scalar level:

```text
u >= 0  =>  u*(a - b*u^alpha) >=? not globally nonnegative,
```

and at least at `u = 0` the reaction does not push below zero when `a >= 0`. Also the chemotaxis flux has a leading `u` factor, so it vanishes when `u = 0`. Those are exactly the structural facts used by a maximum-principle or negative-part-energy proof.

But the raw B-form Duhamel integral is not a pointwise positivity-preserving source integral. The chemotaxis contribution is a divergence-form object rewritten through a conjugate/B-kernel. The flux itself contains `v_x`, which is sign-changing in general, and the B-kernel leg is not simply “nonnegative kernel times nonnegative source”. The repo already has a diagnostic obstruction around the old B-kernel bridge: `hgradB_forces_B_kernel_endpoint_zero` records that the attempted pointwise gradient/B-kernel bridge forces an endpoint vanishing property not expected for a general source.

So the correct statement is:

* **Yes**, nonnegativity should be provable for the actual PDE / truncated B-form route by a maximum principle or negative-part-energy argument.
* **No**, it should not be claimed from a naive nonnegative-kernel Duhamel argument for the raw B-form Picard map.

## (2) Can the PPID tower be run fresh from `u(epsilon)` without uniqueness?

Yes for producing a **new local classical solution starting at `u(epsilon)`**.

No, not by itself for producing a single classical solution on `[0, epsilon + T]` from the original weak datum.

There are two separate goals:

### Fresh restart goal

If you only need

```lean
∃ U V, IsPaper2ClassicalSolution intervalDomain p T U V ∧
  InitialTrace intervalDomain (S.u eps) U
```

then uniqueness is unnecessary. The PPID tower is a fresh IVP at datum `S.u eps`. It does not need to know that it equals the old mild trajectory for later times.

### Patched original-datum goal

`CoupledFluxClassicalLocalExistenceResidual` wants a classical solution with initial trace `u0` at time `0`:

```lean
∃ u v,
  IsPaper2ClassicalSolution intervalDomain p delta u v ∧
  InitialTrace intervalDomain u0 u
```

A fresh restart from `S.u eps` only gives an initial trace at `S.u eps`, not at `u0`. To use it for the residual, one of the following must also be proved:

1. the first segment `[0, eps]` is already classical for all positive times and has initial trace `u0`; and
2. the second PPID solution can be glued at `eps` in a way that preserves the global `IsPaper2ClassicalSolution` predicate; or
3. the final local horizon stops before the seam, so the restart is not needed for the local residual.

For option 2, uniqueness is not the main issue if the patched function is defined piecewise and the second segment has initial trace equal to the first segment’s value at `eps`. The main formal issue is **regularity at the seam**. If `eps` is an interior time of the final interval, `IsPaper2ClassicalSolution` usually asks the PDE/time derivative on all `0 < t < T`, so the seam time is included. A piecewise definition can fail differentiability at `eps` unless the glue lemma is carefully stated or the solution predicate is segmented in the reachable-horizon machinery.

Therefore:

* Fresh PPID restart is valid and useful.
* Fresh PPID restart does not avoid the need for either a first classical segment or a genuine patch/glue theorem.
* For the local residual, the simplest path is to avoid an interior seam if possible: prove the first heat-smoothed weak segment is itself classical on `(0, eps)` and take `delta = eps`.
* If the PPID restart is needed to obtain classicality, then a patching theorem is a real remaining gap.

## (3) Simplest theorem signature that closes the residual

The cleanest close should expose exactly the two remaining V2 gaps and nothing PPID-inside-weak.

I would use a theorem with three inputs:

1. uniform weak-PID mild core produces enough `ConjugateMildSolutionData` for a short first segment;
2. positive-time heat smoothing makes a PPID restart datum;
3. a patch/glue or segment theorem turns the weak first segment plus PPID restart into one local classical solution from `u0`.

A concrete consumer-facing theorem can be:

```lean
import ShenWork.Paper2.IntervalDomainThm11ChiNegResidual
-- imports for the delivered V2 files:
-- import ShenWork.Paper2.<UniformConjugateMildExistenceCore file>
-- import ShenWork.Paper2.<HeatSmoothingBridge file>
-- import ShenWork.Paper2.<PPID spectral bootstrap file>

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.ChiNegResidual

noncomputable section

namespace ShenWork.Paper2.ChiNegResidual

/-- V2 local-existence closure: weak PID -> uniform conjugate mild core ->
positive-time PPID slice -> PPID spectral restart, with the seam handled by
`hPatch`.  No `PaperPositiveInitialDatum` is assumed for the original datum. -/
theorem coupledFluxClassicalLocalExistenceResidual_of_heatSmooth_ppidRestart
    (p : CM2Params) (hα : 1 <= p.α) (hγ : 1 <= p.γ)
    (hUniformCore :
      ∀ M : ℝ, 0 < M -> ∃ Tcore : ℝ, 0 < Tcore ∧
        ∀ {u0 : intervalDomain.Point -> ℝ},
          PositiveInitialDatum intervalDomain u0 ->
          (∀ x, |u0 x| <= M) ->
          ∃ S : ConjugateMildSolutionData p u0,
            S.T = Tcore)
    (hHeatPPID :
      ∀ {u0 : intervalDomain.Point -> ℝ}
        (hu0 : PositiveInitialDatum intervalDomain u0)
        (S : ConjugateMildSolutionData p u0),
        ∃ eps : ℝ, 0 < eps ∧ eps < S.T ∧
          PaperPositiveInitialDatum intervalDomain (S.u eps))
    (hPPIDLocal :
      ∀ {w0 : intervalDomain.Point -> ℝ},
        PaperPositiveInitialDatum intervalDomain w0 ->
        ∃ T : ℝ, 0 < T ∧ ∃ u v : ℝ -> intervalDomain.Point -> ℝ,
          IsPaper2ClassicalSolution intervalDomain p T u v ∧
          InitialTrace intervalDomain w0 u)
    (hPatch :
      ∀ {u0 : intervalDomain.Point -> ℝ}
        (hu0 : PositiveInitialDatum intervalDomain u0)
        (S : ConjugateMildSolutionData p u0)
        {eps T2 : ℝ} (heps0 : 0 < eps) (hepsT : eps < S.T)
        {u2 v2 : ℝ -> intervalDomain.Point -> ℝ},
        IsPaper2ClassicalSolution intervalDomain p T2 u2 v2 ->
        InitialTrace intervalDomain (S.u eps) u2 ->
        ∃ delta : ℝ, 0 < delta ∧
          ∃ u v : ℝ -> intervalDomain.Point -> ℝ,
            IsPaper2ClassicalSolution intervalDomain p delta u v ∧
            InitialTrace intervalDomain u0 u) :
    CoupledFluxClassicalLocalExistenceResidual p := by
  intro M hM
  obtain ⟨Tcore, hTcore, Hcore⟩ := hUniformCore M hM
  refine ⟨Tcore, hTcore, ?_⟩
  intro u0 hu0 hbound
  obtain ⟨S, hST⟩ := Hcore hu0 hbound
  obtain ⟨eps, heps0, hepsT, hPPID⟩ := hHeatPPID hu0 S
  obtain ⟨T2, hT2, u2, v2, hsol2, htrace2⟩ := hPPIDLocal hPPID
  obtain ⟨delta, hdelta, u, v, hsol, htrace⟩ :=
    hPatch hu0 S heps0 hepsT hsol2 htrace2
  exact ⟨u, v, hsol, htrace⟩

end ShenWork.Paper2.ChiNegResidual
```

This is intentionally a consumer theorem: it makes clear that the original datum is weak PID, and that PPID appears only for the positive-time slice `S.u eps`.

### Even simpler if first segment becomes classical

If the heat-smoothing bridge can prove the uniform-core mild solution is already classical on a short interval from time `0`, then the PPID restart and patch are not needed for `CoupledFluxClassicalLocalExistenceResidual`. The theorem becomes:

```lean
theorem coupledFluxClassicalLocalExistenceResidual_of_uniformCore_classicalFirstSegment
    (p : CM2Params) (hα : 1 <= p.α) (hγ : 1 <= p.γ)
    (hFirst :
      ∀ M : ℝ, 0 < M -> ∃ delta : ℝ, 0 < delta ∧
        ∀ {u0 : intervalDomain.Point -> ℝ},
          PositiveInitialDatum intervalDomain u0 ->
          (∀ x, |u0 x| <= M) ->
          ∃ u v,
            IsPaper2ClassicalSolution intervalDomain p delta u v ∧
            InitialTrace intervalDomain u0 u) :
    CoupledFluxClassicalLocalExistenceResidual p :=
  hFirst
```

This is tautological as written, but it identifies the real best target: **classical first segment from weak PID**. If that is attainable from heat smoothing, choose `delta` before any restart seam and avoid patching altogether.

## Recommended dependency order

1. **Fix uniform core positivity fields.** Prove nonnegativity from weak PID via negative-part/truncation or a lower-barrier theorem. Do not rely on raw B-form kernel positivity unless the exact nonnegative-kernel theorem has been proved for the conjugate operator.

2. **Prove heat-smoothing PPID slice.** For each weak datum and uniform-core trajectory, produce `eps > 0` with `PaperPositiveInitialDatum intervalDomain (S.u eps)`. This uses nonnegativity, continuity, heat kernel positivity, and small Duhamel tail/lower-barrier control.

3. **Use existing PPID spectral bootstrap from the smoothed datum.** This is a fresh IVP and does not require uniqueness.

4. **Close patching or avoid it.** If you can stop before the seam with a classical first segment, do that. Otherwise prove a glue theorem strong enough for the exact `IsPaper2ClassicalSolution` predicate.

5. **Wrap to `CoupledFluxClassicalLocalExistenceResidual`.** Use the theorem shape above; it is the narrowest honest consumer of the two remaining gaps.

## Final answer to the three questions

1. Nonnegativity from weak PID is mathematically correct for the PDE, but not by the naive nonnegative-kernel B-form integral argument. Prove it by negative-part/truncation or a lower-barrier/max-principle argument.

2. A fresh PPID tower from `u(epsilon)` avoids uniqueness for the restarted IVP, but it does not by itself give a classical solution with initial trace `u0`. You still need a first classical segment and a seam/glue theorem, or you must choose the local horizon before the seam.

3. The simplest honest residual closer is `coupledFluxClassicalLocalExistenceResidual_of_heatSmooth_ppidRestart`, with explicit hypotheses for uniform-core data, heat-smoothed PPID slice, PPID local classical restart, and patching. If the first weak segment can be made classical directly, use that instead and avoid patching entirely.
