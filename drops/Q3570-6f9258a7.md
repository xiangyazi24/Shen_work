## Executive verdict

Yes: there is an immediate small non-circular route-facing step worth adding for the constant equilibrium / zero-reaction special cases.

The new constant primitive-data producers can be lowered immediately to:

1. `H1ZeroStartPhysicalRHSDataBefore`,
2. `H1LiftDeriv2ZeroSlabRepresentativeBefore`,
3. `H1PhysicalRHSYoungScalarZeroMajorantsBefore`, and
4. conditionally, `H1PhysicalRHSStrictInitialRouteBefore` once `H1PhysicalRHSIdentityBefore` and `H1PhysicalRHSSqrtBoundsBefore` are supplied.

The first three are worth adding: they are small wrappers around already-proved generic reducers and consume only the constant primitive data, the constant classical solution, and `0 < T`. They do not use H¹ bounded-before, sqrt outputs, or downstream RHS outputs.

Do **not** claim a constant H¹ bounded-before/exit through the physical route yet. The route still lacks named constant producers for the physical H¹ identity and physical sqrt bounds, and the final bounded-before wrapper has additional scalar/L² seed/local inputs.

## Strongest useful non-circular constant theorem now

The strongest useful theorem already reachable without proving new H¹ identity/sqrt algebra is:

```lean
H1PhysicalRHSYoungScalarZeroMajorantsBefore p u_const v_const T
```

for the two constant regimes.

This is stronger than merely exposing `H1ZeroStartPhysicalRHSDataBefore` because it reaches the physical strict/initial route’s zero-window majorant input. It still does not claim the full route, because the full strict/initial route also needs:

```lean

```

## Additional assumptions still needed for further route closure

### For `H1PhysicalRHSStrictInitialRouteBefore`

After the constant Young majorant is available, the strict/initial route still needs:

```lean
hId     : H1PhysicalRHSIdentityBefore p u v T
hBounds : H1PhysicalRHSSqrtBoundsBefore p u v T V₁ V₂ M L
```

These are not currently produced for constants by named theorems.

They are mathematically simple for constants: the H¹ energy, `lapL2sq`, and the three physical scalars should all be zero when `u` and `v` are spatially constant and the reaction equilibrium condition holds. But the repo does not currently have named lemmas proving the concrete scalar triple is zero and packaging the identity/sqrt interfaces.

For sqrt bounds, even in the zero-scalar case the current record requires the sign/nonnegativity fields:

```lean
0 ≤ -p.χ₀
0 ≤ V₁, 0 ≤ V₂, 0 ≤ M, 0 ≤ L
```

or one can specialize to `V₁ = V₂ = M = L = 0` and still assume `0 ≤ -p.χ₀`.

### For `IsPaper2BoundedBefore` through the physical H¹ route

The wrapper

```lean
intervalDomain_boundedBefore_of_physicalStrictInitialRoute_before
```

still requires, besides the strict/initial physical route:

```lean
hbounded : IntervalDomainBoundednessHyp params
ha       : 0 < params.a
hu₀      : PaperPositiveInitialDatum intervalDomain u₀
hT       : 0 < T
hsol     : IsPaper2ClassicalSolution intervalDomain params T u v
htrace   : InitialTrace intervalDomain u₀ u
hfrontier : IntervalDomainL2SeedRegularityFrontier T u
hUxxL1   : H1UxxL1ContBefore u T
hcont0   : ContinuousWithinAt (H1energy u) (Set.Ici (0 : ℝ)) 0
```

Some constant ingredients are available or easy (`hsol`, `htrace`, positive datum), but not all are named in route-ready form. In particular, `IntervalDomainL2SeedRegularityFrontier` still needs the zero-right derivative package in `IntervalDomainL2SeedFrontierProducer.lean`, and the bounded-before wrapper also assumes `0 < params.a`, which excludes the `a = 0, b = 0` zero-reaction case.

A direct constant bounded-before theorem is trivial from `intervalDomainSupNorm_const`, but that would bypass the H¹ physical route and is not the same as closing the route-facing frontier.

## Why not code a bounded-before/exit wrapper now?

The physical bounded-before theorem would still need the strict/initial route plus scalar/L² seed inputs. At commit `c631b793`, the bounded-before wrapper still requires data such as:

```lean
IntervalDomainL2SeedRegularityFrontier T u
H1UxxL1ContBefore u T
ContinuousWithinAt (H1energy u) (Set.Ici (0 : ℝ)) 0
```

and parameter/initial-data inputs including:

```lean
IntervalDomainBoundednessHyp params
0 < params.a
PaperPositiveInitialDatum intervalDomain u₀
InitialTrace intervalDomain u₀ u
```

The equilibrium constant case can probably supply many of these with additional small constant lemmas, and direct `IsPaper2BoundedBefore` is trivial from `intervalDomainSupNorm_const`. But that is a different direct special-case closure, not an H¹ route-facing consequence of the new primitive-data producers. The zero-reaction constant case also fails the current bounded-before route’s `0 < params.a` assumption.
