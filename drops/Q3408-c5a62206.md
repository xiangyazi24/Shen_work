# Q3408 route audit: finite mean-ladder retyping

## Verdict

The current `MeanBundleFamily` shape is structurally too strong:

```text
∀ σ : ℝ, ∀ E : TrajectoryHSigmaEnvelope σ t ..., MeanStepBundle ... σ ... E
```

Because `MeanStepBundle` itself contains the band fields

```text
hσ0 : 1 / 2 < σ
hσ1 : σ < 3 / 2
```

the full family asks for bundles at impossible `σ`. The right fix is not to weaken `MeanStepBundle`; the band hypotheses are mathematically meaningful. The fix is to stop asking for a bundle at every real `σ`.

The smallest Lean-friendly retyping is to add a new restricted iterator whose input is a **finite, recursively threaded step supply** tied to the actual current envelope. This is better than making the core recursion consume `Fin n`, because the recursive supply follows the same dependency pattern as the proof: after one step, the next sigma and the next envelope are exactly the ones produced by the previous bundle.

In short:

- add a new finite/nested supply type;
- add a restricted iterator consuming that supply;
- add a restricted H1 reach theorem consuming that supply;
- retype downstream `CarrySeam` data to a finite/nested seam supply, with an adapter to the mean supply;
- keep the old full-family APIs only as compatibility wrappers from full-family input to restricted input, not as the capstone route.

## 1. Definitions to add vs. modify

### Add a recursive sigma function

Do not make the recursive iterator fight the closed form `σ₀ + n * (1 / 4)` at every step. Add a recursive sigma function for the internal API:

```text
meanSigmaAfter 0 σ       = σ
meanSigmaAfter (n+1) σ   = meanSigmaAfter n (σ + 1 / 4)
```

Then separately prove the arithmetic lemma:

```text
meanSigmaAfter n σ = σ + n * (1 / 4)
```

The restricted iterator should compute into `TrajectoryHSigmaEnvelope (meanSigmaAfter n σ₀) ...` internally. A closed-form wrapper can cast this to `TrajectoryHSigmaEnvelope (σ₀ + n * (1 / 4)) ...` using the lemma.

This one design choice prevents sigma arithmetic from polluting every recursive branch.

### Add a nested finite step supply

Use a dependent finite supply, not a global `∀ σ` family:

```text
MeanStepSupply 0 σ E := PUnit

MeanStepSupply (n+1) σ E :=
  Σ B : MeanStepBundle μ σ β χ₀ t ... E,
    MeanStepSupply n (σ + 1 / 4) B.step
```

Equivalently, this can be an inductive type:

```text
nil  : MeanStepSupply 0 σ E
cons : (B : MeanStepBundle μ σ β χ₀ t ... E) ->
       MeanStepSupply n (σ + 1 / 4) B.step ->
       MeanStepSupply (n+1) σ E
```

This is the core retyping I would recommend. It asks for exactly the bundles used by the ladder and no others.

### Add the restricted iterate

Add a new iterator with a name such as:

```text
meanStep_iterate_restricted
meanStep_iterate_finite
meanStep_iterate_supply
```

Prose signature:

```text
Given n, a base sigma σ₀, a base envelope E₀ at σ₀, and a MeanStepSupply n σ₀ E₀,
produce a TrajectoryHSigmaEnvelope (meanSigmaAfter n σ₀) t ... .
```

The recursion is structurally aligned:

```text
n = 0:
  return E₀

n+1:
  unpack B and tail
  recurse on B.step and tail
```

No `Fin` index appears in the core recursion. No bundle is requested at the final sigma.

### Add a closed-form compatibility wrapper

If existing downstream code expects the old closed sigma expression, add:

```text
meanStep_iterate_restricted_closed
```

with conclusion:

```text
TrajectoryHSigmaEnvelope (σ₀ + n * (1 / 4)) t ...
```

This wrapper should just apply `meanStep_iterate_restricted` and cast by `meanSigmaAfter_eq_add_nat_mul`.

### Add a restricted H1 reach theorem

Add a restricted version of the reach theorem:

```text
meanReach_H1_of_base_restricted
```

Prose signature:

```text
Given n, hreach, E₀, and MeanStepSupply n σ₀ E₀,
produce TrajectoryHSigmaEnvelope 1 t ...
by applying antitone to the restricted iterate.
```

The old `hreach` can remain formulated in terms of `σ₀ + n * (1 / 4)` if the closed-form wrapper is used. Alternatively, formulate the new theorem with `meanSigmaAfter n σ₀` and keep a closed-form compatibility theorem.

### Keep old names only as wrappers, if needed

Do not immediately delete the old full-family API if many files depend on it. Instead, add an adapter:

```text
MeanStepSupply.ofFull :
  MeanBundleFamily ... ->
  MeanStepSupply n σ₀ E₀
```

Then define the old iterator and old reach theorem through the restricted versions:

```text
meanStep_iterate Bf n E₀ :=
  meanStep_iterate_restricted n E₀ (MeanStepSupply.ofFull Bf n E₀)

meanReach_H1_of_base n hreach E₀ Bf :=
  meanReach_H1_of_base_restricted n hreach E₀ (MeanStepSupply.ofFull Bf n E₀)
```

This preserves existing APIs while letting the capstone use the new finite route.

## 2. How to avoid painful `Fin` casts

Avoid using `Fin n` in the recursive engine.

A `Fin n` family like

```text
∀ i : Fin n,
  ∀ E : TrajectoryHSigmaEnvelope (σ₀ + (i : ℕ) * (1 / 4)) t ...,
    MeanStepBundle ... (σ₀ + (i : ℕ) * (1 / 4)) ... E
```

is mathematically reasonable, but it is not the best recursion signature in Lean. In the successor case, the tail has indices shifted by one. You then need repeated casts showing that

```text
(σ₀ + 1 / 4) + i * (1 / 4)
= σ₀ + (i + 1) * (1 / 4)
```

and you have to move between `Fin n`, `Fin (n+1)`, `i.val`, and `i.succ`. This is exactly the kind of arithmetic/indexing glue that becomes the proof rather than the proof infrastructure.

The nested supply avoids this. The tail is already indexed by the actual next envelope:

```text
MeanStepSupply n (σ + 1 / 4) B.step
```

so the recursive call is definitionally aimed at the right current state.

If a finite indexed family is useful for upstream producers, keep it as an adapter layer, not the core iterator. The least painful finite-family variant is to index by the recursive sigma function:

```text
MeanBundleFamilyAlong n σ₀ :=
  ∀ i : Fin n,
    ∀ E : TrajectoryHSigmaEnvelope (meanSigmaAfter i.val σ₀) t ...,
      MeanStepBundle ... (meanSigmaAfter i.val σ₀) ... E
```

Then write one adapter:

```text
MeanStepSupply.ofAlong :
  MeanBundleFamilyAlong n σ₀ ->
  MeanStepSupply n σ₀ E₀
```

This localizes the `Fin` shift pain in one place. The iterator itself should not know about `Fin`.

## 3. Downstream `CarrySeam` retyping

The current capstone shape

```text
C : ∀ σ E, CarrySeam ... σ E
```

has the same flaw as `MeanBundleFamily`: if `CarrySeam` contains or produces a `MeanStepBundle`, then asking for all real `σ` is impossible or at least far stronger than the actual ladder needs.

Retype downstream data to a finite seam supply matching the mean supply:

```text
CarrySeamSupply 0 σ E := PUnit

CarrySeamSupply (n+1) σ E :=
  Σ C : CarrySeam ... σ E,
    CarrySeamSupply n (σ + 1 / 4) (C.toMeanStepBundle.step)
```

The exact projection name may differ, but the intended dependency is:

```text
CarrySeam ... σ E
  -> MeanStepBundle ... σ E
  -> next envelope at σ + 1 / 4
  -> tail seams start from that next envelope
```

Then add an adapter:

```text
CarrySeamSupply.toMeanStepSupply :
  CarrySeamSupply n σ₀ E₀ ->
  MeanStepSupply n σ₀ E₀
```

The capstone should consume `CarrySeamSupply n σ₀ E₀`, or a producer that builds such a supply, rather than `∀ σ E, CarrySeam ... σ E`.

If there is already a uniform seam theorem valid for all sigmas in the open band, a second optional compatibility layer is useful:

```text
CarrySeamBandFamily :=
  ∀ σ,
    1 / 2 < σ -> σ < 3 / 2 ->
    ∀ E : TrajectoryHSigmaEnvelope σ t ...,
      CarrySeam ... σ E
```

Then add an adapter from the band family to the finite seam supply, requiring only finite band proofs:

```text
∀ i < n,
  1 / 2 < meanSigmaAfter i σ₀ ∧ meanSigmaAfter i σ₀ < 3 / 2
```

This is sound and often convenient, but it should be an adapter. The capstone itself should not require a seam outside the finite ladder.

## 4. Are old full-family wrappers sound?

Yes, but only in one direction.

A full family

```text
∀ σ E, MeanStepBundle ... σ ... E
```

can always be converted into a finite supply by applying it at the actual current sigma and envelope at each step. Therefore old theorems that accept a full family can be kept as compatibility wrappers over the restricted theorems.

But the converse is false: a finite supply does not give a full family. It gives exactly `n` bundles along one generated trajectory. So do not make the new restricted data an alias for the old full family, and do not advertise compatibility wrappers as the new capstone route.

Recommended naming discipline:

```text
MeanBundleFamily          -- keep temporarily for old overstrong API, or rename to MeanBundleFullFamily
MeanStepSupply            -- new exact finite recursive data
MeanBundleFamilyAlong     -- optional finite indexed adapter
MeanBundleBandFamily      -- optional uniform-in-band adapter
```

For `CarrySeam`, similarly:

```text
CarrySeamSupply           -- new exact finite recursive seam data
CarrySeamBandFamily       -- optional uniform-in-band adapter
```

The old full-family route is logically sound as a theorem assumption, but it is architecturally unsuitable for the headline/capstone because it asks for impossible off-band seams.

## 5. Step-count traps

The ladder needs seams at the starting sigmas, not at the final sigma.

For `n` steps, the required current sigmas are:

```text
σ₀,
σ₀ + 1 * (1 / 4),
σ₀ + 2 * (1 / 4),
...
σ₀ + (n - 1) * (1 / 4)
```

The final envelope is at:

```text
σ₀ + n * (1 / 4)
```

There should be no requirement for a `MeanStepBundle` or `CarrySeam` at the final sigma `σ₀ + n * (1 / 4)`. In the nested supply, this is automatic:

```text
MeanStepSupply 0 σ E := PUnit
MeanStepSupply (n+1) σ E := head step at σ, then tail of length n at σ + 1 / 4
```

That definition asks for exactly one head seam per successor constructor and none in the base case.

If a `Fin` adapter is added, its index type must be `Fin n`, not `Fin (n+1)`. In the successor case, index `0` supplies the head step and indices `1` through `n` supply the tail. Do not accidentally require an index `n`, because that is the final sigma and is not used for a step.

## 6. Band-proof handling

There are two faithful choices:

1. Store band proofs inside each `MeanStepBundle`, as already done. Then the finite supply carries exactly the band proofs at the sigmas that are actually used.

2. If using a `CarrySeamBandFamily`, require explicit finite band proofs for all `i < n` and use them to construct the finite seam supply.

Do not require band proofs for the final sigma unless a downstream theorem independently needs them. The mean-step ladder itself needs band proofs only for the step-producing sigmas.

Also do not replace the finite supply with a global band-family unless the proof genuinely has a uniform seam theorem for every `σ` in the open band and every compatible envelope. A band-family is inhabitable, unlike the current full family, but it may still be stronger than the finite ladder requires.

## 7. Recommended implementation order

1. Add `meanSigmaAfter` and prove the closed-form arithmetic lemma.

2. Add `MeanStepSupply` as the core finite recursive data type.

3. Add `meanStep_iterate_restricted` returning the recursive-sigma target.

4. Add a closed-form wrapper returning the old `σ₀ + n * (1 / 4)` target.

5. Add `meanReach_H1_of_base_restricted`.

6. Add `MeanStepSupply.ofFull` and keep old `meanStep_iterate` / `meanReach_H1_of_base` as wrappers if preserving old APIs is important.

7. Add `CarrySeamSupply` and `CarrySeamSupply.toMeanStepSupply`.

8. Retype the capstone to consume `CarrySeamSupply n σ₀ E₀`, or consume a `CarrySeamBandFamily` plus finite band proofs and immediately convert it to `CarrySeamSupply`.

## Bottom line

The best core signature is not the proposed `Fin n` family. It is a nested finite supply tied to the current sigma and current envelope. This matches the dependent recursion, avoids `Fin` casts, requests only the seams actually evaluated by the ladder, and makes the step-count discipline automatic.

Keep the old full-family APIs only as compatibility wrappers. They are sound as overstrong assumptions, but they should not appear in the capstone or in the faithful route toward the paper theorem.
