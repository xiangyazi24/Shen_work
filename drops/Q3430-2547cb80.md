# Q3430: H¹ component continuity seam audit

## Verdict

For the **existing** package

```lean
H1IdentityRHSComponentsContinuousBefore p u T taxisX uvxx reactX
```

the requested reducer **cannot honestly fill `lap_cont` from only**

```lean
H1LiftDeriv2JointContinuousBefore u T
```

or from

```lean
IsPaper2ClassicalSolution intervalDomain p T u v
```

via the current `H1UxxL1ContBefore` route while carrying only `taxis_cont / uvxx_cont / react_cont`.

The obstruction is exactly the `a = 0` endpoint. `H1IdentityRHSComponentsContinuousBefore.lap_cont` quantifies

```lean
∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
  ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc a b)
```

whereas the landed lap producer is strict-positive-time only:

```lean
theorem lapL2sq_continuousOn_strictWindow_of_liftDeriv2_jointContinuousBefore
    {u : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (h : H1LiftDeriv2JointContinuousBefore u T) :
    ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
      ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc a b)
```

This proves the lap field whenever `0 < a`, but it says nothing about continuity of `lapL2sq` at `τ = 0` on windows `[0,b]`. The current `H1UxxL1ContBefore_of_classical_liftChemotaxisDivPhysicalRep` is also positive-time/L¹: it gives local L¹ continuity of `u_xx` at each `τ ∈ (0,T)`, not continuity of the squared L² lap component at time zero.

The smallest honest options are therefore:

1. add a **strict-window component package** with `0 < a`; or
2. keep the existing `Before` package but add one explicit endpoint input for the lap component at `0`.

The strict package is axiom-clean but does not plug into the current `H1IdentityRHSIntegrableBefore_of_componentsContinuousBefore`, because that integrability package also quantifies `0 ≤ a`. The endpoint-input reducer plugs into the existing package, but it carries one more input than `taxis/uvxx/react`.

## A strict-window reducer that should typecheck

This is axiom-clean and useful if a downstream strict-window RHS package is introduced. It intentionally does **not** claim the current `Before` package.

the exact shape supported by the landed `lapL2sq` theorem. -/

    (T : ℝ) (taxisX uvxx reactX : ℝ → ℝ) : Prop where
  lap_cont : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
    ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc a b)
  taxis_cont : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
    ContinuousOn taxisX (Set.Icc a b)
  uvxx_cont : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
    ContinuousOn uvxx (Set.Icc a b)
  react_cont : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →
    ContinuousOn reactX (Set.Icc a b)

/-- Fill the strict lap component from `H1LiftDeriv2JointContinuousBefore`,
carrying only the other three strict component continuities. -/

    {p : CM2Params} {T : ℝ}
    {u : ℝ → intervalDomainPoint → ℝ}
    {taxisX uvxx reactX : ℝ → ℝ}
    (huxx : H1LiftDeriv2JointContinuousBefore u T)
    (hTaxis : ∀ {a b : ℝ}, 0 < a → a ≤ b → b < T →

      ContinuousOn uvxx (Set.Icc a b))

  refine
    { lap_cont := ?_
      taxis_cont := ?_
      uvxx_cont := ?_
      react_cont := ?_ }
  · intro a b ha hab hbT
    exact
      lapL2sq_continuousOn_strictWindow_of_liftDeriv2_jointContinuousBefore
        (u := u) (T := T) huxx ha hab hbT
  · intro a b ha hab hbT
    exact hTaxis ha hab hbT
  · intro a b ha hab hbT

  · intro a b ha hab hbT

## Reducer for the existing `Before` package with one explicit lap endpoint input

If the new `H1InitialEndpointData` wrapper contains an endpoint statement of this shape, this is the smallest honest reducer for the existing package:

```lean
import ShenWork.Paper2.IntervalChiNegH1Bridge
import ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity

open MeasureTheory Set
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1Energy
open ShenWork.Paper2.IntervalChiNegH1ScalarRegularityProducer
open ShenWork.Paper2.IntervalChiNegH1LapComponentContinuity

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1Bridge

/-- Existing `Before` package from positive-time `u_xx` joint continuity plus an
explicit zero-endpoint lap-component continuity input.  This is the minimal way
to bridge the `0 < a` / `0 ≤ a` mismatch without pretending endpoint continuity
is automatic. -/
theorem H1IdentityRHSComponentsContinuousBefore_of_liftDeriv2_jointContinuousBefore_and_lap_zero
    {p : CM2Params} {T : ℝ}
    {u : ℝ → intervalDomainPoint → ℝ}
    {taxisX uvxx reactX : ℝ → ℝ}
    (huxx : H1LiftDeriv2JointContinuousBefore u T)
    (hLap0 : ∀ {b : ℝ}, 0 ≤ b → b < T →
      ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc (0 : ℝ) b))
    (hTaxis : ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
      ContinuousOn taxisX (Set.Icc a b))
    (hUvxx : ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
      ContinuousOn uvxx (Set.Icc a b))
    (hReact : ∀ {a b : ℝ}, 0 ≤ a → a ≤ b → b < T →
      ContinuousOn reactX (Set.Icc a b)) :
    H1IdentityRHSComponentsContinuousBefore p u T taxisX uvxx reactX := by
  refine
    { lap_cont := ?_
      taxis_cont := hTaxis
      uvxx_cont := hUvxx
      react_cont := hReact }
  intro a b ha hab hbT
  by_cases ha_pos : 0 < a
  · exact
      lapL2sq_continuousOn_strictWindow_of_liftDeriv2_jointContinuousBefore
        (u := u) (T := T) huxx ha_pos hab hbT
  · have ha_eq : a = 0 := le_antisymm (le_of_not_gt ha_pos) ha
    subst a
    exact hLap0 (b := b) hab hbT

end ShenWork.Paper2.IntervalChiNegH1Bridge
```

This theorem is axiom-clean and likely to typecheck. But it is not the requested “carry only taxis/uvxx/react” reducer: it necessarily carries `hLap0`.

## Is this worth implementing now?

A strict reducer is cheap, but it does not feed the current `H1IdentityRHSComponentsContinuousBefore` / `H1IdentityRHSIntegrableBefore` APIs. The endpoint-lap reducer is also cheap and worth adding **only if** the initial endpoint wrapper already provides exactly `hLap0` or an equivalent `ContinuousOn (fun τ => lapL2sq u τ) (Set.Icc 0 b)` field.

Otherwise, the real next frontier remains the other component producers:

```lean
taxis_cont : ContinuousOn taxisX (Set.Icc a b)
uvxx_cont  : ContinuousOn uvxx   (Set.Icc a b)
react_cont : ContinuousOn reactX (Set.Icc a b)
```

The lap component is solved on strict positive windows. The only missing lap piece for the current `Before` package is a genuine `τ = 0` endpoint-continuity statement for `lapL2sq`; it is not supplied by `H1UxxL1ContBefore` and should not be inferred from it.
