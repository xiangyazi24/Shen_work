## Executive conclusion

The proposed `hsrc0`-only reduction is viable and is the right next small layer.
It can choose

```lean
bc σ k := ShenWork.IntervalPicardLimitRestart.limitCoeff p u₀ D.u σ k
```

and derive:

1. `hagree` from
   `ShenWork.Paper2.TimeNhdSubtype.limit_lift_eq_cosineSeries_of_subtypeCont_patched`;
2. `henv` from
   `ShenWork.Paper2.BddAdapterPatched.eigenvalue_mul_abs_limitCoeff_le_uniform_patched`
   and `ShenWork.IntervalPicardLimitBddAdapter.windowEigEnv_summable`;
3. initial coefficient bounds from the existing Task269 theorem
   `initial_cosineCoeffs_bound_of_positiveInitialDatum`.

The one small technical issue is that Task269's initial bound theorem gives only

```lean
∀ k, |cosineCoeffs (intervalDomainLift u₀) k| ≤ 2 * sSup (...)
```

but the patched envelope theorem also needs `0 ≤ M₀`.  Avoid proving nonnegativity
of `2 * sSup ...` again by choosing

```lean
M₀ := max (2 * sSup (Set.range fun x : intervalDomainPoint => |u₀ x|)) 0
```

Then `0 ≤ M₀` is `le_max_right _ _`, and the coefficient bound follows by
`le_trans (initial_cosineCoeffs_bound_of_positiveInitialDatum hu₀ k) (le_max_left _ _)`.

No hidden analytic obstacle was found, provided `hsrc0` is carried at horizon
`D.T` exactly.  A shorter-horizon `hsrc0` would not suffice for all
`σ : (0,D.T)`.

For a compact window `a b` with `0 < a`, `b < D.T`, `a ≤ b`, use:

```lean

  ShenWork.IntervalPicardLimitBddAdapter.windowEigEnv
    M₀ H.hsrc0.M a (H.hsrc0.env (a / 2)) k
```

Summability follows from:


The second `linarith` uses `0 < a`, `a ≤ b`, and `b < D.T` to prove
`a / 2 ≤ D.T`.

Nonnegativity of `E` is best proved by comparison with the left side of the
patched eigenvalue bound at `σ = a`:

```lean
have hleft : 0 ≤ unitIntervalCosineEigenvalue k *
    |ShenWork.IntervalPicardLimitRestart.limitCoeff p u₀ D.u a k| :=
  mul_nonneg (by unfold unitIntervalCosineEigenvalue; positivity) (abs_nonneg _)

  (ShenWork.Paper2.BddAdapterPatched.eigenvalue_mul_abs_limitCoeff_le_uniform_patched
```

This avoids proving termwise nonnegativity of every summand in `windowEigEnv`,
which would require additional facts about `src.env`.

## Notes on likely compilation details

1. The `hagree` proof may require a `simpa` instead of `exact` if Lean unfolds the
   local `limitCoeff` notation differently.  The theorem conclusion is definitionally
   the same series.
2. The `ha2T : a / 2 ≤ D.T` proof should be solved by `linarith` from
   `0 < a`, `a ≤ b`, and `b < D.T`.
3. The nonnegativity proof for `E` should use the already-proved bound at `σ = a`,
   not termwise nonnegativity of `windowEigEnv`.  There is no general public theorem
   saying `src.env (a/2) k ≥ 0`, and it is unnecessary.
4. `hsrc0` must have horizon `D.T`.  If a caller has a bounded-source package on a
   shorter horizon, it cannot directly fill this global `(0,D.T)` surface.

## Feasibility verdict

This is a good next reduction.  It removes `bc`, `hagree`, and `henv` from the
χ₀=0 resolver-source input surface and leaves only `hsrc0` as the genuine source
residual.  The proof is a clean composition of existing theorems:

* `limit_lift_eq_cosineSeries_of_subtypeCont_patched` for representation;
* `eigenvalue_mul_abs_limitCoeff_le_uniform_patched` for uniform weighted bounds;
* `windowEigEnv_summable` for summability;
* Task269's initial coefficient bound, with a `max _ 0` wrapper for nonnegativity.

The only real caveat is that this remains χ₀=0-specific and relies on the patched
bounded-source package at full horizon `D.T`.  It should not be advertised as a
non-χ0 PPID reduction.
