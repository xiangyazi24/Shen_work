# Q4228: positive-time regularity of the truncated Picard limit

## Executive answer

For **(A) joint space-time continuity**, the clean route is straightforward and formalizable:

```text
continuous iterates + uniform convergence on each compact [a,b] × [0,1]
  ⇒ continuous limit on [a,b] × [0,1].
```

The convergence you need is genuinely **uniform on the compact space-time window**, not merely pointwise in `(t,x)`. The existing geometric Picard contraction is the right source if it is stated in a sup norm over time and space, for example

```lean
∀ n, ∀ t ∈ Icc a b, ∀ x ∈ Icc 0 1,
  |lift (U (n+1) t) x - lift (U n t) x| ≤ C * θ^n
```

with `θ < 1`. Then the Weierstrass/M-test Cauchy argument gives uniform convergence of `U n` to `truncatedConjugatePicardLimit` on `[a,b] × [0,1]`.

For **(B) time C¹**, boundedness and continuity of the limit source alone are not enough for the naive kernel-differentiation proof. The derivative kernel contains a nonintegrable `1 / (t-s)`-type singularity near `s = t`. The clean formal route is therefore **spectral**, not raw kernel dominated differentiation: use the source-coefficient floor/EWA to obtain locally uniform `ℓ¹` control of the source coefficients. Then each cosine mode is C¹ in time, and termwise differentiation is justified by uniform summability.

So the combined route is:

```text
(A) uniform-on-compact limit of continuous iterates;
(B) spectral coefficient ODE + locally uniform ℓ¹ source envelope.
```

Do not try to prove (B) from mere bounded joint continuity of the nonlinear source unless you also prove additional Hölder/domain regularity. That would be under-hypothesized.

## 1. Joint continuity of the limit

Let

```lean
F n : ℝ × ℝ → ℝ := fun q => intervalDomainLift (U n q.1) q.2
F∞  : ℝ × ℝ → ℝ := fun q => intervalDomainLift (truncatedConjugatePicardLimit p u₀ q.1) q.2
K   : Set (ℝ × ℝ) := Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1
```

with `0 < a`, `a ≤ b`, and `b ≤ DT.T`.

### Needed hypotheses

You need:

```lean
hcont_n : ∀ n, ContinuousOn (F n) K
```

and uniform convergence on `K`:

```lean
hunif : TendstoUniformlyOn F F∞ atTop K
```

or an equivalent explicit Cauchy/geometric estimate.

A sufficient geometric estimate is:

```lean
∃ C θ : ℝ, 0 ≤ C ∧ 0 ≤ θ ∧ θ < 1 ∧
  ∀ n q, q ∈ K → |F (n+1) q - F n q| ≤ C * θ^n.
```

Then for `m > n`, telescope:

```text
|F m q - F n q| ≤ Σ_{j=n}^{m-1} C θ^j,
```

uniformly in `q ∈ K`, and the tail tends to zero. This proves that `F n` is uniformly Cauchy on `K`. Since the limit is already definitionally or theorem-wise `F∞`, you get `TendstoUniformlyOn F F∞ atTop K`.

### Mathlib-level route

There may be a library theorem with a name like a uniform-limit continuity theorem, but the most robust Lean path is often a direct epsilon proof:

1. Fix `q₀ ∈ K` and `ε > 0`.
2. Choose `N` so that `|F∞ q - F N q| < ε/3` uniformly on `K`.
3. Use continuity of `F N` at `q₀` within `K` to get `|F N q - F N q₀| < ε/3` for `q` near `q₀` in `K`.
4. Triangle inequality gives continuity of `F∞` at `q₀`.

Useful Mathlib ingredients:

```lean
TendstoUniformlyOn
ContinuousOn
ContinuousWithinAt
continuousOn_iff_continuousWithinAt
Metric.tendsto_nhdsWithin_nhdsWithin
```

For the geometric-to-uniform-convergence step, useful facts are:

```lean
summable_geometric_of_norm_lt_one
Summable.tendsto_sum_nat
Finset.sum_Ico_eq_sub
abs_sum_le_sum_abs
```

If your repo already has a Picard geometric theorem like

```lean

truncatedConjugatePicardIter_pointwise_convergent
```

you want to strengthen or wrap it into a compact-uniform version:

```lean

```

Then:

```lean

    (DT : TruncatedConjugateMildExistenceData p u₀)
    {a b : ℝ} (ha : 0 < a) (hab : a ≤ b) (hbT : b ≤ DT.T) :

      (fun q : ℝ × ℝ => intervalDomainLift
        (truncatedConjugatePicardLimit p u₀ q.1) q.2)
      (Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1) := by
  -- uniform limit of continuous functions
```

## 2. Time C¹: use the cosine-mode ODE, not raw kernel differentiation

The safe positive-time route is spectral.

Write the limit as

```text
u(t,x) = Σ' k, u_k(t) cos(kπx),
```

with

```text
u_k(t) = exp(-ν λ_k t) u₀,k
       + ∫₀ᵗ exp(-ν λ_k (t-s)) F_k(s) ds.
```

Here `F_k(s)` is the full nonlinear source coefficient after the B-form/IBP conversion. Then each scalar mode satisfies

```text
u_k'(t) = -ν λ_k u_k(t) + F_k(t).
```

To get pointwise C¹ in time uniformly in `x`, it is enough on every compact `J = [a,b] ⊂ (0,T)` to have:

```lean
hF_cont : ∀ k, ContinuousOn (fun s => Fcoeff k s) (Set.Icc a b)
hF_l1   : Summable fun k => sup_{s ∈ J'} |Fcoeff k s|
```

where `J'` is a slightly larger compact window, e.g. `[a/2,b]`. In a formalization, avoid literal `sup` if inconvenient; use an explicit envelope:

```lean
E : ℕ → ℝ
hE_nonneg : ∀ k, 0 ≤ E k
hE_sum : Summable E
hF_bound : ∀ k s, s ∈ Set.Icc (a/2) b → |Fcoeff k s| ≤ E k
```

This is exactly what the source-coefficient floor should provide: a `WindowSourceEnvelope 0` or equivalent `ℓ¹` coefficient envelope.

### Why this avoids the nonintegrable kernel singularity

For each mode,

```text
ν λ_k ∫₀ᵗ exp(-ν λ_k (t-s)) |F_k(s)| ds ≤ sup_{s≤t} |F_k(s)|.
```

So the differentiated Duhamel series is bounded by

```text
Σ_k ν λ_k exp(-ν λ_k t) |u₀,k|
+ Σ_k |F_k(t)|
+ Σ_k sup_{s≤t} |F_k(s)|.
```

On `[a,b]`, the first sum is finite because the exponential `exp(-ν λ_k a)` kills all polynomial/coefficient growth from bounded initial data. The last two sums are finite by the `ℓ¹` source envelope.

This gives locally uniform convergence of the derivative series:

```lean
fun t => ∑' k, (-ν * λ k * uCoeff k t + Fcoeff k t) * cos(kπx)
```

uniformly for `t ∈ [a,b]` and `x ∈ [0,1]`, because `|cos(kπx)| ≤ 1`.

Then termwise differentiation gives:

```lean
HasDerivAt
  (fun t => intervalDomainLift (truncatedConjugatePicardLimit p u₀ t) x)
  (∑' k, (-ν * λ k * uCoeff k t + Fcoeff k t) * Real.cos (k * Real.pi * x))
  t
```

for `t ∈ Ioo a b`, and the derivative is locally uniformly bounded in `x`.

## 3. What is not enough for (B)

Do not rely on the following statement without extra hypotheses:

```text
F is bounded and jointly continuous
⇒ t ↦ ∫₀ᵗ S(t-s)F(s) ds is C¹ pointwise.
```

For the heat semigroup on a Banach space, a continuous forcing gives a mild solution. To conclude a classical time derivative, one usually needs more, for example:

```text
F is Hölder in time,
```

or

```text
F(t) lies in the generator domain with enough continuity,
```

or a spectral/series condition strong enough to make the generator term summable.

The raw kernel derivative has the schematic bound

```text
|∂_t K(t-s,x,y)| ≈ (t-s)^(-1)
```

near `s=t`, which is not integrable. The familiar `(t-s)^(-1/2)` singularity is for one spatial derivative of the heat kernel; that is integrable and is enough for Lipschitz/gradient estimates, but not by itself for `u_t`.

So the load-bearing improvement for (B) is the **Fourier-ℓ¹ source floor**, not merely joint continuity.

### B. Source coefficient envelope for the limit

You need a limit-level version of the source coefficient floor, not only iterate-level:

```lean
theorem truncatedLimit_sourceCoeff_l1_envelope_positive_window
    (DT : TruncatedConjugateMildExistenceData p u₀)
    {a b : ℝ} (ha : 0 < a) (hab : a ≤ b) (hbT : b ≤ DT.T) :
    ∃ E : ℕ → ℝ,
      (∀ k, 0 ≤ E k) ∧ Summable E ∧
      ∀ k s, s ∈ Set.Icc a b →
        |truncatedLimitSourceCoeff p u₀ s k| ≤ E k
```

This should come from the positive-time Lipschitz bound plus the flux IBP/coefficient estimate.

### C. Modewise C¹ and derivative series

Prove the scalar mode lemma:

```lean
theorem heat_mode_duhamel_hasDerivAt
    {λ ν : ℝ} (hλ : 0 ≤ λ) (hν : 0 < ν)
    {F : ℝ → ℝ} (hF : ContinuousOn F (Set.Icc a b))
    (t : ℝ) (ht : t ∈ Set.Ioo a b) :
    HasDerivAt
      (fun r => Real.exp (-(ν * λ) * r) * u0
        + ∫ s in 0..r, Real.exp (-(ν * λ) * (r-s)) * F s)
      (-(ν * λ) *
        (Real.exp (-(ν * λ) * t) * u0
          + ∫ s in 0..t, Real.exp (-(ν * λ) * (t-s)) * F s)
        + F t)
      t
```

This scalar lemma avoids differentiating the singular kernel as a two-variable object. It is just Leibniz plus the ODE identity for convolution with an exponential kernel.

Likely Mathlib ingredients:

```lean
intervalIntegral.integral_hasDerivAt_right
intervalIntegral.integral_hasDerivAt_of_tendsto_ae
HasDerivAt.integral_of_dominated_loc -- if available in your imports
MeasureTheory.tendsto_integral_of_dominated_convergence
```

If the direct parametric integral derivative lemma is awkward, prove the scalar ODE by rewriting the convolution with an integrating factor:

```text
z(t) = exp(-c t) ∫₀ᵗ exp(c s) F(s) ds.
```

Then differentiation uses only the fundamental theorem for

```text
∫₀ᵗ exp(c s) F(s) ds,
```

whose integrand is continuous. This is much easier in Lean.

### D. Termwise derivative theorem

Use a general theorem of the form:

```lean
theorem hasDerivAt_tsum_of_uniform_summable_deriv
    (hterm_deriv : ∀ k, HasDerivAt (fun t => f k t) (f' k t₀) t₀)
    (hsum : Summable fun k => C k)
    (hderiv_bound : ∀ᶠ t in 𝓝 t₀, ∀ k, |f' k t| ≤ C k)
    (hvalue_summable : Summable fun k => |f k t₀|) :
    HasDerivAt (fun t => ∑' k, f k t) (∑' k, f' k t₀) t₀
```

Mathlib has versions around `HasDerivAt.tsum` / `hasDerivAt_tsum` depending on imports, but if naming is a problem, this is also provable from uniform convergence of difference quotients plus dominated convergence.

For this project, the most practical specialized version is for cosine series:

```lean
theorem cosine_series_time_hasDerivAt_of_l1_mode_derivs
    {uCoeff uCoeff' : ℕ → ℝ → ℝ}
    (hmode : ∀ k, HasDerivAt (fun t => uCoeff k t) (uCoeff' k t₀) t₀)
    (hbound : ∃ E, Summable E ∧
      ∀ᶠ t in 𝓝 t₀, ∀ k, |uCoeff' k t| ≤ E k)
    ... :
    HasDerivAt
      (fun t => ∑' k, uCoeff k t * Real.cos ((k : ℝ) * Real.pi * x))
      (∑' k, uCoeff' k t₀ * Real.cos ((k : ℝ) * Real.pi * x))
      t₀
```

The bound `|cos| ≤ 1` removes all `x` dependence.

## 5. Duhamel representation of the time derivative

Once the modewise derivative is established, the representation is:

```text
∂_t u(t,x)
  = Σ_k [-ν λ_k u_k(t) + F_k(t)] cos(kπx).
```

Equivalently:

```text
∂_t u(t,·) = ν Δ u(t,·) + F(t,·),
```

where both sides are represented by absolutely/uniformly convergent cosine series on compact positive-time windows.

This is stronger and cleaner than the raw kernel formula

```text
νΔ S(t)u₀ + F(t) + ∫₀ᵗ νΔ S(t-s)F(s) ds,
```

because it avoids proving a nonintegrable kernel-differentiation statement. The kernel formula can be recovered from the modewise identity if needed.

## 6. Lean proof-cost ranking

### Cheapest reliable route

```text
A: uniform compact convergence of continuous iterates.
B: spectral mode ODE + ℓ¹ source envelope + termwise differentiation.
```

This aligns with the existing coefficient-bootstrap machinery and avoids singular kernel differentiation.

### More expensive route

```text
Kernel parametric differentiation under the integral.
```

This requires either a cancellation/Hölder argument near `s=t`, or generator-domain assumptions. It is substantially harder in Mathlib and easy to under-hypothesize.

### Not enough

```text
DT continuity/boundedness alone.
```

That gives a mild solution and likely joint continuity, but not automatically classical time C¹.

## Bottom line

(A) is a standard uniform-limit theorem: prove compact-uniform convergence of continuous iterates.

(B) should be proved spectrally. The load-bearing hypothesis is a locally uniform `ℓ¹` envelope for the nonlinear source coefficients on positive-time windows. With that, the cosine coefficients solve scalar C¹ ODEs and termwise differentiation gives a locally uniformly bounded time derivative and the mild/PDE representation.

Do not try to get (B) from mere bounded joint continuity of the source through raw heat-kernel differentiation; the upper-end singularity is not integrable without extra regularity or a spectral summability replacement.
