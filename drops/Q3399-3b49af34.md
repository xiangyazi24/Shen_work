## Why this beats the other remaining frontiers

### 1. Strict-slab `liftDeriv2` joint continuity is local and non-circular

The goal is a compact-slab continuity theorem. For every `a b` with `0 < a`, `a ≤ b`, `b < T`, prove continuity of

```lean
fun z : ℝ × ℝ => deriv (fun y : ℝ => deriv (intervalDomainLift (u z.1)) y) z.2
```

on `Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1`, or whatever exact `liftDeriv2` wrapper the bridge file uses. The proof should use the `u` equation to identify `u_xx` with the continuous physical RHS on the strict slab.

This is real PDE progress: it turns the classical PDE and component continuity already present in `IsPaper2ClassicalSolution` into the H¹ scalar regularity input needed downstream.

### 2. The restricted local H¹ seed is not the lowest-risk next step

The local seed has shape roughly

```lean
∀ τ, τ ∈ Set.Ioc (0 : ℝ) 1 → τ < T → H1energy u τ ≤ Ylocal
```

Even with `H1InitialTraceEnergyTendsto`, the zero endpoint only controls a small right-neighborhood of `0`. If `T < 1`, the interval `(0,T)` has another open endpoint, and strict-slab continuity only gives boundedness on `[δ,b]` for `b < T`, not uniformly as `b → T`. Proving the seed directly therefore needs either a scalar differential inequality/Gronwall argument or a continuity extension up to `T`; neither is as local as the `liftDeriv2` slab target.

So the local seed is a good **second** target after the scalar H¹ regularity chain is improved, but it is not the lowest-risk first target.

### 3. Endpoint trace/compatibility has already been narrowed

Given your Task55/56 split and the new `H1InitialTraceEnergyTendsto` / `H1InitialEnergyCompatibleAtZero` path, the endpoint seam is already in the right shape. The risk now is accidentally reintroducing raw `hcont0` or a closed-box `t = 0` continuity requirement. Do not do that.

### 4. Physical sqrt estimates/component continuity are risky if they use the target conclusion

The sqrt-estimate route is only safe if all inputs are pre-boundedness: mass/L²/local classical facts, resolver positivity, and component continuity. If a proof of a component estimate imports `IsPaper2BoundedBefore`, the H¹ bound, or the 1D Agmon endpoint that itself depends on the H¹ bound, it loops.

The strict-slab `u_xx` continuity theorem avoids that: it uses the PDE identity and component continuity, not the H¹ boundedness conclusion.

```lean
/-- Local algebraic continuity transfer: if the lifted second derivative agrees
on every strict slab with a jointly continuous physical RHS, then the strict-slab
`liftDeriv2` continuity package follows.

This theorem is intentionally local-before-`T`; it does not ask for continuity at
`t = 0` or `t = T`. -/
theorem liftDeriv2_jointContinuousBefore_of_eqOn_continuousOn
    {T : ℝ} {u : ℝ → intervalDomain.Point → ℝ} {F : ℝ → ℝ → ℝ}
    (hF : ∀ a b, 0 < a → a ≤ b → b < T →
      ContinuousOn (Function.uncurry F)
        (Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1))
    (hEq : ∀ a b, 0 < a → a ≤ b → b < T →
      Set.EqOn
        (Function.uncurry
          (fun (t : ℝ) (x : ℝ) =>
            deriv (fun y : ℝ => deriv (intervalDomainLift (u t)) y) x))
        (Function.uncurry F)
        (Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1)) :
    liftDeriv2_jointContinuousBefore u T := by
  intro a b ha hab hbT
  exact (hF a b ha hab hbT).congr
    (fun z hz => (hEq a b ha hab hbT hz).symm)

end ShenWork.Paper2.IntervalChiNegH1LiftDeriv2

end
```

This theorem should be nearly pure `ContinuousOn.congr`. If it does not compile verbatim, the likely mismatch is only the exact definition of `liftDeriv2_jointContinuousBefore`; adjust the final `intro` pattern to that definition, not the mathematical content.

## Routes to avoid

1. **Do not use `IsPaper2BoundedBefore` to prove the physical components.** That makes the H¹ route circular because bounded-before is the endpoint currently being proved.

2. **Do not prove the local H¹ seed from `H1InitialTraceEnergyTendsto` alone.** The tendsto theorem only gives a near-zero bound. It does not control all `τ < min 1 T`, especially when `T < 1` and `τ → T-`.

3. **Do not restore raw `hcont0`.** Task55/56 split endpoint data exists precisely to avoid that overstrong endpoint package.

4. **Do not require closed-box continuity at `t = 0`.** The H¹ scalar route only needs strict positive-time slabs plus the separate initial trace/compatibility theorems.

5. **Do not use spectral/parametric packages as a substitute for physical component continuity.** They are useful after the physical source has been shown continuous/integrable; they should not hide a missing `chemotaxisDiv` or `u_xx` continuity theorem.

6. **Do not expand `v_xx` into a new third-regularity demand.** Use `intervalDomain_v_xx_eq_reaction_lift` / `intervalDomain_v_xx_reaction_jointContinuous`; the elliptic equation already reduces it to `v` and `u^γ`.
