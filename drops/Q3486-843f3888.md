### Proof notes for the two part-continuity lemmas

The two omitted proof bodies are routine but longer.

For `H1PhysicalChemTaxisPart_continuousOn_strictSlab_of_classicalSolution`:

* Set `K := Set.Icc a b ×ˢ Set.Icc (0 : ℝ) 1`.
* Prove `K ⊆ Set.Ioo 0 T ×ˢ Set.Icc 0 1` using `ha` and `hbT`.
* Use:
  * `intervalDomain_dx_u_jointlyContinuous hsol` restricted to `K`;
  * `intervalDomain_dx_v_jointlyContinuous hsol` restricted to `K`;
  * the `v` value continuity field from `hsol.regularity` for the denominator;
  * `hsol.v_nonneg` to get `0 < 1 + v` and hence nonzero denominators.
* Finish with `(hux.mul hvx).div hden_cont hden_ne`, then `simpa [H1PhysicalChemTaxisPart, Function.uncurry]`.

For `H1PhysicalChemUvxxPart_continuousOn_strictSlab_of_classicalSolution`:

* Use the same `K` and subset argument.
* Use:
  * `u` and `v` value continuity from `hsol.regularity`;
  * `intervalDomain_v_xx_reaction_jointContinuous hsol` restricted to `K`;
  * `intervalDomain_dx_v_jointlyContinuous hsol` restricted to `K`;
  * denominator `rpow_const` and nonzero denominator facts from `v_nonneg`.
* Build the two terms:
  * `u * (μ v - ν u^γ) / (1+v)^β`
  * `β * u * v_x^2 / (1+v)^(β+1)`
* Finish with subtraction and `simpa [H1PhysicalChemUvxxPart, Function.uncurry]`.

Likely compile issues are local: naming of `intervalDomain.Point` versus `intervalDomainPoint`, and the exact access path for the `u`/`v` value continuity fields. These are already used repeatedly in `P3MoserDxJointContinuity.lean` and `IntervalChiNegH1ChemDivRepresentative.lean`, so the proof should be copyable from those files.

## Circularity and vacuity checks

* The proposed continuity producer uses only strict positive-time classical regularity and PDE equality. It does not consume bounded-before, full RHS integrability, derivative integrability, or the strict/initial route itself.
* It does not use `IntervalChiNegH1ZeroSlabPhysicalRHS` to claim any zero-start lap representative. The zero-slab file remains a separate bridge for the stronger closed-window route, not for Task88/93’s strict-initial route.
* It does not touch or depend on finite sigma-ladder files.
* It does not hide the physical scalar functions existentially; it keeps Task84’s concrete `H1PhysicalTaxisX`, `H1PhysicalUvxxX`, and `H1PhysicalReactX` exposed.
* It does not assert `H1PhysicalRHSRouteBefore`, because that closed route still includes zero-start component continuity.
