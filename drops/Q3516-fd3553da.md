# Q3516: R4-B identity-frontier audit for the H¹ physical exit

Audit target: `main` at `595354e31017e3dcaa64a2e4daa5b5a7a090dad2`.

## Executive answer

`H1PhysicalRHSIdentityBefore` is **not currently easier** to discharge than the corrected Q3513 zero-window `spatialYoungData` route.

The identity route has a clean wrapper, but it still requires either:

1. the missing mixed gradient-time derivative route, through `hpar : HasDerivAt (H1energy u) (∫ ux*uxt) τ`, plus an exact physical substitution equality; or
2. a Route-C finite-difference derivative, already available up to `u_xx` L¹ continuity, plus a still-missing exact value identity
   `-(∫ u_xx * u_t) = H1IdentityRHSValue` for the concrete physical scalar triple.

By contrast, Q3513's corrected zero-window target is mostly a local integrability/measurability wrapper: add the missing product-measurability assumption, prove `IntervalIntegrable |f*g|` from square integrability plus product measurability, then package the existing spatial Young reducers. That is smaller and less globally entangled.

## 1. Is identity easier than zero-window spatialYoungData/product-meas?

No.

Concrete evidence:

* `IntervalChiNegH1PhysicalRHSScalars.lean` defines the concrete scalars
  `H1PhysicalTaxisX`, `H1PhysicalUvxxX`, and `H1PhysicalReactX`, and its header says the file only packages route data while analytic continuity, bounds, and substitution remain explicit. The identity wrapper is:

  ```lean
  theorem H1PhysicalRHSIdentityBefore_of_parametric_and_substitution
      {p : CM2Params} {T : ℝ}
      {u v : ℝ → intervalDomainPoint → ℝ} {uxt : ℝ → ℝ → ℝ}
      (hpar : ∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
        HasDerivAt (H1energy u)
          (∫ y in (0 : ℝ)..1, ux u τ y * uxt τ y) τ)
      (hsub : ∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
        (∫ y in (0 : ℝ)..1, ux u τ y * uxt τ y) =
          H1IdentityRHSValue p u
            (H1PhysicalTaxisX p u v)
            (H1PhysicalUvxxX p u v)
            (H1PhysicalReactX p u) τ) :
      H1PhysicalRHSIdentityBefore p u v T
  ```

  This is only a wrapper; it does not produce `hpar` or `hsub`.

* `IntervalChiNegH1EnergyCore.lean` has the exact theorem shape that could supply `hpar`:

  ```lean
  theorem H1energy_hasDerivAt_of_gradTime
      {u : ℝ → intervalDomainPoint → ℝ} {τ δ : ℝ}
      {uxt : ℝ → ℝ → ℝ} {bound : ℝ → ℝ}
      ... :
      HasDerivAt (H1energy u)
        (∫ y in (0 : ℝ)..1, ux u τ y * uxt τ y) τ
  ```

  But its assumptions include `hGradDiff`, `hGradBound`, and `hGradEnv`, i.e. a mixed gradient-time derivative field `uxt = ∂ₜ(∂ₓ lift u)` and an integrable envelope. The file header explicitly records this as the carried parabolic frontier: the current classical regularity gives the value time derivative and solution-field continuity, but not the gradient-time field.

* `IntervalChiNegH1EnergyIdentity.lean` has the finite-difference Route-C theorem:

  ```lean
  theorem H1energy_hasDerivAt_of_uxxL1Cont
      (hsol : IsPaper2ClassicalSolution intervalDomain p T u v)
      (hτ : τ ∈ Ioo (0 : ℝ) T)
      (hUxxL1Cont : ∀ ε > 0, ∃ δ > 0,
        ∀ s, |s - τ| < δ → s ∈ Ioo (0 : ℝ) T →
          ∫ x in (0:ℝ)..1,
            ‖liftDeriv2 u s x - liftDeriv2 u τ x‖ ≤ ε) :
      HasDerivAt (H1energy u)
        (-(∫ x in (0:ℝ)..1,
            liftDeriv2 u τ x * liftTimeDeriv u τ x)) τ
  ```

  This avoids `uxt`, and `IntervalChiNegH1ChemDivRepresentative.lean` even proves
  `H1UxxL1ContBefore_of_classical_liftChemotaxisDivPhysicalRep`. However, this Route-C derivative value is not the `∫ ux*uxt` value expected by `H1PhysicalRHSIdentityBefore_of_parametric_and_substitution`. It needs a separate exact equality between `-(∫ u_xx*u_t)` and the concrete physical `H1IdentityRHSValue`.

* `H1EnergyIdentity_of_classicalSolution_and_uxxL1Cont` in `IntervalChiNegH1EnergyIdentity.lean` is not enough for the physical exit. It concludes

  ```lean
  ∃ taxisX uvxx reactX,
    H1EnergyIdentity p u τ taxisX uvxx reactX
  ```

  and the proof chooses bookkeeping witnesses `0, 0, -val + lapL2sq u τ`; it does **not** identify the witnesses with `H1PhysicalTaxisX`, `H1PhysicalUvxxX`, and `H1PhysicalReactX`.

* `IntervalChiNegH1InitialDerivativeRHS.lean` only consumes an existing pointwise identity to build initial-window derivative proxies/integrability. It has theorems such as `H1EnergyDerivativeInitialWindowProxyBefore_of_identityRHSInitialWindow` and `H1EnergyDerivativeInitialWindowIntegrableBefore_of_identityRHSInitialWindow`; none produce `H1PhysicalRHSIdentityBefore`.

So the identity route has more unresolved analytic content than Q3513's corrected wrapper. Q3513 needs a product-measurability-aware integrability helper plus packaging. The identity route still needs a concrete PDE substitution integral identity.

## 3. Smallest non-circular identity producer to add next

The smallest feasible identity-side addition is a **Route-C adapter** that exposes the exact remaining substitution equality. It avoids the missing `uxt` parametric route and uses the already-landed finite-difference H¹ derivative plus the already-landed classical `u_xx` L¹ frontier.

Recommended location: `ShenWork/Paper2/IntervalChiNegH1PhysicalRHSScalars.lean`, after `H1PhysicalRHSIdentityBefore_of_parametric_and_substitution`.

Recommended theorem:

```lean
    {p : CM2Params} {T : ℝ}

          liftDeriv2 u τ x * liftTimeDeriv u τ x) =
        H1IdentityRHSValue p u
          (H1PhysicalTaxisX p u v)
          (H1PhysicalUvxxX p u v)
          (H1PhysicalReactX p u) τ) :

      ∀ s, |s - τ| < δ → s ∈ Set.Ioo (0 : ℝ) T →
```

* The exact `hsub` is a concrete equality, not a vague placeholder: it is precisely the missing integral-level physical PDE substitution for the concrete scalar triple.

## 4. Priority compared with Q3513

Keep Q3513's corrected zero-window route as the next priority.

Reason:

* Q3513's corrected target has a local proof shape: a generic helper
  `IntervalIntegrable |f*g|` from square integrability plus product measurability, then a pure `filter_upwards` packaging theorem for `H1PhysicalRHSComponentSquareSpatialYoungDataBefore`.
* That route does not require exact `pde_u` substitution into the H¹ derivative value, does not involve `uxt`, does not require spectral Parseval/weighted summability, and does not need the final bounded-before package.
* The identity route now has a good adapter theorem to add, but the real analytic missing step remains the exact physical value substitution theorem. That step is more delicate than Q3513 because it crosses PDE pointwise identities, representatives, interval-integral a.e. endpoint handling, and integral algebra.

Recommended next Lean work order:

1. Add the Q3513 generic product-integrability helper in `ShenWork/PDE/GagliardoNirenberg.lean`.

3. Optionally add the small identity adapter `H1PhysicalRHSIdentityBefore_of_classicalSolution_and_uxxUt_substitution` above. This is useful because it makes the remaining identity frontier a single exact equality.
4. Only after that, audit/attempt `H1PhysicalRHSValue_eq_neg_liftDeriv2_mul_liftTimeDeriv_of_classicalSolution`, with explicit attention to product integrability and endpoint-a.e. congruence.
