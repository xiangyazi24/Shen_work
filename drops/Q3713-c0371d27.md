# Q3713 audit: lower resolver-source spatial K2 fields to a coefficient envelope

Audited repo point: `75f730f5db5b916a36f6a30182ea282068707850`
(`Task 259: lower resolver source inputs to joint continuity`).

Relevant current files:

* `ShenWork/Paper2/IntervalResolverSourceWindowInputs.lean`
* `ShenWork/Paper2/IntervalResolverSourceWindowJointInputs.lean`
* `ShenWork/Paper2/IntervalCompactSliceGradientBounds.lean`

Conclusion: the proposed next layer is sound and small.  The existing generic
helpers are strong enough to derive both `hG1` and `hG2` from a **single**
per-compact summable envelope dominating

```lean
unitIntervalCosineEigenvalue n * |bc σ n|
```

for all `σ` in the compact time window.  No `λ_n^2` envelope is needed: the
repo's `unitIntervalCosineEigenvalue n` is already `(nπ)^2`, which controls both
`|nπ sin(nπx)|` and `|(nπ)^2 cos(nπx)|` in the existing helpers.

```lean
/-- Joint-continuity resolver-source inputs with spatial K2 bounds replaced by a
per-compact eigenvalue envelope for the chosen cosine coefficients `bc`. -/
structure ResolverSourceWindowEnvelopeInputs
    (p : CM2Params) {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀) where
  bc : ℝ → ℕ → ℝ
  hbsum : ∀ σ, 0 < σ → σ < D.T →
    Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|)
  hagree : ∀ σ, 0 < σ → σ < D.T →
    Set.EqOn (intervalDomainLift (D.u σ))
      (fun x => ∑' n, bc σ n * cosineMode n x)
      (Set.Icc (0 : ℝ) 1)
  hliftCont :
    ContinuousOn
      (Function.uncurry
        (fun (σ : ℝ) (x : ℝ) => intervalDomainLift (D.u σ) x))
      (Set.Ioo (0 : ℝ) D.T ×ˢ Set.Icc (0 : ℝ) 1)
  /-- On each nonempty compact time window, a summable nonnegative sequence `E`
  dominates `λ_n * |bc σ n|` uniformly in `σ`.  This is exactly what the existing
  generic `grad_series_abs_le` and `grad2_series_abs_le` helpers consume. -/
  henv : ∀ a b, 0 < a → b < D.T → a ≤ b →
    ∃ E : ℕ → ℝ,
      Summable E ∧
      (∀ n, 0 ≤ E n) ∧
      (∀ σ ∈ Set.Icc a b, ∀ n,
        unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n)
  adotPow : ℝ → ℕ → ℝ
  hderivPow : ∀ σ, 0 < σ → σ < D.T → ∀ n,
    HasDerivAt
      (fun r => cosineCoeffs
        (fun x => p.ν * intervalDomainLift (D.u r) x ^ p.γ) n)
      (adotPow σ n) σ
  hadotPowCont : ∀ n, ContinuousOn (fun σ => adotPow σ n) (Set.Ioo 0 D.T)
  hMdotPow : ∀ a b, 0 < a → b < D.T →
    ∃ Mdot, ∀ σ ∈ Set.Icc a b, ∀ n, |adotPow σ n| ≤ Mdot
```

Why keep `hbsum`?  The derivative-transfer identities used at interior points,
namely

```lean
ShenWork.IntervalDuhamelClosedC2.cosineCoeffSeries_grad_hasDerivAt
ShenWork.IntervalDuhamelClosedC2.cosineCoeffSeries_deriv2_eq
```

consume exactly `Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|)`.
The envelope can derive this on singleton windows, but keeping `hbsum` makes this
layer smaller and more likely to compile.  A later tightening can remove `hbsum`
by deriving it from `henv σ σ`.

## 4. Hidden conditions and conservative checks

### Envelope strength

The envelope must dominate

```lean
unitIntervalCosineEigenvalue n * |bc σ n|
```

not `|bc σ n|` alone.  The existing helpers both consume exactly that quantity.

### Nonnegativity of the envelope

`grad_series_abs_le` and `grad2_series_abs_le` do not explicitly require
`∀ n, 0 ≤ E n`; they only need `Summable E` and domination.  But the wrapper proof
needs `0 ≤ ∑' n, E n` for endpoint cases (`abs_zero ≤ ∑ E`).  Therefore include

```lean
(∀ n, 0 ≤ E n)
```

as a field.  One could derive nonnegativity from the domination plus
`0 ≤ λ_n * |bc σ n|` only on nonempty windows, but that would complicate the
empty-window branch and make the layer less robust.

### Empty windows

`ResolverSourceWindowJointInputs.hG1/hG2` quantify over all `a b` with `0 < a` and
`b < D.T`, not just `a ≤ b`.  The envelope field should require `a ≤ b`, and the
producer should handle `¬ a ≤ b` by returning `G1 = 0` / `G2 = 0` with the
`Set.Icc a b` membership contradiction.

### `hagree` is essential

The envelope alone bounds the formal derivative series.  To transfer that bound to
`deriv (intervalDomainLift (D.u σ))` and `deriv (deriv (intervalDomainLift (D.u σ)))`,
the proof still needs

```lean
Set.EqOn (intervalDomainLift (D.u σ))
  (fun x => ∑' n, bc σ n * cosineMode n x)
  (Set.Icc (0 : ℝ) 1)
```

The interior transfer is local on `Ioo 0 1`; endpoints are handled by existing
zero-extension/junk-derivative lemmas.

### `hbsum` is still consumed

The exact derivative-transfer lemmas consume

```lean
Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|)
```

So keep `hbsum` in the first layer.  A later layer can remove it by deriving
`hbsum σ` from `henv σ σ`.

### Non-circularity

This route is non-circular for non-χ0 PPID.  It does not use:

* `GradientMildClassicalFrontierCoreData`;
* `IsPaper2ClassicalSolution`;
* `hpde_u`;
* `HasTimeNeighborhoodSpectralAgreement D.T D.u`.

It only consumes representation/eigenvalue data for the chosen `bc`, joint lift
continuity, and the still-open power-source K1 fields.  The hard non-χ0 work is
therefore isolated in producing `bc/hagree/hbsum/henv/hliftCont` and
`hderivPow/hadotPowCont/hMdotPow` from the actual PPID/B-form Picard-limit
construction, not hidden downstream.

theorem hG1_of_envelope
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {D : GradientMildSolutionData p u₀}
    (bc : ℝ → ℕ → ℝ)
    (hbsum : ∀ σ, 0 < σ → σ < D.T →
      Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|))
    (hagree : ∀ σ, 0 < σ → σ < D.T →
      Set.EqOn (intervalDomainLift (D.u σ))
        (fun x => ∑' n, bc σ n * cosineMode n x)
        (Set.Icc (0 : ℝ) 1))
    (henv : ∀ a b, 0 < a → b < D.T → a ≤ b →
      ∃ E : ℕ → ℝ,
        Summable E ∧
        (∀ n, 0 ≤ E n) ∧
        (∀ σ ∈ Set.Icc a b, ∀ n,
          unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n)) :
    ∀ a b, 0 < a → b < D.T →
      ∃ G1, ∀ σ ∈ Set.Icc a b, ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |deriv (intervalDomainLift (D.u σ)) x| ≤ G1 := by
  intro a b ha hb
  by_cases hab : a ≤ b
  · obtain ⟨E, hEsum, hEnn, hdom⟩ := henv a b ha hb hab
    refine ⟨∑' n, E n, ?_⟩
    intro σ hσ x hx
    have hσpos : 0 < σ := lt_of_lt_of_le ha hσ.1
    have hσT : σ < D.T := lt_of_le_of_lt hσ.2 hb
    have hGnn : 0 ≤ ∑' n, E n := tsum_nonneg hEnn
    have hgrad_bound :
        |∑' n, bc σ n * (-((n : ℝ) * Real.pi) *
            Real.sin ((n : ℝ) * Real.pi * x))| ≤ ∑' n, E n :=
      ShenWork.Paper2.CompactSliceGradientBounds.grad_series_abs_le
        (fun n => bc σ n) E x hEsum (hdom σ hσ)
    rcases eq_or_lt_of_le hx.1 with hx0 | hx0
    · have hval : 0 < intervalDomainLift (D.u σ) 0 := by
        have h0 : (0 : ℝ) ∈ Set.Icc (0 : ℝ) 1 := Set.left_mem_Icc.mpr zero_le_one
        simpa [intervalDomainLift, h0] using
          D.hpos σ hσpos hσT.le ⟨0, h0⟩
      have hnd : ¬ DifferentiableAt ℝ (intervalDomainLift (D.u σ)) x := by
        rw [← hx0]
        exact ShenWork.Paper2.CompactSliceGradientBounds.not_differentiableAt_lift_left
          D.u σ hval
      rw [deriv_zero_of_not_differentiableAt hnd, abs_zero]
      exact hGnn
    · rcases eq_or_lt_of_le hx.2 with hx1 | hx1
      · have hval : 0 < intervalDomainLift (D.u σ) 1 := by
          have h1 : (1 : ℝ) ∈ Set.Icc (0 : ℝ) 1 := Set.right_mem_Icc.mpr zero_le_one
          simpa [intervalDomainLift, h1] using
            D.hpos σ hσpos hσT.le ⟨1, h1⟩
        have hnd : ¬ DifferentiableAt ℝ (intervalDomainLift (D.u σ)) x := by
          rw [hx1]
          exact ShenWork.Paper2.CompactSliceGradientBounds.not_differentiableAt_lift_right
            D.u σ hval
        rw [deriv_zero_of_not_differentiableAt hnd, abs_zero]
        exact hGnn
      · have hxIoo : x ∈ Set.Ioo (0 : ℝ) 1 := ⟨hx0, hx1⟩
        have hEq : intervalDomainLift (D.u σ) =ᶠ[nhds x]
            (fun y => ∑' n, bc σ n * cosineMode n y) := by
          have hmem : Set.Ioo (0 : ℝ) 1 ∈ nhds x := isOpen_Ioo.mem_nhds hxIoo
          filter_upwards [hmem] with y hy
          exact hagree σ hσpos hσT (Set.Ioo_subset_Icc_self hy)
        rw [hEq.deriv_eq,
          (ShenWork.IntervalDuhamelClosedC2.cosineCoeffSeries_grad_hasDerivAt
            (hbsum σ hσpos hσT) x).deriv]
        exact hgrad_bound
  · refine ⟨0, ?_⟩
    intro σ hσ
    exact False.elim (hab (le_trans hσ.1 hσ.2))

## 6. What remains after this layer

This layer reduces only the spatial K2 part.  It does **not** prove the hard
non-χ0 PPID analytic inputs:
