# 4. About `p0 = 2`

`p0 = 2` is admissible exactly under:

```lean
p.γ * (p.N : ℝ) < 2
```

because `rho = 2 * p.γ` gives:

```lean
rho * (p.N : ℝ) / 2 = p.γ * (p.N : ℝ)
```

The Lean proof is:

```lean
have hrewrite :
    (2 * p.γ) * (p.N : ℝ) / 2 = p.γ * (p.N : ℝ) := by
  ring
have hp0 : max (1 : ℝ) ((2 * p.γ) * (p.N : ℝ) / 2) < 2 := by
  rw [hrewrite]
  exact max_lt (by norm_num) hγN_lt_two
```

If the interval specialization has `p.N = 1`, then this is `p.γ < 2`. But the current `Theorem_1_2` critical branch as coded does not expose `p.N = 1` or `p.γ < 2`; it only has `p.m = 1` and `p.χ₀ < chiBeta p`. Therefore the `p0 = 2` seed is not automatically admissible from the critical-branch hypotheses alone.

If `p.γ * p.N ≥ 2`, then the existing L² seed is below the abstract bootstrap threshold. The standard `Corollary_2_1`/`Lemma_2_6` Moser climb cannot be used to manufacture the missing above-threshold seed, because `AbstractLpBootstrapHypothesis` already requires that seed. In that case you need a **separate finite seed-raising lemma**:

```lean
LpPowerBoundedBefore intervalDomain 2 T u →
∀ p0, max 1 ((2*p.γ)*(p.N:ℝ)/2) < p0 →
  LpPowerBoundedBefore intervalDomain p0 T u
```

from direct finite-p energy estimates, finite-horizon smoothing, or an already-available finite-horizon sup bound. I do not see a non-circular committed theorem with exactly this type in the currently inspected files. The committed helper `intervalDomainL2BootstrapSeed_of_L2PowerBoundedBefore` deliberately avoids this by carrying `IntervalDomainBoundednessHyp p`, including `γN < 2`.

# 6. `hcriticalGlobalBound`: exact remaining gap

`hcriticalGlobalBound` is not the same as finite-horizon boundedness. In `Theorem_1_2_intervalDomain_critical_branch_of_corollary21_and_proposition25`, after obtaining a global solution, the theorem constructs:

```lean
hbootstrapAll :
  ∀ T > 0,
    ∃ rho > 0,
      CrossDiffusionBootstrapEstimate intervalDomain p T rho u v ∧
        ∃ p0 > max 1 (rho * (p.N : ℝ) / 2),
          LpPowerBoundedBefore intervalDomain p0 T u
```

and then passes this to:

```lean
hcriticalGlobalBound ... hglobal htrace hbootstrapAll :
  IsPaper2Bounded intervalDomain u
```

The repo also has a wrapper that weakens this to an eventual sup-bound frontier:

```lean
Theorem_1_2_intervalDomain_critical_regime_of_parameter_fields_and_eventual_sup_bound
```

It consumes:

```lean
hcriticalEventualSupBound :
  1 ≤ p.β → p.χ₀ < chiBeta p →
  ∀ u₀, PositiveInitialDatum intervalDomain u₀ →
  ∀ u v,
    IsPaper2GlobalClassicalSolution intervalDomain p u v →
    InitialTrace intervalDomain u₀ u →
    (∀ T > 0,
      ∃ rho > 0,
        CrossDiffusionBootstrapEstimate intervalDomain p T rho u v ∧
          ∃ p0 > max 1 (rho * (p.N : ℝ) / 2),
            LpPowerBoundedBefore intervalDomain p0 T u) →
      ∃ T₀ M, ∀ t, T₀ ≤ t → intervalDomain.supNorm (u t) ≤ M
```

and converts it to `IsPaper2Bounded` using:

```lean
IsPaper2Bounded.of_forall_ge_supNorm_le
```

Thus the exact remaining long-time gap is:

> turn finite-horizon bootstrap-on-each-`T` into a **uniform eventual** sup-norm bound with constants independent of `T`.

`Corollary_2_1` and `Proposition_2_5` only give finite-horizon bounds whose constants may depend on `T`. They do not by themselves prove `IsPaper2Bounded` at `atTop`. So `hcriticalGlobalBound` remains a real dynamical/apriori-long-time frontier unless supplied by `hcriticalEventualSupBound` or a route such as `IntervalDomainMassLpSmoothingAprioriBound.pointwiseBound` with constants uniform enough to choose one eventual `T₀, M`.

# Bottom line

The cross-diffusion estimate is closed. The exact `hcriticalBootstrap` close is now one line through `intervalDomainL2BootstrapSeed_of_L2PowerBoundedBefore`, **provided** the L² seed and `IntervalDomainBoundednessHyp p` are available.

The remaining risks are:

1. `p0 = 2` is only legal under `p.γ * (p.N : ℝ) < 2`; the coded critical branch does not obviously imply that.
2. The L² seed still carries `IntervalDomainL2SeedRegularityFrontier` unless endpoint continuity and zero-time right derivative are already wired.
3. `hcriticalGlobalBound` is a separate eventual-uniform-in-time sup-bound frontier; finite-horizon `Corollary_2_1` + `Proposition_2_5` do not automatically close it.
