# Verdict

For Mathlib **v4.29.1**, the exact fixed-real-exponent chain-rule lemma is

```lean
HasDerivAt.rpow_const
```

and its side condition is the disjunction

```lean
f x ≠ 0 ∨ 1 ≤ p
```

—not simply `0 < f x`.

Consequently:

- for the denominator factor
  \[
  h=(1+R)^{-\beta},
  \]
  use the **left** disjunct, because `1 + R x ≥ 1` implies `1 + R x ≠ 0`;
- for
  \[
  u^{\alpha+1},\qquad \alpha\ge1,
  \]
  use the **right** disjunct `1 ≤ α + 1`, so the first derivative is valid even when `u x = 0`;
- to differentiate that first derivative once more, one differentiates `u^α` and uses the right disjunct `1 ≤ α`; hence the full `C²` logistic tower is also valid at `u x = 0`;
- therefore **strict positivity is not needed for the logistic `C²` source when `α ≥ 1`**. Nonnegativity is needed for identifying the truncated positive-part source with the physical source and for the pointwise bounds, but not for the `rpow_const` derivative tower itself.

There is an important asymmetry:

- `(1+R)^(-β)` has a negative exponent, so it really needs the nonzero denominator hypothesis;
- `u^(α+1)` has exponent at least `2`, so it is `C²` through zero.

# 1. Exact Mathlib declarations in v4.29.1

The scalar theorem in namespace `Real` is:

```lean
theorem Real.hasDerivAt_rpow_const {x p : ℝ}
    (h : x ≠ 0 ∨ 1 ≤ p) :
    HasDerivAt (fun x => x ^ p)
      (p * x ^ (p - 1)) x
```

The composition theorem used by method notation is in the root namespace:

```lean
theorem HasDerivAt.rpow_const
    {f : ℝ → ℝ} {f' x p : ℝ}
    (hf : HasDerivAt f f' x)
    (hx : f x ≠ 0 ∨ 1 ≤ p) :
    HasDerivAt (fun y => f y ^ p)
      (f' * p * f x ^ (p - 1)) x
```

So the exact call is

```lean
hf.rpow_const (p := p) sideCondition
```

or, fully qualified,

```lean
_root_.HasDerivAt.rpow_const hf sideCondition
```

It is **not** named `Real.HasDerivAt.rpow_const`. `HasDerivAt` is a root-level predicate; the uncomposed base theorem is `Real.hasDerivAt_rpow_const`.

The within-set version has the same side condition:

```lean
theorem HasDerivWithinAt.rpow_const
    (hf : HasDerivWithinAt f f' s x)
    (hx : f x ≠ 0 ∨ 1 ≤ p) :
    HasDerivWithinAt (fun y => f y ^ p)
      (f' * p * f x ^ (p - 1)) s x
```

This is the version to use if the endpoint regularity is genuinely one-sided on `Set.Icc 0 1` rather than supplied by a global extension.

Do not confuse this with the variable-exponent theorem

```lean
theorem HasDerivAt.rpow
    (hf : HasDerivAt f f' x)
    (hg : HasDerivAt g g' x)
    (h : 0 < f x) : ...
```

which has the stronger **positive-base** hypothesis. For every expression in this question the exponent is constant, so `rpow_const` is the correct lemma.

Mathlib also provides the direct smoothness theorem

```lean
theorem ContDiff.rpow_const_of_le
    {f : E → ℝ} {m : ℕ} {p : ℝ}
    (hf : ContDiff ℝ m f)
    (h : (m : ℝ) ≤ p) :
    ContDiff ℝ m (fun x => f x ^ p)
```

and the corresponding `ContDiffOn`, `ContDiffAt`, and `ContDiffWithinAt` versions. In particular, with `m = 2` and `p = α + 1`, the condition is

```lean
(2 : ℝ) ≤ α + 1
```

which follows exactly from `1 ≤ α`. This is the shortest qualitative confirmation that `u^(α+1)` is `C²` through zero.

# 2. Denominator tower for `h = (1+R)^(-β)`

Assume at the point `x`:

```lean
hR0 : HasDerivAt R  (R1 x) x
hR1 : HasDerivAt R1 (R2 x) x
hR2 : HasDerivAt R2 (R3 x) x
hden : 1 ≤ 1 + R x
```

Define

```lean
let H : ℝ → ℝ := fun y => (1 + R y) ^ (-β)
let H1 : ℝ → ℝ := fun y =>
  (-β) * (1 + R y) ^ (-β - 1) * R1 y
let H2 : ℝ → ℝ := fun y =>
  β * (β + 1) * (1 + R y) ^ (-β - 2) * (R1 y) ^ 2
    - β * (1 + R y) ^ (-β - 1) * R2 y
```

The first derivative is obtained exactly as follows:

```lean
have hbase : HasDerivAt (fun y => 1 + R y) (R1 x) x := by
  simpa using (hasDerivAt_const x (1 : ℝ)).add hR0

have hbase_pos : 0 < 1 + R x :=
  lt_of_lt_of_le zero_lt_one hden

have hbase_ne : 1 + R x ≠ 0 := hbase_pos.ne'

have hH_raw :
    HasDerivAt (fun y => (1 + R y) ^ (-β))
      (R1 x * (-β) * (1 + R x) ^ ((-β) - 1)) x :=
  hbase.rpow_const (p := -β) (Or.inl hbase_ne)

have hH : HasDerivAt H (H1 x) x := by
  dsimp [H, H1]
  exact hH_raw.congr_deriv (by ring)
```

The side condition passed to `rpow_const` is

```lean
Or.inl hbase_ne
```

because the alternative `1 ≤ -β` is not available in the intended parameter range.

For the second derivative, first differentiate the next negative power:

```lean
have hpow_next :
    HasDerivAt (fun y => (1 + R y) ^ (-β - 1))
      (R1 x * (-β - 1) *
        (1 + R x) ^ ((-β - 1) - 1)) x :=
  hbase.rpow_const (p := -β - 1) (Or.inl hbase_ne)
```

Then use constant multiplication and the product rule:

```lean
have hH1_raw := (hpow_next.const_mul (-β)).mul hR1

have hH1 : HasDerivAt H1 (H2 x) x := by
  dsimp [H1, H2]
  exact hH1_raw.congr_deriv (by ring)
```

The resulting formulas are exactly

\[
H'=-\beta(1+R)^{-\beta-1}R',
\]

and

\[
H''=\beta(\beta+1)(1+R)^{-\beta-2}(R')^2
      -\beta(1+R)^{-\beta-1}R''.
\]

No quotient-rule lemma is necessary after normalizing the denominator to the negative-power form.

# 3. Product towers for `g = R'·h` and `Q = u·g`

Assume additionally

```lean
hu0 : HasDerivAt u  (u1 x) x
hu1 : HasDerivAt u1 (u2 x) x
```

Define the flux core and its first two derivatives:

```lean
let G : ℝ → ℝ := fun y => R1 y * H y
let G1 : ℝ → ℝ := fun y =>
  R2 y * H y + R1 y * H1 y
let G2 : ℝ → ℝ := fun y =>
  R3 y * H y + 2 * R2 y * H1 y + R1 y * H2 y
```

The exact `HasDerivAt.mul`/`add` wiring is:

```lean
have hG : HasDerivAt G (G1 x) x := by
  dsimp [G, G1]
  exact hR1.mul hH

have hG1_raw := (hR2.mul hH).add (hR1.mul hH1)

have hG1 : HasDerivAt G1 (G2 x) x := by
  dsimp [G1, G2]
  exact hG1_raw.congr_deriv (by ring)
```

Thus

\[
g'=R''h+R'h',
\]

\[
g''=R'''h+2R''h'+R'h''.
\]

Now define

```lean
let Q : ℝ → ℝ := fun y => u y * G y
let Q1 : ℝ → ℝ := fun y =>
  u1 y * G y + u y * G1 y
let Q2 : ℝ → ℝ := fun y =>
  u2 y * G y + 2 * u1 y * G1 y + u y * G2 y
```

The outer product tower is:

```lean
have hQ : HasDerivAt Q (Q1 x) x := by
  dsimp [Q, Q1]
  exact hu0.mul hG

have hQ1_raw := (hu1.mul hG).add (hu0.mul hG1)

have hQ1 : HasDerivAt Q1 (Q2 x) x := by
  dsimp [Q1, Q2]
  exact hQ1_raw.congr_deriv (by ring)
```

Therefore the exact second derivative consumed by the algebraic bound is

\[
Q''=u''g+2u'g'+ug''.
\]

Expanding `g`, `g'`, `g''`, `h'`, and `h''` gives the full expression in terms of

```text
u, u1, u2, R1, R2, R3,
(1+R)^(-β), (1+R)^(-β-1), (1+R)^(-β-2).
```

Any scalar chemotaxis coefficient is then added with `.const_mul`; sums and differences use `.add` and `.sub`.

# 4. Logistic tower through `u = 0`

Write the logistic source in the form

\[
S(u)=a\,u-b\,u^{\alpha+1}.
\]

This form makes the zero-base regularity completely transparent.

Define

```lean
let P : ℝ → ℝ := fun y => u y ^ (α + 1)
let P1 : ℝ → ℝ := fun y =>
  u1 y * (α + 1) * u y ^ α
let P2 : ℝ → ℝ := fun y =>
  u2 y * (α + 1) * u y ^ α
    + (u1 y * (α + 1)) *
        (u1 y * α * u y ^ (α - 1))
```

From

```lean
hα : 1 ≤ α
```

obtain

```lean
have hα1 : 1 ≤ α + 1 := by linarith
```

The first derivative of `u^(α+1)` is:

```lean
have hP_raw :
    HasDerivAt (fun y => u y ^ (α + 1))
      (u1 x * (α + 1) *
        u x ^ ((α + 1) - 1)) x :=
  hu0.rpow_const (p := α + 1) (Or.inr hα1)

have hP : HasDerivAt P (P1 x) x := by
  dsimp [P, P1]
  exact hP_raw.congr_deriv (by ring)
```

Crucially, this uses

```lean
Or.inr hα1
```

and therefore remains valid when

```lean
u x = 0.
```

For the second derivative, differentiate `u^α`:

```lean
have hpowα :
    HasDerivAt (fun y => u y ^ α)
      (u1 x * α * u x ^ (α - 1)) x :=
  hu0.rpow_const (p := α) (Or.inr hα)
```

Again, the right disjunct permits `u x = 0`.

Differentiate `P1 = (u1·(α+1))·u^α`:

```lean
have hcoef_u1 :
    HasDerivAt (fun y => u1 y * (α + 1))
      (u2 x * (α + 1)) x :=
  hu1.mul_const (α + 1)

have hP1_raw := hcoef_u1.mul hpowα

have hP1 : HasDerivAt P1 (P2 x) x := by
  dsimp [P1, P2]
  exact hP1_raw.congr_deriv (by ring)
```

Equivalently,

\[
(u^{\alpha+1})''
=(\alpha+1)\alpha\,u^{\alpha-1}(u')^2
 +(\alpha+1)u^\alpha u''.
\]

Now define

```lean
let L : ℝ → ℝ := fun y => a * u y - b * P y
let L1 : ℝ → ℝ := fun y => a * u1 y - b * P1 y
let L2 : ℝ → ℝ := fun y => a * u2 y - b * P2 y
```

Then:

```lean
have hL : HasDerivAt L (L1 x) x := by
  dsimp [L, L1]
  exact (hu0.const_mul a).sub (hP.const_mul b)

have hL1 : HasDerivAt L1 (L2 x) x := by
  dsimp [L1, L2]
  exact (hu1.const_mul a).sub (hP1.const_mul b)
```

After ring normalization,

\[
L''
=-b\alpha(\alpha+1)u^{\alpha-1}(u')^2
 +(a-b(\alpha+1)u^\alpha)u'',
\]

which is the expression consumed by the committed `B_log` algebra bound.

# 5. Exact answer to the zero-base question

The v4.29.1 theorem is:

```lean
hf.rpow_const (p := p) (hx : f x ≠ 0 ∨ 1 ≤ p)
```

Therefore:

## First derivative of `u^(α+1)`

At `u x = 0`, it fires because

```lean
1 ≤ α + 1.
```

The derivative is

```lean
u1 x * (α + 1) * u x ^ α
```

and is zero at the base when `α > 0`.

## Second derivative

To differentiate the first derivative, the only new real-power derivative is that of `u^α`. It fires at `u x = 0` because

```lean
1 ≤ α.
```

Thus **`α ≥ 1` is exactly the threshold needed for the generic `C²` tower of `u^(α+1)` through zero**.

This agrees with the direct smoothness theorem:

```lean
huC2.rpow_const_of_le (m := 2) (p := α + 1) (by exact_mod_cast ...)
```

whose scalar condition is

```lean
(2 : ℝ) ≤ α + 1.
```

At the endpoint case `α = 1`, the factor `u^(α-1)` is `u^0`; Mathlib's formula correctly gives the second derivative of `u²`.

If instead `0 < α < 1`, then `α+1` lies in `(1,2)`: `u^(α+1)` is `C¹` at zero but not generically `C²` there. In that regime a strict floor or special vanishing information would be needed. That is **not** the present parameter regime.

# 6. Why the older strict-positive logistic proof can be weakened

The committed `IntervalLogisticSourceQuantBound.lean` currently uses calls of the form

```lean
hg_has.rpow_const (Or.inl hgx_ne)
```

under `0 < g x`. For `α ≥ 1`, the derivative call itself can instead use

```lean
hg_has.rpow_const (p := α) (Or.inr hα)
```

at zero.

There is, however, another strict-positive dependency in that older proof: it rewrites

```lean
g x * g x ^ (α - 1) = g x ^ α
```

using a nonzero-base `rpow_add_one` lemma. The clean way to remove that artifact is exactly the formulation above:

```lean
a * u - b * u ^ (α + 1)
```

and then differentiate `u^(α+1)` directly. This avoids any nonzero-base rewrite and proves the correct `C²` formula at zero.

Alternatively, retain the original product form but avoid that rewrite until after a zero/nonzero case split. The `u^(α+1)` route is much cleaner.

# 7. Truncated positive-part caveat

The raw truncated definitions contain `positivePart`:

```lean
truncatedLogisticLocal p r =
  r * (p.a - p.b * (positivePart r) ^ p.α)
```

and the truncated flux contains both

```lean
positivePart (lift u)
```

and a resolver driven by

```lean
positivePart ∘ u.
```

One must **not** claim a generic classical `C²` tower for the raw positive-part expressions across a zero of an arbitrary sign-changing function: `positivePart` has a kink at zero.

The correct post-energy route is:

```text
energy gives u ≥ 0 on the slice
→ positivePart u = u on [0,1]
→ the truncated logistic/flux equals the physical logistic/flux on [0,1]
→ differentiate the physical expression using the towers above.
```

At spatial endpoints, if only equality on `Icc 0 1` is known, use the `HasDerivWithinAt` / `ContDiffOn` versions rather than differentiating a zero extension globally. Every `rpow_const`, `mul`, `add`, `sub`, `const_mul`, and `mul_const` step has a corresponding within-set form.

Thus the final circularity audit is:

- chemotaxis denominator tower: needs `1 + R ≠ 0`, supplied unconditionally by `1 + R ≥ 1`;
- logistic `C²` tower: needs only `α ≥ 1`, not `u > 0`;
- `u ≥ 0` is enough to remove the truncation and apply the physical-source tower;
- no Jensen strict-positivity input is needed for the logistic quadratic coefficient decay.

# Bottom line

Use

```lean
HasDerivAt.rpow_const
```

with exact side condition

```lean
f x ≠ 0 ∨ 1 ≤ p.
```

For `(1+R)^(-β)`, supply `Or.inl` from `1+R≥1`. For `u^(α+1)` and the subsequent `u^α` derivative, supply `Or.inr` from `1≤α+1` and `1≤α`. Therefore the logistic second-derivative producer is valid at zeros of `u`; strict positivity is not required.
