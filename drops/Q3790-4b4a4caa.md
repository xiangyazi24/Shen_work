## Why `hgradB` is the wrong operator identity

For a slice `Q`, the true kernel IBP identity is:

```lean
‚ąā‚āď S_N(t) Q(x)
  = -‚ąę‚āÄ¬Ļ Q'(y) * KŐÉ(t,x,y) dy
```

The `B_N` operator currently used by `intervalConjugateKernelOperator` is:

```lean
B_N(t)Q(x) = -‚ąę‚āÄ¬Ļ ‚ąāŠĶß K_N(t,x,y) * Q(y) dy
```

If `Q 0 = Q 1 = 0`, then integration by parts gives roughly

```lean
B_N(t)Q = S_N(t)(Q')
```

which is a **cosine/Neumann** source heat expression.  Meanwhile `‚ąā‚āď S_N(t)Q` is a **sine** heat expression.  They are not the same function.

The endpoint shape exposes the mismatch immediately.  For every source `Q`, the full Neumann semigroup satisfies

```lean
ShenWork.intervalFullSemigroupOperator_deriv_at_zero_eq_zero
ShenWork.intervalFullSemigroupOperator_deriv_at_one_eq_zero
```

so the left side of `hgradB` is forced to be zero at `x = 0` and `x = 1`.  But `intervalConjugateKernelOperator t Q 0` and `intervalConjugateKernelOperator t Q 1` are not generally zero.  The endpoint-zero theorems in `IntervalFullKernelSourceIBP.lean`, such as

```lean
conjugateKernel_integral_at_x_zero
conjugateKernel_integral_at_x_one
```

are for the `KŐÉ` kernel `intervalNeumannConjugateKernel`, **not** for `intervalConjugateKernelOperator`.

## 3. Endpoint / `Icc` vs `Ioo` issues

There are two separate endpoint issues.

### The true `KŐÉ` identity works for all `x`

The theorem

```lean
deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_source_integral_open
```

has conclusion for arbitrary `x : ‚ĄĚ`.  So the true gradient-to-`KŐÉ` source identity has no spatial endpoint problem.  It can support `x ‚ąą Icc 0 1`.

This is because `KŐÉ(t,x,0) = KŐÉ(t,x,1) = 0`, so the source IBP boundary terms vanish without requiring endpoint derivative data for `Q`.

### The requested `hgradB` fails especially at endpoints

The current `hgradB` asks for `x ‚ąą Icc 0 1`.  At `x = 0` and `x = 1`, the gradient semigroup side is zero by Neumann symmetry, but the B-kernel side is not generically zero.  So the `Icc` target is impossible in general.

Restricting `hgradB` to `x ‚ąą Ioo 0 1` would remove the immediate endpoint contradiction, but it would **not** fix the operator mismatch.  Spectrally:

```text
‚ąā‚āď S_N(t) Q          -> sine heat data
B_N(t) Q = -‚ąę‚ąāŠĶßK_N Q -> cosine heat data after IBP when Q endpoints vanish
```

They are different basis outputs.  The correct interior theorem is already the mixed sine/cosine bridge, not `hgradB`.
