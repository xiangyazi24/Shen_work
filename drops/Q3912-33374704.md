## Q1. Is `timeDeriv_rep` correct?

As described, **no, not before proving truncation inactive**.

If the actual object is

```text
truncatedConjugatePicardLimit(t,x) = positivePart(Phi(u)(t,x))
```

while

```text
truncatedPicardCoeff(t,n) = localRestartCoeff ...
```

for the truncated source feeding `Phi`, then the cosine series represents the **pre-positive-part output** `Phi(u)(t,·)`, not the post-truncation function `positivePart(Phi(u)(t,·))`.

Consequently, a field of the form

```text
time derivative of truncatedConjugatePicardLimit
  = Σ truncatedPicardCoeffTimeDeriv n * cos(nπx)
```

is mathematically false in general.  The right-hand side is the derivative of the pre-truncation series.  The derivative of `positivePart(F)` is piecewise:

```text
∂t positivePart(F) = ∂t F     where F > 0,
∂t positivePart(F) = 0        where F < 0,
undefined / needs special proof at F = 0.
```

It becomes correct only after a separate theorem proves that the positive part is inactive on the time window, for example:

```text
0 ≤ Phi(u)(t,x) for all x
```

or, equivalently for the constructed fixed point,

```text
positivePart(Phi(u)(t,x)) = Phi(u)(t,x).
```

Then the post-truncation function equals the pre-truncation function as a function, so its derivative and coefficient reconstruction can be borrowed from the pre-truncation series.

## Q3. Why would differentiability fail only off a countable set?

For a generic `C¹` or `C²` function, it does **not**.  The zero set can be uncountable; Lipschitz functions are differentiable a.e. by Rademacher, but the exceptional set can be uncountable.  Therefore, countability is not a consequence of Lipschitz regularity of `negativePart` or of ordinary spatial `C²` regularity of `u`.

To get “off a countable set,” you need a stronger structural fact, typically real analyticity (or an explicit countability assumption on the zero set).  In one space dimension:

```text
If F is real analytic on an open neighborhood of (0,1), then either
  F ≡ 0 on each connected component,
or
  {x | F(x)=0} is discrete, hence countable on the interval.
```

At points where `F(x) ≠ 0`, `negativePart(F)` is smooth by local sign stability.  At zeroes, the kink of `max(-r,0)` may obstruct differentiability.  If `F ≡ 0` locally, then `negativePart(F)=0` locally and is differentiable there.  The only bad points are nontrivial zeroes, and analyticity makes those isolated/countable.

So the missing theorem for

```lean
test_diff_off_countable
test_deriv_bound
```

is not a generic Lipschitz theorem.  It must be one of:

```lean
-- analytic route
-- theorem countable_nondiff_negativePart_comp_of_realAnalytic
--   (hF_an : AnalyticOn ℝ F (Set.Ioo 0 1)) :
--   Countable {x | ¬ DifferentiableAt ℝ (fun y => negativePart (F y)) x}

-- explicit zero-set route
-- theorem differentiable_negativePart_comp_off_zeroSet
--   (hF_cont : Continuous F) :
--   ∀ x, F x ≠ 0 -> DifferentiableAt ℝ F x ->
--     DifferentiableAt ℝ (fun y => negativePart (F y)) x

-- plus a separate input/proof
-- theorem countable_zeroSet_of_preTruncated_analytic ... :
--   Countable {x | preTruncatedConjugateValue p DT t x = 0}
```

For `test_deriv_bound`, after differentiability is known off the bad set, the scalar bound is easy:

```text
|(negativePart ∘ F)'(x)| ≤ |F'(x)|
```

because the derivative is either `-F'(x)`, `0`, or locally `0` at flat zero points.  But the bound is only meaningful at points where the derivative exists; the countable exceptional set must be supplied first.
