# Q3721 audit: chi-zero power-source K1 reduction after Task268

Audited repo point: `65514df7`.

Relevant current surfaces:

* `IntervalResolverSourceWindowEnvelopeOnlyNoJointInputs.lean`

## Executive answer

Option 1 is **viable as a χ₀ = 0-only adapter**, but only if the adapter carries
(or is separately supplied) the bounded patched logistic source package

```lean
hsrc0 : ShenWork.IntervalPicardLimitRestartBdd.DuhamelSourceBddOn
  (ShenWork.IntervalPicardLimitBddProducer.patchedSource p u₀ D.u) D.T
```

and datum coefficient data (`Continuous u₀` plus `hu₀_bound`, or a small helper
that derives `hu₀_bound` from `PositiveInitialDatum`).

It is **not** a good replacement for the general PPID/non-χ0 surface, and it is
not honest to claim the current `ResolverSourceWindowEnvelopeOnlyNoJointInputs`,
`GradientMildHalfStepLogisticSourceData D`, `HasTimeNeighborhoodSpectralAgreement`,
or `D.u = picardLimit ...` automatically provide `hsrc0`.  They do not.

So the best small Lean reduction is:

1. Add a **core χ0 adapter** file that introduces a no-power input structure
   carrying only `bc/hagree/henv`, then fills
   `ResolverSourceWindowEnvelopeOnlyNoJointInputs` by calling
   `ResolverPowerK1.powerK1_quadruple_of_subtypeCont`.
2. Make `hsrc0` and datum coefficient bound explicit inputs to the adapter, or add
   a very small helper deriving the datum coefficient bound from
   `PositiveInitialDatum`.
3. Only add route-facing χ0 theorem wrappers if the χ0 route already has `hsrc0`
   available or wants to expose it explicitly.  Otherwise this layer is useful but
   too route-specific for the current PPID headline surface.

`GradientMildHalfStepLogisticSourceData D`, and `S` does not imply it.  The

but it requires additional iterate-side data (`Cwin`, `henv_iter`, coefficient

carry `hsrc0` explicitly or be stated only in a context that already has the

`HasTimeNeighborhoodSpectralAgreement D.T D.u` is not needed by
`powerK1_quadruple_of_subtypeCont`; it only supplied `hliftCont` in Task268, which
is not required by the power K1 theorem.  Do not use `Hu` to pretend to build
`hsrc0`.

### Structure

```lean
namespace ShenWork.Paper2.ResolverSourceWindowInput

/-- Resolver-source envelope/no-joint inputs before filling the χ₀=0 power-source
K1 quadruple.  This is just the non-power part: representation and compact
coefficient envelope. -/
structure ResolverSourceWindowChiZeroNoPowerInputs
    (p : CM2Params) {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀) where
  bc : ℝ → ℕ → ℝ
  hagree : ∀ σ, 0 < σ → σ < D.T →
    Set.EqOn (intervalDomainLift (D.u σ))
      (fun x => ∑' n, bc σ n * cosineMode n x)
      (Set.Icc (0 : ℝ) 1)
  henv : ∀ a b, 0 < a → b < D.T → a ≤ b →
    ∃ E : ℕ → ℝ,
      Summable E ∧
      (∀ n, 0 ≤ E n) ∧
      (∀ σ ∈ Set.Icc a b, ∀ n,
        unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n)
```

### Adapter theorem

```lean
/-- In the χ₀=0 route, the existing power-source K1 theorem fills the remaining
power fields of the envelope/no-joint resolver-source input surface. -/
def resolverSourceWindowEnvelopeOnlyNoJointInputs_of_chiZeroNoPowerInputs
    {p : CM2Params} (hχ0 : p.χ₀ = 0)
    {u₀ : intervalDomainPoint → ℝ}
    {D : GradientMildSolutionData p u₀}
    (hα : 1 ≤ p.α) (ha : 0 ≤ p.a) (hb : 0 ≤ p.b)
    (hu₀_cont : Continuous u₀)
    {M₀ : ℝ}
    (hu₀_bound : ∀ k, |cosineCoeffs (intervalDomainLift u₀) k| ≤ M₀)
    (hsrc0 : ShenWork.IntervalPicardLimitRestartBdd.DuhamelSourceBddOn
      (ShenWork.IntervalPicardLimitBddProducer.patchedSource p u₀ D.u) D.T)
    (H : ResolverSourceWindowChiZeroNoPowerInputs p D) :
    ResolverSourceWindowEnvelopeOnlyNoJointInputs p D := by
  let hbsum : ∀ σ, 0 < σ → σ < D.T →
      Summable (fun n => unitIntervalCosineEigenvalue n * |H.bc σ n|) :=
    hbsum_of_envelope H.bc H.henv
  let hG1t : ∀ a' b', 0 < a' → b' < D.T →
      ∃ G1, ∀ σ ∈ Set.Icc a' b', ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |deriv (intervalDomainLift (D.u σ)) x| ≤ G1 :=
    hG1_of_envelope H.bc hbsum H.hagree H.henv
  let hG2t : ∀ a' b', 0 < a' → b' < D.T →
      ∃ G2, ∀ σ ∈ Set.Icc a' b', ∀ x ∈ Set.Icc (0 : ℝ) 1,
        |deriv (deriv (intervalDomainLift (D.u σ))) x| ≤ G2 :=
    hG2_of_envelope H.bc hbsum H.hagree H.henv
  have hfix : ∀ s, 0 < s → s < D.T → ∀ x : ℝ,
      (hx : x ∈ Set.Icc (0 : ℝ) 1) →
        intervalDomainLift (D.u s) x =
          ShenWork.IntervalGradientDuhamelMap.intervalGradientDuhamelMap
            p u₀ D.u s ⟨x, hx⟩ := by
    intro s hs hsT x hx
    simpa [intervalDomainLift, hx] using D.hmild s hs (le_of_lt hsT) ⟨x, hx⟩
  have hpost : ∀ σ, 0 < σ → σ < D.T →
      ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 < intervalDomainLift (D.u σ) x := by
    intro σ hσ hσT x hx
    simp only [intervalDomainLift, dif_pos hx]
    exact D.hpos σ hσ (le_of_lt hσT) ⟨x, hx⟩
  have hubt : ∀ σ, 0 < σ → σ < D.T →
      ∀ x ∈ Set.Icc (0 : ℝ) 1, intervalDomainLift (D.u σ) x ≤ D.M := by
    intro σ hσ hσT x hx
    simp only [intervalDomainLift, dif_pos hx]
    exact le_trans (le_abs_self _) (D.hbound σ hσ (le_of_lt hσT) ⟨x, hx⟩)
  have hLc_ce : ∀ t, 0 < t → t < D.T →
      ∀ s, 0 < s → s ≤ t →
        Continuous (intervalDomainConstExtend (intervalLogisticSource p (D.u s))) := by
    intro t ht htT s hs hst
    exact ShenWork.Paper2.ConstExtendAdapter.logisticSource_constExtend_continuous
      D hs (le_trans hst (le_of_lt htT))
  obtain ⟨hderivPow, hadotPowCont, hMdotPow⟩ :=
    ShenWork.Paper2.ResolverPowerK1.powerK1_quadruple_of_subtypeCont
      (p := p) hχ0 D.u hα ha hb hu₀_cont hu₀_bound hfix hsrc0
      (Msup := D.M) H.bc hbsum H.hagree hpost hubt hG1t hG2t hLc_ce
  exact {
    bc := H.bc
    hagree := H.hagree
    henv := H.henv
    adotPow := ShenWork.Paper2.ResolverPowerK1.adotPowOf p D.u
    hderivPow := hderivPow
    hadotPowCont := hadotPowCont
    hMdotPow := hMdotPow
  }
```

## Optional datum-bound helper

To avoid carrying `hu₀_cont` and `hu₀_bound` explicitly in a route-facing theorem,
add a small helper:

```lean
/-- Positive initial datum supplies bounded cosine coefficients for the lifted
initial slice. -/
theorem exists_initial_cosineCoeff_bound_of_positiveInitialDatum
    {u₀ : intervalDomainPoint → ℝ}
    (hu₀ : PositiveInitialDatum intervalDomain u₀) :
    ∃ M₀ : ℝ, ∀ k, |cosineCoeffs (intervalDomainLift u₀) k| ≤ M₀ := by
  refine ⟨2 * sSup (Set.range fun x : intervalDomainPoint => |u₀ x|), ?_⟩
  -- same proof pattern as the `hu₀_bdF` block in
  -- `IntervalDomainThm11ChiZeroCoreProvider.lean`:
  -- * `hu₀.admissible.1` gives `BddAbove (Set.range fun x => |u₀ x|)`;
  -- * `hu₀.admissible.2` gives subtype continuity;
  -- * transfer to `ContinuousOn (intervalDomainLift u₀) (Icc 0 1)`;
  -- * apply `IntervalMildPicardRegularity.cosineCoeffs_abs_le_of_continuous_bounded`.
```

I would put this helper only if needed by the route wrapper.  It is not necessary
for the core K1 adapter.

## Feasibility verdict

* **Core adapter:** yes, small and likely to compile.  It is a real reduction of
  the χ0 resolver-source input surface: the explicit power fields are replaced by
  one call to the existing theorem.
* **General PPID/non-χ0:** no.  The producer is χ0-specific (`hχ0` and logistic
  restart through `intervalGradientDuhamelMap_eq_of_chi0_zero` internally), so this
  should not be wired into the `χ₀ ≤ 0` PPID route except under the χ0 headline
  specialization.
* **Route-facing wrapper:** useful only if `hsrc0` is already present or acceptable
  as an explicit residual.  `GradientMildHalfStepLogisticSourceData D` and `Hu` do
  not provide `hsrc0`; adding a wrapper that hides this would be misleading.

Therefore I recommend Task269 as the core file
`IntervalResolverSourceWindowChiZeroNoPowerInputs.lean` with the adapter above.
A separate route-facing χ0 wrapper can follow if the current coordinator wants a
headline theorem whose hypothesis no longer mentions the power K1 quadruple fields.
