# Q3974 (cron2): product-rule envelope for `truncatedChemFluxLifted`

The requested theorem, exactly as stated, is not provable from its hypotheses.  In the branch version I inspected, the flux is

```lean
def truncatedChemFluxLifted (p : CM2Params)
    (w : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun y =>
    positivePart (intervalDomainLift w y) * resolverGradReal p w y
      / (1 + intervalDomainLift (intervalNeumannResolverR p w) y) ^ p.β
```

The target theorem only assumes the ball bound and the lift-gradient bound.  It does **not** assume

```lean
|resolverGradReal p w y| ≤ Γ
|resolverLapPhysicalReal p w y| ≤ H
0 ≤ intervalDomainLift (intervalNeumannResolverR p w) y
```

nor any differentiability/derivative identity for `resolverGradReal`, the resolver value, or the positive-part composition.  Since `Γ` and `H` are arbitrary in the stated theorem, the right-hand side can even be negative by choosing `H` very negative.  So the exact statement has to be strengthened.

The right way to organize this in Lean is: first prove a **sorry-free algebraic envelope lemma** from the product-rule identity, then keep all analytic differentiability work in one formula lemma.  This keeps the actual estimate robust.

## 1. Sorry-free algebraic core

This is the part I would actually put next to `truncatedChemFluxLifted`.  It proves the envelope once you have rewritten the derivative into the three product-rule terms.  It does not care how you obtained that formula.

```lean
import ShenWork.Paper2.IntervalBFormNegativePartCron2
import Mathlib.Analysis.SpecialFunctions.Pow.Deriv

open Filter Topology Set
open ShenWork.IntervalDomain (intervalDomainLift intervalDomainPoint)
open ShenWork.PDE (intervalNeumannResolverR)
open ShenWork.Paper2

namespace ShenWork.Paper2.BFormPositiveDatumNegPart

private lemma abs_mul₃_le {a b c A B C : ℝ}
    (ha : |a| ≤ A) (hb : |b| ≤ B) (hc : |c| ≤ C)
    (hA : 0 ≤ A) (hB : 0 ≤ B) :
    |a * b * c| ≤ A * B * C := by
  rw [abs_mul, abs_mul]
  have hab : |a| * |b| ≤ A * B :=
    mul_le_mul ha hb (abs_nonneg b) hA
  exact mul_le_mul hab hc (abs_nonneg c) (mul_nonneg hA hB)

private lemma abs_sq_le_sq_of_abs_le {x A : ℝ} (hx : |x| ≤ A) :
    |x ^ 2| ≤ A ^ 2 := by
  have hx2 : |x| ^ 2 ≤ A ^ 2 := by
    nlinarith [sq_nonneg (A - |x|), abs_nonneg x, hx]
  rw [abs_of_nonneg (sq_nonneg x)]
  rwa [sq_abs] at hx2

/-- Pure three-term envelope.  This is independent of the PDE definitions. -/
private lemma chemFlux_deriv_bound_algebra
    {β M Γ H G uplus uprime grad lap s s₁ : ℝ}
    (hβ : 0 ≤ β) (hM : 0 ≤ M)
    (huplus : |uplus| ≤ M)
    (huprime : |uprime| ≤ G)
    (hgrad : |grad| ≤ Γ)
    (hlap : |lap| ≤ H)
    (hs : |s| ≤ 1)
    (hs₁ : |s₁| ≤ 1) :
    |uprime * grad * s + uplus * lap * s - β * uplus * grad ^ 2 * s₁|
      ≤ (M * H + β * M * Γ ^ 2) + Γ * G := by
  have hGnn : 0 ≤ G := le_trans (abs_nonneg uprime) huprime
  have hΓnn : 0 ≤ Γ := le_trans (abs_nonneg grad) hgrad
  have hHnn : 0 ≤ H := le_trans (abs_nonneg lap) hlap

  let A : ℝ := uprime * grad * s
  let B : ℝ := uplus * lap * s
  let C : ℝ := β * uplus * grad ^ 2 * s₁

  have htri : |A + B - C| ≤ |A| + |B| + |C| := by
    calc
      |A + B - C| = |A + B + (-C)| := by ring
      _ ≤ |A + B| + |-C| := abs_add_le _ _
      _ = |A + B| + |C| := by rw [abs_neg]
      _ ≤ (|A| + |B|) + |C| := by
        exact add_le_add_right (abs_add_le A B) |C|
      _ = |A| + |B| + |C| := by ring

  have hA : |A| ≤ Γ * G := by
    dsimp [A]
    have h := abs_mul₃_le (a := uprime) (b := grad) (c := s)
      (A := G) (B := Γ) (C := 1) huprime hgrad hs hGnn hΓnn
    simpa [mul_comm, mul_left_comm, mul_assoc] using h

  have hB : |B| ≤ M * H := by
    dsimp [B]
    have h := abs_mul₃_le (a := uplus) (b := lap) (c := s)
      (A := M) (B := H) (C := 1) huplus hlap hs hM hHnn
    simpa [mul_comm, mul_left_comm, mul_assoc] using h

  have hβu : |β * uplus| ≤ β * M := by
    rw [abs_mul, abs_of_nonneg hβ]
    exact mul_le_mul_of_nonneg_left huplus hβ
  have hgrad_sq : |grad ^ 2| ≤ Γ ^ 2 :=
    abs_sq_le_sq_of_abs_le hgrad
  have hβMnn : 0 ≤ β * M := mul_nonneg hβ hM
  have hΓsqnn : 0 ≤ Γ ^ 2 := sq_nonneg Γ

  have hC : |C| ≤ β * M * Γ ^ 2 := by
    dsimp [C]
    have h := abs_mul₃_le (a := β * uplus) (b := grad ^ 2) (c := s₁)
      (A := β * M) (B := Γ ^ 2) (C := 1)
      hβu hgrad_sq hs₁ hβMnn hΓsqnn
    simpa [mul_comm, mul_left_comm, mul_assoc] using h

  calc
    |uprime * grad * s + uplus * lap * s - β * uplus * grad ^ 2 * s₁|
        = |A + B - C| := rfl
    _ ≤ |A| + |B| + |C| := htri
    _ ≤ Γ * G + M * H + β * M * Γ ^ 2 := by linarith
    _ = (M * H + β * M * Γ ^ 2) + Γ * G := by ring

/-- Envelope after a product-rule derivative formula has already been proved. -/
private theorem truncatedChemFluxLifted_deriv_abs_le_of_formula
    (p : CM2Params) {w : intervalDomainPoint → ℝ}
    {M Γ H G : ℝ} (hM : 0 < M) (y : ℝ)
    {uplus uprime grad lap s s₁ : ℝ}
    (hformula :
      deriv (truncatedChemFluxLifted p w) y =
        uprime * grad * s + uplus * lap * s
          - p.β * uplus * grad ^ 2 * s₁)
    (huplus : |uplus| ≤ M)
    (huprime : |uprime| ≤ G)
    (hgrad : |grad| ≤ Γ)
    (hlap : |lap| ≤ H)
    (hs : |s| ≤ 1)
    (hs₁ : |s₁| ≤ 1) :
    |deriv (truncatedChemFluxLifted p w) y| ≤
      (M * H + p.β * M * Γ ^ 2) + Γ * G := by
  rw [hformula]
  exact chemFlux_deriv_bound_algebra
    (β := p.β) (M := M) (Γ := Γ) (H := H) (G := G)
    (uplus := uplus) (uprime := uprime) (grad := grad) (lap := lap)
    (s := s) (s₁ := s₁)
    p.hβ hM.le huplus huprime hgrad hlap hs hs₁
```

The intended instantiation is

```lean
uplus  := positivePart (intervalDomainLift w y)
uprime := deriv (fun z => positivePart (intervalDomainLift w z)) y
grad   := resolverGradReal p w y
lap    := resolverLapPhysicalReal p w y
R      := intervalDomainLift (intervalNeumannResolverR p w) y

s₁     := (1 + R) ^ (-p.β - 1)
```

or, if you prefer literal quotients, use

```lean
s  := ((1 + R) ^ p.β)⁻¹
s₁ := ((1 + R) ^ (p.β + 1))⁻¹
```

The negative-power version is usually easier in Lean, because the derivative of the denominator factor is exactly the `rpow` chain rule.

## 2. Product-rule formalization

Do not prove the derivative of `a * b / c` directly.  Rewrite the quotient as multiplication by a negative real power:

```lean
U z := positivePart (intervalDomainLift w z)
R z := intervalDomainLift (intervalNeumannResolverR p w) z
Q z := resolverGradReal p w z
S z := (1 + R z) ^ (-p.β)
```

Then the flux is eventually/pointwise equal to `fun z => U z * Q z * S z`, provided `1 + R z ≠ 0` in the neighborhood where you rewrite using `Real.rpow_neg`.  At the point this follows from `0 ≤ R y`; locally it follows from continuity of `R`.

The hypotheses you want for the clean product rule are:

```lean
(hU : HasDerivAt U Up y)
(hR : HasDerivAt R (Q y) y)
(hQ : HasDerivAt Q Lap y)
(hRnn : 0 ≤ R y)
```

Then:

```lean
have hbase : 0 < 1 + R y := by linarith
have hpow :
    HasDerivAt (fun b : ℝ => b ^ (-p.β))
      (-p.β * (1 + R y) ^ (-p.β - 1)) (1 + R y) :=
  Real.hasDerivAt_rpow_const (Or.inl (ne_of_gt hbase))

have hinner : HasDerivAt (fun z : ℝ => 1 + R z) (Q y) y := by
  simpa using (hasDerivAt_const y (1 : ℝ)).add hR

have hS :
    HasDerivAt S
      ((-p.β * (1 + R y) ^ (-p.β - 1)) * Q y) y := by
  simpa [S] using hpow.comp y hinner

have hprod :
    HasDerivAt (fun z : ℝ => U z * Q z * S z)
      (Up * Q y * S y
        + U y * Lap * S y
        - p.β * U y * (Q y) ^ 2 * (1 + R y) ^ (-p.β - 1)) y := by
  have h := (hU.mul hQ).mul hS
  -- `h` has the same derivative, but with a less convenient associativity shape.
  -- `ring_nf` or `ring` usually closes the scalar normalization.
  convert h using 1 <;> ring
```

From this `HasDerivAt`, get the `deriv` identity by

```lean
have hformula := hprod.deriv
```

and feed it to `truncatedChemFluxLifted_deriv_abs_le_of_formula`.

## 3. Positive part derivative

There is no globally valid derivative formula

```lean
HasDerivAt (fun x => max (f x) 0) ... x
```

without a kink side condition.  At `f x = 0`, `max (f,0)` is generally not differentiable unless the incoming derivative is `0` or you are proving a one-sided/a.e. statement.  For the classical product-rule proof, split away from the kink.

For the two non-kink cases, local equality is the most reliable method:

```lean
private lemma positivePart_comp_hasDerivAt_of_pos
    {f : ℝ → ℝ} {f' y : ℝ}
    (hf : HasDerivAt f f' y) (hy : 0 < f y) :
    HasDerivAt (fun z => positivePart (f z)) f' y := by
  have hnhds : Set.Ioi (0 : ℝ) ∈ 𝓝 (f y) :=
    isOpen_Ioi.mem_nhds hy
  have hev : (fun z => positivePart (f z)) =ᶠ[𝓝 y] f := by
    filter_upwards [hf.continuousAt hnhds] with z hz
    exact positivePart_eq_self_of_nonneg hz.le
  exact hf.congr_of_eventuallyEq hev.symm

private lemma positivePart_comp_hasDerivAt_of_neg
    {f : ℝ → ℝ} {f' y : ℝ}
    (hf : HasDerivAt f f' y) (hy : f y < 0) :
    HasDerivAt (fun z => positivePart (f z)) 0 y := by
  have hnhds : Set.Iio (0 : ℝ) ∈ 𝓝 (f y) :=
    isOpen_Iio.mem_nhds hy
  have hev : (fun z => positivePart (f z)) =ᶠ[𝓝 y] (fun _ : ℝ => 0) := by
    filter_upwards [hf.continuousAt hnhds] with z hz
    exact positivePart_eq_zero_of_nonpos hz.le
  exact (hasDerivAt_const y (0 : ℝ)).congr_of_eventuallyEq hev.symm
```

At the kink, one of these must be true:

* you exclude it with a hypothesis such as `intervalDomainLift w y ≠ 0`;
* you add `deriv (intervalDomainLift w) y = 0` and prove the max derivative is `0`;
* you avoid classical pointwise differentiability and prove an a.e./weak derivative estimate.

For the bound alone, once `hU` is available, you can get

```lean
|Up| ≤ G
```

from either `Up = deriv (intervalDomainLift w) y` in the positive case, or `Up = 0` in the negative case.  In an all-points theorem using Lean's `deriv`, if the flux is not differentiable at the kink then `deriv` is definitionally forced by `deriv_zero_of_not_differentiableAt`, but that is not a product-rule proof and should be handled as a separate case.

## 4. Denominator bounds

For the negative-power setup, assuming `0 ≤ R`, the two denominator factors satisfy:

```lean
have hbase_ge : (1 : ℝ) ≤ 1 + R := by linarith
have hs : |(1 + R) ^ (-p.β)| ≤ 1 := by
  rw [abs_of_nonneg (Real.rpow_nonneg (by linarith) _)]
  exact Real.rpow_le_one_of_one_le_of_nonpos hbase_ge (by linarith [p.hβ])

have hs₁ : |(1 + R) ^ (-p.β - 1)| ≤ 1 := by
  rw [abs_of_nonneg (Real.rpow_nonneg (by linarith) _)]
  exact Real.rpow_le_one_of_one_le_of_nonpos hbase_ge (by linarith [p.hβ])
```

This is cleaner than repeatedly proving `(1 + R)^β ≥ 1` and then bounding the reciprocal.

## 5. Bounds from the ball hypothesis

The ball bound gives the positive-part factor bound by a small helper:

```lean
private lemma abs_positivePart_le_abs (r : ℝ) : |positivePart r| ≤ |r| := by
  by_cases hr : 0 ≤ r
  · simp [positivePart, hr]
  · have hrle : r ≤ 0 := le_of_not_ge hr
    simp [positivePart, hrle, abs_of_nonpos hrle]
```

Then prove

```lean
|positivePart (intervalDomainLift w y)| ≤ M
```

by first proving `|intervalDomainLift w y| ≤ M`.  If `y ∈ Icc 0 1`, this is `hball ⟨y, hy⟩`; if `y ∉ Icc 0 1`, unfold `intervalDomainLift`, where the lift is zero, and use `0 ≤ M` from `hM.le`.

## 6. Recommended final theorem shape

A realistic final theorem should look like this.  The only `sorry`s below are the analytic bridge lemmas that are not supplied by the current statement.

```lean
private theorem truncatedChemFluxLifted_deriv_abs_le_of_ball_grad_fixed
    (p : CM2Params) {w : intervalDomainPoint → ℝ}
    {M Γ H G : ℝ}
    (hM : 0 < M)
    (hball : ∀ x : intervalDomainPoint, |w x| ≤ M)
    (hgrad_lift : ∀ x : ℝ, |deriv (intervalDomainLift w) x| ≤ G)
    (hΓ : |resolverGradReal p w y| ≤ Γ)
    -- Use the actual branch/import name for the Laplacian bridge.  In the bridge
    -- file I would expect this to be `resolverLapPhysicalReal p w y`.
    (hH : |resolverLapPhysicalReal p w y| ≤ H)
    (hRnn : 0 ≤ intervalDomainLift (intervalNeumannResolverR p w) y)
    -- Product-rule derivative formula, obtained from the HasDerivAt lemmas above.
    (hformula :
      deriv (truncatedChemFluxLifted p w) y =
        deriv (fun z => positivePart (intervalDomainLift w z)) y
          * resolverGradReal p w y
          * (1 + intervalDomainLift (intervalNeumannResolverR p w) y) ^ (-p.β)
        + positivePart (intervalDomainLift w y)
          * resolverLapPhysicalReal p w y
          * (1 + intervalDomainLift (intervalNeumannResolverR p w) y) ^ (-p.β)
        - p.β * positivePart (intervalDomainLift w y)
          * (resolverGradReal p w y) ^ 2
          * (1 + intervalDomainLift (intervalNeumannResolverR p w) y) ^ (-p.β - 1))
    (y : ℝ) :
    |deriv (truncatedChemFluxLifted p w) y| ≤
      (M * H + p.β * M * Γ ^ 2) + Γ * G := by
  let R : ℝ := intervalDomainLift (intervalNeumannResolverR p w) y

  have hMnn : 0 ≤ M := hM.le

  have huplus : |positivePart (intervalDomainLift w y)| ≤ M := by
    have hlift : |intervalDomainLift w y| ≤ M := by
      by_cases hy : y ∈ Set.Icc (0 : ℝ) 1
      · simpa [intervalDomainLift, hy] using hball ⟨y, hy⟩
      · simp [intervalDomainLift, hy, hMnn]
    exact (abs_positivePart_le_abs (intervalDomainLift w y)).trans hlift

  have huprime :
      |deriv (fun z => positivePart (intervalDomainLift w z)) y| ≤ G := by
    -- Split on the sign of `intervalDomainLift w y`.
    -- Positive case: derivative equals `deriv (intervalDomainLift w) y`.
    -- Negative case: derivative equals `0`.
    -- Zero/kink case needs one of the extra strategies described above.
    sorry

  have hbase_ge : (1 : ℝ) ≤ 1 + R := by
    dsimp [R]
    linarith [hRnn]

  have hs : |(1 + R) ^ (-p.β)| ≤ 1 := by
    rw [abs_of_nonneg (Real.rpow_nonneg (by linarith) _)]
    exact Real.rpow_le_one_of_one_le_of_nonpos hbase_ge (by linarith [p.hβ])

  have hs₁ : |(1 + R) ^ (-p.β - 1)| ≤ 1 := by
    rw [abs_of_nonneg (Real.rpow_nonneg (by linarith) _)]
    exact Real.rpow_le_one_of_one_le_of_nonpos hbase_ge (by linarith [p.hβ])

  refine truncatedChemFluxLifted_deriv_abs_le_of_formula
    (p := p) (w := w) (M := M) (Γ := Γ) (H := H) (G := G)
    hM y
    (uplus := positivePart (intervalDomainLift w y))
    (uprime := deriv (fun z => positivePart (intervalDomainLift w z)) y)
    (grad := resolverGradReal p w y)
    (lap := resolverLapPhysicalReal p w y)
    (s := (1 + R) ^ (-p.β))
    (s₁ := (1 + R) ^ (-p.β - 1)) ?_ huplus huprime hΓ hH hs hs₁
  simpa [R] using hformula
```

So the best answer to question 4 is: yes, prove the **estimate** without repeating the product rule, but no, you cannot prove a bound on `deriv (truncatedChemFluxLifted p w)` from triangle inequalities on the original expression alone.  You need either a product-rule derivative formula, or you need to switch the theorem to an a.e./weak-derivative formulation where the positive-part kink is handled by Lipschitz/chain-rule infrastructure.
