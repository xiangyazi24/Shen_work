# Source-side zero-start H1 trace reducer audit

Audited the source-side zero-start H1 route around main commit `8bfcf580`.  This audit intentionally ignores downstream bounded-before, sqrt, physical RHS, and physical exit wrappers.

## Short verdict

Pointwise right limits in time are **not sufficient** to glue strict positive-time joint continuity into closed-slab continuity on `[0,b] × [0,1]`.

The clean next source-side theorem should use a **zero-face joint continuity hypothesis**:

```lean
ContinuousWithinAt F (Set.Icc 0 b ×ˢ Set.Icc 0 1) (0, x)
```

for every `x ∈ Set.Icc 0 1`, plus strict-time joint continuity away from zero.  This is mathematically exact, Lean-friendly, and avoids pretending that `InitialTrace` or pointwise vertical limits give derivative traces.

The actual B-form/Picard zero-face C1 trace remains a frontier; the topology gluing reducer itself is feasible to code now.

## Why pointwise right limits are insufficient

Even with strict joint continuity on `(0,b] × [0,1]`, continuous zero-face function `F0`, and equality `F(0,x)=F0 x`, pointwise vertical limits

```lean
∀ x ∈ Set.Icc 0 1, Tendsto (fun t => F (t,x)) (𝓝[>] 0) (𝓝 (F0 x))
```

do not imply joint continuity at the zero face.

Counterexample on the quadrant:

```text
F(t,x) = t*x / (t^2 + x^2)     for t > 0
F(0,x) = 0
F0(x)  = 0
```

For every fixed `x`, `F(t,x) → 0` as `t → 0+`, and `F` is smooth for `t > 0`.  But along `x = t`, `F(t,t) = 1/2`, so `F` is not continuous at `(0,0)`.  Thus a pointwise right-limit API would be too weak and would encode a false source reducer.

A uniform zero-time convergence statement plus continuity of `F0` would be sufficient, for example:

```lean
∀ x₀ ∈ Set.Icc (0 : ℝ) 1, ∀ ε > 0,
  ∃ δ > 0, ∀ t x,
    0 ≤ t → t < δ → x ∈ Set.Icc (0 : ℝ) 1 → |x - x₀| < δ →
      |F (t, x) - F0 x₀| < ε
```

But this is essentially `ContinuousWithinAt F (Set.Icc 0 b ×ˢ Set.Icc 0 1) (0,x₀)` written by hand.  The direct `ContinuousWithinAt` hypothesis is cleaner and exactly matches the target.

### Theorem statement

Use strict continuity on `Ioo 0 T`, not merely `Ioo 0 b`.  The closed slab includes the top face `t=b`; if `hstrict` were only on `Ioo 0 b`, it would not give continuity at `t=b`.  Existing classical/P3Moser data are naturally on `Ioo 0 T`, and callers have `b < T`.

```lean
import ShenWork.Paper2.IntervalChiNegH1ZeroStartInitializedPrimitive
import ShenWork.PDE.P3MoserDxJointContinuity

open MeasureTheory Set
open scoped BigOperators Topology Interval

open ShenWork.IntervalDomain
open ShenWork.Paper2
open ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents

noncomputable section

namespace ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents

/-- Glue strict-positive-time joint continuity with explicit zero-face continuity
into closed-zero-slab continuity.

The hypothesis `hzero` is intentionally `ContinuousWithinAt` for the full slab at
`(0,x)`.  Pointwise right limits in `t` are not enough for joint continuity. -/
theorem continuousOn_zeroClosedSlab_of_strictTime_and_zeroFace
    {F : ℝ × ℝ → ℝ} {T b : ℝ}
    (hb0 : 0 ≤ b) (hbT : b < T)
    (hstrict : ContinuousOn F
      (Set.Ioo (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1))
    (hzero : ∀ x ∈ Set.Icc (0 : ℝ) 1,
      ContinuousWithinAt F
        (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0, x)) :
    ContinuousOn F (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) := by
  intro z hz
  rcases z with ⟨t, x⟩
  rcases hz with ⟨htb, hx⟩
  by_cases ht0 : t = 0
  · subst t
    exact hzero x hx
  · have htpos : 0 < t := lt_of_le_of_ne htb.1 (Ne.symm ht0)
    have hzStrict : (t, x) ∈ Set.Ioo (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1 :=
      ⟨⟨htpos, lt_of_le_of_lt htb.2 hbT⟩, hx⟩
    -- Around a positive-time point, the closed zero slab is locally contained in
    -- the strict-time domain `Ioo 0 T × Icc 0 1`.
    have hposN : Set.Ioi (0 : ℝ) ×ˢ (Set.univ : Set ℝ) ∈ 𝓝 (t, x) :=
      (isOpen_Ioi.prod isOpen_univ).mem_nhds ⟨htpos, trivial⟩
    have hlocal :
        (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) ∩
            (Set.Ioi (0 : ℝ) ×ˢ (Set.univ : Set ℝ))
          ⊆ Set.Ioo (0 : ℝ) T ×ˢ Set.Icc (0 : ℝ) 1 := by
      rintro ⟨s, y⟩ ⟨⟨hsb, hy⟩, hspos, _⟩
      exact ⟨⟨hspos, lt_of_le_of_lt hsb.2 hbT⟩, hy⟩
    refine ContinuousWithinAt.mono_of_mem_nhdsWithin ?_ ?_
    · exact (hstrict (t, x) hzStrict).mono hlocal
    · exact Filter.inter_mem self_mem_nhdsWithin
        (mem_nhdsWithin_of_mem_nhds hposN)

end ShenWork.Paper2.IntervalChiNegH1ZeroStartComponents
```

This theorem is feasible to code now.  The proof pattern matches local gluing already used in `IntervalDomainPiecewiseClassical.lean`: establish a positive-time neighborhood, restrict to the strict domain, then use `ContinuousWithinAt.mono_of_mem_nhdsWithin`.

/-- Zero-face C1/sign traces for an already-initialized primitive pair.
This is not a B-form/Picard producer; it is the exact remaining source input
needed to close zero-start slabs from strict-time classical data. -/
structure H1ZeroStartPrimitiveC1ZeroFaceTrace
    (u v : ℝ → intervalDomainPoint → ℝ) (T : ℝ) : Prop where

    ContinuousWithinAt

        (fun (t : ℝ) (x : ℝ) => deriv (intervalDomainLift (v t)) x))
      (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0, x)

theorem H1ZeroStartInitializedPrimitiveC1SignSource_of_classical_zeroFace

    H1ZeroStartInitializedPrimitiveC1SignSource u₀ v₀ u v T := by
  refine
    { u_zero := hu0
      v_zero := hv0
      u_cont0 := ?_
      v_cont0 := ?_
      ux_cont0 := ?_
      vx_cont0 := ?_
      u_pos0 := ?_

  · intro b hb0 hbT
    have hreg := hsol.regularity

      hreg.2.2.2.2.2.2.1

  · intro b hb0 hbT

## What remains frontier

A true general initialized B-form/Picard source producer would still need an actual paired initialized construction, for example morally:

```lean
uZ t := if t ≤ 0 then u₀ else conjugatePicardLimit p u₀ T t
vZ t := if t ≤ 0 then intervalNeumannResolverR p u₀
        else mildChemicalConcentration p (conjugatePicardLimit p u₀ T) t
```

and then proofs of the zero-face trace fields for `uZ`, `vZ`, `deriv (lift uZ)`, and `deriv (lift vZ)`.  The value-only `InitialTrace` and `patchedSlice` sup-norm continuity are not enough, especially for `ux_zeroFace` and `vx_zeroFace`.

So the recommendation is:

1. **Code now:** the topology lemma, the zero-face trace record, and the reducer from strict-time classical data plus zero-face traces.
2. **Document as frontier:** the B-form/Picard construction of the initialized pair and the actual zero-face C1 trace proofs.
3. **Do not use:** downstream H1 bounded-before, sqrt, RHS, physical reaction-bound, or physical exit outputs as source assumptions.
