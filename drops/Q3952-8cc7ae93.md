# Q3952: operator-level IBP for the Neumann conjugate kernel operator

## Bottom line

The cleanest Lean route is **not** to prove integration by parts directly for
`MeasureTheory.integral` against `intervalMeasure 1`.  Keep the existing
operator definitions, but prove the analytic identity as an `intervalIntegral`
statement on `0..1`, then rewrite the two operator integrals to interval
integrals using the existing `Icc`/`Ioc` bridge.

Concretely, for

```lean
K y = intervalNeumannFullKernel t x y
K' y = deriv (fun y' => intervalNeumannFullKernel t x y') y
```

Mathlib gives

```lean
∫ y in (0:ℝ)..1, K y * deriv Q y
  = K 1 * Q 1 - K 0 * Q 0 - ∫ y in (0:ℝ)..1, K' y * Q y
```

and the endpoint assumptions `Q 0 = 0`, `Q 1 = 0` reduce this to

```lean
∫ y in (0:ℝ)..1, K y * deriv Q y
  = - ∫ y in (0:ℝ)..1, K' y * Q y
```

which is exactly

```lean
intervalConjugateKernelOperator t Q x
  = intervalFullSemigroupOperator t (deriv Q) x
```

after rewriting both sides from `intervalMeasure 1` to `∫ y in 0..1, ...`.

The sign is correct: the leading minus in
`intervalConjugateKernelOperator` is precisely the minus produced when the
`y`-derivative is moved from the kernel onto `Q`.

## The `intervalMeasure` bridge

In this repo, `intervalMeasure 1` is `volume.restrict (Set.Icc 0 1)`, whereas
`intervalIntegral.integral_of_le` rewrites `∫ y in 0..1, f y` as integration
against `volume.restrict (Set.Ioc 0 1)`.  The endpoints are Lebesgue-null, so
use `MeasureTheory.integral_Icc_eq_integral_Ioc`.

This is the same pattern already used in the repo, for example in
`IntervalFullKernelInterchange.lean` and `IntervalFullKernelSourceIBP.lean`.
Package it once:

```lean
import ShenWork.Paper2.IntervalConjugateDuhamelMap
import Mathlib.MeasureTheory.Integral.IntervalIntegral.IntegrationByParts

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.IntervalConjugateDuhamelMap

open ShenWork.IntervalDomain
open ShenWork.IntervalNeumannFullKernel

/-- Convert the concrete interval measure on `[0,1]` to a Mathlib interval integral. -/
lemma intervalMeasure_one_integral_eq_intervalIntegral (f : ℝ → ℝ) :
    (∫ y, f y ∂ intervalMeasure 1) = ∫ y in (0 : ℝ)..1, f y := by
  rw [intervalMeasure, intervalSet]
  change (∫ y in Set.Icc (0 : ℝ) 1, f y ∂ MeasureTheory.volume) =
      ∫ y in (0 : ℝ)..1, f y
  rw [intervalIntegral.integral_of_le (by norm_num : (0 : ℝ) ≤ 1),
    ← MeasureTheory.integral_Icc_eq_integral_Ioc]

end ShenWork.IntervalConjugateDuhamelMap
```

The direction above is intentionally the direction needed after unfolding the
operators:

```lean
rw [intervalMeasure_one_integral_eq_intervalIntegral
  (f := fun y : ℝ => deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y * Q y)]
rw [intervalMeasure_one_integral_eq_intervalIntegral
  (f := fun y : ℝ => intervalNeumannFullKernel t x y * deriv Q y)]
```

No integrability hypothesis is needed for this conversion lemma: it is just
changing the restricting set from `Icc` to `Ioc`, which differs by a null
endpoint.

## Recommended theorem shape

For the existing Mathlib IBP lemma
`intervalIntegral.integral_mul_deriv_eq_deriv_mul_of_hasDerivAt`, the most
convenient hypotheses are not merely “Lipschitz” but the exact data the lemma
consumes:

```lean
(hQ_cont : ContinuousOn Q (Set.uIcc (0 : ℝ) 1))
(hQ_deriv : ∀ y ∈ Set.Ioo (0 : ℝ) 1, HasDerivAt Q (deriv Q y) y)
(hQ_deriv_int : IntervalIntegrable (fun y : ℝ => deriv Q y) MeasureTheory.volume 0 1)
(hQ0 : Q 0 = 0) (hQ1 : Q 1 = 0)
```

For the kernel side, once `ht : 0 < t`, the repo already has the required facts:

```lean
continuousOn_intervalNeumannFullKernel_snd ht x
hasDerivAt_intervalNeumannFullKernel_snd ht x y
intervalIntegrable_deriv_intervalNeumannFullKernel_snd ht x
```

A useful version specialized to `deriv Q` is:

```lean
import ShenWork.Paper2.IntervalConjugateDuhamelMap
import Mathlib.MeasureTheory.Integral.IntervalIntegral.IntegrationByParts

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.IntervalConjugateDuhamelMap

open ShenWork.IntervalDomain
open ShenWork.IntervalNeumannFullKernel

lemma intervalMeasure_one_integral_eq_intervalIntegral (f : ℝ → ℝ) :
    (∫ y, f y ∂ intervalMeasure 1) = ∫ y in (0 : ℝ)..1, f y := by
  rw [intervalMeasure, intervalSet]
  change (∫ y in Set.Icc (0 : ℝ) 1, f y ∂ MeasureTheory.volume) =
      ∫ y in (0 : ℝ)..1, f y
  rw [intervalIntegral.integral_of_le (by norm_num : (0 : ℝ) ≤ 1),
    ← MeasureTheory.integral_Icc_eq_integral_Ioc]

/-- Operator-level integration by parts for the full Neumann kernel in the `y` variable. -/
theorem intervalConjugateKernelOperator_eq_intervalFullSemigroupOperator_deriv
    {t x : ℝ} (ht : 0 < t) {Q : ℝ → ℝ}
    (hQ_cont : ContinuousOn Q (Set.uIcc (0 : ℝ) 1))
    (hQ_deriv : ∀ y ∈ Set.Ioo (0 : ℝ) 1, HasDerivAt Q (deriv Q y) y)
    (hQ_deriv_int :
      IntervalIntegrable (fun y : ℝ => deriv Q y) MeasureTheory.volume 0 1)
    (hQ0 : Q 0 = 0) (hQ1 : Q 1 = 0) :
    intervalConjugateKernelOperator t Q x =
      intervalFullSemigroupOperator t (deriv Q) x := by
  classical
  have h01 : (0 : ℝ) ≤ 1 := by norm_num
  set K : ℝ → ℝ := fun y : ℝ => intervalNeumannFullKernel t x y with hK
  set K' : ℝ → ℝ :=
    fun y : ℝ => deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y with hK'

  have hK_cont : ContinuousOn K (Set.uIcc (0 : ℝ) 1) := by
    rw [Set.uIcc_of_le h01]
    simpa [K] using continuousOn_intervalNeumannFullKernel_snd ht x

  have hK_deriv :
      ∀ y ∈ Set.Ioo (min (0 : ℝ) 1) (max 0 1), HasDerivAt K (K' y) y := by
    intro y _hy
    have h := hasDerivAt_intervalNeumannFullKernel_snd ht x y
    simpa [K, K', h.deriv] using h

  have hQ_deriv_minmax :
      ∀ y ∈ Set.Ioo (min (0 : ℝ) 1) (max 0 1), HasDerivAt Q (deriv Q y) y := by
    intro y hy
    have hy' : y ∈ Set.Ioo (0 : ℝ) 1 := by
      simpa [min_eq_left h01, max_eq_right h01] using hy
    exact hQ_deriv y hy'

  have hK'_int : IntervalIntegrable K' MeasureTheory.volume 0 1 := by
    simpa [K'] using intervalIntegrable_deriv_intervalNeumannFullKernel_snd ht x

  have hIBP :
      (∫ y in (0 : ℝ)..1, K y * deriv Q y) =
        K 1 * Q 1 - K 0 * Q 0 - ∫ y in (0 : ℝ)..1, K' y * Q y := by
    exact intervalIntegral.integral_mul_deriv_eq_deriv_mul_of_hasDerivAt
      hK_cont hQ_cont hK_deriv hQ_deriv_minmax hK'_int hQ_deriv_int

  have hIBP_zero :
      (∫ y in (0 : ℝ)..1, K y * deriv Q y) =
        - ∫ y in (0 : ℝ)..1, K' y * Q y := by
    simpa [hQ0, hQ1] using hIBP

  unfold intervalConjugateKernelOperator intervalFullSemigroupOperator
  rw [intervalMeasure_one_integral_eq_intervalIntegral
    (f := fun y : ℝ =>
      deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y * Q y)]
  rw [intervalMeasure_one_integral_eq_intervalIntegral
    (f := fun y : ℝ => intervalNeumannFullKernel t x y * deriv Q y)]
  simpa [K, K'] using hIBP_zero.symm

end ShenWork.IntervalConjugateDuhamelMap
```

Depending on the exact imported namespace state, the final `unfold` may need the
fully-qualified semigroup name:

```lean
unfold intervalConjugateKernelOperator
unfold ShenWork.IntervalNeumannFullKernel.intervalFullSemigroupOperator
```

The proof above is the intended shape.  I have not run Lean here; the two places
most likely to need small local edits are namespace qualification for
`intervalFullSemigroupOperator` and the simplification line
`simpa [K, K', h.deriv] using h`.  If that simplification does not close, replace
it by an explicit `rw [K, K']; simpa [h.deriv] using h` block.

## What hypotheses are actually needed?

For the clean Mathlib proof above, use:

1. `Q` continuous on `Set.uIcc 0 1`.
2. `Q` has derivative `deriv Q y` for every `y ∈ Set.Ioo 0 1`.
3. `deriv Q` is `IntervalIntegrable` on `0..1`.
4. `Q 0 = 0` and `Q 1 = 0`.
5. `0 < t` for the kernel regularity and kernel derivative integrability.

This is stronger than bare Lipschitz continuity.  A Lipschitz function is not
classically differentiable at every interior point, so it does not directly feed
`integral_mul_deriv_eq_deriv_mul_of_hasDerivAt`.  It is mathematically enough if
you develop the absolutely-continuous/a.e.-derivative version of IBP, but that is
a different API.

A practical hierarchy is:

```text
C¹ on [0,1] or continuous on [0,1] + HasDerivAt on (0,1) + derivative integrable
    easiest with existing Mathlib interval IBP

AbsolutelyContinuousOn Q [0,1] + a.e. derivative Q'
    mathematically sufficient, but requires an AC/product-FTC IBP bridge

LipschitzOnWith C Q [0,1]
    mathematically implies AC and a.e. differentiability, but not pointwise HasDerivAt;
    use only after proving/using the AC bridge, or add pointwise differentiability as
    a separate hypothesis
```

For the chemotaxis flux, your endpoint lemmas
`truncatedChemFluxLifted_zero_left/right` provide exactly `hQ0` and `hQ1`.
The positive-time regularity/bootstrap side should provide either the C¹-style
hypotheses above or a named derivative witness theorem for the stronger reusable
`Q'` version.

## Can the `intervalMeasure` issue be avoided?

Yes.  Do not reason about `intervalMeasure` during IBP.  The clean proof order is:

1. Unfold only conceptually, set `K` and `K'`.
2. Prove the interval-integral identity with Mathlib IBP.
3. Simplify boundary terms using `Q 0 = 0`, `Q 1 = 0`.
4. Unfold `intervalConjugateKernelOperator` and `intervalFullSemigroupOperator`.
5. Rewrite each `∫ ... ∂ intervalMeasure 1` using
   `intervalMeasure_one_integral_eq_intervalIntegral`.
6. Finish by `simpa [K, K'] using hIBP_zero.symm`.

That keeps the measure restriction nuisance completely localized to one lemma.

## Sign convention

With `K(y) = intervalNeumannFullKernel t x y`, IBP says

```text
∫₀¹ K(y) Q'(y) dy
  = K(1)Q(1) - K(0)Q(0) - ∫₀¹ K'(y)Q(y) dy.
```

Because `Q(0)=Q(1)=0`, this becomes

```text
∫₀¹ K(y) Q'(y) dy = -∫₀¹ K'(y)Q(y) dy.
```

The definition of `intervalConjugateKernelOperator` is exactly the right-hand
side.  There is no extra sign change from the measure conversion because the
interval is oriented as `0..1` and `0 ≤ 1` is used in
`intervalIntegral.integral_of_le`.
