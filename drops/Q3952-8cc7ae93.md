# Q3952: operator-level IBP for `intervalConjugateKernelOperator`

## Bottom line

The clean Lean proof is: **do the IBP as an `intervalIntegral` on `0..1`, not directly as a `MeasureTheory.integral` against `intervalMeasure 1`**.

The repository already uses exactly this bridge pattern in `ShenWork/PDE/IntervalFullKernelSourceIBP.lean` and `ShenWork/PDE/IntervalFullKernelInitialIBP.lean`:

```lean
simp only [intervalMeasure, intervalSet]
rw [MeasureTheory.integral_Icc_eq_integral_Ioc,
  ← intervalIntegral.integral_of_le h01]
```

After that rewrite, Mathlib's interval-integral integration-by-parts lemma gives the desired identity immediately. The leading minus in

```lean
intervalConjugateKernelOperator t Q x
```

is exactly the correct sign.

For fixed `t > 0` and `x`, set

```lean
K y  := intervalNeumannFullKernel t x y
K' y := deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y
```

Then IBP gives

```lean
∫ y in (0 : ℝ)..1, Q y * K' y
  = Q 1 * K 1 - Q 0 * K 0
    - ∫ y in (0 : ℝ)..1, deriv Q y * K y
```

If `Q 0 = 0` and `Q 1 = 0`, this reduces to

```lean
- ∫ y in (0 : ℝ)..1, Q y * K' y
  = ∫ y in (0 : ℝ)..1, deriv Q y * K y
```

which is precisely

```lean
intervalConjugateKernelOperator t Q x
  = intervalFullSemigroupOperator t (deriv Q) x
```

because both definitions integrate over `intervalMeasure 1`.

## Recommended theorem shape

Use the hypothesis package that matches the existing repo IBP proofs:

```lean
(hQ_deriv : ∀ y ∈ Set.uIcc (0 : ℝ) 1,
  HasDerivAt Q (deriv Q y) y)
(hQ_deriv_int : IntervalIntegrable (fun y : ℝ => deriv Q y)
  MeasureTheory.volume (0 : ℝ) 1)
(hQ0 : Q 0 = 0)
(hQ1 : Q 1 = 0)
```

This is more Lean-friendly than stating only “`Q` is Lipschitz”. A Lipschitz function gives differentiability a.e. and absolute continuity, but the Mathlib lemma used in the current repo wants pointwise `HasDerivAt` on the interval package, plus interval integrability of the derivative. If your flux theorem only has a Lipschitz statement, prove or use a small regularity lemma upstream that supplies this `HasDerivAt`/`IntervalIntegrable` package for the particular flux, or switch to a genuinely absolutely-continuous/a.e.-derivative IBP theorem, which is a larger measure-theory detour.

For the kernel side, `IntervalConjugateDuhamelMap.lean` already contains the required positive-time facts:

```lean
hasDerivAt_intervalNeumannFullKernel_snd
intervalIntegrable_deriv_intervalNeumannFullKernel_snd
```

So the production theorem only needs `ht : 0 < t` plus the `Q` assumptions above.

## Drop-in Lean proof

This is the theorem I would add after the definition of `intervalConjugateKernelOperator`, or in a new imported file. The imports below are intentionally explicit.

```lean
import ShenWork.Paper2.IntervalConjugateDuhamelMap
import Mathlib.MeasureTheory.Integral.IntervalIntegral.IntegrationByParts

open MeasureTheory
open scoped Topology

noncomputable section

namespace ShenWork.IntervalConjugateDuhamelMap

open ShenWork.IntervalDomain
open ShenWork.IntervalNeumannFullKernel

/-- Operator-level integration by parts for the Neumann full kernel:
`-∫ ∂ᵧK_N(t,x,y) Q(y) dy = ∫ K_N(t,x,y) Q'(y) dy`, assuming the
source vanishes at both endpoints.

This is the exact operator identity
`intervalConjugateKernelOperator t Q x = intervalFullSemigroupOperator t (deriv Q) x`.
-/
theorem intervalConjugateKernelOperator_eq_intervalFullSemigroupOperator_deriv
    {t : ℝ} (ht : 0 < t) (Q : ℝ → ℝ) (x : ℝ)
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0)
    (hQ_deriv : ∀ y ∈ Set.uIcc (0 : ℝ) 1,
      HasDerivAt Q (deriv Q y) y)
    (hQ_deriv_int : IntervalIntegrable
      (fun y : ℝ => deriv Q y) MeasureTheory.volume (0 : ℝ) 1) :
    intervalConjugateKernelOperator t Q x =
      intervalFullSemigroupOperator t (deriv Q) x := by
  have h01 : (0 : ℝ) ≤ 1 := by norm_num

  -- Convert the conjugate operator's `intervalMeasure 1` integral to `∫ in 0..1`.
  have hμ_left :
      (∫ y,
          deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y * Q y
          ∂ intervalMeasure 1)
        = ∫ y in (0 : ℝ)..1,
            Q y * deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y := by
    simp only [intervalMeasure, intervalSet]
    rw [MeasureTheory.integral_Icc_eq_integral_Ioc,
      ← intervalIntegral.integral_of_le h01]
    exact intervalIntegral.integral_congr (fun y _ => by ring)

  -- Convert the semigroup side to the same interval-integral convention.
  have hμ_right :
      (∫ y,
          intervalNeumannFullKernel t x y * deriv Q y
          ∂ intervalMeasure 1)
        = ∫ y in (0 : ℝ)..1,
            deriv Q y * intervalNeumannFullKernel t x y := by
    simp only [intervalMeasure, intervalSet]
    rw [MeasureTheory.integral_Icc_eq_integral_Ioc,
      ← intervalIntegral.integral_of_le h01]
    exact intervalIntegral.integral_congr (fun y _ => by ring)

  -- Positive-time differentiability of `y ↦ K_N(t,x,y)`, with the derivative
  -- rewritten to Lean's canonical `deriv` expression.
  have hK_deriv : ∀ y ∈ Set.uIcc (0 : ℝ) 1,
      HasDerivAt
        (fun y' : ℝ => intervalNeumannFullKernel t x y')
        (deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y)
        y := by
    intro y _
    have h := hasDerivAt_intervalNeumannFullKernel_snd ht x y
    simpa [← h.deriv] using h

  have hK_deriv_int : IntervalIntegrable
      (fun y : ℝ =>
        deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y)
      MeasureTheory.volume (0 : ℝ) 1 :=
    intervalIntegrable_deriv_intervalNeumannFullKernel_snd ht x

  -- Integration by parts with `f = Q` and
  -- `g = fun y => intervalNeumannFullKernel t x y`.
  have hibp :
      (∫ y in (0 : ℝ)..1,
          Q y * deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y)
        = Q 1 * intervalNeumannFullKernel t x 1
          - Q 0 * intervalNeumannFullKernel t x 0
          - ∫ y in (0 : ℝ)..1,
              deriv Q y * intervalNeumannFullKernel t x y := by
    simpa using
      (intervalIntegral.integral_mul_deriv_eq_deriv_mul
        hQ_deriv hK_deriv hQ_deriv_int hK_deriv_int)

  unfold intervalConjugateKernelOperator intervalFullSemigroupOperator
  rw [hμ_left, hμ_right]
  rw [hibp]
  simp [hQ0, hQ1]

end ShenWork.IntervalConjugateDuhamelMap
```

## If Lean chooses the `_of_hasDerivAt` lemma instead

Some Mathlib versions expose the theorem primarily as

```lean
intervalIntegral.integral_mul_deriv_eq_deriv_mul_of_hasDerivAt
```

rather than the shorter

```lean
intervalIntegral.integral_mul_deriv_eq_deriv_mul
```

The repository already uses the shorter one in `IntervalFullKernelSourceIBP.lean`, so I would first try the proof above. If the local import graph does not expose that theorem, the mathematical proof is unchanged: call the `_of_hasDerivAt` variant with

```lean
u  := Q
v  := fun y : ℝ => intervalNeumannFullKernel t x y
u' := fun y : ℝ => deriv Q y
v' := fun y : ℝ => deriv (fun y' : ℝ => intervalNeumannFullKernel t x y') y
```

and supply the same derivative and interval-integrability hypotheses. Depending on the exact theorem signature, you may also need closed-interval continuity hypotheses; these follow from the `HasDerivAt` assumptions plus the kernel continuity lemmas, but using the shorter theorem avoids this extra boilerplate.

## Answer to the four questions

### 1. Cleanest approach

Prove the identity after rewriting both operator integrals to `intervalIntegral`. This follows the existing repository pattern:

```lean
simp only [intervalMeasure, intervalSet]
rw [MeasureTheory.integral_Icc_eq_integral_Ioc,
  ← intervalIntegral.integral_of_le h01]
```

Then call `intervalIntegral.integral_mul_deriv_eq_deriv_mul` with `f = Q` and `g = K`.

### 2. Hypotheses on `Q`

For the fastest Lean proof, use:

```lean
∀ y ∈ Set.uIcc (0 : ℝ) 1, HasDerivAt Q (deriv Q y) y
IntervalIntegrable (fun y : ℝ => deriv Q y) volume 0 1
Q 0 = 0
Q 1 = 0
```

Mathematically, absolute continuity is the natural minimal assumption. In Lean, Lipschitz alone is not the right direct input to this IBP lemma because it gives differentiability only a.e.; you would then need an a.e.-derivative version of IBP and an a.e. identification with `deriv Q`.

### 3. Avoiding the `intervalMeasure` issue

Yes. Do not prove an IBP theorem for `intervalMeasure 1`. Rewrite the two concrete operator integrals to interval integrals and use Mathlib's interval-integral theorem. This is exactly what the repo already does for nearby initial/source IBP lemmas.

### 4. Sign convention

No sign bug. IBP gives

```lean
∫ Q * K' = Q 1 * K 1 - Q 0 * K 0 - ∫ Q' * K
```

The endpoint term vanishes because `Q 0 = Q 1 = 0`, so

```lean
-∫ Q * K' = ∫ Q' * K
```

The definition of `intervalConjugateKernelOperator` is precisely the left-hand side, and `intervalFullSemigroupOperator t (deriv Q) x` is precisely the right-hand side after the same `intervalMeasure`-to-`intervalIntegral` conversion.
