# Audit: next non-circular theorem for the general initialized zero-start route

Audited `xiangyazi24/Shen_work` at main commit `c631b793`.

## Verdict

There is **no existing initialized/patched `(u,v)` construction** in the audited source that can non-circularly supply

```lean
H1ZeroStartInitializedPrimitiveC1SignSource u₀ v₀ u v T
```

for the general B-form/Picard local solution.

The repo currently has:

- a **u-only** patch, `patchedSlice`, for value-level source/coefficient work;
- strict-positive-time B-form/classical/dx regularity for raw `conjugatePicardLimit` and its resolver;
- constant-special-case zero-start producers;
- no general paired initialized `v` patch and no general first-spatial-derivative trace theorem at `t = 0`.

So the next step should be a **source-facing reducer/frontier record**, not a claimed construction producer.

## Why current B-form/Picard data do not close the general source guard

### Raw B-form output still has the bad zero slice

`conjugatePicardLimit` in `IntervalConjugatePicard` is zero outside `(0,T]`:

```lean
def conjugatePicardLimit ... (t : ℝ) ... :=
  if 0 < t ∧ t ≤ T then atTop.limUnder ... else 0
```

So raw `conjugatePicardLimit p u₀ T` cannot satisfy `u_zero : u 0 = u₀` for a positive datum and cannot satisfy `u_pos0` at `t = 0`.

### `patchedSlice` is u-only and value-level

`ShenWork/Paper2/IntervalPicardLimitBddHcontP.lean` defines:

```lean
noncomputable def patchedSlice (u₀ : intervalDomainPoint → ℝ)
    (u : ℝ → intervalDomainPoint → ℝ) (s : ℝ) : intervalDomainPoint → ℝ :=
  if s ≤ 0 then u₀ else u s
```

with:

```lean

patchedSlice_of_pos
patchedSource_eq_coeff_slice
patchedSource_continuousOn_Icc
```

This gives only a patched `u` slice and source-coefficient continuity reducers. It does not define a paired initialized chemical trajectory `vZ`, and it does not prove closed-zero-slab primitive C1 continuity.

`ShenWork/Paper2/IntervalPicardLimitSliceTimeContinuity.lean` proves value/sup-norm continuity for patched slices, including:

```lean

patchedSlice_timeContinuousAt_pos

```

These are about `∀ y, |patchedSlice ... s y - patchedSlice ... s₀ y| < ε`; they do not imply first-spatial-derivative traces and are not a paired `(u,v)` source producer.

### dx continuity is strict-time only

`ShenWork/PDE/P3MoserDxJointContinuity.lean` proves:


both on


These are useful strict-time near-producers, but they cannot supply `ux_cont0` or `vx_cont0` on `Set.Icc 0 b ×ˢ Set.Icc 0 1` without genuine derivative trace fields at `t = 0`.

### B-form local data are also strict-time

The supporting facts such as `bform_u_pos`, `bform_u_closedC2_endpointDerivs`, `bform_u_neumann_left`, and `bform_u_neumann_right` all require `0 < t`.

`ShenWork/Paper2/IntervalBFormPositiveDatumLocalExistenceSq.lean` has the same limitation: `PositiveDatumBFormLocalComponentsSq.strictPos` is positive-time only, and `PositiveDatumBFormLocalComponentsSq.localClassicalSolution` returns the raw `conjugatePicardLimit` plus `InitialTrace`, not an initialized pair.

## Hidden circularities to avoid

Do **not** use these as source producers for `H1ZeroStartInitializedPrimitiveC1SignSource`:

1. `H1ZeroStartPhysicalPrimitiveDataBefore`: it already contains the primitive fields plus PDE seam fields; using it would be circular.
2. `H1ZeroStartPhysicalRHSDataBefore` or `H1LiftDeriv2ZeroSlabRepresentativeBefore`: downstream zero-start RHS/representative outputs depend on primitive source data.
3. sqrt/reaction-bound/H1 bounded-before files: downstream of the source frontier and out of scope for producing primitive regularity.
4. `InitialTrace intervalDomain u₀ u`: this is a sup-norm right-limit only, not `u 0 = u₀` and not a derivative trace.
5. `patchedSlice_timeContinuousAt_zero` or `hsliceTC_of_mild_restart`: value/sup-norm continuity does not provide `deriv (lift u)` or resolver-gradient traces.

## Frontier that should only be documented, not claimed as code now

A true general B-form producer would need a paired initialized construction, for example:

```lean
uZ t := if t ≤ 0 then u₀ else conjugatePicardLimit p u₀ T t
vZ t := if t ≤ 0 then intervalNeumannResolverR p u₀
        else mildChemicalConcentration p (conjugatePicardLimit p u₀ T) t
```

Then it would need to prove the `H1ZeroStartPrimitiveC1ZeroFaceTrace` fields above for `(uZ,vZ)`. The missing analytic content is exactly the value and first-spatial-derivative zero-face traces for both `uZ` and `vZ`, especially `ux_zeroFace` and `vx_zeroFace`. Existing `InitialTrace` and `patchedSlice` value continuity are insufficient.
