# Definitive verdict

The coefficient-time-differentiation step is small **once the correct space-time derivative fields are available**, and the repository already contains nearly all of the required machinery:

```lean
ShenWork.IntervalMildPicardRegularity.cosineCoeffs_hasDerivAt_of_smooth_param
ShenWork.IntervalLogisticSourceTimeC1.logisticReaction_comp_hasDerivAt
ShenWork.IntervalCoupledRegularityBootstrap.CoupledChemDivLocalChainRule
ShenWork.IntervalCoupledRegularityBootstrap.coupledChemDivCoeff_hasDerivAt_of_fields
ShenWork.IntervalCoupledRegularityBootstrap.chemDivAdot_continuousOn_of_jointCont
```

There are two precise qualifications.

1. `DuhamelSourceTimeC1On` does **not** require a summable envelope for the time derivatives. It requires one scalar constant, independent of both time and mode:

```lean
derivBound : ℝ
hderivBound : ∀ s ∈ Set.Icc lo hi, ∀ n, |adot s n| ≤ derivBound
```

Thus it requires more than merely a separate finite bound `B_n` for each mode, but much less than `Summable B_n`.

2. The currently committed definition

```lean
MildSolutionSliceHasDerivAtTime S
```

at commit `2347cb0365108ea74d7a64607cbdd4850c593c18` gives:

```lean
∃ udot,
  (∀ t, 0 < t → t < S.T → ∀ x,
      HasDerivAt (fun r => S.u r x) (udot t x) t) ∧
  (∀ t, 0 < t → t < S.T → Continuous (udot t)) ∧
  (∀ c T', 0 < c → T' < S.T → ∃ B ≥ 0,
      ∀ t ∈ Set.Icc c T', ∀ x, |udot t x| ≤ B)
```

The second field is **spatial continuity of `udot t` for each fixed `t`**. It is not continuity of `t ↦ udot t x`, and not joint continuity of `(t,x) ↦ udot t x`. Therefore `MildSolutionSliceHasDerivAtTime` by itself supplies pointwise chain rules and a uniform bound, but it does not by itself prove the `hadotcont` field. One additional small seam is needed: joint continuity of the relevant source-time-derivative field on each compact positive-time slab. Under the question's stronger phrase “continuity of `∂ₜu` is given,” interpreted as this joint continuity, the rest is wiring.

# 1. Exact structure fields: no summable derivative envelope

The current window-local structure is:

```lean

```

Only the **value** envelope is summable:

```lean
Summable envelope
```

For the derivative side the exact target is:

```text
∃ Mdot : ℝ, ∀ t in the window, ∀ k, |∂ₜ s_k(t)| ≤ Mdot.
```

There is no field

```lean

```

and no such fact should be proved merely to fill `DuhamelSourceTimeC1On`.

The global `DuhamelSourceTimeC1` has the same distinction: a summable source-value envelope and only one mode-uniform derivative constant.

# 2. Generic cosine-coefficient time Leibniz rule

Let `F : ℝ → ℝ → ℝ` be a source field and `G = ∂ₜF`. With the repository normalization,

```text
cosineCoeffs (F t) 0 = ∫₀¹ F(t,x) dx,
cosineCoeffs (F t) k = 2∫₀¹ cos(kπx) F(t,x) dx       (k > 0).
```

Define

```lean
a t k := cosineCoeffs (F t) k
adot t k := cosineCoeffs (G t) k
```

The committed theorem

```lean
cosineCoeffs_hasDerivAt_of_smooth_param
```

has exactly the right interface. Around each time `τ`, it asks for:

```lean
∃ δ > 0,
  (∀ᶠ s in 𝓝 τ, IntervalIntegrable (F s) volume 0 1) ∧
  (∀ x ∈ Set.Ioo 0 1, ∀ s ∈ Metric.ball τ δ,
      HasDerivAt (fun r => F r x) (G s x) s) ∧
  ContinuousOn (Function.uncurry G)
    (Set.Icc (τ - δ) (τ + δ) ×ˢ Set.Icc 0 1)
```

and concludes

```lean
HasDerivAt
  (fun t => cosineCoeffs (F t) k)
  (cosineCoeffs (G τ) k)
  τ.
```

The domination needed to differentiate under the integral is obtained internally from continuity of `G` on the compact slab. In other words, the formal proof should not re-prove a mode-specific dominated convergence theorem. Supply the pointwise chain rule and joint continuity to this existing theorem.

For a window package, restrict the resulting global derivative:

```lean
(hCoeff τ k).hasDerivWithinAt
```

or prove it directly in the within-set form.

## Uniform mode bound from a physical-field bound

Suppose on `K = [c,T'] × [0,1]`:

```text
|G(t,x)| ≤ BG.
```

Then for every mode,

```text
|cosineCoeffs (G t) k| ≤ 2 BG.
```

The factor `2` uniformly covers positive modes; the zero mode actually has the sharper bound `BG`. Thus one may set

```lean
derivBound := 2 * BG.
```

This is the simplest route to `hderivBound`. Joint continuity of `G` on a compact slab itself gives some finite `BG`; for an explicit quantitative theorem, carry a pointwise bound instead.

# 3. Logistic source: exact derivative, continuity, and bound

Write

```text
L(t,x) = u(t,x) (a - b u(t,x)^α).
```

Let `w = ∂ₜu`. Pointwise in space,

```text
∂ₜL
 = w (a - b u^α) + u (-b α u^(α-1) w)
 = w (a - b(1+α)u^α).
```

The first expanded expression is the one produced directly by the repository theorem

```lean
logisticReaction_comp_hasDerivAt
```

and the second is an algebraic simplification.

Define the physical derivative field and its coefficient:

```lean
logisticTimeField p u udot t x :=
  udot t x * (p.a - p.b * (1 + p.α) * (u t x) ^ p.α)

logisticAdot p u udot t k :=
  cosineCoeffs (logisticTimeField p u udot t) k
```

Then the time-Leibniz theorem gives

```lean
HasDerivAt
  (fun r => cosineCoeffs
    (fun x => u r x * (p.a - p.b * (u r x)^p.α)) k)
  (logisticAdot p u udot t k)
  t.
```

## Positivity and the `δ` floor

For noninteger `rpow`, the clean Lean chain rule uses the positive branch:

```lean
HasDerivAt.rpow_const (Or.inl (ne_of_gt hu_pos))
```

so use the positive-time compact floor `u ≥ δ > 0`. For the **numerical bound** below, when `α ≥ 0`, no negative power occurs in the simplified formula, so only `u ≤ M` is needed. The floor is used by the differentiation and continuity proof, not by the final estimate.

## Joint continuity

If both

```text
(t,x) ↦ u(t,x),
(t,x) ↦ udot(t,x)
```

are continuous on `[c,T'] × [0,1]` and `u ≥ δ > 0`, then

```text
(t,x) ↦ udot(t,x) (a - b(1+α)u(t,x)^α)
```

is jointly continuous by `ContinuousOn.rpow_const_of_ne`, products, sums, and scalar multiplication. Consequently

```lean
∀ k, ContinuousOn (fun t => logisticAdot p u udot t k) (Set.Icc c T')
```

follows from the committed compact-integral continuity lemma

```lean
cosineCoeffs_continuousOn_of_jointContinuousOn_Icc.
```

## Uniform bound

Assume on the window

```text
0 < δ ≤ u(t,x) ≤ M,
|udot(t,x)| ≤ Bu,
0 ≤ a,
0 ≤ b,
0 ≤ α.
```

Then

```text
|∂ₜL(t,x)|
  ≤ Bu * (a + b(1+α) M^α).
```

Hence one valid derivative constant is

```text
Mdot_log := 2 * Bu * (a + b * (1 + α) * M^α).
```

This gives, uniformly in both `t` and `k`,

```text
|logisticAdot t k| ≤ Mdot_log.
```

This logistic time half is genuinely small.

# 4. Resolver time derivative from linearity

Let `v = R[u]`. In the repository the resolver equation is

```text
-vₓₓ + μ v = ν u^γ.
```

Differentiating in time gives

```text
-(vₜ)ₓₓ + μ vₜ = ν γ u^(γ-1) uₜ,
(vₜ)ₓ(0) = (vₜ)ₓ(1) = 0.
```

At the coefficient level,

```text
widehat(vₜ)_j
  = cosineCoeff(ν γ u^(γ-1)uₜ, j) / (μ + λ_j).
```

For the alternative equation `(μ - νₑ∂ₓₓ)v = f(u)`, replace the denominator by

```text
μ + νₑ λ_j
```

and the numerator by the cosine coefficient of `f'(u)uₜ`.

The power-source coefficient derivative is already formalized in

```lean
ShenWork.Wiener.EWA.SourcePowerCoeffDeriv
```

with

```lean
adotPow p v vdotL t k :=
  cosineCoeffs
    (fun x => p.ν * p.γ
      * (intervalDomainLift (v t) x)^(p.γ - 1)
      * vdotL t x) k.
```

The static resolver multiplier then transfers this time-C¹ coefficient data to `vₜ`. No nonlinear resolver differentiation is needed: it is linear modewise division.

# 5. Chemotaxis-divergence source: exact derivative field

Let

```text
v = R[u],
d = 1 + v,
Q = u vₓ d^(-β),
C = ∂ₓQ.
```

Writing `w = uₜ` and `z = vₜ`, the flux time derivative is

```text
Qₜ
 = w vₓ d^(-β)
   + u zₓ d^(-β)
   - β u vₓ z d^(-β-1).
```

Therefore

```text
∂ₜC = ∂ₓQₜ.
```

This is exactly the repository definition

```lean
coupledChemDivTimeDerivativeLift p u t x
```

which is, schematically,

```lean
deriv (fun y =>
    u_t(t,y) * v_x(t,y) / (1 + v(t,y))^β
  + u(t,y) * (v_t)_x(t,y) / (1 + v(t,y))^β
  - β * u(t,y) * v_x(t,y) * v_t(t,y)
      / (1 + v(t,y))^(β+1)) x.
```

The canonical coefficient derivative is already fixed in the repo:

```lean
coupledChemDivAdot p u t k :=
  cosineCoeffs (coupledChemDivTimeDerivativeLift p u t) k.
```

Thus the exact coefficient formula is

```text
∂ₜ [coupledChemDivSourceCoeffs p u t k]
  = coupledChemDivAdot p u t k.
```

Equivalently, since `Qₜ` also vanishes at the endpoints (`vₓ = 0` and `(vₜ)ₓ = 0`), the divergence identity gives for positive modes

```text
coupledChemDivAdot(t,k)
  = (kπ) * sineCoeffs(Qₜ(t,·),k).
```

For the time-C¹ package, however, the direct cosine coefficient of `∂ₓQₜ` is the most convenient form.

## Uniform bound

If on the compact positive-time slab

```text
|coupledChemDivTimeDerivativeLift p u t x| ≤ Bchem,
```

then

```text
|coupledChemDivAdot p u t k| ≤ 2 Bchem
```

for every `t` and `k`. Therefore set

```text
Mdot_chem := 2 Bchem.
```

Joint continuity of the complete field `coupledChemDivTimeDerivativeLift` on the compact slab implies existence of such a `Bchem` by compactness. An explicit bound can instead be assembled from window bounds on the factors in `∂ₓQₜ`.

This is where the chemDiv branch is slightly more than the mere statement “`uₜ` is bounded.” The field `∂ₓQₜ` contains spatial derivatives of the time-dependent factors. A sup bound on `uₜ` alone does not control `∂ₓQₜ`. The committed `CoupledChemDivLocalChainRule`/joint-continuity package is the correct additional analytic interface.

# 6. Why the committed `MildSolutionSliceHasDerivAtTime` is not alone sufficient

The committed type supplies:

```text
uₜ(t,·) continuous in x for each fixed t,
|uₜ(t,x)| uniformly bounded on compact positive-time windows.
```

It does not supply:

```text
t ↦ uₜ(t,x) continuous,
(t,x) ↦ uₜ(t,x) jointly continuous,
spatial differentiability of uₜ,
joint continuity or a uniform bound for ∂ₓQₜ.
```

Therefore the exact feasibility statement is:

- **Logistic `hderiv` and `derivBound`:** follow from the committed pointwise `HasDerivAt`, positivity, and uniform `|uₜ|` bound.
- **Logistic `hadotcont`:** needs one extra joint/time-continuity seam for `uₜ`.
- **ChemDiv `hderiv`:** follows once `CoupledChemDivLocalChainRule` is assembled from `uₜ`, resolver time regularity, and the spatial chain rule.
- **ChemDiv `hadotcont` and `derivBound`:** follow from joint continuity/boundedness of `coupledChemDivTimeDerivativeLift` on the compact positive-time slab.

If “continuity of `∂ₜu` is given” means joint continuity on these slabs and the chemDiv chain-rule field has been assembled, then this is bounded wiring. If it means only the fields literally present in `MildSolutionSliceHasDerivAtTime`, one small continuity interface remains.

# 7. Total B-form source assembly

The B-form source coefficients have the form

```text
bFormSourceCoeffs p u t k
  = logisticCoeff(t,k) - p.χ₀ * chemDivCoeff(t,k).
```

Define

```text
adot_total(t,k)
  = adot_log(t,k) - p.χ₀ * coupledChemDivAdot p u t k.
```

Then

```text
|adot_total(t,k)|
  ≤ Mdot_log + |p.χ₀| * Mdot_chem.
```

Set

```text
Mdot_total := Mdot_log + |p.χ₀| * Mdot_chem.
```

The package-level assembly uses the already committed closure operations:

```lean
DuhamelSourceTimeC1On.const_mul
DuhamelSourceTimeC1On.add
```

or their global counterparts. Schematically:

```lean
have hlogOn  : DuhamelSourceTimeC1On logCoeff c T' := ...

have htotalOn := hlogOn.add (hchemOn.const_mul (-p.χ₀))
```

The summable `envelope` fields of `hlogOn` and `hchemOn` come from the previously established spatial source ladder. The present time-differentiation step fills only:

# Final read

The time-coefficient half is not a new PDE regularity theorem. It is:

```text
pointwise source chain rule
+ local joint continuity
+ existing cosine-coefficient Leibniz theorem
+ compact-slab coefficient bound
+ record assembly.
```

The logistic branch is almost trivial once `uₜ` is available. The resolver time derivative is a linear cosine multiplier. The chemDiv branch uses the explicit committed field `coupledChemDivTimeDerivativeLift` and `CoupledChemDivLocalChainRule`.

The decisive bookkeeping fact is:

> `DuhamelSourceTimeC1On` needs a single mode-uniform derivative constant, not a summable derivative envelope. But it does require that single constant to be independent of `k`; per-mode compactness alone is insufficient. A uniform physical-space bound on the source time-derivative field gives it immediately.
