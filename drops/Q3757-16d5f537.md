# Q3757 audit: Q3755 global B-form cosine theorem is missing the gradient-to-B-kernel bridge

I checked GitHub `main`; it is identical to `8aad19545318329f77d786e9afdf2e4520e034c5`.

## Verdict

Q3755's proposed theorem

```lean
gradientMildSolution_bForm_global_cosine_of_sourceBridge
```

is incomplete as stated.  `D.hmild` plus `hsource_bridge` is not enough to prove the global B-form cosine representation for a `GradientMildSolutionData`.

The missing input is exactly the equality between the gradient-map chemotaxis leg and the conjugate/B-form chemotaxis leg.  Without that equality, the proof silently changes the mild equation being used.


## What `IntervalFullKernelSourceIBP.lean` actually proves

`ShenWork/PDE/IntervalFullKernelSourceIBP.lean` proves the source-IBP identity

```lean
theorem deriv_intervalFullSemigroupOperator_eq_neg_conjugateKernel_source_integral
    {t : ŌäØ} (ht : 0 < t)

      -(Ōł½ y in (0 : ŌäØ)..1, Q' y * intervalNeumannConjugateKernel t x y)
```

This is not the same as `intervalConjugateKernelOperator t Q x`.  The right side uses:

```lean
Q' y * intervalNeumannConjugateKernel t x y
```

where `intervalNeumannConjugateKernel` is the signed kernel `K╠ā` satisfying `ŌłéßĄ¦ K╠ā = ŌłéŌéō K_N`.  But `intervalConjugateKernelOperator` is

```lean
-Ōł½ y, deriv (fun y' => intervalNeumannFullKernel t x y') y * Q y Ōłé intervalMeasure 1
```

So the existing IBP theorem does not provide

```lean
deriv (fun z => intervalFullSemigroupOperator t Q z) x
  = intervalConjugateKernelOperator t Q x
```

and Q3755's proposed global representation cannot be proved from `D.hmild + hsource_bridge` alone.

## Hidden circularity / invalid shortcut

Do not prove the missing equality by using `hpde_u`, `GradientMildClassicalFrontierCoreData`, or `IsPaper2ClassicalSolution`.  That would be circular, because the whole purpose of the new `hpde_u_of_generalChi_sourceSpectralData` lemma is to prove `hpde_u` from source spectral data.

The safe inputs are only kernel identities, source/flux differentiability, integrability, coefficient/Fourier regularity, and the mild fixed-point equation `D.hmild`.

## After that theorem

After the open IBP theorem, there is still one more kernel-identification issue before Q3755's route works with the existing B-form source bridge.  The open IBP theorem gives a `K╠ā * Q'` expression:

```lean
-(Ōł½ y in (0 : ŌäØ)..1,
  Q' y * intervalNeumannConjugateKernel r x y)
```

whereas `hsource_bridge` consumes

```lean
intervalConjugateKernelOperator r Q x
  = -Ōł½ y, deriv (fun y' => intervalNeumannFullKernel r x y') y * Q y
      Ōłé intervalMeasure 1
```

So the corrected proof path must either:

1. prove the additional kernel/source identity identifying the open-IBP expression with the B-kernel leg, if true under the intended hypotheses; or
2. bypass `intervalConjugateKernelOperator` and prove a gradient-specific source bridge directly:

```lean
(-p.ŽćŌéĆ) * deriv (fun z => intervalFullSemigroupOperator r
    (chemFluxLifted p (u s)) z) x
  + intervalFullSemigroupOperator r (logisticLifted p (u s)) x
= unitIntervalCosineHeatValue r (bFormSourceCoeffs p u s) x
```

The second option is cleaner for the actual `GradientMildSolutionData` route, because it matches `intervalGradientDuhamelMap` directly and avoids silently changing the chemotaxis kernel.

## Bottom line

Q3755's proposed theorem is missing a necessary `hgradB`/IBP chemotaxis-leg bridge.  `D.hmild + hsource_bridge` alone cannot prove the global B-form cosine representation for a gradient mild solution.

The next core theorem should be the endpoint-safe/open source-IBP identity for the gradient chemotaxis leg, followed by either a genuine kernel-identification theorem to the existing B-kernel source bridge or a direct gradient-source bridge.  Do not proceed with the Q3755 global representation theorem until that bridge is explicit.
