# Q3791 route selector: replace false B-kernel/global-cosine route with mixed sine/cosine source data

Audit basis: current source around `3027a6f2` plus the stated new obstruction theorem `hgradB_forces_B_kernel_endpoint_zero`.

## Verdict

The next true core target is **not** another B-form/global-cosine producer. The old interface

```lean
IntervalGradientBFormGlobalCosine.gradientMildSolution_bForm_global_cosine_of_gradB_sourceBridge
```

is based on the false conversion

```lean
∫ deriv (fun z => intervalFullSemigroupOperator (t-s) Q z) x
  = ∫ intervalConjugateKernelOperator (t-s) Q x
```

and the new endpoint obstruction shows why this cannot be the headline path. The gradient mild chemotaxis leg is the **Ktilde/source-derivative/sine route**, not the B-kernel/cosine-total-source route.

So the replacement interface should be a new **mixed sine/cosine spectral PDE producer**:

* cosine part for the homogeneous solution and logistic source;
* sine part for the chem-div source produced by `∂ₓ S_N Q`;
* no `intervalConjugateKernelOperator` and no `hgradB`.

The smallest useful Task314-style file is:

```text
ShenWork/Paper2/IntervalDomainPdeUMixedSineCosine.lean
```

It should prove the analogue of `IntervalDomainPdeUGeneralChiOn.hpde_u_of_generalChi_sourceSpectralDataOn`, but with mixed sine/cosine local restart data.

```lean
theorem hpde_u_of_mixedSineCosine_sourceSpectralDataOn
    (p : CM2Params) {T : ℝ} {u : ℝ → intervalDomainPoint → ℝ}
    (Hmixed : ∀ t₀, 0 < t₀ → t₀ < T →
      ∀ {x : intervalDomainPoint}, x.1 ∈ Set.Ioo (0 : ℝ) 1 →
      ∃ (a₀ : ℕ → ℝ) (M : ℝ) (_ : 0 ≤ M) (_ : ∀ n, |a₀ n| ≤ M)

        (aC : ℝ → ℕ → ℝ) (WC : ℝ) (_ : DuhamelSourceTimeC1On aC 0 WC)
        (offset : ℝ) (_ : 0 < t₀ - offset)
        (_ : t₀ - offset < WL) (_ : t₀ - offset < WC)
        (_ : LogisticCosineFourierData p u t₀)
        (_ : ChemDivSineFourierDataIoo p (u t₀)
          (coupledChemicalConcentration p u t₀)),
        (∀ᶠ s in 𝓝 t₀, ∀ y : intervalDomainPoint,
          u s y =
            (∑' n : ℕ,
              localRestartCoeff a₀ aL (s - offset) n * cosineMode n y.1)
            + (-p.χ₀) *
              (∑' n : ℕ,
                duhamelSpectralCoeff aC (s - offset) n *
                  Real.sin ((n : ℝ) * Real.pi * y.1))) ∧

        (∀ n, aC (t₀ - offset) n =
          sineCoeffs (coupledChemDivSourceLift p u t₀) n) ∧
        Summable (fun n => unitIntervalCosineEigenvalue n *
          |localRestartCoeff a₀ aL (t₀ - offset) n|) ∧
        Summable (fun n => unitIntervalCosineEigenvalue n *
          |duhamelSpectralCoeff aC (t₀ - offset) n|)) :
    ∀ t x, 0 < t → t < T → x ∈ intervalDomain.inside →
      intervalDomain.timeDeriv u t x =
        intervalDomain.laplacian (u t) x
          - p.χ₀ * intervalDomain.chemotaxisDiv p (u t)
              (mildChemicalConcentration p u t) x
          + u t x * (p.a - p.b * (u t x) ^ p.α)
```

## How the mixed producer proves `hpde_u`

At time `t` and interior point `x`:

1. Obtain from `Hmixed` local data at `t`.
2. Differentiate the local representation in time:
   * cosine part: reuse the same windowed `localRestartCoeff` derivative machinery used by `IntervalDomainPdeUGeneralChiOn` / `timeDeriv_eq_of_rep_on_public`, or factor out the cosine half if needed;
   * sine part: use `sineDuhamelSeries_hasDerivAt_time_of_on`.
3. Compute the interior Laplacian of the local representation using `laplacian_eq_of_mixed_cosine_sine_rep`.
4. Cancel the `-λ` terms between time derivative and Laplacian.
5. Use:
   ```lean
   coupledLogistic_cosineFourier_convergence
   coupledChemDiv_sineFourier_convergence_Ioo
   ```
   to identify the two source series with the PDE source terms.
6. Finish with the same algebra pattern as:
   ```lean
   IntervalConjugateDuhamelMap.hpde_u_core_general_chi
   ```
