## (1) Can `hg_holder` be derived from only `GradientMildSolutionData` and existing bounds?

For the current theorem as stated, with arbitrary `0 < θ < 1`: **no, not from the existing bounds and theorem surface**.

`GradientMildSolutionData` gives bounded, nonnegative, continuous time slices and joint measurability. In the current repo this is enough for:

```lean
resolverSourceCoeff_re_sq_summable_of_continuousOn
source_coeffL2Norm_le_of_bounded
resolver_sineSeries_summable_of_sourceL2
resolverGrad_sup_le_of_bounded
```

That is, it gives source coefficient `ℓ²`, pointwise gradient-series summability, and a uniform `L∞` bound for `resolverGradReal`. It does **not** give a uniform spatial modulus of continuity for `resolverGradReal`.

The existing per-time source-decay route does give spatial Holder:

```lean
resolverGradReal_holder_Icc_of_sourceDecay
resolverGradReal_holder_Icc_mild
resolverGradReal_holder_Icc_mild_of_halfStepRestart
```

but all of these return an existential `Hg` per time slice. They do not produce a single uniform constant on `0 < s ≤ D.T`.

So the current remaining frontier is real: a uniform-in-time gradient Holder modulus is not discharged by the weak order-box data currently exposed by `GradientMildSolutionData`.

## (3) Does the direct cosine-series route from weak bounded data work?

### For resolver value: yes, and Task170 already added it

Task170's value proof works because

```text
R(x)-R(y) = Σ a_k/(μ+λ_k) · (cos(kπx)-cos(kπy))
```

and the mode difference gives one factor of `kπ|x-y|`, so the Cauchy-Schwarz square weight is

```text
(kπ/(μ+λ_k))² ~ 1/k²,
```

which is summable. This is exactly the same gradient-weight constant used by
`resolverGrad_sup_le_of_bounded`.

### For resolver gradient with the same naive Lipschitz-mode estimate: no

For the gradient series,

```text
resolverGradReal(x) = Σ a_k/(μ+λ_k) · (−kπ sin(kπx)).
```

A direct Lipschitz estimate gives

```text
|−kπ sin(kπx) + kπ sin(kπy)| ≤ (kπ)^2 |x-y|.
```

Then the Cauchy-Schwarz square weight is

```text
((kπ)^2/(μ+λ_k))² ~ 1,
```

which is **not summable**. This is the precise obstruction. It is not a Lean artifact; it is the real loss of one derivative.

The existing derivative bridge also cannot help under weak data:

```lean
resolverR_hasDerivAt_grad
```

requires an `ℓ¹` gradient majorant, which is currently obtained from `SourceCoeffQuadraticDecay`, not from weak bounded/continuous order-box data.

### Fractional exception

There is a mathematically plausible weaker direct spectral route for `θ < 1/2` using

```text
|sin X - sin Y| ≤ Cθ |X-Y|^θ
```

instead of the Lipschitz estimate. The square weight becomes

```text
((kπ)^(1+θ)/(μ+λ_k))² ~ 1/k^(2-2θ),
```

which is summable iff `θ < 1/2`.

So:

- For arbitrary `0 < θ < 1`, weak bounded/continuous data are not enough by this direct Cauchy-Schwarz route.
- For a restricted small exponent, e.g. `0 < θ < 1/2`, a new weak spectral theorem is plausible.
- For full `θ < 1`, the clean route is stronger source regularity: uniform source-decay / uniform `resolverGrad2Real` bound.

A dyadic/min estimate using `min(2, kπ|x-y|)` can also give the endpoint exponent `1/2` at the analytic level, but it is more involved in Lean. The fractional-weight theorem for `θ < 1/2` is more modular.

```lean
/-- Fractional sine-mode difference bound.  This is the single-mode input for

theorem resolverGrad_sineMode_sub_le_rpow
    {θ : ℝ} (hθ0 : 0 < θ) (hθ1 : θ ≤ 1)
    (k : ℕ) (x y : ℝ) :
    |(-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * x)) -
      (-((k : ℝ) * Real.pi) * Real.sin ((k : ℝ) * Real.pi * y))| ≤
      (2 : ℝ) ^ (1 - θ) * (((k : ℝ) * Real.pi) ^ (1 + θ)) *
        |x - y| ^ θ
```

Proof plan:

1. Rewrite as `(kπ) * |sin(kπx)-sin(kπy)|`.
2. Use `|sin A - sin B| ≤ min 2 |A-B|` from `Real.abs_sin_le_one` and the usual Lipschitz bound, then interpolate
   `min 2 r ≤ 2^(1-θ) r^θ` for `0 < θ ≤ 1`.
3. Rewrite `|kπx-kπy| = kπ |x-y|`, and use rpow multiplication with nonnegative bases.

Then the theorem that actually produces `hg_holder`:


2. Convert to raw source `ℓ²` using `intervalNeumannResolverSourceCoeff_zero`.
3. Use `resolver_sineSeries_summable_of_sourceL2` at `x` and `y` to rewrite the difference of `resolverGradReal` as a `tsum` of term differences.

   using `resolverGrad_sineMode_sub_le_rpow`.
6. Apply

   as in `intervalNeumannResolverR_lift_holder_Icc_of_bounded`.

   `2 * (p.ν * M ^ p.γ)`.

```lean
open ShenWork.IntervalDomain (intervalDomainPoint)

namespace ShenWork.Paper2

noncomputable section

/-- Uniform C¹ control of the resolver gradient on a time window.

The `source_decay` field supplies the derivative bridge
`resolverGradReal_hasDerivAt_of_sourceDecay`; the `grad2_bound` field supplies the
single constant needed for a uniform mean-value estimate. -/
structure UniformResolverGrad2RealBoundOn
    (p : CM2Params) (u : ℝ → intervalDomainPoint → ℝ)
    (T G2 : ℝ) : Prop where
  G2_nonneg : 0 ≤ G2
  source_decay : ∀ s, 0 < s → s ≤ T → SourceCoeffQuadraticDecay p (u s)
  grad2_bound : ∀ s, 0 < s → s ≤ T → ∀ x : ℝ,
    x ∈ Set.Icc (0 : ℝ) 1 → |resolverGrad2Real p (u s) x| ≤ G2

/-- Uniform second-derivative bound implies uniform Holder control of
`resolverGradReal`. -/
theorem resolverGradReal_holder_Icc_of_uniform_grad2Real_bound
    {p : CM2Params} {u : ℝ → intervalDomainPoint → ℝ}
    {T G2 θ : ℝ}
    (H : UniformResolverGrad2RealBoundOn p u T G2)
    (hθ0 : 0 < θ) (hθ1 : θ ≤ 1) :
    ∀ s, 0 < s → s ≤ T → ∀ a b : ℝ,
      a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
        |resolverGradReal p (u s) a - resolverGradReal p (u s) b| ≤
          G2 * |a - b| ^ θ
```

Proof plan:

1. For fixed `s`, obtain
   ```lean
   hderiv : ∀ z, HasDerivAt (fun y => resolverGradReal p (u s) y)
      (resolverGrad2Real p (u s) z) z
   ```
   from
   ```lean
   resolverGradReal_hasDerivAt_of_sourceDecay (H.source_decay s hs0 hsT)
   ```
2. Build
   ```lean
   hdiffAt : ∀ z ∈ Icc 0 1, DifferentiableAt ℝ (fun y => resolverGradReal p (u s) y) z
   ```
3. Rewrite `deriv` with `hderiv z).deriv` and use `H.grad2_bound` to get
   ```lean
   ∀ z ∈ Icc 0 1, ‖deriv (fun y => resolverGradReal p (u s) y) z‖ ≤ G2
   ```
4. Apply `Convex.norm_image_sub_le_of_norm_deriv_le` on `convex_Icc 0 1`.
5. Upgrade Lipschitz to `θ`-Holder on `[0,1]` via `|a-b| ≤ 1` and `θ ≤ 1`, exactly as in
   `resolverGradReal_holder_Icc_of_sourceDecay`.

This theorem is small and Lean-friendly because it copies the already checked proof pattern in
`resolverGradReal_holder_Icc_of_sourceDecay`, replacing the per-time compactness-produced bound by a uniform record field.

## If pursuing a uniform source-decay producer

A later producer from uniform source-decay should target `UniformResolverGradC1On`. The key quantitative lemma is:

```lean
/-- Uniform quadratic source-coefficient decay gives a uniform bound on
`resolverGrad2Real`. -/
theorem uniformResolverGradC1On_of_uniformSourceCoeffQuadraticDecayOn
    {p : CM2Params} {u : ℝ → intervalDomainPoint → ℝ}
    {T Csrc : ℝ}
    (H : UniformSourceCoeffQuadraticDecayOn p u T Csrc) :
    ∃ G2 : ℝ, UniformResolverGradC1On p u T G2
```

Suggested explicit constant:

```lean
G2 = Csrc * (∑' n : ℕ, 1 / (((n.succ : ℝ) * Real.pi) ^ 2))
```

Reason: in `resolverGrad2Real`, the `k = 0` term is zero, and for `k ≥ 1`,

```text
|(resolverCoeff p (u s) k).re| * (kπ)^2
  = |sourceCoeff_k|/(μ+λ_k) * (kπ)^2
  ≤ |sourceCoeff_k|
  ≤ Csrc/(kπ)^2.
```

Thus the second-derivative cosine series is uniformly absolutely bounded by the fixed summable p-series. The derivative bridge is supplied per time by

```lean
resolverGradReal_hasDerivAt_of_sourceDecay (H.sourceDecay s hs0 hsT)
```

where `H.sourceDecay` packages the same uniform `Csrc` into a `SourceCoeffQuadraticDecay` record.
