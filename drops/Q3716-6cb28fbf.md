The exact statement should be:

```lean
/-- A compact-window envelope implies the pointwise eigenvalue-weighted
summability by applying it to the singleton window `[σ,σ]`. -/
theorem hbsum_of_envelope
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    {D : GradientMildSolutionData p u₀}
    (bc : ℝ → ℕ → ℝ)
    (henv : ∀ a b, 0 < a → b < D.T → a ≤ b →
      ∃ E : ℕ → ℝ,
        Summable E ∧
        (∀ n, 0 ≤ E n) ∧
        (∀ σ ∈ Set.Icc a b, ∀ n,
          unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n)) :
    ∀ σ, 0 < σ → σ < D.T →
      Summable (fun n => unitIntervalCosineEigenvalue n * |bc σ n|) := by
  intro σ hσ0 hσT
  obtain ⟨E, hEsum, _hEnn, hdom⟩ := henv σ σ hσ0 hσT le_rfl
  exact hEsum.of_nonneg_of_le
    (fun n => mul_nonneg
      (by unfold unitIntervalCosineEigenvalue; positivity)
      (abs_nonneg _))
    (fun n => hdom σ ⟨le_rfl, le_rfl⟩ n)
```

This uses only standard ingredients already used elsewhere in the repo:

* eigenvalue nonnegativity:

```lean
by unfold unitIntervalCosineEigenvalue; positivity
```

* absolute value nonnegativity:

```lean
abs_nonneg _
```

* singleton-window membership:

```lean
(⟨le_rfl, le_rfl⟩ : σ ∈ Set.Icc σ σ)
```

* summability comparison:

```lean
hEsum.of_nonneg_of_le ...
```

If Lean does not resolve the method form in this exact context, the equivalent

```lean
Compared with `ResolverSourceWindowEnvelopeInputs`, this removes the redundant
pointwise `hbsum`; it is derivable from `henv` on singleton windows. -/
structure ResolverSourceWindowEnvelopeOnlyInputs
    (p : CM2Params) {u₀ : intervalDomainPoint → ℝ}
    (D : GradientMildSolutionData p u₀) where
  bc : ℝ → ℕ → ℝ
  hagree : ∀ σ, 0 < σ → σ < D.T →
    Set.EqOn (intervalDomainLift (D.u σ))
      (fun x => ∑' n, bc σ n * cosineMode n x)
      (Set.Icc (0 : ℝ) 1)
  hliftCont :
    ContinuousOn
      (Function.uncurry
        (fun (σ : ℝ) (x : ℝ) => intervalDomainLift (D.u σ) x))
      (Set.Ioo (0 : ℝ) D.T ×ˢ Set.Icc (0 : ℝ) 1)
  henv : ∀ a b, 0 < a → b < D.T → a ≤ b →
    ∃ E : ℕ → ℝ,
      Summable E ∧
      (∀ n, 0 ≤ E n) ∧
      (∀ σ ∈ Set.Icc a b, ∀ n,
        unitIntervalCosineEigenvalue n * |bc σ n| ≤ E n)
  adotPow : ℝ → ℕ → ℝ
  hderivPow : ∀ σ, 0 < σ → σ < D.T → ∀ n,
    HasDerivAt
      (fun r => cosineCoeffs
        (fun x => p.ν * intervalDomainLift (D.u r) x ^ p.γ) n)
      (adotPow σ n) σ
  hadotPowCont : ∀ n, ContinuousOn (fun σ => adotPow σ n) (Set.Ioo 0 D.T)
  hMdotPow : ∀ a b, 0 < a → b < D.T →
    ∃ Mdot, ∀ σ ∈ Set.Icc a b, ∀ n, |adotPow σ n| ≤ Mdot
```

This is a real residual reduction: it removes one redundant field from the
primitive interface without adding any analytic assumptions.

## 5. Pitfalls checked

### Singleton `Icc` membership

Use:

```lean
(⟨le_rfl, le_rfl⟩ : σ ∈ Set.Icc σ σ)
```

No interval arithmetic is needed.

### `a ≤ b` for singleton windows

Use `le_rfl` when calling `henv σ σ hσ0 hσT le_rfl`.

### Nonnegativity of the dominated sequence

For `Summable.of_nonneg_of_le`, the target sequence is

```lean
fun n => unitIntervalCosineEigenvalue n * |bc σ n|
```

Nonnegativity proof:

```lean
fun n => mul_nonneg
  (by unfold unitIntervalCosineEigenvalue; positivity)
  (abs_nonneg _)
```

The `hEnn` field of the envelope is not needed for `hbsum_of_envelope`, but it is
still needed downstream in `hG1_of_envelope`/`hG2_of_envelope` for the endpoint
cases via `tsum_nonneg hEnn`.  Do not remove it.

### Keeping `henv` exactly as-is

No change to the envelope shape is needed.  It already includes all data needed to
derive `hbsum`.

### No circularity

This tightening remains non-circular.  It only uses the coefficient envelope itself
and elementary summability comparison.  It does not use:

* `GradientMildClassicalFrontierCoreData`;
* `IsPaper2ClassicalSolution`;
* `hpde_u`;
* `HasTimeNeighborhoodSpectralAgreement D.T D.u`.
