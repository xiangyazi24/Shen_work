# Q3749: Hu chart to `LocalRestartWeak` for power K1

## Verdict

**No, not with `œÑ := C.R.offset` as stated.**  A `LocalRestartWeak`-from-Hu-chart lemma is the right real next target, but the proposed field assignment has two hard Lean/mathematical obstructions:

1. `LocalRestartWeak` requires

```lean
hœÑpos : 0 < œÑ
hœÉœÑ : œÑ < œÉ
hrep : ‚àÄ r, r ‚àà Set.Ioo œÑ d -> ...
```

while `HuRestartData` gives only

```lean
C.R.hœÑ : 0 < œÉ - C.R.offset
```

This proves `C.R.offset < œÉ`, but **not** `0 < C.R.offset`.

2. `HuRestartLocalCoverChart.hU` gives representation only on an open neighborhood `C.U` of the center `œÉ`.  Even if `C.R.offset > 0`, choosing `œÑ := C.R.offset` makes `hrep` demand representation on all of `Set.Ioo C.R.offset d`, which can include times below the chart neighborhood `C.U`.  The chart does not provide that.

So the direct constructor must first **shrink to a subwindow inside `C.U`** and then **rebase** the restart coefficients from `C.R.offset` to a new positive time `œÑ` with

```lean
0 < œÑ ‚àß œÑ < œÉ ‚àß œÉ < d ‚àß d < D.T ‚àß Set.Ioo œÑ d ‚äÜ C.U.
```

The corrected construction sets

```lean
base := œÑ - C.R.offset
W    := d - œÑ
a0'  := fun n => localRestartCoeff C.R.a0 C.R.a base n
aC'  := fun s n => C.R.a (base + s) n
```

Then `C.hU r` gives the chart representation with time `r - C.R.offset`, and a rebasing lemma rewrites

```lean
localRestartCoeff C.R.a0 C.R.a (r - C.R.offset) n
=
localRestartCoeff a0' aC' (r - œÑ) n.
```

## First small lemmas to prove

### 1. Subwindow inside one Hu chart

This is pure topology/order.  It is the first compile-friendly lemma.

```lean
/-- Shrink a Hu local cover chart around `œÉ` to a positive two-sided interval
`(œÑ,d)` contained in the chart and in `(0,T)`. -/
theorem exists_positive_subwindow_subset_huRestartLocalCoverChart
    {T : ‚Ñù} {u : ‚Ñù ‚Üí intervalDomainPoint ‚Üí ‚Ñù}
    {Hu : HasTimeNeighborhoodSpectralAgreement T u}
    {œÉ : ‚Ñù} (hœÉ0 : 0 < œÉ) (hœÉT : œÉ < T)
    (C : HuRestartLocalCoverChart T u Hu œÉ) :
    ‚àÉ œÑ d : ‚Ñù,
      0 < œÑ ‚àß œÑ < œÉ ‚àß œÉ < d ‚àß d < T ‚àß
      œÑ ‚àà C.U ‚àß Set.Ioo œÑ d ‚äÜ C.U := by
  -- Use `C.hU_open.mem_nhds C.hœÉ‚ÇÄ_mem` to get an interval around œÉ inside C.U.
  -- Intersect with `Set.Ioo 0 T`; choose small `Œ¥ > 0`, set
  --   œÑ := œÉ - Œ¥/2, d := œÉ + Œ¥/2.
  -- Keep `œÑ ‚àà C.U` because the closed/half-open lower point is inside the chosen ball.
  -- This is the simplest non-analytic lemma.
  exact by
    -- implement with `Metric.mem_nhds_iff` or `isOpen.mem_nhds` + real interval basis
    -- no PDE input needed
    admit
```

When implementing, do not keep the `admit`; replace it with the usual metric-neighborhood shrink.  This lemma is intentionally isolated because it removes the second obstruction above.

### 2. Rebase a restart coefficient

This is the main algebraic atom.  It uses `duhamelSpectralCoeff_general_split_on`.

```lean
/-- Rebase `localRestartCoeff` from absolute restart time `0` to a later local
start `base`.  This is the coefficient identity needed when a Hu chart is valid
only near `œÉ`, not on the whole interval from the original `offset`. -/
theorem localRestartCoeff_rebase
    {a0 : ‚Ñï ‚Üí ‚Ñù} {a : ‚Ñù ‚Üí ‚Ñï ‚Üí ‚Ñù}
    (src : DuhamelSourceTimeC1 a)
    {base q : ‚Ñù} (hbase : 0 ‚â§ base) (hq : 0 ‚â§ q) (n : ‚Ñï) :
    localRestartCoeff a0 a (base + q) n =
      localRestartCoeff
        (fun n => localRestartCoeff a0 a base n)
        (fun s n => a (base + s) n)
        q n := by
  -- Expand both sides.  Use
  --   duhamelSpectralCoeff_general_split_on
  -- with `œÑ := base`, `t := base + q`, `T := base + q`.
  -- Continuity comes from `src.hderiv`:
  --   continuous_iff_continuousAt.2 (fun s => (src.hderiv s n).continuousAt)
  -- restricted to `Set.Icc 0 (base+q)`.
  -- Then finish by `ring` after rewriting exponentials:
  --   exp (-(base+q)Œª) = exp(-qŒª) * exp(-baseŒª).
  exact by
    admit
```

This atom is the precise fix for the failed `œÑ := offset` plan.

  -- Use nonnegativity of `src.envelope n`, derived from `src.henv_bound 0 le_rfl n`,

## Lean caveats

1. **Do not set `œÑ := C.R.offset`.**  It may not be positive, and `Set.Ioo offset d` is not known to lie in `C.U`.

2. **Do not set `aC := C.R.a` after choosing a new `œÑ`.**  The chart formula uses elapsed time `r - C.R.offset`; `LocalRestartWeak` uses elapsed time `r - œÑ`.  You must rebase with

```lean
aC' s n := C.R.a ((œÑ - C.R.offset) + s) n.
```

3. **Use the existing shift lemma.**  `IntervalDuhamelSourceShift.lean` already defines

```lean
DuhamelSourceTimeC1.shift_nonneg
```

so the source shift should not be reproved.

4. **Fill `M/ha‚ÇÄ` for the rebased initial coefficients.**  This is easy to forget.  The original `C.R.ha0` bounds `C.R.a0`, not

```lean
localRestartCoeff C.R.a0 C.R.a (œÑ - C.R.offset).
```

5. **The post-`mkL` power K1 proof should be abstracted.**  Otherwise any Hu-specific theorem will duplicate the long continuity/bound proof from `powerK1_quadruple_of_subtypeCont`.  Extracting

```lean
powerK1_quadruple_of_localRestartWeak
```

is not a headline wrapper; it removes the chi-zero construction dependency from the field-level K1 proof.

## Bottom line

The real next target is:

```text
HuRestartLocalCoverChart + shrink + rebase  ==>  LocalRestartWeak
```

not the unrebased `œÑ := offset` version.  The first lemma to prove is either the pure topological subwindow lemma or the coefficient rebase lemma; the coefficient rebase lemma is the first analytic floor.  After that, the field-removal theorem for `ResolverSourceWindowHuNoEnvelopeInputs` follows by reusing `LocalRestartWeak.hasDerivAt_powerCoeff` and the chi-free tail abstracted from `powerK1_quadruple_of_subtypeCont`.
