First define a sine heat value:

```lean
noncomputable def unitIntervalSineHeatValue
    (r : ℝ) (a : ℕ → ℝ) (x : ℝ) : ℝ :=
  ∑' n : ℕ,
    Real.exp (-r * unitIntervalCosineEigenvalue n) * a n *
      Real.sin ((n : ℝ) * Real.pi * x)
```

Then prove the pointwise kernel theorem:

```lean
theorem intervalNeumannConjugateKernel_eq_sineKernel_nat
    {r : ℝ} (hr : 0 < r) (x y : ℝ) :
    intervalNeumannConjugateKernel r x y =
      -∑' n : ℕ,
        ShenWork.IntervalConjugateCosineSeries.neumannCosineWeight n *
          Real.exp (-r * unitIntervalCosineEigenvalue n) *
          Real.sin ((n : ℝ) * Real.pi * x) *
          Real.sin ((n : ℝ) * Real.pi * y)
```

Proof plan: unfold `intervalNeumannConjugateKernel`; rewrite it as `-gaussianLatticeSum r (x-y) + gaussianLatticeSum r (x+y)`; use `gaussianLatticeSum_poisson` twice; combine the two integer sums using the existing summability lemmas; apply the trig identity for `cos(n*pi*(x+y)) - cos(n*pi*(x-y))`; fold the even integer sum to natural modes using the same `tsum_int_eq_zero_add_two_mul_tsum_pnat` pattern already used in the repo.  The zero mode vanishes and positive modes have weight `2`.

This theorem is the smallest likely compile target because it is pointwise and does not yet require interval-integral interchange.

## Normalization and sign checks

* `sineCoeffs` already has zero mode `0` and positive-mode factor `2`; do not add another factor of `2` in `unitIntervalSineHeatValue`.
* The pointwise Ktilde expansion has a leading minus.  Therefore the value theorem should be stated with a leading minus on the integral.
* `sqrt(lam)` / `k*pi` belongs to the B-form divergence-mode identity, not to the direct Ktilde value theorem.
* `bFormSourceCoeffs` is correct for `intervalConjugateKernelOperator`; it is the wrong coefficient object for the Task294 gradient leg.
