# Q3793 route audit: lowering the Task316 conjugate/B-form provider

## Verdict

The smallest real next Lean module is **not** another `hpde_u` wrapper.  It should be a public restart-coefficient summability producer:

`ShenWork/Paper2/IntervalBFormRestartCoeffSummability.lean`

Its core theorem should prove

```lean
Summable (fun n => |localRestartCoeff aInit aB t n|)
```

from an initial coefficient bound and `DuhamelSourceTimeC1On aB 0 T` (plus a global `DuhamelSourceTimeC1` corollary).  This directly removes Task316's explicit `hB_global_summable` assumption, and the proof is already present only as private/inline work inside the existing B-form `hpde_u` banked proofs.  Extracting it is a genuine analytic input theorem: it uses the homogeneous heat trace summability plus the Duhamel envelope estimate, and it does not consume `hpde_u`, `IsPaper2ClassicalSolution`, the final regularity bootstrap, `GradientMildSolutionData`, or any `hgradB` bridge.

There is an even smaller projection from `ChemDivCosineFourierDataIoo` to the `hdiv_rep` source-bridge hypothesis, but I would not make that the next move: it is mostly field projection.  The restart summability lemma is the smallest non-wrapper theorem that actually lowers a Task316 analytic hypothesis toward the committed banked producers.

    (haInit : ∀ n, |aInit n| ≤ MInit) :
    Summable (fun n : ℕ => |localRestartCoeff aInit aB t n|) := by
  have hhom : Summable (fun n : ℕ =>
      |Real.exp (-t * unitIntervalCosineEigenvalue n) * aInit n|) := by
    refine Summable.of_nonneg_of_le (fun n => abs_nonneg _) (fun n => ?_)
      ((ShenWork.IntervalSemigroupComposition.expEigSummable ht).mul_right
        MInit)
    rw [abs_mul, abs_of_pos (Real.exp_pos _)]

  have hduh : Summable (fun n : ℕ =>
      |duhamelSpectralCoeff aB t n|) := by

      (hsrc.henv_summable.mul_left t)

    calc ‖∫ s in (0:ℝ)..t,
          Real.exp (-(t - s) * unitIntervalCosineEigenvalue n) * aB s n‖
        ≤ hsrc.envelope n * |t - 0| := by
          apply intervalIntegral.norm_integral_le_of_norm_le_const
          intro s hs
          rw [Set.uIoc_of_le ht.le] at hs
          rw [Real.norm_eq_abs, abs_mul, abs_of_pos (Real.exp_pos _)]
          calc Real.exp (-(t - s) * unitIntervalCosineEigenvalue n) *
                |aB s n|
              ≤ 1 * |aB s n| := by
                apply mul_le_mul_of_nonneg_right _ (abs_nonneg _)
                rw [Real.exp_le_one_iff]
                have hts : 0 ≤ t - s := by linarith [hs.2]
                have hlam : 0 ≤ unitIntervalCosineEigenvalue n := by
                  unfold unitIntervalCosineEigenvalue
                  positivity
                nlinarith [mul_nonneg hts hlam]

                hsrc.henv_bound s ⟨le_of_lt hs.1, le_trans hs.2 htT⟩ n
        _ = t * hsrc.envelope n := by
          rw [sub_zero, abs_of_pos ht]
          ring

    (fun n => abs_nonneg _)

## Why this is the right next theorem

`IntervalConjugatePdeUProvider.lean` currently composes two real pieces: the endpoint-safe B-form source bridge produces the global B-form cosine representation, and `IntervalDomainPdeUGeneralChiProvider.lean` converts that B-form source/spectral data into the PDE identity.  The only purely restart-analytic hypothesis still exposed by Task316 is `hB_global_summable`.  The proposed lemma proves exactly that from source-time data and initial coefficient boundedness.

This theorem is upstream of the PDE core.  It only uses:

* the definition of `localRestartCoeff`;
* heat-trace summability for the homogeneous part;
* the `DuhamelSourceTimeC1On` envelope bound for the Duhamel part;
* triangle inequality.

It therefore cannot be circular with `hpde_u`, `IsPaper2ClassicalSolution`, or the direct regularity assembly.

## Existing producers versus missing inputs

### Already available or effectively available

* `hfix` for the conjugate Picard limit is produced by `conjugateMildSolutionData_of_data D`, whose `hmild` field is upstream Picard fixed-point data.
* `hu₀_bound` is already a banked input in the B-form bank modules (`MInit`/`haInit`).
* `hB_global` is produced on the correct route by `conjugatePicardLimit_hB_global_of_open_sourceBridgeRepresentativeSubtypeLogisticData`; it uses the open representative source bridge and does not route through `GradientMildSolutionData` or `hgradB`.
* `hB_global_summable` is exactly what the proposed theorem should make public.  The proof is already present inline in the banked `hpde_u` path, but trapped inside a final PDE theorem.
* `hsource_split` for `bFormSourceCoeffs` is definitionally `rfl`.
* `bFormSource_duhamelSourceTimeC1` and `bFormSource_duhamelSourceTimeC1On` already combine logistic and chem-div source regularity by add/scalar closure.
* `coupledChemDivSource_duhamelSourceTimeC1` already produces the chem-div source global `DuhamelSourceTimeC1` from raw weak-H²/time-C¹ coefficient data.
* `LogisticCosineFourierData` can be built in the existing banked path by `logisticCosineFourierData_constExtend` from the bank's logistic continuity and Fourier summability fields.
* `ChemDivCosineFourierDataIoo` is the right chem-div Fourier package.  The endpoint-safe builder `chemDivCosineFourierDataIoo_of_repr` is already present, and the banked B-form path carries `hchemIoo` directly.
* `hdiv_rep` can be obtained from `ChemDivCosineFourierDataIoo` without any PDE conclusion: take `Gdiv := H.representative`, use `H.continuous_representative.continuousOn`, and reverse the `Ioo` agreement.  This is safe, but it is only a small adapter/projection.

### Genuinely missing or not yet publicly produced

* A global `DuhamelSourceTimeC1` for `bFormSourceCoeffs p (conjugatePicardLimit p u₀ T)` is not supplied by the current banked B-form path; the banked path supplies `DuhamelSourceTimeC1On ... 0 T`.  Do not pretend `On` gives global two-sided time C¹.  For Task316 unchanged, either prove a real global source tower or add an `On` variant of the general-χ provider.
* I did not find a committed direct global producer for `DuhamelSourceTimeC1 (coupledLogisticSourceCoeffs p u)` analogous to the chem-div producer.  The B-form closure theorem can use it once available, but it is not itself the logistic source proof.
* `hB_int` and `hlog_int` are still exposed by the open source bridge.  They look analytically dischargeable from semigroup/kernel boundedness plus source continuity/envelopes, but I did not find a public producer in the inspected files.
* `hchem_cont`, `hlog_cont`, `hlog_bound`, and `hchem_bound` are still source-bridge side conditions.  Some should be derivable from existing Fourier/source packages, but there is no obvious committed theorem that packages them for the conjugate limit.
* `hQderiv` is a real spatial derivative statement for the chem flux.  It must be proved from pre-`hpde_u` spatial resolver/u regularity, not from `IsPaper2ClassicalSolution` or any final bootstrap.
* `ChemDivCosineFourierDataIoo` itself still needs a continuous interior-agreeing representative with Fourier summability.  The builder is present; the analytic construction of the representative is the real source/Fourier data work when it is not banked.
