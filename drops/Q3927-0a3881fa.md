# Q3927 — coefficient bootstrap for truncated Picard positive-time ℓ¹ regularity

## Executive answer

The raw Duhamel estimate is not enough:

```text
chemDiv_k(s) = kπ · sineInner(Q(s), k) = O(k)
Duhamel gain = 1/λ_k = O(1/k²)
coefficient contribution = O(1/k)  -- not ℓ¹
```

So the first pass of the coefficient ladder cannot start from mere continuity of the chemotaxis flux.

The correct route is a **positive-time Picard bootstrap on compact time windows**:

1. Work on a window `[τ, T₁]` with `0 < τ < T₁ ≤ T`, usually `τ = t / 2` for a fixed target time `t`.
2. Prove a uniform `W^{1,1}` / BV bound for the chemotaxis flux on that window.
3. BV gives
   ```text
   |kπ · sineInner(Q(s), k)| ≤ C
   ```
   uniformly in `s ∈ [τ,T₁]`; hence the full truncated source has a `WindowSourceEnvelope 0`.
4. Feed this bounded source envelope into the existing coefficient ladder:
   ```text
   WindowSourceEnvelope 0
     → ladder_pass_gain_envelope repeated
     → WindowCoefficientEnvelope pass4
     → eigenvalue_weighted_summable_of_pass4
   ```
5. The pass-4 envelope gives positive-time coefficient `ℓ¹`, gradient-weighted `ℓ¹`, and the laplacian/test summability fields.
6. Establish the flux BV seed uniformly for Picard iterates, pass to the limit using the existing contraction/uniform convergence of truncated Picard iterates, then instantiate the pass-4 envelope for the limit.

This route is non-circular because the bootstrap is performed on **Picard iterates** on a positive-time window.  Iterate 0 is heat-smoothed and analytic at positive time; the induction step is:

```text
flux W¹,¹ bound for iterate n
  → bounded full source for iterate n+1
  → pass-4 coefficient envelope for iterate n+1
  → C¹ / gradient bound for iterate n+1
  → flux W¹,¹ bound for iterate n+1
```

The constants are uniform on `[τ,T₁]` because the truncated Picard construction carries a uniform ball bound and the window is bounded away from time `0`.

## Answers to the specific questions

### (A) Uniform ℓ¹ bounds for Picard iterates?

Yes, but prove them on compact positive-time windows and by induction through a flux-BV seed, not by the raw continuity estimate.

The induction seed is iterate 0:

```lean
truncatedConjugatePicardIter p u₀ 0 t x
  = intervalFullSemigroupOperator t (intervalDomainLift u₀) x.1
```

At `t ≥ τ > 0`, the heat coefficients have exponential decay, so all coefficient envelopes needed by the ladder are available.  This gives smoothness and flux BV for iterate 0.

The induction step uses:

```text
C¹ u_n on [τ,T₁]
  ⇒ truncatedChemFlux(u_n) ∈ W¹,¹ uniformly
  ⇒ chemDiv source coefficients bounded uniformly
  ⇒ full source has WindowSourceEnvelope 0
  ⇒ pass-4 WindowCoefficientEnvelope for u_{n+1}
  ⇒ C¹ u_{n+1}
```

Then use truncated Picard convergence to transfer the pass-4 envelope to the limit.

### (B) Can Lipschitz regularity come from the heat kernel gradient estimate?

Not directly for the full B-form mild equation.  The logistic heat leg is fine:

```text
‖∂ₓ S(t-s)F(s)‖∞ ≤ C (t-s)^(-1/2) ‖F(s)‖∞
```

but the chemotaxis B-form/conjugate leg already contains a derivative kernel.  Differentiating it once more produces a second heat-kernel derivative with a `(t-s)^(-1)` singularity, which is not integrable near `s=t` unless the flux has extra spatial regularity.

So the safe route is not:

```text
bounded source ⇒ direct Lipschitz u
```

but rather:

```text
flux BV ⇒ bounded chemDiv coefficients ⇒ coefficient ladder ⇒ Lipschitz u
```

Lipschitz regularity is part of the bootstrap output, not the raw starting point.

### (C) Does the flux derivative blow up because of Hessian dependence?

No, if it is organized correctly.  The derivative of

```text
positivePart(u) * resolverGrad / (1 + R)^β
```

uses:

```text
(positivePart u)' * resolverGrad / (1+R)^β
+ positivePart u * (resolverGrad / (1+R)^β)'
```

The second term involves the **second derivative of the elliptic resolver**, not the Hessian of `u`.  In 1D, the resolver equation gives:

```text
R'' = μ R - ν positivePart(u)^γ
```

so `R''` is bounded from `R` and bounded `u`.  The only derivative of `u` needed in the flux derivative is the first derivative from `(positivePart u)'`.  Thus a `C¹`/Lipschitz bound for `u`, plus boundedness of `u` and resolver bounds, suffices for a flux `W¹,¹` bound.

### (D) Does `IntervalCompactSliceGradientBounds.lean` provide a template?

Yes.  Use it as a template after replacing the source seed.

For the non-truncated/logistic-only path, the source envelope starts from logistic coefficients.  For the truncated chemotaxis path, first prove the flux-BV lemma that turns the chemotaxis divergence coefficients into a bounded source envelope.  Then the rest is the same pattern:

```text
bounded source envelope
  → ladder_pass_gain_envelope
  → WindowCoefficientEnvelope 4
  → eigenvalue_weighted_summable_of_pass4
  → gradient/laplace summability
```

Do not describe the chemotaxis part as a bounded perturbation until after proving flux BV.  Once BV gives `|chemDiv_k| ≤ C`, it is a bounded perturbation of the logistic source and the ladder machinery can be reused directly.

# Final recommendation

The minimal non-circular route is:

```text
Picard iterate positive-time C¹ bootstrap
  → flux W¹,¹ uniformly on [t/2,t]
  → chemDiv source coefficients bounded
  → WindowSourceEnvelope 0 for full truncated source
  → existing coefficient ladder pass4
  → eigenvalue_weighted_summable_of_pass4
  → coeff ℓ¹, gradient ℓ¹, lap_summable, deriv_l1
  → pass to truncated Picard limit
```

Do not attempt to start the ladder from mere continuity of the chemotaxis flux.  Do not rely on a direct heat-gradient estimate for the chemotaxis B-form leg.  Use flux `W¹,¹` as the missing seed that converts the dangerous `kπ` factor into a bounded source coefficient.
