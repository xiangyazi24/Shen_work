# Q3972: integrated weak ODE identity for the interval Neumann resolver

## Bottom line

The clean Lean route is the **spectral testing route**:

1. take the real part of the coefficient equation
   \[
   (\mu + \lambda_k)c_k = a_k,
   \qquad \lambda_k = (k\pi)^2,
   \]
2. test it against the interval indicator `1_[a,b]`, i.e. against the scalar
   \[
   I_k(a,b) := \int_a^b \cos(k\pi x)\,dx,
   \]
3. identify the `-λ_k c_k I_k(a,b)` sum with the endpoint difference of the already-defined gradient sine series.

Do **not** try to prove `source x = ∑' k, a_k cos(kπx)` pointwise.  The source is only naturally an `L²` object at this stage, and the right formal statement is the pairing identity
\[
\int_a^b source(x)\,dx
  = \sum_k a_k \int_a^b \cos(k\pi x)\,dx.
\]
This is the only serious Parseval/completeness atom.  Everything else is algebra, elementary trigonometric integration, and `Summable` bookkeeping.

In repo notation, the identity should use the spectral gradient
`intervalNeumannResolverRGrad p u`, not initially `deriv (intervalDomainLift (intervalNeumannResolverR p u))`.  If a later consumer needs the actual derivative, bridge it with `resolverR_hasDerivAt_grad` from `ShenWork.PDE.IntervalResolverGradientBridge` under the stronger gradient-majorant hypotheses.

## Atom 2: elementary cosine-mode integration

You want the version with `λ_k` already multiplied in, because it avoids a separate `k = 0` division problem.

```lean
lemma eigen_cos_intervalIntegral
    (k : ℕ) (a b c : ℝ) :
    (∫ x in a..b, lam k * c * phi k x) =
      c * ((k : ℝ) * Real.pi) *
        (Real.sin ((k : ℝ) * Real.pi * b) -
         Real.sin ((k : ℝ) * Real.pi * a)) := by
  by_cases hk : k = 0
  · subst k
    simp [lam, phi, unitIntervalCosineMode]
  · set κ : ℝ := (k : ℝ) * Real.pi with hκ
    have hκ_ne : κ ≠ 0 := by
      have hk' : (k : ℝ) ≠ 0 := by exact_mod_cast hk
      exact mul_ne_zero hk' (ne_of_gt Real.pi_pos)
    have hlam : lam k = κ ^ 2 := by
      simp [lam, unitIntervalNeumannSpectrum, hκ]
      ring
    -- Preferred tactic: integrate `cos (κ*x)` as the derivative of `sin(κ*x)/κ`.
    -- Depending on the Mathlib version, use either
    -- `intervalIntegral.integral_eq_sub_of_hasDerivAt` or
    -- `intervalIntegral.integral_deriv_eq_sub`.
    have hcos :
        (∫ x in a..b, Real.cos (κ * x)) =
          (Real.sin (κ * b) - Real.sin (κ * a)) / κ := by
      -- Skeleton:
      --   have hderiv : ∀ x, HasDerivAt (fun y => Real.sin (κ*y) / κ)
      --       (Real.cos (κ*x)) x := ... using `(Real.hasDerivAt_sin _).comp ...`
      --   simpa [div_eq_mul_inv, hκ_ne] using
      --     intervalIntegral.integral_eq_sub_of_hasDerivAt hderiv ...
      sorry
    calc
      (∫ x in a..b, lam k * c * phi k x)
          = lam k * c * ∫ x in a..b, Real.cos (κ * x) := by
              simp [phi, unitIntervalCosineMode, hκ, intervalIntegral.integral_const_mul]
      _ = c * κ * (Real.sin (κ * b) - Real.sin (κ * a)) := by
              rw [hcos, hlam]
              field_simp [hκ_ne]
              ring
      _ = c * ((k : ℝ) * Real.pi) *
            (Real.sin ((k : ℝ) * Real.pi * b) -
             Real.sin ((k : ℝ) * Real.pi * a)) := by
              simp [hκ]
```

The `sorry` above is not mathematically hard; it is just the usual trig-antiderivative proof.  The key trick is: prove the antiderivative for `κ ≠ 0`, while the `k = 0` case is discharged before any division appears.

lemma resolver_coeff_abs_summable_of_sourceL2
    (p : CM2Params) {u : intervalDomainPoint → ℝ}

    intervalNeumannResolver_denom_pos p k

## Atom 6: endpoint gradient difference equals the integrated `-λC` sum

Use the existing endpoint sine-series summability.  No source pairing is involved here.

```lean
lemma resolverGrad_endpointDiff_eq_neg_lam_tsum
    (p : CM2Params) (u : intervalDomainPoint → ℝ)
    {a b : ℝ} (ha : a ∈ Set.Icc (0 : ℝ) 1) (hb : b ∈ Set.Icc (0 : ℝ) 1)
    (hl2 : Summable fun k : ℕ => (A p u k) ^ 2) :
    resolverGCoord p u b - resolverGCoord p u a =
      ∑' k : ℕ,
        (-(lam k * C p u k * (∫ x in a..b, phi k x))) := by
  have hgb : Summable fun k : ℕ => C p u k * psi k b := by
    simpa [C, psi] using
      resolver_sineSeries_summable_of_sourceL2 p (u := u) hl2 b
  have hga : Summable fun k : ℕ => C p u k * psi k a := by
    simpa [C, psi] using
      resolver_sineSeries_summable_of_sourceL2 p (u := u) hl2 a
  have hdiff : Summable fun k : ℕ => C p u k * psi k b - C p u k * psi k a :=
    hgb.sub hga
  have hterm : ∀ k : ℕ,
      C p u k * psi k b - C p u k * psi k a =
        -(lam k * C p u k * (∫ x in a..b, phi k x)) := by
    intro k
    have h := eigen_cos_intervalIntegral k a b (C p u k)
    -- `h` gives `∫ lam*C*phi = C*kπ*(sin_b-sin_a)`.
    -- Unfold `psi` and rearrange signs.
    simp [psi] at *
    rw [← h]
    ring
  calc
    resolverGCoord p u b - resolverGCoord p u a
        = (∑' k : ℕ, C p u k * psi k b) -
          (∑' k : ℕ, C p u k * psi k a) := by
            simp [resolverGCoord, ha, hb, intervalNeumannResolverRGrad, C, psi]
    _ = ∑' k : ℕ, (C p u k * psi k b - C p u k * psi k a) := by
            exact (hgb.tsum_sub hga).symm
    _ = ∑' k : ℕ,
          (-(lam k * C p u k * (∫ x in a..b, phi k x))) := by
            exact tsum_congr hterm
```

The `k = 0` mode vanishes automatically: `psi 0 y = 0` and `lam 0 = 0`.

## Final theorem skeleton

After the atoms above, the final proof is short.

```lean
theorem resolver_integratedWeakODE
    (p : CM2Params) (u : intervalDomainPoint → ℝ)
    {a b : ℝ} (ha0 : 0 ≤ a) (hab : a ≤ b) (hb1 : b ≤ 1)
    (hsrcCoeffL2 : Summable fun k : ℕ => (A p u k) ^ 2)
    (hsrcInt : IntervalIntegrable
      (fun x : ℝ => ((resolverSourceCoord p u x : ℝ) : ℂ)) volume 0 1)
    (hsrcL2 : MemLp
      (unitIntervalEvenReflection
        (fun x : ℝ => ((resolverSourceCoord p u x : ℝ) : ℂ))) 2
      (volume.restrict (Set.Ioc (-1 : ℝ) 1)))
    (hsrcSq : IntervalIntegrable
      (fun x : ℝ => ‖((resolverSourceCoord p u x : ℝ) : ℂ)‖ ^ 2) volume 0 1) :
    resolverGCoord p u b - resolverGCoord p u a =
      ∫ x in a..b, (p.μ * resolverRCoord p u x - resolverSourceCoord p u x) := by
  have ha01 : a ∈ Set.Icc (0 : ℝ) 1 := ⟨ha0, le_trans hab hb1⟩
  have hb01 : b ∈ Set.Icc (0 : ℝ) 1 := ⟨le_trans ha0 hab, hb1⟩

  set I : ℕ → ℝ := fun k => ∫ x in a..b, phi k x with hI

  have hG : resolverGCoord p u b - resolverGCoord p u a =
      ∑' k : ℕ, (-(lam k * C p u k * I k)) := by
    simpa [I, hI] using
      resolverGrad_endpointDiff_eq_neg_lam_tsum
        p u ha01 hb01 hsrcCoeffL2

  have hR : (∫ x in a..b, resolverRCoord p u x) =
      ∑' k : ℕ, C p u k * I k := by
    simpa [I, hI] using
      resolverR_intervalIntegral_eq_tsum
        p u ha0 hab hb1 hsrcCoeffL2

  have hS : (∫ x in a..b, resolverSourceCoord p u x) =
      ∑' k : ℕ, A p u k * I k := by
    simpa [I, hI] using
      resolverSource_intervalIntegral_eq_tsum
        p u ha0 hab hb1 hsrcInt hsrcL2 hsrcSq

  have hcoeff : ∀ k : ℕ,
      -(lam k * C p u k * I k) = (p.μ * C p u k - A p u k) * I k := by
    intro k
    have hk := resolverCoeff_rearranged p u k
    nlinarith

  have hsum_alg :
      (∑' k : ℕ, (p.μ * C p u k - A p u k) * I k) =
        p.μ * (∑' k : ℕ, C p u k * I k) -
          (∑' k : ℕ, A p u k * I k) := by
    -- Summability inputs:
    -- * `C_k*I_k` from `resolver_coeff_abs_summable_of_sourceL2` and bounded `I_k`.
    -- * `A_k*I_k` from the source-pairing lemma / Cauchy--Schwarz.
    -- Then use `Summable.tsum_sub`, `Summable.tsum_mul_left`, and `tsum_congr`.
    sorry

  calc
    resolverGCoord p u b - resolverGCoord p u a
        = ∑' k : ℕ, (-(lam k * C p u k * I k)) := hG
    _ = ∑' k : ℕ, (p.μ * C p u k - A p u k) * I k := by
          exact tsum_congr hcoeff
    _ = p.μ * (∑' k : ℕ, C p u k * I k) -
          (∑' k : ℕ, A p u k * I k) := hsum_alg
    _ = p.μ * (∫ x in a..b, resolverRCoord p u x) -
          (∫ x in a..b, resolverSourceCoord p u x) := by
          rw [hR, hS]
    _ = ∫ x in a..b, (p.μ * resolverRCoord p u x - resolverSourceCoord p u x) := by
          -- Use `intervalIntegral.integral_const_mul` and
          -- `intervalIntegral.integral_sub`.  Integrability of `resolverRCoord`
          -- follows from the absolute-convergent series atom; source integrability
          -- follows from `hsrcInt` after taking real parts.
          sorry

end
end ShenWork.IntervalResolverWeakODE

    (∑' k : ℕ, (p.μ * C p u k - A p u k) * I k)
        = ∑' k : ℕ, (p.μ * (C p u k * I k) - A p u k * I k) := by
            refine tsum_congr ?_
            intro k
            ring
    _ = (∑' k : ℕ, p.μ * (C p u k * I k)) -
        (∑' k : ℕ, A p u k * I k) := by
            exact hμCI.tsum_sub hAI
    _ = p.μ * (∑' k : ℕ, C p u k * I k) -
        (∑' k : ℕ, A p u k * I k) := by
            rw [Summable.tsum_mul_left]
```

## Why this is better than a pointwise `R''` route

The identity you want is weak/integrated.  The spectral coefficient equation is already exactly the weak elliptic equation.  Testing it against `1_[a,b]` gives the integrated identity without requiring a pointwise second derivative of `R`, without proving pointwise convergence of the source cosine series, and without upgrading the source beyond `L²`.

If you already have `SourceCoeffQuadraticDecay` and want a classical proof, then `IntervalResolverSpatialC2.resolverR_contDiff_two` plus the fundamental theorem of calculus is shorter.  But for the weak resolver framework, the spectral-pairing proof above is the one that should be feasible and robust in Lean 4.
