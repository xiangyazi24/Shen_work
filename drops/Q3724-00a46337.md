## 2. Is finite-cover `henv` from `Hu` feasible?

Yes, but do not start with the full compact-window theorem. There is a subtle choice issue.

`huRestartCoeff Hu σ n` is chosen independently for every `σ` using `huRestartData Hu σ ...`. A local restart chart from `Hu.exists_data σ₀ ...` gives coefficients valid for nearby times, but those chart coefficients are not definitionally the same as the separately chosen `huRestartCoeff Hu σ n` at nearby `σ`.

Therefore the first Lean task should be a coefficient identity, not the finite-cover theorem.

Recommended new file:

`ShenWork/Paper2/IntervalHuRestartCoeffIdentity.lean`

- `ShenWork.Paper2.IntervalResolverSourceWindowHuCoeffInputs`
- `ShenWork.Paper2.IntervalBFormRestart`

`ResolverSourceWindowInput.cosineCoeffs_eq_huRestartCoeff`

`∀ σ, 0 < σ → σ < T → ∀ k, cosineCoeffs (intervalDomainLift (u σ)) k = huRestartCoeff Hu σ k`.

Proof route:

1. Use `huRestartCoeff_agree Hu` for representation on `[0,1]`.
