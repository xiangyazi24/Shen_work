# Q3483 audit: next non-circular H¹ route-C step after Tasks 90--93

## Executive verdict

Task93 is a useful frontier, but it is still an interface frontier, not a PDE estimate. Its `H1PhysicalRHSYoungScalarZeroMajorantsBefore` predicate is non-circular **only when** its four time-majorant functions are produced from primitive zero-window spatial estimates. The interface itself does not consume bounded-before, full RHS integrability, derivative integrability, or the strict/initial route. The danger is not in the Task93 lowering theorem; the danger is in a future producer smuggling one of those downstream packages into the hypotheses used to fill `Glap`, `Gtaxis`, `Guvxx`, `Greact`, or `hRHS_meas`.

The best Task94 frontier is therefore not another global route wrapper. It should expose the next strictly local analytic layer: component-square zero-window data and/or a generic interval-integral Young/Cauchy estimate. That moves the obligation from “bound the physical scalar RHS terms” to “bound explicit spatial square profiles on one zero window,” which is much harder to accidentally prove from the H¹ route itself.

## 1. Hostile audit of Task93

### What is honestly non-circular

Verified from the current files:

* `H1PhysicalRHSYoungScalarZeroMajorantsBefore` asks for one `δ` with `0 < δ` and `δ < T`, integrable nonnegative functions `Glap`, `Gtaxis`, `Guvxx`, `Greact` on `(0,δ]`, assembled-RHS measurability on `(0,δ]`, and a.e. bounds
  * `‖lapL2sq u r‖ ≤ Glap r`,
  * `‖H1PhysicalTaxisX p u v r‖ ≤ (1/2) * Glap r + Gtaxis r`,
  * `‖H1PhysicalUvxxX p u v r‖ ≤ (1/2) * Glap r + Guvxx r`,
  * `‖H1PhysicalReactX p u r‖ ≤ (1/2) * Glap r + Greact r`.
* `H1PhysicalRHSAdditiveNonnegScalarZeroMajorantsBefore_of_young` is just algebra/integrability closure: it replaces each product term majorant by `(1/2) * Glap + remainder` and preserves nonnegativity/integrability.
* `H1PhysicalRHSZeroWindowMajorantBefore_of_additiveScalarMajorants` is term-sum norm algebra using `H1IdentityRHSValue_norm_le_scalar_sum`.
* `H1PhysicalRHSStrictInitialRouteBefore_of_youngScalar_zeroWindow` then adds separately supplied identity, sqrt bounds, and strict positive-time continuity.

So Task93 is non-circular **as an adapter**. It gives a clean target for a producer that knows only local-in-time integrability of the lap term and local square remainders.

### The hidden hazards

1. **The `hRHS_meas` field is easy to fill tautologically.** It must not be obtained from `H1IdentityRHSInitialWindowIntegrableBefore`, `H1IdentityRHSIntegrableBefore`, `H1PhysicalRHSInitialWindowMajorantBefore`, or the strict/initial route. It should be derived from component measurability of `lapL2sq`, `H1PhysicalTaxisX`, `H1PhysicalUvxxX`, and `H1PhysicalReactX`.

2. **`Glap` is the load-bearing field.** If `Glap` is produced from `H1EnergyDerivativeInitialWindowIntegrableBefore`, full RHS integrability, or bounded-before, the proof is circular. The source of `Glap` should be primitive local `u_xx` square-integrability / lap-dissipation data on `(0,δ]`, not the H¹ derivative route.

3. **The global initial-window majorant theorem contains a deliberate tautological repackaging step.** `H1IdentityRHSInitialWindowMajorantBefore_of_zeroWindow_strict` first derives initial-window integrability from the zero window plus strict continuity and then uses `H1IdentityRHSInitialWindowMajorantBefore_of_initialWindowIntegrable`, which majorizes the RHS by itself. This is acceptable only as a downstream bridge after the independent zero-window estimate. It is not a source lemma for producing zero-window data.

4. **`H1PhysicalRHSSqrtBoundsBefore` should not be used to prove Task93.** It is a downstream pointwise DI package for the same H¹ route. Even if it does not syntactically assert bounded-before, it is too route-facing: using it to prove the zero-window majorants would blur the separation between the sup-bound differential inequality and the RHS-integrability producer.

## 2. Best Task94 frontier

### Recommended order

Do **not** first write a producer that directly proves scalar estimates for `H1PhysicalTaxisX`, `H1PhysicalUvxxX`, and `H1PhysicalReactX` from broad “bounds before `T`” hypotheses. That invites accidentally using global bounded-before-type information.

The less circular order is:

1. Add a component measurability helper for the assembled RHS.
2. Add a component-square zero-window interface and a lowering theorem from that interface to Task93.
3. Separately prove the generic interval-integral Young/Cauchy lemma, then specialize it to the three physical scalar integrals.

This order keeps the real analytic obligation explicit: produce local integrability of concrete square profiles and local lap-square integrability on one `(0,δ]`.

## 4. The hard analytic task for a prover/subagent

After Task94-A/B/C, the real PDE target should be:


Do not make this too broad in the first attempt. A good subagent spec is:

*Given a single zero window `(0,δ]`, prove time-integrability of `lapL2sq u` and of the three square profiles `H1PhysicalTaxisPartSq`, `H1PhysicalUvxxPartSq`, `H1PhysicalReactPartSq`, plus the spatial product Young estimates, from primitive local solution estimates only.*

Honest primitive inputs include:

* a local zero-window `δ` from local existence/classical regularity, not from bounded-before;
* local lower bound for `1 + v` on `[0,δ] × [0,1]`, coming from positive initial data / local continuity, not from global bounded-before;
* local `L∞` bounds for `u`, `v`, `v_x`, and powers of `u` on `[0,δ] × [0,1]`, again local only;
* local `H¹` seed or zero-start continuity for `H1energy u`, sufficient to show `r ↦ ∫ (u_x)^2` is integrable near zero;
* primitive `u_xx` square time-integrability on `(0,δ]`, i.e. `IntervalIntegrable (lapL2sq u) volume 0 δ`, from local strong-solution/H² data or an independent dissipation estimate;
* spatial integrability/measurability facts for the physical non-lap factors.

## 5. Scalar-product estimates vs component-square majorants

The next producer should **first define component-square zero-window majorants**, then prove scalar product estimates as local lemmas that lower those squares to Task93.

Reason: direct scalar estimates like

```lean
‖H1PhysicalTaxisX p u v r‖ ≤ ((1 : ℝ) / 2) * Glap r + Gtaxis r
```

are too easy to satisfy with opaque `Gtaxis` from a downstream RHS majorant. By contrast, a square profile such as

```lean
fun r => ∫ x in (0 : ℝ)..1,
  (H1PhysicalChemTaxisPart p u v r x) ^ 2
```

forces the proof to talk about explicit physical factors. That is the correct non-circular frontier. The scalar product estimate should be a theorem from those square profiles, not the primitive source hypothesis.

## 7. Final warning: the tautology boundary

The red line is simple:

* It is fine to use zero-window majorants plus strict positive-time continuity to derive full initial-window integrability.
* It is not fine to use full initial-window integrability to prove the zero-window majorants.
* It is fine to use the H¹ identity after RHS integrability has been independently produced to obtain derivative integrability.
* It is not fine to use derivative integrability to produce RHS integrability or RHS majorants.
* It is fine to carry local zero-window `u_xx` square-integrability from independent local strong-solution regularity.
* It is not fine to carry any route-level package whose only known consumer is the bounded-before wrapper and then use it to fill Task93.

Task94 should therefore be a square-profile/interval-Young frontier, not another strict/initial route constructor.
