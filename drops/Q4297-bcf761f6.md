# 1. Direct coefficient-decay thresholds

Write

\[
\lambda_k=(k\pi)^2\asymp k^2,
\qquad k\ge 1.
\]

Suppose a coefficient sequence satisfies

\[
|d_k|\le C k^{-p}.
\]

Then

\[
\sum_{k\ge1}\lambda_k|d_k|
\lesssim
C\sum_{k\ge1}k^{2-p}.
\]

This converges exactly when

\[
2-p<-1,
\qquad\text{i.e.}\qquad p>3.
\]

Therefore:

| Goal for the same coefficient sequence | Required real decay exponent | Minimum integer envelope order |
|---|---:|---:|
| `∑ |d_k| < ∞` | `p > 1` | `2` |
| `∑ λ_k |d_k| < ∞` | `p > 3` | `4` |

The zero mode causes no difficulty: `λ_0=0`, so it contributes zero to the eigenvalue-weighted series and is handled separately in ordinary envelope estimates.

## Consequence for the source-loss map

The nonlinear engine is typed as

```lean
SourceFromSolutionEnvelopePass S :
  ∀ {m}, 1 ≤ m →
    WindowCoefficientEnvelope m c T' uCoeff →
    WindowSourceEnvelope (m - 1) c T' sourceCoeff
```

Thus:

- `WCE 3 ⟶ WSE 2`;
- `WCE 4 ⟶ WSE 3`;
- `WCE 5 ⟶ WSE 4`.

Consequently, `WCE 4` does **not** give eigenvalue-weighted `ℓ¹` of the source by direct decay: it gives source decay `O(k^{-3})`, for which

\[
\lambda_k|s_k|=O(k^{-1}),
\]

and the harmonic series diverges.  A literal weighted-`ℓ¹` source target needs `WCE 5 ⟶ WSE 4`.

# 2. Why the actual representation target needs only `WSE 2`

Let the Duhamel coefficient be

\[
b_k(\tau)
=
\int_0^\tau e^{-\nu\lambda_k(\tau-s)}s_k(s)\,ds,
\qquad \nu>0,
\]

and suppose a single time-uniform envelope satisfies

\[
|s_k(s)|\le E_k
\quad\text{on the window}.
\]

For `k≥1`,

\[
\begin{aligned}
|b_k(\tau)|
&\le E_k\int_0^\tau e^{-\nu\lambda_k(\tau-s)}\,ds\\
&=
E_k\frac{1-e^{-\nu\lambda_k\tau}}{\nu\lambda_k}.
\end{aligned}
\]

Hence

\[
\boxed{
\lambda_k|b_k(\tau)|\le \frac1\nu E_k.
}
\]

Therefore

\[
\sum_k\lambda_k|b_k(\tau)|
\le
\frac1\nu\sum_k E_k.
\]

So the Duhamel output has eigenvalue-weighted `ℓ¹` as soon as the **source envelope is ordinarily `ℓ¹`**.  If

\[
E_k\le Ck^{-q},
\]

ordinary summability needs only `q>1`; the minimum integer envelope is

```text
WindowSourceEnvelope 2.
```

The restarted homogeneous term is easier:

\[
\lambda_k e^{-\nu\tau\lambda_k}|a_{0,k}|
\]

is summable for every positive restart gap `τ>0`, even when the restart coefficients are merely bounded.  Exponential heat smoothing beats every polynomial weight.

This is exactly the mathematical content behind the repository theorem name

```lean
restartCoeff_eigenvalue_weighted_summable_of_pass2_envelope
```

and explains why the current source ladder stops at pass-2 source decay.

# 4. Exact relation between `WindowCoefficientEnvelope m` and `H^s`

Assume

\[
|c_k|\le Ck^{-m}.
\]

The one-dimensional cosine `H^s` norm has the spectral form

\[
\sum_k(1+\lambda_k)^s|c_k|^2
\asymp
\sum_{k\ge1}k^{2s}|c_k|^2.
\]

The pointwise envelope implies

\[
k^{2s}|c_k|^2\lesssim k^{2s-2m},
\]

which is summable exactly when

\[
2s-2m<-1,
\qquad\text{i.e.}\qquad
\boxed{s<m-\tfrac12.}
\]

Therefore

```text
WCE m  ⟹  H^s for every s < m - 1/2.
```

In particular:

```text
WCE 3 ⟹ H^s for every s < 5/2, but not the needed strict s>5/2 threshold;
WCE 4 ⟹ H^s for every s < 7/2, hence certainly some s>5/2.
```

Conversely, `H^s` does **not** mean pointwise `O(k^{-s-1/2})`.  From the fact that every nonnegative summand is bounded by the total `H^s` energy, one obtains only

\[
|c_k|\le C(1+\lambda_k)^{-s/2}=O(k^{-s}).
\]

The exponent `s+1/2` is the borderline heuristic obtained by asking when a pure power belongs to the weighted square sum:

\[
|c_k|=O(k^{-p})\quad\Longrightarrow\quad c\in H^s
\quad\text{if }p>s+\tfrac12.
\]

Equality at the borderline gives a harmonic series and is not enough in general.

## Why `H^s`, `s>5/2`, suffices for eigenvalue-weighted `ℓ¹`

By Cauchy–Schwarz,

\[
\begin{aligned}
\sum_k\lambda_k|c_k|
&=
\sum_k
(1+\lambda_k)^{s/2}|c_k|
\cdot
\lambda_k(1+\lambda_k)^{-s/2}\\
&\le
\left(\sum_k(1+\lambda_k)^s|c_k|^2\right)^{1/2}
\left(\sum_k\lambda_k^2(1+\lambda_k)^{-s}\right)^{1/2}.
\end{aligned}
\]

The second series behaves like

\[
\sum_k k^{4-2s},
\]

which converges exactly when

\[
\boxed{s>\tfrac52.}
\]

Thus the two useful sufficient conditions are:

```text
pointwise route: WCE 4;
Hilbert route:   H^s for any s > 5/2.
```

They are not equivalent.  `WCE 4` is a convenient stronger integer-decay condition; `H^{5/2+ε}` can imply the weighted `ℓ¹` conclusion without giving a `WCE 4` pointwise bound.

# 5. Exact base-versus-ladder split

Use the abbreviations

```text
C_m := WindowCoefficientEnvelope m
S_q := WindowSourceEnvelope q.
```

The generic nonlinear/heat pass is

```text
C_m  --source loss 1-->  S_{m-1}
S_{m-1}  --heat gain 2-->  C_{m+1},       m ≥ 1.
```

So its net effect is `C_m ⟶ C_{m+1}`.

The unconditional bounded-slice base is

```lean
windowCoefficientEnvelope_zero_of_bound : C_0
```

with the mode bound `|u_k|≤2*S.M`.  Since `SourceFromSolutionEnvelopePass` requires `1≤m`, the `m=0` case cannot be fed directly into it.  The first positive-time step is therefore a distinct smoothing lemma:

```text
C_0  --special positive-time smoothing-->  C_1.
```

This base lemma must control the **whole mild slice**, not only the homogeneous heat term.  The homogeneous term is infinitely smoothing; the bounded divergence-form Duhamel term supplies the first spatial derivative.  It is fine to call this the “free/base smoothing step,” provided the Duhamel contribution is not silently omitted.

# 7. Conservative `α<1` gain still terminates

The landed conservative engine is naturally stated in the Sobolev scale:

```text
H^s → H^{s+α},       0 < α < 1.
```

For eigenvalue-weighted `ℓ¹` of the solution coefficients, the exact Sobolev target is

\[
s>\frac52.
\]

Starting at `s_0`, the minimum number of conservative steps is

\[
\boxed{
N=\left\lfloor\frac{\frac52-s_0}{\alpha}\right\rfloor+1
}
\]

when `s_0≤5/2`; the `+1` enforces the strict inequality if the quotient is an integer.

Examples:

- from the `H^0` seed:
  \[
  N=\left\lfloor\frac{5}{2\alpha}\right\rfloor+1;
  \]
- after a separately established `H^1` base:
  \[
  N=\left\lfloor\frac{3}{2\alpha}\right\rfloor+1.
  \]
- for `α=9/10`, three steps from `H^0` give `H^{2.7}`, already above `5/2`.

Thus finite termination is immediate for every fixed `α>0`.

The estimate “about `4/α` passes” is a safe overshoot only if one insists on reaching a literal pointwise `WCE 4`-type exponent.  It is not the sharp count for the repository's `MemHSigma` engine.  The actual Hilbert-space target is `s>5/2`, so the sharp conservative count is approximately `(5/2)/α`, or `(3/2)/α` after an `H^1` base.

Also, one should not silently identify an `H^{s+α}` gain with a `WCE` pointwise-exponent gain.  They are related by the strict half-derivative summability conversion described above, but they are not definitionally the same ladder.
