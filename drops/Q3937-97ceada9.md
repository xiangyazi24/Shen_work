# Q3937 — adapting eigenvalue-weighted summability to the truncated B-form source

## Short answer

Use **Option A as the outer proof**, and use **Option B only for the positive-time tail**, but with one important correction:

> The tail cannot be proved from the stated Level 2 theorem alone.  A pointwise bound
> ```lean
> ∃ C, ∀ k, |src s k| ≤ C
> ```
> at positive times is not enough for
> ```lean
> Summable (fun k => λ_k * |duhamelCoeff k|)
> ```
> because `λ_k * |duh_k| ≤ C` is not summable.

For the tail `[t/2,t]`, you need a **positive-window summable envelope** for the truncated source coefficients:

```lean
∃ env : ℕ → ℝ,
  Summable env ∧
  ∀ s ∈ Set.Icc (t/2) t, ∀ k,
    |truncatedBFormSourceCoeff p U s k| ≤ env k
```

This is exactly the role played by the `env` field in the existing non-truncated `DuhamelSourceBddOn` theorem.  The existing theorem `summable_eigenvalue_mul_abs_limitCoeff_bdd` already uses the split internally:

```lean
duhamelSpectralCoeff a t k
  = exp (-(t - t₂) * λ_k) * duhamelSpectralCoeff a t₂ k
    + duhamelSpectralCoeff (fun σ k => a (t₂ + σ) k) (t - t₂) k
```

The head gets exponential damping; the tail is controlled by a positive-window summable envelope.  For truncated chemotaxis, copy this proof shape, but replace the global source package with a two-part truncated package:

```text
head:  raw O(k) source bound on [0,t/2]
tail:  summable positive-window source envelope on [t/2,t]
```

Avoid Option C.  It reintroduces the old `fullAgreement` trap: the truncated limit should be used as the actual solution, not identified by name with `conjugatePicardLimit` unless a separate uniqueness theorem has already been proved.  It is not the clean path for this local summability theorem.

---

## Why Level 2 alone is insufficient

For the shifted tail with `L = t - t₂`, suppose only

```lean
∀ σ ∈ Set.Icc 0 L, ∀ k,
  |truncatedBFormSourceCoeff p U (t₂ + σ) k| ≤ C
```

Then the standard parabolic gain gives only

```lean
λ_k * |duhamelSpectralCoeff shifted L k| ≤ C
```

which does **not** imply summability over `k`.  Therefore the Level 3 theorem needs either:

1. a positive-window source envelope with summable decay; or
2. a pass-4 `WindowCoefficientEnvelope` / source-ladder output on `[t/2,t]` that implies such an envelope.

So the Level 2 theorem is useful for plain coefficient `ℓ¹`, but not enough for eigenvalue-weighted summability.

# 2. Raw head bound near zero: allow `O(k)` growth

The head interval `[0,t/2]` may see only continuous flux at `0`, so prove an `O(k+1)` bound.

```lean
/-- Crude source growth near zero: the logistic part is `O(1)` and the chem-div
part is `O(k)` because of the explicit `kπ` factor. -/
theorem truncatedSource_linear_growth_on_initial_window
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (DT : TruncatedConjugateMildExistenceData p u₀)
    {T₀ : ℝ} (hT₀ : 0 ≤ T₀) (hT₀T : T₀ ≤ DT.T) :
    ∃ C : ℝ, 0 ≤ C ∧

          (truncatedConjugatePicardLimit p u₀ DT.T) s k|

  -- Chem flux: bounded flux gives `|intervalSineInner Q k| ≤ C`, hence

```

# 7. Final target theorem

Now the target theorem is a short wrapper.

```lean
/-- Level 3: positive-time eigenvalue-weighted summability of truncated Picard
coefficients. -/
theorem truncatedPicardCoeff_eigenvalue_weighted_summable_positive_time
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ}
    (DT : TruncatedConjugateMildExistenceData p u₀)
    {t : ℝ} (ht : 0 < t) (htT : t ≤ DT.T) :
    Summable (fun k => unitIntervalCosineEigenvalue k *
      |truncatedPicardCoeff p u₀
        (truncatedConjugatePicardLimit p u₀ DT.T) t k|) := by
  let U : ℝ → intervalDomainPoint → ℝ :=
    truncatedConjugatePicardLimit p u₀ DT.T
  let a : ℝ → ℕ → ℝ := fun s k => truncatedBFormSourceCoeff p U s k
  set t₂ : ℝ := t / 2 with ht₂def
  have ht₂ : 0 < t₂ := by rw [ht₂def]; linarith
  have ht₂t : t₂ ≤ t := by rw [ht₂def]; linarith
  have ht₂T : t₂ ≤ DT.T := le_trans ht₂t htT

  -- Initial coefficient bound.
  obtain ⟨M₀, hM₀, ha₀⟩ := truncatedInitialCoeff_bound p u₀

  -- Source continuity on `[0,t]`.
  have hacont : ∀ k, ContinuousOn (fun s => a s k) (Set.Icc (0 : ℝ) t) := by
    -- from continuity of truncated source coefficients; positivePart is continuous,
    -- and the patched value at `0` should be handled by the truncated Picard definition.
    exact truncatedBFormSourceCoeff_continuousOn_DT DT ht htT

  -- Head `O(k)` bound on `[0,t/2]`.
  obtain ⟨Chead, hChead, hhead⟩ :=
    truncatedSource_linear_growth_on_initial_window DT (T₀ := t₂) ht₂.le ht₂T

  -- Tail positive-window summable envelope on `[t/2,t]`.
  have Htail : TruncatedSourcePositiveWindowEnvelope p U t₂ t :=
    truncatedSource_positiveWindowEnvelope_of_existenceData DT ht₂ ht₂t htT

  -- Apply split theorem after unfolding `truncatedPicardCoeff`.
  have hsplitSum :=
    summable_eigenvalue_mul_abs_localRestartCoeff_split_linear_head
      (a₀ := cosineCoeffs (intervalDomainLift u₀))
      (a := a) ht hM₀ hChead ha₀ hacont hhead
      (Htail.toPositiveWindowEnvelopeForSource)

  simpa [truncatedPicardCoeff, U, a] using hsplitSum
```

Names such as `truncatedInitialCoeff_bound`, `truncatedBFormSourceCoeff_continuousOn_DT`, and `.toPositiveWindowEnvelopeForSource` are adapters you should add if they do not already exist.

## Option-by-option verdict

### Option A: split at `t/2`

**Best outer proof.**  It handles the non-bounded behavior at `s=0` correctly.  The head only needs an `O(k)` source bound because the leftover heat factor `exp(-(t/2)λ_k)` kills every polynomial.

Needs:

```lean
truncatedSource_linear_growth_on_initial_window
summable_eigenvalue_mul_exp_head_duhamel_linear_growth
duhamelSpectralCoeff_general_split_on
```

### Option B: shifted source

**Good for the tail only**, but not from Level 2 alone.  You need a summable positive-window envelope, not just a constant pointwise bound.  Implement it as:

```lean
TruncatedSourcePositiveWindowEnvelope p U (t/2) t
```

and then apply:

```lean
summable_eigenvalue_mul_abs_duhamel_tail_of_positiveWindowEnvelope
```

### Option C: identify with conjugate Picard coefficients

**Avoid.**  It depends on name-level agreement that the current design intentionally avoids.  Even if the truncated limit is later packaged as a full mild solution, that does not give a cheap definitional equality of coefficient formulas.  It is more brittle than the local split proof.
