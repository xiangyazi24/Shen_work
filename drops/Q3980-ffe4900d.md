## Important correction to the proposed spectral plan

The `μR` half is exactly what `resolverValueSeriesReal_integral_tsum` solves. However, the proposed source half cannot be obtained from the same `integral_tsum_pairing` under only `ContinuousOn` of `u`. The source coefficients are only available as an `ℓ²` sequence from Bessel/Parseval; they are not absolutely summable in general for a merely continuous source. Pairing against the interval indicator `1_(a,b]` is still true, but it is an `L²` tested-pairing theorem, not the absolute-summability M-test theorem.

So the spectral proof needs an additional lemma of this shape:

```lean
theorem resolverPhysicalSourceReal_integral_sourceCoeff_tsum_Ioc
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    (hUnonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1, 0 ≤ intervalDomainLift u x)
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    (∫ x in a..b, resolverPhysicalSourceReal p u x)
      = ∑' k : ℕ,
          (intervalNeumannResolverSourceCoeff p u k).re *
            ∫ x in a..b, unitIntervalCosineMode k x
```

This lemma should be proved by Hilbert-space/Parseval pairing, because

* `ρ(u) ∈ L²(0,1)` by continuity/boundedness on `[0,1]`,
* `1_(a,b] ∈ L²(0,1)`, and
* the cosine basis is complete in `L²(0,1)`.

The hypothesis `hUnonneg` is used only to rewrite

```lean
positivePart (intervalDomainLift u x) = intervalDomainLift u x
```

on `[0,1]`, so that the physical source matches the coefficient source.

## Practical proof that closes the local sorry now

If the goal is to close exactly this sorry in `IntervalResolverWeakODEBridge.lean`, the shortest non-spectral proof is to use the already imported derivative theorem from `IntervalResolverWeakLapBound.lean`:


That theorem gives the pointwise derivative of `resolverGradReal`; then the interval-integral identity is just the FTC on the compact segment between `a` and `b`. The new `hUnonneg` argument is unused in this shortcut.

Replace the sorry with:

```lean

  let q : ℝ → ℝ := resolverLapPhysicalReal p u

  have hseg_Ioo : ∀ x ∈ Set.uIcc a b, x ∈ Set.Ioo (0 : ℝ) 1 := by
    intro x hx
    rcases hx with hx | hx
    · exact ⟨lt_of_lt_of_le ha.1 hx.1, lt_of_le_of_lt hx.2 hb.2⟩
    · exact ⟨lt_of_lt_of_le hb.1 hx.1, lt_of_le_of_lt hx.2 ha.2⟩

  have hderiv :
      ∀ x ∈ Set.uIcc a b, HasDerivAt F (q x) x := by
    intro x hx
    have hxIoo : x ∈ Set.Ioo (0 : ℝ) 1 := hseg_Ioo x hx
    have hxIcc : x ∈ Set.Icc (0 : ℝ) 1 := Set.Ioo_subset_Icc_self hxIoo

      resolverGradReal_hasDerivAt_physicalLap_of_continuousOn

    -- inside `(0,1)`.
    simpa [F, q, resolverLapPhysicalReal, hxIcc] using h

    intro x hx

    exact intervalIntegral.integral_eq_sub_of_hasDerivAt hderiv hq_int
```

## If you insist on the spectral proof

Do not try to prove the source pairing via `integral_tsum_pairing`. That theorem has the hypothesis

```lean
Summable (fun n => |c n| * B n)
```

and with `B n = 1`, it requires absolute summability of the source coefficients. Continuity of the source does not give this. The correct spectral path is:

1. Use `resolverValueSeriesReal_integral_tsum` for the `μR` term.
2. Add the missing `L²` tested-source pairing lemma above.
3. Prove the gradient boundary-minus-boundary identity by a summable derivative-series lemma, not by differentiating twice.
4. Use `intervalNeumannResolverCoeff_elliptic` modewise.
5. Convert `resolverLapPhysicalReal` to `p.μ * resolverValueSeriesReal - resolverPhysicalSourceReal` on `Set.uIcc a b` using `ha`, `hb`, and `hUnonneg`.

The blocker is only step 2. Once that lemma exists, the spectral proof becomes a bookkeeping `tsum_congr`/`tsum_sub`/`tsum_mul_left` proof.
