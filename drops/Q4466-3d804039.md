## Key correction before pasting

The current `Statements.lean` `Theorem_1_1` does **not** quantify over the weaker
`PositiveInitialDatum`. It quantifies over the paper-faithful stronger datum:

```lean

```

So the projection API should use `PaperPositiveInitialDatum`. A theorem with only
`PositiveInitialDatum` is not a one-line projection from the headline; it would need an
additional, generally false bridge
`PositiveInitialDatum intervalDomain u‚āÄ ‚Üí PaperPositiveInitialDatum intervalDomain u‚āÄ`.

Also, the current strict-logistic branch of `Theorem_1_1` gives a **finite-horizon sup
bound** on the witness interval `0 < t < Tmax`, plus global existence when `1 ‚Č§ p.m`.
It does **not** by itself give `IsPaper2Bounded` / an eventual-atTop global bound for the
global solution. So the boundedness API below is the honest finite-horizon bound exactly
projected from the headline.
