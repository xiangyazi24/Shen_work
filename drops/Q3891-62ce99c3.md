# Q3891 — ground-level proof strategy for the two remaining Paper 2 χ<0 gaps

## Executive answer

There are two different tasks here, and they should be kept separate.

1. `NegativePartStandardHeatSemigroupDuhamelFacts` is the negative-part weak-test bridge.  Its file currently states a “standard analytic facts” interface with nine fields.  The cheapest proof should **not** try to differentiate the truncated map in physical space.  Prove a coefficient-route theorem that fills the final weak identity directly, then either populate the existing nine-field structure as a compatibility wrapper or bypass it with a new constructor.

2. `HSpectral` / `BFormMildSpectralBootstrapData` for arbitrary `ConjugateMildSolutionData` should be obtained by **parameterizing the existing windowed spectral provider over an arbitrary mild solution `u`**.  The actual theorem chain is already in place: `DuhamelSourceTimeC1On`, `duhamelSpectralCoeff_hasDerivAt_of_on`, `localRestartCoeff_eigenvalue_summable_of_on`, `restartCosineSeries_hasDerivAt_time_of_on`, `pde_u_of_localized_data_with_hpost_on`, `HasTimeNeighborhoodSpectralAgreement`, and `HasResolverDirectSpectralData`.  The remaining work is not new spectral analysis; it is replacing hardwired occurrences of `conjugatePicardLimit p u₀ D.T` by an arbitrary `u` carried by `ConjugateMildSolutionData`.

## Can the fields be proved from (a) coefficient mild identity, (b) joint continuity, (c) cosine spectral machinery?

Yes, but the recommended proof is sharper:

* The **last four fields** should be proved directly by coefficient ODEs and cosine weak testing.
* The **first five fields** can be proved from spectral estimates and joint continuity, but they are no longer necessary if we add a coefficient-route constructor.  Keeping them as fields is harmless, but proving them is extra work.

The correct proof route is coefficient route (A), not physical mild route (B): do not differentiate `positivePart (u t x)` or `truncatedChemFluxLifted p (u t)` through zero crossings.  Instead prove coefficient ODEs for the full truncated source coefficients.  The source is only `C⁰` in time as a map into coefficients, but that is enough for the Duhamel coefficient FTC:

# Recommended proof order

## For gap (1)

1. Prove coefficient continuity for the truncated logistic and flux source coefficients from joint continuity of `u` and Lipschitz/continuous `positivePart`/`negativePart`.
2. Prove/use the C⁰ Duhamel coefficient FTC (`IntervalDuhamelCoeffFTC`).
3. Prove the coefficient ODE for the truncated mild fixed point:
   `b' = aL - χ aQ - λ b`.
4. Prove one-mode weak-gradient pairing for the negative-part test.
5. Prove `negativePartMildSemigroupWeakAfterFluxTestDuality_of_coefficients`.
6. Only if the old record must remain, wrap this theorem into `NegativePartStandardHeatSemigroupDuhamelFacts` or keep its first five fields as separately supplied spectral estimates.

## For gap (2)

1. Create generic versions of `pde_u_of_localized_data_with_hpost_on` and `hasBFormSpectralPdeAgreement_conjugatePicardLimit_of_localized_data` over arbitrary `u` and `ConjugateMildSolutionData`.
2. Use `DuhamelSourceTimeC1On.add`, `.const_mul`, `.shift_zero`, and `.restrict_hi` to build the B-form source package on restart windows.
3. Use `localRestartCoeff_eigenvalue_summable_of_on` and `restartCosineSeries_hasDerivAt_time_of_on` inside the generic PDE proof.
4. Use `TimeNhdLocalized.Hu_of_restart_localized` (or B-form analogue) for `HasTimeNeighborhoodSpectralAgreement`.
5. Use `hasResolverDirectSpectralData_of_clamped_perT0` plus `IntervalResolverDirectTimeRegularity` for resolver data.
6. Package all three fields into `BFormMildSpectralBootstrapData`.

## Short final answer

* `NegativePartStandardHeatSemigroupDuhamelFacts` has nine fields: one gradient estimate, two endpoint facts, two DCT dominators, three leg weak identities, and one tested mild decomposition.  The correct proof should be coefficient-level: use the truncated mild coefficient identity + Duhamel coefficient FTC + cosine weak-gradient pairing.  This avoids differentiating `u_+`.

* `HSpectral` is mostly already solved in the tree for the canonical Picard limit.  Make the existing windowed spectral provider generic in `u : ℝ → intervalDomainPoint → ℝ` with `MD : ConjugateMildSolutionData p T u₀ u`.  Then chain:

```lean

→ duhamelSpectralCoeff_hasDerivAt_of_on
→ localRestartCoeff_eigenvalue_summable_of_on
→ restartCosineSeries_hasDerivAt_time_of_on
→ pde_u_of_localized_data_with_hpost_on_for_mild
→ HasBFormSpectralPdeAgreement p T u

TimeNhdLocalized.Hu_of_restart_localized
→ HasTimeNeighborhoodSpectralAgreement T u

hasResolverDirectSpectralData_of_clamped_perT0
→ HasResolverDirectSpectralData T (mildChemicalConcentration p u) p
```

These are local rewrites/parameterizations, not new analytic theory.
