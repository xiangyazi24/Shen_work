## 5. Why the exact statement is false

### 5.1 Re-anchoring is a committed operation

`PDE/P3MoserEnergyContinuity.lean` defines:

The proof works because the classical-solution predicate is local on `(0,T)`.

### 5.2 Concrete, non-vacuous countermodel

Use the already committed parameter witness:


For these parameters:

```text

```

The real unit interval admits the strict-positive-time stationary solution

```text
u(t,x) = 1,
v(t,x) = 1,
```

by `zeroReaction_isPaper2ClassicalSolution`.

Now re-anchor only the stored cell-density slice:

```lean

  (fun _ : intervalDomain.Point => 2)
  (fun _ _ => (1 : ℝ))
let v : ℝ → intervalDomain.Point → ℝ := fun _ _ => 1
```

Then all exact Part 4 hypotheses hold:

1. `a = b = 0`, `m = 1`, `β = 1`, `χ₀ = 1/4`, and the strict threshold inequality hold by normalization.
2. `u,v` are a positive global bounded solution, because they equal the stationary solution for every `t > 0`; `intervalDomain_globalClassical_withInitialSlice` proves the global classical part.
3. `HasInitialMass intervalDomain u 2` holds because `u 0` is the constant `2` and the unit interval has volume `1`.
4. The actual positive-time physical mass is `1`, not `2`.
5. `liminfInfValue intervalDomain v = 1` because `v` is the constant function `1` at all times.

Take the existing concrete constants

```lean

```

for which

```lean

C.eventualMinimalUBound 2 = 1.
```

Because `p.γ = 1`, the displayed Part 4 lower formula is

```text
minimalVLowerFormula 1 1 2 1
= 1 * (2 * 1^(1-1))
= 2.
```

The conclusion is therefore `2 ≤ 1`, false.

This is not an objection to the analytic signal-floor theorem: that theorem correctly uses the physical positive-time mass `1` and produces a floor compatible with `v = 1`. The false conclusion appears only because the exact statement feeds the unrelated stored mass `2` into the formula.

### 5.3 Stronger version: no `Paper3Constants` instance can repair the exact statement

Let `C : Paper3Constants intervalDomain theorem21Part4CounterParams` be arbitrary, and write

```text

```

Use the same positive-time orbit `u = v = 1`, but set the stored zero slice to the constant

```text

```

Then `HasInitialMass intervalDomain u uStar` holds. Since `γ = 1`, the power of `C.eventualMinimalUBound uStar` is zero, so

```text

```

while the `v` liminf remains `1`. Hence the exact interval specialization of `Theorem_2_1_part4` is false for **every** valid `Paper3Constants` instance, not merely for an inconvenient choice of constants.

## 6. The precise missing leaf

There is no genuinely missing analytic PDE estimate between the current minimal upper bound and the current signal floor. The impossible missing bridge is the statement-layer implication

```lean

HasInitialMass intervalDomain u uStar →
HasEquilibriumMassOnPositiveTimes intervalDomain u uStar
```

in the zero-reaction branch.

That implication is false under the current definitions, by the re-anchored countermodel above. It cannot be filled by more Moser iteration, elliptic regularity, mass conservation on `(0,∞)`, or a liminf lemma, because the premise supplies no relationship between `u 0` and the positive-time orbit.

A faithful repair must do one of the following:

1. Replace `HasInitialMass D u uStar` in Part 4 by `HasEquilibriumMassOnPositiveTimes D u uStar`.
2. Quantify an actual initial datum `u₀`, require `InitialTrace D u₀ u`, prescribe the mass of `u₀`, and use a mass-conservation theorem for positive times.
3. Strengthen the solution interface with exact zero-slice/trace compatibility and then prove mass conservation from that datum.

The smallest correction is option 1, which is already the interface used by the committed minimal eventual-stability chain.
