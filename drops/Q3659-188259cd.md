

3. Prove `bound` is interval-integrable exactly as in `chemLeg_interior_hasDerivAt`:
   ```lean

   have hcomp : IntervalIntegrable (fun s : ℝ => s ^ (-1 + θ / 2 : ℝ)) volume 0 t₀ :=

   ```
4. Prove the a.e. domination on `Set.uIoc 0 t₀`. Use the endpoint-dropping pattern already used in `chemLeg_interior_hasDerivAt` and `chemLitLeg₂_eq_chemDuhamelLeg_Icc`:

## Is there a semantic trap?

Yes, but it is manageable.

### Trap 1: direct spectral coefficient estimates are too singular

Do **not** try to prove integrability by bounding `unitIntervalCosineHeatSecondValue` only from the uniform coefficient bound `hd.hQcoeff`. That gives a raw second-derivative heat-series bound behaving like `(t₀-s)^{-2}` or similarly nonintegrable near `s=t₀`. The integrable bound comes from `C^θ` cancellation, via the literal Hessian estimate `neumannHeatSecondDeriv_Ctheta_to_Linfty`.

### Trap 2: spectral/literal equality is only positive-time

`intervalFullSemigroupOperator_secondDeriv_eq_secondValue_Icc` needs `0 < t₀ - s`, so it holds for `s ∈ Ioo 0 t₀`, not at the endpoint `s = t₀`. Use a.e. congruence, not pointwise `IntervalIntegrable.congr` over all of `uIoc`.

### Trap 3: `ChemLegData` has enough, but every field matters

`hQmeas`, `hQint`, and `hQbdd` are needed for a.e.-measurability of the literal second-derivative integrand. `hQholder`/`hHQ_nn` are needed for the brick-3 integrable domination. `hQcont`/`hQcoeff` are needed only for the spectral/literal equality. If any of these are removed from `ChemLegData`, the theorem would require replacement hypotheses.

### Trap 4: stay local

This should remain in `ChemMildC1etaUncond.lean` or, if preferred, a small bridge file importing it. It should not touch folded-kernel files or SourceDecay.
