# Q3695 hostile route audit: strict χ<0, per-datum EWA local existence, and anti-Zeno continuation

Repository state audited: `xiangyazi24/Shen_work`, requested commit `52cf8838` / Task236.

## Short answer

A route that uses **only** per-datum local EWA existence, with each restart lifespan depending on the current slice's own PPID floor, does **not** globalize from the current interfaces. It can Zeno: the slice theorem gives a fresh positive floor at each restart, but gives no quantitative lower bound on those floors and no lower bound on the sum of the corresponding local lifespans.

There **is** a viable route that avoids the impossible all-PPID common floor, but it is not “per-datum local existence alone.” It needs an anti-Zeno input: a positive lower floor/min-persistence estimate away from `t = 0`. The repo already has the right mechanism:

```text
ThresholdQuantitativeLocalExistence p
+ ClassicalMinPersistence p
+ per-datum seed local existence
=> finite threshold restarts reach a fixed horizon
=> global reachability / Theorem_1_1
```

The next real Lean task should therefore be a **PPID-typed anti-Zeno reachability theorem** that reuses `IntervalDomainQuantFromThreshold.reaches_fixed_horizon`, not another all-PPID uniform-floor interface.

These do not say that an individual solution's floor must decay to zero. They say the **datum class** does not supply a common floor. Therefore, if the restart proof only says “each slice is PPID,” the floors may be arbitrarily small from the standpoint of the interface. A sequence of local lifespans depending on those floors may shrink summably. That is the Zeno gap.

Formally: `classicalSolution_slice_paperPositiveInitialDatum` gives the type needed to restart a PPID local theorem, but it gives no estimate of the form

```text
∀ target T, ∃ c > 0, ∀ restart times τ in [t₁, T], ∀ x, c ≤ u τ x.
```

That missing estimate is exactly what the min-persistence machinery is designed to provide.

## Existing anti-Zeno route in the repo

The key file is:


It defines:

```lean
def ThresholdQuantitativeLocalExistence (p : CM2Params) : Prop :=
  ∀ M c : ℝ, 0 < M → 0 < c → ∃ δ : ℝ, 0 < δ ∧
    ∀ w : intervalDomainPoint → ℝ,
      PositiveInitialDatum intervalDomain w →
      (∀ x, |w x| ≤ M) →
      (∀ x, c ≤ w x) →
      ∃ uw vw,

        InitialTrace intervalDomain w uw
```

and:

```lean
def ClassicalMinPersistence (p : CM2Params) : Prop :=
  ∀ u₀ : intervalDomainPoint → ℝ,
    PositiveInitialDatum intervalDomain u₀ →
  ∀ δ t₁ : ℝ, 0 < t₁ → t₁ < δ →
  ∃ c : ℝ, 0 < c ∧
    ∀ T : ℝ, t₁ < T → T ≤ δ →
    ∀ u v,
      IsPaper2ClassicalSolution intervalDomain p T u v →
      InitialTrace intervalDomain u₀ u →
      ∀ t, t₁ ≤ t → t < T → ∀ x, c ≤ u t x
```

The anti-Zeno step is already implemented as:

```lean
theorem reaches_fixed_horizon ... :
  ∀ n : ℕ, ∃ u v,
    IsPaper2ClassicalSolution intervalDomain p
      (min δ (T₀ + n * (δc / 2))) u v ∧
    InitialTrace intervalDomain u₀ u
```

This is exactly what variable-floor restarts lack. Once `ClassicalMinPersistence` supplies one positive threshold `c` on `[t₁, target]`, `ThresholdQuantitativeLocalExistence` supplies one fixed positive restart duration `δc = δ(M', c)`, and `reaches_fixed_horizon` reaches the target in finitely many restarts. The restart count may depend on the datum and target; that is fine.

The file already packages this into:

```lean

```

and:

```lean

```

But these are PID-typed: `hlocal` is required for every `PositiveInitialDatum`, not just PPID. That is too strong for the strict EWA route if EWA local existence is only available for closed-domain floored PPID data.

## Answer to Question A

### A pure per-datum local-EWA restart route: no

The current per-datum route cannot globalize by merely restarting at slices and using their individual PPID floors. `classicalSolution_slice_paperPositiveInitialDatum` gives existence of a new floor but not a lower bound along the restart chain. The current EWA fixed-point constants are floor-dependent, and Tasks235/236 rule out a uniform floor over the bounded PPID class. Without a positive lower floor/min-persistence estimate along the already-built solution, the variable-step restart proof has a Zeno hole.

### A route avoiding all-PPID common floor: yes, via threshold + min-persistence

The viable route is:

```text
per-datum PPID local seed
+ threshold local existence on {|w|≤M, c≤w}
+ ClassicalMinPersistence for the original datum/window
=> fixed positive restart step δc after t₁
=> finite restarts reach any prescribed target
=> ReachableArbitrarilyLong / global part
```

This avoids the refuted all-PPID common floor because `c` is not uniform over the whole bounded PPID class. It depends on the original datum and the target window, exactly as `ClassicalMinPersistence` permits.
