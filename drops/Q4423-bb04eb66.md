## Verdict

At the current `Statements.lean` API, the strict-logistic branch of

```lean

```

provides one coherent witness package

```lean
∃ Tmax > 0, ∃ u v,
  IsPaper2ClassicalSolution intervalDomain p Tmax u v ∧
  InitialTrace intervalDomain u₀ u ∧
  (∀ t, 0 < t → t < Tmax →
    intervalDomain.supNorm (u t) ≤
      max (intervalDomain.supNorm u₀) ((p.a / p.b) ^ (1 / p.α))) ∧
```

Therefore the following are literal projections:

- global existence, assuming `1 ≤ p.m`;
- initial trace;
- the displayed **finite-`Tmax`** sup-norm estimate;
- nonnegativity/strict positivity on `(0,Tmax)` from `IsPaper2ClassicalSolution`;
- classical regularity on `Tmax` from `IsPaper2ClassicalSolution`.

There is one load-bearing limitation:

# Recommended witness-coherent API

Do not publish five unrelated existential theorems: Lean could choose different `u,v` witnesses in each theorem. Package the single branch witness once, and make the five requested declarations projections from that package.

The following core block can be added immediately. It accepts an arbitrary proof
`h11 : Theorem_1_1 intervalDomain p`, so it works with every existing conditional headline today and with the eventual unconditional headline without changing the projection layer.

```lean
import ShenWork.Paper2.IntervalDomainChiNonposHeadline

open ShenWork.IntervalDomain (intervalDomain)

noncomputable section

namespace ShenWork.Paper2

/-- The exact strict-logistic witness returned by the nonminimal branch of
`Theorem_1_1 intervalDomain p`.

`boundedOnTmax` is intentionally the finite-horizon bound present in the
current statement.  It is not `IsPaper2Bounded intervalDomain u`. -/
structure IntervalChiNonposStrictLogisticSolution
    (p : CM2Params) (u₀ : intervalDomain.Point → ℝ) where
  Tmax : ℝ
  hTmax : 0 < Tmax
  u v : ℝ → intervalDomain.Point → ℝ
  classicalSolution :
    IsPaper2ClassicalSolution intervalDomain p Tmax u v
  initialTrace :
    InitialTrace intervalDomain u₀ u
  boundedOnTmax :
    ∀ t, 0 < t → t < Tmax →
      intervalDomain.supNorm (u t) ≤
        max (intervalDomain.supNorm u₀)
          ((p.a / p.b) ^ (1 / p.α))
  globalOfMGeOne :
    1 ≤ p.m →
      IsPaper2GlobalClassicalSolution intervalDomain p u v

/-- Extract the exact strict-logistic branch witness from Paper 2 Theorem 1.1.

Notice that `1 ≤ α` and `1 ≤ γ` are not logical inputs to this projection:
they are inputs used by the current interval headline to construct `h11`.
The `Theorem_1_1` statement itself only needs `hχ`, `ha`, and `hb` at this
projection point. -/
theorem intervalChiNonposStrictLogisticSolution_of_theorem_1_1
    (p : CM2Params)
    (h11 : Theorem_1_1 intervalDomain p)
    (hχ : p.χ₀ ≤ 0)
    (ha : 0 < p.a) (hb : 0 < p.b)
    (u₀ : intervalDomain.Point → ℝ)
    (hu₀ : PaperPositiveInitialDatum intervalDomain u₀) :
    Nonempty (IntervalChiNonposStrictLogisticSolution p u₀) := by
  rcases (h11 hχ).1 ha hb u₀ hu₀ with
    ⟨Tmax, hTmax, u, v, hsol, htrace, hbound, hglobal⟩
  exact ⟨
    { Tmax := Tmax
      hTmax := hTmax
      u := u
      v := v
      classicalSolution := hsol
      initialTrace := htrace
      boundedOnTmax := hbound
      globalOfMGeOne := hglobal }⟩

/-- Requested API 1: global classical existence for the selected witness. -/
theorem globalExistence_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀)
    (hm : 1 ≤ p.m) :
    IsPaper2GlobalClassicalSolution intervalDomain p S.u S.v :=
  S.globalOfMGeOne hm

/-- Requested API 2: exact nonnegativity projected from the local classical
solution.  The source field is actually strict positivity on the closed spatial
domain. -/
theorem nonneg_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀) :
    ∀ t x, 0 < t → t < S.Tmax → 0 ≤ S.u t x :=
  fun t x ht0 htT => (S.classicalSolution.u_pos' ht0 htT).le

/-- Requested API 3: the exact boundedness conclusion currently stored in
`Theorem_1_1`; this is a bound on `(0,Tmax)`, not an eventual/global bound. -/
theorem boundedness_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀) :
    ∀ t, 0 < t → t < S.Tmax →
      intervalDomain.supNorm (S.u t) ≤
        max (intervalDomain.supNorm u₀)
          ((p.a / p.b) ^ (1 / p.α)) :=
  S.boundedOnTmax

/-- Requested API 4: initial trace of the selected witness. -/
theorem initialTrace_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀) :
    InitialTrace intervalDomain u₀ S.u :=
  S.initialTrace

/-- Requested API 5: the exact regularity field of the local classical solution. -/
theorem regularity_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀) :
    intervalDomain.classicalRegularity S.Tmax S.u S.v :=
  S.classicalSolution.regularity

end ShenWork.Paper2
```

These are the literal one-line projections requested. The constructor is the only place that destructs the existential returned by `Theorem_1_1`, so all five declarations refer to the same solution.

# Global corollaries from `globalExistence_interval_chiNeg`

For a consumer operating after `hm : 1 ≤ p.m`, it is useful to expose positivity and regularity on every positive horizon. These are consequences of the global-classical field, not additional χ-sign estimates.

```lean
namespace ShenWork.Paper2

/-- Closed-domain nonnegativity for every positive time. -/
theorem nonneg_global_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀)
    (hm : 1 ≤ p.m) :
    ∀ t x, 0 < t → 0 ≤ S.u t x := by
  intro t x ht
  have hglobal := globalExistence_interval_chiNeg S hm
  have hsol := hglobal.classical (show 0 < t + 1 by linarith)
  exact (hsol.u_pos' ht (by linarith)).le

/-- Classical regularity on every positive finite horizon. -/
theorem regularity_global_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀)
    (hm : 1 ≤ p.m) :
    ∀ T > 0, intervalDomain.classicalRegularity T S.u S.v :=
  fun T hT => (globalExistence_interval_chiNeg S hm).regularity hT

end ShenWork.Paper2
```

# The P3.1 boundedness gate

Paper 3's `PositiveGlobalBoundedSolution` and Proposition 1.2 consume:

```lean
IsPaper2Bounded intervalDomain u
```

whose definition is

```lean
∃ M, ∀ᶠ t in Filter.atTop, intervalDomain.supNorm (u t) ≤ M.
```

That is not the conclusion of `boundedness_interval_chiNeg` above. It cannot be obtained by changing a projection index: an actual theorem is missing at the statement boundary.

The concrete χ≤0 proof should therefore export one additional result, for the same selected witness, in either of these equivalent forms:

```lean
∀ t, 0 < t →
  intervalDomain.supNorm (S.u t) ≤
    max (intervalDomain.supNorm u₀)
      ((p.a / p.b) ^ (1 / p.α))
```

or directly

```lean
IsPaper2Bounded intervalDomain S.u
```

From the all-positive-time pointwise bound, the conversion to `IsPaper2Bounded` is routine:

```lean
open Filter

namespace ShenWork.Paper2

theorem isPaper2Bounded_interval_of_forall_pos_bound
    {u : ℝ → intervalDomain.Point → ℝ} {B : ℝ}
    (hB : ∀ t, 0 < t → intervalDomain.supNorm (u t) ≤ B) :
    IsPaper2Bounded intervalDomain u :=
  IsPaper2Bounded.of_forall_ge_supNorm_le
    (T := (1 : ℝ)) (M := B)
    (fun t ht => hB t (lt_of_lt_of_le zero_lt_one ht))

end ShenWork.Paper2
```

Once that extra concrete theorem is proved, the P3-facing name should be reserved for the genuinely global conclusion:

```lean
theorem boundedness_global_interval_chiNeg
    {p : CM2Params} {u₀ : intervalDomain.Point → ℝ}
    (S : IntervalChiNonposStrictLogisticSolution p u₀)
    (hglobalBound :
      ∀ t, 0 < t →
        intervalDomain.supNorm (S.u t) ≤
          max (intervalDomain.supNorm u₀)
            ((p.a / p.b) ^ (1 / p.α))) :
    IsPaper2Bounded intervalDomain S.u :=
  isPaper2Bounded_interval_of_forall_pos_bound hglobalBound
```

Then Paper 3 can form its canonical solution package in one line:

```lean

```

6. The literal headline bound is only on `0 < t < Tmax`.
