## Does P3.7 reduce to global stability?

**No — not as the repo is structured.**

The small-data global existence fields are **inputs to the local stability/sectorial branch**, not consequences of global stability.  The file `IntervalDomainSectorialNonlinearBridges.lean` already shows the cheap route: if you have a full interval-domain global Cauchy solver, then both small-data existence fields follow immediately; the small radius is only a restriction on admissible initial data:

```lean
theorem intervalDomain_smallDataGlobal_of_globalSolutionExists
    (p : CM2Params)
    (hglobal : IntervalDomainGlobalSolutionExists p)
    (hm : 1 ≤ p.m) :
    ∀ uStar, ∀ delta > 0,
      SmallDataGlobalExistence intervalDomain p uStar delta
```

and similarly for `MassConstrainedSmallDataGlobalExistence`.

The small-χ minimal tail branch is also **not a consequence of global stability**.  It is part of the persistence/lower-bound machinery that global stability later consumes.  Proving it by invoking Theorems 2.3–2.5 would be circular, because those global-stability statements depend on persistence/compactness inputs in the existing statement-layer design.

So the correct dependency is:

```text
Paper2/P3.1 global existence + boundedness
  + absorbing/eventual upper bound u ≤ uBar
  + Harnack/Gaussian lower propagation / elliptic lower transfer
  => IntervalDomainUniformPersistencePart4PointwiseRaw p uBar
  => UniformPersistencePart4Raw intervalDomain p (fun _ => uBar) 1
  => Theorem_2_1_part4
  => Theorem_2_1
  => used downstream by P3.6 global stability
```

For small-data Cauchy existence:

```text

  => SmallDataGlobalExistence intervalDomain p uStar delta
  => MassConstrainedSmallDataGlobalExistence intervalDomain p uStar delta
  => local exponential stability branch / Theorem 2.2 sectorial wrappers
```

### Analytic ingredients still needed for the small-χ tail leaf

For `IntervalDomainUniformPersistencePart4PointwiseRaw p uBar`, the real analytic work is:

1. **Eventual upper/absorbing bound** for the minimal model: an eventual `u ≤ uBar` bound, with `uBar` matching the constants package field `eventualMinimalUBound := fun _ => uBar`.

2. **Mass persistence** from `HasInitialMass intervalDomain u uStar` in the minimal case `a=b=0`.  In the minimal model, the total mass should be conserved, or at least uniformly bounded below by the initial mass.

3. **Parabolic Harnack / Gaussian lower propagation** or an equivalent persistence engine.  This is the shared dynamical core: positive mass plus eventual upper regularity should imply a pointwise lower tail, or at least enough lower input for the elliptic resolver.

4. **Elliptic lower transfer** from the `u` information to the resolver `v`, yielding the displayed constant:

```lean
minimalVLowerFormula 1 p.γ uStar uBar
```

The repo already has interval elliptic lower-bound machinery (`IntervalDomainPersistenceElliptic.lean`) and a persistence-logistic wrapper (`IntervalDomainPersistenceLogistic.lean`), but the actual pointwise part-4 raw lower-tail estimate remains the leaf.
