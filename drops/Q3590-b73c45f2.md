## 4. Patches that only move assumptions sideways or duplicate wrappers

Avoid these as the next patch:

* Adding another final wrapper around
  `FinalWiring.paper2_theorem_1_1_from_quant` without exposing a new hQuant producer.  That is cosmetic.
* Duplicating the already-landed
  `BFormPositiveDatumLocal.quantitativeLocalExistence_of_picardLimitFrontier_boundary_of_BForm`.  That would not reduce assumptions.
* Replacing `PositiveDatumBFormLocalHyp p` with any squared-barrier `PositiveDatumBFormLocalHypSq* p` package in the hQuant route.  Those packages are for `PaperPositiveInitialDatum`, not all `PositiveInitialDatum`; using them for hQuant would either be ill-typed or would require a false positivity-floor bridge.
* Trying to prove `BoundaryMinPersistenceBound p` by simply restating `ClassicalMinPersistence p`: the repo already has the correct one-way reduction, and reversing it would be assumption movement, not proof.
* Treating `PicardRestartFrontier p` as an improvement over `PicardLimitRestartFrontier p` for the preferred route.  The unified `PicardLimitRestartFrontier` is the stronger shared residual that closes both cone and threshold Picard constructions; introducing the older residual is only a regression unless a theorem actually proves it first.
