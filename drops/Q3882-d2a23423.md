# Q3882 — critical design fix for `fullAgreement`

## Decision

Choose **Option B**.

Do **not** prove

```lean
truncatedConjugatePicardLimit = conjugatePicardLimit
```

by iterate induction.  The critique is correct: truncated iterates are not expected to be nonnegative, because the conjugate/gradient kernel leg is sign-indefinite.  So the iterate-level proof plan is unsound.

Also do **not** add sign-free full-map contraction fields to `UniformConjugateMildExistenceCore` unless a later consumer truly requires name-level equality with `conjugatePicardLimit`.  That is a much larger refactor.

The cheap and correct fix is:

> Once the truncated limit is proved nonnegative, show that it satisfies the **full** B-form mild equation, package it as the `u` field of `ConjugateMildSolutionData`, and feed that `u` directly to the bootstrap/classical-solution consumer.

This works because the bootstrap consumer is parametric in the solution field `u`; it does not need `u` to be definitionally or propositionally equal to the canonical name `conjugatePicardLimit`.

---

## Why Option B is cheaper

### Option A cost

Option A requires adding sign-free full-map contraction data and proving a uniqueness/fixed-point theorem for arbitrary full mild fixed points in the full ball, independent of the nonnegative cone.  That means touching the fixed-point core, the full Picard data records, and the full-map convergence/uniqueness proof.

Concretely, Option A would need at least:

```lean
sign_free_hcontr :
  ∀ u w d,
    fullBall u → fullBall w →
    HasContinuousSlices T u → HasContinuousSlices T w →
    HasJointMeasurability u → HasJointMeasurability w →
    (∀ t, 0 < t → t ≤ T → ∀ x, |u t x - w t x| ≤ d) →
    ∀ t, 0 < t → t ≤ T → ∀ x,
      |intervalConjugateDuhamelMap p u₀ u t x
        - intervalConjugateDuhamelMap p u₀ w t x| ≤ K * d
```

and then a sign-free analogue of the existing uniqueness theorem.  That is not a local V5 assembly fix.

### Option B cost

Option B needs only one local theorem plus V5 field rewiring:

1. prove the truncated fixed point is a full mild solution after nonnegativity;
2. build `ConjugateMildSolutionData` for
   `uT := truncatedConjugatePicardLimit p u₀ DT.T`;
3. call the existing bootstrap consumer with `u := uT`.

No full Picard iterate agreement, no full Picard convergence proof, no new contraction fields.

  calc

### 2. Make the solution name local and truncated

## Final answer

**Option B is cheaper and is the correct V5 fix.**

The assembly should stop trying to prove or use

```lean
truncatedPicardLimit = conjugatePicardLimit
```

and instead treat

```lean
uT := truncatedConjugatePicardLimit p u₀ DT.T
```

as the actual constructed solution.  After nonnegativity, `uT` satisfies the full B-form mild equation, so build `ConjugateMildSolutionData` for `uT` and feed `uT` directly into `isClassicalSolution_of_conjugateMild_spectral`.

This removes the false iterate-nonnegativity dependency and avoids the much larger sign-free contraction refactor.
