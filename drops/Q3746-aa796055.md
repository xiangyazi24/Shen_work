# Q3746 audit: weak-PID B-form component fields

## Bottom line

```lean

```

for weak


1. keeping an equivalent analytic assumption under another name; or

The first genuine analytic floor is the **weak-PID lower-barrier / positive-time initial comparison** needed for strict positivity. The PPID square route gets this from `paperPositiveFloor`; weak `PositiveInitialDatum` does not have such a floor. A safe preparatory target exists (`SquareHeatSeed` from weak PID), but it does not by itself remove a `PositiveDatumBFormLocalComponents` field, because the missing comparison inequality remains.

## Is there any small non-cheating target at all?

### Option B: switch the component to the squared-heat strict-positivity route

A weak-safe squared route would need a positive-time initial comparison for every restart strip:

```lean
∀ t, 0 < t → t < DB.T →
  ∃ τ, 0 < τ ∧ τ < t ∧
    0 < DB.T - τ ∧
    NeumannLinearDriftCoefficientsRegular (DB.T - τ)
      (restartTimeShift τ drift) (restartTimeShift τ react) ∧
    IsClassicalNeumannLinearDriftSuperSolution (DB.T - τ)
      (restartTimeShift τ drift) (restartTimeShift τ react)
      (restartTimeShift τ (bformConjugatePicardLift p DB)) ∧
    A ^ 2 / 2 + D ≤ M ∧
    drift/reaction bounds ∧
    (∀ x ∈ Set.Icc 0 1,
      squareHeatBarrier M f τ x ≤ bformConjugatePicardLift p DB τ x)
```

Then the already-proved theorem

```lean
theorem bform_strictPos_via_t0_restart_semigroup
```

would give strict positivity. File reference: `IntervalBFormSquareHeatT0RestartDerivativeData.lean:50-85`.

The missing weak-PID part is the last initial comparison at positive `τ`. PPID proves it with `paperPositiveFloor`; weak PID has no such floor. Proving that comparison is the analytic floor.

## PPID/squared route adaptation audit

The PPID/squared routes cannot be adapted to weak PID by a mere theorem call.

Reasons:

1. `PositiveDatumBFormLocalComponentsSq`, `PositiveDatumBFormLocalComponentsSqRegular`, and `PositiveDatumBFormSqDeepestHypotheses` all carry `PaperPositiveInitialDatum` or a bank that carries it.
2. The strict positivity theorem `bform_strictPos_closed` assumes `hu₀ : PaperPositiveInitialDatum intervalDomain u₀` and uses `paperPositiveFloor hu₀`.
3. `BFormBankedInputs` itself has `huPaper : PaperPositiveInitialDatum intervalDomain u₀`, so direct classical discharges through `BFormDirectFrontier` are PPID-only.

Relevant file references:

* `PositiveDatumBFormLocalComponentsSq.huPaper`: `IntervalBFormPositiveDatumLocalExistenceSq.lean:36-44`.
* `PositiveDatumBFormLocalComponentsSqRegular.huPaper`: `IntervalBFormPositiveDatumLocalExistenceSqRegular.lean:40-48`.
* `PositiveDatumBFormSqDeepestHypotheses.bank` uses `BFormBankedInputs`: `IntervalBFormPositiveDatumLocalExistenceSqDeepest.lean:38-58`.
* `BFormBankedInputs.huPaper`: `IntervalBFormDirectClassical.lean:75-83`.
* `bform_strictPos_closed` requires PPID: `IntervalBFormStrictPosClosed.lean:5-18` in the fetched final chunk.

Thus any attempt to instantiate a PPID theorem from weak `PositiveInitialDatum` is a false route.
