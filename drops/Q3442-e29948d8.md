# Q3442: zero-slab `liftDeriv2` representative audit

Audited the requested current main commit:

`7094530f879d9ace06ba7b3c098d340395737cf2` — `Task 74: package H1 zero slab lap frontier`.

## Verdict

A proof of

```lean
H1LiftDeriv2HasZeroSlabRepresentative u
```

or the stronger

```lean
H1LiftDeriv2ZeroSlabRepresentativeBefore u T F
```

is **not** now just existing wiring from the physical-RHS / transfer files.

Task74 added the correct target records and downstream bridge wrappers, but the physical/transfer side still only proves **strict positive-time** representative continuity/equality. The zero-slab records require data on a slab whose time set contains `0`, and their `EqOn` domain includes `t = 0`:

```lean
Set.Icc (0 : ℝ) b ×ˢ Set.Ioo (0 : ℝ) 1
```

The current physical/transfer proofs give domains of the form

```lean
Set.Icc a b ×ˢ Set.Ioo (0 : ℝ) 1
```

with `0 < a`. That does not cover the endpoint `t = 0`. This is the real gap.

## Shortest attempted chain and where it fails

The shortest conceptual chain would be:

1. choose the representative

```lean
F := liftDeriv2PhysicalRHSWithChemRep p u
       (liftChemotaxisDivPhysicalRep p u v)
```

2. prove `ContinuousOn (Function.uncurry F) (Set.Icc 0 b ×ˢ Set.Icc 0 1)` from the component constructor;
3. prove

```lean
Set.EqOn
  (Function.uncurry (fun t x => liftDeriv2 u t x))
  (Function.uncurry F)
  (Set.Icc (0 : ℝ) b ×ˢ Set.Ioo (0 : ℝ) 1)
```

4. package `⟨F, b, ...⟩` as `H1LiftDeriv2HasZeroSlabRepresentative u`.

Step 2 currently fails because the available time-derivative, logistic-reaction, and chem-representative continuity producers are only strict slabs `0 < a`.

Step 3 currently fails more seriously because the PDE equality theorem requires `t ∈ Set.Ioo 0 T`, and the zero-slab EqOn includes `t = 0`. There is no theorem in the checked files identifying `liftDeriv2 u 0 x` with the physical RHS representative at `t = 0` for `x ∈ Set.Ioo 0 1`.

This is why it is not just wiring.

## Adversarial checks
