# 1. Exact pointwise `C²` computation

Let

```text
F(z) = z * (a - b*z^α) = a*z - b*z^(α+1).
```

On the positive branch `z > 0`, real-power differentiation gives

```text
F'(z)  = a - b*(α+1)*z^α,
F''(z) = -b*α*(α+1)*z^(α-1).
```

Therefore

```text
L'(x)
  = u'(x) * (a - b*(α+1)*u(x)^α),
```

and

```text
L''(x)
  = -b*α*(α+1)*u(x)^(α-1)*(u'(x))^2
      + (a - b*(α+1)*u(x)^α)*u''(x).
```

Equivalently, this is the ordinary one-dimensional second-order chain rule

```text
(F ∘ u)'' = F''(u)*(u')^2 + F'(u)*u''.
```

No general Faà di Bruno development is needed at order two.

Assume

```text
0 < u(x) ≤ M,
|u'(x)| ≤ G1,
|u''(x)| ≤ G2
```

on `[0,1]`, together with `1 ≤ α`, `0 ≤ a`, and `0 ≤ b`. Since `α-1 ≥ 0`,

```text
u(x)^(α-1) ≤ M^(α-1),
u(x)^α     ≤ M^α.
```

Hence

```text
|L''(x)|
≤ b*α*(1+α)*M^(α-1)*G1^2
   + (a + b*(1+α)*M^α)*G2.
```

Define exactly as the repository does:

```lean
def B_log (a b α M G1 G2 : ℝ) : ℝ :=
  b * α * (1 + α) * M ^ (α - 1) * G1 ^ 2
    + (a + b * (1 + α) * M ^ α) * G2
```

Then

```text
∀ x ∈ [0,1], |L''(x)| ≤ B_log(a,b,α,M,G1,G2).
```

Because the interval has length one,

```text
∫₀¹ |L''(x)| dx ≤ B_log(a,b,α,M,G1,G2).
```

The repository proves these two statements as

```lean
F2_abs_le_B_log
logisticSourceFun_secondDeriv_abs_integral_le
```

in `IntervalLogisticSourceQuantBound.lean`.

## Dependence on the positive floor `δ`

For the paper's actual hypothesis `α ≥ 1`, **the numerical second-derivative bound does not require `δ`**. Positivity is still useful in Lean because Mathlib's real-`rpow` derivative lemmas are easiest to invoke on the nonzero positive branch, but the coefficient

```text
u^(α-1)
```

has a nonnegative exponent and is bounded above by `M^(α-1)`.

For the more general range `0 < α < 1`, the exponent `α-1` is negative. Then the lower bound

```text
δ ≤ u(x)
```

is essential, and monotonicity reverses in the expected way:

```text
u(x)^(α-1) ≤ δ^(α-1).
```

The corresponding bound becomes

```text
B_log^δ
  = b*α*(1+α)*δ^(α-1)*G1^2
      + (a + b*(1+α)*M^α)*G2.
```

Thus the clean piecewise coefficient is

```text
P(α,δ,M) =
  M^(α-1),   if α ≥ 1,
  δ^(α-1),   if 0 < α < 1,
```

and

```text
|L''|
≤ b*α*(1+α)*P(α,δ,M)*G1^2
   + (a+b*(1+α)*M^α)*G2.
```

For `CM2Params`, the landed theorem uses `p.hα : 1 ≤ p.α`, so the existing `B_log` with `M^(α-1)` is exactly the right specialization.

## Important correction about a linear `C²`-norm estimate

There is a quadratic gradient term `(u')²`. Consequently one cannot, with a constant independent of `u`, generally prove

```text
‖L''‖∞ ≤ C(M,δ,a,b,α) * (1 + ‖u‖C²)
```

as a globally linear estimate in `‖u‖C²`.

If

```text
N := max(‖u'‖∞, ‖u''‖∞),
```

then the natural estimate is

```text
‖L''‖∞
≤ A(M,δ,a,b,α)*N^2 + B(M,a,b,α)*N
≤ (A+B)*(1+N^2).
```

On a fixed positive-time window, if the bootstrap already provides an a priori bound `N ≤ K`, then one may absorb the square:

```text
N^2 ≤ K*N,
```

and obtain a window-linear estimate

```text
‖L''‖∞ ≤ (A*K+B)*N ≤ (A*K+B)*(1+N).
```

That linearized constant necessarily depends on the pre-existing window bound `K`. The repository wisely keeps the sharper explicit `G1² + G2` form instead of hiding this dependence.

# 2. Neumann inheritance

The first derivative is

```text
L'(x) = u'(x) * (a - b*(1+α)*u(x)^α).
```

Therefore

```text
u'(0) = 0  ⇒  L'(0) = 0,
u'(1) = 0  ⇒  L'(1) = 0.
```

So the logistic source inherits the Neumann boundary condition directly from the solution slice. No resolver or no-flux argument is involved.

# 5. The actual `WindowSourceEnvelope 2`, including the zero mode

The positive-mode estimate alone gives the `1/(kπ)²` tail. A full window envelope usually also carries a uniform zeroth-mode bound.

For `0 < u ≤ M`, `a,b ≥ 0`, and `α ≥ 1`,

```text
|L(x)|
= |u(x)*(a-b*u(x)^α)|
≤ M*(a+b*M^α).
```

Since the zero cosine mode is the unscaled integral over a unit interval,

```text
|cosineCoeffs L 0| ≤ M*(a+b*M^α).
```

Thus choose

```text
C_log := max
  (2 * B_log(a,b,α,M,G1,G2))
  (M * (a + b*M^α)).
```

Then

```text
k = 0:
  |cosineCoeffs L 0| ≤ C_log,

k ≥ 1:
  |cosineCoeffs L k| ≤ C_log / ((k:ℝ)*π)^2.
```

This exact constant is already defined in

```text
ShenWork/Paper2/IntervalPicardWeightedC2Bootstrap.lean
```

as

```lean
def windowSourceConst (p : CM2Params) (M G1 G2 : ℝ) : ℝ :=
  max
    (2 * B_log p.a p.b p.α M G1 G2)
    (M * (p.a + p.b * M ^ p.α))
```

and `windowSourceConst_nonneg` proves its nonnegativity.

Therefore the logistic half of the source pass is exactly:

```text
window-uniform bounds on u, u', u''
+ positivity of u
+ Neumann u'(0)=u'(1)=0
    ↓
explicit B_log bound on L''
    ↓
positive-mode coefficient decay 2*B_log/(kπ)^2
+ separate zero-mode sup bound
    ↓
WindowSourceEnvelope 2 for the logistic coefficient family.
```

The logistic branch is not an open analytic difficulty. For `α ≥ 1`, it is a short, fully formalized chain:

```text
u ∈ C², 0<u≤M, |u'|≤G1, |u''|≤G2
  ⇒ |L''|≤B_log
```

The substantive source-ladder work remains in the chemotaxis-divergence branch. The logistic half is already landed, explicit, axiom-clean, and should be reused rather than reproved.
