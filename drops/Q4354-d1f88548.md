# Truncated chemotaxis flux: the correct δ-free pointwise `C²` bound

Audit basis: current `Shen_work` code at commit `2347cb0365108ea74d7a64607cbdd4850c593c18`.

## Verdict

The intended **δ-free flux algebra is correct**, but the bound as stated needs one correction:

> From only `0 ≤ u ≤ M` and `R ∈ C³`, one cannot bound the full flux
> \(Q=u\,R_x(1+R)^{-\beta}\) in `C²` by a constant depending only on
> \(M,\beta,\|R\|_{C^3}\).

Indeed,

\[
Q''=u''g+2u'g'+ug'',
\qquad
 g=R'(1+R)^{-\beta},
\]

so bounds for `u'` and `u''` are load-bearing. What **is** true is the following.

Let

\[
U_j:=\|\partial_x^j u\|_{L^\infty(0,1)},\qquad j=0,1,2,
\]

and

\[
K_j:=\|\partial_x^j R\|_{L^\infty(0,1)},\qquad j=1,2,3.
\]

Assume

\[
\beta\ge0,\qquad R\ge0,
\]

so that `1 + R ≥ 1`. Then the full pointwise `C²` flux bound depends on

\[
(U_0,U_1,U_2,\beta,K_1,K_2,K_3),
\]

and **contains no positive lower bound on `u`**. In particular, if one common constant `U` controls all three solution derivatives,

\[
U_0,U_1,U_2\le U,
\]

and `K := ‖R‖_{C³}` controls `K₁,K₂,K₃`, then, for the max convention

\[
\|Q\|_{C^2,\max}
 :=\max\{\|Q\|_\infty,\|Q'\|_\infty,\|Q''\|_\infty\},
\]

one may take the explicit polynomial constant

\[
\boxed{
\|Q\|_{C^2,\max}
\le
U\Bigl(4K+5\beta K^2+\beta(\beta+1)K^3\Bigr).
}
\]

Thus the desired `C(M, β, ‖R‖_{C³})` form is valid if the symbol `M` means a **common `C²` envelope for `u`**, not merely an `L∞` bound.

# 1. Denominator derivatives

Set

\[
A:=1+R,
\qquad
h:=A^{-\beta}=(1+R)^{-\beta}.
\]

Since `R ≥ 0`,

\[
A\ge1.
\]

Since `β ≥ 0`, all exponents `-β`, `-β-1`, and `-β-2` are nonpositive. Hence

\[
0\le A^{-\beta-j}\le1,
\qquad j=0,1,2.
\]

The first two derivatives are

\[
h'
 =-\beta A^{-\beta-1}R',
\]

and

\[
h''
 =\beta(\beta+1)A^{-\beta-2}(R')^2
   -\beta A^{-\beta-1}R''.
\]

Consequently,

\[
|h|\le1,
\]

\[
|h'|\le \beta K_1,
\]

and

\[
|h''|
\le \beta(\beta+1)K_1^2+\beta K_2.
\]

This exact second-derivative identity and its pointwise bound are already represented in the repository by

```lean
ShenWork.Paper2.IntervalDenomSecondDerivBound.denomSecondDeriv_hasDerivAt
ShenWork.Paper2.IntervalDenomSecondDerivBound.denomSecondDeriv_abs_le
```

The latter uses the Mathlib inequality

```lean
Real.rpow_le_one_of_one_le_of_nonpos
```

together with

```lean
Real.rpow_nonneg
```

to dispose of all denominator powers by the fixed lower bound `1 + R ≥ 1`. No lower bound `δ ≤ u` occurs here.

# 2. Chemical-gradient factor `g`

Define

\[
g:=R'h=R'(1+R)^{-\beta}.
\]

The first derivative is

\[
\begin{aligned}
g'
 &=R''h+R'h'\\
 &=R''A^{-\beta}
   -\beta(R')^2A^{-\beta-1}.
\end{aligned}
\]

Thus

\[
\boxed{|g|\le K_1,}
\]

and

\[
\boxed{|g'|\le K_2+\beta K_1^2.}
\]

The first-derivative formula is the same chain rule used in

```lean
ShenWork.MinPersistenceAtoms.flux_integrand_hasDerivAt
```

from `IntervalDomainFluxIntegrandDeriv.lean`.

For the second derivative,

\[
\begin{aligned}
g''
 &=R'''h+2R''h'+R'h''\\
 &=R'''A^{-\beta}
   -3\beta R'R''A^{-\beta-1}
   +\beta(\beta+1)(R')^3A^{-\beta-2}.
\end{aligned}
\]

Therefore

\[
\boxed{
|g''|
\le
K_3+3\beta K_1K_2+\beta(\beta+1)K_1^3.
}
\]

If

\[
K_1,K_2,K_3\le K,
\]

then the convenient common bounds are

\[
G_0:=K,
\]

\[
G_1:=K+\beta K^2,
\]

and

\[
G_2:=K+3\beta K^2+\beta(\beta+1)K^3,
\]

with

\[
|g|\le G_0,
\qquad
|g'|\le G_1,
\qquad
|g''|\le G_2.
\]

Again, this is completely independent of a positive floor for `u`.

# 3. Product `Q = u g`

Now let

\[
Q:=ug.
\]

The derivative identities are

\[
Q'=u'g+ug',
\]

and

\[
Q''=u''g+2u'g'+ug''.
\]

Hence, pointwise,

\[
|Q|
\le U_0K_1,
\]

\[
|Q'|
\le U_1K_1+U_0(K_2+\beta K_1^2),
\]

and

\[
\boxed{
\begin{aligned}
|Q''|
\le{}& U_2K_1
 +2U_1(K_2+\beta K_1^2)\\
&+U_0\bigl(
K_3+3\beta K_1K_2+\beta(\beta+1)K_1^3
\bigr).
\end{aligned}
}
\]

This is the precise non-collapsed constant.

Using one resolver constant `K` gives

\[
|Q|\le U_0K,
\]

\[
|Q'|\le U_1K+U_0(K+\beta K^2),
\]

and

\[
\boxed{
|Q''|
\le
U_2K+2U_1(K+\beta K^2)
+U_0\bigl(K+3\beta K^2+\beta(\beta+1)K^3\bigr).
}
\]

A formal max-norm constant is therefore

\[
\begin{aligned}
C_{Q,2}(U_0,U_1,U_2,\beta,K)
:=\max\{&U_0K,\\
&U_1K+U_0(K+\beta K^2),\\
&U_2K+2U_1(K+\beta K^2)\\
&\quad+U_0(K+3\beta K^2+\beta(\beta+1)K^3)
\}.
\end{aligned}
\]

Then

\[
\|Q\|_{C^2,\max}\le C_{Q,2}.
\]

## Common `C²` solution envelope

If

\[
U_0,U_1,U_2\le U,
\]

then

\[
|Q''|
\le
U\left[
K+2(K+\beta K^2)
 +(K+3\beta K^2+\beta(\beta+1)K^3)
\right],
\]

hence

\[
|Q''|
\le
U\Bigl(4K+5\beta K^2+\beta(\beta+1)K^3\Bigr).
\]

This last expression also dominates the displayed `Q` and `Q'` bounds, so

\[
\boxed{
\|Q\|_{C^2,\max}
\le
U\Bigl(4K+5\beta K^2+\beta(\beta+1)K^3\Bigr).
}
\]

If the project defines the `C²` norm by the sum

\[
\|Q\|_{C^2,\Sigma}
 :=\|Q\|_\infty+\|Q'\|_\infty+\|Q''\|_\infty,
\]

then the same common envelope gives

\[
\boxed{
\|Q\|_{C^2,\Sigma}
\le
U\Bigl(7K+6\beta K^2+\beta(\beta+1)K^3\Bigr).
}
\]

# 4. Why an `L∞` bound on `u` is not enough

The occurrence of `u'` and `u''` is not removable by an estimate. It is a genuine derivative-counting requirement.

The repository already reflects the first-order version of this fact. The theorem

```lean
ShenWork.Paper2.TruncatedPositiveTimeBootstrap.
  truncatedChemFluxLifted_deriv_abs_le_of_ball_grad
```

requires both a ball bound and a separate gradient envelope. Its schematic conclusion is

```text
|Q'| ≤ M·H + β·M·Γ² + Γ·G,
```

where `G` bounds the solution gradient. Thus even `Q'` is not controlled by `‖u‖∞` alone; `Q''` additionally needs a second-derivative envelope for `u`.

## The elliptic identity does not repair the `C³` deficit

Suppose, as in the question, that

\[
u=\mu R-\nu R''.
\]

Then

\[
u'=\mu R'-\nu R''',
\]

so `R ∈ C³` controls `u'`. But

\[
u''=\mu R''-\nu R'''',
\]

so controlling `u''` requires `R ∈ C⁴`, not merely `R ∈ C³`.

There is no cancellation of the `R''''` term in `Q''`: it appears in `u''g` with coefficient `-νg`.

A concrete high-frequency family shows the failure. Take, for large `n`,

\[
R_n(x)=C+a\cos(\pi x)+n^{-3}\cos(n\pi x),
\]

with `C` large. These profiles satisfy Neumann boundary conditions and have uniformly bounded `C³` norms. The corresponding

\[
u_n=\mu R_n-\nu R_n''
\]

can be kept nonnegative and uniformly bounded, while `u_n''` has a component of order `n`. At points where the low-frequency part of `R_n'` is nonzero, `u_n''g_n` makes `Q_n''` unbounded. Hence no universal

\[
C(M,\beta,\|R\|_{C^3})
\]

exists under the literal hypotheses of the question.

There are two valid repairs:

1. carry explicit `U₁` and `U₂` solution derivative bounds; or
2. assume `R ∈ C⁴` and use the elliptic identity to derive `U₂`.

# 5. The δ-floor issue

## What is genuinely δ-free

Conditional on

```text
u ∈ C²,
0 ≤ u,
R ∈ C³,
R ≥ 0,
β ≥ 0,
```

the entire algebra above is δ-free. It uses only

```text
1 + R ≥ 1,
```

not `δ ≤ u`.

The fixed denominator floor is supplied by resolver nonnegativity:

\[
R\ge0\quad\Longrightarrow\quad1+R\ge1.
\]

Thus no factor of `δ⁻¹`, `δ^(γ-2)`, or any other solution lower-floor constant enters the flux-product estimate.

## Where δ may enter upstream

In the actual chemotaxis system, the resolver source is typically a real power such as `u^γ`. If `1 ≤ γ < 2`, producing a `C²` bound for that **bare power source** requires `u` to stay away from zero, since

\[
(u^\gamma)''
=\gamma(\gamma-1)u^{\gamma-2}(u')^2
 +\gamma u^{\gamma-1}u''.
\]

Therefore a positive floor may be needed upstream to produce the asserted `R ∈ C³` or stronger resolver bound. But once the `R` derivative bounds are handed to the flux lemma, the flux estimate itself has no δ dependence.

So the precise statement is:

> The **flux algebra and multiplier estimate** are δ-free; the theorem that supplies the resolver regularity may use δ if its elliptic source is a noninteger real power at an order where that power is singular at zero.

# 6. Truncation `u = ρ₊`

There is another important distinction.

The scalar map

\[
r\mapsto r_+=\max(r,0)
\]

is only Lipschitz, not `C²` at zero. Thus from a `C²` profile `ρ` one cannot conclude that `ρ₊` is pointwise `C²` before proving nonnegativity or imposing special zero-contact conditions.

The correct architecture is:

```text
weak energy argument
    ⇒ ρ ≥ 0 on the active window
    ⇒ ρ₊ = ρ
    ⇒ rewrite the truncated flux as the ordinary flux
    ⇒ differentiate the ordinary C² profile ρ.
```

The repository already contains the exact rewrite:

```lean
ShenWork.Paper2.BFormPositiveDatumNegPart.
  truncatedChemFluxLifted_eq_chemFluxLifted_of_nonneg
```

in `IntervalBFormFaithfulBridgeProducer.lean`.

Therefore:

- **no strict floor** `δ ≤ ρ` is needed to remove the truncation;
- ordinary nonnegativity `0 ≤ ρ` is enough;
- but one should rewrite to the untruncated flux before invoking classical derivative formulas.

Before nonnegativity, only weak/Lipschitz positive-part chain rules are available. The repository's `IntervalTransversalZeroSet.lean` explicitly reflects this limitation.

# 7. Lean organization

A clean formalization should avoid repeatedly rewriting nested `deriv` expressions. Define the expected derivative tower explicitly, as the repository does for the power source.

```lean
def denom0 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x => (1 + R x) ^ (-β)

def denom1 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x => -β * (1 + R x) ^ (-β - 1) * deriv R x

def denom2 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x =>
    β * (β + 1) * (1 + R x) ^ (-β - 2) * (deriv R x) ^ 2
      - β * (1 + R x) ^ (-β - 1) * deriv (deriv R) x

def chemGrad0 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x => deriv R x * denom0 β R x

def chemGrad1 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x =>
    deriv (deriv R) x * denom0 β R x
      + deriv R x * denom1 β R x

def chemGrad2 (β : ℝ) (R : ℝ → ℝ) : ℝ → ℝ :=
  fun x =>
    deriv (deriv (deriv R)) x * denom0 β R x
      + 2 * deriv (deriv R) x * denom1 β R x
      + deriv R x * denom2 β R x

def flux0 (β : ℝ) (u R : ℝ → ℝ) : ℝ → ℝ :=
  fun x => u x * chemGrad0 β R x

def flux1 (β : ℝ) (u R : ℝ → ℝ) : ℝ → ℝ :=
  fun x =>
    deriv u x * chemGrad0 β R x
      + u x * chemGrad1 β R x

def flux2 (β : ℝ) (u R : ℝ → ℝ) : ℝ → ℝ :=
  fun x =>
    deriv (deriv u) x * chemGrad0 β R x
      + 2 * deriv u x * chemGrad1 β R x
      + u x * chemGrad2 β R x
```

Then prove, pointwise:

```lean
HasDerivAt (denom0 β R) (denom1 β R x) x
HasDerivAt (denom1 β R) (denom2 β R x) x
HasDerivAt (chemGrad0 β R) (chemGrad1 β R x) x
HasDerivAt (chemGrad1 β R) (chemGrad2 β R x) x
HasDerivAt (flux0 β u R) (flux1 β u R x) x
HasDerivAt (flux1 β u R) (flux2 β u R x) x
```

using only the standard combinators

```lean
HasDerivAt.rpow_const
HasDerivAt.mul
HasDerivAt.add
HasDerivAt.sub
HasDerivAt.const_mul
ContDiff.deriv'
```

and the already committed denominator theorem

```lean

```

for the second line.

For regularity of the denominator itself, the repository already has

```lean
ShenWork.Paper2.IntervalCkComposition.contDiff_two_one_add_rpow_neg
```

which proves that `(1 + R)^(-β)` is `C²` from `R ∈ C²` and `R ≥ 0`.

For the pointwise inequalities, the useful Mathlib tools are

```lean

abs_mul
Real.rpow_nonneg
Real.rpow_le_one_of_one_le_of_nonpos
mul_le_mul

```

followed by `ring`/`ring_nf` and `nlinarith` for the polynomial rearrangements.

A convenient Lean constant layer is:

```lean
def chemGradC0 (K : ℝ) : ℝ := K

def chemGradC1 (β K : ℝ) : ℝ :=
  K + β * K ^ 2

def chemGradC2 (β K : ℝ) : ℝ :=
  K + 3 * β * K ^ 2 + β * (β + 1) * K ^ 3

def fluxC2PointConst
    (U0 U1 U2 β K : ℝ) : ℝ :=
  U2 * chemGradC0 K
    + 2 * U1 * chemGradC1 β K
    + U0 * chemGradC2 β K

def fluxC2CommonConst (U β K : ℝ) : ℝ :=
  U * (4 * K + 5 * β * K ^ 2 + β * (β + 1) * K ^ 3)
```

The central pointwise theorem should conclude

```lean
|flux2 β u R x| ≤ fluxC2PointConst U0 U1 U2 β K
```

from the six pointwise derivative bounds and nonnegativity hypotheses. The common-envelope corollary then concludes

```lean
|flux2 β u R x| ≤ fluxC2CommonConst U β K
```

when `U0,U1,U2 ≤ U`.

# 8. Scope relative to the source ladder

This result gives

```text
u ∈ C², R ∈ C³  ⇒  Q ∈ C²  ⇒  ∂ₓQ ∈ C¹.
```

It does **not** by itself give the chemotaxis-divergence source `∂ₓQ` in `C²`. For that stronger target one needs `Q ∈ C³`, hence one additional derivative on the factors—typically `u ∈ C³` and `R ∈ C⁴`, or the corresponding weak `W^{3,1}` package.

That derivative count should be matched to the precise `WindowSourceEnvelope` pass being discharged. The present `C²` flux estimate is nevertheless the correct δ-free algebraic component at its stated order.

# Final answer

The exact δ-free factor bounds are

\[
|h|\le1,
\qquad
|h'|\le\beta K,
\qquad
|h''|\le\beta(\beta+1)K^2+\beta K,
\]

\[
|g|\le K,
\qquad
|g'|\le K+\beta K^2,
\qquad
|g''|\le K+3\beta K^2+\beta(\beta+1)K^3.
\]

Consequently,

\[
\boxed{
|Q''|
\le
U_2K+2U_1(K+\beta K^2)
+U_0\bigl(K+3\beta K^2+\beta(\beta+1)K^3\bigr).
}
\]

If one common `C²` solution bound `U` controls `u,u',u''`, then

\[
\boxed{
\|Q\|_{C^2,\max}
\le
U\Bigl(4K+5\beta K^2+\beta(\beta+1)K^3\Bigr),
\qquad K=\|R\|_{C^3}.
}
\]

This constant has no δ-floor dependence. But with only `0 ≤ u ≤ M`, the claimed `C(M,β,‖R‖_{C³})` bound is false: one must also carry `u'` and `u''` bounds, or strengthen the elliptic input to `R ∈ C⁴`.
