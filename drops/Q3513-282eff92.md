## 1. Will the Q3509 theorem compile as stated?

Probably **not as stated**. The problem is not the Young inequality. The problem is measurability.

The proposed theorem only assumes per-time square-integrability of

* `fun x => (liftDeriv2 u r x)^2`,
* `fun x => (H1PhysicalChemTaxisPart p u v r x)^2`,
* `fun x => (H1PhysicalChemUvxxPart p u v r x)^2`,
* `fun x => (H1PhysicalLogisticReactionPart p u r x)^2`.

That is not enough, in Lean or mathematically, to prove

```lean
IntervalIntegrable (fun x => |f x * g x|) volume (0 : ℝ) 1
```

because `IntervalIntegrable (fun x => f x ^ 2)` gives measurability/integrability of `f^2`, not measurability of `f` or `|f*g|`. A nonmeasurable sign function can have measurable square, so this is a real missing hypothesis, not just an API annoyance.

This is also visible in the current local APIs: `GagliardoNirenberg.norm_integral_mul_le_half_sq_sum` takes product integrability explicitly:

```lean
(hfg : IntervalIntegrable (fun y => |f y * g y|) volume 0 L)
```

and the existing physical fixed-time wrappers in `IntervalChiNegH1PhysicalYoungSpatial.lean` pass exactly that product-integrability input. The spatial-data structure also already stores three product-integrability fields.

So the corrected theorem should either add **product a.e.-strong measurability fields** or add **factor a.e.-strong measurability fields** from which product measurability can be built. The minimal corrected shape is product-measurability:

```lean
(hTaxisProd_meas :
  ∀ᵐ r ∂volume.restrict (Set.Ioc (0 : ℝ) δ),
    AEStronglyMeasurable
      (fun x => |liftDeriv2 u r x * H1PhysicalChemTaxisPart p u v r x|)
      (volume.restrict (Set.Ioc (0 : ℝ) 1)))
(hUvxxProd_meas :
  ∀ᵐ r ∂volume.restrict (Set.Ioc (0 : ℝ) δ),
    AEStronglyMeasurable
      (fun x => |liftDeriv2 u r x * H1PhysicalChemUvxxPart p u v r x|)
      (volume.restrict (Set.Ioc (0 : ℝ) 1)))
(hReactProd_meas :
  ∀ᵐ r ∂volume.restrict (Set.Ioc (0 : ℝ) δ),
    AEStronglyMeasurable
      (fun x => |liftDeriv2 u r x * H1PhysicalLogisticReactionPart p u r x|)
      (volume.restrict (Set.Ioc (0 : ℝ) 1)))
```

A stronger but often easier source-facing shape is to assume factor measurability a.e. in `r`, for example measurability of `fun x => liftDeriv2 u r x` and of each physical non-lap factor. Then each product field follows by `(hf_meas.mul hg_meas).norm`, with `simpa [Real.norm_eq_abs]`.

## 2. Strongest reusable helper statement

The best reusable helper is generic, interval-based, and only adds the exact missing measurability. It should not mention H¹ or physical pieces.

I recommend adding this to `ShenWork/PDE/GagliardoNirenberg.lean`, inside namespace `ShenWork.GagliardoNirenberg`, near the existing Cauchy/Young lemmas.

```lean
import ShenWork.PDE.GagliardoNirenberg

open MeasureTheory Set intervalIntegral
open scoped ENNReal Interval

noncomputable section

namespace ShenWork.GagliardoNirenberg

/-- Product integrability from square integrability, assuming the product is
measurable on the interval.  The measurability assumption is essential: square
integrability of `f^2` and `g^2` alone does not imply measurability of `f` or `g`. -/
theorem intervalIntegrable_abs_mul_of_sq_integrable_of_aestronglyMeasurable
    {a b : ℝ} (hab : a ≤ b)
    {f g : ℝ → ℝ}
    (hf_sq : IntervalIntegrable (fun x => f x ^ 2) volume a b)
    (hg_sq : IntervalIntegrable (fun x => g x ^ 2) volume a b)
    (hfg_meas : AEStronglyMeasurable (fun x => |f x * g x|)
      (volume.restrict (Set.Ioc a b))) :
    IntervalIntegrable (fun x => |f x * g x|) volume a b := by
  let dom : ℝ → ℝ := fun x =>
    ((1 : ℝ) / 2) * f x ^ 2 + ((1 : ℝ) / 2) * g x ^ 2
  have hdom_interval : IntervalIntegrable dom volume a b := by
    dsimp [dom]
    exact (hf_sq.const_mul ((1 : ℝ) / 2)).add
      (hg_sq.const_mul ((1 : ℝ) / 2))
  have hbound : ∀ᵐ x ∂volume.restrict (Set.Ioc a b),
      ‖|f x * g x|‖ ≤ ‖dom x‖ := by
    refine ae_of_all _ ?_
    intro x
    have hprod_nonneg : 0 ≤ |f x * g x| := abs_nonneg _
    have hdom_nonneg : 0 ≤ dom x := by
      dsimp [dom]
      nlinarith [sq_nonneg (f x), sq_nonneg (g x)]
    have hpt : |f x * g x| ≤ dom x := by
      dsimp [dom]
      rw [abs_mul]
      nlinarith [sq_nonneg (|f x| - |g x|), sq_abs (f x), sq_abs (g x)]
    rw [Real.norm_eq_abs, abs_of_nonneg hprod_nonneg,
      Real.norm_eq_abs, abs_of_nonneg hdom_nonneg]
    exact hpt
  rw [intervalIntegrable_iff_integrableOn_Ioc_of_le hab]
  rw [IntegrableOn]
  have hdom_integrable : Integrable dom (volume.restrict (Set.Ioc a b)) := by
    rw [intervalIntegrable_iff_integrableOn_Ioc_of_le hab] at hdom_interval
    simpa [IntegrableOn] using hdom_interval
  exact Integrable.mono' hdom_integrable hfg_meas hbound

/-- Source-friendly corollary: factor measurability implies the product
measurability required by
`intervalIntegrable_abs_mul_of_sq_integrable_of_aestronglyMeasurable`. -/
theorem intervalIntegrable_abs_mul_of_sq_integrable_of_factor_aestronglyMeasurable
    {a b : ℝ} (hab : a ≤ b)
    {f g : ℝ → ℝ}
    (hf_sq : IntervalIntegrable (fun x => f x ^ 2) volume a b)
    (hg_sq : IntervalIntegrable (fun x => g x ^ 2) volume a b)
    (hf_meas : AEStronglyMeasurable f (volume.restrict (Set.Ioc a b)))
    (hg_meas : AEStronglyMeasurable g (volume.restrict (Set.Ioc a b))) :
    IntervalIntegrable (fun x => |f x * g x|) volume a b := by
  refine intervalIntegrable_abs_mul_of_sq_integrable_of_aestronglyMeasurable
    (a := a) (b := b) (f := f) (g := g) hab hf_sq hg_sq ?_
  simpa [Real.norm_eq_abs] using (hf_meas.mul hg_meas).norm

#print axioms intervalIntegrable_abs_mul_of_sq_integrable_of_aestronglyMeasurable
#print axioms intervalIntegrable_abs_mul_of_sq_integrable_of_factor_aestronglyMeasurable

end ShenWork.GagliardoNirenberg
```

This proof shape mirrors existing local patterns:

* `.const_mul` and `.add` are already used for interval-integrability algebra.
* `intervalIntegrable_iff_integrableOn_Ioc_of_le` plus `Integrable.mono'` is already the local majorant pattern.
* `ae_of_all _` is already used for pointwise majorants.
* The pointwise estimate is pure real arithmetic: `|f*g| ≤ (f^2 + g^2)/2`.

If the final `simpa [Real.norm_eq_abs] using (hf_meas.mul hg_meas).norm` is brittle, replace it with an explicit `have hp := hf_meas.mul hg_meas; simpa [Real.norm_eq_abs] using hp.norm`.

## 4. File placement and imports/cycles

Recommended placement:

1. Put the generic helper in `ShenWork/PDE/GagliardoNirenberg.lean`, namespace `ShenWork.GagliardoNirenberg`, near the existing interval Young lemmas.
2. Put the physical packaging theorem in `ShenWork/Paper2/IntervalChiNegH1PhysicalYoungSpatial.lean`.

This avoids cycles because `IntervalChiNegH1PhysicalYoungSpatial.lean` already imports `ShenWork.PDE.GagliardoNirenberg`. No new helper file is necessary.

Putting the helper directly in `IntervalChiNegH1PhysicalYoungSpatial.lean` would also avoid cycles, but it is less reusable and duplicates the generic nature of the argument. A new helper file such as `IntervalIntegrableYoung.lean` is possible but unnecessary import churn for this small lemma.

## Bottom line

The Q3509 proof idea is right, but the theorem statement is missing product measurability. The corrected local wrapper is very likely small once the generic helper above exists. The proof should be pure packaging plus `filter_upwards`, and it should not touch bounded-before, RHS integrability, derivative-integrability producers, or any downstream H¹ route package.
