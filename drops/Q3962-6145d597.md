# Q3962: correct source constants for the truncated gradient window

## Bottom line

The constants `A_L = A_F = B_F = 0` are not admissible. They make

```lean
truncWindowSourceCL A_L A_F B_F p.χ₀ G = 0
```

and therefore force `Src n s y = 0`, which is false for both the logistic and chemotaxis pieces.

There are two mathematically consistent ways to wire the source envelope.

The safest one, and the one matching the quantitative resolver-gradient envelopes already present in the current repository, is an **M-only flux bound**:

```lean
A_L = M * (p.a + p.b * M ^ p.α)
A_F = M * Fg(p, M)
B_F = 0
```

where `Fg(p,M)` is a uniform bound for `|resolverGradReal p u y|` on the `M`-ball. In the current repository resolver, whose elliptic source is `p.ν * u ^ p.γ`, the explicit spectral value is

```lean
Fg(p, M) =
  Real.sqrt (∑' k : ℕ,
    (ShenWork.PDE.intervalNeumannResolverGradWeight p k) ^ 2)
    * (2 * (p.ν * M ^ p.γ))
```

If your local cron1 branch has the normalized resolver source `(u₊)^m` instead of the current repository's `p.ν * u^p.γ`, replace `p.ν * M ^ p.γ` by `M ^ p.m` in this formula, with the corresponding normalized gradient-resolvent weight.

This gives

```lean
|Src n s y| ≤ A_L + |p.χ₀| * A_F
           = truncWindowSourceCL A_L A_F 0 p.χ₀ G
```

and the gradient part of the affine recurrence has `B_F = 0`, so `truncWindowB = 0 < 1` automatically.

A genuine nonzero `B_F` is possible only after adding a separate theorem bounding the resolver gradient by the iterate gradient, for example

```lean
|resolverGradReal p (U n s) y| ≤ K_R(p,M) * G
```

on the window. Then one may take

```lean
A_F = 0
B_F = M * K_R(p, M)
```

but this is an extra analytic input. It is not implied by the `M`-ball alone, and for powers below `1` it needs a positive lower bound or a different truncation argument.

## Lean-shape constants

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow
import ShenWork.Paper2.IntervalBFormNegativePartCron2
import ShenWork.Paper2.IntervalDomainResolverSupQuantitative

open scoped BigOperators Real

noncomputable section

namespace ShenWork.Paper2.TruncatedSourceEnvelope

/-- Current-repository resolver-gradient envelope.

The current repository's `intervalNeumannResolverR` is driven by the elliptic
source `p.ν * u ^ p.γ`.  Its quantitative gradient envelope is the spectral
operator norm times the source `L²`/cosine coefficient bound `2 * p.ν * M^γ`. -/
def resolverGradEnvelopeCurrent (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ,
      (ShenWork.PDE.intervalNeumannResolverGradWeight p k) ^ 2)
    * (2 * (p.ν * M ^ p.γ))

/-- Logistic source envelope on `|u| ≤ M`. -/
def truncLogisticEnvelope (p : CM2Params) (M : ℝ) : ℝ :=
  M * (p.a + p.b * M ^ p.α)

/-- Flux envelope using only the `M`-ball and the elliptic resolver gradient bound. -/
def truncFluxEnvelopeCurrent (p : CM2Params) (M : ℝ) : ℝ :=
  M * resolverGradEnvelopeCurrent p M

/-- Recommended constants for the already-formalized current-repository route. -/
def sourceAL (p : CM2Params) (M : ℝ) : ℝ :=
  truncLogisticEnvelope p M

def sourceAF_current (p : CM2Params) (M : ℝ) : ℝ :=
  truncFluxEnvelopeCurrent p M

def sourceBF_current (_p : CM2Params) (_M : ℝ) : ℝ :=
  0

/-- If the local branch uses a normalized resolver source `(u₊)^m`, use this
shape instead: replace `p.ν * M^p.γ` by `M^p.m`.  The weight symbol here should
be replaced by the actual normalized resolver gradient weight in that branch. -/
def resolverGradEnvelopePowerShape
    (resolverGradWeight : CM2Params → ℕ → ℝ) (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ, (resolverGradWeight p k) ^ 2) * (2 * M ^ p.m)

end ShenWork.Paper2.TruncatedSourceEnvelope
```

## 1. Logistic term

Let

```lean
u = intervalDomainLift (U n s) y
u₊ = positivePart u
```

Assume `0 ≤ M` and `|u| ≤ M`. Then `0 ≤ u₊ ≤ M`. Since `p.a ≥ 0`, `p.b ≥ 0`, and `p.α > 0`,

```lean
|truncatedLogisticLifted p (U n s) y|
  = |u * (p.a - p.b * u₊ ^ p.α)|
  ≤ |u| * (p.a + p.b * u₊ ^ p.α)
  ≤ M * (p.a + p.b * M ^ p.α)
```

So the correct logistic constant is

```lean
def A_L (p : CM2Params) (M : ℝ) : ℝ :=
  M * (p.a + p.b * M ^ p.α)
```

This is the first place where `A_L = 0` fails: even if `χ₀ = 0`, the logistic source is generally nonzero.

## 2. Chemotaxis flux term

Write

```lean
R  = intervalDomainLift (ShenWork.PDE.intervalNeumannResolverR p (U n s)) y
Rx = resolverGradReal p (U n s) y
F  = truncatedChemFluxLifted p (U n s) y
   = u₊ * Rx / (1 + R) ^ p.β
```

Under the standard resolver positivity `0 ≤ R` and `p.β ≥ 0`,

```lean
1 ≤ (1 + R) ^ p.β
```

so the denominator does not increase the absolute value. Hence

```lean
|F| ≤ u₊ * |Rx| ≤ M * |Rx|.
```

Using the repository's quantitative resolver-gradient envelope,

```lean
|Rx| ≤ Fg(p,M)
```

where

```lean
def Fg (p : CM2Params) (M : ℝ) : ℝ :=
  Real.sqrt (∑' k : ℕ,
    (ShenWork.PDE.intervalNeumannResolverGradWeight p k) ^ 2)
      * (2 * (p.ν * M ^ p.γ))
```

gives

```lean
|truncatedChemFluxLifted p (U n s) y| ≤ M * Fg(p,M).
```

Thus the recommended flux constants are

```lean
def A_F (p : CM2Params) (M : ℝ) : ℝ :=
  M * Fg p M

def B_F (_p : CM2Params) (_M : ℝ) : ℝ :=
  0
```

This still has the requested affine form

```lean
|F| ≤ A_F + B_F * |deriv (intervalDomainLift (U n s)) y|
```

because `B_F = 0`. The iterate gradient hypothesis is simply unused in `hsource_of_grad`.

## 3. Combined source bound

For

```lean
Src n s y =
  truncatedLogisticLifted p (U n s) y
    - p.χ₀ * truncatedChemFluxLifted p (U n s) y
```

the triangle inequality gives

```lean
|Src n s y|
  ≤ |truncatedLogisticLifted p (U n s) y|
      + |p.χ₀| * |truncatedChemFluxLifted p (U n s) y|
  ≤ A_L + |p.χ₀| * (A_F + B_F * G)
  = truncWindowSourceCL A_L A_F B_F p.χ₀ G.
```

In Lean-shape form, the proof should be routed through the already existing algebra lemma:

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow
import ShenWork.Paper2.IntervalBFormNegativePartCron2
import ShenWork.Paper2.IntervalDomainResolverSupQuantitative

open ShenWork.Paper2.TruncatedGradientWindow
open ShenWork.Paper2.BFormPositiveDatumNegPart

noncomputable section

namespace ShenWork.Paper2.TruncatedSourceEnvelope

variable {p : CM2Params}
variable {U : ℕ → ℝ → ShenWork.IntervalDomain.intervalDomainPoint → ℝ}
variable {M G : ℝ}

-- Mathematical shape, not a ready-to-paste proof term.
theorem source_bound_shape
    (hL : ∀ n s y,
      |truncatedLogisticLifted p (U n s) y| ≤ sourceAL p M)
    (hF : ∀ n s y,
      |truncatedChemFluxLifted p (U n s) y| ≤ sourceAF_current p M)
    (n : ℕ) (s y : ℝ) :
    |truncatedLogisticLifted p (U n s) y
        - p.χ₀ * truncatedChemFluxLifted p (U n s) y|
      ≤ truncWindowSourceCL
          (sourceAL p M) (sourceAF_current p M) 0 p.χ₀ G := by
  -- Conceptually:
  --   exact abs_logistic_sub_chi_flux_le (hL n s y) ?flux
  -- where `?flux` is `hF n s y` rewritten as
  --   |F| ≤ A_F + 0 * G.
  have hF' :
      |truncatedChemFluxLifted p (U n s) y|
        ≤ sourceAF_current p M + 0 * G := by
    simpa using hF n s y
  exact abs_logistic_sub_chi_flux_le (hL n s y) hF'

end ShenWork.Paper2.TruncatedSourceEnvelope
```

The key replacement for the bad `hsource_of_grad` is therefore:

```lean
let A_L : ℝ := M * (p.a + p.b * M ^ p.α)
let A_F : ℝ := M * resolverGradEnvelopeCurrent p M
let B_F : ℝ := 0
```

not `0, 0, 0`.

## 4. If you insist on a nonzero `B_F`

A nonzero `B_F` is a different analytic route. It requires a theorem of the following form on the window:

```lean
-- Extra theorem required; not a consequence of the M-ball alone.
have hresolver_grad_from_u_grad :
  ∀ n s y,
    a ≤ s → s ≤ hi →
    |resolverGradReal p (U n s) y| ≤ K_R p M * G := by
  -- elliptic differentiated-resolver estimate goes here
  sorry
```

Then

```lean
|truncatedChemFluxLifted p (U n s) y|
  ≤ M * |resolverGradReal p (U n s) y|
  ≤ M * K_R(p,M) * G.
```

So the affine constants become

```lean
A_F = 0
B_F = M * K_R(p,M)
```

For a normalized one-dimensional resolver of `(u₊)^m`, a typical differentiated-source estimate would have the schematic form

```lean
K_R(p,M) = C_R * p.m * M ^ (p.m - 1)
B_F      = C_R * p.m * M ^ p.m
```

where `C_R` is the `L∞ → L∞` operator norm for the differentiated elliptic resolver. This formula needs the chain-rule estimate for `(u₊)^m`, so it is safe only with an additional hypothesis such as `1 ≤ p.m`, or with a positive lower bound away from zero. Without that, the M-only envelope above is the robust choice.

For the current repository resolver source `p.ν * u^p.γ`, the analogous schematic differentiated-source version would be

```lean
K_R(p,M) = C_R(p) * p.ν * p.γ * M ^ (p.γ - 1)
B_F      = C_R(p) * p.ν * p.γ * M ^ p.γ
```

again only after proving the required differentiated-resolver and power-chain estimates.

## 5. Small-window contraction

The gradient recurrence in `IntervalTruncatedGradientWindow` is

```lean
G_{n+1}
  ≤ Cg * M / sqrt (lo - a)
      + Cg * (2 * sqrt (hi - a)) * (A_L + |p.χ₀| * A_F)
      + Cg * (2 * sqrt (hi - a)) * |p.χ₀| * B_F * G
```

that is,

```lean
G_{n+1} ≤ truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G.
```

The contraction coefficient is exactly

```lean
def Bwin : ℝ :=
  truncWindowB B_F p.χ₀ a hi
-- = Cg * (2 * sqrt (hi - a)) * |p.χ₀| * B_F
```

If `B_F = 0`, then

```lean
Bwin = 0 < 1
```

so no additional smallness is needed from this part of the argument.

If using the nonzero-`B_F` route, choose the window so that

```lean
Cg * (2 * sqrt (hi - a)) * |p.χ₀| * B_F < 1.
```

Equivalently, when `Cg * |p.χ₀| * B_F > 0`, it is enough to impose

```lean
hi - a < (1 / (2 * Cg * |p.χ₀| * B_F)) ^ 2.
```

Then use the fixed point value, not the old raw `truncWindowA` value:

```lean
import ShenWork.Paper2.IntervalTruncatedGradientWindow

open ShenWork.Paper2.TruncatedGradientWindow

noncomputable section

namespace ShenWork.Paper2.TruncatedSourceEnvelope

-- Mathematical wiring pattern.
def correctedWindowG
    (M A_L A_F B_F χ a lo hi : ℝ) : ℝ :=
  truncWindowFixedG M A_L A_F B_F χ a lo hi

example
    {M A_L A_F B_F χ a lo hi : ℝ}
    (hB : truncWindowB B_F χ a hi < 1) :
    truncWindowAffine M A_L A_F B_F χ a lo hi
        (correctedWindowG M A_L A_F B_F χ a lo hi)
      ≤ correctedWindowG M A_L A_F B_F χ a lo hi := by
  unfold correctedWindowG truncWindowFixedG truncWindowAffine
  exact affine_fixed_closes
    (A := truncWindowA M A_L A_F χ a lo hi)
    (B := truncWindowB B_F χ a hi)
    hB

end ShenWork.Paper2.TruncatedSourceEnvelope
```

Therefore, for the concrete `a = t/4`, `lo = t/2`, `hi = t`, the zero-source value

```lean
G = truncWindowA M 0 0 p.χ₀ a lo hi
```

must be replaced by either

```lean
G = truncWindowFixedG M A_L A_F 0 p.χ₀ a lo hi
```

for the M-only flux envelope, or by

```lean
G = truncWindowFixedG M A_L A_F B_F p.χ₀ a lo hi
```

for the differentiated-resolver route. Keeping `A_L = A_F = B_F = 0` is mathematically false; keeping `G = truncWindowA ...` while making `B_F` nonzero loses the contraction closure.
