## Risks and circularities to audit

1. **Do not use chi-zero K1 producers for strict negative.**  Anything requiring `p.χ₀ = 0`, including `powerK1_quadruple_of_subtypeCont` as currently stated, is not a valid strict-negative producer.

2. **PPID vs `PositiveInitialDatum` mismatch.**  The PPID headline is routed through wrappers whose per-datum hypotheses still quantify `PositiveInitialDatum intervalDomain u₀`.  Do not replace this by weak/BForm `PositiveInitialDatum` unless there is an explicit PPID-to-that bridge in the exact route.

3. **Avoid BForm residuals for the PPID headline.**  BForm theorems may be useful analytically, but they do not directly close `Theorem_1_1` in the PPID path described here.

4. **Do not re-add the compact coefficient envelope.**  `IntervalHuRestartCoeffFiniteCoverProducer.lean` already proves the envelope from `Hu`.  A new theorem that asks for `henv` again is a regression.

5. **Do not hide K1 inside `DuhamelSourceTimeC1` for the same source.**  The old circularity was using a source-time-`C¹` package that already contains derivative data with the same shape as the conclusion.  The Hu restart chart may contain `DuhamelSourceTimeC1` for the u-side restart source; that is acceptable only if it is not the target power-source `ν*u^γ` K1 package being produced.
