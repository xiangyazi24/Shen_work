# Verdict

The proposed split-Duhamel argument is mathematically correct, with one essential qualification:

> “The source is `ℓ¹`” must mean that the source coefficients admit a **single summable envelope, uniform on the compact tail window** `[t/2,t]`.

A statement only at the single time `t/2`,

```lean
Summable (fun k => |s (t / 2) k|)
```

is not enough to control

```text
∫_{t/2}^t exp(-(t-r)λ_k) s_k(r) dr.
```

Even the pointwise-in-time assertion

```lean
∀ r ∈ [t/2,t], Summable (fun k => |s r k|)
```

does not by itself produce a summable sequence dominating all those slices. The clean Lean hypothesis is

```lean
∃ E : ℕ → ℝ,
  Summable E ∧
  ∀ r, t / 2 ≤ r → r ≤ t → ∀ k, |s r k| ≤ E k.
```

Under that window-envelope hypothesis, the exact majorant is

```text
λ_k |c_k(t)|
  ≤ M₀ λ_k exp(-νtλ_k)
    + C₀ (t/2) λ_k exp(-ν(t/2)λ_k)
    + E_k/ν,
```

where

```text
|c_k(0)| ≤ M₀,
|s_k(r)| ≤ C₀              for 0 ≤ r ≤ t/2,
|s_k(r)| ≤ E_k             for t/2 ≤ r ≤ t.
```

All three sequences on the right are summable. This gives

```lean
Summable (fun k => λ k * |c k t|).
```

For the current repository normalization, the heat factor is `exp (-(t-r) * λ_k)`, so `ν=1`; the tail bound is simply `E k`.


# 1. Exact analytic estimate

Let

```text
λ_k := (kπ)²,
τ := t/2,
```

and write

```text
c_k(t)
  = exp(-νλ_k t)c_k(0)
    + D_k(t),

D_k(t)
  := ∫₀ᵗ exp(-νλ_k(t-r)) s_k(r) dr.
```

Assume throughout

```text
ν>0,
t>0.
```

## 1.1 Duhamel split

Split at an arbitrary `0<τ<t` first. Then

```text
D_k(t)
 = ∫₀^τ exp(-νλ_k(t-r))s_k(r)dr
   + ∫_τ^t exp(-νλ_k(t-r))s_k(r)dr.
```

Equivalently, after factoring the semigroup over the first interval,

```text
D_k(t)
 = exp(-νλ_k(t-τ)) D_k(τ)
   + D_k^shift(t-τ),
```

where

```text
D_k^shift(h)
 := ∫₀ʰ exp(-νλ_k(h-σ)) s_k(τ+σ)dσ.
```

For `τ=t/2`, both elapsed times are `t/2`.

This is exactly the algebra packaged in the repository lemma

```lean
duhamelSpectralCoeff_general_split_on
```

for the normalized `ν=1` coefficient.

## 1.2 Head estimate

Suppose

```text
|s_k(r)| ≤ C₀
```

for every `k` and every `r∈[0,τ]`, with `C₀≥0`. Then

```text
|D_k(τ)| ≤ τ C₀.
```

Consequently,

```text
λ_k exp(-νλ_k(t-τ)) |D_k(τ)|
  ≤ τ C₀ λ_k exp(-νλ_k(t-τ)).
```

At `τ=t/2`, this becomes

```text
λ_k · |head_k|
  ≤ (t/2) C₀ λ_k exp(-νλ_k t/2).
```

Since

```text
λ_k = |kπ|²,
```

the right side is a scalar multiple of

```lean
fun k => |(k : ℝ) * Real.pi| ^ 2 *
  Real.exp (-(ν * (t / 2)) * unitIntervalCosineEigenvalue k),
```

which is summable by

```lean
frequency_pow_mul_exp_summable 2
  (mul_pos hν (half_pos ht)).
```

There is also a sharper exact head estimate. For `k≥1`,

```text
λ_k ∫₀^τ exp(-νλ_k(t-r))dr
 = exp(-νλ_k(t-τ)) · (1-exp(-νλ_kτ))/ν
 ≤ exp(-νλ_k(t-τ))/ν.
```

Thus

```text
λ_k |head_k|
 ≤ (C₀/ν) exp(-νλ_k(t-τ)).
```

The cruder `τC₀ λ_k exp(-νλ_k(t-τ))` estimate is usually easier to formalize because it needs only the elementary integral-length bound and `frequency_pow_mul_exp_summable 2`.

## 1.3 Tail eigenvalue gain

Suppose there is a sequence `E : ℕ → ℝ` such that

```text
Summable E,
|s_k(r)| ≤ E_k                  for τ ≤ r ≤ t.
```

For `k≥1`, so that `λ_k>0`,

```text
λ_k |tail_k|
 ≤ E_k λ_k ∫_τ^t exp(-νλ_k(t-r))dr
 = E_k · (1-exp(-νλ_k(t-τ)))/ν
 ≤ E_k/ν.
```

For `k=0`, `λ_0=0`, so the weighted tail term is exactly zero. Therefore, for every `k`,

```text
λ_k |tail_k| ≤ E_k/ν.
```

Since scalar multiplication preserves summability,

```lean
henv.mul_left (1 / ν)
```

proves summability of the tail majorant.

In the repository's normalized `ν=1` model, the exact landed estimate is

```lean
eigenvalue_mul_abs_duhamelSpectralCoeff_le_envelope
```

with conclusion

```lean
unitIntervalCosineEigenvalue k *
    |duhamelSpectralCoeff a h k|
  ≤ src.envelope k.
```

That theorem already handles `k=0` separately and proves the exact identity

```text
∫₀ʰ λ_k exp(-(h-r)λ_k)dr
  = 1-exp(-hλ_k).
```

## 1.4 Homogeneous term

Suppose

```text
|c_k(0)| ≤ M₀.
```

Then

```text
λ_k |exp(-νλ_k t)c_k(0)|
 ≤ M₀ λ_k exp(-νtλ_k).
```

This sequence is summable by either of the repository lemmas

```lean
frequency_pow_mul_exp_summable 2 (mul_pos hν ht)
```

or, more directly in the normalized model,

```lean
unitIntervalCosineEigenvalue_mul_exp_summable ht.
```

The current truncated file already proves the needed boundedness of the initial coefficients:

```lean
truncatedPicardInitialCoeff_abs_le_two_M DT :
  ∀ k, |truncatedPicardInitialCoeff u₀ k| ≤ 2 * DT.M.
```

So one can take

```text
M₀ = 2·DT.M.
```

## 1.5 Final majorant

By the split identity and `abs_add_le`,

```text
λ_k |c_k(t)|
 ≤ M₀ λ_k exp(-νtλ_k)
   + τ C₀ λ_k exp(-ν(t-τ)λ_k)
   + E_k/ν.
```

At `τ=t/2`, define

```text
B_k :=
  M₀ λ_k exp(-νtλ_k)
  + (t/2) C₀ λ_k exp(-ν(t/2)λ_k)
  + E_k/ν.
```

Then

```text
0 ≤ λ_k|c_k(t)|,
λ_k|c_k(t)| ≤ B_k,
Summable B.
```

The conclusion is one application of

```lean
Summable.of_nonneg_of_le
```

together with two `.add` operations for the three majorants.

# 2. What “source is ℓ¹” must mean

There are three different assertions that should not be conflated.

## 2.1 Single-time ℓ¹

```lean
Summable (fun k => |s t₀ k|)
```

controls only one source slice. It gives no estimate for a Duhamel integral over a time interval.

This is exactly the current mismatch in Bootstrap #5: the proof has

```lean
have _hl1_src :=
  truncatedBFormSourceCoeff_summable_positive_time
    DT (show 0 < t / 2 by linarith) ...
```

but this theorem returns summability only at `s=t/2`.

## 2.2 Pointwise-in-time ℓ¹

```lean
∀ r ∈ Set.Icc τ t, Summable (fun k => |s r k|)
```

is still not enough for the clean termwise domination argument. In general one cannot infer

```text
∑ₖ sup_{r∈[τ,t]} |s_k(r)| < ∞.
```

Continuity of each coordinate `r ↦ s_k(r)` does not repair this inference.

## 2.3 Uniform compact-window ℓ¹ envelope

The useful condition is

```lean
∃ E : ℕ → ℝ,
  Summable E ∧
  ∀ r, τ ≤ r → r ≤ t → ∀ k, |s r k| ≤ E k.
```

This is exactly what the shifted tail source needs in a `DuhamelSourceL1ContOn` package.

A weaker but also sufficient analytic alternative is an integrated ℓ¹ condition such as

```text
∫_τ^t ∑ₖ |s_k(r)| dr < ∞.
```

Then Tonelli gives direct summability of the tails. That route is less convenient in Lean because it requires a sum-integral interchange and a jointly measurable nonnegative family. The uniform envelope is the cleanest project interface.

# 8. Final assembly skeleton

After obtaining `hsplit`, `hhom_sum`, `hhead_sum`, and `htail_sum`, the final proof is just domination.

A faithful normalized skeleton is:

```lean
let U := truncatedConjugatePicardLimit p u₀ DT.T
let a : ℝ → ℕ → ℝ :=
  fun s k => truncatedBFormSourceCoeff p U s k
let a₀ : ℕ → ℝ := truncatedPicardInitialCoeff u₀
let τ : ℝ := t / 2
let ashift : ℝ → ℕ → ℝ := fun σ k => a (τ + σ) k

have hτpos : 0 < τ := by dsimp [τ]; linarith
have hτt : τ < t := by dsimp [τ]; linarith

    duhamelSpectralCoeff a t k
      = Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k) *
          duhamelSpectralCoeff a τ k
        + duhamelSpectralCoeff ashift (t - τ) k := by
  intro k
  exact duhamelSpectralCoeff_general_split_on
    hsource_cont τ.le hτt.le htT k

have hmajorant : Summable (fun k =>
    (2 * DT.M) *
      (unitIntervalCosineEigenvalue k *
        Real.exp (-t * unitIntervalCosineEigenvalue k))
    + τ * C₀ *
      (unitIntervalCosineEigenvalue k *
        Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k))
    + E k) :=
  (((unitIntervalCosineEigenvalue_mul_exp_summable ht).mul_left
      (2 * DT.M)).add
    ((unitIntervalCosineEigenvalue_mul_exp_summable
      (by linarith : 0 < t - τ)).mul_left (τ * C₀))).add hEsum

refine Summable.of_nonneg_of_le
  (fun k => mul_nonneg
    (by unfold unitIntervalCosineEigenvalue; positivity)
    (abs_nonneg _))
  (fun k => ?_)
  hmajorant

-- unfold the coefficient into homogeneous + Duhamel
simp only [truncatedPicardCoeff, localRestartCoeff]
rw [hsplit k]

have hlam : 0 ≤ unitIntervalCosineEigenvalue k := by
  unfold unitIntervalCosineEigenvalue
  positivity

-- `λ |hom + head + tail| ≤ λ|hom| + λ|head| + λ|tail|`
calc
  unitIntervalCosineEigenvalue k *
      |Real.exp (-t * unitIntervalCosineEigenvalue k) * a₀ k
        + (Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k) *
            duhamelSpectralCoeff a τ k
          + duhamelSpectralCoeff ashift (t - τ) k)|
    ≤ unitIntervalCosineEigenvalue k *
        |Real.exp (-t * unitIntervalCosineEigenvalue k) * a₀ k|
      + unitIntervalCosineEigenvalue k *
        |Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k) *
          duhamelSpectralCoeff a τ k|
      + unitIntervalCosineEigenvalue k *
        |duhamelSpectralCoeff ashift (t - τ) k| := by
          nlinarith [abs_add_le
            (Real.exp (-t * unitIntervalCosineEigenvalue k) * a₀ k)
            (Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k) *
                duhamelSpectralCoeff a τ k
              + duhamelSpectralCoeff ashift (t - τ) k),
            abs_add_le
              (Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k) *
                duhamelSpectralCoeff a τ k)
              (duhamelSpectralCoeff ashift (t - τ) k)]
  _ ≤ (2 * DT.M) *
        (unitIntervalCosineEigenvalue k *
          Real.exp (-t * unitIntervalCosineEigenvalue k))
      + τ * C₀ *
        (unitIntervalCosineEigenvalue k *
          Real.exp (-(t - τ) * unitIntervalCosineEigenvalue k))
      + E k := by
        exact add_le_add
          (add_le_add (hhom_le k) (hhead_le k))
          (htail_le k)
```
