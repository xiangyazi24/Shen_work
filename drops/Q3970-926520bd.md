# Q3970: continuous source ⇒ classical second derivative for the 1D Neumann resolver

## Verdict

Do **not** try to prove

```lean
HasDerivAt (fun z => resolverGradReal p u z)
  (resolverLapPhysical p u y) x
```

by differentiating the gradient sine series termwise under only continuous source data. That route is mathematically false at the summability level you identified: continuity gives source coefficients in `ℓ²`, hence resolved coefficients roughly `c_k ~ a_k / k²`, but it does not give

```text
∑ k, |c_k| * (kπ)^2 < ∞.
```

So `cosineCoeffSeries_contDiff_two`, `sineSeries_hasDerivAt_of_grad2Summable`, or any direct Weierstrass-M-test proof of the second derivative is the wrong interface unless you assume extra source decay such as `SourceCoeffQuadraticDecay`.

The feasible Lean route is:

1. Prove one **integrated weak ODE identity** for the resolver gradient,
   ```text
   resolverGradReal p u b - resolverGradReal p u a
     = ∫ t in a..b, resolverLapPhysicalReal p u t,
   ```
   for `a,b ∈ (0,1)` or at least locally around the target point `x`.
2. Prove `resolverLapPhysicalReal p u` is continuous at the interior point `x`.
3. Apply Mathlib FTC-1, specifically the interval-integral theorem for
   `z ↦ ∫ t in x..z, q t`.

After the integrated identity is available, the final `HasDerivAt` proof is a short Mathlib-FTC proof. The analytic work should be factored into the integrated identity, not into a fake `C²` termwise-differentiation theorem.

## Answers to the three route questions

### 1. Mathlib Sobolev route

Mathlib now has Sobolev/Bessel-potential infrastructure and a Gagliardo-Nirenberg-Sobolev inequality file, but it is not a convenient one-line theorem of the form

```lean
H2_on_Icc_0_1_to_C1 : H^2(0,1) → C^1([0,1])
```

for your concrete cosine-series resolver representative. To use route 3 formally, you would still need to build a large bridge:

```text
coefficient ℓ² control of (kπ)^2 c_k
  ⇒ Sobolev membership of the represented function
  ⇒ choose the continuous representative on [0,1]
  ⇒ identify that representative with intervalNeumannResolverR
  ⇒ identify its weak derivative with resolverGradReal
  ⇒ upgrade weak R'' = μR - ρ(u) to HasDerivAt of R'.
```

That is a good long-term library development, but it is not the shortest way to close this target.

### 2. Mathlib ODE regularity route

Mathlib has ODE existence/uniqueness files and very strong interval-integral FTC lemmas. It does **not** currently give the black-box theorem you would want:

```lean
weak_second_order_ode_with_continuous_forcing_is_classical
```

So the useful Mathlib part is not an ODE regularity theorem; it is the FTC theorem after you have converted the weak/spectral equation into an integral identity for `resolverGradReal`.

### 3. Explicit cosine route

The explicit series route is still the most feasible, but not by differentiating

```text
R'(x) = ∑ c_k * (−kπ sin(kπx))
```

termwise. Instead, use the first derivative series only at the **endpoints** `a` and `b`, where the gradient series is already absolutely summable, and use the elliptic coefficient identity in an integrated form.

The identity to prove is:

```text
R'(b) - R'(a) = ∫_a^b (μR(t) - source(t)) dt.
```

For each mode,

```text
∫_a^b −(kπ)^2 cos(kπt) dt
  = −kπ * (sin(kπb) - sin(kπa)).
```

Thus the dangerous factor `(kπ)^2` is always paired with the interval integral of the cosine mode. After integration it becomes a first-gradient endpoint expression, controlled by the already available gradient absolute summability. This is exactly the reason the integrated identity can be true under continuous source data even though the pointwise second-derivative series need not converge absolutely.

## The bridge theorem to add

Add a real-coordinate version of the physical Laplacian so the FTC statement does not have dependent interval points in the integrand.

```lean
import ShenWork.Paper2.IntervalResolverWeakLapBound
import Mathlib.MeasureTheory.Integral.IntervalIntegral.FundThmCalculus

open MeasureTheory intervalIntegral
open ShenWork.IntervalDomain
open ShenWork.PDE
open ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

/-- Physical Laplacian as a plain real-coordinate function.
Inside `[0,1]` it is `μR - source`; outside it is irrelevant. -/
def resolverLapPhysicalReal (p : CM2Params)
    (u : intervalDomainPoint → ℝ) (x : ℝ) : ℝ :=
  if hx : x ∈ Set.Icc (0 : ℝ) 1 then
    resolverLapPhysical p u ⟨x, hx⟩
  else
    0

/-- Continuity of the physical Laplacian at an interior point.
This should use:
* continuity of `intervalDomainLift u` on `[0,1]`,
* continuity of `positivePart` and `rpow` with `p.hγ : 0 < γ`,
* continuity of the resolver value series `intervalNeumannResolverR p u`, obtained from
  the existing absolute/uniform cosine-series summability route. -/
theorem resolverLapPhysicalReal_continuousAt_of_continuousOn
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    ContinuousAt (resolverLapPhysicalReal p u) x := by
  -- This is a small continuity lemma once a value-continuity lemma for
  -- `intervalNeumannResolverR p u` is available.
  -- Do not prove differentiability here.
  -- Suggested supporting lemma:
  --   resolverValueReal_continuousAt_of_continuousOn
  --     : ContinuousAt (fun z => intervalNeumannResolverR p u ⟨z, hz⟩) x
  -- for `x ∈ Ioo 0 1`, implemented using the existing absolute cosine-series
  -- summability and `continuous_cos`.
  sorry

/-- Integrated weak ODE identity. This is the main analytic bridge.
It is the correct replacement for illegal termwise differentiation of `R'`. -/
theorem resolverGradReal_sub_eq_integral_lapPhysicalReal
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {a b : ℝ} (ha : a ∈ Set.Ioo (0 : ℝ) 1) (hb : b ∈ Set.Ioo (0 : ℝ) 1) :
    resolverGradReal p u b - resolverGradReal p u a
      = ∫ t in a..b, resolverLapPhysicalReal p u t := by
  -- Prove this from the spectral weak equation, not from pointwise `R''` series.
  -- Sketch:
  --   1. Pair the coefficient equation `(μ + λ_k)c_k = a_k` with the cosine
  --      coefficients of `Set.indicator (Set.uIcc a b) 1`.
  --   2. Use the L²/cosine Parseval bridge for the source/R terms.
  --   3. For the λ-part, use
  --        λ_k * ∫_a^b cos(kπt) dt
  --          = kπ * (sin(kπb) - sin(kπa))
  --      with the correct sign.
  --   4. The resulting endpoint sine sums are exactly the two gradient series,
  --      and their convergence is justified by `resolver_sineSeries_summable_of_sourceL2`.
  --
  -- This proof uses only integrated mode identities and endpoint gradient
  -- summability. It never needs `∑ |c_k| (kπ)^2`.
  sorry

/-- Local primitive form, convenient for FTC. -/
theorem resolverGradReal_eventually_eq_primitive
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    (fun z : ℝ => resolverGradReal p u z)
      =ᶠ[𝓝 x]
    (fun z : ℝ => resolverGradReal p u x
      + ∫ t in x..z, resolverLapPhysicalReal p u t) := by
  have hIoo_mem : Set.Ioo (0 : ℝ) 1 ∈ 𝓝 x :=
    IsOpen.mem_nhds isOpen_Ioo hx
  filter_upwards [hIoo_mem] with z hz
  have h := resolverGradReal_sub_eq_integral_lapPhysicalReal
    p hUcont hx hz
  linarith

end ShenWork.IntervalResolverWeakBounds
```

The two `sorry`s above are not meant as new assumptions; they mark the two reusable lemmas to prove once. The second one is the real mathematical payload. If the repo already has a full cosine Parseval/inversion equality for L² pairings, the spectral proof is the most direct. If not, prove the same integrated identity using the closed-form Neumann Green kernel.

## The final target after the bridge is available

Once `resolverGradReal_eventually_eq_primitive` and `resolverLapPhysicalReal_continuousAt_of_continuousOn` are proved, the desired theorem is short.

```lean
import ShenWork.Paper2.IntervalResolverWeakLapBound
import Mathlib.MeasureTheory.Integral.IntervalIntegral.FundThmCalculus

open MeasureTheory intervalIntegral
open ShenWork.IntervalDomain
open ShenWork.PDE
open ShenWork.Paper2
open ShenWork.IntervalResolverWeakBounds
open scoped Topology BigOperators

noncomputable section

namespace ShenWork.IntervalResolverWeakBounds

/-- Continuous-source weak resolver regularity, expressed only as the derivative
needed downstream: the derivative of the gradient is the physical Laplacian. -/
theorem resolverGradReal_hasDerivAt_physicalLap_of_continuousOn_via_FTC
    (p : CM2Params) {u : intervalDomainPoint → ℝ}
    (hUcont : ContinuousOn (intervalDomainLift u) (Set.Icc (0 : ℝ) 1))
    {x : ℝ} (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    HasDerivAt (fun z : ℝ => resolverGradReal p u z)
      (resolverLapPhysical p u ⟨x, Set.Ioo_subset_Icc_self hx⟩) x := by
  let q : ℝ → ℝ := resolverLapPhysicalReal p u

  have hq_cont : ContinuousAt q x := by
    simpa [q] using
      resolverLapPhysicalReal_continuousAt_of_continuousOn p hUcont hx

  -- For the actual Lean proof, discharge this from local continuity on a small
  -- compact interval around `x`, or use the degenerate interval form if accepted
  -- by simplification in your Mathlib version.
  have hq_int : IntervalIntegrable q volume x x := by
    -- continuous-at plus a local continuous-on lemma is enough; this is not the
    -- analytic issue.
    simpa using (intervalIntegrable_const (a := x) (b := x) (c := (0 : ℝ)))

  have hq_meas : StronglyMeasurableAtFilter q (𝓝 x) :=
    hq_cont.stronglyMeasurableAtFilter

  have hFTC :
      HasDerivAt (fun z : ℝ => ∫ t in x..z, q t) (q x) x :=
    intervalIntegral.integral_hasDerivAt_right
      (a := x) (b := x) hq_int hq_meas hq_cont

  have hprim :
      HasDerivAt
        (fun z : ℝ => resolverGradReal p u x + ∫ t in x..z, q t)
        (q x) x := by
    simpa using hFTC.const_add (resolverGradReal p u x)

  have hev :
      (fun z : ℝ => resolverGradReal p u z)
        =ᶠ[𝓝 x]
      (fun z : ℝ => resolverGradReal p u x + ∫ t in x..z, q t) := by
    simpa [q] using resolverGradReal_eventually_eq_primitive p hUcont hx

  have hq_x : q x = resolverLapPhysical p u ⟨x, Set.Ioo_subset_Icc_self hx⟩ := by
    simp [q, resolverLapPhysicalReal, Set.Ioo_subset_Icc_self hx]

  simpa [hq_x] using hprim.congr_of_eventuallyEq hev.symm

end ShenWork.IntervalResolverWeakBounds
```

Depending on the exact Mathlib version, the last line may require one of the nearby congruence lemmas for `HasDerivAt`, for example rewriting the function by eventual equality before applying `hprim`. The conceptual proof is stable: local equality with an antiderivative plus FTC.

## Why this is the right cut point

The target theorem is not really a theorem about Fourier series being twice differentiable. It is a theorem about a weak solution of a one-dimensional elliptic equation whose forcing is continuous. In Lean, the clean cut is therefore:

```lean
-- mathematical bridge
resolverGradReal_sub_eq_integral_lapPhysicalReal

-- calculus bridge supplied by Mathlib
intervalIntegral.integral_hasDerivAt_right
```

Trying to make the second derivative series converge uniformly asks for a false hypothesis. Trying to import a Sobolev embedding theorem asks Mathlib for infrastructure it does not currently package in the needed interval/cosine-representative form. The integrated weak ODE identity is both mathematically sharp and closest to the artifacts already present in this repo: coefficient elliptic identity, `ℓ²` source coefficients, absolute convergence of the gradient sine series, and interval-integral FTC.

## If choosing Green kernel instead

If the L² cosine Parseval equality for pairings is not yet available, the Green-kernel route may be less abstract. Define

```text
α = sqrt μ,
G(y,z) = cosh(α * min y z) * cosh(α * (1 - max y z)) / (α * sinh α).
```

Then prove

```text
R(y) = ∫ z in 0..1, G(y,z) * source(z) dz,
```

split the integral at `z = y`, and use `intervalIntegral.integral_hasDerivAt_right/left` plus the jump identity for `∂_y G` at `z = y`. This gives directly

```text
HasDerivAt R' (μ * R y - source y) y.
```

This is analytically robust, but the initial kernel algebra and split-integral proof will likely be longer than the spectral integrated-identity bridge if your cosine Parseval tools are already close.
