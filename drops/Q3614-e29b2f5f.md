## 4. Circular/vacuous routes to avoid

Avoid these routes:

1. **Do not reintroduce boundary persistence as an assumption.**  Any new headline wrapper that again takes `BoundaryMinPersistenceWindowBound p`, `BoundaryMinPersistenceBound p`, or endpoint chem-div limits is strictly weaker than Task152.  Use `BFormPositiveDatumLocal.boundaryMinPersistenceWindowBound_chiNonpos` from `IntervalDomainBoundaryChemDivLimit.lean` instead.

2. **Do not pretend `PositiveInitialDatum → PaperPositiveInitialDatum`.**  `BFormDirectClassical.BFormPaperLocalFrontier` and `BFormDirectClassical.paperPositive_localExistence_of_BFormDirect` in `ShenWork/Paper2/IntervalBFormDirectClassical.lean` are for `PaperPositiveInitialDatum`.  The actual final B-form theorem `BFormDirectClassical.paper2_theorem_1_1_general_chi_bform` requires local existence for the weaker `PositiveInitialDatum` interface.  The negative-part file `IntervalBFormPositiveDatumNegPartFrontier.lean` was introduced precisely to avoid smuggling this false conversion.

3. **Do not prove a B-form local frontier by appealing to the final headline.**  `PositiveDatumBFormLocal.positiveDatum_localExistence_of_BForm` and `BFormPositiveDatumNegPart.positiveDatum_localExistence_of_BForm` are consumers of the B-form packages; they are not producers of those packages.  A proof of `PositiveDatumBFormLocalHyp p` or `BFormPositiveLocalFrontier p` must construct the per-datum B-form data directly.

4. **Do not count `paper2_theorem_1_1_general_chi_bform_from_quant` or `paper2_theorem_1_1_general_chi_bform_negpart_from_quant` as producers.**  Both are valid consumers in `IntervalBFormPositiveDatumQuantWiring.lean`, but each takes the quantitative local-existence factory `hQuant` as an explicit input.  Using either as a substitute for producing the factory would be circular/vacuous.

5. **Do not spend effort on the Picard-restart version if `hPLF` is available.**  The exact conversion `ConeQuantBridge.picardRestartFrontier_of_picardLimitFrontier` already exists in `IntervalDomainConeQuantBridge.lean`, and Task152 already has Picard-limit wrappers.  Use the Picard-limit wrappers as the shorter non-circular route.
