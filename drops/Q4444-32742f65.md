# Paper 3 steps 4–5: compactness + persistence execution prep

## Verdict

P3.5 and P3.2 are **not** finished by Paper 2’s finite-window existence/smoothing alone. Paper 2 gives the local-in-time classical/mild machinery and boundedness inputs; Paper 3 needs **tail-uniform dynamical estimates**: constants that are independent of the restart time once `t ≥ T*`.

The shared engine should be organized as:

```lean
TailDynamicalCore =
  absorbing_set        -- eventual L∞ / source bounds, uniform for t ≥ T*
  rolling_smoothing    -- uniform positive-length window Schauder/heat smoothing
  tail_compactness     -- Arzelà/Aubin-Lions from the rolling bounds
  mass_or_average_persistence
  harnack_or_lower_propagation
```

P3.5 consumes primarily `absorbing_set + rolling_smoothing + compactness`. P3.2 consumes those plus the lower-bound part: `mass_or_average_persistence + harnack_or_lower_propagation + elliptic lower transfer`. So they share the same tail engine, but the **true persistence-specific hard lemma is the Harnack/lower-propagation core**.

## 1. P3.5 compactness: exact eventual / uniform-in-tail estimate

The useful estimate is not merely:

```lean
∀ 0 < c, ∀ T' < T, ∃ C, ∀ t ∈ Icc c T', regularity_bound u t ≤ C
```

That is a finite-window Paper 2 style estimate. Paper 3 compactness needs the rolling-window version:

```lean
∃ Tstar C θ τ, 0 < τ ∧ 0 < θ ∧
  ∀ u v, PositiveGlobalBoundedSolution intervalDomain p u v →
    ∀ t, Tstar ≤ t →
      ParabolicHolderBoundOnWindow θ C
        (fun s x => u (t + s) x)
        (fun s x => v (t + s) x)
        (Set.Icc (0 : ℝ) τ ×ˢ Set.Icc (0 : ℝ) 1)
```

or, if you want to avoid naming Hölder spaces at first, an Arzelà-ready formulation:

```lean

    (locallyConverges : (ℕ → ℝ → intervalDomain.Point → ℝ) →
      (ℝ → intervalDomain.Point → ℝ) → Prop) : Prop where
  tail_bound :
    ∃ Tstar Mtail, ∀ u v,
      PositiveGlobalBoundedSolution intervalDomain p u v →
        ∀ t, Tstar ≤ t →
          intervalDomain.supNorm (u t) ≤ Mtail ∧
          intervalDomain.supNorm (v t) ≤ Mtail
  rolling_equicontinuity :
    ∀ R > 0, ∃ Tstar Cmod,

    ∀ u v, PositiveGlobalBoundedSolution intervalDomain p u v →
```

### Where the rolling estimate comes from

Yes: the intended route is

```text
logistic damping / absorbing set
  ⇒ ∃T*, ∀t≥T*, ||u(t)||∞ ≤ Mtail
  ⇒ elliptic resolver bounds for v(t), v_x(t), ... uniformly for t≥T*
  ⇒ source bounds on every rolling window [t, t+τ]
  ⇒ heat/Schauder smoothing on [t+τ/2, t+τ]
  ⇒ uniform C^θ or C^{2+θ,1+θ/2} bounds independent of t
  ⇒ time-translate compactness.
```

Paper 2’s finite-window smoothing can be reused only after adding a **restart/rolling-window wrapper**: restart at `t`, use the absorbing bound as the new initial-size datum, and run a fixed-length smoothing step. The constants must depend on `Mtail`, `p`, and the chosen window length `τ`, not on the absolute restart time.

## 2. P3.2 persistence: mechanism and target theorem

The persistence target is stronger than positive mass. It is eventually pointwise:

```lean
∃ δ > 0, ∀ᶠ t in atTop,
  ∀ x : intervalDomain.Point, δ ≤ u t x
```

and, for the chemotactic signal, the elliptic transfer:

```lean
∀ᶠ t in atTop,
  ∀ x : intervalDomain.Point,
    (p.ν / p.μ) * δ ^ p.γ ≤ v t x
```

```lean
structure IntervalDomainPersistenceCore (p : CM2Params) : Prop where
  absorbing :
    ∀ u v, PositiveGlobalBoundedSolution intervalDomain p u v →
      ∃ Tstar Mtail, ∀ t, Tstar ≤ t →
        intervalDomain.supNorm (u t) ≤ Mtail
  average_part1 :

      ∀ u v, PositiveGlobalBoundedSolution intervalDomain p u v →
        ∃ m0 > 0, ∀ᶠ t in atTop,

    0 < p.a → 0 < p.b → 0 < p.χ₀ → 1 < p.m → 1 ≤ p.β →

        ∀ᶠ t in atTop,
          theorem21Part3LowerU p ≤ intervalDomain.infValue (u t)
  harnack_mass_to_floor :
    ∀ {m0 Mtail : ℝ}, 0 < m0 → 0 ≤ Mtail →
      ∃ τ δ, 0 < τ ∧ 0 < δ ∧
        ∀ u v t,
          PositiveGlobalBoundedSolution intervalDomain p u v →
          intervalDomain.integral (u t) ≥ m0 →
          (∀ s ∈ Set.Icc t (t + τ), intervalDomain.supNorm (u s) ≤ Mtail) →
          ∀ x : intervalDomain.Point, δ ≤ u (t + τ) x

    ∀ {δ : ℝ}, 0 < δ →
```

### Which persistence mechanism is right?

For Lean, do **not** rely on “positive mass alone implies uniform pointwise lower bound.” That is false without a smoothing/propagation estimate: mass can concentrate. The correct mechanism is:

```text
average persistence / scalar ODE lower bound
  + eventual L∞ absorbing set
  + parabolic Harnack or positivity-spreading on a fixed rolling window
  ⇒ eventual pointwise lower bound for u
  + elliptic maximum principle/resolvent lower bound
  ⇒ eventual pointwise lower bound for v.
```

For the interval, the Harnack input can be stated in an elementary kernel/comparison style instead of full Moser-Harnack if the equation is in divergence form with bounded coefficients:

```lean
structure IntervalDomainLowerPropagationCore (p : CM2Params) : Prop where
  bounded_coefficients_from_absorbing : ...
  parabolic_harnack_or_kernel_lower :
    ∀ Mtail m0, 0 < m0 →
      ∃ τ δ, 0 < τ ∧ 0 < δ ∧
        ∀ u v t,
          PositiveGlobalBoundedSolution intervalDomain p u v →
          tailBoundOnWindow Mtail u v t (t+τ) →
          m0 ≤ intervalDomain.integral (u t) →
          ∀ x, δ ≤ u (t+τ) x
```

This is the same conceptual object as the Jensen/barrier strict-positivity core, but with constants uniform over all large restart times. The uniformity comes from the absorbing bounds, not from the initial datum.

## 3. Do P3.5 and P3.2 reduce to the same absorbing-set + Harnack core?

Mostly yes, but with a nuance:

* **P3.5 compactness** needs `absorbing_set + rolling_smoothing + Arzelà/Aubin-Lions`. It does not need a positive lower floor unless the chosen coefficient estimates require it. For compactness of `u` and `v`, eventual upper bounds and parabolic/elliptic regularity are the main ingredients.
* **P3.2 persistence** needs the same absorbing/rolling regularity, plus a genuine lower-bound engine: Harnack/positivity-spreading or an equivalent subsolution comparison.

So the shared core should be split as:

```lean
structure IntervalDomainTailAbsorbingSmoothingCore (p : CM2Params) : Prop where
  absorbing_Linf : ...
  rolling_parabolic_regularization : ...
  rolling_elliptic_regularization : ...
  time_translate_compactness : ...

  mass_or_average_persistence : ...
```

## 5. Recommended execution order

1. **Define the concrete tail convergence topology** for `K.locallyConverges` on `intervalDomain`, probably local uniform convergence on compact time windows and `[0,1]` in space. This gives a target for `TimeTranslateCompactnessRaw`.

2. **Prove the absorbing-set theorem**:

   ```lean
   theorem intervalDomain_eventual_Linf_absorbing
       (p : CM2Params) :
       ∃ Mtail, ∀ u v,
         PositiveGlobalBoundedSolution intervalDomain p u v →
           ∀ᶠ t in atTop, intervalDomain.supNorm (u t) ≤ Mtail
   ```

   For the negative-sensitivity branch this should reuse Paper 2’s energy/boundedness as much as possible; for the positive/paper3 branches it is part of the Lp/Moser mountain.

3. **Prove rolling-window smoothing from absorbing**:

   ```lean
   theorem intervalDomain_rolling_parabolic_regularization_of_absorbing
       (p : CM2Params) :
       IntervalDomainTailAbsorbingSmoothingCore p
   ```

   This is Paper 2 finite-window smoothing plus a restart lemma and constants independent of the restart time.

4. **Derive `TimeTranslateCompactnessRaw`** by Arzelà/diagonal extraction:

   ```lean
   theorem intervalDomain_timeTranslateCompactnessRaw_of_tailCore
       (H : IntervalDomainTailAbsorbingSmoothingCore p) :
       TimeTranslateCompactnessRaw intervalDomain p K.locallyConverges
   ```

5. **Prove lower propagation / Harnack**:

   ```lean
   theorem intervalDomain_mass_to_eventual_floor_of_tailCore
       (Hsmooth : IntervalDomainTailAbsorbingSmoothingCore p)
       (Havg : IntervalDomainAveragePersistenceCore p) :
       IntervalDomainTailLowerPropagationCore p
   ```

6. **Assemble `IntervalDomainLogisticPersistenceInputs`** and then Theorem 2.1:

   ```lean
       (Hsmooth : IntervalDomainTailAbsorbingSmoothingCore p)

       IntervalDomainLogisticPersistenceInputs p
   ```

   then use `.to_persistence`, `.to_persistenceFrontiers`, and the existing stability-chain wrappers.

## Bottom line

The right shared core is **not** “Paper 2 smoothing, copied to Paper 3.” It is:

```text
eventual absorbing bounds
+ restart/rolling-window smoothing with constants independent of restart time
+ compactness extraction
+ Harnack/positivity-spreading for lower bounds
```

P3.5 consumes the first three. P3.2 consumes all four, plus scalar/average persistence and elliptic lower transfer. The single most important new analytic brick is the **uniform-in-tail lower-propagation/Harnack theorem**; the single most important compactness brick is the **rolling-window Schauder/Arzelà theorem**. Together they are the Paper 3 dynamical engine.
