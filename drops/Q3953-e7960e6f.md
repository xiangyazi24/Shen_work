# Q3953: small-window Lipschitz contraction for truncated 1D Keller--Segel iterates

## Executive recommendation

Use a lightweight `Prop` predicate for the actual per-iterate gradient bound, and a separate `Prop` structure for the window wiring and scalar gate. Do not make the one-field gradient bound itself a heavy structure unless you already know you will add fields to it.

The clean split is:

1. Define explicit scalar constants:
   * `truncWindowSourceCL G = A_L + |χ₀| * (A_F + B_F * G)`;
   * `truncWindowA = Cg*M/sqrt(lo-a) + 2*Cg*sqrt(hi-a)*(A_L + |χ₀|*A_F)`;
   * `truncWindowB = 2*Cg*sqrt(hi-a)*|χ₀|*B_F`;
   * `truncWindowAffine G = truncWindowA + truncWindowB * G`.
2. Carry the scalar self-map condition as a field:
   `truncWindowAffine G ≤ G`.
   Then the step proof is just `raw_bound.trans hclosed`.
3. Prove a separate algebra lemma showing that `G = A/(1-B)` works when `B < 1`. The main Picard theorem should consume `hclosed`, not unfold the division every time.
4. Prove the iterate theorem by induction over `n`, not by invoking a fixed-point theorem.

This mirrors the existing `IntervalPicardIterateUniform.lean` style: explicit profiles/constant definitions, a smallness/gate condition, a `Prop` wiring bundle, then `_zero`, `_succ`, `_all` induction theorems.

## Critical window-design warning

With the restart fixed as

```text
a = lo/2,
```

the contraction coefficient is

```text
B_win = 2 * Cg * sqrt(hi - lo/2) * |χ₀| * B_F.
```

For fixed `lo`, shrinking the displayed window length `hi - lo` does **not** force `B_win` to zero, because as `hi ↓ lo` one still has

```text
hi - lo/2 ↓ lo/2.
```

So the statement “short enough windows imply `B_win < 1`” is true only if the restart distance `hi-a` is also short. For a robust formalization, parameterize the window by a general restart time

```text
a < lo ≤ hi,
```

and specialize to `a = lo/2` only when that is the intended positive-time bootstrap window. The real smallness parameter is `hi - a`, not `hi - lo`.

This also matters for the source bound: the Duhamel integral runs over `[a,t]`, so the flux-derivative estimate needs a gradient bound for the previous iterate on the **source interval** `[a,hi]`, not merely on the displayed target interval `[lo,hi]`. The Lean carrier should make this explicit.

## 1. What structure should carry the bound?

Use this pattern:

* an `abbrev`/`def` for the actual gradient assertion;
* a `structure ... : Prop` for the window assumptions, source-wiring hypotheses, and scalar gate.

A one-field structure such as

```lean
structure TruncatedIterateGradientWindow ... : Prop where
  hGrad : ...
```

is usable, but it buys little. Prefer the lighter predicate:

```lean
abbrev IterGradOnWindow ... : Prop := ...
```

and reserve structures for packages with several fields, for example `TruncatedGradientWindowWiring`.

The important reason is that the step is not only “`G_n ≤ G` implies `G_{n+1} ≤ G`”. Because the source starts at `a`, the step is really:

```text
previous gradient bound on [a,hi]
  -> source bound on [a,hi]
  -> raw next gradient bound on [lo,hi]
  -> closed next gradient bound on [lo,hi].
```

So the carrier should separate:

* left/source-interval supply;
* source estimate from gradient bound;
* kernel/Duhamel raw estimate;
* scalar closure.

## 2. How to handle finite covers?

First prove the per-window theorem. Do not start with a compactness/finite-cover formalization.

For the global step later, avoid a topological finite-cover argument. Use an explicit finite chain of windows as data, with fields saying each window is short enough and each source interval is supplied by either an earlier window or a seed positive-time bound. This is much easier in Lean than extracting a finite subcover from open intervals.

Recommended order:

1. Land the per-window theorem:

   ```text
   WindowWiring a lo hi G -> ∀ n, gradient_bound_on [lo,hi] n G.
   ```

2. Later, define a `WindowChain` only if the final theorem needs a bound on all of `[lo,T]`.

3. In the chain theorem, pass the left-strip supply as an explicit field. Do not hide it in automation.

A typical chain field is:

```text
for each i, source interval [a_i, hi_i] is covered by
  already supplied left part [a_i, lo_i]
  plus current target part [lo_i, hi_i].
```

If you keep `a_i = lo_i/2`, a finite chain may not make the contraction coefficient small on later large windows. For a genuine “small window” cover, use general restarts `a_i < lo_i` with `hi_i - a_i ≤ δ`.

## 3. Clean truncated profile

Do not mutate the existing χ₀ = 0 `G1profile`. It is correctly specialized to the logistic-only source

```text
CL = M * (a + b*M^α).
```

For the truncated case, define a new window source profile with a gradient parameter `G`:

```text
CL_trunc(G) = A_L + |χ₀| * (A_F + B_F * G).
```

Then split it into an affine self-map:

```text
A_win = Cg*M/sqrt(lo-a)
          + 2*Cg*sqrt(hi-a)*(A_L + |χ₀|*A_F),

B_win = 2*Cg*sqrt(hi-a)*|χ₀|*B_F,

G_{n+1} ≤ A_win + B_win*G_n.
```

This split is much cleaner in Lean than repeatedly unfolding a source constant containing `G`. It also makes the scalar gate reusable:

```text
A_win + B_win * G ≤ G.
```

When you want a canonical choice, define

```text
G_fixed = A_win / (1 - B_win)
```

and prove once that it closes when `B_win < 1`.

## 4. Induction or fixed-point theorem?

Use induction over `n`.

The small-window argument is a scalar invariant closure, not a Banach fixed-point proof. The Picard convergence/fixed-point theorem is already a separate layer. For the uniform gradient bound, the Lean proof should be:

```text
base:  gradient bound for iterate 0 on the window;
step:  assume bound for iterate n;
       obtain source bound for source_n;
       apply gradient Duhamel kernel estimate;
       close by A_win + B_win*G ≤ G;
all:   Nat.rec / induction n.
```

This is exactly the style of `picardIterateUniformData_all` in the existing χ₀ = 0 infrastructure.

## Suggested Lean skeleton

The following is a skeleton for a new file such as

```text
ShenWork/Paper2/IntervalTruncatedGradientWindow.lean
```

The source/split fields are deliberately abstract. You can instantiate them with the already-proved operator-level IBP theorem and the flux derivative estimate. This keeps the first landed theorem small and prevents the window algebra from depending on the details of the B-form/conjugate kernel implementation.

```lean
import ShenWork.Paper2.IntervalPicardIterateUniform
import ShenWork.Paper2.IntervalBFormCron2TruncatedPicard
import ShenWork.PDE.IntervalGradDuhamelBound

open MeasureTheory Filter Topology
open ShenWork.IntervalDomain (intervalDomainLift intervalDomainPoint intervalMeasure)
open ShenWork.IntervalGradDuhamelBound (gradDuhamel_sup_bound)
open ShenWork.HeatKernelGradientEstimates
  (heatGradientLinftyLinftyConstant heatGradientLinftyLinftyConstant_nonneg)

noncomputable section

namespace ShenWork.Paper2.TruncatedGradientWindow

/-- Effective truncated source bound for a gradient envelope `G`:
`|L - χ₀ F'| ≤ A_L + |χ₀|*(A_F + B_F*G)`. -/
def truncWindowSourceCL (A_L A_F B_F χ G : ℝ) : ℝ :=
  A_L + |χ| * (A_F + B_F * G)

/-- Constant part of the shifted-window first-gradient estimate.  The general
restart is `a`; the special positive-window choice is `a = lo/2`. -/
def truncWindowA (M A_L A_F χ a lo hi : ℝ) : ℝ :=
  heatGradientLinftyLinftyConstant / Real.sqrt (lo - a) * M
    + heatGradientLinftyLinftyConstant * (2 * Real.sqrt (hi - a))
        * (A_L + |χ| * A_F)

/-- Linear coefficient multiplying the previous gradient envelope. -/
def truncWindowB (B_F χ a hi : ℝ) : ℝ :=
  heatGradientLinftyLinftyConstant * (2 * Real.sqrt (hi - a)) * |χ| * B_F

/-- The affine scalar recurrence for the window. -/
def truncWindowAffine
    (M A_L A_F B_F χ a lo hi G : ℝ) : ℝ :=
  truncWindowA M A_L A_F χ a lo hi
    + truncWindowB B_F χ a hi * G

/-- The canonical scalar fixed envelope when `B < 1`.  The main window theorem
should consume the inequality `truncWindowAffine ... G ≤ G`; this definition is
only a convenient constructor for that inequality. -/
def truncWindowFixedG (M A_L A_F B_F χ a lo hi : ℝ) : ℝ :=
  truncWindowA M A_L A_F χ a lo hi /
    (1 - truncWindowB B_F χ a hi)

/-- Pure scalar closure lemma.  Keep this separate from PDE arguments. -/
theorem affine_fixed_closes {A B : ℝ} (hB : B < 1) :
    A + B * (A / (1 - B)) ≤ A / (1 - B) := by
  have hden_pos : 0 < 1 - B := sub_pos.mpr hB
  have hden_ne : 1 - B ≠ 0 := ne_of_gt hden_pos
  have hEq : A + B * (A / (1 - B)) = A / (1 - B) := by
    field_simp [hden_ne]
    ring
  exact le_of_eq hEq

/-- Per-iterate first-gradient envelope on a closed time window. -/
abbrev IterGradOnWindow
    (U : ℕ → ℝ → intervalDomainPoint → ℝ)
    (lo hi : ℝ) (n : ℕ) (G : ℝ) : Prop :=
  ∀ t, lo ≤ t → t ≤ hi → ∀ x : ℝ,
    |deriv (intervalDomainLift (U n t)) x| ≤ G

/-- Monotonicity in the scalar envelope. -/
theorem IterGradOnWindow.mono
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {lo hi : ℝ} {n : ℕ} {G₁ G₂ : ℝ}
    (hG : G₁ ≤ G₂) (H : IterGradOnWindow U lo hi n G₁) :
    IterGradOnWindow U lo hi n G₂ := by
  intro t htlo hthi x
  exact (H t htlo hthi x).trans hG

/-- Glue a left/source strip `[a,lo]` and a target strip `[lo,hi]` into a source
interval `[a,hi]`.  This is the key lemma that prevents the restart interval
from being hidden. -/
theorem IterGradOnWindow.glue_left
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {a lo hi G : ℝ} {n : ℕ}
    (Hleft : IterGradOnWindow U a lo n G)
    (Hright : IterGradOnWindow U lo hi n G) :
    IterGradOnWindow U a hi n G := by
  intro t hta hthi x
  by_cases htlo : t ≤ lo
  · exact Hleft t hta htlo x
  · have hlot : lo ≤ t := le_of_lt (lt_of_not_ge htlo)
    exact Hright t hlot hthi x

/-- Scalar bound for the combined truncated source from separate logistic and
flux-derivative estimates. -/
theorem abs_logistic_sub_chi_flux_le
    {L Fp A_L A_F B_F χ G : ℝ}
    (hL : |L| ≤ A_L)
    (hF : |Fp| ≤ A_F + B_F * G) :
    |L - χ * Fp| ≤ truncWindowSourceCL A_L A_F B_F χ G := by
  unfold truncWindowSourceCL
  calc
    |L - χ * Fp| ≤ |L| + |χ * Fp| := abs_sub_le L (χ * Fp)
    _ = |L| + |χ| * |Fp| := by rw [abs_mul]
    _ ≤ A_L + |χ| * (A_F + B_F * G) := by
      exact add_le_add hL (mul_le_mul_of_nonneg_left hF (abs_nonneg χ))

/-- Window wiring package.

`Src n s y` is the already-IBP'd source entering the shifted heat Duhamel formula
on `[a,t]`.  In the concrete truncated Keller--Segel application it is the
function corresponding to `L_n - χ₀ F'_n` after your operator-level IBP bridge.

The field `hleft` is intentional: the Duhamel source is integrated from `a`, not
from `lo`, so source control on `[a,lo]` must be supplied by a previous window or
by a seed positive-time estimate. -/
structure TruncatedGradientWindowWiring
    (p : CM2Params)
    (U : ℕ → ℝ → intervalDomainPoint → ℝ)
    (Src : ℕ → ℝ → ℝ → ℝ)
    (M A_L A_F B_F a lo hi G : ℝ) : Prop where
  hM_nonneg : 0 ≤ M
  hAL_nonneg : 0 ≤ A_L
  hAF_nonneg : 0 ≤ A_F
  hBF_nonneg : 0 ≤ B_F
  hG_nonneg : 0 ≤ G
  ha_lt_lo : a < lo
  hlo_le_hi : lo ≤ hi
  /-- Scalar window gate.  This can be built from `B_win < 1` and
  `G = A_win/(1-B_win)`, but the induction theorem should only need this form. -/
  hclosed : truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G ≤ G
  /-- Bound for the left/source strip `[a,lo]`, supplied by a previous window or
  by a seed positive-time estimate. -/
  hleft : ∀ n : ℕ, IterGradOnWindow U a lo n G
  /-- Base iterate bound on the target window. -/
  hbase : IterGradOnWindow U lo hi 0 G
  /-- Analytic source estimate from a gradient envelope on the full source
  interval `[a,hi]`.  This is where the flux derivative estimate
  `|F'_n| ≤ A_F + B_F*G` is used. -/
  hsource_of_grad : ∀ n : ℕ,
    IterGradOnWindow U a hi n G →
      ∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
        |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G
  /-- Kernel/Duhamel raw step.  This should be proved from the shifted mild
  identity, the homogeneous heat gradient estimate, and `gradDuhamel_sup_bound`.
  It deliberately returns the affine bound before scalar closure. -/
  hkernel_step : ∀ n : ℕ,
    (∀ s, a ≤ s → s ≤ hi → ∀ y : ℝ,
      |Src n s y| ≤ truncWindowSourceCL A_L A_F B_F p.χ₀ G) →
      IterGradOnWindow U lo hi (n + 1)
        (truncWindowAffine M A_L A_F B_F p.χ₀ a lo hi G)

/-- One Picard step closes the target window. -/
theorem truncatedGradientWindow_succ
    {p : CM2Params}
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {Src : ℕ → ℝ → ℝ → ℝ}
    {M A_L A_F B_F a lo hi G : ℝ}
    (W : TruncatedGradientWindowWiring p U Src M A_L A_F B_F a lo hi G)
    {n : ℕ}
    (IH : IterGradOnWindow U lo hi n G) :
    IterGradOnWindow U lo hi (n + 1) G := by
  have HsrcInterval : IterGradOnWindow U a hi n G :=
    IterGradOnWindow.glue_left (W.hleft n) IH
  have Hsrc := W.hsource_of_grad n HsrcInterval
  have Hraw := W.hkernel_step n Hsrc
  exact IterGradOnWindow.mono W.hclosed Hraw

/-- Uniform target-window gradient bound for all Picard iterates. -/
theorem truncatedGradientWindow_all
    {p : CM2Params}
    {U : ℕ → ℝ → intervalDomainPoint → ℝ}
    {Src : ℕ → ℝ → ℝ → ℝ}
    {M A_L A_F B_F a lo hi G : ℝ}
    (W : TruncatedGradientWindowWiring p U Src M A_L A_F B_F a lo hi G) :
    ∀ n : ℕ, IterGradOnWindow U lo hi n G := by
  intro n
  induction n with
  | zero => exact W.hbase
  | succ n IH => exact truncatedGradientWindow_succ W IH

end ShenWork.Paper2.TruncatedGradientWindow
```

## How to prove `hkernel_step`

For each `t ∈ [lo,hi]`, use the shifted mild identity

```text
u_{n+1}(t) = S(t-a)u_n(a) + ∫_a^t S(t-s) Src_n(s) ds.
```

After differentiating in `x`, estimate the homogeneous term by

```text
|∂x S(t-a) u_n(a)| ≤ Cg * M / sqrt(t-a)
                   ≤ Cg * M / sqrt(lo-a),
```

using `a < lo ≤ t` and the uniform ball bound `|u_n(a)| ≤ M`.

Estimate the Duhamel term by `gradDuhamel_sup_bound` shifted from `[0,t-a]` to `[a,t]`:

```text
|∫_a^t ∂x S(t-s) Src_n(s) ds|
  ≤ Cg * 2 * sqrt(t-a) * Csrc
  ≤ Cg * 2 * sqrt(hi-a) * Csrc.
```

Here

```text
Csrc = A_L + |χ₀| * (A_F + B_F*G).
```

Expanding gives exactly

```text
truncWindowA + truncWindowB * G.
```

If the current `gradDuhamel_sup_bound` is only stated on `[0,t]`, prove a shifted wrapper once. Do not reprove the kernel estimate. The shifted wrapper can be stated abstractly with `qShift r y = Src n (a+r) y` and horizon `t-a`.

## How to prove `hsource_of_grad`

This is the flux derivative estimate:

```text
|L_n(s,y)| ≤ A_L,
|F'_n(s,y)| ≤ A_F + B_F*G,
```

so

```text
|L_n(s,y) - χ₀ F'_n(s,y)|
  ≤ A_L + |χ₀|*(A_F + B_F*G).
```

The small Lean lemma `abs_logistic_sub_chi_flux_le` above is the only scalar algebra needed. All PDE work should go into proving the two inputs `|L| ≤ A_L` and `|F'| ≤ A_F + B_F*G`.

For the flux derivative:

```text
F_n = positivePart(u_n) * v_{n,x} / (1+R)^β,
```

use the already available endpoint vanishing only for the IBP/divergence-to-source bridge. For this Lipschitz window estimate, the relevant local bound is the pointwise derivative bound

```text
|F'_n| ≤ A_F + B_F * |u_{n,x}|,
```

where the constants absorb the elliptic bounds on `v_n`, `v_{n,x}`, and `v_{n,xx}` from the `L∞` ball. Then `IterGradOnWindow U a hi n G` gives the displayed `A_F + B_F*G`.

## Covering `[lo,T]` later

When the per-window theorem is landed, a later finite-chain theorem should be data-driven. A good structure is:

```text
structure WindowChain where
  N        : ℕ
  left     : Fin (N+1) -> ℝ
  right    : Fin N -> ℝ
  restart  : Fin N -> ℝ
  covers   : ...
  short    : ∀ i, right i - restart i ≤ δ
  leftFeed : each [restart i, left i] is already supplied
  gates    : each window has A_i + B_i*G_i ≤ G_i
```

But do not formalize this now unless a downstream theorem actually requires it. The per-window theorem is the reusable atom. In practice, most later proofs can consume:

```text
∀ lo hi, admissible_window lo hi -> ∃ G, ∀ n, gradient_bound_on [lo,hi] n G
```

rather than a single monolithic global-cover theorem.

## Direct answers

1. **Structure:** use `abbrev IterGradOnWindow ... : Prop` for the one-field bound, and a `Prop` structure such as `TruncatedGradientWindowWiring` for the constants, source estimate, kernel step, left-strip supply, base case, and scalar gate.

2. **Cover:** prove the per-window theorem first. If a global cover is later needed, use an explicit finite `WindowChain` with shortness and left-feed fields. Do not use a topological finite-subcover proof. Also, for a genuine short-window cover, use a general restart `a < lo`; `a = lo/2` does not become short by shrinking `hi-lo`.

3. **Profile:** define a new truncated profile, not by modifying `G1profile`:

   ```text
   CL_trunc(G) = A_L + |χ₀|*(A_F + B_F*G),
   A_win = homogeneous + Duhamel constant part,
   B_win = Duhamel gradient coefficient,
   recurrence = A_win + B_win*G.
   ```

4. **Proof method:** use induction over `n`. The only fixed-point-like ingredient should be the scalar algebra lemma showing `A+B*G≤G`, with `G=A/(1-B)` as one possible constructor when `B<1`. A Banach fixed-point theorem is unnecessary for the uniform gradient bound.
