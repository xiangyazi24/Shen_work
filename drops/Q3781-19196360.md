# Q3781 route selector: next core theorem after Task305

Audit basis: origin/main commit `ec5d5a6e` / Task305.

## Verdict

The next smallest honest core theorem is **not** a direct producer for `GradientMildClassicalCoreData.hpde_u`. It is the **time-integrated/a.e. source bridge** obtained by integrating Task305’s per-slice equality over `s ∈ (0,t)` and discarding the endpoint `s = t` where `r = t - s = 0`.

More precisely, I would land it in two layers:

1. first, the truly minimal lemma:

```lean
gradientMildSourceIntegrand_eq_mixedSpectralSource_ae_of_classical_physicalRep
```

which is only an a.e. equality of the two `s`-integrands on `volume.restrict (Set.uIoc 0 t)`;

2. then, as a direct corollary with explicit integrability hypotheses:

```lean
gradientMildDuhamelTerms_eq_integral_mixedSpectralSource_of_classical_physicalRep
```

which rewrites

```lean
gradientMildChemotaxisDuhamelTerm p u t x +
gradientMildLogisticDuhamelTerm p u t x
```

as the interval integral of the mixed sine/cosine source expression.

This is candidate **1**, factored at the right granularity. The a.e. integrand bridge is the smallest core theorem; the integrated theorem is the useful consumer once integrability is available.

## Why not candidate 2 now?

A direct `hpde_u` producer for `GradientMildClassicalCoreData.hpde_u` is much larger and risks circularity. Task305’s theorem assumes

```lean
IsPaper2ClassicalSolution intervalDomain p T u (coupledChemicalConcentration p u)
```

so using it directly to prove `hpde_u` for the same mild solution would assume the classical solution whose `hpde_u` is one of the target components. That is valid only as a **classical-consistency** lemma, not as the producer of the mild solution’s missing PDE field.

A non-circular `hpde_u` producer would need the full differentiation of the weak gradient Duhamel map in time, including singular behavior near `s = t`, source time regularity, integration-by-parts/Leibniz justifications, and spectral inversion of the mixed sine/cosine source. That is not the next smallest theorem.

## Core theorem 1: a.e. integrand bridge

This is the smallest theorem. It consumes the real input from Task305 and handles the endpoint `s = t` by a.e. restriction. It does not need interval-integrability.

```lean
import ShenWork.Paper2.IntervalGradientSourceBridgePhysicalRep
import ShenWork.Paper2.IntervalMildToLocalExistence

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.IntervalMildToLocalExistence

open ShenWork.IntervalDomain
  (intervalDomain intervalDomainPoint)
open ShenWork.IntervalGradientDuhamelMap
  (chemFluxLifted logisticLifted)
open ShenWork.IntervalNeumannFullKernel
  (intervalFullSemigroupOperator)
open ShenWork.IntervalCoupledRegularityBootstrap
  (coupledChemicalConcentration coupledChemDivSourceLift coupledLogisticSourceCoeffs)
open ShenWork.Paper2.IntervalGradientSourceBridgeOpen

/-- A.e. in time, Task305 rewrites the weak gradient/logistic integrand as the
mixed sine/cosine source integrand.  The endpoint `s = t` is discarded, since the
per-slice bridge requires `0 < t - s`. -/
theorem gradientMildSourceIntegrand_eq_mixedSpectralSource_ae_of_classical_physicalRep
    {p : CM2Params} {T t x : ℝ} {u : ℝ → intervalDomainPoint → ℝ}
    (ht0 : 0 < t) (htT : t < T) (hx : x ∈ Set.Ioo (0 : ℝ) 1)
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u
      (coupledChemicalConcentration p u)) :
    (fun s : ℝ =>
      (-p.χ₀) *
          deriv (fun z : ℝ =>
            intervalFullSemigroupOperator (t - s) (chemFluxLifted p (u s)) z) x
        + intervalFullSemigroupOperator (t - s) (logisticLifted p (u s)) x)
      =ᵐ[volume.restrict (Set.uIoc (0 : ℝ) t)]
    (fun s : ℝ =>
      (-p.χ₀) *
          unitIntervalSineHeatValue (t - s)
            (sineCoeffs (coupledChemDivSourceLift p u s)) x
        + unitIntervalCosineHeatValue (t - s)
            (coupledLogisticSourceCoeffs p u s) x) := by
  rw [Set.uIoc_of_le ht0.le]
  filter_upwards
    [(MeasureTheory.Ioo_ae_eq_Ioc
      (a := (0 : ℝ)) (b := t) (μ := volume)).symm] with s hs_ae hsIoc
  have hsIoo_t : s ∈ Set.Ioo (0 : ℝ) t := hs_ae.mp hsIoc
  have hr : 0 < t - s := sub_pos.mpr hsIoo_t.2
  have hsT : s ∈ Set.Ioo (0 : ℝ) T :=
    ⟨hsIoo_t.1, lt_trans hsIoo_t.2 htT⟩
  exact
    gradient_source_bridge_slice_open_of_classical_physicalRep
      (p := p) (T := T) (u := u) (r := t - s) (x := x) (s := s)
      hr hx hsol hsT

end ShenWork.IntervalMildToLocalExistence
```

### Why this should compile

The proof is almost identical to the endpoint-erasing pattern already used in `IntervalGradientSourceBridgeOpen.lean`:

```lean
MeasureTheory.Ioo_ae_eq_Ioc
Set.uIoc_of_le
filter_upwards ... with s ...
```

For `s ∈ Ioo 0 t`, we get both:

```lean
0 < t - s
s ∈ Ioo 0 T
```

using `htT : t < T`.

## Core theorem 2: integrated Duhamel bridge

This should be the immediate follow-up. It needs exactly the integrability inputs required by `intervalIntegral.integral_add` and `intervalIntegral.integral_const_mul`. Keep those inputs explicit at first; do not hide new analytic frontiers inside the theorem.

```lean
import ShenWork.Paper2.IntervalGradientSourceBridgeIntegrated

open MeasureTheory intervalIntegral
open scoped Topology

noncomputable section

namespace ShenWork.IntervalMildToLocalExistence

open ShenWork.IntervalDomain
  (intervalDomain intervalDomainPoint)
open ShenWork.IntervalGradientDuhamelMap
  (chemFluxLifted logisticLifted)
open ShenWork.IntervalNeumannFullKernel
  (intervalFullSemigroupOperator)
open ShenWork.IntervalCoupledRegularityBootstrap
  (coupledChemicalConcentration coupledChemDivSourceLift coupledLogisticSourceCoeffs)
open ShenWork.Paper2.IntervalGradientSourceBridgeOpen

/-- Integrated form of the Task305 slice bridge for the two Duhamel source terms.
The integrability hypotheses are kept explicit: producing them is a separate
analytic input, not part of the endpoint-insensitive bridge. -/
theorem gradientMildDuhamelTerms_eq_integral_mixedSpectralSource_of_classical_physicalRep
    {p : CM2Params} {T t x : ℝ} {u : ℝ → intervalDomainPoint → ℝ}
    (ht0 : 0 < t) (htT : t < T) (hx : x ∈ Set.Ioo (0 : ℝ) 1)
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u
      (coupledChemicalConcentration p u))
    (hchem_int :
      IntervalIntegrable
        (fun s : ℝ =>
          deriv (fun z : ℝ =>
            intervalFullSemigroupOperator (t - s) (chemFluxLifted p (u s)) z) x)
        volume (0 : ℝ) t)
    (hlog_int :
      IntervalIntegrable
        (fun s : ℝ =>
          intervalFullSemigroupOperator (t - s) (logisticLifted p (u s)) x)
        volume (0 : ℝ) t) :
    gradientMildChemotaxisDuhamelTerm p u t x
      + gradientMildLogisticDuhamelTerm p u t x
      =
    ∫ s in (0 : ℝ)..t,
      ((-p.χ₀) *
          unitIntervalSineHeatValue (t - s)
            (sineCoeffs (coupledChemDivSourceLift p u s)) x
        + unitIntervalCosineHeatValue (t - s)
            (coupledLogisticSourceCoeffs p u s) x) := by
  let A : ℝ → ℝ := fun s : ℝ =>
    deriv (fun z : ℝ =>
      intervalFullSemigroupOperator (t - s) (chemFluxLifted p (u s)) z) x
  let B : ℝ → ℝ := fun s : ℝ =>
    intervalFullSemigroupOperator (t - s) (logisticLifted p (u s)) x
  let C : ℝ → ℝ := fun s : ℝ =>
    (-p.χ₀) *
        unitIntervalSineHeatValue (t - s)
          (sineCoeffs (coupledChemDivSourceLift p u s)) x
      + unitIntervalCosineHeatValue (t - s)
          (coupledLogisticSourceCoeffs p u s) x
  have hleft :
      gradientMildChemotaxisDuhamelTerm p u t x
        + gradientMildLogisticDuhamelTerm p u t x
        = ∫ s in (0 : ℝ)..t, ((-p.χ₀) * A s + B s) := by
    -- `intervalIntegral.integral_add` needs `(hchem_int.const_mul (-p.χ₀))`
    -- and `hlog_int`.  `intervalIntegral.integral_const_mul` rewrites the first
    -- integral back to `(-χ₀) * ∫ A`.
    rw [intervalIntegral.integral_add (hchem_int.const_mul (-p.χ₀)) hlog_int]
    rw [intervalIntegral.integral_const_mul]
    simp [gradientMildChemotaxisDuhamelTerm, gradientMildLogisticDuhamelTerm,
      A, B]
  have hae :
      (fun s : ℝ => (-p.χ₀) * A s + B s)
        =ᵐ[volume.restrict (Set.uIoc (0 : ℝ) t)] C := by
    simpa [A, B, C] using
      gradientMildSourceIntegrand_eq_mixedSpectralSource_ae_of_classical_physicalRep
        (p := p) (T := T) (u := u) (t := t) (x := x)
        ht0 htT hx hsol
  calc
    gradientMildChemotaxisDuhamelTerm p u t x
        + gradientMildLogisticDuhamelTerm p u t x
        = ∫ s in (0 : ℝ)..t, ((-p.χ₀) * A s + B s) := hleft
    _ = ∫ s in (0 : ℝ)..t, C s := by
        exact intervalIntegral.integral_congr_ae hae
    _ = ∫ s in (0 : ℝ)..t,
        ((-p.χ₀) *
            unitIntervalSineHeatValue (t - s)
              (sineCoeffs (coupledChemDivSourceLift p u s)) x
          + unitIntervalCosineHeatValue (t - s)
              (coupledLogisticSourceCoeffs p u s) x) := rfl

end ShenWork.IntervalMildToLocalExistence
```

If the exact method name `hchem_int.const_mul` is not available, grep for examples of `IntervalIntegrable` scalar multiplication. Existing code uses patterns like `intervalIntegral.integral_add hf hg`, `intervalIntegral.integral_const_mul`, and `.neg`; the local replacement is likely one of:

```lean
hchem_int.const_mul (-p.χ₀)
(hchem_int.mul_const (-p.χ₀))
intervalIntegrable_const_mul
```

The theorem should keep `hchem_int` and `hlog_int` explicit. A later analytic lemma can produce them from existing gradient/logistic Duhamel bound files.

## Candidate evaluation

### 1. Time-integrated bridge

This is the right next core path, with the a.e. integrand lemma first. It consumes Task305, handles the endpoint `s = t`, and moves from pointwise slices to the actual weak mild-map Duhamel terms. It is not a wrapper because it changes the quantifier/integration level and discharges the endpoint issue.

Likely Lean obstacles:

* `r = t - s` is positive only a.e.; use `Ioo_ae_eq_Ioc` on `volume.restrict (Set.uIoc 0 t)`.
* `intervalIntegral.integral_add` requires integrability of both summands. Keep `hchem_int` and `hlog_int` explicit.
* `intervalIntegral.integral_congr_ae` expects a.e. equality on the interval-integral domain, which is why the first theorem states equality under `volume.restrict (Set.uIoc 0 t)`.
* Do not attempt to evaluate the slice theorem at `s = t`.

### 2. Direct `hpde_u` producer

Not next. It would require a full time-differentiation theorem for the weak Duhamel map or a mixed spectral PDE agreement theorem. Existing Task305 is classical-consistency-shaped because it assumes `IsPaper2ClassicalSolution`. Using it to produce the same solution’s `hpde_u` would be circular unless the theorem is explicitly framed as “already-classical mild map agrees with mixed spectral Duhamel source.”

A non-circular `hpde_u` route should probably be staged after the integrated bridge, by proving a mixed spectral analog of the `χ₀ = 0` producer. That later theorem will need source time regularity for

```lean
fun s n =>
  (-p.χ₀) * ?sineCoeffSource s n + coupledLogisticSourceCoeffs p u s n
```

or a precise mixed sine/cosine restart representation. That is larger than the immediate integrated congruence.

### 3. Existing theorem already bridges it

No general theorem appears to exist. The closest existing theorem is

```lean
ShenWork.IntervalDomainPdeUProducer.mildSolution_pde_u_of_spectral
```

but it assumes `hχ0 : p.χ₀ = 0` and only handles the logistic/cosine source. It does not consume the new Task305 mixed gradient-source bridge.
