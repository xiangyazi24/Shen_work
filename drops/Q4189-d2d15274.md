# Q4189 — restart identity for truncated nonlinear heat Duhamel Picard iterate

## Executive answer

The clean split is:

1. The **linear restart algebra** is closable and should be isolated as a separate theorem.
2. The **conjugate-kernel-to-source conversion** is not closable from continuity/boundedness/measurability of the Picard iterates alone.
3. You can avoid **pointwise everywhere differentiability** and the `not_bad` condition by proving a weaker **AC / weak-derivative integration-by-parts** version of the conjugate-kernel identity, but that still requires more than the current DT data. It needs at least `chemFlux(U n s)` to be absolutely continuous or in a one-dimensional `W^{1,1}` class, with zero traces and an integrable weak derivative.

So the restart theorem, in the form whose future-window source is

```lean
logistic (U n s) - χ₀ * deriv_y (chemFlux (U n s))
```

is effectively under-hypothesized if the only nonlinear regularity inputs are DT continuity, boundedness, and measurability.

The right architecture is to prove a **linear restart lemma with a source-compatibility hypothesis**, then separately prove the compatibility either from C¹ + `not_bad`, or from AC / weak derivative, or assume it as part of the data for this theorem.

## What exactly blocks the direct proof

The original iterate has the chemotaxis leg in conjugate-kernel form:

```lean
-χ₀ * ∫ s in 0..t,
  intervalConjugateKernelOperator (t - s) (chemFlux (U n s)) x
```

The target restart identity has the source form:

```lean
∫ s in a..τ,
  intervalFullSemigroupOperator (τ - s)
    (logistic (U n s) - χ₀ * deriv_y (chemFlux (U n s))) x
```

For the logistic part this is just the semigroup Duhamel restart. For the chemotaxis part, however, you need a theorem of the shape:

```lean
intervalConjugateKernelOperator t Q x
  = intervalFullSemigroupOperator t (fun y => deriv Q y) x
```

or the corresponding statement with an explicitly supplied derivative `q'`.

Your available lemma `(ii)` gives exactly this, but only under strong pointwise assumptions:

```lean
Q ContinuousOn [0,1]
HasDerivAt Q (Q' y) y for y ∈ Ioo 0 1
Q' IntervalIntegrable
Q 0 = 0
Q 1 = 0
```

To instantiate it with

```lean
Q := chemFlux (U n s)
```

you need differentiability of the chemFlux on the whole open interval. The available chemFlux differentiability lemma `(iii)` requires differentiability of `lift (U n s)` and the `not_bad` condition:

```lean
lift (U n s) y = 0 → deriv (lift (U n s)) y = 0
```

The DT package, as described, does not carry either C¹ regularity or nonnegativity / `not_bad`. Therefore the direct route is circular if you try to obtain these facts from the same differentiability apparatus that depends on the restart theorem.

## Can pointwise `not_bad` be avoided?

Yes, mathematically, but not for free.

The real issue is not pointwise differentiability at every zero of `positivePart`. The correct weak statement is:

```lean
Q ∈ AC([0,1])
Q 0 = 0
Q 1 = 0
q' is the a.e. derivative of Q
q' ∈ L¹(0,1)
```

Then the integration-by-parts identity is valid:

```lean
∫ y in 0..1, ∂_y K_full(t,x,y) * Q y
  = ∫ y in 0..1, K_full(t,x,y) * q' y
```

up to the sign convention used in your definition of `K_conj`. This would produce an AC/weak-derivative version of `(ii)`:

```lean
theorem intervalConjugateKernelOperator_eq_semigroup_aeDeriv
    {Q q' : ℝ → ℝ} {t x : ℝ}
    (hQ_ac : AbsolutelyContinuousOn Q (Set.Icc 0 1))
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0)
    (hQ_deriv_ae : ∀ᵐ y ∂volume.restrict (Set.Ioo 0 1),
      HasDerivAt Q (q' y) y) -- or the local a.e.-derivative formulation you use
    (hq'_int : IntervalIntegrable q' volume 0 1) :
    intervalConjugateKernelOperator t Q x =
      intervalFullSemigroupOperator t q' x := by
  -- integrate by parts in y, using zero boundary traces
  -- then identify the right side with the full semigroup operator
  ...
```

This avoids the `not_bad` condition because `positivePart ∘ f` is absolutely continuous when `f` is absolutely continuous, and its a.e. derivative is `indicator {0 < f} * f'`. The kink at zero is harmless in the weak/a.e. formulation.

But this still requires `lift (U n s)` to be at least AC in `y`, and the remaining multiplicative factors in `chemFlux` to have enough regularity. DT continuity/boundedness alone does not imply that. A continuous function composed with `positivePart` is continuous, not necessarily AC or weakly differentiable.

So the AC route is a good mathematical design, but it is not a way to close the present theorem from the current DT hypotheses only.

## A useful observation: C¹ + nonnegativity makes `not_bad` elementary

If later you can prove

```lean
0 ≤ lift (U n s) y
```

for all interior `y`, then `not_bad` follows from ordinary calculus, not from a strong maximum principle. A differentiable nonnegative function has derivative zero at any interior zero, because the zero is a local minimum:

```lean
lemma deriv_eq_zero_of_nonneg_of_eq_zero
    {f : ℝ → ℝ} {y : ℝ}
    (hy0 : 0 < y) (hy1 : y < 1)
    (hf_nonneg : ∀ z ∈ Set.Ioo 0 1, 0 ≤ f z)
    (hf_deriv : DifferentiableAt ℝ f y)
    (hfy : f y = 0) :
    deriv f y = 0 := by
  -- Fermat/local-minimum argument
  -- use `IsLocalMin` + derivative-at-local-min if available
  ...
```

This avoids a strong maximum principle, but still requires C¹ and nonnegativity. The current DT data does not include those either.

## The linear restart lemma you should prove first

The robust architecture is a linear theorem that does not mention `chemFlux` at all. It should take the source compatibility as a hypothesis.

A good target theorem shape is:

```lean
theorem duhamel_restart_from_source_compat
    {a τ hi T : ℝ} (ha : 0 < a) (haτ : a < τ) (hτhi : τ ≤ hi) (hτT : τ ≤ T)
    {u0 : CUnit} -- whatever function space you use
    {Q : ℝ → CUnit}
    {L : ℝ → CUnit}
    {Src : ℝ → CUnit}
    (hQ_int : IntervalIntegrable Q volume 0 τ)
    (hL_int : IntervalIntegrable L volume 0 τ)
    (hSrc_int : IntervalIntegrable Src volume a τ)
    -- Past commutation for the conjugate leg:
    (hConj_restart :
      S (τ - a) (∫ s in 0..a, K_conj (a - s) (Q s)) =
        ∫ s in 0..a, K_conj (τ - s) (Q s))
    -- Future compatibility with the source used in the statement:
    (hSrc_compat : ∀ᵐ s ∂volume.restrict (Set.Icc a τ),
      Src s = L s - χ₀ • derivChemFluxSource s)
    (hConj_to_src : ∀ᵐ s ∂volume.restrict (Set.Icc a τ),
      K_conj (τ - s) (Q s) = S (τ - s) (derivChemFluxSource s)) :
    U τ = S (τ - a) (U a) + ∫ s in a..τ, S (τ - s) (Src s) := by
  -- split integrals at a
  -- commute S through past integrals
  -- use semigroup composition
  -- use compatibility only on [a, τ]
  ...
```

In practice, you may not want both `hSrc_compat` and `hConj_to_src`; you can merge them into a single statement:

```lean
(hFuture_source : ∀ᵐ s ∂volume.restrict (Set.Icc a τ),
  -χ₀ • K_conj (τ - s) (Q s) + S (τ - s) (L s)
    = S (τ - s) (Src s))
```

This is the cleanest way to avoid requiring the restart theorem itself to know how `chemFlux` differentiates.

## How to handle the past conjugate leg without chemFlux differentiability

Do not convert the old interval `[0,a]` conjugate term to a derivative source. You only need to transport it from time `a` to time `τ`:

```lean
S (τ - a) (∫ s in 0..a, K_conj (a - s) (Q s))
  = ∫ s in 0..a, K_conj (τ - s) (Q s)
```

The ideal lemma is:

```lean
theorem intervalFullSemigroupOperator_comp_conjugate
    {r t : ℝ} (hr : 0 ≤ r) (ht : 0 ≤ t) (Q : CUnit) :
    S r (K_conj t Q) = K_conj (r + t) Q := by
  -- direct kernel/series semigroup identity
  ...
```

This lemma only needs `Q` as a function/integrable input, not differentiability of `Q`. It is much better for the restart proof than using `(ii)` on the past leg.

If this lemma does not exist, it is still the right infrastructure to add. It is linear kernel algebra, not nonlinear regularity. It should be proved once for the conjugate kernel from the periodized-image kernel identities.

## S ∘ ∫ interchange: cleanest Lean route

The best route is not product-measure Fubini if you can avoid it. Make the heat semigroup a continuous linear map on the function space in which the time integral lives.

For a Banach space of functions, package:

```lean
noncomputable def intervalFullSemigroupCLM (r : ℝ) : CUnit →L[ℝ] CUnit := ...
```

Then use the continuous-linear-map integral commutation theorem. The exact Mathlib name can vary by import, so check these names with `#check`:

```lean
#check ContinuousLinearMap.integral_comp_comm
#check ContinuousLinearMap.map_integral
```

The intended use is:

```lean
have hmap :
    intervalFullSemigroupCLM r (∫ s in 0..a, F s) =
      ∫ s in 0..a, intervalFullSemigroupCLM r (F s) := by
  -- rewrite interval integral to a measure integral if needed
  -- then use the continuous linear map lemma
  simpa using
    (intervalFullSemigroupCLM r).integral_comp_comm
      (μ := volume.restrict (Set.Icc 0 a))
      (f := fun s => F s)
```

For interval-integral notation, the common pattern is:

```lean
rw [intervalIntegral.integral_of_le ha_nonneg]
-- apply `ContinuousLinearMap.integral_comp_comm` to the restricted measure integral
```

or prove a local wrapper:

```lean
lemma ContinuousLinearMap.intervalIntegral_comp_comm
    (L : E →L[ℝ] F) {a b : ℝ} (hab : a ≤ b)
    {f : ℝ → E} (hf : IntervalIntegrable f volume a b) :
    L (∫ s in a..b, f s) = ∫ s in a..b, L (f s) := by
  rw [intervalIntegral.integral_of_le hab]
  -- reduce to Bochner integral over `volume.restrict (Set.Icc a b)`
  simpa using L.integral_comp_comm (μ := volume.restrict (Set.Icc a b)) (f := f)
```

The exact restricted-measure set may be `Ioc` depending on how your local `intervalIntegral` normalizes; use whatever `intervalIntegral.integral_of_le` exposes.

Once you have this, the logistic restart proof is just:

```lean
calc
  S (τ - a) (∫ s in 0..a, S (a - s) (L s))
      = ∫ s in 0..a, S (τ - a) (S (a - s) (L s)) := by
          exact S_intervalIntegral_comp_comm ...
  _ = ∫ s in 0..a, S (τ - s) (L s) := by
          apply intervalIntegral.integral_congr
          intro s hs
          rw [intervalFullSemigroupOperator_comp]
          ring_nf -- `(τ-a)+(a-s)=τ-s`
```

This is far cleaner than doing a two-dimensional kernel Fubini proof at every restart use.

## If you cannot package `S` as a continuous linear map

Then use scalar Fubini at fixed `x`. The ingredients are:

```lean
intervalIntegral.integral_congr
intervalIntegral.integral_add
intervalIntegral.integral_sub
intervalIntegral.integral_const_mul
MeasureTheory.integral_prod
MeasureTheory.Integrable.swap
MeasureTheory.Integrable.prod_left_ae
MeasureTheory.Integrable.prod_right_ae
```

The proof shape is:

```lean
S r (fun y => ∫ s in 0..a, F s y) x
  = ∫ y in 0..1, K r x y * (∫ s in 0..a, F s y) dy
  = ∫ y in 0..1, ∫ s in 0..a, K r x y * F s y ds dy
  = ∫ s in 0..a, ∫ y in 0..1, K r x y * F s y dy ds
  = ∫ s in 0..a, S r (F s) x ds
```

On finite rectangles, the integrability proof should use boundedness:

```lean
|K r x y * F s y| ≤ C_K * C_F
```

and then finite measure of `[0,a] × [0,1]`. But this route is substantially more verbose than the continuous-linear-map route.

## Why DT continuity/boundedness alone cannot prove the derivative-source restart

Suppose the theorem only knows:

```lean
ContinuousOn (lift (U n s)) (Icc 0 1)
Bounded / measurable / integrable data
```

Then `chemFlux(U n s)` is at best continuous. A continuous function does not have a classical derivative or an L¹ weak derivative. Therefore there is no way to justify the identity:

```lean
K_conj t (chemFlux(U n s)) = S t (deriv_y chemFlux(U n s))
```

from those hypotheses alone.

In Lean, a formula involving `deriv` does not solve this. `deriv f y` is a total function, but without a `HasDerivAt` or weak-derivative hypothesis it does not certify that it is the derivative relevant for integration by parts. A hypothesis like

```lean
hSrc : Src s y = logistic (U n s) y - χ₀ * deriv (chemFlux (U n s)) y
```

is only algebraic unless it is accompanied by the regularity theorem that makes `deriv` meaningful in the integral identity.

## Recommended theorem stack

### Layer 1: pure linear restart

No nonlinear regularity. Prove once:

```lean
theorem linear_duhamel_restart
    -- semigroup composition
    (hS_comp : ...)
    -- S maps Bochner integrals
    (hS_int : ...)
    -- conjugate past restart or source compatibility
    (hConj_past : ...)
    (hFuture_source : ...)
    : restart_identity := by
  -- split integrals and commute S through the past integrals
  ...
```

This is the theorem the current sorry should ideally reduce to.

### Layer 2: conjugate-source compatibility, strong version

Use your current lemma `(ii)`:

```lean
theorem chemFlux_conjugate_source_compat_C1
    (hQ_cont : ContinuousOn Q (Set.Icc 0 1))
    (hQ_deriv : ∀ y ∈ Set.Ioo 0 1, HasDerivAt Q (q' y) y)
    (hq'_int : IntervalIntegrable q' volume 0 1)
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0) :
    K_conj t Q = S t q' := by
  exact intervalConjugateKernelOperator_eq_semigroup_deriv ...
```

### Layer 3: conjugate-source compatibility, weak version

Better mathematically, but new infrastructure:

```lean
theorem chemFlux_conjugate_source_compat_AC
    (hQ_ac : AbsolutelyContinuousOn Q (Set.Icc 0 1))
    (hQ_weak_deriv : ...)
    (hq'_int : IntervalIntegrable q' volume 0 1)
    (hQ0 : Q 0 = 0)
    (hQ1 : Q 1 = 0) :
    K_conj t Q = S t q' := by
  -- weak integration by parts
  ...
```

This avoids `not_bad`, but still requires AC/weak derivative.

### Layer 4: nonlinear chemFlux regularity

This is where you prove either:

```lean
chemFlux(U n s) is C¹ and not_bad
```

or:

```lean
chemFlux(U n s) is AC with weak derivative q'
```

This layer cannot be derived from DT continuity/boundedness alone.

## What to close first

If the immediate goal is to close one `sorry`, do not start by proving the nonlinear chemFlux regularity. Close a **linear** sub-lemma first:

```lean
S_intervalIntegral_comp_comm
```

or

```lean
linear_duhamel_restart_from_compat
```

These are genuinely reusable and do not depend on the circular differentiability apparatus.

If the existing theorem statement has no source-compatibility or regularity hypothesis, then the full restart identity with derivative source is not a good first target. It is not a matter of missing Fubini; it is missing analytic content.

## Bottom line

There is a clean sidestep of **pointwise `not_bad`**: use an AC / weak-derivative integration-by-parts theorem for the conjugate kernel.

There is no clean sidestep of **all chemFlux regularity** if the goal is written with `deriv_y (chemFlux(U n s))` as the source. Continuity and boundedness of iterates are not enough.

The restart proof should be refactored so that the current theorem consumes a source-compatibility hypothesis:

```lean
K_conj(t)(chemFlux(U n s)) = S(t)(deriv-source(s))
```

or an already-windowed source identity, and the proof of that compatibility should live outside the restart algebra. This breaks the definitional cycle and makes the restart `sorry` a linear semigroup/Duhamel proof rather than a hidden nonlinear regularity theorem.
