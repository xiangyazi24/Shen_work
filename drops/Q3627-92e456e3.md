# Q3627 — chemFlux Hölder frontier audit for `PatchedSliceDerivUniformApproachAtZero`

Repo inspected: `xiangyazi24/Shen_work`, current main at `f36d73122ef55971a3efb1ecafc6c5f1ab63fb14`.

Focus question: whether current source regularity can provide a **uniform spatial `C^θ` / Hölder modulus** for

```lean
chemFluxLifted p (D.u s) : ℝ → ℝ
```

on `[0,1]` for small positive `s`, so that the chemotaxis differentiated leg can feed

```lean
neumannHeatSecondDeriv_Ctheta_to_Linfty
```

or a future full-kernel second-derivative Duhamel zero-time theorem.

## Verdict

No existing theorem currently produces the needed uniform Hölder source package for

```lean
s ↦ chemFluxLifted p (D.u s)
```

near `s = 0`.

The repo has strong **consumer** and **kernel** machinery:

- full-kernel `C^θ → L∞` second-derivative smoothing,
- full-kernel `C^θ → C^η` chemotaxis-leg machinery,
- algebraic product Hölder lemmas,
- sup bounds and measurability for `chemFluxLifted`,
- positive-time Hölder regularity for `u` itself,

but it does **not** yet have a non-circular theorem producing

```lean
∀ s, 0 < s → s ≤ T →
  ∀ a b ∈ Set.Icc (0 : ℝ) 1,
    |chemFluxLifted p (D.u s) a - chemFluxLifted p (D.u s) b|
      ≤ HQ * |a - b| ^ θ
```

with a single constant `HQ` uniform for small `s`.

## 1. Existing relevant facts

### The full-kernel mild map uses the right operator

`ShenWork/Paper2/IntervalGradientDuhamelMap.lean` defines the weak divergence-form mild map with the **full Neumann kernel**:

```lean
def chemFluxLifted (p : CM2Params) (w : intervalDomainPoint → ℝ) : ℝ → ℝ :=
  fun y => intervalDomainLift w y * resolverGradReal p w y
    / (1 + intervalDomainLift (intervalNeumannResolverR p w) y) ^ p.β

def intervalGradientDuhamelMap ... :=
  intervalFullSemigroupOperator t (intervalDomainLift u₀) x.1

```

This is faithful to the full-kernel route.  The route below should stay in this file family and should not switch to the helper `intervalSemigroupOperator 1` scaffold.

### The full-kernel second-derivative consumer exists

`ShenWork/PDE/IntervalFullKernelSecondDerivCtheta.lean` proves the key kernel-side estimate:

```lean
theorem neumannHeatSecondDeriv_Ctheta_to_Linfty {t θ : ℝ} (ht : 0 < t)
    (hθ0 : 0 < θ) (hθ1 : θ < 1) {h : ℝ → ℝ}
    (hh_meas : AEStronglyMeasurable h (intervalMeasure 1))
    {Ch : ℝ} (hh : ∀ y, |h y| ≤ Ch) {Hh : ℝ} (hHh_nn : 0 ≤ Hh)
    (hHh : ∀ a b, a ∈ Set.Icc (0:ℝ) 1 → b ∈ Set.Icc (0:ℝ) 1 →
      |h a - h b| ≤ Hh * |a - b| ^ θ)
    {x : ℝ} (hx : x ∈ Set.Icc (0:ℝ) 1) :
    |deriv (fun z : ℝ => deriv
        (fun w : ℝ => intervalFullSemigroupOperator t h w) z) x|
      ≤ weightedHeatHessConst θ * t ^ (-1 + θ / 2 : ℝ) * Hh
```

This theorem is exactly the per-slice kernel estimate the chemotaxis differentiated leg wants.  It still requires the input source `h` to be uniformly bounded and uniformly Hölder on `[0,1]`.

## 2. Shortest non-circular route if one tried to use existing facts

There is no complete existing instantiation route.

The closest available partial route is:

1. Use `mild_orderBox_positiveTime_holder` to get `D.u s` Hölder on `[τ, D.T]` for fixed `τ > 0`.
2. Use a future elliptic regularity theorem to turn `D.u s ∈ C^θ` into spatial Hölder bounds for:
   - `intervalDomainLift (intervalNeumannResolverR p (D.u s))`,
   - `resolverGradReal p (D.u s)`.

Step 2 is not present, and step 1 is not uniform down to `s = 0`.  Therefore this does not solve the requested small-time source frontier.

## 4. Component-level theorem worth adding before the full producer

The smallest algebraic theorem that would genuinely reduce future work is:

```lean
/-- Component Hölder bounds for `u`, `resolverGradReal`, and `R`, plus positivity
of `R`, imply a Hölder bound for `chemFluxLifted`.  This is pure source algebra;
it does not use any Duhamel target. -/
theorem chemFluxLifted_holder_of_component_holder
    {p : CM2Params} {w : intervalDomainPoint → ℝ}
    {θ U G V Hu Hg Hv : ℝ}
    (hθ0 : 0 < θ)
    (hU_nonneg : 0 ≤ U) (hG_nonneg : 0 ≤ G)
    (hHu_nonneg : 0 ≤ Hu) (hHg_nonneg : 0 ≤ Hg) (hHv_nonneg : 0 ≤ Hv)
    (hu_bound : ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |intervalDomainLift w x| ≤ U)
    (hg_bound : ∀ x ∈ Set.Icc (0 : ℝ) 1,
      |resolverGradReal p w x| ≤ G)
    (hR_nonneg : ∀ x ∈ Set.Icc (0 : ℝ) 1,
      0 ≤ intervalDomainLift (ShenWork.PDE.intervalNeumannResolverR p w) x)
    (hu_holder : ∀ a b, a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
      |intervalDomainLift w a - intervalDomainLift w b| ≤ Hu * |a - b| ^ θ)
    (hg_holder : ∀ a b, a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
      |resolverGradReal p w a - resolverGradReal p w b| ≤ Hg * |a - b| ^ θ)
    (hR_holder : ∀ a b, a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
      |intervalDomainLift (ShenWork.PDE.intervalNeumannResolverR p w) a -
        intervalDomainLift (ShenWork.PDE.intervalNeumannResolverR p w) b| ≤
        Hv * |a - b| ^ θ) :
    ∀ a b, a ∈ Set.Icc (0 : ℝ) 1 → b ∈ Set.Icc (0 : ℝ) 1 →
      |chemFluxLifted p w a - chemFluxLifted p w b| ≤
        (U * Hg + Hu * G + U * G * p.β * Hv) * |a - b| ^ θ
```

This theorem should live near the frontier record, or in a small algebra file imported by it.  It uses only:

- `chemFluxLifted` definition,
- `holder_mul`-style product algebra,
- `IntervalChemFluxLipschitz.oneAddRpow_neg_lipschitz` for the denominator factor.

The genuinely missing analytic component producers would then be named separately, for example:

```lean

theorem resolverGradReal_holder_of_source_holder ...
```

Those are elliptic regularity/spectral multiplier estimates.  They are not currently present.

## 5. Why not use current positive-time or Wiener routes?

- `mild_orderBox_positiveTime_holder` is a positive-time regularization theorem for `u`, with a lower time cut `τ > 0`.  It does not provide a small-time uniform source modulus for `Q(s)` down to `s = 0`.

- `ChemMildC1eta` and `ChemMildDifferentiableOn` are the correct full-kernel consumer side for the `Q` source.  They assume `hQholder` and do not prove it for `Q = chemFluxLifted p (D.u s)`.

- `SourceFluxSpatialReg` is interior differentiability under strong Wiener/EWA source-decay assumptions.  It is not a uniform `C^θ` package on `[0,1]` for the weak mild fixed-point data.

- The helper semigroup `intervalSemigroupOperator 1` should not be used for this frontier.  The relevant files here (`IntervalGradientDuhamelMap`, `IntervalFullKernelSecondDerivCtheta`, `ChemMildC1eta`, `IntervalGradDuhamelBound`) are full-kernel files and use `intervalFullSemigroupOperator`.

## Bottom line

The next honest frontier is not `PatchedSliceDerivUniformApproachAtZero` itself and not `ux_zeroFace`.  It is a source-side `ChemFluxCthetaSourceOn` package, plus a small algebraic reducer showing that component Hölder bounds for

```lean
intervalDomainLift (u s),
resolverGradReal p (u s),
intervalDomainLift (intervalNeumannResolverR p (u s))
```

imply the needed Hölder bound for

```lean
chemFluxLifted p (u s).
```

The missing analytic work is to prove those component Hölder bounds uniformly on the small-time interval for the actual `D.u` trajectory, using only the full-kernel route.
