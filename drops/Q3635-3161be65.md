## Verdict

Current status:

- `flux_meas`: already available from `HasJointMeasurability u` via the public theorem `chemFluxLifted_uncurry_measurable` in `ChemMildHolderBootstrap.lean`.
- `flux_int`: already available slice-by-slice from `HasContinuousSlices T u`, `hbound`, and `hnonneg`, via the public theorem `chemFluxLifted_integrable_of_continuous` in `IntervalDuhamelIntegrability.lean`.

There is also a public existential bound `chemFluxLifted_bounded_of_continuous`, but it is per-slice and does not by itself give the uniform `CQ` required by `ChemFluxCthetaSourceOn`.
- `flux_cont`: analytically available from the same continuity/resolver-continuity argument embedded inside `chemFluxLifted_integrable_of_continuous` and `chemFluxLifted_bounded_of_continuous`, but I did not find a public theorem with conclusion `Continuous (chemFluxLifted p w)`.

### Continuity is analytically available but not public as a standalone theorem

`chemFluxLifted_integrable_of_continuous` and `chemFluxLifted_bounded_of_continuous` both prove the continuity of the flux on the interval internally, using:

```lean
resolverGradReal_continuous_of_continuousOn
resolverValueReal_continuous_of_continuousOn
intervalNeumannResolverR_nonneg_of_nonneg_source
```

But I did not find a public theorem with conclusion:

```lean
Continuous (chemFluxLifted p w)
```

or even:

```lean
ContinuousOn (chemFluxLifted p w) (Set.Icc (0 : ‚ĄĚ) 1)
```

For `ChemFluxCthetaSourceOn.flux_cont`, the record asks for global `Continuous (chemFluxLifted p (u s))`.  This is stronger than closed-interval continuity.  Because `chemFluxLifted` is zero outside `[0,1]`, global continuity at the endpoints requires compatibility of the flux value with zero at the endpoints.  That is not automatic from mere continuity of `w`; it would require endpoint vanishing of the flux, for example via `resolverGradReal` zero at endpoints or an equivalent Neumann/odd-sine fact.

This is a faithfulness issue: the existing integrability proof only needs closed-interval continuity plus zero outside for boundedness/integrability; it should not be silently treated as global continuity of the zero extension.

Therefore the smallest honest wrapper should either:

- change/relax the frontier record in a future patch to `flux_contOn : ‚ąÄ s, ... ContinuousOn (chemFluxLifted p (u s)) (Set.Icc 0 1)`, or
- prove and expose the true endpoint-compatible global continuity theorem for `chemFluxLifted` under source decay / resolver-gradient endpoint vanishing.
