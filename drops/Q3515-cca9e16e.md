# R4-A source/API audit: H1 physical exit product measurability

Audited at commit `595354e31017e3dcaa64a2e4daa5b5a7a090dad2`.

## 1. Can current APIs provide the missing measurability non-circularly?

**Direct exact API at this HEAD: no.** I did not find an existing theorem that outputs the zero-window/a.e. spatial product-measurability or factor-measurability fields needed by the corrected `H1PhysicalRHSComponentSquareSpatialYoungDataBefore_of_squareData_and_productMeas` route.

**Source material to prove it non-circularly: yes.** The classical strict-slab continuity APIs are enough. They use only `IsPaper2ClassicalSolution` regularity and physical representatives; they do not use bounded-before, RHS integrability, or the target Young zero-window theorem.

For the three non-lap factors, existing strict-slab continuity gives fixed-time spatial continuity on `[0,1]`, hence `AEStronglyMeasurable` on `volume.restrict (Set.Ioc 0 1)` for a.e. `r` in any `Set.Ioc 0 δ` with `δ < T`:

- `H1PhysicalChemTaxisPart p u v r`
- `H1PhysicalChemUvxxPart p u v r`
- `H1PhysicalLogisticReactionPart p u r`

For the `liftDeriv2 u r` factor, there is no direct closed-spatial-slab continuity theorem for literal `liftDeriv2`; the available route is the endpoint-safe representative:

- continuity of `liftDeriv2PhysicalRHSWithChemRep p u (liftChemotaxisDivPhysicalRep p u v)` on strict closed slabs;
- equality with literal `liftDeriv2 u` only on the spatial interior `Set.Ioo 0 1`.

That is still enough for `AEStronglyMeasurable` on `volume.restrict (Set.Ioc 0 1)`, because the only missing endpoint is `{1}`, a null set. So the issue is an API gap, not a new analytic obstacle.

## 4. Route attractiveness

This measurability issue does **not** make the zero-window/spatial-Young route less attractive than identity/sqrt. It is a thin API producer missing between already-proved strict-slab continuity and the zero-window square-data route. The identity and sqrt frontiers remain genuinely analytic and problem-specific. By contrast, the product-measurability gap is routine: classical strict-slab continuity plus representative interior equality should close it without using bounded-before or RHS integrability.
