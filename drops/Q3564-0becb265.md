# Q3564 audit: next non-circular producer for `H1PhysicalChemResolverSupBefore`

## Executive conclusion

I do **not** see a landed theorem that honestly produces

```lean
H1PhysicalChemResolverSupBefore p u v T V₁ V₂ M
```

from `hsol : IsPaper2ClassicalSolution intervalDomain p T u v` alone, nor from `IsPaper2BoundedBefore` alone.  That would be too strong: classical regularity gives compactness on each strict slab `[a,b] × [0,1]` with `0 < a ≤ b < T`, but the target asks for fixed-before-`T` constants on the whole open time interval `(0,T)`.  The latter is not compact and needs a genuine uniform source, such as a finite-horizon `L∞`/sup bound plus additional control of the `uvxx` core.

The smallest honest reducer now available is:

* assume a uniform pointwise upper bound on `u` on `(0,T) × [0,1]`;
* use the landed quantitative resolver theorem `resolverGrad_sup_le_of_ub` to produce the `resolver_grad_le` field;
* derive `u_abs_le` from the same upper bound plus positivity of the classical solution;
* keep the exact `uvxx_core_le` field as the remaining source residual.

That reducer is non-circular: it does not use `H1PhysicalChemResolverSupBefore`, factor bounds, chem L² data, H¹ sqrt bounds, or bounded-before H¹ data.

  Real.sqrt (∑' k : ℕ, (intervalNeumannResolverGradWeight p k) ^ 2) *
    (2 * (p.ν * M ^ p.γ))

    (p : CM2Params) {M : ℝ} (hM : 0 ≤ M) :
    0 ≤ H1PhysicalChemResolverGradCap p M := by
  unfold H1PhysicalChemResolverGradCap
  exact mul_nonneg (Real.sqrt_nonneg _) (by
    have hpow : 0 ≤ M ^ p.γ := Real.rpow_nonneg hM p.γ
    have hνpow : 0 ≤ p.ν * M ^ p.γ := mul_nonneg p.hν.le hpow

      solution_lift_pos (p := p) (T := T) (u := u) (v := v) hsol hτ x hx

      resolverGrad_sup_le_of_ub

## Why `IsPaper2BoundedBefore` is not enough by itself

`IsPaper2BoundedBefore intervalDomain T u` is the right high-level source of a uniform `u` bound, and it is produced in the negative-sensitivity branches by `boundedBefore_nonminimal_of_corrected_initial_approach` and `boundedBefore_minimal_of_corrected_initial_approach`.  But for the current theorem shape, two additional bridges are still needed:

1. a concrete interval-domain bridge from `intervalDomain.supNorm (u τ) ≤ M` to

   ```lean
   ∀ x, x ∈ Set.Icc (0 : ℝ) 1 → intervalDomainLift (u τ) x ≤ M
   ```

2. a producer of the exact `uvxx_core_le` expression, uniformly on `(0,T) × [0,1]`.

The first is probably easy but should be named explicitly.  The second is the substantive remaining source estimate.

A safe later bridge, once the sup-norm pointwise lemma is located or proved, would look like this:

## What remains for `uvxx_core_le`

The exact field is:

```lean
∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
  ∀ x, x ∈ Set.Icc (0 : ℝ) 1 →
    |(p.μ * intervalDomainLift (v τ) x -
        p.ν * (intervalDomainLift (u τ) x) ^ p.γ) /
        (1 + intervalDomainLift (v τ) x) ^ p.β -
      p.β * (deriv (intervalDomainLift (v τ)) x) ^ 2 /
        (1 + intervalDomainLift (v τ) x) ^ (p.β + 1)| ≤ V₂
```

A future non-circular producer should probably split this into more primitive value/gradient data:

```lean
structure H1PhysicalChemValueGradientBoundsBefore
    (p : CM2Params) (u v : ℝ → intervalDomainPoint → ℝ)
    (T M V₀ V₁ : ℝ) : Prop where
  hM : 0 ≤ M
  hV0 : 0 ≤ V₀
  hV1 : 0 ≤ V₁
  u_le : ∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
    ∀ x, x ∈ Set.Icc (0 : ℝ) 1 →
      intervalDomainLift (u τ) x ≤ M
  v_le : ∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
    ∀ x, x ∈ Set.Icc (0 : ℝ) 1 →
      intervalDomainLift (v τ) x ≤ V₀
  vx_le : ∀ τ, τ ∈ Set.Ioo (0 : ℝ) T →
    ∀ x, x ∈ Set.Icc (0 : ℝ) 1 →
      |deriv (intervalDomainLift (v τ)) x| ≤ V₁
```

Then prove an algebraic reducer to `uvxx_core_le`, using:

* `solution_lift_v_nonneg_Icc` for `0 ≤ v`;
* `Real.one_le_rpow` for denominator lower bounds;
* `u_le` plus positivity to bound `u^γ ≤ M^γ`;
* `v_le` to bound the `μ v` numerator term when `β < 1`;
* `vx_le` to bound the quadratic gradient term.

The likely cap is conservative, for example:

```lean
p.μ * V₀ + p.ν * M ^ p.γ + p.β * V₁ ^ 2
```

This is mathematically easy but Lean-algebraically longer than the resolver-gradient reducer.  It should be a separate patch.
