## What `Hsource` asks for

For each `t₀ ∈ (0,T)` and interior spatial point, `Hsource` asks for:

```lean
∃ (a₀ : ℕ → ℝ) (M : ℝ) (_ : 0 ≤ M) (_ : ∀ n, |a₀ n| ≤ M)
  (a : ℝ → ℕ → ℝ) (_ : DuhamelSourceTimeC1 a)
  (offset : ℝ) (_ : 0 < t₀ - offset)
  (_ : LogisticCosineFourierData p u t₀)
  (_ : ChemDivCosineFourierDataIoo p (u t₀)
    (coupledChemicalConcentration p u t₀)),
  (∀ᶠ s in 𝓝 t₀, ∀ y : intervalDomainPoint,
    u s y = ∑' n, localRestartCoeff a₀ a (s - offset) n * cosineMode n y.1) ∧
  (∀ n, a (t₀ - offset) n =
    coupledLogisticSourceCoeffs p u t₀ n
      - p.χ₀ * coupledChemDivSourceCoeffs p u t₀ n) ∧
  Summable (fun n => unitIntervalCosineEigenvalue n
    * |localRestartCoeff a₀ a (t₀ - offset) n|)
```

The right intended choices are:

```lean
τ      := t₀ / 2
offset := τ
a₀     := cosineCoeffs (intervalDomainLift (D.u τ))
a      := fun σ n => bFormSourceCoeffs p D.u (τ + σ) n
```

or a soft-clamped variant of `a` if only windowed source-`C¹` data is available.

### `hsource_split`

This is already essentially solved once the source family is chosen as the total B-form source. In `IntervalBFormSpectralHtime.lean`:

```lean
def bFormSourceCoeffs (p : CM2Params)
    (u : ℝ → intervalDomainPoint → ℝ) : ℝ → ℕ → ℝ :=
  fun s n => coupledLogisticSourceCoeffs p u s n
    - p.χ₀ * coupledChemDivSourceCoeffs p u s n
```

So for `a σ n = bFormSourceCoeffs p D.u (τ + σ) n`, the proof of

```lean
∀ n, a (t₀ - τ) n =
  coupledLogisticSourceCoeffs p D.u t₀ n
    - p.χ₀ * coupledChemDivSourceCoeffs p D.u t₀ n
```

is just `τ + (t₀ - τ) = t₀` plus `simp [bFormSourceCoeffs]`.

## Circularity check

A proof of `Hsource` must not use:

```lean
hpde_u_of_generalChi_sourceSpectralData
GradientMildClassicalFrontierCoreData p D
IsPaper2ClassicalSolution ...
```

Using any of those to produce the source representation would be circular, because `Hsource` is precisely the input intended to prove `hpde_u`.

The non-circular data that may be used are:

```lean
D.hmild
D.hbound
D.hcont
D.hnonneg
D.hpos
D.hmeas
```

plus independent source/flux regularity facts such as source `DuhamelSourceTimeC1`, Fourier packages, and the open source bridge. In particular, `IntervalSourceBridgeOpen.source_bridge_slice_open` is safe: it is a flux/source identity, not a consequence of `hpde_u`.
