# Q3560: initial-time PDE trace audit for zero-start H¹ physical route

Audit target: default branch `8504f594`.

Target field:

```lean

  Set.EqOn
    (fun x => liftDeriv2 u (0 : ℝ) x)
```

## Verdict

For a general `u v`, the `eq0Interior` field is **not already derivable** from the current source/construction APIs. The repository has several positive-time or scalar-integrability bridges, and it has special constant-solution construction lemmas, but no existing theorem gives the literal pointwise initial-time `u`-equation/laplacian trace needed here.

The positive-time branch is already closed by

```lean

```

in `IntervalChiNegH1ZeroStartComponents.lean`: it uses

```lean
liftDeriv2_eq_liftDeriv2PhysicalRHS_interior_of_classicalSolution
lift_chemotaxisDiv_eq_liftChemotaxisDivPhysicalRep_interior
```

for `0 < t`. The `t = 0` branch is exactly `hinit.eq0Interior`; no theorem in the audited construction layer replaces it.

## Source-grounded findings

### 1. The current target and positive-time bridge

`IntervalChiNegH1ZeroStartComponents.lean` defines the target package

```lean
H1ZeroStartPhysicalPDEInitialCompatibilityBefore p u v T
```

and the bridge

```lean
theorem H1ZeroStartPhysicalPDESeamBefore_of_classicalSolution_initialCompatibility
    {p : CM2Params} {u v : ℝ → intervalDomainPoint → ℝ} {T : ℝ}
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u v)
    (hinit : H1ZeroStartPhysicalPDEInitialCompatibilityBefore p u v T) :
    H1ZeroStartPhysicalPDESeamBefore p u v T
```

The proof splits on `t = 0`. At `t = 0`, it uses exactly `hinit.eq0Interior`. At `0 < t`, it uses the positive-time classical PDE equality and the positive-time chemotaxis representative equality. Therefore the remaining source problem is solely the initial trace.

### 2. Classical PDE equality is strict-time only

`Statements.lean` defines `IsPaper2ClassicalSolution`; the `u` PDE field is stored only under strict hypotheses:

```lean
∀ t x, 0 < t → t < T → x ∈ D.inside →
  D.timeDeriv u t x =
    D.laplacian (u t) x
      - p.χ₀ * D.chemotaxisDiv p (u t) (v t) x
      + u t x * (p.a - p.b * (u t x) ^ p.α)
```

The positive-time H¹ bridge in `IntervalChiNegH1LiftDeriv2Transfer.lean` is therefore also strict-time:

```lean
theorem liftDeriv2_eq_liftDeriv2PhysicalRHS_interior_of_classicalSolution
    (hsol : IsPaper2ClassicalSolution intervalDomain p T u v)
    (ht : t ∈ Set.Ioo (0 : ℝ) T) (hx : x ∈ Set.Ioo (0 : ℝ) 1) :
    liftDeriv2 u t x = liftDeriv2PhysicalRHS p u v t x
```

and its slab version requires `0 < a`:

```lean
theorem liftDeriv2_eq_liftDeriv2PhysicalRHS_strictSlab_interior_of_classicalSolution
    (ha : 0 < a) ... :
    Set.EqOn ... (Set.Icc a b ×ˢ Set.Ioo (0 : ℝ) 1)
```

So `IsPaper2ClassicalSolution` and its derived strict-slab APIs cannot prove `eq0Interior`.

### 3. Chemotaxis representative equality is also strict-time only

`IntervalChiNegH1ChemDivRepresentative.lean` gives:

```lean
lift_chemotaxisDiv_eq_liftChemotaxisDivPhysicalRep_interior
lift_chemotaxisDiv_eq_liftChemotaxisDivPhysicalRep_strictSlab_interior
```

Both require `t ∈ Ioo 0 T` or a strict slab `0 < a`. They do not supply the initial equality between the literal lifted chemotaxis divergence and `liftChemotaxisDivPhysicalRep p u v 0`.

This matters because even an initial literal PDE trace with `intervalDomain.chemotaxisDiv` would still need an initial-time chem-representative compatibility theorem to reach the target’s `liftChemotaxisDivPhysicalRep` RHS.

### 4. Mild/Duhamel construction data does not give the trace

`IntervalDomainExistence.lean` defines `intervalDuhamelOperator`, but it is currently the logistic-only Duhamel operator:

```lean
intervalSemigroupOperator 1 t (intervalDomainLift u₀) x.1 +
  ∫ s in Set.Icc 0 t,
    intervalSemigroupOperator 1 (t - s)
      (intervalDomainLift (intervalLogisticSource p (u s))) x.1
```

It is not the full chemotaxis PDE trace at `t = 0`. The construction predicate

```lean
IsMildSolutionData p T u₀ u v
```

has a Duhamel fixed-point field for `0 ≤ t ≤ T`, but its pointwise PDE field is still only

```lean
∀ t x, 0 < t → t < T → x ∈ intervalDomain.inside → ...
```

Thus `IsMildSolutionData` does not imply `eq0Interior` either. It explicitly routes to `IsPaper2ClassicalSolution`, which is also strict at time zero.

### 5. Reinitialized actual-zero-slice trajectory does not solve this

`P3MoserEnergyContinuity.lean` defines

```lean
intervalDomainWithInitialSlice u₀ u
```

and proves:

```lean
intervalDomainWithInitialSlice_eq_raw_of_pos
intervalDomain_classical_withInitialSlice
intervalDomain_globalClassical_withInitialSlice
```

These theorems deliberately preserve strict-positive-time behavior and set the stored zero slice. They do not prove time differentiability, laplacian compatibility, or the PDE at the new stored zero slice. In fact the reanchoring is exactly a point modification at `t = 0`; it is useful for energy/trace bookkeeping, but it is not a pointwise initial PDE trace theorem.

So using the reinitialized zero slice to claim `eq0Interior` would be a false route unless additional compatibility data about `u₀`, `u_t(0)`, `u₀,xx`, and the chemotaxis representative is supplied.

### 6. P3/Moser/FTC/global-PDE-initial data remains scalar/integral-level

`P3MoserInitialTimeRegularity.lean` explicitly says current global classical data does not produce componentwise initial-window integrability, because the classical PDE and strongest continuity fields are open in time at `0`. It introduces scalar packages such as

```lean
IntervalDomainLpPDETermInitialWindowContinuity
intervalDomain_lpPDETermInitialWindowIntegrability_of_initialWindowContinuity
```

`P3MoserPDECombinedInitialProducer.lean` gives scalar initial-window integrability, and the key theorem says the equality is used only on `Ioc 0 b`, so the endpoint value at `0` is irrelevant:

```lean
intervalDomain_lpWeightedTimeTermInitialWindowIntegrability_of_moserDerivativeInitial
```

This is the wrong shape for `eq0Interior`, which is a pointwise `t = 0` equality on the spatial interior.

### 7. B-form / divergence-mode / spectral coefficient identities are not pointwise initial PDE trace

`IntervalConjugateChemFluxIntegrable.lean` addresses B-form conjugate-kernel time integrability:

```lean
s ↦ B_N(t−s) Q(w s) x
```

and proves measurability/integrability style facts. This is not a theorem about `liftDeriv2 u 0` or the `u` PDE at `t = 0`.

`IntervalDivergenceModeIdentity.lean` proves spectral/IBP identities such as

```lean
rawCosCoeff_deriv_eq_kpi_rawSinCoeff
cosineCoeffs_deriv_eq_sqrtLambda_sineCoeff
chemFlux_cosineCoeff_deriv_eq_engine
chemotaxisDuhamel_cosineCoeff_eq_engine
```

These identify coefficients of a spatial divergence under boundary hypotheses. They do not identify `u_xx(0)` with `u_t(0) + χ₀*chemRep(0) - reaction(0)` pointwise.

Similarly, the mixed chem-div representative files can at most help with `time_cont0` or chem-div time-derivative regularity; they are about `coupledChemDivTimeDerivativeLift`, not the initial `u`-equation trace.

### 8. Special constant solutions are a non-circular special case, not the general producer

`IntervalDomainExistence.lean` has constant-solution infrastructure:

```lean

intervalDomainChemotaxisDiv_const_zero

equilibrium_reaction_zero

```

For those very special equilibrium/zero-reaction constant trajectories, a non-circular special-case `eq0Interior` theorem should be provable by unfolding `liftDeriv2PhysicalRHSWithChemRep` and showing all derivative/chem/reaction pieces vanish. However, no existing theorem is exported as the target `eq0Interior`, and this does not help the general H¹ physical route for arbitrary constructed solutions.

## Partial bridges

No existing theorem gives `eq0Interior` for the general source/construction setting.

A possible partial bridge exists only if the source layer supplies a **literal initial PDE trace** plus an **initial chem-representative equality**. This would be non-circular and would not use H¹ bounded-before/sqrt/zero-start RHS outputs.

Dependencies would be source-facing, not H¹-route outputs:

1. an initial `u` PDE trace at `t = 0` on the spatial interior, in the literal interval-domain PDE form;
2. an initial equality between the literal lifted chemotaxis divergence and `liftChemotaxisDivPhysicalRep p u v 0`;
3. for construction-specific `v = coupledChemicalConcentration p u`, an explicit identification of the route’s arbitrary `v` with that constructed concentration at least at `t = 0` and for the derivatives entering `liftChemotaxisDivPhysicalRep`.

No audited file currently supplies those fields for general `u v`.

## Smallest honest source-facing record

For the H¹ route itself, the current `eq0Interior` field is already the smallest possible interface: it is exactly the literal equality needed.

For a construction-facing bridge, the smallest honest decomposition is a two-field initial trace record: one field for the literal initial `u`-PDE, one field for the initial chemotaxis representative equality. This is closer to construction data than directly carrying `eq0Interior`, while still implying the literal target.

## False/circular routes to avoid

* Do not use H¹ bounded-before, `H1SupBoundSqrtRHSIntegrableBefore`, sqrt bounds, or zero-start physical RHS outputs; those are downstream.
* Do not use P3/Moser scalar initial-window integrability as if it were a pointwise `t = 0` PDE trace.
* Do not use `coupledChemDivTimeDerivativeLift` regularity as if it were the `u`-equation trace.
* Do not claim `intervalDomainWithInitialSlice` creates a PDE-compatible zero slice. It only changes the stored value at `t = 0` and preserves positive-time classical properties.
* Do not apply positive-time classical PDE or chem-rep equality at `t = 0`; every relevant theorem requires `0 < t` or `0 < a`.

## Recommended next audit/proof direction

The honest next source question is not “can P3/Moser/Duhamel prove `eq0Interior`?” but:

> What construction-level initial compatibility record gives a pointwise initial `u`-PDE trace and a physical chemotaxis-representative equality for the chosen zero slice?

Until such a record exists, the current `eq0Interior` field remains the minimal safe carry for the H¹ route.
