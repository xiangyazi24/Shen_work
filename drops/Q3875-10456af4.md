## 1. `energy_continuous`

### Target

```lean

  ContinuousOn

    (Set.Icc (0 : ℝ) T)
```

### Strategy overview

Split `[0,T]` into positive time and zero.

* For `t₀ ∈ (0,T]`, use joint continuity of `u` on positive-time slabs and dominated convergence on a compact time window `[t₀/2, min T (t₀+δ)]`.
* For `t₀ = 0`, use `u 0 = 0` by definition of the Picard limit plus the initial trace `u(t) → u₀` uniformly, with `u₀ ≥ 0`.  Then `u_-(t) → 0` uniformly, so `∫ (u_-)² → 0`.

The zero-time argument is the missing point.  It should not use joint continuity, because the available joint continuity is only on positive time.

## 3. `jensenStrictPosData`

### Target

Produce the strict-positivity restart data at a positive time `s`, using:

1. restart seed `f` with `f² ≤ u(s)`,
2. Jensen `(Sτ f)² ≤ Sτ(f²)`,
3. heat positivity `Sτ f > 0`,
4. reaction-discounted lower bound
   `u(s+σ) ≥ exp(-D*σ) * Sσ(u(s))`.

This is the non-circular positivity route: nonnegativity plus positive mass gives a nonzero seed at restart time.

### Seed definition

For fixed restart time `s` with `0 < s` and `s < T`, define:

```lean
def restartSqrtSeed
    (u : ℝ → intervalDomainPoint → ℝ) (s : ℝ) : ℝ → ℝ :=
  fun x => Real.sqrt (max 0 (intervalDomainLift (u s) x))
```

If you already have pointwise nonnegativity of `u s`, use the simpler version:

```lean
def restartSqrtSeed
    (u : ℝ → intervalDomainPoint → ℝ) (s : ℝ) : ℝ → ℝ :=
  fun x => Real.sqrt (intervalDomainLift (u s) x)
```

I recommend the `max 0` version because it is robust and simplifies off-support cases.

```lean
  -- integral_congr over s∈0..t; use hnonneg s hs hst.
  -- logistic: positivePart_eq_self_of_nonneg.
  -- chem flux: previous flux equality.
  congr 1

    sorry
  · -- logistic leg
    apply intervalIntegral.integral_congr
```

### Iterate equality by induction

The clean theorem:

```lean
theorem truncatedConjugatePicardIter_eq_conjugatePicardIter_of_nonneg_iterates
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ} {T : ℝ}
    (hiter_nonneg : ∀ n t, 0 < t → t ≤ T → ∀ x,
      0 ≤ truncatedConjugatePicardIter p u₀ n t x) :
    ∀ n t, 0 < t → t ≤ T → ∀ x,
      truncatedConjugatePicardIter p u₀ n t x =
        conjugatePicardIter p u₀ n t x := by
  intro n
  induction n with
  | zero =>
      intro t ht htT x
      simp [truncatedConjugatePicardIter, conjugatePicardIter]
  | succ n ih =>
      intro t ht htT x
      simp [truncatedConjugatePicardIter, conjugatePicardIter]
      have hmap := truncatedConjugateDuhamelMap_eq_intervalConjugateDuhamelMap_of_nonneg
        (p := p) (u₀ := u₀) (w := truncatedConjugatePicardIter p u₀ n)
        (T := T) (hiter_nonneg n)
      -- rewrite the full map input by function extensionality using ih
      have hfun : truncatedConjugatePicardIter p u₀ n = conjugatePicardIter p u₀ n := by
        funext t x
        by_cases htpos : 0 < t
        · by_cases htT : t ≤ T
          · exact ih t htpos htT x
          · -- outside time window, both iterates may not be definitionally zero; if not, state equality only under map integral window.
            sorry
        · sorry
      rw [hmap t ht htT x, hfun]
```

Better: avoid needing global function equality.  Prove map congruence under the integral directly using `ih s hs hsT`, because only `s ∈ [0,t]` matters.  This avoids outside-window issues.

### Limit equality

Once iterate equality is pointwise on the time window, the limits are immediate by unfolding:

```lean
theorem truncatedConjugatePicardLimit_eq_conjugatePicardLimit_of_iter_eq
    {p : CM2Params} {u₀ : intervalDomainPoint → ℝ} {T : ℝ}
    (hiter_eq : ∀ n t, 0 < t → t ≤ T → ∀ x,
      truncatedConjugatePicardIter p u₀ n t x =
        conjugatePicardIter p u₀ n t x) :
    ∀ t, 0 < t → t ≤ T → ∀ x,
      truncatedConjugatePicardLimit p u₀ T t x =
        conjugatePicardLimit p u₀ T t x := by
  intro t ht htT x
  unfold truncatedConjugatePicardLimit conjugatePicardLimit
  simp only [ht, htT, and_self, ite_true]
  congr
  funext n
  exact hiter_eq n t ht htT x
```

This is the main reason no `ConjugateMildExistenceData` is needed: `limUnder` sees equal sequences.

### Required nonnegativity input

The theorem should accept nonnegativity of truncated iterates as a hypothesis if not already proved:

```lean
hiter_nonneg : ∀ n t, 0 < t → t ≤ T → ∀ x,
  0 ≤ truncatedConjugatePicardIter p u₀ n t x
```

If final nonnegativity has only been proved for the truncated **limit**, do not claim it implies iterate nonnegativity.  Instead prove `hiter_nonneg` separately by induction from the truncated map’s positivity-preserving property, or change the agreement proof to use uniqueness of fixed points in the contraction ball.  The iterate route is simpler and does not need existence data.
