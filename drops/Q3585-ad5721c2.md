## Executive conclusion

The derivative frontier is real.  The current source tree has:

1. strict positive-time joint continuity of `u_x` and `v_x` on `Set.Ioo 0 T ×ˢ Set.Icc 0 1`, and
2. value/sign zero-face producers for patched slices and patched chemical data,

but it does **not** have a theorem proving the hard zero-face derivative fields

```lean
ux_zeroFace : ∀ {b}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,
  ContinuousWithinAt
    (Function.uncurry
      (fun (t : ℝ) (x : ℝ) => deriv (intervalDomainLift (u t)) x))
    (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0, x)

vx_zeroFace : ∀ {b}, 0 ≤ b → b < T → ∀ x ∈ Set.Icc (0 : ℝ) 1,
  ContinuousWithinAt
    (Function.uncurry
      (fun (t : ℝ) (x : ℝ) => deriv (intervalDomainLift (v t)) x))
    (Set.Icc (0 : ℝ) b ×ˢ Set.Icc (0 : ℝ) 1) (0, x)
```

The smallest non-circular next theorem should therefore be a **u-side derivative zero-face reducer**: from an explicit uniform-in-space derivative approach at `t = 0` plus spatial continuity of the initial derivative, prove the `ux_zeroFace` field for `patchedSlice`.  This is small, honest, and non-circular.  It does not solve the analytic derivative-convergence problem, but it gives the exact bounded source-side target that future analytic work must feed.

The v-side resolver/chemical derivative is larger: the repo has fixed-slice resolver derivative machinery, but I found no zero-time theorem saying the resolver gradient of `u(t)` converges to the resolver gradient of `u₀` as `t → 0+`.

## 3. Which derivative side is smaller to attack first?

Attack `ux_zeroFace` first.

Reason: the u-side can be reduced by a very small theorem that mirrors the already-proved value theorem `patchedSlice_lift_zeroFace_of_timeContinuousAt_zero`.  The only genuinely new input is exactly the missing analytic content: uniform-in-space convergence of the spatial derivative of the patched slice to the spatial derivative of the datum.

The v-side would first require a resolver-gradient continuity theorem.  The repo has static series differentiation and positive-time elliptic regularity, but no Lipschitz/continuity theorem for the gradient of `intervalNeumannResolverR` as the input slice varies toward `u₀`.  That makes `vx_zeroFace` the larger analytic project.

- `patchedSlice_timeContinuousAt_zero` and `patchedSlice_lift_zeroFace_of_timeContinuousAt_zero` are value-level facts.  They do not control derivatives.

- Positive-time P3 Moser derivative continuity theorems, `intervalDomain_dx_u_jointlyContinuous` and `intervalDomain_dx_v_jointlyContinuous`, are time-open at zero.  They can be glued to a closed slab only after the zero-face derivative traces are independently produced.

- Downstream H1 bounded-before routes, physical-RHS routes, sqrt-bound routes, and constant-route files should not be used to produce this general source-side derivative trace.  They either consume the same regularity/sign package downstream or solve only a special case.
